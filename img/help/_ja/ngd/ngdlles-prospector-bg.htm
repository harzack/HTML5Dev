OpenText Prospectorsは、情報の検索と利用に役立つ Content Server アイテムです。このアイテムはフィルターのように機能し、Content Server のインデックスの作成時にデータをスキャンして、トピックに一致する情報を検索します。Prospectorsは継続的に実行され、正確で最新の結果リストを維持しています。

Prospectorsは、Content Server によってインデックスが作成されるすべてのデータ ソースをスキャンできます。例えば、Content Server リポジトリ、ローカル ディレクトリ、Lotus Notes データベースなどをスキャンできます。Content Server サイトでインデックスが作成されるデータ ソースの詳細については、管理者に問い合わせてください。

Prospectors を作成するには、 マップ タイトル (LQL) を使用して、必要な情報のタイプを検索するクエリを構成します。

Prospectors のクエリを構成した後、プロスペクタでスキャンするスライスを指定します。これにより、プロスペクタはクエリを実行し、インデックスに追加される新しいアイテムが指定された検索条件に一致するかどうかを確認します。検索条件に一致する場合、プロスペクタは、結果リストに対応するアイテムへのリンクを追加します。アイテムがインデックスに追加されたり、インデックスから削除されると、プロスペクタは、常に結果リストを更新します。Prospectorsは、管理者がクエリの実行対象として設定したインデックスのみをスキャンできます。

Prospectors を作成する時は、その結果リスト オプションも設定します。このオプションには、表示する結果の数と表示順序が含まれます。また、Content Server における結果リスト アイテムのサマリーの表示/非表示を選択することもできます。

Prospectors は、Content Server 内のほとんどの場所で作成できます。例えば、ワークスペース、プロジェクト ワークスペース、およびパーソナル ワークスペース で作成できます。
