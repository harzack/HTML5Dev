詳細検索を実行するには

  1. [Enterprise Connect] メニューまたはタブで [検索] をクリックし、次に [検索Content Server] をクリックします。

     

    ヒント:
    またエクスプローラーや Microsoft Outlook で、[Enterprise Connect] ツールバーの [詳細検索] をクリックしても、[詳細Content Server 検索] ページにアクセスできます。また、簡単アクセス ペイン、または [Enterprise Connect にコピー/移動]、[Enterprise Connect オプション]、および [Enterprise Connect 名前を付けて保存] の各ダイアログ ボックスの検索ボックスからも [詳細検索] にアクセスできます。このボタンが有効になるのは、検索に対応したコンテナーで、なおかつ、ユーザーに検索の権限が与えられているコンテナーに限られます。たとえば、[詳細検索] は、Enterprise Connect ルート ノード、検索結果、[お気に入り] フォルダー、[パーソナル] フォルダーに加えて、仮想フォルダーなどの物理的に存在しないコンテナーでも使用できません。

     

  2. 次の表の情報に基づいて検索条件を指定します。

     

    注意:
    ユーザーが使用できる Content Server モジュールによっては、追加的にオプションを使用できることがあります。Content Server での検索の詳細については、OpenText Content Server ユーザー オンライン ヘルプ - Content Server の検索 (LLESWBB-H-UGD)を参照してください。

     

    目的 操作
    検索する場所の指定 [Add to Search Form] 領域で [ロケーション] をクリックします。次に [ロケーション] 領域で、[Content Server の参照] をクリックして、検索する場所を選択します。
     

    注意:

    • 検索する場所を指定しない場合、Enterprise Connect は Content Server リポジトリ全体を検索します。

      [ 検索 Content Server] ダイアログ ボックスを閉じると、現在の検索条件はクリアされます。

    • Outlook で詳細検索を実行している場合、フィールドの選択機能を使用して選択したその他の列は、同じタイプの 1 つ以上のコンテナーを検索するときに常に表示されるようになります。ただし、異なるタイプの複数のコンテナーで詳細検索を実行する場合は、既定の列のみが表示されます。

     
     

    ヒント:
    検索をさらに絞り込むには、[コピー] をクリックして、リストから適切なブール演算子を選択し、追加的な検索場所を指定できます。

     
    検索方法の指定 (すべての語、いずれかの語、同じフレーズの検索など) [フル テキスト] 領域で [検索対象] リスト内のオプションをクリックし、検索語を入力します。
    複合クエリの実行 [フル テキスト] 領域で [検索対象] リスト内の [複合クエリ] をクリックし、検索語を入力します。詳細については、複合クエリを使用した検索 を参照してください。
    検索を拡張するための修飾語の指定 [フル テキスト] 領域で [Modifier] リスト内のオプションをクリックし、検索語を入力します。
    アイテムのコンテンツ、メタデータ、またはその両方に検索を制限するかどうかの指定 [フル テキスト] 領域で [Within] リスト内のオプションをクリックし、検索語を入力します。
    文またはフレーズを使用した検索 [Add to Search Form] 領域で [自然言語クエリ] をクリックします。使用する検索語を入力して、[モード] リストから適切なオプションを選択します。自然言語クエリの詳細については、 自然言語クエリを使用した検索を参照してください。
    システム属性検索の実行 [Add to Search Form] 領域で [システム属性] をクリックします。[追加の属性] リストで検索に含める属性をクリックしてから、検索する属性値を指定します。システム属性の使用の詳細については、 カテゴリと属性の操作を参照してください。
    カテゴリ検索の実行 [Add to Search Form] 領域で [カテゴリ] をクリックし、[カテゴリの選択] ダイアログ ボックスで使用するカテゴリを選択します。[カテゴリ] 領域に、使用するカテゴリ属性の値を指定します。カテゴリの使用の詳細については、 カテゴリと属性の操作を参照してください。
    Classifications 検索の実行 [Add to Search Form] 領域で [Classifications] をクリックします。[Classifications] 領域で [参照Classifications] をクリックして検索する Classifications を見つけるか、お気に入りフォルダーから Classifications を選択できるように [私のお気に入り] をクリックします。検索にサブ クラシフィケーションを含める場合は、[サブ クラシフィケーションを含める] チェック ボックスをオンにします。サブ クラシフィケーションを含めることによって、選択した Classifications とその子孫のすべてを検索することができます。Classifications の使用の詳細については、 クラシフィケーションの使用を参照してください。
     

    注意:
    このオプションは、環境に Classifications がインストールされている場合にのみ使用できます。

     
    電子メールの検索 (基本検索) [Add to Search Form] 領域で [電子メール] をクリックします。[OTEmailSubject]、[OTEmailReceivedDate]、および [OTEmailSentDate] などの検索条件を指定します。電子メール送信者および受信者の名前を [検索] ボックスに指定することもできます。
     

    注意:

    • [検索] ボックスで複数の検索条件エントリを区切るには、セミコロンを使用します。

    • ワイルドカード検索はサポートされていません。

     
    電子メールの詳細検索の実行 [検索フォームに追加] 領域で [Email (Advanced)] をクリックします。[追加の属性] リストをクリックして検索条件となる列を選択してから、該当する値を指定します。
    物理アイテム タイプの検索 [詳細のオプション] 領域で [物理アイテム タイプ] をクリックします。[物理アイテム タイプ] 領域で [Select Physical Item Type] リストをクリックして、[物理アイテム タイプの参照] をクリックします。[物理アイテム タイプの参照] ページで [選択] をクリックして、適切なアイテム タイプを選択します。
     

    注意:
    このオプションは、環境に物理オブジェクトがインストールされている場合にのみ使用できます。

     
    物理アイテム属性の検索 [Add to Search Form] 領域で [物理アイテム属性] をクリックします。[物理アイテム属性] 領域で、検索する属性を指定します。[格納場所 (ホーム)] リストは既定で提供されますが、必要に応じて削除できます。検索する属性をさらに追加するには、[追加の属性] リストをクリックします。
     

    注意:
    このオプションは、環境に物理オブジェクトがインストールされている場合にのみ使用できます。

     
    Records Management 属性検索の実行 [Add to Search Form] 領域で [Records Management属性] をクリックします。[ Records Management 属性] 領域で、[追加の属性] リストをクリックして選択し、検索する属性を定義します。
     

    注意:
    このオプションは、環境に Records Management がインストールされている場合にのみ使用できます。

     
    テンプレートを使用して検索するには [Use this Search Form] リストで、使用するテンプレートをクリックします。[検索] ページが開いて、選択したテンプレートで指定した検索条件が表示されます。必要に応じて、検索条件を編集できます。
    その他の条件の追加 使用できる場合は、[コピー] をクリックし、リストでブール演算子をクリックして、その他の条件を指定します。
    条件の削除 使用できる場合は、削除する条件を対象として [削除] をクリックします。
    検索フォームからのオプションの削除 削除するオプションを対象として、[コンポーネントを削除] をクリックします。
    すべての検索条件の削除 [クリア] をクリックします。

  3. [検索] をクリックします。

 

注意:

  • システム属性の日付範囲やカテゴリ内のカスタム属性を指定する時には、[開始日] または [終了日] リストの日付を空白のままにできます。[開始] リストを空白にした場合は、[終了] リストの日付よりも前の日付すべてを対象に、Content Server で検索が実行されます。[終了日] リストを空白にした場合は、[開始日] リストの日付から今日までを対象にして、Content Server が検索を実行します。

  • 検索結果リストの一部のアイテムでは、コンテキスト メニューの [ブラウザーで起動] オプションが、使用できる唯一のオプションになることがあります。この状況は、返されたアイテムの親フォルダーにアクセスする権限がない場合、または、[Enterprise Connect] を使用してボリュームを表示できない場合に発生します。別のブラウザー インスタンスで選択したアイテムを操作する場合、[ブラウザーで起動] をクリックします。この動作は、[Enterprise Connect] 検索とクイック サーチにも適用されます。

  • 環境によっては、[表示オプションの編集] リンクをクリックすることによって、検索結果の表示オプションを設定できる場合があります。表示オプションのページで利用可能なオプションについては、 表示オプションの設定を参照してください。
