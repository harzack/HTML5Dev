アイテムが入っているフォルダーにアクセスできる適切な権限がないために、状況によっては、検索で閲覧できないアイテムが返されることがあります。これらのタイプのアイテムは、親がないアイテムとして知られています。検索結果リストで親がないアイテムを開くと、選択したアイテムは、Enterprise Connect ツリーの [追加アイテム] フォルダー内のフォルダー構造に表示されます。

親がないアイテムと [追加アイテム] フォルダーを操作する場合は、次の項目に注意してください。

  • 検索結果リストで親がないアイテムを開いたり、[元のロケーションに移動] コマンドを実行したりすると、[追加アイテム] の場所が使用されます。

  • Content Server Web UI で親がないアイテムに対して [ここから Enterprise Connect] コマンドを実行した場合、Enterprise Connect は、[追加アイテム] フォルダー内の場所からアイテムを開きます。

  • [追加アイテム] フォルダーは、必要な場合に、Enterprise Connect ツリー内でのみ表示されます。

  • 作成すると、[追加アイテム] フォルダーは、Enterprise Connect セッションの実行中は、Enterprise Connect 内で使用可能の状態で維持されます。

  • 複数の Content Server リポジトリがある場合、それぞれのリポジトリ内には、[追加アイテム] フォルダーの個別のインスタンスが存在しています。

  • [追加アイテム] フォルダーにアイテムを直接追加することはできません。ただし、[追加アイテム] フォルダーに表示されるフォルダーにはアイテムを追加できます。

  • [追加アイテム] フォルダーは、セッションの完了時に Enterprise Connect ツリーから削除されます。

  • [追加アイテム] フォルダーは削除できません。

  • 十分な権限がある場合、[追加アイテム] フォルダーからアイテムを削除できます。この操作では、[追加アイテム] フォルダーからアイテムが削除され、Content Server 内の元の場所からアイテムが削除されます。
