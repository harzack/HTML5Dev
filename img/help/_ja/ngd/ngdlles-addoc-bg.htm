ドキュメントは、コンピューターから Content Server データベースにアップロードされる電子的なファイルです。ドキュメントの例としては、スプレッドシート、メモ、HTML ページ、画像、音声ファイルなどがあります。各機能を使用して、ドキュメントを Content Server に追加する、ドキュメントの内容を編集する、開いて確認する、バージョンやジェネレーションを管理するといった作業を行います。

 

注意:
これらの機能を使用するには、追加するドキュメントに対して適切な権限が必要です。権限の設定の詳細については、権限の設定を参照してください。

 

ドキュメントの追加

コンピューター上に既に存在するドキュメント ファイルをアップロードして、Content Server にドキュメントを追加できます。また、新しいドキュメントを作成することもできます。

適切な権限を持っている場合は、オリジナルのドキュメントのメジャーまたはマイナー バージョンとしてドキュメントを追加できます。ドキュメントをバージョンとして追加する権限がない場合は、ドキュメントをメジャーまたはマイナー バージョンとして追加するオプションが表示されません。

重複するドキュメント

ドキュメントを Content Server に追加しており、同じ名前のドキュメントが目的の場所に既に存在していた場合、重複するドキュメントを処理する方法を指定できます。例えば、新しいドキュメントは、目的の場所に既に存在しているドキュメントの 1 バージョンとして追加できます。複数のドキュメントを Content Server に追加している場合は、バッチ内の全重複ドキュメントを既存のドキュメントの新バージョンとして追加するように指定できます。また新しいドキュメントのみを Content Server に追加して、重複するドキュメントは追加しないことも指定できます。

[ドキュメント概要] ページの使用

[ドキュメント概要] ページには、説明や関連情報など、ドキュメントの情報が表示されます。タスクの実行権限がある場合、[ドキュメント概要] ページでは、バージョンを編集する、ダウンロードする、開く、追加するなど、多くの機能をドキュメントに対して実行できます。

ドキュメントの編集

[編集] を使用するなど、ドキュメントを編集する方法はいくつかあります。Microsoft Office ドキュメントまたは Adobe Acrobat ドキュメントを編集する場合、ドキュメントは適切なアプリケーションで開かれます。アプリケーションを閉じるまで、アイテムは Content Server 内で予約済みとしてマークされます。Content Server では、ユーザーが変更を保存するたびに、ドキュメントの現在のバージョンが更新されます。

開いたり編集したりするドキュメントの権限レベルにより、ユーザーが使用できるドキュメントのバージョンが決まります。ドキュメントの権限を変更した場合、そのドキュメントの最新バージョンにアクセスできるようになります。ドキュメントのステータスに応じて、ドキュメントはメジャー バージョンまたはマイナー バージョンになります。ドキュメントに対して読み取り専用の権限を許可されている場合、そのドキュメントの最新メジャー バージョンのみにアクセスできます。

 

注意:
適切なドキュメントを右クリックして [開く] をクリックすると、編集のために一部のドキュメントを開くことができます。

 
 

ヒント:
設定によっては、Enterprise Connect 内で Microsoft Office ドキュメントまたはそのドキュメントのショートカットをダブルクリックすることにより、ドキュメントを編集のために開くことができます。
