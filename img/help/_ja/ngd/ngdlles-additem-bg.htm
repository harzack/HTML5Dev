[アイテムの追加] メニューは、アイテムを追加する権限を持っているあらゆる場所で使用できます。アイテムを追加する時に使用できるオプションは、アイテムのタイプによって異なります。アイテムの追加の詳細については、特定のアイテム タイプのヘルプ トピックを参照してください。

アイテムを追加する時は、権限により、アクセスできるアイテムが決まることに注意してください。既定では、追加するすべてのアイテムは追加先のコンテナーから権限を継承します。例えば、グループにフォルダーに対する内容の参照権限が割り当てられている場合、そのグループのメンバーはフォルダーに追加されるすべてのドキュメントを参照する権限を持ちます。また、フォルダー内の個々のアイテムの権限を変更することもできます。アイテムを追加したユーザーはアイテムの所有者になり、それらのアイテムに対するすべての権限が与えられる場合があります。権限の設定の詳細については、権限の設定を参照してください。

メタデータの提供

環境の設定に応じて、Content Server フォルダーまたはその他のタイプのコンテナーに追加するアイテムについて、カテゴリ属性など、メタデータを提供する必要があります。状況によっては、一部のメタデータはアイテムから自動的に抽出され、[追加] ダイアログ ボックス内のボックスに表示されます。設定に応じて、これらのボックスを編集できることがあります。

複数のアイテムを Content Server コンテナーに追加している場合、[類似したタイプのすべてのアイテムに適用する] チェック ボックスが表示されることがあります。このチェック ボックスをオンにすると、そのアイテムに指定したメタデータは、それ以降、グループ内にある同じタイプのすべてのアイテムに適用されます。

[追加] ダイアログ ボックスでは、追加するほぼすべてのアイテムに共通するパラメーターがいくつか存在しています。次の表にこれらのパラメーターを示しますが、これらは Content Server コンテナーにアイテムを正常に追加するための必須パラメーターとは限りません。

パラメーター 説明
名前 Content Server では、ユーザーが追加するすべてのアイテムには既定の名前が自動的に付けられます。例えば、ドキュメントの場合は、追加されるファイルの名前が使用されますが、[名前] ボックスにわかりやすい名前を入力しておくと、アイテムの内容と目的が名前から分かりやすくなります。また、分かりやすい名前にすることで、検索クエリや LiveReport で正確に識別される可能性が高くなります。
説明 説明を追加しておくと、検索クエリや LiveReport でアイテムが正確に識別される可能性が高くなります。Content Server がインデックスを作成する時に、[説明] ボックス内のテキストはアイテムのメタデータの一部として使用されます。アイテムのメタデータが多ければ多いほど、Content Server はアイテムの内容と目的を識別しやすくなります。
カテゴリ 追加するアイテムに 1 つ以上の属性を適用するには、[カテゴリ] ボックスを使用します。カテゴリの操作の詳細については、カテゴリの使用を参照してください。
作成場所 既定では、アイテムは現在のコンテナーに追加されますが、異なるコンテナーにアイテムを追加することも可能です。

次の例の手順では、アイテムを Content Server コンテナーに追加する時に検出される可能性があるボックスを示します。設定や使用できるモジュールに応じて、追加的なボックスが表示されることがあります。

  1. 追加するアイテムを選択して、適切な Content Server フォルダーにこれらのアイテムをドラッグします。

  2. [追加] ページで、次のいずれかの作業を実行します。

    • [説明] ボックスに、アイテムの説明を入力します。

    • 可能な場合は、アイテムに対して適切なクラシフィケーションを選択します。

    • 必要に応じて、[編集] をクリックしてカテゴリ情報を入力します。

    • それ以降、グループ内にある同じタイプのすべてのアイテムにこれらの値を適用する場合は、[類似したタイプのすべてのアイテムに適用する] チェック ボックスをオンにします。

       

      注意:
      アイテムをコンテナーに追加している時に [キャンセル] をクリックすると、処理が停止されてそれ以降のアイテムが目的の場所に追加されなくなります。既に追加されたファイルは削除されません。

       

  3. [OK] をクリックします。
