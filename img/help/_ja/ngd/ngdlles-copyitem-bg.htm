同じ Content Server リポジトリ内では、[Enterprise Connect にコピー/移動] コマンドを使用して、個々のアイテムまたは複数のアイテムを特定の場所から別の場所にコピーできます。ドキュメントまたはコンテナー以外のアイテムをコピーしている場合、Enterprise Connect の標準的なコピー コマンドを使用できます。フォルダーなど、コンテナーをコピーしている場合は、コンテキスト メニューを使用して、[Enterprise Connect に移動/コピー] コマンドにアクセスできます。

 

ヒント:
またエクスプローラーで作業している場合は、右マウス ボタンでもドキュメントとフォルダーをコピーできます。コピーするアイテムを選択して、右マウス ボタンで目的の場所までアイテムをドラッグし、[コピー] をクリックします。このアクションが使用可能になるのは、ローカル フォルダーから Enterprise Connect にアイテムをドラッグしている場合、同じ Content Server リポジトリ内で特定の場所から別の場所にアイテムをドラッグしている場合、Enterprise Connect の検索結果から、同じ Content Server リポジトリ内または異なるプラグイン内のフォルダーにアイテムをドラッグしている場合です。

 

アイテムを新しい場所にコピーする場合、コピーしようとしている 1 つ以上のアイテムで、元々割り当てられていたカテゴリをそのまま継承する、コピー先に割り当てられているカテゴリを継承する、コピー元とコピー先の両方のカテゴリを継承するなど、これらのいずれかを指定するように指示されることがあります。また、システムの設定方法によっては、割り当てられた OpenText Records Management クラシフィケーションを、アイテムで継承するかどうかを指定するように指示されることもあります。

同じファイル名のアイテムが既に目的の場所に存在していた場合、名前の競合の解決方法を指定するように指示されることがあります。設定に応じて、次のオプションが使用できることがあります。

  • 自動的に生成されたファイル名で現在のアイテムを目的の場所にコピーすると指定できます。この設定では、すべてのバージョン、監査履歴、またはアイテムと関連付けられたメタデータを維持できます。複数のアイテムをコピーしている場合は、名前の競合をすべてこの方法で解決するように指定することもできます。

     

    ヒント:
    十分な権限がある場合は、コピー アクションの完了後にファイルの名前を変更できます。

     

  • 現在のアイテムを目的の場所にコピーしないことを指定できます。複数のアイテムをコピーしている場合は、バッチ内の重複アイテムをコピーしないことを指定することもできます。

  • コピー アクションを続行しないと指定できます。既に正常にコピーされたすべてのアイテムは目的の場所に残されます。

次のドキュメント以外のアイテム タイプ、コンテナー以外のアイテム タイプでは、アイテムの個別のインスタンスまたは複数のインスタンスを Content Server リポジトリ内の新しい場所にコピーできます。

  • カスタム ビュー

  • アンケート

  • ショートカット

  • URL

  • ワークフロー マップ

  • ドキュメント

  • テキスト ドキュメント

  • wiki ページなど、Content Server モジュールで提供されるサブタイプ。

 

注意:

  • 電子メール フォルダー、プロジェクト、コンパウンド ドキュメント、ワークフロー ステータス、タスク リスト、チャンネル、マイルストーンなど、フォルダーまたはフォルダーと同等の Content Server アイテムをコピーするには、コンテキスト メニューを使用します。

  • この機能を使用するには適切な権限が必要です。権限の設定の詳細については、権限の設定を参照してください。

 
 

ヒント:
使用可能なコマンドなど、Enterprise Connect を使用してアイテムをコピーする方法の詳細については、OpenText Enterprise Connect Plug-in for Content Server - ユーザ ガイド (NGDCORE-UGD)を参照してください。
