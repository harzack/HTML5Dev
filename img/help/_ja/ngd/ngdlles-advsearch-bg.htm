[検索Content Server] ダイアログ ボックスでは、キーワードを含むクエリや、 Live Query Language (LQL) で構成された複合クエリを構成できます。ここでは、検索範囲を拡張または絞り込むことができるコンポーネントについても説明します。検索範囲を拡張するには、修飾語を選択し、検索に含める関連語の種類を指定します。検索範囲を絞り込むには、アイテムのコンテンツ内を検索するか、メタデータ内を検索するか、またはその両方を指定します。コンテンツとは、ドキュメントのテキストなど、アイテム内の情報です。メタデータとは、ドキュメントが追加された日付など、各 Content Server アイテムに付属する情報です。また、ブール演算子を使用して、クエリ内の検索条件の関連を正確に定義することで、検索範囲を絞り込むこともできます。

キーワードは、特定の語またはフレーズを含むアイテムを検索する時に指定する、語またはフレーズです。例えば、business plan model というキーワードを指定した場合、Content Server はこれら 3 つの語のすべてか、いずれかの語、あるいは完全に一致するフレーズを含むアイテムのリストを返します。複数のキーワードを引用符で囲んだ場合、Content Server は引用符内に指定された通りのフレーズを検索します。

クエリ内の語に関連する用語を含むアイテムを検索する時は、検索範囲を拡張します。修飾語を選択すると、関連用語の型を指定して検索に含めることができます。例えば、修飾語を選択して、指定したキーワードに似た単語を含むアイテムを検索することができます。また、指定したキーワードの語幹と関連した単語を含むアイテムを検索することもできます。例えば、interface というキーワードの語幹と関連した単語を検索すると、interface、interfaces、interfacing、interfaced などの単語を含むアイテムのリストが返されます。

キーワードを修飾語と併用する場合は、修飾語はすべての語に適用されます。フレーズと完全に一致するアイテムを検索する場合は、修飾語はフレーズ全体に適用されます。複合クエリでは、クエリ内にある LQL キーワード以外のすべての語に修飾語が適用されます。LQL の詳細については、複合クエリを使用した検索を参照してください。

Content Server が検索で使用する検索条件の数を増やして、検索を絞り込みます。これらの検索条件には、ブール演算子の AND、OR、NOT が含まれます。これらは、キーワードの相互の関連性を変更する基本的なロジックです。条件には、アイテムのコンテンツ、メタデータ、またはその両方を検索するオプションも含まれます。その他の検索条件の型には、スコープ、システム属性、カテゴリのカスタム属性、XML タイプなどの詳細オプションがあります。次の表では、使用可能な検索条件について説明します。

条件 検索のタイプ
検索範囲: すべて ドキュメントのテキストなど、各アイテムに含まれている情報だけでなく、管理者が検索可能と設定したメタデータも検索します。
検索範囲: コンテンツ ドキュメントのテキストなど、各アイテムに含まれている情報を検索します。
検索範囲: メタデータ ドキュメントが追加された日付など、各アイテムに付属する情報を検索します。メタデータ内を検索する場合は、管理者が検索可能として設定した (既定で検索とマークが付けられた) メタデータ情報に検索が絞り込まれます。この場合、アイテムに関連づけられたすべてのメタデータが含まれることもあれば、含まれないこともあります。
ブール演算子 クエリ内の検索条件間の関連を、正確に定義します。例えば、plan と test がキーワードの場合、plan AND test を検索すると、plan と test の両方が含まれるアイテムが返されます。plan OR test を検索すると、plan または test が含まれるアイテム、あるいはこの両方が含まれるアイテムが返されます。plan NOT test を検索すると、plan は含まれ、test は含まれていないすべてのアイテムが返されます。
システム属性 ある属性を持つアイテムを検索します。例えば、作成者属性が特定のユーザーに設定されているすべてのアイテムを検索できます。

また Content Server で作成し、維持しているパーソナルおよびシステム テンプレートを使用して、Enterprise Connect で詳細検索を実行できます。Enterprise Connect で特定のテンプレートを選択すると、[検索] ページには、テンプレートで指定した検索条件とフォームが表示されます。テンプレートの操作の詳細については、 検索フォームの操作を参照してください。

 

注意:
テンプレートに Enterprise Connect で提供されていない検索フォームが含まれている場合、そのコンポーネントは [検索] ページには表示されません。[フル テキスト]、[システム属性]、[カテゴリ]、[メール]、[場所]、[物理アイテム タイプ] が現在サポートされています。
