外観はドキュメントと HTML コードを格納するコンテナーです。外観では、Web ページのデザインと同じように、特定のロケーションをカスタマイズできます。

 

重要項目:
Enterprise Connect を使用して外観を作成したり編集したりできますが、これらは Enterprise Connect では表示できません。またこれは、Content Server Web インターフェイスで作成した外観にも適用されます。

 

インターフェイスをカスタマイズするために追加できる外観には、以下の 2 種類があります。

  • グローバル外観: Content Server 全体の外観として特定のユーザーに適用されます。グローバル外観は外観ボリュームにのみ追加可能です。このボリュームに追加することで、グローバル外観はシステム全体で使用できるようになります。

  • ロケーションベースの外観。

ロケーションベースの外観: 以下の 2 通りの方法で適用できます。

  • カスケード表示をしない: 外観が追加された ワークスペース、プロジェクトまたはフォルダーにのみ外観を適用します。

  • カスケード表示をする: 外観が追加された親 ワークスペース、プロジェクトまたはフォルダー、およびすべての子プロジェクトまたはフォルダーに外観を適用します。この種類の外観は、特定のユーザーまたはグループの集合にのみ適用することもできます。

外観は、[概要] ページと [ワークスペース] ページに分割され、外観の定義や編集はこのページで行います。外観を追加した後、[概要] ページで設定およびレイアウトの定義を行い、画像などのコンテンツを [ワークスペース] に追加します。通常、外観の追加および編集機能は制限されています。
