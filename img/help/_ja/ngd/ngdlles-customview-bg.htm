カスタム ビューは、HTML コードを含むアイテム タイプです。カスタム ビューを ワークスペース やフォルダー、プロジェクト、タスク リスト、コンパウンド ドキュメントに追加すると、Content Server に、カスタム ビューに含まれる HTML コードがインターフェイスの一部として表示されます。カスタム ビューは、コンテナーや ワークスペース でナビゲーション メニューとアイテムの間に挿入されます。

カスタム ビューを追加する場合は、まずカスタム ビューを定義する HTML ファイルと画像ファイルやスタイル シートなどその他のファイルを作成します。デフォルトでカスタム コードに使用できるコードは以下の通りです。

  • HTML、JavaScript、VBScript、Java アプレット、または Web ブラウザーで解釈可能なその他のコード。ただし一部のコンテンツについては、管理者が追加できないように制限している場合があります。

  • Content Server 内の他のアイテムや他の Web ページへのリンク。

  • Content Server、またはアクセス可能な他の Web サーバー上のディレクトリに格納された画像や音声などその他のファイルへの参照。

カスタム ビューのコンテンツはインターフェイスを表示する HTML ファイル内に挿入されるため、Web ブラウザーでは <TITLE> や <BODY>、<FRAMESET> などのタグが無視されます。このため、カスタム ビューで使用できるコードのタイプには制限があります。例えば、カスタム ビューにフレームを作成することはできません。

 

注意:
カスタム ビューと同じロケーションにリソース フォルダーを追加すると、カスタム ビューで使用する画像およびその他のファイルをそのフォルダーに格納できます。これらのファイルが誤って削除されるのを防ぎ、また ワークスペース を整理された状態に保つには、カスタム ビューとリソース フォルダーを非表示にすることを検討してください。

 

通常、カスタム ビューを作成および管理する機能は、一定のユーザーまたはグループに制限されています。カスタム ビューを追加するには、カスタム ビューのオブジェクト作成特権および適切なコンテナーに対するアイテム追加権限と編集権限が必要となります。ただし、カスタム ビューが影響を及ぼす可能性のあるサブアイテムを変更するには権限は必要ありません。他のユーザーが追加したカスタム ビューを変更するには、カスタム ビューに対する編集権限が必要です。

カスタム ビューを適用するユーザーまたはグループを決定するには、カスタム ビューに対する権限を設定します。
