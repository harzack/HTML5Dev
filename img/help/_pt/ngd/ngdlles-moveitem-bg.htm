You can use the Copy/Move to Enterprise Connect command to move individual or multiple items from one location to another within the same Content Server repository. If you are moving a document or non-container item, you can use the standard Enterprise Connect move commands, such as dragging. If you are moving a container, such as a folder, you can use the context menu to access the Copy/Move to Enterprise Connect command.

 

Tip:
If you are working in Windows Explorer, you can also move documents and folders using the right mouse button. Select the items that you want to move, drag the items to the destination using the right mouse button, and then click Move. This action is available when you drag items from a local folder to Enterprise Connect, when you drag items from one location to another in the same Content Server repository, and when you drag items from an Enterprise Connect search result to a folder either in the same Content Server repository or in a different plug-in.

 

When you move an item to a new location, you may be prompted to specify if the item or items that you are moving will keep any originally assigned Categories, if they should assume the Categories that have been assigned to the destination, or if they should assume the Categories of both the original and destination. Depending on how your system is configured, you may also be prompted to specify if the items should inherit any assigned OpenText Records Management Classifications.

If an item already exists in the destination with the same file name, you may be prompted to specify how you want to resolve the naming conflict. Depending on your configuration, the following options may be available:

  • You can specify that the current item be moved to the destination with an automatically generated file name. This enables you to preserve any versions, audit trails, or metadata associated with the item. If you are moving multiple items, you can also specify that all naming conflicts be resolved in this manner.

     

    Tip:
    If you have sufficient permissions, you can rename the file after the move action is complete.

     

  • You can specify that you do not want to move the current item to the destination. If you are moving multiple items, you can also specify that no duplicates in the batch are moved.

  • You can specify that you do not want to continue with the move action. Any items that have already been successfully moved will remain in the destination.

You can move individual or multiple instances of the following non-document, non-container item types to a new location in the Content Server repository:

  • Custom Views

  • Polls

  • Shortcuts

  • URLs

  • Workflow Maps

  • Documents

  • Text Documents

  • Subtypes that are provided by Content Server modules, such as wiki pages.

 

Notes:

  • Use the context menu to move Folders or Folder-equivalent Content Server items such as Email Folders, Projects, Compound Documents, Workflow Status, Task Lists, Channels, or Milestones.

  • You must have sufficient permissions to perform this function. For more information about setting permissions, see Setting Permissions.

 
 

Tip:
For more information about moving items using Enterprise Connect, including available commands, see OpenText Enterprise Connect - User Getting Started Guide (NGDCORE-UGD).
