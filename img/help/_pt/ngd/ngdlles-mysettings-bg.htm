When you add emails to Content Server using Enterprise Connect, naming conventions are applied to the email. If your administrator has enabled the My Settings node in the Enterprise Connect Tree, you will be able to specify rules for email names. The initial values in the Name Generation for E-mails page have been set by your administrator. You can do the following on the Name Generation for E-mails page:

  • Specify which message properties to use when generating a Content Server name for email.

  • Define the separator string to be used between the individual message properties that make up the name of the email.

  • Specify any strings that you want to remove from the generated email name.

  • Specify the format used for dates and times.

  • Specify how duplicate emails should be handled.

 

Note:
If Enterprise Connect generates a name for the email that has already been used in the Content Server container, you will be prompted to resolve the naming conflict before the email can be added. For more information about resolving file name conflicts, see Working with Email.
