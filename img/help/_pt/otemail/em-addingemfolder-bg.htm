You use E-mail Folders to organize your work in Content Server the same way that you work with folders in your email system.

E-mail Folders are similar to regular Content Server Folders. E-mail Folders, however, enable you to view large numbers of emails on one page at a time. When you store emails in an E-mail Folder, the following additional columns of information are provided by default. These columns may also appear in regular Content Server Folders, if the Administrator has enabled the option.

  • From

  • To

  • Subject

  • Sent Date

  • Received Date

You should use E-mail Folders when the primary contents stored in the Folder are email messages and the Folder will contain a large number of email messages.

Adding an E-mail Folder

You can add an E-mail Folder to every Workspace or container page where you have the permission to add items.

When you add an E-mail Folder to Content Server, you can provide a description of the E-mail Folder, modify its Categories Attributes, and add the E-mail Folder to a different container. For more information about adding items, see Working with General Items.

Permissions

By default, any item that you add to an E-mail Folder inherits its permissions from the E-mail Folder you put it in. For example, if a group has See Contents permission on an E-mail Folder, members of that group will have permission to see any item you add to that E-mail Folder. However, it is possible to modify the permissions of individual items in an E-mail Folder. For more information, see Understanding Permissions.
