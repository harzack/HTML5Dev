Important:
Any user given See Contents permission to a Virtual Folder can view that Virtual Folder. However, you can only create Virtual Folders if your administrator has given you permission to create Virtual Folders. If you are unable to create a Virtual Folder, see your administrator.

 

A Virtual Folder is an object in Content Server which defines both:

  1. a selection of facets, and

  2. a location within Content Server to which that selection will be applied.

Users with See Contents permission to a Virtual Folder can select it to display a Virtual Folder View of the faceted browse that the user who created the Virtual Folder has defined.

Because Virtual Folders are saved faceted browsing selections, it is necessary that you have a clear understanding of Using Faceted Browsing before you begin to view, create or edit Virtual Folders.

 

Note:
Administrators can find instructions about how to give a user permission to create a Virtual Folder in Configuring Virtual Folders. Expand the help topic Server Configuration, then select Configuring Virtual Folders.

 

Virtual Folders provides two views:

  • The Virtual Folder View of a Virtual Folder will display the names of all items in Content Server which both:

    1. conform to the settings that the creator of the Virtual Folder defined for that Virtual Folder, and

    2. that the user has See Contents to view.

  • The Static View will display the settings that were applied to that Virtual Folder. It is only accessible by Knowledge Managers. This view is the Virtual Folder's Specific tab of its Properties page. The Static View allows users, with the appropriate permissions, to edit the settings of the Virtual Folder.

Example 5-1: Example of a Virtual Folder

Using the example found in Using Faceted Browsing, the administrator has given a user permission to create a Virtual Folder. That user has created a Virtual Folder, called DeptMemo, with the following settings:

  • DeptMemo will only display items which have been saved to the Administration folder, or any child Folder of the Administration folder

  • DeptMemo will display all Content Server items which have the value Document assigned to the Type facet

  • DeptMemo will display all Content Server items which have the value John Smith assigned to the Owner facet

When a user clicks the name of the DeptMemo Virtual Folder, the Virtual Folder View of the DeptMemo Virtual Folder displays the names of all Documents owned by John Smith which were saved to the Administration folder, provided:

  1. the user has been given See Contents permission to view the Virtual Folder, and

  2. the user has been given See Contents permission to view the Documents owned by John Smith which were saved to the Administration folder.

Provided a user has permission to access the Static View, the Static View of the DeptMemo Virtual Folder displays:

  1. the Selected Location from which the browse begins: “Administration”

  2. a list of the names of the facets which will be browsed: “Content Type: Document” and “Owner: John Smith”.

  3. options that the user with edit permissions can apply to this Virtual Folder. For more information about these options, see To Review or Edit a Virtual Folder's Settings.
