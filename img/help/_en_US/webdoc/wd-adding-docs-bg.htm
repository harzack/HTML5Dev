There are two ways to add documents to Content Server:

  • by uploading a document file that already exists on your computer

  • by creating a new document in Content Server

     

    Note:
    Creating a new document requires that you use the Microsoft Internet Explorer Web browser.

     

When you add an existing document to Content Server, you can specify a name and description, apply Categories and Attributes, and choose the location in Content Server where you want to store the document. If the document you are adding is of a type that has more than one Category associated with the document type, you will be taken to a second page where you can choose which category you want to assign to the document. For example, suppose your administrator has associated JPG and TIF files with Categories called “Scanned Image” and “Digital Photograph”. When you add files with the .jpg or .tif extensions, you will be asked to select the Scanned Image and / or Digital Photograph Category on a second page after you add the item.

 

Note:
This functionality is only available if your administrator has enabled the Advanced Add Item process and the type of document you are adding is a MIME type that your administrator has associated with more than one Category. You will be prompted to choose a Category, or multiple Categories if there is more than one category associated with the MIME type.

 

When you add a new document to Content Server, you select a document type, specify information, and then compose its contents in an external application. Document types that you can add are set by the administrator, but can include Microsoft Word, Excel and PowerPoint files. Once you add the new document, a blank version opens in its associated desktop program where you can compose its contents.

If you have the proper permissions, you can add a document as a major or minor version of the original. If you do not have permission to add a document as a version, the choice to add a document as a major or minor version does not appear. For more information about advanced versioning, see Working with Versions.

 

Note:
For information about using a Macintosh computer with a Netscape browser to add documents to Content Server, see Adding Macintosh Files to Content Server.

 

If you use Microsoft Internet Explorer, you can also edit documents in Content Server. You do this by starting an edit session. An edit session does the following things for you automatically:

  • reserves the document and downloads a temporary file to your computer, and

  • opens the temporary file in its associated desktop program, so that you can modify it.

When you are finished making changes, you end the edit session. Ending an edit session involves choosing one of the following options:

  • saving the changes to Content Server as a minor version of the original Document; or

  • saving the changes to the local file on your computer, leaving the document on Content Server as it is; or

  • discarding the changes altogether.

 

Note:
When you are working in a Content Server edit session, do not close the Web browser until you are finished editing the document. If you close the Web browser without saving your changes, they may be lost.

 
 

Tip:
To see your changes in Content Server, refresh the Web browser when you end an edit session. There is a Refresh the browser hosting this Edit session check box which is selected by default. Leave this check box selected to refresh the Web browser when you end your edit session.
