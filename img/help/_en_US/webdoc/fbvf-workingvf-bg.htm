Once a Virtual Folder has been created, users who have been granted See Contents permissions to that Virtual Folder can use it to browse the Virtual Folder's predefined facets.

You will see a browse banner above the sidebar when:

  • a user selects a facet from the available facets in the Content Filter panel of the sidebar, or

  • a user selects a Virtual Folder

The Browse Banner

The browse banner appears across the entire page, above the sidebar. It will display whenever you are accessing a faceted browse view of Content Server's contents. The options that the browse banner displays change dynamically, depending on your permissions.

  • The browse banner displays the heading Virtual Folder View when a user clicks a Virtual Folder's name. This view displays the names of the items in Content Server which conform to the settings that were defined for that Virtual Folder.

    The Virtual Folder settings, which were set by the Virtual Folder's creator, may be displayed in the Content Filter panel of the sidebar, depending on how the Virtual Folder was configured.

    Depending on your permissions, the Virtual Folder View browse banner may also display the Edit Virtual Folder link. This is only displayed if the user has permission to edit the Virtual Folder. For more information about editing a Virtual Folder, see To Review or Edit a Virtual Folder's Settings.

  • The browse banner displays the heading Dynamic View when a user selects a facet from the Content Filter panel on the sidebar.

    The Dynamic View browse banner may also display:

    • The Remove all filters link. This is displayed when a user selects a facet from the Content Filter panel on the sidebar.

    • The Save as Virtual Folder link. This is only displayed if the user has permission to create a Virtual Folder. For more information about creating a Virtual Folder, see To Create a Virtual Folder.

Rules for Users Creating and Managing Virtual Folders

Only users who have been granted permissions by the administrator can create Virtual Folders. If the option to create or manage a Virtual Folder is not available to you, see your administrator.

To facilitate managing Virtual Folders which are intended to be used company-wide, your administrator may have designated a Virtual Folders container for you to store Virtual Folders you create.

  1. You can create a Virtual Folder in any container that you:

    • have permissions to create an object

    • can create a Folder

       

      Note:
      You can create a Virtual Folder within a Virtual Folder from the Add Item menu or using the Create In field of any Add: Virtual Folder page. For more information, see To Create a Virtual Folder within a Virtual Folder.

       

  2. You can only store a Virtual Folder, an Appearance or a Custom View in a Virtual Folder. No other item can be stored in a Virtual Folder.

  3. You can store a Virtual Folder in a Virtual Folder. The sub-Virtual Folder will be displayed, by default, as a feature item when a user views the parent Virtual Folder. The user with permission to edit the Virtual Folder can change this default setting. See To Review or Edit a Virtual Folder's Settings, for more information.

  4. When a Virtual Folder appears in a directory list, it will not display a value for size. The number of items inside the Virtual Folder is not displayed. When the user clicks the Virtual Folder, each item displayed by the Virtual Folder will display a value for size.

  5. When you create a Virtual Folder for your users, the users will only see the location of the container from which you selected the saved facets if you select the Show Selected Location check box.

    If this check box is selected, the location will display in the Content Filter panel on the sidebar when the end user selects that Virtual Folder.

  6. When you create a Virtual Folder for your users, the users will only see the list of the facets that you selected for this saved facet browse if you select the Show Selected Facets check box.

    If this check box is selected, the facets will display in a list in the Content Filter panel on the sidebar when the user selects that Virtual Folder.

  7. When you create a Virtual Folder for your users, the users will only see the location, in Content Server, of each item which satisfies that Virtual Folder's criteria if you select the Show Location Column check box.

    If this check box is selected, the location of each item will display in the main browse view under the column heading Location. If this check box is not selected, the Location column does not appear.

  8. If you provide a user with edit permissions to your created Virtual Folder, but that user does not have permission to view one of the facets that you defined for your Virtual Folder, when that user views the Specific tab of your Virtual Folder, they will only see listed those facets they have permission to view. If the user attempts to edit that Virtual Folder, a message will display to inform the user that they do not have access to one or more of the facets that define the Virtual Folder.

    This means that a user to whom you granted edit permission to your Virtual Folder, could change the definition of your Virtual Folder without being able to see that Virtual Folder's full list of defined facets.

    Under these circumstances, the user will have the option of removing all facets to which they have not been granted See Contents permission, from the definition of the Virtual Folder.

    OpenText recommends that you ensure that the users to whom you grant edit permissions to your Virtual Folder have See Contents permission to all facets which define your Virtual Folder.

  9. It is possible to give a user See Contents access to a Virtual Folder and then define facets in that Virtual Folder that the user does not have See Contents permission to view. In the event that this occurs, that user will be able to access the Virtual Folder but the Virtual Folder will be empty.

     

    Caution
    Because Virtual Folders have separate permissions from the facets which define them, you can theoretically grant a user Edit permission to your Virtual Folder without that user having See Contents permission to any of the facets which define that Virtual Folder. In that case, if a user with Edit permission to that Virtual Folder updates it in edit mode, the facets which defined that Virtual Folder will be removed, and the Virtual Folder will be empty.
