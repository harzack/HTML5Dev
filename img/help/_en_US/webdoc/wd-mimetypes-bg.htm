In most cases, Content Server can automatically identify the type of any file added to its database. Whenever a Document is added to Content Server, its MIME type is stored as a part of its metadata. MIME, which stands for Multipurpose Internet Media Extensions, is a standard for describing file types. Examples of file types include text, image or audio.

For example, the MIME type application/msword denotes an MS Word file, and the MIME type text/richtext can denote a Plain-text Document. In Content Server, icons are used to identify file types. Examples of icons Content Server uses to identify file types include the Microsoft Word icon, the Text Document icon and the Document icon.

Examples of common MIME types:

  • application/msword

  • application/pdf

  • audio/x-wav

  • image/jpeg

  • text/richtext

  • video/mpeg
