If you have the item creation privilege for communities, you can create a community anywhere in Content Server where you have permission to add items. When you create a community, you also perform the following basic configuration tasks:

  • Setting the community type.

  • Specifying a unique name for the community.

  • Choosing a community template on which to base the community.

  • Choosing the directories in which to list your community in the community directory.

  • Specifying a mission and purpose for the community.

  • Subscribing roles to community email.

  • Creating a submission folder.

  • Specifying whether the community will receive external questions.

  • Adding members and specifying what type of members will receive community email.

 

Tip:
You can also create a community within a community, which is called a sub-community. The basic settings you configure when you create a sub-community are the same as for a community.

 

Setting the Community Type

The community type determines who can see the community and how members join. There are three types of communities:

  • Public: The full contents of these communities are visible to all users, and all users can request membership, but the facilitator must approve or reject requests.

  • Private: The names and missions of these communities are visible to all users – in workspaces and in the community directory – but their contents can only be viewed by members. All users can request membership, but the facilitator must approve or reject requests.

  • Unlisted: These communities are only visible to members. Users cannot request membership – it must be granted by the facilitator.

Specifying a Unique Name

The unique name that you specify when you create a community determines its email address, for example, unique_name@yourdomain.com, and the URL that accesses the community, for example, http://intranet.yourdomain.com/OTCS/livelink.exe/open/unique_name.

The email functions are carried out by the eLink module. For information about eLink, see Understanding eLink.

 

Note:
You cannot change the unique name of a community after it has been created.

 

Choosing a Community Template

Community templates provide a default structure and content for communities. You add and configure community templates to suit certain purposes, for example, a product development group, so that it is easier to configure newly created communities. Community templates are stored in the community templates volume, which makes them available system-wide. You can organize your community templates in the volume by storing them in community template folders.

 

Note:
Community templates are created and maintained by your administrator or users with the system administration rights privilege.

 

Choosing Where to List Your Community

The community directory is a listing of communities and their related elements, such as questions and answers, organized in a hierarchy of directories. It provides a way for users to browse for and find communities, and target questions. For more information about questions, see Using Questions and Answers.

When you create a community, you must choose one or more directories in which to list the community. You can add or change directory listings at any time. For more information about changing listings, see Configuring Communities.

 

Note:
Directories and directory trees, which organize one or more related directories, can be created only by the administrator or other privileged users.

 

Specifying a Mission and Purpose

The mission and purpose that you specify describe the intended goals of your community and the benefits that it will provide to members. This text is visible on the Community Homepage if you set the community to display the community info content module and on the Community Directory page. For more information about displaying content modules, see Configuring Communities.

Subscribing Roles

In eLink-enabled communities, the facilitator can automatically subscribe all users who have a particular role to receive all emails sent to the community email address. These emails are also stored in the mail archive. For information about subscribing using eLink, see Understanding eLink.

Creating a Submission Folder

A submission folder provides users with a convenient way to email content directly into the community. The submission folder is an eLink-enabled folder, and its email address is the unique name you specify when you create it, for example, unique_name@yourdomain.com.

If you do not create a submission folder when you create a community, and then want to add one, you must do so manually. For more information about adding items using eLink, see Adding Items Using eLink.

Allowing External Questions

External questions are questions asked through a directory under which the community is listed. When a community receives external questions, it lists them in community question reports on the Community Homepage, for example, the new questions report along with standard questions asked inside the community.

Adding Members

You can add members when you create a community or when you configure the community.

When you add members, you can also specify what type of members will receive emails sent to the community email address. You can choose to send email to all members who have a subscribed role, or only to direct members who have a subscribed role. For more information about direct and indirect members, see Administering Membership.
