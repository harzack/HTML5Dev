The native import/export capabilities in Content Server enable you to convert certain objects and information contained within a community into an XML file. This file can then be imported to another community elsewhere in Content Server, or to a community in a different Content Server instance. It is also possible to export a community itself.

Exporting

The following community object types, also known as nodes, can be exported:

  • Community

  • Sub-community

  • Community templates

     

    Note:
    It is possible to export an entire community and import it into the community templates volume, thereby making the imported community a community template. You cannot copy a community into the community templates volume to make it a template. For more information about community templates, see Administering Community Templates.

     

  • Forums

  • Blogs

  • Mail archive

     

    Note:
    If you export a mail archive and then import it into an existing community, the imported mail archive will be merged with the existing destination community.

     

  • Library

     

    Note:
    A community library is exported as a folder. This means that you can export a library and import its contents into another folder or into a different community library, where the imported library will appear as a new folder.

     

  • Content modules

  • My Workspaces in a community

  • FAQs

  • Comments

 

Note:
Expert groups cannot be exported individually; however, they can be exported as part of a total community export.

 

When exporting a community or content within a community, you can specify the following parameters:

  • Scope of the export, which includes the following:

    • This node only, which is just the item itself.

    • This node and only the first level children, which is the item and its immediate exportable contents, but not including items contained within those items.

    • This node and all child nodes, which is the item and all of its exportable contents, including all exportable items contained within those items.

  • Category and attribute information. For more information about categories and attributes, see Working With Categories and Attributes.

  • Version information, that is, export only the current version information for an item or all versions. For more information about versions, see Working with Versions.

  • Version text type, that is, export version content as plain text or as base64-encoded text.

  • Permission information, that is, include an item's permission information in the exported data. For more information about permissions, see Understanding Permissions.

Importing

You can perform an XML import into the following object types or locations:

  • Communities

  • Community templates

  • Template folders

  • The community templates volume

For more information about community templates, see Administering Community Templates.

The only restriction on importing is that the object specified by the XML file must conform to the native XML schema in Content Server.
