Transfers are used to track boxes when they are moved to a new location. When you create a transfer, you can specify the name of the user responsible for the transfer, an estimated size for the transfer, the date range of the content in the transfer, and a description of the transfer's contents.

Once you have added transfers, you can send and receive them, delete or rename them, view a list of assigned boxes for a transfer, calculate the size of the boxes currently in a transfer to update the transfer's estimated size, and synchronize the date of a transfer, which updates the content start and end dates for a transfer to match the content date of the oldest and newest boxes in the transfer. You can also assign physical items to a transfer.

To perform procedures associated with transfers, you access the Transfers page in the Storage Management section of the Physical Objects workspace. With the exception of creating a new transfer, you use the following parameters in the Date Search section to search for transfers:

  • All, which returns a list of all available transfers

  • Transfers not Sent, which returns a list of transfers that have not been sent

  • Transfers Sent not Received, which returns a list of transfers that have been sent but have not yet been received

  • Transfers Received, which lists transfers that have been sent and received

 

Note:
Once a list of transfers is returned, you can view additional information related to the transfer, such as items that are included in the transfer, by clicking the transfer's Show Details link.
