You can create Return Listing Reports that are compiled lists of borrowed physical items. The reports include information about whether the borrowed items are overdue and, if they are, the number of days they are overdue. It also includes information about the box or container (including the box number, a description, and company name), the contents of the box or container (including record numbers and descriptions, the From, To, and Status Dates associated with the contents), its location, and destruction date of the files associated with the box or container. You can also create Request Listing Reports, which include a list of requested items and the associated request dates.

When creating a Return Listing Report, you can specify the user who performed the borrows and a date range that the borrows were performed.

When creating a Request Listing Report, you can specify the user who performed the requests that the report is based on, the date range the requests were performed, the type of request, the home location of the items, and the location where the items are currently stored. You can sort the report results in ascending or descending order based on the following: Request Date, Home Location, Offsite Storage ID, or Unique ID.

When creating a Box Listing Report or Container Listing Report for a single box or container, you can browse to the box or container for which you are creating the report. If you are creating a report for multiple boxes or containers, you should enter a list of Unique IDs or scan a list of the barcodes.
