You can borrow a single physical item or multiple physical items, or you can borrow physical items in a Collection. You can also search for physical items, and then borrow the items that have been requested.

Borrowing a Single Physical Item

When you borrow a single physical item or physical item copy, you can specify comments and the name of the user who is borrowing the physical item. In some circumstances, a user may borrow a physical item on behalf of another user (for example, an Executive Assistant might borrow a physical item for an Executive Vice-President). Using this example, the physical item is “Borrowed By” the Executive Vice-President and “Obtained By” the Executive Assistant.

 

Notes:

  • The user performing the borrow must have See Contents permissions. The user assigned the borrowed item must also have See Contents permissions. If the user assigned the borrowed item does not have See Contents permissions but has See permissions, a permission override permits the borrow but does not change the user's permissions. Users without See permissions cannot borrow items.

  • You cannot borrow a physical item that has been identified as an item type that cannot be borrowed. For information about specifying physical item types that cannot be borrowed, see Administering Physical Item Types.

 

Borrowing Multiple Physical Items

When you borrow multiple physical items or physical items in a Collection, you can specify the location where you want the physical items to be placed and add comments.

Searching for Requested Physical Items

When searching for physical items that have been requested, you can specify the start and end date for the search, the user who requested the physical items, the location where the physical items are stored, and the current location where the physical items are stored. After the search, you can specify the location where you want the physical items to be placed and add comments.

Requesting Acknowledgement of Physical Items

To ensure responsibility for physical objects that are borrowed, Records Managers can enable the acknowledge functionality. This functionality enables the system to allow users to acknowledge receipt of an item when it is assigned or passed to them from the system or another user. When a user has a physical object assigned or passed to them, they must acknowledge that they have received it. If they fail to acknowledge the physical object, a reminder notification will be sent through email; if they still do not acknowledge that they have received the item, an escalation email will be sent to the Records Manager. External users are also required to acknowledge a physical object when it is sent to them.

The following options are available for items being borrowed:

  • Required, which will send out a notice to the Borrowed By user to acknowledge that they have received the item.

  • Not Required, which will not send out a notice to the Borrowed By user.

  • Confirmed, which will simultaneously borrow the item and confirm that it is acknowledged. This option is useful in cases where the Records Officer wants to automatically confirm that the borrow has received the item. For example, when a borrower is physically picking up materials at a records office.
