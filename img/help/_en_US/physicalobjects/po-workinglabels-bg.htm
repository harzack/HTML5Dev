Labels are blocks of information used for Physical Objects, and can include information such as the name or unique ID of the physical item, storage locator, or user.

You can generate or print a single label for a physical item, locator, or user. The group of objects or users that meet specified search criteria is called a batch. First you generate a label or batch of labels, and then you can print the labels. You can also generate a label for items in a Collection or set of Search results.

The criteria that appears when you search or generate labels in batches depends on the object type you have chosen. For every object type, you can specify the number of copies you want to generate. The following list details the parameters you can specify for each object type:

  • Physical Items: Generated By, Generated Date, Offsite Storage ID, Transfer ID, Physical Item Type, Facility, Area, Locator, Unique ID Range From\To, Label Stock and Type, and the starting position for printing the labels.

  • Locators: Facility, Area, Locator, and Label Type

  • Users: Users or Groups and Label Type

To print or remove labels in batches, you can search for the physical item or locator labels by specifying the label stock and type.

 

Note:
If you are a Records Manager (or user with by-pass permissions) and do not specify any search criteria when generating or printing batch labels, labels will be generated or printed for all physical items in Content Server. For all other users, labels will be generated for physical items for which you have the See permission; labels will be printed only for those you generated.

 

For information about designing the content that is generated on labels, see the OpenText Physical Objects Label Design Guide
