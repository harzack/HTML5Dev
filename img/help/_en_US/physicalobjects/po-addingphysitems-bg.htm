In Physical Objects, you can create a physical item that you can use to organize physical objects. A physical item is a virtual representation of a physical object and is associated with a physical item type.

When creating a physical item, you can specify a name, description, unique ID, keywords, categories, and the location where you are creating the physical item. The location where you create the Physical item is called the home location, if the home location is changed after the physical item is created, the current location is only updated if the value is the same as the old location.

If you are creating a physical item with a box or container physical item type, you can also specify the creation-date range for the items contained in or assigned to the box or container. The To and From dates for a container automatically update to correspond to the To and From dates of the contents within the container, which means if you specify dates and then add or remove items that have earlier or later To and From dates, the original dates for the container will change.

You can specify the frequency at which a box is accessed, an ID for boxes stored off site, and the user that is responsible for the box. If the physical item is managed, you can classify it.

You can create another physical item that contains the prepopulated metadata of an existing physical item. As well, once you create a physical item, you can create copies of it.

 

Note:
When you add a physical item, you can specify a unique ID for the item. If you do not specify a unique ID, one will automatically be assigned to it. If your administrator has enabled you to do so, the unique ID will appear after you create a physical item.
