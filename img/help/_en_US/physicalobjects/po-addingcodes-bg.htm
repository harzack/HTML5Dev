Physical Objects lets you enter codes associated with physical items. These codes allow you to set specific options users can choose when working with or managing physical items. You can add codes to the following tables on the Table Maintenance page:

  • Facility, which lets you enter codes that allow users to track boxes that are sent to off-site storage facilities. Facilities codes can be for managed or non-managed facilities. When adding codes to the Facility table, you can specify a description and address. You can set a managed facilities match rules for reference rate, storage type, length, depth and height. The match rules ensure that the properties for a facility and the properties for the facility's assigned boxes are the same.

  • Area, which lets you add codes that define specific areas in a storage facility. When adding a code to the Area table, you can specify a description.

  • Custodian Site, which lets you enter codes that allow users to circulate items within their custodian site but view items in other custodian sites.

  • Locator Type, which lets you add codes that are references to a single spot on a shelf that fits one box, an entire shelf that fits multiple boxes, or another physical location within an area where boxes can be stored. When adding a code to the Locator Type table, you can specify a description and the dimensions for the location.

  • External Client, which lets you enter codes for use in borrowing physical items. This table is useful if you want to assign physical items to non-Content Server users but do not want to have the information entered free-form. When adding codes to the External Client table, you can specify a description and the client email address (or multiple email addresses, separated by a semi-colon), physical address, and telephone number. You can specify if external client is External Borrower or Box Client. Box Client can be assigned to a box as a person or company responsible for a box.

  • Location, which lets you enter codes for specifying the location of physical items. You can use this table if you have set your system to disallow free-form entry of location fields. When adding codes to the Location table, you can specify a description and an e-mail address or multiple e-mail addresses, separated by a semi-colon. You can also add one custodian site or disable the location code so that it is unavailable in the Home Location drop-down list for physical items.

  • Recurrence, which designates the time interval between requests. For more information about requesting physical items, see Requesting Physical Items.

  • Reference Rate, which lets you add codes that describe the frequency at which a box or storage locator is accessed. When adding codes to the Reference Rate table, you can specify a description of the code.

  • Request Type, which lets you add codes for specifying the type of request. When adding a request type code, you can specify a description of the code.

 

Note:
In addition to being able to add codes on the Table Maintenance page, you can also create Table Maintenance codes on the page which the code is to be used. This feature is useful because it enables you to create codes on-the-fly when a code does not exist, or when the code required has not yet been created. The option to create a new code on the page where it is to be used is not available on the following pages:
