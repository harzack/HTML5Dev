Physical item types are definitions that users associate to physical items, enabling the user to identify the type of physical item being managed. When creating a physical item type, you can specify a name and description.

Once you have created the physical item type, you can edit its general properties, such as its name, description, and display settings, and edit its icon and unique ID. As well, you can specify that a physical item type cannot be borrowed, and you can hide fields from the Add Physical Item page and the Specific tab for a physical item type.

 

Note:
If the Unique ID, From Date, or To Date field is mandatory, you cannot hide the field.

 

On the Specific tab of a physical item's Properties page, you can view and modify information for a physical item such as its type, to/from dates, current location, locator type, as well as storage information and its temporary ID, if one was specified. You can also manually synchronize container dates by clicking the Dates icon in the From/To Dates field. Manual synchronization is useful when events occur that cause data to be unsynchronized, such as items being moved or deleted. When you manually synchronize container dates, all items in the container will be updated with maximum and minimum date values. Manually synchronizing dates only updates the current container, not the entire tree.

You can also add physical properties to the physical item types you create. Physical properties are additional fields associated with a specific physical item type that display when a user creates a physical item and assigns that physical item type. Physical Item properties can be viewed or modified on the Physical Properties tab.

 

Note:
When you add and additional property to a Physical Item type, you need to click the Add button before you click the Update button.

 

The Physical Properties tab allows you to enter information specific to a physical item type. For example, if you modify the physical properties for a book item type and add an author, title, and ISBN number, every time a physical item with the book type is created, the fields will automatically populate on the Create page.
