The following Physical Objects default settings can be modified.

Table 1-1: Physical Object Settings

Setting Description Default Setting
Number of Days to Borrow The number of days users can borrow Physical Items. 14
Midnight Notification of Overdue Borrowed Items Enables an e-mail notification to be sent at midnight when borrowed items are overdue. No
Number of Days Between E-mail Reminders This number specifies the increment in days that users with overdue borrowed items will recieve e-mail reminders. 5
Number of E-mail Reminders Before Escalation The number of e-mail reminders a user receives for overdue borrowed items before an e-mail is sent to the Record Manager. 2
Acknowledge Feature on Borrow Activities Enables the system to allow a user to acknowledge receipt of Physical Objects that they have borrowed. For more information, see Managing Borrowed Physical Items. No
Default Home Location for New Physical Items The location that automatically populates the Location field when users create new Physical Items. None specified
Borrowing Children Enables users to borrow children objects from parent objects or boxes that are already borrowed. Yes
Security Override Determines whether the Security Override parameter is selected by default when users borrow Physical Items. No
Free Form for Location Allows users to type any text into the Location field, instead of having to browse for a location. Yes
Verify RSI Match when Boxing
 

Note:
This setting is only available when the Records Management module is installed.

 
 Determines whether the Record Series Indentifier (RSI) must match when boxing Physical Items. Yes
Re-classification Rules
 

Note:
This setting is only available when the Records Management module is installed.

 
 Determines whether classification of items in a box should change when parent classification is changed. Yes
Synchronize Transfer Dates Determines whether the Transfer From Date and To Date should be automatically updated with the item's From Date and To Date. Yes
Delete all Contents of a Box Deletes box and all contents even if sub-items are assigned to a different box. No
Make From and To Dates Mandatory Determines whether From and To Dates of Physical Item containers are mandatory. No
Synchronize Container Dates when Creating Contents Determines whether From and To Dates are automatically updated with contents From and To Dates when content is created. No
Synchronize Box Dates Determines whether Box From and To Dates are automatically updated with contents From and To Dates. No
Restrict Assigning Items to a Box Restricts items from being placed in a box if the items do not have date range within the box's date range. No
Make Unique ID Required Field Determines whether the Unique ID field is required when users create Physical Items. No
Show Unique ID when Creating Physical Item Determines whether the Unique ID will display when a user creates a Physical Item. No
Object Name Generation Using Unique ID Determines whether to append or replace a manually entered name value with the unique ID for the object, and specifies whether the unique ID replaces the manually entered name value, or if the unique ID is placed in front or behind the manually entered name value.
The object name will appear in the language specified by the user or system. If no value is entered for the object name in the specified language, the default English value will be used.
 No
Always Send E-mail when Creating a Request Specifies whether to send an e-mail when a user creates a request for a Physical Item. No
Send E-Mail when Item is Flagged for Pick Up Determines whether an e-mail is sent to the default e-mail address when items are flagged for pick up. No
Remove Item from the Locator Determines whether the Physical Item is removed from the Locator when it is borrowed. No
Update Status and Location Determines whether the Update Status of all Sub-Items and Update Location of all Sub-Items check boxes are selected by default when a user assigns content to a box. No
Restrict Purging Circulation History Restricts the ability to purge the Circulation History of Physical Items to users with Administration rights only. This prevents the Circulation History from being purged accidentally by users who can view the history.
 

Notes:

  • If the Records Management module is installed, and the Functional Access feature is enabled, users with Functional Access will also be able to purge the Circulation History.

  • If purging of the audit logs is disabled in Content Server, the options to purge the Circulation History are not available.

 
 No

Physical Objects Action in Browse View
Choose whether multi-select actions appear for Physical Object functions in the Browse View. If selected, users can apply Physical Item metadata; request, borrow or return Physical Items; generate Labels, assign a Box or Locator, remove a Locator, and assign a Transfer to multiple objects at one time.
None selected
Reply to E-mail Address Automatically populates the Reply to e-mail address field when users create new Physical Items. The default address used is defined in the Configure Notification section of the Content Server Administration page. You can specify an alternate e-mail address for Physical Items. None specified

You can add the Borrowed By and Home Location metadata as columns, which enables the information to display beside the object when navigating in the Browse View. The columns must be configured in the Facets Volume. For more information, see Using Faceted Browsing.
