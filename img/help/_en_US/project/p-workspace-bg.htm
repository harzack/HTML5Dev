The Project Workspace is a container that stores information added by Project participants. Participants with the proper permissions can add any type of Content Server item to a Project or sub-Project. A Project Workspace is similar to any other Content Server Workspace. However, you can customize a Project Workspace's appearance by specifying which page appears upon opening the Project, adding banner images and Project News, or by defining the navigation style for the Workspace. The Project Workspace comes with a Project menu to access the Project Overview, Workspace, My Tasks, News, Notifications, Outline, Participants, and Reports specific to the Project. For more information about customizing a Project Workspace, see Customizing a Project Workspace.

The Project Overview page

The Project Overview page gives each participant a personalized view of items in a Project. By default, every Project created in Content Server uses the Project Workspace as its start page. However, the settings can be changed so that the Project Overview page appears when a participant opens a Project.

The Project Overview page is divided into sections, and each section can contain up to five items by default. The Project Coordinator can configure this page to display fewer sections or to change the number of items that appear within each section. For more information about changing the Project Overview page and making other customizations, see Customizing a Project Workspace. The following table describes the sections that appear on the Project Overview page by default.

Project Overview Section Description
Project Summary Displays a Project's current status and its start and target dates. The Status Indicator in the Project Summary section resembles a traffic light. It uses colors to provide the Project's current status:
•

green, On Target

•

yellow, Caution

•

red, Critical

•

no color, Pending

The information that appears in this section is the information specified in the Content section when the Project was created.
Sub-Projects Lists all sub-Projects, and includes their status and target dates.
Newest Polls Displays the most recently added Polls. This section shows whether each Poll is open or closed, whether you have voted, and when the Poll closes.
Late Tasks Lists all Tasks in the Project with expired due dates.
Recently Visited Items Lists the items in the Project that you have most recently opened.
Documents of Interest Lists the Documents in the Project that you and other participants have opened. These Documents are sorted by the most frequently accessed to the least frequently accessed. A Document in this section will contain a rating if a participant has recommended it. For more information, see Using Recommendations and Ratings.
My Tasks Lists all Tasks assigned to you for the Project.
Current News Displays the current headlines and News for all active Channels in a Project.
Current Milestones Lists a Project's current Milestones, which are sorted by date.
Unassigned Tasks Lists all Tasks in the Project that are unassigned. The Tasks are sorted by due date.
Most Active Items Lists the items in the Project that have been opened more frequently than other items. An item in this section will contain a rating if a participant has recommended it. For more information, see Using Recommendations and Ratings.
Top Picks Lists the items in the Project that have been accessed most frequently and have the highest ratings.

Project Report pages

A Project Workspace contains report pages that provide information specific to you or the Project. With the exception of the Project Outline report page, every report page functions the same way that your Personal Report pages function. However, while a Personal Report page contains information from anywhere within Content Server that is specific to you, a Project Report page contains information only from a Project.

Many Project Report pages are divided into tabs, which allow you to organize them according to your preferences. For more information about Report page tabs, see Managing Report Page Tabs.

The following table describes the Report pages that are available for Projects.

Report Page Name This report page... For more information, see...
My Project Tasks Lists all Tasks in the Project that are assigned to you. Working with Tasks and Task Lists
Project News Lists all Channels in the Project, provided you have the See Contents permission to the Project. Working with Channels and News
Project Notification Lists Notifications for events that occur in the Project. The Project Notifications can only be set up by the Project Coordinator or other participants with the proper permissions. Using Notifications
Project Outline Lists selected items in the Project or sub-Project in a tree-like structure. Participants can filter any of the following items that they want included in the list: Task Lists, Discussions, Channels, Documents, URLs, Compound Documents, and Folders.  
Project Participants Lists the name, log-in ID, department, and role for all participants in the Project. Managing Project Participants
Project Reports Lists all Queries and Snapshots in the Project, and the same LiveReports that appear in the LiveReports Volume. The LiveReports that appear on this page will not provide search results that are specific to the Project; these LiveReports search the entire Content Server system. Working With LiveReports and Searching Content Server
