You can use Workflow Map roles to help you assign users and groups to steps. After you add a role, you can assign it to steps in a Workflow Map so that when you assign a user or group to the role, the steps assigned to the role are sent to that user or group. For example, if you create a Workflow Map for a publications group, you can define a role called Reviewer. When you assign a user to the role, that user will be responsible for completing all the steps assigned to Reviewer.

You can add and manage roles only if you specified Map Based role implementation in the Workflow Map General properties. For more information about the Workflow Map's General properties, see Defining General Properties.
