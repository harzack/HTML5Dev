If you want to take advantage of processing performed in another Workflow Map, you can use the Sub-Workflow step in your Workflow Map. For example, if part of your Workflow Map requires editorial approval of a document, you can use the Sub-Workflow step to embed the Workflow Map that performs the editorial review in your Workflow Map. When the embedded Workflow Map is finished, processing returns to your main Workflow Map.

When Content Server executes a Sub-Workflow step, it pauses the main Workflow Map processing to initiate a Workflow Map instance based on the Sub-Workflow. When work on the Sub-Workflow instance is completed, Content Server resumes processing of the main Workflow Map. You specify how the main Workflow Map's work package and information are exchanged with the Sub-Workflow so that work completed in the embedded Workflow Map is returned to the main Workflow Map.
