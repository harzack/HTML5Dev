You define the tasks and events in your business process in Workflow Map steps. You use links between steps to control the order in which steps are processed. The default steps in the Workflow Map Designer toolbox are the following:

  • Start step

  • User step

  • Initiator step

  • Evaluate step

  • Milestone step

  • Item Handler step

  • Sub-Workflow step

  • Process step

After you add a step to the Workflow Map Designer, you can alter the name of the step to better identify each step. Step names appear inside the step, except for the Start, Evaluate, and Milestone steps, which is rendered below the shape. When you save the Workflow Map, the ID of the step is shown in the step name, as well as the step details. In both cases, the ID is added at the end of the step name in braces []. The due date, if applicable, and done date for a step now appears below the text for all steps. For more details on adding a step to the Workflow Map Designer, see Exploring the Workflow Map Designer.

The number of steps executed at one send on is restricted to 70. In case more than 70 steps are executed at one send on, all other steps are executed in background.
