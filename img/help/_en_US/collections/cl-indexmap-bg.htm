When you create a Disk Image, in addition to copying the specified Collection items to disk, Content Server also generates a Disk Image Manifest file. This Disk Image Manifest file provides information specific to the Disk Image creation, which allows you to quickly review details about the Disk Image, the Collection that the items came from, and about each individual item included in the Disk Image.

The Disk Image Manifest file is named MANIFEST.HTM, and is located at the root of each Disk Image directory. For example, if your Disk Image output location was \\server1\filecache\7, the MANIFEST.HTM file is located in the directory \\server1\filecache\7\D1\MANIFEST.

If you have OpenText Enterprise Library items and you collected the attributes, you view the property types for OpenText Enterprise Library in the MANIFEST.HTM file. In Content Server, OpenText Enterprise Library property types are implemented as category attributes.

Manifest Summary tab

The Manifest Summary tab has information about the Collection used to create the Disk Image, and some of the key settings used to create them, and consists of several sections.

The About this Collection section:

Field Name Description
Collection Name The name and hyperlink to the Collection in Content Server from which the Disk Image was created.
Collection ID The Content Server identifier of the Collection. This ID cannot be modified.
Description The description, if entered in Content Server, when the Collection was created.
Created The date the Collection was created.
Modified The date the Collection was last modified.
Collection Size The number of items in the Collection.
Name Filter The name of the filter used to select items based on their Content Server object name.
Type Filter The type of filter used to select items based on their Content Server object name.
Entire Collection The state of the Entire Collection check box on the browse page at the time of Disk Image creation:
•

Yes – the entire collection, or the entire filtered collection with respect to defined Name Filter and/or the Type Filter values is included

•

No – the entire Collection is not included

Information in the About this Disk Image section:

Field Name Description
Notes The supplementary information for the consumer of the Disk Image, such as a case number or procedural steps to follow.
Transaction ID The unique transaction ID assigned to each Make Disk Image request.
Disk Image Manifest The current Disk Image number out of the total number of Disks.
Sort Order The sort order selected when the Disk Image was created:
•

Create Date – date the object was added to Content Server

•

Data ID – internal Content Server identifier for the object

•

Created By – user ID of the person creating the object

•

Owner – user ID of the person who currently owns or manages the object

•

Name – user modifiable label for the object

•

Parent Data ID – Content Server identifier for the location in which the object is located, typically a folder

•

Parent Name – the user modifiable label for the location in which the object is located, typically a folder

Range Start The first object in this Image based on the sort order.
Range End The last object in this Image based on the sort order.
Image Created The date and time the Disk Image was created.
File System The file name and folder restrictions, based on the file system used for the exported data.
Item Count The number of Content Server objects included in the Disk Image. The count may include multiple versions of objects. The number of distinct objects is included in brackets. If more than one image is created from a Collection, a total count will be included in the final image.

Information in the Disk Image Settings section:

Field Name Description
Metadata Language The language used to create the Disk Image.
Include EDRM XML 1.2: Indicator if a description of the disk image, using the Electronic Discovery Reference Model (EDRM) standard was included.
 

Note:
The number of items described in the EDRM may be significantly less than the number of items in the Disk Image. Only versioned objects with content stored in Content Server are represented in an EDRM description. Content Server objects such as folders, Categories, and Classifications and many other object types are not files, and therefore are not included in the EDRM description.

 
Version Mode The versions of objects selected to be exported. Multiple options may be selected, and include:
•

Collected Versions – the exact versions of objects added to the Collection.

•

All Versions – every available version of an object, regardless of the Collected version.

•

Current Version – the most current versions of the objects.

•

Latest Major Version – the latest Major version of Content Server objects. If there are no Major versions, then the current Minor version is exported. For more information, see Working with Versions.

Information in the Chain of Custody Notes section:

Field Name Description
Environment
Software The product and version number information for your Content Server installation.
Licensed to The customer name of your organization as entered in your Content Server license key.
Collection Owner The Content Server user ID of the owner of the Collection being exported.
Disk Image Initiator The Content Server user ID of the person who created the Disk Image.
Collection Creation
First item Collected The date and time of the first ‘add to Collection’ operation for objects in this Image, using the format MM/DD/YYYY HH:MM.
Last item Collected The date and time of the last ‘add to Collection’ operation for objects in this Image.
Uncollectable search results The number of Search Results objects that could not be included in this Collection. This may happen if items are removed from Content Server and the collection operation occurs before the search index has been updated.
Completeness
Discovery rights enabled Indicator if eDiscovery Mode was used by the Disk Image Initiator. This mode enables the person who created the Disk Image to ‘see’ and access all objects for Search, Collection and Browse operations.
Removed from Collection The number of objects removed from the Collection before the Disk Image was created. If all objects are removed from a Collection, this value is reset to 0. All versions of an object must be removed from the Collection for an object to be considered removed. This count is based on Collection Audit entries for the entire Collection, not just this Disk Image. If Collection auditing is disabled or if the audit history was purged, this count will display Not Applicable.
Deleted Objects
The number of objects in Content Server referenced by the Collection that were deleted between the time the object was Collected, and the time the Disk Image was created. Objects must be fully purged before being counted as deleted.
This data is based on Content Server audit records for the entire Collection, not just this Disk Image. If the audit settings were not enabled, this field will display Not Applicable. For more information, see Reviewing the Collection Items Audit.
Failed Output Items The total number of Collection objects that were not exported for this Disk Image because objects could not be retrieved from Content Server and written to disk. When multiple versions fail for a single object, this count will display 1. If an object cannot be retrieved from Content Server and written to disk, this results in an object export failure. The total number of objects with failures for this Disk Image is recorded here. Multiple versions failing for a single object counts as 1, and are identified with the error message: Unable to save the file. If multiple images are created, the final image will also contain a total count of failures in all Images.

Information in the Collection Categories section:

Field Name Description
The Content Server Categories and Attributes (custom metadata) attached to the Collection, if they exist.

Disk Image Contents tab

The Disk Image Contents tab contains detailed information about each object included in the Disk Image. As a convenience, checkboxes under the tab header will hide or show various sections with details for all objects.

Field Name Description
General
Name The Content Server name of the object. Often, it is the name of the file or the subject line of an email message, although any item may have an arbitrary name.
Image Directory The file name and path to the object as saved in the Disk Image.
Created The time the object was added to Content Server.
Modified The time the object was last modified in Content Server, such as a new version being added. In some cases, changes to metadata about the object may also change the Modified time.
Size The approximate size of the object as saved on disk. Rounded to KB or MB as appropriate.
Type The Content Server “subtype”. There are hundreds of possible subtypes in Content Server, such as documents, folders, search forms, task lists, or email volumes. Usually, files stored in Content Server such as text, spreadsheets, presentations, faxes or PDF files are of the “document” subtype. The numeric value of the subtype as used by Content Server is included in brackets.
Created By The Content Server user ID of the person who added the object to Content Server. This may be different from the user who currently owns the object, or who last modified the object. For automated processes, such as journaling email, this field does not indicate that a user explicitly added the object to Content Server.
Content Server Location The Content Server location of the object at the time the Disk Image was created.
Collected Versions The versions of the object in the Collection from which the Disk Image was created. Based on version export options, the versions in the Disk Image are not always identical to the collected versions.
Description An optional field in Content Server where users can enter descriptions of objects. It is common for this field to be empty.
Categories
The Content Server Categories and Attributes (custom metadata) for each object contained in the Collection which exists in this particular disk image. Click Show to display the metadata.
Versions
The exported versions of objects in the Collection, if they exist. Objects marked in red were not exported, so they have an error message, a File Name link to the object in the disk image, and an optional Description for each version. Click Show to display the versions. For more information, see Collecting Items from the Search Results Page.
Additional Files
The supplemental files included in Content Server as part of an object’s definition, if they exist. Click Show to display the files.
Audit Records
The audit records detail the material events that created or modified the Collection. Audit records are only included in the first Disk Image of a set. Click Show to display the records.

Search Queries tab

The Search Queries tab contains information about the search query used to create the Collection being exported to this Disk Image. This tab is only available when the Collect All Search Results button is used after a search query to create a Collection. For details, see Collecting Items from the Search Results Page. The collected items are also available on the Collection Items Audit tab of the Collection’s Properties page. For details, see To View the Collection Items Audit.

 

Note:
Detailed user and search query information is disabled by default due to potential privacy issues that may exist in some countries.

 

Field Name Description
Date The date and time this Disk Image was created, using the format MM/DD/YYYY HH:MM.
User The Content Server user ID of the person who created the Disk Image.
Search Manager The Content Server ID of the Search Manager used to create the Disk Image.
Items Collected The number of items collected and exported in this Disk Image.
 

Note:
The number of items described in the EDRM may be significantly less than the number of items in the Disk Image. Only proper files stored in Content Server are represented in an EDRM description. Content Server objects such as folders, messages, wiki entries and many other object types are not files, and therefore are not included in the EDRM description.

 
eDiscovery Mode The indicator if the Disk Image Initiator used eDiscovery Mode, which enables them to ‘see’ and access all objects for Search, Collection and Browse operations.
Search Query The entire WHERE clause used in the search query to create the Collection exported to this Disk Image, for example, [region "OTCurrentVersion"] != "FALSE" AND "report".

Help tab

A generic Help file is included, by default, on the Help tab of the Manifest file when you create a Disk Image. Your Content Server administrator may edit or replace this Help file, if needed. This file is intended to help users of the Manifest file who are not familiar with Content Server, or with specified types of documents or information that has been assembled and copied to a Collection of items on disk.
