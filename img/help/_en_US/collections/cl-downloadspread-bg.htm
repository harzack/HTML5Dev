Once a Collection is created, and items have been saved to that Collection, you can download that Collection as a spreadsheet using the Download as Spreadsheet option found in the More Actions list.

You can use the multi-select check boxes to select specific individual items within the Collection to download, or you can download the entire Collection. If you select the entire Collection, you can specify the type of item from the Collection to download using the object type filter.

 

Note:
The Download as Spreadsheet feature depends on the privileges assigned by your administrator and may not be available.

 

When downloading a large Collection, a progress bar shows the percentage completed and time remaining to complete the operation.

 

Note:
Only items that meet the filter criteria will be included in the spreadsheet.

Only the metadata for items in the Collection is placed in the spreadsheet. The actual items are not extracted from Content Server.

 

Each time a Collection is downloaded as a spreadsheet, information is recorded in the Collection Items Audit tab for the Collection. You can therefore view all past events relating to the Collection, including the items that have been included in the current and past spreadsheet downloads. Administrators can see more information about the Collection Items Audit in the Reviewing the Collection Items Audit.

Downloaded Spreadsheet

The downloaded spreadsheet of a Collection is written in a Comma-Separated Variable format, and is saved with a .csv suffix. The naming convention used to save the spreadsheet will be Collection export <your_collection_name> <YYYY>-<MM>-<DD> <HHMM>.csv, where:

  • <your_collection_name> is the name you typed for your Collection.

  • <YYYY>-<MM>-<DD> is the year in four digits, the month in two digits and the day in two digits on which this downloaded spreadsheet of a Collection was created.

  • <HHMM> is the hour and minute, is displayed using the 24-hour clock, that this downloaded spreadsheet of a Collection was created. For example, 7:15 AM is displayed as 0715, while 7:15 PM is displayed as 1915.

An example of a filename of a downloaded spreadsheet of a Collection is: Collection export My_Collection 2012-01-14 1605.csv

Only the metadata for items in the Collection is placed in the spreadsheet. The actual items are not extracted from Content Server.

The spreadsheet displays the information in a number of columns in two different formats.

First Format: Header

The first format, the header, displays at the top of the spreadsheet as follows:

The title appears at the top of the spreadsheet: Content Server Exported Collection Report

Collection Name: <collection_name> Collection ID: <xxx>
Export Date: MM/DD/YYYY Export Time: HH:MM AM/PM
User ID:
<user_id>
The userid of the person who created the Collection.
 User Name:
<user_name>
The username of the person who created the Collection.
Items In Collection:
<n>
The number of items in the Collection.
 Items Selected:
<n>
The number of items selected.
Name Filter:
<filter_name>
The name of the filter used to filter Collection items.
 Type Filter:
<filter_type>
The type of the filter used to filter Collection items.
Section:
<n>
The number of the section.
 Starting Item:
<n>
The number of the starting item in the Collection.

Second Format: Body

In the second format found in the spreadsheet, one row is created for each item exported to the spreadsheet. Each row contains all the relevant metadata imported from the Collection.

The second format, the body of the spreadsheet, displays the following information:

  • Object ID

    The ID of the object in Content Server.

  • Name

    The name of the object in Content Server.

  • Type ID

    The type ID of the object in Content Server.

  • Type

    The type of the object in Content Server.

  • Created

    The date the object was created in Content Server, in the format MM/DD/YYYY.

  • Modified

    The last date on which the object was modified in Content Server, in the format MM/DD/YYYY.

  • Created By

    The user ID of user who created the object in Content Server.

  • Modified By

    The user ID of user who last modified the object in Content Server.

  • Owner

    The user ID of the owner of the object in Content Server.

  • Description

    The description of the object in Content Server, if one was entered for the object.

  • Size

    The size of the object in Content Server.

  • Full Path

    The path of the object's location in Content Server.

  • Version

    The number of version of the object in Content Server.

  • Current

    One of “YES” or “NO” to indicate if this is the current version of the object in Content Server.

  • File Name

    The file name of the object in Content Server.

  • File Size (in bytes)

    The size of the object, in bytes, in Content Server.

  • Version Created

    The date on which the version of the object was created in Content Server, in the format MM/DD/YYYY.

  • Version Created By

    The user id of the person who created the version of the object in Content Server.

  • Version Comment

    If this is a version, and if a version comment was created for that version, the version comment is listed here.

The Spreadsheet Footer Row

The spreadsheet footer row has the following column:

  • Items Exported

In the footer, the value for the “Items Exported” heading will match the number of item rows in the body of the spreadsheet.
