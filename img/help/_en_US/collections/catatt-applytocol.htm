You can apply categories to one or more items in a Collection. From a Collection page, from the More Actions list, select Apply Categories. When applying a Category to individual items, the Apply Categories page appears with two sections; a Supported Objects section that lists selected items to which you can apply Categories, and an Unsupported Objects section that lists selected items to which you cannot apply Categories.

The following information appears on the Apply Categories page:

  • Type

  • Name

  • Location

 

Note:
In the Unsupported Objects section, an explanation is available for each item, in the Reason column, explaining why you cannot apply Categories to that item.

 

In addition to the above information, the following options are available:

  • The Categories field, where you select the categories to be applied.

  • The Add Version check box that allows you to create new versions of versionable items when Categories are applied.

  • The Clear Existing Categories check box that allows you to clear any existing Categories from the items.

When you apply Categories to all items in the Collection, the Supported Objects and Unsupported Objects areas on the Apply Categories page, contain a count of item types to which you can or cannot apply categories. In addition to the options listed above, you can also send email notification to users, groups, or third party recipients notifying them that one or more categories have been applied to the items in the Collection. For example, if the Collection is used as a central location for a team project, you can use email notification to notify other members of the team when categories have been applied.

If you apply Categories to a Collection that already has Categories applied, these Categories are listed as default values on the Edit Categories page. If newer versions of currently applied Categories are available, the Upgrade button appears, and you can apply the latest version of the Category to the items. If you choose not to apply the latest version of the Category, any items that already have the latest version are not affected. If you apply a different Category to an item that already has a category applied, the new Category is added.

 

Note:
If for any reason an attempt to apply a Category to selected items fails, and if you have enabled email notification, the specified recipients receive notification of the error.
