To queue a Collection for indexing:

 

Caution
Indexing a Collection should be used with caution because this is a potentially performance-expensive operation which may require hours or days to complete.

 

  1. To access the Collection you want to re-index, do one of the following:

    • From any workspace or folder which contains a Collection, click the name of the Collection you want to re-index.

    • On the Global Menu bar, from the Personal menu, select Collections. On the Manage Collections page, click the name of the Collection you want to re-index.

  2. On the <collection_name> page, using the multi-select check boxes, select the check box next to each item in the Collection you want re-indexed. You can also select the Select/Deselect All check box to select all items in the Collection.

  3. From the More Actions... menu, select Queue for Indexing.

  4. In the Queue Collection for Indexing Confirmation dialog box, the Description, Collection Name and Summary fields are informational only, and are not editable.

  5. In the Notification area:

    1. In the Recipient field, type the email address of the individual who will receive this Notification.

    2. In the Message Subject field, type a message subject.

    3. Optional In the 3rd Party Notifications area, select one of the two available option buttons:

      1. Select No Additional Notification if you do not want to add additional email recipients.

      2. Select Group to add additional email recipients. Next, do any, or all, of the following:

        • In the User field, click Find a user to select a Content Server user to whom you want to send an email notification.

        • In the Group field, click Find a group to select a Content Server group to whom you want to send an email notification.

        • In the Other Email field, enter an email address to send an email notification to an email address not available in Content Server.

  6. Click Submit.
