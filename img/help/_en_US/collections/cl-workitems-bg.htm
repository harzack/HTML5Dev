From any workspace, you can add one or more items to a Collection. You can also collect items from a Search Results page. Once you have collected an item, you can then access the item, or group of items, from the central location of the Collection.

If you have the appropriate permissions, you can also collect individual versions of an item, including major and minor versions, from a Search Results page. This can be useful if you are working on a team project where you must have access to specific versions of an item from within the Collection. For more information about versions, see Working with Versions.

On new and upgraded installations of Content Server Update 10.5, or later, the Version column indicates the specific version of the object that is marked for inclusion in the Collection. The current version is always marked, whether from Browse or Search.

You can use the Browse Content Server... button to select items on the Items to Collect page, or use the pop-up menus that contain Recently Used Collections, your Favorites Collections, and your Personal 10 most recently used Collections.

If you have the appropriate permissions, you can also collect OpenText Enterprise Library items. Collections are stored in the Content Server Volume.

Collecting Items from a Workspace

You can add items from any workspace to a Collection. You can perform a bulk add operation by selecting and adding multiple items to a Collection at one time using the multi-select check boxes in the Detail View browse view of any workspace.

You can include OpenText Enterprise Library items in bulk add operations; however, you cannot perform some of the basic Content Server functions for the original items from within the Collection because they are read-only. Examples of basic Content Server functions include Copy, Move and Delete. For more information about managing items from within a Collection, refer to Managing Collected Items.

 

Important:
You must use the Detail View browse view to add items to a Collection from a workspace. For more information about browse views, see Understanding Workspaces and Containers.

 
 

Note:
You must have the appropriate permissions to perform a bulk add operation.

 

Collecting Items from the Search Results Page

After you perform a search, from the Search Results page, you can:

  • collect an individual item

  • collect multiple items using a bulk add operation

  • collect all search results

 

Notes:

  • You must have permissions to perform a bulk add operation or to collect all search results. You can perform a bulk add operation by selecting and adding multiple items to a Collection at one time using the multi-select check boxes in the Detail View browse view.

  • You can also include OpenText Enterprise Library items in bulk add operations; however, you cannot perform some of the basic Content Server functions for the original items from within the Collection because they are read-only. Examples of basic Content Server functions include Copy, Move and Delete. For more information about managing items from within a Collection, refer to Managing Collected Items.

 

If you perform a search on the Enterprise [All Versions] slice, and multiple versions of an item are returned, you can select specific versions of the item from the search results to collect. For example, if you perform a search and Versions 2, 3 and 6 are returned, you can choose to select and collect only Versions 2 and 3 from the list. If you choose to collect all search results, all versions included in the search result are collected.

If you collect an item that doesn't support versioning, such as a folder, 0 (zero) appears in the Version column on the Collect Search Results page, and the item itself is collected. The message “Version 0 does not exist; object collected OK” appears in the Status column on the Collecting Search Results Items to page. For a list of versionable items, see Understanding Versions.

 

Note:
In some circumstances, a recently deleted version of an item may appear in the search results. If you attempt to collect that recently deleted version, the message “Version <n> does not exist” appears, where <n> represents the version number; however, the item is collected.

 

When you collect all search results using the Collect All Search Results button, you can send an email notification to a user, a group, or a third-party recipient notifying them that the Collection has been created. For example, if you collect a group of documents for a team project, you can notify other team members about the Collection, its contents, and location.

 

Note:
Email notification must have been enabled by the administrator in the Notifications Administration area of the Content Server administration pages before any notification you enable for Collections will be sent.

If your notifications from Collections are not sent, contact your Administrator to enable Notifications.

 

The email notification contains the following information:

Initiated by The login ID of the user who created the Collection.
Collection Name The name of the Collection.
Started At The day, month, year and time the collection action started.
Completed At The day, month, year and time the collection action completed.
Total Versions Collected The total number of versions of all objects collected.
Total Objects Collected The total number of objects collected. Objects are defined as items with distinct Data IDs.
Total Items Collected The total number of items collected. This includes total number of versions and total number of objects collected.
Updated Collection Size The total number of items now found in the Collection.
The Collection can be accessed here A URL which links to the Collection in Content Server.

 

Note:
The Total Versions Collected and Total Objects Collected may not match. For example, if you are adding items to a new, empty Collection, if one of the items is not versionable, the item is not counted in the Total Versions Collected but is counted in the Total Objects Collected. In addition, if you collect multiple versions of an item, each version is counted in Total Versions Collected, but only the item itself is counted in the Total Objects Collected.

 

Recipients receive the email message once all items have been collected.
