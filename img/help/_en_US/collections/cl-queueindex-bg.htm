Note:
The Queue for Index menu option is only available if your administrator has given you the appropriate usage privileges.

 
 

  1. OpenText recommends that you do not use this feature if you have read-only search partitions configured.

  2. You should only queue a Collection for indexing to correct index errors. This option should not be used in normal operation.

 

There are situations in which a sub-set of content in Content Server needs to be re-indexed. For example:

  • OTContentStatus identified some problems, which are fixed, and the objects need re-indexing.

  • Indexing settings are changed and need to be applied.

Beginning with Content Server 10, Service Pack 2, administrators can designate users to select specific content in Content Server in a Collection, and re-index that content.

 

Note:
Indexing content in Content Server writes information about that content to an index used by Content Server search. By re-indexing specific content, users can now update the index used by the search function.

 
 

Caution
Indexing a Collection should be used with caution because this is a potentially performance-expensive operation which may require hours or days to complete. Before re-indexing a Collection, users should think carefully about:

  • when to re-index, for example choosing a time period with low activity on the system; and

  • consider how much content is in the Collection they have selected to re-index. The larger the content of the Collection, the more time the re-indexing process will take.
