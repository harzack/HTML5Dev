OpenText Content Server Disk Image

This help is intended to provide context for consumers of a Content Server Disk Image who are not familiar with Content Server.  This is not the official OpenText Content Server product documentation.

The information here is generic. Depending on installation or configuration settings, there may be features described in this help text not included in a particular Disk Image.

Disk Image Concepts

Introduction

Within OpenText Content Server, a feature known as Collections allows links to managed objects to be gathered.  This can be done from searching or browsing within Content Server.  All or a subset of the Collected objects can then be exported using the “Make Disk Image” function.

The term “Disk Image” is used because the process has the ability to package the exported objects in subsets based on their ability to fit on disks of various sizes.  An exported Collection may span multiple Disk Images.

Manifest File Overview

The Manifest file contains a summary of the Collection and the Make Disk Image settings, plus detailed information about each exported object within a Disk Image.  The Manifest is in the MANIFEST folder of the Disk Image, and is formatted as HTML so it can be viewed with a web browser.  If multiple Disk Images were created, each image will contain a separate Manifest.

Version Notes

Some objects in Content Server do not have versions, such as folders.  For non-versioned objects, the text “Not Applicable” will appear instead of version numbers.

It is possible for a Collected objected to be deleted from Content Server after being added to a Collection.  In this case, the object is also removed from the Collection.  If appropriate settings are enabled in Content Server, the number of deleted objects will be noted in the Manifest.

In some cases, “None” may appear in the list of Collected versions.  This occurs when the Collected version of the object has been deleted from Content Server.  If other versions of the object still exist, the current version of the object is exported instead.

Audit Information

Content Server has the ability to audit operations on objects.  The initiator of a Disk Image can optionally include the audit information in the Manifest.  The presence of audit data is based upon the auditing interests that are active within Content Server over the life of the exported objects.  Audit information is associated with objects in Content Server, not with versions of objects.

Permissions

Content Server imposes permission restrictions on access to objects.  The permissions of the user creating a Disk Image may limit the objects which may be exported.  If the user initiating a Disk Image has fewer permissions than the person(s) creating the Collection, then some of the Collected items may not be available for export.

Content Server can be configured such that deleted objects can be restored.   Depending on Content Server configuration, these deleted objects may not be searchable, or may not be accessible for export.

To ensure that all Collected items are included in a Disk Image, the user initiating the Disk Image must meet at least one of the following requirements:

·         be the same user that created the Collection;

·         use eDiscovery Mode within Content Server;

·         have System Administration privileges;

·         log in as the Content Server “Admin” user.

Exported Files

The root folder of a Disk Image contains the Manifest folder and a folder for the exported objects.  The structure of the folders in the Disk Image mimics the folder structure of Content Server.  To avoid potential naming conflicts, files and folders are prefaced with their Content Server Object ID and version.

The manifest file contains relative path UNC links to the exported files in the disk image.

The Make Disk Image process is designed to adjust file names to meet the restrictions of target file systems such as character sets or name lengths.  This means that file and folder names in the Disk Image may not be identical to those in Content Server.  To meet restrictions on the number of objects allowed in folders, extra folders may be added to the Disk Image that do not exist in Content Server.

Non-File Objects

Most objects managed by Content Server are based on files, such as email messages, faxes, PDF files or office documents.  These files are included in the Disk Image.

There are other types of information in Content Server that are not based on files.  For example, a Wiki entry or Folder may be nothing but entries in a database table.  Wherever possible, the disk image will contain an XML representation of a non-file object.  This representation is based on a Content Server feature known as “XML Export”, and may not be available for every object type.

 

Header

The top section of the Manifest provides basic information about the current Disk Image, and contains tabs for displaying the components of the Manifest.

Image #

A Collection may be too large for a single Disk Image.  The total number of images is only known for the last image.  For example, if there are two images, the first will be identified as “1 of …”, the second will be “2 of 2”.  This value is also in the Manifest Summary.

Transaction ID

To facilitate tracking the relationships between Disk Images, each “Make Disk Image” request has a unique transaction ID.  This value is also in the Manifest Summary.

 

Manifest Summary Tab

 

About this Collection

This tab displays information about the Collection used to create the Disk Image, and some of the key settings used in creating the Collection and the Disk Image.  This information is useful for understanding the contents of the Disk Image and establishing Chain of Custody in litigation scenarios.  The Summary is also available as a separate file named SUMMARY.HTM in the MANIFEST folder of the Disk Image.

Description

The optional description of the Collection within Content Server

Collection

The name of the Collection in Content Server from which the Disk Image was generated.  This is an editable field in Content Server.  Collections may be temporary objects, used to gather items to be included in a Disk Image, and reused or deleted upon completion of the task.

Collection ID

The non-modifiable Content Server ID of the Collection.

Collection Size

The number of items in the Collection from which the Disk Image was created.  The Collection Size is not necessarily the same as the number of items in the Disk Image.

Type Filter

If the selected items in a Collection were filtered based upon their Content Server type when making the Disk Image, the type filter is displayed here.

Name Filter

If the selected items in a Collection were filtered based upon their object name in the Content Server database when making the Disk Image, the name filter is displayed here.

About this Disk Image

Notes

The Notes field may be added during the Make Disk Image operation.  It has no corresponding field in Content Server.  The notes allow the initiator to communicate supplementary information to the consumer of the Disk Image, such as a case number or procedural steps to follow.

Sort Order

When creating a Disk Image, there are a number of sorting options available.  The selected sort order is recorded here.  Options for sorting include:

·         Create Date: when the object was added to Content Server

·         Data ID: internal Content Server identifier for the object

·         Created By: the Content Server user ID of the person creating the object

·         Owner: the Content Server user ID of the person that currently owns/manages the object

·         Name: the user modifiable label for the object

·         Parent Data ID: the Content Server identifier for the location in which the object is located, typically a folder.

·         Parent Name: the user modifiable label for the location in which the object is located, typically a folder.

Range

The first and last objects in this image based on the sort order.

Image Created

Date and time the Make Disk Image process was performed.

File System

The exported data may have file name and folder restrictions based on the selected file system.  Several variations of ISO Disk Images are supported.

Item Count

The number of Content Server objects that were included in this Disk Image.  The count may include multiple versions of objects.  The number of distinct objects is included in brackets.  If more than 1 images are created from a Collection, a total count will be included in the final image.

Disk Image Settings

This section contains additional settings used to create the Disk Image.

Sample Size

When a sample size is requested, randomly selected items from the Collection are written to the Disk Image.   The total number of objects in all Disk Images may be less than the sample size if the user creating the image does not have permission to enough objects.

Metadata Language

Content Server may be configured to have a multi-language user interface.  In this case, some fields such as the name or description of an object may exist in more than one language, and the user can set a preference for the value to include in the Disk Image.  If a value does not exist for the preferred language, then a language value that does exist will be selected.

Version Mode

When a Disk Image is created, the initiator selects which versions of objects should be exported.  Multiple options may be selected.  The options include:

·         Collected Versions – export the exact versions of objects that were added to the Collection.

·         All Versions – export every available version of an object, regardless of the Collected version.

·         Current Version – regardless of which version was Collected, export the most current version of the object.

·         Latest Major Version – Content Server has two versioning schemes.  If the advanced versioning scheme is used, Minor versions are often considered unfinished work in progress.  This version option selects only the latest Major version of an object.  If there are no Major versions, then the current Minor version is exported.

Chain of Custody Notes

This information is useful for establishing Chain of Custody in litigation scenarios, and only included in the Disk Image when requested.

System

Product and version number information about Content Server.

Licensed to

The customer name as entered in the license key for Content Server.

Image Requestor

The Content Server user ID for the person that initiated the creation of the Disk Image.

Collection Owner

The Content Server ID for the owner of the Collection being exported.

Deleted Objects

The deleted count identifies how many objects in Content Server referenced by the Collection were deleted between the time the object was Collected, and the time the Disk Image was created.  Objects must be fully purged before being counted as deleted.  This data is derived from Content Server audit records for the entire Collection, not just this Disk Image.  If appropriate auditing interests were not enabled, this field will display “Not Applicable”.

Removed from Collection

The removed object count identifies how many objects were removed from the Collection before the Disk Image was created.  If all objects are removed from a Collection, this value is reset to 0.  All versions of an object must be removed from the Collection for an object to be considered removed.  This count is based on Collection Audit entries for the entire Collection, not just this Disk Image.  If Collection auditing is disabled or the audit history was purged, this count will display “Not Applicable”.

Failed Output Items

If an object cannot be retrieved from Content Server and written to disk, this results in an object export failure.  The total number of objects with failures for this Disk Image is recorded here.  Multiple versions failing for a single object counts as 1. In the details for items that fail to output, these are identified with the error message: “Unable to save the file”.  If multiple images are created, the final image will also contain a total count of failures in all images.

Collection Categories

If Content Server Categories and Attributes (custom metadata) are attached to the collection, they are listed here.

Collection Audit Records

This data is optional.  If present, the audit records detail the material events that created or modified the Collection.  Audit records are only included in the first Disk Image of a set.  Clicking on the header bar will display the records.

 

Disk Image Contents Tab

This section contains detailed information about each object included in the Disk Image.  As a convenience, checkboxes under the Tab will hide or show various details for all objects.

General

Name

This is the Content Server name of the object.  Often, it is the name of the file or the subject line of an email message, although any item may have an arbitrary name.

Image Directory

The file name and path to the object as saved in the disk image.

Created

This is the time the object was added to Content Server.

Modified

This is the time the object was last modified in Content Server, such as a new version being added.  In some cases, changes to metadata about the object may also change the Modified time.

Size

The approximate size of the object as saved on disk.  Rounded to KB or MB as appropriate.

Type

This is the Content Server “subtype”.  There are hundreds of possible subtypes in Content Server, such as documents, folders, search forms, task lists, or email volumes.  Usually, files stored in Content Server such as text, spreadsheets, presentations, faxes or PDF files are of the “document” subtype.  The numeric value of the subtype as used by Content Server is included in brackets.

Created By

The Content Server user ID of the person that added the object to Content Server.  This may be different from the user that currently owns the object, or who last modified the object.  For automated processes, such as journaling email, the Created By field does not indicate that a user explicitly added the object to Content Server.

Content Server Folder

Content Server objects are usually stored in a folder hierarchy.  This is the location of the object at the time the Disk Image was created.

Collected Versions

The versions of the object in the Collection from which the Disk Image was created.  Based on version export options, the versions in the Disk Image are not always identical to the collected versions.

Description

An optional field in Content Server where users can enter descriptions of objects.  It is common for this field to be empty.

Categories

Categories and Attributes are a Content Server feature that allows the definition and attachment of custom metadata to objects.  If Categories and Attributes are defined for the object and requested for export, the key / value pairs are listed here.

Versions

If the object is versioned, each exported version is listed here.  If an object fails to export, it will be marked in red with an error message below it.  The File Name includes a link to the object in the disk image.  It is possible for a separate Description to be attached to a version, although typically this is empty.

Additional Files

Some objects in Content Server may have supplemental files as part of their definition.  In this situation, the supplemental files are listed here.

Audit Records

If requested, the audit records for the object are listed here.  The types of records depend on the auditing interests enabled in Content Server.
