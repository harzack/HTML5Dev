When XML actions are configured to process messages with stylesheets, all outgoing messages are wrapped in sendxml elements before being sent to the stylesheet for transformation. The stylesheet must produce a sendxml element if it wants the transformed message to be sent. This allows a single stylesheet to distinguish between the incoming and outgoing cases and gives stylesheets some ability to control message processing as well as message format.

XML Representation

<sendxml filename="…" contenttype="…">…</sendxml>

The sendxml wrapper is a sendxml element that contains the message to be sent.

XML Namespaces (if used)

<sendxml xmlns="http://www.opentext.com/llx/">

HTML Representation

The sendxml wrapper is not used in HTML exchanges.

Contents of Message

The following items are included in a sendxml wrapper:

  • Filename

  • Contenttype

  • Any message

A sendxml wrapper must contain a single child element for the message to be sent. The filename and contenttype attributes are optional and can be used to modify message processing. If the message is being sent to a directory using the HTTP PUT protocol, the filename attribute contains the filename to be used for storing the message in that directory. The contenttype attribute contains the MIME type of the message to be sent. The default value is text/xml, but it can be changed by a stylesheet. If the stylesheet changes the content type to text/html, the message is written using the XSLT output rules for HTML.
