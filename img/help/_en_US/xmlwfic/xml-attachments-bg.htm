Workflow attachments information is provided when available.

XML Representation

<attachments folderid="…"/>

The XML format for the attachments work package includes only the node ID of the folder in which the attachments are stored. To actually transfer attachments requires a separate request into Content Server using other APIs.

 

Note:
To include attachment information that includes actual attached documents in the task’s work package, use an HTML form data message. For more information, see the HTML Representation, below.

 

XML Namespaces (if used)

<attachments xmlns="http://www.opentext.com/llx/workflow/">

HTML Representation

attachments_folderid=…

attachments_documents_name_1=…

attachments_documents_content_1=…

…

In HTML form data, the attachments information can include the actual attached documents. To have a send action include the documents or an accept action accept attached documents, you must customize the attachments field using the action's format customization tab. Attached documents are never included in OK messages returned in response to a request.

Transferred attachments include only the document nodes in the attachments folder. Each node produces a numbered group of two fields:

  • A name field, which gives the name of the document as it appears in Content Server.

  • A content field, which gives the document content. The content field conforms to the format used by Web browsers to upload files.
