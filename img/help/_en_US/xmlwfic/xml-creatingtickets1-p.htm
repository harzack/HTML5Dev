To add a remote initiation ticket:

  1. Select Remote Initiation on the Map menu.

  2. On the Remote Initiation tab, click Add Ticket.

  3. Edit the ticket name in the Ticket Name field.

  4. Select a date in the Expiry Date lists.

     

    Note:
    Any messages sent to the access URL on or after the expiration date will be rejected.

     

  5. In the Initiation Action area, select one of the following:

    • Accept an XML Initiation Message, which specifies that the initiation messages must be in XML format. For example, you can use this format to let the Workflow Map be initiated by a Workflow Interchange step in another Content Server system that is configured to send an XML initiation message when ready.

    • Accept an HTML Initiation Message, which specifies that the initiation messages must be in HTML form data format. For example, you can use this format to accept messages from legacy systems, HTML forms, or documents attached to the Workflow by other systems.

  6. In the Trusted IPs area, click one of the following radio buttons:

    • All IP Addresses, which lets you accept initiation messages from any IP address.

    • Only from the following IP Addresses, which lets you restrict the IP addresses allowed to send initiation messages. Specify the IP addresses by clicking the Add Value icon and typing an IP address in the available field.

  7. In the Initiator area, click the Choose User or Group icon and then find and select a user.

    The user is assigned to the Initiator role when the Workflow is initiated with this ticket.

  8. Click Accept.

  9. Click Add to Workflow Definition and save the Workflow.

For more information related to this procedure, see Creating Remote Initiation Tickets.
