Configurable properties for actions are distributed onto several different pages. On the General page, you set parameters that determine how the message is named, sent, or received. You can configure any of the following properties:

  • XML Accept Action Properties, which can be configured in remote initiation tickets and Workflow Interchange tasks. On the General page for these types of actions, you determine how errors are reported to the sender. Other tabs allow you to change the format and content of the messages themselves.

  • XML and HTML Initiation Action Properties, which, when a Workflow Interchange step becomes ready, sends a message to a specific URL by configuring the XML or HTML Initiation Action properties for that event. In the General properties page for these types of actions, you set parameters that control where and how the message is sent.

  • XML Response to Remote Initiation Action Properties, which, when a Workflow Interchange step becomes ready, sends a response message back to the source task that initiated the Workflow if the Workflow was initiated remotely and an XML response remote initiation action has been configured to occur. On the General page for this type of action, you can set the action's name and determine whether or not the source task should be forwarded.
