A task edit message can be sent by a "send an XML response to a remote initiator" action and accepted by an "accept an XML task edit message" action or an "accept an HTML task edit message" action.

XML Representation

<exec><edittask status="…" query= "…">...</edittask></exec>

In the default XML format, an initiation message consists of an edittask element inside an exec element. The edittask element holds the contents of the message.

If an accept action receives a task edit message with the query="yes" attribute, it returns updated values for all valid fields.

XML Namespaces (if used)

<exec xmlns="http://www.opentext.com/llx/">

<edittask xmlns="http://www.opentext.com/llx/Workflow/">

When an action is configured to use and require XML namespaces, different namespaces are used for the exec element and the edittask elements.

HTML Representation

In HTML form data, a task edit message is represented by its contents.

Contents of Message

The following items are included in a task edit message:

  • Task status

  • Work packages

A task edit message contains a subset of the information that an initiation message can contain. It carries only work package data and status information about the task being edited. For more information about these items, see Task Information and Work Packages.
