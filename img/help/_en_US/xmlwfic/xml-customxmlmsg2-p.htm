To format XML message content:

  1. Double-click the Workflow Interchange step and click the Actions tab.

  2. Click the Edit icon for Accept an XML Initiation Message.

  3. Click the Format tab.

  4. In the Format area, select Custom Format in the list.

  5. Do any of the following:

    • Type the name and path of an XSLT stylesheet in any of the fields to transform the message with XSLT stylesheets.

       

      Notes:

      • For XML send actions, you specify separate stylesheets for outgoing XML messages and for incoming acknowledgements from the target system. For XML accept actions, you specify stylesheets for incoming XML messages and outgoing acknowledgements that the action returns. The specified path is relative to the support directory.

      • The XSLT stylesheet must be placed in the Content Server support directory by the system administrator.

       

    • Type the name and value of a parameter in the Name and Value fields and then click the Add Value icon to set the parameters if you specify stylesheets that accept XSL parameters.

       

      Important:
      XSL parameter values are XSL expressions, not strings. If you want to pass a string value to the stylesheet, it must be enclosed in quotes. Consult the XSLT specification at http://www.w3.org for a description of XSL expressions.

       

  6. Select the Use and require XML Namespaces check box to ensure that the elements in all XML messages produced by the action are assigned their default XML namespaces.

     

    Notes:

    • The default namespaces are required on all XML messages that the action accepts, in accordance with the XML Namespaces Recommendation at http://www.w3.org. For more information about the proper namespaces for each element, see Message Fields and Formats.

    • Namespaces can make XML processing somewhat more complicated, but they are useful in advanced XML systems to manage the integration of XML components from various sources. If you are using XML Workflow Interchange to connect Content Server Workflows, you should clear the Use and require XML namespaces check box unless the other Content Server system is set up to require namespaces.

     

  7. Click Apply.

For more information related to this procedure, see Customizing XML Messages.
