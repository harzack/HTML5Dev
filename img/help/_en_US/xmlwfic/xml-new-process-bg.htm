New process information is returned in reply to an initiation message when that message starts a process on the receiving system.

New process information includes two variable fields:

  • An opaque handle, which identifies the new process within the target system. If present, this is guaranteed never to be reused.

  • A URL, which is available if the status of the new process can be checked with a Web interface.

XML Representation

<newproc handle="…">

<target url="…" type="browser"/>

</newproc>

In XML, the new process information consists of a newproc element that contains one or more target elements. Target elements are distinguished by their type attributes.

Currently, only browser targets are supported, so all newproc elements have exactly one target child element. However, systems that process these XML messages should anticipate additional target types in the future.
