XML Workflow Interchange performs all of its messaging interactions using the HTTP protocol. Each interaction has two parts:

  • A client connects to the target server over the Internet and sends an HTTP request to a specific URL on that server. The request indicates an operation that the client wants the target server to perform and provides any additional information the server may require to complete the request.

  • The target server replies by sending an HTTP response to the client over the same connection. The response indicates whether or not the requested operation was performed successfully and provides the operational results to the client.

The entire exchange is synchronous-some client processes may wait for a reply from the server. When XML Workflow Interchange sends an HTTP request, it signals an error and closes the connection to the server if it does not receive a reply within one minute.
