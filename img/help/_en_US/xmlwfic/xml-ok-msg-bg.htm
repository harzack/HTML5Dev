An OK message can be returned in response to an HTTP request and indicates that the request was successful.

In the default XML format, an OK message is an OK element. The contents of the element can be used to return updated fields or new information.

XML Namespaces (if used)

HTML Representation

An HTML OK message can be an HTML document or a multipart/form-data document that carries updated fields or returned information.

Contents of Message

The following items are included in an OK message:

  • Task status

  • New process

  • Work packages

The contents of an OK message must be in the same format as the contents of the initiation or task edit message in the corresponding request.

An OK message generated by an XML Workflow Interchange action will have contents under two circumstances:

  • If there is a successful response to a remote initiation request and the response includes new process information.

  • If there is a successful response to an XML request that sets the query="yes" attribute and the response includes updated work package information.

When responding to an initiation message generated by an XML Workflow Interchange, a custom process, such as a CGI or servlet, can also include task status information to explicitly send on the source task and work package information to update fields in the source task.
