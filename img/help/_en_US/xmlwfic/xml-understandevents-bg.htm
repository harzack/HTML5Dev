The messages sent and received by XML Workflow Interchange allow a source process on a source machine to start a target process on a target machine. The target process then returns results back to the source process. Either process can be a Content Server Workflow.

Because the target process can involve manual interaction by users, it is not expected to complete immediately, and therefore does not need an open communication channel while the target process runs. When Workflows are connected in this way, the following two events cause the exchange of messages:

  • An initiation occurs when the source process wants to start the target process. For example, if the source process is a Content Server Workflow and an XML Interchange task becomes ready, the source task sends an Initiation Message to the target system, which then starts the target process. The target system immediately returns either an Initiation Reply containing information about the new process it started or an error indicating the reason why the process could not start. If the target is a remote initiation URL in a Content Server Workflow Map, the initiation message initiates a new Workflow using that map. A successful reply includes a URL that can be used to check the status of the Workflow that has started.

  • A task edit occurs when the newly created target process reaches a milestone or completes and is ready to return information or status to the source process. As a result, the target process sends a Task Edit Message to the source process. This message can update information in the source task as well as send the task on. If the target process is a Content Server Workflow, the task edit message is sent to the URL provided in the initiation message from the source task.

If both the source and target processes are Content Server Workflows, both these events are triggered when a Workflow Interchange step becomes ready. When a Workflow Interchange step in the source Workflow is ready, it sends an Initiation Message to the target system. When the Workflow Interchange step at the end of the target Workflow is ready, it responds with a Task Edit Message. After the Initiation Message is sent, the Workflow Interchange step in the source listens for a Task Edit Message.

The Content Server Workflow Designer lets you determine what happens when each Workflow Interchange step in your Workflow becomes ready: active steps receive Task Edit Messages, and Workflow Maps receive Initiation Messages. The Content Server Workflow Designer allows for significant deviations from the basic interchange model and lets other types of systems assume the source or target role.
