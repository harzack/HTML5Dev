Every HTTP request or response has a content type that determines the basic format of the data it carries. Different types of XML Workflow Interchange actions support different content types for exchanges.

HTML Form Data Exchanges

HTML sending actions send requests using the multipart/form-data content type, as defined in Internet Request for Comments 1867. The request contains data fields for an initiation message or task edit message as determined by the action's message type and customization parameters.

There are three possible replies to this type of request:

  • HTTP error status code (between 400 and 599) with a status message to indicate an error.

  • Successful status code (between 200 and 299) with no data or with HTML data to indicate success.

  • Successful status code and multipart/form-data content type used to indicate success and update fields in the initiating task. This type of response is an HTML OK message and supports the same fields in the same format as the request.

HTML accepting actions are designed to process requests from HTML sending actions and will accept POST requests using the multipart/form-data content type. The action's message type and customization parameters determine which fields are processed. Any fields that are not processed are ignored.

An HTML accepting action generates one of the following responses:

  • HTTP error status with a status message to indicate an error.

  • OK status (code 200) and HTML data to indicate success.

  • OK status (code 200) and multipart/form-data to indicate success and return new information to the caller. This sort of response is returned from successfully accepted initiation messages to provide the caller with information about the Workflow that is initiated.

HTML accepting actions also accept GET requests that contain no data. In response, they return multipart/form-data containing the current values for most of the fields filled in with a POST. This is the kind of request a Web browser performs if you click the URL link. The browser downloads the form data, which you can use as a convenient template for a future upload.

XML Exchanges

XML sending actions send requests using the text/xml content type, indicating that the data is an XML document conforming to the XML 1.0 Recommendation (http://www.w3.org). The request is an XML initiation message or task edit message formatted according to any customized parameters of the action.

There are three possible replies to this type of request:

  • HTTP error status code (between 400 and 599) with a status message to indicate an error.

  • Successful status code (between 200 and 299) with no data or with non-XML data to indicate success.

  • Successful status code and text/xml content type with an XML error message or an XML OK message. An error message indicates a processing failure and can include a status string describing the error. An OK message indicates success and updates fields in the calling task. The action accepts the same XML format for responses that it generates for the request, as determined by its message type and customization parameters.

XML accepting actions are designed to process requests from XML sending actions. They accept POST requests using the text/xml content type carrying an XML message in the format determined by the action's message type and customization parameters. An XML accept action returns either an HTTP error condition, an XML error message to indicate failure, or an XML OK message to indicate success.
