An initiation message can be sent by a "send XML initiation message" or a "send HTML initiation message" action and accepted by an "accept XML initiation message" or "accept HTML initiation message" action for remote Workflow initiation.

XML Representation

<exec><initiate status="…" query="…">…</initiate></exec>

In the default XML format, an initiation message consists of an "initiate element inside an exec" element. The initiate element holds the contents of the message.

If an accept action receives an initiation message with the query="yes" attribute, then it will return updated values for all valid fields.

XML Namespaces (if used)

<exec xmlns="http://www.opentext.com/llx/">

<initiate xmlns="http://www.opentext.com/llx/Workflow/">

When the action is configured to use and require XML namespaces, different namespaces are used for the exec and initiate elements.

HTML Representation

In HTML form data, an initiation message is represented by its contents.

Contents of Message

The following items are included in an initiation message:

  • Task status

  • Task title

  • Source task

  • Instructions

  • Work packages

For more information about these items, see Task Information and Work Packages.
