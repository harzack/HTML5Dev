Stylesheets used for processing messages in XML Workflow Interchange must conform to version 1.0 of the XSLT specification (http://www.w3.org).

When used for translating an incoming message or acknowledgement, the stylesheet receives the message from the external system as input. As output, the stylesheet produces one of the following:

  • A message in the default XML format for the action, which is then processed by XML Workflow Interchange.

  • An error element that describes a problem with the incoming message. If the action is part of a Workflow Interchange step, the error is logged and displayed to the assignee for the step.

  • An OK element that indicates success. No processing of the incoming message is performed, but the action is considered successful. A stylesheet can return OK to ignore an incoming initiation or task edit message.

When a stylesheet is used for translating an outgoing message or acknowledgement, the Workflow Interchange system will generate the outgoing message in the default XML format, encapsulate it in a sendxml element, and pass it to the stylesheet as input. Because outgoing messages are distinguished by the sendxml element, the same stylesheet can often be used for both incoming and outgoing messages.

As output, the stylesheet produces one of the following:

  • A sendxml element containing the new outgoing message. This message is sent to the target system.

  • A reply in the default XML format for the action. No message is sent to the target system, but the stylesheet output is processed as if it were received in acknowledgement to a successful send.

  • An error element that describes a problem with the outgoing message. The error is logged and displayed to the assignee for the step, and no message is sent.

  • An OK element that indicates success. No outgoing message is sent.

For more information on the default message formats and the sendxml, error, and OK elements, see Message Fields and Formats.
