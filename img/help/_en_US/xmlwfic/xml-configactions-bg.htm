You can configure actions for a Workflow Interchange step, which allows you to determine the actions that will take place when a Workflow Interchange step becomes ready or when an active task receives a task edit request.

To configure Workflow Interchange step actions:

  1. On the Workflow Map, double-click a Workflow Interchange step.

  2. Click the Actions tab.

  3. In the Step Ready area, select one of the following actions in the When this step becomes ready list:

    • Send an XML Initiation Message, which sends an initiation message in XML format to a specific URL when the step becomes ready; for example, to initiate a Workflow in another Content Server system.

    • Send an HTML Initiation Message, which sends an initiation message in HTML form data format to a specific URL when the step becomes ready; for example, to send Workflow data to a custom servlet.

    • Send an XML Response to a Remote Initiator, which sends a task edit message to the callback URL when the step becomes ready, if the Workflow is initiated using the remote initiation mechanism. The remote initiator provides a callback URL in the initiation message. The parameters for this action include updated attribute information and a request to send on the source task. Use this action at the end of Workflows that support remote initiation. For more information about remote initiation, see Enabling the Remote Initiation Work Package.

     

    Note:
    If Send an XML Initiation Message or Send an HTML Initiation Message option was selected, you need to click the Edit icon to define the target URL of the action. For information about configuring actions, see Configuring XML and HTML Initiation Action Properties.

     

  4. Repeat step 3 for each action you want to add.

  5. In the Step Completion area, select one of the following options in the When the above actions have been performed list:

    • Wait for a request

    • Finish task automatically

  6. If you select the Wait for a request option, select one of the following options from the When a request is received list:

    • Reject It, which rejects any requests received. Any attempt to send a request to the task will return an HTTP error.

    • Accept an XML Task Edit Message, which lets the task process XML task edit messages; for example, to accept XML responses from Workflows you initiate remotely in other Content Server systems.

    • Accept an HTML Task Edit Message, which lets the task process HTML form data task edit messages; for example, to accept messages from legacy systems or HTML forms, or if you want other systems to attach documents to the Workflow.

  7. Click Add to Workflow Definition and save the Workflow.

For more information related to this procedure, see Configuring Workflow Interchange Step Actions.
