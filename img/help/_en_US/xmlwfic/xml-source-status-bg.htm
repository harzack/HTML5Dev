Source task information is carried in an initiation message that informs the target system about the client process.

Source task information includes two variable fields:

  • An opaque handle, which identifies the task within the source system. If present, this field is guaranteed never to be reused for another source task. A target system might use this value to detect duplicate messages.

  • An HTTP URL, which can have messages sent to it. This field is only available if the source task can be configured to accept task edit messages over HTTP.

XML Representation

<sourcetask handle="…">

<target url="…" type="HTTP"/>

</sourcetask>

In XML, the source task information consists of a sourcetask element that contains one or more target elements. Target elements are distinguished by their type attributes.

Only HTTP targets are supported. All sourcetask elements have exactly one target child element, but systems that process these XML messages should anticipate additional target types in the future.
