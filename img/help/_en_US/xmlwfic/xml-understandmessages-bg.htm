All messages sent or received by XML Workflow Interchange consist of either XML or HTML form data (MIME type multipart/form-data) transferred using the HTTP protocol. When a message is sent by a Workflow Interchange task, it can include data from Forms, Comments, Attachments, and Attributes work packages associated with the task. The recipient of a message returns a status indication and, optionally, new values for some of or all the data.

 

Note:
Comments from the source map cannot be seen in a remotely initiated Workflow.

 

The type of data that can be transferred in a message and reply depends on the type of interchange event occurring.

During an Initiation Event, the following information can be exchanged:

  • Source task information (message only), including the URL used to send task edit messages back to the source task.

  • New process information (reply only), including the URL used to check the status of the process that was created.

  • Task status information (reply only). The reply can set the task status to Done to complete the source task without sending a task edit message.

  • Comments (message and reply).

  • Workflow Attributes (message and reply).

  • Workflow Forms (message and reply).

  • Workflow Attachments (message only).

During a Task Edit Event, the following information can be exchanged:

  • Task status information (message only). The message can set the task status to Done to complete it.

  • Comments (message and reply).

  • Workflow Attributes (message and reply).

  • Workflow Forms (message and reply).

  • Workflow Attachments (message only).

XML and HTML form data messages and replies carry approximately the same information, and each format can be customized in the Workflow Designer. Choosing a format depends on the needs of your application. When choosing a format, consider the following:

  • HTML form data is easiest to process in technologies traditionally used to interact with HTML forms, such as servlets, CGI applications, and .asp pages.

  • Documents attached to a Workflow can only be sent in HTML format messages. The XML format includes only the attachment folder ID.

  • The Content Server Workflow Designer lets you customize simple HTML form data message formats. For example, you can change the names of the form fields used to transfer each data item.

  • XML message formats are more likely to be compatible with current and future versions of XML Workflow Interchange.

  • XML messages can contain information about the range of values that can be entered in attributes and forms.

  • When completing attributes and forms, XML messages let you control which fields are inherited from the previous versions.

  • XML messages can delete the existing values of an attribute.

OpenText recommends using the default XML messages for connecting Content Server Workflows and HTML form data to send initiation messages to custom Java Servlets and CGI applications.
