If the forms work package is included in the map, then messages can include data from all forms that are available to the step. Each action involved in the exchange will only process forms that are made available in its step. The following rules apply:

  • An action that accepts a remote initiation message only updates forms that are made available in the start step of the Workflow.

  • An action that accepts a task edit message only updates forms that are made available in the action's task.

  • An action that sends a message will only send data for forms that are made available in the action's task.

XML Representation

<forms>

<form name="my form">

<attribute name="my att">

<value>…</value>

…

</attribute>

</form>

</forms>

In XML, the forms package is a forms element that contains one form child element for each available form. Each form element gives the name of the form and contains attribute elements for each of the form's attributes. Each attribute element gives the name of the attribute and has one child element for each defined attribute value. For attributes of type set, each value contains more attribute child elements.

When the forms package is updated, unmentioned forms retain their previous values, and unmentioned attributes within a form retain their previous values. To delete the values of an attribute, you must provide an attribute element with no value child elements.

XML Namespaces (if used)

<forms xmlns="http://www.opentext.com/llx/workflow/">

HTML Representation

forms_my_form_my_att_1=…

forms_my_form_my_att_2=…

…

In HTML form data, form attributes have field names that are generated from their attribute names and the name of the forms they appear in. The HTML base names for all attributes appear in the custom format configuration tab for any action in the Workflow step.

Because Content Server supports multiple values for attributes, each value of each attribute is numbered. Values of sub-attributes inside set attributes will have two indexes. For example, forms_my_form_setatt_textatt_2_3 names the third value for the textatt attribute in the second value for the setatt attribute.

When form attributes are updated with an HTML form data message, you can overwrite individual values with the appropriately numbered field, but cannot delete values that already exist.
