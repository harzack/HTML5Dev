Each remote initiation ticket that you create is associated with a URL to which remote initiation messages can be sent. Tickets also have parameters that control the initiation process.

Remote initiation can be disabled. However, disabling remote initiation saves the configuration information of all the remote initiation tickets in case you want to re-enable the remote initiation.

 

Note:
The successful receipt of an initiation message initiates the Workflow Map. Information about the newly initiated Workflow, including a URL that can be used to check its status, is returned in the HTTP response.
