A Workflow Interchange task that is active in a Workflow represents a process that is running on a remote system that is required to return a response. However, every Workflow Interchange step is assigned to a local Content Server user who oversees the execution of the step and manages difficulties that might arise. If the target system fails, for example, the required response might never arrive. Without the intervention of the assignee, a source Workflow is delayed while waiting for a task edit message to complete the task.

Active interchange tasks appear in the assignee's list of assignments. While the task is active, the assignee can check the progress of the various messages that are sent, retry messages, see the status of any task edit messages received, or complete the task before receiving a response.

The assignee can also see work package data and edit it manually, like the assignee of a User step.
