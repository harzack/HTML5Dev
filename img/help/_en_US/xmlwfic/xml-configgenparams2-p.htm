To configure XML and HTML initiation action properties:

  1. Double-click the Workflow Interchange step and click the Actions tab.

  2. In the Step Ready area, select the Perform Action list and select Send an XML Initiation Message or Send an HTML Initiation Message.

  3. Click an action's Edit icon .

  4. Type the name of the action in the Name field.

     

    Note:
    The name appears in the list of actions configured for the step but does not appear in any messages. You can use it to describe the purpose of a send action or to distinguish between multiple actions of the same type that are configured on the same step.

     

  5. Type the absolute URL to which the message should be sent in the Target URL field. For example, http://www.opentext.com.

  6. If the target system is a Web server that supports HTTP basic authentication or a Content Server instance that supports Content Server cookie-based authentication, type a username and password in the Authentication area. These credentials are supplied to the target Web server if required.

     

    Note:
    If credentials are required, OpenText recommends creating a user for this specific purpose.Â 

     

  7. In the HTTP Method area, click one of the following options to choose the method used to send the message to the target system:

    • POST, which is used by a Web browser to submit forms and should be selected when you are sending data to a server process.

    • PUT, which is used to save documents into a writable Web server directory.

       

      Note:
      When PUT is selected from the list in the HTTP Method area, the last segment of the target URL (the part after the last forward slash) is the name of the file created in the destination directory. If the target URL ends with a forward slash, a default filename is generated automatically. Default filenames generated in this way are different for every Workflow Interchange task in every initiated Workflow.

       

  8. Optional If you know that the target system requires custom HTTP headers to properly process the message, you can add those headers in the Additional Headers area. Type the name and value of the header in the Name and Value fields, and then click the Add value icon .

  9. Click Apply.

For more information related to this procedure, see Configuring General Action Parameters.
