Workflow attributes are captured in the attributes package.

XML Representation

<attributes>

<attribute name="my att">

<value>…</value>

…

</attribute>

…

In XML, the attributes package is an attributes element with one attribute child element for each attribute. Each attribute element gives the name of the attribute and has one child element for each defined attribute value.

When the attributes package is updated, unmentioned attributes retain their previous values. To delete the values of an attribute, you must provide an attribute element with no value child elements.

XML Namespaces (if used)

<attributes xmlns="http://www.opentext.com/llx/workflow/">

HTML Representation

attributes_my_att_1=…

attributes_my_att_2=…

…

In HTML form data, attributes have field names that are generated from their attribute names. Since Content Server supports multiple values for attributes, each value of each attribute is numbered. You can see the HTML base names for all attributes in the custom format configuration tab for any action in the Workflow step.

When attributes are updated with an HTML form data message, you can overwrite individual values with the appropriately numbered field, but you cannot delete values that already exist.
