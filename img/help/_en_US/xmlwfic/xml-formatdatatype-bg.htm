Most elementary field values in both HTML and XML messages are simple, arbitrary strings, but there are a few types of values with special formats.

Boolean Values

In XML, the Boolean values are true and false. In HTML form data, they default to 1 and 0, but can be set to true and false in the format customization tab for HTML actions.

Dates

In XML, date values include both date and time and are written in the following form: YYYY/MM/DD:hh:mm:ss. For example, 2000/01/02:03:04:05 indicates January 2 of the year 2000 at 3:04 and 5 seconds AM.

In HTML form data, you can elect to use this format by selecting Single Field in the Format of Date Values list on the Format page for editing HTML send and accept messages. By default, however, dates are written in a six-field numeric format. The names of the fields used are derived from the single field name by appending the name of each date part.

Instead of date_value_1=2000/01/02:03:04:05, the six field format would be:

date_value_1_year=2000

date_value_1_month=1

date_value_1_day=2

date_value_1_hour=3

date_value_1_minute=4

date_value_1_second=5

 

Note:
When a complete field name is generated from the HTML base name, date parts are appended after index numbers.
