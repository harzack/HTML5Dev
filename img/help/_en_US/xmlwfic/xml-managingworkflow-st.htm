When a Workflow Interchange task is active, it appears in the assignments list belonging to the task's assignee.

You can open an assignment list from any Content Server page. Once you open an assignment list, you can edit any task in the list. You can also view and edit data, and you can send a task immediately without sending or receiving any further messages.

The general work package lets you manage the interchange behavior of a task. In addition to the common task information, the following sections are provided on this page:

  • External Access URL, which displays the URL that external systems should use to access the task. If the task is configured to accept task edit messages, they are sent here. This URL is included by default in any initiation messages sent by the task. The URL is displayed as a link so that it can be easily copied and pasted in other applications. If the task is configured to accept task edit messages, you can click this link to retrieve the current values of all the fields that a task edit message might set and to retrieve the format that the action accepts.

  • Accept Status, which displays information about the last five requests sent to the external access URL including the time the request was received and either an error message or an indication that the request was processed successfully.

  • Send Status, which displays the status of the send actions that were configured to occur when the step became ready. You can select the Select check boxes for any of the actions and click Retry Selected to perform the actions again.

Other work packages are available for manual editing by the assignee and use the same procedures as a standard user task.
