If the DoD Supplemental Markings setting is enabled, you can edit the supplemental markings on supported object types.

If the DoD Supplemental Markings and Security Clearance Levels setting is enabled, you can do the following to the security clearance level on supported object types:

  • Upgrade, which allows you to set the security clearance level to a higher level than what was originally set.

  • Declassify, which allows you to set the security clearance level to the lowest possible level available. If the Administrator has changed the declassify time frame, and all objects have not been updated, you, the Security Officer, will see a Recalculate Declassify on Date link. When you click the link, the Recalculate Declassify on Date page appears and you can click the Recalculate button to update all objects and view summary statistics. Summary statistics only appear for the records for which you have the See and See Contents permission.

  • Downgrade, which allows you to set the security clearance level to a lower level than what was originally set, and specify when the downgrade should happen.

When you change the security levels or supplemental markings of a container, the changes can be applied to any of the following:

  • The item only

  • The item and classified items

  • The item and sub-items

  • The item, sub-items, and classified items

When you update sub-items and classified items, the action is controlled by rules. If an item does not meeting one of the following conditions, it will not be updated:

  • The user lacks the proper security clearance and cannot view the item

  • The user is not a member of the Security Officer group and does not have the Edit permissions for the item

  • The current security level on the sub-item or classified item is higher than the old security level on the original item. This rule only applies when the item is downgraded.

  • The sub-type of the item is not a supported sub-type.

Security Clearance with DoD settings enabled supports the following object types:

  • Documents (144)

  • Emails (557)

  • Physical Objects (411)

  • Physical Object Copies (419)

  • RM Folders (426)

  • RM Classifications (551)
