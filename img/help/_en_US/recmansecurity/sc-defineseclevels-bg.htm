Security clearance levels are implemented by assigning a security clearance level to objects and users. These levels, which are hierarchical, provide access to individual objects. The levels are also used to filter searches and provide users with lists of objects that only they can see based on their assigned security clearance level.

Security clearance levels are a single attribute attached to each individual object. Typically, this type of filtering is performed before a check of the object's Access Control List (ACL).

Users assigned a high security clearance level can see objects with lower security clearance levels, but users assigned a low security clearance level cannot see objects with higher security clearance levels.

Table 1-1: Security Clearance Levels

Name Level
Top Secret 90
Secret 50
Confidential - A 40
Confidential - B 30
Confidential - C 20

Using the examples from Table 1-1, Top Secret can see level 90 and all levels below level 90; Secret can see level 50 and all levels below level 50, etc.

 

Note:
If you change the Security Level on a container and choose Apply to this Item and Sub-items, the new Security Level is applied to the container and sub-containers, as well as the items in those containers. However, if the Security Clearance Level you assign to the container is lower than the Security Clearance Level currently assigned to a sub-container or item, the new Security Clearance Level will not be assigned to those sub-containers and items. The Security Clearance Level can be changed on these sub-containers individually.

Example 1-1: Security Clearance Level is Higher on a Sub-container or Item than the Parent Container

Assume Folder 1 is assigned a Security Clearance Level of 60, Folder 2 (child container in Folder 1) is assigned a Security Clearance Level of 30, and Folder 3 (child container in Folder 1) is assigned a Security Clearance Level of 90.

If you change the Security Clearance Level of Folder 1 from 60 to 70, select the Upgrade and Downgrade Inheritance option, and choose Apply to this Item and Sub-items, Folder 1 and Folder 2 will be assigned a Security Clearance Level of 70, but Folder 3 will retain its current higher Security Clearance Level of 90.
