To assign an object security clearance when the DoD Supplemental Markings and Security Clearance setting is enabled:

  1. Click the Functions icon for an object, click Properties, and then choose Security Clearance.

  2. On the Security Clearance tab, modify the settings in the Supplemental Marking section.

  3. In the Security Clearance Levels section, click a security clearance level in the Current Security Clearance Level drop-down list.

  4. Type information in at least one of the following fields:

    • Derived From, which lets you specify information about how the security clearance was derived

    • Classified By, which lets you specify the name or personal identifier, and position of the original classification authority

  5. Type information citing the applicable security categories in the Reasons for Security field.

     

    Note:
    The Reason for Security field is only mandatory if you type information in the Classified By field.

     

  6. Type the name of the agency and the office of origin in the Classifying Agency field.

     

    Note:
    A classifying agency is any executive agency as defined in 5 U.S.C 105 and any other entity within the executive branch that comes into possession of classified information.

     

  7. In the Declassify On section, click one of the following radio buttons:

    • Date, which lets you specify the date on which to declassify the record

       

      Note:
      If the Declassify on Time Frame value has been updated for an object with Security Clearance set and the Recalculate Declassify on Date has not been run, an error message appears next to the date. You can manually change the date, but the message will appear until the recalculation is run.

       

    • Event, which lets you specify the event that needs to occur before the record can be declassified

    • Date and Event, which lets you specify the date on which to declassify the record and the event that needs to occur before the record can be declassified

    • Exemption Categories, which lets you specify a date on which to declassify the record and that exceeds the date set in the Security Clearance Administration pages, and then specify a category that explains why the date is exceeded.

  8. Click the Update button.

 

Note:
If the document is restricted, the information in the Declassify On section is disregarded.

Buttons with the names of existing Security Clearance guides appear on the Security Clearance tab. If you click the button for an existing guide, the Guide Title, Guide Date, and Originating Organization Guide metadata are displayed in the Derived From field. The Supplemental Markings for that guide, if you have permission to see them, are also displayed in the object's Supplemental Markings field.
