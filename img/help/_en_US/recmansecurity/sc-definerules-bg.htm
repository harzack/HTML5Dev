The Security Clearance tab contains two sub-tabs: Set Security Clearance and Defined Rules. The Set Security Clearance sub-tab allows you to assign Supplemental Markings and Security Clearance levels to managed items and RM Classifications. For more information, see Assigning Security Clearance and Supplemental Markings to Objects.

The Defined Rules sub-tab allows you to assign existing Supplemental Markings and Security Clearance levels to managed items and RM Classifications on a specific date. You can assign Defined Rules to individual items, containers, and sub-items in a container.

 

Note:
If you make changes on the Set Security Clearance or Defined Rules tab, you must save your changes before you access the other tab. If you click the link for the other tab without saving your changes, a warning message appears.

 

The following actions are available on the Defined Rules page: Assign Supplemental Marking, Remove Supplemental Marking, and Update Security Clearance Level. The page is refreshed with additional fields, based on the action you choose to perform. When you select an action, you must also select the date you want the action to occur, and if you are assigning the rule to a container, you must choose whether you want the rule assigned to sub-items in the container. Defined Rules must be unique; if you attempt to add a rule that already exists, an error message is displayed.

When an item is assigned Security Clearance using Defined Rules, or a Defined Rule is added, changed, or removed, the action is captured in the item’s audit trail. You can also search for items based on the action in the Security Clearance Attributes section on the Defined Rules sub-tab (Action to Perform, Supplemental Marking, Security Clearance Level, Date to Perform, or Reason for Rule) using the standard search functionality in Content Server.

 

Note:
The Defined Rules sub-tab is not displayed if DoD setting is enabled.

You must have the Edit permission or be a member of the Security Clearance Officers group to modify the Define Rules sub-tab.

 

When you add new items to a container, they will inherit the Defined Rules of the parent object if the Inherit From

  • Apply changes to current item only, which applies the action to the container only.

  • Apply changes to current item and sub-items, which applies the action to the container and all items within the container.

When the items are updated, the Update Results message provides you with the number of sub-items that were updated.

 

Note:
When you add items to a container, the newly created items will inherit the Defined Rules that are assigned to the container, as long as the Inherit From option is enabled on the Supported Object Type administration settings. This setting is enabled by default, but can be changed by the administrator.

When an item is assigned Security Clearance using Defined Rules, the action is captured in the item’s audit trail. You can also search for items based on the action in the Security Clearance Attributes section on the Defined Rules sub-tab (Action to Perform, Supplemental Marking, Security Clearance Level, Date to Perform, or Reason for Rule) using the standard search functionality in Content Server.
