You can enable eLink for Content Server items such as discussions, containers, and Workflow Maps that you have authored to interact with these items using an email program. You can enable or disable eLink functionality for these items.

As a discussion owner or administrator, you can enable eLink for a discussion so that users with Read permissions who subscribe to the discussion can participate using an email program. You can also enable eLink for a project, folder, or compound document so that users can add items to the container using an email program.

Choosing Content to Store

When you enable eLink for an item, you must specify the content that you want stored in the container when an emailed document is received. You can specify one of the following options:

  • Only the files attached to the email message are added as documents; the email message, including message body, headers, and subject line are ignored.

  • The entire email message is added as a document, including the message body, headers, subject line, and any attachments.
