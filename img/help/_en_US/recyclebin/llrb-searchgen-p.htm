Note:
Only System Administrators or users with Administrator rights may create a new search slice.

 

To Create the New Recycle Bin Slice:

  1. Go to the Content Server Search page Tools : Search.

  2. If Scope is not visible, it can be left off for now.

  3. Change the fields in the Full Text area as follows:

    • Set Look For to Complex Query.

    • Set Modifier to <None>.

    • Set Within to All.

    • Set Text Field to OTLocation:xxxx, where xxxx is the Recycle Bin volume ID.

  4. Click the Save Your Search button.

  5. Name the search “Recycle Bin” (without the quotes) and click the Browse Content Server button.

  6. Select the Slice Folder, which can be found in Content Server : Content Server System (you can select Content Server from the pull-down menu at the top left-hand side of the page).

  7. Click the Add button or click Continue should the Category screen appear and then click the Add button again.

  8. Once the “Recycle Bin slice” is created, it must be correctly defined:

    • Select the Functions menu from the Recycle Bin slice and select Properties : Specific.

    • Be certain that the values specified in step 3 are present:

      • Look For is Complex Query.

      • Modifier is <None>.

      • Within is All.

      • Text Field is OTLocation: xxxx where xxxx is the Recycle Bin volume ID.

    • Click Scope in the left-hand menu if this is not shown on your screen. Be certain that Scope is set to Enterprise Search Manager.

    • Click the Update button.

To Exclude the Recycle Bin from the Existing Enterprise Slice:

 

Note:
To avoid having items in the Recycle Bin appear when searching normal slices, the Enterprise slice and the Enterprise [All Versions] slice need to be modified.

 

  1. Select the Functions menu for the Enterprise slice and select Properties : Specific. This should provide you with the information about the Enterprise slice.

  2. Press the plus sign beside the Within All option once.

  3. Change the fields as follows without removing any Full Text options already available:

    • Set Look For to Complex Query.

    • Set Modifier to <None>.

    • Set Within to All.

    • Set Text Field tonot "<OTCurrentVersion>FALSE".

    • Set Operator Drop-down to And.

    • Set Look For to Complex Query.

    • Set Modifier to <None>.

    • Set Within to All.

    • Set Text Field to (NOT OTLocation : xxxx) where xxxx is the Recycle Bin volume ID.

  4. Click Scope in the left-hand menu if this is not shown on your screen. Be certain that Scope is set to Enterprise Search Manager.

  5. Click the Update button.

  6. Do the same for the Enterprise [All Versions] slice. Select Properties : Specific from the Functions menu. By default, you do not need to press the+ sign, as the Full Text search line should still be empty. Be certain that the scope is set to Enterprise Search Manager and then click the Update button.

The steps outlined above should ensure that your Search performance is not adversely affected by Content Server Recycle Bin. If this is not the case (if, for example, your Recycle Bin contains large amounts of data), it may be advisable to establish a separate extractor process for the Recycle Bin. Contact your OpenText Account Manager or Sales Representative for details.

 

Note:
When an object is recycled and restored and then a search is performed with the scope set to Enterprise or Enterprise [All Versions], no results may be returned. This is because the Admin Process needs time to run before the object's location is updated. Best practice in this situation is to wait five (5) minutes (more time may be required depending upon the number of items that the indexer processes), then repeat the search.
