Content Server Recycle Bin addresses the problems associated with users accidentally deleting documents from Content Server. Content Server 10.0.0 contains a Document Undelete module, but deleted documents are only accessible to the Administrator. The Recycle Bin application solves this problem by introducing a Recycle Bin to which supported objects (not just documents) are moved upon deletion and from which they can be retrieved by regular users.

The Content Server Recycle Bin Package replaces the standard document delete operation. The new functionality moves documents that are “deleted” in Content Server to a Recycle Bin Volume. This volume is structured such that all documents deleted on a particular day are found in one folder (i.e., a folder structure is created, year:month:day).

Documents in the Recycle Bin can have their versions, permissions, and attributes maintained. Objects can be restored through the Content Server interface by manually moving the document from the Recycle Bin back to its original location or to another location. The Administrator should implement a policy of regularly clearing out the Recycle Bin. A typical policy is to permanently delete documents that have reached a pre-defined expiration date.
