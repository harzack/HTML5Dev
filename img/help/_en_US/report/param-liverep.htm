When you add a LiveReport, you must specify parameters, and you can specify up to five input types. Parameters determine what information a LiveReport returns, and work in conjunction with the SQL statement. The administrator specifies which parameter types are available when you add a LiveReport. The following table describes the most common parameter types.

Parameter Type Returns...
Filter <item> type Records of the specified item type, or database subtype.
Filter permissions LiveReport only returns results for those items that the user has been assigned the See permission.
Filter <volume> Only records from the specified volume.
Object Type One of the available item types.
Status Code The status of an item, such as pending or cancelled for a Task.
<Time> The specified date/time, relative to when the LiveReport is opened.
UserID A Content Server user ID number.
User Input<#> The value of the input parameters defined in the Inputs section.

Input Type Requirements – Input types require the user who opens the report to supply a value, item type, or other input type at a prompt that you specify. For each input type that you add, you must define an input parameter and include a reference to the parameter in your SQL statement. The following list describes the available input types for LiveReports.

Input Type Prompts users to...
Container Browse for and select an item that can contain other items, such as a Folder, Channel, or Discussion.
Date Supply a date in a provided field.
NodeID Supply the ID number of the item.
Number Supply an integer in a provided field.
Object Select an item type from the provided list. Object types that appear in this list update dynamically, depending on available object types.
Project Select a Project.
InsertString Allows any string to be passed as an input to the LiveReport and inserted directly into the SQL query without any conversion or modification, such as putting quotes around the string. For more information, see Insert String.
String Type a character string, which can contain special characters, such as dashes, in a provided field.
SQLList
Allows a list of values to be specified in the format: <value1>, <value2>,<value3>
The values can be any valid SQL type. For example: 'text value1', 'text value2', 'text value3'
When an SQLList input parameter is resolved, it is enclosed in parenthesis so it can be used in an SQL “IN” clause. For example: ... WHERE Name IN %1 ...
Could resolve to: WHERE Name IN ('text value1', 'text value2', 'text value3')
User Select a user.
