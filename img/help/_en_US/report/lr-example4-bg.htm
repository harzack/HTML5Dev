Using Templates within Templates: Besides specifying templates for inclusion in the main SQL field, it is also possible to use templates within other templates. The syntax is the same, for example “~2”; however, a template can only use a template with a lower number. For example, template ~2 can include a reference to template ~1 but not to itself or template ~3, or any number above 2.

Using the Template View Icon: Each template includes a special icon that can be used to View Template Behavior. Clicking this icon opens a window that contains a series of statements about the template in question. These statements attempt to clarify, for each template, how it will behave based on the selected condition and the source that has been specified. The View Template Behavior icon is found at the end of each template row.

Example 1-8: Viewing behavior for the condition: Not all inputs set to flag

The conditions Not all inputs set to flag and No Inputs set to flag both include a table of conditions which is designed to help understand how User Input parameters in the template will affect the inclusion or exclusion of the template. In the example below, the table shows that if all User Input parameters are set to an empty string, the template is excluded but if any User Input parameter is not set to an empty string then the template will be included.

In the Template Setup, in the Templates area, define ~1:

  • Do not select the Auto-Where box.

  • In the SQL Source field, type: “%1,%2,%3”

  • Select the Auto-Comma box.

  • From the Include IF list, select “No inputs set to flag”.

  • In the Options field, ensure the Flag Value field is empty.

If you click the View Template Behavior icon found at the end of the ~1 template row, a page containing the following information will appear:

Explanation Window – The combination of fields in this row are interpreted as follows:

  • The parameters: %1,%2,%3 were found in the SQL source field for this template.

  • The Condition Type is set to “No inputs set to flag”. This means that: These parameters will be tested to see if the flag: (An empty string) has been passed as a value.

  • If none of the parameters in the SQL source field are set to An empty string then this template will be inserted anywhere in the SQL where the ~1 symbol is found.

  • Conversely, if ANY of the parameters in the source field (%1,%2,%3) are set to the flag value (An empty string) then template ~1 will not be included in the SQL source.

This table indicates the expected results for different settings of parameters:

Parm 1 Parm 2 etc. Result
An empty string An empty string EXCLUDE
(any other value) An empty string EXCLUDE
(any other value) (any other value) INCLUDE

Click Close to close this explanation window.
