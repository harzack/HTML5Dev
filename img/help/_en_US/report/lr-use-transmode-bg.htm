In standard LiveReports, and by default with LiveReport Extensions, all SQL queries run within a database transaction which locks the tables affected by the query. Where a query is altering the database it is essential that it runs within a transaction.

 

Note:
OpenText does not recommend that you use LiveReports to make changes to the Content Server Database.

 

It is most often the case that LiveReport queries are only reporting on data, and not modifying it. For queries which are only reading data, running the query within a transaction is less desirable as it locks the tables involved. (It is beyond the scope of this document to describe the concept of Transactions.)

 

Note:
For the reasons already explained, certain functions such as Allow Database Modification are not available once Use Transaction mode is cleared.
