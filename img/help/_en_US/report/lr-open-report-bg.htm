Obtaining information using LiveReports is different from using Content Server Search. LiveReports query the Content Server database, rather than search the Content Server repository. For example, the Items Reserved By LiveReport provides a list of the items in the database that are reserved by a specific user. When you open it, the LiveReport automatically prompts you for the parameter (a Content Server user), and then executes the SQL statement.

You can open a LiveReport if you have the See Contents permission on it. For more information about permissions, see Permission Types.

The LiveReports Volume contains a set of default LiveReports. You can see any LiveReports in the LiveReports Volume to which you have at least the See permission on the LiveReports tab of your My Reports page.

 

Note:
LiveReports can reside in any container, but only LiveReports that are stored in the LiveReports Volume appear on your My Reports page.

 

A LiveReport can be set up so that, when you open it, you are asked to provide additional information, such as your name. The way you provide additional information depends on the LiveReport parameters. In some cases, you can browse for information that you need. In other cases, you must type information into a box. In still other cases, Content Server supplies the information for you.

Certain LiveReport formats allow you to view information differently after you open the report. For example, if the information is presented in a pie chart, you can rotate the chart, separate and reattach individual segments of the pie chart, and separate segments from the pie. For more information about LiveReport formats, see Understanding LiveReport Formats.
