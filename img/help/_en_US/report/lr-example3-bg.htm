There are two Include IF conditions that are dependent on other templates. These two conditions are: Template Condition True and Template Condition False. In each case, a Template # field is provided in order to input the number of whichever template's condition will be used to control this one. For more information about these conditions, see the “Include IF” row in SQL Template Fields and Include IF.

Example 1-7: Adding two templates based on a single condition

This example shows two templates where one template is conditional on &inputlabel1 not being set to an empty string, and the second template is dependent on the first template. In this example we show a situation where the developer has decided that if the subtype has not been included as a filter clause, then it will not be required as a column. If &inputlabel1 is not a blank string then both the Where template and the column Select template are included.

  1. In the Main SQL Source, in the SQL field, type:

    select name~2
from sampleTable
~1

  2. In the Template Setup, in the Templates area, define ~1:

    • Do not select the Auto-Where box.

    • In the SQL Source field, type: “where subtype = %1”

    • Do not select the Auto-Comma box.

    • From the Include IF list, select “No inputs set to flag”.

    • In the Options field, ensure the Flag Value field is empty.

  3. In the Template Setup, in the Templates area, define ~2:

    • Do not select the Auto-Where box.

    • In the SQL Source field, type: “, subtype”

    • Do not select the Auto-Comma box.

    • From the Include IF list, select “Template condition true”.

    • In the Options field, in the Template # field, type “1”.

  4. In the Inputs area, include a generic input. For example, include an input of type STRING with a Prompt called “Subtype”.

If you access the following URL:

...&inputLabel1=grommet&nextURL=...

The resulting SQL will contain, in the SQL Source field, “select name , subtype from sampleTable where subtype ='grommet'”.
