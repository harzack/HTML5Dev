Using the Items Reserved by [User] LiveReport as an example, this page describes the LiveReport-specific fields that appear when you add or edit a LiveReport. When you add a report, additional general item settings appear. For more information, see Working with General Items.

To bring up the page containing the fields discussed below, from the Global Menu Bar, select Reports from the Personal menu. On the My Reports page, select the LiveReports tab. Scroll down to the Items Reserved by [User] LiveReport. From the Items Reserved by [User] Functions menu, select Edit.

  1. Title text box, which contains the title value of the LiveReport you selected. In this example, the text box contains the title: “Items Reserved by [User]”.

    The title text appears on the LiveReport's results page. The bracketed value, [User], will be replaced with the name of the user that opens the report.

  2. Record Limit number field which requires a positive integer. This field contains the default value 100.

    This number represents the maximum number of results that a LiveReport will return. This prevents Content Server from searching for more information than necessary. If the specified limit is reached when a LiveReport opens, a note appears on the result page.

  3. Show Prompts check box.

    Selecting this check box requires users to supply parameters when this LiveReport is used as a subreport. This check box is not selected by default, which causes certain report values to automatically pass on to subreports.

  4. Inputs area containing multiple lines, each of which includes a Type list and a Prompt text box.

    There are multiple input types available in the Type list. For more information about input types, see Input Type Requirements. If the “User” type is selected from the Type list, “User Name:” appears when a user opens a report. The input type, “User”, enables the user who opens the report to browse for the name of the user about whom they want information.

  5. SQL text box.

    This text box contains the SQL statement that defines the parameters and input values from the report. A LiveReport cannot be executed without a valid SQL statement. In this example, the SQL statement requests a LiveReport to return information for items that are reserved by a specified user.

  6. Allow Database Modification check box.

    This check box is cleared by default, which prevents the SQL statement from modifying the database.

     

    Note:
    The Allow Database Modification check box does not appear if your administrator has enabled LiveReports Security.

     

  7. Param %<number> lists.

    The parameters that you choose from these lists need to be defined in the SQL statement. In this example, the Param %1 list selected by default is User Input 1. This means that the value displayed in the LiveReport is determined at the time it is opened, depending on the user input. The Param %2 list selected by default is Filter Permissions. This means that the LiveReport only returns results for those items that the user has been assigned the See permission. The Param %3 list selected by default is Report Fields. This parameter type is required by the LiveReport format, unless your SQL statement contains the statement SELECT * FROM DTree. For more information about parameters, see Parameter and Input Types and LiveReports and Auto LiveReports.

  8. Report Format list and Configure button.

    In this example, the results of the SQL statement appear in the “LiveReport” format, which displays results in a list. It is often helpful to select the Auto LiveReport format first so that you can inspect the results in a list before adapting your SQL statement to produce a customized graph.

  9. Click-thru Sub-Report text box and Browse Content Server... button.

    The Browse Content Server... button enables you to select a subreport which you can associate with this LiveReport. In this example, a subreport is not associated with the LiveReport.

  10. Sub-Report Param <number> text boxes.

    If, in the Click-thru Sub-Report text box, you associated a subreport with this LiveReport, you must type the name of an existing parameter that you want to pass on to the subreport in these fields. These subreport parameters must correspond to the input parameters of the subreport.

  11. Display Columns area containing multiple lines, each of which includes a Field text box and a Column Title text box.

    This instructs Content Server, when displaying the LiveReport results, to display the value you enter in the Column Title text box as the column title of the column you identify in the Field text box, instead of using the terminology from the database table.
