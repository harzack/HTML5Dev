Multi-File Output makes it simple and fast to download, email, or print either single or multiple documents from Content Server. Installing and configuring Multi-File Output adds the following buttons to the Content Server interface:

  • Zip & Download – compress one or more Content Server objects into a ZIP file for download.

  • Zip & E-mail – compress, download, and attach one or more Content Server objects into a new email message, or save the resulting ZIP file back to Content Server and email a hyperlinked URL.

  • Print – download and print one or more documents.

Multi-File Output works with any supported document type for which the user has at least See Contents permissions. It can extract the contents of entire folders, compound documents, or other supported containers in Content Server in one action. You can also use Multi-File Output with Content Server Collections to gather information from anywhere in Content Server and then output it all at once (one page at a time), mimicking the e-commerce shopping cart approach.

Multi-File Output also supports OpenText Enterprise Library objects. When you perform a Multi-File Output task on OpenText Enterprise Library objects, Content Server uses the default variant for content.

 

Notes:

  • To enable Multi-File Output to work with minor document versions, the user must have Reserve permissions.

  • Containers within Content Server can include other containers, such as folders within folders. If any of the objects selected is a container that includes other containers, the action initiated will not extend to the subcontainers, which means that only objects at the highest level are acted upon. To perform an action on the contents of a subcontainer, go to the Functions menu for that container.

  • The container types supported by Multi-File Output are Compound Documents, Folders, Projects, Tasks, and Workflows.

  • Multi-File Output does not support Compound Documents for Enterprise Library objects.

 

To perform an action, select the check box beside one or more object names and then click the appropriate action button. For example, select the check box for a Document, select the check box for a Spreadsheet, and then click the Zip & Download button. If an object has not been configured for the action selected, an error message is returned.

If an object has been configured for one or more of the Multi-File Output functions, it is also possible to initiate that functionality directly from its Functions menu. If the object type is not configured for the Zip & Download, Zip & E-mail, and Print functions, those functions do not appear in the Functions menu.
