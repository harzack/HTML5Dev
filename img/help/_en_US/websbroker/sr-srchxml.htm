Content Server indexes contain regions that store different types of information. For example, they contain a location region that stores an item's location in Content Server, a size region that stores the size of an item, and a date region that stores the date on which an item was created. Many of these regions have corresponding system attributes that you can search on the Content Server Search page.

For Content Server indexes that contain XML documents, an administrator can make XML regions, ortypes, searchable. XML types store information about XML documents in Content Server, for example, the XML elements, their content, and their attributes, and are defined by XML DTD files, which are also stored in Content Server. The XML types that are available for you to search depend on the XML documents in the index and the types that the administrator has made searchable.

If an XML type that you want to search is not available in the XML Types section on the Content Server Search page, you can search for the corresponding XML content by using other components on the Content Server Search page. For example, you can use a metadata search to search for documents created on a certain date.

Understanding XML types is important when performing searches with basic Queries. For more information about using basic Queries, see Searching with Basic Queries.
