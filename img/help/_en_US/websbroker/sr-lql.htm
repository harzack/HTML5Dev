You can search Content Server using complex Queries that precisely target the data you want to retrieve. When you create a complex Query, you can save it for reuse like any other Query. For information about saving Queries, see Working with Saved Searches.

Using Live Query Language, LQL

A complex Query is a single statement, constructed with LQL, that precisely targets the data you want to retrieve. For example, to find Content Server items that contain the word report but do not contain the word expense, type the complex Query report AND-NOT expense.

LQL queries initiated from the Advanced Search page will return OpenText Enterprise Library items provided that you specify an OpenText Enterprise Library slice in the Slices section.

Using LQL Conventions

LQL conventions define the syntax of a complex Query. They specify how to use wildcards, define the order in which Content Server processes Query terms, and define the syntax for constructing phrases. For more information about LQL conventions, see Understanding LQL Conventions.

Using LQL Keywords

LQL keywords are words that LQL reserves and assigns special meaning to. When used with LQL conventions, LQL keywords allow you to construct complex Queries. LQL keywords are not case sensitive. For more information about LQL keywords, see Understanding LQL Keywords.

Specifying Attributes Using LQL

LQL provides an alternative way to specify attributes when you search for Content Server items. Each Content Server item has certain attributes that contain additional information about it. By default, Content Server does not search item attributes. However, in addition to being able to specify attributes in a basic Query, you can use LQL to search for a specific attribute or set of attributes. For more information about attributes, see Working With Categories and Attributes. For more information about using LQL to specify attributes, see Specifying Query Attributes Using LQL.

 

Note:
Attribute names are case sensitive. For a list of available attributes and their proper capitalization, contact your administrator.

 

Working with Regular Expressions Using LQL

You can work with regular expressions using LQL to find Content Server items that contain a regular expression expansion of the specified query expression. You use query expressions with the QLREGEX keyword and enclose them in double quotation marks “ ”. For information about using regular expressions, see Working with Regular Expressions Using LQL.

LQL Tutorial

The query language allows users to construct search requests that can be processed by the OpenText Search Engine. These search requests are described in Search Bar Query Tutorial.
