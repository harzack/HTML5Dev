Search Filters are a feature of Content Server that can assist with understanding and refining Search Results. When they are enabled, Search Filters are displayed in a panel beside Search Results. Similar to Browse Filters, you can hide or show the Search Filter panel by clicking on the orange bar that displays when your mouse is over the edge of the Search Results display area. The location on the left or right is based on your preferences, specified by the Sidebar Location setting on your My General Settings page. For details, see Changing Your Personal Settings.

A Search Filter is comprised of a title bar and a list of values. The title bar shows the “Display Name” of the metadata field used for the Search Filter. The behavior of the list of values will differ based on the type of metadata displayed in the filter, but in general is comprised of a value for the metadata and a count which indicates how frequently that value shows up in the results. When you select a value in a Search Filter, a new search request is generated which includes the additional restrictions of the selection.

 

Note:
Search Filters are configured by your Content Server system administrator. There may be significant differences between your actual system and the typical configurations described here.

When using Search Filters, Remote Search results are excluded from the filter values and counts.

Search Filters currently does not support Aggregate-Text regions.

 

When Search Filters are applied, they are displayed at the top of the panel. You can remove a filter by clicking the small x icon beside it, or remove all the current filters by clicking the Remove all filters link. On then Advanced Search page, active Search Filters are also displayed, and can be likewise removed.

Content Server asks the search system to provide information for Filters when the search query is performed. In some cases, there may be no results for the requested filters, in which case the Filter is not displayed. This can result in Filters not appearing, and is normal behavior.

Restricted Filters

If an attribute filter is applied, and you do not have permissions for the attribute's Category, you will see (Restricted Filters) displayed in the active Filters. This indicates there are some filters applied that affect the Search Results that you don't have permissions to see. You cannot remove a single restricted Filter, but you can click the Remove all filters link.

Hidden Containers

You can introduce Search Filters constraints into a saved Query. For information about saved Queries, see Working with Saved Searches. Some constraints may contain information that you don't have permissions to see. If you run a saved Query where you don't have permission for the Filter data, for example, if there is a Filter on the location “Folder A”, and you don't have permissions to see this folder, the data is marked as (hidden) on the Search Results and Advanced Search pages.

Differences from Browse Filters

Search Filters may often be used for the same purposes as Content Filters, but there are some important differences between the two, and the differences in behavior are normal.

You should expect to see different filters and value counts when using Search Filters. The Search Filters are based upon the capabilities of a full text Search Engine, and the Content Filters are based on system information. For example, the Search Filters might display information from Microsoft Office documents that is not known to Content Filters.

The values and counts in Search Filters are estimates. You will often encounter situations where selecting a filter value will display more or fewer results than indicated by the estimate. You may even get no results. There are many reasons that the estimates may be different than the actual values, including:

  • Time delays between adding or changing objects and the search index being updated.

  • The estimating process cannot anticipate permission restrictions.

  • The search index operates on word matching, not on exact strings.

  • The Search Engine ignores most punctuation and special symbols.

  • For performance optimization, the Search Engine will limit the number of unique values that it will consider when generating Search Filters.

Types of Search Filters

  • Text –

    If the metadata field is comprised of Text in the Search Engine, it is displayed as a Text style Search Filter. Text Filters display values for the metadata fields that occur most frequently in the current Search Results. Text Filters are configured to display few or many filter values, and you can switch between these states using the Less… or More… links in the panel. Text filters are optimized for strings of 32 characters or less.

  • Dates

    If the metadata field is of type Date in the Search Engine, the Search Filter is enhanced to include tab sheets. Date filters represent recent dates, from the current day backwards. The default tab sheet has convenient representations for intervals of time. If you know more specifically which date is of interest, additional tabs contain specific recent days, weeks, months and years that can be used to refine the search query.

  • Classifications and Categories

    When used as Search Filters, these metadata fields are checked for permissions. Depending on the situation, these will be displayed as (hidden) or removed from the Filters entirely.

  • Container

    Several metadata fields represent locations in Content Server. Filters for these fields display a string that shows the name of the folder, plus the name of the next higher folder, in the form …grandparent : foldername. If the string does not fit in the available space, it is truncated on the left and preceded by an ellipsis (…). In some cases, if you do not have permission to see the name of the folder, it may display as (hidden).

  • Integers

    Metadata fields that contain integers will display the most common integer values for the current Search Results. More… and Less… links control the number of values to display. Note that some Text fields may appear to be Integer fields. Usually, there is little difference. One difference is that a Text field will treat 007 and 7 as distinct values, whereas an Integer filter will consider these both as the number 7.
