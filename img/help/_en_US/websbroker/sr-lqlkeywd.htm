AND

Description

The AND keyword has the same function as the & keyword. It finds Content Server items that contain both specified query expressions.

Syntax

<expression> AND <expression>

Example

Type fish AND bird to find Content Server items that contain both fish and bird.

&

Description

The & keyword has the same function as the AND keyword. It finds Content Server items that contain both specified query expressions.

Syntax

<expression>  &  <expression>

Example

Type fish & bird to find Content Server items that contain both fish and bird.

AND-NOT

Description

The AND-NOT keyword has the same function as the &! keyword. It finds Content Server items that contain the expression to the left of the keyword and do not contain the expression to the right of the keyword.

Syntax

<expression>  AND-NOT <expression>

Example

Type fish AND-NOT bird to find Content Server items that contain the word fish but do not contain the word bird.

&!

Description

The &! keyword has the same function as the AND-NOT keyword. It finds Content Server items that contain the expression to the left of the keyword and do not contain the expression to the right of the keyword.

Syntax

<expression>  &! <expression>

Example

Type fish &! bird to find Content Server items that contain the word fish but do not contain the word bird.

OR

Description

The OR keyword has the same function as the | keyword. It finds Content Server items that contain the expression to the left of the keyword, or those that contain the expression to the right of the keyword, or those that contain both expressions.

Syntax

<expression>  OR  <expression>

Example

Type fish OR bird to find Content Server items that contain the word fish, or Content Server items that contain the word bird, or Content Server items that contain both fish and bird.

|

Description

The | keyword has the same function as the OR keyword. It finds Content Server items that contain the expression to the left of the keyword, or those that contain the expression to the right of the keyword, or those that contain both expressions.

Syntax

<expression>  | <expression>

Example

Type fish | bird to find Content Server items that contain the word fish, or Content Server items that contain the word bird, or Content Server items that contain both fish and bird.

SOR

Description

The SOR keyword has the same function as the OR keyword but its relevancy ranking algorithm considers the expressions to the left and right of the SOR keyword as synonyms and treats them as a single term. This prevents synonyms from carrying more weight in the calculation of the score for a search result item. The ranked scores and order of the results produced by queries that use the SOR keyword differ from the ranked scores and order of the results produced by queries that use the OR keyword.

Syntax

<expression>  SOR  <expression>

Example

Type "U.K." SOR "United Kingdom" to find Content Server items that contain at least one instance of the word U.K., or the word United Kingdom, or both. Content Server lists the results of this Query in a different order than the results of "U.K." OR "United Kingdom".

XOR

Description

The XOR keyword finds Content Server items that contain one of the two specified query expressions, but not both. This keyword is also known as the exclusive <OR> keyword.

Syntax

<expression>  XOR  <expression>

Example

Type fish XOR bird to find Content Server items that contain the word fish or the word bird, but not both.

=

Description

The = keyword finds Content Server items that contain the specified phrase, as follows:

  • For metadata text regions, the search term or phrase must match the entire region, or an entire component value of a multivalued region, to be considered a match.

    For example, [qlregion "CityName"] ="York" would match “York” in the CityName region, but would not match “New York”.

  • For content searches, the search term or phrase may appear anywhere in the object content.

    For example, ="York" would match an object with City of York in its content.

  • Searches are not case-sensitive.

    For example, ="York" would match “YORK”.

  • Searches are not sensitive to extraneous whitespace or punctuation (as long as they do not break the individual search terms).

    For example, [qlregion "CityName"] ="York" would match a CityName of “: York :”.

  • Searches of multivalued metadata fields only require one component of the multivalue to match.

    For example, if the input XML of an object included <CityName>London</CityName><CityName>New York</CityName>, then a search for [qlregion "CityName"] ="New York" will match the object because “New York” matches an entire component of the multivalued CityName region, and a search for [qlregion "CityName"] ="York" would not match.

Syntax

=<term>

For example, type =York to find Content Server items that either have a metadata region value of “York” or contain “York” in the item's content.

Example

Type = York to find Content Server items that contain the word York.

NEAR

Description

The NEAR keyword has the same function as the QLNEAR keyword and the PROX keyword.

Syntax

<phrase> NEAR <phrase>

Example

Type race NEAR car to find Content Server items that contain the word race and the word car up to a distance of 10 tokens from one-another, in any order.

NOT

Description

The NOT keyword has the same function as the ! keyword and the != keyword. It finds Content Server items that do not contain the specified phrase.

Syntax

NOT <expression>

Example

Type NOT bird to find Content Server items that do not contain the word bird.

!

Description

The ! keyword has the same function as the NOT keyword and the != keyword. It finds Content Server items that do not contain the specified phrase.

Syntax

! <expression>

Example

Type ! bird to find Content Server items that do not contain the word bird.

!=

Description

The != keyword has the same function as the NOT keyword and ! keyword. It finds Content Server items that do not contain the specified phrase.

Syntax

!= <expression>

Example

Type != bird to find Content Server items that do not contain the word bird.

<

Description

The < keyword finds Content Server items that contain numbers or words that are less than the specified number or word. If the specified term is an integer or real number, Content Server uses numeric ordering. Otherwise, it uses lexicographic ordering.

Syntax

< <expression>

Example

Type < 20140630 to find Content Server items that contain at least one number that is less than 20140630. If you restrict this Query to a date attribute, it finds Content Server items dated before June 30, 2014.

<=

Description

The <= keyword finds Content Server items that contain numbers or words that are less than or equal to the specified number or word. If the specified term is an integer or real number, Content Server uses numeric ordering. Otherwise, it uses lexicographic ordering.

Syntax

<= <expression>

Example

Type <= 20140630 to find Content Server items that contain at least one number or word that is less than or equal to 20140630. If you restrict this Query to a Date attribute, it finds Content Server items dated June 30, 2014, or earlier.

>

Description

The > keyword finds Content Server items that contain numbers or words that are greater than the specified number or word. If the specified term is an integer or real number, Content Server uses numeric ordering. Otherwise, it uses lexicographic ordering.

Syntax

> <expression>

Example

Type > 20140630 to find Content Server items that contain at least one number or word that is greater than 20140630. If you restrict this query to a date attribute, it finds Content Server items dated after June 30, 2014.

>=

Description

The >= keyword finds Content Server items that contain numbers or words that are greater than or equal to the specified number or word. If the specified term is an integer or real number, Content Server uses numeric ordering. Otherwise, it uses lexicographic ordering.

Syntax

>= <expression>

Example

Type >= 20140630 to find Content Server items that contain at least one number or word that is greater than or equal to 20140630. If you restrict this Query to a date attribute, it finds Content Server items dated June 30, 2014, or later.

PROX

Description

The prox[n,t/f] keyword finds Content Server items that contain all specified query phrases that are within a designated distance (n) of one-another, and where the ordering is either ordered (t) or unordered (f). Distance (n) is calculated as the number of tokens between the last token in the first phrase and the first token in the second phrase. The specification of the second parameter (t/f) is optional and defaults to f.

 

Note:
The PROX operator is limited strictly to simple search terms and exact phrases. It is not generic.

 

Syntax

<phrase> prox[n,t/f] <phrase>

Example

Type race prox[10,t] car to find Content Server items that contain the word race preceding the word car up to a distance of 10 tokens of one-another.

QLLIKE

Description

The QLLIKE keyword finds search results in text metadata regions that have common structures found in part numbers, URLs, and file names.

 

Note:
QLLIKE must be configured by your administrator. If QLLIKE is used on a region without this configuration, it behaves like an AND operation. For more information about the search configuration at your site, contact your administrator.

When another modifier, for example, QLPHONETIC, is specified, then it takes precedence.

 

Syntax

qllike “<expression>”

Examples

Type [qlregion "OTFileName"] qllike “MyDocument99” to find all Content Server objects with names containing words similar to MyDocument99.

If your part number is in the file name, then the OTName region needs the “LIKE”configuration. The query [qlregion "OTFileName"] qllike "ZA3009-17" will find a file named ZA-17F12_-PN3009.xls.

QLATTRIBUTE

Description

The QLATTRIBUTE keyword restricts QLREGION expressions to region values containing a specific attribute.

All OTName and OTDComment field values in Content Server receive a lang="x" attribute, where x is the metadata language code. QLATTRIBUTE is used in conjunction with the QLREGION keyword to narrow searches to region values with a particular attribute. lang is the only region attribute that is currently supported. The two operators used with QLATTRIBUTE are: =, is null.

Syntax

QLATTRIBUTE <expression>

Examples

Type [qlregion "OTName"] [qlattribute "lang" = "fr"] "voiture" to find all Content Server objects with French names containing the word voiture.

Type [qlregion "OTDComment"] [qlattribute "lang" is null] * to find all objects with a description defined that does not have a language associated with it.

QLPHONETIC

Description

The QLPHONETIC keyword finds Content Server items that contain words that sound like the specified query expression. This keyword is based on a popular algorithm called Soundex. The Soundex algorithm takes an English word and produces a four-digit representation of the word, designed to match the phonetic pronunciation of the original word. Soundex is often used for vague searches, where a close match is required.

Syntax

QLPHONETIC <expression>

Example

Type QLPHONETIC sail to find Content Server items that contain words like sail or sale.

QLREGION

Description

The QLREGION keyword finds Content Server items that contain search terms within a specific index region. Each region stores a different kind of information about indexed items, including the content and metadata of documents. For a list of regions, contact your Content Server administrator.

Syntax

[QLREGION <region>] <expression>

Example

Type [QLREGION OTCreatedByName] jdoe to find Content Server items that contain the word jdoe in the OTCreatedByName region.

QLSTEM

Description

The QLSTEM keyword finds Content Server items that are likely noun plural or singular forms of the specified query expression.

Syntax

QLSTEM <expression>

Example

Type QLSTEM tax to find Content Server items that contain either the word tax or taxes.

QLTHESAURUS

Description

The QLTHESAURUS keyword finds Content Server items that contain thesaurus entries that are associated with the specified query expression.

Syntax

QLTHESAURUS <expression>

Example

Type QLTHESAURUS stone to find Content Server items that contain at least one of the thesaurus entries associated with the word stone, such as stone, pit, seed, or kernel.

QLRIGHT-TRUNCATION

Description

The QLRIGHT-TRUNCATION keyword finds Content Server items that contain words whose prefixes match the specified query expression.

Syntax

QLRIGHT-TRUNCATION <expression>

Example

Type QLRIGHT-TRUNCATION post to find Content Server items that contain at least one word that begins with post, such as postpone, postman, or posture.

QLLEFT-TRUNCATION

Description

The QLLEFT-TRUNCATION keyword finds Content Server items that contain words whose suffixes match the specified query expression.

Syntax

QLLEFT-TRUNCATION <expression>

Example

Type QLLEFT-TRUNCATION thing to find Content Server items that contain at least one word that ends with thing, such as thing, anything, or soothing.

QLRANGE

Description

The QLRANGE keyword finds Content Server items that contain words in the lexicographic or numeric range between two specified query expressions. The terms that you use to specify the range are also included in the range. If the specified terms are integers or real numbers, Content Server uses numeric ordering. Otherwise, it uses lexicographic ordering.

You must enclose the range in double quotation marks, " ", and separate the terms that you use to specify the range with a tilde, ~.

Syntax

QLRANGE < "expression >~ <expression">

Example

Type QLRANGE "giraffe ~ gnu" to find Content Server items that contain at least one word in the lexicographic range from giraffe to gnu, such as giraffe, giraffes, give, glow, gnome, or gnu.

Type QLRANGE "20140801 ~ 20140901" to find items that contain at least one number between 20140801 and 20140901, for example, dates between August 1, 2014, and September 1, 2014, inclusive.

QLREGEX

Description

The QLREGEX keyword finds Content Server items that contain a regular expression expansion of the specified query expression. Query expressions used with the QLREGEX keyword must be enclosed in double quotation marks, " ". This keyword does not work with phrases (across tokens). For information about using regular expressions, see Working with Regular Expressions Using LQL.

Syntax

QLREGEX <"expression">

Example

Type QLREGEX "^the" to find Content Server items that contain at least one of the regular expression expansions of ^the, such as the, their, them, then, there, therefore, thesaurus, these, or they.

QLNEAR

Description

The QLNEAR keyword has the same function as the PROX keyword and the NEAR keyword, but it is set to a default of prox[10,f].

Syntax

<phrase>QLNEAR <phrase>

Example

Type race QLNEAR car to find Content Server items that contain the word race and the word car up to a distance of 10 tokens from one-another, in any order.
