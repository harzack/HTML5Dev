To create a custom view search:

  1. Navigate to the location where a saved Search is located in Content Server, click the Search's Functions menu, and then choose Make Custom View Search.

     

    Tip:
    You can display all the Searches that are stored in your Personal Workspace or your Project Workspace by choosing Reports on the Personal or Project menu.

     

  2. On the Make Custom View Search page, in the Full Text section, to enable the text box in which to enter search terms, select the Show check box.

  3. In the Natural Language Query section, to enable the text box in which to type a question or text, select the Show check box.

  4. In the Slices section, to enable which Content Server slice to search, select the Show check box. If you do not explicitly select a Content Server slice, the Enterprise slice is searched by default.

     

    Note:
    If a slice is deleted, it may still appear in the Slices drop-down list, and may be returned in a func=search.getslices function call. This is because of the caching characteristics in Content Server. Once the cache is cleared by all active threads the slice will no longer be returned.

     

  5. In the Location section, to choose a Content Server location to enter in the text box, select the Show check box.

  6. In the Options section, select each Show check box to:

    • Display a Custom View Search Title. Enter the text to display.

    • Display Custom View Search Instructions. Enter the intructions to display.

    • Display the Show Search Component Headers. These are the titles of the sections described above.

    • Select the Search Results Format in the Select Display Style drop-down list. For more information, see Setting Display Options.

  7. Click Save.

For more information about Custom Views, see Working with Custom Views.
