Content Server provides features that allow you to work with search results to assist you in finding the information you are searching for. For example, you can sort the list of results by relevance, date, or size. You can view an item that has been marked as a Best Bet. A Best Bet is an item that Content Server suggests is a particularly good match for your search. You can also view instances of your search terms in search results using hit highlighting. It is possible to save search results, refine search results based on recurring patterns in the results, and find items that are similar to a search result.

 

Note:
The availability of these features varies depending on the selected display style of the search results and settings controlled by your Content Server administrator.

 

Sorting Search Results

In the Sort drop-down list on the Search Results page, you can sort the list of search results by the following criteria:

Multi-Field

You can specify multiple fields to be used to sort results and specify the order the criteria should be applied. For example, you can specify Date in descending order as sort criteria and then add Size in ascending order as sort criteria. The results will then be sorted by the most recently modified items and then by size so that the newest, smallest items appear first in the search results.

Relevance

The score that Content Server has assigned to the result based on how closely the result matches your search criteria.

Date

The date the item was last modified. You can sort the list in either ascending or descending date order. Descending order lists the most recently modified items first. Ascending order lists the least recently modified item first.

Size

The size, in bytes, of the item. For the purposes of sorting, all items that have a size in bytes. For example, Documents are considered larger than items that have a size in items, for example, Folders. You can sort the list in either ascending or descending size order. Descending order lists the largest items first. Ascending order lists the smallest items first.

 

Note:
Depending on settings configured by your administrator, other options may be available in the Sort drop-down list.

When the Physical Objects module is installed, you can sort Search results by the following Physical Objects circulation fields: Borrowed By, Obtained By, Comment, Borrowed Date, and Return Date.

 

Changing the Number of Search Results on a Page

You can change the number of search results that are displayed on one page using the Show list on the Search Results page. This enables you to easily increase or decrease the number of results you are viewing depending on your needs for an individual search.

 

Tip:
You can change the number of results that appear by default, by modifying the Number of Results setting on the Display Options page and then save your setting as a Search Form. For more information, see Working with Forms and the Search Results Page

 

Filtering Search Results

You can generate a new search request, based on your Search Results, by selecting values in the Search Filters that include additional restrictions. Search Filters are displayed in a panel beside your Search Results, comprised of a title bar and a list of values, showing a number for the metadata and a count which indicates how frequently that value shows up in the results. For details, see Using Search Filters.

Working with Best Bets

The Best Bets section on the Search Results page allows you access to items that Content Server suggests are the best matches for your search term. For example, you may be updating current design templates and need to search for all associated documentation. If the phrase design templates has been applied to the documents as a best-bets value, the required documents for the project will be displayed in the Best Bets section of the Search Results page. Best-bets values are determined by best bets administrators, knowledge managers, or other persons with appropriate privileges.

If you have best-bets administration privileges, you can apply a best-bets value to Content Server items, which allows you to determine the items to be displayed in the Best Bets section when a user's search matches a best-bets value. You want to apply values that enhance the search; therefore, you should avoid using generic values such as template or human resources. If you are using a phrase as a best-bets value, you must enclose it in quotation marks. If you apply a phrase without quotation marks, such as design templates, the search will return items containing design or templates or design templates. You can also apply a best-bets expiry date to a Content Server item. Both the values and the expiry date can be changed or removed at any time.

 

Notes:

  • If you are using the best-bets system attribute to set the expiry date, the only acceptable date format is YYYYMMDD. If you use any other format, the best-bets results will not be returned.

  • Best Bets does not support OpenText Enterprise Library items.

 

Using Hit Highlighting

Hit highlighting allows you to quickly scan the relevant portions of an item's content to see if it contains the information you need. This is particularly useful when a search result is a large Document. You can hit highlight Documents, Text Documents, Discussion Topics, Discussion Replies, and XML DTDs. You can also hit highlight email messages and their attachments.

If you search with a modifier, Content Server also highlights words that are related to the words in your Query. For example, if you search for words related to the stem form of the word interface, Content Server highlights words such as interface, interfaces, interfacing, and interfaced.

 

Note:
If a search generates results that contain 100 or more search terms, Content Server hit highlights only the first 100 terms.

 

Using Advanced Search

Advanced Search lets you make changes to your Query on the Content Server Search page and run it again. If you then save the Content Server Search page as a Search Form, it will include the theme criteria that you may have specified previously. For information about Search Forms, see Working with Search Forms.

Saving Search Results

Saving search results lets you access that list of results at any time. When you save search results, you create a Search Result Snapshot, and that Snapshot is added to Content Server as an item. Consequently, you can perform many of the same operations on Search Result Snapshots that you perform on other Content Server items. For example, you can set permissions for the Search Result Snapshot so that other Content Server users and groups can access it. For more information about the operations that you can perform on Content Server items, see Working with General Items.

When you want to display the list of search results in a Search Result Snapshot, you open the Snapshot. Search Result Snapshots are not updated as items are added to or removed from Content Server. Instead, Search Result Snapshots reflect the page of search results obtained by a particular search performed at a particular time. If you want to update a Search Result Snapshot, you must perform a search using the same search criteria, and then save the search results, overwriting the out-of-date Snapshot.

 

Notes:

  • When you create a Search Result Snapshot, you save a page of search results and not the entire set of results. To change the number of results that are saved in a Snapshot, change the display options for the number of results displayed on the Search Result page. To save several pages of search results, create a Search Result Snapshot for each page. For more information about display options, see Searching with Basic Queries.

  • You can include OpenText Enterprise Library items when you save search results if these items are returned in the search result.

 

Collecting Search Results

Collecting search results allows you add shortcuts to an individual Content Server item, or you can collect an entire set of results. For information about collecting individual items see, Collecting Items.

When collecting an entire set of results, you can enable email notification; the default email address is called from the User's profile. If there is no address defined in the User profile, you can add an address in the Recipient field.

 

Tip:
If you want to update the email address in your profile, click the Personal menu and then click My Profile.

 

You can collect OpenText Enterprise Library items from the search results page by performing bulk operations or clicking the Collect All Search Results button.

Downloading a Collection as a Spreadsheet

Once a Collection is created, and items are collected, you can download a Collection as a spreadsheet. You can select individual items, or the entire Collection, and specify the type of item to download using the object type filter. For more information, see Downloading a Collection as a Spreadsheet.

Finding Similar Documents

You can search for Documents and Text Documents similar to one found in a page of search results just as you can search for Documents and Text Documents that are similar to one in a Workspace. For more information, see Finding Similar Documents.

Setting Display Options

The Search Form you search with determines how the results of your Query appear on the Search Results page. For more information about setting display options for the Search Results page, see Searching with Basic Queries. For more information about Search Forms, see Setting Display Options.
