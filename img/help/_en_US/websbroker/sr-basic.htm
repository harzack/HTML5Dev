You can search Content Server by creating a basic Query that contains keywords. A basic Query can also contain a combination of keywords and other criteria that expands or narrows your search. When you create a basic Query, you can save it for reuse like any other Query. For information about other types of Queries, see Searching with Natural Language Queries or Searching with Complex Queries. For information about saving Queries, see Working with Saved Searches.

You can search for OpenText Enterprise Library items provided you have the appropriate permissions to see OpenText Enterprise Library slices. For more information about searching OpenText Enterprise Library items, see Using the Search Bar to Find Open Text Enterprise Library Services Items.

Working with Full Text

The Full Text section of the Content Server Search page lets you construct Queries that contain keywords or complex Queries constructed with the Content Server Query Language. This section also includes components that let you expand or narrow your search. You expand your search by selecting a modifier, which specifies the kind of related terms that you want included in the search. You narrow your search by specifying whether you want to search within the content or metadata of items, or both. Content is the information contained in Content Server items, such as the text of a document; metadata is the information accompanying each Content Server item, such as the date that a document was added to Content Server. You also narrow your search by using Boolean operators to define precisely the relationship between search criteria in your Query. For more information, see Searching Content Server.

 

Note:
If you search within metadata, you narrow your search to the metadata information that your administrator has made searchable, that is, marked search by default. This may or may not include all metadata associated with an item. When you search within both content and metadata, you search the metadata information that your administrator has made searchable as well as the content.

 

Working with Keywords

Keywords are the words or phrases that you specify when searching for Content Server items that contain those words or phrases. For example, if you specify the keywords business plan model, Content Server can return a list of Content Server items that contain all three words, any of these words, or the exact phrase. If two or more keywords are enclosed in quotation marks, Content Server searches for the exact phrase as it appears within the quotation marks.

Expanding a Search

You expand a search when you want to search for Content Server items that contain words that are related to the words in your Query. Selecting a modifier lets you specify the type of related words to include in the search. For example, you can select a modifier that returns Content Server items containing words that sound like the keyword you specify. You can also select a modifier that returns Content Server items containing words that are related to the stem form of the specified keyword. For example, searching for words related to the stem form of the keyword interface returns a list of Content Server items that contain words such as interface, interfaces, interfacing, and interfaced.

When you use keywords with a modifier, the modifier is applied to all the words. When you search for an exact phrase, the modifier is applied to the phrase as a whole. In a complex Query, the modifier is applied to all the words in that Query that are not Content Server Query Language keywords. For more information about the Content Server Query Language, see Searching with Complex Queries.

Narrowing a Search

You narrow a search by increasing the number of specific criteria that Content Server uses in a search. These criteria include the use of the Boolean operators AND, OR, and NOT. These operators apply basic logic that changes how the keywords are related. The criteria also include the option of searching within an item's content, metadata, or both. Other types of search criteria include the following advanced options: slices, system attributes, custom attributes within a Category, and XML types. The following table describes the available search criteria.

Table 1-1: Search Criteria

Criteria Lets you...
Within All Search the information contained within each Content Server item, such as within the text of documents as well as the metadata information that your administrator has made searchable.
Within Content Search the information contained within each Content Server item, such as within the text of documents.
Within Metadata Search the information accompanying each Content Server item, such as the date that a document was added to Content Server. If you search within metadata, you narrow your search to the metadata information that your administrator has made searchable, that is, marked search by default. This may or may not include all metadata associated with an item.
Boolean Operators Precisely define the relationships between search criteria in your Query. For example, if plan and test are keywords, searching for plan AND test returns Content Server items that contain both plan and test; plan OR test returns Content Server items that contain plan, or test, or both; plan NOT test returns all Content Server items that contain plan, but do not contain test.
Slices Search a particular collection, or slice, of Content Server information. For example, choose the Help slice to search the Content Server User Online Help for information.
System Attributes Search for Content Server items with certain attributes. For example, you can search for all Content Server items with a Created By attribute that is set to a certain user, or for a specific language when searching for a Name or Description. For information about attributes, see Working With Categories and Attributes.
 

Note:
In the Date drop-down list, if you specify Past 3 Months or Past 6 Months, the date range specified by the query is not exactly 3 or 6 months. Instead, the calculation is performed using 90 days and 180 days, respectively. Since some months may have more or less than 30 days, the range over which the query operates may miss some items that would otherwise be found.

You cannot search XML files through a System Attributes search. To perform a search on an XML region, use the XML Types component to select the DTD region.

When choosing options from the Content Type or MIME Type drop-down lists, your query is not modified until you select an option. For example, when you select Any Type from the Content Type drop-down list, or Any MIME Type from the MIME Type drop-down list, the region is defined. When you select No Type Defined from the Content Type drop-down list, or No MIME Type Defined from the MIME Type drop-down list, the region is not defined. The default state is “blank”.

 
Categories Search for Content Server items with certain custom attributes within a Category. For example, you can search for all Content Server items that have their Company Name custom attribute set to Open Text and their File Number custom attribute set to 93761634 within the Company Information Category. For information about categories, see Working With Categories and Attributes.
XML types Search for XML content in Content Server. Choose an XML DTD that defines the XML types, or XML regions, that you want to search. For information about XML types, see Understanding XML Types.
Location Search a specific area in Content Server, such as a particular Folder, Project, or Workspace.

Specifying System Attributes

Many system attributes appear by default on the Content Server Search page, but some must be enabled by an administrator to appear. For example, an administrator can enable the OTDocumentRating attribute so that users can search for Documents with a particular rating. An administrator will often enable new attributes when aContent Server module or a new version of Content Server is installed. For more information about attributes, see Working With Categories and Attributes.

The following table contains examples of attributes an administrator may enable.

 

Note:
