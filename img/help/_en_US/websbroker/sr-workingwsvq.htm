Saving a Search that you have constructed lets you reuse a particular set of search criteria. Running a saved Search then allows you to update the search results obtained from that Search. Modifying a saved Search allows you to make changes to a Search without having to rebuild it. If you want a framework for constructing Searches, you use Search Forms. For more information about Search Forms, see Working with Search Forms. For information about constructing a Search, see Searching with Basic Queries.

When you save a Search, you add it to a Folder or Workspace in Content Server. You can then perform many of the same operations that you perform on other types of Content Server items. For example, you can set permissions for the Search so that other Content Server users and groups can have access to it. For more information about the operations you can perform on Content Server items, see Working with General Items.

You can also create a Custom View Search in a particular Folder. This allows:

  • other users to add it to their Favorites

  • other users to reuse your search criteria

  • you to define the display priority of custom search forms

For more information, see To Create a Custom View Search, To Order Custom Views, and Working with Custom Views.

If you created a Hold Query in Records Management, you can save the Query on the Hold Criteria page or on the Advanced Search page. The Save as Query for Hold button appears in the Save Options section only if you have the proper permissions or are a designated Records Management user. After you save a Hold Query on the Advanced Search page, you are automatically returned to the Hold Criteria page in Records Management. For more information about using Search Enabled Holds in Records Management, see Managing Holds.
