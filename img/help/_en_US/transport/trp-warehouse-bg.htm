The Transport Warehouse is an area of Content Server that is designated for use by Warehouse Managers, and is reserved for functions that are exclusive to Transport. You access the Transport Warehouse from the Enterprise menu.

 

Note:
The Transport Warehouse option on the Enterprise menu is only visible to users that have been given access to the Warehouse by the Administrator. By default, only the Administrator can access the Warehouse.

 

The Warehouse is made up of the following components:

  • Warehouse Folder, which is a container used to store the items you are preparing to transport. Warehouse Administrators can create Warehouse Folders in the Transport Warehouse and give permissions to specific users so they can access the folders.

  • Workbench, which is made up of Content Server items. Once an item in Content Server is moved to the Workbench, it is called a Transport item. Workbenches can be accessed by users with the proper permissions. For more information, see Using the Transport Workbench.

    Workbenches can be added directly to a Warehouse, or to other Warehouse Folders. Content Server items must be added to a Workbench; they cannot be added directly to the Warehouse.

  • Transport Box, which is the container where items in Workbenches are added when they are ready to transport to the target system.

    When preparing the Workbench, you can add properties files and instructions in addition to the other items. These instructions can be useful for deployment, especially when a different Administrator or Warehouse Administrator is responsible for deploying the contents on the target system.

     

    Note:
    Transport Boxes can only be used by Warehouse Administrators.

     

Faceted Browsing is available for the items within a Warehouse, which means you can quickly view items based on the type of Warehouse item (Workbench, Transport Box, Transport Items, or Warehouse Folders).
