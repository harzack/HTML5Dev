The functionality that is available to you is dependent upon your assigned role. Transport users can be Administrators, Warehouse Managers, or Developers.

 

Note:
Transport Roles are hierarchical, which means that users can perform the functions allowed by the privilege they are assigned, as well as the functions for the privileges assigned to the roles at the levels below them.

 

For more information about Content Server Permissions and Privileges, see Managing Permissions.

Administrator

The Administrator has primary control over Content Server, and can perform many tasks that standard users cannot because of security settings and permissions. The Administrator specifies which users have the Warehouse Manager permission.

 

Note:
To set Warehouse Manager permissions, you must access the Object and Usage Privileges page from the Content Server Administration page.

 

Warehouse Manager

The Warehouse Manager role is designated by the Administrator. A Warehouse Manager is responsible for managing the items that will be imported, exported, tracked, and moved to other Content Server instances. Warehouse Managers can see all Workbenches and perform all functions in the Workbench. They are also the only users that have access to the default Transport Box Folder, which is required to transport items.

Developer

A Developer is the user responsible for building the transportable applications. Developers can add and organize Workbenches, add items to Workbenches, and prepare the sets of files to be transported. These users have limited access to some areas of the Warehouse. Developers are designated by the Warehouse Manager or Administrator.
