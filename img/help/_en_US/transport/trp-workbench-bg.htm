The Workbench is the primary location on the source system where users store and work with items that will be transported. Workbenches are also used on the target system to review and deploy the items that were transported.

The Workbench can be used by multiple users, as long as they have permission to access it. Users can view the Workbenches they have permissions to on the My Workbenches page, which is accessed from the Personal menu. If you have permissions, you can also click the Warehouse link in the Workbench’s Location column to access the Warehouse that contains the Workbench.

When you open a Workbench, the Workbench Overview page shows a list of all items contained in that Workbench. The Workbench Overview page provides details about the items in the Workbench, such as when it was added to the Workbench and the last time it was added to a Transport Box.

Adding Items to a Workbench

You can add folders, Documents and Compound Documents, URLs, Custom Views, and Projects to a Workbench. If you add a folder to a Workbench, you can include all of the folder’s contents as well. You cannot choose specific items in the folder unless you add them individually; if you add a folder and select the Include Contents option, all items in the folder are added to the Workbench. Transport automatically checks to see if items already exist in the Workbench before they are added. If duplicate item exists, you can overwrite the existing item or ignore the duplicate and leave the existing item as is.

 

Note:
Unlike most items in Content Server, items in a Workbench cannot be copied or moved.

 

Adding Items to a Transport Box

 

Note:
Transport Boxes appear in the Warehouse and can only be accessed by the Warehouse Administrator.

 

A Transport Box is a container that stores the set of items that are ready to be transferred to the target system. The Transport Boxes folder is the default container that appears in the Warehouse automatically. Transport Boxes are created from the Warehouse, or from a Workbench if you have the proper permissions, and added to the Transport Box container by default. If a Transport Box is created from the Workbench, a message appears stating that the box is now available for the Warehouse Manager.

You can add additional boxes to the Transport Boxes folder. General information for Transport Boxes, such as the name, description, size, when and by whom it was created, and what instance of Content Server the box originated from, appear on the Transport Box Overview page. The overview page appears when you open a box, either by clicking the box’s link, or clicking Open on the box’s Functions menu.

Adding a Workbench to a Transport Box

Once a Workbench contains all the items required for transport and the Workbench has been reviewed and tested, the items are added to a box. Adding to a box means that you are finalizing the items for transport to the target system. Once the box is finalized, it is moved to the Transport Boxes area of the Warehouse, which is restricted to users with administration permissions.

The Add to Transport Box function is the final Workbench task for non-Administrator users; however, the Warehouse Administrator still uses the transport box to begin the deployment process. For more information, see Deploying a Workbench.
