The Transport feature allows users to package items together from their current instance of Content Server (source system), download them as a ZIP file, and then import and deploy them to a different instance of Content Server (target system).

Transport has a dedicated workspace in Content Server, called the Transport Warehouse. The Warehouse contains tools that are exclusive to Transport, which allow users to work with Content Server items and prepare the files to be exported and deployed to the target system. Transport functionality is based on users’ roles, which means users must have a specific role assigned to them in order to perform certain tasks. For more information about Transport roles, see Understanding Transport Roles.

Items that are added to a Warehouse, changed, or transported and deployed to a target system are included in the audit trail. The users that perform the tasks are also tracked, and can be viewed based on the specific user or by role. Auditing information for Transport items is shown on the item’s Properties pages. For information about Properties pages for Content Server items, see Viewing and Modifying Item Properties.

 

Note:
Certain Transport settings are managed on the Warehouse Settings administration pages, and can only be changed by the Administrator.
