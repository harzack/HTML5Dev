A Task List is a container that provides a simple way to manage and track the details of individual and collective work assignments. A Task List can contain Tasks, Milestones, and Task Groups, and it can be added anywhere. Using Task Lists to manage assignments in a Project is especially useful, since it allows the members of a Project to remain informed of a Task's status.

Tasks enable you to provide a user or team of users with a priority or due date for an assignment, as well as detailed instructions for completing an assignment. You must create a Task List before you can add Tasks or Task Groups. You can add as many Tasks as necessary to a Task List, another Task, or a Task Group.

A Task Group is used to organize a large number of related Tasks. You can perform functions on groups of Tasks that reside in a Task Group. In addition, you can add multiple Tasks to a Task Group, and then add multiple Task Groups to a Task List. You can also create specific Task Groups for each user, and give each separate Milestones.

A Milestone represents a target date in a project's development process, such as a deadline for completing certain tasks. Milestones help track the status and target dates for all Tasks assigned to a specific Milestone. The most efficient way to assign a Milestone to a Task or Task Group is to add it to the Task List before you add a Task or Task Group. You can also assign Milestones or change previously assigned Milestones at a later time by editing the Milestone attribute.

You can access specific information about Task Lists by accessing the associated information pages. For example, you can view the Task List Summary page, which provides a summary for all items in a Task List, or the Milestone Detail page, which provides information about the Milestones associated with the Task List.
