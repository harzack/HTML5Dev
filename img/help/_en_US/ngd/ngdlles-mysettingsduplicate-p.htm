To handle emails with the same name

  1. In the Enterprise Connect Tree, click a Content Server container, Personal, and then My Settings.

  2. In the Name Generation for E-mails section, go to the When an email with the same name is added to a folder area and do one of the following:

    1. Click Generate a name with a suffix if you want Enterprise Connect to automatically generate a new name for the duplicate email. A new instance of the email, with a suffix to distinguish it from the instance that already resides in the destination, will be added to the destination. In the adjacent list, you must then select one of the following options:

      • Select Prompt to decide if you want Enterprise Connect to ask you if you want to add the email that was automatically renamed to the destination location. This is the default setting.

      • Select Generate name automatically if you want Enterprise Connect to automatically generate a new name for the duplicate email without first confirming with you. You will not be prompted to resolve the naming conflict.

    2. Click Enter a new name if you want to be able to enter a new name for the duplicate email.

  3. Click Submit to apply the changes.

 

Note:
These settings apply to emails, Compound Emails, and email stubs when working with Windows Explorer, Microsoft Outlook, Lotus Notes, and GroupWise.

 
 

Tip:
Click Reset to revert to the most recently saved values for email naming conventions. Click Use Defaults to revert to the settings provided by the administrator.
