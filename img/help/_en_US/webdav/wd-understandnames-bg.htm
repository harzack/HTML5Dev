In WebDAV, every Content Server item has two names. The name of an item as it appears in the Content Server user interface is the item's display name. The name of the item as it appears in the WebDAV URL is the item's URL name.

Some WebDAV client software use display names instead of URL names in the user interface, but a URL name is always used in the URL that accesses the item. By default, the URL names for items are the same as the display names, with the following exceptions:

  • The display names for the Enterprise Workspace, Reports Volume (the location of system-wide LiveReports), and the Categories Volume (the location of custom system-wide categories) are localized (translated) in non-English versions of Content Server, but their URL names are always enterprise, reports, and categories.

  • The display name for a user's Personal Workspace is ~username Home, or the localized equivalent that appears in the Content Server user interface.

  • The URL names for document resources have appropriate extensions added if the extension is not present in the display name. For example, the default URL name for a Microsoft Word document is docdisplayname.doc. Some WebDAV clients need this extension to determine the resource type.
