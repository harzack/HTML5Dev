There are two separate WebDAV clients on Windows XP. While both clients let you drag-and-drop files and folders in Content Server from Windows Explorer, accessing WebDAV from Windows XP is a different experience, depending on which client you are using. The following are the WebDAV clients available on Windows XP:

  • WebDAV redirector, which is a file system driver that allows you to access WebDAV servers with UNC paths, like \\myhost\livelinkdav. To the rest of the system, the WebDAV server looks just like a network drive. You can map drive letters to it, and you can access your WebDAV server from applications that know nothing at all about WebDAV.

  • WebFolders, which is a WebDAV client identical to the one provided in Windows 2000, Office 2003/2007/2010/2013. It allows you to browse your WebDAV server in Windows Explorer or Internet Explorer. Instead of letting you access to the WebDAV server with UNC paths, it uses WebDAV HTTP URLs.

Working with the WebDAV Redirector

The following should be taken into consideration when choosing the WebDAV redirector as your WebDAV client:

  • It provides WebDAV access to all applications.

  • It has bugs that inhibit its compatibility with many WebDAV servers. While WebDAV 3.1.0 includes explicit work-arounds whenever possible, certain sequences of operations may still fail and have to be retried.

  • When you open a document using the WebDAV redirector, the application accesses the document as if it were on a local disk or a network drive and does not know that the document is really in a remote location. This may cause inappropriate actions to be performed, like creating temporary files in the same directory.

  • Microsoft Office applications and some other third-party applications have good built-in WebDAV support. They can communicate with WebDAV servers using the HTTP URLs and know how to behave appropriately in these remote locations. For example, they will reserve a document when you are editing it. When you open a document with the WebDAV redirector, these features are not enabled because the application does not know that the document is located on a WebDAV server.

  • When the WebDAV redirector is used, background operations, such as examining the contents of all the documents in a folder to generate thumbnail views, can cause poor response times because they must be performed across the network.

Working with WebFolders

The following should be taken into consideration when choosing WebFolders as your WebDAV client:

  • Although the WebFolders client does not provide WebDAV capabilities to all applications, it is a mature client with few bugs and integrates very well with Microsoft Office applications.

  • When you open a Microsoft Office document through WebFolders, WebFolders tells the application to open the HTTP URL. Because the application knows that the document is located on a WebDAV server, it will reserve it while you are editing it and will refrain from creating temporary files in the same folder.

  • Browsing using WebFolders is efficient. Because Windows Explorer knows that the folders are remote, it minimizes accesses to the folder contents to reduce network traffic. Operations that it would normally perform in the background, like examining the contents of all the documents in the folder to generate thumbnail views, are not performed by the WebFolders client.

For most purposes, the WebFolder client is more appropriate than the WebDAV redirector for accessing WebDAV.

How a WebDAV client is chosen

When you open a Microsoft Office document for editing using the WebDAV Edit function from Content Server, the application is invoked with the HTTP URL in the same way that it would be invoked from the WebFolders client. The application knows that the document is remote and behaves appropriately in Content Server.

When you click a WebDAV link, activate a WebDAV Folder View from Content Server, or explicitly open a WebDAV location in Windows Explorer, Windows Explorer will use the WebDAV redirector if possible and convert the HTTP URL to a UNC path. You cannot explicitly tell Windows Explorer to use WebFolders instead, but there are situations in which it will choose WebFolders automatically:

  • The WebDAV redirector only works with HTTP (not HTTPS) on the default port 80. When you connect to a folder on a secure server or a server that uses a different port, Windows Explorer uses the WebFolders client.

  • The WebDAV redirector is implemented by the WebClient service in Windows XP. If this service is disabled, the WebDAV redirector is not available for use, and Windows Explorer uses the WebFolders client instead.

Disabling the WebClient Service

The domain administrator can set up a script to disable the WebClient Service for all users. However, some applications may require the WebClient service for complete and proper operation. For example, the Microsoft Sharepoint Web Client explicitly requests the WebDAV redirector to generate its Explorer View of shared folders wherever possible. This view is unavailable if the WebClient service is disabled.
