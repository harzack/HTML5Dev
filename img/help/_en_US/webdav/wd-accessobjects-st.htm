Objects that are accessed with WebDAV are called resources. Resources are assigned URLs, which allow them to be accessed like Web pages. A WebDAV URL points to a Web server, a virtual directory, and a path in Content Server. The following is a typical WebDAV URL:

http://www.mycompany.com/livelinkdav/enterprise/somedoc.doc

Where www.mycompany.com identifies the Web server on which Content Server WebDAV is installed, livelinkdav is the virtual directory that was assigned to WebDAV on installation, and enterprise/somedoc.doc is the path to a specific Content Server Document.
