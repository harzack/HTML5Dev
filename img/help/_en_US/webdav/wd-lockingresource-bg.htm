WebDAV has a locking feature that prevents multiple authors from modifying a document or collection simultaneously. Many WebDAV clients use this feature to automatically lock or reserve a Content Server item as soon as it is opened. The item will remain locked, or reserved, until it is closed.

If a Content Server item is locked by a WebDAV client, it appears in Content Server as a reserved item with a red check mark beside it.

If a WebDAV user leaves a Content Server item locked after editing it, users with the Modify permission can remove the lock by unreserving the item in Content Server.

 

Note:
For more information about Content Server's Reserve and Unreserve functions, see Accessing Documents and Text Documents.
