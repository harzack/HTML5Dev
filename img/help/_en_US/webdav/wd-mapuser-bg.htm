When you attempt to access a WebDAV resource, you may be required to authenticate your identity with a user name and password. WebDAV supports authentication mechanisms that includeBasic/Content Server Authentication and Other Authentication mechanisms.

 

Note:
For further information about what authentication mechanism the WebDAV Resource you are accessing employs, please consult your administrator.

 

Basic / Content Server Authentication

When accessing a WebDAV Resource a dialog prompts you to enter a user name and password. For Basic /Content Server Authentication, you enter the same user name and password as your Content Server user name and password.

Other Authentication Mechanisms

When accessing a WebDAV Resource a dialog prompts you to enter a user name and password. You enter a Principal Name as the user name e.g. me@THEDOMAIN.NET. For this particular example, you would enter the password for the logon 'me' in the 'THEDOMAIN.NET' domain as the Password.

 

Note:
Before you can log in to the WebDAV resource, your Internet Explorer browser must be configured accordingly. For information about configuring Internet Explorer, see the WebDAV Installation and Administration Guide.

 

How Principal Names are mapped to Content Server users

When you logon to WebDAV with a Principal Name that successfully authenticates, WebDAV will match the Principal Name to a Content Server user. WebDAV is configurable by your administrator to use one of two methods to do the matching. These methods include the Same Name and the User Managed mapping methods, respectively.

Same Name

In the Same Name mapping method, the Content Server user name is the same name as the domain logon name of the Principal Name. For example, the Principal Name “me@THEDOMAIN.NET” would map to a Content Server user called “me”.

User Managed

In the User Managed mapping method, each Content Server user manages a list of Principal Names mapped to themselves.

To determine if WebDAV is configured for User Managed mapping:

  1. In Content Server, click Tools from the Global Menu bar, then select Settings.

  2. WebDAV is configured for User Managed mapping if the WebDAV User Mappings tab is visible.

To view and edit the Principal Names mapped to you:

  1. In Content Server, click Tools from the Global Menu bar, then select Settings.

  2. Select the WebDAV User Mappings tab. The page will display all the Principal Names mapped to you.

  3. Click Edit Mappings.

  4. The Delete Mapping page opens for editing the mappings.

To view and edit the Principal Names that are mapped to a user:

 

Note:
This functionality is only available to users with administrative privileges.

 

  1. In Content Server, click Enterprise from the Global Menu bar, then select Users and Groups.

  2. Select a user.

  3. Click Edit for the user, and then select the General tab.

  4. The page will display all the Principal Names mapped to the user.

  5. Click Edit Mappings.

  6. The Delete Mapping page opens for editing the mappings.
