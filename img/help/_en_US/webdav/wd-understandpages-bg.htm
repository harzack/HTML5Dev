WebDAV resources are like Web pages, and the Content Server interface is, essentially, a Web page. However, WebDAV resources differ from Content Server Web pages in that they allow anyone with the proper permissions to open and edit a Content Server item in its native program.

How WebDAV Clients Retrieve Content Server Document Resources

A WebDAV client retrieves Content Server document resources with WebDAV exactly the same way as Web browsers retrieve Web pages. You can use any Web browser to download Content Server items by typing the Content Server item's WebDAV URL in the browser's address field, or by clicking the link on the Content Server item's WebDAV Properties page.

To a Web browser, a document resource looks just like a Web page. Unfortunately, because Web browsers cannot distinguish between document resources and Web pages, the Web browser will not always know that a document is on a WebDAV server. When this information does not get passed along to the client, the item may not automatically open in its native application.

How WebDAV Clients Retrieve Content Server Collection Resources

When the WebDAV module is installed on a Content Server server, the Server becomes a WebDAV collection that contains all the content on the Content Server server. A collection resource in WebDAV refers to the following items: Folders, Compound Documents, Channels, Discussions, Projects, Tasks, and Task lists.

WebDAV collection resources are accessed differently than document resources. If you retrieve a collection by typing its URL in a Web browser, your browser informs you that the item is a WebDAV collection and should be accessed with a WebDAV client.

The Web page displays the same WebDAV URL link that appears on the WebDAV Properties page for the item. Some browsers automatically open the resource with WebDAV when you follow the WebDAV URL link.

With most WebDAV clients, you connect to the WebDAV server using the URL of a collection that contains the items you want to retrieve.
