This section details the sub-tags: EACH, EMAILINFO, ENCSTR, ESCAPECSV, ESCAPEHTML, ESCAPESTR, ESCAPEURL, ESCAPEWR, ESCAPEXML, EVAL, EVEN, and EXCLUDE.

EACH

Syntax

EACH:<sub-tags>

Description

The EACH sub-tag operates on an iterable data type, such as a List or RecArray. The sub-tag accepts a single parameter containing a sub-tag chain to apply to each item in the iterable data. The results are returned as a List.

Examples

For example, [LL_REPTAG_"{'one','two'}" EACH:"UPPER" /] returns {'ONE','TWO'}.

Sub-tags can also be chained, [LL_REPTAG_"{2000,2001}" EACH:"NODEINFO:NAME UPPER" /] returns {'ENTERPRISE','LIVELINK SYSTEM'}.

Related Information

For more information, see LIST and RECARRAY.

EMAILINFO

Syntax

EMAILINFO:<property>

Description

Expects a valid dataID for an email message and returns email data for that node, similar to the view in an Email Folder. The email needs to have been uploaded through the Content Server Explorer/Outlook plugin in order to make use of this tag.

Parameters

Format Description
EMAILINFO:ATTACHMENT
Returns True or False based on the “Attachments” field of the email. If the email has an attachment “true” is returned; if the email does not have an attachment “false” is returned.
EMAILINFO:BCC
Returns the “BCC” field of the email.
EMAILINFO:CC
Returns the “CC” field of the email.
EMAILINFO:FROM
Returns the “From” field of the email.
EMAILINFO:ONBEHALF
Returns the “On Behalf Of” field of the email.
EMAILINFO:RECDATE
Returns the “Received Date” field of the email.
EMAILINFO:SENTDATE
Returns the “Sent Date” field of the email.
EMAILINFO:TO
Returns the “To” field of the email.

ESCAPECSV

Syntax

ESCAPECSV

or

ESCAPECSV:ALLCELLS

Description

Converts text characters so that they are appropriate to use as a cell of data in a CSV export. This means that the following rules are followed:

  • If a comma exists in the text then the group of characters is enclosed in double quotes.

  • If an end of line character exists in the text then the group of characters is enclosed in double quotes.

  • If a double quote character exists in the text then it is converted to two double quotes and the whole group of characters is enclosed in double quotes.

Example

For example, if [LL_REPTAG_1 /] normally returns one "two", three, then [LL_REPTAG_1 ESCAPECSV /] would return: "one ""two"", three". If the optional parameter: ALLCELLS is specified then all cells will be quoted regardless of the contents of the data cell.

ESCAPEHTML

Syntax

ESCAPEHTML

Description

Converts characters such as less than or greater than to HTML escape syntax.

Example

For example, if [LL_REPTAG_1 /] normally returns <hello>, then [LL_REPTAG_1 ESCAPEHTML /] will return: &lt;Hello&gt;

ESCAPESTR

Syntax

ESCAPESTR

Description

Converts characters such as line feeds, carriage returns, and quotes to the standard C language syntax, for example, \r for carriage returns. This sub-tag is essential for any script variable assignments as carriage returns and other characters like quotes will cause an error.

Example

For example, Rows[x] = "[LL_REPTAG_1 ESCAPESTR /]";

ESCAPEURL

Syntax

ESCAPEURL

Description

Converts non-alphanumeric characters into URL hexadecimal sequences, and converts spaces into the plus-sign, +, character.

Example

For example, if a tag normally returns the string: “http://www.bogus.com/somefile”, then [LL_REPTAG_1 ESCAPEURL /] will return: “http%3A%2F%2Fwww%2Ebogus%2Ecom%2Fsomefile”

ESCAPEWR

Syntax

ESCAPEWR

Description

Prevents any WebReport's tag syntax that may be included in the data from being interpreted as valid tag syntax.

ESCAPEXML

Syntax

ESCAPEXML

Description

Converts data to XML compatible format by escaping special characters, such as greater-than, >, less-than, <, ampersand, &, and apostrophe, '.

EVAL

Syntax

EVAL

Description

Processes any Webreport tags found in the main tag.

Example

The following example:

[LL_REPTAG_&p /] // with &p=[LL_REPTAG_'uppercase' UPPER /]

will return “[LL_REPTAG_'uppercase' UPPER /]”

Adding the EVAL sub-tag will evaluate any sub-tags found in the data. In other words,

[LL_REPTAG_&p EVAL /] // with &p=[LL_REPTAG_'uppercase' UPPER /]

will return “UPPERCASE”.

EVEN

Syntax

EVEN

Description

Returns true if the data returned by the main tag is an even number. Returns false otherwise. This can be used with a ROWNUM tag to create code that can alternate from row to row, for example, row colour.

Example

[LL_WEBREPORT_IF "TRUE" == "[LL_REPTAG_ROWNUM ODD /]" /]
   <TR class=oddrow>
[LL_WEBREPORT_ENDIF /]
[LL_WEBREPORT_IF "TRUE" == "[LL_REPTAG_ROWNUM EVEN /]" /]
   <TR class=evenrow>
[LL_WEBREPORT_ENDIF /]

EXCLUDE

Syntax

EXCLUDE:<condition>

Description

This sub-tag operates on a RecArray and returns a new RecArray with the records removed that meet the condition. An exclude condition must contain a column name, operator, and value, and takes the form: “<column_name> <operator> <value>”. Spaces are required between the column_name, operator, and value.

Example

For example, a RecArray containing a USERID column could be filtered to exclude the current user with:

[LL_REPTAG_DATASOURCE EXCLUDE:'"USERID" == "[LL_REPTAG_USERID /]"' /]

This sub-tag is similar to the FILTER sub-tag, except that it excludes records meeting the condition instead of including them.
