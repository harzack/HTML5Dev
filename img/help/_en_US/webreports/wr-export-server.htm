This feature is only available to users logged in as the Content Server admin user. It allows the admin user to define an export location on the server and export the report to that place. If this is set as the default behavior, WebReports users with permission to run reports will be able to execute the report but they will not be able to change the existing server path or define a new export path. The Server File Path should be specified like any other path on the file system. For example C:\export_testing\test_export.xml

For more information, see Exporting to the Server - Parameters and Example.
