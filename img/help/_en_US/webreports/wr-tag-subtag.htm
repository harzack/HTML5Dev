Sub-tags, because they are subordinate to data tags, are always prefixed with: [LL_REPTAG

Any of the data tags can contain a list of formatting sub-tags, separated by spaces, which allow actions to be taken on the actual data returned by the tag before it is inserted into the output.

For example: [LL_REPTAG_1 UPPER /]

Multiple sub-tags can be used for any tag and they are processed from left to right so that the result of each tag becomes the input for the next tag in the list.

Some sub-tags use additional parameters. For example, in the following tag a sub-tag called REPLACE is used which requires 2 parameters, one for the string to be replaced, and one for the string to replace it: [LL_REPTAG_1 REPLACE:<param1>:<param2> /]

[LL_REPTAG_1 REPLACE:apple:pear /]

Any sub-tag parameters are added using the colon, :, symbol. Some parameters may need to use quotes if they include spaces, or colons. For example, if the string to be replaced was “test value”, the syntax would be: [LL_REPTAG_1 REPLACE:"test value":"new value" /]
