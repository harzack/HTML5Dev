This section details the sub-tags: IMEMBERSHIP, INC, INDEXOF, INSPECT, INT, INTLIST, and ISMODULEINSTALLED.

IMEMBERSHIP

Syntax

IMEMBERSHIP

Description

Returns a comma-separated list of Content Server groups of which the user is a direct or indirect member.

INC

Syntax

INC

or

INC:<increment_number>

Description

Increments a numeric value by 1 or by a specified <increment_number>. By default, this sub-tag returns the current tag value plus one. This sub-tag will return errors if the tag does not contain a valid integer. If the optional <increment_number> has been specified, then the current tag plus the <increment_number> is returned.

INDEXOF

Syntax

INDEXOF:<parameter>

Description

For a main tag that returns a list, the INDEXOF sub-tag returns the 1-based index of <parameter>. Returns 0 if not found.

INSPECT

Syntax

INSPECT

Description

The INSPECT sub-tag is useful during development to visualize the hierarchy of a complex data structure. It can be used with LIST, ASSOC, RECORD, and RECARRAY.

Related Information

For more information, see LIST, ASSOC, RECORD, and RECARRAY

INT

Syntax

INT

Description

Explicitly casts data as type integer.

INTLIST

Syntax

INTLIST

Description

The INTLIST sub-tag accepts an integer and returns a list of integers from 1 to that value.

Example

[LL_REPTAG_'5' INTLIST /]

returns: “{1,2,3,4,5}”.

Related Information

For more information, see LIST, TOLIST, and FOR.

ISMODULEINSTALLED

Syntax

ISMODULEINSTALLED

Description

This sub-tag will return true if the module name in the main tag is installed, otherwise it will return false.

Example

For example, [LL_REPTAG_'WebReports' ISMODULEINSTALLED /] will return true, while [LL_REPTAG_'NonExistentModule' ISMODULEINSTALLED /] will return false.
