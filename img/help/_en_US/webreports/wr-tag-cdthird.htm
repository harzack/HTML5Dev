This section details the column data tags COLNAME, CR, and CRLF.

COLNAME

Syntax

[LL_REPTAG_COLNAME<column number> /]

or

[LL_REPTAG_COLNAME++ /]

Description

This tag returns the name of a column from the associated data source, based on the number which is supplied after COLNAME.

This tag is useful when a generic Reportview is being developed. Rather than hard coding a column name into the Reportview, this tag will always provide the correct column name. The default Reportview uses this tag for all column names.

The COLNAME++ variation of this tag will automatically provide the next column number.

Example

An example using [LL_REPTAG_COLNAME<column number> /]: [LL_REPTAG_COLNAME1 /] would return the word: “DATAID” in a WebReport that is dumping the Content Server DTREE using an Auto LiveReport.

An example using [LL_REPTAG_COLNAME++ /]:

  • If the previous COLNAME tag is COLNAME3 then COLNAME++ will return the value of COLNAME4.

  • If COLNAME++ is used again, then the value of COLNAME5 will be returned.

  • If COLNAME1 is then used, COLNAME++ continues from the last column number and will therefore return the value of COLNAME2.

CR

Syntax

[LL_REPTAG_CR /]

Description

Returns a carriage-return character constant.

CRLF

Syntax

[LL_REPTAG_CRLF /]

Description

Returns a carriage-return, linefeed constant.
