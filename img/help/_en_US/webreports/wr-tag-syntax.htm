There are two ways to access information about the tags that can be used in WebReports:

  • You can access the static tag guide that ships as part of the documentation. For more information, see Static Tag Guide.

  • There is a dynamic tag guide that ships as part of the module. This tag guide allows developers to drag and drop tags from the guide to the UI. For information about how to access this dynamically created tag guide, see To Access the Dynamic Tag Guide.

Static Tag Guide

This section describes the different types of tags that can be used in WebReports:

  1. Content Control Tags: these tags are always prefixed with: [LL_WEBREPORT_. They are used to control the way in which the output content is built. For example, it is possible to enable or disable the standard Content Server header and footer. The most important content control tags are used to define which part of the reportview is applied to each row of data source Data.

  2. Column Data Tags: data tags are always prefixed with: [LL_REPTAG. They can further be broken down into Column Data Tags and Static Data Tags:

    1. Column Data Tags: for every column in the associated data source, column data tags exist to reference which pieces of report data are to be included in the output. These tags allow either the column number, for example [LL_REPTAG_2 /], or the column name, for example [LL_REPTAG=DATAID /], to be used.

    2. Static Data Tags: static data tags are used to insert information from Content Server that is not associated with report rows. For example, the date and time, the name of the user running the WebReport, or various URLs that might help with building hyperlinks in the report output. For example, [LL_REPTAG_NEXTURL /]).

  3. Sub-Tags: sub-tags are used to provide formatting or additional functions with any of the [LL_REPTAG ... /] tags. These sub-tags, separated by spaces, allow actions to be taken on the actual data pulled from the data source. Further information about sub-tags can be found in section 6.2.3.

    All tags are case sensitive, always use uppercase, and space sensitive, ensure there is a space before the “/]” at the end of the tag; however, column names can be mixed case.
