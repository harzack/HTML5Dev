This table illustrates the different parameter types which can be selected on the Parameters tab and the effect they have on the end report.

Type
Description
String
The String parameter type will prompt the user with a free text field.
ObjectID
The ObjectID parameter type allows the report developer to specify types of Content Server objects from which the user can select on the prompt screen. One or more items can be selected from the multi-value list, and at least one type must be selected before the developer can browse for a default value.
User
The User parameter type allows the report developer to create a user prompt. The developer can decide whether the user is allowed to browse for Users, Groups or Users and Groups.
Number
The Number parameter type is similar to the String parameter. However, the Number type forces the user to enter a numeric value.
Object
The Object parameter type allows the user to select a Content Server subtype. For example, select Document (144) or Folder (0).
Date
The Date parameter type prompts the user with a date field. The developer can decide whether the time is displayed or not. The developer can also decide whether the user is prompted with a blank date, the current date, or a specific date in time.
Custom
The Custom parameter type allows the developer to browse for a WebReport that is being used to provide a custom prompt. The developer might use this to create a custom pick list, for example.
RSI
If the Records Management module is installed, the developer can browse for a Record Series Identifier, RSI.
