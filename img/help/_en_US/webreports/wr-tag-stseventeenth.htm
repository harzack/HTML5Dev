This section details the sub-tags: TABS, TIMESINCE, TODATE, TOJSON, TOLIST, TRIM, TRUNC and TZOFFSET.

TABS

Syntax

TABS

Description

Returns an oscript representation of the Content Server tabs associated to an object. These are the same tabs you see when you go to any object's Functions menu and then select Properties. This sub-tag expects a valid dataID in the main tag. The tabs will be returned in their proper order as seen in the Content Server interface.

Example

The following example displays the Oscript representation of the General and Specific tabs of a document. Example output of the TABS sub-tag:

{A<1,?,'Name'=X,'URL'='/Livelink971/livelink.exe?func=ll&objId=1389412&objAction=properties&nexturl=%2FLivelink971%2Flivelink%2Eexe%3Ffunc%3Dll%26objid%3D1388531%26objAction%3Dbrowse%26sort%3Dname'>,A<1,?,'Name'=X,'URL'='/Livelink971/livelink.exe?func=ll&objId=1389412&objAction=info&nexturl=%2FLivelink971%2Flivelink%2Eexe%3Ffunc%3Dll%26objid%3D1388531%26objAction%3Dbrowse%26sort%3Dname'>...

Related Information

See the Requests as Data Sources advanced help for how to use the output of this and related tags as the data source to another WebReport.

See the TOJSON sub-tag for how to convert this into a data structure that can be interpreted by JavaScript in the client. The JSON structure can then be parsed and the appropriate HTML can be implemented to display the tabs.

TIMESINCE

Syntax

TIMESINCE:<ref date>:<precision>

Description

This sub-tag expects a date in the main tag and returns the time delta, <ref date> minus the <main tag date>, in an easy-to read-format. If no <ref date> is specified, then the current date is used. Negative values will be returned if the <ref date> is before the <main tag date>.

Dates must be specified in one of the following formats:

  • 'MMM DD YYYY'. For example, 'Jan 12 2010'.

  • 'D/YYYY/MM/DD'. For example, 'D/2009/12/16'.

  • 'DayOfWeek MMM DD HH:MM:SS YYYY'. For example, 'Fri Jan 01 01:04:24 2010''.

  • 'YYYY-MM-DD HH:MM:SS'. For example, '2038-01-19 03:14:07'.

The main tag can also contain an integer representing a time interval in seconds. In this case, the <ref date> parameter is ignored and the value is returned in an easy-to-read format.

A second, and optional, <precision> parameter specifies how many units of precision to return. A default value of 2 is used when the optional <precision> parameter is omitted.

Example

Tag
Output
[LL_REPTAG_'May 1 2010' TIMESINCE /]
11 months, 1 week
[LL_REPTAG_'May 1 2010' TIMESINCE:"May 2 2010" /]
1 day
[LL_REPTAG_'2011-05-01 00:00:00' TIMESINCE:'2011-05-01 01:01:23':3 /]
1 hour, 1 minute, 23 seconds
[LL_REPTAG_'3661' TIMESINCE /]
1 hour, 1 minute
[LL_REPTAG_'3661' TIMESINCE:3 /]
1 hour, 1 minute, 1 second

Related Information

For more information, see DATE and TODATE.

TODATE

Syntax

TODATE:“<format list>”

Description

This sub-tag converts string formatted date information into a Content Server format date. This means the output can be used in logical expressions such as to compare a custom date with a date parameter or possibly the current date. The output of TODATE can also be piped into the DATE sub-tag for further manipulation prior to display.

If the main tag is an integer, and no parameters are passed to TODATE, then the sub-tag returns a date which is the epoch, “1970-01-01 00:00:00”, plus the value of the main tag.

Parameters

Modifier
Description
%%
A percentage sign
%b
The three-character abbreviated month name. For example: “Jan” and “Feb”.
%d
The two-digit day of the month, from 01 to 31. For example, “05” to represent the fifth day of the month.
%m
The two-digit month, from 01 to 12. For example, “05” to represent the month of May.
%p
AM or PM
%y
The two-digit year. For example, “93” to represent 1993 or “05” to represent 2005.
%B
The full month name, for example “January”.
%H
The two-digit hour on a 24-hour clock, from 00 to 23.
%I
The two-digit hour, from 01 through 12.
%M
The minutes past the hour, from 00 to 59.
%S
The seconds past the minute, from 00 to 59.
%Y
The year, including the century, for example 1993.

Example

The following is an example of using TODATE with DATE to get from a custom date format to the Content Server short date format, as defined in the Content Server admin pages:

Imagine you are starting with a custom date stored in the format: “2006-02-15T00:00:00”. Use [LL_REPTAG=MY_DATE_COLUMN TODATE:"%Y-%m-%dT%H:%M:%S" /] to put the date into a string represntation of a Content Server date. At this point, the date will look like “Wed Feb 15 00:00:00 2006” which may or may not be useful.

Now use the DATE sub-tag [LL_REPTAG=MY_DATE_COLUMN TODATE:"%Y-%m-%dT%H:%M:%S" DATE:SHORT /] to change the output so that the date is in the same format as, for example, the modified date in a Content Server folder browse. At this point the date might look like “February 15, 2006”, though this will depend on your system settings in the Content Server admin pages.

Related Information

For output formatting see the DATE and DATEINTEGER sub-tags. For the current date, see DATE and DATETIME tags.

TOJSON

Syntax

TOJSON

Description

For a tag that returns an OSCRIPT type data structure, this tag converts the structure into a JSON (JavaScript Object Notation) structure that can be processed by a JavaScript interpreter. This sub-tag converts OSCRIPT structures as follows:

OSCRIPT Structure OSCRIPT Example JSON Structure JSON Example
LIST {'one','two'} Array [“one”,“two”]
ASSOC A<1,?,'one'=1,'two'=2> Object {“one”:1, “two”:2}
RECORD R<'one'=1,'two'=2> Object {“one”:1, “two”:2}
RECARRAY V{<'fruit','color'><'apple','green'><'banana','yellow'>} Array of Objects [{“fruit”:“apple”, “color”:“green”}, {“fruit”:“banana”, “color”:“yellow”}]

TOLIST

Syntax

TOLIST:<delimiter>

or

TOLIST:<delimiter>:<index>

Description

For a string which has a known delimiter, such as a comma, the string can be converted into a list. This list is similar to a programming array and is in the form: {1,2,3} or {'one','two','three'}. A list in this format can be used by the LIST and RANGE sub-tags; however, the TOLIST sub-tag also supports an optional <index> value that specifies which element in the newly created list to return.

The TOLIST sub-tag can also operate on an Assoc or Record and returns a List containing the values stored in the specified structure. The <delimiter> parameter is ignored when the TOLIST sub-tag is used on such data types.

Example

If the first column of a data table returns the following string: 'apple','orange','pear';, then use the tag: [LL_REPTAG_1 TOLIST:, /] to return the list: {'apple','orange','pear'}

If the optional <index> parameter is also used, the syntax: [LL_REPTAG_1 TOLIST:",":2 /] would return the single element: orange

Related Information

See also LIST and RANGE.

TRIM

Syntax

TRIM

Description

Removes all leading and trailing white space from the specified string. White space is defined as any combination of spaces, form feeds, new lines, carriage returns, and horizontal or vertical tabs.

TRUNC

Syntax

TRUNC:<format>

Description

Returns a truncated string.

Parameters

Format Description
TRUNC:<width> Truncates the field to the number of characters specified in the <width>, starting from the first character.
TRUNC:<width>:<continuation characters> Set <continuation characters> which will be appended to the return string to show that the data has been truncated. The length of the return string, including the <continuation characters>, will not exceed the value of the <width> argument, unless the <width> value is less than the length of the <continuation characters>.
TRUNC:<width>:<continuation characters>:APPEND Append <continuation characters> to the return string if the data has been truncated.

Examples

[LL_REPTAG_'123456789' TRUNC:5 /] returns 12345

[LL_REPTAG_'123456789' TRUNC:5:"..." /] returns 12...

[LL_REPTAG_'123456789' TRUNC:5:"...":APPEND /] returns 12345...

TZOFFSET

Syntax

TZOFFSET

Description

Applies a time zone offset to dates. Date tags such as [LL_REPTAG_DATETIME /] and date columns [LL_REPTAG=CREATEDATE /] return the time at the server. The TZOFFSET sub-tag can be used to apply the appropriate time zone offset for the specific user viewing the output.
