This section details the column data tags: URLDOMAIN, URLPREFIX, URLPREFIXFULL, URLPROTOCOL, USERFULLNAME, USERID and USERNAME.

URLDOMAIN

Syntax

[LL_REPTAG_URLDOMAIN /]

Description

Returns the domain name used to access the instance. For example: www.theDomain.com.

This data is only available if the WebReport is run from the Content Server GUI. The tag will resolve to an empty string if the WebReport is run from an API call, such as via WRTrigger, or from a workflow instance.

Related Information

For more information, see URLPROTOCOL, URLPREFIX, and URLPREFIXFULL.

URLPREFIX

Syntax

[LL_REPTAG_URLPREFIX /]

Description

URL required to call the Content Server CGI. For example: Livelink/livelink.exe

This can be used to construct a hyperlink in the WebReport that runs a Content Server command. For example, open a form for the data in the WebReport.

This data is only available if the WebReport is run from the Content Server GUI. The tag will resolve to empty string if the WebReport is run from an API call, such as WRTrigger, or from a workflow instance.

Related Information

For more information, see URLPROTOCOL, URLDOMAIN, and URLPREFIXFULL.

URLPREFIXFULL

Syntax

[LL_REPTAG_URLPREFIXFULL /]

Description

Returns the full URL required to call the Content Server CGI including protocol and domain. For example: http://www.theDomain.com/<instance_name>/cs.exe. This data is only available if the WebReport is run from the Content Server GUI. If the WebReport is run through an API call, such as through WRTrigger or a workflow instance, then the tag will attempt to retrieve the data from the Content Server Home Page URL setting, as defined in the Configure Notification administration page.

Related Information

See also URLPROTOCOL, URLPREFIX, and URLDOMAIN.

URLPROTOCOL

Syntax

[LL_REPTAG_URLPROTOCOL /]

Description

Returns the protocol used in the request to access the WebReport. For example: http:// or https://

This data is only available if the WebReport is run from the Content Server GUI. A default value of http:// is returned if the protocol cannot be determined from the request. For example, if the WebReport is run through an API call, such as through WRTrigger, or a workflow instance.

Related Information

See also URLPREFIX, URLPREFIX, and URLPREFIXFULL.

USERFULLNAME

Syntax

[LL_REPTAG_USERFULLNAME /]

Description

Full name of the user executing the WebReport.

USERID

Syntax

[LL_REPTAG_USERID /]

Description

User ID for the user executing the WebReport. If the RunAs feature is set, this tag will return the user ID of the RunAs user. Use the [LL_REPTAG_ACTUALUSERID /] tag to obtain the actual user ID.

Example

For example, if User 1 runs a WebReport that has RunAs set to User2, the ACTUALUSERID tag will return the ID of User1, whereas the USERID tag will return the ID of User2.

USERNAME

Syntax

[LL_REPTAG_USERNAME /]

Description

Username of the user executing the WebReport.
