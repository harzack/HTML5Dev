This section details the sub-tags: DATE, DATEDEC, DATEDIFF, DATEINC, DATEINTEGER, DEC, DECODE, DECSTR, DEF, DISTINCT, DIVIDE and DMEMBERSHIP.

DATE

Syntax

DATE

or

DATE:<parm>

Description

This sub-tag converts date formatted data into a user defined format. The output can be formatted using any of the modifiers below. For example, [LL_REPTAG=CREATEDATE DATE:"%a - %b %d, %Y" /] will provide a date similar to: “Mon - Jan 31, 2004”.

This sub-tag supports DATETIME, DATE, and Content Server formatted dates. An example of a Content Server formatted date is: “D/2006/3/27:0:0:0”

Parameters

Modifier
Description
DATE:INPUT
 Input date format as specified on the admin pages.
DATE:SHORT
 Short date format as specified on the admin pages.
DATE:LONG
 Long date format as specified on the admin pages.
DATE:JS
DATE:JAVASCRIPT
Format that can be used with Javascript. For example, var now = new Date("[LL_REPTAG_DATETIME DATE:JS /]");
DATE:%%
 A percentage sign.
DATE:%a
 The three-character abbreviated weekday name, for example, Mon, Tue, and so on.
DATE:%b
 The three-character abbreviated month name, for example, Jan, Feb, and so on.
DATE:%d
 The two-digit day of the month, from 01 through 31.
DATE:%j
 The three-digit day of year, from 001 through 366.
DATE:%m
 The two-digit month, from 01 through 12.
DATE:%p
 AM or PM
DATE:%w
 The one-digit weekday, from 1 through 7, where 1= Sunday.
DATE:%y
 The two-digit year, for example, 93.
DATE:%A
 The full weekday name, for example, Monday.
DATE:%B
 The full month name, for example, January.
DATE:%H
 The two-digit hour on a 24-hour clock, from 00 through 23.

 The two-digit hour, from 01 through 12.
DATE:%M

DATE:%Y

Related Information

For manipulating custom date formats see TODATE. For the current date see DATE and DATETIME tags.

DATEDEC

Syntax

DATEDEC:<value>:<units>

Description

Decrements the date returned by the main tag by the value and units specified. Valid date formats for the main tag include [LL_REPTAG_DATETIME /], [LL_REPTAG_DATE /], and Content Server-formatted dates such as “Sat Apr 16 00:00:00 2011”, “Feb 20 2010” and “D/2012/12/25:00:00:00”. The <value> parameter should be a positive integer.

Parameters

The <units> parameter will be one of the following:

Units
Description
DAYS
This option will decrement the date by <value> number of days.
WEEKS
This option will decrement the date by <value> number of weeks.
SECONDS
This option will decrement the date by <value> number of seconds.

Example

[// Here DATETIME and DATE tags return Mon Apr 11 00:00:00 2011
[LL_REPTAG_DATETIME DATEDEC:2:DAYS /] [// Returns Sat Apr 09 00:00:00 2011
[LL_REPTAG_DATE DATEDEC:2:WEEKS /] [// Returns Mon Mar 28 00:00:00 2011
[LL_REPTAG_'Sat Apr 16 10:23:32 2011' DATEDEC:86400:SECONDS /] [// Returns Fri Apr 15 10:23:32 2011
[LL_REPTAG_'D/2012/12/25:00:00:00' DATEDEC:4:WEEKS /] [// Tue Nov 27 00:00:00 2012

[// Here dueDate is a column from the data source and returns the date: Tue May 24 00:00:00 2011
[LL_REPTAG=dueDate DATEDEC:6:DAYS /] [// Returns Wed May 18 00:00:00 2011

An empty string is returned if the main tag is empty. An error message will be returned if the main tag is not a valid date.

Related Information

See also DATE, DATEDIFF, DATEINC, DATEINTEGER, TODATE, and TIMESINCE.

DATEDIFF

Syntax

DATEDIFF:<reference date>:<units>

or

DATEDIFF

or

DATEDIFF:<units>

or

DATEDIFF:<reference date>

Description

Returns the difference between the date in the main tag and the reference date, in other words, the <reference date> minus the <main tag date>. If the reference date is omitted then the server's current date and time are used.

An optional <units> parameter will convert the seconds to minutes, hours, days, and so on. If no <units> parameter is provided, the difference will be returned in days by default. If the measured difference can not be specified in a round number using the specified unit then a 2 digit decimal value is returned. For example, if the difference between two dates is 75 minutes and the parameter HOURS has been specified, then the value 1.25 will be returned.

Valid date formats for the main tag include [LL_REPTAG_DATETIME /], [LL_REPTAG_DATE /], and Content Server formatted dates. Examples of Content Server formatted dates include: “Sat Apr 16 00:00:00 2011”, “Feb 20 2010”, and “D/2012/12/25:00:00:00”. The <reference date> parameter should also be a valid Content Server formatted date.

The <units> parameter can be one of the following: SECONDS, MINUTES, HOURS, DAYS, WEEKS, MONTHS, YEARS, WEEKDAYS. The WEEKDAYS unit returns days but only non-weekend days. Thus, if DAYS normally returns 10 but spans two weekends, WEEKDAYS would only return 6. If the lower date starts on a non-weekday, the calculation starts from the following Monday. If the higher date ends on a non-weekday, then the calculation ends on the preceding Friday.

Examples

Tag
Result
[LL_REPTAG_"D/2010/5/30:0:0:0" DATEDIFF:"D/2011/5/30:0:0:0":DAYS /] 365
[LL_REPTAG=modifyDate DATEDIFF:HOURS /] Number of hours between the modified date, as returned from the DB, and today's date.
[LL_REPTAG=modifyDate DATEDIFF:"[LL_REPTAG_&dateparm /]":DAYS /] The number of days between the modified date and a passed date parm.

DATEINC

Syntax

DATEINC:<value>:<units>

Description

Increments the date returned by the main tag by the value and units specified. Valid date formats for the main tag include [LL_REPTAG_DATETIME /], [LL_REPTAG_DATE /], and Content Server-formatted dates such as “Sat Apr 16 00:00:00 2011”, “Feb 20 2010” and “D/2012/12/25:00:00:00”. The <value>> parameter should be an integer greater than zero.

Parameters

The <units> parameter will be one of the following:

Units
Description
DAYS
 This option will increment the date by <value> number of days.
WEEKS
 This option will increment the date by <value> number of weeks.
SECONDS
 This option will increment the date by <value> number of seconds.
WEEKDAYS
 This option will increment the date by <value> number of days, excluding weekends.

Example

[// Here DATETIME and DATE tags return Mon Apr 11 00:00:00 2011
[LL_REPTAG_DATETIME DATEINC:2:DAYS /]     [// Returns Wed Apr 13 00:00:00 2011
[LL_REPTAG_DATE DATEINC:2:WEEKS /]        [// Returns Mon Apr 25 00:00:00 2011

[LL_REPTAG_'Sat Apr 16 10:23:32 2011' DATEINC:86400:SECONDS /]   [// Returns Sun Apr 17 10:23:32 2011
[LL_REPTAG_'Feb 20 2010' DATEINC:10:WEEKDAYS /]     [// Returns Fri Mar 05 00:00:00 2010
[LL_REPTAG_'D/2012/12/25:00:00:00' DATEINC:4:WEEKS /]       [// Tue Jan 22 00:00:00 2013

[// Here dueDate is a column from the data source and returns the date: Tue May 24 00:00:00 2011
[LL_REPTAG=dueDate DATEINC:60:DAYS /]               [// Returns Sat Jul 23 00:00:00 2011

An empty string is returned if the main tag is empty. An error message will be returned if the main tag is not a valid date.

Related Information

For more information, see DATE, DATEDEC, DATEDIFF, DATEINTEGER, TODATE, and TIMESINCE.

DATEINTEGER

Syntax

DATEINTEGER

Description

This sub-tag operates on a date and returns the number of seconds since the epoch: 1970-01-01 00:00:00. Due to the 32-bit integer limit of Content Server, this tag only works with dates before 2038-01-19 03:14:07.

Related Information

For more information, see TODATE.

DEC

Syntax

DEC

or

DEC:<decrement number>

Description

Decrements a numeric value by 1 or a specified decrement number. By default this sub-tag returns the current tag value minus one and errors if the tag does not contain a valid integer. If the optional decrement number has been specified, then the current tag minus the decrement number is returned.

DECODE

Syntax

DECODE:<if1>:<then1>

or

DECODE:<if1>:<then1>:<if2>:<then2>:...:<else>

Description

Performs a value by value substitution, similar to the Oracle DECODE statement.DECODE's parameters consist of pairs of strings that specify a match and a replacement value. There must be at least one pair of strings specified, and it can have as many as 98 pairs in total.

The <else> string is optional.

The DECODE sub-tag looks to see if the data returned by the data tag matches any of the <if> strings specified in the parameter list. If it finds a match, it replaces the data with the <then> string. If it does not find a match, it will replace the data with the optional <else> string, if specified.

For example, if a data tag, [LL_REPTAG=STATUS /], is returning numeric strings that indicate a status such as 0, 1, 2, and so on. It would be useful to present these as meaningful text, such as “Completed”, “Pending Review”, and “Open”. The following DECODE statement will acheive this: DECODE:"0":"Completed":"1":"Pending Review":"2":"Open":"Unknown"

The final “Unknown” parameter means that if none of the values 0, 1 or 2 is found, the string “Unknown” is substituted.

DEF

Syntax

DEF:<value>

Description

If the value returned by the main tag is undefined, the value specified here will be used.

DISTINCT

Syntax

DISTINCT:<value>

Description

The DISTINCT sub-tag operates on a list, and removes any duplicate values.

Example

[LL_REPTAG_'{1,1,2,3,3}' DISTINCT /]

returns: “{1,2,3}”

DIVIDE

Syntax

DIVIDE:<value>

Description

Performs a division operation. Divides an integer returned in the main tag by the <value> specified. Other sub-tags can be used in conjunction with this tag, like ROUND to provide greater precision.

Example

For example: [LL_REPTAG_"10" DIVIDE:2 ROUND:2 /] will resolve to 5.00.

Related Information

For more information, see ADD, MULTIPLY, MODULUS, ROUND, SUBTRACT, and TIMESINCE.

DMEMBERSHIP

Syntax

DMEMBERSHIP

Description

Returns a comma-separated list of Content Server Groups of which the user is a direct member.
