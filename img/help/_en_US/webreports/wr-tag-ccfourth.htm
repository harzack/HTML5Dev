This section details the content control tags: FOR, FORCEUTF8, and FORMPARSEOFF.

FOR

Syntax

[LL_WEBREPORT_FOR VAR:<varName> DATA:<collection> /]

     [LL_REPTAG_%<varName> CURRENTVAL /]

[LL_WEBREPORT_ENDFOR /]

Description

This tag is used to loop over a collection of data. Valid collections include Lists, RecArrays, Assocs, Records, and Strings.

Parameters

The tag has two required parameters:

  • DATA:<collection>: The collection to iterate, which can be a List, RecArray, Assoc, Record, or String.

  • VAR:<varName>: The looping variable name. This variable becomes visible in the looping block. A second variable named <varName>_index is also defined. The <varName>_index is the index of the element.

Example

To output the elements of a list:

[LL_WEBREPORT_FOR DATA:{'apple','pear','orange'} VAR:value /]
     [LL_REPTAG_%value_index CURRENTVAL /] - [LL_REPTAG_%value CURRENTVAL /]
[LL_WEBREPORT_ENDFOR /]

returns:

1 - apple
2 - pear
3 - orange

FOR loops can also contain other FOR loops. For example, the following will output each field in each row of the data source:

[LL_WEBREPORT_FOR DATA:[LL_REPTAG_DATASOURCE /] VAR:i /]
   [LL_WEBREPORT_FOR DATA:[LL_REPTAG_%i CURRENTVAL TOLIST /] VAR:j /]
      [LL_REPTAG_%j CURRENTVAL /]
   [LL_WEBREPORT_ENDFOR /]
[LL_WEBREPORT_ENDFOR /]

Optionally include the [LL_WEBREPORT_BREAK /] subtag to break out of a FOR loop. For example:

[LL_WEBREPORT_FOR DATA:[LL_REPTAG=DataID CAT:RAW /] VAR:Category /]
   [// Loop through the category data until we find the Fruit category
   [LL_WEBREPORT_IF "(LL_REPTAG_%Category CURRENTVAL ASSOC:DisplayName /)" == "Fruit" /]
      [// Display something
      This document contains the Fruit category.
      [// Break out of the FOR loop
      [LL_WEBREPORT_BREAK /]
   [LL_WEBREPORT_ENDIF /]
[LL_WEBREPORT_ENDFOR /]

Related Information

For more information see DATASOURCE, CURRENTVAL, and TOLIST:delimiter.

FORCEUTF8

Syntax

[LL_WEBREPORT_FORCEUTF8 /]

Description

This tag forces a UTF-8 Byte Order Marker (BOM) to be added to the beginning of the output from the WebReport. The BOM is made of the hex characters: #FE #BB #BF. Some client applications may require this marker to correctly handle UTF-8 content.

FORMPARSEOFF

Syntax

[LL_WEBREPORT_FORMPARSEOFF /]

Description

This tag is only used when working with WebReports in forms, through the WR Power View option in form templates. When this tag is included in a WR Power View, the normal WebForms tag parsing code will not be invoked and instead a simplified parser, built into the WebReports engine, is used. This simple parser will handle substitution of WebForms tag syntax, such as [LL_FormTag_... /] type tags, but will not handle any LOOP tags or any form tags associated with WebForms loops.

This tag also disables the WebForms feature that automatically sets HTML SELECT and CHECKBOX form fields with default values. In some cases this feature of WebForms can be problematic and may actually fail when non-standard HTML is used for fields like SELECT statements. In these situations the FORMPARSEOFF tag should be used to prevent any conflict with the WebForms code.

Related Information

For more information see WR Power View.
