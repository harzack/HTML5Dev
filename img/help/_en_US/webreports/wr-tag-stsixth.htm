This section details the sub-tags FETCHURL, FIELDS, FILTER, and FORMAT, FROMJSON, and FROMLIST.

FETCHURL

Syntax

FETCHURL

Description

Returns the HTML content of a URL.

FETCHURL can only access non-secured external web resources, as the underlying component does not have the ability to authenticate. For this reason, requests to Content Server instances are not supported.

 

Note:
FETCHURL requests are performed synchronously, so multiple requests can affect the performance of a WebReport. The number of FETCHURL requests should be limited as much as possible.

 

FIELDS

Syntax

FIELDS

Description

This tag accepts a RecArray, Record, or Assoc and returns the fieldnames of the structure as a List.

Related Information

For more information, see VALUES.

FILTER

Syntax

FILTER:<condition>

Description

This sub-tag operates on a RecArray and returns a new RecArray with the records removed that do not meet the condition. A filter condition must contain a column_name, operator, and value. Spaces are required between the <column_name>, <operator>, and <value>.

This sub-tag is similar to the EXCLUDE sub-tag, except that it includes records meeting the condition instead of excluding them.

Example

For example, a RecArray containing a USERID column could be filtered for the current user with:

[LL_REPTAG_DATASOURCE FILTER:'"USERID" == "[LL_REPTAG_USERID /]"' /]

Related Information

For more information, see:

  • The list of Operators under Logical Expressions.

  • The EXCLUDE sub-tag.

FORMAT

Syntax

FORMAT

or

FORMAT:<rule>

Description

This sub-tag automatically formats the main tag into a readable format, when possible. Various checks are made on the context and data to determine how to format the data. The sub-tag is useful whenever automatic formatting is desired. It has no effect if a formatting rule cannot be determined.

The sub-tag also accepts an optional parameter to force the formatting rule. This is useful when a data type is known or a specific rule is desired. Valid formatting rules include USER, GROUP, DATE, SUBTYPE, PERMISSION, and BOOLEAN.

Example

Tag
Return Value
[LL_REPTAG_'true' FORMAT /]
  
[LL_REPTAG=SUBTYPE FORMAT /]
Folder
[LL_REPTAG_'144' FORMAT:SUBTYPE /]
Document
[LL_REPTAG=UPERMISSIONS FORMAT /]
See, See Contents
[LL_REPTAG_'12321' FORMAT:USER /]
John Doe

FROMJSON

Syntax

FROMJSON

Description

Decodes a JSON string to an Assoc or List.

Example

[LL_REPTAG_'{"firstname":"John","lastname":"Doe"}' FROMJSON  /]

returns: A<1,N,'firstname'='John','lastname'='Doe'>

Example

A specific use of the tag might be to decode a JSON string from an AJAX POST request. For example, if a request is made with: &names=["John","Bob","Sally"], the value of the parameter can be fetched with the [LL_REPTAG_&names /] tag. To get the second element, use FROMJSON and LIST:

[LL_REPTAG_&names FROMJSON LIST:2 /] [// returns "Bob"

Related Information

For more information, see ASSOC, LIST, and TOJSON.

FROMLIST

Syntax

FROMLIST:<delimiter>

Description

Accepts a List and returns a catenation of the elements. An optional delimiter can be specified to override the default space and comma “,” delimiter.

Example

[LL_REPTAG_"{'one','two','three'}" FROMLIST:";" /]

returns: one;two;three.

Related Information

For more information, see LIST.
