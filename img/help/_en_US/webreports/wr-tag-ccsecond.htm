This section details the content control tags CALL and COMPILER.

CALL

Syntax

[LL_WEBREPORT_CALL /]

or

[LL_WEBREPORT_CALL NAME:<script name> PARM:<arg1>:<arg2>:<argN> /]

Description

Calls a server side Oscript function defined by STARTSCRIPT and ENDSCRIPT tags. The particular script to call is identified by the <script name> parameter. The script may optionally be passed a list of arguments seperated by semi-colons. Note that server side scripting must be enabled by an administrator at the server level and for each WebReport each time a new reportview is added. Scripts are rendered inactive unless enabled. Server side scripting is powerful but must be used with great care. Users should read all the related documentation before using server side scripting.

Related Information

See Server Side Scripting for a summary.

COMPILER

Syntax

[LL_WEBREPORT_COMPILER /]

or

[LL_WEBREPORT_COMPILER RUNTIMETAGS:TRUE /]

or

[LL_WEBREPORT_COMPILER RUNTIMETAGS:FALSE /]

Description

Determines if constants and parameters in the report should be evaluated at runtime for tags. This is required if any tags are defined in the constants tab. It is effectively the same as having an EVAL sub-tag on each parameter and constant.

Example

For example, having [LL_REPTAG_&p /] in a report with &p=[LL_REPTAG_"literal" UPPER /] in the URL will output:

[LL_REPTAG_"literal" UPPER /]

If you enable RUNTIMETAGS with [LL_WEBREPORT_COMPILER RUNTIMETAGS:TRUE /], the output will be:

LITERAL

Related Information

See the EVAL sub-tag.
