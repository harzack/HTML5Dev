This section details the sub-tags: UNESCAPESTR, UNESCAPEURL, UPPER, URLPARSE, URLTOPOST, USERACTION, USERINFO, USERINGROUP, USERPREF and USERPREFACTION.

UNESCAPESTR

Syntax

UNESCAPESTR

Description

Converts escaped characters to their equivalents. For example, a \r would be converted to an actual carriage return character.

Example

[LL_REPTAG=NAME UNESCAPESTR /]

UNESCAPEURL

Syntax

UNESCAPEURL

Description

Unescapes a previously escaped URL. The plus-sign character (+) will be converted to a space. In addition, hexadecimal non-alphanumeric characters, represented by %<xx>, where <xx> is hexadecimal notation, will be converted to their ASCII equivalent.

Example

[LL_REPTAG_"http%3A%2F%2Fjoe%2Efred%2Ecom" UNESCAPEURL /] returns “http://joe.fred.com”.

UPPER

Syntax

UPPER

Description

Sets all text in the field to upper case.

URLPARSE

Syntax

URLPARSE

Description

Parses a URL and returns the anchor, domain, parameters as an ASSOC, path, port, protocol, and query as an ASSOC.

The URLPARSE sub-tag can be combined with the [LL_REPTAG_MYURL /] data tag to fetch attributes of the current URL.

Example

[LL_REPTAG_"http://localhost:8080/livelink/livelink.exe?func=llworkspace#anchor" URLPARSE /]

returns:

A<1,?,'anchor'='anchor','domain'='localhost','parameters'=A<1,?,'func'='llworkspace'>,'path'='/livelink/livelink.exe','port'='8080','protocol'='http','query'='func=llworkspace'>

Example: The following tag in a standard WebReport, [LL_REPTAG_MYURL URLPARSE ASSOC:"parameters" ASSOC:"objaction" /] returns “RunReport”.

Description

For more information, see ASSOC and URLTOPOST.

URLTOPOST

Syntax

URLTOPOST

Description

For any valid URL, converts the GET form URL into a POST request by creating a series of appropriately named FORM fields with corresponding values. If the URL contains a prefix, prior to any parameters, the prefix is abandoned and only the parameters are converted into form fields.

Example

For example, the following URL: /Livelink971/livelink.exe?func=ll&objId=157220&objAction=RunReport&reqcacheid=1005851343 converts into the following lines of HTML:

<INPUT TYPE="HIDDEN" NAME="func" VALUE="ll">
<INPUT TYPE="HIDDEN" NAME="objId" VALUE="157220">
<INPUT TYPE="HIDDEN" NAME="objAction" VALUE="RunReport">
<INPUT TYPE="HIDDEN" NAME="reqcacheid" VALUE="1005851343">

Related Information

For more information, see URLPARSE.

USERACTION

Syntax

USERACTION:<operation>

Description

This sub-tag provides the ability to manipulate users and groups in large volumes at the server. Users running a report containing this sub-tag are only able to perform operations with the same level of permissions/privileges as through the regular user interface.

Parameters

Action
Description
USERACTION:ADD:<group or user ID>
Adds a user or group to the group returned from the main tag.
USERACTION:CREATEGROUP
Creates a new group with the name returned from the main tag. Supports an optional parameter, LEADERID:<userID> to set the group leader.
USERACTION:CREATEUSER:<password>:<department>
Creates a new user with default permissions, Login and Public Access, using the name returned from the main tag.
USERACTION:DELETE
Deletes the user or group returned by the main tag.
USERACTION:DEPARTMENT:<group ID or name>
Changes the Department group for the user returned by the main tag.
USERACTION:GROUPLEADER:<group or user ID>
Sets the user or group as group leader of the group returned from the main tag.
USERACTION:PRIVILEGE:<privilege>:TRUE/FALSE
Sets a privilege value for the user returned by the main tag. Privilege values correspond to the checkboxes in the Privileges section on the Edit User page. This sub-tag accepts the following values for <privilege>:
•

LoginEnabled: corresponds to the Log-in enabled checkbox.

•

PublicAccessEnabled: corresponds to the Public Access enabled checkbox.

•

CanCreateModifyUsers: corresponds to the Can create/modify users checkbox.

•

CanCreateModifyGroups: corresponds to the Can create/modify groups checkbox.

•

UserAdministrationRights: corresponds to the User administration rights checkbox.

•

SystemAdministrationRights: corresponds to the System administration rights checkbox.

This sub-tag will only work for users with the User Administration privilege, and users will only be able to set privileges that they have themselves.
USERACTION:REMOVE:<group or user ID>
Removes a user or group from the group returned in the main tag.
 

Note:
You can't remove a user from a group whose Department is the same group. In Content Server a user must be part of a Department.

 
USERACTION:RENAME:<name>
Renames the user or group.
USERACTION:UNSETGROUPLEADER
Unsets the group leader of the group returned from the main tag.

Example

[LL_REPTAG=GROUPID USERACTION:ADD:38832 /] adds the user with ID=38832 to the group returned in the GROUPID column.

Example: [LL_REPTAG=newname USERACTION:CREATEGROUP:LEADERID:4526 /] creates a new group called whatever is in the “newname” column and sets the leader of that group to be the user “4526”.

Example: [LL_REPTAG_'jbloggs' USERACTION:CREATEUSER:'aardv4rk':'DefaultGroup' /] creates a new user with the username “jbloggs”, the password “aardv4rk” and the department “DefaultGroup”.

Example: [LL_REPTAG_'1234' USERACTION:GROUPLEADER:9876 /] sets the user or group with ID=9876 to be the group leader of the group with ID=1234.

Example: [LL_REPTAG_'1234' USERACTION:UNSETGROUPLEADER /] unsets the group leader of the group with ID=1234.

USERINFO

Syntax

USERINFO:<format>

Description

For a valid Content Server user ID, user login name, group ID, or group name, this sub-tag will return user or group metadata corresponding to the <format> parameter that has been provided.

Parameters

Format
Description
USERINFO
Returns an Assoc containing metadata for the user or group, with the exception of the UserPWD, password, field.
Can be chained with the ASSOC sub-tag to retrieve specific information not available from the other USERINFO tags.
USERINFO:CONTACT
Returns the contact information for the user.
USERINFO:EDITABLE
Returns TRUE if the user running the WebReport has permission to edit the user or group in the main tag, else it will return FALSE.
USERINFO:FIRSTNAME
Returns the user's first name.
USERINFO:FULLNAME
Returns the full name of the user, using the format which has been globally defined for the Content Server system, for example “Joseph R. Smith”.
USERINFO:GROUPID
Returns the ID of the user's base group.
USERINFO:GROUPNAME
Returns the name of this user's base group.
USERINFO:ID
Returns the ID of the user or group.
USERINFO:LASTNAME
Returns the user's last name.
USERINFO:LEADERID
If the main tag contains a group ID or name, the user ID of the user who is the leader of the group is returned.
USERINFO:MAILADDRESS
Returns the user's email address.
USERINFO:MEMBERS
Returns the group's members as a RecArray.
USERINFO:MIDDLENAME
Returns the user's middle name.
USERINFO:NAME
Returns the login ID of a user or the name of a group.
USERINFO:PRIVILEGES
Returns an Assoc containing boolean values corresponding to the user's privileges.
Can be chained with the ASSOC sub-tag to a value for a specific privilege. For a list of available values for the optional <privilege> parameter, see the Parameters section of the USERACTION sub-tag.
This sub-tag will only return data if the WebReport is run by a user with User Administration privileges.
USERINFO:PWS
Returns the objectID of the Personal Workspace of the user specified in the main tag.
The USERINFO:PWS sub-tag can return a link to a user's Personal Workspace if it is used with the LLURL sub-tag.
USERINFO:TITLE
Returns the user's title.

Examples

[LL_REPTAG_'1000' USERINFO ASSOC:Gender /]

Example: [LL_REPTAG_USERID USERINFO:PRIVILEGES ASSOC:UserAdministrationRights /] will return TRUE or FALSE based on the value of the “UserAdministrationRights” boolean.

Example: [LL_REPTAG_USERID USERINFO:PWS LLURL:BROWSE /] will provide a URL to access the Personal Workspace of the user executing the WebReport.

Related Information

For more information, see ASSOC, LLURL, and the Parameters section of USERACTION.

USERINGROUP

Syntax

USERINGROUP

Description

Returns TRUE if the current user, the user running the WebReport, is a member of the group returned by the main tag. If the main tag is not associated with a valid group ID, then FALSE is returned.

Example

[LL_WEBREPORT_IF "[LL_REPTAG=ID USERINGROUP /]" == "TRUE" /]
   I AM IN GROUP: [LL_REPTAG=NAME /]
[LL_WEBREPORT_ENDIF /]

USERPREF

Syntax

USERPREF:<tab>:<setting>

Description

For a valid Content Server user ID this sub-tag will return the user's settings found when you select Settings from the Tools menu.

Some sub-tags return an Oscript Assoc and can be further manipulated using the TOJSON sub-tag.

Parameters

Format
Description
GENERAL
Returns an Oscript Assoc containing the values of all the settings from the General tab of the My General Settings page.
GENERAL:DFTSTARTPAGE
Returns the user's setting in the Default Start Page field on the General tab of the My General Settings page.
GENERAL:ENHANCEDKYBD
Returns TRUE or FALSE based on the user's setting in the Enhanced Keyboard Accessibility Mode field on the General tab of the My General Settings page.
GENERAL:LANGUAGE:CODE
Returns the code of the user's preferred language. Will default to the system default language if not specified.
Examples of return values include “_en_US” and “_fr”.
GENERAL:LANGUAGE:NAME
Returns the display name of the user's preferred language. Will default to the system default language if not specified.
Examples of return values include: “English ( United States )” and “French”.
GENERAL:METADATALANGUAGE:CODE
Returns the code of the user's preferred metadata language as set in the Metadata Language field on the General tab of the My General Settings page. Will default to the system default metadata language if not specified.
Examples of return values include “_en_GB” and “_fr”.
GENERAL:METADATALANGUAGE:NAME
Returns the display name of the user's preferred metadata language as set in the Metadata Language field on the General tab of the My General Settings page. Will default to the system default metadata language if not specified.
Examples of return values include “English ( United Kingdom )” and “French”.
GENERAL:MODDUR
Returns the number of days set in the Modified Indicator Duration field on the General tab of the My General Settings page.
GENERAL:NAVSTYLE
Returns the user's setting in the Navigation Style field on the General tab of the My General Settings page. This will return one of three values:
•

a blank string, if no preference is selected

•

trail

•

dropdown

GENERAL:NEWDUR
Returns the number of days set in the New Indicator Duration field on the General tab of the My General Settings page.
GENERAL:SBLOCATION
Returns left or right based on the user's setting in the Sidebar Location field on the General tab of the My General Settings page. Only available in Content Server 10.0 and higher.
GENERAL:SIDEBAR
Returns TRUE or FALSE based on the user's setting in the Show Sidebar in Browse field on the General tab of the My General Settings page. Only available in Content Server 10.0 and higher.
GENERAL:SHOWITEMDESC
Returns TRUE or FALSE based on the user's setting in the Show Item Descriptions on Detail View field on the General tab of the My General Settings page.
COLORS
Returns an Oscript Assoc containing the values of all the settings from the Color tab of the My Color Settings page.
COLORS:ROW1
Returns the hexidecimal value from the Row1 field on the Color tab of the My Color Settings page.
COLORS:ROW2
Returns the hexidecimal value from the Row2 field on the Color tab of the My Color Settings page.
DISCUSSION

DISCUSSION:DFTPERIODFROM
Returns the user's setting from the Default Period From field on the Discussion tab of the My Discussion Settings page.
DISCUSSION:DFTVIEW
Returns the user's setting from the Default View field on the Discussion tab of the My Discussion Settings page.
DISCUSSION:FILTERPREF
Returns the user's setting from the Filter Preference field on the Discussion tab of the My Discussion Settings page.
DISCUSSION:POSTINGPREF
Returns TRUE or FALSE based on the user's setting from the Posting Preference field on the Discussion tab of the My Discussion Settings page.
DISCUSSION:REPLYPREF
Returns TRUE or FALSE based on the user's setting from the Reply Preference field on the Discussion tab of the My Discussion Settings page.
DISCUSSION:VIEWITEMOPTIONS
Returns TRUE or FALSE based on the user's setting from the View Item Options field on the Discussion tab of the My Discussion Settings page.
WORKFLOW
Returns an Oscript Assoc containing the values of all the settings from the Workflow tab of the My Workflow Settings page.
WORKFLOW:WFSTARTPAGE
Returns the user's setting from the Starting Page field of the Workflow Assignments section on the Workflow tab of the My Workflow Settings page.
WORKFLOW:PROXY
Returns the user's setting from the Proxy field of the Workflow Assignments section on the Workflow tab of the My Workflow Settings page.
If workflow user proxies are disallowed, or the user does not have a proxy value set, this sub-tag will return the user ID in the main tag.
WORKFLOW:ASSIGNMENTSSHOW
Returns an Oscript Assoc containing the values of all the settings, including any expression data, from the Show field of the Workflow Assignments section on the Workflow tab of the My Workflow Settings page.

Returns TRUE or FALSE based on whether the Automatically open the next assignment in the same workflow if assigned to me box is selected in the Options section of the Workflow Assignments section of the Workflow tab on the My Workflow Settings page.
WORKFLOW:INCLUDEWFASSGN
Returns TRUE or FALSE based on whether the Include workflow assignments on 'My Assignments' page box is selected in the Options section of the Workflow Assignments section of the Workflow tab on the My Workflow Settings page.
WORKFLOW:STATUSSHOW
Returns an Oscript Assoc containing the values of all the settings, including any expression data, from the Show field of the Workflow Statuses section on the Workflow tab of the My Workflow Settings page.
WORKFLOW:SORTORDER
Returns the user's setting from the Sort Order field of the Workflow Statuses section of the Workflow tab on the My Workflow Settings page.
WORKFLOW:DFTWFTAB
Returns the user's setting from the Default My Workflows Tab field of the Workflow tab on the My Workflow Settings page.
WORKFLOW:MAXITEMS
Returns the user's setting from the Maximum Items Per Page field of the Workflow tab on the My Workflow Settings page.

Example

[LL_REPTAG=USERID USERPREF:GENERAL:STARTPAGE /] will return the user's Start Page setting.

Possible return values from this sub-tag are:

  • Enterprise Workspace

  • My Workspace

  • About Content Server

  • My Home

Related Information

Some sub-tags return an Oscript Assoc, and can therefore be further manipulated using the TOJSON sub-tag.

USERPREFACTION

Syntax

USERPREFACTION<operation>

Description

For a valid Content Server user ID, this sub-tag will update the user's settings found on the Settings page. You access the Settings page from the Content Server menu bar by selecting Settings from the Tools menu.

This sub-tag will only work if run by a user who has been granted sufficient privileges.

Parameters

Format
Description
USERPREFACTION:WORKFLOW:PROXY:<UserID>
Sets the value of the Proxy field of the Workflow Assignments section on the Workflow tab of the My Workflow Settings page.
The value of the Proxy field will be set to the user ID specified in the <UserID> parameter.
USERPREFACTION:WORKFLOW:PROXY:“none”
Clears the value from the Proxy field of the Workflow Assignments section on the Workflow tab of the My Workflow Settings page.
The value of the Proxy field will be set to “none”.

Example

[LL_REPTAG=<XXX> USERPREFACTION:WORKFLOW:PROXY:1000 /] will set the value of the Proxy user to “1000”, the Admin user.

Example: [LL_REPTAG=<XXX> USERPREFACTION:WORKFLOW:PROXY:"none" /] will clear the value in the Proxy field.
