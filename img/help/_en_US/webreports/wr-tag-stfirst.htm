A2UTF8

A2UTF8

Description

Converts an ASCII string to a UTF-8 string.

ADD

Syntax

ADD:<value>

Description

Adds a value to the main tag. The exact operation depends on the types. The return value of the expression, [LL_REPTAG_'<x>' ADD:<y> /], will depend on the types of <x> and <y>.

Parameters

Type of x Type of y Returns Example
Integer or Real Integer or Real x + y
[LL_REPTAG_'5' ADD:2.2 /] returns 7.2
Date Integer (seconds) x + y
[LL_REPTAG_'May 1 2011' ADD:86400 /] returns Mon May 02 00:00:00 2011
List List {@x,@y}
[LL_REPTAG_'{1,2,3}' ADD:'{4,5,6}' /] returns {1,2,3,4,5,6}
List any type {@x,y}
[LL_REPTAG_'{1,2,3}' ADD:'four' /] returns {1,2,3,'four'}
Assoc Assoc Merges x & y. y takes precedence on duplicate keys.
[LL_REPTAG_"A<?,1,'key1'=1>" ADD:"A<?,1,'key1'=2,'key2'=5>" /] returns A<1,1,'key1'=2,'key2'=5>
Record Record Merges x & y. y takes precedence on duplicate keys. Column order is lost.
[LL_REPTAG_"R<'key1'=1>" ADD:"R<'key1'=2,'key2'=5>" /] returns R<'key1'=2,'key2'=5>
RecArray RecArray Appends y to x, as long as x & y have the same fields.
[LL_REPTAG_"V{<'fruit','color'><'banana','yellow'>}" ADD:"V{<'fruit','color'><'orange','orange'>}" /] returns V{<'fruit','color'><'banana','yellow'><'orange','orange'>}
anything else anything else String(x) + String(y)
[LL_REPTAG_"abc" ADD:"3" /] returns abc3

Related Information

For more information, see DIVIDE, MULTIPLY, MODULUS, SUBTRACT, and TIMESINCE.

ADDVAR

Syntax

ADDVAR:<variable name>

Description

Adds the value of the sub-tag, after formatting by any previous sub-tags, with the current contents of a variable which can later be referenced using the specified variable name. This sub-tag only works when numeric values are returned by a tag. The first time this sub-tag is used, it behaves as if the variable already existed with the value of zero, 0.

Example

For example, [LL_REPTAG=COST ADDVAR:subtotal /]

In this example, if the values 26, 34, and 14 were returned as values from the tag “cost” then the variable tag LL_REPTAG_%variablename would return: 74

Related Information

More information on using ADDVAR, variable tags and other sub-tags associated with managing variables can be found in Using Variables.

See also SETVAR, CONCATVAR and CURRENTVAL.

APPPROPERTY

Syntax

APPPROPERTY:<property name>

Description

Returns a string from the application's properties file for the user's default metadata language. If a properties file does not exist for the user's default metadata language then the string will be returned from the application's properties file for the system default language. If a properties file does not exist for the system default language then the string will be returned from the application's en_US properties file. If the property is not found in the properties file, or the application has no properties files, then an error message will be returned.

This sub-tag expects an application name in the main tag.

Example

Example

[LL_REPTAG_"myapplication" APPPROPERTY:"MYAPPLICATION_LABEL.Instructions" /]

will return the value of the MYAPPLICATION_LABEL.Instructions string from the myapplication_<language>.properties file for the application.

ASSOC

Syntax

ASSOC:<key> or ASSOC:[index]

Description

Returns the value of the key field if the returned datatype is a Content Server ASSOC. Either the key itself or an integer representing the key location within the ASSOC can be used. Note, this sub-tag can be used repetitively where ASSOCs may be nested.

Related Information

See also ASSOCACTION, LIST, RECORD and RECARRAY.

ASSOCACTION

Syntax

ASSOCACTION:<parameter>:<value>

Description

This sub-tag allows the creation and manipulation of data in a Content Server Assoc data type. Use the sub-tag SETVAR to store the Assoc in order to allow it to be accessed by subsequent tags.

Parameters

Format Description
ASSOCACTION:ADD:<key>:<value> Adds a <key>/<value> pair to the Assoc defined in the main tag, and returns the updated Assoc.
ASSOCACTION:CHANGE:<key>:<value> Changes the value of a <key> in the Assoc defined in the main tag, and returns the updated Assoc.
ASSOCACTION:CREATE Creates an Assoc. If the main tag is an empty string, then an empty Assoc will be created. If the main tag contains a list of <key>/<value> pairs, then the data will be used to populate the Assoc. Returns the resultant Assoc.
ASSOCACTION:DELETE:<key> Deletes a <key> from the Assoc defined in the main tag, and returns the updated Assoc.
ASSOCACTION:FROMRECORD Returns an Assoc created from the Record defined in the main tag.
ASSOCACTION:MERGE:<Assoc_arg> Merges the <Assoc_arg> supplied as an argument with the Assoc defined in the main tag, and returns the updated Assoc.
ASSOCACTION:TORECORD Returns a Record created from the Assoc defined in the main tag.

Examples

Example: [LL_REPTAG_!myAssoc ASSOCACTION:ADD:"Name":"Bob" SETVAR:myAssoc /] will add the <key> “Name” with the <value> “Bob” to the Assoc defined in the myAssoc variable. It will then save the updated Assoc back to the myAssoc variable.

Example: [LL_REPTAG_!myAssoc ASSOCACTION:CHANGE:"Name":"Fred" SETVAR:myAssoc /] will change the value of the <key> “Name” to the value “Fred” in the Assoc defined in the myAssoc variable. It will then save the updated Assoc back to the myAssoc variable.

Example

[LL_REPTAG_'' ASSOCACTION:CREATE SETVAR:myAssoc /] will create an empty Assoc and save it to the myAssoc variable.

[LL_REPTAG_'{{"Name","Bob"},{"Fruit","Apple"}}' ASSOCACTION:CREATE SETVAR:myAssoc /] will create an Assoc and populate it with the <key>s “Name” and “Fruit”. The <key> “Name” will be populated with the value “Bob”. The <key> “Fruit” will be populated with the value “Apple”. It will then save the created Assoc to the myAssoc variable.

Example: [LL_REPTAG_!myAssoc ASSOCACTION:DELETE:"Fruit" SETVAR:myAssoc /] will delete the <key> “Fruit” from the Assoc defined in the myAssoc variable. It will then save the updated Assoc back to the myAssoc variable.

Example: [LL_REPTAG_DATASOURCE RECARRAY:1 ASSOCACTION:FROMRECORD SETVAR:myAssoc /] will create an Assoc from the first row of the data source. It will then save the created Assoc to the myAssoc variable.

Example: [LL_REPTAG_!myAssoc ASSOCACTION:MERGE[LL_REPTAG_!myOtherAssoc /] SETVAR:myAssoc /] will merge the data from the Assoc defined in the myOtherAssoc variable with the data in the Assoc defined in the myAssoc variable. It will then save the updated Assoc back to the myAssoc variable.

Example: [LL_REPTAG_!myAssoc ASSOCACTION:TORECORD SETVAR:myRecord /] will create a Record from the Assoc defined in the myAssoc variable. It will then save the Record to the myRecord variable.

Related Information

See also ASSOC and SETVAR.

AUDITACTION

Syntax

AUDITACTION:<operation>

Description

Expects a valid Content Server ID, either a user/group ID or a DataID for a Content Server item, and performs an action to generate an audit event.

The result is a new row added to the DAuditNew Content Server database table and optional rows in the DAuditMore table. If the specified ID is a DataID then the DataID field of the DAuditNew record will be set, otherwise if the ID is a user or user Group ID then the UserID field will be set instead. Whichever field is not used will be set to NULL.

The PerformerID field for the new audit record will always be set to the ID for whichever user context was active when the sub-tag is executed. This happens even if the “Run As” feature has been used to run the WebReport in a different user context. In this case, a row will be added to the DAuditMore table, linked using the eventID key, that shows the ID of the selected “Run As” user.

In addition to the UserID, DataID and PerformerID columns, values can be specified for several other fields as specified in this table.

Append the SHOW sub-tag to return the eventID of the audit event.

 

Note:
The Custom Audit Event checkbox must be ticked on the Set Audit Interests administration page in order for the AUDITACTION sub-tag to work.

 

Parameters

Format
Description
AUDITACTION:<value>
Adds an entry to the DAuditNew table with the value specified set to the “value1” column.
AUDITACTION:<fixed key>:<value>:<fixed key>:<value> ...
Adds an entry to the DAuditNew table with multiple values set to any of the fixed keys, column names in DAuditNew, which can be one of “value1”, “value2” or “valuekey”.
AUDITACTION:<user key>:<value>:<user key>:<value> ...
Adds key-value pairs into the DAuditMore table. Value(s) specified in <user key> will be added to the KeyStr column, while value(s) specified in <value> will be added to the ValueStr column.

Related Information

For more information, see SHOW.

AUDITINFO

Syntax

AUDITINFO:<operation>

Description

Expects a valid Content Server node ID. Returns details from the most recent Audit Event for the object, and allows you to specify which column from DAuditNew to display. It supports custom audit entries generated by the AUDITACTION sub-tag, as well as the RunAs feature.

Parameters

Format
Description
AUDITINFO:ACTUALUSER
For “WebReports Run” events, if the WebReport was executed by the RunAs feature, the ID of the RunAs user will be displayed.
AUDITINFO:AUDITDATE
Returns the AuditDate value of the event from the DAuditNew table.
AUDITINFO:AUDITID
Returns the AuditID value of the event from the DAuditNew table.
AUDITINFO:AUDITSTR
Returns the AuditStr value of the event from the DAuditNew table.
AUDITINFO:EVENTID
Returns the EventID value of the event from the DAuditNew table.
AUDITINFO:KEYSTR:<key>
This option looks up the <key> value in the KeyStr column of DAuditMore, and returns the corresponding value of the ValueStr column. Useful to return the value of ValueStr for a specific custom value.
AUDITINFO:PERFORMERID
Returns the PerformerID value of the event from the DAuditNew table. For WebReports Run events, use the :ACTUALUSER option to display the ID of the RunAs user.
AUDITINFO:SUBTYPE
Returns the SUBTYPE value of the event from the DAuditNew table.
AUDITINFO:USERID
Returns the UserID value of the event from the DAuditNew table.
AUDITINFO:VALUE1
Returns the Value1 value of the event from the DAuditNew table.
AUDITINFO:VALUE2
Returns the Value2 value of the event from the DAuditNew table.
AUDITINFO:VALUEKEY
Returns the ValueKey value of the event from the DAuditNew table.
AUDITINFO:VALUESTR
Returns the ValueStr value of the event from the DAuditMore table.

Related Information

For more information, see AUDITACTION.

AVERAGE

Syntax

AVERAGE

Description
