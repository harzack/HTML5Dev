This section details the sub-tags: VALUES, VERINFO, VERSIONACTION, and VERSIONCAT.

VALUES

Syntax

VALUES

Description

For a main tag that represents a Record or Assoc, the VALUES sub-tag returns the values of the tag as a List.

Example

[LL_REPTAG_"R<'key1'='value1','key2'='value2'>" VALUES /] returns “{'value1','value2'}”.

 

Note:
The field order might differ.

 

Related Information

For more information, see ASSOC, FIELDS, LIST, and RECORD.

VERINFO

Syntax

VERINFO:<format>:<version number>

Description

This sub-tag will return information about a version of a nodeid. Version number is an optional parameter. If it is appended to the end of the sub-tag arguments then information about that specific version will be returned, otherwise the information about the latest version will be returned.

Returns comments assigned to the version.
 

Note:
Comments assigned to the version are not the same as comments assigned to the node.

Example

[LL_REPTAG=DATAID VERINFO:MIMETYPE:5 /] will return the mimetype of the 5th version of an object, whereas [LL_REPTAG=DATAID VERINFO:MIMETYPE /] will return the mimetype for the latest version.

VERSIONACTION

Syntax

VERSIONACTION:<operation>

Description

This sub-tag offers capabilities to perform various actions against versions of a given node within Content Server.

Parameters

Syntax Description
VERSIONACTION:UPDATE:DESCRIPTION:<new description>:<version number>
Updates the Description field of a node's latest version, provided <versionNum> is not specified. To update a specific version, supply the <version number> as a parameter after the <new description>.
If <version number> is not supplied, or the version number is “0”, the latest version of the node will be updated.

VERSIONCAT

Syntax

VERSIONCAT:<parms>

Description

Returns information about the Categories and Attributes associated with a specific version of the current object ID. This sub-tag supports both attribute sets and multi-value attributes through use of the MV and SET operators.

Parameters

Syntax Description
VERSIONCAT:<vernum> Returns TRUE or FALSE based on the presence of one or more categories assigned to the version of the node.
VERSIONCAT:<vernum>:JSARRAY
This is a unique option for the VERSIONCAT sub-tag which results in a JavaScript object representation of all the category and attribute data, including full support for sets and multi-values, for the version of the node.
For detailed information on this feature, see the CAT:JSARRAY Examples advanced help.
VERSIONCAT:<vernum>:RAW
This option results in a flat Oscript representation of all the category and attribute data associated with the version of the node.
See Requests as Data Sources for how to use the output of this, and related tags, as the data source to another WebReport.
See the TOJSON sub-tag for how to convert this into a data structure that can be interpreted by JavaScript in the client.
See related tags LLURL:FUNCTIONMENU:RAW and LLURL:ADDITEM:ITEMMENU:RAW.
VERSIONCAT:<vernum>:DISPLAY Will return a string containing a list of semi-colon-separated, ;, category names assigned to the version of the node. Any category containing mandatory attributes will be suffixed with an exclamation in square brackets: [!].
VERSIONCAT:<vernum>:MAND Returns TRUE or FALSE based on whether a category containing a mandatory attribute is assigned to the version of the node.
VERSIONCAT:<vernum>:<catname> Returns TRUE or FALSE based on the presence of the <catname> category.
VERSIONCAT:<vernum>:<catname>:DISPLAY
Will return a string containing a list of semi-colon-separated, ;, attribute values for the category specified in <catname>. If either attribute sets or multi-value attributes are contained in the category, they will be displayed in the following syntax: 'attval1;attval2;[SET3:attval4,attval5,[MV6:attval7,attval8]];attval9'
Here, attval1, attval2 and attval9 are regular attribute values.
[SET3: represents the start of an attribute set, where 3 is the attribute ID of the set, see note below. Contained in the attribute set are two regular attributes, attval4 and attval5, and a multi-value attribute, starting [MV6:, with 6 being the attribute ID of the multi-value attribute. The multi-value attribute contains two values for attribute 6, these are: attval7 and attval8.
 

Note:
Attribute IDs can be identified by looking at the browser status bar whilst hovering over the attribute name in the category definition page.

 
VERSIONCAT:<vernum>:<catname>:ID Will return the ID of <catname>, if the category is present.
VERSIONCAT:<vernum>:<catname>:<attname> Returns TRUE or FALSE based on whether an attribute, <attname>, exists with a non-blank value in the category, <catname>. If an attribute set is specified in <attname>, a single entry in any one of the set's attribute values will cause this to return TRUE. If <attname> is a multi-value attribute, a single entry in any one will also cause this to return TRUE. <attname> can also be nested within an attribute set, either as a regular attribute or a multi-value attribute.
VERSIONCAT:<vernum>:<catname>:<attname>:DISPLAY Will return the value in the attribute, <attname>, for the category, <catname>. If <attname> is an attribute set, or a multi-value attribute, a comma-separated list of values will be returned.
VERSIONCAT:<vernum>:<catname>:<attname>:ID Will return the ID of the attribute, <attname>, for the category, <catname> if it is present. If <attname> exists inside an attribute set, it will be returned if no attribute matching <attname> exists at the top level in the category. If <attname> exists at the top level of the category, and within an attribute set of the same category, the attribute ID at the top level, not within the set, will take precedence.
VERSIONCAT:<vernum>:<catname>:<attname>:<attval> Returns TRUE or FALSE based on whether an attribute, <attname>, exists with a value equal to <attval> in the category, <catname>. The <attname> parameter may be an attribute within an attribute set, including a multi-value, or any element within a multi-value attribute.
VERSIONCAT:<vernum>:<catname>:MV:<attname>:COUNT Will return the number of multi-value attributes in use.
VERSIONCAT:<vernum>:<catname>:MV:<attname>:<n> Returns TRUE or FALSE based on the presence of a specific non-blank element in a multi-value attribute, <attname>, for the category, <catname> at the index location <n>. The value of <n> must be a positive integer between 1 and <n>, where <n> does not exceed the length of the multi-value attribute. If <n> exceeds the length of the multi-value attribute, an empty string will be returned.
VERSIONCAT:<vernum>:<catname>:MV:<attname>:<n>:<attval> Returns TRUE or FALSE based on the presence of <attval> in a multi-value attribute, <attname>, for the category, <catname> at at the index location <n>. The value of <n> must be a positive integer between 1 and <n>, where <n> does not exceed the length of the multi-value attribute.
VERSIONCAT:<vernum>:<catname>:MV:<attname>:<n>:DISPLAY Will return the value in the multi-value attribute, <attname>, for the category, <catname> at the index location <n>, where <n> is a positive integer between 1 and <n>. The value of <n> must be a positive integer between 1 and <n>, where <n> does not exceed the length of the multi-value attribute.
VERSIONCAT:<vernum>:<catname>:MV:<attname>:INDEX:<attval> Will return the first index of the multi-value attribute, <attname>, for the category, <catname> where <attval> is equal to the value contained in the multi-value attribute at index. For example, if <attval> exists in the third element of <attname>, the number 3 will be returned.
VERSIONCAT:<vernum>:<catname>:SET:<setname>:<attname> Uses the SET operator allowing attribute sets to be explicitly referenced. This example will return TRUE if the attribute, <attname>, within the set, <setname>, is non-blank. Only attributes within <setname> can be referenced, so if <attname> exists outside the set, it will not be found. If <attname> is a multi-value attribute, TRUE will be returned if any field within it is non-blank.
VERSIONCAT:<vernum>:<catname>:SET:<setname>:<attname>:DISPLAY Will return the value in the attribute, <attname>, for the category, <catname>, if <attname> is an attribute within <setname>. If <attname> is a multi-value attribute, a comma-separated list of the defined values will be returned. If no defined value exists, either in a multi-value attribute, or a standard attribute, an empty list will be returned.
VERSIONCAT:<vernum>:<catname>:SET:<setname>:<attname>:ID Will return the ID of the attribute, <attname>, within the set, <setname>. If <attname> exists outside <setname>, it will not be returned. <attname> can be either a regular or multi-value attribute.
VERSIONCAT:<vernum>:<catname>:SET:<setname>:<attname>:<attval> Returns TRUE or FALSE based on whether an attribute, <attname>, exists with a value equal to <attval> in the set, <setname>. If <attname> is a multi-value attribute, all elements will be checked for equality against <attval>, if any match, TRUE will be returned.
VERSIONCAT:<vernum>:<catname>:SET:<setname>:MV:<attname>:COUNT Will return number of multi-value attributes that have been populated.
VERSIONCAT:<vernum>:<catname>:SET:<setname>:MV:<attname>:<n> Returns TRUE or FALSE based on the presence of a specific non-blank element in a multi-value attribute, <attname>, within a set, for the category, <catname> at the index location <n>. The value of <n> must be a positive integer between 1 and <n>, where <n> does not exceed the length of the multi-value attribute.
VERSIONCAT:<vernum>:<catname>:SET:<setname>:MV:<attname>:<n>:<attval> Returns TRUE or FALSE based on the presence of <attval> in a multi-value attribute, <attname>, within a set, for the category, <catname>, at the index location <n>. The value of <n> must be a positive integer between 1 and <n>, where <n> does not exceed the length of the multi-value attribute.
VERSIONCAT:<vernum>:<catname>:SET:<setname>:MV:<attname>:<n>:DISPLAY Will return the value in the multi-value attribute, <attname>, within a set, for the category, <catname>, at the index location <n>, where <n> is a positive integer between 1 and <n>. The value of <n> must be a positive integer between 1 and <n>, where <n> does not exceed the length of the multi-value attribute.
VERSIONCAT:<vernum>:<catname>:SET:<setname>:MV:<attname>:INDEX:<attval> Will return the first index of the multi-value attribute, <attname>, within a set, for the category, <catname>, where <attval> is equal to the value contained in the multi-value attribute at index.
VERSIONCAT:<vernum>:<catname>:MVSET:<setname>:<n>:<attname> Returns TRUE or FALSE based on whether the multi-value set, <setname>, has a non-blank attribute, <attname>, at the index location <n>, within the the category, <catname>.
VERSIONCAT:<vernum>:<catname>:MVSET:<setname>:<n>:<attname>:DISPLAY Will display the value of an attribute <attname> within a multi-value set, <setname> at the index location <n>, within the category, <catname>.
VERSIONCAT:<vernum>:<catname>:MVSET:<setname>:<n>:<attname>:ID Will display the ID of an attribute, <attname>, within a multi-value set, <setname>, at the index location <n>, within the the category, <catname>.
VERSIONCAT:<vernum>:<catname>:MVSET:<setname>:<n>:<attname>:<attval> Returns TRUE or FALSE based on whether the multi-value set, <setname>, has a attribute, <attname>, at the index location <n>, within the the category, <catname> that is equal to the value specified in <attval>.
VERSIONCAT:<vernum>:<catname>:MVSET:<setname>:<n>:MV:<attname>:COUNT Returns the number of multi-value attributes in use in the multi-value set.
VERSIONCAT:<vernum>:<catname>:MVSET:<setname>:<n>:MV:<attname>:<m> Returns TRUE or FALSE based on whether the multi-value set, <setname>, at index location <n> has a multi-value attribute, <attname>, at the index location <m>, that is non-blank.
VERSIONCAT:<vernum>:<catname>:MVSET:<setname>:<n>:MV:<attname>:<m>:DISPLAY Will display the value of a multi-value attribute, <attname>, at the index location <m>, within a multi-value set, <setname>, at the index location <n>, within the the category, <catname>.
VERSIONCAT:<vernum>:<catname>:MVSET:<setname>:<n>:MV:<attname>:<m>:<attval> Returns TRUE or FALSE based on whether the multi-value set, <setname>, at the index location <n>, has a multi-value-attribute, <attname>, at the index location <m> whose value is equal to the value specified in <attval>.
VERSIONCAT:<vernum>:<catname>:MVSET:<setname>:<n>:MV:<attname>:INDEX:<attval> Will return the first index of the multi-value attribute, <attname>, within a multi-value set, for the category, <catname>, where <attval> is equal to the value contained in the multi-value attribute at index.

Example

[LL_REPTAG=DATAID VERSIONCAT:<vernum>:<catname>:<attname>:DISPLAY /] will return the attribute stored in the category whose display name or ID matches the <catname> value and the attribute whose display name or ID matches the <attname> value for any attribute type, for the version <vernum> of the node.

If <attname> is a multi-value attribute, or an attribute set, a comma-separated list of values will be returned.

Related Information

To return Category data for the latest version of a node see the CAT sub-tag. The VERSIONCAT sub-tag is essentially the same as the CAT sub-tag, but returns data for a specific version of a node whereas the CAT sub-tag returns data for the latest version of a node. A detailed example of the CAT sub-tag and several other methods of Categories and Attributes reporting can be found in Categories and Attributes.

See Requests as Data Sources for how to use the output from certain variations of this sub-tag, and related tags, as the data source to another WebReport.

See the TOJSON sub-tag for how to convert the output from certain variations of this sub-tag into a data structure that can be interpreted by JavaScript in the client.

For related information, see CAT:JSARRAY Examples, LLURL:FUNCTIONMENU:RAW, and LLURL:ADDITEM:ITEMMENU:RAW.
