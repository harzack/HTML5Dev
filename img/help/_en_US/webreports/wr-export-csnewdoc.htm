It is possible to export the report output to a new Content Server document. This can either be done in one of two ways:

  1. Select Export to Content Server from the WebReport Export option.

  2. Set the default Output Destination as Content Server and then run the WebReport.

In the Export to Content Server New Document screen:

  1. From the Output Destination list, select Content Server.

  2. Set the Export Type option to Content Server Node.

  3. In the Name field, ensure that you type a unique filename. If a name that is not unique is provided, a unique postfix will be appended. For example, “New WebReport Snap Shot 5”.

     

    Note:
    The default name will be the WebReport name followed by “Snap Shot”.

     

  4. In the Duplicate Name Action field, set an option to determine how the export handles duplicate node names:

    • Select Delete Original if you want to delete the original node, if it already exists, and create a new node with exactly the same name.

    • Select Add Version to Original if you want to add a version to that node, if the node name already exists. If the node doesn't exist it will be created.

    • Select Generate Unique Number if you want to create a new document every time the export occurs. A number will be appended to the node name in order to distinguish it from the previous exports.

  5. You can optionally type a description of the resultant document in the Description field.

  6. In the Create In field, click Browse Content Server, navigate to the container you want to store this exported document, then click that container's Select link.

     

    Note:
    The default location will be the folder containing the WebReport.

     

  7. You can optionally select a category to apply to this exported document in the Categories field.

  8. In the Export MIME Type field, you can optionally specify the MIME type of the resultant document.

  9. You can optionally select Use External Conversion Engine.

  10. Optionally, if you want to return immediately to your browser, select Run in Background.

For more information, see Exporting to a Content Server Node - Parameters and Example.
