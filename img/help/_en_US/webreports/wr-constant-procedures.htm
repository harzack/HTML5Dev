The following procedures detail how to:

  1. To Manually Add a New Constant for the WebReport: use this procedure when you have constants you want to define, but have not yet used, in the WebReport. Once defined here, you can reference them in the reportview.

  2. To Automatically Add a New Constant for the WebReport: use this procedure when you have made reference to a constant, or multiple constants, in the reportview and you want to quickly populate them on the Constants tab.

    This procedure saves time on the procedure above as it allows all the constants to be populated at once. It will also, where possible, determine the constant type programmatically. Where this is not possible, the constant will be of type string.

  3. To Drag and Drop a Constant to the WebReport Reportview: if you are working in the WebReports online editor, it is possible to use the drag-and-drop interface to speed up some activities.

  4. To Delete a WebReport Constant.
