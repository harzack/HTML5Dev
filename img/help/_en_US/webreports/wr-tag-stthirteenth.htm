This section details the sub-tags: ODD, ODDEVEN, and ONERROR.

ODD

Syntax

ODD

Description

Returns true if the data returned by the main tag is an odd number, otherwise returns false. This can also be used as per the example for the EVEN tag.

ODDEVEN

Syntax

ODDEVEN

Description

Returns either ODD or EVEN based on the data returned by the main tag. If the main tag data is not a valid integer then this sub-tag returns an empty string. This sub-tag can be used with a tag like ROWNUM to set an appropriate style for each row. The ODDEVEN sub-tag also allows two additional parameters to be specified. If provided, these two parameters will be returned alternately instead of ODD/EVEN.

Example

This example shows how to use the standard Content Server styles, BrowseRow1 and BrowseRow2 to allow each row to have an alternating colour. The class statement for each row would alternate between BrowseRow1 and BrowseRow2.

Each row includes the following TR statement:

<TR class=[LL_REPTAG_ROWNUM ODDEVEN:BrowseRow1:BrowseRow2 /]>

Each row would resolve to either:

<TR class=BrowseRow1>

or:

<TR class=BrowseRow2>

ONERROR

Syntax

ONERROR:<parm>

Description

A sub-tag error typically occurs when a sub-tag receives invalid data or an invalid parameter. The ONERROR sub-tag allows the developer to change the default error message when an error occurs. The position of the ONERROR sub-tag in the list of sub-tags is irrelevant, and applies to every sub-tag in the list.

The optional BREAK parameter prevents subsequent sub-tags from being processed as soon as the first error occurs.

Parameters

The following options for using this sub-tag include:

Error Type
Description
ONERROR:BLANK[:BREAK]
This option will replace any standard error message with a blank string, useful for preventing error messages from being displayed.
ONERROR:SHORT[:BREAK]
This option will display a short version of the error message consisting of the sub-tag in question and an error number.
ONERROR:LONG[:BREAK]
This option will display a full error message for the sub-tag in question.
ONERROR:CUSTOM:<error message>[:BREAK]
This option allows the developer to specify their own error message for a sub-tag.

Example

For example: [LL_REPTAG=VALUE LIST:1 ONERROR:CUSTOM:"Invalid list" INC:1 ONERROR:LONG /]

In this example the message “Invalid list” would be returned for this tag if the LIST sub-tag had returned an error. If the LIST sub-tag does not cause an error but the INC sub-tag does, then the standard error message would be returned.

Example

The following is example using the optional BREAK parameter. Consider the following sub-tag:

[LL_REPTAG_"somestring" LIST:4 INC /]

The LIST:4 sub-tag produces an error because the data, in this case “somestring”, is not a valid list. The INC sub-tag receives the error message from the LIST sub-tag, which is also invalid, in other words is not an integer, and therefore produces an error:

* INC: Invalid (non-integer) data value *

The error message can be customized in a number of ways, as detailed below. Notice that the placement of ONERROR is irrelevant:

[LL_REPTAG_"somestring" LIST:4 ONERROR:SHORT INC /]

Returns: * INC: 1002 *

[LL_REPTAG_"somestring" LIST:4 INC ONERROR:CUSTOM:"Custom Error Message" /]

Returns: * INC: Custom Error Message *

Adding the BREAK parameter causes processing to stop as soon as the first error occurs. In the following example, the error occurs in the LIST sub-tag, and therefore the INC sub-tag isn't processed:

[LL_REPTAG_"somestring" LIST:4 INC ONERROR:LONG:BREAK /]

Returns: * LIST: Data is not a valid list *
