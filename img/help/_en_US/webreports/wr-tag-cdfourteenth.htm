This section details the column data tags: TIMEZONE, TOTALROWS, TOTALSOURCEROWS, TRIGGER, TRIGGERADDEDCATEGORIES, TRIGGERCATEGORYVALUECHANGES, TRIGGERDELETEDCATEGORIES, TRIGGERDESTPARENTID, TRIGGERID, TRIGGERNEWID, TRIGGERNEWNAME, TRIGGEROLDNAME and TRIGGERSOURCEPARENTID.

TIMEZONE

Syntax

[LL_REPTAG_TIMEZONE /]

Description

Returns the server time zone if the optional Electronic Signatures Content Server module is installed. Returns an empty string otherwise.

TOTALROWS

Syntax

[LL_REPTAG_TOTALROWS /]

Description

Total number of rows returned by the Data Source, after DSstartrow and DSendrow. This tag returns the number of rows originally available, regardless of the INCLUDEIF tag. If the user requires a counter for the number of rows that have been included after INCLUDEIF, the ACTUALROWS parameter should be used. This tag does not take account of the effect of the special data source parameters DSstartrow and DSendrow see TOTALSOURCEROWS.

Related Information

For more information, see ACTUALROWS, FILTEREDROWS, and TOTALSOURCEROWS.

TOTALSOURCEROWS

Syntax

[LL_REPTAG_TOTALSOURCEROWS /]

Description

Total number of rows returned by the Data Source, before DSstartrow and DSendrow. This tag is closely related to TOTALROWS and ACTUALROWS. The difference being that it takes account of the special data source parameters DSstartrow and DSendrow, making it very useful for pagination.

Related Information

See Data Source Parameters for more information.

TRIGGER

Syntax

[LL_REPTAG_TRIGGER /]

Description

The trigger event that initiated the WebReport as specified for the WR Trigger. This tag will only return a value when a WebReport was triggered using the WRTrigger feature. The WRTrigger feature must be enabled in the admin pages on a per subtype basis. By default, it is off for all subtypes. If the feature is not enabled for a given subtype no WebReport will be triggered, even if the node is within a hierarchy that has a cascading trigger applied.

Parameters

Possible trigger values are:

Trigger Value
Description
Add Version
Fired when a new version is added to an existing document or when a new document is added.
Category Update
Fired when a Content Server category is applied, removed or updated on a given node.
Copy Destination
Fired when an item is copied. Can also be associated with a container where it will fire for any item that is copied to the container. In other words, the container is the destination for the copy action.
Copy Source
Fired when an item is copied. Can also be associated with a container where it will fire for any item that is copied from the container. In other words, the container is the source for the copy action.
Create
Fired when a node is created.
Delete
Fired when a node is deleted.
Move Destination
Fired when an item is moved. Can also be associated with a container where it will fire for any item that is moved to the container. In other words, the container is the destination for the move action.
Move Source
Fired when an item is moved. Can also be associated with a container where it will fire for any item that is moved from the container. In other words, the container is the source for the move action.
Rename
Fired when a node is renamed.
Update
Fired when a node is updated. For example, a node is updated when the description is changed.
Workflow Initiated
Fired when a workflow is initiated.
Workflow Completed
Fired when a workflow is completed.
Workflow Suspended
Fired when a workflow is suspended.
Workflow Resumed
Fired when a workflow is resumed.
Workflow Stopped
Fired when a workflow is stopped.
Workflow Deleted
Fired when a workflow is deleted.
Workflow Archived
Fired when a workflow is archived.

Related Information

See general information on WR Trigger.

TRIGGERADDEDCATEGORIES

Syntax

[LL_REPTAG_TRIGGERADDEDCATEGORIES /]

Description

Returns a RecArray containing details of categories that have been added when a category update event occurs.

TRIGGERCATEGORYVALUECHANGES

Syntax

[LL_REPTAG_TRIGGERCATEGORYVALUECHANGES /]

Description

Returns a RecArray containing the old and new values of category attributes when a category update event occurs.

The RecArray will contain the following columns:

Column Name
Description
CATID
The ID of the category that was updated.
ATTRID
The ID of the attribute that was updated.
OLDVALUE
The value of the attribute prior to the update. Will return ? (undefined) if the no value was present. For example, if a new row was added.
NEWVALUE
The new value of the attribute after the update.
SETNAME
The name of the set, if the attribute is contained in a set. Will return ? (undefined) otherwise.
SETINDEX
The set index, if the attribute is contained in a set. Will return ? (undefined) otherwise.
ATTRINDEX
The index of the attribute. Will always return a value. The index will be 1 if the attribute is not a multi-value attribute.

Related Information

For more information, see RECARRAY and RECORD.

TRIGGERDELETEDCATEGORIES

Syntax

[LL_REPTAG_TRIGGERDELETEDCATEGORIES /]

Description

Returns a RecArray containing details of categories that have been deleted when a category update event occurs.

TRIGGERDESTPARENTID

Syntax

[LL_REPTAG_TRIGGERDESTPARENTID /]

Description

The nodeid of the destination container in a copy or move operation.

TRIGGERID

Syntax

[LL_REPTAG_TRIGGERID /]

Description

The nodeid of the item that initiated the trigger event. If the trigger event was initiated from a workflow then the subworkid of the workflow instance will be returned instead.

TRIGGERNEWID

Syntax

[LL_REPTAG_TRIGGERNEWID /]

Description

The new nodeid of the item copied when a Copy event occurs.

TRIGGERNEWNAME

Syntax

[LL_REPTAG_TRIGGERNEWNAME /]

Description

The new name of the item renamed when a Rename event occurs.

TRIGGEROLDNAME

Syntax

[LL_REPTAG_TRIGGEROLDNAME /]

Description

The old name of the item renamed when a Rename event occurs.

TRIGGERSOURCEPARENTID

Syntax

[LL_REPTAG_TRIGGERSOURCEPARENTID /]

Description

The nodeid of the source container in a copy or move operation
