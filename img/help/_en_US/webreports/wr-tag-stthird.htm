CAT

Syntax

CAT:<parm>

Description

Returns information about the Categories and Attributes associated with the current object ID. This sub-tag supports both attribute sets and multi-value attributes through use of the MV and SET operators.

For example, [LL_REPTAG=DATAID CAT:<catname>:<attname>:DISPLAY /] will return the attribute value stored in the category whose display name or ID matches the <catname> value and the attribute whose display name or ID matches the <attname> value for any attribute type. If <attname> is a multi-value attribute, or an attribute set, then a comma-separated list of values will be returned.

If using a WebReport to update category values using the CATACTION sub-tag with the NOCOMMIT argument, you can retrieve the uncommitted category values from the cache by appending the argument :FROMCACHE to your CAT syntax.

To return Category data for a specific version of a node see the VERSIONCAT sub-tag.

Parameters

Syntax Description
CAT Returns true or false based on the presence of one or more categories assigned to the node.
CAT:JSARRAY This is a unique option for the CAT sub-tag which results in a JavaScript object representation of all the category and attribute data, including full support for sets and multi-values, for a specific node. For detailed information on this feature see CAT:JSARRAY Examples.
CAT:RAW
This option results in a flat oscript representation of all the category and attribute data associated with a specific node.
See Requests as Data Sources for how to use the output of this, and related tags, as the data source to another WebReport.
See the TOJSON sub-tag for how to convert this into a data structure that can be interpreted by JavaScript in the client.
See related tags LLURL:FUNCTIONMENU:RAW and LLURL:ADDITEM:ITEMMENU:RAW.

CAT:MAND Returns true or false based on whether a category containing a mandatory attribute is assigned to the node.

CAT:<catname>:DISPLAY
Will return a string containing a list of semi-colon-separated attribute values for the category specified in <catname>.
A category containing an attribute set or a multi-value attribute, will be displayed in the following syntax:
'attval1;attval2;[SET3:attval4,attval5,[MV6:attval7,attval8]];attval9'

•

•

•

Contained in the attribute set are two regular attributes, attval4 and attval5, and a multi-value attribute, which starts with: [MV6:, where 6 is the attribute ID of the multi-value attribute.

•

The multi-value attribute contains two values for attribute 6, these are: attval7 and attval8.

Attribute IDs can be identified by looking at the browser status bar whilst hovering over the attribute name in the category definition page.

CAT:<catname>:UPGRADEABLE Will return TRUE or FALSE depending on whether the category, <catname>, is upgradeable or not.
CAT:<catname>:VERSION Will return the version number of <catname>, if the category, <catname>, is present.
CAT:<catname>:<attname> Returns true or false based on whether an attribute, <attname>, exists with a non-blank value in the category, <catname>. If an attribute set is specified in <attname>, a single entry in any one of the set's attribute values will cause this to return TRUE. If <attname> is a multi-value attribute, a single entry in any one will also cause this to return TRUE. <attname> can also be nested within an attribute set, either as a regular attribute or as a multi-value attribute.
CAT:<catname>:<attname>:DISPLAY Will return the value in the attribute, <attname>, for the category, <catname>. If <attname> is an attribute set or a multi-value attribute a comma-separated list of values will be returned.
CAT:<catname>:<attname>:ID Will return the ID of the attribute, <attname>, for the category, <catname>, if it is present. If <attname> exists inside an attribute set, it will be returned if no attribute matching <attname> exists at the top level in the category. If <attname> exists at the top level of the category and within an attribute set of the same category, the attribute ID at the top level, not within the set, will take precedence.
CAT:<catname>:<attname>:<attval> Returns true or false based on whether an attribute, <attname>, exists with a value equal to <attval> in the category, <catname>. <attname> may be an attribute within an attribute set, including a multi-value, or any element within a multi-value attribute.
CAT:<catname>:MV:<attname>:COUNT Will return number of multi-value attributes in use.
CAT:<catname>:MV:<attname>:<n> Returns true or false for the presence of a specific non-blank element in a multi-value attribute, <attname>, for the category, <catname> at the index location <n>. The value of <n> must be a positive integer between 1 and <n>, where <n> does not exceed the length of the multi-value attribute. If <n> exceeds the length of the multi-value attribute, an empty string will be returned.
CAT:<catname>:MV:<attname>:<n>:<attval> Returns true or false based on the presence of <attval> in a multi-value attribute, <attname>, for the category, <catname> at the index location <n>. The value of <n> must be a positive integer between 1 and <n>, where <n> does not exceed the length of the multi-value attribute.
CAT:<catname>:MV:<attname>:<n>:DISPLAY Will return the value in the multi-value attribute, <attname>, for the category, <catname> at the index location <n>, where <n> is a positive integer between 1 and <n>. The value of <n> must be a positive integer between 1 and <n>, where <n> does not exceed the length of the multi-value attribute.
CAT:<catname>:MV:<attname>:INDEX:<attval> Will return the first index of the multi-value attribute, <attname>, for the category, <catname>, where <attval> is equal to the value contained in the multi-value attribute at index. For example, if <attval> exists in the third element of <attname>, the number 3 will be returned.
CAT:<catname>:SET:<setname>:<attname> Uses the <SET> operator allowing attribute sets to be explicitly referenced. This example will return true if the attribute <attname>, within the set <setname>, is non-blank. Only attributes within <setname> can be referenced, so if <attname> exists outside the set, it will not be found. If <attname> is a multi-value attribute, TRUE will be returned if any field within it is non-blank.
CAT:<catname>:SET:<setname>:<attname>:DISPLAY Will return the value in the attribute, <attname>, for the category, <catname>, if <attname> is an attribute within <setname>. If <attname> is a multi-value attribute, a comma-separated list of the defined values will be returned. If no defined value exists, either in a multi-value attribute, or a standard attribute, an empty list will be returned.
CAT:<catname>:SET:<setname>:<attname>:ID Will return the ID of the attribute <attname> within the set <setname>. If <attname> exists outside <setname> it will not be returned. <attname> can be either a regular or multi-value attribute.
CAT:<catname>:SET:<setname>:<attname>:<attval> Returns true or false based on whether an attribute,<attname>, exists with a value equal to <attval> in the set, <setname>. If <attname> is a multi-value attribute, all elements will be checked for equality against <attval>, if any match, TRUE will be returned.
CAT:<catname>:SET:<setname>:MV:<attname>:COUNT Will return number of multi-value attributes that have been populated.
CAT:<catname>:SET:<setname>:MV:<attname>:<n> Returns true or false for the presence of a specific non-blank element in a multi-value attribute, <attname>, within a set, <setname>, for the category, <catname> at the index location <n>. The value of <n> must be a positive integer between 1 and <n>, where <n> does not exceed the length of the multi-value attribute.
CAT:<catname>:SET:<setname>:MV:<attname>:<n>:<attval> Returns true or false for the presence of <attval> in a multi-value attribute, <attname>, within a set, <setname>, for the category, <catname> at the index location <n>. The value of <n> must be a positive integer between 1 and <n> where <n> does not exceed the length of the multi-value attribute.
CAT:<catname>:SET:<setname>:MV:<attname>:<n>:DISPLAY Will return the value in the multi-value attribute, <attname>, within a set, <setname>, for the category, <catname>, at the index location, <n>. The value of <n> must be a positive integer between 1 and <n> where <n> does not exceed the length of the multi-value attribute.
CAT:<catname>:SET:<setname>:MV:<attname>:INDEX:<attval> Will return the first index of the multi-value attribute, <attname>, within a set, <setname>, for the category, <catname> where <attval> is equal to the value contained in the multi-value attribute at index.
CAT:<catname>:MVSET:<mvsetname>:ATTRIBUTE:<attname>:INDEX:<attval> Will return the first index of the multi-value set, <mvsetname>, for the category, <catname> and attribute, <attname>, containing a value equal to the value specified in <attval>.
CAT:<catname>:MVSET:<mvsetname>:COUNT Returns the number of rows in the multi-value set, <mvsetname>.
CAT:<catname>:MVSET:<mvsetname>:<n>:<attname> Returns true or false based on whether the multi-value set, <mvsetname>, has a non-blank attribute, <attname>, at the index location <n>, within the category, <catname>.
CAT:<catname>:MVSET:<mvsetname>:<n>:<attname>:DISPLAY Will display the value of an attribute <attname> within a multi-value set, <mvsetname> at the index location <n>, within the the category, <catname>.
CAT:<catname>:MVSET:<mvsetname>:<n>:<attname>:ID Will display the ID of an attribute <attname> within a multi-value set, <mvsetname>, at the index location <n>, within the the category, <catname>.
CAT:<catname>:MVSET:<mvsetname>:<n>:<attname>:<attval> Returns true or false based on whether the multi-value set, <mvsetname>, has a attribute, <attname>, at the index location <n>, within the the category, <catname> that is equal to the value specified in <attval>.
CAT:<catname>:MVSET:<mvsetname>:<n>:MV:<attname>:COUNT Returns the number of multi-value attributes in use in the multi-value set, <mvsetname>.
CAT:<catname>:MVSET:<mvsetname>:<n>:MV:<mvattname>:<m> Returns true or false based on whether the multi-value set, <mvsetname>, at index location <n> has a multi-value attribute, <mvattname>, at the index location <m>, that is non-blank.
CAT:<catname>:MVSET:<mvsetname>:<n>:MV:<mvattname>:<m>:DISPLAY Will display the value of an multi-value attribute <mvattname>, at the index location <m>, within a multi-value set, <mvsetname> at the index location <n>, within the the category, <catname>.
CAT:<catname>:MVSET:<mvsetname>:<n>:MV:<mvattname>:<m>:<attval> Returns true or false based on whether the multi-value set, <mvsetname>, at the index location <n>, has a multi-value-attribute, <mvattname>, at the index location <m> whose value is equal to the value specified in <attval>.
CAT:<catname>:MVSET:<mvsetname>:<n>:MV:<mvattname>:INDEX:<attval> Will return the first index of the multi-value attribute, <mvattname>, within a multi-value set, <mvsetname>, for the category, <catname>, where <attval> is equal to the value contained in the multi-value attribute at index.

Examples

Example

[LL_REPTAG=DATAID CATACTION:SETVALUE:<catname>:<attname>:<new_value>:NOCOMMIT /]

[LL_REPTAG=DATAID CAT:><catname>:<attname>:DISPLAY /] will display the original value of the attribute, as the CATACTION change hasn't been committed yet.

[LL_REPTAG=DATAID CAT:<catname>:<attname>:DISPLAY:FROMCACHE /] will display the new value of the attribute from the cache.

CATACTION

Syntax

CATACTION:<options>

Description

This sub-tag provides capabilities to modify category and attribute metadata. With this tag you can:

  • add or remove a category from a node; and

  • add, set, or remove the value of a specific category attribute.

The <catname> parameter accepts the DataID or name of the category. The <attname> and <mvattname> parameters accept the attribute ID or name of the attribute. The <setname> and <mvsetname> parameters accept the ID or name of the set.

[:Index] and [:Set Index] are optional parameters to target a specific attribute in a multi-valued attribute and multi-valued set. These values default to 1 if not set. You must give a value of [:Index] when using the [:Set Index] parameter. The following table describes the syntax for the various usages of CATACTION. Each use case expects the DataID of the target node on which CATACTION will work.

The parts in angle brackets, <>, are required; the parts in square brackets, [], are optional.

Parameters

Table 1-3: Setting Attribute Values

Syntax
Description
CATACTION:SETVALUE:<catname>:<attname>[:Index][:Set Index]:<attval>
Sets the attribute value, <attval>, at a given attribute and set index. To add a new row to a multi-value set, specify the ADDSETROW keyword for the [:Set Index] parameter. Once added, the new row can be targeted using the NEWROW keyword for [:Set Index].
See CATACTION:SETVALUE:, CATACTION:SETVALUE:, and CATACTION:SETVALUE/ADDVALUE.
CATACTION:ADDVALUE:<catname>:<attname>[:Index][:Set Index]:<attval>
Appends a value, <attval>, to the end of multi-valued attribute, <attname>. This tag ignores the [:Index] parameter.
See CATACTION:SETVALUE/ADDVALUE.
CATACTION:INSERTVALUE:<catname>:<attname>[:Index][:Set Index]:<attval>
Inserts a value, <attval>, at the given attribute index. This causes the existing attribute at that index and all subsequent attributes to shift down.
:NOCOMMIT
Optionally, append :NOCOMMIT to SETVALUE, ADDVALUE, or INSERTVALUE statements to delay the category update from being committed. This option is particularly useful if there are multiple CATACTION statements updating several attributes within a category. Omit :NOCOMMIT from the final CATACTION statement for the category in order to commit the update.

Example

[LL_REPTAG_MYID CATACTION:SETVALUE:"Cat 2 Mandatory Attributes":"Attr01":"a":NOCOMMIT /]
[LL_REPTAG_MYID CATACTION:SETVALUE:"Cat 2 Mandatory Attributes":"Attr02":"b" /]

Example: This example will update a category containing two mandatory attributes. The first line will set the value of the Attr01 attribute but not commit it. The second line will set the value of the Attr02 attribute, and then commit both changes at once.

.
Table 1-4: Removing Attribute Values

Syntax
Description
CATACTION:REMOVEVALUE:<catname>:<attname>[:Index][:Set Index]
Removes the attribute at a given index. This causes the number of values in a multi-valued attribute, <attname>, to be reduced by one. If the attribute only has one value, the tag will have no effect.
Use ALL as the index value to remove all of the attribute values. The first value will be cleared, rather than removed.
CATACTION:REMOVEVALUE:<catname>:<setname>[:Set Index]
Removes the row at the given set index from the given attribute set, <setname>. This causes the number of rows in <setname> to be reduced by one. If the attribute set contains only one row then the tag will have no effect.
CATACTION:CLEARVALUE:<catname>:<attname>[:Index][:Set Index]
Sets the attribute value at a given index to null. This does not change the number of values in a multi-valued attribute.
:NOCOMMIT
Optionally, append :NOCOMMIT to REMOVEVALUE or CLEARVALUE statements to delay the category update from being committed. This option is particularly useful if there are multiple CATACTION statements updating several attributes within a category. Omit :NOCOMMIT from the final CATACTION statement for the category in order to commit the update.

Example

[LL_REPTAG_MYID CATACTION:CLEARVALUE:"MyCategory":"Attr01":NOCOMMIT /]
[LL_REPTAG_MYID CATACTION:CLEARVALUE:"MyCategory":"Attr02" /]

Example: This example will clear the values in two attributes, “Attr01” and “Attr02”, in a category, “MyCategory”. The second line of the example will clear the value of the Attr02 attribute, and then commit both changes at once.

CATINFO

Syntax

CATINFO:<format>

Description

For a valid Content Server category ID, this sub-tag will return category information according to the format parameters that have been provided.

Parameters

Format
Description
CATINFO:DEFINITION:
Returns an assoc containing the category definition. Use the LIST and ASSOC sub-tags to extract specific information.
CATINFO:VALIDVALUES:<attribute ID>
Returns a list. If the attribute is a popup the list will contain the valid values defined in the attribute definition, otherwise the list will be empty.

CONCATVAR

Syntax

CONCATVAR:<variable name>

or

CONCATVAR:<varname>:<joinstr>

or

CONCATVAR:<varname>:<joinstr>:NOBLANKS

or

CONCATVAR:<varname>:<joinstr>:UNIQUE

Description

Concatenates the value of the tag, after formatting by any previous sub-tags, with the current contents of a variable which can be referenced using the specified variable name.

The second, optional, parameter can be used to specify a character or string of characters that are used before each new value added to the variable.

If the special string @EMAIL is used as the “join string” the standard email separator as defined for the Content Server system will be automatically inserted.

The third optional parameter can only be used if the join string has been supplied. When used, the syntax is always: NOBLANKS. When NOBLANKS is used, no join strings will be added for empty data cells.

Alternatively, instead of NOBLANKS, the parameter UNIQUE can be used to ensure that only unique values are added to the concatenated variable. The first time this sub-tag is used, it behaves as if the variable already existed with an empty string. For example: [LL_REPTAG=NAME CONCATVAR:names:";" /]

Example

For example, if the values “Joe”, “Bob” and “Smith” were returned consecutively by: [LL_REPTAG=NAME CONCATVAR:names:";" /], the variable: [LL_REPTAG_%names /] would return: “Joe;Bob;Smith;”.

When this sub-tag is used, the associated tag output will not be displayed unless the SHOW sub-tag is also used.

Related Information

More information on using CONCATVAR, variable tags and other sub-tags associated with managing variables can be found in Using Variables. See also ADDVAR,SETVAR, and CURRENTVAL.

CSAPPLICATIONINSTALLED

Syntax

CSAPPLICATIONINSTALLED:<fully installed>

or

CSAPPLICATIONINSTALLED:<fully installed>:<version>

Description

Returns true or false depending on whether an application matching the application name, and optionally the version as well, has been installed. This sub-tag expects an application name in the main tag.

The <fully_installed> parameter must be either true or false. If it is passed in as false, then an additional check will be performed to see if the application has been partially installed. For example, in a multi-server environment the application has been installed on at least one server, resulting in the application's Content Server nodes being created.

Examples

Example

[LL_REPTAG_'myapplication' CSAPPLICATIONINSTALLED:'TRUE' /]

Example: Will return true if the application is installed, else it will return false.

[LL_REPTAG_'myapplication' CSAPPLICATIONINSTALLED:'FALSE' /]

Example: Will return true if the application is installed or partially installed, else it will return false.

[LL_REPTAG_'myapplication' CSAPPLICATIONINSTALLED:'TRUE':'1.0' /]

Example: Will perform the same checks as above with an additional check for the version number.

CURRENTVAL

Syntax

CURRENTVAL

Description

Displays the current value of a variable according to its position within the reportview. Variable sub-tags are processed starting at the top of the file and working towards the bottom, rows are also processed in order from 1 to <n>. Normally a variable tag, [LL_REPTAG_%test /], returns the final value of a variable regardless of where it was positioned in the reportview; however, if the CURRENTVAL sub-tag is included in a variable tag, the tag will display its value according to its position in the Reportview. For example, [LL_REPTAG_%test CURRENTVAL /] shows the current value of the variable called “test”.

The output of the CURRENTVAL sub-tag can be formatted with other sub-tags. For example, [LL_REPTAG_%test CURRENTVAL CAPITALIZE /] shows the capitalized value of the variable called “test”.

CURRENTVAL can also be used with other variable sub-tags, for example SETVAR, CONCATVAR or ADDVAR, to return the value of whatever variable they are modifying after the changes have been made. When used in this fashion, the CURRENTVAL variable must be immediately to the right of the variable sub-tag. For example, [LL_REPTAG=number ADDVAR:total CURRENTVAL /] shows the value of a variable called “total” after ADDVAR has added the number value to it.

CURRENTVAL also makes it possible to use the value of a variable to feed another action type sub-tag, for example SETFORM.[LL_REPTAG_%test CURRENTVAL CAPITALIZE SETFORM:field1 /] takes the current value of the variable called “test”, capitalizes it, and uses it to set a form field called “field1”.

Related Information

More information on using CURRENTVAL, variable tags and other sub-tags associated with managing variables can be found in Using Variables. See also, SETVAR, CONCATVAR, ADDVAR and SETFORM.
