As with all Content Server items, WebReports have a Properties page, accessible from the item's function drop down menu. In addition to the standard Content Server tabs there are some unique tabs that allow control over WebReport sources, destinations, constants and parameters. The basic features of these tabs, and the Specific tab are described here.

Specific

The Specific tab for a WebReport is similar in functionality to the Specific tab for a document. The document properties such as Current Version, Max Versions and MIME Type are used to manage version control for the Reportview associated with the WebReport. In addition to these fields, WebReports has the following unique fields:

  • Run As: this option allows the WebReport in question to execute as a specific user, irrespective of the userid of the person executing the report. Because this capability can surface data the person running the report might not normally be able to see, the ability to change this field is controlled by a privilege with the Usage Type of “WebReports” and the Usage Name of “Run As”.

    For more information, see WebReports Run As Feature.

  • Run/Export Audits Enabled: this option is cleared by default and allows each WebReport to have additional audit events enabled so that every RUN or EXPORT event will generate an entry in the AUDIT log.

     

    Important:
    This option is cleared by default due to the volume of audits that can be generated if this feature is used without careful consideration.

     

  • Oscript Scripting Enabled: this option will only appear when the Reportview includes tags related to the Server Side Scripting feature. It is used to enable or disable the execution of any scripts that may be defined and executed with the Reportview.

    For more information on this feature see Server Side Scripting.

Constants

The Constants tab allows the creation and management of values that can be used in the Reportview, or other designated fields. For more information, see Procedures Defining How to Use Constants.

Destination

The Destination tab allows the developer to specify the location to which the report is delivered. This could be email, workflow initiation, or any of several other destinations. For more information, see Exporting Report Data.

Parameters

The Parameters tab allows the developer to define parameter behavior for the WebReport. This is used to control things like the prompt order, the default value and whether a parameter is mandatory or not. For more information, see WebReport Parameters Procedures.

Source

The Source tab allows the developer to specify different types of data sources and the format in which the source data will appear. For more information, see Data Source Configuration.
