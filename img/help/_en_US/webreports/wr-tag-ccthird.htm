This section details the content control tags: ELSE, ENDIF, ENDROW, ENDSCRIPT, EXCLUDEADDITEM, EXCLUDECHANNEL, EXCLUDEFOOTER, EXCLUDEHEADER, EXCLUDEHTML, EXCLUDELOCATIONDISPLAY, EXCLUDEMENU, EXCLUDESEARCH, EXCLUDETITLE and EXITIF.

ELSE

Syntax

[LL_WEBREPORT_ELSE /]

Description

This tag is optional, and, if used, it must be used between IF and ENDIF tags. If included, then everything between the ELSE tag and the ENDIF tag are included if the specified condition evaluates to false.

Example

[LL_WEBREPORT_IF (expression) /]
[LL_WEBREPORT_ELSE /] (optional)
[LL_WEBREPORT_ENDIF /]

Related Information

See also IF and ENDIF for more detail.

ENDIF

Syntax

[LL_WEBREPORT_ENDIF /]

Description

Tag used to mark the end of an [LL_WEBREPORT_IF /] clause.

Example

[LL_WEBREPORT_IF (expression) /]
[LL_WEBREPORT_ELSE /] (optional)
[LL_WEBREPORT_ENDIF /]

Related Information

See also IF and ELSE for more detail.

ENDROW

Syntax

[LL_WEBREPORT_ENDROW /]

Description

Tag used to mark the end of the report row template.

When online editing is used, this tag is automatically inserted and is not visible in the source of the reportview.

ENDSCRIPT

Syntax

[LL_WEBREPORT_ENDSCRIPT /]

Description

This tag ends a defined block of server side Oscript that can be called by the CALL tag.

Example

[LL_WEBREPORT_STARTSCRIPT NAME:<script name> /]
function String anyName(Dynamic c, List args)
String result
...
return result
end
[LL_WEBREPORT_ENDSCRIPT /]

Related Information

See also STARTSCRIPT.

EXCLUDEHTML

Syntax

[LL_WEBREPORT_EXCLUDEHTML /]

Description

Excludes all the HTML, JavaScript and Style definitions that Content Server uses to wrap any given page. This can be used to good effect when exporting, for example XML or CSV as the Content Server page wrapping is no longer relevant. When this option is used all other EXCLUDE tags will be ignored as they are not relevant unless in the context of a Content Server page.

 

Note:
When using [LL_WEBREPORT_EXCLUDEHTML /] in a WebReport that is also using the Using the Predefined Functions the following variable will need to be defined in a script block. The reason is the baseURL variable is defined within the Content Server page code that is inserted into the WebReport output but this is excluded by the tag. The following code will ensure this variable is defined and prevent the AJAX (Asynchronous JavaScript and XML) library from breaking.

 

Example

<script>
   baseURL = "[LL_REPTAG_URLPREFIX /]";
</script>

EXITIF

Syntax

[LL_WEBREPORT_EXITIF /]

or

[LL_WEBREPORT_EXITIF (expression) [:USECASE] /]

Parameters

Format for (expression): "parmA" <operator> "parmB" <optional clauses>

Description

Causes the processing of rows to stop when a condition is met. If the specified expression evaluates to TRUE, then no more rows will be included in the output. The current row being processed is not included if the expression evaluates to TRUE.

 

  1. This is the only conditional tag where it is safe to use the ROWNUM tag in an expression. This tag only works in the row section, in other words, between the STARTROW and ENDROW tags.

  2. This tag does not change the TOTALROWS tag which will always reflect the number of rows that were actually available from the database table. To access the number of rows which are actually returned, those that pass the conditional test, the ACTUALROWS tag should be used. See the specific syntax information for these tags.

  3. parmA and parmB can be either text or another report tag, all other tags are resolved first.

  4. The parameters must be enclosed inside double quotation marks.

  5. See the section on Logical Expressions for a list of Operators.

  6. Additional, optional clauses are added using one of the following operators:

    • AND (logical AND)

    • && (alternative AND)

    • OR (logical OR)

    • || (alternative OR)

 

Examples

Example

For example: [LL_WEBREPORT_EXITIF "[LL_REPTAG_ROWNUM /]" == "10" /]

In this example, rows numbered 1 to 9 will be output but not row 10.

Example

[LL_WEBREPORT_EXITIF "[LL_REPTAG_&var1 /]" != "[LL_REPTAG=DATAID /]" /]
[LL_WEBREPORT_EXITIF "[LL_REPTAG_ROWNUM /]" == "101" /]
[LL_WEBREPORT_EXITIF "[LL_REPTAG=dataid /]" NOTCHILDOF "[LL_REPTAG_&rootid /]" OR "[LL_REPTAG=parentid /]"
NOTCHILDOF "[LL_REPTAG_&rootid /]" /]

Related Information

For more information, see:

  • See the Notes in the Description section of INCLUDEIF.

  • See the section on Logical Expressions for a list of Operators.
