In addition to being displayed in the browser, the resulting output from running a WebReport can be exported to any of the following locations:

  • Content Server: exporting to Content Server itself is only available if you are the admin user.

  • Exporting to Desktop

  • Exporting to a Content Server New Document

  • Exporting to a Content Server New Document Version

  • Exporting to Email

  • Exporting to a WebForm

  • Exporting to a Workflow

  • Exporting to the Server

  • Exporting to an FTP Server

You can access the export functionality by:

  • selecting Export from the functions menu, or

  • by setting Export to be the default report behavior. You set this from the functions menu by selecting Properties, then selecting Destination.

When exporting to either Content Server or e-mail it is possible to separate the web browser from the report so the user can carry on working whilst large reports are executed in the background. This behavior is accessed by selecting the Run in Background box.

Configuring Default WebReport Behavior

It is possible to change the default action resulting from selecting a WebReport in Content Server, so that rather than running the report and displaying the output in the browser, the output is redirected to an export operation. For example, a WebReport could be configured to add a new version to a Content Server document each time that WebReport is selected. This behavior is specified from the Destination tab which you access from the WebReport's functions menu by selecting Properties, then selecting Destination. The fields for specifying this default behavior are exactly the same as described for explicit exports as described in the following pages.
