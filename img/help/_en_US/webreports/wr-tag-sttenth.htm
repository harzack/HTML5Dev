This section details the sub-tags: LENGTH, LIST, LLURL, LOWER and LPAD.

LENGTH

Syntax

LENGTH

Description

Returns the length of the data returned in the main tag. For example, if the data resolved by the main tag was 10 characters long, using this sub-tag would cause the characters 10 to be returned instead. This sub-tag can be useful for determining data validity, greater than a minimum number of characters, and HTML SIZE parameters.

Related Information

For more information, see AVERAGE, MAX, MIN, and SUM.

LIST

Syntax

LIST:<index>

Description

For a Content Server <LIST> data type the specified index can be retrieved. Index starts at 1.

Example

For example, [LL_REPTAG_1 LIST:3> /] would return the third item in a list.

Related Information

See also: ASSOC, RECORD, RECARRAY, and TOLIST.

LLURL

Syntax

LLURL:<parms>

Description

This sub-tag allows URLs to be easily built and/or snippets of Content Server functionality to be easily inserted. For example, if your report needs to contain an image representing the node type, the following could be used: [LL_REPTAG=DATAID LLURL:GIF /] resulting in if DATAID was a Word document. For this sub-tag to function correctly, the data returned from the main tag must be a Content Server object ID, for example, the DATAID column of the DTREE table.

 

Note:
For URLs, the default behavior is that a URL will be built without verifying that a valid DataID has been provided, and without verifying if the user running the WebReport has sufficient permissions to execute the URL. To force checking of a valid DataID, and that the user has at least See Contents permission, add :PERMCHECK to the end of the LLURL syntax. For example, LLURL:BROWSE:PERMCHECK. For more specific permission levels, the PERMCHECK sub-tag can be used with the [LL_WEBREPORTS_IF ... /] statement.

Inserts a workflow function menu. Requires that the main tag returns a subworkid.
[LL_REPTAG=SUBWORKID LLURL:WFFUNCTIONMENU /]

Example

Generally speaking this sub-tag can be broken down into two types of output. One group that returns part of a URL or link which can be used within your own HTML code, examples of this include LLURL:BROWSE and LLURL:DEFAULTLINK. In this situation the sub-tag would usually be used in conjunction with URLPREFIX. Example of building an HTML anchor tag:

<a href=      "[LL_REPTAG_URLPREFIX /][LL_REPTAG=DATAID LLURL:BROWSE /]">my browse link</a>

Which, following execution should resolve to something like:

<a href= "http://localhost/ll9.5sp1/livelink.exe?func=ll&objId=41680&objAction=browse&sort=name"> my browse link</a>

The second group insert complete images or links as they would appear in Content Server. Examples of this second group include LLURL:FUNCTIONMENU, LLURL:DROPDOWNLIST, and LLURL:GIF. A function menu might be added with the following syntax: [LL_REPTAG=DATAID LLURL:FUNCTIONMENU /] and if viewed in a WebReport the output of this tag would simply appear as a fully operational function menu ( [Functions] ). However, in order for the menu to operate WebReports inserts something similar to the following:

<a href="#" onclick="javascript:showFunctionMenu2(document.location.search.slice(document.location.search.indexOf('&nexturl')),41842,event);return false;"><img src="/img_main/actions.gif" id="x41842" border="0" title="Functions" name="x41842"></a>
<div id="z41842" style="visibility:hidden; position: absolute; min-width:1px; text-align: left; background-color: #eeeeee;"></div>

Related Information

For more information, see URLPREFIX, CAT, and TOJSON.
