This section details the content control tag JSLIBS.

JSLIBS

Syntax

[LL_WEBREPORT_JSLIBS /]

[LL_WEBREPORT_JSLIBS <list of libraries> /]

Description

This tag supports one or more quoted library selectors that determine which JavaScript libraries to insert into the output. An example of a library selector is COREUTIL.

Parameters

The libraries currently available for Content Server 10.0.0 and greater are:

 

Note:
COREMENU and PAGINATION are obsolete. Use BROWSE and COREAJAX instead.

 

BASE
(webdav/global.js, menu.js)
Basic libraries used for standard Content Server menus and functions.
BASICBROWSE
(checkall.js)
Functionality related to multi-object selection.
BROWSE
(webnode/browse.js and webnode/browsecoretable_<locale>.js
An example of <locale> is browsecoretable_en_US.js.
Functions used for browseview.
When the page loads, the browse.js library requires an additional function that is in the core ajax_dhtml_util.js library. If [LL_WEBREPORT_EXCLUDEHTML /] is being used, you will need to call the COREAJAX library first to avoid JavaScript errors. For example, [LL_WEBREPORT_JSLIBS "COREAJAX" "BROWSE" /]
You may also need to include the core jQuery libraries in this scenario. These are located in the <SupportDir>/core/ directory.
COREAJAX
(core/ajax_dhtml_util.js)
Ajax utilties.
COREUTIL
(core/ot_utilities.js)
Miscellaneous utilities.
RKTJSLIBS
(webreports/rktjslibs.js)
RKT JavaScript loader.
SEARCHBROKER
(websbroker/search.js, websbroker/sortwidget.js)
Search result utilities
