This section details the sub-tags: WFACTION, WFATTR, WFCOMMENTS, WFDBLOOKUP, WFFORM, WFINFO, WFTASKACTION, and WFTASKINFO.

WFACTION

Syntax

WFACTION:<operation>

Description

This sub-tag offers powerful capabilities to manipulate workflows in large volumes at the server. Users are only able to manipulate workflows with the same level of permissions as through the regular user interface.

Parameters

Action
Description
WFACTION:ARCHIVE
Changes the status of a Stopped or Completed workflow to Archived.
WFACTION:DELETE
Deletes a Stopped or Completed workflow.
WFACTION:RESUME
Changes the status of a Suspended workflow to Executing.
WFACTION:STOP
Changes the status of an Executing or Suspended workflow to Stopped.
WFACTION:SUSPEND
Changes the status of an Executing workflow to Suspended.

Example

[LL_REPTAG=WORKID WFACTION:SUSPEND /] changes the status of an Executing workflow, identified by the WORKID column returned from the data source, to Suspended.

Related Information

For more information, see NODEACTION:operation and USERACTION:operation.

WFATTR

Syntax

WFATTR:<format>

Description

For a valid Content Server Workflow, SUBWORKID, the WFATTR sub-tag will return attribute information for that workflow. If the argument :USECACHE is appended to the end of the tag then the SUBWORKID will be ignored, and the workflow data will be loaded from the cache instead. This can be used if you wanted to display attribute data in a Form PowerView on the workflow start step.

Parameters

Format
Description
WFATTR
When no optional parameters are defined, this sub-tag returns TRUE or FALSE based on the presence of the parameters package in the workflow.
WFATTR:<attname>
Returns TRUE or FALSE based on whether the named workflow attribute has a non-blank value.
WFATTR:<attname>:DISPLAY
Will display the value in the named workflow attribute, <attname>. If <attname> is a multi-value, all values will be shown in a comma-separated format.
WFATTR:<attname>:ID
Will display the ID of the named attribute, <attname>.
WFATTR:<attname>:COUNT
Will return the number of populated entries in a multi-value workflow attribute, <attname>.
WFATTR:<attname>:<attval>
Returns TRUE or FALSE based on whether the specified value, <attval>, is found in the workflow attribute, <attname>.
WFATTR:<attname>:<n>
Returns TRUE or FALSE based on whether there is a value in the <n>th element of a multi-value workflow attribute, <attname>.
WFATTR:<attname>:<n>:DISPLAY
Will display the <n>th value in the named multi-value workflow attribute, <attname>.
WFATTR:<attname>:<n>:<attval>
Returns TRUE if the value <attval> is found at the index <n> within a multi-value workflow attribute, <attname>. Otherwise, this sub-tag returns FALSE.
WFATTR:<attname>:INDEX:<attval>
Will return the index of the specified value, <attval>, within a multi-value workflow attribute, <attname>.

Example

[LL_REPTAG_'' WFATTR:<attname>:DISPLAY:USECACHE /] will display the value of the <attname> attribute in a PowerView if the workflow hasn't been initiated yet.

WFCOMMENTS

Syntax

WFCOMMENTS:<format>

Description

For a valid Content Server Workflow, SUBWORKID, the WFATTR sub-tag will return the comments data in a RecArray.

Parameters

Format
Description

WFDBLOOKUP

Syntax

WFDBLOOKUP:<table ref>:<column ref>

or

WFDBLOOKUP:<table ref>:<column ref>:<column select>

or

WFDBLOOKUP:<table ref>:<column ref>:<column select>:MULTIMATCH

Description

This sub-tag is used to look up data from any database table created by the Content Server WebForms module. It expects a valid key to be supplied by the main, data tag and accepts parameters to specify the appropriate table, <table ref>, and column, <column ref>, to reference. The key, table, and column information are then used to look up one or more rows from the specified WebForms table.

An additional parameter, <column select>, can also be included to specify that a particular column of data is returned.

If more than one column is returned, the data is provided in an Oscript RECARRAY structure, a proprietary Content Server data structure. This RECARRAY structure can be dereferenced using the RECARRAY sub-tag or converted into JSON format using a TOJSON sub-tag. If only one match is found, the RECARRAY will only have one record, but if more than one row of data is returned, the data is provided in multiple records within the RECARRAY structure.

The required <table ref> parameter can be either a form template ID or a form ID. Either of these objects is used to determine the correct SQL table name.

 

Note:
The user running the WebReport must have the appropriate level of permissions to be able to See Contents on the form/template objects.

 

The required <column ref> parameter is an identifier that specifies the name of the column that will be searched for the key supplied by the main, data tag.

The optional <column select> parameter is used to narrow the data returned from the query down to a single, specific column.

Normally, if only one column is being returned, this sub-tag only returns the most recently added item in a single field of text, in other words, not in a RECARRAY. However, the optional MULTIMATCH parameter allows more than one row of data to be returned, even though only one column has been selected. If no column has been selected, the data is always returned in a Recarray structure. This structure can be dereferenced using the RECARRAY sub-tag, or the TOJSON sub-tag can be used to convert the structure into JSON format, which the client can then use to create JavaScript variables.

The various methods of returning data are described in the following bullets. Generally speaking, a RECARRAY is always returned unless the optional <column select> parameter has been specified without a MULTIMATCH option, which results in a single text field.

  • If the optional <column select> parameter has not been specified, all columns for the data source are returned for as many rows as are matched by the query.

  • If the optional <column select> parameter has been specified, only the data for the specified column in the first matched row is returned.

  • If the optional <column select> parameter has been specified with the optional MULTIMATCH parameter, then only the specified column is returned for any matched rows. This allows for more than one row of data, but the data is always returned in a RECARRAY, even if only one row is found.

Related Information

For more information, see RECARRAY and TOJSON.

WFFORM

Syntax

WFFORM:<parms>

Description

Returns information about the workflow form fields associated with a specific workflow, SUBWORKID. This sub-tag supports both sets and multi-value fields through use of the MV and SET operators.

Parameters

Syntax
Description
WFFORM:<formname>:<fieldname>
Returns TRUE or FALSE based on whether a field, <fieldname>, exists with a non-blank value in the form, <formname>. If a set is specified in <fieldname>, a single entry in any one of the set's field values will cause this to return TRUE. If <fieldname> is a multi-value field, a single entry in any one of those fields will also cause this to return TRUE. <fieldname> can also be nested within a set, either as a regular field or as a multi-value field.
WFFORM:<formname>:<fieldname>:DISPLAY
Will return the value in the field, <fieldname>, for the form, <formname>. If <fieldname> is a set or a multi-value field, a comma-separated list of values will be returned.
WFFORM:<formname>:<fieldname>:ID
Will return the ID of the field, <fieldname>, for the form, <formname>, if it is present. If <fieldname> exists inside a set, it will be returned if no field matching <fieldname> exists at the top level in the form. If <fieldname> exists at the top level of the form, and within a set of the same form, the field ID at the top level, not within the set, will take precedence.
WFFORM:<formname>:<fieldname>:<fieldval>
Returns TRUE or FALSE based on whether a field, <fieldname>, exists with a value equal to <fieldval> in the form, <formname>. <fieldname> may be a field within a set, including a multi-value, or any element within a multi-value field.
WFFORM:<formname>:MV:<fieldname>:COUNT
Will return the number of multi-value fields in use.
WFFORM:<formname>:MV:<fieldname>:<n>
Returns TRUE or FALSE based on the presence of a specific non-blank element in a multi-value field, <fieldname>, for the form, <formname> at the index location <n>. The value of <n> must be a positive integer between 1 and <n>, where <n> does not exceed the length of the multi-value field. If <n> exceeds the length of the multi-value field, an empty string will be returned.
WFFORM:<formname>:MV:<fieldname>:<n>:<fieldval>
Returns TRUE or FALSE based on the presence of <fieldval> in a multi-value field, <fieldname>, for the form, <formname> at the index location <n>. The value of <n> must be a positive integer between 1 and <n>, where <n> does not exceed the length of the multi-value field.
WFFORM:<formname>:MV:<fieldname>:<n>:DISPLAY
Will return the value in the multi-value field, <fieldname>, for the form, <formname> at the index location <n>, where <n> is a positive integer between 1 and <n>. The value of <n> must be a positive integer between 1 and <n>, where <n> does not exceed the length of the multi-value field.
WFFORM:<formname>:MV:<fieldname>:INDEX:<fieldval>
Will return the first index of the multi-value field, <fieldname>, for the form, <formname>, where <fieldval> is equal to the value contained in the multi-value field at index. For example, if <fieldval>exists in the third element of <fieldname>, the number 3 will be returned.
WFFORM:<formname>:MV:<setname>:COUNT
Will return the number of multi-value sets in use.
WFFORM:<formname>:SET:<setname>:<fieldname>
Uses the SET operator allowing field sets to be explicitly referenced. This example will return TRUE if the field, <fieldname>, within the set, <setname>, is non-blank. Only fields within <setname> can be referenced, so if <fieldname> exists outside the set, it will not be found. If <fieldname> is a multi-value field, TRUE will be returned if any field within it is non-blank.
WFFORM:<formname>:SET:<setname>:<fieldname>:DISPLAY
Will return the value in the field, <fieldname>, for the form, <formname>, if <fieldname> is a field within <setname>. If <fieldname> is a multi-value field, a comma-separated list of the defined values will be returned. If no defined value exists, either in a multi-value field, or a standard field, an empty list will be returned.
WFFORM:<formname>:SET:<setname>:<fieldname>:ID
Will return the ID of the field, <fieldname>, within the set, <setname>. If <fieldname> exists outside <setname>, the ID of the field will not be returned. <fieldname> can be either a regular or multi-value field.
WFFORM:<formname>:SET:<setname>:<fieldname>:<fieldval>
Returns TRUE or FALSE based on whether a field, <fieldname>, exists with a value equal to <fieldval> in the set, <setname>. If <fieldname> is a multi-value field, all elements will be checked for equality against <fieldval>, if any match, TRUE will be returned.
WFFORM:<formname>:SET:<setname>:MV:<fieldname>:COUNT
Will return the number of multi-value fields that have been populated.
WFFORM:<formname>:SET:<setname>:MV:<fieldname>:<n>
Returns TRUE or FALSE based on the presence of a specific non-blank element in a multi-value field, <fieldname>, within a set, for the form, <formname> at the index location <n>. The value of <n> must be a positive integer between 1 and <n>, where <n> does not exceed the length of the multi-value field.
WFFORM:<formname>:SET:<setname>:MV:<fieldname>:<n>:<fieldval>
Returns TRUE or FALSE based on the presence of <fieldval> in a multi-value field, <fieldname>, within a set, for the form, <formname> at the index location <n>. The value of <n> must be a positive integer between 1 and <n>, where <n> does not exceed the length of the multi-value field.
WFFORM:<formname>:SET:<setname>:MV:<fieldname>:<n>:DISPLAY
Will return the value in the multi-value field, <fieldname>, within a set, <setname>, for the form, <formname> at the index location <n>. The value of <n> must be a positive integer between 1 and <n>, where <n> does not exceed the length of the multi-value field.
WFFORM:<formname>:SET:<setname>:MV:<fieldname>:INDEX:<fieldval>
Will return the first index of the multi-value field, <fieldname>, within a set, <setname>, for the form, <formname>, where <fieldval> is equal to the value contained in the multi-value field at index.
WFFORM:<formname>:MVSET:<setname>:<n>:<fieldname>
Returns TRUE or FALSE based on whether the multi-value set, <setname>, has a non-blank field, <fieldname>, at the index location <n>, within the form, <formname>.
WFFORM:<formname>:MVSET:<setname>:<n>:<fieldname>:DISPLAY
Will display the value of a field, <fieldname>, within a multi-value set, <setname>, at the index location <n>, within the the form, <formname>.
WFFORM:<formname>:MVSET:<setname>:<n>:<fieldname>:ID
Will display the ID of a field, <fieldname>, within a multi-value set, <setname>, at the index location <n>, within the the form, <formname>.
WFFORM:<formname>:MVSET:<setname>:<n>:<fieldname>:<fieldval>
Returns TRUE or FALSE based on whether the multi-value set, <setname>, has a field, <fieldname>, at the index location <n>, within the form, <formname>, that is equal to the value specified in <attval>.
WFFORM:<formname>:MVSET:<setname>:<n>:MV:<fieldname>:COUNT
Returns the number of multi-value fields in use in the multi-value set, <setname>.
WFFORM:<formname>:MVSET:<setname>:<n>:MV:<fieldname>:<m>
Returns TRUE or FALSE based on whether the multi-value set, <setname>, at index location, <n>, has a multi-value field, <fieldname>, at the index location, <m>, that is non-blank.
WFFORM:<formname>:MVSET:<setname>:<n>:MV:<fieldname>:<m>:DISPLAY
Will display the value of an multi-value field <fieldname>, at the index location <m>, within a multi-value set, <setname>, at the index location <n>, within the the form, <formname>.
WFFORM:<formname>:MVSET:<setname>:<n>:MV:<fieldname>:<m>:<fieldval>
Returns TRUE or FALSE based on whether the multi-value set, <setname>, at the index location <n>, has a multi-value-field, <fieldname>, at the index location <m>, whose value is equal to the value specified in <attval>.
WFFORM:<formname>:MVSET:<setname>:<n>:MV:<fieldname>:INDEX:<fieldval>
Will return the first index of the multi-value field, <fieldname>, within a multi-value set, <setname>, for the form, <formname>, where <fieldval> is equal to the value contained in the multi-value field at index.

Example

[LL_REPTAG=SUBWORKID WFFORM:<formname>:<fieldname>:DISPLAY /] will return the field value stored in the form <formname> and the field <fieldname> for any field type. If <fieldname> is a multi-value field or a set, a comma-separated list of values will be returned.

Syntax

WFINFO:<format>

Description

For a valid Content Server Workflow, SUBWORKID, this sub-tag will return information associated with that executing workflow.

Parameters

Format
Description

WFTASKACTION

Syntax

WFTASKACTION:<format>

Description

This sub-tag will update data for a workflow step. The main tag must contain a list containing the workflow task's <workID>, <subworkID> and <taskID>.

Parameters

Format
Description
WFTASKACTION:REASSIGN:<user/group>
Will reassign the workflow step to the user or group specified in <user/group>. Acceptable arguments are the userID, user login name, or groupID.

Example

An example of the main tag is:

[LL_REPTAG_{ <workID>, <subworkID>, <taskID>} /]

An example of the main tag with the sub-tag is:

[LL_REPTAG_{98765, 56789, 3} WFTASKACTION:REASSIGN:1234 /] will reassign the workflow step with a workID of 98765, a subworkID of 56789, and a taskID of 3, to the user or group with the ID of 1234.

Related Information

The WFTASKACTION sub-tag is one of a handful of sub-tags that are called Action Sub-Tags. Additional information on using this, and other action sub-tags, can be found in Action Sub-tags.

WFTASKINFO

Syntax

WFTASKINFO:<format>

Description

This sub-tag will return information associated with a valid Content Server workflow task. The main tag must contain a list containing the workflow task's <workid>, <subworkid> and <taskid>.

Parameters

Format
Description
WFTASKINFO:ATTACHMENTSINFO
Returns an Assoc containing information about the Attachments section of the task.
WFTASKINFO:ATTRIBUTESINFO
Returns an Assoc containing information about the Attributes section of the task.
WFTASKINFO:BUTTONS
Returns a List containing information about the disposition buttons for the task.
WFTASKINFO:COMMENTSINFO
Returns an Assoc containing information about the Comments section of the task.
WFTASKINFO:DELEGATE
Returns TRUE if the current user has permission to delegate the task, else returns FALSE.
WFTASKINFO:FORMSINFO
Returns an Assoc containing information about the Forms section of the task.
WFTASKINFO:PANEINDEX:<Work Item Name>
Returns an integer containing the PaneIndex value corresponding to the work item.
WFTASKINFO:REVIEW
Returns TRUE if the current user has permission to send the task for review, else returns FALSE.
WFTASKINFO:TASKINFO
Returns a Record containing general information about the task.
WFTASKINFO:WEBREPORTSINFO
Returns an Assoc containing information about the WebReports section of the task.
WFTASKINFO:WORKITEMS
Returns a List containing information about the workitems that make up the task.

Example

[LL_REPTAG_'{<workid>, <subworkid>, <taskid>}' /]

Related Information

For more information, see ASSOC, LIST, and RECORD.
