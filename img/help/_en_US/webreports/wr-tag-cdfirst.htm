This section details the column data tag LL_REPTAG=column name, [LL_REPTAG_!variablename /], LL_REPTAG_$constname, LL_REPTAG_%variablename, LL_REPTAG_&parmname, LL_REPTAG_column number, LL_REPTAG_literal text, and LL_REPTAG_@function.

LL_REPTAG=column name

Syntax

[LL_REPTAG=<column name> /]

Description

Causes WebReports to insert the source data specified by the <column name> into the output report. The tag should only be used in the row section of a WebReport unless it is being used as an argument for the INSERTJSON tag. The optional list of sub-tags is used to specify formatting options. These sub-tags should be separated by spaces. See the sub-tag definition table for information on these tags.

The use of single or double quotes to delimit a column name is supported. Using quotes is only mandatory for referencing data source column names that contain a space character. This also applies if the LiveReport renames a column to a name containing a space character. Double quotes should not be used if the LL_REPTAG is inside a conditional tag.

Example

[LL_REPTAG=NAME /]
[LL_REPTAG="NAME" /]
[LL_REPTAG="FIRST NAME" /]
[LL_WEBREPORT_INCLUDEIF "John" != "[LL_REPTAG='FIRST NAME' /]" /]

[LL_REPTAG_!variablename /]

Syntax

[LL_REPTAG_!<variablename> /]

Description

This tag is a shortcut to [LL_REPTAG_%<variablename> CURRENTVAL /], LL_REPTAG_%variablename.

Example

The following two tags are equal:

[LL_REPTAG_%myVar CURRENTVAL /]
[LL_REPTAG_!myVar /]

Related Information

For more information, see ADDVAR, CONCATVAR, and CURRENTVAL.

LL_REPTAG_$constname

Syntax

[LL_REPTAG_$<constname> /]

Description

This tag provides a way to access constants that have been defined using the Constants tab of a WebReports Properties page. For more information, see WebReports Constants. The constant name in the Reportview should exactly match the name that has been specified on the Constants page. The tag is replaced by the value that has been defined for the corresponding constant.

Constants are particularly useful for referencing Sub-WebReports and other Content Server objects as it means that objectids don't need to be hardcoded within the Reportview. For more information, see Calling Sub-WebReports.

Example

For example, if a constant called MyConst has been defined with a constant value of 12345, the following tag will return '12345':

[LL_REPTAG_$MyConst /]

Related Information

For more information, see Using Sub-WebReports and WebReports Constants.

LL_REPTAG_%variablename

Syntax

[LL_REPTAG_%<variablename> /]

Description

This tag provides a way to reference variables that have been defined by using SETVAR, ADDVAR or CONCATVAR.

Example

If, for example, ADDVAR had been used in the row section to accumulate a mathematical total in a variable called MyVar, the final total could be accessed by using:

[LL_REPTAG_%MyVar /]

Related Information

More information on using variable tags and the associated sub-tags can be found in Using Variables. For more information, see SETVAR, ADDVAR, CONCATVAR and CURRENTVAL.

LL_REPTAG_&parmname

Syntax

[LL_REPTAG_&<parmname> /]

Description

This tag provides a means of accessing any parameter in the URL used to invoke the WebReport. Any values in the URL prefixed by an ampersand “&” can be referenced in this way. This means that existing parameters such as &ObjAction can be referenced as well as any number of bespoke parameters.

Example

For example, if the following URL were executed:

...?func=ll&objId=47478&objAction=RunReport&myparm=testdata&nexturl=....

The parameter myparm can be accessed by using [LL_REPTAG_&myparm /]. This tag will resolve to the value in the URL, in the example above that value is testdata, where it can be used as output or as part of a conditional statement.

For example, [LL_WEBREPORT_INCLUDEIF "[LL_REPTAG_&myparm /]" == "testdata" /]

If a parameter is referenced in the Reportview but is not provided in the URL a blank string will be inserted in the output unless a parameter default is set on the Parameters tab of a WebReports Properties page.

Related Information

For more information, see Using Parameters.

LL_REPTAG_column number

Syntax

[LL_REPTAG_<column number> /]

Description

Reference column data with the column number. Content Server columns are numbered from left to right starting at 1.

 

Note:
Pay particular attention to the use of an underscore “_” instead of an equal “=” sign in this form of LL_REPTAG. This roughly matches the Content Server WebForm tag model. This form of the tag can be useful if one Reportview is to be used in multiple WebReports where the column titles may vary.

 

LL_REPTAG_“literal text”

Syntax

[LL_REPTAG_“<literal text>” /]

or

[LL_REPTAG_'<literal text>' /]

Description

This tag provides a means of specifying a string of text to be inserted in the Reportview. Any sub-tags included in this tag definition will operate on the text specified between the two quotes. Either double quotes or single quotes may be used but must be matched.

Example

[LL_REPTAG_"this text will be displayed in capitals" UPPER /]

LL_REPTAG_@function

Syntax

[LL_REPTAG_@<function> /]

[LL_REPTAG_@<function><reference options> /]

Description

This tag type provides data functions. The result of each function is inserted into the output; however, these tags are currently only supported in the header or footer of a Reportview. In other words, before the STARTROW or after the ENDROW tag.

For each function there are optional characters following the function name which reference either a specific row of data or a column or both. These reference characters are:

  • [r]: this optional parameter, only used by @DATA, specifies the row index.

     

    Note:
    The reserved word LAST can be used instead of r.

     

    [r] can be followed by:

  • _c: specifies a column number, or

  • _<colname>: specifies a column name.

When one of the functions below is specified, the function will be executed against which ever data has been referenced. For any math tags, if non-numeric values are found in the column then an error is returned.

 

Note:
There are sub-tags which will help to format the values returned by these functions.

 

Parameters

Parameter
Description
[LL_REPTAG_@AVERAGE_<column> /]
Returns the average of all valid numbers in the specified column.
[LL_REPTAG_@DATA[r]_<column> /]
Returns a single cell of data specified by the row and column.
[LL_REPTAG_@MIN_<column> /]
Returns the lowest, minimum, value from all the values in the specified column.
[LL_REPTAG_@MAX_<column> /]
Returns the greatest, maximum, value from all the values in the specified column.
[LL_REPTAG_@RESPONSEDATA.<data field name> /]
Returns fields returned by Content Server in the response to a request. These fields are normally used in Content Server pages to display data. For example, some Content Server browse pages use a field in the response from a browse request called: hasStagedContent. This value could be referenced in a WebReport using: [LL_REPTAG_@RESPONSEDATA.hasStagedContent /]
[LL_REPTAG_@SUM_<column> /]
Returns the sum of all the values in the specified column.

Example

If the average of column 2 is required, the following syntax could be used: [LL_REPTAG_@AVERAGE_2 /]

If the sum of a column called “hours” is required, the following syntax could be used: [LL_REPTAG_@SUM_hours /]

If the contents of a column called dataid from the last row of data is required, the following syntax could be used: [LL_REPTAG_@DATA[last]_dataid /]

If the contents of a column 2 from row 1 of the data is required, the following syntax could be used: [LL_REPTAG_@DATA[ 1 ]_2 /]

If the contents of a field in the Content Server request response is required, the following syntax could be used: [LL_REPTAG_@RESPONSEDATA.HASSTAGEDCONTENT /]
