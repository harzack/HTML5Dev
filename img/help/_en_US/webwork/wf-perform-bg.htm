When Workflow processing reaches an email-enabled step, Content Server sends a notification message to the users assigned to work on the step. In addition to notifying step assignees that the step is ready for work to begin, the notification message provides instructions about how to complete the task. Depending on the way the step assignment is defined, the message can contain instructions about how to work with attachments, forms, and step comments. Content Server includes any Workflow attachments to the step assignment message as email attachments, and may include optional step features, such as permission to delegate the step or send the step for review. For example, if a step is late or configured as high priority, the email message is sent with high priority.

 

Important:
Before you can receive Workflow-related email messages, you or the Content Server administrator must configure eLink settings for Workflow events. For more information about eLink settings, see Configuring eLink Alerts and Notifications

 

If you are assigned a step by email, you can review the assignment instructions, complete the assignment work, and then indicate that the work is complete by responding to the step assignment email message. Although some email-enabled steps can be completed entirely through email, you have the option to work on the step in Content Server as you would a step that is not email-enabled. All step notification email messages include a link to the step page in Content Server, where you can view or work with the step. Also, email-enabled steps appear in your My Assignments page along with other assignments.

Some email-enabled steps cannot be completed through email. For example, if a step requires you to provide Workflow attribute values or authentication, it cannot be completed by email because Content Server does not support updating Workflow attributes or authentication by email. If you are assigned to an email-enabled step that cannot be completed by email, you will receive a notification email when the step is ready, but you must sign into Content Server to complete the assignment.

Completing a Step Assignment

One of the actions in each Workflow step is to indicate when the step work is complete. Depending on the way the step is defined, the notification message may include either a Send On button or a choice of disposition buttons. Disposition buttons include of the following:

  • Delegate To, which allows you to reassign the step by selecting another user to perform the work.

  • Send for Review, which allows you to forward your work to another user for review before you complete the step.

When you are ready to complete a step assignment by email, you click the appropriate button and then send an email message confirming you have completed the work.

 

Note:
Some email-enabled steps cannot be completed through email. For example, if a step requires you to provide required attribute information or authentication, you must sign into Content Server to provide the information.

 

If Content Server receives and processes the step completed message, it returns a confirmation message. If Content Server cannot process the message, it returns an error message.

Working with Attachments

If the step instructs you to modify documents attached to the notification email message, you must attach your modified versions when you complete work on the step.

 

Important:
If the step instructs you to modify an attachment, do not change the attachment's file name. Changing the attachment's file name will attach a second document to the Workflow instead of version of the first. This causes confusion when distinguishing the attachments.

 

If you submit an attachment to a Workflow that does not permit attachments, attempt to attach a file type that has been blocked, or if you do not have permission to add attachments to the Workflow attachments folder, the attachment will not be added to the Workflow. Content Server returns a status message that explains why the attachment was rejected.

Performing Steps Assigned to Groups

If an email-enabled Workflow step is assigned to a group, all group members receive a step notification message. Depending on the way the step is configured, the step may require only one person to accept and perform the work or it may instruct all members to complete the work. If the step is configured to use the first member accept feature, the first group member to accept the assignment by replying to the step notification message will then receive the step assignment in a separate email message. The other group members receive a message that indicates that the step assignment has been accepted. Also, if another group member attempts to accept the step action message after it is already assigned to another user, the group member receives an email error message explaining that the step is no longer available.

If the step is assigned to a group and requires every member of the group to complete the step, the notification message is sent to all group members. Each group member must complete the step assignment before the step can be completed.

For more information about group assignments, see Completing Group Tasks.

Completing a Step Online

If you want to view more detailed information about a step, or if you want to perform the assignment online in a browser rather than by email, you can click the Workflow Step Page link at the bottom of the step action email message.

 

Important:
If you complete a step online, do not send an Action Completed message or reply to the Action Requested message.
