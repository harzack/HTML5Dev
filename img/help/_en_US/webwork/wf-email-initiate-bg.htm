If the Workflow author has email-enabled a Workflow Map, you can initiate the Workflow by composing and sending an email message to an email address associated with the Workflow Map. An email-enabled Workflow Map generally uses the email address Workflow_Map_Name@eLink_email_server, where Workflow_Map_Name is the Workflow Map name specified by the Workflow Map author, and eLink_email_server is the name of the eLink email server that handles Workflow Map requests for your organization. For more information about whether a Workflow Map is email-enabled and the email address you should use, contact the Workflow Map author or system administrator.

 

Important:
Before you can receive Workflow-related email messages, you or the Content Server administrator must configure eLink settings for Workflow events. For more information about eLink settings, see Configuring eLink Alerts and Notifications

Important:
Before you can receive Workflow-related email messages, you or the system administrator must configure eLink settings for Workflow events. For more information about eLink settings, see Configuring eLink Alerts and Notifications.

 
 

Notes:

  • If you submit an attachment to a Workflow that does not permit attachments, attempt to attach a file type that has been blocked, or if you do not have permission to add attachments to the Workflow attachments folder, the attachment will not be added to the Workflow. Content Server returns a status message that explains why the attachment was rejected.

  • Workflow attributes or form field values cannot be specified by email message.

 

After Content Server finishes processing a Workflow, the system returns a confirmation email message to the initiator and any users who have permission to manage the Workflow. The completed email message contains summary information about each step in the Workflow. If your request cannot be processed, Content Server returns an error message explaining the problem encountered.
