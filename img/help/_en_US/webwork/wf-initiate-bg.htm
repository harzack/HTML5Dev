When you are ready to start work on a project or business process defined by a Workflow, you initiate it. When you initiate a Workflow, Content Server creates a working copy (or instance) of the Workflow Map and begins to execute it. The work that occurs during the execution of the business process is stored within a Workflow instance. Each time the same project or business process needs to be performed, you initiate another instance of the Workflow Map.

You can initiate a Workflow within Content Server or by sending an email if the Workflow Map author has email-enabled the Workflow Map. For more information about initiating a Workflow within Content Server, see To Initiate a Workflow. For more information about initiating a Workflow by email, see Initiating a Workflow by E-mail.

Depending on the way the Workflow Map author has configured the Workflow, you may be required to do the following:

  • Add or modify the work package, which is composed of the instructions, tasks, files and other information used in the Workflow. For more information about the work package, see Working with the Work Package.

  • Provide a new name for the Workflow in the Initiate As field.

  • Specify a Workflow due date in the Due Date field.

  • Supply your Content Server user name and password.

After initiation, the active Workflow appears in your My Workflows page or in a Content Server Workflow Status item, if available. Also, the Workflow's managers can monitor its progress and manage it. For more information on monitoring and managing Workflows, see Managing Workflows. For more information on the Content Server Workflow Status item, see Using a Content Server Workflow Status Item.

 

Note:
You must have the See Contents permission for a Workflow to initiate it. For more information about permissions, see Understanding Permissions.

 

Initiating Signing Workflows

If the Workflow you want to initiate is a signing Workflow that allows you to circulate a Content Server Document to be approved and signed or rejected, you may need to perform other tasks before you can initiate the Workflow. For more information about initiating a signing Workflow, see Initiating Signing Workflows.

Initiating a Workflow by Email

If the Workflow author has email-enabled a Workflow Map, you can initiate the Workflow by composing and sending an email message to the Workflow Map. An email-enabled Workflow Map uses the email address Workflow_Map_Name@eLink_Server, where Workflow_Map_Name is the Workflow Map name specified by the Workflow Map author, and eLink_server is the name of the eLink Server. For more information about whether a Workflow Map is email-enabled, see the Workflow Map author or the administrator. For more information about email-enabled Workflows, see Working with Email-Enabled Workflows.
