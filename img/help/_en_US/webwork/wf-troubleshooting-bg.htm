This section explains the causes and recommended responses to the following email-enabled Workflow problems:

  • Workflow Failed to Initiate

  • Action Completed Message Failed

  • Action Confirmed but Attachments not Listed as Received

  • Contacting the Content Server Administrator

Workflow Failed to Initiate

If an error or other problem occurs after you attempt to initiate a Workflow by email, you will receive an email response message that describes the problem encountered. The problem may be caused by one of the following conditions:

  • The Workflow email address may not be correct. If the error message says it cannot find the email address or email box for the address you used, verify that you have the correct email address for the Workflow and resubmit your request.

  • The Content Server administrator or Workflow author may have disabled email initiation for this Workflow. If email initiation has been disabled, the only way to initiate the Workflow is to initiate it in a browser.

  • The Workflow may require information that cannot be supplied in an email message, such as attribute values or user authentication. If the Workflow requires information that cannot be specified in an email message, the only way to initiate the Workflow is to initiate it in a browser.

  • If you attached a document to the initiation email message and the Workflow is not configured to accept attachments, the request will fail. Try to resubmit the request without the attachment.

If you encounter one of these conditions and cannot resolve it, contact your system administrator.

Action Completed Message Failed

If an error or other problem occurs after you attempt to send an Action Completed message when you have completed work on a step assigned to you, you will receive a response email message that describes the problem encountered. The problem may be caused by one of the following conditions:

  • The Workflow may require some type of information that cannot be supplied in an email message, such as attribute values or user authentication. If the Workflow requires information that cannot be specified in an email message, the only way to complete the Workflow step is to perform the work in a browser.

  • If you have attempted to perform a step that has already been completed, you will receive an error message explaining that the step has already been performed. This error will not affect Workflow processing.

If you encounter any of these conditions and cannot resolve them, contact your system administrator.

Action Confirmed but Attachments Not Listed as Received

After you complete a Workflow step through email, you should receive the Action Confirmed email message to confirm that Content Server has received your work. Any attachments that you added or modified for the process should be listed as received in the Action Confirmed email message. However, there are several situations which may result in some or all of your attachments not being accepted by Content Server even though your step work is processed successfully. The following list includes some of the common reasons why attachments are not accepted by Content Server:

  • The Workflow may not be configured to accept attachments.

  • You might not have permissions to modify or add attachments to this step or Workflow process.

  • The Content Server administrator may have configured some restrictions on the types of files that can be accepted.

The Action Confirmed email message will not identify a cause if your attachments are not accepted. Contact the Workflow author or system administrator to resolve your issue.

Contacting the Content Server Administrator

If you have questions about a Workflow message, you can send an email message to your system administrator using the Email Administrator link, which appears near the bottom of Workflow email messages.
