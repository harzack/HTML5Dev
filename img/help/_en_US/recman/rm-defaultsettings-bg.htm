Table 1-1: Default System Settings

Setting This setting allows you to...
Default Classification
Specify a default Classification. If a default Classification is not specified, all objects can be classified by inheritance. If the parent object does not have the Inherit From flag specified, it will not inherit the default RM Classification. users can still choose an RM Classification different from the inherited one, which will override the default RM Classification. For information about setting default Classifications for specific object types, see the OpenText Records Management Installation and Administration guide.
Default Location for Items Created within the Classification Workspace
 Specify a default location for items created within the Classifications Workspace. These items include Appearances, Classification Trees, Custom Views, and RM Classifications.
RM Classification Titling
Set whether to use titling for Managed Objects and what RM Classification to append them to.
If titling is enabled, and a managed object has the Use Titling option selected, when users create a managed object and specify a Classification, the name of the RM Classification will be appended to the name or description of the managed object. The object name will appear in the language specified by the user or system. If no value is entered for the object name in the specified language, the default English value will be used.
Number of Search Results to Return
Specify the number of search results to return on one page.
File Number Separator
 Specify a default character to display between file numbers for different Classification levels.
RM Classification Display Mode
Specify whether RM Classifications appear in List mode or Featured mode when browsing the Classifications volume. The List mode places items in a vertical list; Featured mode places items in a separate section of a container to make them stand out from other items.
RSI and Holds Browsing
Set the mode for RSI and Holds browsing. This determines the behavior of the RSI and Holds pages when entered from the Records Management Workspace. In Standard Browse mode, the RSI and Holds pages will be automatically populated with the list of available RSIs or Holds. In Advanced Search mode, the user will be required to perform a search to list the available RSIs or Holds. If there is a large number of RSIs or Holds, the Advanced Search mode should be enabled.
Hide Icons
Hide official and classified icons when browsing. If the icons are hidden, the Classified and Official icons do not appear next to items that are classified or marked official in the Browse view. If the icons are displayed, users can click them and be directed to the Records Detail tab.
Records Management Actions in Browse View
Choose whether multi-select actions appear for Records Management functions in the Browse View. If selected, users can apply RM Metadata, apply or remove Holds, apply and remove Classifications, assign Cross-References, and finalize records for multiple managed items at one time.
