Permission Settings for Classified Objects and Objects Marked as Official

The Access Permissions tab enables you to force permissions settings on objects when they are classified or marked official. Users must have sufficient standard permissions to mark items official, remove the official marking, or finalize a record. You set the permissions for these functions on the Access Permissions tab. For more information about the standard Content Server permissions, see Understanding Permissions.

The following behaviors can be applied to objects when they are classified:

  • Use existing access restrictions, which does not change an object's Access List permissions when a user classifies the object

  • Add to access list, which adds the permissions from the RM Classification Access List to an object's existing Access List permissions when a user classifies the object.

  • Replace access list, which removes an object's existing Access List permissions and replaces them with the RM Classification Access List permissions when a user classifies the object

The Mark Official functionality (and associated rules) can be applied to managed items, with the exception of Polls and Generations of items, that have the read/write permissions. The standard ACL permissions map to the read/write permissions: See (None), See/See Content (Read), Modify (Write), and Edit (Administer). The following behavior can be applied to objects that are marked official:

  • Use existing access restrictions, which downgrades an object's existing Access List permissions to See, See Contents when a user marks the object as official. The Records Manager group is also added with full rights when permissions are downgraded.

  • Add to access list, which adds the permissions from the Primary RM Classification Access List to an object's existing Access List permissions, and then downgrades them to See, See Contents when a user marks the object as official. The Records Managers group is also added with full rights when permissions are downgraded.

  • Replace access list, which removes an object's existing Access List permissions, replaces them with the Primary RM Classification Access List permissions, and then downgrades them to See, See Contents when a user marks the object as official. The Records Manager group is also added with full rights when permissions are downgraded.

     

    Note:
    Downgrades are applied only to subtypes that have the Downgrade Permissions setting enabled.

    Since Polls and Generations of items in Content Server cannot be reserved, the Mark Official functionality cannot be completed for these types of items.
