The Records Manager or users with the proper permissions can create Holds in the Hold Maintenance container or on the fly from the Holds properties tab. Holds are based on the codes created. For more information, see Maintaining Codes.

When creating Holds you can specify comments and a date the Hold can be removed. Once you have created Holds, you can modify them in the Hold Maintenance container. Modifying a Hold allows you to edit, suspend, reactivate, or delete it. When you suspend a Hold, the Hold becomes inactive, which means that the metadata for the Hold is still applied to items, but items with the applied Hold are no longer prevented from being deleted.

You can export the contents of a Hold to a directory that is designated by your Administrator. You can export the current version only or all versions of the Hold. If you export more than one version of a document, the versions and corresponding file names and version numbers appear below the document; if only the current version is exported, the document is displayed. Versions are sorted in descending order from newest to oldest. The index.html file in the export folder is a list of the exported Holds and contents. You can click a file name to be redirected to the actual exported file. If the managed object contains multilingual metadata in the Name and Description fields, the contents of those fields are exported in the same way RM metadata is exported. For more information about the Multilingual Metadata Editor, see Working with General Items.

Using Hold Groups

You can organize Holds by creating Hold Groups. A Hold Group, like Holds, can be added on the Hold Maintenance page in the Records Management workspace. Hold Groups can only be added at the top level of the Hold Maintenance page, and can only contain Holds, which means only one level of grouping is allowed per Hold group. You cannot move or copy existing Holds to Hold Groups.

Once a Hold Group is created and contains Holds, you can export all records associated with the Hold Group.
