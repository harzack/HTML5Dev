A complex Query is a single statement, constructed with Live Query Language (LQL) , that precisely targets the data you want to retrieve. For example, to find items that contain the word report but do not contain the word expense, type the complex Query report AND-NOT expense.

LQL keywords are words that LQL reserves and assigns special meaning to. When used with LQL conventions, LQL keywords allow you to construct complex Queries. LQL keywords are not case sensitive.

LQL provides an alternative way to specify attributes when you search for items. Each item has certain attributes that contain additional information about it. By default, Content Server does not search item attributes. However, in addition to being able to specify attributes in a basic Query, you can use LQL to search for a specific attribute or set of attributes.

 

Note:
Attribute names are case sensitive. For a list of available attributes and their proper capitalization, contact your administrator.
