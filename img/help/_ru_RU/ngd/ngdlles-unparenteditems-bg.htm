In some cases, a search may return items that you cannot browse to because you do not have the appropriate permissions to access the folders in which the items reside. These types of items are known as unparented. When you open an unparented item in a search result list, the item that you selected appears in the folder structure in the Additional Items folder in the Enterprise Connect tree.

Note the following when working with unparented items and the Additional Items folder:

  • If you perform an open or the Go To Original Location command on an unparented item in the search result list, the Additional Items location is used.

  • If you perform the Enterprise Connect From Here command on an unparented item in the Content Server Web UI, Enterprise Connect will open the item from its location in the Additional Items folder.

  • The Additional Items folder only appears in the Enterprise Connect tree if it is required.

  • Once it has been created, the Additional Items folder will remain available in Enterprise Connect for the duration of your Enterprise Connect session.

  • If you have multiple Content Server repositories, there will be a separate instance of the Additional Items folder in each repository.

  • You cannot directly add items to the Additional Items folder. However, you can add items to folders that appear in the Additional Items folder.

  • The Additional Items folder is removed from the Enterprise Connect tree when you complete your session.

  • You cannot delete the Additional Items folder.

  • If you have sufficient permissions, you can delete items from the Additional Items folder. This action removes the item from the Additional Items folder and deletes it from its original location in Content Server.
