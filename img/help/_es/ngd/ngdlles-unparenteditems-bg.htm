En algunos casos, una búsqueda puede devolver elementos que no pueden examinarse porque no se dispone de los permisos necesarios para acceder a las carpetas en las que residen dichos elementos. Estos tipos de elementos se conocen como no emparentados. Al abrir un elemento no emparentado en una lista de resultados de búsqueda, el elemento seleccionado aparece en la estructura de carpetas en la carpeta Elementos adicionales del árbol de Enterprise Connect.

Tenga en cuenta lo siguiente al trabajar con elementos no emparentados y la carpeta Elementos adicionales:

  • Si abre un elemento o utiliza el comando Ir a la ubicación original con un elemento no emparentado de la lista de resultados de búsqueda, se emplea la ubicación de Elementos adicionales.

  • Si utiliza el comando Enterprise Connect desde aquí con un elemento no emparentado en la interfaz de usuario web de Content Server, Enterprise Connect abrirá el elemento desde su ubicación en la carpeta Elementos adicionales.

  • La carpeta Elementos adicionales sólo aparece en el árbol de Enterprise Connect en caso necesario.

  • Una vez creada, la carpeta Elementos adicionales seguirá estando disponible en Enterprise Connect mientras dure la sesión.

  • Si dispone de varios repositorios de Content Server, habrá una instancia independiente de la carpeta Elementos adicionales en cada repositorio.

  • No se puede agregar directamente los elementos a la carpeta Elementos adicionales. Sin embargo, sí puede agregar elementos a las carpetas que se encuentran en la carpeta Elementos adicionales.

  • Dicha carpeta Elementos adicionales se elimina del árbol de Enterprise Connect al finalizar la sesión.

  • No se puede suprimir esta carpeta.

  • Si tiene permisos suficientes, puede eliminar los elementos de la carpeta Elementos adicionales. Esta acción quita el elemento de la carpeta Elementos adicionales y lo elimina de su ubicación original en Content Server.
