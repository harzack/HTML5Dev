Live Query Language Las convenciones de (LQL) definen la sintaxis de una consulta compleja.

Una frase es una secuencia fija de una palabra o más palabras que se escriben entre comillas dobles (""). Por ejemplo, "bueno" y "para bien o para mal" son frases. Sin embargo, puede omitir las comillas dobles en frases de una sola palabra (por ejemplo, arroz).

Cuando crea frases, se aplican las reglas siguientes:

  • Un carácter de espacio (por ejemplo, una tabulación o un espacio) indican el final de una palabra.

  • Content Server interpreta las cadenas que incluyen puntuación pero ningún espacio como una frase (por ejemplo, "www.opentext.com" ).

  • Puede omitir las comillas dobles en frases de una sola palabra (por ejemplo, arroz).

  • No puede omitir las comillas dobles en palabras clave reservadas (por ejemplo, qlnear) si desea tratar las palabras clave como criterios de búsqueda. Por ejemplo, para buscar en el sistema de ayuda en línea información acerca de la palabra clave qlnear , debe especificar "qlnear" como consulta y seleccionar la porción adecuada de la ayuda.

Puede usar un asterisco (*) como marcador de posición para caracteres en consultas. Por ejemplo, para utilizar un asterisco que represente el final de una palabra que tiene varias terminaciones distintas, escriba cont* para buscar los elementos que contienen las palabras contar , contado, cantable, contabilidad, etc.

De manera predeterminada, Content Server procesa los términos de consulta de izquierda a derecha. Utilice paréntesis () para sustituir el orden predeterminado y hacer que Content Server evalúe las expresiones entre paréntesis primero. Por ejemplo, escriba (crecer O granja) Y (trigo O grano) para buscar elementos que contengan al menos una instancia de las palabras crecer o bien granja y por lo menos una instancia de las palabras trigo o bien grano .
