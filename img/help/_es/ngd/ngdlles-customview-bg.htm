Una vista personalizada es un tipo de elemento que contiene código HTML. Cuando se agrega una vista personalizada a un Espacio de trabajo, una carpeta, un proyecto, una lista de tareas o un documento compuesto, Content Server muestra dicho código HTML como parte de la interfaz. La vista personalizada se inserta entre los menús de navegación y los elementos del contenedor o el Espacio de trabajo.

El primer paso al agregar una vista personalizada es crear el archivo HTML que define la vista personalizada junto con cualquier otro archivo como archivos de imagen u hojas de estilo. De manera predeterminada, una vista personalizada puede contener los siguientes códigos:

  • HTML, JavaScript, VBScript, applets Java o cualquier otro código que pueda interpretar un navegador web. Sin embargo, el administrador puede restringir la capacidad de agregar determinado contenido.

  • Vínculos a otros elementos de Content Server u otras páginas web.

  • Referencias a imágenes, audio u otros archivos que se almacenan en Content Server o en un directorio de cualquier otro servidor web al que se pueda acceder.

Debido a que el contenido de una vista personalizada se inserta en un archivo HTML que proporciona la interfaz, los navegadores web ignoran etiquetas como <TITLE>, <BODY> o <FRAMESET>. Esto significa que hay límites en los tipos de código que puede incluir. Por ejemplo, no es posible crear marcos en una vista personalizada.

 

Nota:
Puede agregar una carpeta de recursos a la misma ubicación que la vista personalizada para almacenar imágenes y otros archivos que utiliza la vista personalizada. Para evitar la eliminación accidental de estos archivos y mantener un Espacio de trabajo organizada, debe considerar la posibilidad de ocultar la vista personalizada y la carpeta de recursos.

 

La capacidad de crear y gestionar vistas personalizadas está restringida normalmente a ciertos usuarios o grupos. Para agregar una vista personalizada, debe tener el privilegio de creación de objetos para vistas personalizadas, además de los permisos Agregar elementos y Modificar en los contenedores adecuados. Sin embargo, no necesita permisos para modificar cualquier subelemento que se pueda ver afectado por la vista personalizada. Para modificar una vista personalizada agregada por otro usuario, debe tener el permiso Modificar en la vista personalizada.

Puede determinar a qué usuarios o grupos se debe aplicar una vista personalizada mediante la definición de permisos en la vista personalizada.
