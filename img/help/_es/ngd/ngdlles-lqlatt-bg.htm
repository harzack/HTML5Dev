La sintaxis siguiente describe cómo especificar atributos al crear consultas complejas:

<nombre_región>  : <frase>

donde <nombre_región> es el nombre de una región de índice asociada al atributo que desea consultar. Todos los atributos tienen unas o más regiones asociadas. Para obtener una lista de regiones asociadas a atributos, póngase en contacto con el administrador.

Por ejemplo, OTName : "versión 1.0" busca todos los elementos de Content Server cuyo atributo de nombre contenga la frase versión 1.0.

Para especificar criterios de búsqueda más complejos en un atributo, utilice las palabras clave de Live Query Language  (LQL) e incluya la expresión a la derecha de los dos puntos entre paréntesis:

<nombre_región>  : <(expresión_LQL)>

donde <nombre_región> es el nombre de una región de índice asociada al atributo que desea consultar.

Por ejemplo, OTModifyDate : (> 19990601) busca todos los elementos de Content Server modificados después del 1 de junio de 1999.

Es importante especificar atributos de consulta mediante el LQL al realizar búsquedas con consultas complejas. Para obtener más información acerca del uso de las consultas complejas, consulte Búsquedas con consultas complejas.
