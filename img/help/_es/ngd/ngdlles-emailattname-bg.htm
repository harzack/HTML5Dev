Cuando intenta agregar un correo electrónico con archivos adjuntos a un destino, es posible que ya haya un archivo adjunto con el mismo nombre en la carpeta. Puede especificar cómo Enterprise Connect gestiona los archivos adjunto de correo electrónico con el mismo nombre.

Puede especificar que Enterprise Connect le pregunte si desea agregar el archivo adjunto con el nombre duplicado como una nueva versión del documento existente. También puede especificar que esto suceda automáticamente, cada vez que Enterprise Connect detecte un documento con el mismo nombre. Puede especificar que Enterprise Connect le pregunte si desea agregar el archivo adjunto con el nombre duplicado como un documento nuevo. En este caso, el documento duplicado mantendrá su nombre, pero se agregará un sufijo numérico para distinguirlo del documento original. Por ejemplo, taxform(2).doc. También puede especificar que esto suceda automáticamente, cada vez que Enterprise Connect detecte un documento con el mismo nombre.

Si tiene Records Management (RM) instalado, y crea una referencia cruzada para los archivos adjuntos de correo electrónico, verá referencias cruzadas para la instancia adecuada del documento. Por ejemplo, suponga que movió un correo electrónico con un archivo adjunto llamado legalform.doc a una carpeta en Content Server, y ya existía un documento con el mismo nombre en esa carpeta. Ha seleccionado Añadir automáticamente archivos adjuntos como una nueva versión de documento. Con las referencias cruzadas habilitadas, se agregará una nueva versión de legalform.doc al documento existente. Cuando abra el correo electrónico desde la nueva ubicación, habrá un vínculo a la instancia adecuada de legalform.doc. En este caso, será la segunda versión de legalform.doc. Para obtener más información sobre la configuración de referencias cruzadas de RM, consulte OpenText Records Management - Installation and Administration Guide (LLESRCM-IGD) .
