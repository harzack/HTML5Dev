Una consulta compleja es una única sentencia, construida con el Live Query Language (LQL), que localiza de manera precisa los datos que desea recuperar. Por ejemplo, para buscar elementos que contienen la palabra informe pero no contienen la palabra gasto, escriba la consulta compleja report AND-NOT expense.

Las palabras claves del LQL son las palabras que LQL reserva y a las que asigna un significado especial. Cuando se utilizan con convenciones del LQL, las palabras claves del LQL permiten que crear consultas complejas. Las palabras claves del LQL no distinguen entre mayúsculas y minúsculas.

El LQL proporciona un método alternativo para especificar atributos al buscar elementos. Todos los elementos tienen determinados atributos que contienen información adicional acerca de ellos. De manera predeterminada, Content Server no busca atributos de elemento. Sin embargo, además de poder especificar atributos en una consulta básica, puede utilizar el LQL para buscar un atributo o conjunto de atributos específico.

 

Nota:
En los nombres de atributos no se distingue entre mayúsculas y minúsculas. Para obtener una lista de los atributos disponibles y sus mayúsculas/minúsculas adecuadas, póngase en contacto con el administrador.
