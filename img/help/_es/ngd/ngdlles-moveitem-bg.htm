El comando Copiar/Mover a Enterprise Connect permite mover un solo elemento o varios de una ubicación a otra dentro del mismo repositorio de Content Server. Si va a mover un documento o un elemento distinto de un contenedor, puede usar los comandos estándar para mover de Enterprise Connect, como arrastrar. Si va a mover un contenedor, como puede ser una carpeta, utilice el menú contextual para acceder al comando Copiar/Mover a Enterprise Connect.

 

Sugerencia:
Si está trabajando en el Explorador de Windows, también puede mover documentos y carpetas con el botón derecho del ratón. Seleccione los elementos que desee mover, arrástrelos al destino con el botón derecho de ratón y, a continuación, haga clic en Mover. Esta acción está disponible al arrastrar elementos de una carpeta local a Enterprise Connect, al arrastrar elementos de una ubicación a otra en el mismo repositorio de Content Server y cuando se arrastran elementos de un resultado de búsqueda de Enterprise Connect a una carpeta del mismo repositorio de Content Server o en otro módulo complementario.

 

Cuando se mueve un elemento a una nueva ubicación, puede que tenga que especificar si los elementos que va a mover mantendrán las categorías asignadas en un principio, si deberán asumir las categorías que han sido asignadas al destino o si deberán hacerlo con las categorías del original y del destino. Según cómo esté configurado el sistema, puede que tenga que especificar si los elementos deben heredar las clasificaciones de OpenText Records Management asignadas.

Si ya existe un elemento con el mismo nombre de archivo en el destino, puede que tenga que especificar el modo en que desea solucionar el conflicto de nomenclatura. Dependiendo de la configuración, pueden estar disponibles las opciones siguientes:

  • Puede especificar que el elemento actual se mueva al destino con un nombre de archivo generado automáticamente. Esto permite conservar las versiones, las auditorías o los metadatos asociados con el elemento. Si va a mover varios elementos, puede especificar además que todos los conflictos de nomenclatura se resuelvan de ese modo.

     

    Sugerencia:
    Si tiene permisos suficientes, puede cambiar el nombre al archivo tras moverlo.

     

  • Puede especificar si no desea mover el elemento actual al destino. Si va a mover varios elementos, puede especificar además que no se muevan duplicados en el lote.

  • Puede especificar si no desea continuar con la acción de mover. Cualquier elemento que ya se haya movido correctamente se mantendrá en el destino.

Puede mover instancias individuales o múltiples de los siguientes tipos de elemento distintos de documentos y contenedores a una nueva ubicación en el repositorio de Content Server:

  • vistas personalizadas

  • sondeos

  • métodos abreviados

  • direcciones de Internet (URL)

  • Esquema de Workflows

  • documentos

  • documentos de texto

  • Subtipos proporcionados por módulos de Content Server, como páginas wiki.

 

Notas:

  • Utilice el menú contextual para mover carpetas o elementos equivalentes a carpetas de Content Server como carpetas de correo electrónico, proyectos, documentos compuestos, Estado de workflow, listas de tareas, canales o hitos.

  • Debe tener los permisos necesarios para ejecutar esta función. Para obtener más información acerca de la configuración de permisos, consulte Configuración de permisos.

 
 

Sugerencia:
Para obtener más información acerca de cómo mover elementos mediante Enterprise Connect, incluidos los comandos disponibles, consulte OpenText Enterprise Connect - User Getting Started Guide (NGDCORE-UGD).
