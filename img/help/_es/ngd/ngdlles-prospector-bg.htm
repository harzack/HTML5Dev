Los prospectores son elementos de Content Server que ayudan a localizar y utilizar la información. Actúan como filtros; exploran datos a medida que Content Server los indexa para localizar información que se corresponda con sus intereses. Los prospectores se ejecutan continuamente, por lo que mantienen listas precisas y actualizadas de sus resultados.

Pueden explorar cualquier origen de datos que Content Server esté indexando. Por ejemplo, pueden explorar el repositorio de Content Server, los directorios locales o una base de datos de Lotus Notes. Para obtener más información acerca de los orígenes de datos que se están indexando en el sitio de Content Server, póngase en contacto con el administrador.

Puede crear prospectores mediante el uso de Live Query Language  (LQL) para generar una consulta que localice el tipo de información que desea recuperar.

Una vez creada la consulta de prospectores, identifique la porción que desee que explore el prospector. A continuación, el prospector ejecuta su consulta para ver si los nuevos elementos que se han añadido al índice coinciden con los criterios de búsqueda especificados. Si existe una coincidencia, el prospector añade un enlace al elemento correspondiente en la lista de resultados. El prospector continúa actualizando la lista de resultados mientras que se añadan o eliminen elementos del índice. Los prospectores solamente pueden explorar los índices que el administrador ha configurado para su prospección.

Al crear prospectores, también define las opciones de la lista de resultados, que incluyen el número de resultados mostrados y el orden en el que aparecen. También puede elegir si desea que Content Server muestre u oculte los resúmenes de los elementos de la lista de resultados.

Puede crear prospectores en la mayoría de las ubicaciones de Content Server. Por ejemplo, puede crearlos en el Espacio de trabajo, en el Espacio de trabajo del proyecto y en su Espacio de trabajo personal.
