Un documento compuesto es un contenedor que almacena varios documentos en orden secuencial. La gestión de documentos compuestos permite ejecutar funciones de documentos compuestos desde el Explorador de Windows. Por ejemplo, puede crear un lanzamiento o una revisión, o reorganizar los subdocumentos.

Los elementos se agregan a un documento compuesto de la misma manera que se agregan a otros contenedores. Sin embargo, los documentos compuestos solamente pueden contener los siguientes tipos de elementos:

  • documentos

  • documentos de texto

  • mensajes de correo electrónico

  • documentos compuestos

  • Métodos abreviados y generaciones de documentos, documentos de texto y otros documentos compuestos

Cada elemento que se agrega a un documento compuesto se numera de manera secuencial, según el orden en el que se ha agregado. Sin embargo, puede cambiar el orden de los elementos en cualquier momento. Además, un documento compuesto puede incluir un documento maestro, que puede designar al reorganizar los elementos del documento compuesto. El documento maestro podría ser la tabla de contenido del libro, la página de índice del sitio web o la carta adjunta de un informe. El documento maestro es siempre el primer elemento de un documento compuesto y el elemento situado más arriba de la estructura.

Los documentos compuestos no requieren numeración secuencial; es válido tener los elementos numerados con 1, 17 y 339. Al reorganizar los elementos, puede volver a numerarlos automáticamente para quitar cualquier espacio de la secuencia.

Aunque Content Server mantiene las versiones de los elementos que contiene un documento compuesto, los documentos compuestos tienen sus propias clases de versiones denominadas lanzamientos y revisiones. Cada lanzamiento y revisión es una versión estática de un documento compuesto, creada en un determinado punto en el pasado. Por ejemplo, si tiene un documento compuesto que representa un manual de software, podía haber creado un lanzamiento cuando se publicó el manual. Puede continuar trabajando en el documento compuesto original porque el lanzamiento representa el manual que se está suministrando actualmente a los clientes. Los lanzamientos se numeran 1.0, 2.0, 3.0 y así sucesivamente. Una revisión es similar a un lanzamiento, pero es conveniente cuando desea conservar solamente modificaciones menores en un documento compuesto y no necesita crear un nuevo lanzamiento. La numeración de revisiones se basa en la enumeración de lanzamientos. La revisión 1.2 es la segunda revisión del primer lanzamiento y así sucesivamente.

Con documentos compuestos grandes o complejos, puede ser difícil gestionar la estructura global viendo el contenido de un nivel cada vez. Por ello, tiene la opción de ver un esquema del documento compuesto. El esquema muestra la estructura completa de un documento compuesto en una página, por orden secuencial y según su jerarquía. También proporciona un menú Funciones para cada elemento del documento compuesto, que permite trabajar con cada elemento desde allí. Por ejemplo, puede abrir un documento para ver su contenido.
