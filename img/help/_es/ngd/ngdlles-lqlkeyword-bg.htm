El Live Query Language (LQL) consta de las siguientes palabras clave:

AND

La palabra clave AND tiene la misma función que la palabra clave &. Busca elementos que contengan las dos expresiones de consulta especificadas.

Sintaxis

<expresión> AND <expresión>

Ejemplo

Escriba pez AND pájaro para buscar elementos que contengan tanto pez como pájaro.

&

La palabra clave & tiene la misma función que la palabra clave AND. Busca elementos que contengan las dos expresiones de consulta especificadas.

Sintaxis

<expresión>  &  <expresión>

Ejemplo

Escriba pez & pájaro para buscar elementos que contengan tanto pez como pájaro.

AND-NOT

La palabra clave AND-NOT tiene la misma función que la palabra clave &!. Busca elementos que contengan la expresión a la izquierda de la palabra clave y no contengan la expresión a la derecha.

Sintaxis

<expresión>  AND-NOT <expresión>

Ejemplo

Escriba pez AND-NOT pájaro para buscar elementos que contengan la palabra pez pero no la palabra pájaro.

&!

La palabra clave &! tiene la misma función que la palabra clave AND-NOT. Busca elementos que contengan la expresión a la izquierda de la palabra clave y no contengan la expresión a la derecha.

Sintaxis

<expresión>  &! <expresión>

Ejemplo

Escriba pez &! pájaro para buscar elementos que contengan la palabra pez pero no la palabra pájaro.

OR

La palabra clave OR tiene la misma función que la palabra clave |. Busca elementos que contengan la expresión a la izquierda de la palabra clave, los que contengan la expresión a la derecha o los que contengan ambas expresiones.

Sintaxis

<expresión>  OR  <expresión>

Ejemplo

Escriba pez OR pájaro para buscar elementos que contengan la palabra pez, elementos que contengan la palabra pájaro o elementos que contengan tanto pez como pájaro.

|

La palabra clave | tiene la misma función que la palabra clave OR. Busca elementos que contengan la expresión a la izquierda de la palabra clave, los que contengan la expresión a la derecha o los que contengan ambas expresiones.

Sintaxis

<expresión>  | <expresión>

Ejemplo

Escriba pez | pájaro para buscar elementos que contengan la palabra pez, elementos que contengan la palabra pájaro o elementos que contengan tanto pez como pájaro.

SOR

La palabra clave SOR tiene la misma función que la palabra clave OR pero su algoritmo de clasificación de relevancia considera las expresiones a la izquierda y derecha de la palabra clave SOR como sinónimos y los trata como un único término. Esto evita que los sinónimos tengan más peso en el cálculo de la puntuación para un elemento de resultado de la búsqueda. Las puntuaciones clasificadas y el orden de los resultados producidos por las consultas que utilizan la palabra clave SOR son diferente de las puntuaciones clasificadas y el orden de los resultados producidos por las consultas que utilizan la palabra clave O.

Sintaxis

<expresión>  SOR  <expresión>

Ejemplo

Escriba "U.K." SOR "United Kingdom" para buscar elementos que contengan al menos una instancia de la palabra U.K.. , la palabra United Kingdom o ambas. Content Server muestra los resultados de esta consulta en un orden distinto del de los resultados de "R.U." OR "Reino Unido".

XOR

La palabra clave XOR busca elementos que contengan una de las dos expresiones de consulta especificadas pero no ambas. Esta palabra clave también se conoce como palabra clave <OR> exclusivo.

Sintaxis

<expresión>  XOR  <expresión>

Ejemplo

Escriba pez XOR pájaro para buscar elementos que contengan la palabra pez o la palabra pájaro pero no ambas.

=

La palabra clave = busca elementos que contengan la frase especificada. Puede omitir esta palabra clave al crear consultas en Live Query Language porque es el comparador predeterminado.

Sintaxis

= <expresión>

Ejemplo

Escriba = pájaro para buscar elementos que contengan la palabra pájaro.

NOT

La palabra clave NOT tiene la misma función que la palabra clave ! y la palabra clave !=. Busca elementos que no contengan la frase especificada.

Sintaxis

<expresión> NOT

Ejemplo

Escriba NOT pájaro para buscar elementos que no contengan la palabra pájaro.

!

La palabra clave ! tiene la misma función que la palabra clave NOT y la palabra clave !=. Busca elementos que no contengan la frase especificada.

Sintaxis

<expresión> !

Ejemplo

Escriba ! pájaro para buscar elementos que no contengan la palabra pájaro.

!=

La palabra clave != tiene la misma función que la palabra clave NOT y la palabra clave !. Busca elementos que no contengan la frase especificada.

Sintaxis

<expresión> !=

Ejemplo

Escriba != pájaro para buscar elementos que no contengan la palabra pájaro.

<

La palabra clave < busca elementos que contengan números o palabras que son menores que el número o la palabra especificado. Si el término especificado es un número entero o real, Content Server utiliza un orden numérico. De lo contrario, utiliza un orden lexicográfico.

Sintaxis

< <expresión>

Ejemplo

Escriba < 20000630 para buscar elementos que contengan al menos un número menor que 20000630. Si restringe esta consulta a un atributo de fecha, busca elementos con fecha anterior al 30 de junio de 2000.

<=

La palabra clave <= busca elementos que contengan números o palabras que son menores o iguales que el número o la palabra especificado. Si el término especificado es un número entero o real, Content Server utiliza un orden numérico. De lo contrario, utiliza un orden lexicográfico.

Sintaxis

<expresión> <=

Ejemplo

Escriba <= 20000630 para buscar elementos que contengan al menos un número o una palabra que sea menor o igual que 20000630. Si restringe esta consulta a un atributo de fecha, busca elementos con fecha de 30 de junio de 2000 o anterior.

>

La palabra clave > busca elementos que contengan números o palabras que son mayores que el número o la palabra especificado. Si el término especificado es un número entero o real, Content Server utiliza un orden numérico. De lo contrario, utiliza un orden lexicográfico.

Sintaxis

<expresión> >

Ejemplo

Escriba > 20000630 para buscar elementos que contengan al menos un número o una palabra que sea mayor que 20000630. Si restringe esta consulta a un atributo de fecha, busca elementos con fecha posterior al 30 de junio de 2000.

>=

La palabra clave >= busca elementos que contengan números o palabras que son mayores o iguales que el número o la palabra especificado. Si el término especificado es un número entero o real, Content Server utiliza un orden numérico. De lo contrario, utiliza un orden lexicográfico.

Sintaxis

<expresión> >=

Ejemplo

Escriba >= 20000630 para buscar elementos que contengan al menos un número o una palabra que sea mayor o igual que 20000630. Si restringe esta consulta a un atributo de fecha, busca elementos de Content Server con fecha de 30 de junio de 2000 o posterior.

QLPHONETIC

La palabra clave QLPHONETIC busca elementos que contengan palabras que suenen como la expresión de consulta especificada. Esta palabra clave se basa en un algoritmo muy utilizado llamado Soundex. El algoritmo Soundex toma una palabra inglesa y produce una representación de cuatro dígitos de la palabra, diseñada para coincidir con la pronunciación fonética de la palabra original. Soundex se suele utilizar para búsquedas imprecisas, donde se requiere una coincidencia aproximada.

Sintaxis

<Expresión> QLPHONETIC

Ejemplo

Escriba QLPHONETIC vaca para buscar elementos que contengan palabras como vaca o baca.

QLREGION

La palabra clave QLREGION busca elementos que contengan términos de búsqueda en una región de índice específica. Cada región almacena una un tipo distinto de información acerca de elementos indexados, que incluye el contenido y los metadatos de documentos. Para obtener una lista de las regiones, póngase en contacto con el administrador.

Sintaxis

<expresión>[QLREGION <región>]

Ejemplo

Escriba [QLREGION OTCreatedByName] jdoe para buscar elementos que contengan la palabra jdoe en la región OTCreatedByName.

QLSTEM

La palabra clave QLSTEM busca elementos que contengan palabras derivadas de la raíz de la expresión de consulta especificada.

Sintaxis

<expresión> QLSTEM

Ejemplo

Escriba QLSTEM navegando para buscar elementos que contengan al menos una palabra derivada de la palabra navegar como navegando, navegado o navega.

QLTHESAURUS

La palabra clave QLTHESAURUS busca elementos que contengan entradas del tesauro que estén asociadas a la expresión de consulta especificada.

Sintaxis

<expresión> QLTHESAURUS

Ejemplo

Escriba QLTHESAURUS piedra para buscar elementos que contengan como mínimo una de las entradas del tesauro asociadas a la palabra piedra como, por ejemplo, piedra, hoyo, semilla o grano.

QLRIGHT-TRUNCATION

La palabra clave QLRIGHT-TRUNCATION busca elementos que contengan palabras cuyos prefijos coincidan con la expresión de consulta especificada.

Sintaxis

<expresión> QLRIGHT-TRUNCATION

Ejemplo

Escriba QLRIGHT-TRUNCATION pos para buscar elementos que contengan al menos una palabra que empiece por pos como, por ejemplo, posponer, posguerra o postura.

QLLEFT-TRUNCATION

La palabra clave QLLEFT-TRUNCATION busca elementos que contengan palabras cuyos sufijos coincidan con la expresión de consulta especificada.

Sintaxis

<expresión> QLLEFT-TRUNCATION

Ejemplo

Escriba QLLEFT-TRUNCATION mente para buscar elementos que contengan al menos una palabra que termine con mente como, por ejemplo, mente, anteriormente o posteriormente.

QLRANGE

La palabra clave QLRANGE busca elementos que contengan palabras del intervalo lexicográfico o numérico entre dos expresiones de consulta especificadas. Los términos utilizados para especificar el intervalo también se incluyen en el mismo. Si los términos especificados son números enteros o reales, Content Server utiliza un orden numérico. De lo contrario, utiliza un orden lexicográfico.

Debe incluir el intervalo entre comillas dobles ("") y separar los términos utilizados para especificar el intervalo con una tilde (~).

Sintaxis

QLRANGE < "expresión >~ <expresión">

Ejemplo

Escriba QLRANGE "jamba ~ juzgar" para buscar elementos que contengan al menos una palabra del intervalo lexicográfico entre jamba y juzgar como, por ejemplo, jamba, jambas, jilguero, jota, junta o juzgar.

Escriba QLRANGE "20000801 ~ 20000901 para buscar elementos que contengan al menos un número entre 20000801 y 20000901 (por ejemplo, fechas entre el 1 de agosto de 2000 y el 1 de septiembre de 2000, ambos inclusive).

QLREGEX

La palabra clave QLREGEX busca elementos que contengan una ampliación de expresión regular de la expresión de consulta especificada. Las expresiones de consulta utilizadas con la palabra clave QLREGEX deben estar incluidas entre comillas dobles ("").

Sintaxis

<"expresión"> QLREGEX

Ejemplo

Escriba QLREGEX "^el" para buscar elementos que contengan al menos una de las ampliaciones de expresión regular de ^el como, por ejemplo, el, ele, elemental, elementales, elemento, elevar, ella o ellas

PROX

La palabra clave prox[n,t/f] busca elementos que contengan todas las frases de consulta especificadas que estén a una determinada distancia (n) unas de otras y donde el orden sea ordenadas (t) o no ordenadas (f). La distancia (n) se calcula como el número de símbolos entre el último símbolo de la primera frase y el primer símbolo de la segunda frase. La especificación del segundo parámetro (t/f) es opcional y es pro defecto f.

Sintaxis

<frase> prox[n,t/f] <frase>

Ejemplo

Escriba carrera prox[10,t] coche para buscar elementos que contengan la palabra carrera delante de la palabra coche a una distancia máxima de 10 unidades la una de la otra.

QLNEAR

La palabra clave QLNEAR tiene la misma función que la palabra clave PROX pero se define en el valor predeterminado prox[10,f].

Sintaxis

<frase >QLNEAR <frase>

Ejemplo

Escriba carrera QLNEAR coche para buscar elementos que contengan la palabra carrera y la palabra coche a una distancia máxima de 10 unidades la una de la otra, en cualquier orden.
