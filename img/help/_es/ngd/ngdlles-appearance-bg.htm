Una apariencia es un contenedor que almacena documentos y código HTML. Las apariencias permiten personalizar ciertas ubicaciones hasta tal punto que podría diseñar una página web.

 

Importante:
Aunque pueden crearse y editarse apariencias mediante Enterprise Connect, estas no pueden verse en Enterprise Connect. Esto también se aplica a apariencias creadas en la interfaz web de Content Server.

 

Hay dos tipos de apariencias que puede agregar para ayudarle a personalizar la interfaz:

  • Apariencias globales, que se aplican sistemáticamente a través de Content Server para usuarios específicos. Las apariencias globales se pueden agregar solamente al volumen de apariencias y éste hace que estén disponibles en todo el sistema.

  • Apariencias basadas en la ubicación.

Las apariencias basadas en la ubicación se pueden aplicar de dos formas:

  • Sin cascada, que aplica la apariencia solo al Espacio de trabajo, al proyecto o a la carpeta a la que se agrega.

  • Cascada, que aplica la apariencia al Espacio de trabajo, al proyecto o a la carpeta padre a la que se agrega y a cualquier proyecto o carpeta hijo. Este tipo de apariencia solo puede aplicarse a un conjunto específico de usuarios o grupos.

Una apariencia se divide en una página Visión general y una página Espacio de trabajo, en las que define o edita la apariencia. Después de agregar una apariencia, puede definir la configuración y el esquema en la página Visión general y agregar contenido, como imágenes, a su Espacio de trabajo. La capacidad de agregar y editar apariencias normalmente está restringida.
