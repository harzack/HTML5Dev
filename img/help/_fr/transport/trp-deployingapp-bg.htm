Note :
Vous devez être un administrateur d'entrepôt pour déployer un atelier.

 

Pour déployer un atelier vers le système cible, procédez comme suit :

  • Exportez le lot de transport depuis le système source et enregistrez le fichier vers un emplacement externe (par exemple, une clé USB ou un emplacement réseau).

  • Créez un atelier dans l'entrepôt de transport sur le système cible.

  • Chargez le lot de transport vers le système cible via le menu Ajouter un élément.

  • Décompressez le lot de transport vers un atelier sur le système cible. Après décompression du lot, l'atelier vers lequel le lot a été décompressé apparaît. Vous pouvez ainsi voir les fichiers transportés.

     

    Note :
    Si plusieurs lots contiennent des éléments que vous souhaitez déployer vers le système cible, vous pouvez les décompresser vers le même atelier et les déployer vers Content Server en même temps.

     

  • Vérifiez l'atelier pour vous assurer que vous les dépendances des éléments sont respectées pour chaque élément.

  • Déployer l'atelier. La page Déployer l'atelier affiche le nombre d'éléments dans l'atelier, ainsi qu'une description de l'atelier, si elle a été fournie lors de la création de l'atelier. La page Déployer l'atelier affiche également la liste des éléments qui seront déployés, ainsi que les informations suivantes pour chaque élément : type, nom, emplacement dans lequel sera créé l'élément et état de l'élément (si l'élément est prêt pour le déploiement ou s'il manque des dépendances). S'il manque des dépendances à un élément, ou si, pour une toute autre raison, celui-ci n'est pas prêt pour le déploiement, il ne sera pas créé lors du déploiement de l'atelier. Si un élément en cours de déploiement possède le même nom qu'un élément situé dans le recueil parent auquel il doit être ajouté, il sera ajouté en tant que nouvelle version.

Traitement des dépendances de transport

La fonction Transport analyse le système pour vérifier que les dépendances de l'élément sont respectées. Par exemple, si des autorisations ou un classement ont été attribués à un élément, la fonction Transport analyse Content Server pour vérifier l'existence de l'autorisation et du classement. Si un atelier est déployé avec un utilisateur ou un groupe qui n'existe pas sur le système cible, les autorisations sur l'atelier sont supprimées. Si l'utilisateur ou le groupe existe sur le système cible, les autorisations sur l'atelier sont attribuées automatiquement.

Lorsque vous ouvrez un élément sur l'atelier, la page Vue d'ensemble apparaît et comprend les informations suivantes :

  • Nom : nom actuel de l'élément

  • Version : numéro de version actuelle de l'élément à déployer

  • Taille : taille de l'élément

  • Créé par : nom de l'utilisateur qui a créé l'élément

  • Créé le : date à laquelle l'élément d'origine a été créé

  • Parent : dossier dans lequel l'élément d'origine était situé

  • Instance d'origine : instance Content Server dans laquelle l'élément d'origine était situé

  • Description : description de l'élément, si elle a été fournie lors de la création de l'élément

Si l'élément que vous déployez a des dépendances sur d'autres éléments, elles sont répertoriées sur la page Vue d'ensemble de l'élément. Les informations suivantes sont disponibles pour chaque élément dépendant : le type et le nom de l'élément, si l'élément existe sur le système cible, et si l'élément existe sur l'atelier actuel. Si l'élément dépendant existe sur l'atelier, une coche verte apparaît indiquant que l'élément est prêt pour le déploiement.

 

Important :
Tous les éléments dépendants doivent figurer dans le système cible pour déployer un atelier. Le bouton Déployer sera désactivé jusqu'à ce que toutes les dépendances soient respectées.

 

Si un élément a une dépendance sur un ou plusieurs éléments de même type et même nom d'élément, il ne peut être déployé. Si un élément n'est pas prêt pour le déploiement, vous pouvez utiliser le bouton Modifier pour apporter des changements avant le début du déploiement. Si le recueil parent d'un élément n'existe pas sur le système cible, vous pouvez remplacer ce recueil parent par un recueil existant.

 

Note :
Les colonnes de dépendance Existe dans l'instance et Existe dans l'atelier sont visibles uniquement pour les administrateurs d'entrepôt. De même, seuls les administrateurs d'entrepôt peuvent modifier les dépendances des éléments. Le bouton Modifier ne sera visible que si vous êtes un administrateur d'entrepôt.
