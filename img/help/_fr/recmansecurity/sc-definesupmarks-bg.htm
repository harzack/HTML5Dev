Les marquages supplémentaires ne sont pas hiérarchiques. Ils sont mis en œuvre pour donner accès aux métadonnées d'objets individuels. Ils sont également utilisés pour filtrer les recherches et fournir aux utilisateurs des listes d'objets qu'ils peuvent voir selon le marquage supplémentaire qui leur a été assigné.

Les marquages supplémentaires possèdent des codes d'attributs multiples associés à des objets individuels. Ce type de filtrage est généralement effectué avant vérification de la liste de contrôle d'accès de l'objet, comme c'est le cas avec les niveaux d'habilitation de sécurité.

Description Code
Audit In Confidence (audit en confiance) AIC
Commercial In Confidence (commercial en confiance) CIC
For Official Use Only (pour usage officiel uniquement) FOUO
Restricted Data (données restreintes) RD
Security In Confidence (sécurité en confiance) SIC
