Vous pouvez ajouter un ou plusieurs éléments à une collection depuis l'espace de travail de votre choix. Vous pouvez également collecter des éléments depuis la page Résultats de recherche. Une fois un élément collecté, vous pouvez alors accéder à cet élément ou à ce groupe d'éléments depuis l'emplacement central de la collection.

Si vous disposez des autorisations appropriées, vous pouvez également collecter des versions individuelles d'un élément, notamment des versions secondaires et principales, à partir d'une page Résultats de recherche. Cela peut être utile si vous travaillez sur un projet d'équipe pour lequel vous devez avoir accès aux versions spécifiques d'un élément de la collection. Pour de plus amples informations sur les versions, reportez-vous à la rubrique Utilisation des versions.

Sur les nouvelles installations mises à niveau de Content Server mise à jour 10.5 ou ultérieure, la colonne Version indique la version spécifique de l'objet marqué pour l'inclusion dans la collection. La version actuelle est toujours marquée, que ce soit dans Parcourir ou Rechercher.

Vous pouvez utiliser le bouton Parcourir Content Server... pour sélectionner les éléments de la page Eléments à collecter, ou utiliser les menus contextuels contenant les collections Utilisées récemment, vos collections Favorites et vos 10 collections Personnelles les plus récemment utilisées.

Si vous détenez les autorisations appropriées, vous pouvez également collecter des éléments d'OpenText Enterprise Library. Les collections sont stockées dans le volume Content Server.

Collecte d'éléments depuis un espace de travail

Vous pouvez ajouter des éléments à une collection depuis n'importe quel espace de travail. Une opération d'ajout globale consiste à sélectionner et à ajouter en une seule fois plusieurs éléments à une collection grâce aux cases à cocher de la fenêtre Vue détaillée d'un espace de travail.

L'opération d'ajout globale vous permet d'intégrer des éléments OpenText Enterprise Library. Cependant, vous ne pouvez pas effectuer certaines fonctions de base de Content Server sur les éléments d'origine à partir de la collection, ces derniers étant en lecture seule. Les exemples de fonctions de base de Content Server incluent Copier, Déplacer et Supprimer. Pour plus d'informations sur la gestion des éléments à partir d'une collection, reportez-vous à la rubrique Gestion des éléments collectés.

 

Important :
Vous devez utiliser la fenêtre Vue détaillée pour ajouter des éléments à une collection depuis un espace de travail. Pour plus d'informations sur les fenêtres de navigation, reportez-vous à la rubrique Espaces de travail et recueils.

 
 

Note :
Vous devez disposer des autorisations appropriées pour effectuer une opération d'ajout globale.

 

Collecte des éléments depuis la page Résultats de recherche

Après avoir effectué une recherche, depuis la page Résultats de recherche, vous pouvez :

  • collecter un élément spécifique ;

  • collecter plusieurs éléments par une opération d'ajout globale ;

  • collecter tous les résultats de recherche.

 

Notes :

  • Vous devez disposer des autorisations requises pour effectuer une opération d'ajout globale ou pour collecter tous les résultats de recherche. Une opération d'ajout globale consiste à sélectionner et à ajouter en une seule fois plusieurs éléments à une collection grâce aux cases à cocher de la fenêtre Vue détaillée.

  • L'opération d'ajout globale vous permet également d'intégrer des éléments OpenText Enterprise Library. Cependant, vous ne pouvez pas effectuer certaines fonctions de base de Content Server sur les éléments d'origine à partir de la collection, ces derniers étant en lecture seule. Les exemples de fonctions de base de Content Server incluent Copier, Déplacer et Supprimer. Pour plus d'informations sur la gestion des éléments à partir d'une collection, reportez-vous à la rubrique Gestion des éléments collectés.

 

Si vous effectuez une recherche sur la vue Enterprise [Toutes les versions] et que plusieurs versions d'un élément sont mises en évidence, vous pouvez sélectionner des versions spécifiques de l'élément depuis les résultats de recherche à collecter. Par exemple, si vous effectuez une recherche et que les versions 2, 3 et 6 sont mises en évidence, vous avez la possibilité de choisir et de collecter uniquement les versions 2 et 3 dans la liste. Si vous choisissez de collecter tous les résultats de recherche, toutes les versions comprises dans ces résultats sont collectées.

Si vous collectez un élément ne prenant pas en charge la création de version (un dossier, par exemple), 0 (zéro) s'affiche dans la colonne Version de la page Collecter les résultats de recherche et l'élément lui-même est collecté. Le message « Version 0 inexistante ; objet collecté OK » s'affiche dans la colonne Statut de la page Collecte des éléments des résultats de recherche vers. Pour une liste d'éléments multi-versions, reportez-vous à la rubrique Versions.

 

Note :
Dans certaines circonstances, la version récemment supprimée d'un élément peut apparaître dans les résultats de recherche. Si vous essayez de collecter cette version récemment supprimée, le message « Version <n> n'existe pas » s'affiche, <n> représentant le numéro de version ; l'élément sera cependant collecté.

 

Lorsque vous collectez tous les résultats de recherche à l'aide du bouton Collecter tous les résultats de recherche, vous pouvez informer par e-mail un utilisateur, un groupe ou un destinataire tiers de la création de la collection. Par exemple, si vous collectez un groupe de documents pour un projet d'équipe, vous pouvez notifiez aux autres membres de l'équipe le contenu et l'emplacement de la collection.

 

Note :
La notification par e-mail doit avoir été activée par l'administrateur dans la zone Administration des notifications des Content Server pages d'administration avant que toute notification activée pour la collection soit envoyée.

Si vos notifications issues de collections ne sont pas envoyées, contactez votre administrateur pour activer les notifications.

 

La notification par e-mail contient les informations suivantes :

Lancé par ID de connexion de l'utilisateur ayant créé la collection.
Nom de la collection Le nom de la collection.
Démarré à Le jour, le mois, l'année et l'heure où le processus de collecte a démarré.
Achevé à Le jour, le mois, l'année et l'heure où le processus de collecte s'est terminé.
Total des versions collectées Le nombre total de versions de l'ensemble des objets collectés.
Total des objets collectés Le nombre total d'objets collectés. Les objets sont définis comme des éléments avec des ID de données distincts.
Total des éléments collectés Le nombre total d'éléments collectés. Il comprend le nombre total de versions et le nombre total d'objets collectés.
Taille de la collection mise à jour Le nombre total d'éléments est désormais disponible dans la collection.
La collection peut être consultée ici. Une URL permet d'accéder à la collection dans Content Server.

 

Note :
Le total des versions collectées et le total des objets collectés peuvent ne pas correspondre. Par exemple, si vous ajoutez des éléments à une nouvelle collection vide, si l'un des éléments ne peut contenir de version, l'élément n'est pas comptabilisé dans le Total des versions collectées, mais il est comptabilisé dans le Total des objets collectés. En outre, si vous collectez plusieurs versions d'un élément, chaque version est comptabilisée dans le total des versions collectées, mais seul l'élément lui-même est comptabilisé dans le total des objets collectés.

 

Les destinataires reçoivent l'e-mail une fois tous les éléments collectés.
