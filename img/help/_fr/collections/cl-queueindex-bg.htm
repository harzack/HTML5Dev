Note :
L'option de menu File d'attente pour l'indexation n'est disponible que si votre administrateur vous a accordé les privilèges d'utilisation appropriés.

 
 

  1. OpenText vous recommande de ne pas utiliser cette fonction si des partitions de recherche en lecture seule sont configurées.

  2. Vous devez uniquement mettre en attente une collection pour l'indexation afin de corriger les erreurs d'indexation. Cette option ne doit pas être utilisée pour le fonctionnement normal.

 

Dans certaines situations, un sous-ensemble de contenu dans Content Server doit être indexé à nouveau. Par exemple :

  • OTContentStatus a identifié certains problèmes, qui ont été corrigés, et les objets doivent être indexés à nouveau.

  • Les paramètres d'indexation ont été modifiés et doivent être appliqués.

En commençant avec Content Server 10, Service Pack 2, les administrateurs peuvent désigner les utilisateurs pour sélectionner du contenu spécifique dans Content Server dans une collection, puis indexer à nouveau le contenu.

 

Note :
L'indexation de contenu dans Content Server écrit les informations sur ce contenu dans un index utilisé par la recherche de Content Server. Grâce à la nouvelle indexation du contenu spécifique, les utilisateurs peuvent désormais mettre à jour l'index utilisé par la fonction de recherche.

 
 

Attention
L'indexation d'une collection doit être utilisée avec précaution, car il s'agit d'une opération dont les performances sont parfois coûteuses et pour laquelle plusieurs heures ou plusieurs jours peuvent être nécessaires. Avant d'indexer à nouveau une collection, les utilisateurs doivent réfléchir aux points suivants :

  • le bon moment pour effectuer la réindexation, par exemple pendant une période de faible activité du système ; et

  • réfléchir à la quantité de contenu dans la collection qu'ils ont sélectionnée pour la réindexation. Plus le contenu de la collection est important, plus le processus de réindexation est long.
