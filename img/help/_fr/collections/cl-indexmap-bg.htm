Lorsque vous créez une image de disque, en plus de copier les éléments de la collection spécifiés vers le disque, Content Server génère également un fichier de manifeste de l'image de disque. Ce fichier de manifeste de l'image de disque fournit des informations spécifiques sur la création de l'image de disque, vous permettant ainsi de consulter rapidement des informations détaillées sur l'image de disque, la collection dont sont issus les éléments et sur chaque élément compris dans cette image.

Le fichier de manifeste de l'image de disque est nommé index.html et est situé à la racine de chaque répertoire d'image. Par exemple, si l'emplacement de sortie de votre image de disque est \\server1\filecache\7, le fichierindex.html est situé dans le répertoire \\server1\filecache\7.

Si vous disposez d'éléments OpenText Enterprise Library et que vous avez collecté les attributs, vous pouvez afficher les types de propriétés OpenText Enterprise Library dans le fichier index.html. Dans Content Server, les types de propriété OpenText Enterprise Library sont implémentés en tant qu'attributs de catégorie.

La première section du fichier de manifeste de l'image de texte fournit des informations détaillées sur la création de l'image de disque et sur la collection dont l'image est issue. Elle comprend les champs suivants :

Nom du champ Description
Nom de la collection Nom et lien hypertexte de la collection de Content Server dont provient l'image de disque créée.
Créé Date de création de la collection.
Modifié Date de dernière modification de la création.
Taille de la collection Nombre d'éléments dans la collection.
Image créée Date de création de l'image de disque.
Système de fichiers Format du système de fichiers du disque d'où proviennent les informations.
Taille d'image Nombre d'éléments dans l'image de disque.
Mode de version actuelle Option sélectionnée dans le paramètre Gestion de la version actuelle lors de la création de l'image de disque.

En plus des informations sur la collection et l'image de disque, des informations détaillées sont également disponibles pour chaque élément compris dans l'image de disque, notamment :

Nom du champ Description
Type
Type de nœud Content Server de l'élément.
Créé
Date de création de l'élément.
Modifié
La date de la dernière modification de l'élément.
Taille
Taille en Ko de l'élément.
Nom
Nom de l'élément et un lien hypertexte vers l'élément de Content Server.
Créé par
ID de connexion de l'utilisateur ayant créé l'élément.
Chemin complet
Chemin complet vers l'objet de Content Server.

 

Note :
Si plusieurs versions d'un objet sont collectées, le lien hypertexte dans la colonne Nom ouvre la version la plus récente du document, si cette dernière a été intégrée à l'image de disque. Si la version la plus récente de l'objet n'a pas été intégrée et que la collection de l'objet n'a pas été forcée, aucun lien hypertexte n'est disponible.

 

Si les versions ont été collectées, une section Versions supplémentaire apparaît pour chaque élément, comprenant les informations suivantes pour chaque version de l'élément :

Nom du champ Description
Version
Numéro de la version collectée. Le numéro de version est un lien qui permet d'ouvrir la version spécifique de l'élément.
Nom du fichier
Nom de l'élément de la version.
Créé
Date de création de la version dans Content Server.
Créé par
L'utilisateur ayant créé la version dans Content Server.
Commentaire
Commentaire : (contexte de l'image de disque)
Description
La description, si elle a été saisie dans Content Server, une fois la version créée.
