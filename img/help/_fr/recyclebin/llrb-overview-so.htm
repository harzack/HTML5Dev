Content Server Recycle Bin prend en charge les problèmes associés à la suppression accidentelle de documents Content Server par les utilisateurs. Content Server 10.0.0 contient un module de récupération de document, mais les documents supprimés ne sont accessibles que par l'administrateur. L'application Corbeille résoud ce problème en introduisant une corbeille aux objets pris en charge (et pas uniquement aux documents) dans laquelle sont placés les éléments à supprimer et à partir de laquelle les utilisateurs peuvent les récupérer.

Le package Corbeille Content Server remplace l'opération standard de suppression de document. La nouvelle fonctionnalité déplace les documents « supprimés » dans Content Server vers le volume de la corbeille. Ce volume est structuré de manière telle que tous les documents supprimés durant une journée se trouvent dans un dossier (en d'autres mots, la structure du dossier créé est année:mois:jour).

Il est possible de maintenir les versions, permissions et attributs des documents dans la corbeille. Les objets peuvent être récupérés via l'interface Content Server en déplaçant manuellement les éléments de la corbeille vers leur emplacement d'origine ou un autre emplacement. L'administrateur doit mettre en place une politique pour vider régulièrement la corbeille. Une politique courante consiste à supprimer définitivement les documents qui ont atteint une date d'expiration prédéfinie.
