L'utilisateur peut restaurer des éléments placés dans la corbeille vers n'importe quel emplacement de Content Server pour lequel il dispose de permissions. Par défaut, l'emplacement sera celui à partir duquel l'objet a été supprimé. Attention cependant : lorsqu'un dossier contenant un document est supprimé et que l'on essaie de restaurer ce seul document, l'emplacement par défaut est vide. Les données d'emplacement ainsi que le nom de l'élément peuvent être modifiés. N'oubliez pas que les objets déplacés n'héritent pas des autorisations du nouvel emplacement sauf s'ils ont été déplacés dans un volume (projet, liste de tâches, chaîne, etc.). Une fois un objet restauré, vous devrez peut-être modifier manuellement les droits d'accès.

 

Note :
Lorsqu'un élément portant le même nom est supprimé plusieurs fois durant la même journée, un suffixe de valeur numérique est automatiquement ajouté au nom (c'est également le cas de documents de même nom situés à des emplacements différents dans Content Server). Lorsque deux instances d'un document appelé lisezmoi.txt situés à des emplacements différents dans Content Server (par ex. l'un dans l'espace de travail Entreprise et l'autre dans l'espace de travail Personnel) sont supprimés, le premier document est déplacé dans la corbeille sous le nom lisezmoi.txt tandis que le second est renommé pour y inclure une valeur numérique (lisezmoi.txt2). Si vous choisissez de restaurer un document qui possède une extension numérique, il est restauré sous son nouveau nom. Si vous souhaitez restaurer l'élément sous son nom d'origine, veillez à supprimer le suffixe numérique dans le champ Eléments.
