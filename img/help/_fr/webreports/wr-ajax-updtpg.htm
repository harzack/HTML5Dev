• Cette fonction permet d'appeler une URL spécifique ou un WebReport qui mettra à jour un composant HTML sur la page actuelle. Le WebReport cible renvoie généralement un ensemble HTML à insérer dans un composant HTML de la page actuelle.

  • Traite les arguments suivants :

    • URLreference (number/String) : paramètre obligatoire. URL, objectId numérique (dataId) ou surnom (String) d'un WebReport. Si le surnom est numérique, il doit être mis entre guillemets.

    • HTMLelementID : paramètre obligatoire. Chaîne qui correspond à l'ID d'un élément HTML dans la page actuelle à mettre à jour avec le WebReport cible.

    • optionalParms (String) : paramètre facultatif. Tout paramètre à transmettre au WebReport cible en cours d'exécution. optionalParms présente le format suivant : &parm1=value1,&parm2=value2...

    • JSfunction : paramètre facultatif. Fonction facultative à exécuter après la mise à jour du composant HTML spécifié. Cette fonction est transmise à l'objet HTML (désigné par HTMLelementID) sous la forme d'un paramètre.

  • HTMLelementID représente un élément HTML dont l'attribut innerHTML est inscriptible et utilisable.

  • D'autres informations peuvent être transmises à JSfunction sous forme de paramètres. Pour en savoir plus, voir Exemple d'utilisation de la fonction getJSData.

  • Si vous exécutez le WebReport cible seul, tout ce que vous verrez dans le navigateur sera renvoyé au WebReport appelant. N'oubliez donc pas d'utiliser [LL_WEBREPORT_EXCLUDEHTML /] pour désactiver les en-têtes, pieds de page et fichiers à inclure standard Content Server.

  • En cas d'erreur, un message d'alerte descriptif apparaît dans le navigateur.

  • Vous trouverez un autre exemple dans Cas 1 - Mise à jour d'une vue personnalisée avec un WebReport.
