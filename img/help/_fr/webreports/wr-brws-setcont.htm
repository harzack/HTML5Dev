• Fonction utilisée pour initialiser un ensemble de variables globales, dans browse.js, utilisé par différentes fonctions d'utilitaires. Par exemple, pour gérer les colonnes de tri.

  • Traite les arguments suivants :

    • support_Dir : paramètre obligatoire. Emplacement du répertoire de support : [LL_REPTAG_SUPPORTDIR /]. Par exemple : /Livelink971support/

    • url_Prefix : paramètre obligatoire. [LL_REPTAG_URLPREFIX /]. Par exemple : /Livelink971/livelink.exe

    • my_URL : paramètre obligatoire. Emplacement actuel : [LL_REPTAG_SOURCEID LLURL:BROWSE /]. Par exemple : ?func=ll&objId=<xxx>&objaction=browse

      Où <xxx> est un objectId.

    • next_URL : paramètre obligatoire. Emplacement de l'URL suivante : [LL_REPTAG_NEXTURL ESCAPEURL /]

    • def_SortCol : paramètre obligatoire. Valeur actuelle de la colonne de tri. Par exemple, name. En général, spécifié au moyen d'une balise de paramètre comme [LL_REPTAG_&SORT /]. En présence de plusieurs colonnes de tri possibles, vous pouvez spécifier un tableau au format JSON avec les valeurs actuelles de toutes les colonnes. Les fonctions setColumn recherchent la première valeur non vide. Cela présente généralement l'apparence suivante : ['[LL_REPTAG_&AVSORT /]','[LL_REPTAG_&SORT /]'].

    • sort_Dir : paramètre obligatoire. Direction de tri par défaut. Par exemple : '' ou desc.

    • form_name : paramètre facultatif. Nom du formulaire dans le modèle/l'affichage de rapport ; par défaut, prend la valeur BrowseViewFrm.

    • style_Type : paramètre facultatif. Paramètre booléen facultatif (true/false) pour indiquer si vous souhaitez le style CS10 (true) ou pré-CS10 (false) pour les colonnes du rapport.

  • Cette fonction doit être appelée avant l'utilisation de toute autre fonction dans browse.js.
