Le module WebReports permet de créer des rapports au sein de Content Server qui fournissent des informations à partir de toute table Content Server, standard ou personnalisée, sous tout format de présentation souhaité. Voici quelques exemples de tables Content Server : données Formulaires, données Workflow, données Tâche, métadonnées de documents et données utilisateur. Par exemple, WebReports peut être utilisé pour créer :

  • de simples rapports exigeant des couleurs, un format, un style ou une personnalisation particuliers ;

  • des rapports contenant des liens hypertexte vers votre intranet, Internet ou d'autres emplacements dans Content Server ;

  • des rapports destinés à présenter des données WebForms, y compris des boutons permettant de modifier les données des formulaires ;

  • des rapports imprimables ;

  • des applications de type base de données intégrées, s'exécutant sous Content Server, WebReports en combinaison avec WebForms ;

  • des rapports sophistiqués nécessitant une manipulation de données complexe.

WebReports fournit également des fonctions utiles relatives à l'exécution et au stockage des données des rapports. Par exemple, WebReports propose :

  • l'option d'exporter la sortie du rapport depuis Content Server vers un nœud Content Server, une version Content Server, un WebForm Content Server ou un workflow Content Server ;

  • l'option d'exporter la sortie du rapport en externe vers un e-mail, vers le bureau de l'utilisateur et, si vous êtes l'utilisateur Admin de Content Server, vers Content Server proprement dit ;

  • la possibilité de définir différents fonctionnements par défaut pour l'exécution ;

  • l'option de programmer des rapports pour une fourniture des données en temps opportun, par e-mail ou dans le stockage de Content Server ;

  • l'option d'exécuter des rapports en arrière-plan, sans retarder le navigateur de l'utilisateur.

Fonctionnement de WebReports

WebReports utilise des sources de données telles que des rapports LiveReport pour extraire les données requises de la base de données. La présentation des rapports est contrôlée au moyen d'un modèle, appelé un affichage de rapport, qui contient généralement du code HTML standard et, si souhaité, JavaScript. Lorsqu'il est exécuté, le WebReport insère les données source dans l'affichage de rapport et envoie le résultat obtenu au navigateur ou à une autre destination. Cette technologie de production de rapports est donc extrêmement puissante et flexible, et elle peut être utilisée en toute sécurité par les utilisateurs commerciaux de Content Server.

LiveReports permet d'accéder directement à la base de données de Content Server ; l'autorisation de créer des rapports LiveReport est donc presque toujours réservée aux administrateurs. En revanche, un WebReport ne permet pas d'accéder directement à la base de données : il utilise des rapports LiveReport existants, des requêtes de recherche ou d'autres sources de données. WebReports peut donc être utilisé en toute sécurité par des utilisateurs commerciaux et des développeurs qui ne sont pas des administrateurs.
