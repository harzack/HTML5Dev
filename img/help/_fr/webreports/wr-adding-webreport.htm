Vous pouvez ajouter un WebReport en sélectionnant WebReport à partir du menu standard deContent Server, Ajouter un nouvel élément. Cette option affiche la page Ajouter : WebReport. Vous accédez à l'écran Ajouter des éléments... WebReport :

Ce formulaire doit être renseigné comme suit :

  1. Modifiez le champ Nom avec un titre approprié pour le nouveau WebReport.

  2. Comme pour tous les éléments Content Server, une Description facultative peut être ajoutée. Ces informations n'apparaissent nulle part dans la sortie du WebReport.

  3. La Source de données est un champ facultatif qui permet de sélectionner une source de rapport. Pour renseigner ce champ, cliquez sur Parcourir Content Server, recherchez une source de données appropriée, puis cliquez sur son lien Sélectionner. Les options valides sont les suivantes :

    • LiveReport : utilisation des résultats d'une requête LiveReport.

    • Requête enregistrée : utilisation des résultats d'une requête enregistrée. Une requête enregistrée est créée à partir d'une recherche avancée.

    • Modèle de formulaire : utilisation des résultats d'une requête pour le renvoi de toutes les données de la table SQL associées au modèle sélectionné.

    • Formulaire : utilisation des résultats d'une requête pour le renvoi de toutes les données de la table SQL par le formulaire sélectionné.

    • Document : utilisation d'un document avec une table CSV ou une table HTML basée sur le contenu.

    • Dossier, document composite, liste de tâches, groupe de tâches, chaîne, discussion, tous affichent les enfants lorsqu'ils sont sélectionnés.

    • Application externe, généralement via HTTP.

     

    Note :
    Si aucune source de données n'est sélectionnée, la section de ligne de l'affichage de rapport est ignorée.

     

  4. Si l'option Aller à l'onglet source est sélectionnée, après avoir cliqué sur le bouton Ajouter, l'onglet Source apparaît permettant de sélectionner d'autres sources de données nonContent Server, ou de modifier davantage de paramètres de sources de données spécifiques.

  5. Le fichier d'Affichage de rapport est un champ obligatoire. La sélection d'un affichage de rapport à partir de la liste Utiliser un affichage de rapport par défaut permet d'utiliser un simple affichage de rapport préécrit. Cet affichage de rapport par défaut peut être modifié grâce à l'option de fonction Modifier l'affichage du rapport une fois le WebReport créé. Si aucun affichage de rapport par défaut n'a été sélectionné, le bouton Parcourir peut être utilisé pour sélectionner un affichage de rapport préécrit à partir du bureau de l'utilisateur. Lorsqu'un fichier est sélectionné, les détails du fichier sont automatiquement insérés dans le champ de formulaire correspondant.

  6. Le champ Catégories est éventuellement utilisé pour associer tout attribut prédéfini au WebReport nouvellement créé.

  7. Le champ Créer dans permet éventuellement à l'utilisateur de sélectionner un autre emplacement Content Server pour stocker le nouveau WebReport.
