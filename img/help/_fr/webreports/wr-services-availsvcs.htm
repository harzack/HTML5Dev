Tableau 1-52 : Valeurs valides de &servicetype=

&servicetype=
Description
Gettagdata
Ce service permet l'exécution des balises statiques et/ou des sous-balises à partir du moteur WebReports appelé directement. Cette fonction permet d'exploiter la majorité des fonctions utiles disponibles via le moteur et notamment d'accéder à plus de 60 sous-balises et à plusieurs balises statiques. La plupart des sous-balises permettent d'accéder à des fonctions Content Server utiles telles que la recherche de catégories et d'attributs, le test d'appartenance à un groupe et même la réalisation d'actions Content Server, sous réserve des contrôles d'autorisation normaux. Pour plus d'informations sur les autres paramètres utilisés par ce service, voir Tableau 1-53.
Getstatictags
Ce service renvoie toutes les valeurs statiques appropriées du moteur WebReports dans une structure de données conformément au paramètre responsetype ; toutefois, seul le type de réponse json est actuellement pris en charge. Les balises statiques correspondent à l'ensemble des balises du guide sur les balises WebReports accessibles sous l'intitulé « balises de données » et qui présentent des noms fixes tels que : [LL_REPTAG_DATETIME /].
Liststatictags
Ce service fournit une liste simple de toutes les balises statiques utilisables dans les services gettagdata et getstatictags. Les développeurs l'utilisent principalement pour des raisons pratiques.

Obligatoire
Description
TAGDATA
Oui, si aucune balise STATIC n'est spécifiée ou si STATICTAG renvoie une chaîne vide.
Ce paramètre sert à spécifier les données à utiliser dans toutes les sous-balises spécifiées. Si une balise statique est également spécifiée, les données de balise sont ignorées sauf si la balise statique renvoie une chaîne vide. Cela permet d'utiliser tagdata par défaut lorsque la balise statique ne renvoie pas une valeur utile.
STATICTAG
Oui, si TAGDATA n'est pas spécifié.
Ce paramètre permet de spécifier l'une des balises statiques WebReports qui peuvent être appelées. Si la balise spécifiée renvoie une chaîne vide, la valeur de tagdata, le cas échéant, est utilisée à la place. Sinon, le résultat de la balise statique est utilisé par les sous-balises spécifiées. Le paramètre de balise statique n'exige pas la syntaxe WebReports complète ; seul le nom de la balise statique est utilisé. Par exemple, la balise [LL_REPTAG_USERID /] peut être appelée à l'aide de : &statictag=USERID
 

Note :
Les services WebReports prennent uniquement en charge un petit sous-ensemble des balises statiques disponibles. En général, les balises statiques utilisées pour renvoyer des informations sur un WebReport en cours d'exécution ne sont pas valides, car aucun objet WebReport n'est associé à un appel de service. Pour connaître les balises statiques disponibles, vous pouvez utiliser l'option liststatictags spéciale comme suit : <prefix>?func=webreports.runservice&servicetype=liststatictags
