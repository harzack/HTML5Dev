Le script peut accéder aux données provenant de la source de données et transmises au WebReport. Le premier paramètre transmis à la première fonction Oscript déclarée dans le script est toujours un Assoc appelé Context. S'il existe une source de données pour le WebReport, l'Assoc Context contient une clé appelée data comportant un RecArray, un tableau à deux dimensions représentant toutes les données transmises au WebReport par la source de données. 

[LL_WEBREPORT_STARTSCRIPT NAME:myFunc /]
function String anyName(Dynamic context)
  String cell = ""
    if isDefined (context.data)  // check there is a data source
      cell = context.data[1][1] // return the first cell of data
    end
  return cell
end
[LL_WEBREPORT_ENDSCRIPT /]

Notez que le script vérifie si context.data est bien défini. En l'absence de données renvoyées par la source de données ou en l'absence de source de données, context.data n'est pas défini. Si cette éventualité n'est pas vérifiée, le traitement de l'intégralité du WebReport échoue. Il incombe toujours au script de vérifier que les données qu'il attend ont réellement été transmises.

Le script peut lire ce RecArray, mais il peut aussi y écrire. Cela signifie qu'un script appelé dans la section d'en-tête peut modifier les données provenant de la source de données avant qu'elles soient sorties dans la section des lignes du WebReport.
