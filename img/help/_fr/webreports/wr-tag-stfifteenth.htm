Cette section apporte des informations sur les sous-balises suivantes : RANGE, RECARRAY, RECORD, REDIRECT, REPLACE, RMACTION, RMCLASS, RMHOLDACTION, RMHOLDINFO, RMINFO, RMTYPE, ROUND, RPAD et ISA.

RANGE

Syntaxe

RANGE:<index de début>

ou

RANGE:<index de début>:<index de fin>

Description

Pour toute liste au format : {'item1','item2'}, cette sous-balise renvoie une plage de la liste d'origine, selon l'index de début et l'index de fin spécifiés, ce dernier étant facultatif. Cette sous-balise est identique à la sous-balise SLICE, mais elle fonctionne uniquement avec des LIST. Tous les index s'appliquent à partir de la valeur 1 (un), si bien que la plage inclut tous les caractères depuis l'<index de début>, celui-ci étant compris, jusqu'à l'<index de fin>, celui-ci étant compris, sous réserve que ce dernier soit spécifié. Si l'index de fin est omis ou dépasse la longueur totale de la chaîne, le reste de la chaîne est utilisé. Une LIST respecte la convention LIST de Content Server, où le premier caractère est une {, le dernier caractère une } et chaque élément est séparé par une virgule. La sortie de cette sous-balise est une liste au même format. Une LIST peut être déréférencée au moyen de la sous-balise LIST.

Exemple

Ces différents exemples de la sous-balise RANGE illustrent les différents résultats produits, en supposant que la balise associée contienne la liste suivante : {'Apple','Pear','Orange','Mango'}

RANGE:3 renvoie : {'Orange','Mango'}

RANGE:2:3 renvoie : {'Pear','Orange'}

RANGE:3:100 renvoie : {'Orange','Mango'}

RANGE:2:3 LIST:1 renvoie : 'Pear'

RECARRAY

Syntaxe

RECARRAY:<index_ligne>:<index_colonne>

ou

RECARRAY:<index_ligne>:<nom_champ>

Description

Permet l'indexation du type de données RECARRAY de Content Server. Si la colonne extraite de la source de données est un RECARRAY, c'est-à-dire une structure de données Content Server constituée d'une LIST de RECORD, la valeur du champ spécifié par <index_ligne> et <index_colonne> ou <nom_champ> est renvoyée, et cette valeur uniquement. Notez que cette sous-balise peut être utilisée de manière répétitive et/ou en combinaison avec d'autres sous-balises, comme ASSOC et LIST, pour accéder à des informations dans des structures de données complexes.

Informations associées

Voir également : ASSOC, LIST et RECORD.

RECORD

Syntaxe

RECORD:<index>

ou

RECORD:<nom_champ>

Description

Permet l'indexation du type de données RECORD de Content Server. Si la colonne extraite d'une source de données est un RECORD, une structure de données Content Server, la valeur du champ spécifié par <index> ou par <nom_champ> est renvoyée, et cette valeur uniquement. Notez que cette sous-balise peut être utilisée de manière répétitive et/ou en combinaison avec d'autres sous-balises, comme ASSOC et LIST, pour accéder à des informations dans des structures de données complexes.

Informations associées

Voir également : ASSOC, LIST et RECARRAY.

RECORD:<nom_champ>

Description

REPLACE

Syntaxe

REPLACE:<param1>:<param2>

Description

Effectue une action « remplacer tout » sur les éléments de données provenant de la source de données. Toutes les occurrences de la chaîne <param1> sont remplacées par <param2>. Par exemple, si l'expression [LL_REPTAG=PRIORITY REPLACE:3:Low /] est utilisée, chaque occurrence du nombre « 3 » dans le champ Priorité est remplacée par le mot « Faible ». Notez l'absence d'espace entre REPLACE, <param1> et <param2>. Utilisez uniquement les deux-points en guise de séparateur.

RMCLASS

Syntaxe

RMCLASS:<param>

Description

Fournit des informations de métadonnées uniques aux objets de classement RM. Nécessite que le dataid du classement RM proprement dit soit renvoyé à partir de la balise principale. Pour accéder aux détails d'un classement RM à partir du nœud auquel il est appliqué, vous pouvez utiliser la sous-balise RMINFO : [LL_REPTAG=DATAID RMINFO:CLASSID RMCLASS:<param> /]. Vous pouvez accéder d'une façon similaire aux détails d'ISA au moyen de la sous-balise ISA. Par exemple, [LL_REPTAG=DATAID RMINFO:CLASSID RMCLASS:RSI RSI:URL /] obtient l'URL pour ouvrir un ISA seulement sur la base du dataid du nœud auquel l'ISA/le classement RM est appliqué. Notez qu'un classement RM est simplement un type de nœud Content Server standard et donc que les métadonnées de base associées au nœud sont disponibles au moyen de la sous-balise NODEINFO Par exemple, [LL_REPTAG=DATAID NODEINFO:CREATEDATE /] renvoie la date de création du classement RM. En revanche, RMCLASS renvoie des métadonnées propres au sous-type d'objet de classement RM, qui sont présentes uniquement sur l'onglet spécifique de tels objets. Si le module Records Management n'est pas installé, cette sous-balise ne renvoie rien, quelles que soient les options indiquées.

RMHOLDACTION

Syntaxe

RMHOLDACTION:<params>

Description

Permet d'effectuer différentes actions relatives aux attentes, comme créer, appliquer et enlever des attentes ou créer des types d'attentes.

Si le module Records Management n'est pas installé, cette sous-balise ne renvoie rien, quelles que soient les options indiquées.

Paramètres

  • Modificateur : RMHOLDACTION:APPLY:<ID ou nom d'attente>

    Description : applique une attente au nœud spécifié dans la balise principale.

  • Modificateur : RMHOLDACTION:REMOVE:<ID ou nom d'attente>

    Description : enlève une attente au nœud spécifié dans la balise principale.

  • Modificateur : RMHOLDACTION:CREATE:HOLD:<params>

    Description : crée une nouvelle attente. Les données présentes dans la balise principale sont ignorées. Vous pouvez utiliser les paramètres suivants :

    Nom du paramètre
    Détails
    Nom de l'attente
    Obligatoire. Est converti en majuscules
    Date d'application
    Obligatoire. Doit être une chaîne de date valide, par exemple « yyyy-mm-dd ».
    Type d'attente
    Obligatoire. Doit correspondre à l'un des types d'attentes existants
    Autre ID d'attente
    Facultatif.
    Commentaire
    Facultatif.
    Date à interrompre
    Facultatif. Si ce paramètre est défini, il doit être une chaîne de date valide, par exemple « yyyy-mm-dd ».

    Exemple

    L'instruction suivante crée une attente appelée « Attente 1 », avec la date d'application « 1er mars 2012 », du type « Type d'attente 1 » :

    [LL_REPTAG_'123456' RMHOLDACTION:CREATE:HOLD:"Hold 1":"2012-03-01":"Hold Type 1" /]

    L'exemple suivant crée une attente appelée « Attente 2 », avec une date d'application du « 1er mars 2012 », de type « Type d'attente 2 », avec un « autre ID d'attente » vide, un commentaire « un commentaire » et une date à interrompre du « 15 mars 2012 ».

    [LL_REPTAG_'123456' RMHOLDACTION:CREATE:HOLD:"Hold 2":"2012-03-01":"Hold Type 1":"":"a comment":"2012-03-15" /]

  • Modificateur : RMHOLDACTION:CREATE:TYPE:<params>

    Description : crée un nouveau type d'attente. Vous pouvez utiliser les paramètres suivants :

    Nom du paramètre
    Détails
    Code
    Obligatoire. Est converti en majuscules.
    Période par défaut
    Obligatoire. Doit être un entier.
    Description
    Facultatif. Est converti en majuscules

    Exemple

    Cet exemple crée un type d'attente appelé « TYPE D'ATTENTE 1 » avec une période par défaut de 30 :

    [LL_REPTAG_'123456' RMHOLDACTION:CREATE:TYPE:"Hold Type 1":30 /]

    L'exemple suivant crée un type d'attente appelé « TYPE D'ATTENTE 2 », avec une période par défaut de 30 et la description « DESCRIPTION DE TYPE D'ATTENTE 2 ».

    [LL_REPTAG_'123456' RMHOLDACTION:CREATE:TYPE:"Hold Type 2":30:"Hold Type 2 Description /]

Informations associées

Voir également RMACTION, RMCLASS, RMHOLDINFO, RMINFO, RMTYPE et ISA pour d'autres fonctionnalités Records Management.

ROUND

Syntaxe

ROUND:<valeur d'arrondi>

Description

Arrondit une valeur numérique selon la valeur spécifiée par le paramètre de ROUND. Si le paramètre de ROUND n'est pas spécifié, il prend la valeur 2 par défaut.

Le paramètre de ROUND doit être un entier ou un réel. Cet entier ou ce réel doit être supérieur ou égal à 0. Une erreur survient si la balise ne contient pas un entier ou un réel valide. En l'absence de portion décimale, la valeur arrondie est complétée par des zéros, conformément au paramètre de ROUND spécifié. Voir le deuxième exemple ci-dessous.

Exemple

25.479 ROUND:2 renvoie : « 25.48 »

45 ROUND:3 renvoie : « 45.000 » // des zéros sont ajoutés

50.99 ROUND:0 renvoie : « 51 »

-4.545 ROUND:1 renvoie : « -4.5 »

RPAD

Syntaxe

RPAD:<caractère de complément>:<largeur>

Description

Ajoute des caractères de complément à droite de la donnée existante. Ajoute suffisamment de caractères pour remplir le champ jusqu'à la largeur spécifiée. En présence d'une mantisse, seule la largeur du nombre suivant les points décimaux est utilisée.

Exemple

Par exemple, soit une balise de donnée renvoyant le nombre 123.456. L'expression [LL_REPTAG_1 RPAD:9:5 /] renvoie : 123.45699.

Si la balise de donnée renvoie le nombre 123, l'expression [LL_REPTAG_1 RPAD:9:5 /] renvoie : 12399.

ISA

Syntaxe

RSI:<param>

Description

Un identificateur de séries d'archives (ISA) permet aux gestionnaires de documents d'appliquer des informations de conservation et de destruction à des archives électroniques dans Livelink. Vous pouvez définir de nouveaux ISA ou modifier des ISA existants au moyen du menu Entreprise, option Records Management suivie par le lien ISA. Cette sous-balise permet d'accéder aux métadonnées associées à l'ISA souhaité. Pour plus d'informations sur l'accès aux données ISA à partir du nœud de classement RM, voir la sous-balise RMCLASS. Si le module Records Management n'est pas installé, cette sous-balise ne renvoie rien, quelles que soient les options indiquées.
