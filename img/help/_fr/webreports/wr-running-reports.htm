Par défaut, l'exécution d'un WebReport entraîne la fourniture de la sortie combinée de l'affichage de rapport et de toute source de données au navigateur de l'utilisateur. Les WebReports peuvent également être « fournis » vers d'autres destinations à l'aide de nombreuses options d'exportation. Les trois méthodes utilisées le plus fréquemment lors de l'exécution d'un WebReport sont :

  1. La sélection du lien WebReport dans Content Server : emplacement dans lequel un WebReport est visible dans un recueil Content Server. L'utilisateur peut cliquer sur le lien du nom afin d'exécuter la version actuelle du WebReport. Cette approche peut également être utilisée pour les versions répertoriées dans le menu Fonctions d'un WebReport en sélectionnant Propriétés, puis Versions. Elle n'est pas la même que celle utilisée pour les documents normaux pour lesquels une option de visualisation ou d'ouverture serait exécutée.

     

    Note :
    Le comportement du lien WebReport peut être modifié en changeant les paramètres par défaut à partir du menu Fonctions d'un WebReport en sélectionnant Propriétés, puis Destination.

     

  2. Sélection de l'option Ouvrir dans le menu Fonctions : permet d'exécuter la même action qu'en cliquant sur le lien du nom.

  3. Appel de l'URL du WebReport approprié : cette option peut être utilisée afin de créer des relations complexes entre différents objets dans Content Server. Voici deux exemples possibles :

    • Une vue personnalisée Content Server peut appeler un WebReport afin de collecter des données à afficher.

    • Un WebForm Content Server peut appeler un WebReport pour créer des informations de formulaire telles que des listes de sélection.

    • Un WebReport peut appeler un autre WebRport afin de créer des applications de type base de données relationnelle.

    Cette option peut également être utilisée afin d'appeler un WebReport à partir d'un objet URL Content Server. L'utilisation de cette approche permet de créer des URL uniques comprenant des paramètres qui peuvent être utilisés dans l'affichage de rapport. Tous les paramètres transmis dans l'URL peuvent être référencés dans l'affichage de rapport à l'aide de balises spécifiques. Par exemple, [LL_REPTAG_&parmname /]. Ces URL multiples peuvent être créées afin de permettre à un seul WebReport de se comporter de manières différentes. Il est également possible d'inclure des paramètres d'entrée à transmettre directement aux sources de données sous-jacentes. Pour en savoir plus, voir Utilisation des paramètres.

Transmission de paramètres

Lors de l'exécution d'un WebReport, les paramètres peuvent être inclus dans l'URL pour le WebReport lui-même, ainsi que pour la source de données associée. Pour en savoir plus, voir Utilisation des paramètres.
