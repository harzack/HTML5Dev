Un WebReport peut utiliser une catégorie Content Server comme source de données. Un utilisateur peut sélectionner une ou plusieurs catégories Content Server, puis un ou plusieurs attributs dans chaque catégorie. Le WebReport va traiter une requête SQL par rapport à la base de données DTREE Content Server et aux tables LLATTRDATA, puis renvoyer un enregistrement pour chaque élémentContent Server avec les catégories sélectionnées appliquées.

Chaque enregistrement renvoyé à partir de la requête contiendra les valeurs suivantes :

Champ Descriptions
DataID Le DataID de l'objet tel que stocké dans la table de base de données DTREE Content Server.
ParentID Le ParentID de l'objet tel que stocké dans la table de base de données DTREE Content Server.
Sous-type : Le sous-type de l'objet tel que stocké dans la table de base de données DTREE Content Server.
Nom Le nom de l'objet tel que stocké dans la table de base de données DTREE Content Server.
Valeurs d'attributs La valeur actuelle de chaque attribut sélectionné tel que stockée dans la table LLAttrData Content Server. L'en-tête de la colonne sera le nom de l'attribut système. Par exemple, le nom d'origine de l'attribut avec les espaces remplacées par des traits de soulignement.

Fonctionnalités d'une catégorie comme source de données

Une fois la source de données Catégorie Content Server sélectionnée, les fonctionnalités suivantes sont disponibles :

  • Utilisation du bouton Parcourir Content Server pour sélectionner une catégorie Content Server. Après la sélection d'une catégorie, la page est actualisée et affiche tous les attributs de la catégorie.

  • Sélection d'attributs : sélectionnez la case à cocher en regard d'un attribut spécifique pour inclure l'attribut dans les résultats de la requête. Après la sélection d'un attribut, le menu Paramètres s'affiche pour cet attribut. Si un attribut est désélectionné, le menu Paramètres disparaît pour cet attribut.

  • Utilisation des boutons Cocher tout et Décocher tout pour cocher ou décocher toutes les cases à cocher de la catégorie spécifiée.

  • Utilisation de l'icône Supprimer la catégorie, , pour supprimer une catégorie spécifiée.

  • Utilisation de l'icône Ajouter une catégorie, , pour ajouter une autre catégorie. L'icône Ajouter une catégorie sera uniquement affichée à la fin de la liste actuelle de catégories. L'administrareur Content Server peut configurer le nombre de catégories pouvant être incluses dans la liste Source de données, ainsi l'icône Ajouter une catégorie ne s'affiche pas une fois cette limite atteinte.

  • Opérateur de jointure Et/Ou de la catégorie : pour chaque nouvelle catégorie, sélectionnez le bouton radio Et/Ou pour spécifier si les catégories seront jointes dans la requête par une jointure interne (Et) ou par une jointure externe gauche (Ou).

Utilisation des paramètres avec une catégorie comme source de données

L'éditeur de WebReport peut définir une catégorie/des attributs comme paramètres WebReport et spécifier un opérateur de requête à partir de la page Source de données WebReports. Quand une catégorie/un attribut a été coché sur la page Source de données, une nouvelle icône menu Ajouter un paramètre, ,s'affiche. A partir de celle-ci, un opérateur de requête peut être sélectionné. Quand un opérateur de requête a été sélectionné dans le menu Paramètres, l'attribut est alors disponible via le processus « Extraction de paramètre » dans l'onglet Paramètres.

Quand un attribut a un paramètre opérateur de requête sélectionné, l'icône menu Editer un paramètre, , s'affiche et une coche, , s'affiche en regard de l'opérateur de requête sélectionné.

Si un attribut est actuellement utilisé comme paramètre et si l'opérateur de requête est modifié sur la page Source de données, la requête est désormais basée sur l'opérateur de requête nouvellement sélectionné. Une option proposera au concepteur du WebReport de mettre à jour le texte affiché sur la page Paramètres ; quel que soit le texte affiché sur la page Paramètres, la requête SQL traitée utilisera toujours l'opérateur de requête tel que sélectionné dans le menu Paramètres sur la page Source de données. Pour désactiver un opérateur de requête sélectionné dans le menu Paramètres :

  • sélectionnez un autre opérateur de requête à partir du même menu pour définir un nouvel opérateur de requête,

  • ou sélectionnez l'opérateur de requête existant pour reconfigurer l'opérateur de requête comme indéfini.

Actions source de données catégorie

Trois actions supplémentaires sont disponibles pour l'éditeur de WebReport quand une catégorie Content Server est la source de données sélectionnée.

  1. Sélectionnez le bouton Afficher SQL pour voir le SQL généré pour les catégories, les attributs et les paramètres définis actuellement sélectionnés.

     

    Note :
    Seul un éditeur WebReports avec des privilèges Modification peut afficher le SQL.

     

  2. Sélectionnez la case à cocher Filtrer les résultats par autorisations utilisateur pour vous assurer que les résultats de la requête seront limités en fonction des autorisations des utilisateurs qui exécutent le WebReport. L'utilisateur doit avoir l'autorisation Voir pour voir un élément spécifique.

     

    Note :
    La valeur par défaut pour cette fonctionnalité est COCHEE ; seul un utilisateur avec des privilèges Administrateur Content Server peut modifier cette fonctionnalité.

    Le filtre des autorisations est appliqué aux résultats de requête comme processus séparé, une fois la requête de base de données terminée.

     

  3. Définissez le Nombre maximal de résultats sur une valeur de nombre entier indiquant le nombre maximal de résultats que la requête de base de données doit renvoyer. Notez que le nombre d'éléments affichés peut être inférieur au Nombre maximal de résultats quand le filtre des autorisations est appliqué après le renvoi de la requête. L'éditeur WebReports ne peut sélectionner une valeur supérieure à celle définie par l'administrateur Content Server.
