Cette section apporte des informations sur les sous-balises suivantes : LENGTH, LIST, LLURL, LOWER et LPAD.

LENGTH

Syntaxe

LENGTH

Description

Indique la longueur des données renvoyée par la balise principale. Par exemple, si les données résolues par la balise principale ont une longueur de 10 caractères, cette sous-balise renvoie à la place les 10 caractères. Elle peut être utile pour déterminer la validité de données, c'est-à-dire comptant un nombre de caractères supérieur au minimum indiqué, ainsi que les paramètres HTML SIZE.

LIST

Syntaxe

LIST:<index>

Description

Permet de récupérer l'index spécifié pour le type de données Content Server <LISTE>. L'index débute à 1.

Exemple

Par exemple, [LL_REPTAG_1 LIST:3> /] renvoie le troisième élément de la liste.

Informations associées

Voir également : ASSOC, RECORD et RECARRAY.

LLURL

Syntaxe

LLURL:<params>

Description

Cette sous-balise permet de facilement créer des URL et/ou d'insérer des extraits de code de fonctionnalité Content Server. Par exemple, si votre rapport doit contenir une image représentant le type de nœud, vous pouvez utiliser la sous-balise suivante : [LL_REPTAG=DATAID LLURL:GIF /], avec pour résultat si DATAID est un document Word. Pour que cette sous-balise fonctionne correctement, les données renvoyées par la balise principale doivent être un ID d'objet Content Server. Par exemple, la colonne DATAID de la table DTREE. Notez que, pour les URL, le comportement par défaut est que l'URL soit créée sans vérifier qu'une valeur DataID valide a été fournie et sans vérifier si l'utilisateur qui exécute le WebReport dispose des autorisations appropriées pour exécuter l'URL. Pour forcer la vérification du DataID et s'assurer que l'utilisateur dispose au minimum de l'autorisation Voir le contenu, ajoutez :PERMCHECK à la fin de la syntaxe LLURL. Par exemple, LLURL:BROWSE:PERMCHECK. Pour spécifier des niveaux d'autorisation plus précis, vous pouvez utiliser la sous-balise PERMCHECK avec l'instruction [LL_WEBREPORTS_IF ... /].

Exemple

De manière générale, cette sous-balise peut être divisée en deux types de sortie. Un groupe renvoie une portion d'URL ou de lien pouvant être utilisée dans votre propre code HTML, par exemple LLURL:BROWSE et LLURL:DEFAULTLINK. Dans ce cas, la sous-balise est généralement utilisée conjointement avec URLPREFIX. Exemple de création d'une balise d'ancrage HTML :

< A HREF="[LL_REPTAG_URLPREFIX /][LL_REPTAG=DATAID LLURL:BROWSE /]">my browse link</A>

La résolution après exécution est similaire à ce qui suit :

< A HREF="http://localhost/ll9.5sp1/livelink.exe?func=ll&objId=41680&objAction=browse&sort=name" >my browse link</A>

Le second groupe insère des images ou des liens complets, tels qu'ils apparaîtront dans Content Server. Par exemple, LLURL:FUNCTIONMENU, LLURL:DROPDOWNLIST et LLURL:GIF. Un menu Fonctions peut être ajouté avec la syntaxe suivante : [LL_REPTAG=DATAID LLURL:FUNCTIONMENU /] et, si elle est affichée dans un WebReport, la sortie de cette balise apparaîtra tout simplement sous la forme d'un menu Fonctions parfaitement opérationnel ().
