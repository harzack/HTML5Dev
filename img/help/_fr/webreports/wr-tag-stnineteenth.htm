Cette section apporte des informations sur les sous-balises suivantes : VERINFO et VERSIONCAT.

VERINFO

Syntaxe

VERINFO:<format>:<numéro de version>

Description

Cette sous-balise renvoie des informations sur la version d'un ID de nœud. Le numéro de version est un paramètre facultatif. S'il est ajouté à la fin des arguments de la sous-balise, les informations sur la version concernée sont renvoyées, sinon les informations sur la version la plus récente seront renvoyées.

Paramètres

Format
Description
COMMENT
Commentaires associés à la version, à ne pas confondre avec les commentaires associés au nœud.
CREATEDATE
Date de création de la version.
EFSPATH
Emplacement du fichier de version dans l'EFS. Cette option renvoie une valeur uniquement si l'utilisateur administrateur exécute le rapport.
FILECREATEDATE
Date de création du fichier de version.
FILECREATOR
Auteur de la version.
FILEDATASIZE
Taille de la version.
FILENAME
Nom du fichier de version.
FILETYPE
Type de fichier.
GIF
Insère le code HTML complet pour l'image de nœud par défaut. Prend en charge le paramètre facultatif PATH, qui renvoie le chemin d'accès de l'image, au lieu de la source HTML complète.
ID
ID unique de la version.
MIMETYPE
Type MIME de la version.
MODIFYDATE
Date de dernière modification de la version.
NODEID
ID du nœud contenant cette version.
NUMBER
Numéro de version.
OWNER
Propriétaire de la version.
PROVIDERID
ID du fournisseur de stockage.
PROVIDERNAME
Nom du fournisseur de stockage.
RENDITION:<type>:<propriété>
Fournit l'accès aux propriétés de présentation, par exemple, VERINFO:RENDITION:PDF:MIMETYPE.
RENDITIONCOUNT
Renvoie le nombre de présentations disponibles dans la dernière version du nœud.
RENDITIONDEFAULT
Renvoie le type de présentation par défaut pour le nœud en fonction de la priorité définie dans les pages Administration des présentations.
TYPE
Type de version.

Exemple

[LL_REPTAG=DATAID VERINFO:MIMETYPE:5 /] renvoie le type MIME de la 5e version d'un objet, tandis que [LL_REPTAG=DATAID VERINFO:MIMETYPE /] renvoie le type MIME de la dernière version.

VERSIONCAT

Syntaxe

VERSIONCAT:<params>

Description

Renvoie des informations sur les catégories et attributs associés à une version spécifique de l'ID d'objet actuel. Cette sous-balise prend en charge les ensembles d'attributs et les attributs à valeurs multiples, au moyen des opérateurs MV et SET.

Paramètres

Syntaxe Description
VERSIONCAT:<num_ver> Renvoie Vrai si une ou plusieurs catégories assignées à la version du nœud sont présentes ; renvoie Faux si tel n'est pas le cas.
VERSIONCAT:<num_ver>:JSARRAY Option unique pour la sous-balise VERSIONCAT, qui résulte en une représentation d'objet JavaScript de toutes les données de catégories et d'attributs, avec prise en charge complète des ensembles et des valeurs multiples, pour une version de nœud spécifique. Pour plus d'informations sur cette fonction, voir l'aide avancée JSARRAY des catégories.
VERSIONCAT:<num_ver>:RAW
Cette option résulte en une représentation Oscript plate de toutes les données de catégories et d'attributs associées à une version de nœud particulière.
Voir Requêtes en tant que sources de données pour savoir comment utiliser cette sortie et les balises associées en tant que sources de données pour un autre WebReport.
Voir la sous-balise TOJSON pour savoir comment convertir cette fonction en une structure de données pouvant être interprétée par JavaScript au niveau du client.
Voir les balises connexes LLURL:FUNCTIONMENU:RAW et LLURL:ADDITEM:ITEMMENU:RAW.
VERSIONCAT:<num_ver>:DISPLAY Renvoie une chaîne contenant la liste des noms des catégories, séparées par des points-virgules, assignées à la version du nœud. Toute catégorie contenant des attributs obligatoires est suivie d'un point d'exclamation entre crochets : [!].
VERSIONCAT:<num_ver>:MAND Renvoie Vrai si une catégorie contenant un attribut obligatoire est assignée à la version du nœud ; renvoie Faux si tel n'est pas le cas.
VERSIONCAT:<num_ver>:<nom_catég> Renvoie Vrai si la catégorie <nom_catég> est présente ; renvoie Faux si tel n'est pas le cas.
VERSIONCAT:<num_ver>:<nom_catég>:DISPLAY
Renvoie une chaîne contenant la liste des valeurs d'attributs, séparées par des points-virgules, pour la catégorie spécifiée par <nom_catég>. Si des ensembles d'attributs ou des attributs à valeurs multiples sont contenus dans la catégorie, ils sont affichés conformément à la syntaxe suivante : 'attval1;attval2;[SET3:attval4,attval5,[MV6:attval7,attval8]];attval9'
Ici, attval1, attval2 et attval9 sont des valeurs d'attribut normales.
[SET3: représente le début d'un ensemble d'attributs où 3 est l'ID d'attribut de l'ensemble (voir la remarque ci-dessous). L'ensemble d'attributs contient deux attributs normaux, attval4 et attval5, et un attribut à valeurs multiples, commençant par [MV6:, où 6 est l'ID de l'attribut à valeurs multiples. L'attribut à valeurs multiples contient deux valeurs pour l'attribut 6, soit : attval7 et attval8.
 

Note :
Pour identifier un ID d'attribut, consultez la barre de statut du navigateur tout en passant la souris sur le nom de l'attribut dans la page de définition de catégorie.

 
VERSIONCAT:<num_ver>:<nom_catég>:ID Renvoie l'ID de la catégorie <nom_catég>, si celle-ci est présente.
VERSIONCAT:<num_ver>:<nom_catég>:<nom_attr> Renvoie Vrai si l'attribut <nom_attr> existe avec une valeur non vide dans la catégorie <nom_catég> ; renvoie Faux si tel n'est pas le cas. Si <nom_attr> spécifie un ensemble d'attributs, il suffit d'une seule entrée parmi les valeurs des attributs de l'ensemble pour que la sous-balise renvoie Vrai. Si <nom_attr> spécifie un attribut à valeurs multiples, il suffit d'une seule entrée parmi ces valeurs pour que cette sous-balise renvoie Vrai. Le paramètre <nom_attr> peut également être imbriqué dans un ensemble d'attributs, sous forme d'attribut standard ou d'attribut à valeurs multiples.
VERSIONCAT:<num_ver>:<nom_catég>:<nom_attr>:DISPLAY Renvoie la valeur de l'attribut <nom_attr> pour la catégorie <nom_catég>. Si <nom_attr> est un ensemble d'attributs ou un attribut à valeurs multiples, une liste de valeurs séparées par des virgules est renvoyée.
VERSIONCAT:<num_ver>:<nom_catég>:<nom_attr>:ID Renvoie l'ID de l'attribut <nom_attr> pour la catégorie <nom_catég>, s'il est présent. Si <nom_attr> se trouve dans un ensemble d'attributs, l'ID est renvoyé si aucun attribut de nom <nom_attr> n'est présent au niveau supérieur dans la catégorie. Si l'attribut <nom_attr> est présent au niveau supérieur de la catégorie et dans un ensemble d'attributs de la même catégorie, l'ID de l'attribut au niveau supérieur, hors de l'ensemble, a la priorité.
VERSIONCAT:<num_ver>:<nom_catég>:<nom_attr>:<val_attr> Renvoie Vrai si l'attribut <nom_attr> existe avec une valeur égale à <val_attr> dans la catégorie <nom_catég> ; renvoie Faux si tel n'est pas le cas. Le paramètre <nom_attr> peut être un attribut d'un ensemble d'attributs, y compris un attribut à valeurs multiples ou un élément dans un attribut à valeurs multiples.
VERSIONCAT:<num_ver>:<nom_catég>:MV:<nom_attr>:COUNT Renvoie le nombre d'attributs à valeurs multiples utilisés.
VERSIONCAT:<num_ver>:<nom_catég>:MV:<nom_attr>:<n> Renvoie Vrai si un élément spécifique non vide est présent dans l'attribut à valeurs multiples <nom_attr>, pour la catégorie <nom_catég>, à l'emplacement d'index <n> ; renvoie Faux si tel n'est pas le cas. La valeur de <n> doit être un entier positif compris entre 1 et <n>, où <n> ne dépasse pas la longueur de l'attribut à valeurs multiples. Si <n> dépasse la longueur de l'attribut à valeurs multiples, une chaîne vide est renvoyée.
VERSIONCAT:<num_ver>:<nom_catég>:MV:<nom_attr>:<n>:<val_attr> Renvoie Vrai si la valeur <val_attr> est présente dans l'attribut à valeurs multiples <nom_attr>, pour la catégorie <nom_catég>, à l'emplacement d'index <n> ; renvoie Faux si tel n'est pas le cas. La valeur de <n> doit être un entier positif compris entre 1 et <n>, où <n> ne dépasse pas la longueur de l'attribut à valeurs multiples.
VERSIONCAT:<num_ver>:<nom_catég>:MV:<nom_attr>:<n>:DISPLAY Renvoie la valeur présente dans l'attribut à valeurs multiples <nom_attr>, pour la catégorie <nom_catég>, à l'emplacement d'index <n>, où <n> est un entier positif compris entre 1 et <n>. La valeur de <n> doit être un entier positif compris entre 1 et <n>, où <n> ne dépasse pas la longueur de l'attribut à valeurs multiples.
VERSIONCAT:<num_ver>:<nom_catég>:MV:<nom_attr>:INDEX:<val_attr> Renvoie le premier index dans l'attribut à valeurs multiples, <nom_attr>, pour la catégorie, <nom_catég> où la valeur <val_attr> est égale à la valeur contenue dans l'attribut à valeurs multiples au niveau de l'index. Par exemple, si la valeur <val_attr> est présente pour la première fois au troisième élément de <nom_attr>, le nombre 3 est renvoyé.
VERSIONCAT:<num_ver>:<nom_catég>:SET:<nom_ensemble>:<nom_attr> Utilise l'opérateur SET permettant de référencer explicitement des ensembles d'attributs. Cet exemple renvoie Vrai si l'attribut <nom_attr> dans l'ensemble <nom_ensemble> n'est pas vide. Seuls les attributs présents dans <nom_ensemble> peuvent être référencés ; si <nom_attr> existe en dehors de l'ensemble, il n'est pas trouvé. Si <nom_attr> est un attribut à valeurs multiples, la valeur Vrai est renvoyée sous réserve qu'au moins un champ de cet attribut ne soit pas vide.
VERSIONCAT:<num_ver>:<nom_catég>:SET:<nom_ensemble>:<nom_attr>:DISPLAY Renvoie la valeur présente dans l'attribut <nom_attr> pour la catégorie <nom_catég> si <nom_attr> est un attribut dans <nom_ensemble>. Si <nom_attr> est un attribut à valeurs multiples, une liste des valeurs définies, séparées par des virgules, est renvoyée. En l'absence de valeurs définies, soit dans un attribut à valeurs multiples, soit dans un attribut standard, une liste vide est renvoyée.
VERSIONCAT:<num_ver>:<nom_catég>:SET:<nom_ensemble>:<nom_attr>:ID Renvoie l'ID de l'attribut <nom_attr> dans l'ensemble <nom_ensemble>. Si l'attribut <nom_attr> existe en dehors de l'ensemble <nom_ensemble>, il n'est pas renvoyé. L'attribut <nom_attr> peut être un attribut standard ou à valeurs multiples.
VERSIONCAT:<num_ver>:<nom_catég>:SET:<nom_ensemble>:<nom_attr>:<val_attr> Renvoie Vrai si l'attribut <nom_attr> existe avec une valeur égale à <val_attr> dans l'ensemble <nom_ensemble> ; renvoie Faux si tel n'est pas le cas. Si l'attribut <nom_attr> est un attribut à valeurs multiples, chaque élément est comparé à <val_attr> et Vrai est renvoyé si l'un d'entre eux est égal à cette valeur.
VERSIONCAT:<num_ver>:<nom_catég>:SET:<nom_ensemble>:MV:<nom_attr>:COUNT Renvoie le nombre d'attributs à valeurs multiples renseignés.
VERSIONCAT:<num_ver>:<nom_catég>:SET:<nom_ensemble>:MV:<nom_attr>:<n> Renvoie Vrai si un élément spécifique non vide est présent dans l'attribut à valeurs multiples <nom_attr>, dans un ensemble, pour la catégorie <nom_catég>, à l'emplacement d'index <n> ; renvoie Faux si tel n'est pas le cas. La valeur de <n> doit être un entier positif compris entre 1 et <n>, où <n> ne dépasse pas la longueur de l'attribut à valeurs multiples.
VERSIONCAT:<num_ver>:<nom_catég>:SET:<nom_ensemble>:MV:<nom_attr>:<n>:<val_attr> Renvoie Vrai si la valeur <val_attr> est présente dans l'attribut à valeurs multiples <nom_attr>, dans un ensemble, pour la catégorie <nom_catég>, à l'emplacement d'index <n> ; renvoie Faux si tel n'est pas le cas. La valeur de <n> doit être un entier positif compris entre 1 et <n>, où <n> ne dépasse pas la longueur de l'attribut à valeurs multiples.
VERSIONCAT:<num_ver>:<nom_catég>:SET:<nom_ensemble>:MV:<nom_attr>:<n>:DISPLAY Renvoie la valeur présente dans l'attribut à valeurs multiples <nom_attr>, dans un ensemble, pour la catégorie <nom_catég>, à l'emplacement d'index <n>, où <n> est un entier positif compris entre 1 et <n>. La valeur de <n> doit être un entier positif compris entre 1 et <n>, où <n> ne dépasse pas la longueur de l'attribut à valeurs multiples.
VERSIONCAT:<num_ver>:<nom_catég>:SET:<nom_ensemble>:MV:<nom_attr>:INDEX:<val_attr> Renvoie le premier index dans l'attribut à valeurs multiples <nom_attr>, dans un ensemble, pour la catégorie <nom_catég> où la valeur <val_attr> est égale à la valeur contenue dans l'attribut.
VERSIONCAT:<num_ver>:<nom_catég>:MVSET:<nom_ensemble>:<n>:<nom_attr> Renvoie Vrai si l'ensemble à valeurs multiples <nom_ensemble> comporte un attribut <nom_attr> non vide, à l'emplacement d'index <n>, dans la catégorie <nom_catég> ; renvoie Faux si tel n'est pas le cas.
VERSIONCAT:<num_ver>:<nom_catég>:MVSET:<nom_ensemble>:<n>:<nom_attr>:DISPLAY Affiche la valeur de l'attribut <nom_attr>, dans l'ensemble à valeurs multiples <nom_ensemble>, à l'emplacement d'index <n>, pour la catégorie <nom_catég>.
VERSIONCAT:<num_ver>:<nom_catég>:MVSET:<nom_ensemble>:<n>:<nom_attr>:ID Affiche l'ID de l'attribut <nom_attr>, dans l'ensemble à valeurs multiples <nom_ensemble>, à l'emplacement d'index <n>, pour la catégorie <nom_catég>.
VERSIONCAT:<num_ver>:<nom_catég>:MVSET:<nom_ensemble>:<n>:<nom_attr>:<val_attr> Renvoie Vrai si l'ensemble à valeurs multiples <nom_ensemble> comporte l'attribut <nom_attr>, à l'emplacement d'index <n>, pour la catégorie <nom_catég>, égal à la valeur <val_attr> ; renvoie Faux si tel n'est pas le cas.
VERSIONCAT:<num_ver>:<nom_catég>:MVSET:<nom_ensemble>:<n>:MV:<nom_attr>:COUNT Renvoie le nombre d'attributs à valeurs multiples utilisés dans l'ensemble à valeurs multiples.
VERSIONCAT:<num_ver>:<nom_catég>:MVSET:<nom_ensemble>:<n>:MV:<nom_attr>:<m> Renvoie Vrai si l'ensemble à valeurs multiples <nom_ensemble>, à l'emplacement d'index <n>, comporte un attribut à valeurs multiples <nom_attr> non vide, à l'emplacement d'index <m> ; renvoie Faux si tel n'est pas le cas.
VERSIONCAT:<num_ver>:<nom_catég>:MVSET:<nom_ensemble>:<n>:MV:<nom_attr>:<m>:DISPLAY Affiche la valeur de l'attribut à valeurs multiples <nom_attr>, à l'emplacement d'index <m>, dans l'ensemble à valeurs multiples <nom_ensemble>, à l'emplacement d'index <n>, pour la catégorie <nom_catég>.
VERSIONCAT:<num_ver>:<nom_catég>:MVSET:<nom_ensemble>:<n>:MV:<nom_attr>:<m>:<val_attr> Renvoie Vrai si l'ensemble à valeurs multiples <nom_ensemble>, à l'emplacement d'index <n>, comporte un attribut à valeurs multiples <nom_attr>, à l'emplacement d'index <m>, dont la valeur est égale à la valeur spécifiée dans <val_attr> ; renvoie Faux si tel n'est pas le cas.
VERSIONCAT:<num_ver>:<nom_catég>:MVSET:<nom_ensemble>:<n>:MV:<nom_attr>:INDEX:<val_attr> Renvoie le premier index dans l'attribut à valeurs multiples <nom_attr>, dans un ensemble à valeurs multiples et pour la catégorie <nom_catég> où la valeur <val_attr> est égale à la valeur contenue dans l'attribut à valeurs multiples.

Exemple

[LL_REPTAG=DATAID VERSIONCAT:<num_ver>:<nom_catég>:<nom_attr>:DISPLAY /] renvoie la valeur d'attribut stockée dans la catégorie <nom_catég> et l'attribut <nom_attr> pour tout type d'attribut pour la version <num_ver> du nœud. Si <nom_attr> est un attribut à valeurs multiples ou un ensemble d'attributs, une liste de valeurs séparées par des virgules est renvoyée.

Informations associées

Si vous souhaitez renvoyer des données de catégorie pour la dernière version d'un nœud, voir la sous-balise CAT.

La sous-balise VERSIONCAT est essentiellement identique à la sous-balise CAT, mais elle renvoie les données d'une version spécifique d'un nœud tandis que la sous-balise CAT renvoie les données de la dernière version d'un nœud. Exemple détaillé de la sous-balise CAT et plusieurs autres méthodes de génération de rapports sur les catégories et attributs.
