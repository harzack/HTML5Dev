Cette section apporte des informations sur les balises de données de colonne LL_REPTAG=nom colonne, LL_REPTAG_numéro colonne, LL_REPTAG_texte littéral, LL_REPTAG_&nom_param, LL_REPTAG_$nom_const, LL_REPTAG_%nom_variable et LL_REPTAG_@fonction.

LL_REPTAG=nom colonne

Syntaxe

[LL_REPTAG=<nom colonne> <sous-balise> <sous-balise> ... /]

Description

Cette balise indique à WebReports d'insérer les données source spécifiée par <nom colonne> dans le rapport de sortie. Elle doit être placée entre les balises STARTROW et ENDROW. La liste facultative des sous-balises est utilisée pour configurer les options de mise en forme. Ces sous-balises doivent être séparées par des espaces. Pour plus d'informations sur ces balises, voir le tableau de définition des sous-balises. L'utilisation de guillemets doubles ou simples est prise en charge pour délimiter les noms de colonne. L'utilisation de guillemets est obligatoire uniquement pour le référencement des noms de colonne de source de données contenant un caractère d'espacement. Cette règle s'applique également lorsque le rapport LiveReport renomme une colonne avec un nom contenant un caractère d'espacement. N'utilisez pas de guillemets doubles lorsque la balise LL_REPTAG est située à l'intérieur d'une balise conditionnelle.

Exemple

[LL_REPTAG=name /]
[LL_REPTAG="name" /]
[LL_REPTAG="first name" /]
[LL_WEBREPORT_INCLUDEIF "John" != "[LL_REPTAG='first name' /]" /]

LL_REPTAG_numéro colonne

Syntaxe

[LL_REPTAG_<numéro colonne> <sous-balise> <sous-balise> ... /]

Description

Cette balise référence les données de colonne à l'aide du numéro de colonne. Les colonnes Content Server sont numérotées de la gauche vers la droite, en commençant par le n°1.

 

Note :
Attention : veillez à utiliser un trait de soulignement (« _ ») au lieu du symbole égal (« = ») dans cette forme de LL_REPTAG. Cela correspond approximativement au modèle de balise WebForm Content Server. Cette forme de balise peut être utile si vous prévoyez d'utiliser un même affichage de rapport dans plusieurs WebReports, avec des titres de colonne parfois différents.

 

LL_REPTAG_« texte littéral »

Syntaxe

[LL_REPTAG_« <texte littéral> » /]

ou

[LL_REPTAG_'<texte littéral>' /]

Description

Cette balise permet de spécifier une chaîne de texte à insérer dans l'affichage de rapport. Toute sous-balise incluse dans la définition de balise agira sur le texte spécifié entre les deux guillemets. Vous pouvez utiliser des guillemets simples ou doubles, dans la mesure où les guillemets entrants et sortants correspondent.

Exemple

[LL_REPTAG_"this text will be displayed in capitals" UPPER /]

LL_REPTAG_&nom_param

Syntaxe

[LL_REPTAG_&<nom_param> /]

Description

Cette balise permet d'accéder aux paramètres de l'URL utilisée pour appeler le WebReport. Vous pouvez référencer de cette manière toutes les valeurs de l'URL comprenant une esperluette (« & ») en préfixe. Cela signifie que vous pouvez référencer des paramètres existants, tels que &ObjAction, ou tout paramètre personnalisé.

Exemple

Par exemple, si l'URL suivante a été exécutée :

...?func=ll&objId=47478&objAction=RunReport&myparm=testdata&nexturl=....

Vous pouvez accéder au paramètre myparm à l'aide de [LL_REPTAG_&myparm /]. Cette balise effectue la résolution de la valeur dans l'URL ; dans l'exemple ci-dessus, cette valeur est testdata et peut être utilisée en sortie ou dans une condition.

Par exemple, [LL_WEBREPORT_INCLUDEIF "[LL_REPTAG_&myparm /]" == "testdata" /]

Si un paramètre est référencé dans l'affichage de rapport mais ne figure pas dans l'URL, une chaîne vide est insérée dans la sortie, excepté si la valeur par défaut du paramètre est configurée dans l'onglet Paramètres de la page Propriétés d'un WebReports. Pour en savoir plus, voir Utilisation des paramètres.

LL_REPTAG_$nom_const

Syntaxe

[LL_REPTAG_$<nom_const> /]

Description

Cette balise permet d'accéder aux constantes définies par le biais de l'onglet Constantes de la page Propriétés d'un WebReports. Pour en savoir plus, voir Constantes WebReports. Le nom de constante figurant dans l'affichage de rapport doit correspondre exactement à celui spécifié sur la page Constantes. La balise est remplacée par la valeur définie pour la constante correspondante.

Exemple

Par exemple, si une constante appelée MyConst a été définie avec une valeur de « 12345 », la balise suivante renvoie « 12345 » : [LL_REPTAG_$MyConst /].

Les constantes sont particulièrement utiles pour référencer les sous-rapports WebReports et autres objets Content Server, car elles permettent d'éviter de coder en dur les objectID dans l'affichage de rapport. Pour en savoir plus, voir Appel de sous-rapports WebReport.

Informations associées

Lorsque vous utilisez XML pour exporter des applications WebReports, les constantes permettent de « rétablir le lien » des objets avec l'import XML, dans le nouveau système. Pour plus d'informations sur les constantes, voir Constantes WebReports.

LL_REPTAG_%nom_variable

Syntaxe

[LL_REPTAG_%<nom_variable> /]

Description

Cette balise permet de référencer les variables définies en utilisant SETVAR, ADDVAR ou CONCATVAR. Par exemple, si vous avez utilisé ADDVAR dans la section de ligne pour cumuler un total mathématique dans une variable appelée MyVar, le total final est accessible via : [LL_REPTAG_%MyVar /]

Informations associées

Pour plus d'informations sur l'utilisation des balises de variable et des sous-balises associées, voir L'ordre des variables définies par les balises WebReports. Voir également ADDVAR, CONCATVAR et CURRENTVAL.

LL_REPTAG_@fonction

Syntaxe

[LL_REPTAG_@<fonction><options référence > /]

Description

Ce type de balise offre des fonctions de données. Le résultat de chaque fonction est inséré dans la sortie ; toutefois, ces balises sont actuellement prises en charge uniquement dans l'en-tête et dans le pied de page d'un affichage de rapport. En d'autres termes, devant la balise STARTROW ou derrière la balise ENDROW. Il existe pour chaque fonction des caractères facultatifs, qui suivent le nom de la fonction et font référence à une ligne ou à une colonne de données spécifique, ou les deux. Ces caractères de référence sont :

  • [r] : ce paramètre facultatif, utilisé uniquement par @DATA, spécifie l'index de ligne.

     

    Note :
    Le mot réservé LAST peut être utilisé à la place de r.

     

    [r] peut être suivi des éléments suivants :

  • _c : spécifie un numéro de colonne ; ou

  • _<nom_col> : spécifie un nom de colonne.

Lorsque vous configurez l'une des fonctions ci-dessous, celle-ci est exécutée sur les données référencées, quelles qu'elles soient. Dans le cas des balises mathématiques, si la colonne contient des valeurs non numériques, une erreur est renvoyée. Des sous-balises permettent de mettre en forme les valeurs renvoyées par ces fonctions.

Paramètres

Paramètre
Description
@AVERAGE_<colonne>
Renvoie la valeur moyenne de tous les chiffres valides contenus dans la colonne spécifiée.
@DATA[r]_<colonne>
Renvoie une cellule de données unique, définie par sa ligne et sa colonne.
@MIN_<colonne>
Renvoie la valeur la plus basse, sur toutes les valeurs contenues dans la colonne spécifiée.
@MAX_<colonne>
Renvoie la valeur la plus haute, sur toutes les valeurs contenues dans la colonne spécifiée.
@RESPONSEDATA.<nom champ données>
Indique les champs renvoyés par Content Server en réponse à une requête. Ces champs sont normalement utilisés dans les pages Content Server pour afficher des données. Par exemple, certaines pages de navigation Content Server utilisent, dans la réponse à une requête de navigation, un champ appelé : hasStagedContent. Cette valeur peut être référencée dans un WebReport via : [LL_REPTAG_@RESPONSEDATA.hasStagedContent /]
@SUM_<colonne>
Renvoie la somme de toutes les valeurs valides contenues dans la colonne spécifiée.

Exemple

Pour demander la moyenne de la colonne 2, utilisez la syntaxe suivante : [LL_REPTAG_@AVERAGE_2 /]

Pour demander la somme des valeurs d'une colonne appelée « heures », utilisez la syntaxe suivante : [LL_REPTAG_@SUM_hours /]

Pour demander le contenu d'une colonne appelée dataid, à partir de la dernière ligne, utilisez la syntaxe suivante : [LL_REPTAG_@DATA[last]_dataid /]

Pour demander le contenu de la colonne 2, à partir de la ligne 1, utilisez la syntaxe suivante : [LL_REPTAG_@DATA[1]_2 /]

Pour demander le contenu d'un champ de la réponse à une requête Content Server, utilisez la syntaxe suivante : [LL_REPTAG_@RESPONSEDATA.HASSTAGEDCONTENT /]
