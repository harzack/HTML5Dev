JSON (JavaScript Object Notation) représente un moyen efficace pour spécifier les données à envoyer des serveurs aux applications client basées sur un navigateur. La syntaxe réellement utilisée pour créer un ensemble de données JSON n'a pas d'importance, tant que le logiciel du serveur le construit correctement. Dans la plupart des cas, si ce n'est dans tous, la structure JSON est évaluée immédiatement dans le client de navigateur, en général au moyen de la fonction JavaScript eval(), qui la convertit en une structure de type objet/tableau JavaScript normale. Cette fonctionnalité a pour avantage principal de réduire la bande passante et les traitements supplémentaires entre le serveur et le client. JSON permet d'encapsuler un objet afin qu'il soit rapidement analysé par des applications logicielles tout en restant compréhensible pour les humains. Cette notation est particulièrement bien adaptée à l'échange de données entre clients et serveurs utilisant une technologie de type AJAX, avec des demandes de messages asynchrones. A partir de Livelink 9.7.1, JSON est utilisé pour faciliter l'envoi de données de recueil associées au navigateur au moyen d'appels AJAX.

Pour prendre en charge cette technologie, WebReports comporte un mécanisme permettant à la plupart des données normalement récupérées au moyen de balises WebReports distinctes d'être fournies sous forme d'une structure JSON.
