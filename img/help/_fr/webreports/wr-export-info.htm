Cette section fournit des informations complémentaires sur l'utilisation des destinations d'exportation. Elle fournit notamment des détails sur la manière de contrôler différents types d'exportation à l'aide de paramètres d'URL. Cela est particulièrement utile pour les développements où des exports WebReport peuvent être appelés à partir de pages Web personnalisées plutôt que d'utiliser une interface d'exportation standard décrite dans les procédures d'exportation de base. Pour chaque type d'exportation, tous les paramètres d'URL disponibles sont décrits et accompagnés d'un exemple d'utilisation.

Chaque destination d'exportation possède un tableau explicatif avec les colonnes suivantes :

  • Nom du paramètre : le nom du paramètre tel qu'il apparaît dans l'URL.

  • Nom dans l'interface : il s'agit du nom qui apparaîtra dans la case grise à gauche de l'écran d'exportation.

  • Obligatoire : indique si le paramètre est obligatoire ou non. L'exportation échouera si les paramètres obligatoires sont manquants.

  • Valeurs valables : il s'agit des valeurs prises en charge pour le paramètre spécifié.

     

    Note :
    Lorsqu'une valeur contenant une espace ou un autre caractère spécial est transmise, elle doit être insérée de manière appropriée afin d'être utilisée dans une URL.

     

  • Balises de remplacement : certains paramètres prennent en charge la transmission des balises de remplacement WebReports dans l'URL. Ce champ indique si la balise de remplacement est prise en charge pour le paramètre.

     

    Note :
    Lorsqu'une balise de remplacement contenant une espace ou un autre caractère spécial est transmise, elle doit être insérée de manière appropriée afin d'être utilisée dans une URL.
