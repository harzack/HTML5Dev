Les constantes globales constituent une extension des constantes standard et permettent de gérer les constantes de manière centralisée pour un ensemble particulier de WebReports qui repose sur les mêmes données. Ceci est particulièrement utile lorsque l'utilisateur dispose d'un groupe de WebReports, de formulaires et d'autres objetsContent Server qui fonctionnent ensemble pour créer une application. Chaque WebReport peut faire référence à un seul rapport Constante globale défini en haut de sa page Constantes.

En effet, tous les WebReports d'une application particulière pointent vers un seul WebReport utilisé pour gérer les constantes de toute l'application. L'avantage est que lorsqu'une constante est définie dans votre rapport global, elle est disponible pour tous les WebReports qui pointent vers ce WebReport. Ceci représente un gain de temps considérable lorsque l'on considère qu'une application WebReports peut être conçue à partir de 50 WebReports et autres objets, ou plus, qui doivent tous faire référence les uns aux autres. Les constantes globales peuvent être « chaînées » pour qu'un WebReport pointe vers un autre, etc.

Même dans ce cas, les constantes globales respectent les autorisations Content Server ; par conséquent, si un utilisateur n'est pas autorisé à voir un WebReport donné, il ne pourra pas utiliser les constantes définies dans celui-ci. Lors de l'utilisation de l'éditeur en ligne, un astérisque, *, apparaît à côté des constantes globales dans la liste.
