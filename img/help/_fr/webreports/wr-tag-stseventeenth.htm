Cette section apporte des informations sur les sous-balises suivantes : TABS, TIMESINCE, TODATE, TOLIST, TOJSON, TRIM, TRUNC et TZOFFSET.

TABS

Syntaxe

TABS

Description

Renvoie une représentation Oscript des onglets Content Server associés à un objet. Ce sont les mêmes onglets que vous voyez lorsque vous allez sur tout menu Fonctions de l'objet, puis que vous sélectionnez Propriétés. Cette sous-balise attend un dataid valide dans la balise principale. Les onglets sont renvoyés dans l'ordre approprié, comme illustré dans l'interface Content Server.

Exemple

L'exemple suivant affiche la représentation Oscript des onglets Général et Spécifique d'un document. Exemple de sortie de la sous-balise TABS :

{A<1,?,'Name'=X,'URL'='/Livelink971/livelink.exe?func=ll&objId=1389412&objAction=properties&nexturl=%2FLivelink971%2Flivelink%2Eexe%3Ffunc%3Dll%26objid%3D1388531%26objAction%3Dbrowse%26sort%3Dname'>, A<1,?,'Name'=X,'URL'='/Livelink971/livelink.exe?func=ll&objId=1389412&objAction=info&nexturl=%2FLivelink971%2Flivelink%2Eexe%3Ffunc%3Dll%26objid%3D1388531%26objAction%3Dbrowse%26sort%3Dname'>...

Informations associées

Voir l'aide avancée Requêtes en tant que sources de données pour savoir comment utiliser cette sortie et les balises associées en tant que sources de données pour un autre WebReport. Voir la sous-balise TOJSON pour savoir comment convertir cette fonction en une structure de données pouvant être interprétée par JavaScript au niveau du client. La structure JSON peut ensuite être analysée et le format HTML approprié peut être appliqué pour afficher les onglets.

TIMESINCE

Syntaxe

TIMESINCE:<date de réf>

Description

Cette sous-balise attend une date dans la balise principale et renvoie l'écart de temps, <date de réf> moins <date de la balise principale>, dans un format facile à lire. Si aucune <date de réf> n'est spécifiée, la date actuelle est alors utilisée. Les valeurs négatives sont renvoyées si la <date de réf> se trouve devant la <date de la balise principale>. Les dates doivent être spécifiées dans l'un des formats suivants :

  • 'MMM DD YYYY'. Par exemple, 'Jan 12 2010'.

  • 'D/YYYY/MM/DD'. Par exemple, 'J/2009/12/16'.

  • 'DayOfWeek MMM DD HH:MM:SS YYYY'. Par exemple, 'Ven Jan 01 01:04:24 2010''.

  • 'YYYY-MM-DD HH:MM:SS'. Par exemple, '2038-01-19 03:14:07'.

La balise principale peut aussi contenir un entier représentant un intervalle de temps en secondes. Dans ce cas, le paramètre <date de réf> est ignoré et la valeur est renvoyée dans un format facile à lire

Un second paramètre, facultatif, indique le nombre d'unités de précision à renvoyer. Une valeur par défaut de 2 est utilisée lorsque ce paramètre facultatif est omis.

Exemple

Balise
Sortie
[LL_REPTAG_'May 1 2010' TIMESINCE /]
11 mois, 1 semaine
[LL_REPTAG_'May 1 2010' TIMESINCE:"May 2 2010" /]
1 jour
[LL_REPTAG_'2011-05-01 00:00:00' TIMESINCE:'2011-05-01 01:01:23':3 /]
1 heure, 1 minute, 23 secondes
[LL_REPTAG_'3661' TIMESINCE /]
1 heure, 1 minute
[LL_REPTAG_'3661' TIMESINCE:3 /]
1 heure, 1 minute, 1 seconde

TODATE

Syntaxe

TODATE:« <liste de formats> »

Description

Cette sous-balise convertit les informations formatées de date en chaîne au format de date Content Server. Cela signifie que la sortie peut être utilisée dans des expressions logiques comme pour comparer une date personnalisée avec un paramètre de date ou éventuellement avec la date actuelle. La sortie de TODATE peut aussi être contenue dans la sous-balise DATE pour plus de manipulation avant l'affichage.

Paramètres

Opérateur
Description
%%
Correspond à un signe de pourcentage
%b
Nom du mois abrégé avec trois caractères. Par exemple : « Jan » et « Fév ».
%d
Les deux chiffres du jour du mois, de 01 à 31. Par exemple, « 05 » représente le cinquième jour du mois.
%m
Les deux chiffres du mois, de 01 à 12. Par exemple, « 05 » représente le mois de mai.
%p
Matin ou soir (AM ou PM)
%y
Année à deux chiffres. Par exemple, « 93 » représente 1993 ou « 05 » représente 2005.
%B
Le nom complet du mois, par exemple « Janvier ».
%H
Les deux chiffres de l'heure sur 24 heures, de 00 à 23.
%I
Les deux chiffres de l'heure, de 01 à 12.
%M
Les minutes après les heures, de 00 à 59.
%S
Les secondes après les minutes, de 00 à 59.
%Y
L'année, incluant le siècle, par exemple 1993.

Exemple

Voici un exemple d'utilisation de TODATE avec DATE pour passer du format de date personnalisé au format de date court Content Server, comme défini dans les pages Administration Content Server :

Imaginez que vous commencez avec une date personnalisée stockée au format : « 2006-02-15T00:00:00 ». Utilisez [LL_REPTAG=MY_DATE_COLUMN TODATE:"%Y-%m-%dT%H:%M:%S" /] pour mettre la date dans une représentation de type chaîne d'une date Content Server. A ce stade, la date est « Mer Fév 15 00:00:00 2006 », ce qui peut parfois être utile.

Utilisez maintenant la sous-balise DATE [LL_REPTAG=MY_DATE_COLUMN TODATE:"%Y-%m-%dT%H:%M:%S" DATE:SHORT /] pour changer la sortie afin que la date soit au même format que, par exemple, la date modifiée dans un navigateur de dossier Content Server. A ce stade, la date peut être « Février 15, 2006 », même si cela dépend de votre paramétrage système dans les pages Administration Content Server.

Informations associées

Pour le formatage de sortie, voir la sous-balise DATE. Pour la date actuelle, voir les balises DATE et DATETIME.

TOLIST

Syntaxe

TOLIST:<délimiteur> TOLIST:<délimiteur>:<index>

Description

Pour une chaîne ayant un délimiteur connu, comme une virgule, la chaîne peut être convertie en liste. Cette liste est semblable à un tableau de programmation et se trouve dans le formulaire : {1,2,3} ou {'one','two','three'}. Une liste dans ce format peut être utilisée par les sous-balises LIST et RANGE, toutefois, la sous-balise TOLIST prend elle aussi en charge une valeur d'index facultative spécifiant l'élément à renvoyer dans la liste créée récemment.

Exemple

Si la première colonne d'une table de données renvoie la chaîne suivante :

'apple','orange','pear';

il faut utiliser la balise : [LL_REPTAG_1 TOLIST:, /] pour renvoyer la liste :

{'apple','orange','pear'}

Si le paramètre d'index facultatif est lui aussi utilisé, la syntaxe :

[LL_REPTAG_1 TOLIST:,:2 /] renvoie seulement : orange

Informations associées

Voir également LIST et RANGE.

TOJSON

Syntaxe

TOJSON

Description

Pour une balise qui renvoie une structure de données de type OSCRIPT, cette balise convertit la structure en JSON, JavaScript Object Notation, une structure qui peut être traitée par un interpréteur JavaScript. Cette sous-balise convertit les structures OSCRIPT comme suit :

Structure OSCRIPT Exemple OSCRIPT Structure JSON Exemple JSON
LIST {'one','two'} Tableau [« un »,« deux »]
ASSOC A<1,?,'un'=1,'deux'=2> Object {« un »:1, « deux »:2}
RECORD R<'un'=1,'deux'=2> Object {« un »:1, « deux »:2}
RECARRAY V{<'fruit','couleur'><'pomme','vert'><'banane','jaune'>} Tableau des objets [{« fruit »:« pomme », « couleur »:« vert »}, {« fruit »:« banane », « couleur »:« jaune »}]

TRIM

Syntaxe

TRIM

Description

Supprime l'ensemble des espaces de début et de fin de la chaîne spécifiée. L'espace est définie de la même manière que les combinaisons d'espaces, les sauts de page, les nouvelles lignes, les retours chariot, et les onglets horizontaux ou verticaux.

TRUNC

Syntaxe

TRUNC:<largeur>

Description

Tronque le champ au nombre de caractères spécifié dans la largeur (en commençant par le premier caractère).

TZOFFSET

Syntaxe

TZOFFSET

Description

Applique un décalage horaire aux dates. Les balises de date comme [LL_REPTAG_DATETIME /] et les colonnes de date [LL_REPTAG=CREATEDATE /] renvoient le temps au serveur. La sous-balise TZOFFSET peut être utilisée pour appliquer le décalage horaire approprié afin que l'utilisateur spécifique puisse afficher la sortie.
