WebReports utilise un schéma de restrictions pour sécuriser davantage Oscript dans un affichage de rapport. Les voici :

  • Aucun accès aux variables globales au moyen du symbole dollar ($).

  • Aucune déclaration de variables Object, ObjRef ou Frame.

  • Aucune utilisation de parenthèses comme « .() » pour évaluer le contenu d'une variable. Par exemple, le code suivant n'est pas autorisé car « .(s) » n'est pas autorisé :

    Assoc       myAssoc
String      s = 'item1'
myAssoc.item1 = 'first item'
echo ( myAssoc.(s) )

  • Tous les packages du générateur sont restreints, sauf ceux répertoriés ci-dessous.

  • Certaines fonctions de certains packages restreints du générateur sont autorisées, voir la liste ci-dessous.

Les packages suivants du générateur sont autorisés par défaut. Cependant, l'administrateur Content Server peut ajouter ou retirer tout élément de son choix dans la liste des packages ou des fonctions de packages autorisés.
