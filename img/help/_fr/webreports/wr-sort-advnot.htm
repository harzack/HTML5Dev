Cette balise est toujours placée dans une section de ligne, par exemple après [LL_WEBREPORT_STARTROW /] et avant [LL_WEBREPORT_ENDROW /]. Vous pouvez trouver plusieurs balises SORT dans une section de ligne, ce qui n'est généralement pas nécessaire puisque la balise SORT prend en charge plusieurs clés.

Lorsque vous utilisez une balise ou des sous-balises WebReports en tant que paramètre de tri, tous les résultats numériques ou sous forme de date valide comme définis dans le système Content Server seront respectivement triés en tant que nombres entiers ou dates.

Lorsque vous utilisez la balise SORT, vous pouvez utiliser les balises de paramètre ou de constante dans la syntaxe de la balise. Cela permet de trier le contenu en fonction d'une valeur qui a été transmise en tant que paramètre ou constante. Par exemple :

[LL_WEBREPORT_SORT"[LL_REPTAG=DATAID NODEINFO:[LL_REPTAG_&nodefield/]/]":[LL_REPTAG_&direction/]/]

Dans l'exemple ci-dessus, un paramètre URL pourrait être transmis pour spécifier quel NODEINFO: doit être utilisé. Un ordre de tri pourrait également être transmis dans cet exemple.

 

Note :
Pour le paramètre &nodefield, la balise ne fonctionnera pas correctement si aucune valeur n'a été transmise en tant que champ de nœud. La fonction Paramètre par défaut doit être utilisée pour fournir un paramètre par défaut dans ce type de situation.
