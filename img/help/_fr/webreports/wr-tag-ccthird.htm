Cette section apporte des informations sur les balises de contrôle de contenu : ELSE, ENDIF, ENDROW, ENDSCRIPT, EXCLUDEADDITEM, EXCLUDECHANNEL, EXCLUDEFOOTER, EXCLUDEHEADER, EXCLUDEHTML, EXCLUDELOCATIONDISPLAY, EXCLUDEMENU, EXCLUDESEARCH, EXCLUDETITLE et EXITIF.

ELSE

Syntaxe

[LL_WEBREPORT_ELSE /]

Description

Cette balise facultative doit être utilisée entre les balises IF et ENDIF. Si elle est incluse, tous les éléments situés entre la balise ELSE et la balise ENDIF sont inclus si la condition spécifiée est considérée comme fausse.

Exemple

[LL_WEBREPORT_IF (expression) /]
[LL_WEBREPORT_ELSE /] (optional)
[LL_WEBREPORT_ENDIF /]

Informations associées

Voir également IF et ENDIF pour plus de détails.

ENDIF

Syntaxe

ENDROW

Syntaxe

[LL_WEBREPORT_ENDROW /]

Description

Balise utilisée afin de marquer la fin du modèle de ligne de rapport.

Lorsque la modification en ligne est utilisée, cette balise est automatiquement insérée et n'est pas visible dans la source de l'affichage de rapport.

ENDSCRIPT

Syntaxe

[LL_WEBREPORT_ENDSCRIPT /]

Description

Cette balise se trouve à la fin d'un bloc défini Oscript côté serveur qui peut être appelé par la balise CALL.

Exemple

[LL_WEBREPORT_STARTSCRIPT NAME:<script name> /]
function String anyName(Dynamic c, List args)
String result
... 
return result
end
[LL_WEBREPORT_ENDSCRIPT /]

Informations associées

Voir aussi STARTSCRIPT.

EXCLUDEHTML

Syntaxe

[LL_WEBREPORT_EXCLUDEHTML /]

Description

Exclut toutes les définitions HTML, JavaScript et Style que Content Server utilise pour intégrer une page donnée. Cette méthode peut être utilisée à bon escient lors de l'exportation, par exemple XML ou CSV, car inclure la page Content Server n'est plus pertinent. Lorsque cette option est utilisée, toutes les autres balises EXCLUDE sont ignorées car elles ne sont pas pertinentes, sauf dans le contexte d'une page Content Server.

 

Note :
Lors de l'utilisation de [LL_WEBREPORT_EXCLUDEHTML /] dans un WebReport qui utilise également les Utilisation des fonctions prédéfinies, la variable suivante ne doit pas être définie dans un bloc de script, car la variable baseURL est définie dans le code de page Content Server qui est inséré dans la sortie WebReport, mais elle est exclue à l'aide de la balise. Le code suivant garantit la définition de la variable et empêche la rupture de la bibliothèque AJAX.

 

Exemple

<script>
   baseURL = "[LL_REPTAG_URLPREFIX /]";
</script>

EXITIF

Syntaxe

[LL_WEBREPORT_RUNIF (expression) /]

Description

Entraîne l'arrêt du traitement des lignes lorsqu'une condition est remplie. Si l'expression spécifiée est considérée comme Vrai, plus aucune ligne ne sera incluse dans la sortie. La ligne actuelle traitée n'est pas incluse si l'expression est considérée comme Vrai.

 

  1. Il s'agit de la seule balise conditionnelle pour laquelle il est juste d'utiliser la balise ROWNUM dans une expression. Cette balise fonctionne uniquement dans la section de ligne, ou en d'autres termes, entre les balises STARTROW et ENDROW.

  2. Cette balise ne modifie pas la balise TOTALROWS, qui reflétera toujours le nombre de lignes réellement disponibles dans la table de base données. Pour obtenir le nombre de lignes réellement renvoyées, c'est-à-dire celles qui répondent aux critères du test conditionnel, utilisez la balise ACTUALROWS. Voir les informations spécifiques relatives à la syntaxe de ces balises.

  3. parmA et parmB peuvent être soit du texte, soit une balise de rapport, toutes les autres balises sont résolues en premier.

  4. Les paramètres doivent être mis entre des guillemets doubles.

  5. Des clauses facultatives complémentaires sont ajoutées à l'aide de l'un des opérateurs suivants :

    • AND (AND logique)

    • && (AND alternatif)

    • OR (OR logique)

    • || (OR alternatif)

 

Paramètres

Format pour (expression) : "parmA" <opérateur> "parmB" <clauses facultatives>

Exemples

Exemple

Par exemple : [LL_WEBREPORT_EXITIF "[LL_REPTAG_ROWNUM /]" == "10" /]

Dans cet exemple, les lignes numérotées de 1 à 9 seront générées, mais pas la ligne 10.

Exemple

[LL_WEBREPORT_EXITIF "[LL_REPTAG_&var1 /]" != "[LL_REPTAG=DATAID /]" /]
[LL_WEBREPORT_EXITIF "[LL_REPTAG_ROWNUM /]" == "101" /]
[LL_WEBREPORT_EXITIF "[LL_REPTAG=dataid /]" NOTCHILDOF "[LL_REPTAG_&rootid /]" OR "[LL_REPTAG=parentid /]" NOTCHILDOF "[LL_REPTAG_&rootid /]" /]

Informations associées

Voir la section Notes sous INCLUDEIF ou Expressions logiques pour plus d'informations. Voir la section relative à Expressions logiques pour obtenir une liste des Opérateurs.
