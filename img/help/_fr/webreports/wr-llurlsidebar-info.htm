LLURL:SIDEBAR insère la sortie générée à partir des panneaux de facette vers un recueil de barre latérale dans un affichage de rapport ou un modèle. En raison de la nature de cette fonction, elle repose en grande partie sur une structure HTML prédéterminée qui implique une série de formulaires, de tables imbriquées et de DIV. L'affichage de rapport par défaut 13_navigation_flexible_cs10_avec_barre_latérale.txt comprend les éléments nécessaires pour que la fonctionnalité Barre latérale fonctionne. Il est donc conseillé de s'en servir comme exemple ou point de départ afin de créer des affichages de rapport personnalisés utilisant la fonction Barre latérale. Vous trouverez ci-dessous un schéma présentant les éléments HTML nécessaires et la mise en page suggérée pour un modèle/affichage de rapport.

Ce qui est inséré

La sous-balise LLURL:SIDEBAR attend le DataID d'un objet de recueil et renvoie le contenu de la barre latérale (y compris les panneaux de facette et le contenu de facette) pour ce recueil. Un certain nombre d'éléments de formulaires supplémentaires/d'appels de fonction Javascript dans CS10 doivent être définis sur le même ID afin de gérer le comportement cohérent de la barre latérale. Ils sont tous décrits et mis à jour avec les valeurs appropriées dans le modèle/l'affichage de rapport par défaut incluant la barre latérale. Le code LLURL:SIDEBAR renvoie les données facette/barre latérale renvoyées dans une cellule de tableau « sidebar-wrapper ». Par exemple <td valign="top" class="sidebar-wrapper">

Dans WebReports, la variable SBCONTAINER est définie en haut de l'affichage de rapport et est utilisée aux emplacements appropriés dans l'affichage de rapport. Pour utiliser un ID différent du dossier parent du WebReport, il est conseillé de mettre à jour cette variable avec le nouvel ID, comme cette dernière se trouve déjà à la position souhaitée. Par exemple, pour modifier la barre latérale et charger les données de facette pour un recueil avec un ID de 12345, définissez la variable SBCONTAINER comme suit : [LL_REPTAG_'12345' SETVAR:SBCONTAINER /].

Exemple d'utilisation dans WebReports : [LL_REPTAG_%SBCONTAINER LLURL:SIDEBAR /]

Exemple d'utilisation dans ActiveView : [LL_REPTAG_%SBCONTAINER LLURL:SIDEBAR /]

Améliorations USERPREF

Deux nouvelles options sont disponibles dans la sous-balise USERPREF et permettent à la barre latérale de se comporter conformément aux paramètres de chaque utilisateur. Ces nouveaux paramètres dans Content Server 10 se trouvent dans le menu Outils -> Paramètres, dans l'onglet Général et peuvent être affichés à l'aide des sous-balises suivantes :

  • USERPREF:GENERAL:SBLOCATION : renvoie droite ou gauche en fonction du paramètre Emplacement de la barre latérale

  • USERPREF:GENERAL:SBLOCATION : renvoie vrai ou faux en fonction du paramètre Afficher la barre latérale dans la navigation

Exemple

Utilisez [LL_REPTAG_USERID USERPREF:GENERAL:SIDEBAR /] afin de déterminer si l'utilisateur exécutant le WebReport souhaite que la barre latérale s'affiche lors de la navigation.
