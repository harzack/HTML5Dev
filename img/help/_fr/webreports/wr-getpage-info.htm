Le fichier page.js fourni avec WebReports inclut une fonction utile getPage pour vous aider par rapport aux mécanismes de pagination lorsque vous souhaitez afficher un important ensemble de résultats. Cette fonction est appelée lorsqu'un utilisateur clique sur les boutons Premier, Précédent, Suivant ou Dernier (voir illustration ci-dessous). Cette fonction est également disponible afin d'être utilisée dans toute application de génération de rapports WebReport. Le fichier page.js est accédé par l'inclusion d'un SCRIPT comme suit : <SCRIPT SRC="/<support dir>/webreports/library/browse.js"></SCRIPT>

Si ce fichier est inclus dans un WebReport, la balise LIBPATH peut être utilisée comme suit : <SCRIPT SRC="[LL_REPTAG_LIBPATH /]page.js"></SCRIPT>

getPage( baseURL, page, dataCount, pageSize, ip1, ip2, startParm, endParm )

  • Cette fonction est appelée lorsqu'un utilisateur clique sur les boutons Premier, Précédent, Suivant ou Dernier. Elle crée et exécute l'URL pour la prochaine vue de données à afficher pour l'utilisateur en fonction du bouton sur lequel il a cliqué.

  • Traite les arguments suivants :

    • baseURL : paramètre obligatoire. L'URL de base du rapport que nous voulons exécuter. Pour cela, utilisez la balise WebReport [LL_REPTAG_MYID LLURL:REPORT /] qui devient : ?func=ll&objId=xyz&objAction=RunReport

    • page : paramètre obligatoire. Une chaîne indiquant sur quel bouton l'utilisateur a cliqué. Il doit correspondre à l'une des valeurs suivantes : premier, préc., suivant ou dernier.

    • dataCount : paramètre obligatoire. Un nombre représentant le nombre total de lignes à utiliser pour la pagination.

    • pageSize : paramètre obligatoire.

    • ip1 : paramètre obligatoire. Le numéro actuel de début de ligne. Exemple : Si 'ip1=1' et 'ip2=20' sont spécifiés, les 20 premières lignes sont affichées lorsque l'utilisateur utilise le WebReport.

    • ip2 : paramètre obligatoire. Le numéro actuel de fin de ligne. Exemple : Si 'ip1=1' et 'ip2=20' sont spécifiés, les 20 premières lignes sont affichées lorsque l'utilisateur utilise le WebReport.

    • startParm : paramètre facultatif. Correspond au nom du paramètre de début. S'il n'est pas spécifié, il est défini par défaut sur 'inputLabel1'. Il s'agit généralement du nom du paramètre 'ip1'. Ce paramètre est utilisé dans l'URL de la page suivante à afficher dont la valeur est définie sur le numéro du début de ligne.

    • endParm : paramètre facultatif. Correspond au nom du paramètre de fin. S'il n'est pas spécifié, il est défini par défaut sur 'inputLabel2'. Il s'agit généralement du nom du paramètre 'ip2'. Ce paramètre est utilisé dans l'URL de la page suivante à afficher dont la valeur est définie sur le numéro de fin de ligne.
