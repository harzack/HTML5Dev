Cette section apporte des informations sur les balises de contrôle de contenu : FORCEUTF8 et FORMPARSEOFF.

FORCEUTF8

Syntaxe

[LL_WEBREPORT_FORCEUTF8 /]

Description

Cette balise oblige un indicateur d'ordre des octets UTF-8 (BOM, Byte Order Marker) à être ajouté au début de la sortie du WebReport. Le BOM est composé de caractères hexadécimaux : #FE #BB #BF. Certaines applications client nécessitent parfois que cet indicateur traite le contenu UTF-8.

FORMPARSEOFF

Syntaxe

[LL_WEBREPORT_FORMPARSEOFF /]

Description

Cette balise est uniquement utilisée lors de l'utilisation de WebReports dans des formulaires, grâce à l'option WR Power View dans des modèles de formulaire. Lorsque cette balise est incluse dans une option WR Power View, le code d'analyse de balises de WebForms normal n'est pas appelé. Un analyseur simplifié intégré au moteur WebReports est alors utilisé. Cet analyseur simple traite la substitution de la syntaxe de la balise WebForms, telle que les balises de type [LL_FormTag_ ... /], mais ne traite pas les balises LOOP, ainsi que toute balise de formulaire associée à des boucles WebForms. Cette balise désactive également la fonction WebForms qui définit automatiquement les champs de formulaire HTML SELECT et CHECKBOX avec des valeurs par défaut. Dans certains cas, cette fonction de WebForms peut être problématique et échouer lorsqu'un code HTML non standard est utilisé pour les champs tels que les instructions SELECT. Dans ces situations, la balise FORMPARSEOFF doit être utilisée afin d'empêcher tout conflit avec le code WebForms.
