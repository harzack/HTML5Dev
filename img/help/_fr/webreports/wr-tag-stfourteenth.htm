Cette section apporte des informations sur les sous-balises suivantes : PATCHANGE, PATFIND, PERMACTION, PERMCHECK, PERMINFO, POCOPY, POINFO et PREV.

PATCHANGE

Syntaxe

PATCHANGE:<modèle de recherche>:<nouveau modèle>

ou

PATCHANGE:<modèle de recherche>:<nouveau modèle>:<ignorer la casse ?>

Description

Renvoie une version modifiée de la balise principale en fonction des modèles de recherche et remplacement spécifiés.

La variable <modèle de recherche> est indiquée à l'aide d'une série d'opérateurs spéciaux conformément au tableau ci-dessous. Il s'agit d'un sous-ensemble simplifié et légèrement modifié de correspondance de modèles d'expressions régulières UNIX. Pour les développeurs de Content Server, cette syntaxe s'appuie sur les paramètres intégrés PATFIND et PATCHANGE d'Oscript. Elle est presque identique à la syntaxe PATFIND, mais permet de spécifier les parties du modèle de recherche qui peuvent faire l'objet d'une substitution dans le modèle de remplacement. Si aucune correspondance n'est trouvée, le contenu des balises d'origine reste inchangé.

La variable <modèle de remplacement> peut également inclure les caractères spéciaux présentés ci-dessous. Le modèle de remplacement est normalement un modèle de caractères fixe, mais les caractères issus du modèle de recherche sont compris dans les caractères <...> puis insérés dans le modèle de remplacement à l'aide du symbole dièse, #,. Par défaut, cette sous-balise ignore la casse des caractères en cas de correspondance. Si vous définissez le troisième paramètre sur FALSE, l'opération de remplacement devient sensible à la casse.

Paramètres

Tableau 1-30 : Caractères spéciaux du modèle de recherche

Caractère
Description
c
Correspond à un caractère exact. c représente ici n'importe quel caractère.
@c
Correspond à un caractère d'échappement, tel que @%, @[, @*.
?
Correspond à n'importe quel caractère à l'exception d'une fin de ligne.
%
Correspond à un début de ligne.
$
Correspond à une fin de ligne. Chaîne nulle avant la fin de ligne.
[...]
Correspond à une classe de caractères. L'un de « ces » caractères, tels que [abc] ou [a-z].
[!...]
Correspond à une classe de caractères réfutée. Tout sauf ces caractères, tels que [! ] pour indiquer aucune espace.
*
Correspond à zéro ou plusieurs occurrences de l'élément précédent, tel que [0-9]* pour zéro ou plusieurs chiffres.
+
Correspond à zéro ou plusieurs occurrences de l'élément précédent, tel que [A-Za-z]* pour une ou plusieurs lettres.
<...>
Les crochets indiquent un élément spécialement extrait à des fins de référence ou d'utilisation future.

Tableau 1-31 : Caractères spéciaux du modèle de remplacement

Caractère
Description
&
Idem (chaîne entière).
@
Caractère d'échappement, tel que @&.
#n
Insertion d'une sous-chaîne balisée.

Exemple

Cet exemple de sous-balise PATCHANGE suppose que la balise associée contient la chaîne : « This is a test sentence 123, end »

PATCHANGE:"?* <t?+t> *sen[!0-9]+<[0-9]+>?*":"Word:#1; Number: #2"

renvoie : Word: test; Number: 123

Dans l'exemple ci-dessus :

  1. Les caractères correspondants, mais non compris entre des chevrons, <>, ne seront pas inclus dans le remplacement.

  2. <t?+t> : trouve une correspondance avec le mot « test ». Comme il s'agit de la première « sous-chaîne balisée », ce mot remplace #1 dans la chaîne de remplacement.

  3. <[0-9]+> : trouve une correspondance avec les numéros « 123 ». Comme il s'agit de la seconde « sous-chaîne balisée », ces caractères remplacent #2 dans la chaîne de remplacement.

  4. La partie du modèle de recherche : sen[!0-9]+ trouve les caractères « sen » suivis de 1 ou plusieurs caractères NON numériques.

PATFIND:<modèle de recherche>:<ignorer la casse ?>

La variable <modèle de recherche> est indiquée à l'aide d'une série d'opérateurs spéciaux conformément au tableau ci-dessous. Il s'agit d'un sous-ensemble simplifié et légèrement modifié de correspondance de modèles d'expressions régulières UNIX. Pour les développeurs de Content Server, cette syntaxe s'appuie sur les paramètres intégrés PATFIND et PATCHANGE d'Oscript. Par défaut, cette sous-balise ignore la casse des caractères en cas de correspondance. Si vous définissez le second paramètre sur FALSE, l'opération de recherche devient sensible à la casse.

PERMACTION

Syntaxe

PERMACTION:<opération>

Description

Cette sous-balise fournit la flexibilité requise pour ajouter, supprimer et modifier des autorisations de contenu dans Content Server. En raison de la nature puissante de cette sous-balise, vous devez bien comprendre ses effets avant de l'utiliser. La sous-balise se comporte comme les autres sous-balises WebReport puisque vous ne pouvez pas modifier les autorisations des nœuds si vous n'êtes pas autorisé à le faire via l'interface standard. La différence principale est que l'utilisation de WebReports permet d'automatiser ces modifications de façon à les effectuer plus rapidement que via l'interface. Content Server prend en charge 3 modèles d'autorisation :

  • le modèle d'autorisation standard (SPM), par exemple les objets Content Server tels que les documents et dossiers ;

  • le modèle d'autorisation des projets (PPM), par exemple, les projets ;

  • et le modèle d'autorisation des éléments de travail (WPM), par exemple, les discussions, chaînes ou listes de tâches.

La sous-balise PERMACTION permet de mettre à jour les 3 modèles d'autorisation. Les tableaux ci-dessous décrivent l'utilisation de la sous-balise PERMACTION et ses répercussions sur les objets. Nous ferons référence au modèle d'autorisation et aux objets concernés.

Paramètres

PERMACTION:ACL:<ID_utilisateur>:UPDATE:<autors>
Met à jour les autorisations spécifiées par <autors> pour l'utilisateur ou le groupe spécifié par <ID_utilisateur> dans une liste de contrôle d'accès des autorisations d'un objet Content Server. Si l'utilisateur ou le groupe spécifié ne se trouve pas dans la liste de contrôle d'accès, il y est ajouté. Cette opération est prise en charge par les objets SPM et WPM.

Où DATAID renvoie le dataID du projet et MEMBER est un mot-clé pour les objets PPM.

PERMCHECK

Syntaxe

PERMCHECK:<param>

ou

Description

Vérifie si l'utilisateur actuel ou spécifié est autorisé à effectuer l'action spécifiée pour l'objet. Renvoie la valeur Vrai ou Faux en fonction du résultat de cette vérification.

 

  1. Lorsque cette balise est utilisée, la valeur normalement renvoyée par la balise principale est remplacée par Vrai ou Faux.

  2. Cette balise est valable uniquement lorsqu'elle est utilisée avec une balise qui renvoie normalement un ID d'objet Content Server valide. Par exemple, ID, DATAID et VOLUMEID. Une balise de rapport incorrecte renvoie la valeur Faux pour cette sous-balise.

Pour utiliser ce tableau, trouvez le niveau supérieur d'autorisation Content Server que l'utilisateur s'attend à obtenir pour que la balise renvoie Vrai. Utilisez ensuite le paramètre répertorié sous Action vérification_autor.

PERMINFO

Syntaxe

PERMINFO:<format>

Description

Pour un ID de nœud valide, cette sous-balise renvoie une liste de propriétés Assoc contenant les données d'autorisation du nœud en fonction du paramètre de format fourni. Elle peut être associée en chaîne avec les sous-balises LIST et ASSOC pour renvoyer des informations plus spécifiques. Elle renvoie des données uniquement si l'utilisateur dispose d'autorisations de modification sur le nœud, sinon elle renvoie une chaîne vide.

Paramètres

Format
Description
PERMINFO
Renvoie une liste contenant les propriétés Assoc séparées pour le propriétaire, le groupe de propriétaires, l'accès public et les entrées Accès assigné. La première propriété Assoc contient les données d'autorisation du propriétaire. La deuxième propriété Assoc contient les données d'autorisation du groupe de propriétaires. La troisième propriété Assoc contient les données d'autorisation de l'accès public. S'il existe des entrées Accès assigné, une propriété Assoc distincte est fournie pour chaque entrée à partir de la quatrième entrée de liste.
PERMINFO:OWNER
Renvoie une liste contenant une propriété Assoc qui contient les données d'autorisation du propriétaire.
PERMINFO:OWNERGROUP
Renvoie une liste contenant une propriété Assoc qui contient les données d'autorisation du groupe de propriétaires.
PERMINFO:PUBLICACCESS
Renvoie une liste contenant une propriété Assoc qui contient les données d'autorisation de l'accès public.
PERMINFO:ASSIGNEDACCESS
Renvoie une liste contenant les propriétés Assoc distinctes pour chaque entrée Accès assigné.

Exemple

PERMINFO LIST:1 ASSOC:NAME

Renvoie le nom du nœud Propriétaire.

PERMINFO:PUBLICACCESS LIST:1 ASSOC:PERMASSOC ASSOC:RESERVE

Renvoie la valeur Vrai ou Faux si l'accès public comporte une autorisation Réserver sur le nœud ou non.
