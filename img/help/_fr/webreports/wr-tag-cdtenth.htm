Cette section apporte des informations sur les sous-balises suivantes : ORIGROWNUM, OVERRIDEOBJID et OWNERID.

ORIGROWNUM

Syntaxe

[LL_REPTAG_ORIGROWNUM /]

Description

Renvoie le numéro de ligne de la source de données après le tri de tout WebReport, mais avant le filtrage des WebReports. Cette balise est différente de la balise ROWNUM dans la mesure où elle renvoie toujours le numéro de ligne d'origine de la source de données, quel que soit le nombre de lignes réellement renvoyées par le WebReport. Elle est utile pour les conditions INCLUDEIF qui doivent réagir à un numéro de ligne puisque la balise ROWNUM normale ne peut pas être utilisée dans une instruction INCLUDEIF. Cette balise est similaire à la balise SOURCEROWNUM. Cependant, elle est différente de la balise SOURCEROWNUM car elle reflète l'ordre des lignes après le traitement de toute balise SORT WebReport, alors que SOURCEROWNUM reflète l'ordre des lignes avant le traitement de toute balise SORT WebReports.

OVERRIDEOBJID

Syntaxe

[LL_REPTAG_OVERRIDEOBJID /]

Description

Renvoie le dataID de l'objet Content Server actuellement modifié par un modèle ActiveView.

OWNERID

Syntaxe

[LL_REPTAG_OWNERID /]

Description

Renvoie le ownerID de l'objet Content Server.
