Lorsqu'une constante a été définie, la valeur correspondante peut être insérée dans l'affichage de rapport autant de fois que nécessaire. Il n'est pas indispensable de définir une constante avant d'ajouter la balise correspondante dans l'affichage de rapport. Il peut être plus rapide de les définir via la méthode automatique ci-dessus lorsque vous avez terminé de modifier l'affichage de rapport. Par exemple, si une constante nommée reportID a été stockée pour référencer un autre WebReport, la syntaxe utilisée peut être la suivante :

?func=ll&objid=[LL_REPTAG_$reportID /]&objaction=runreport

Cependant, il peut être plus approprié d'utiliser la même constante avec LLURL comme suit :

[LL_REPTAG_$reportID LLURL:REPORT /]

qui devient ?func=ll&objid=xyz&objaction=runreport, où xyz est la valeur de la constante reportID.

L'avantage de cette approche est que si Content Server modifie le fonctionnement de cette URL, le rapport continue à fonctionner, tandis que la première méthode nécessiterait d'apporter des modifications au rapport. Si vous travaillez dans l'éditeur en ligne WebReports, il est possible d'utiliser l'interface glisser-déposer pour accélérer certaines activités. Pour plus d'informations sur l'utilisation de l'interface glisser-déposer, voir Pour faire glisser et déposer une constante dans l'affichage de rapport WebReport.
