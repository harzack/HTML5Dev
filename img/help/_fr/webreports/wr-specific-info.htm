Comme pour tous les éléments Content Server, les WebReports possèdent une page Propriétés, accessible à partir du menu déroulant Fonctions de l'élément. En plus des onglets standard Content Server, il existe des onglets uniques qui permettent de contrôler les sources, destinations, constantes et paramètres du WebReport. Les fonctionnalités de base de ces onglets et de l'onglet Spécifique sont décrites ici.

Spécifique

L'onglet Spécifique d'un WebReport a les mêmes fonctionnalités que l'onglet Spécifique d'un document. Les propriétés d'un document comme Version actuelle, Nbre max de versions et Type MIME sont utilisées pour gérer le contrôle de version de l'affichage de rapport associé au WebReport. En plus de ces options, il y a toujours une option supplémentaire appelée Exécution/Exportation des audits activée. Cette option est désélectionnée par défaut et permet d'activer des événements d'audit supplémentaires pour chaque WebReport afin que tous les événements RUN ou EXPORT génèrent une entrée dans le journal AUDIT.

Il peut également y avoir une option supplémentaire appelée Ecriture de script Oscript activée. Cette option n'est disponible que lorsque l'affichage de rapport inclut les onglets relatifs à la fonctionnalité Ecriture de script côté serveur. Elle est utilisée pour activer ou désactiver l'exécution des scripts qui peuvent être définis et exécutés avec l'affichage de rapport.

Constantes

L'onglet Constantes permet la création et la gestion de valeurs pouvant être utilisées dans l'affichage de rapport, ou d'autres champs définis.

Destination

L'onglet Destination permet au développeur de spécifier l'emplacement vers lequel le rapport est envoyé. Cela peut être un e-mail, un lancement de workflow ou toute autre destination.

Paramètres

L'onglet Paramètres permet au développeur de définir le comportement des paramètres du WebReport. Il est utilisé pour contrôler des éléments comme l'ordre d'apparition des invites, la valeur par défaut et si un paramètre est obligatoire ou non.

Source

L'onglet Source permet au développeur de spécifier différents types de sources de données, ainsi que le format dans lequel les données source vont apparaître.
