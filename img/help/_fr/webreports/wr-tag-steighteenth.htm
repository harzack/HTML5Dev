Cette section apporte des informations sur les sous-balises suivantes : UNESCAPESTR, UPPER, URLTOPOST, USERACTION, USERINFO, USERINGROUP et USERPREF.

UNESCAPESTR

Syntaxe

UNESCAPESTR

Description

Convertit les caractères d'échappement dans leur équivalent. Par exemple, \r sera converti en caractère de retour chariot.

Exemple

[LL_REPTAG=NAME UNESCAPESTR /]"

UPPER

Syntaxe

UPPER

Description

Met en majuscules tout le texte contenu dans le champ.

URLTOPOST

Syntaxe

URLTOPOST

Description

Pour toute URL valide, convertit l'URL de type GET en une requête POST, en créant une série de champs FORM au nom approprié, avec les valeurs correspondantes. Si l'URL contient un préfixe avant les paramètres, celui-ci est abandonné et seuls les paramètres sont convertis en champs de formulaire.

Exemple

Prenons par exemple l'URL suivante : /Livelink971/livelink.exe?func=ll&objId=157220&objAction=RunReport&reqcacheid=1005851343, qui est convertie en HTML, avec les lignes suivantes :

<INPUT TYPE="HIDDEN" NAME="func" VALUE="ll">
<INPUT TYPE="HIDDEN" NAME="objId" VALUE="157220">
<INPUT TYPE="HIDDEN" NAME="objAction" VALUE="RunReport">
<INPUT TYPE="HIDDEN" NAME="reqcacheid" VALUE="1005851343">

USERACTION

Syntaxe

USERACTION:<opération>

Description

Cette sous-balise permet de manipuler de gros volumes d'utilisateurs et de groupes, au niveau du serveur. Les utilisateurs qui exécutent un rapport contenant cette sous-balise ne sont autorisés à exécuter que les opérations bénéficiant du même niveau d'autorisations/de privilèges que l'interface utilisateur standard.

Paramètres

Action
Description
USERACTION:ADD:<id_utilisateur>
Ajoute un utilisateur ou un groupe au groupe renvoyé par la balise principale.
USERACTION:CREATEGROUP
Crée un nouveau groupe, portant le nom renvoyé par la balise principale. Un paramètre facultatif est pris en charge, pour définir le chef de groupe.
USERACTION:CREATEUSER:<mot_de_passe>:<service>
Crée un nouvel utilisateur bénéficiant des autorisations par défaut (accès public et avec identification), portant le nom renvoyé par la balise principale.
USERACTION:DELETE
Supprime l'utilisateur ou le groupe renvoyé par la balise principale.
USERACTION:DEPARTMENT:<ID de groupe ou nom>
Modifie le groupe de service pour l'utilisateur renvoyé par la balise principale.
USERACTION:PRIVILEGE:<privilège>:<vrai ou faux>
Définit une valeur de privilège pour l'utilisateur renvoyé par la balise principale. Les valeurs de privilège correspondent aux options disponibles dans la section Privilèges de la page Modifier l'utilisateur. Cette sous-balise accepte les valeurs suivantes :
•

LoginEnabled : correspond à la case à cocher Connexion autorisée.

•

PublicAccessEnabled : correspond à la case à cocher Accès public activé.

•

CanCreateModifyUsers : correspond à la case à cocher Peut créer/modifier des utilisateurs.

•

CanCreateModifyGroups : correspond à la case à cocher Peut créer/modifier des groupes.

•

UserAdministrationRights : correspond à la case à cocher Droits d'administration utilisateur.

•

SystemAdministrationRights : correspond à la case à cocher Droits d'administration système.

Cette sous-balise fonctionne uniquement pour les utilisateurs disposant de droits d'administration utilisateur. Les utilisateurs sont autorisés à définir uniquement les privilèges qu'eux-mêmes possèdent.
USERACTION:REMOVE:<id_utilisateur>
Supprime un utilisateur ou un groupe du groupe renvoyé par la balise principale. Notez que vous ne pouvez pas supprimer d'utilisateur d'un groupe dont le service est le même groupe. Dans Content Server, tout utilisateur doit faire partie d'un service.
USERACTION:RENAME:<nom>
Renomme le groupe ou l'utilisateur ciblé.

Exemple

[LL_REPTAG=GROUPID USERACTION:ADD:38832 /] ajoute l'utilisateur dont l'ID est « 38832 » au groupe renvoyé dans la colonne GROUPID.

[LL_REPTAG=newname USERACTION:CREATEGROUP:LEADERID:4526 /] crée un nouveau groupe, qui portera le nom indiqué dans la colonne « nouveau_nom », et spécifie comme chef de ce groupe l'utilisateur « 4526 ».

[LL_REPTAG_'jbloggs' USERACTION:CREATEUSER:'aardv4rk':'DefaultGroup' /] crée un nouvel utilisateur portant le nom « jbloggs », dont le mot de passe est « aardv4rk » et le service « Groupe_par_défaut ».

USERINFO

Syntaxe

USERINFO:<format>

Description

Dans le cas d'un ID d'utilisateur Content Server valide, cette sous-balise renvoie les informations utilisateur au format indiqué dans le paramètre spécifié.

Paramètres

Format
Description
USERINFO
Renvoie une Assoc contenant les métadonnées de l'utilisateur. Peut être associée en chaîne avec la sous-balise ASSOC, pour récupérer des informations spécifiques, non disponibles via les autres balises USERINFO, par exemple [LL_REPTAG_'1000' USERINFO ASSOC:Gender /].
USERINFO:CONTACT
Coordonnées de l'utilisateur.
USERINFO:FIRSTNAME
Prénom de l'utilisateur.
USERINFO:FULLNAME
Nom complet de l'utilisateur, au format globalement spécifié pour le système Content Server ; par exemple, « Joseph Durand ».
USERINFO:GROUPID
ID du groupe principal de l'utilisateur.
USERINFO:GROUPNAME
Nom du groupe principal de l'utilisateur.
USERINFO:LASTNAME
Nom de famille de l'utilisateur.
USERINFO:LEADERID
Si l'ID spécifié correspond à un groupe, l'ID de l'utilisateur chef de ce groupe est renvoyé.
USERINFO:MAILADDRESS
Adresse e-mail de l'utilisateur.
USERINFO:MIDDLENAME
Deuxième prénom de l'utilisateur.
USERINFO:NAME
ID de connexion de l'utilisateur ou nom du groupe.
USERINFO:PRIVILEGES
Renvoie une Assoc contenant les valeurs booléennes correspondant aux privilèges de l'utilisateur. Peut être associée en chaîne à une autre valeur, à l'aide de la sous-balise ASSOC, pour définir un privilège particulier, par exemple [LL_REPTAG_USERID USERINFO:PRIVILEGES ASSOC:UserAdministrationRights /], qui renvoie vrai ou faux en fonction de la valeur de la propriété booléenne « UserAdministrationRights ». Cette sous-balise renvoie des données uniquement si le WebReport est exécuté par un utilisateur disposant de droits d'administration utilisateur.
USERINFO:PWS
ObjectID de l'espace de travail Personnel de l'utilisateur. Cette sous-balise peut être utilisée pour créer un lien vers l'espace de travail Personnel d'un utilisateur, en l'associant à la sous-balise LLURL. Par exemple, [LL_REPTAG_USERID USERINFO:PWS LLURL:BROWSE /] fournit une URL pour accéder à l'espace de travail Personnel de l'utilisateur qui exécute le WebReport.
USERINFO:TITLE
Titre de l'utilisateur.

USERINGROUP

Syntaxe

USERINGROUP

Description

Renvoie Vrai si l'utilisateur actuel, c'est-à-dire celui qui exécute le WebReport, est membre du groupe renvoyé par la balise principale. Si la balise principale n'est associée à aucun ID de groupe valide, la valeur Faux est renvoyée.

Exemple

[LL_WEBREPORT_IF "[LL_REPTAG=id USERINGROUP /]" == "TRUE" /]

I AM IN GROUP: [LL_REPTAG=NAME /]

[LL_WEBREPORT_ENDIF /]

USERPREF

Syntaxe

USERPREF:<onglet>:<paramètre>

Description

Dans le cas d'un ID utilisateur Content Server valide, cette sous-balise renvoie les paramètres utilisateur trouvés lorsque vous sélectionnez Paramètres dans le menu Outils. Certaines sous-balises renvoient une Assoc Oscript et peuvent être associées à la sous-balise TOJSON.

Paramètres

Format
Description
GENERAL
Renvoie une Assoc Oscript contenant les valeurs de l'onglet Général.
GENERAL:DFTSTARTPAGE
Page d'ouverture par défaut de l'utilisateur.
GENERAL:ENHANCEDKYBD
Renvoie Vrai ou Faux en fonction du paramètre Mode d'accès au clavier étendu.
GENERAL:LANGUAGE:CODE
Renvoie le code de la langue préférée de l'utilisateur. Si aucune langue n'a été spécifiée, la langue système par défaut est renvoyée. Voici quelques exemples de valeurs renvoyées possibles : _en_US et _fr.
GENERAL:LANGUAGE:NAME
Renvoie le nom d'affichage de la langue préférée de l'utilisateur. Si aucune langue n'a été spécifiée, la langue système par défaut est renvoyée. Voici quelques exemples de valeurs renvoyées possibles : Anglais (Etats-Unis) et Français.
GENERAL:MODDUR
Durée de l'indicateur Modifié, en nombre de jours.
GENERAL:NAVSTYLE
Renvoie le style de navigation de l'utilisateur. Il s'agit de l'une des trois valeurs suivantes :
•

chaîne vide, si aucune préférence n'a été définie ;

•

bandeau ;

•

liste déroulante.

GENERAL:NEWDUR
Renvoie la durée de l'indicateur Nouveau, en nombre de jours.
GENERAL:SBLOCATION
Renvoie left ou right, en fonction du paramètre Emplacement de la barre latérale. Disponible uniquement dans les versions Content Server 10.0 et supérieures.
GENERAL:SIDEBAR
Renvoie true ou false, en fonction du paramètre Afficher la barre latérale dans la navigation. Disponible uniquement dans les versions Content Server 10.0 et supérieures.
GENERAL:SHOWITEMDESC
Renvoie true ou false, en fonction du paramètre Présenter les descriptions d'élément dans l'affichage détaillé.
COLORS
Renvoie une Assoc Oscript contenant tous les paramètres de l'onglet Couleur.
COLORS:ROW1
Valeur hexadécimale du paramètre Row1.
COLORS:ROW2
Valeur hexidécimale du paramètre Row2.
DISCUSSION
Renvoie une Assoc Oscript contenant tous les paramètres de l'onglet Discussion.
DISCUSSION:DFTPERIODFROM
Paramètre Valeur par défaut de la zone Depuis de l'onglet Discussion.
DISCUSSION:DFTVIEW
Paramètre Affichage par défaut de l'onglet Discussion.
DISCUSSION:FILTERPREF
Paramètre Préférences de filtre de l'onglet Discussion.
DISCUSSION:POSTINGPREF
Renvoie true ou false en fonction du paramètre Préférences de publication dans l'onglet Discussion.
DISCUSSION:REPLYPREF
Renvoie true ou false en fonction du paramètre Préférences de réponses dans l'onglet Discussion.
DISCUSSION:VIEWITEMOPTIONS
Renvoie true ou false en fonction du paramètre Afficher les options de l'élément dans l'onglet Discussion.
WORKFLOW
Renvoie une Assoc Oscript contenant tous les paramètres de l'onglet Workflow.
WORKFLOW:WFSTARTPAGE
Paramètre Tâches assignées du workflow > Page de départ de l'onglet Workflow.
WORKFLOW:PROXY
Paramètre Tâches assignées du workflow > Proxy de l'onglet Workflow.
WORKFLOW:ASSIGNMENTSSHOW
Renvoie une Assoc Oscript contenant tous les paramètres Tâches assignées du workflow > Afficher, y compris les données d'expression, le cas échéant.
WORKFLOW:AUTOOPEN
Renvoie true ou false en fonction du paramètre Ouvrir automatiquement la tâche suivante dans le même workflow si elle m'est assignée dans l'onglet Workflow.
WORKFLOW:INCLUDEWFASSGN
Renvoie true ou false en fonction du paramètre Inclure les tâches assignées du workflow dans la page 'Mes tâches assignées' dans l'onglet Workflow.
WORKFLOW:STATUSSHOW
Renvoie une Assoc Oscript contenant tous les paramètres de l'onglet Statuts du workflow > Afficher dans le workflow, y compris les données d'expression, le cas échéant.
WORKFLOW:SORTORDER
Paramètre Statuts de workflow > Ordre de classement de l'onglet Workflow.
WORKFLOW:DFTWFTAB
Paramètres Onglet 'Mes workflows' par défaut de l'onglet Workflow.
WORKFLOW:MAXITEMS
Paramètre Nombre maximum d'éléments par page de l'onglet Workflow.

Exemple

[LL_REPTAG=USERID USERPREF:GENERAL:STARTPAGE /] renvoie le paramètre Page de départ de l'utilisateur.

Les autres valeurs pouvant être utilisées à la place de STARTPAGE sont :

  • Espace de travail Entreprise

  • Mon espace de travail

  • A propos de Content Server

  • Ma page d'accueil
