Bien que WebReports fournisse de nombreuses balises pour contrôler ce qui est inclus et exclus d'un WebReport, certaines tâches nécessitent encore la puissance d'un langage d'écriture de script complet au niveau du serveur, avant que le rapport atteigne le client. A partir de la version 4.0 de WebReports, il est possible d'insérer une section de script dans un affichage de rapport.

Cette fonction apporte une puissance et une flexibilité considérables pour la production de rapport ou le développement d'applications Web. Comme c'est souvent le cas pour les fonctions puissantes, celle-ci comporte des risques potentiels si elle n'est pas utilisée correctement. Pour réduire au mieux ces risques, l'écriture de script WebReports est soigneusement sécurisée et contenue, et ces restrictions sont configurables : chaque instance de Content Server peut ainsi choisir précisément les fonctions de script activées ou non.

Actuellement, Oscript est le seul langage de script pris en charge. Oscript présente l'avantage de s'exécuter en natif sur la plate-forme Content Server et donc d'apporter d'excellentes performances. D'un point de vue strictement technique, le générateur Content Server n'est pas obligatoire pour créer des WebReports avec Oscript. Cependant, les développeurs pouvant accéder au SDK Content Server peuvent bénéficier des possibilités de débogage de ce générateur. En pratique, sauf pour les scripts relativement simples, l'accès au débogueur du générateur Content Server est essentiel pour que les développeurs puissent exécuter leur code pas à pas et le corriger.

Ce guide de l'utilisateur décrit l'implémentation d'un script dans WebReport, mais ne couvre pas le codage ni la syntaxe d'Oscript. Pour plus d'informations sur Oscript, consultez la documentation du SDK Content Server, telle que l'aide du générateur Content Server, ou contactez Open Text pour plus de détails sur la formation au SDK Content Server.
