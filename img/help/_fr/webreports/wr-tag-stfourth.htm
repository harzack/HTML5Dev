Cette section apporte des informations sur les sous-balises suivantes : DATE, DATEDEC, DATEDIFF, DATEINC, DEC, DECODE, DECSTR, DEF, DIVIDE et DMEMBERSHIP.

DATE

Syntaxe

DATE

ou

DATE :« <liste de formats> »

Description

Cette sous-balise convertit les données formatées de la date au format défini par l'utilisateur. La sortie peut être formatée à l'aide des modificateurs ci-dessous. Par exemple, [LL_REPTAG=createdate DATE:"%a - %b %d, %Y" /] fournit une date similaire à : « Lun - Jan 31, 2004. »

Cette sous-balise prend en charge les dates formatées Content Server [LL_REPTAG_DATETIME /], [LL_REPTAG_DATE /]. Par exemple, « J/2006/3/27:0:0:0 »

Paramètres

Opérateur

 Le format d'entrée de date comme spécifié sur les pages Administration.
SHORT
 Le format de date court comme spécifié sur les pages Administration.
LONG
 Le format de date long comme spécifié sur les pages Administration.
%%
 Un signe de pourcentage.
%a
 Les trois premiers caractères du jour de la semaine comme Lun, Mar, etc.
%b
 Les trois premiers caractères du mois comme Jan, Fév, etc.
%d
 Les deux chiffres du jour du mois, de 01 à 31 comme 01-31.
%j
 Les trois chiffres du jour de l'année, de 001 à 366.

 Les deux chiffres du mois, comme 01 à 12

 Matin ou soir (AM ou PM)

 Le chiffre du jour de la semaine, de 1 à 7, 1 correspondant à dimanche.

 Les deux chiffres de l'année, comme 93.

 Le nom complet du jour de la semaine, comme Lundi.
%B
 Le nom complet du mois, comme Janvier.

 Les deux chiffres de l'heure, de 01 à 12.

Informations associées

Pour la manipulation des formats de date personnalisés, voir TODATE. Pour la date actuelle, voir les balises DATE et DATETIME.

DATEDEC

Syntaxe

DATEDEC :<valeur> :<unités>

Description

Décrémente la date renvoyée par la balise principale par la valeur et les unités spécifiées. Les formats de date valides pour la balise principale incluent les dates formatées [LL_REPTAG_DATETIME /], [LL_REPTAG_DATE /] et Content Server telles que « Sam Avr 16 00:00:00 2011 » , « Fév 20 2010 » et « J/2012/12/25:00:00:00 ». Le paramètre <valeur> doit être un entier positif.

Paramètres

Le paramètre <unités> est l'un des suivants :

Unités
Description
DAYS
Cette option décrémente la date d'un nombre <x> de jours.
WEEKS
Cette option décrémente la date d'un nombre <x> de semaines.
SECONDS
Cette option décrémente la date d'un nombre <x> de secondes.

Exemple

[// Here DATETIME and DATE tags return Mon Apr 11 00:00:00 2011
[LL_REPTAG_DATETIME DATEDEC:2:DAYS /] [// Returns Sat Apr 09 00:00:00 2011
[LL_REPTAG_DATE DATEDEC:2:WEEKS /] [// Returns Mon Mar 28 00:00:00 2011
[LL_REPTAG_'Sat Apr 16 10:23:32 2011' DATEDEC:86400:SECONDS /] [// Returns Fri Apr 15 10:23:32 2011
[LL_REPTAG_'D/2012/12/25:00:00:00' DATEDEC:4:WEEKS /] [// Tue Nov 27 00:00:00 2012

[// Here dueDate is a column from the data source and returns the date: Tue May 24 00:00:00 2011
[LL_REPTAG=dueDate DATEDEC:6:DAYS /] [// Returns Wed May 18 00:00:00 2011

Une chaîne vide est renvoyée si la balise principale est vide. Un message d'erreur est envoyé si la balise principale n'est pas une date valide.

Informations associées

Voir aussi DATEINC.

DATEDIFF

Syntaxe

DATEDIFF :<date de référence> :<unités>

ou

DATEDIFF

ou

DATEDIFF :<unités>

ou

DATEDIFF :<date de référence>

Description

Renvoie la différence entre la date dans la balise principale et la date de référence, <date de référence> moins <date de la balise principale>. Si la date de référence est omise, alors la date et l'heure actuelles du serveur sont utilisées. Un paramètre <unités> facultatif convertit les secondes en minutes, heures, jours, etc. Si aucun paramètre <unités> n'est fourni, la différence est par défaut renvoyée en jours. Si la différence mesurée ne peut pas être spécifiée dans un nombre rond à l'aide de l'unité spécifiée, alors une valeur décimale à 2 chiffres est renvoyée. Par exemple, si la différence entre deux dates est de 75 minutes et que le paramètre HOURS a été spécifié, alors la valeur 1.25 est renvoyée. Les formats de date valides pour la balise principale incluent les dates formatées [LL_REPTAG_DATETIME /], [LL_REPTAG_DATE /] et Content Server. Par exemple : « Sam Avr 16 00:00:00 2011 », « Fév 20 2010 », « J/2012/12/25:00:00:00 ». Le paramètre <date de référence> doit également être une date formatée Content Server valide. Le paramètre <unités> peut être l'un des suivants : SECONDS, MINUTES, HOURS, DAYS, WEEKS, MONTHS, YEARS et WEEKDAYS. L'unité WEEKDAYS renvoie les jours, à l'exception des week-ends. Donc, si DAYS renvoie normalement 10 et couvre deux week-ends, WEEKDAYS renvoie uniquement 6. Si la date antérieure commence pendant le week-end, le calcul commence à partir du lundi suivant. Si la date ultérieure se termine pendant un week-end, alors le calcul prend fin le vendredi précédent.

Paramètres

Balise
Résultat
[LL_REPTAG_"D/2010/5/30:0:0:0" DATEDIFF:"D/2011/5/30:0:0:0":DAYS /] 365
[LL_REPTAG=modifyDate DATEDIFF:HOURS /] Nombre d'heures entre la date modifiée, comme renvoyé à partir de la BD, et la date d'aujourd'hui.
[LL_REPTAG=modifyDate DATEDIFF:"[LL_REPTAG_&dateparm /]":DAYS /] Le nombre de jours entre la date modifiée et le paramètre de date transmis.

DATEINC

Syntaxe

DATEINC :<valeur> :<unités>

Description

Incrémente la date renvoyée par la balise principale par la valeur et les unités spécifiées. Les formats de date valides pour la balise principale incluent les dates formatées [LL_REPTAG_DATETIME /], [LL_REPTAG_DATE /] et Content Server telles que « Sam Avr 16 00:00:00 2011 » , « Fév 20 2010 » et « J/2012/12/25:00:00:00 ». Le paramètre <valeur>> doit être un entier supérieur à zéro.

Paramètres

Le paramètre <unités> est l'un des suivants :

Unités
Description
DAYS
 Cette option incrémente la date d'un nombre <x> de jours.
WEEKS
 Cette option incrémente la date d'un nombre <x> de semaines.
SECONDS
 Cette option incrémente la date d'un nombre <x> de secondes.
WEEKDAYS
 Cette option incrémente la date d'un nombre <x> de jours, à l'exception des week-ends.

Exemple

[// Here DATETIME and DATE tags return Mon Apr 11 00:00:00 2011
[LL_REPTAG_DATETIME DATEINC:2:DAYS /]     [// Returns Wed Apr 13 00:00:00 2011
[LL_REPTAG_DATE DATEINC:2:WEEKS /]        [// Returns Mon Apr 25 00:00:00 2011

[LL_REPTAG_'Sat Apr 16 10:23:32 2011' DATEINC:86400:SECONDS /]   [// Returns Sun Apr 17 10:23:32 2011
[LL_REPTAG_'Feb 20 2010' DATEINC:10:WEEKDAYS /]     [// Returns Fri Mar 05 00:00:00 2010
[LL_REPTAG_'D/2012/12/25:00:00:00' DATEINC:4:WEEKS /]       [// Tue Jan 22 00:00:00 2013

[// Here dueDate is a column from the data source and returns the date: Tue May 24 00:00:00 2011
[LL_REPTAG=dueDate DATEINC:60:DAYS /]               [// Returns Sat Jul 23 00:00:00 2011

Une chaîne vide est renvoyée si la balise principale est vide. Un message d'erreur est envoyé si la balise principale n'est pas une date valide.

Informations associées

Voir aussi DATEDEC.

DEC

Syntaxe

DEC

ou

DEC :<nombre de décrémentation>

Description

Décrémente une valeur numérique de 1 ou d'un nombre de décrémentation spécifié. Par défaut, cette sous-balise renvoie la valeur de balise actuelle moins un et des erreurs si la balise ne contient pas un entier valide. Si le nombre de décrémentation facultatif a été spécifié, alors la balise actuelle à laquelle le nombre de décrémentation a été oté est renvoyée.

DECODE

Syntaxe

DECODE:<if1>:<then1>

ou

DECODE:<if1>:<then1>:<if2> :<then2>:...:<else>

Description

Effectue une substitution valeur par valeur, similaire à l'instruction DECODE Oracle. Les paramètres de DECODE se composent de paires de chaînes spécifiant une valeur de correspondance et de remplacement. Au moins une paire de chaînes doit être spécifiée, cette dernière pouvant atteindre un total de 98 paires. La chaîne <else> est facultative. La sous-balise DECODE recherche si les données renvoyées par la balise de données correspondent à l'une des chaînes <if> spécifiées dans la liste de paramètres. Si elle trouve une correspondance, elle remplace les données par la chaîne <then>. Si elle ne trouve pas de correspondance, elle remplace les données par la chaîne <else> facultative, si spécifiée. Par exemple, si une balise de données [LL_REPTAG=STATUS /] renvoie des chaînes numériques indiquant un statut tel que 0, 1, 2, etc. Nous souhaitons présenter cela sous forme de texte explicite comme « Achevé », « Révision en attente » et « Ouvrir », respectivement. Grâce à l'instruction DECODE, le résultat ci-dessous est obtenu : DECODE:"0":"Completed":"1":"Pending Review":"2":"Open":"Unknown"

Le paramètre « Inconnu » (Unknow) signifie qu'aucune des valeurs 0, 1 ou 2 n'est trouvée, la chaîne « Inconnu » est substituée.

DECSTR

Syntaxe

DECSTR :<type_codage>

Description

Décode la chaîne spécifiée à l'aide de la méthode de décodage sélectionnée. Actuellement, BASE64 est le seul type pris en charge.

DEF

Syntaxe

DEF :<valeur>

Description

Si la valeur renvoyée par la balise principale est indéfinie, la valeur spécifiée ici est utilisée.

DIVIDE

Syntaxe

DIVIDE :<valeur>

Description

Divise un entier renvoyé dans la balise principale par la valeur spécifiée. D'autres sous-balises peuvent être utilisées en association avec cette balise, telle que la balise ROUND pour offrir plus de précision.

Exemple

Par exemple : [LL_REPTAG_"10" DIVIDE:2 ROUND:2 /] devient 5.00.

DMEMBERSHIP

Syntaxe

DMEMBERSHIP

Description

Renvoie une liste séparée par des virgules des groupes Content Server dont l'utilisateur est un membre direct.
