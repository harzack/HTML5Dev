Plusieurs couches de sécurité assurent que l'écriture de script WebReports est aussi sûre que possible :

  1. Si la fonction d'écriture de script n'est pas requise, elle peut être désactivée, entièrement et globalement, par l'administrateur Content Server. Avant de tenter d'utiliser cette fonction, il est souhaitable que les utilisateurs vérifient auprès de leur administrateur si elle est activée ou non.

  2. Même si cette fonction est activée globalement, elle doit être activée pour chaque WebReport qui l'utilise, chaque fois qu'une nouvelle version d'un affichage de rapport est ajoutée au WebReport. Tout utilisateur pouvant créer un WebReport peut inclure des balises pour définir et appeler des scripts, mais ces balises restent désactivées jusqu'à ce qu'un administrateur système vérifie et active l'écriture de script pour le rapport. Les rapports désactivés s'exécutent, mais ils ignorent les scripts et ne les exécutent pas.

  3. Par défaut, les fonctions fournies par Oscript sont fortement limitées. Par exemple, le cas échéant, vous ne pouvez pas accéder aux variables globales ou à de nombreux packages intégrés au générateur comme, entre autres, CAPI, DAPI et File. La disponibilité des fonctions peut être configurée par l'administrateur Content Server.
