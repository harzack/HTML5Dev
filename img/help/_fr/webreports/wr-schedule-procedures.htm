Les développeurs WebReports, ou les utilisateurs disposant de l'autorisation Réserver, peuvent définir des programmes d'exécution pour chaque WebReport, sans intervention de l'utilisateur. Les rapports programmés s'exécutent en arrière-plan et envoient les résultats vers un nœud de document Content Server ou un e-mail. Le programme permet l'exécution des rapports un certain nombre de fois ou tout le temps, à partir d'une date et d'une heure spécifiques. Le temps entre l'exécution répétée d'un rapport peut varier de 5 minutes à 520 semaines. Un programme peut être défini par utilisateur et par WebReport.

Parfois, il peut être nécessaire d'exécuter un WebReport le même jour de chaque mois. Par exemple, le rapport des heures supplémentaires d'un salarié peut être nécessaire chaque 25du mois. En raison de l'intervalle de programme fixe indiqué par les WebReports, ceci ne peut pas être réalisé facilement via l'interface. Vous devez plutôt utiliser les sous-rapports WebReports. Voir Pour définir l'exécution d'un WebReport à la même date ou le même jour.

Notes importantes et recommandations

  1. Les utilisateurs doivent être vigilants quant au moment et à la fréquence de programmation d'un rapport qui renvoie un ensemble de données particulièrement important. Ces rapports peuvent nécessiter d'importantes ressources et impacter les performances système s'ils sont exécutés trop souvent ou pendant les heures de pointe.

  2. Si les services Content Server sont désactivés ou que les agents Content Server sont indisponibles pour une tout autre raison, les rapports programmés ne s'exécuteront pas. Lorsqu'ils sont à nouveau activés, WebReports exécutera un rapport le plus tôt possible si un rapport est manquant, sans tenir compte du programme. Ensuite, WebReports exécutera le rapport lors du prochain programme prévu. Si vous observez des temps d'exécution irréguliers, ceci en est probablement la cause.

  3. Les paramètres de destination de sortie, e-mail ou Content Server, sont les mêmes pour la programmation que pour l'action par défaut pour l'utilisateur cliquant sur le lien WebReport. Si ces paramètres doivent être différents, OpenText recommande d'utiliser deux WebReports avec le même affichage de rapport. Le rapport programmé peut être masqué ou placé dans un dossier autorisé.

  4. Les WebReports programmés s'exécutent comme s'ils étaient exécutés par l'utilisateur ayant défini le programme. Par conséquent, lorsqu'un filtrage par autorisations est utilisé dans une source de données, comme un rapport LiveReport ou une requête de recherche, il renverra les résultats que cet utilisateur a l'autorisation de voir. Toujours envoyer les rapports sensibles vers un emplacement de Content Server possédant les autorisations adéquates.
