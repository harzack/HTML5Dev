Cette section apporte des informations sur les balises de données de colonne : SOURCENAME, SOURCEID, SOURCEROWNUM, SUBVERSION et SUPPORTDIR.

SOURCENAME

Syntaxe

[LL_REPTAG_SOURCENAME /]

Description

Renvoie le nom de la source de données.

SOURCEID

Syntaxe

[LL_REPTAG_SOURCEID /]

Description

Renvoie l'objectid de la source de données. Cette balise peut être utilisée avec la sous-balise NODEINFO afin de récupérer le nom de la source de données.

Exemple

Par exemple, [LL_REPTAG_SOURCEID NODEINFO:NAME /]

SOURCEROWNUM

Syntaxe

[LL_REPTAG_SOURCEROWNUM /]

Description

Renvoie le numéro de ligne de la source de données. Cette balise est différente de la balise ROWNUM dans la mesure où elle renvoie toujours le numéro de ligne d'origine de la source de données, quels que soient le nombre de lignes réellement renvoyées par le WebReport et les tris de WebReport effectués. Cela est utile pour les conditions INCLUDEIF qui doivent porter sur les lignes de la source de données d'origine avant le traitement de toute ligne de WebReports.

Informations associées

Cette balise est similaire à ORIGROWNUM. Toutefois, la balise ORIGROWNUM fournit la référence du numéro de ligne après tout tri.

SUBVERSION

Syntaxe

[LL_REPTAG_SUBVERSION /]

Description

Renvoie les informations de sous-version pour le logiciel Resonate KT, ce qui est particulièrement utile pour les administrateurs et lorsque vous signalez un problème auprès du service d'assistance.

Exemple

Exemple de sortie : (Livelink 9.7.1) (rktengine 10.0.1.21_Enabled) (webreports 10.0.2.27)

SUPPORTDIR

Syntaxe

[LL_REPTAG_SUPPORTDIR /]

Description

Fournit le chemin correct afin de référencer le répertoire de support Content Server. Ce répertoire et/ou dossier contient des fichiers de support Content Server standard tels que des images.

Exemple

Par exemple, dans de nombreux cas, la balise renvoie : /LivelinkSupport/.
