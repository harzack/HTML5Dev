Un WebReport peut appeler un autre WebReport (un « sous-rapport WebReport ») dont les résultats seront renvoyés et insérés dans le rapport parent, à l'endroit où la balise est placée. Les sous-rapports WebReport peuvent être considérés comme des « sous-routines », qui font partie des résultats de rapport globaux. Cette balise est particulièrement utile lorsque vous souhaitez collecter les résultats de plusieurs requêtes différentes (par exemple des requêtes SQL LiveReport) et les présenter dans un rapport unique. Ainsi, il est souvent plus facile d'utiliser plusieurs requêtes SQL simples à la place d'une seule requête complexe. Vous pouvez également préférer combiner les résultats d'une requête SQL avec ceux d'une autre requête de recherche.

Il existe deux façons d'appeler un sous-rapport WebReport :

  • Si le moteur RKT est activé : [LL_REPTAG_'<Sub-WebReport ID>' RUNSWR /]

  • Si le moteur RKT n'est ni activé, ni installé : [LL_WEBREPORT_SUBWEBREPORT NODEID:<Sub-WebReport ID> /]

Les deux approches ont le même résultat, sauf que le moteur RKT permet l'ajout d'autres sous-balises après RUNSWR. Par exemple : [LL_REPTAG_'<Sub-WebReport ID>' RUNSWR SHOW SETVAR:SubWRValue /].
