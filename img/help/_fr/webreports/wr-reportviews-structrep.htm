L'affichage de rapport comprend les éléments suivants :

  1. Section en-tête : cette section est utilisée pour communiquer tout code devant être exécuté au début du WebReport. Elle est générée une seule fois, quel que soit le nombre de lignes de rapport qui sont lues à partir de la source de données sous-jacente. Par exemple, si vous produisez un document HTML, toutes les balises d'ouverture comme <TABLE> doivent figurer dans cette section. Cependant, il n'est pas nécessaire d'inclure <HTML> ou <BODY> si l'en-tête Content Server standard est inclus. L'en-tête Content Server est inclus par défaut, sauf si la balise [LL_WEBREPORT_EXCLUDEHEADER /] est utilisée.

  2. [LL_WEBREPORT_STARTROW /] : dans l'éditeur en ligne, cette ligne n'est pas modifiable. Elle est fournie uniquement pour indiquer l'endroit où la balise de début de ligne sera insérée afin que les utilisateurs n'essayent pas d'en ajouter une autre.

  3. Section ligne : cette section est utilisée pour définir tout code devant être appliqué à toutes les lignes de données. Toutes les balises utilisées pour désigner les données de la colonne de source de données ne fonctionneront que dans cette section. Il est important de noter que tout dans cette section sera répété pour chaque ligne de la source de données. Par exemple, si 10 lignes de données du rapport doivent être renvoyées, tout dans cette section apparaîtra 10 fois dans la sortie du WebReport. La seule exception est l'utilisation de balises conditionnelles pour exclure certaines données de la sortie. Pour en savoir plus, voir Balises d'affichage de rapport.

  4. [LL_WEBREPORT_ENDROW /] : dans l'éditeur en ligne, cette ligne n'est pas modifiable. Elle est fournie uniquement pour indiquer l'endroit où la balise de fin de ligne sera insérée afin que les utilisateurs n'essayent pas d'en ajouter une autre.

  5. Section pied de page : cette section est utilisée pour communiquer tout code devant être exécuté à la fin du WebReport. Elle est générée une seule fois, quel que soit le nombre de lignes de rapport qui sont lues à partir de la source de données sous-jacente. Toutes les balises HTML de fermeture comme </TABLE> figurent normalement dans cette section.
