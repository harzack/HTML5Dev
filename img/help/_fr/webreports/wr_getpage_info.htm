The page.js file packaged with WebReports includes a useful function 'getPage' to assist in the mechanics of pagination when you want to display a large set of results. This function is called when the user clicks the First, Previous, Next, or Last buttons (See image below). In addition this function is available to use in any WebReport reporting application. The page.js file is accessed using the a SCRIPT include like this:

<SCRIPT SRC="/<support dir>/webreports/library/page.js"></SCRIPT>

If this file is being included into a WebReport the LIBPATH tag can be used like this: 

<SCRIPT SRC="[LL_REPTAG_LIBPATH /]page.js"></SCRIPT>

Example Pagination - First, Previous, Next, and Last Buttons



A description of the 'getPage' function and its parameters is provided below:

getPage( baseURL, page, dataCount, pageSize, ip1, ip2, startParm, endParm )

  • A function that is invoked when the user clicks the First, Previous, Next, or Last buttons. It builds and executes the URL for the next slice of data to display to the user based on the button that is clicked.
  • Takes arguments: baseURL (required), page (required), dataCount (required), pageSize (required), ip1 (required), ip2 (required), startParm (optional), endParm (optional)
    • baseURL - the base URL of the report we want to run. This can be achieved using the WebReport tag "[LL_REPTAG_MYID LLURL:REPORT /]" which would resolve to something like this: '?func=ll&objId=xyz&objAction=RunReport'
    • page - a String indicating which button is clicked. It must be one of the following values: 'first', 'prev', 'next', or 'last'.
    • dataCount - a number representing the total number of rows to paginate through.
    • ip1 - the current start row number. See example below.
    • ip2 - the current end row number. See example below.
    • Example: If 'ip1=1' and 'ip2=20' is specified, it would result in the first 20 rows being displayed when the user runs the WebReport.
    • startParm - an optional parameter that represents the start parameter name. If not specified, it defaults to 'inputLabel1'. Generally this is the name for the parameter 'ip1'. This parameter is used in the URL of the next page to display with its value set to the start row number.
    • endParm - and optional parameter that represents the end parameter name. If not specified, it defaults to 'inputLabel2'. Generally this is the name for the parameter 'ip2'. This parameter is used in the URL of the next page to display with its value set to the end row number.
