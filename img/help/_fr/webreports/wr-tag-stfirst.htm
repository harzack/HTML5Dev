Cette section apporte des informations sur les sous-balises suivantes : A2UTF8, ABS, ADD, ADDVAR, ASSOC, AUDITACTION et AUDITINFO.

A2UTF8

Syntaxe

A2UTF8

Description

Convertit une chaîne ASCII en chaîne UTF-8.

ABS

Syntaxe

ABS

Description

Affiche la valeur absolue d'un nombre.

ADD

Syntaxe

ADD

Description

Ajoute une valeur à la balise principale. L'opération exacte dépend des types de valeur concernés. La valeur renvoyée de l'expression suivante dépend des types de valeur :

[LL_REPTAG_'X' ADD:Y /]

Paramètres

Type de X Type de Y Renvoie Exemple
Entier ou Réel Entier ou Réel X + Y
[LL_REPTAG_'5' ADD:2.2 /]
renvoie 7.2
Date Entier (secondes) X + Y
[LL_REPTAG_'May 1 2011' ADD:86400 /]
renvoie Lun Mai 02 00:00:00 2011
Liste Liste {@X,@Y}
[LL_REPTAG_'{1,2,3}' ADD:'{4,5,6}' /]
renvoie {1,2,3,4,5,6}
Liste de tous les types {@X,Y}
[LL_REPTAG_'{1,2,3}' ADD:'four' /]
renvoie {1,2,3,'four'}
Assoc Assoc Fusionne X et Y. Y a priorité sur les clés en double.
[LL_REPTAG_« A<?,1,'clé1'=1> » ADD:« A<?,1,'clé1'=2,'clé2'=5> » /]
renvoie A<1,1,'clé1'=2,'clé2'=5>
Enregistrement Enregistrement Fusionne X et Y. Y a priorité sur les clés en double. L'ordre de la colonne est perdu.
[LL_REPTAG_« R<'clé1'=1> » ADD:« R<'clé1'=2,'clé2'=5> » /]
renvoie R<'clé1'=2,'clé2'=5>
RecArray RecArray Additionne Y et X, dans la mesure où X et Y présentent les mêmes champs.
[LL_REPTAG_« V{<'fruit','couleur'><'banane','jaune'> »
renvoie V{<'fruit','couleur'><'banane','jaune'><'orange','orange'>}
autre chose autre chose Chaîne(X) + Chaîne(Y)
[LL_REPTAG_« abc » ADD:« 3 » /]
renvoie abc3

ADDVAR

Syntaxe

ADDVAR:<nom_variable>

Description

Ajoute la valeur de la balise, après avoir mis en forme toutes les éventuelles sous-balises précédentes, avec le contenu actuel d'une variable qui pourra être référencée ultérieurement à l'aide du nom de variable spécifié. Cette balise fonctionne uniquement lorsque des valeurs numériques sont renvoyées par une balise. Lors de sa première utilisation, cette sous-balise se comporte comme s'il existait déjà une variable de valeur zéro (0).

Exemple

Par exemple, [LL_REPTAG=cost ADDVAR:subtotal /]

Dans cet exemple, si les valeurs 26, 34 et 14 étaient renvoyées comme valeurs de la balise « cost », la balise variable LL_REPTAG_%nom_variable renverrait : 74

Informations associées

Pour plus d'informations concernant l'utilisation de ADDVAR, des balises de variables et d'autres sous-balises associées à la gestion des variables, voir la section Informations avancées : Utilisation des variables.

Voir également SETVAR, CONCATVAR et CURRENTVAL.

ASSOC

Syntaxe

ASSOC:<clé> ou ASSOC:[index]

Description

Renvoie la valeur du champ clé, si le type de données renvoyé est une ASSOC Content Server. Vous pouvez utiliser aussi bien la clé elle-même qu'un nombre entier représentant l'emplacement de la clé au sein de l'ASSOC. Notez que cette sous-balise peut être utilisée de manière répétée lorsque plusieurs ASSOC sont imbriquées.

Informations associées

Voir également LIST, RECORD et RECARRAY.

AUDITACTION

Syntaxe

AUDITACTION:<opération>

Description

Attend une valeur d'ID Content Server valide, que ce soit un ID d'utilisateur/de groupe ou le DataID d'un élément Content Server, et exécute une action pour générer un audit d'événement.

Le résultat est l'ajout d'une nouvelle ligne dans la table de base de données DAuditNew Content Server et de lignes facultatives dans la table DAuditMore. Si l'ID spécifié est un DataID, le champ DataID de l'enregistrement DAuditNew sera configuré ; s'il s'agit d'un ID d'utilisateur ou de groupe, le champ UserID sera configuré à la place. Le champ non utilisé prendra la valeur NULL.

Le champ PerformerID pour le nouvel enregistrement d'audit est toujours renseigné avec l'ID de l'utilisateur dont le contexte était activé lors de l'exécution de la sous-balise. C'est le cas même lorsque la fonction « Exécuter en tant que » a été utilisée pour exécuter le WebReport dans un contexte utilisateur différent. Dans ce cas, une ligne est ajoutée à la table DAuditMore, liée avec la clé eventID, indiquant l'ID de l'utilisateur « Exécuté en tant que » sélectionné.

Outre les colonnes UserID, DataID et PerformerID, vous pouvez spécifier des valeurs pour plusieurs autres champs, tel qu'indiqué dans cette table. Notez que la case à cocher Evénement d'audit personnalisé doit être activée, dans la page Administration Définir les événements à inclure dans l'audit, pour que la sous-balise AUDITACTION puisse fonctionner.

Paramètres

Format
Description
AUDITACTION:<valeur>
Ajoute une entrée dans la table DAuditNew, avec la valeur spécifiée dans la colonne « valeur1 ».
AUDITACTION:<clé fixe>:<valeur>:<clé fixe>:<valeur> ...
Ajoute une entrée dans la table DAuditNew, avec plusieurs valeurs définies sur une clé fixe, les noms de colonne de la table DAuditNew, qui peuvent être soit « valeur1 », « valeur2 » ou « valeur_clé ».
AUDITACTION:<clé utilisateur>:<valeur>:<clé utilisateur>:<valeur> ...
Ajoute des paires de valeurs clés dans la table DAuditMore. Les valeurs spécifiées dans la table sont ajoutées à la colonne KeyStr, tandis que la valeur spécifiée sera ajoutée à la colonne ValueStr.

AUDITINFO

Syntaxe

AUDITINFO:<Nom_colonne>

Description

Attend un ID de nœud Content Server valide. Renvoie le détail de l'événement d'audit le plus récent pour l'objet et vous permet de spécifier quelle colonne de la table DAuditNew afficher. Cette balise prend en charge les entrées d'audit personnalisées générées par la sous-balise AUDITACTION, ainsi que la fonction RunAs.

Paramètres

Format
Description
AUDITINFO:ACTUALUSER
Pour les événements « Exécution WebReports », si le WebReport a été exécuté par la fonction RunAs, l'ID de l'utilisateur RunAs sera affiché.
AUDITINFO:AUDITDATE
Renvoie la valeur AuditDate de l'événement à partir de la table DAuditNew.
AUDITINFO:AUDITID
Renvoie la valeur AuditID de l'événement à partir de la table DAuditNew.
AUDITINFO:AUDITSTR
Renvoie la valeur AuditStr de l'événement à partir de la table DAuditNew.
AUDITINFO:EVENTID
Renvoie la valeur EventID de l'événement à partir de la table DAuditNew.
AUDITINFO:KEYSTR:<clé>
Cette option recherche la valeur « clé » dans la colonne KeyStr de la table DAuditMore et renvoie la valeur correspondante pour la colonne ValueStr. Cette fonction est utile pour renvoyer la valeur ValueStr pour une valeur personnalisée donnée.
AUDITINFO:PERFORMERID
Renvoie la valeur PerformerID de l'événement à partir de la table DAuditNew. Pour les événements Exécution WebReports, utilisez l'option :ACTUALUSER afin d'afficher l'ID de l'utilisateur RunAs.
AUDITINFO:SUBTYPE
Renvoie la valeur SUBTYPE de l'événement à partir de la table DAuditNew.
AUDITINFO:USERID
Renvoie la valeur UserID de l'événement à partir de la table DAuditNew.
AUDITINFO:VALUE1
Renvoie la valeur Value1 de l'événement à partir de la table DAuditNew.
AUDITINFO:VALUE2
Renvoie la valeur Value2 de l'événement à partir de la table DAuditNew.
AUDITINFO:VALUEKEY
Renvoie la valeur ValueKey de l'événement à partir de la table DAuditNew.
AUDITINFO:VALUESTR
Renvoie la valeur ValueStr de l'événement à partir de la table DAuditNew.
