Broadly speaking there are three types of tags in WebReports:

Content Control Tags
These tags are always prefixed with: [LL_WEBREPORT_. They are used to control the way in which the output content is built. For example, it is possible to enable or disable the standard Livelink header and footer. The most important content control tags are used to define which part of the reportview is applied to each row of data source Data.

Data Tags
Data tags are always prefixed with: [LL_REPTAG. They can further be broken down into Column Data Tags and Static Tags. For every column in the associated data source, column data tags exist to reference which pieces of report data are to be included in the output. These tags allow either the column number (e.g. [LL_REPTAG_2 /]) or the column name (e.g. [LL_REPTAG=DATAID /]) to be used.

Static tags are used to insert information from Content Server which is not associated with report rows. For example, the date and time, the name of the user running the WebReport, or various URLs that might help with building hyperlinks in the report output. (E.g. [LL_REPTAG_NEXTURL /])
Note: WebReport tags are case sensitive (always use uppercase) and space sensitive (ensure there is a space before the '/]' at the end of the tag).

Sub-tags
Any Content Server data tags e.g. [LL_REPTAG... can contain a list of formatting sub-tags separated by spaces which allow actions to be taken on the actual data pulled from the data source. Several of these sub-tags can be used for the same tag if required. If multiple sub-tags are used they are processed from left to right so that the output of each sub-tag operation is passed to the next sub-tag. E.g.In the following tag the output would be lowercase. [LL_REPTAG_1 UPPER LOWER /]

Some sub-tags use additional parameters. For example, in the following tag a sub-tag called REPLACE is used which requires 2 parameters (one for the string to be replaced, and one for the string to replace with. [LL_REPTAG_1 REPLACE:_dummy:genius /]
Any sub-tag parameters are added using the colon (:) symbol. Some parameters may need to use quotes to specify parameters that require spaces. In the previous example if the string to be replaced was 'test value', the syntax would be: [LL_REPTAG_1 REPLACE:"test value":"new value" /]
