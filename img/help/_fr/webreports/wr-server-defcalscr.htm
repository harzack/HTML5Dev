Les scripts sont définis et appelés au sein du même affichage de rapport. La définition d'un script peut apparaître partout dans l'affichage de rapport et un script peut être appelé avant sa définition. En effet, les définitions des scripts sont analysées, compilées et mises en cache avant l'exécution d'un WebReport, dans la plupart des cas au moment où la nouvelle version de l'affichage de rapport est ajoutée. Cependant, pour faciliter la compréhension des affichages de rapports, il convient de placer les définitions de scripts dans la section d'en-tête.

Pour définir un script, utilisez le code suivant :

[LL_WEBREPORT_STARTSCRIPT NAME:myFunc /]
function String anyName(Dynamic c)
  String s = 'Testing'
  return s
end
[LL_WEBREPORT_ENDSCRIPT /]

Lorsqu'il est appelé, ce simple script renvoie la chaîne « Testing » à l'affichage de rapport. Cette chaîne est insérée dans le résultat du rapport, là où le script a été appelé. Pour appeler le script, utilisez le code : [LL_WEBREPORT_CALL NAME:myFunc /]

Cette balise entraîne l'exécution du script appelé myFunc et tout résultat renvoyé est inséré dans la sortie du rapport, à la place de la balise. Les scripts peuvent être appelés partout dans l'affichage de rapport.
