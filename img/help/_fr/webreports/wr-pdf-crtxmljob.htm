Pour créer un bon de travail XML, veuillez respecter les éléments suivants :

  1. Les bons de travail XML doivent être valides pour être utilisés. Les règles suivantes doivent s'appliquer à chaque bon de travail pour qu'il soit valide :

    1. Doit contenir un en-tête XML tout en haut. Par exemple, <?xml version="1.0" encoding="ISO-8859-1"?>

    2. L'en-tête XML doit être suivi par la balise d'en-tête Adlib qui contient la version à laquelle le bon Adlib se conforme. Par exemple, <?AdLibeXpress applanguage="USA" appversion="2.7.0" dtdversion="1.8"?>

    3. Doit contenir une balise <JOBS> avec des attributs facultatifs.

    4. Doit contenir une seule balise <JOB>. Notez que la balise <JOB> n'a pas d'attributs. Même si Adlib prend en charge plusieurs travaux dans un seul bon, ceci n'est pas utile avec la conversion WebReports. Chaque document converti aura son propre bon de travail.

    5. Doit contenir une balise <JOB:DOCINPUTS> avec une balise <JOB:DOCINPUT ... /> imbriquée dans celle-ci. On y trouve des informations sur l'emplacement où Adlib peut trouver le fichier à convertir.

    6. Doit contenir une balise <JOB:DOCOUTPUTS> avec une balise <JOB:DOCOUTPUT ... /> imbriquée dans celle-ci. On y trouve des informations pour qu'Adlib définisse où placer le fichier converti.

    7. Même si plusieurs attributs peuvent être ajoutés aux balises <JOB:DOCINPUT ... /> et <JOB:DOCOUTPUT ... />, chacune d'entre elles doit contenir l'attribut FOLDER pour être valide.

    Exemple

    <?xml version="1.0" encoding="ISO-8859-1" ?>
<?AdLibeXpress applanguage="USA" appversion="2.7.0" dtdversion="1.8" ?>
<!DOCTYPE JOBS SYSTEM "C:\AdLib eXpress\DTD\AdLibeXpress.dtd">
   <JOBS xmlns:JOBS="http://www.adlibsys.com" xmlns:JOB="http://www.adlibsys.com">
      <JOB>
         <JOB:DOCINPUTS>
            <JOB:DOCINPUT FILENAME="" FOLDER="$DEFAULT_PATH" />
         </JOB:DOCINPUTS>
         <JOB:DOCOUTPUTS>
            <JOB:DOCOUTPUT FILENAME="" FOLDER="$DEFAULT_PATH" DOCTYPE="PDF" />
         </JOB:DOCOUTPUTS>
      </JOB>
   </JOBS>

  2. Tout attribut FILENAME trouvé dans les balises <JOB:DOCINPUT ... /> et <JOB:DOCOUTPUT ... /> sera remplacé par un nom de fichier généré de façon dynamique ; l'attribut FILENAME ; bien qu'il soit facultatif, n'est donc pas nécessaire et n'a aucune incidence sur le traitement du fichier à convertir.

  3. L'attribut FOLDER doit contenir un chemin d'accès valide pour que les fichiers puissent être traités et fournis correctement. La seule exception lors de l'utilisation d'un chemin d'accès valide est l'utilisation de la nouvelle valeur $DEFAULT_PATH. Pour plus d'informations sur la valeur $DEFAULT_PATH, voir $DEFAULT_PATH

  4. La création d'un bon de travail XML prend en charge l'utilisation des balises WebReports à l'intérieur du fichier de bon de travail. La documentation Adlib Express vous donne accès à plus d'informations sur les fonctions Bon de travail, étant donné que la disponibilité d'une fonction dépend de la version d'Adlib Express que vous utilisez.
