Cette section apporte des informations sur les sous-balises suivantes : WFACTION, WFATTR, WFCOMMENTS, WFDBLOOKUP, WFFORM et WFINFO.

WFACTION

Syntaxe

WFACTION:<opération>

Description

Cette sous-balise fournit des fonctions puissantes qui permettent de manipuler les workflows dans d'importants volumes sur le serveur. Les utilisateurs peuvent seulement manipuler les workflows présentant le même niveau d'autorisations dans l'interface utilisateur standard.

Paramètres

Action
Description
WFACTION:ARCHIVE
Modifie le statut Stopped ou Completed d'un workflow en Archived.
WFACTION:DELETE
Supprime un workflow Archived ou Completed.
WFACTION:RESUME
Modifie le statut Suspended d'un workflow en Executing.
WFACTION:STOP
Modifie le statut Executing ou Executing d'un workflow en Stopped.
WFACTION:SUSPEND
Modifie le statut Executing d'un workflow en Suspended.

Exemple

[LL_REPTAG=SUBWORKID WFACTION:SUSPEND /] modifie le statut Executing d'un workflow, identifié par la colonne SUBWORKID renvoyée à partir de la source de données, en Suspended.

Informations associées

Voir également NODEACTION:opération et USERACTION:opération.

WFATTR

Syntaxe

WFATTR:<format>

Description

Pour un workflow Content Server valide, SUBWORKID, la sous-balise WFATTR renvoie les informations d'attribut correspondantes. Si l'argument :USECACHE est ajouté à la fin de la balise, le workflow subworkid est ignoré et les données de workflow sont chargées à partir de la mémoire cache. Cela peut être utile si vous souhaitez afficher les données d'attribut dans un formulaire PowerView dans l'étape Départ du workflow.

Paramètres

Format
Description
WFATTR
Si aucun paramètre facultatif n'est défini, cette sous-balise renvoie Vrai ou Faux en fonction de la présence du lot de paramètres dans le workflow.
WFATTR:<nom_attr>
Renvoie Vrai si l'attribut de workflow nommé comporte une valeur non vide ; renvoie Faux si tel n'est pas le cas.
WFATTR:<nom_attr>:DISPLAY
Affiche la valeur dans l'attribut de workflow nommé. Si l'attribut est un attribut à valeurs multiples, toutes les valeurs sont affichées au format séparé par une virgule.
WFATTR:<nom_attr>:ID
Affiche l'ID de l'attribut nommé.
WFATTR:<nom_attr>:COUNT
Renvoie le nombre d'entrées renseignées dans un attribut de workflow à valeurs multiples.
WFATTR:<nom_attr>:<val_attr>
Renvoie Vrai si la valeur spécifiée se trouve dans l'attribut de workflow ; renvoie Faux si tel n'est pas le cas.
WFATTR:<nom_attr>:<n>
Renvoie Vrai si une valeur se trouve dans le <n>ème élément d'un attribut de workflow à valeurs multiples ; renvoie Faux si tel n'est pas le cas.
WFATTR:<nom_attr>:<n>:DISPLAY
Affiche la <n>ème valeur dans l'attribut de workflow à valeurs multiples nommé.
WFATTR:<nom_attr>:<n>:<val_attr>
Renvoie Vrai si la valeur <val_attr> se trouve à l'index <n> dans un attribut de workflow à valeurs multiples ; renvoie Faux si tel n'est pas le cas.
WFATTR:<nom_attr>:INDEX:<val_attr>
Renvoie l'index de la valeur spécifiée dans un attribut de workflow à valeurs multiples.

Exemple

[LL_REPTAG_'' WFATTR:<nom_attr>:DISPLAY:USECACHE /] affiche la valeur de l'attribut <nom_attr> dans un objet PowerView si le workflow n'a pas encore été lancé.

Informations associées

Voir également NODEACTION et USERACTION.

WFCOMMENTS

Syntaxe

WFCOMMENTS:<format>

Description

Pour un workflow Content Server valide, SUBWORKID, la sous-balise WFATTR renvoie les données de commentaire dans une structure RecArray.

Paramètres

Format
Description
WFCOMMENTS
Si aucun paramètre facultatif n'est défini, cette sous-balise renvoie toutes les données de commentaire du workflow.
WFCOMMENTS:<taskID>
Renvoie les données de commentaire pour l'étape de workflow correspondant à <taskID>.

WFDBLOOKUP

Syntaxe

WFDBLOOKUP:<réf table>:<réf colonne>WFDBLOOKUP:<réf table>:<réf colonne>:<sélect colonne>WFDBLOOKUP:<réf table>:<réf colonne>:<sélect colonne>:MULTIMATCH

Description

Cette sous-balise sert à rechercher des données dans une table de base de données créée avec le module WebForms de Content Server. Elle attend que la balise de données envoie une clé valide et accepte les paramètres pour indiquer la table et la colonne correspondantes à référencer. Les informations sur la clé, la table et la colonne servent ensuite à rechercher une ou plusieurs lignes dans la table WebForms spécifiée. Un autre paramètre peut également être inclus pour désigner une colonne de données à renvoyer. Si plusieurs colonnes sont renvoyées, les données apparaissent dans une structure RECARRAY OSCRIPT, la structure de données Content Server propriétaire. Cette structure RECARRAY peut être déréférencée à l'aide de la sous-balise RECARRAY ou convertie au format JSON à l'aide de la sous-balise TOJSON. Si une seule correspondance est trouvée, la structure RECARRAY comporte un seul enregistrement, mais si plusieurs lignes de données sont renvoyées, plusieurs enregistrements sont affichés dans la structure RECARRAY. Le paramètre <réf table> peut correspondre à un ID de modèle de formulaire ou à un ID de formulaire, qui permet de déterminer le nom correct de la table SQL. Notez que l'utilisateur qui exécute le WebReport doit disposer du niveau d'autorisations approprié pour pouvoir Voir le contenu des objets du formulaire/modèle. L'identifiant <réf colonne> indique le nom d'une colonne dans laquelle la clé est recherchée. Le paramètre facultatif <sélect colonne> sert à restreindre les données renvoyées par la requête dans une seule colonne.

Normalement, si une seule colonne est renvoyée, cette sous-balise renvoie uniquement le dernier élément ajouté dans un champ de texte unique, autrement dit, pas dans une structure RECARRAY. Toutefois, l'option MULTIMATCH permet de renvoyer plusieurs lignes de données, même si une seule colonne a été sélectionnée. Si aucune colonne n'a été sélectionnée, les données sont toujours renvoyées dans une structure Recarray.

Cette structure peut être déréférencée à l'aide de la sous-balise RECARRAY et/ou la sous-balise TOJSON peut servir à convertir la structure au format JSON ; le client peut ensuite l'utiliser pour créer des variables JavaScript. Les différentes méthodes de renvoi des données sont décrites ci-dessous. En règle générale, une structure RECARRAY est toujours renvoyée, sauf si sélect colonne a été spécifié sans option MULTIMATCH, ce qui génère un champ de texte unique.

  • Si l'option sélect colonne n'a pas été spécifiée, toutes les colonnes de la source de données sont renvoyées en fonction du nombre de lignes correspondant à la requête.

  • Si l'option sélect colonne a été spécifiée, seules les données de la colonne indiquée dans la première ligne correspondante sont renvoyées.

  • Si l'option sélect colonne a été spécifiée avec l'option MULTIMATCH, seule la colonne indiquée est renvoyée pour les lignes correspondantes. Cette option permet de rechercher dans plusieurs lignes de données, mais les données sont toujours renvoyées dans une structure RECARRAY, même si une seule ligne est trouvée.

WFFORM

Syntaxe

WFFORM:<params>

Description

Renvoie des informations sur les champs de formulaire associés à un workflow, SUBWORKID. Cette sous-balise prend en charge les ensembles et les champs à valeurs multiples à l'aide des opérateurs MV et SET.

Paramètres

Syntaxe
Description
WFFORM:<nom_formulaire>:<nom_champ>
Renvoie Vrai si le champ, <nom_champ>, comporte une valeur non vide dans le formulaire, <nom_formulaire> ; renvoie Faux si tel n'est pas le cas. Si un ensemble est spécifié dans <nom_champ>, une seule entrée dans l'une des valeurs de champ de l'ensemble entraîne le renvoi de la valeur Vrai. Si <nom_champ> spécifie un champ à valeurs multiples, il suffit d'une seule entrée parmi ces valeurs pour que cette sous-balise renvoie Vrai. Le paramètre <nom_champ> peut également être imbriqué dans un ensemble de champs, sous forme de champ standard ou de champ à valeurs multiples.
WFFORM:<nom_formulaire>:<nom_champ>:DISPLAY
Renvoie la valeur du champ, <nom_champ>, pour le formulaire, <nom_formulaire>. Si <nom_champ> est un ensemble de champs ou un champ à valeurs multiples, une liste de valeurs séparées par des virgules est renvoyée.
WFFORM:<nom_formulaire>:<nom_champ>:ID
Renvoie l'ID du champ, <nom_champ>, pour le formulaire <nom_formulaire>, s'il est présent. Si <nom_champ> se trouve dans un ensemble de champs, il est renvoyé si aucun champ de nom <nom_champ> n'est présent au niveau supérieur du formulaire. Si le champ <nom_champ> est présent au niveau supérieur du formulaire et dans un ensemble de champs du même formulaire, l'ID de champ au niveau supérieur, hors de l'ensemble, a la priorité.
WFFORM:<nom_formulaire>:<nom_champ>:<val_champ>
Renvoie Vrai si un champ, <nom_champ>, comporte une valeur égale à <val_champ> dans le formulaire <nom_formulaire> ; renvoie Faux si tel n'est pas le cas. <nom_champ> peut être un champ d'un ensemble, y compris un champ à valeurs multiples, ou tout élément d'un champ à valeurs multiples.
WFFORM:<nom_formulaire>:MV:<nom_champ>:COUNT
Renvoie le nombre de champs à valeurs multiples utilisés.
WFFORM:<nom_formulaire>:MV:<nom_champ>:<n>
Renvoie Vrai ou Faux en fonction de la présence d'un élément non vide spécifique dans un champ à valeurs multiples, <nom_champ>, pour le formulaire <nom_formulaire> à l'emplacement d'index <n>. La valeur de <n> doit être un entier positif compris entre 1 et <n> où <n> ne dépasse pas la longueur du champ à valeurs multiples. Si <n> dépasse la longueur du champ à valeurs multiples, une chaîne vide est renvoyée.
WFFORM:<nom_formulaire>:MV:<nom_champ>:<n>:<val_champ>
Renvoie Vrai ou Faux en fonction de la présence de <val_champ> dans un champ à valeurs multiples, <nom_champ>, pour le formulaire <nom_formulaire> à l'emplacement d'index <n>. La valeur de <n> doit être un entier positif compris entre 1 et <n> où <n> ne dépasse pas la longueur du champ à valeurs multiples.
WFFORM:<nom_formulaire>:MV:<nom_champ>:<n>:DISPLAY
Renvoie la valeur présente dans le champ à valeurs multiples, <nom_champ>, pour le formulaire <nom_formulaire> à l'emplacement d'index <n> où <n> est un entier positif compris entre 1 et <n>. La valeur de <n> doit être un entier positif compris entre 1 et <n> où <n> ne dépasse pas la longueur du champ à valeurs multiples.
WFFORM:<nom_formulaire>:MV:<nom_champ>:INDEX:<val_champ>
Renvoie le premier index du champ à valeurs multiples, <val_champ>, pour le formulaire <nom_formulaire> où <val_champ> est égal à la valeur contenue dans le champ à valeurs multiples au niveau de l'index. Par exemple, si <val_champ> se trouve dans le troisième élément de <nom_champ>, le chiffre 3 est renvoyé.
WFFORM:<nom_formulaire>:SET:<nom_ensemble>:<nom_champ>
Utilise l'opérateur SET permettant de référencer explicitement des ensembles de champs. Cet exemple renvoie Vrai si le champ, <nom_champ>, de l'ensemble, <nom_ensemble>, n'est pas vide. Seuls les champs présents dans l'ensemble <nom_ensemble> peuvent être référencés ; si <nom_champ> existe en dehors de l'ensemble, il n'est pas trouvé. Si <nom_champ> est un champ à valeurs multiples, la valeur Vrai est renvoyée si l'un des champs n'est pas vide.
WFFORM:<nom_formulaire>:SET:<nom_ensemble>:<nom_champ>:DISPLAY
Renvoie la valeur du champ <nom_champ>, pour le formulaire <nom_formulaire>, si <nom_champ> est un champ de l'ensemble <nom_ensemble>. Si <nom_champ> est un champ à valeurs multiples, une liste des valeurs définies séparées par des virgules est renvoyée. En l'absence de valeurs définies, soit dans un champ à valeurs multiples, soit dans un champ standard, une liste vide est renvoyée.
WFFORM:<nom_formulaire>:SET:<nom_ensemble>:<nom_champ>:ID
Renvoie l'ID du champ <nom_champ>, dans l'ensemble, <nom_ensemble>. Si <nom_champ> existe en dehors de l'ensemble <nom_ensemble>, il n'est pas renvoyé. <nom_champ> peut être un champ standard ou à valeurs multiples.
WFFORM:<nom_formulaire> :SET:<nom_ensemble>:<nom_champ>:<val_champ>
Renvoie Vrai si un champ, <nom_champ>, comporte une valeur égale à <val_champ> dans l'ensemble <nom_ensemble> ; renvoie Faux si tel n'est pas le cas. Si le champ <nom_champ> est un champ à valeurs multiples, chaque élément est comparé par rapport à <val_champ>, et la valeur TRUE est renvoyée si l'un d'entre eux est égal à cette valeur.
WFFORM:<nom_formulaire>:SET:<nom_ensemble>:MV:<nom_champ>:COUNT
Renvoie le nombre de champs à valeurs multiples renseignés.
WFFORM:<nom_formulaire>:SET:<nom_ensemble>:MV:<nom_champ>:<n>
Renvoie Vrai ou Faux en fonction de la présence d'un élément non vide spécifique dans un champ à valeurs multiples, <nom_champ>, dans un ensemble pour le formulaire <nom_formulaire> à l'emplacement d'index <n>. La valeur de <n> doit être un entier positif compris entre 1 et <n> où <n> ne dépasse pas la longueur du champ à valeurs multiples.
WFFORM:<nom_formulaire>:SET:<nom_ensemble>:MV:<nom_champ>:<n>:<val_champ>
Renvoie Vrai ou Faux en fonction de la présence de <val_champ> dans un champ à valeurs multiples, <nom_champ>, dans un ensemble pour le formulaire <nom_formulaire> à l'emplacement d'index <n>. La valeur de <n> doit être un entier positif compris entre 1 et <n> où <n> ne dépasse pas la longueur du champ à valeurs multiples.
WFFORM:<nom_formulaire>:SET:<nom_ensemble>:MV:<nom_champ>:<n>:DISPLAY
Renvoie la valeur présente dans le champ à valeurs multiples, <nom_champ>, dans un ensemble pour le formulaire <nom_formulaire> à l'emplacement d'index <n> où <n> est un entier positif compris entre 1 et <n>. La valeur de <n> doit être un entier positif compris entre 1 et <n> où <n> ne dépasse pas la longueur du champ à valeurs multiples.
WFFORM:<nom_formulaire> :SET:<nom_ensemble>:MV:<nom_champ> :INDEX:<val_champ>
Renvoie le premier index du champ à valeurs multiples, <nom_champ>, dans un ensemble pour le formulaire <nom_formulaire> où <val_champ> est égal à la valeur contenue dans le champ à valeurs multiples au niveau de l'index.
WFFORM:<nom_formulaire>:MVSET:<nom_ensemble>:<n>:<nom_champ>
Renvoie Vrai si l'ensemble à valeurs multiples, <nom_ensemble>, comporte un champ non vide, <nom_champ>, à l'emplacement d'index <n>, dans le formulaire <nom_formulaire> ; renvoie Faux si tel n'est pas le cas.
WFFORM:<nom_formulaire>:MVSET:<nom_ensemble>:<n>:<nom_champ>:DISPLAY
Affiche la valeur d'un champ <nom_champ> dans un ensemble à valeurs multiples, <nom_ensemble>, à l'emplacement d'index <n>, dans le formulaire <nom_formulaire>.
WFFORM:<nom_formulaire>:MVSET:<nom_ensemble>:<n>:<nom_champ>:ID
Affiche l'ID d'un champ <nom_champ> dans un ensemble à valeurs multiples, <nom_ensemble>, à l'emplacement d'index <n>, dans le formulaire <nom_formulaire>.
WFFORM:<nom_formulaire>:MVSET:<nom_ensemble>:<n>:<nom_champ>:<val_champ>
Renvoie Vrai si l'ensemble à valeurs multiples, <nom_ensemble>, comporte un champ, <nom_champ>, à l'emplacement d'index <n>, dans le formulaire <nom_formulaire> qui est égal à la valeur spécifiée dans <val_champ> ; renvoie Faux si tel n'est pas le cas.
WFFORM:<nom_formulaire>:MVSET:<nom_ensemble>:<n>:MV:<nom_champ>:COUNT
Renvoie le nombre de champs à valeurs multiples utilisés dans l'ensemble à valeurs multiples.
WFFORM:<nom_formulaire>:MVSET:<nom_ensemble>:<n>:MV:<nom_champ>:<m>
Renvoie Vrai si l'ensemble à valeurs multiples, <nom_ensemble>, à l'emplacement d'index <n> comporte un champ à valeurs multiples, <nom_champ>, à l'emplacement d'index <m>, qui est non vide ; renvoie Faux si tel n'est pas le cas.
WFFORM:<nom_formulaire>:MVSET:<nom_ensemble>:<n>:MV:<nom_champ>:<m>:DISPLAY
Affiche la valeur d'un champ à valeurs multiples <nom_champ>, à l'emplacement d'index <m>, dans un ensemble à valeurs multiples <nom_ensemble> à l'emplacement d'index <n>, dans le formulaire <nom_formulaire>.
WFFORM:<nom_formulaire>:MVSET:<nom_ensemble>:<n>:MV:<nom_champ>:<m>:<val_champ>
Renvoie Vrai si l'ensemble à valeurs multiples, <nom_ensemble>, à l'emplacement d'index <n> comporte un champ à valeurs multiples, <nom_champ>, à l'emplacement d'index <m>, dont la valeur est égale à celle spécifiée dans <val_champ> ; renvoie Faux si tel n'est pas le cas.
WFFORM:<nom_formulaire>:MVSET:<nom_ensemble>:<n>:MV:<nom_champ>:INDEX:<val_champ>
Renvoie le premier index du champ à valeurs multiples, <nom_champ>, dans un ensemble à valeurs multiples pour le formulaire <nom_formulaire> où <val_champ> est égal à la valeur contenue dans le champ à valeurs multiples au niveau de l'index.

Exemple

[LL_REPTAG=SUBWORKID WFFORM:<nom_formulaire>:<nom_champ>:DISPLAY /] renvoie la valeur de champ stockée dans le formulaire <nom_formulaire> et le champ <nom_champ> pour tout type de champ. Si <nom_champ> désigne un champ à valeurs multiples ou un ensemble, une liste de valeurs séparées par des virgules est renvoyée.

WFINFO

Syntaxe

WFINFO:<format>

Description

Pour un workflow Content Server valide, SUBWORKID, cette sous-balise renvoie les informations associées au workflow en cours d'exécution.

Paramètres

Format
Description
ATTACHFOLDER
Renvoie l'identifiant objectid du dossier Pièces jointes associé au workflow en cours d'exécution.
FLAGS
Indicateurs.
DATEDUE_MIN
Date d'échéance minimum.
DATEDUE_MAX
Date d'échéance maximum.
DATEINITIATED
Date de début du workflow.
DATECOMPLETED
Date d'achèvement du workflow.
OWNERID
ID utilisateur du propriétaire du workflow.
MANAGERID
ID utilisateur du responsable du workflow.
STATUS
Statut actuel du workflow. Voir également la sous-balise STATUS:paramètre, avec l'option WORK.
TITLE
Titre du workflow.
PROJECT
Projet.
USERDATA
Données utilisateur.
CUSTOMDATA
Données personnalisées.
OWNERPERMS
Autorisations du propriétaire.
