Cette méthode requiert une certaine connaissance du stockage des catégories et attributs Content Server dans la base de données. Il existe deux tables clés, LLAttrData et LLAttrBlobData, qui présentent chacune leurs propres difficultés pour la production de rapports. LLAttrData comporte souvent des dizaines de millions de lignes et LLAttrBlobData contient des structures de données internes à Content Server, tels que des Assoc et des RecArray.

Voici certaines colonnes LLAttrData importantes :

Colonnes LLAttrData
Description
ID
DATAID de l'élément auquel cette catégorie est appliquée.
VERNUMD
Numéro de version de l'élément auquel cette catégorie est appliquée.
DEFID
DATAID de la catégorie appliquée à l'élément.
DEFVERN
Version de la catégorie appliquée.
ATTRID
ID de l'attribut.
ATTRTYPE
Type de donnée de cet attribut.
VALINT
Données propres au type pour les entiers, utilisateurs et booléens.
VALREAL
Données propres au type pour les nombres réels.
VALDATE
Données propres au type pour les dates.
VALSTR
Données propres au type pour les chaînes.
