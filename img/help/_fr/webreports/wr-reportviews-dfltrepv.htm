Lorsque l'utilisateur ajoute un nouvel objet WebReport à Content Server, il peut sélectionner un affichage de rapport par défaut. Les affichages de rapports par défaut fournissent différents points de départ génériques qui peuvent servir à développer un WebReport. Certains affichages de rapports peuvent se révéler immédiatement utiles, mais dans d'autres cas, il est nécessaire de les modifier en fonction d'exigences spécifiques. Lorsqu'un WebReport a été créé à l'aide d'un affichage de rapport par défaut, l'utilisateur peut modifier l'affichage de rapport en ligne ou le télécharger à des fins de modification sur son bureau. Ces affichages de rapports par défaut sont fournis lorsque le module WebReports est installé ; toutefois, un administrateur Content Server peut les modifier pour intégrer l'apparence d'une entreprise.

Note :
Seuls les affichages de rapports contenus dans le dossier defaultreportviews peuvent être sélectionnés lorsqu'un utilisateur crée un WebReport. L'administrateur peut modifier les affichages de rapports présents dans ce dossier. A titre d'exemple, vous pouvez déplacer un ou tous les affichages de rapports du dossier examplereportviews dans le dossier defaultreportviews afin qu'ils soient tous disponibles dans l'écran Ajouter un nouveau WebReport.

Le champ d'emplacement dans le tableau ci-dessus fait référence à un dossier de Content Server. Il est accessible via <install path>\module\webreports_10_x_x\ et disponible uniquement pour l'administrateur de Content Server.
