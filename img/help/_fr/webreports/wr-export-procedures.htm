En plus d'être affichée dans le navigateur, la sortie générée lors de l'exécution d'un WebReport peut être exportée vers les emplacements suivants :

  • Content Server

    L'exportation vers Content Server lui-même n'est possible que si vous êtes l'utilisateur administrateur.

  • Content Server - Nouveau document

  • Content Server - Version

  • Content Server - Workflow

  • Content Server - WebForm

  • Bureau

  • Adresse e-mail

Vous pouvez accéder à la fonction d'exportation des façons suivantes :

  • en sélectionnant Exporter à partir du menu Fonctions, ou

  • en définissant le comportement par défaut du rapport sur Exporter. Pour cela, sélectionnez Propriétés dans le menu Fonctions, puis Destination.

Lorsque vous exportez vers Content Server ou un e-mail, vous pouvez séparer le navigateur web du rapport, pour permettre à l'utilisateur de poursuivre son travail pendant que des rapports volumineux sont exécutés en arrière-plan. Pour définir ce comportement, sélectionnez la case à cocher Exécuter en arrière-plan.

Configuration du comportement par défaut d'un WebReport

Vous pouvez modifier l'action par défaut générée par la sélection d'un WebReport dans Content Server. Ainsi, au lieu d'exécuter le rapport et d'afficher la sortie dans le navigateur, la sortie est redirigée vers une opération d'exportation. Par exemple, un WebReport pourrait être configuré de façon à ajouter une nouvelle version à un document Content Server chaque fois que ce WebReport est sélectionné. Ce comportement est spécifié dans l'onglet Destination, accessible à partir de menu Fonctions du WebReport en sélectionnant Propriétés puis Destination. Les champs utilisés pour spécifier ce comportement par défaut sont identiques à ceux indiqués pour les exportations explicites comme décrit dans les pages suivantes.
