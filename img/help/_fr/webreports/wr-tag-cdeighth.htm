Cette section apporte des informations sur les balises de données de colonne : MYCACHEURL, MYID et MYURL.

MYCACHEURL

Syntaxe

[LL_REPTAG_MYCACHEURL /]

Description

Renvoie une URL pour le WebReport en cours d'exécution, mais tout paramètre d'URL qui serait normalement inclus dans l'URL est placé dans la mémoire cache et l'ID de la mémoire cache est ajouté à l'URL à la place. L'ID de la mémoire cache est ajouté à l'URL au format : &reqcacheid=<nnnnnnnn>

Si cette URL est utilisée pour exécuter une requête vers Content Server, tous les paramètres mis en mémoire cache sont ajoutés à la requête précédente de tout traitement WebReports.

MYID

Syntaxe

[LL_REPTAG_MYID /]

Description

Renvoie l'ObjectID du WebReport en cours d'exécution.

MYURL

Syntaxe

[LL_REPTAG_MYURL /]

Description

Renvoie l'URL qui a été utilisée pour exécuter le WebReport en cours d'exécution. Conjointement avec la sous-balise ESCAPEURL, cette balise peut être utilisée en ajoutant un lien hypertexte à un autre élément Content Server pour permettre une nouvelle exécution du WebReport d'origine. Lorsque cette balise est utilisée pour spécifier un paramètre nexturl, la sous-balise ESCAPEURL doit être utilisée pour générer le bon format d'intégration d'URL correct.

Exemple

La dernière partie d'un lien hypertexte peut inclure : nexturl=[LL_REPTAG_MYURL ESCAPEURL /]
