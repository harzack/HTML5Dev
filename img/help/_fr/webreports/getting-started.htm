Le module WebReports fournit un nouveau type de nœud Content Server appelé WebReport qui peut être ajouté à un espace de travail Content Server à partir du menu déroulant Ajouter un nouvel élément. Pour créer un nouveau WebReport, l'utilisateur est invité à :

  1. Choisir une source de données sur laquelle reposera le WebReport.

  2. Sélectionner un affichage de rapport à utiliser en tant que modèle de présentation pour la source de données.

  3. Un WebReport présente toutes les propriétés standard d'autres nœuds Content Server, comme un nœud de document, mais il inclut également des fonctions appropriées au rapport. Par exemple, Modifier l'affichage du rapport.

Présentation des procédures WebReports

Procédures destinées aux utilisateurs finaux WebReports : Pour les utilisateurs finaux de WebReports, il existe des éléments importants relatifs à l'exécution de WebReports pour afficher ou exporter des données. Les éléments les plus utiles pour les utilisateurs du WebReport sont :

  • Exportation des données du rapport

  • Exécution de WebReports

  • Définition de programmes pour WebReports

Procédures destinées aux développeurs WebReports : Pour les développeurs WebReports certaines des procédures les plus utiles comprennent :

  • Création d'un WebReport

  • Modification d'un affichage de rapport hors ligne

  • Modification d'un affichage de rapport en ligne

  • Configuration de la source de données

  • Définition de programmes pour WebReports

  • Constantes WebReports

  • Utilisation des paramètres

  • Balises d'affichage de rapport

  • Informations avancées sur WebReports

  • Définition des autorisations WebReports
