WebReports fournit un éditeur en ligne permettant de modifier rapidement et facilement l'affichage de rapport. Cet éditeur est utilisé pour mettre en forme le WebReport et insérer les balises sélectionnées WebReports dans le code. Si vous venez de créer un WebReport à l'aide de l'affichage de rapport par défaut, vous souhaitez peut-être utiliser l'éditeur afin de modifier le rapport.

L'option Modifier l'affichage du rapport dans le menu Fonctions du WebReport ouvre l'éditeur d'affichage de rapport. L'éditeur présente le fichier d'affichage de rapport en trois sections : l'en-tête, le modèle de ligne et le pied de page. Voir la capture d'écran ci-dessous.

Après avoir effectué les modifications souhaitées à l'affichage de rapport, un clic sur Ajouter une version ou sur Ajouter une version et continuer génère une nouvelle version de l'affichage de rapport en utilisant le texte nouvellement modifié. Le nouvel affichage de rapport sera utilisé lors de la prochaine exécution du WebReport.

L'écran Modifier l'affichage du rapport :

  1. Les boutons Ajouter une version, Ajouter une version et continuer, Réinitialiser et Annuler sont affichés en haut et en bas de ce formulaire. Ajouter une version permet l'envoi des informations révisées à Content Server en tant que nouvelles versions et redirige l'utilisateur sur l'écran précédent. Ajouter une version et continuer permet également l'envoi des informations révisées à Content Server en tant que nouvelles versions et donne à l'utilisateur la possibilité de continuer à travailler sur l'affichage de rapport. Le bouton Réinitialiser supprime les modifications effectuées depuis l'ouverture initiale du formulaire. Annuler fait simplement disparaître le formulaire sans enregistrer une nouvelle version.

  2. Section en-tête : cette section est utilisée pour communiquer tout code devant être exécuté au début du WebReport. Elle est générée une seule fois, quel que soit le nombre de lignes de rapport qui sont lues à partir de la source de données sous-jacente. Toutes les balises d'ouverture comme <TABLE> doivent figurer dans cette section. Cependant, il n'est pas nécessaire d'inclure <HTML> ou <BODY> si l'en-tête Content Server standard est inclus. L'en-tête Content Server est inclus par défaut, sauf si la balise [LL_WEBREPORT_EXCLUDEHEADER /] est utilisée.

  3. [LL_WEBREPORT_STARTROW /] : cette ligne ne peut pas être modifiée. Elle est fournie uniquement pour indiquer l'endroit où la balise de début de ligne sera insérée afin que les utilisateurs n'essayent pas d'en ajouter une autre.

  4. Section ligne : cette section est utilisée pour définir tout code devant être appliqué à toutes les lignes de données. Toutes les balises utilisées pour désigner les données de la colonne de source de données ne fonctionneront que dans cette section. Il est important de noter que tout dans cette section sera répété pour chaque ligne de la source de données. Par exemple, si 10 lignes de données du rapport doivent être renvoyées, tout dans cette section apparaîtra 10 fois dans la sortie du WebReport. La seule exception est l'utilisation de balises conditionnelles pour exclure certaines données de la sortie. Pour en savoir plus, voir Balises d'affichage de rapport.

  5. [LL_WEBREPORT_ENDROW /] : cette ligne ne peut pas être modifiée. Elle est fournie uniquement pour indiquer l'endroit où la balise de fin de ligne sera insérée afin que les utilisateurs n'essayent pas d'en ajouter une autre.

  6. Section pied de page : cette section est utilisée pour communiquer tout code devant être exécuté à la fin du WebReport. Elle est générée une seule fois, quel que soit le nombre de lignes de rapport qui sont lues à partir de la source de données sous-jacente. Toutes les balises de fermeture comme </TABLE> doivent figurer dans cette section.
