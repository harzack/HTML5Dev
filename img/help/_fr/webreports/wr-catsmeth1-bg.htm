La sous-balise CAT peut être considérée comme une macro WebReports. Elle opère sur un identificateur d'objet Content Server et peut récupérer toute donnée de catégorie et d'attribut stockée relativement à l'objet correspondant. Ces données peuvent ensuite être utilisées en sortie ou former la base d'un filtrage ou d'un tri. Nous nous intéressons ici uniquement à l'affichage de ces données. Avec CAT, nous avons également besoin d'une source de données. Dans ce cas, nous utilisons un LiveReport. 

select LLAttrData.ID DATAID
	from
		LLAttrData, DTree
	where
		LLAttrData.DEFID = 338223 and
		LLAttrData.ATTRID = 1 and
		LLAttrData.VERNUM = DTree.VERSIONNUM and
		LLAttrData.ID = DTree.DATAID

Cette simple requête renvoie une colonne unique de dataids Content Server auxquels la catégorie 338223 a été appliquée, à leur version la plus récente. Evidemment, tout critère de filtre peut être utilisé ; il est important ici que nous disposions d'une liste de dataids dont nous souhaitons voir les informations de catégories. Cela étant obtenu ci-dessus, nous pouvons créer un WebReport pour extraire des informations de catégories : 

<TABLE CELLSPACING="1" STYLE="font-size:11px" BORDER="0">

	<TD> Assignee </TD>
	<TD> Start Date </TD>
	<TD> End Date </TD>
	<TD> Complete </TD>
</TR>
[LL_WEBREPORT_STARTROW /]
<TR BGCOLOR="ffffff">
	<TD> [LL_REPTAG=DATAID CAT:"Test Category":"Trial Name":DISPLAY /]</TD>
	<TD> [LL_REPTAG=DATAID CAT:"Test Category":"Number of Failures":DISPLAY /]</TD>
	<TD> [LL_REPTAG=DATAID CAT:"Test Category":"Assignee":DISPLAY /]</TD>
	<TD> [LL_REPTAG=DATAID CAT:"Test Category":"Start Date":DISPLAY /]</TD>
	<TD> [LL_REPTAG=DATAID CAT:"Test Category":"End Date":DISPLAY /]</TD>
	<TD> [LL_REPTAG=DATAID CAT:"Test Category":"Complete":DISPLAY /]</TD>
</TR>
[LL_WEBREPORT_ENDROW /]
</TABLE>

Cet affichage de rapport est conçu pour être aussi simple que possible. Il utilise la colonne DATAID renvoyée par la source de données afin d'extraire chaque attribut souhaité pour la catégorie Catégorie de test. Les résultats sont insérés dans un tableau HTML qui ressemble à une sortie LiveReport, bien que nous puissions utiliser tout type de formatage.

Notez que la valeur renvoyée pour Achevé est TRUE, car c'est ainsi que l'utilisateur voit cette valeur dans la catégorie. Cela est différent des résultats de la méthode SQL, car la base de données stocke les valeurs sous forme de 0 ou 1 et Content Server convertit cela en TRUE ou FALSE pour affichage dans le type d'attribut de case à cocher.

Selon la taille du système Content Server et son utilisation réelle, les étapes ci-dessus peuvent suffire pour fournir un rapport terminé. Toutefois, s'il est nécessaire d'améliorer les performances, consultez la section Mise en cache de résultats avec WebReports.
