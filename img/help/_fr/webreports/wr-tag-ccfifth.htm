Cette section apporte des informations sur les balises de contrôle de contenu : IF, INCLUDEDISTINCT, INCLUDEIF, INCLUDERANGE, INSERTJSARRAY et INSERTJSON.

IF

Syntaxe

[LL_WEBREPORT_IF <expression> /]

[LL_WEBREPORT_ENDIF /]

Description

Lorsqu'une condition est satisfaite, inclut le contenu dans la sortie. Tous les éléments placés entre les balises IF et ENDIF seront inclus uniquement si la condition spécifiée est considérée comme vraie (c'est-à-dire satisfaite). Si une clause facultative ELSE est incluse, tous les éléments situés entre la balise ELSE et la balise ENDIF sont inclus si la condition spécifiée est considérée comme fausse.

Cette balise diffère de la balise INCLUDEIF, car sa structure permet d'inclure ou d'exclure sous condition certaines portions de la ligne. Cette balise peut être utilisée n'importe où dans l'affichage de rapport. Lorsque vous incluez ou excluez la section de ligne entière, utilisez toujours la balise INCLUDEIF au lieu de IF ; les performances seront meilleures.

Paramètres

Format pour <expression> :

"parmA" <opérateur> "parmB"

Exemple

[LL_WEBREPORT_IF (expression) /]
[LL_WEBREPORT_ELSE /] (optional)
[LL_WEBREPORT_ENDIF /]

Informations associées

Voir la section relative à Expressions logiques pour obtenir une liste des Opérateurs. Voir également les sections Remarques dans EXITIF et INCLUDEIF pour plus d'informations. Voir également Expressions logiques pour plus de détails. Pour plus d'exemples d'expression, voir Exemple.

INCLUDEDISTINCT

Syntaxe

[LL_WEBREPORT_INCLUDEDISTINCT « <clé distincte> » /]

Description

Utilisée pour limiter le nombre de lignes renvoyées par le WebReport, afin de renvoyer uniquement des valeurs de clés uniques.

La valeur « <clé distincte> » indique quelle colonne, ou balise de données WebReports, doit être unique. Dans la source de données, seules les lignes générant une valeur unique pour cette clé seront incluses. Toutes les lignes générant la même valeur pour la clé spécifiée seront rejetées.

 

  1. Les valeurs de clé distinctes sont sensibles à la casse.

  2. Lorsque des lignes présentent des clés en double, la première ligne de l'ordre en cours est conservée, et toutes les lignes suivantes sont rejetées.

  3. L'ordre de lignes utilisé pour la comparaison des éléments en double est basé sur l'ordre de la source de données d'origine, ainsi que sur les balises WebReports SORT incluses dans la section de ligne.

  4. Le filtrage des lignes effectué à l'aide de cette balise a lieu avant l'application de toute autre balise de filtrage de ligne, telle que INCLUDEIF, EXITIF ou INCLUDERANGE.

 

Paramètres

La valeur « <clé distincte> » peut correspondre à un nom de colonne, un numéro de colonne ou une combinaison balise/sous-balise WebReports.

Exemple

Considérons par exemple l'ensemble de données suivant :

DataID
NomFruit

Si la section de ligne dispose d'un filtre INCLUDEDISTINCT, tel que [LL_WEBREPORT_INCLUDEDISTINCT "FruitName" /], le résultat contiendra uniquement les lignes ayant une valeur unique dans la colonne « FruitName », comme suit :

12345

12346
Orange
12348

Le même résultat est obtenu lorsque le numéro de colonne 1 est utilisé, par exemple, [LL_WEBREPORT_INCLUDEDISTINCT "1" /]. Cette balise prend également en charge l'utilisation des combinaisons balise/sous-balise WebReports. Par exemple, si l'on reprend l'exemple de données initial et la balise [LL_WEBREPORT_INCLUDEDISTINCT "[LL_REPTAG=FruitName DECODE:Banana:apple /]" /], il reste les lignes suivantes après l'exécution du filtre :

12345
Apple
12346
Orange

Dans cet exemple, la sous-balise WebReports DECODE convertit en « apple » toutes les données correspondant à la chaîne « Banana ». Etant donné que la valeur « Apple » existe déjà dans une ligne précédente, la ligne qui contient normalement la valeur « Banana » dans cette colonne perd son caractère unique. Cette fonction est particulièrement utile lorsque les sous-balises de type NODEINFO et/ou CAT sont utilisées pour traduire des champs de dataID dans d'autres valeurs de données connexes, qui ne sont pas nécessairement uniques.

INCLUDEIF

Syntaxe

[LL_WEBREPORT_INCLUDEIF <expression> /]

Description

Utilisée pour déterminer si une ligne de données doit ou non être incluse. Si la condition est Vrai, la ligne entière est incluse ; dans le cas contraire, elle est ignorée. Il est possible d'utiliser plusieurs balises INCLUDEIF pour créer une série des barrières conditionnelles. Lorsque vous utilisez plusieurs balises INCLUDEIF, celles-ci se comportent comme si les conditions étaient cumulatives (AND) et non exclusives (OR). Ainsi, plus vous utilisez de balises INCLUDEIF, plus les restrictions appliquées au rapport seront importantes. Cette balise fonctionne uniquement dans la section de ligne. Par exemple, entre les balises [LL_WEBREPORT_STARTROW /] et [LL_WEBREPORT_ENDROW /].

 

  1. Cette balise affecte le nombre de lignes réellement renvoyées et affichées. C'est la raison pour laquelle cette balise a un impact sur la balise ROWNUM, qui ne doit pas être intégrée à la logique conditionnelle pour INCLUDEIF.

  2. Cette balise ne modifie pas la balise TOTALROWS, qui reflétera toujours le nombre de lignes réellement disponibles dans la table de base données. Pour obtenir le nombre de lignes réellement renvoyées, c'est-à-dire celles qui répondent aux critères du test conditionnel, utilisez la balise ACTUALROWS. Voir les informations spécifiques relatives à la syntaxe de ces balises.

 

Paramètres

Format pour <expression> : "parmA" <opérateur> "parmB" <clauses facultatives>

Exemple

[LL_WEBREPORT_INCLUDEIF "[LL_REPTAG_&var1 /]" != "[LL_REPTAG=DATAID /]" /]
[LL_WEBREPORT_INCLUDEIF "Admin" == "[LL_REPTAG_USERNAME /]" /]
[LL_WEBREPORT_INCLUDEIF "1234" NOTIN "[LL_REPTAG_1 /]" /]
[LL_WEBREPORT_INCLUDEIF "*DELETED*" IN "[LL_REPTAG=keyfield /]" /]
[LL_WEBREPORT_INCLUDEIF "[LL_REPTAG_1 LENGTH /]" >= "10" AND "[LL_REPTAG=dataid /]" CHILDOF "[LL_REPTAG_&rootid /]" /]

Informations associées

Voir la section Remarques dans EXITIF. Voir également Expressions logiques et Opérateurs.

INCLUDERANGE

Syntaxe

[LL_WEBREPORT_INCLUDERANGE STARTROW:<n> ENDROW:<n> MAXROWS:<n> /]

Description

Utilisée pour limiter le nombre de lignes renvoyées par le WebReport, de manière à respecter les paramètres spécifiés.

Paramètres

Utilisez au moins l'un des paramètres facultatifs suivants : STARTROW:<n> , ENDROW:<n> ou MAXROWS:<n>.

  • Si vous spécifiez uniquement STARTROW:<n>, et qu'aucun paramètre MAXROWS:<n> ou ENDROW:<n> n'est inclus, ENDROW:<n> sera considéré comme étant la dernière ligne de l'ensemble de données.

  • Si vous ne spécifiez aucun paramètre STARTROW:<n>, la première ligne de l'ensemble de données sera considérée comme le point de départ.

  • Si vous spécifiez à la fois MAXROWS:<n> et ENDROW:<n>, le plus restrictif de ces deux paramètres sera appliqué. Par exemple, si STARTROW=1, ENDROW=10 et MAXROWS=5, seules les cinq premières lignes seront renvoyées, au lieu de dix.

  • Si le paramètre STARTROW:<n> est inférieur à 1, la valeur 1 sera utilisée.

  • Si la valeur du paramètre STARTROW:<n> est supérieure au nombre total de lignes dans l'ensemble de données, aucune ligne ne sera renvoyée.

  • Si la valeur du paramètre ENDROW:<n> est inférieure à celle du paramètre STARTROW:<n>, aucune ligne ne sera renvoyée.

Exemple

La plage de lignes renvoyée est calculée après évaluation des autres filtres de ligne, tels que INCLUDEIF ou EXITIF. Examinez les ensembles de données suivants :

N° ligne source
Données

3

Si la section de ligne dispose d'un filtre INCLUDEIF, tel que [LL_WEBREPORT_INCLUDEIF "[LL_REPTAG_SOURCEROWNUM ODDEVEN /]" == "EVEN" /], seuls les numéros de ligne impairs seront renvoyés. Par exemple :

Si un filtre INCLUDERANGE est également utilisé, [LL_WEBREPORT_INCLUDERANGE STARTROW:2 MAXROWS:2 /], cet ensemble de lignes est encore réduit. Ainsi, deux lignes seulement seront renvoyées, en commençant par la ligne 2 de l'ensemble de données, renvoyée après traitement des éventuelles balises INCLUDEIF et EXITIF. En reprenant les exemples de table précédents, cet ensemble serait le suivant :

4
Banana
6
Mango

INSERTJSARRAY

Syntaxe

[LL_WEBREPORT_INSERTJSARRAY ARRAYNAME:<Nom de tableau facultatif> MULTIVALUE:<Nom de colonne> /]

Description

Utilisée pour insérer automatiquement un tableau JavaScript contenant toutes les données du rapport, organisées sous forme de tableau multidimensionnel.

Cette balise crée normalement un tableau appelé « repData », indexé numériquement par colonne et par ligne. Elle crée également un ensemble de constantes, qui permettent de référencer les colonnes. Par exemple, pour référencer la colonne dataid pour une ligne de données, vous pouvez utiliser l'élément suivant : document.write(repData[row][DATAID]);

Il est également possible d'utiliser un nom de tableau alternatif à l'aide du paramètre ARRAYNAME:. Le paramètre MULTIVALUE: peut être utilisé lorsqu'une seule colonne du tableau est unique et que les autres colonnes se répètent. Le paramètre SCRIPT: peut être utilisé pour ajouter ou supprimer les balises d'encadrement <SCRIPT> situées autour du tableau JavaScript créé, en spécifiant la valeur Vrai ou Faux.

Informations associées

Pour plus d'informations sur cette balise, voir Tableaux JavaScript.

INSERTJSON

Syntaxe

[LL_WEBREPORT_INSERTJSON <directives multiples> /]

Description

Utilisée pour insérer automatiquement des données Content Server dans la sortie, dans le format d'une structure JSON.

Paramètres

Cette balise prend en charge diverses « directives », qui spécifient les options relatives aux données et aux formats à utiliser.

@ADDJSVAR
@ALLDATASOURCE
@ALLSTATIC
@BROWSECOLUMNS
@ESCAPETEXT
@EXCLUSIVE
@FIELDS
@ROWCOLUMNS

Exemple

[LL_REPTAG_&PARM1 MODULUS:7 /] renvoie 6 si PARM1 contient « 2001 ».

Informations associées

Pour plus d'informations sur ces directives, voir Insertion de données JSON.
