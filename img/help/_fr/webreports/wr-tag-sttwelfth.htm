Cette section apporte des informations sur les sous-balises suivantes : NEXT, NODEACTION et NODEINFO.

NEXT

Syntaxe

NEXT

Description

Renvoie la même colonne à partir de la ligne de données suivante. Si vous utilisez cette sous-balise dans la dernière ligne de données, une chaîne vide est renvoyée.

NODEACTION

Syntaxe

NODEACTION:<opération>

Description

Cette sous-balise offre une puissante fonctionnalité permettant d'exécuter diverses opérations sur les nœuds Content Server ; il est donc essentiel de bien comprendre son fonctionnement et ses effets avant de l'utiliser. La sous-balise elle-même se comporte comme toutes les autres sous-balises WebReports, à ceci près que NODEACTION ne permet pas à l'utilisateur qui exécute le rapport d'effectuer une action, par exemple un déplacement, qu'il serait incapable d'exécuter depuis l'interface standard.

Tableau 1-9 : Action : COPY

Opération
Description
NODEACTION:COPY:<recueil>
Attend un dataID renvoyé par la balise principale. Copie ce nœud à l'emplacement spécifié par le paramètre recueil. Utilise le même nom pour toutes les versions.
NODEACTION:COPY:<recueil>:CURRENT
Balise identique à la précédente, excepté qu'elle utilise uniquement la dernière version la plus à jour du document source. Si l'objet source est un recueil, seule la dernière version des éléments contenus dans ce recueil sera copiée.
NODEACTION:COPY:<recueil>:INHERITATTRS:DESTINATION
Hérite des données de catégorie et d'attribut de l'emplacement de destination.
NODEACTION:COPY:<recueil>:INHERITATTRS:MERGED
Fusionne les données de catégorie et d'attribut du nœud source et de l'emplacement de destination.
NODEACTION:COPY:<recueil>:INHERITATTRS:SOURCE
Conserve les données de catégorie et d'attribut source lors de la copie vers l'emplacement de destination.
NODEACTION:COPY:<recueil>:INHERITPERMS
Hérite des autorisations du recueil de destination.
NODEACTION:COPY:<recueil>:INHERITRMOFF
Le comportement par défaut de NODEACTION est le même que si l'opération avait été effectuée via l'interface Content Server : un nœud copié hérite des classements RM du nouveau recueil parent. Cette option peut être utilisée pour stopper la transmission des classements RM. Lorsqu'un classement RM est déjà appliqué au nœud à copier, ce classement RM est conservé une fois l'événement de copie terminé. Cette option peut être utilisée pour remplacer les options « Administration de Records Management », « Objets gérés » et « Hériter à partir de », mais pas pour forcer la transmission (héritage) d'un classement RM si les paramètres RM appropriés n'ont pas été spécifiés.
NODEACTION:COPY:<recueil>:NEWNAME:<nom>
Utilisez le paramètre facultatif NEWNAME pour spécifier un nouveau nom pour le nœud de destination.
NODEACTION:COPY:<recueil>:NOCHILDREN
Copie uniquement le nœud spécifié, sans copier les nœuds enfants.
NODEACTION:COPY:<recueil>:UNIQUEONLY
Si le nom du nœud n'est pas unique au sein du recueil de destination, le nœud ne sera pas copié. Cette option peut être utilisée avec ou sans l'option NEWNAME.

Vous pouvez utiliser les paramètres de copie dans n'importe quelle combinaison et dans n'importe quel ordre. Voir Tableau 1-15 ci-dessous pour obtenir un exemple de combinaison de plusieurs paramètres.

Tableau 1-10 : Action : CREATE

Opération
Description
NODEACTION:CREATE:COMPOUNDDOC:<recueil>:<nom>
Crée un nouveau document composite dans le recueil spécifié. Les données présentes dans la balise principale sont ignorées.
NODEACTION:CREATE:DOC:<recueil>:<nom>
Attend un dataID de document dans la balise principale. Le contenu de la dernière version de ce document est utilisé comme source pour un nouveau document, créé dans le recueil spécifié.
NODEACTION:CREATE:FOLDER:<recueil>:<nom>
Crée un nouveau dossier dans le recueil spécifié. Les données présentes dans la balise principale sont ignorées.
NODEACTION:CREATE:GENERATION:<recueil>:<nom>
Crée une génération à partir de la dernière version du nœud spécifié dans la balise principale. Vous pouvez également ajouter :VERSIONNUM:<numéro de version> pour créer une génération à partir d'une version particulière du nœud. Par exemple, NODEACTION:CREATE:GENERATION:<recueil>:<nom>:VERSIONNUM:2 créera une génération à partir de la version 2 du nœud.
NODEACTION:CREATE:RELEASE
Crée une version du document composite spécifié dans la balise principale. Vous pouvez également ajouter un nom pour cette version. Par exemple, NODEACTION:CREATE:RELEASE créera une version utilisant le nom par défaut : « 1.0 », « 2.0 ». NODEACTION:CREATE:RELEASE:"Release Name" créera une version appelée « Release Name ».
NODEACTION:CREATE:SHORTCUT:<recueil>:<nom>
Crée un raccourci pointant vers le nœud spécifié dans la balise principale.
NODEACTION:CREATE:URL:<container>:<name>
Attend une URL renvoyée par la balise principale. L'URL doit être complète, par exemple, http://. Cette option crée également un nouvel objet d'URL dans le recueil spécifié.

Toutes les options NODEACTION:CREATE, excepté RELEASE et REVISION, prennent en charge INHERITATTRS, INHERITPERMS et UNIQUEONLY. INHERITPERMS se comporte de la même manière que dans NODEACTION:COPY et NODEACTION:MOVE. INHERITATTRS se comporte de manière légèrement différente, car il n'y a pas de concept de catégories existantes, excepté dans le cas d'un document utilisant un nœud existant pour obtenir le contenu. INHERITATTRS prend en charge les options SOURCE et DESTINATION pour les documents et DESTINATION uniquement pour les autres types d'objet. Sans précision de votre part, aucune catégorie n'est appliquée. MERGE n'est pas prise en charge pour NODEACTION:CREATE. UNIQUEONLY renverra l'un des éléments suivants :

  • l'ID du nouveau nœud créé, le cas échéant ;

  • l'ID d'un nœud existant, de même nom et de même sous-type, si l'existence de ce nœud a empêché la création d'un nouveau nœud ;

  • une chaîne vide contenant un message d'erreur vierge indiquant que la création du nouveau nœud a échoué, car un nœud portant le même nom mais présentant un sous-type différent a empêché cette création.

Le message d'erreur peut être affiché à la place de la chaîne vide à l'aide de la balise ONERROR:LONG.

Tableau 1-14 : Action : MAXVERS

Opération
Description
NODEACTION:MAXVERS:<versions>
Définit le nombre maximum de versions pour un document ou un affichage personnalisé, de la même manière que ce paramètre est défini dans l'onglet d'un objet donné. Content Server fournit un paramètre opentext.ini, VersionLimitDefault, qui définit le nombre maximum de versions par défaut. Cette valeur peut être configurée pour un nœud à l'aide du paramètre facultatif DEFAULT. Par exemple, [LL_REPTAG=DATAID NODEACTION:MAXVERS:DEFAULT /]. Pour configurer un nombre illimité de versions pour un nœud donné, servez-vous de l'option NOLIMIT. Par exemple, [LL_REPTAG=DATAID NODEACTION:MAXVERS:NOLIMIT /]. Voir également NODEACTION:PURGEVERSIONS.

Tableau 1-15 : Action : MOVE

Opération
Description
NODEACTION:MOVE:<recueil>
Attend un dataID renvoyé par la balise principale. Déplace ce nœud à l'emplacement spécifié par le paramètre « recueil ».
NODEACTION:MOVE:<recueil>:INHERITATTRS:DESTINATION
Déplace le nœud dans le « recueil » spécifié, dont il hérite des catégories et des attributs.
NODEACTION:MOVE:<recueil>:INHERITATTRS:MERGED
Déplace le nœud dans le « recueil » spécifié et fusionne les catégories et les attributs du nœud source et de ce « recueil ».
NODEACTION:MOVE:<recueil>:INHERITATTRS:SOURCE
Déplace le nœud dans le « recueil » spécifié, en conservant les catégories et les attributs du nœud source.
NODEACTION:MOVE:<conteneur>:INHERITPERMS
Déplace le nœud dans le « recueil » spécifié, dont il hérite des autorisations.
NODEACTION:MOVE:<recueil>:INHERITRMOFF
Le comportement par défaut de NODEACTION est le même que si l'opération avait été effectuée via l'interface Content Server : un nœud déplacé hérite des classements RM du nouveau recueil parent. Cette option peut être utilisée pour stopper la transmission des classements RM. Lorsqu'un classement RM est déjà appliqué au nœud à déplacer, ce classement RM est conservé une fois l'événement de déplacement terminé. Cette option peut être utilisée pour remplacer les options « Administration de Records Management », « Objets gérés » et « Hériter à partir de », mais pas pour forcer la transmission (héritage) d'un classement RM si les paramètres RM appropriés n'ont pas été spécifiés.
NODEACTION:MOVE:<recueil>:NEWNAME:<nom>
Option facultative. Renomme le nœud déplacé.
NODEACTION:MOVE:<recueil>:UNIQUEONLY
Si le nom du nœud n'est pas unique au sein du recueil de destination, le nœud ne sera pas déplacé. Cette option peut être utilisée avec ou sans l'option NEWNAME.

Vous pouvez utiliser les paramètres de déplacement ci-dessus dans n'importe quelle combinaison et dans n'importe quel ordre. Par exemple, utilisez [LL_REPTAG=DATAID NODEACTION:MOVE:1234:INHERITATTRS:MERGED:INHERITPERMS:NEWNAME:"my node" /] pour combiner les catégories de source et de destination, hériter des autorisations de la destination et renommer le nœud déplacé avec le nom « mon nœud ». Pour joindre des éléments Content Server à un workflow lancé via un WebReport, dans l'onglet de destination, voir SETWFATTACH. Pour plus d'informations, consultez Sous-balises d'action.

Tableau 1-16 : Action : MOVEPROVIDER

Opération
Description
NODEACTION:MOVEPROVIDER:<nom du fournisseur>:<numéro de version>
Modifie le fournisseur de stockage pour la version. Les fournisseurs de stockage disponibles peuvent être identifiés dans la page Administration Configurer les fournisseurs de stockage. Pour sélectionner le fournisseur de stockage par défaut, utilisez DEFAULT comme nom de fournisseur. Si aucun numéro de version n'a été spécifié, la dernière version sera assignée au nouveau fournisseur. Si ALL est spécifié à la place du numéro de version, toutes les versions du nœud seront assignées au nouveau fournisseur. Les conséquences de cette sous-balise en termes de performances doivent être soigneusement étudiées avant toute utilisation ; en effet, celle-ci ne met pas seulement à jour la base de données, elle modifie également l'emplacement de stockage sous-jacent. Cela signifie qu'un nœud simple avec 30 versions entraînera le déplacement de 30 fichiers, et si chacun d'eux est un document de 1 Mo, la charge peut être considérable pour le système. Il est donc recommandé d'effectuer ce type d'opération en dehors des heures de bureau et de manière progressive.

Tableau 1-22 : Action : RENAME

Opération
Description
NODEACTION:RENAME:<nouveau nom>
Renomme le nœud renvoyé par la balise principale. Dans les environnements multilingues de la version Content Server 10, cette commande définit le nom dans la langue par défaut du système.
NODEACTION:RENAME:<nouveau nom>:<langue>
Dans les environnements multilingues de la version Content Server 10, cette commande renomme le nœud renvoyé par la balise principale, dans la langue spécifiée. Par exemple, NODEACTION:RENAME:"new name":"en_US" modifiera le nom anglais (US) du nœud. Notez que le paramètre <langue> est sensible à la casse et doit correspondre à une valeur de la colonne « Code de langue » de la page Administration Configurer les métadonnées multilingues.
NODEACTION:RENAME:<nouveau nom>:USERLANG
Dans les environnements multilingues de la version Content Server 10, cette commande renomme le nœud renvoyé par la balise principale, dans la langue par défaut de l'utilisateur. Par exemple, dans le cas d'un utilisateur dont la langue par défaut est l'anglais, NODEACTION:RENAME:"new name":USERLANG modifiera le nom anglais du nœud.
NODEACTION:RENAME:<nouveau nom1>:<langue1>:<nouveau nom2>:<langue2>
Dans les environnements multilingues de la version Content Server 10, cette commande renomme le nœud renvoyé par la balise principale, dans plusieurs langues. Par exemple, NODEACTION:RENAME:"new name":"en_US":"nouveau nom":"fr" remplacera le nom anglais (US) du nœud par « new name » et le nom français du nœud par « nouveau nom ».
Notez que les paramètres <langue> sont sensibles à la casse et doivent correspondre à une valeur de la colonne Code de langue de la page Administration Configurer les métadonnées multilingues.

NODEINFO

Syntaxe

NODEINFO:<format>

Description

Cette sous-balise renvoie des informations sur l'objet ou le nœud en cours, en fonction du paramètre de format spécifié. Le paramètre de format accepte comme valeur la plupart des colonnes de la table DTREE de la base de données Content Server, ainsi que divers autres paramètres uniques aux WebReports. Les principaux champs sont répertoriés ci-dessous. La sous-balise agit soit sur l'ID de nœud, soit sur le surnom Content Server.

Paramètres

Format
Description
ASSIGNEDTO
ID de l'utilisateur qui a été assigné à cet objet, généralement une tâche.
CATALOG
Renvoie une valeur numérique représentant la valeur de catalogue de l'objet.
0 = Répertorié, 1 = Mise en valeur et 2 = Masqué.
CHILDCOUNT
Nombre d'enfants du nœud.
CLASSIFICATION
Renvoie un élément RecArray contenant l'ID et le nom de chaque classement du nœud dont l'ID a été spécifié dans la balise principale.
COMMENT
Description du nœud.
CREATEDBY
ID de l'utilisateur ayant créé le nœud.
CREATEDATE
Date de création du nœud.
DATEASSIGNED
Date à laquelle le nœud a été assigné, généralement une tâche.
DATECOMPLETED
Date à laquelle le nœud a été achevé, généralement une tâche.
DATEDUE
Date d'échéance du nœud, généralement une tâche.
DATESTARTED
Date de démarrage du nœud.
DEPTH
Renvoie la profondeur hiérarchique d'un objet donné dans l'entreprise.
EXTENDEDDATA
Données spécifiquement stockées pour le nœud. Pour pouvoir extraire des informations utiles de ces données, vous devez utiliser des sous-balises complémentaires, par exemple ASSOC.
GROUPID
ID du groupe propriétaire du nœud.
ID
Toutes les commandes NODEINFO peuvent être lancées sur la base d'un surnom ou d'un ID de nœud. Cela signifie que l'utilisateur ne connaît pas nécessairement l'ID de nœud d'un élément ; dans ce cas, NODEINFO:ID lui permet de le récupérer en fournissant son surnom.
JSON
Exporte l'enregistrement NODEINFO sous forme de chaîne JSON.
LOCATION
Renvoie une liste des ID de nœud séparés par des virgules, qui commence par l'entreprise et se termine par le nœud en cours.
MAXVERSION
Nombre maximum de versions pour le nœud.
MODIFYDATE
Date de modification de l'objet.
NAME
Nom du nœud.
NICKNAME
Surnom du nœud.
ORIGID
Récupère le dataID du nœud d'origine d'un raccourci ou d'une génération.
PARENTID
ID du parent du nœud.
PATH
Indique le chemin d'accès complet d'un nœud Content Server, à partir du nœud le plus haut. Cette information est donnée sous la forme d'une chaîne de texte, délimitée par des virgules et commençant par le nœud le plus haut. Par exemple : Enterprise:Customers:ABC company
PRIORITY
Priorité d'un nœud, généralement une tâche.
RESERVED
Statut de réservation numérique du nœud.
RESERVEDBY
ID de l'utilisateur ayant réservé le nœud.
RESERVEDDATE
Date de réservation du nœud.
SIZE
Taille du nœud en cours. Si l'option DETAIL a été spécifiée, NODEINFO:SIZE:DETAIL, la taille exacte est renvoyée. Vous pouvez également utiliser NODEINFO:SIZE:BYTES pour extraire la taille exacte en octets, sous la forme d'un entier, ce qui offre davantage d'options de mise en forme.
STATUS
Statut d'un nœud, généralement une tâche.
SUBNAME
Type de nœud exprimé en mots, par exemple Document ou Dossier.
SUBTYPE
Type de nœud exprimé au format numérique ; parfois appelé sous-type.
USERID
ID de l'utilisateur propriétaire du nœud.
VERSIONCONTROL
Renvoie « Standard » ou « Avancée » en fonction du paramètre de contrôle de la version spécifié pour le nœud.
VERSIONNUM
Numéro de la version en cours du nœud.
VOLUMEID
ID de volume du nœud. L'ID de volume correspond généralement au recueil de plus haut niveau pour un objet. Ce champ est parfois appelé OWNERID. Pour les objets stockés dans l'espace de travail Projet, un ID de projet négatif est renvoyé pour l'ID de volume.
