Lors de l'exécution d'un WebReport, les paramètres peuvent être inclus dans l'URL pour le WebReport lui-même, ainsi que pour la source de données associée. Ces deux types de paramètres, les paramètres du WebReport et les paramètres de source de données, sont décrits ci-dessous.

Paramètres de WebReport

Plusieurs paramètres permettent de contrôler le comportement du WebReport. La plupart d'entre eux sont conçus pour permettre aux concepteurs de WebReport de créer des URL uniques afin de modifier le comportement du WebReport en fonction des exigences de l'application. Outre ces paramètres intégrés, le module WebReports prend également en charge la création et l'utilisation de paramètres uniques définis par les développeurs de WebReport. Pour les transmettre, il suffit de placer &parmname=data dans la liste des paramètres de l'URL où parmname peut être une chaîne quelconque de caractères alphanumériques standard.

Exemple

?func=ll&objId=47478&objAction=RunReport&myparm=testdata&nexturl=

Ces données sont alors accessibles dans l'affichage de rapport à l'aide de la syntaxe de balise définie. Dans cet exemple, [LL_REPTAG_&myparm /] renvoie « testdata ». Des sous-balises peuvent être utilisées pour modifier les données renvoyées par une balise de paramètre.

Paramètres de source de données

Les sources de données, les rapports LiveReport et les requêtes de recherche permettent de définir et de collecter les paramètres d'entrée à l'exécution du rapport :

  1. Lorsqu'un rapport LiveReport est utilisé comme source de données, l'invite standard du rapport LiveReport apparaît avant l'exécution du WebReport si aucun paramètre n'est indiqué dans l'URL. Dans ce scénario, à l'exécution de l'objet WebReport, l'utilisateur reçoit les invites du rapport LiveReport comme s'il avait exécuté directement le rapport LiveReport. Une fois les invites requises complétées, le WebReport continue normalement.

  2. Lorsqu'un WebReport a été défini à l'aide d'une requête enregistrée comme source de données, il est possible d'insérer des variables dans la spécification de recherche de texte intégral de la requête enregistrée. Par exemple, le terme %1 peut être placé dans le champ Termes de la recherche de texte intégral de toute zone de recherche de texte intégral. Ces variables seront remplacées par les paramètres correspondants dans l'URL. Les paramètres URL suivent la même convention que dans un rapport LiveReport.

    Par exemple, inputlabel<x>=value, où <x> est un nombre compris entre 1 et 99 qui correspond aux variables %1, %2, etc.

Exemple

...?func=ll&objId=47478&objAction=RunReport&inputLabel1=WebReport&nexturl=...

Dans cet exemple, une variable telle que %1 serait remplacée par le mot « WebReport » à l'aide d'un paramètre URL sous la forme : inputlabel1=WebReport.

Ces variables peuvent également servir dans des requêtes complexes du langage de requête Content Server, LQL. D'autres informations sur l'utilisation de la recherche sont disponibles dans l'aide en ligne.

Si un WebReport est exécuté en appelant son URL à partir d'une page développée par un utilisateur, par exemple un document HTML ou un autre WebReport, il est alors possible de fournir les paramètres d'entrée requis pour le WebReport et la source de données dans l'URL utilisée pour appeler le WebReport. Si tous les paramètres de rapport LiveReport requis sont transmis dans l'URL, l'utilisateur n'est pas invité à saisir les paramètres d'entrée de façon normale.

WebReports fournit une fonction qui permet de définir des valeurs par défaut pour les paramètres normalement transmis via l'URL. Lorsqu'une valeur par défaut est définie pour un paramètre, elle est alors utilisée si aucun paramètre correspondant ne se trouve dans l'URL.
