Seuls les utilisateurs bénéficiant des droits d'administration système peuvent activer l'utilisation de scripts pour un WebReport donné. Pour ce faire, trois méthodes différentes sont possibles :

  1. Ajoutez la nouvelle version de l'affichage de rapport. Lorsqu'un administrateur système ajoute un affichage de rapport contenant l'écriture de script, celle-ci est activée automatiquement.

  2. Dans le menu Fonctions du WebReport, sélectionnez Propriétés, puis Spécifique. Dans l'onglet Spécifique du WebReport, sélectionnez la case à cocher Ecriture de script Oscript activée, qui apparaît si l'écriture de script est présente dans l'affichage de rapport.

  3. Utilisez la page d'administration WebReports Gérer l'écriture de script WebReports pour vérifier tous les WebReports contenant l'écriture de script. Activez-les ou désactivez-les ensuite en sélectionnant ou en désélectionnant la case à cocher.

En fait, il est fastidieux pour les utilisateurs ne bénéficiant pas des droits d'administration système de développer un WebReport contenant des scripts, puisque chaque nouvelle version de leur WebReport doit être activée par quelqu'un disposant de ces droits. C'est l'une des raisons pour lesquelles il convient de développer des WebReports contenant des scripts sur une instance de simulation, où il est possible d'attribuer des droits d'administration système au développeur. Pour en savoir plus, voir Recommandations de développement d'affichages de rapports avec scripts.

Seule exception à cette règle, tout utilisateur peut créer un WebReport à partir d'un affichage de rapport par défaut contenant un oscript. Un tel WebReport est activé automatiquement, sans nécessiter l'intervention d'un administrateur système. Si, ultérieurement, l'affichage de rapport de ce WebReport est modifié ou une nouvelle version est ajoutée, son oscript est désactivé sauf s'il est réactivé par un administrateur système. Comme les affichages de rapports par défaut sont préactivés, il est essentiel que tous les affichages de rapports contenant Oscript et destinés à devenir des affichages de rapports par défaut soient testés et approuvés. Pour qu'un affichage de rapport devienne un affichage de rapport par défaut, placez-le dans le dossier des rapports d'affichages par défaut. Pour plus d'informations, reportez-vous au OpenText WebReports - Installation and Administration Guide (LLESWEBR-IGD).
