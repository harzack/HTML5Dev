It is possible to reference settings from the opentext.ini file with a WebReport, using the standard constant tag syntax. This feature is subject to the following limitations:

  • Only items contained under the [WebReports] section of the opentext.ini file can be referenced
  • Some standard options cannot be referenced. This is a development decision based on potential security issues. If in doubt, try to reference a particular setting as defined below. If the setting has been blocked and undefined constant error message will be displayed
  • If a constant has been defined for a WebReport with the same name as a preference, the constant takes precedence

The most common application of this feature is to define unique preferences that will only be used with this feature, i.e. preferences that do not already exist in the [WebReports] section. This allows a system (server) level setting that can be referenced by all WebReports. Because the opentext.ini file can vary from server to server, this feature also allows that WebReports can use logic based on a particular server. Here are a few examples of how this feature may be used:

  • Define system level messages, titles or labels
  • Define file paths or URLs. This is effectively creating a globally (system wide) available constant
  • Define parameters that identify a particular Livelink server. For example, "SERVERTYPE=SECURE". In this way logic can be added to WebReports to control their behavior according to which Livelink server they are being run from.

For more information on using the constant tag syntax see the section called Referencing Constants in the WebReport Reportview in the Using Constants help page.
