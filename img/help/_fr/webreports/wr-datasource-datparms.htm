Nom du paramètre
Description
sourceid
Le paramètre sourceid permet à une autre source de données d'être spécifiée en tant que dataid. Le dataid doit correspondre à l'un des types de nœud actuellement pris en charge par WebReports en tant que sources de données.
DSstartrow
Le paramètre DSstartrow permet de spécifier la ligne de début de la source de données à l'aide d'un nombre compris entre 1 et <N>. Ceci signifie que toute ligne avant le paramètre DSstartrow spécifié ne sera pas utilisée dans le rapport. Par exemple, si la source de données possède normalement 100 lignes de données et si un DSstartrow de 21 lignes est spécifié sans aucun paramètre DSendrow ou DSmaxrows, alors seules les lignes de 21 à 100 sont utilisées. Le paramètre DSstartrow peut être associé aux paramètres DSendrow ou DSmaxrows.
DSendrow
Le paramètre DSendrow permet de spécifier la dernière ligne de la source de données à l'aide d'un nombre compris entre 1 et <N>. Ceci signifie que toute ligne après le paramètre DSendrow spécifié ne sera pas utilisée dans le rapport.
Si le paramètre DSendrow est spécifié en tant que nombre négatif, alors zéro (0) ligne n'est renvoyée. Si le paramètre DSendrow spécifie un nombre de ligne inférieur au paramètre DSstartrow, alors zéro ligne n'est renvoyée.
Le paramètre DSendrow peut être associé aux paramètres DSstartrow ou DSmaxrows. Lorsque le paramètre DSmaxrows est utilisé, tout paramètre avec l'ensemble de données le plus limité sera prioritaire.
DSmaxrows
A l'aide d'un nombre compris entre 1 et <N>, le paramètre DSmaxrows spécifie le nombre maximum de lignes à utiliser à partir de la source de données. Ceci signifie que le nombre de lignes ne sera jamais supérieur à celui spécifié dans DSmaxrows. Comme indiqué pour DSendrows, DSmaxrows spécifie un nombre maximum de lignes, mais le véritable nombre de lignes peut être inférieur à DSmaxrows si les paramètres DSstartrow et DSendrow se combinent pour spécifier un ensemble de données plus petit. Si le paramètre DSmaxrows est spécifié en tant que nombre négatif, alors zéro (0) ligne n'est renvoyée.
