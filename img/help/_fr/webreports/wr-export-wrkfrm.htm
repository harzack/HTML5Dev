Lors de l'exportation vers un WebForm, l'utilisateur peut récupérer les résultats d'un rapport, ou en sélectionner des parties, et insérer ces résultats dans un WebForm Content Server. Ceci fournit au développeur de rapport une puissante fonction dans laquelle il peut importer des données provenant d'autres systèmes vers un WebForm, ou d'un WebForm vers un autre WebForm dans le même système. Les utilisateurs chevronnés peuvent utiliser cette fonction pour créer une mémoire cache à partir d'un ensemble de données plus important et exécuter d'autres rapports sur la base de cette mémoire cache régulièrement mise à jour.

L'exportation vers une fonction WebForm permet à l'utilisateur de mettre à jour et de remplacer les données existantes du WebForm, ou bien d'ajouter des données aux données existantes du WebForm.

Exemple

Pour spécifier quelles données du WebReport sont insérées dans le WebForm, vous devez utiliser la sous-balise SETFORM :

[LL_REPTAG=USERID SETFORM:USER /] va prendre l'ID de l'utilisateur et l'insérer dans la colonne USER.

Exemple

Vous pouvez également utiliser des sous-balises qui vont agir sur les données avant de les insérer dans le WebForm :

[LL_REPTAG=USERID USERINFO:GROUPID SETFORM:GROUP /] utilise USERINFO avec le paramètre GROUPID pour récupérer le groupe de l'actuel utilisateur. SETFORM prend alors ce groupe et l'insère dans le champ du WebForm appelé GROUP. Vous pouvez utiliser SETFORM pour insérer les données pour tous les champs nécessaires.
