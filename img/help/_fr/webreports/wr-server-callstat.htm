Il est parfois utile d'obtenir la valeur renvoyée par une balise de donnée statique WebReport dans une section d'Oscript. Notez que les balises de données statiques n'incluent pas de balises telles que [LL_REPTAG=<colonne> /] ou [LL_REPTAG_%<nom variable> /].

Par exemple, vous pouvez avoir besoin de connaître la valeur d'une constante ou d'un paramètre WebReport. Vous pouvez transmettre ces éléments sous forme de paramètres lorsque le script est appelé, mais vous pouvez aussi y accéder directement au moyen de la fonction point intégrée ._repTag dont la syntaxe est la suivante : String ._repTag( String tag )

Fonction Paramètres Renvoie
String ._repTag( String tag ) tag : valeur de chaîne représentant la balise, puis des sous-balises facultatives. Une valeur de chaîne contenant la valeur finale de la balise et des sous-balises spécifiées.

.
Exemple 1-32 : Voici deux exemples renvoyant la valeur d'une constante WebReport appelée report :

  1. String value1 = ._repTag('$report')

  2. String value2 = ._repTag('[LL_REPTAG_$report /]')

Notez que vous pouvez spécifier le nom complet de la balise ou une version raccourcie qui omet la mention d'ouverture [LL_REPTAG_ et le crochet fermant /].

Exemple 1-33 : Voici deux autres exemples :

  1. String value3 = ._repTag('&myNode NODEINFO:NAME')

  2. String value4 = ._repTag('USERID USERINFO:MAILADDRESS')

Notez que certaines sous-balises ne sont pas prises en charge par ._repTag. En général, les sous-balises qui exécutent une autre action que renvoyer des données formatées ne fonctionnent pas.

Les sous-balises suivantes ne sont pas prises en charge : ADDVAR, CONCATVAR, NEXT, PREV, SETFORM, SETVAR, SETWFATTR, SETWFFORM.
