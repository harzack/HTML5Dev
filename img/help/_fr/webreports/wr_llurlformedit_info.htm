LLURL:FORMEDIT may be used with forms residing inside a workflow as well as standalone forms outside of workflows. From WebReports 4.0.1 a change was made that effects WebReports running on Livelink 9.5 or 9.6 that use LLURL:FORMEDIT for a workflow form: It is no longer necessary to manually append &editable=true at the end of a URL created by LLURL:FORMEDIT as this parameter is now included automatically. Any extra "&editable=true" should be removed.

Standalone Forms

An optional parameter VIEWID is supported to specify the nodeid of the form view that should be used when displaying the form for editing. The viewids may be found by using the template's function menu and selecting Properties-Views. Hover the mouse pointer over the links to the view and look for the &objid= parameter in the browser status bar. If VIEWID is not specified, the view selected in the form's specific tab is used.

Usage example:

[LL_REPTAG=DATAID LLURL:FORMEDIT:VIEWID:1234 /]

Workflow Forms

When used with forms inside a workflow LLURL:FORMEDIT supports the optional parameters VIEWNAME and FORMID. FORMID specifies an integer that identifies (if there is more than one form) which form in the workflow should be edited. If no FORMID parameter is provided, a value of 1 is used by default. To discover the value required for FORMID you need to open the forms tab of the workflow and hover the mouse pointer over the "Edit" link to the form you require. Examine the URL for this link (either in the browser status bar or open the form in a new window and inspect the address bar). Look for the &FormID= parameter in the URL - this is id you need. VIEWNAME specifies the name (not the nodeid) of the view to use when editing the form. The available view names may be found by using the Template's function menu and selecting Properties-Views. If VIEWNAME is not specified, no view will be applied.

Further details on LLURL and the FORMEDIT option can be found in the Tag Guide.
