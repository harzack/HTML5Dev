Plusieurs destinations WebReports prennent en charge la conversion de documents tiers qui peut être utilisée pour créer des rapports au format PDF. Les destinations sont :

  • E-mail

  • Nœud Content Server

  • Version Content Server

  • Serveur

Dans les versions antérieures de WebReports, la conversion PDF était prise en charge d'une manière légèrement différente. L'utilisateur pouvait sélectionner un type MIME PDF qui, une fois sélectionné, générait l'affichage de rapport à convertir au format PDF. Une des limites résidait dans le fait que l'affichage de rapport converti au format PDF ne pouvait être qu'au format HTML.

A partir de WebReports 4.0, le type MIME et le processus de conversion ont été découplés. Ceci signifie que l'utilisateur peut utiliser WebReports pour créer tout type de document, par exemple WordML ou SpreadsheetML, puis exécuter le processus de conversion sur ce document. Ceci apporte beaucoup plus de flexibilité à l'aspect du document à convertir.
