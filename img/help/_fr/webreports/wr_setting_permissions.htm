WebReport permissions have some unique considerations. WebReports are effectively an amalgamation of three different object types in Content Server:

  • Documents (used to store and manage the reportviews)
  • Text Documents (used to support online reportview editing)
  • LiveReports (used to produce the report output)

The following table indicates the most common roles for WebReport users and what the recommended permissions should be.

WebReport User Role Minimum Permissions Associated Commands
General users who are not allowed to run or edit a report See Make Shortcut
General end user that is allowed to run the WebReport. This will be the main permissions setting for all end users of WebReports. This differs from normal documents where end users would need to View, Open, or Download. With WebReports these commands are only used for the users who actually create or edit the reportviews. See Contents RunReport, (also standard See Contents commands such as Set Notification, Add to Favorites etc.)
Users who might create or edit other WebReports but who are not allowed or required to have access to this specific WebReport. Modify Copy, Open, Download, View
Users who are responsible for modifying the reportviews and or selecting different associated data sources. These are typically the WebReport owners and administrators; however, they do not necessarily need to have delete and or permissions access. Reserve Add Reportview Version,
Edit Reportview,

Administrative users As Required (normal permissions behavior) E.g. Delete, Permissions, etc.
