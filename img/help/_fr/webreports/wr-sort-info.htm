La balise de contrôle de contenu du WebReport [LL_WEBREPORT_SORT <Clés de tri> <Ordre général> /] permet de trier les données en utilisant une ou plusieurs colonnes de la source de données spécifiée en tant que clés de tri. Il existe trois manières différentes de spécifier la référence de la colonne pour chaque clé de tri :

  • Spécifiez un numéro de colonne à partir de la source de données. Vous pouvez utiliser une valeur numérique représentant le numéro de colonne de gauche à droite dans la source de données. Par exemple, [LL_WEBREPORT_SORT "1" /].

  • Spécifiez un nom de colonne à partir de la source de données. Vous pouvez utiliser une valeur de texte représentant le nom de l'une des colonnes utilisées dans la source de données. Par exemple, [LL_WEBREPORT_SORT "dataid" /].

  • Spécifiez une balise et des sous-balises WebReports à utiliser en tant que clés de tri. Cette fonction permet de trier selon différents critères renvoyés par les données normales via la colonne. Par exemple : [LL_WEBREPORT_SORT "[LL_REPTAG=DATAID NODEINFO:NAME /]" /]. Dans cet exemple, le tri ne sera pas effectué en fonction de la colonne DataId, mais en fonction du nom de l'objet DTree renvoyé par NODEINFO:NAME.

Plusieurs colonnes de tri peuvent être spécifiées, permettant à l'utilisateur d'effectuer un tri dans un autre tri, etc. Nous pourrions, par exemple, trier en fonction de la première colonne, puis trier cette colonne en fonction de la deuxième colonne, et celle-ci en fonction de la troisième colonne. Les méthodes ci-dessus peuvent être combinées pour permettre le tri de plusieurs valeurs clés, comme le montrent les exemples suivants :

  1. [LL_WEBREPORT_SORT "1":ASC "name" "[LL_REPTAG_3 DECODE:1:Active:2:Inactive /]":ASC /]

    Le tri est effectué selon :

    1. la première colonne de façon explicite dans l'ordre croissant,

    2. puis la colonne nom, également dans l'ordre croissant par défaut,

    3. ensuite, en fonction de la valeur dans la colonne 3, le tri est effectué sur la colonne Active ou Inactive, également dans l'ordre croissant.

  2. [LL_WEBREPORT_SORT "1" "name":DESC "[LL_REPTAG_3DECODE:1:Active:2:Inactive/]" /]

    Le tri est effectué selon :

    1. la première colonne dans l'ordre croissant, par défaut,

    2. puis la colonne nom dans l'ordre décroissant,

    3. ensuite la troisième colonne, en ayant d'abord converti les valeurs 1 en Active et les valeurs 2 en Inactive.

    En plus du paramètre d'ordre qui peut éventuellement être spécifié pour chaque clé de tri, la fonction SORT prend également en charge un paramètre d'ordre général permettant de spécifier l'ordre de tri, croissant ou décroissant, qui doit être appliqué à toutes les clés de tri pour lesquelles aucun ordre n'a été spécifié. DESC ou ASC doivent être utilisés. Certaines syntaxes plus anciennes sont encore prises en charge. Si ce paramètre n'est pas spécifié, l'ordre de tri par défaut est ASC (ascending). Par exemple :

    [LL_WEBREPORT_SORT "1" "2" DESC /]

    Dans cet exemple, l'ensemble de données serait trié dans l'ordre décroissant pour la colonne 1, puis dans l'ordre décroissant pour la colonne 2.
