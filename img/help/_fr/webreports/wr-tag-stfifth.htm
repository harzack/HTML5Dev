Cette section apporte des informations sur les sous-balises suivantes : EMAILINFO, ENCSTR, ESCAPECSV, ESCAPEHTML, ESCAPESTR, ESCAPEURL, ESCAPEWR, ESCAPEXML et EVEN.

EMAILINFO

Syntaxe

EMAILINFO :<propriété>

Description

Attend un dataID valide pour un e-mail et renvoie les données de l'e-mail pour ce nœud, de la même manière que la vue dans un dossier d'e-mail. L'e-mail doit avoir été chargé via le plug-in Livelink Explorer/Outlook afin d'utiliser cette balise.

Paramètres

Format Description
ATTACHMENT
Renvoie Vrai ou Faux en fonction du champ « Pièces jointes » de l'e-mail. Si l'e-mail contient une pièce jointe, la valeur « Vrai » est renvoyée ; s'il ne contient pas de pièce jointe, la valeur« Faux » est renvoyée.
BCC
Renvoie le champ « CCi » de l'e-mail.
CC
Renvoie le champ « Cc » de l'e-mail.
FROM
Renvoie le champ « A partir de » de l'e-mail.
ONBEHALF
Renvoie le champ « Au nom de » de l'e-mail.
RECDATE
Renvoie le champ « Date de réception » de l'e-mail.
SENTDATE
Renvoie le champ « Date d'envoi » de l'e-mail.
TO
Renvoie le champ « A » de l'e-mail.

ENCSTR

Syntaxe

ENCSTR :<type_codage>

Description

Fournit un codage de chaîne à l'aide du format spécifié. Actuellement, BASE64 est le seul type de codage pris en charge.

ESCAPECSV

Syntaxe

ESCAPECSV

ou

ESCAPECSV:ALLCELLS

Description

Convertit les caractères de texte afin qu'ils puissent être utilisés en tant que cellules de données dans une exportation CSV. Cela signifie que les règles suivantes sont suivies :

  • Si une virgule est présente dans le texte, alors le groupe de caractères est mis entre guillemets doubles.

  • Si un caractère de fin de ligne est présent dans le texte, alors le groupe de caractères est mis entre guillemets doubles.

  • Si un caractère de guillemets doubles est présent dans le texte, alors il est converti en deux guillemets doubles et l'ensemble du groupe de caractères est mis entre guillemets doubles.

Exemple

Par exemple, si [LL_REPTAG_1 /] renvoie normalement one "two", three, alors [LL_REPTAG_1 ESCAPECSV /] renvoie : "one ""two"", three". Si le paramètre facultatif : ALLCELLS est spécifié, alors toutes les cellules sont mises entre guillemets, quel que soit le contenu de la cellule de données.

ESCAPEHTML

Syntaxe

ESCAPEHTML

Description

Convertit des caractères comme inférieur à ou supérieur à pour respecter la syntaxe d'échappement HTML.

Exemple

Par exemple, si [LL_REPTAG_1 /] renvoie normalement <bonjour>, [LL_REPTAG_1 ESCAPEHTML /] renvoie : &lt;Bonjour&gt;

ESCAPESTR

Syntaxe

ESCAPESTR

Description

Convertit des caractères tels que les sauts de lignes, les retours chariot et les guillemets en syntaxe de langage C standard, comme \r pour les retours chariot. Cette sous-balise est essentielle pour toute assignation de variable de script puisque les retours chariot, ainsi que d'autres caractères tels que des guillemets entraînent une erreur.

Exemple

Par exemple, Rows[x] = "[LL_REPTAG_1 ESCAPESTR /]";

ESCAPEURL

Syntaxe

ESCAPEURL

Description

Convertit les caractères non alphanumériques en séquences hexadécimales d'URL et convertit les espaces en signe plus, +.

Exemple

Par exemple, si une balise renvoie normalement la chaîne : « http://www.bogus.com/somefile » , [LL_REPTAG_1 ESCAPEURL /] renvoie : « http%3A%2F%2Fwww%2Ebogus%2Ecom%2Fsomefile »

ESCAPEWR

Syntaxe

ESCAPEWR

Description

Empêche qu'une syntaxe de balise de WebReport éventuellement incluse dans les données soit interprétée comme une syntaxe de balise valide.

ESCAPEXML

Syntaxe

ESCAPEXML

Description

Convertit les données en format compatible XML par l'intégration de caractères spéciaux tels que supérieur à, inférieur à, esperluette et apostrophe.

EVEN

Syntaxe

EVEN

Description

Renvoie Vrai si les données renvoyées par la balise principale sont un nombre pair. Renvoie Faux dans le cas contraire. Cela peut être utilisé avec une balise ROWNUM afin de créer un code pouvant varier d'une ligne à l'autre, par exemple la couleur de ligne.

Exemple

[LL_WEBREPORT_IF "TRUE" == "[LL_REPTAG_ROWNUM ODD /]" /]
<TR class=oddrow>
[LL_WEBREPORT_ENDIF /]
[LL_WEBREPORT_IF "TRUE" == "[LL_REPTAG_ROWNUM EVEN /]" /]
<TR class=evenrow>
[LL_WEBREPORT_ENDIF /]
