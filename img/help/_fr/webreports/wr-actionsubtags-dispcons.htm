Normalement, l'utilisation d'une sous-balise d'action dans une balise signifie que les données de la balise ne sont pas renvoyées dans la sortie du WebReport. Si les données de la balise doivent également être affichées dans la sortie du rapport, la sous-balise SHOW doit être utilisée. Celle-ci est généralement placée la plus à droite dans la liste des sous-balises. Notez que cette sous-balise affiche la valeur d'origine de la balise, avant que l'action soit effectuée. Pour les sous-balises d'action sur les variables, il est important de noter qu'aucun calcul sur les variables n'est inclus dans les données affichées. Les exemples ci-dessous décrivent ces faits plus en détail.

 

Note :
Les exemples ci-dessous utilisent tous une « balise littérale » qui renvoie la valeur fixe spécifiée entre guillemets. Cette balise est parfois utile lorsqu'un nombre donné doit être utilisé par les sous-balises d'action, par exemple pour ajouter un nombre particulier. Ici, elle est principalement utilisée pour simplifier les exemples.

 

Exemple
Description
[LL_REPTAG_"2" SETVAR:test /]
Une variable appelée test est créée et définie sur la valeur : 2. Elle n'est pas affichée dans le rapport. Les exemples successifs supposent que la variable test est définie sur cette valeur.
[LL_REPTAG_"123" ADDVAR:test /]
La valeur 123 est ajoutée à la variable nommée test ; la nouvelle valeur de test est désormais 125. Rien n'est affiché dans le rapport.
[LL_REPTAG_"123" ADDVAR:test SHOW /]
La valeur 123 est ajoutée à la variable nommée test, en supposant que cette variable test est toujours définie sur 2. La nouvelle valeur de test est égale à 125. La valeur de la balise, soit 123, s'affiche dans le rapport.
[LL_REPTAG_"123" DEC ADDVAR:test SHOW /]
La valeur 123 est décrémentée par la sous-balise DEC, si bien que la valeur devient 122. Elle est ensuite ajoutée à la variable nommée test ; la nouvelle valeur de test est désormais 124. La valeur de la balise, une fois modifiée par la sous-balise DEC, soit 122, s'affiche dans le rapport.
