Cette section apporte des informations sur les sous-balises suivantes : ODD, ODDEVEN et ONERROR.

ODD

Syntaxe

ODD

Description

Renvoie Vrai si les données renvoyées par la balise principale sont un nombre impair ; renvoie Faux dans le cas contraire. Cela peut également être utilisé comme dans l'exemple pour la balise EVEN.

ODDEVEN

Syntaxe

ODDEVEN

Description

Renvoie soit ODD, soit EVEN en fonction des données renvoyées par la balise principale. Si les données de la balise principale ne sont pas un entier valide, alors cette sous-balise renvoie une chaîne vide. Cette sous-balise peut être utilisée avec une balise telle que ROWNUM afin de définir un style approprié pour chaque ligne. La sous-balise ODDEVEN permet à deux paramètres supplémentaires d'être spécifiés. S'ils sont fournis, ces deux paramètres seront renvoyés d'une manière différente de ODD/EVEN. L'exemple ci-dessous indique comment utiliser les styles Content Server, ParcourirLigne1 et ParcourirLigne2 standard afin de permettre à chaque colonne d'avoir une couleur alternative. L'instruction de classe pour chaque ligne alterne entre ParcourirLigne1 et ParcourirLigne2.

Chaque ligne inclut l'instruction TR suivante :

<TR class=[LL_REPTAG_ROWNUM ODDEVEN:BrowseRow1:BrowseRow2 /]>

Chaque ligne devient l'un des résultats suivants :

<TR class=BrowseRow1>

ou :

<TR class=BrowseRow2>

ONERROR

Syntaxe

ONERROR:<param>

Description

Utilisé afin de spécifier le texte alternatif à utiliser pour toute erreur de sous-balise. Certaines sous-balises renvoient un message d'erreur si les données et/ou l'un des paramètres attendus de la sous-balise n'a pas la valeur ou le type correct nécessaire afin d'exécuter la fonction de la sous-balise. Cette sous-balise permet au message d'erreur par défaut pour une balise particulière d'être affiché de différentes manières définies ou d'être défini par le développeur. Cette sous-balise apparaît généralement après la sous-balise pouvant générer une condition d'erreur. Il est possible d'avoir plus d'une sous-balise ONERROR afin de traiter les erreurs de manière différente pour plusieurs sous-balises. Dans le cas où une sous-balise ONERROR a été fournie, la position dans la liste de sous-balises n'est pas importante et la sous-balise ONERROR s'applique à toute erreur, pour toute sous-balise dans la liste.

Paramètres

Les options suivantes pour utiliser cette sous-balise incluent :

Type d'erreurs
Description
ONERROR:BLANK
Cette option remplace tout message d'erreur standard avec une chaîne vide et est utile pour empêcher l'affichage des messages d'erreur.
ONERROR:SHORT
Cette option affiche une version courte du message d'erreur composé de la sous-balise en question et d'un numéro d'erreur.
ONERROR:LONG
Cette option affiche un message d'erreur complet pour la sous-balise en question.
ONERROR:CUSTOM :<message d'erreur>
Cette option permet au développeur de spécifier son propre message d'erreur pour une sous-balise.

Exemple

Par exemple : [LL_REPTAG=VALUE LIST:1 ONERROR:CUSTOM:"Invalid list" INC:1 ONERROR:LONG /]

Dans cet exemple, le message « Liste non valable » est renvoyé pour cette balise si la sous-balise LIST a renvoyé une erreur. Si la sous-balise LIST n'entraîne pas d'erreur, contrairement à la sous-balise INC, un message d'erreur standard est renvoyé.
