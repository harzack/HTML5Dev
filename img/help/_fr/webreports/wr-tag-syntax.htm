Deux méthodes permettent d'accéder aux informations sur les balises dans WebReports :

  • Vous pouvez accéder au guide sur les balises statiques fourni avec la documentation. Pour en savoir plus, voir Guide sur les balises statiques.

  • Un guide sur les balises dynamiques est fourni avec le module. Ce guide permet aux développeurs de glisser-déposer les balises du guide vers l'interface utilisateur. Pour plus d'informations sur l'accès au guide sur les balises dynamiques, voir Pour accéder au guide sur les balises dynamiques.

Guide sur les balises statiques

Cette section décrit les différents types de balises utilisables dans WebReports :

  1. Balises de contrôle de contenu : ces balises sont toujours précédées de : [LL_WEBREPORT_. Elles servent à contrôler le mode de création du contenu de sortie. Par exemple, il est possible d'activer ou de désactiver l'en-tête et le pied de page Content Server standard. Les principales balises de contrôle de contenu servent à définir la partie de l'affichage de rapport appliquée à chaque ligne de données de la source de données.

  2. Balises de données : les balises de données sont toujours précédées de : [LL_REPTAG. Elles peuvent être décomposées en Balises de données de colonne et en Balises de données statiques :

    1. Balises de données de colonne : pour chaque colonne présente dans la source de données associée, des balises de données de colonne référencent les données de rapport à inclure dans le résultat. Ces balises permettent d'utiliser le numéro de colonne, par exemple [LL_REPTAG_2 /], ou le nom de colonne, par exemple [LL_REPTAG=DATAID /].

    2. Balises de données statiques : les balises de données statiques servent à insérer des informations de Content Server qui ne sont pas associées à des lignes du rapport. Par exemple, la date et l'heure, le nom de l'utilisateur qui exécute le WebReport ou différentes URL qui peuvent servir à créer des liens hypertextes dans le résultat du rapport. Par exemple, [LL_REPTAG_NEXTURL /]).

  3. Sous-balises : les sous-balises servent à fournir des fonctions de mise en forme ou autre avec l'une des balises [LL_REPTAG ... /]. Ces sous-balises, séparées par des espaces, permettent d'effectuer des actions sur les données réelles extraites de la source de données. Pour plus d'informations sur les sous-balises, voir la section 6.2.3.

    Toutes les balises sont sensibles à la casse et utilisent toujours des lettres majuscules. Elles sont aussi sensibles aux espaces. Veillez à placer une espace devant « /] » à la fin de la balise. En revanche, les noms de colonne peuvent comporter indifféremment des majuscules et des minuscules. Les tableaux des sections Balises de contrôle de contenu, Balises de données de colonne et Sous-balises répertorient toutes les balises actuellement disponibles dans WebReports et décrivent leur mode d'utilisation.

Les tableaux suivants décrivent les balises reconnues par WebReports. Les balises ne sont pas toutes prises en charge dans l'intégralité des sections de l'affichage de rapport. Les sections prises en charge (en-tête, modèle de ligne, pied de page) sont indiquées sous chaque nom de balise.
