• Crée une colonne dans un ActiveView/WebReport pouvant être triée lorsque l'utilisateur clique sur le titre. Un ensemble d'attributs de formatage peut également être spécifié et appliqué à la colonne.

  • Traite les arguments suivants :

    • title : paramètre obligatoire. Titre à afficher de la colonne. Si aucun titre n'est spécifié, par exemple ", un simple diviseur est ajouté.

    • HTML Attributes : paramètre facultatif. Paramètre facultatif représentant une liste d'attributs de formatage HTML, séparés par des virgules, pour la colonne. Par exemple « align=right,width=10%,etc... »

    • Sort Reference : paramètre facultatif. Chaîne facultative représentant le nom de la colonne sur laquelle effectuer le tri, à partir de la source de données. Vous pouvez spécifier ici des noms de colonnes simples. For example, « subtype » ou « name ». si vous souhaitez trier selon la valeur d'une sous-balise WebReport, il vous suffit de spécifier ici une simple « reference key » et la sous-balise peut être définie par la directive @PREDEFKEY dans la balise SORT. Par exemple, si parm3='catValue', la balise SORT contient la valeur suivante : @PREDEFKEY REF:catValue PARM:"[LL_REPTAG=DATAID CAT:'somecat':'someattr':DISPLAY /]")

    • CS Sorting : paramètre facultatif. Paramètre booléen facultatif, true/false, pour indiquer si vous souhaitez utiliser la syntaxe du tri Content Server, par exemple &sort=name, &sort=-name, ou celle du tri WebReport, par exemple &sort=name&direction=asc, &sort=name&direction=desc. La valeur par défaut est false. Les noms de colonnes simples doivent utiliser le tri Content Server (true). Les tris basés sur des combinaisons complexes de balises et de sous-balises doivent utiliser le tri WebReport (false).

    • Alternate Sort Parm : paramètre facultatif. Chaîne facultative vous permettant de définir votre paramètre de tri personnalisé dans l'URL. Utilisée conjointement à la directive @PARMNAMES, qui doit être définie dans la balise SORT. Exemple : Si parm5='AVsort', pour effectuer le tri, la balise SORT recherche le paramètre '&AVsort' plutôt que le paramètre par défaut '&sort'.

    • Current Value for Alt Parm : paramètre facultatif. Chaîne facultative spécifiant la valeur de tri indiquée par l'autre paramètre de tri. Il s'agit généralement d'une balise de paramètre utilisant le nom dans l'autre paramètre de tri. Par exemple, [LL_REPTAG_&AVSORT /]

  • Cette fonction exige que la fonction 'setupContext' ait été exécutée au préalable, car elle fait référence à certaines variables. Par exemple : supportDir, urlPrefix, myURL et sortCol.

  • Pour plus d'informations sur les directives @PREDEFKEY et @PARMNAMES, voir Tri des résultats.

  • Exemple :

    setColumn( '' );								// No title is specified, just add a divider
setColumn( 'Type', 'width=2%', 'subtype', true );				// Simple usage. Sort by the column 'subtype' returned from the datasource using the Content Server sort. 
setColumn( 'Name', 'align=left', 'name', true );				// Simple usage. Sort by the column 'name' returned from the datasource using the Content Server sort. 

// Sort the column by the value of '=DATAID NODEINFO:VERSIONNUM' tag/sub-tag. Here 'verNum' is a reference key for the @PREDEFKEY directive defined in the SORT tag below. Here we are using the WebReport sort. 
setColumn( 'Version', 'align=right,width=15%', 'verNum' );

// Sort the column by a category-attribute value. 'catValue' is a reference key and 'AVsort' is a custom sort parameter defined in @PARMNAMES directive in the SORT tag below. 
setColumn( 'Attribute', '', 'catValue', false, 'AVsort' );
