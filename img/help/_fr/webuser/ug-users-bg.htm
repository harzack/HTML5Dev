Content Server vous permet de travailler avec d'autres personnes de différentes façons. La plupart des utilisateurs ne peuvent faire que rechercher des informations sur d'autres utilisateurs afin de communiquer ou collaborer avec eux. Si vous disposez des privilèges appropriés, vous pouvez également modifier les informations utilisateur et créer ou supprimer des utilisateurs. Vous pouvez afficher et utiliser les informations sur les utilisateurs sur la page Utilisateurs et groupes.

Création d'utilisateurs

Lorsque vous créez un utilisateur, Content Server vous demande de fournir un nom d'utilisateur pour le compte et de spécifier un groupe de service pour le nouvel utilisateur.

Vous devez également indiquer les privilèges système dont l'utilisateur disposera. Par défaut, tous les utilisateurs ajoutés à Content Server reçoivent les privilèges Connexion autorisée et Accès public activé.

Si votre administrateur a activé plusieurs langues sur votre système, une option permettant de définir la langue préférée de l'utilisateur apparaît. Vous pouvez ainsi spécifier une langue pour un utilisateur particulier qui peut être différente de la langue par défaut du système.

Privilèges système

Les privilèges sont assignés à un compte utilisateur au moment de son ajout. Contrairement aux autorisations, qui ne fonctionnent que pour des éléments spécifiques, les privilèges s'appliquent à l'ensemble du système. Le tableau suivant décrit les privilèges système dans Content Server.

Privilège Description
Connexion autorisée Active le compte utilisateur.
Accès public activé Permet à l'utilisateur d'accéder à n'importe quel élément du système pour lequel l'autorisation Accès public a été activée.
Peut créer/modifier des utilisateurs Permet à l'utilisateur de créer des utilisateurs et de modifier ou de supprimer ceux qu'il a créés.
 

Note :
Si vous disposez seulement du privilège Création/modification d'utilisateurs, vous pouvez ajouter des utilisateurs seulement dans les groupes dont vous êtes le chef. Pour créer un utilisateur n'importe où dans Content Server, vous devez disposer du privilège Droits d'administration utilisateur.

 
Peut créer/modifier des groupes Permet à l'utilisateur de créer des groupes et de modifier ou de supprimer ceux qu'il a créés.
Droits d'administration utilisateur Permet à l'utilisateur de créer, de modifier ou de supprimer n'importe quel utilisateur ou groupe.
Droits d'administration système, Permet à l'utilisateur d'accéder à toutes les fonctions d'administration de Content Server s'il connaît le mot de passe d'administration. Ce privilège permet en outre à l'utilisateur qui le possède de bénéficier de l'accès complet à tous les éléments du système, sans aucun filtrage. Ce privilège n'inclut pas la capacité de gérer les utilisateurs ou les groupes.

 

Note :
Lorsque vous créez ou modifiez un utilisateur, vous ne pouvez pas lui accorder des privilèges dont vous ne disposez pas vous-même. Par exemple, vous ne pouvez pas accorder le privilège Droits d'administration système si vous ne possédez pas vous-même ce privilège.

 

Affichage des informations sur les utilisateurs

Content Server affiche les informations sur les utilisateurs sur les pages suivantes :

  • la page Infos générales pour : la page <nom_utilisateur>, qui contient le nom de connexion de l'utilisateur, son mot de passe et ses principales coordonnées. Si vous disposez des privilèges nécessaires pour modifier un utilisateur spécifique, cela affiche également les privilèges système dont l'utilisateur dispose.

  • la page Infos générales pour : la page <nom_utilisateur>, qui contient des informations telles qu'une photographie, une biographie et d'autres informations sur le contact. Toutes les informations contenues sur cette page sont facultatives.

Vous pouvez afficher vos propres informations utilisateur, générales et personnelles, sur la page Mon profil. Pour en savoir plus, voir Affichage des pages de rapports personnels.

Modification des informations utilisateur

Si vous disposez des privilèges adéquats, vous pouvez modifier les informations se trouvant sur les pages Infos générales ou Informations personnelles. L'icône Modifier, , indique que vous disposez du privilège nécessaire pour modifier les informations d'un utilisateur. Tous les utilisateurs peuvent modifier leurs propres informations personnelles.

Suppression d\'utilisateurs

Il est relativement simple de supprimer un utilisateur ou un groupe. Cependant, avant de supprimer un utilisateur, allez dans l'espace de travail Personnel de l'utilisateur et éliminez-en le contenu en supprimant les éléments indésirables et en déplaçant les éléments importants vers d'autres emplacements. Il est important d'effectuer cette opération avant de supprimer l'utilisateur, car il est difficile d'accéder à l'espace de travail Personnel d'un utilisateur supprimé.

 

Attention
Une fois un utilisateur supprimé de Content Server, cet utilisateur n'est plus accessible à personne, y compris l'administrateur, sur la page Utilisateurs et groupes. Content Server est conçu de cette manière pour qu'il soit impossible d'attribuer des tâches ou des étapes de workflow à un utilisateur supprimé. Cependant, l'espace de travail Personnel reste intact et peut être consulté par l'administrateur si ce dernier peut accéder à l'ID de l'utilisateur supprimé ou s'il sait comment rechercher un élément contenu dans l'espace de travail Personnel de l'utilisateur supprimé. Pour en savoir plus sur l'accès à l'espace de travail Personnel d'un utilisateur supprimé, voir Pour afficher l`espace de travail personnel d`un utilisateur supprimé.
