Chaque utilisateur de Content Server est membre d'au moins un groupe et peut appartenir à plusieurs groupes. Un groupe peut contenir des sous-groupes, qui sont des groupes au sein d'un groupe. Il est essentiel d'organiser les utilisateurs en groupes pour configurer un système Content Server afin d'en simplifier l'utilisation et la maintenance. Une communauté d'utilisateurs bien organisée permet à l'administrateur de :

  • administrer les privilèges système ;

  • gérer les autorisations pour les éléments de Content Server ;

  • gérer les tâches assignées, telles que les tâches des étapes du workflow et les listes de tâches ;

  • configurer les rapports de notification par défaut pour les nouveaux utilisateurs.
