Note :
Vous pourrez uniquement accéder à ces paramètres si votre administrateur a fait en sorte que la page Mes paramètres soit disponible.

 

Lorsque vous essayez d'ajouter un e-mail avec des pièces jointes à une destination, vous pouvez vous apercevoir qu'une pièce jointe portant un nom identique se trouve déjà dans le dossier. Vous pouvez spécifier la façon dont Enterprise Connect gère les pièces jointes d'e-mail portant le même nom.

Vous pouvez spécifier qu'Enterprise Connect vous demande d'ajouter la pièce jointe avec le nom en double en tant que nouvelle version du document existant. Vous pouvez également spécifier que cela se produit automatiquement, chaque fois qu'Enterprise Connect détecte un document portant le même nom. Vous pouvez spécifier qu'Enterprise Connect vous demande si vous voulez ajouter la pièce jointe avec le nom en double en tant que nouveau document. Dans ce cas, le document en double conservera son nom, mais un suffixe numérique sera ajouté pour le distinguer du document original. Par exemple, formulaire(2).doc. Vous pouvez également spécifier que cela se produit automatiquement, chaque fois qu'Enterprise Connect détecte un document portant le même nom.

Si vous avez installé Records Management (RM) et que vous créez un renvoi pour les pièces jointes d'e-mails, vous verrez des renvois vers l'instance appropriée de la pièce jointe. Par exemple, supposons que vous déplaciez un e-mail avec une pièce jointe intitulée formulairelegal.doc vers un dossier dans Content Server, et un document portant un nom identique se trouve déjà dans ce dossier. Vous avez sélectionné Ajouter automatiquement des pièces jointes en tant que nouvelle version de document. Les renvois étant activés, une nouvelle version de formulairelegal.doc sera ajoutée au document existant. Lorsque vous ouvrez l'e-mail à partir du nouvel emplacement, celui-ci contient un lien vers l'instance appropriée de formulairelégal.doc. Dans ce cas, il s'agit de la seconde version de formulairelégal.doc. Pour plus d'informations sur la configuration des renvois RM, voir OpenText Records Management - Installation and Administration Guide (LLESRCM-IGD) .
