Une apparence est un recueil qui stocke les documents et le code HTML. Elle permet de personnaliser certains emplacements, un peu comme vous pouvez concevoir une page Web.

 

Important :
Bien que vous puissiez créer et modifier des apparences à l'aide d'Enterprise Connect, celles-ci ne sont pas visibles dans Enterprise Connect. Cela s'applique également aux apparences créées dans l'interface Web de Content Server.

 

Il existe deux types d'apparences que vous pouvez ajouter pour personnaliser l'interface :

  • Les apparences générales, appliquées de manière homogène dans Content Server pour des utilisateurs spécifiques. Les apparences générales peuvent être appliquées uniquement au volume des apparences, ce qui les rend disponibles dans tout le système.

  • Apparences basées sur l'emplacement.

Les apparences basées sur l'emplacement peuvent être appliquées de deux façons :

  • Pas de cascade. L'apparence est appliquée uniquement à l'Espace de travail, au projet ou au dossier auquel elle est ajoutée.

  • En cascade. L'apparence est appliquée à un Espace de travail, au projet ou au dossier parent dans lequel elle est ajoutée et à tous les projets ou dossiers enfant. Ce type d'apparence peut être également appliqué à un ensemble d'utilisateurs spécifique ou à des groupes uniquement.

Vous pouvez gérer les apparences via les pages Vue d'ensemble et Espace de travail, sur lesquelles vous pouvez définir ou modifier l'apparence. Après avoir ajouté une apparence, vous devez définir les paramètres et la disposition sur la page Vue d'ensemble, puis ajouter du contenu, tel que des images, à la page Espace de travail. L'ajout et la modification d'apparences sont soumis à des restrictions.
