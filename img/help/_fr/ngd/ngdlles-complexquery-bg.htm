Une requête complexe est une instruction unique, formulée en langage Live Query Language (LQL), qui vous permet de cibler avec précision les données que vous souhaitez récupérer. Par exemple, pour rechercher des éléments contenant le mot rapport mais pas le mot frais, saisissez la requête complexe report AND-NOT expense.

Les mots clés LQL sont des mots que le langage de requête Livelink se réserve et auxquels il attribue une signification particulière. Lorsque les mots clés LQL sont utilisés avec les conventions du langage de requête Livelink, ils vous permettent de créer des requêtes complexes. Les mots clés LQL ne sont pas sensibles à la casse.

Le langage LQL constitue une autre méthode de spécification des attributs lors de la recherche d'éléments. Chaque élément possède certains attributs contenant des informations supplémentaires sur cet élément. Par défaut, les recherches Content Server ne portent pas sur les attributs des éléments. Toutefois, vous avez la possibilité de spécifier des attributs dans les requêtes de base ; vous pouvez également utiliser le langage LQL pour rechercher un attribut en particulier ou encore un ensemble d'attributs.

 

Note :
Les noms d'attributs sont sensibles à la casse. Pour obtenir la liste des attributs disponibles et leur casse correcte, contactez votre administrateur.
