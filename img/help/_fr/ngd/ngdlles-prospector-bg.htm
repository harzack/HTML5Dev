Les OpenText Prospectors sont des éléments Content Server qui aident les utilisateurs à localiser et utiliser les informations. Ils agissent en tant que filtres, en analysant les données indexées par Content Server afin de localiser les informations susceptibles d'intéresser les utilisateurs. Les Prospectors sont exécutés en continu, de sorte que les listes de résultats qu'ils gèrent sont toujours précises et à jour.

Les Prospectors peuvent analyser toute source de données indexée par Content Server. Ils peuvent, par exemple, effectuer des recherches sur le référentiel Content Server, sur des répertoires locaux ou sur une base de données Lotus Notes. Pour plus d'informations sur les sources de données indexées sur votre site Content Server, contactez votre administrateur.

Pour créer des Prospectors, vous utilisez le Live Query Language (LQL) pour construire une requête qui vous permet de localiser le type d'informations que vous souhaitez récupérer.

Une fois la requête du Prospectors créée, vous choisissez la vue de recherche que le prospecteur devra analyser. Le prospecteur exécute ensuite sa requête pour voir si les nouveaux éléments ajoutés à l'index correspondent aux critères de recherche indiqués. Si c'est le cas, le prospecteur ajoute un lien à l'élément correspondant dans sa liste de résultats. Il continue de mettre à jour sa liste de résultats lorsque des éléments sont ajoutés ou enlevés de l'index. Les Prospectors peuvent uniquement analyser les index créés pour la recherche par un administrateur.

Lorsque vous créez des Prospectors, vous définissez également les options de la liste de résultats : le nombre de résultats et leur ordre d'affichage. Vous pouvez également choisir d'afficher ou de masquer les résumés des éléments de la liste des résultats dans Content Server.

Il est possible de créer des Prospectors dans la plupart des emplacements Content Server. Vous pouvez, par exemple, les créer dans l'Espace de travail, dans un Espace de travail Projet ou dans votre Espace de travail Personnel.
