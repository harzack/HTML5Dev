Vous pouvez utiliser la commande Copier/Déplacer vers Enterprise Connect pour déplacer un ou plusieurs éléments d'un emplacement à un autre dans le même référentiel Content Server. Si vous déplacez un élément de document ou de non-recueil, vous pouvez utiliser les commandes de déplacement Enterprise Connect standard, telles que glisser-déposer. Si vous déplacez un conteneur, un dossier par exemple, vous pouvez utiliser le menu contextuel pour accéder à la commande Copier/Déplacer vers Enterprise Connect.

 

Astuce :
Si vous utilisez l'Explorateur Windows, vous pouvez également déplacer des documents et des dossiers en cliquant sur le bouton droit de la souris. Sélectionnez les éléments à déplacer, faites-les glisser vers la destination à l'aide du bouton droit de la souris, puis cliquez sur Déplacer. Cette action est disponible lorsque vous faites glisser des éléments d'un dossier local vers Enterprise Connect, d'un emplacement à l'autre dans un même référentiel Content Server et des résultats d'une recherche Enterprise Connect vers un dossier du même référentiel Content Server ou d'un différent plug-in.

 

Lorsque vous déplacez un élément vers un nouvel emplacement, vous pouvez être invité à spécifier si le ou les éléments que vous déplacez doivent conserver les catégories assignées à l'origine, si les catégories sont supposées être assignées à la destination ou si les catégories sont supposées être assignées à l'origine et à la destination. En fonction de la configuration de votre système, vous pouvez également être invité à spécifier si les éléments doivent hériter de classements OpenText Records Management assignés.

Si un élément portant le même nom de fichier existe déjà dans la destination, vous pouvez être invité à spécifier la méthode résolution du conflit d'attribution de noms. Selon votre configuration, les options suivantes peuvent être disponibles :

  • Vous pouvez spécifier que l'élément actuel soit déplacé vers la destination avec un nom de fichier généré automatiquement. Ceci vous permet de conserver toutes les versions, historiques d'expertise ou métadonnées associés à l'élément. Si vous déplacez plusieurs éléments, vous pouvez également spécifier que tous les conflits d'attribution de noms doivent être résolus de cette manière.

     

    Astuce :
    Si vous disposez d'autorisations suffisantes, vous pouvez renommer le fichier une fois le déplacement terminé.

     

  • Vous pouvez spécifier que vous ne souhaitez pas déplacer l'élément actuel vers la destination. Si vous déplacez plusieurs éléments, vous pouvez également spécifier qu'aucun des doublons du lot ne soit déplacé.

  • Vous pouvez spécifier que vous ne souhaitez pas poursuivre le déplacement. Les éléments déjà déplacés vers la destination y sont conservés.

Vous pouvez déplacer une ou plusieurs instances des types d'élément non-document, non-recueil suivants vers un nouvel emplacement du référentiel Content Server :

  • Affichages personnalisés

  • Sondages

  • Raccourcis

  • URL

  • Schéma de Workflows

  • Documents

  • Documents texte

  • Sous-types fournis par les modules Content Server, des pages wiki par exemple.

 

Notes :

  • Utilisez le menu contextuel pour déplacer des éléments Content Server Dossiers ou équivalents de Dossier tels que les dossiers d'e-mails, projets, documents composites, Statut du workflow, listes de tâches, chaînes ou étapes clés.

  • Vous devez disposer des autorisations appropriées pour exécuter cette fonction. Pour plus d'informations sur la définition des autorisations, reportez-vous à la rubrique Définition des autorisations.

 
 

Astuce :
Pour plus d'informations sur le déplacement d'éléments à l'aide de Enterprise Connect, notamment les commandes disponibles, reportez-vous à la rubrique OpenText Enterprise Connect Plug-in for Content Server - Manuel de l'utilisateur (NGDCORE-UGD).
