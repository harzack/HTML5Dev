La Live Query Language (LQL) est constitué des mots clés suivants :

AND

Le mot clé AND a la même fonction que le mot clé &. Il permet de rechercher les éléments contenant les deux expressions indiquées dans la requête.

Syntaxe

<expression> AND <expression>

Exemple

Saisissez fish AND bird pour rechercher les éléments contenant à la fois poisson et oiseau.

&

Le mot clé & a la même fonction que le mot clé AND. Il permet de rechercher les éléments contenant les deux expressions indiquées dans la requête.

Syntaxe

<expression>  &  <expression>

Exemple

Saisissez fish & bird pour rechercher les éléments contenant à la fois poisson et oiseau.

AND-NOT

Le mot clé AND-NOT a la même fonction que le mot clé &! . Il permet de rechercher les éléments contenant l'expression indiquée à gauche du mot clé mais pas l'expression située à sa droite.

Syntaxe

<expression>  AND-NOT <expression>

Exemple

Saisissez fish AND-NOT bird pour rechercher les éléments contenant le mot poisson mais pas le mot oiseau.

&!

Le mot clé &! a la même fonction que le mot clé AND-NOT. Il permet de rechercher les éléments contenant l'expression indiquée à gauche du mot clé mais pas l'expression située à sa droite.

Syntaxe

<expression>  &! <expression>

Exemple

Saisissez fish &! bird pour rechercher les éléments contenant le mot poisson mais pas le mot oiseau.

OR

Le mot clé OR a la même fonction que le mot clé |. Il permet de rechercher les éléments contenant l'expression indiquée à gauche du mot clé, ceux contenant l'expression située à sa droite ou ceux qui contiennent les deux expressions.

Syntaxe

<expression>  OR  <expression>

Exemple

Saisissez fish OR bird pour rechercher des éléments contenant le mot poisson ou des éléments contenant le mot oiseau ou encore des éléments qui contiennent à la fois poisson et oiseau.

|

Le mot clé | a la même fonction que le mot clé OR. Il permet de rechercher les éléments contenant l'expression indiquée à gauche du mot clé, ceux contenant l'expression située à sa droite ou ceux qui contiennent les deux expressions.

Syntaxe

<expression>  | <expression>

Exemple

Saisissez fish | bird pour rechercher des éléments contenant le mot poisson ou des éléments contenant le mot oiseau ou encore des éléments qui contiennent à la fois poisson et oiseau.

SOR

Le mot clé SOR a la même fonction que le mot clé OR mais son algorithme de classement de pertinence considère que les expressions situées à la gauche et à la droite du mot clé SOR sont synonymes et les traite comme un terme unique. Cela permet d'éviter que les synonymes aient plus de poids dans le calcul des taux de pertinence pour un élément du résultat de la recherche. Les taux et l'ordre de classement des résultats des requêtes effectuées à l'aide du mot clé SOR sont différents de ceux issus des requêtes effectuées à l'aide du mot clé OR.

Syntaxe

<expression>  SOR  <expression>

Exemple

Saisissez "U.K." SOR "United Kingdom" pour rechercher les éléments contenant au moins une fois le mot R.U. ou le mot Royaume-Uni ou les deux. Content Server affiche les résultats de cette requête dans un ordre différent de ceux de la requête "R.U." OR "Royaume-Uni".

XOR

Le mot clé XOR recherche les éléments contenant une des deux expressions indiquées dans la requête mais pas les deux. Ce mot clé est également appelé <OR> exclusif.

Syntaxe

<expression>  XOR  <expression>

Exemple

Saisissez fish XOR bird pour rechercher les éléments contenant le mot poisson ou le mot oiseau, mais pas les deux.

=

Le mot clé = permet de rechercher les éléments contenant l'expression spécifiée. Vous pouvez ignorer ce mot clé lorsque vous créez une requête à l'aide de la Live Query Language car il s'agit du comparateur par défaut.

Syntaxe

= <expression>

Exemple

Saisissez = oiseau pour rechercher les éléments contenant le mot oiseau.

NOT

Le mot clé NOT a la même fonction que le mot clé ! et le mot clé !=. Il recherche les objets ne contenant pas l'expression spécifiée.

Syntaxe

NOT <expression>

Exemple

Saisissez NOT bird pour rechercher les éléments ne contenant pas le mot oiseau.

!

Le mot clé ! a la même fonction que le mot clé NOT et le mot clé !=. Il recherche les objets ne contenant pas l'expression spécifiée.

Syntaxe

! <expression>

Exemple

Saisissez ! bird pour rechercher les éléments ne contenant pas le mot oiseau.

!=

Le mot clé != a la même fonction que les mots clés NOT et !. Il recherche les objets ne contenant pas l'expression spécifiée.

Syntaxe

!= <expression>

Exemple

Saisissez != bird pour rechercher les éléments ne contenant pas le mot oiseau.

<

Le mot clé < permet de rechercher les éléments contenant des nombres ou des mots inférieurs à celui que vous avez indiqué. Si le terme indiqué est un nombre entier ou un nombre réel, Content Server utilise la position numérique. Autrement, il utilise la position lexicographique.

Syntaxe

< <expression>

Exemple

Saisissez < 20000630 pour rechercher des éléments contenant au moins un nombre inférieur à 20000630. Si vous restreignez cette requête à un attribut de date, elle permet de rechercher les éléments qui sont datés d'avant le 30 juin 2000.

<=

Le mot clé <= permet de rechercher les éléments contenant des nombres ou des mots inférieurs ou égaux à celui que vous avez indiqué. Si le terme indiqué est un nombre entier ou un nombre réel, Content Server utilise la position numérique. Autrement, il utilise la position lexicographique.

Syntaxe

<= <expression>

Exemple

Saisissez <= 20000630 pour rechercher des éléments contenant au moins un nombre inférieur ou égal à 20000630. Si vous restreignez cette requête à un attribut de date, elle permet de rechercher les éléments qui sont datés du 30 juin 2000 ou d'avant.

>

Le mot clé > permet de rechercher les éléments contenant des nombres ou des mots supérieurs à celui que vous avez indiqué. Si le terme indiqué est un nombre entier ou un nombre réel, Content Server utilise la position numérique. Autrement, il utilise la position lexicographique.

Syntaxe

> <expression>

Exemple

Saisissez > 20000630 pour rechercher des éléments contenant au moins un nombre ou un mot supérieur à 20000630. Si vous restreignez cette requête à un attribut de date, elle permet de rechercher les éléments qui sont datés après le 30 juin 2000.

>=

Le mot clé >= permet de rechercher les éléments contenant des nombres ou des mots supérieurs ou égaux à celui que vous avez indiqué. Si le terme indiqué est un nombre entier ou un nombre réel, Content Server utilise la position numérique. Autrement, il utilise la position lexicographique.

Syntaxe

>= <expression>

Exemple

Saisissez >= 20000630 pour rechercher des éléments contenant au moins un nombre ou un mot supérieur ou égal à 20000630. Si vous restreignez cette requête à un attribut de date, elle permet de rechercher les éléments Content Server qui sont datés du 30 juin 2000 ou d'après.

QLPHONETIC

Le mot clé QLPHONETIC recherche les éléments contenant des mots qui ressemblent phonétiquement à l'expression indiquée dans la requête. Ce mot clé est basé sur un algorithme usuel appelé Soundex. L'algorithme Soundex crée une représentation à 4 chiffres à partir d'un mot ; celle-ci est conçue pour correspondre à la prononciation phonétique du mot d'origine. Soundex est souvent utilisé pour des recherches vagues dans lesquelles une correspondance proche est requise.

Syntaxe

QLPHONETIC <expression>

Exemple

Saisissez QLPHONETIC sail pour rechercher les éléments contenant les mots comme conte ou compte.

QLREGION

Le mot clé QLREGION recherche les éléments contenant les termes de la recherche compris dans une région d'index spécifique. Chaque région regroupe un certain type d'informations concernant les éléments indexés, notamment le contenu et les métadonnées des documents. Pour obtenir la liste des régions, contactez votre administrateur.

Syntaxe

[QLREGION <région>] <expression>

Exemple

Saisissez [QLREGION OTCreatedByName] jdoe pour rechercher les éléments contenant le mot gdurand dans la région OTCreatedByName.

QLSTEM

Le mot clé QLSTEM recherche les éléments contenant des mots dérivés de la racine de l'expression spécifiée dans la requête.

Syntaxe

QLSTEM <expression>

Exemple

Saisissez QLSTEM sailing pour rechercher les éléments contenant au moins un mot dérivé de charge, comme chargement, décharger ou charges.

QLTHESAURUS

Le mot clé QLTHESAURUS recherche les éléments contenant les entrées de thésaurus associées à l'expression spécifiée dans la requête.

Syntaxe

QLTHESAURUS <expression>

Exemple

Saisissez QLTHESAURUS stone pour rechercher les éléments contenant au moins une entrée de thésaurus associée au mot noyau, comme noyau, pépin, graine ou noix.

QLRIGHT-TRUNCATION

Le mot clé QLRIGHT-TRUNCATION recherche les éléments contenant des mots dont le préfixe correspond à l'expression spécifiée dans la requête.

Syntaxe

QLRIGHT-TRUNCATION <expression>

Exemple

Saisissez QLRIGHT-TRUNCATION post pour rechercher les éléments contenant au moins un mot qui commence par part, comme particulier, partage ou partenaire.

QLLEFT-TRUNCATION

Le mot clé QLLEFT-TRUNCATION recherche les éléments contenant des mots dont le suffixe correspond à l'expression de requête indiquée.

Syntaxe

QLLEFT-TRUNCATION <expression>

Exemple

Saisissez QLLEFT-TRUNCATION thing pour rechercher les éléments contenant au moins un mot qui finit par tion, comme action, définition ou spécification.

QLRANGE

Le mot clé QLRANGE recherche les éléments contenant des mots (ou nombres) compris dans l'intervalle lexicographique (ou numérique) formé par les deux expressions indiquées dans la requête. Les termes que vous indiquez comme bornes de l'intervalle de recherche sont inclus dans cet intervalle. Si les termes indiqués sont des nombres entiers ou des nombres réels, Content Server utilise la position numérique. Autrement, il utilise la position lexicographique.

Vous devez placer l'intervalle de recherche entre guillemets droits (" ") et séparer les termes qui servent de bornes par un tilde (~).

Syntaxe

QLRANGE <  "expression  >~ <expression">

Exemple

Saisissez QLRANGE "giraffe ~ gnou" pour rechercher les éléments contenant au moins un mot de l'intervalle lexicographique qui va de girafe à gnou, comme girafe, girafes, girolle, glace, gnome ou gnou.

Saisissez QLRANGE "20000801 ~ 20000901 pour rechercher les éléments contenant au moins un nombre compris entre 20000801 et 20000901 (par exemple, entre le 1er août 2000 et le 1er septembre 2000 compris).

QLREGEX

Le mot clé QLREGEX recherche les éléments contenant une extension de l'expression régulière spécifiée dans la requête. Les expressions de requête utilisées avec le mot clé QLREGEX doivent être délimitées par des guillemets droits (" " ").

Syntaxe

QLREGEX <"expression">

Exemple

Saisissez QLREGEX "^the" pour rechercher les éléments contenant au moins l'une des extensions de l'expression régulière ^cha, par exemple chat, chatte, chaton, char, chant, chaîne, chanoine, chagrin ou chapiteau.

PROX

Le mot clé prox[n,t/f] permet de rechercher les éléments contenant toutes les locutions de requête spécifiées se trouvant à une distance désignée (n) l'une de l'autre, et où l'ordre est classé (t) ou non (f). La distance (n) est calculée comme le nombre d'unités lexicales entre la dernière unité de la première locution et la première de la seconde. La spécification du second paramètre (t/f) est facultative et a par défaut la valeur f.

Syntaxe

<expression> prox[n,t/f] <expression>

Exemple

Saisissez race prox[10,t] car pour trouver les éléments contenant le mot course précédant le mot voitures jusqu'à une distance de 10 unités lexicales d'écart.

QLNEAR

Le mot clé QLNEAR a la même fonction que le mot clé PROX, mais il est défini sur la valeur par défaut prox[10,f].

Syntaxe

<expression >QLNEAR <expression>

Exemple

Saisissez race QLNEAR car pour trouver les éléments contenant le mot course et le mot voitures jusqu'à une distance de 10 unités lexicales d'écart, dans n'importe quel ordre.
