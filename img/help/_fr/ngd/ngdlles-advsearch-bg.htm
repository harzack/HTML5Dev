La boîte de dialogue Recherche Content Server vous permet de créer des requêtes contenant des mots clés ou des requêtes complexes créées avec Live Query Language (LQL). Cette section comprend également des composants qui vous permettent d'étendre ou de limiter votre recherche. Pour étendre votre recherche, sélectionnez un opérateur qui indique la relation que vous souhaitez inclure entre le terme saisi et votre recherche. Pour limiter votre recherche, précisez si vous souhaitez effectuer la recherche dans le contenu, dans les métadonnées des éléments ou dans les deux. Le contenu correspond aux informations contenues dans les éléments, par exemple le texte dans un document. Les métadonnées correspondent aux informations accompagnant les différents éléments Content Server, par exemple la date d'ajout d'un document. Vous pouvez également limiter votre recherche en utilisant des opérateurs booléens pour définir précisément la relation entre les différents critères de recherche de votre requête.

Les mots clés sont des mots ou des groupes de mots que vous spécifiez pour rechercher des éléments contenant ces mots ou ces groupes de mots. Par exemple, si vous spécifiez les mots clés modèle de plan de développement, Content Server peut renvoyer une liste d'éléments contenant ces trois mots, n'importe lequel de ces mots ou l'expression exacte. Si plusieurs mots sont indiqués entre guillemets, Content Server recherche la locution exacte mentionnée entre guillemets.

Vous pouvez étendre votre recherche à des éléments contenant des mots associés aux mots spécifiés dans la requête. Certains opérateurs vous permettent ainsi de spécifier le type de mots associés à inclure dans la recherche. Par exemple, vous pouvez sélectionner un opérateur permettant de rechercher les éléments contenant des mots qui, phonétiquement, ressemblent au mot clé spécifié. Vous pouvez également sélectionner un opérateur permettant de rechercher les éléments contenant des mots dérivés de la racine du mot clé spécifié. Par exemple, si vous recherchez les termes ayant la même racine que le mot clé communiquer vous obtenez la liste des éléments contenant les termes communiquer , communicant, communiquent , puis communiqué.

Lorsque vous utilisez des mots clés avec un opérateur, celui-ci s'applique à tous les mots. Lorsque vous recherchez une la locution exacte, l'opérateur s'applique à l'ensemble de l'expression. Dans une requête complexe, l'opérateur s'applique à tous les mots de la recherche qui ne sont pas des mots clés LQL. Pour plus d'informations sur le langage LQL, consultez la section Recherche à l'aide de requêtes complexes.

Pour limiter une recherche, vous pouvez augmenter le nombre de critères spécifiques utilisés par Content Server dans le cadre de cette recherche. Ces critères regroupent l'utilisation d'opérateurs booléens (Et (AND), Ou (OR) et Sauf (NOT) : logique élémentaire permettant de modifier la relation entre les mots clés) et la possibilité de faire porter la recherche sur le contenu ou les métadonnées d'un élément (ou les deux). Les autres types de critères de recherche sont les options avancées : portée, attributs système, attributs personnalisés au sein d'une catégorie et types XML. Le tableau ci-dessous décrit les critères de recherche disponibles.

Critères Type de recherche
Dans Tous Permet de rechercher les informations contenues dans chaque élément, comme dans le texte de documents ou dans les informations de métadonnées dont la recherche a été autorisée par votre administrateur.
Contenu Permet de rechercher les informations contenues dans les différents éléments, par exemple dans le texte des documents.
Métadonnées Permet de rechercher les informations accompagnant les différents éléments, par exemple la date d'ajout d'un document. Si vous effectuez des recherches dans des métadonnées, votre recherche se limite aux métadonnées pour lesquelles votre administrateur a autorisé les recherches (c'est-à-dire, s'il a appliqué la recherche par défaut ). Il est possible que cette option n'inclue pas toutes les métadonnées associées à un élément.
Opérateurs booléens Permet de définir avec précision la relation entre les différents critères de recherche mentionnés dans la requête. Par exemple, si les mots clés indiqués sont plan et test vous pouvez rechercher uniquement les éléments contenant les mots plan AND test renvoie les éléments contenant plan et test; plan OR test permet de rechercher les éléments contenant planou testou les deux ; plan NOT test permet de rechercher les éléments contenant le terme plan, mais pas le terme test .
Attributs système Permet de rechercher les éléments possédant certains attributs. Par exemple, vous pouvez rechercher tous les éléments avec un attribut Créé par défini pour un utilisateur spécifique.

Vous pouvez également utiliser les modèles personnels et systèmes que vous créez et gérez dans Content Server pour effectuer des recherches avancées dans Enterprise Connect. Lorsque vous sélectionnez un modèle dans Enterprise Connect, la page Rechercher est renseignée avec les critères et formulaires de recherche spécifiés dans le modèle. Pour plus d'informations sur l'utilisation des modèles, consultez la section Utilisation des formulaires de recherche.

 

Note :
Si un modèle contient un formulaire de recherche non pris en charge dans Enterprise Connect, ce composant n'apparaîtra pas sur la page Rechercher. Texte intégral, Attributs système, Catégories, E-mail, Emplacement et Types d'élément physique sont actuellement pris en charge.
