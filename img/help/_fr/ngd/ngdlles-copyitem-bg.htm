Vous pouvez utiliser la commande Copier/Déplacer vers Enterprise Connect pour copier un ou plusieurs éléments d'un emplacement à un autre dans le même référentiel Content Server. Si vous copiez un élément de document ou de non-recueil, vous pouvez utiliser les commandes de copie Enterprise Connect standard. Si vous copiez un conteneur, un dossier par exemple, vous pouvez utiliser le menu contextuel pour accéder à la commande Copier/Déplacer vers Enterprise Connect.

 

Astuce :
Si vous utilisez l'Explorateur Windows, vous pouvez également copier des documents et des dossiers en cliquant sur le bouton droit de la souris. Sélectionnez les éléments à copier, faites-les glisser vers la destination à l'aide du bouton droit de la souris, puis cliquez sur Copier. Cette action est disponible lorsque vous faites glisser des éléments d'un dossier local vers Enterprise Connect, d'un emplacement à l'autre dans un même référentiel Content Server et des résultats d'une recherche Enterprise Connect vers un dossier du même référentiel Content Server ou d'un différent plug-in.

 

Lorsque vous copiez un élément dans un nouvel emplacement, vous pouvez être invité à spécifier si le ou les éléments que vous copiez doivent conserver les catégories assignées à l'origine, si les catégories sont supposées être assignées à la destination ou si les catégories sont supposées être assignées à l'origine et à la destination. En fonction de la configuration de votre système, vous pouvez également être invité à spécifier si les éléments doivent hériter de classements OpenText Records Management assignés.

Si un élément portant le même nom de fichier existe déjà dans la destination, vous pouvez être invité à spécifier la méthode résolution du conflit d'attribution de noms. Selon votre configuration, les options suivantes peuvent être disponibles :

  • Vous pouvez spécifier que l'élément actuel soit copié vers la destination avec un nom de fichier généré automatiquement. Ceci vous permet de conserver toutes les versions, historiques d'expertise ou métadonnées associés à l'élément. Si vous copiez plusieurs éléments, vous pouvez également spécifier que tous les conflits d'attribution de noms soient résolus de cette manière.

     

    Astuce :
    Si vous disposez d'autorisations suffisantes, vous pouvez renommer le fichier une fois la copie terminée.

     

  • Vous pouvez spécifier que vous ne souhaitez pas copier l'élément actuel vers la destination. Si vous copiez plusieurs éléments, vous pouvez également spécifier qu'aucun des doublons du lot ne soit copié.

  • Vous pouvez spécifier que vous ne souhaitez pas poursuivre la copie. Les éléments déjà copiés vers la destination y sont conservés.

Vous pouvez copier une ou plusieurs instances des types d'élément non-document, non-recueil suivants dans un nouvel emplacement du référentiel Content Server :

  • Affichages personnalisés

  • Sondages

  • Raccourcis

  • URL

  • Schéma de Workflows

  • Documents

  • Documents texte

  • Sous-types fournis par les modules Content Server, des pages wiki par exemple.

 

Notes :

  • Utilisez le menu contextuel pour copier des éléments Content Server Dossiers ou équivalents de Dossier tels que les dossiers d'e-mails, projets, documents composites, Statut du workflow, listes de tâches, chaînes ou étapes clés.

  • Vous devez disposer des autorisations appropriées pour exécuter cette fonction. Pour plus d'informations sur la définition des autorisations, reportez-vous à la rubrique Définition des autorisations.

 
 

Astuce :
Pour plus d'informations sur la copie d'éléments à l'aide de Enterprise Connect, notamment les commandes disponibles, reportez-vous à la rubrique OpenText Enterprise Connect Plug-in for Content Server - Manuel de l'utilisateur (NGDCORE-UGD).
