Live Query Language (LQL) permettent de définir la syntaxe des requêtes complexes.

Une expression est une séquence fixe d'un ou plusieurs mots délimitée par des guillemets droits (" "). Par exemple, "bon" et "pour le meilleur et pour le pire". Vous pouvez toutefois ignorer les guillemets droits encadrant les expressions composées d'un seul mot (par exemple, riz).

Lors de la création de locutions, les règles suivantes s'appliquent :

  • Un caractère d'espacement (par exemple, une tabulation ou un espace) indique la fin d'un mot.

  • Content Server interprète les chaînes comportant des signes de ponctuation mais aucun espace comme une seule expression (par exemple, "www.opentext.com").

  • Vous pouvez ignorer les guillemets droits encadrant les expressions d'un seul mot (par exemple, riz).

  • N'oubliez pas les guillemets droits encadrant les mots clés réservés (qlnear, par exemple) lorsque vous souhaitez utiliser les mots clés comme critères de recherche. Par exemple, pour rechercher des informations sur le mot clé qlnear dans l'aide en ligne, votre requête doit être "qlnear" et vous devez cliquer sur la vue de recherche appropriée.

Pour vos requêtes, vous pouvez utiliser un astérisque (*) en tant qu'espace réservé pour certains caractères. Par exemple, pour indiquer que vous souhaitez rechercher un mot comportant plusieurs terminaisons possibles, saisissez account* pour rechercher les éléments contenant les mots compte, comptes, comptable, comptabilité, etc.

Par défaut, Content Server traite les termes de la requête de gauche à droite. Utilisez les parenthèses (()) pour ne pas tenir compte de l'ordre par défaut et obliger Content Server à évaluer d'abord les expressions entre parenthèses. Par exemple, saisissez (culture OR ferme) AND (blé OR grain) pour rechercher les éléments contenant au moins une occurrence des mots culture ou ferme et au moins une occurrence des mots blé ou grain.
