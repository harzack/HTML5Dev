Un document composite est un recueil qui stocke plusieurs documents dans un ordre séquentiel. La gestion des documents composites vous permet d'exécuter des fonctions liées aux documents composites à partir de l'Explorateur Windows. Vous pouvez par exemple créer une version ou une révision du document ou encore réorganiser les sous-documents.

La manière d'ajouter des éléments à un document composite est la même que pour les autres recueils. Toutefois, les documents composites ne peuvent contenir que les types d'éléments suivants :

  • Documents

  • Documents texte

  • E-mails

  • Documents composites

  • Raccourcis et générations de documents, de documents texte et d'autres documents composites

Chaque élément que vous ajoutez à un document composite est numéroté séquentiellement en fonction de l'ordre dans lequel vous l'avez ajouté. Toutefois, vous pouvez à tout moment modifier l'ordre des éléments. De plus, un document composite peut inclure un document principal, que vous pouvez désigner lorsque vous réorganisez les éléments dans le document composite. Le document principal peut être la table des matières d'un livre, la page d'index d'un site Web ou encore la lettre de présentation d'un rapport. Le document principal est toujours le premier élément d'un document composite et celui situé le plus en haut de la structure.

Les documents composites ne requièrent pas de numérotation séquentielle, vous pouvez parfaitement avoir des éléments numérotés 1, 17 , puis 339. Lorsque vous réorganisez les éléments, ils peuvent être renumérotés automatiquement pour enlever les intervalles vides.

Content Server conserve les versions des éléments contenus dans un document composite. En outre, les documents composites disposent de leurs propres types de versions appelées versions et révisions. Chaque version et révision est une version statique d'un document composite, créée à un moment donné dans le passé. Par exemple, si vous possédez un document composite contenant le manuel d'un logiciel, vous pouvez créer une version au moment de la publication du logiciel. Vous pouvez continuer à utiliser le document composite d'origine car la version représente le manuel tel qu'il est actuellement présenté à vos clients. Les versions sont numérotées en 1.0, 2.0, 3.0, etc. Une révision est comparable à une version mais cette option est préférable lorsque vous souhaitez ne conserver que certaines modifications mineures apportées à un document composite et que vous n'avez pas besoin de créer une nouvelle version. Les révisions sont numérotées en fonction de la version. La révision 1.2 correspond à la deuxième révision de la première version, etc.

Il est parfois difficile de gérer l'ensemble de la structure des gros documents composites en regardant les contenus niveau par niveau. C'est pourquoi vous pouvez afficher un aperçu du document composite. L'aperçu, qui est hiérarchique, présente l'intégralité de la structure d'un document composite sur une page, avec les éléments les uns à la suite des autres. En outre, il possède un menu Fonctions pour chaque élément du document composite, qui vous permet d'utiliser chaque élément à partir de ce document. Par exemple, vous pouvez ouvrir un document pour voir son contenu.
