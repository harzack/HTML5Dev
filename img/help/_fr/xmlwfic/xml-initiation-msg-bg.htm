Un message de lancement peut être envoyé par une action d'« envoi de message de lancement XML » ou d'« envoi de message de lancement HTML », et accepté par une action d'« acceptation de message de lancement XML » ou d'« acceptation de message de lancement HTML » pour les lancements de workflows à distance.

Représentation XML

<exec><initiate status="…" query="…">…</initiate></exec>

Au format XML par défaut, un message de lancement est constitué d'un élément « initiate situé à l'intérieur d'un élément exec ». L'élément initiate contient le message.

Si une action d'acceptation reçoit un message de lancement contenant l'attribut query="yes", elle renvoie les valeurs mises à jour pour tous les champs valides.

Espaces de noms XML (le cas échéant)

<exec xmlns="http://www.opentext.com/llx/">

<initiate xmlns="http://www.opentext.com/llx/Workflow/">

Lorsque l'action est configurée de façon à utiliser et demander des espaces de noms XML, différents espaces de noms sont utilisés pour les éléments exec et initiate.

Représentation HTML

Dans les données de formulaire HTML, un message de lancement est représenté par son contenu.

Contenu du message

Un message de lancement contient les éléments suivants :

  • statut de la tâche ;

  • titre de la tâche ;

  • tâche source ;

  • Instructions

  • lots de travail.

Pour plus d'informations sur ces éléments, reportez-vous à Informations sur la tâche et Lots de travail.
