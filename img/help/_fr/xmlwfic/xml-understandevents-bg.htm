Les messages envoyés et reçus par XML Workflow Interchange permettent à un processus source d'une machine source de lancer un processus cible sur une machine cible. Le processus cible renvoie ensuite les résultats au processus source. Les deux processus peuvent être un workflow Content Server.

Etant donné que le processus cible peut impliquer une interaction manuelle des utilisateurs, ce processus n'est pas censé se terminer immédiatement et n'a donc pas besoin d'un canal de communication ouvert pendant l'exécution du processus cible. Lorsque les workflows sont connectés de cette façon, les deux événements suivants entraînent l'échange de messages :

  • Un lancement se produit lorsque le processus source souhaite démarrer le processus cible. Par exemple, si le processus source est un workflow Content Server et qu'une tâche XML Interchange est prête, la tâche source envoie un message de lancement au système cible, qui démarre alors le processus cible. Le système cible renvoie immédiatement une réponse de lancement contenant des informations sur le nouveau processus qu'il a démarré ou une erreur indiquant la raison pour laquelle le processus ne peut pas démarrer. Si la cible est une URL de lancement à distance dans un schéma de workflow Content Server, le message de lancement démarre un nouveau workflow en utilisant ce schéma. Les réponses indiquant un lancement réussi comprennent une URL qui peut être utilisée pour vérifier le statut du workflow lancé.

  • Une modification de tâche se produit lorsque le nouveau processus cible atteint une étape ou se termine et est prêt à renvoyer des informations ou le statut au processus source. Le processus cible envoie alors un message de modification de la tâche au processus source. Ce message peut mettre à jour les informations dans la tâche source et transférer la tâche. Si le processus cible est un workflow Content Server, le message de modification de la tâche est envoyé à l'URL fournie dans le message de lancement par la tâche source.

Si les processus source et cible sont des workflows Content Server, ces deux événements sont déclenchés lorsqu'une étape Workflow Interchange est prête. Lorsqu'une étape Workflow Interchange du workflow source est prête, elle envoie un message de lancement au système cible. Lorsque l'étape Workflow Interchange qui se trouve à la fin du workflow cible est prête, elle répond en envoyant un message de modification de la tâche. Une fois le message de lancement envoyé, l'étape Workflow Interchange de la source attend un message de modification de la tâche.

L'éditeur de workflow Content Server permet de déterminer ce qui se produit lorsque chaque étape Workflow Interchange de votre workflow est prête, que les étapes actives reçoivent des messages de modification de tâche et que les schémas de workflow reçoivent des messages de lancement. Le concepteur de workflow Content Server permet de s'écarter de façon significative du modèle d'échange standard et permet à d'autres types de systèmes de jouer le rôle de source ou de cible.
