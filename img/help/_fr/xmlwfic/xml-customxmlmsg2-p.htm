Pour formater le contenu d'un message XML :

  1. Double-cliquez sur l'étape Workflow Interchange, puis cliquez sur l'onglet Actions.

  2. Cliquez sur l'icône Modifier  pour Accepter un message de lancement XML.

  3. Cliquez sur l'onglet Format.

  4. Dans la liste de la zone Format, sélectionnez Format personnalisé.

  5. Effectuez l'une des actions suivantes :

    • Saisissez le nom et le chemin d'accès d'une feuille de style XSLT dans l'un des champs pour convertir le message via des feuilles de style XSLT.

       

      Notes :

      • Pour les actions d'envoi XML, spécifiez des feuilles de style séparées pour les messages XML sortants et pour les accusés de réception entrants provenant du système cible. Pour les actions d'acceptation XML, spécifiez les feuilles de style pour les messages XML entrants et les accusés de réception sortants que l'action renvoie. Le chemin spécifié est relatif au répertoire support.

      • La feuille de style XSLT doit être placée dans le répertoire support de Content Server par l'administrateur système.

       

    • Saisissez le nom et la valeur d'un paramètre dans les champs Nom et Valeur, puis cliquez sur l'icône Ajouter une valeur  pour définir les paramètres si vous spécifiez des feuilles de style qui acceptent les paramètres XSL.

       

      Important :
      Les valeurs des paramètres XSL sont des expressions XSL et non pas des chaînes. Si vous souhaitez insérer une valeur de chaîne dans la feuille de style, elle doit être placée entre guillemets. Consultez la section « XSLT specification » à l'adresse http://www.w3.org pour obtenir une description des expressions XSL.

       

  6. Sélectionnez la case à cocher Utiliser et exiger des espaces de noms XML pour garantir que les espaces de noms XML par défaut sont affectés aux éléments de tous les messages XML produits par l'action.

     

    Notes :

    • Les espaces de noms par défaut sont nécessaires pour tous les messages XML que l'action accepte, conformément à la section « XML Namespaces Recommendation » du site http://www.w3.org. Pour plus d'informations sur les espaces de noms devant être attribués à chaque élément, reportez-vous à Champs et formats de messages.

    • Les espaces de noms peuvent compliquer le traitement XML mais ils sont utiles dans les systèmes XML avancés pour gérer l'intégration de composants XML provenant de plusieurs sources. Si vous utilisez XML Workflow Interchange pour connecter des workflows Content Server, désélectionnez la case à cocher Utiliser et exiger des espaces de noms XML, sauf si l'autre système Content Server exige des espaces de noms.

     

  7. Cliquez sur Appliquer.

Pour plus d'informations sur cette procédure, reportez-vous à la rubrique Personnalisation des messages XML.
