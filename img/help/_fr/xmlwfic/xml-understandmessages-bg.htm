Tous les messages envoyés ou reçus par XML Workflow Interchange contiennent des données de formulaire XML ou HTML (formulaires en plusieurs parties de type MIME) transférées à l'aide du protocole HTTP. Lorsqu'un message est envoyé par une tâche Workflow Interchange, il peut comprendre des données provenant de lots de travail formulaires, commentaires, pièces jointes et attributs associés à la tâche. Le destinataire d'un message renvoie une indication du statut, le cas échéant, les nouvelles valeurs de certaines données ou de toutes les données.

 

Note :
les commentaires du schéma source ne sont pas visibles sur un workflow lancé à distance.

 

Le type de données qui peut être transféré dans un message et une réponse dépend du type d'événement d'échange qui se produit.

Lors d'un événement de lancement, les informations suivantes peuvent être échangées :

  • Informations de la tâche source (message uniquement), comprenant l'URL utilisée pour envoyer des messages de modification de tâche à la tâche source.

  • Informations sur le nouveau processus (réponse uniquement), y compris l'URL utilisée pour vérifier le statut du processus créé.

  • Informations sur le statut de la tâche (réponse uniquement). La réponse peut définir le statut de la tâche sur Terminé pour terminer la tâche source sans envoyer de message de modification de tâche.

  • Commentaires (message et réponse).

  • Attributs de workflow (message et réponse).

  • Formulaires de workflow (message et réponse).

  • Pièces jointes au workflow (message uniquement).

Lors d'un événement de modification de tâche, les informations suivantes peuvent être échangées :

  • Informations sur le statut de la tâche (message uniquement). Le message peut définir le statut de la tâche sur Terminé pour la terminer.

  • Commentaires (message et réponse).

  • Attributs de workflow (message et réponse).

  • Formulaires de workflow (message et réponse).

  • Pièces jointes au workflow (message uniquement).

Les messages et réponses de données de formulaire XML et HTML contiennent plus ou moins les mêmes informations ; chaque format peut être personnalisé dans le concepteur de workflow. Le choix du format dépend de votre application. Lorsque vous choisissez un format, tenez compte des points suivants :

  • Les données de formulaire HTML sont plus faciles à traiter par les technologies traditionnellement utilisées pour interagir avec les formulaires HTML, comme les servlets, les applications CGI et les pages .asp.

  • Les documents joints à un workflow peuvent être envoyés uniquement au format HTML. Le format XML comprend uniquement l'ID de dossier de la pièce jointe.

  • Le concepteur de workflow Content Server permet de personnaliser des formats de messages de données de formulaire HTML simples. Par exemple, vous pouvez changer le nom des champs du formulaire utilisés pour transférer chaque élément de données.

  • Les formats de message XML sont plus susceptibles d'êtres compatibles avec les versions actuelles et futures de XML Workflow Interchange.

  • Les messages XML peuvent contenir des informations sur la plage des valeurs autorisées dans les attributs et les formulaires.

  • Lorsque vous renseignez des attributs et des formulaires, les messages XML permettent de contrôler les champs hérités de versions précédentes.

  • Dans les messages XML, il est possible de supprimer les valeurs existantes d'un attribut.

OpenText recommande l'utilisation des messages XML par défaut pour la connexion des workflows Content Server et des données de formulaire HTML par défaut pour l'envoi de messages de lancement aux servlets Java et applications CGI personnalisés.
