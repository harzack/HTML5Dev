Les propriétés configurables des actions sont réparties sur plusieurs pages. La page Général permet de configurer les paramètres déterminant la façon de nommer, d'envoyer et de recevoir le message. Vous pouvez configurer les propriétés suivantes :

  • Propriétés des actions d'acceptation XML, qui peuvent être configurées dans des tickets de lancement à distance et des tâches Workflow Interchange. Sur la page Général de ces types d'actions, déterminez la façon dont les erreurs sont rapportées à l'expéditeur. Les autres onglets permettent de changer le format et le contenu des messages.

  • Propriétés des actions de lancement XML et HTML, qui, lorsqu'une étape Workflow Interchange est prête, envoient un message à une URL spécifique en configurant les propriétés des actions de lancement XML ou HTML pour cet événement. Sur la page de propriétés Général de ces types d'actions, définissez les paramètres qui contrôlent l'emplacement où les messages sont envoyés et la façon dont ils sont envoyés.

  • Propriétés de réponse XML aux actions de lancement à distance, qui, lorsqu'une étape Workflow Interchange est prête, renvoient un message de réponse à la tâche source qui a lancé le workflow si celui-ci a été lancé à distance et qu'une action de lancement à distance de réponse XML a été configurée. Sur la page Général de ce type d'action, vous pouvez définir le nom de l'action et déterminer si la tâche source doit ou non être transférée.
