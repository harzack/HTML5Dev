Cette version de XML Workflow Interchange peut être facilement personnalisée et propose des formats de message par défaut légèrement différents des versions précédentes. Pour échanger des messages XML avec des versions précédentes de XML Workflow Interchange, tenez compte des points suivants :

  • Les versions précédentes utilisent toujours des espaces de noms XML. Sur la page Format de chaque action qui prévoit l'échange de messages avec une version antérieure à la version 1.1.0 de XML Workflow Interchange, sélectionnez toujours la case à cocher Utiliser et exiger des espaces de noms XML.

  • Les versions précédentes convertissent les espaces en traits de soulignement dans les noms d'attributs Content Server à valeurs multiples. Vérifiez toujours que les noms des attributs échangés avec des versions antérieures à la version 1.1.0 ne contiennent pas d'espaces.

  • Les versions précédentes produisent toujours des schémas d'attributs sur les messages sortants mais n'exigent pas que ces schémas soient présents dans les messages entrants. Si vous transmettez des messages XML à des processus personnalisés rédigés en vue d'être compatibles avec une version précédente et si ces processus nécessitent ces informations, cochez la case Inclure des schémas d'attribut dans les messages sortants pour les actions qui envoient ces messages.

  • Si vous utilisez ou importez un schéma de workflow qui comprend des étapes Workflow Interchange créées avec des versions précédentes, les options appropriées sont automatiquement sélectionnées pour garantir la compatibilité.
