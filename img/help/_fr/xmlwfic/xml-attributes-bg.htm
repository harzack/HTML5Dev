Les attributs de workflow sont placés dans le lot d'attributs.

Représentation XML

<attributes>

<attribute name="my att">

<value>…</value>

…

</attribute>

…

En XML, le lot d'attributs est un élément attributes ayant un élément attribute enfant pour chaque attribut. Chaque élément attribut fournit le nom de l'attribut et dispose d'un élément enfant pour chaque valeur d'attribut définie.

Lorsque le lot d'attributs est mis à jour, les attributs non mentionnés conservent leurs valeurs précédentes. Pour supprimer les valeurs d'un attribut, vous devez fournir un élément attribut avec des éléments enfant sans valeur.

Espaces de noms XML (le cas échéant)

<attributes xmlns="http://www.opentext.com/llx/workflow/">

Représentation HTML

attributes_my_att_1=…

attributes_my_att_2=…

…

Dans les données de formulaire HTML, les noms de champs des attributs sont générés à partir du nom des attributs correspondants. Etant donné que Content Server accepte les valeurs multiples pour les attributs, chaque valeur de chaque attribut est numérotée. Les noms de base HTML de tous les attributs sont présents dans l'onglet de configuration de format personnalisé pour toutes les actions de l'étape du workflow.

Lorsque les attributs sont mis à jour avec un message de données de formulaire HTML, vous pouvez écraser les valeurs avec le champ numéroté approprié mais vous ne pouvez pas supprimer les valeurs qui existent déjà.
