Les informations relatives aux pièces jointes du workflow sont fournies le cas échéant.

Représentation XML

<attachments folderid="…"/>

Le format XML du lot de travail de pièces jointes comprend uniquement l'ID d'objet du dossier dans lequel se trouvent les pièces jointes. Pour réellement transférer les pièces jointes, une requête séparée est nécessaire dans Content Server en utilisant d'autres API.

 

Note :
Pour inclure des informations relatives aux pièces jointes qui comprennent des pièces jointes elles-mêmes dans le lot de travail de la tâche, utilisez un message de données de formulaire HTML. Pour plus d'informations, reportez-vous à Représentation HTML ci-après.

 

Espaces de noms XML (le cas échéant)

<attachments xmlns="http://www.opentext.com/llx/workflow/">

Représentation HTML

attachments_folderid=…

attachments_documents_name_1=…

attachments_documents_content_1=…

…

Dans les données de formulaire HTML, les informations relatives aux pièces jointes peuvent inclure les pièces jointes elles-mêmes. Pour qu'une action d'envoi comprenne les documents ou qu'une action d'acceptation accepte les documents en pièce jointe, vous devez personnaliser le champ des pièces jointes en utilisant l'onglet de personnalisation du format de l'action. Les documents en pièce jointe ne sont jamais inclus dans les messages OK renvoyés en réponse à une requête.

Les pièces jointes transférées comprennent uniquement les objets des documents inclus dans le dossier des pièces jointes. Chaque objet produit un groupe numéroté de deux champs :

  • un champ de nom, qui affiche le nom du document tel qu'il apparaît dans Content Server ;

  • un champ de contenu, qui fournit le contenu du document. Le champ de contenu est conforme au format utilisé par les navigateurs Web pour charger les fichiers.
