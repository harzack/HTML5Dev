Pour filtrer le contenu d'un message XML :

  1. Double-cliquez sur l'étape Workflow Interchange, puis cliquez sur l'onglet Actions.

  2. Cliquez sur l'icône Modifier  pour Accepter un message de lancement XML.

  3. Cliquez sur l'onglet Filtre.

  4. Sélectionnez la case à cocher en regard de chaque champ à inclure. Deux types d'éléments apparaissent : les champs et les groupes de champs. Un groupe de champs est un ensemble de champs et de sous-groupes qui forment une unité logique.

     

    Note :
    Les champs que vous incluez apparaissent dans les messages sous forme d'élément ou d'attribut XML dans son format par défaut. Pour plus d'informations sur le format par défaut des champs XML, reportez-vous à Champs et formats de messages.

     

  5. Facultatif Décochez la case Statut de la tâche pour ne pas inclure un champ dans tous les messages que l'action envoie et ignorer le champ dans tous les messages que l'action reçoit.

  6. Sélectionnez la case à cocher Titre de la tâche pour ajouter le titre de la tâche à tous les messages envoyés et reçus.

     

    Note :
    Dans l'Etape de départ du schéma de workflow, la case à cocher Demander un titre à l'initialisation doit être sélectionnée pour cet élément du workflow.

     

  7. Dans la liste, cliquez sur un format pour chaque groupe de champs. Tous les éléments du workflow valides dans les messages apparaissent de façon individuelle.

  8. Facultatif Sélectionnez la case à cocher Inclure des schémas d'attribut dans les messages sortants pour spécifier que chaque ensemble d'attributs à valeurs multiples Content Server comporte un contenu XML supplémentaire qui décrit le type et le nombre de valeurs autorisés pour chaque attribut.

  9. Cliquez sur Appliquer.

Pour plus d'informations sur cette procédure, reportez-vous à la rubrique Personnalisation des messages XML.
