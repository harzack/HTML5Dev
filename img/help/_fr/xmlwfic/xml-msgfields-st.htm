XML Workflow Interchange effectue tous ses échanges de messages via le protocole HTTP. Chaque interaction est constituée de deux parties :

  • Un client se connecte au serveur cible via Internet et envoie une requête HTTP à une URL spécifique sur le serveur en question. La requête définit une opération que le client souhaite que le serveur cible réalise et fournit toutes les informations supplémentaires dont le serveur pourrait avoir besoin pour traiter la requête.

  • Le serveur cible envoie une réponse HTTP au client via la même connexion. La réponse indique si l'opération demandée a ou non réussi et fournit les résultats de l'opération au client.

L'ensemble de l'échange est synchrone. Certains processus du client peuvent attendre une réponse du serveur. Lorsque XML Workflow Interchange envoie une requête HTTP, il signale une erreur et ferme la connexion au serveur s'il ne reçoit pas de réponse dans la minute qui suit.
