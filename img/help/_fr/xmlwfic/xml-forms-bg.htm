Si le lot de travail de formulaires est inclus dans le schéma, les messages peuvent inclure des données de tous les formulaires qui sont disponibles pour l'étape. Chaque action concernée par l'échange traite uniquement les formulaires disponibles à l'étape correspondante. Les règles suivantes s'appliquent :

  • Une action qui accepte un message de lancement à distance met uniquement à jour les formulaires disponibles à l'étape de départ du workflow.

  • Une action acceptant un message de modification de tâche met uniquement à jour les formulaires disponibles dans la tâche de l'action.

  • Une action qui accepte un message de modification de tâche met uniquement à jour les formulaires disponibles dans la tâche de l'action.

Représentation XML

<forms>

<form name="my form">

<attribute name="my att">

<value>…</value>

…

</attribute>

</form>

</forms>

En XML, le lot de formulaires est un élément forms qui contient un élément form enfant pour chaque formulaire disponible. Chaque élément form détermine le nom du formulaire et contient des éléments attribute pour chaque attribut du formulaire. Chaque élément attribut fournit le nom de l'attribut et dispose d'un élément enfant pour chaque valeur d'attribut définie. Pour les attributs d'un ensemble de types, chaque valeur contient davantage d'éléments attribute enfant.

Lorsque le lot de formulaire est mis à jour, les formulaires non mentionnés conservent leurs valeurs précédentes et les attributs non mentionnés au sein d'un formulaire conservent leurs valeurs précédentes. Pour supprimer les valeurs d'un attribut, vous devez fournir un élément attribut avec des éléments enfant sans valeur.

Espaces de noms XML (le cas échéant)

<forms xmlns="http://www.opentext.com/llx/workflow/">

Représentation HTML

forms_my_form_my_att_1=…

forms_my_form_my_att_2=…

…

Dans les données de formulaire HTML, les noms des champs des attributs de formulaires sont générés à partir du nom de leurs attributs et du nom des formulaires dans lesquels ils apparaissent. Les noms de base HTML de tous les attributs apparaissent dans l'onglet de configuration du format personnalisé pour toutes les actions de l'étape du workflow.

Etant donné que Content Server accepte les valeurs multiples pour les attributs, chaque valeur de chaque attribut est numérotée. Les valeurs des sous-attributs dans les attributs définis ont deux index. Par exemple, forms_my_form_setatt_textatt_2_3 nomme la troisième valeur de l'attribut textatt dans la seconde valeur de l'attribut setatt.

Lorsque les attributs de formulaire sont mis à jour avec un message de données de formulaire HTML, vous pouvez écraser les valeurs avec le champ numéroté approprié, mais vous ne pouvez pas supprimer les valeurs qui existent déjà.
