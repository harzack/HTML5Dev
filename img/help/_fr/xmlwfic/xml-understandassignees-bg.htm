Une tâche Workflow Interchange active dans un workflow représente un processus qui s'exécute sur un système distant nécessaire pour renvoyer une réponse. Toutefois, chaque étape Workflow Interchange est affectée à un utilisateur Content Server local qui contrôle l'exécution de l'étape et gère les éventuelles difficultés. Par exemple, si le système cible rencontre un problème, la réponse attendue risque de ne jamais arriver. Sans l'intervention de l'utilisateur, le workflow source est retardé et reste en attente d'un message de modification de tâche pour terminer la tâche.

Les tâches d'échange actives apparaissent dans la liste des affectations de l'utilisateur. Lorsque la tâche est active, l'utilisateur peut vérifier la progression des différents messages envoyés, renvoyer des messages, afficher le statut de tous les messages de modification de tâche reçus ou terminer la tâche avant de recevoir une réponse.

L'utilisateur avec une tâche assignée peut également afficher les données de lots de travail et les modifier manuellement, de la même façon que celui d'une étape Utilisateur.
