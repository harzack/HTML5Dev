Les informations sur le nouveau processus sont renvoyées en réponse à un message de lancement lorsque ce message démarre un processus sur le système récepteur.

Les informations sur le nouveau processus comprennent deux champs de variables :

  • Un descripteur opaque, qui identifie le nouveau processus au sein du système cible. Si cette variable est présente, elle ne sera jamais réutilisée.

  • Une URL, qui est disponible si le statut du nouveau processus peut être vérifié avec une interface Web.

Représentation XML

<newproc handle="…">

<target url="…" type="browser"/>

</newproc>

En XML, les informations sur le nouveau processus sont constituées d'un élément newproc contenant un ou plusieurs éléments cibles. Les éléments cibles se distinguent par leurs attributs.

Pour le moment, seuls les navigateurs sont pris en charge. Tous les éléments newproc ont donc exactement un élément enfant cible. Les systèmes qui traitent ces messages XML doivent toutefois anticiper la possibilité d'autres types cibles à l'avenir.

Espaces de noms XML (le cas échéant)

<newproc xmlns="http://www.opentext.com/llx/Workflow/">

<target xmlns="http://www.opentext.com/llx/">

Représentation HTML

newprocess_handle=…

newprocess_browser_url =…
