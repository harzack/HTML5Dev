Les feuilles de style utilisées pour traiter les messages dans XML Workflow Interchange doivent être conformes à la version 1.0 de la spécification XSLT (http://www.w3.org).

Lorsqu'elle est utilisée pour traduire un message ou un accusé de réception entrant, la feuille de style reçoit le message du système externe en tant qu'entrée. A la sortie, la feuille de style produit l'un des éléments ci-dessous :

  • Un message au format XML par défaut pour l'action, lequel est ensuite traité par XML Workflow Interchange.

  • Un élément error qui décrit un problème lié au message entrant. Si l'action fait partie d'une étape Workflow Interchange, l'erreur est consignée dans le journal et s'affiche pour l'utilisateur de l'étape en cours.

  • Un élément OK qui indique une réussite. Le message entrant n'est pas traité, mais l'action est considérée comme réussie. Une feuille de style peut renvoyer un message OK pour ignorer un message de lancement ou de modification de tâche entrant.

Lorsqu'une feuille de style est utilisée pour traduire un message ou un accusé de réception sortant, le système Workflow Interchange génère le message sortant au format XML par défaut, l'encapsule dans un élément sendxml et le transmet à la feuille de style en tant qu'entrée. Etant donné que les messages sortants se distinguent grâce à l'élément sendxml, la même feuille de style peut souvent être utilisée pour les messages entrants et sortants.

A la sortie, la feuille de style produit l'un des éléments ci-dessous :

  • Un élément sendxml contenant le nouveau message sortant. Ce message est envoyé au système cible.

  • Une réponse au format XML par défaut pour l'action. Aucun message n'est envoyé au système cible, mais le résultat de la feuille de style est traité comme s'il était reçu en tant qu'accusé de réception d'un envoi réussi.

  • Un élément error qui décrit un problème lié au message sortant. L'erreur est consignée dans le journal et s'affiche pour l'utilisateur de l'étape en cours ; aucun message n'est envoyé.

  • Un élément OK qui indique une réussite. Aucun message sortant n'est envoyé.

Pour plus d'informations sur les formats de messages par défaut et les éléments sendxml, error et OK, reportez-vous à Champs et formats de messages.
