Vous avez la possibilité de configurer les actions d'une étape Workflow Interchange. Cela vous permet de déterminer les actions réalisées lorsqu'une étape Workflow Interchange est prête ou lorsqu'une tâche active reçoit une requête de modification de tâche.

Pour configurer des actions d'une étape de Workflow Interchange :

  1. Dans l'éditeur de schéma de workflow, double-cliquez sur une étape Workflow Interchange.

  2. Cliquez sur l'onglet Actions.

  3. Dans la zone Etape prête, sélectionnez l'une des actions suivantes dans la liste Lorsque cette étape est en cours de préparation :

    • Envoyer un message de lancement XML, qui envoie un message de lancement au format XML à une URL spécifique une fois l'étape prête, par exemple pour lancer un workflow dans un autre système Content Server ;

    • Envoyer un message de lancement HTML, qui envoie un message de lancement au format de données de formulaire HTML à une URL spécifique une fois l'étape prête, par exemple pour envoyer des données de workflow à un servlet personnalisé ;

    • Envoyer une réponse XML à l'initiateur à distance, qui envoie un message de modification de tâche à l'URL de rappel une fois l'étape prête, si le workflow est lancé en utilisant le mécanisme de lancement à distance. L'initiateur à distance fournit une URL de rappel dans le message de lancement. Les paramètres de cette action comprennent les informations mises à jour sur les attributs et une demande de suivi de la tâche source. Utilisez cette action à la fin des workflows prenant en charge le lancement à distance. Pour plus d'informations sur le lancement à distance, reportez-vous à Activation du lot de travail de lancement à distance.

     

    Note :
    Si l'option Envoyer un message de lancement XML ou Envoyer un message de lancement HTML était sélectionnée, vous devez cliquer sur l'icône Modifier  pour définir l'URL cible de l'action. Pour plus d'informations sur la configuration d'actions, voir Configuration des propriétés d'une action de lancement XML et HTML.

     

  4. Répétez l'étape 3 pour chaque action à ajouter.

  5. Dans la zone Exécution de l'étape, sélectionnez l'une des options suivantes dans la liste Lorsque les actions ci-dessus ont été exécutées :

    • Attente d'une requête

    • Terminer la tâche automatiquement

  6. Si vous sélectionnez l'option Attente d'une requête, sélectionnez l'une des options suivantes dans la liste Lorsqu'une requête est reçue :

    • Rejeter, qui rejette toutes les requêtes reçues. Toute tentative d'envoi d'une requête à une tâche renverra une erreur HTTP.

    • Accepter un message de modification de la tâche XML, qui permet à la tâche de traiter les messages de modification de tâche XML, par exemple pour accepter les réponses XML des workflows que vous lancez à distance sur d'autres systèmes Content Server.

    • Accepter un message de modification de la tâche HTML, qui permet à la tâche de traiter les messages de modification de tâche avec données de formulaires HTML, par exemple pour accepter les messages de systèmes hérités ou de formulaires HTML, ou si vous souhaitez qu'il soit possible de joindre des documents d'autres systèmes au workflow.

  7. Cliquez sur Ajouter à la définition du workflow, puis enregistrez le workflow.

Pour plus d'informations sur cette procédure, reportez-vous à la rubrique Configuration des actions d'une étape de Workflow Interchange.
