Un message OK peut être renvoyé en réponse à une requête HTTP ; il indique que la requête a réussi.

Représentation XML

<ok>…</ok>

Au format XML par défaut, un message OK est un élément OK. Le contenu de l'élément peut être utilisé pour renvoyer des champs mis à jour ou de nouvelles informations.

Espaces de noms XML (le cas échéant)

<ok xmlns="http://www.opentext.com/llx/">

Représentation HTML

Un message OK HTML peut être un document HTML ou un document contenant des données de formulaires en plusieurs parties transportant les champs mis à jour ou les informations renvoyées.

Contenu du message

Un message OK contient les éléments suivants :

  • statut de la tâche ;

  • Nouveau processus

  • lots de travail.

Le contenu d'un message OK doit être au même format que le contenu du message de lancement ou de modification de tâche de la requête correspondante.

Un message OK généré par une action XML Workflow Interchange aura un contenu dans deux circonstances :

  • en cas de réponse de réussite à une requête de lancement à distance et si la réponse comporte des informations sur le nouveau processus ;

  • en cas de réponse de réussite à une requête XML définissant l'attribut query="yes" et si la réponse comporte des informations sur le lot de travail mis à jour.

Lorsqu'il répond à un message de lancement généré par XML Workflow Interchange, un processus personnalisé (par exemple un CGI ou un servlet) peut également comporter des informations sur le statut de la tâche de façon à transférer explicitement la tâche source et les informations sur le lot de travail pour mettre à jour les champs de la tâche source.
