Lorsque des actions XML sont configurées de façon à traiter des messages avec des feuilles de style, tous les messages sortants sont inclus dans des éléments sendxml avant d'être envoyés à la feuille de style pour être convertis. La feuille de style doit produire un élément sendxml pour pouvoir envoyer le message converti. Ceci permet à une feuille de style de distinguer les messages entrants et sortants, et de contrôler dans une certaine mesure le traitement des messages en plus de leur format.

Représentation XML

<sendxml filename="…" contenttype="…">…</sendxml>

L'enveloppe sendxml est un élément sendxml qui contient le message à envoyer.

Espaces de noms XML (le cas échéant)

<sendxml xmlns="http://www.opentext.com/llx/">

Représentation HTML

L'enveloppe sendxml n'est pas utilisée dans les échanges HTML.

Contenu du message

Une enveloppe sendxml contient les éléments suivants :

  • Nom de fichier

  • type de contenu ;

  • message.

Une enveloppe sendxml doit contenir un élément enfant unique pour le message à envoyer. Les attributs filename et contenttype sont facultatifs et peuvent être utilisés pour modifier le traitement du message. Si le message est envoyé à un répertoire à l'aide du protocole PUT HTTP, l'attribut filename contient le nom de fichier à utiliser pour stocker le message dans ce répertoire. L'attribut contenttype contient le type MIME du message à envoyer. La valeur par défaut est text/xml, mais elle peut être modifiée par une feuille de style. Si la feuille de style change le type de contenu en text/html, le message est écrit en utilisant les règles de sortie XSLT pour HTML.
