Un message de modification de tâche peut être envoyé par une action « d'envoi d'une réponse XML à l'initiateur à distance » et accepté par une action « d'acceptation d'un message de modification de la tâche XML » ou « d'acceptation d'un message de modification de la tâche HTML ».

Représentation XML

<exec><edittask status="…" query= "…">...</edittask></exec>

Au format XML par défaut, un message de lancement est constitué d'un élément edittask situé à l'intérieur d'un élément exec. L'élément edittask contient le message.

Si une action d'acceptation reçoit un message de modification de tâche portant l'attribut query="yes", elle renverra des valeurs mises à jour pour tous les champs valides.

Espaces de noms XML (le cas échéant)

<exec xmlns="http://www.opentext.com/llx/">

<edittask xmlns="http://www.opentext.com/llx/Workflow/">

Lorsque l'action est configurée de façon à utiliser et demander des espaces de noms XML, différents espaces de noms sont utilisés pour les éléments exec et edittask.

Représentation HTML

Dans les données de formulaire HTML, un message de modification de tâche est représenté par son contenu.

Contenu du message

Un message de modification de tâche contient les éléments suivants :

  • statut de la tâche ;

  • lots de travail.

Un message de modification de tâche contient un sous-ensemble des informations qu'un message de lancement peut contenir. Il transporte uniquement les données du lot de travail et les informations de statut relatives à la tâche modifiée. Pour plus d'informations sur ces éléments, reportez-vous à Informations sur la tâche et Lots de travail.
