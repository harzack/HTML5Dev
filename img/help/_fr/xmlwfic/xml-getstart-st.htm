XML Workflow Interchange permet aux workflows Content Server d'interagir avec d'autres systèmes grâce à un échange de messages HTTP. Le module dispose des fonctionnalités suivantes qui, ensemble, prennent en charge de nombreux scénarios de connectivité :

  • L'étape Workflow Interchange, qui est ajoutée aux workflows Content Server à l'aide de l'éditeur de workflow. Une étape peut être configurée de façon à envoyer des messages lorsqu'elle est prête dans un workflow actif et à accepter des messages lorsqu'elle est active.

  • La fonctionnalité de lancement à distance, qui est activée pour chaque schéma de workflow. Les schémas peuvent être lancés au moyen d'un message HTTP envoyé à une URL spéciale. Le lancement à distance accepte les types de messages que les étapes Workflow Interchange peuvent envoyer. Ainsi, les workflows d'un système Content Server peuvent être utilisés comme sous-schémas d'un autre système.
