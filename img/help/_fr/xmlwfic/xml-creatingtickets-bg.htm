Chaque ticket de lancement à distance que vous créez est associé à une URL à laquelle les messages de lancement à distance peuvent être envoyés. Les tickets ont également des paramètres qui permettent de contrôler le processus de lancement.

Le lancement à distance peut être désactivé. Cependant, lorsque vous désactivez le lancement à distance, les informations de configuration de tous les tickets de lancement à distance sont enregistrées, de telle sorte que vous pouvez les réutiliser si vous souhaitez réactiver le lancement à distance.

 

Note :
Le schéma de workflow est lancé à réception d'un message de lancement. Les informations sur le nouveau workflow sont renvoyées dans la réponse HTTP, avec une URL qui peut être utilisée pour vérifier son statut.
