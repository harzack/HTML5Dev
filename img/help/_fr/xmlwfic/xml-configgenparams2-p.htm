Pour configurer les propriétés d'une action de lancement XML et HTML :

  1. Double-cliquez sur l'étape Workflow Interchange, puis cliquez sur l'onglet Actions.

  2. Dans la zone Etape prête, sélectionnez la liste Exécuter l'action, puis sélectionnez Envoyer un message de lancement XML ou Envoyer un message de lancement HTML.

  3. Cliquez sur l'icône Modifier  d'une action.

  4. Saisissez le nom de l'action dans le champ Nom.

     

    Note :
    le nom apparaît dans la liste des actions configurées pour l'étape mais pas dans les messages. Vous pouvez l'utiliser pour décrire l'objectif d'une action d'envoi ou pour faire la distinction entre les différentes actions du même type configurées pour la même étape.

     

  5. Dans le champ URL cible, saisissez l'URL absolue à laquelle le message doit être envoyé. Par exemple http://www.opentext.com.

  6. Si le système cible est un serveur Web prenant en charge l'authentification HTTP Basic ou une instance Content Server prenant en charge l'authentification par cookies Content Server, saisissez un nom d'utilisateur et un mot de passe dans la zone Authentification. Ces informations d'authentification sont fournies au serveur Web cible le cas échéant.

     

    Note :
    si les informations d'authentification sont requises, OpenText vous recommande de créer un utilisateur dans ce but précis. 

     

  7. Dans la zone Méthode HTTP, cliquez sur l'une des options suivantes pour choisir la méthode à utiliser pour envoyer le message au système cible :

    • POST, utilisée par un navigateur Web pour soumettre des formulaires. Cette option doit être sélectionnée lorsque vous envoyez des données à un processus de serveur.

    • PUT, utilisée pour enregistrer les documents dans un répertoire inscriptible du serveur Web.

       

      Note :
      si vous sélectionnez PUT dans la liste de la zone Méthode HTTP, le dernier segment de l'URL cible (la partie située après la dernière barre oblique vers l'avant) correspond au nom du fichier créé dans le répertoire de destination. Si l'URL cible se termine par une barre oblique vers l'avant, un nom de fichier par défaut est généré automatiquement. Les noms de fichier par défaut générés de cette façon sont différents pour toutes les tâches Workflow Interchange de chaque workflow lancé.

       

  8. Facultatif Si vous savez que le système cible nécessite des en-têtes HTTP personnalisés pour traiter correctement le message, vous pouvez les ajouter dans la zone En-têtes supplémentaires. Saisissez le nom et la valeur de l'en-tête dans les champs Nom et Valeur, puis cliquez sur l'icône Ajouter une valeur .

  9. Cliquez sur Appliquer.

Pour plus d'informations sur cette procédure, reportez-vous à la rubrique Configuration des paramètres d'action généraux.
