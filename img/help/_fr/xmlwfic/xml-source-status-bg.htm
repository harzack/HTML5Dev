Les informations sur la tâche source sont transportées dans un message de lancement qui donne des informations au système cible concernant le processus client.

Les informations sur la tâche source comprennent deux champs de variables :

  • Un descripteur opaque, qui identifie la tâche au sein du système source. Si ce champ est présent, il ne sera jamais réutilisé pour une autre tâche source. Un système cible peut utiliser cette valeur pour détecter les messages en double.

  • Une URL HTTP, à laquelle des messages peuvent être envoyés. Ce champ n'est disponible que si la tâche source peut être configurée de façon à accepter des messages de modification de tâche via HTTP.

Représentation XML

<sourcetask handle="…">

<target url="…" type="HTTP"/>

</sourcetask>

En XML, les informations sur la tâche source sont constituées d'un élément sourcetask contenant un ou plusieurs éléments cibles. Les éléments cibles se distinguent par leurs attributs.

Seules les cibles HTTP sont prises en charge. Tous les éléments sourcetask ont exactement un élément enfant cible. Les systèmes qui traitent ces messages XML doivent toutefois anticiper la possibilité d'autres types cibles à l'avenir.

Espaces de noms XML (le cas échéant)

<sourcetask xmlns="http://www.opentext.com/llx/Workflow/">

<target xmlns="http://www.opentext.com/llx/">

Représentation HTML

sourcetask_handle=…

sourcetask_http_url=…
