Une pièce jointe de schéma de workflow est un élément Content Server ou un fichier qui contient des informations relatives au schéma de workflow. Les intervenants du schéma de workflow peuvent avoir pour instructions de consulter, de modifier ou d'ajouter des pièces jointes au schéma de workflow. Par exemple, une pièce jointe peut être un document contenant des instructions concernant la tâche ou une feuille de calcul que l'utilisateur avec une tâche assignée doit remplir pour terminer le travail défini pour l'étape.

Lorsqu'une pièce jointe est ajoutée au schéma de workflow, Content Server place une copie de cet élément dans le dossier Pièces jointes. Les concepteurs de schéma de workflow et les intervenants peuvent ajouter, consulter, modifier et exécuter d'autres opérations sur les pièces jointes de la même manière qu'avec des éléments placés dans un autre emplacement de Content Server.

Vous ne pouvez pas utiliser de pièces jointes avant de les activer dans les propriétés générales du schéma de workflow. Pour plus d'informations sur l'activation des pièces jointes dans un schéma de workflow, reportez-vous à la rubrique Définition des propriétés générales.

Les pièces jointes à un schéma de workflow peuvent être les éléments suivants :

  • Des éléments Content Server que vous créez dans le dossier Pièces jointes.

  • Des éléments copiés à partir de Content Server vers le dossier Pièces jointes.

  • Des fichiers que vous ajoutez depuis votre ordinateur ou un système de fichiers réseau.

 

Note :
Si vous préférez que les intervenants du schéma de workflow utilisent les éléments d'origine, ajoutez un raccourci vers le dossier Pièces jointes. Un raccourci offre un accès direct à l'élément d'origine.

 

Définition des autorisations

Lorsqu'un élément est ajouté en tant que pièce jointe, il hérite des autorisations du dossier Pièces jointes du schéma de workflow. Par défaut, il s'agit de toutes les autorisations. Tous les utilisateurs peuvent effectuer une recherche dans les pièces jointes du schéma de workflow, même s'ils n'ont pas l'autorisation sur le schéma de workflow. Vous pouvez modifier les autorisations de pièces jointes dans un workflow pour que seuls les utilisateurs ou les groupes qui utilisent le schéma de workflow ou y participent puissent accéder à l'élément. L'administrateur peut, en outre, autoriser les pièces jointes pour tous les schémas de workflow actifs ou interrompus.

 

Note :
Les utilisateurs ne disposant pas de l'autorisation Voir pour des pièces jointes ne peuvent pas lancer le schéma de workflow, ni participer aux étapes de schéma de workflow qui leur sont envoyées via les options Déléguer ou Envoyer pour révision.
