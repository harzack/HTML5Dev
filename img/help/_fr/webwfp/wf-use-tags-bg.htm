Vous pouvez utiliser des balises de remplacement dynamiques dans un schéma de workflow pour personnaliser les workflows lancés.

Exemple: Placez la balise <Initiator /> dans le titre d'un workflow, de manière à ce qu'il apparaisse sous le nom « schéma de workflow de Jean » après le lancement. Pour ce faire, saisissez <schéma de workflow de Initiator /> dans le champ Titre du schéma de workflow, sur la page Paramètres généraux.

Les balises de remplacement dynamique suivantes sont disponibles :

<Initiateur />

Indique le nom d'affichage de l'initiateur.

<InitiatedDate />

Indique la date à laquelle le workflow a été lancé.

<InitiatorMail />

Indique l'adresse e-mail de l'initiateur.

<WorkflowTitle />

Indique le titre du workflow.

<WorkID />

Indique le numéro du workflow.

<ParentTitle />

Indique le titre du workflow principal d'un sous-workflow.

<SubInitiatedDate />

Indique la date à laquelle un sous-workflow a été lancé.

<SubWorkTitle />

Indique le titre du sous-workflow.

<DataType_1_3_AttrName />

Balise de substitution d'un type de données d'attribut à valeurs multiples. Par exemple, si vous avez un attribut Langues auquel sont assignées les valeurs Français, Allemand et Anglais, vous pouvez l'utiliser de la même manière que les autres balises. La syntaxe pour un attribut à valeur unique est la même : <DataType_1_3_AttrName />.

Balise Résultat
<DataType_1_3_Languages[3] />  Anglais
<DataType_1_3_Langues[*] /> Français, Allemand, Anglais
<DataType_1_3_Langues /> Français, Allemand, Anglais

Vous ne pouvez saisir les balises de remplacement dynamiques que sous la forme ci-dessus.

 

Note :
Les balises de remplacement dynamiques <SubWorkID /> ne sont pas prises en charge actuellement.

 
 

Important :
Le remplacement dynamique est effectué au lancement du workflow, et les valeurs de remplacement ne sont pas mises à jour en cours de workflow, même en cas de modification des valeurs de base par des utilisateurs ou événements système. Pour que le remplacement dynamique réussisse, la balise de remplacement dynamique doit faire référence à une variable possédant une valeur au moment où le workflow est lancé.

Exemple: Utilisez la balise <DataType_1_3_AttrName /> dans les commentaires de processus de workflow. Le remplacement dynamique ne peut réussir si la valeur d'attribut est nulle au moment où le workflow est lancé, et que cette valeur n'est définie qu'au cours de l'exécution du workflow. La valeur doit être spécifiée avant le démarrage du workflow.

Utilisation des paramètres de substitution de type de données d'attribut multi-valeur

Les balises de substitution d'un type de données d'attribut multi-valeur utilisent la syntaxe suivante :

<DataType_<x>_<y>_<NomAttr> />

où <x> représente le type, <y> le sous-type et <NomAttr> le nom de l'attribut. La valeur d'un type pour tous les lots de données du schéma de workflow (comme les pièces jointes, les commentaires, les attributs et les formulaires) est toujours 1. Le sous-type identifie le type de données. Par exemple, la valeur du sous-type pour des attributs de schéma de workflow est 3 et la valeur du sous-type d'un champ de formulaire est 4.

Utilisation des champs de formulaire

Si vous travaillez avec des champs de formulaire, les balises de substitution d'un type de données d'attribut à valeurs multiples utilisent la syntaxe suivante :

<DataType_<x>_<y>_<z>_<NomChamp> />

où <x> est le nombre ordinal du formulaire dans le schéma de workflow, <y> est 4 et <z> identifie le formulaire. Par exemple, si un schéma de workflow inclut trois formulaires, les valeurs valides pour z sont 1, 2 et 3. De même, si un schéma de workflow inclut un formulaire et que ce formulaire contient un champ nommé « Auteur », la balise de remplacement des données est <DataType_1_4_1_Auteur />.
