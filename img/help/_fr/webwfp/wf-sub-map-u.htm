Pour bénéficier du traitement exécuté dans un autre schéma de workflow, vous pouvez utiliser l'étape Sous-workflow dans votre schéma de workflow. Par exemple, si une partie de votre schéma de workflow nécessite une validation éditoriale, vous pouvez utiliser l'étape Sous-workflow pour intégrer un schéma de workflow qui exécute une révision éditoriale dans votre schéma de workflow. Lorsque le schéma de workflow intégré est terminé, le processus retourne à votre schéma de workflow principal.

Lorsque Content Server exécute une étape Sous-workflow, elle suspend le traitement du schéma de workflow principal pour lancer une instance de schéma de workflow basée sur le sous-workflow. Lorsque le travail de l'instance du sous-workflow est terminé, Content Server reprend le traitement du schéma de workflow principal. Vous indiquez la manière dont les informations et le lot de travail du schéma de workflow principal sont échangées avec le sous-workflow pour que le travail effectué dans le schéma de workflow intégré soit transmis au schéma de workflow principal.
