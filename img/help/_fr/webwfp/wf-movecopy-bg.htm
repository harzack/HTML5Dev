Vous pouvez configurer un schéma de workflow pour déplacer ou copier automatiquement des éléments Content Server et des pièces jointes au schéma de workflow vers un recueil Content Server (un dossier ou un projet, par exemple). Lorsque le processus du schéma de workflow atteint une étape Gestionnaire d'élément, il déplace ou copie les pièces jointes et les éléments en fonction des demandes (appelées définitions déplacer/copier) que vous avez définies dans son onglet Déplacer/copier. Une fois les définitions déplacer/copier créées, vous pouvez les modifier si vous le souhaitez.

Une définition déplacer/copier est composée de deux parties : la source, l'élément que vous souhaitez que l'étape Gestionnaire d'élément déplace ou copie, et la cible, le recueil Content Server vers lequel vous souhaitez que l'étape Gestionnaire d'élément déplace ou copie l'élément source.

La source peut être :

  • une pièce jointe du schéma de workflow, y compris le dossier Pièces jointes du schéma de workflow ;

  • un élément Content Server ;

  • une définition de dossier définie sur la même étape ; Pour plus d'informations sur les définitions de dossier, voir Création de dossier.

  • un attribut de référence d'élément qui renvoie à un élément Content Server.

La cible peut être un des éléments de recueil suivants :

  • le dossier Pièces jointes du schéma de workflow ou un de ses sous-dossiers ;

  • un élément de recueil Content Server, comme un espace de travail, un projet ou un dossier ;

  • une définition de dossier définie sur la même étape ;

  • un attribut de référence d'élément qui renvoie à un élément de recueil Content Server.

 

Notes :

  • Les définitions de dossier sont uniquement disponibles si vous avez défini plus d'une définition de dossier pour la même étape Gestionnaire d'élément.

  • Un attribut de référence d'élément peut uniquement servir de cible si vous en avez configuré un pour le schéma de workflow.

 

Définition des options de la source

Vous pouvez définir la manière dont l'étape déplace ou copie un élément de recueil et la manière dont elle nomme les éléments créés suite à la demande.

Utilisation des espaces de travail comme sources

Vous ne pouvez ni déplacer ni copier un espace de travail Entreprise ou Personnel, mais vous pouvez les sélectionner en tant que sources dans une demande de copie ou de déplacement d'étape Gestionnaire d'élément si vous souhaitez exécuter des opérations uniquement sur leur contenu. Par exemple, si vous sélectionnez un espace de travail Personnel comme source d'une demande de copie ou de déplacement et spécifiez que vous souhaitez que l'opération s'applique uniquement aux éléments qu'il contient (et non sur l'espace de travail lui-même), l'étape Gestionnaire d'élément déplace ou copie le contenu de l'espace de travail vers le recueil spécifié.

Utilisation du dossier Pièces jointes du schéma de workflow comme source

Vous pouvez spécifier le dossier Pièces jointes du schéma de workflow dans une demande de copie ou de déplacement de l'étape Gestionnaire d'élément, mais l'opération de déplacement produit le même résultat que l'opération de copie. Le dossier Pièces jointes du schéma de workflow est un recueil fixe disponible pour tous les schémas de workflow. Par conséquent, Content Server ne vous autorise pas à le déplacer. Si vous spécifiez le dossier Pièces jointes du schéma de workflow comme source dans une demande de déplacement, le système crée une copie du dossier Pièces jointes du schéma de workflow au sein du recueil spécifié.

Utilisation du contenu d'un recueil

Si la source est un recueil Content Server, tel qu'un dossier ou un espace de travail, vous pouvez spécifier que seul le contenu du recueil soit déplacé ou copié.

Changement de nom d'un élément déplacé ou copié

Vous pouvez indiquer un nouveau nom pour l'élément créé suite au déplacement ou à la copie.

 

Note :
Il est impossible de renommer les éléments Content Server si vous décidez de déplacer ou de copier uniquement le contenu d'un recueil.

 

Synchronisation des recueils

Si vous déplacez ou copiez un recueil, vous pouvez synchroniser le contenu du recueil cible. Cela permet de garantir que la dernière version de chaque élément qui existe dans les recueils source et cible est ajoutée au recueil cible. Par exemple, si un document existe à la fois dans l'emplacement source et dans l'emplacement cible, le document source est ajouté en tant que nouvelle version du document cible. En outre, si un document existe dans le recueil source mais pas dans l'emplacement cible, il est déplacé ou copié vers l'emplacement cible selon l'opération de transfert que vous spécifiez.

Création de versions principales de documents

Vous pouvez configurer une demande de déplacement ou de copie pour créer des versions principales de documents qui utilisent la création de version avancée dans le recueil cible, à condition que votre demande inclut les opérations suivantes :

  • Synchronisation des recueils

  • Déplacement ou copie du contenu du recueil uniquement

Supposons par exemple que vous vouliez synchroniser le contenu de deux dossiers appelés « Dossier 1 » (le recueil source) et « Dossier 2 » (le recueil cible) qui contiennent, entre autres éléments, un document nommé « RapportFinal » qui utilise la création de version avancée. Si vous configurez la demande de déplacement ou de copie pour créer des versions principales, le système ajoute la version actuelle de « RapportFinal » du « Dossier 1 » en tant que version principale de « RapportFinal » dans le « Dossier 2 » chaque fois que l'étape est traitée.

Définition des autorisations

Vous devez choisir l'une des méthodes suivantes pour indiquer quelles autorisations sont attribuées aux éléments déplacés ou copiés :

  • Original, qui attribue les autorisations de l'élément source à l'élément déplacé ou copié.

  • Destination, qui attribue les autorisations du recueil cible à l'élément déplacé ou copié.

 

Note :
Lorsque vous utilisez l'étape Gestionnaire d'élément pour déplacer un dossier du lot de travail du workflow vers un dossier dans l'espace de travail Entreprise à l'aide de l'option d'autorisations Destination, l'étape Gestionnaire d'élément hérite uniquement des autorisations pour le premier niveau de dossier.

 

Pour de plus amples informations sur les autorisations, reportez-vous à la section Autorisations.

Gestion des catégories

Vous devez choisir l'une des méthodes suivantes pour indiquer la manière dont sont gérés les catégories et les attributs Content Server dans les éléments déplacés ou copiés :

  • Original, qui applique les catégories de l'élément source à l'élément déplacé ou copié.

  • Destination, qui applique les catégories de l'élément cible à l'élément déplacé ou copié.

  • Fusionné, qui applique une combinaison des catégories de l'élément source et de celles de l'élément cible à l'élément déplacé ou copié.

 

Note :
Lorsqu'une catégorie existe dans l'élément source et l'élément cible, la catégorie source remplace la catégorie cible.

 

Pour plus d'informations sur les catégories et attributs, voir Utilisation des catégories et des attributs dans les schémas de workflow.

Utilisation de plusieurs attributs de référence d'élément

Si le schéma de workflow contient un attribut de référence d'élément, vous pouvez éventuellement l'utiliser comme source ou cible dans des demandes de déplacement ou de copie. Toutefois, vous ne pouvez pas utiliser plusieurs attributs de référence d'élément comme source et cible de la même demande. Par exemple, si vous spécifiez un attribut de référence d'élément qui contient des références à cinq documents et choisissez un dossier comme cible, l'étape déplace ou copie les cinq documents vers le dossier cible. Si vous spécifiez le même attribut de référence d'élément comme source, mais que vous spécifiez ensuite un autre attribut de référence d'élément se référant à plusieurs dossiers comme cible, une erreur se produit.

Par ailleurs, si vous spécifiez un attribut de référence d'élément de valeur comme cible dans une demande de déplacement, la source sera déplacée vers chaque élément référé par l'attribut source jusqu'à ce qu'elle atteigne le dernier élément où elle sera stockée. Par exemple, si vous spécifiez un document comme source et un attribut de référence d'élément se référant à trois dossiers comme cible, l'étape commence par déplacer le document vers le premier dossier, puis le déplace ensuite vers le deuxième dossier, et enfin vers le troisième dossier avant la fin du traitement de l'étape. L'historique d'expertise du document indique que le document a été déplacé vers chaque dossier ; chaque dossier est mis à jour, mais seul le troisième dossier contient le document source.

Pour plus d'informations sur les attributs de référence d'élément, reportez-vous à la rubrique Définition des attributs.

Copie de pièces jointes à l'aide d'une étape Processus

Vous pouvez utiliser une étape Processus pour copier une pièce jointe Content Server dans un recueil Content Server. Vous précisez la pièce jointe et le recueil Content Server dans l'onglet Pièces jointes de l'étape.

Vous devez indiquer les autorisations à utiliser au moment de la création de l'élément copié dans le recueil Content Server.

 

Note :
OpenText vous recommande vivement d'utiliser l'étape Gestionnaire d'élément pour copier des pièces jointes. Vous pouvez copier des pièces jointes à l'aide de l'étape Processus mais cette étape risque de ne plus être prise en charge dans les versions de Livelink à venir.
