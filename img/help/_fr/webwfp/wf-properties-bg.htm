Lorsque vous créez un schéma de workflow, vous pouvez définir les caractéristiques générales suivantes du schéma de workflow et de son lot de travail :

  • définir les informations générales associées au schéma de workflow, telles que son titre et sa description ;

  • configurer le schéma de workflow pour qu'il utilise les rôles du schéma de workflow (mise en œuvre de type Fondé sur schéma). Pour plus d'informations sur les rôles, reportez-vous à la rubrique Utilisation des rôles.

  • Déterminer si Content Server archive ou supprime les instances en cours d'exécution d'un schéma de workflow lorsqu'elles sont terminées.

  • Indiquer la manière dont sont pris en compte les week-ends pour le calcul des échéances du schéma de workflow.

  • Choisir d'afficher un message standard ou personnalisé (qui peut comprendre du code HTML de base) au lancement d'un schéma de workflow.

  • Sélectionner le type d'informations que vous souhaitez inclure dans le lot de travail : attributs, pièces jointes, commentaires et formulaires (disponibles uniquement si le module facultatif Content Server Forms est installé).

 

Note :
Certains modules optionnels proposent un autre type de schéma de workflow. Si votre administrateur a créé un type personnalisé de schéma de workflow, vous devez choisir le type de schéma de workflow que vous souhaitez utiliser dans la liste Type de workflow. Le type de schéma de workflow Content Server par défaut est Standard.

 

Spécification des propriétés d'un schéma de workflow avec signature

Si vous souhaitez créer un schéma de workflow dans lequel un document peut être approuvé et signé ou refusé, vous devez spécifier des propriétés de signature. Pour plus d'informations sur les propriétés de schéma de workflow avec signature, voir Gestion des workflows avec signature.
