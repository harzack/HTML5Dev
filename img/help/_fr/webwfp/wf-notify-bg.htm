OpenText recommande d'encourager les utilisateurs gérant les schémas de workflow ou y participant à utiliser les paramètres OpenText eLink afin de faciliter le suivi des schémas de workflow. Par exemple, les utilisateurs peuvent configurer un évènement eLink pour recevoir une alerte par e-mail chaque fois qu'une étape qui leur est assignée est prête. Content Server propose le traitement de notifications par e-mail pour les évènements de schéma de workflow suivants :

  • Un schéma de workflow que vous gérez est terminé ou a du retard.

  • Une étape qui vous est assignée est prête ou a du retard.

Pour plus d'informations sur la configuration des préférences pour les évènements du schéma de workflow, voir Configuration des préférences pour les évènements du workflow.

En outre, l'administrateur peut contrôler les paramètres eLink pour les préférences des événements du schéma de workflow en configurant les paramètres de schéma de workflow eLink généraux. Les paramètres de schéma de workflow eLink généraux de l'administrateur remplacent les paramètres eLink de l'utilisateur. Par exemple, même si les utilisateurs désactivent un paramètre eLink pour recevoir un e-mail lorsqu'une étape de schéma de workflow qui leur est assignée est en retard, l'administrateur peut configurer un paramètre eLink global pour que tous les utilisateurs reçoivent ces e-mails. Pour plus d'informations sur la gestion des préférences générales des événements de schéma de workflow, consultez votre administrateur. Voir également « Administration de paramètres généraux du workflow eLink », dans l'aide en ligne de l'administrateur Content Server.

 

Important :
L'administrateur doit configurer les paramètres de schéma de workflow eLink généraux ou les utilisateurs doivent configurer leurs paramètres de schéma de workflow eLink locaux afin de recevoir des e-mails concernant les évènements des schémas de workflow activés par e-mail.
