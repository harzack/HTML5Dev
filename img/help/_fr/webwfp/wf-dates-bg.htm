Les intervenants du workflow peuvent programmer et suivre la progression d'un workflow en cours si vous configurez le workflow et ses étapes pour l'utilisation de durées (dates d'échéance) et de dates de début. Vous pouvez également configurer Content Server pour que la date d'échéance du workflow soit recalculée après l'exécution de n'importe quelle étape Initiateur, Utilisateur, Evaluer ou Gestionnaire d'élément.

Spécification d'échéances

Content Server calcule la date d'échéance d'une étape dès le lancement du workflow en ajoutant à la date de début du workflow les durées que vous avez indiquées pour les étapes Initiateur, Utilisateur, Gestionnaire d'élément et Sous-workflow. Par exemple, si un workflow comporte trois étapes Utilisateur avec une durée d'une journée et que le workflow est lancé le 22 septembre, sa date d'échéance sera le 25 septembre. Les Etapes clés et les Etapes du processus s'exécutent automatiquement et n'ont pas de durée.

Si le workflow inclut une ou plusieurs étapes Evaluer, Content Server calcule les échéances pour toutes les branches de chaque étape Evaluer et utilise la date la plus éloignée comme échéance du workflow.

Si vous attribuez une date de début explicite à une étape de workflow, Content Server inclut la durée précédant le début de l'étape dans ses calculs d'échéance. Si vous ne définissez pas de date de début explicite, Content Server calcule les échéances à partir du moment où l'étape arrive sur la page Tâches assignées d'un utilisateur.

 

Notes :

  • Vous pouvez préciser si Content Server doit calculer l'échéance d'une tâche d'étape sur la base d'une semaine de travail standard (en excluant les week-ends) dans les propriétés générales du schéma de workflow. Pour plus d'informations sur les propriétés générales, reportez-vous à la rubrique Définition des propriétés générales.

  • Les utilisateurs disposant des autorisations appropriées peuvent modifier les dates d'échéances dans une instance de workflow en cours sur la page Statut de workflow. Pour plus d'informations sur la gestion des autorisations, reportez-vous à la rubrique Configuration des autorisations de gestion. Pour plus d'informations sur le suivi des workflows, reportez-vous à la rubrique Gestion des workflows et des tâches assignées.

 

Recalcul automatique de la date d'échéance

Vous pouvez indiquer à Content Server de recalculer automatiquement la date d'échéance une fois que les étapes Initiateur, Evaluer, Sous-workflow, Gestionnaire d'élément et Utilisateur sont terminées. Par exemple, si une étape Utilisateur prend moins de temps que prévu dans sa durée, Content Server recalcule automatiquement l'échéance du workflow en retirant le temps économisé dans l'étape.

 

Note :
Si vous spécifiez l'option de recalcul des dates d'échéance au cours d'une étape sans spécifier de durée, le système ne peut pas calculer la date d'échéance du workflow ni la date d'échéance de l'étape. Si vous ne spécifiez pas la durée de l'étape, le workflow affiche la date d'échéance <Aucune> ou <Aucune date d'échéance>.

 

Spécification d'une date de commencement

Vous pouvez spécifier une date de début, à savoir une date explicite pour le lancement des étapes Initiateur, Utilisateur et Gestionnaire d'élément. Une étape n'apparaît sur la page Tâches assignées d'un utilisateur avec une tâche assignée qu'à sa date de début.

Calcul des dates d'échéances avec des sous-workflows

Lorsqu'un workflow est lancé, les échéances pour le workflow principal et le sous-workflow sont calculées sur la base de la durée et des dates de début fournies pour chaque étape. Dès qu'une étape est terminée (envoyée) le moteur du workflow recherche toute étape non terminée configurée pour utiliser l'option de recalcul des échéances. Les étapes d'un sous-workflow peuvent donc influencer les dates d'échéance des étapes du workflow principal si elles sont configurées pour utiliser l'option de recalcul des échéances. Par exemple, les étapes d'un sous-workflow qui prennent plus de temps que prévu provoquent un retard dans le lancement des étapes restantes dans le workflow principal.

Calcul des dates d'échéances avec plusieurs étapes Evaluer

Si vous demandez que l'échéance du workflow soit calculée par Content Server et que le workflow comporte une ou plusieurs étapes Evaluer, Content Server calcule les échéances pour les deux branches de chaque étape Evaluer et définit la date la plus éloignée comme date d'échéance du workflow.
