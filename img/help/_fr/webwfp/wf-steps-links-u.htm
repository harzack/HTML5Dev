Les tâches et les événements de votre processus commercial sont définis dans les étapes du schéma de workflow. Les liens entre les étapes servent à contrôler l'ordre dans lequel les étapes sont traitées. Les étapes par défaut disponibles dans la boîte à outils de l'éditeur de schéma de workflow sont les suivantes :

  • Etape Départ

  • Etape Utilisateur

  • Etape Initiateur

  • Etape Evaluation

  • Etape Clé

  • Etape Gestionnaire d'élément

  • Etape Sous-workflow

  • Etape Processus

Une fois que vous avez ajouté une étape dans l'éditeur de schéma de workflow, vous pouvez en modifier le nom afin de l'identifier de manière pertinente. Le nom d'une étape apparaît à l'intérieur de celle-ci, excepté pour les étapes Départ, Evaluer et Clé, dont le rendu s'effectue sous la structure visible. Lorsque vous enregistrez le schéma de workflow, l'ID d'étape s'affiche dans le nom de l'étape, ainsi que dans ses informations détaillées. Dans les deux cas, l'ID est ajouté à la fin du nom de l'étape, entre crochets []. La date d'échéance, si elle a été définie, et la date d'achèvement apparaissent désormais en dessous du texte, pour toutes les étapes. Pour plus d'informations sur l'ajout d'une étape dans l'éditeur de schéma de workflow, voir Exploration de l'éditeur de schéma de workflow.

Le nombre d'étapes exécutées lors d'un cycle de suivi est limité à 70. S'il dépasse 70, les étapes supplémentaires sont exécutées en arrière-plan.
