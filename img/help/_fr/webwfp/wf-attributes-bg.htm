Vous définissez des attributs pour collecter des informations sur les intervenants et stocker les informations dont vous souhaitez effectuer le suivi tout au long du processus de travail. Par exemple, ils peuvent vous permettre d'enregistrer les réponses à une question ou à un problème. Ces réponses peuvent s'afficher ultérieurement dans les étapes de schéma de workflow ou servir de critères pour déterminer le chemin d'exécution d'un schéma de workflow dans une étape Evaluer.

Vous ne pouvez pas utiliser d'attributs avant de les activer dans le schéma de schéma de workflow. Pour plus d'informations sur l'activation des attributs dans un schéma de workflow, reportez-vous à la rubrique Définition des propriétés générales.

Vous pouvez définir un attribut sous forme de champ, de zone de liste déroulante ou de case à cocher, ce qui vous permet ainsi de déterminer la meilleure méthode de saisie des données pour les intervenants du schéma de workflow. Par exemple, si vous voulez savoir si ces intervenants du schéma de workflow souhaitent des cartes de visite en relief ou imprimées, définissez un attribut intitulé Méthode d'impression sous la forme d'une case à cocher et configurez le schéma de workflow pour permettre aux utilisateurs de sélectionner la case à cocher s'ils souhaitent des cartes de visite imprimées ou de la désactiver s'ils veulent des cartes en relief.

Une fois les attributs pour un schéma de workflow définis, ces derniers sont disponibles pour chaque étape du schéma de workflow. Vous pouvez leur attribuer la valeur Non disponible, Modifiable, Saisie obligatoire ou Lecture seule pour chaque étape.

Si des objets physiques sont installés, une valeur Ignorer l'action supplémentaire est disponible. Pour en savoir plus, voir Configuration de l'accès aux attributs.

Spécification des propriétés d'attribut

Certaines propriétés se trouvent dans plusieurs attributs et d'autres sont particulières à un attribut. Désactivation de la création de versions

Par défaut, Content Server affiche la valeur actuelle des attributs du schéma de workflow, mais stocke la version actuelle et toutes les versions précédentes des valeurs d'attribut dans la base de données Content Server. Si vous souhaitez afficher les versions précédentes d'une valeur d'attribut (par exemple, si vous souhaitez effectuer l'audit d'une instance de schéma de workflow), vous pouvez créer un rapport LiveReport pour récupérer toutes les versions d'une valeur d'attribut à partir de la base de données Content Server. Si vous souhaitez que Content Server stocke uniquement la valeur d'attribut actuelle, vous pouvez désactiver la création de version des attributs du schéma de workflow.
