Si vous souhaitez permettre aux utilisateurs de lancer des schémas de workflow par e-mail, vous pouvez activer par e-mail un schéma de workflow. Lorsque vous activez par e-mail un schéma de workflow, Content Server lui affecte une adresse e-mail du type :

Nom_schéma_de_workflow@Serveur_eLink

où Nom_schéma_de_workflow correspond au nom du schéma de workflow que vous spécifiez (il n'est pas obligatoire que la valeur du nom corresponde au nom du schéma de workflow) et Serveur_eLink est le nom du serveur eLink. Par exemple, l'adresse e-mail par défaut d'un workflow activé par e-mail nommé Mon_schéma_de_workflow sur un serveur Content Server utilisant un serveur eLink appelé eLink_Serveur.MaSociété.com est Mon_schéma_de_workflow@eLink_Serveur.MaSociété.com.

L'administrateur peut vous interdire d'activer par e-mail les schémas de workflow en désactivant cette fonction dans le module OpenText eLink. Pour plus d'informations sur l'activation ou la désactivation de la fonction de lancement par e-mail, consultez votre administrateur.

 

Important :
Content Server ne permet pas de lancer un schéma de workflow par e-mail si l'étape Départ requiert l'une des données suivantes :

  • Valeurs d'attributs

  • Valeurs des champs du formulaire

  • Authentification

 

Une fois un schéma de workflow activé par e-mail, vous pouvez rendre l'adresse e-mail du schéma de workflow disponible pour les utilisateurs autorisés à lancer un schéma de workflow en envoyant un e-mail de demande à l'adresse du schéma de workflow. Les informations contenues dans le message de l'utilisateur sont transférées sur le lot de travail du schéma de workflow. Par exemple, l'objet de l'e-mail devient le nom du schéma de workflow, les pièces jointes sont copiées dans le dossier des pièces jointes du schéma de workflow et le texte du corps du message est intégré à la section Commentaires de l'étape Départ.

 

Note :
Les utilisateurs peuvent utiliser les mêmes méthodes pour lancer un schéma de workflow comme schéma de workflow, qu'il soit activé par e-mail ou non.

 

Pour plus d'informations sur le lancement d'un schéma de workflow par e-mail, voir Lancement d`un workflow.
