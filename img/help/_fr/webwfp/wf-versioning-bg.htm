Vous pouvez configurer votre schéma de workflow afin d'effectuer automatiquement des opérations de création de versions pour des documents ou des documents composites dans Content Server ou dans le dossier Pièces jointes du schéma de workflow. Lorsque le processus du schéma de workflow atteint une étape Gestionnaire d'élément, il exécute les opérations de création de version que vous avez définies dans son onglet Création de version. Par exemple, si vous ajoutez un document au dossier Pièces jointes du schéma de workflow et que ce document est modifié lors de l'exécution du schéma de workflow, vous pouvez configurer l'étape pour qu'elle renvoie le document modifié en tant que version du document d'origine.

Pour les documents et les documents composites ajoutés aux schémas de workflow en tant que pièces jointes à partir de Content Server, l'étape peut exécuter de manière globale des opérations de création de version sur les éléments Content Server d'origine, en utilisant les pièces jointes au schéma de workflow comme sources. L'étape peut également effectuer des opérations de création de version sur des documents ou des documents composites spécifiques ajoutés en tant que pièces jointes du schéma de workflow ou stockés dans Content Server. Dans les deux cas, pour toute option de création de version indiquée explicitement et qui implique un document, l'étape crée toujours implicitement une version du document.

Vous pouvez également configurer l'étape pour créer :

  • des générations à partir de documents ou de documents composites ;

  • des révisions ou des versions de documents composites ;

  • des versions principales de documents qui utilisent la création de versions avancée.

Vous pouvez aussi déverrouiller les documents ou les documents composites d'origine.

Pour plus d'informations sur la création de version de documents et de documents composites, voir Utilisation des documents et des documents texte et Utilisation des versions et des révisions.

 

Notes :

  • Il est impossible de créer une génération à partir d'un document composite qui ne possède pas une version ou une révision.

  • Lorsque vous créez manuellement une génération à partir d'un document ou d'un document composite dans Content Server, vous devez indiquer explicitement l'emplacement cible où la génération sera créée. Cette règle ne s'applique pas à l'étape Gestionnaire d'élément qui crée la génération dans le recueil où l'élément cible sélectionné est stocké.

 

Utilisation des documents composites

Lorsque vous effectuez des opérations sur des documents composites, l'étape Gestionnaire d'élément exécute des opérations sur les éléments qui portent des noms identiques. Par exemple, si un document composite source nommé « MesDocuments » contient les documents « DocumentA.doc », «  DocumentB.doc » et « DocumentC.doc », et que la cible est un document composite qui porte le même nom et qui contient les documents «  DocumentA.doc », « DocumentB.doc » et « DocumentX.doc », l'étape exécute les demandes de création de version pour « DocumentA.doc » et « DocumentB.doc », mais ignore « DocumentC.doc » et « DocumentX.doc ».

Si vous souhaitez créer une version de « DocumentX.doc » à partir de « DocumentC.doc », vous pouvez indiquer cette opération comme cible spécifique.

Création de versions principales

Lorsque vous utilisez l'étape Gestionnaire d'élément pour créer des versions d'un document, vous pouvez configurer l'étape pour qu'elle crée une version principale si le document utilise la création de version avancée. Le document défini comme source dans une demande de création d'une version principale peut utiliser la création de version standard ou avancée, mais le document défini comme cible doit utiliser la création de version avancée. Si vous configurez une étape pour qu'elle ajoute une version principale à un document qui utilise la création de version standard, l'étape Gestionnaire d'élément ignore la demande de création de versions principale et crée une version standard.

Par exemple, si un document A utilise la création de version avancée et qu'un document B utilise la création de version standard, vous pouvez ajouter le document B en tant que version principale du document A. Toutefois, si vous souhaitez ajouter un document A en tant que version d'un document B, l'étape Gestionnaire d'élément peut uniquement ajouter le document A en tant que version standard, car le document B n'utilise pas la création de version avancée. Par ailleurs, vous ne pouvez pas utiliser l'étape Gestionnaire d'élément pour modifier le type de création de version d'un document.

Pour plus d'informations sur la création de versions avancée, reportez-vous à la rubrique Utilisation des versions.

Par défaut, la fonction de création de versions avancée est activée sur Content Server, mais l'administrateur peut la désactiver. Même si la fonction de création de version avancée est désactivée, l'onglet Création de version affiche la case à cocher Créer une version principale, qui vous permet de spécifier les demandes de versions principales. Si Content Server traite une étape Gestionnaire d'élément contenant une demande de version principale alors que la création de version avancée est désactivée, le système crée des versions standard au lieu de versions principales.

 

Note :
Avant que l'étape Gestionnaire d'élément ne puisse créer une version principale d'un document, le compte utilisateur Content Server devant traiter l'étape doit disposer de l'autorisation Modifier pour le document.

 

Configuration des opérations sur les versions pour les pièces jointes au schéma de workflow

Spécifiez les opérations de création de version pour les documents et les documents composites de Content Server que vous avez ajoutés au dossier Pièces jointes du schéma de workflow. Pour toutes les pièces jointes de documents et de documents composites, vous pouvez spécifier les options suivantes :

  • Aucune, pour indiquer qu'aucune nouvelle version de document ou de document composite d'origine n'est renvoyée vers Content Server.

  • Si modifié, pour indiquer que seuls les documents ou les documents composites modifiés sont renvoyés vers Content Server comme nouvelles versions des éléments d'origine.

  • Tous, pour indiquer que tous les documents ou les documents composites sont renvoyés vers Content Server comme nouvelles versions des éléments d'origine.

Vous pouvez également configurer l'étape Gestionnaire d'élément pour qu'elle crée une génération à partir de tous les documents et qu'elle exécute l'une des opérations suivantes pour tous les documents composites :

  • Aucune, qui n'exécute aucune opération sur les documents composites.

  • Créer une version, qui crée une version à partir des documents composites.

  • Créer une révision, qui crée une révision à partir des documents composites.

  • Créer une génération, qui crée une génération à partir des documents composites.

  • Créer une version et une génération, qui crée une version et une génération à partir des documents composites.

  • Créer une révision et une génération, qui crée une révision et une génération à partir des documents composites.

Configuration des opérations sur les versions pour les cibles spécifiques

Vous pouvez configurer l'étape Gestionnaire d'élément pour exécuter des opérations de création de version sur des documents ou des documents composites ajoutés comme pièces jointes au schéma de workflow ou stockés dans Content Server. Configurez une demande de création de version pour chaque document ou document composite que vous souhaitez traiter.

Une demande de création de version est composée de l'élément source (le document ou le document composite qui représente la version) et d'un élément cible (le document ou le document composite sur lequel la source est appliquée comme version). La source et la cible peuvent être :

  • des pièces jointes au schéma de workflow ;

  • élément Content Server

  • un attribut de référence d'élément.

Déverrouillage de cibles

Si l'élément cible est verrouillé, vous pouvez le déverrouiller lorsque l'opération est terminée.

Création de générations

En complément de la version, vous pouvez créer une génération à partir de l'élément source. Vous pouvez également spécifier un nom pour la génération en remplaçant le nom par défaut.

Création de versions et de révisions

En complément d'une version ou d'une génération, vous pouvez créer une version ou une révision à partir de l'élément source. Pour remplacer le nom de version ou de révision par défaut, saisissez un nouveau nom pour la version ou la révision.
