Vous pouvez configurer un schéma de workflow pour créer automatiquement des dossiers dans un recueil Content Server ou dans le dossier Pièces jointes du schéma de workflow. Vous pouvez également créer une définition de sous-dossier à l'intérieur d'une définition de dossier. Lorsque le processus du schéma de workflow atteint une étape Gestionnaire d'élément, il crée les dossiers que vous avez définis dans son onglet Définitions de dossier. Chaque demande de création de dossier est appelée définition de dossier. Vous pouvez également utiliser des définitions de dossier comme cible ou source d'autres opérations de l'étape Gestionnaire d'élément. Par exemple, vous pouvez configurer une demande de déplacement ou de copie dans une étape Gestionnaire d'élément pour déplacer ou copier un élément Content Server dans un dossier créé par une définition de dossier. Pour plus d'informations sur la configuration de l'étape Gestionnaire d'élément pour déplacer ou copier des éléments, voir Copie ou déplacement d'éléments. Une fois la définition de fichier créée, vous pouvez la modifier.

 

Notes :

  • Il est possible que l'étape Gestionnaire d'élément ne traite pas les définitions de dossier dans l'ordre dans lequel elles apparaissent dans l'onglet Définitions de dossier.

  • Les dossiers créés par l'étape Gestionnaire d'élément n'héritent pas des catégories du recueil Content Server dans lequel ils sont créés. Pour plus d'informations sur les catégories et attributs Content Server, voir Utilisation des catégories et des attributs.

 

Utilisation des dossiers existants

Vous devez indiquer la manière dont l'étape Gestionnaire d'élément doit réagir si elle rencontre un dossier existant qui porte le même nom que celui qui sera créé dans une définition de dossier. Vous avez le choix entre les actions suivantes :

  • Réutiliser le dossier existant avec le même nom, pour indiquer que l'étape utilisera le dossier existant au lieu de créer un dossier.

  • Créer un nouveau dossier avec un numéro ajouté, pour créer le dossier et y ajouter un numéro (le système commence à 2 et augmente de 1) pour que le nom de dossier soit unique.

Choix du dossier cible

La cible du dossier est l'emplacement où vous souhaitez que l'étape Gestionnaire d'élément crée un dossier. Il peut s'agir d'un des éléments suivants :

  • le dossier Pièces jointes du schéma de workflow ou un de ses sous-dossiers ;

  • un recueil Content Server, comme un espace de travail, un projet ou un dossier ;

  • une définition de dossier définie sur la même étape ;

  • un attribut de référence d'élément qui renvoie à un recueil Content Server.

 

Notes :

  • Les définitions de dossier peuvent servir de dossiers cible uniquement lorsque plus d'une définition de dossier est définie pour la même étape.

  • Un attribut de référence d'élément peut servir de dossier cible uniquement si vous en avez configuré plus d'un qui renvoie à un recueil Content Server pour le schéma de workflow. Par ailleurs, un attribut de référence d'élément défini comme cible ne peut pas contenir plusieurs valeurs.
