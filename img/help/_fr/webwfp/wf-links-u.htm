Lorsque vous souhaitez définir le chemin qu'un schéma de workflow emprunte pour exécuter un processus de travail, vous créez des liens entre les étapes. Pour ce faire, vous placez le curseur de la souris sur l'étape et faites glisser la flèche qui s'affiche jusqu'à l'autre à relier. Content Server traite un schéma de workflow en suivant les liens et en exécutant les étapes qu'il rencontre dans l'ordre où elles apparaissent dans les liens. Lorsque vous reliez deux étapes dans l'éditeur de schéma de workflow, une ligne connecte ces étapes. Si vous souhaitez rompre le flux de traitement entre deux étapes, supprimez le lien.

En fonction des besoins de votre schéma de workflow, vous pouvez créer l'un des types de liens suivants entre les étapes :

  • Liens série

  • Liens parallèles

  • Liens Rendezvous

  • Liens conditionnels

  • Liens de retour arrière

  • Liens de retour arrière conditionnels

Une fois que vous avez défini des liens dans votre schéma de workflow, vous pouvez vérifier ce dernier pour vous assurer que les liens ne forment pas d'opérations non valables ou ne provoquent pas d'erreurs. Pour en savoir plus, voir Vérification des schémas de workflow.

Liens série

Les liens série relient deux étapes dans une seule ligne.

Liens parallèles

Les liens parallèles connectent une même étape à deux autres étapes ou plus d'un type quelconque. Un lien parallèle permet l'exécution de plusieurs étapes simultanément.

Liens Rendezvous

Les liens Rendezvous connectent plusieurs étapes à une seule autre étape.

Liens conditionnels

Les liens conditionnels connectent une étape Evaluation à d'autres étapes par l'intermédiaire des chemins VRAI et FAUX. Une fois un chemin défini, cliquez sur la ligne et sélectionnez Vrai ou Faux. Vous pouvez utiliser plusieurs chemins VRAI et FAUX pour quitter une étape Evaluation, à condition d'en créer au moins un de chaque. Les liens de type VRAI sont verts. Les liens de type FAUX sont rouges.

Les critères définis pour l'étape Evaluation déterminent si le schéma de workflow suit le chemin VRAI ou FAUX. Pour plus d'informations sur les étapes Evaluation, voir Configuration d'une étape Evaluer.

 

Note :
Si le schéma de workflow inclut une ou plusieurs étapes Evaluation, Content Server calcule les échéances pour les deux chemins de chaque étape Evaluation et utilise la date la plus éloignée pour déterminer l'échéance du workflow.

 

Liens de retour arrière

Un lien Boucle de rappel connecte une étape à une étape précédente du schéma de workflow, vous permettant ainsi de répéter une ou plusieurs étapes précédentes. Les liens de boucle de rappel sont bleus.

Une connexion ne peut pas effectuer un lien de boucle de rappel à l'étape Départ. Si vous tentez de créer un lien illégal, vous recevrez un message d'erreur. Les liens Boucle de rappel doivent répondre aux règles suivantes :

  • Un lien Boucle de rappel doit former une boucle autonome, c'est-à-dire que la boucle ne peut pas comprendre des étapes qui sont reliées avec des étapes extérieures au lien de boucle de rappel. De même, les étapes en dehors du lien de boucle de rappel ne peuvent pas être reliées à des étapes qui font partie de la boucle. Content Server vérifie que vos liens respectent cette règle lorsque vous enregistrez un schéma de workflow.

  • Toutefois, cette règle ne s'applique pas si une étape du lien Boucle de rappel est elle-même reliée à un autre lien Boucle de rappel. Utilisez l'éditeur de schéma de workflow pour vérifier que vos liens Boucle de rappel respectent cette règle.

Liens de retour arrière conditionnels

Vous pouvez créer un lien Boucle de rappel conditionnel opposé au chemin Vrai ou Faux d'une étape Evaluation. Un lien Boucle de rappel conditionnel prend la valeur Vrai ou Faux opposée à celle de l'autre lien. Bien que vous ne puissiez créer qu'un seul lien Boucle de rappel conditionnel, vous pouvez créer plusieurs liens parallèles pour le lien Vrai ou pour le lien Faux.

Les critères définis pour l'étape Evaluation déterminent le chemin suivi par le schéma de workflow.

Pour plus d'informations sur les liaisons, voir Définition de liens entre les étapes.
