Lorsque vous définissez des étapes Sous-workflow, vous devez spécifier les informations suivantes :

  • des informations générales comme le nom de l'étape, le schéma de workflow que vous souhaitez intégrer, un nom de lancement, la durée et les préférences de calcul d'échéances ;

  • les mappages et la disponibilité des données, pour déterminer la manière dont fonctionnent les informations du lot (les commentaires, les attributs et les pièces jointes) et les autres données partagées entre le schéma de workflow principal et le schéma de workflow que vous intégrez.

Pour plus d'informations sur les étapes Sous-workflow, voir Etapes Sous-workflow.

Définition des informations générales

Les informations générales que vous pouvez définir pour une étape Sous-workflow sont :

  • le nom de l'étape ;

  • le schéma de workflow que vous souhaitez intégrer dans le schéma de workflow principal ;

  • le nom attribué à une instance du sous-workflow lors de son lancement ;

  • le délai (durée) dans lequel vous souhaitez que le sous-workflow soit exécuté (en jours ou en heures) ;

  • si Content Server doit recalculer les dates d'échéance du schéma de workflow une fois l'étape terminée.

Pour plus d'informations sur les durées et les dates d'échéances, reportez-vous à la rubrique Dates et priorités.

Définition des mappages et de la disponibilité des données

Vous devez indiquer la manière dont le lot de travail et les autres données du schéma de workflow principal sont partagées avec le schéma de workflow intégré. Par exemple, si le schéma de workflow principal contient un attribut « Titre » et que le sous-workflow utilise un attribut « Nom du document », vous pouvez créer un mappage entre les deux attributs pour que les valeurs du schéma de workflow principal soient disponibles dans l'attribut du schéma de workflow intégré. Le schéma de workflow intégré peut ainsi modifier la valeur de l'attribut. En fonction des informations activées dans le lot de travail du schéma de workflow, vous pouvez indiquer la disponibilité et les mappages pour les types d'informations suivants :

  • Rôles

  • Pièces jointes

  • Commentaires

  • Attributs

  • Formulaires

Echange de rôles

Si le workflow principal et le workflow intégré utilisent des rôles, vous pouvez indiquer les correspondances des rôles entre les deux schémas de workflow en créant un mappage des rôles. Par exemple, si le schéma de workflow principal contient un rôle « Responsable »» et que le schéma de workflow intégré contient un rôle similaire appelé « Chef », vous pouvez créer un mappage entre les deux rôles.

 

Note :
Tous les rôles du schéma de workflow principal doivent être mappés avec un rôle du schéma de workflow intégré.

 

Echange de commentaires

Vous pouvez décider si les intervenants au schéma de workflow principal peuvent afficher les commentaires inscrits dans l'un ou l'autre des workflows.

Echange d'attributs

Vous pouvez mapper les attributs du workflow principal avec ceux du sous-workflow afin que les valeurs d'attribut soient transmises entre les deux schémas de workflow. Lorsque le sous-workflow s'achève, les valeurs de ses attributs mappés sont transmises au schéma de workflow principal.

Lorsque vous mappez des attributs, le type de données de chaque attribut du schéma de workflow principal doit être compatible avec celui de l'attribut du sous-workflow sur lequel vous le mappez. Le sous-workflow doit également comporter au moins un type d'attribut défini.

Echange de pièces jointes

Si vous souhaitez que les intervenants au schéma de workflow intégré aient accès aux pièces jointes du schéma de workflow principal, Content Server crée un raccourci vers le dossier Pièces jointes du schéma de workflow principal dans le dossier Pièces jointes du schéma de workflow intégré. Les éléments placés dans le dossier représenté par le raccourci lors du traitement du schéma de workflow intégré sont accessibles depuis le schéma de workflow principal ; ceux qui sont placés en dehors du dossier ne le sont pas dans le schéma de workflow principal. Si vous sélectionnez des éléments individuels, Content Server crée un raccourci dans le dossier Pièces jointes du schéma de workflow intégré.

 

Astuce :
Pour organiser de manière efficace l'échange des pièces jointes entre les schémas de workflow, vous pouvez créer un dossier contenant uniquement les pièces jointes à échanger dans le dossier Pièces jointes du schéma de workflow principal, puis rendre ce dossier disponible dans le schéma de workflow intégré.

 

Echange de formulaires

Vous pouvez mapper des champs de formulaire du schéma de workflow principal avec des champs de formulaire du sous-workflow afin que les valeurs de champ soient transmises entre les deux schémas de workflow. Lorsque le sous-workflow s'achève, les valeurs de ses champs mappés sont transmises au schéma de workflow principal.

Les formulaires ne sont disponibles que si le module facultatif OpenText Forms est installé. Pour plus d'informations sur l'utilisation des formulaires dans les schémas de workflow, voir Utilisation de formulaires dans les schémas de workflow.
