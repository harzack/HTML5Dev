Après avoir ajouté un modèle de formulaire, vous pouvez l'utiliser comme tout autre élément. Par exemple, vous pouvez supprimer, renommer, copier, réserver ou déplacer le formulaire. Vous pouvez également créer un raccourci vers ce formulaire, en faire une nouvelle, définir une notification, afficher ses propriétés ou l'ajouter à votre page Favoris. De plus, vous pouvez créer des versions du modèle de formulaire. Pour plus d'informations sur l'utilisation des éléments, voir Aide du Guide de prise en main. Pour plus d'informations sur les nouvelles, reportez-vous à la rubrique Utilisation des chaînes et des nouvelles.

Gestion des modèles de formulaire

Les modèles de formulaire sont des éléments contenant une version. Chaque fois que vous enregistrez un modèle de formulaire, le système en crée automatiquement une version. Vous pouvez supprimer une version ou afficher les propriétés d'une version. Pour protéger la version et empêcher qu'elle soit remplacée, vous avez la possibilité de la verrouiller. Pour enlever plusieurs versions, vous pouvez en éliminer autant que vous souhaitez en une seule fois. Vous pouvez aussi associer une version à une table de base de données pour mettre les mécanismes de révision et d'envoi Table SQL à la disposition des utilisateurs qui basent leurs formulaires sur le modèle.

Si vous avez installé les modules PDF Forms ou eForms, vous pouvez ajouter un formulaire PDF Forms ou eForm en tant que vue personnalisée (version du modèle de formulaire).

Pour plus d'informations sur l'utilisation des formulaires PDF Forms, voir Utilisation de formulaires PDF.

Pour plus d'informations sur l'utilisation des formulaires électroniques, reportez-vous à la section Utilisation de eForms Management.

Suppression des modèles de formulaire associés à une table de base de données

Lorsque vous supprimez un modèle de formulaire associé à la table SQL, le système supprime le modèle de formulaire, mais pas la table SQL qui lui est associée. Il ne supprime pas la table SQL au cas où d'autres modèles de formulaire utilisent cette même table SQL pour stocker leurs données. Si vous souhaitez supprimer une table SQL associée à un modèle de formulaire, vous devez la supprimer manuellement.

 

Note :
Les données enregistrées d'un formulaire peuvent être perdues si vous ajoutez une version du modèle qui exclut les champs existant dans les versions précédentes.

Il n'est pas possible actuellement de déterminer si d'autres formulaires utilisent la même table SQL.

 

Affichage d'informations spécifiques

Vous pouvez afficher les informations de modèle de formulaire suivantes. Cliquez sur l'icône Fonctions du modèle, sélectionnez Propriétés, puis Infos spécifiques.

  • Version actuelle – Numéro de la version la plus récente du modèle de formulaire et nombre de versions actuellement disponibles.

  • Nbre max. de versions – Nombre maximal de versions de ce modèle de formulaire stockées par Content Server. Lorsque le nombre maximal de versions est atteint et qu'un utilisateur en ajoute une autre, le système supprime automatiquement la version la plus ancienne.

  • Table SQL associée – Nom de la table de base de données associée au modèle de formulaire. Si aucune table SQL n'est associée au modèle de formulaire, ce champ est vide.

 

Note :
Si vous avez l'autorisation Supprimer les versions pour le modèle, vous pouvez modifier certaines informations spécifiques.
