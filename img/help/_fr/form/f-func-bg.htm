Après avoir ajouté un formulaire, vous pouvez l'utiliser de la même façon que les autres éléments. Par exemple, vous pouvez supprimer, renommer, copier, réserver ou déplacer le formulaire. Vous pouvez également créer un raccourci vers ce formulaire, en faire une nouvelle, définir une notification, afficher ses propriétés ou l'ajouter à votre page Favoris. De plus, vous pouvez créer des versions du formulaire. Pour plus d'informations sur l'utilisation des éléments Content Server, reportez-vous à la rubrique Aide du Guide de prise en main. Pour plus d'informations sur les nouvelles, reportez-vous à la rubrique Utilisation des chaînes et des nouvelles.

Affichage d'informations spécifiques

Vous pouvez afficher les informations suivantes du formulaire, que vous pouvez modifier si vous possédez l'autorisation Gérer pour le formulaire :

  • Version actuelle – Numéro de la version la plus récente du formulaire et nombre de versions actuellement disponibles.

  • Nbre max. de versions – Nombre maximal de versions de ce formulaire stockées par Content Server. Lorsque le nombre maximal de versions est atteint et qu'un utilisateur en ajoute une nouvelle, Content Server supprime automatiquement la plus ancienne.

  • Modèle – Nom du modèle de formulaire qui détermine la présentation du formulaire.

  • Vue personnalisée – Vues personnalisées, s'il en existe, associées à ce formulaire.

  • Mécanisme de révision – Décrit ce qui se passe lorsque vous cliquez sur le bouton du formulaire permettant de transférer les données à Content Server. Un mécanisme de révision fournit un stockage intermédiaire des données, ce qui permet à Content Server d'afficher le formulaire avec les données enregistrées précédemment lorsque vous l'ouvrez de nouveau.

  • mécanisme d'envoi – Décrit ce qu'il se passe lorsque vous cliquez sur l'icône Fonctions du formulaire et que vous choisissez Envoyer. Un mécanisme d'envoi permet d'envoyer les données finales du formulaire. Lorsque vous rouvrez le formulaire, Content Server le rétablit à son état d'origine.

  • Workflow Map – (disponible uniquement avec le mécanisme d'envoi Lancer le workflow) Nom du schéma de workflow que Content Server lance lors de l'envoi du formulaire.

  • Modèle prérenseigné – Indique si le formulaire est un modèle prérenseigné ou non. Pour plus d'informations sur l'utilisation d'un modèle prérenseigné, reportez-vous à la rubrique Définition d'un modèle prérenseigné.

Modification d'un formulaire

Après avoir ajouté un formulaire, vous avez la possibilité de le modifier. Lorsque vous modifiez un formulaire, il s'ouvre dans l'éditeur qui lui est associé. Par exemple, si vous modifiez un formulaire PDF, le formulaire s'ouvre dans Adobe Acrobat.

Pour plus d'informations sur le module optionnel PDF Forms, consultez Utilisation de formulaires PDF.

Si vous n'avez que l'autorisation Voir pour le formulaire, vous ne pouvez pas modifier le formulaire. Pour éditer un formulaire, vous devez disposer de l'autorisation Utiliser ...« ... » le mécanisme de révision pour ce formulaire s'il possède un mécanisme de révision et de l'autorisation Utiliser « ... » le mécanisme d'envoi dans le cas contraire. Pour de plus amples informations sur les autorisations, reportez-vous à la section Description des autorisations pour les modèles de formulaire et les formulaires.

Suppression d'un formulaire

Si vous supprimez un formulaire, Content Server supprime ses données révisables (le cas échéant), mais ne supprime pas automatiquement les données envoyées du formulaire. L'accès aux données envoyées après suppression du formulaire dépend de l'emplacement où le mécanisme d'envoi stocke les données. Vous pouvez, par exemple, accéder directement aux données du formulaire qui ont été envoyées vers une table SQL en créant des commandes SQL dans la base de données, mais vous ne pouvez pas accéder aux données du formulaire qui étaient stockées dans le lot de travail d'un schéma de workflow après suppression du formulaire.

 

Notes :

  • Si vous souhaitez supprimer un formulaire utilisant un mécanisme d'envoi, tel que Versions ou Table SQL, vous devez d'abord supprimer les données stockées. Si vous supprimez le formulaire avant d'avoir supprimé ses données stockées, vous ne pouvez plus supprimer les données stockées de Content Server. Pour plus d'informations sur la suppression des données de formulaire, reportez-vous à la section Utilisation des données de formulaire.

  • Pour plus d'informations sur la suppression d'un formulaire d'un schéma de workflow, reportez-vous à la section Pour supprimer un formulaire du lot de travail.
