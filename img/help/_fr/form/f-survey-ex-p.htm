Supposons que votre entreprise effectue une enquête auprès des employés. Vous décidez que personne ne doit voir les informations saisies par les employés dans le formulaire. Cependant, vous voulez que toutes les données du formulaire soient stockées dans une table de base de données, car vous prévoyez de générer un rapport LiveReport qui évalue et présente les données.

Pour configurer un formulaire d'enquête :

  1. Créez un modèle de formulaire qui contient le texte, les champs et les boutons que vous souhaitez voir apparaître sur l'enquête.

  2. Cliquez sur l'icône Fonctions du modèle de formulaire et sélectionnez Gérer la table relationnelle. Définissez les paramètres suivants :

    • Dans la zone Nom de la table SQL, saisissez un nom pour la table des données de l'enquête.

    • Dans la liste Créer une table SQL, cliquez sur Créer une table de base de données avec le nom indiqué ci-dessus. Pour plus d'informations sur la définition d'un modèle de formulaire, reportez-vous à la rubrique Définition d'un modèle de formulaire.

  3. Ajoutez un formulaire basé sur un modèle de formulaire. Lorsque vous définissez un formulaire, utilisez les paramètres suivants :

    • Dans la liste Mécanisme de révision, cliquez sur <Aucun>.

    • Dans la liste Mécanisme d'envoi, cliquez sur Table SQL.

    Pour plus d'informations sur la définition des formulaires, reportez-vous à la rubrique Définition d'un formulaire.

Utilisation de l'exemple d'enquête

Remplissez plusieurs formulaires d'enquête (comme si vous étiez plusieurs employés fictifs) et cliquez sur le bouton Envoyer du formulaire. Remarque : vous ne pouvez pas accéder aux informations qu'un autre utilisateur a envoyées. Lorsque toutes les enquêtes sont renvoyées, créez et exécutez un rapport LiveReport qui résume et évalue les données.

Pour plus d'informations sur cette procédure, consultez la section Utilisation des exemples de formulaires.
