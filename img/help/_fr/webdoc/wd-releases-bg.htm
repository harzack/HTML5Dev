Content Server gère les versions des éléments contenus dans un document composite. En outre, les documents composites disposent de leurs propres types de versions appelées versions et révisions. Chaque version et révision est une version statique d'un document composite, créée à un moment donné dans le passé. Par exemple, si vous possédez un document composite contenant le manuel d'un logiciel, vous pouvez créer une version au moment de la publication du logiciel. Vous pouvez continuer à utiliser le document composite d'origine car la version représente le manuel tel qu'il est actuellement présenté à vos clients. Les versions sont numérotées en 1.0, 2.0, 3.0, etc.

Une révision est comparable à une version mais cette option est préférable lorsque vous souhaitez ne conserver que certaines modifications mineures apportées à un document composite et que vous n'avez pas besoin de créer une nouvelle version. Les révisions sont numérotées en fonction de la version. La révision 1.2 correspond à la deuxième révision de la première version, etc.

Accès aux versions et révisions

Vous pouvez accéder rapidement aux versions et aux révisions et les utiliser à partir de l'onglet Versions de la page Propriétés d'un document composite. Chaque version dispose de son propre menu Fonctions qui vous permet d'ouvrir, de visualiser, de télécharger, de supprimer ou de verrouiller une version ou une révision. Le verrouillage empêche la suppression d'une version ou d'une révision. Si un document composite possède un document qui utilise une création de versions avancée, les fonctions de la version et de la révision utilisent la version principale du document. Pour plus d'informations sur la création de versions avancée et sur les documents composites, voir Fonctionnement de la création de versions avancée pour les fonctions de Content Server.
