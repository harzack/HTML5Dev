Dans Content Server, l'objectif de la création de versions est de conserver un élément existant lorsqu'il est amené à être modifié. Etant donné que la création de versions est régie par des autorisations, seuls les utilisateurs disposant des autorisations appropriées peuvent ajouter manuellement une version à un élément. Pour de plus amples informations sur les autorisations, reportez-vous à la section Gestion des autorisations.

Les éléments pouvant être enregistrés sous plusieurs versions sont appelés éléments multi-versions. Content Server gère les versions des types d'éléments suivants :

  • Documents

  • Documents composites

  • Documents texte

  • Catégories

  • Schémas de workflow

  • Modèles de projet

  • Documents DTD (Document Type Definitions) XML (Extensible Markup Language)

  • Affichages personnalisés

Content Server utilise deux types de systèmes de création de versions : création de versions standard et création de versions avancée. La création de versions standard est utilisée pour tous les types d'éléments Content Server versionnables ; la création de versions avancée est utilisée pour les documents et documents texte. La création de versions avancée est activée par défaut mais peut être désactivée par votre administrateur. Si la création de versions avancée est désactivée, la création de versions standard sera utilisée pour tous les éléments Content Server, y compris les documents et documents texte.

 

Note :
Les documents composites possèdent leur propre type de versions appelées versions et révisions. Pour en savoir plus, voir Utilisation des versions et des révisions.

 

Utilisation de la création de versions standard

La création de versions standard, également appelée création de versions linéaire, affiche les versions à l'aide de nombres entiers. Par exemple : 1, 2, 3, 4. Lorsque vous ajoutez un élément versionnable à Content Server, celui-ci devient à la fois un objet du système et la version de cet élément. Si vous ajoutez ensuite une version à l'élément, celle-ci devient la version actuelle et représente l'objet dans Content Server. Par exemple, lorsque vous ajoutez une feuille de calcul à Content Server, celle-ci existe en tant que document et première version de ce document. Si vous ajoutez une version modifiée de la feuille de calcul, celle-ci devient à la fois le document objet et la deuxième version. La feuille de calcul d'origine est conservée dans Content Server en tant que première version du document.

Lorsque des changements sont apportés à un élément, Content Server ajoute automatiquement une nouvelle version. Vous pouvez également ajouter une nouvelle version d'un élément lorsque vous annulez sa réservation. Pour plus d'informations sur la réservation et l'annulation de la réservation d'éléments Content Server, consultez la rubrique Utilisation des éléments généraux.

 

Note :
Les versions d'un élément ou d'un document ne peuvent pas être réservées. Si vous souhaitez empêcher que les autres utilisateurs de modifier ou télécharger une version d'un élément ou d'un document, vous devez la verrouiller.

 

Accès aux versions standard

Vous pouvez rapidement ouvrir et utiliser les versions précédentes d'un élément contenant plusieurs versions à partir de l'onglet Versions de la page Propriétés de l'élément. Chaque version a sa propre icône Fonctions que vous pouvez utiliser pour ouvrir, visualiser, télécharger, supprimer, verrouiller ou déverrouiller une version.

 

Note :
Lorsque vous supprimez un élément qui dispose d'une version standard, vous supprimez la version actuelle et toutes les versions précédentes.

 

Par défaut, Content Server conserve un nombre illimité de versions précédentes, mais vous pouvez limiter le nombre de versions individuellement pour chaque élément. Lorsque la limite de version est atteinte, Content Server élimine automatiquement une ancienne version de l'élément lorsqu'une nouvelle version est ajoutée. Vous pouvez également éliminer des versions manuellement.

Vous pouvez verrouiller une version pour qu'elle ne soit ni modifiée ni éliminée. Content Server n'inclut pas les versions verrouillées dans le nombre maximum de versions à conserver.

Utilisation de la création de versions avancée

La création de versions avancée permet aux utilisateurs d'ajouter un document ou un sous-type de document, tel qu'un document texte, à Content Server en tant que version principale ou secondaire lors de la création du document. L'onglet Version de la page Propriétés d'un document affiche les versions principales à l'aide de nombres entiers et les versions secondaires à l'aide de décimales. Par exemple, les versions principales s'affichent sous la forme 1.0, 2.0 et 3.0 ; les versions secondaires s'affichent sous la forme 0.1, 0.2 et 0.3. Lorsque vous ajoutez un document, celui-ci est ajouté en tant que version secondaire et porte le nombre 0.1. Si une version est ajoutée à un document existant, le numéro de sa version sera le numéro de version secondaire suivant. Par exemple, si le dernier numéro de version d'un document est 1.2 et qu'une version est ajoutée, la nouvelle version portera le numéro 1.3.

 

Note :
La création de versions avancée est activée par défaut ; cependant, l'administrateur peut la désactiver.

 

Si la création de versions avancée n'est pas activée, les versions sont automatiquement ajoutées à l'aide de la création de versions standard. Lorsque la création de versions avancée est désactivée sur un élément existant auquel est associée une version principale ou secondaire, les numéros des versions seront affichés à l'aide de la numérotation utilisée pour la création de versions standard. Par exemple, 1, 2, 3 et 4. Cependant, les numéros des versions principales ou secondaires existants sont conservés dans le cas où la création de versions avancée serait réactivée plus tard.

Accès aux versions avancées

Les fonctions exécutées dans une version avancée dépendent des autorisations dont vous disposez pour cet élément particulier. Tout comme les versions standard, chaque version principale ou secondaire d'un document a sa propre icône Fonctions que vous pouvez utiliser pour ouvrir, visualiser, télécharger, supprimer ou verrouiller les versions. Si un document est réservé, vous pouvez ajouter une version principale ou secondaire lorsque vous annulez la réservation. Le tableau suivant décrit les autorisations requises pour exécuter certaines fonctions sur des versions.

Les utilisateurs ayant l'autorisation de modifier un dossier ou un projet peut forcer l'ensemble des documents ajoutés au recueil d'utiliser la création de versions avancée. Si cette option est sélectionnée, l'option permettant de choisir un type de création de versions pour un document que vous ajoutez n'apparaît pas. L'option qui permet de forcer la création de versions avancée peut être appliquée à un élément spécifique, uniquement à des dossiers ou projets dans l'élément ou à l'élément spécifique et aux dossiers et projets dans l'élément. De plus, les utilisateurs ayant l'autorisation de modification peuvent transformer les versions secondaires et principales d'un document en version principale supérieure. Les versions principales ne peuvent pas « devenir » des versions secondaires.

 

Note :
Un document qui a été ajouté à Content Server en tant que version secondaire apparaît dans Content Server avec une icône de document générique et son type MIME n'est pas spécifié jusqu'à ce qu'une version principale du document soit ajoutée ou que celui-ci soit transformé en version principale.

 

Vous pouvez afficher les versions d'un élément dans l'onglet Version de la page Propriétés de l'élément. Les informations affichées sur cette page dépendent de vos autorisations. Si vous avez :

  • Un Accès complet, ce qui signifie que l'utilisateur est autorisé à modifier un élément, l'onglet Version de la page Propriétés de l'élément affiche toutes les versions principales et secondaires.

  • Un Accès limité, ce qui signifie que l'utilisateur est autorisé à voir le contenu un élément, l'onglet Version de la page Propriétés de l'élément affiche uniquement les versions principales.

L'administrateur a également la possibilité d'autoriser les utilisateurs à ne voir que les versions principales d'un document. Si cette option est activée et qu'aucune version secondaire n'existe, vous pourrez soit voir les documents sans pouvoir y accéder, soit ne pas les voir car ces derniers seront masqués. Par défaut, les utilisateurs n'ayant accès qu'aux versions principales peuvent voir que les versions secondaires existent mais ne peuvent pas les visualiser, les ouvrir, les télécharger ou y exécuter des fonctions.

Comme pour la création de versions standard, Content Server conserve un nombre illimité de versions précédentes mais vous pouvez limiter le nombre de versions qui sont conservées pour chaque élément. Lorsqu'un document ayant des versions principales et secondaires atteint la limite maximum de versions, Content Server élimine automatiquement la version secondaire la plus ancienne. Vous pouvez éliminer manuellement et de la même façon des versions secondaires et principales. Vous devez disposer des autorisations appropriées pour pouvoir supprimer ou éliminer les versions secondaires ou pour pouvoir supprimer les versions principales. Pour supprimer les versions secondaires, vous devez disposer de l'autorisation Supprimer les versions pour un document. Pour supprimer des versions principales d'un document, vous devez disposer de l'autorisation Supprimer.

Etant donné que les versions verrouillées ne peuvent pas être supprimées, vous pouvez empêcher que des versions principales et secondaires ne soient supprimées en les verrouillant. Verrouiller des versions secondaires permet également d'empêcher qu'elles ne soient éliminées. Les versions principales ne peuvent pas être éliminées, qu'elles soient verrouillées ou non ; elles ne peuvent être que supprimées.

Le fonctionnement de certaines fonctions dans Content Server, comme la copie et le déplacement d'un élément, est affecté par la création de versions avancée. Pour plus d'informations sur les fonctions affectées par la création de versions avancée, reportez-vous à la rubrique Fonctionnement de la création de versions avancée pour les fonctions de Content Server.
