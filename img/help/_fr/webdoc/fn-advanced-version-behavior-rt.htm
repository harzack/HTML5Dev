En général, les règles de création de versions avancée s'appliquent aux documents et documents texte. Cependant, comme l'expliquent les sections suivantes, la création de versions avancée affecte certaines fonctions de Content Server.

Pour plus d'informations sur la création de versions standard et avancée, reportez-vous à la rubrique Utilisation des versions.

Déplacement et copie de documents de création de versions avancée

Lorsque vous essayez de déplacer ou de copier un document avec la création de versions avancée activée, les versions déplacées ou copiées dépendent de vos autorisations sur cet élément. Les utilisateurs devant disposer des autorisations adéquates, ils ne peuvent pas facilement accéder aux versions secondaires. Les autorisations suivantes s'appliquent lorsque les opérations de déplacement et de copie sont effectuées sur un document ayant des versions principales et secondaires :

  • L'autorisation Voir le contenu s'applique aux opérations de copie. Si un utilisateur disposant de l'autorisation Voir le contenu copie un document avec la création de versions avancée activée, la copie du document contient uniquement ses versions principales.

  • L'autorisation Réserver ou Supprimer les versions s'applique aux opérations de copie. Si un utilisateur disposant de l'autorisation Réserver ou Supprimer les versions copie un document avec la création de versions avancée activée, la copie du document contient toutes ses versions.

  • L'autorisation Supprimer s'applique aux opérations de déplacement. Si un utilisateur disposant de l'autorisation Supprimer déplace un document avec la création de versions avancée activée, le document original et toutes ses versions sont déplacés.

Définition de la notification pour les documents avec création de versions avancée

Comme pour les éléments de création de versions standard, vous pouvez définir la notification pour les documents pour lesquels la création de versions avancée est activée. Toutefois, les utilisateurs seulement autorisés à afficher les versions principales du document recevront uniquement les notifications relatives aux versions principales. Par exemple, les utilisateurs seulement autorisés à afficher les versions principales du document recevront uniquement une notification en cas d'ajout d'une version principale ou de changement de statut d'une version. Si la fonction de notification est définie pour un élément sans version principale et que l'utilisateur a pour seule autorisation l'affichage des versions principales, il ne reçoit pas de notification. Cependant, dès qu'une version principale existe, il reçoit une notification.

Pour de plus amples informations sur les notifications, reportez-vous à la section Utilisation des notifications.

Apparences avec la création de versions avancée

Les apparences sont traitées comme des documents standard et peuvent donc avoir des versions principales et secondaires. Les utilisateurs peuvent afficher les versions principales et secondaires des apparences, à condition de disposer des autorisations appropriées.

Pour plus d'informations sur les apparences, reportez-vous à la rubrique Utilisation des apparences.

Fonctionnement de la création de versions avancée avec les documents composites

Les fonctions Version et Révision d'un document composite utilisent la version principale active du document. Si un document placé dans un document composite utilise la création de versions avancée, les fonctions Version et Révision utilisent la version principale active du document.

Si un document placé dans un document composite utilise la création de versions avancée et n'a pas de version principale, les fonctions Version et Révision n'incluront pas ce document.

Pour plus d'informations sur les documents composites, reportez-vous à la rubrique Utilisation des documents composites.

Joindre des documents avec création de versions avancée aux messages et réponses

Seuls les documents ayant des versions principales peuvent être joints à des messages ou réponses de discussion. Si la version la plus récente du document est une version secondaire et qu'une version principale existe, la version principale la plus récente du document est jointe. Si aucune version principale n'existe pour un document et que vous essayez de joindre ce document à un message ou à une réponse, un message d'erreur vous indique que vous ne pouvez pas ajouter le document car aucune version principale n'existe.

Pour plus d'informations sur les discussions, reportez-vous à la rubrique Communication à l`aide des discussions.

Utilisation des modèles de projet avec les documents pour lesquels la création de versions avancée est activée

Les projets peuvent comporter des documents pour lesquels la création de versions avancée est activée. Vous pouvez donc créer des modèles de projet contenant des documents avec création de versions avancée. Si le modèle de projet comprend un document avec création de versions avancée, seules les versions que l'utilisateur a l'autorisation de Voir sont comprises dans le modèle. Si un document avec création de versions avancée est compris dans le modèle et que l'utilisateur spécifie que toutes les versions du document doivent être incluses au moment de l'instanciation du modèle, le document qui en résulte utilisera la création de versions avancée et comprendra toutes les versions principales et secondaires du document. Si un utilisateur instancie le modèle et spécifie que seule la version active doit être incluse, le document qui en résulte utilisera la création de versions avancée et inclura les versions principales actives du modèle.

Pour plus d'informations sur les modèles de projet, reportez-vous à la rubrique Utilisation des modèles de projet.
