Le contrôle de version vous permet de conserver et de gérer plusieurs versions d'éléments. Lorsque vous modifiez un élément existant, Content Server stocke les éléments modifiés en tant que nouvelle version de l'élément d'origine.

Une génération est un élément pointant vers une version spécifique (une version antérieure généralement) d'un élément et peut être stockée dans n'importe quel recueil de Content Server. La génération permet en outre de protéger une version en l'empêchant d'être modifiée ou supprimée.

Par conséquent, les versions et les générations servent d'enregistrements de l'état d'un élément au fil du temps. Elles permettent de conserver une trace des modifications effectuées et de savoir quand et par qui elles ont été effectuées.
