Content Server vous permet d'améliorer la gestion des connaissances en fournissant plusieurs types d'éléments de travail. Les informations que vous stockez dans Content Server sont présentées sous plusieurs formes. Les différents types d'éléments prennent en charge divers types d'informations. La plupart des éléments, tels que les documents et les nouvelles, correspondent à un type d'information particulier. Voici quelques exemples d'autres éléments : recueils, éléments de référence et modèles d'élément.

Les types d'éléments disponibles dans Content Server peuvent varier selon vos autorisations, vos privilèges, vos modules facultatifs et les personnalisations effectuées par votre administrateur.

Les sections suivantes décrivent les types d'éléments les plus courants, ainsi que leur utilisation.

Tableau 3-1 : Eléments de recueil

Icône Nom Description
Apparence Recueil spécifique stockant des documents et des documents de texte spécifiques contenant du code HTML utilisé pour personnaliser l'interface utilisateur de Content Server.
Dossier Recueil qui stocke d'autres éléments. Les dossiers peuvent contenir la plupart des types d'éléments. Un dossier situé dans un autre dossier est appelé sous-dossier.
Document composite Recueil qui stocke plusieurs documents (comme des chapitres de livre) selon un ordre séquentiel. Les documents composites peuvent uniquement contenir des documents, des raccourcis, des générations et d'autres documents composites.
Canal Recueil permettant de stocker et de publier des nouvelles.
Projet Environnement de travail destiné aux équipes qui collaborent suivant un objectif commun.
Discussion Recueil contenant un ensemble de messages et de réponses et permettant aux utilisateurs d'échanger des informations sur un même sujet.
Liste de tâches Recueil permettant de stocker et d'organiser les tâches.
Groupe de tâches Recueil appartenant à une liste de tâches et permettant d'organiser des groupes de tâches connexes volumineux.
Catégorie Recueil permettant de stocker et d'organiser des attributs.
Vue personnalisée Recueil spécifique servant à stocker le code HTML qui permet de personnaliser l'apparence et le fonctionnement de Content Server.

.
Tableau 3-2 : Eléments de référence

Icône Nom Description
Raccourci Raccourci vers un élément stocké ailleurs dans Content Server.
URL Adresse d'une page Web externe. Vous pouvez utiliser une URL pour renvoyer à un autre élément Content Server, mais il est préférable d'utiliser un raccourci.
Génération Pointeur vers une version particulière d'un élément (généralement une version précédente).
Version Type de version spécifique d'un document composite. Une version est une copie en lecture seule d'un document composite (et de tout son contenu) à un moment donné.
Révision Type de version spécifique d'un document composite. Une révision est une copie en lecture seule d'un document composite (et de tout son contenu) à un moment donné. Une révision est comparable à une version mais cette option est préférable lorsque vous souhaitez ne conserver que certaines modifications mineures apportées à un document composite et que vous n'avez pas besoin de créer une nouvelle version.

.
Tableau 3-3 : Eléments généraux

Icône Nom Description
Document Document dont le type n'est pas défini. Par défaut, Content Server reconnaît de nombreux types de fichiers, y compris les documents Microsoft Word, , les feuilles de calcul Microsoft Excel, , et les documents HTML, .
Document de texte Les fichiers de document sont au format texte brut.
Nouvelles Nouvelle ajoutée à une chaîne et pouvant être diffusée dans un bandeau de nouvelles. Une nouvelle est composée d'un titre et de contenu.
Sujet Début d'un fil de discussion.
Réponse Suite d'un fil de discussion.
Tâche Mission confiée à un utilisateur ou à un groupe.
Etape clé Pointeur utilisé pour associer une tâche à une étape particulière d'un projet ou d'une date.
Schéma de workflow Représentation graphique d'un processus de travail structuré.
Sondage Elément permettant aux utilisateurs de voter sur des sujets spécifiques.
Rapport LiveReport Requête directe exécutée dans la base de données Content Server. Le rapport fournit des statistiques et des informations sur la base de données et les informations qu'elle contient.
Requête Ensemble de critères de recherche enregistrés qui peuvent être réutilisés.
Lot de résultats Ensemble de résultats de recherche enregistrés. L'affichage d'un lot de résultats n'exécute pas de nouveau la recherche qui l'a généré.
Version Version précédente d'un document, d'un schéma de workflow ou d'un autre type d'élément multi-version géré par Content Server.

Tableau 3-4 : Eléments de modèle

Icône Nom Description
Modèle de recherche Configuration de la page Recherche Content Server que vous créez et enregistrez. Elle permet de réutiliser une configuration particulière de la page Recherche sans avoir à la créer à chaque fois.
Modèle de projet Elément contenant une hiérarchie d'objets préconfigurée et de contenu que vous pouvez utiliser comme modèle pour les projets que vous créez.
Modèle de résultats de recherche Configuration de la page Recherche que vous créez et enregistrez. Elle permet de réutiliser une configuration particulière de la page Recherche sans avoir à la créer à chaque fois.
