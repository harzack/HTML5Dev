Comme les différents types d'éléments sont contrôlés par différents types d'autorisations, il arrive que les autorisations ne soient pas transférées lorsque les éléments sont ajoutés, déplacés ou copiés. Par exemple, lorsque vous ajoutez une chaîne (contrôlée par les autorisations des éléments de travail) à un dossier (contrôlé par les autorisations de gestion de documents) ou que vous l'y déplacez, Content Server doit mapper les autorisations du dossier avec les autorisations les plus proches pour une chaîne.

Les tableaux suivants décrivent le mappage des différents types d'autorisation entre eux.

Autorisation des éléments de travail Autorisations de gestion de document
Aucun Aucune autorisation.
Lecture Afficher, Afficher le contenu
Ecriture Voir, Voir le contenu, Modifier, Modifier les attributs, Ajouter les éléments
Gérer Voir, Voir le contenu, Modifier, Modifier les attributs, Ajouter des éléments, Supprimer, Modifier les autorisations

 

Autorisation basée sur les rôles Autorisations de gestion de document
Invité Afficher, Afficher le contenu
Membre Voir, Voir le contenu, Modifier, Modifier les attributs, Ajouter des éléments, Réserver, Supprimer les versions
Coordinateur Voir, Voir le contenu, Modifier, Modifier les attributs, Ajouter des éléments, Réserver, Supprimer les versions, Supprimer, Modifier les autorisations

 

Note :
Le mappage des autorisations peut affecter les autorisations des utilisateurs et des groupes de la liste de contrôle d'accès d'un élément. Par exemple, si vous disposez des autorisations Voir, Voir le contenu et Ajouter des éléments sur un dossier et si un projet est ajouté à ce dossier, vous devenez automatiquement membre du projet. En plus de vos autorisations d'origine, vous êtes autorisé à réserver, modifier, modifier les attributs et supprimer les versions des éléments du projet.
