Il existe trois types d'autorisations différents mais associés :

  • Autorisations des éléments de travail, qui s'appliquent aux chaînes, discussions et listes de tâches.

  • Autorisations de gestion de document, qui s'appliquent à la plupart des types d'éléments.

  • Les autorisations basées sur les rôles, qui s'appliquent aux projets. Pour plus d'informations sur les autorisations basées sur les rôles et la manière dont elles s'appliquent aux projets, reportez-vous à la rubrique Autorisations de projet basées sur les rôles.

Les divers types d'éléments étant contrôlés par différentes autorisations, les autorisations ne sont pas toujours totalement transférées lorsque vous déplacez ou copiez des éléments. Dans ce cas, Content Server doit mapper, ou associer, les autorisations d'un type sur l'autre. Pour en savoir plus, voir Mappage des autorisations.

Autorisations des éléments de travail

Les autorisations des éléments de travail s'appliquent aux chaînes, nouvelles, discussions, messages et réponses, listes de tâches et tâches. Ces types d'éléments ne possèdent pas leurs propres autorisations. Ils possèdent uniquement les autorisations qu'ils héritent du recueil dans lequel ils sont hébergés. Les règles suivantes sont donc appliquées :

  • les nouvelles ne peuvent être ajoutées qu'aux chaînes ;

  • les messages et réponses ne peuvent être ajoutés qu'aux discussions ;

  • les tâches ne peuvent être ajoutées qu'aux listes de tâches.

Le tableau ci-dessous décrit les quatre autorisations des éléments de travail.

Autorisation Avec l'autorisation. . .
Aucun Vous ne pouvez pas afficher l'élément.
Lecture Vous pouvez afficher l'élément.
Ecriture Vous pouvez afficher et modifier l'élément.
Gérer Vous disposez d'un accès complet à l'élément. Vous pouvez notamment lire, écrire et supprimer. Vous pouvez également modifier les autorisations de l'élément.

Les autorisations des éléments de travail sont inclusives. Ainsi, si vous possédez l'autorisation Gérer, vous avez également les autorisations Lecture et Ecriture.

Autorisations des listes de tâches

Le fonctionnement des autorisations des listes de tâches diffère légèrement de celui des chaînes et des discussions car les autorisations s'appliquent au créateur de la tâche et à son utilisateur (l'utilisateur auquel la tâche est assignée). Par conséquent, les règles suivantes s'appliquent :

  • Si vous avez l'autorisation Gérer pour une liste de tâches, vous avez également toutes les autorisations pour les tâches de la liste.

  • Vous possédez toutes les autorisations pour une tâche si vous avez créé la tâche et si vous bénéficiez de l'autorisation Ecriture pour la liste de tâches. L'autorisation Ecriture pour une liste de tâches vous permet de modifier les informations de toutes les tâches de la liste.

  • Vous possédez des autorisations partielles pour une tâche si vous bénéficiez de l'autorisation Ecriture sur la liste de tâches et si la tâche vous est assignée. Cela signifie que vous ne pouvez modifier que les champs Commentaires et Statut associés à cette tâche. Vous devez disposer de l'autorisation Ecriture sur une liste de tâches pour qu'une tâche contenue dans cette liste puisse vous être assignée.

Autorisations de gestion de document

Les autorisations de gestion des documents s'appliquent à la plupart des types d'éléments Content Server, y compris les documents, les dossiers et les rapports LiveReport. Ces autorisations de gestion des documents sont plus détaillées que les autorisations des éléments de travail et permettent de contrôler plus précisément les opérations effectuées sur les éléments.

Le tableau suivant décrit chaque autorisation de gestion des documents.

Autorisation Avec l'autorisation. . .
Voir Vous pouvez voir que l'élément existe dans son recueil, mais vous ne pouvez pas l'ouvrir pour voir son contenu.
Voir le contenu Vous pouvez ouvrir l'élément et voir son contenu.
Modifier Vous pouvez renommer l'élément et modifier certaines de ses propriétés. Pour les éléments de recueil, tels que les dossiers, cela inclut les propriétés répertoriées dans l'onglet Présentation de la page Propriétés de l'élément. Pour en savoir plus, voir Personnalisation des espaces de travail et des recueils.
Modifier les attributs Vous pouvez appliquer des catégories à l'élément, modifier les catégories de l'élément ou modifier les attributs contenus dans la catégorie.
Ajouter des éléments Vous pouvez ajouter des éléments à l'élément. Cette autorisation est disponible uniquement pour les éléments de recueils.
Réserver Vous pouvez réserver l'élément (pour empêcher les utilisateurs de le modifier), le modifier, puis le déverrouiller. Vous pouvez également ajouter des versions aux éléments que vous déverrouillez. Cette autorisation n'est disponible que pour les éléments qui peuvent être réservés.
Supprimer les versions Vous pouvez supprimer des versions de l'élément. Cette autorisation n'est disponible que pour les éléments comprenant des versions, tels que les documents et les schémas de workflow.
Supprimer Vous pouvez supprimer l'élément.
Modifier les autorisations Vous pouvez modifier les autorisations que les autres utilisateurs ou groupes possèdent sur l'élément.

Toutes les autorisations sont imbriquées au sein de l'autorisation Voir. ce qui signifie que si vous ne disposez pas de cette autorisation, vous ne pouvez pas modifier ou supprimer un élément que vous ne pouvez pas voir.

 

Note :
Les autorisations Supprimer des versions et Réserver ne s'appliquent pas directement aux dossiers ou aux documents composites. Elles sont disponibles dans l'ensemble d'autorisations principalement pour les dossiers et les documents composites, de sorte que vous pouvez spécifier des autorisations par défaut pour les éléments ajoutés au dossier ou au document composite.

 

Lorsque vous accordez une autorisation, Content Server s'assure que le jeu d'autorisations de base correspondant à cette autorisation est également sélectionné.
