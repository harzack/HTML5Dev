Les autorisations affectent chaque élément et emplacement de Content Server. Il est important de comprendre le fonctionnement des autorisations, ainsi que leur impact sur votre travail. Pour plus d'informations sur les autorisations, reportez-vous à la rubrique Autorisations.

Chaque élément dans Content Server possède une liste de contrôle d'accès (ACL), qui détermine quels utilisateurs et groupes ont accès à l'élément et définit leurs autorisations. Par défaut, la liste de contrôle d'accès de chaque élément comprend les autorisations suivantes :

  • Propriétaire,  : l'utilisateur possède toutes les autorisations sur l'élément. Par défaut, le propriétaire est l'utilisateur qui a créé l'élément. Toutefois, la propriété d'un élément peut être transférée. Selon les besoins de votre entreprise, cet utilisateur n'a pas nécessairement toutes les autorisations sur l'élément.

  • Groupe de propriétaires,  : un groupe auquel vous souhaitez accorder des autorisations supplémentaires sur l'élément. Par défaut, l'élément hérite du groupe de propriétaires du recueil auquel il appartient.

  • Accès public,  : indique que tous les utilisateurs ayant le privilège Accès public peuvent accéder à l'élément. Cette option permet de définir un ensemble d'autorisations générales adapté à une utilisation classique. Le privilège Accès public étant valable dans tout le système, vous ne pouvez pas modifier les utilisateurs de ce groupe.

Par défaut, les ACL de tous les éléments contiennent ces trois entrées. Toutefois, dans certains cas, vous pouvez interdire l'accès à un élément au propriétaire, au groupe de propriétaires ou à toute personne ayant uniquement le privilège Accès public. Pour en savoir plus, contactez l'administrateur.

Modification des autorisations d'un élément

Définir des autorisations spécifiques pour chaque élément ajouté à Content Server peut s'avérer long et fastidieux. C'est pourquoi, dans la plupart des cas, l'élément hérite de ses autorisations (selon le recueil auquel il est ajouté). Toutefois, vous (ou un autre utilisateur) pouvez modifier les autorisations d'un élément à n'importe quel moment après l'avoir ajouté.

Par défaut, tout utilisateur ayant l'autorisation Modifier les autorisations sur un élément peut :

  • accorder l'accès à un élément à des utilisateurs ou des groupes ;

  • modifier les autorisations d'un utilisateur ou d'un groupe pour un élément ;

  • supprimer l'accès à un élément d'un utilisateur ou d'un groupe ;

  • modifier le propriétaire ou le groupe de propriétaires d'un élément ;

  • activer ou désactiver l'accès public.

Votre administrateur peut définir des options de contrôle d'accès supplémentaires limitant votre capacité à modifier la liste de contrôle d'accès de l'élément. Pour en savoir plus, contactez l'administrateur.

Problèmes lors de la copie et du déplacement d'éléments

Les autorisations d'un élément étant affectées par son recueil, la copie et le déplacement de l'élément peuvent entraîner la modification de ses autorisations. Lorsque vous copiez un élément dans un autre recueil, l'élément copié hérite généralement des autorisations du recueil de destination. Toutefois, l'élément déplacé conserve ses autorisations lorsqu'il est déplacé comme suit :

  • copie ou déplacement vers un emplacement situé dans le même espace de travail, y compris un espace de travail Projet ;

     

    Note :
    Si vous déplacez un dossier d'un projet à un autre, celui-ci hérite ses autorisations du projet de destination.

     

  • copie ou déplacement de l'espace de travail Entreprise vers Mon espace de travail ;

  • copie ou déplacement de Mon espace de travail vers l'espace de travail Entreprise.

Par exemple, si vous ajoutez un document dans Mon espace de travail (où vous disposez de toutes les autorisations), puis le déplacez dans l'espace de travail Entreprise (où vos autorisations sont probablement plus limitées), ce document hérite des autorisations de l'espace de travail Entreprise. Dans ce cas de figure, par exemple, vous pourriez perdre votre autorisation de suppression du document.

 

Note :
Dans certains cas, votre administrateur peut modifier ce fonctionnement.

 

Certains éléments, tels que les nouvelles, les tâches, les messages et les réponses ne peuvent pas être copiés ou déplacés. Cependant, ces éléments sont déplacés si vous déplacez le recueil dans lequel ils sont hébergés. Si vous copiez un recueil contenant des éléments ne pouvant pas être copiés, Content Server copie le recueil tout en ignorant les éléments qui ne peuvent pas être copiés.

 

Note :
Comme les différents types d'élément sont gouvernés par différents types d'autorisations, les autorisations ne se transfèrent pas toujours lorsque les éléments sont ajoutés, déplacés ou copiés. Dans ces cas précis, Content Server doit mapper les autorisations d'origine de l'élément vers les autorisations correspondantes les plus proches de sa destination. Pour plus d'informations sur la manière dont les différents types d'autorisation se mappent entre eux, reportez-vous à la rubrique Mappage des autorisations.

 

Modification des autorisations d'un sous-élément

Lorsque vous modifiez les autorisations sur un recueil, ses sous-éléments gardent leurs autorisations d'origine à moins que vous n'en décidiez autrement. Lorsque vous modifiez les autorisations des sous-éléments, vous disposez des options suivantes :

  • Remplacer, qui remplace les autorisations actuelles des sous-éléments par celles indiquées. Remplacer est la seule option disponible pour les utilisateurs propriétaires, groupe de propriétaires et accès public.

  • Ajouter, qui ajoute le nouvel utilisateur ou le groupe aux ACL des sous-éléments, ainsi que les autorisations sélectionnées.

  • Ajouter et remplacer, qui ajoute un nouvel utilisateur ou groupe à la liste de contrôle d'accès des sous-éléments et remplace tous les utilisateurs, groupes et autorisations actuels par ceux spécifiés.

  • Enlever l'utilisateur, qui enlève l'utilisateur ou le groupe de la liste d'accès et supprime ses autorisations.

 

Note :
Lorsque vous modifiez les autorisations d'un recueil contenant une liste des tâches, une chaîne ou une discussion, Content Server mappe automatiquement les autorisations du recueil avec l'autorisation de l'élément de travail correspondant le plus.
