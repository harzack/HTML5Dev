Lorsque vous travaillez en équipe, il est important de suivre les modifications apportées aux informations. Lorsque vous partagez des documents avec d'autres utilisateurs, vous devez vous assurer que :

  • les utilisateurs ne peuvent pas modifier les documents, sauf s'ils bénéficient des autorisations appropriées ;

  • une seule personne peut modifier un document à un instant donné ;

  • vous pouvez accéder aux informations relatives aux modifications des versions des documents ;

  • vous pouvez récupérer les versions précédentes des documents si nécessaire.

Content Server fournit un environnement contrôlé par version dans lequel les groupes d'auteurs peuvent s'informer du travail de chacun. Les versions servent également d'historique des modifications effectuées et de copies de sauvegarde pouvant être facilement récupérées et réutilisées.

Les éléments pouvant comporter plusieurs versions sont appelés éléments multi-versions. Les types d'éléments suivants peuvent comporter plusieurs versions :

  • Catégories

  • Documents composites

  • Affichages personnalisés

  • Documents

  • Modèles de projet

  • Requêtes du prospecteur

  • Documents texte

  • Schémas de workflow

Lorsque vous ajoutez un élément multi-version, tel qu'un document, à Content Server, il devient la première version de l'élément dans Content Server. Lorsqu'un utilisateur révise le document, les modifications sont implémentées en tant que nouvelle version. La nouvelle version devient la version actuelle du document. Par défaut, vous travaillez toujours avec la version actuelle de l'élément. Les versions précédentes ne sont pas affectées par les modifications que vous effectuez. Par défaut, Content Server peut stocker un nombre illimité de versions, mais les utilisateurs de votre entreprise ayant les autorisations appropriées peuvent restreindre le nombre de versions des éléments.

Pour plus d'informations sur les versions, reportez-vous à la rubrique Gestion des versions et des générations.
