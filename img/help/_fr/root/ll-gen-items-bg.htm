Ajout des métadonnées multilingues

L'Editeur de métadonnées multilingues, , permet de saisir des valeurs dans les champs Nom et Description dans n'importe quelle langue activée par votre administrateur. L'Editeur de données multilingues s'affiche lorsque vous ajoutez, renommez, déplacez, copiez ou configurez un élément. L'éditeur apparaît également sur la page Propriétés d'un élément. Il fonctionne de manière identique avec la plupart des pages sur lesquelles il s'affiche. Il est composé d'un champ Nom sur le premier écran et d'un champ Description sur l'écran suivant pour chaque langue activée. Sur certaines pages (les pages Configurer, par exemple), le champ Description n'apparaît pas.

Les métadonnées que vous ajoutez aux éléments sont capturées dans les historiques d'expertise et peuvent faire l'objet d'une recherche.

Ouverture d'éléments

L'ouverture est l'action par défaut pour la plupart des éléments lorsque vous cliquez sur le lien <nom_élément>. Pour les éléments qui ne disposent pas de la commande Ouvrir, l'action par défaut pour accéder ou afficher l'élément est effectuée.

Ajout d'éléments

Le menu Ajouter un élément s'affiche lorsque vous avez l'autorisation d'ajouter des éléments à un recueil. Les options disponibles pour l'ajout d'un élément varient selon le type d'élément. Pour plus d'informations sur l'ajout d'un type d'élément spécifique, reportez-vous à la rubrique d'aide consacrée à cet élément. Vous pouvez également ajouter des éléments à certains recueils à l'aide d'un glisser-déposer de votre bureau vers la zone Navigation du recueil. Pour en savoir plus, voir Ajout d'éléments avec la fonction Glisser-déposer.

Plusieurs paramètres sont communs à la plupart des éléments que vous ajoutez. Ces paramètres, répertoriés dans le tableau suivant, ne sont pas nécessaires pour ajouter un élément.

Paramètre Description
Nom Pour tous les types d'objet, le champ Nom doit être obligatoirement renseigné. Pour les documents, si le champ du nom n'est pas renseigné avant la sélection d'un fichier, le nom de fichier est automatiquement ajouté à ce champ. Il peut être utile d'attribuer un nom descriptif pour fournir des informations à l'utilisateur sur le contenu et l'utilisation de l'élément. Cela permet également d'identifier correctement l'élément lors d'une requête de recherche ou d'un rapport LiveReport.
Description Une description permet d'augmenter vos chances d'identifier correctement l'élément lors d'une requête de recherche ou d'un rapport LiveReport. Content Server indexe le texte du champ Description comme partie des métadonnées de l'élément. Les métadonnées sont les informations sur l'élément. Plus l'élément contient de métadonnées, plus il est facile pour Content Server d'identifier son contenu et son utilisation.
MacBinary (définir avant de choisir un fichier) Si vous utilisez un ordinateur Macintosh et qu'il n'exécute pas le système d'exploitation OSX, la page Ajouter un document peut afficher une case à cocher MacBinary (définir avant de choisir un fichier). Pour plus d'informations sur les fichiers Macbinary, reportez-vous à la rubrique Ajout de fichiers Macintosh dans Content Server.
Catégories Le champ Catégories permet d'appliquer un ou plusieurs attributs à un élément que vous ajoutez. Pour plus d'informations, reportez-vous à la rubrique Utilisation des catégories et des attributs.
Créer dans Par défaut, un élément est ajouté au recueil actuel. Cependant, vous pouvez utiliser le champ Créer dans pour ajouter l'élément à un recueil différent.

Rôle des autorisations lors de l'ajout d'éléments

Lors de l'ajout d'éléments, n'oubliez pas que l'accès aux éléments est contrôlé par des autorisations. Par défaut, un élément que vous ajoutez à un recueil hérite des autorisations de ce recueil. Par exemple, si un groupe possède l'autorisation Voir le contenu sur un dossier, les membres de ce groupe auront l'autorisation de voir tous les documents que vous ajoutez à ce dossier. Toutefois, il est possible de modifier les autorisations d'un élément spécifique d'un dossier. Vous êtes propriétaire des éléments que vous ajoutez. Par conséquent vous possédez toutes les autorisations pour ces éléments. Pour en savoir plus, voir Gestion des autorisations.

Ajout d'éléments avec la fonction Glisser-déposer

Les utilisateurs disposant de l'autorisation Ajouter un élément peuvent ajouter des éléments à l'aide d'un glisser-déposer de leur bureau vers la zone de navigation d'un recueil. L'icône Glisser-déposer des fichiers, indiquant que la fonction glisser-déposer est disponible pour ce recueil, apparaît sur la barre de menu :

  • pour tous les recueils de Content Server auxquels des attributs système ou de catégories obligatoires ne sont pas associés ;

  • pour les recueils comportant des données obligatoires définies avec des valeurs par défaut, mais sans aucun contenu.

 

Note :
Lorsque vous faites glisser un élément de votre bureau vers un recueil, l'élément s'affiche automatiquement en mode navigation, sauf si des attributs système obligatoires ou des catégories avec des attributs obligatoires sont associés à ce recueil. Les éléments comportant des attributs obligatoires ne peuvent pas être consultés par d'autres utilisateurs tant que les données d'attribut nécessaires n'ont pas été complétées. Votre administrateur peut définir des notifications pour que vous soyez informé par e-mail lorsque des éléments sont incomplets. Pour plus d'informations sur les éléments incomplets et les éléments comportant des informations obligatoires, reportez-vous à la rubrique Utilisation des catégories.

Si un recueil est supprimé alors qu'il contient des éléments comportant des attributs obligatoires incomplets, ces éléments ne sont pas supprimés. Ils sont transférés dans le dossier Eléments incomplets et sont accessibles au moyen de votre page Tâches assignées. Les autres utilisateurs n'ont pas accès à ces éléments tant qu'ils n'ont pas été placés dans un autre recueil. Les éléments dont les attributs obligatoires sont incomplets peuvent être supprimés individuellement.

 

Suppression d'éléments

Par défaut, lorsque vous supprimez un élément, celui-ci est enlevé définitivement de la base de données Content Server. Cependant, votre administrateur peut récupérer les éléments supprimés ou vous permettre de les récupérer. Pour en savoir plus, contactez l'administrateur.

Certains éléments ne peuvent pas être supprimés. Par exemple, vous ne pouvez pas supprimer un document si l'une de ses versions est verrouillée. Vous ne pouvez pas supprimer un élément si vous ne bénéficiez pas des autorisations nécessaires.

 

Note :
La suppression d'un élément n'entraîne pas la suppression des raccourcis vers cet élément. Vous devez les supprimer manuellement. Vous pouvez afficher une liste des raccourcis vers un élément dans l'onglet Références de la page Propriétés de cet élément. Pour en savoir plus, voir Affichage et modification des propriétés d'un élément.

 

Changement du nom des éléments

Vous pouvez renommer la plupart des éléments, en fonction du type d'élément et de vos autorisations. Toutefois, un nom descriptif permet de fournir des informations utiles sur le contenu et l'utilisation de l'élément. Cela permet également d'identifier correctement l'élément lors d'une requête de recherche ou d'un rapport LiveReport.

Copie d'éléments

Vous pouvez copier la plupart des éléments, en fonction du type d'élément et de vos autorisations. Vous pouvez copier un ou plusieurs éléments à la fois.

La copie d'un élément consiste à créer une copie de l'original et à l'enregistrer ailleurs dans Content Server. L'élément d'origine reste à son emplacement actuel. Vous pouvez également ajouter une version à l'élément copié.

Certains éléments ne peuvent pas être copiés. Par exemple, si vous ne bénéficiez pas de l'autorisation Ajouter un élément pour un recueil, vous ne pouvez pas copier un élément dans ce recueil.

 

Note :
Comme les éléments héritent les autorisations de leur recueil, le fait de copier un élément d'un recueil à un autre peut modifier les autorisations de cet élément. Pour en savoir plus, voir Gestion des autorisations.

Si vous copiez un élément auquel des classements RM ont été attribués, l'élément copié hérite du classement RM. Pour en savoir plus, voir OpenText Records Management.

 

Conséquences de la copie d'éléments sur les catégories

La copie d'éléments peut affecter l'assignation des catégories des éléments. Lorsque vous copiez un élément, vous disposez de trois options pour ses catégories. Vous pouvez :

  • conserver les catégories d'origine de la copie dans le nouveau recueil ;

  • appliquer les catégories de la destination, le nouveau recueil, à la copie ;

  • conserver les catégories d'origine de la copie et les fusionner avec les catégories du nouveau recueil.

Pour en savoir plus, voir Utilisation des catégories et des attributs.

Déplacement d'éléments

Vous pouvez déplacer la plupart des éléments, en fonction du type d'élément et de vos autorisations. Vous pouvez déplacer un ou plusieurs éléments à la fois.

Lorsque vous déplacez un élément vers un autre recueil, cet élément est enlevé de son recueil d'origine. Tous les raccourcis ou générations qui pointent vers l'élément sont automatiquement mis à jour afin de renvoyer à l'élément dans son nouveau recueil. Vous pouvez également ajouter une version à l'élément placé dans son nouveau recueil.

Certains éléments ne peuvent pas être déplacés. Par exemple, vous ne pouvez pas déplacer un élément vers un recueil pour lequel vous n'avez pas l'autorisation Ajouter un élément.

 

Note :
Comme les éléments héritent les autorisations de leur recueil, le fait de déplacer un élément d'un recueil à un autre peut modifier les autorisations de cet élément. Pour en savoir plus, voir Gestion des autorisations.

Si vous déplacez un élément auquel des classements RM ont été attribués, l'élément déplacé hérite du classement RM. Pour en savoir plus, voir OpenText Records Management.

 

Conséquences du déplacement d'un élément sur ses catégories

Le déplacement d'un élément peut affecter l'assignation de ses catégories. Lorsque vous déplacez un élément, vous disposez de trois options pour ses catégories. Vous pouvez :

  • conserver les catégories d'origine de l'élément dans le nouveau recueil ;

  • appliquer les catégories de la destination, le nouveau recueil, à l'élément ;

  • conserver les catégories d'origine et les fusionner avec les catégories du nouveau recueil.

Pour en savoir plus, voir Utilisation des catégories et des attributs.

Réservation et annulation de la réservation d'éléments

Vous pouvez empêcher d'autres utilisateurs de modifier un élément sur lequel vous travaillez en réservant l'élément pour vous-même, ou pour un groupe auquel vous appartenez. Si vous réservez un élément pour un groupe, tous les utilisateurs faisant partie de ce groupe peuvent le déverrouiller.

Il est possible de réserver les éléments suivants :

  • Documents

  • Documents texte

  • Définitions DTD (Document Type Definitions) XML (Extensible Markup Language)

  • Modèles d'élément

  • Modèles de projet

  • Affichages personnalisés

  • Requêtes du prospecteur

Lorsque vous réservez un élément, les autres utilisateurs peuvent ouvrir l'élément en lecture seule uniquement ; il leur est impossible d'y apporter des modifications tant que vous n'avez pas annulé la réservation. Si l'élément réservé est un type d'élément multi-version, vous pouvez ajouter une version de l'élément lorsque vous annulez la réservation. Pour de plus amples informations sur les versions, reportez-vous à la section Utilisation des versions.
