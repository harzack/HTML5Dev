Pour modifier les autorisations d'un sous-élément

  1. Cliquez sur l'icône Fonctions d'un élément et choisissez Autorisations.

  2. Sur la page Autorisations, cliquez sur l'icône Modifier les autorisations affectées aux sous-éléments, , située sur le côté gauche de la page.

  3. Sur la page Modifier les autorisations affectées aux sous-éléments, les utilisateurs et/ou groupes auxquels des autorisations ont été assignées précédemment s'affichent, groupés dans un tableau. Toute modification apportée est appliquée à chaque sous-élément.

    Dans la liste Opération d'autorisation, sélectionnez une option.

  4. Activez ou désactivez les cases relatives aux autorisations en fonction des autorisations que vous souhaitez appliquer aux sous-éléments.

  5. Facultatif Dans le champ Inclure, sélectionnez les cases à cocher Liste de tâches, Chaîne, Discussion ou Collection pour configurer Content Server afin qu'il mappe les changements d'autorisation si nécessaire.

  6. Sélectionnez la case à cocher Vider les listes d'accès actuelles des sous-éléments pour configurer les sous-éléments sur l'accès par défaut d'un : propriétaire, d'un groupe de propriétaires et d'un accès public.

  7. Sélectionnez la case à cocher Définir <propriétaire_recueil> comme propriétaire des sous-éléments pour définir le propriétaire du recueil comme propriétaire de tous les sous-éléments.

  8. Sélectionnez la case à cocher Définir <groupe_propriétaires_recueil> comme groupe de propriétaires des sous-éléments pour définir le groupe de propriétaires du recueil comme groupe de propriétaires de tous les sous-éléments.

  9. Cliquez sur Mettre à jour.

 

Note :
Des cases à cocher peuvent également s'afficher pour des types d'élément supplémentaires en fonction des modules optionnels installés dans Content Server.
