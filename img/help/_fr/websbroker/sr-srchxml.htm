Les index Content Server contiennent des régions qui stockent différents types d'informations. Par exemple, ils contiennent une région Emplacement qui stocke l'emplacement d'un objet dans Content Server, une région Taille qui stocke la taille d'un objet et une région Date qui stocke la date de création d'un objet. Plusieurs de ces régions disposent d'attributs système correspondants que vous pouvez rechercher sur la page de recherche Content Server.

Pour les index Content Server contenant des documents XML, un administrateur peut autoriser les recherches dans les régions ou types XML. Les types XML stockent les informations relatives aux documents XML de Content Server (par exemple, les éléments XML, leur contenu et leurs attributs) et sont définis par des fichiers DTD XML également stockés dans Content Server. Les types XML dans lesquels il est possible de faire des recherches dépendent des documents XML présents dans l'index et des types dans lesquels votre administrateur a autorisé les recherches.

Si vous ne pouvez pas effectuer de recherche dans un type XML via la section Types XML de la page Recherche Content Server, vous pouvez rechercher le contenu XML correspondant en utilisant d'autres composants de la page Recherche Content Server. Vous pouvez, par exemple, effectuer une recherche de métadonnées pour rechercher des documents créés à une date spécifique.

Il est important de comprendre le fonctionnement des types XML lorsque vous effectuez des recherches à l'aide de requêtes de base. Pour plus d'informations sur l'utilisation des requêtes de base, reportez-vous à la rubrique Recherche à l'aide de requêtes de base.
