Vous utilisez des mots clés LQL lorsque vous effectuez des recherches à l'aide de requêtes complexes. Pour plus d'informations sur l'utilisation des requêtes complexes, reportez-vous à la rubrique Recherche à l'aide de requêtes complexes.

Le langage de requête Content Server (LQL) est constitué des mots clés suivants :

ET

Description

Le mot clé AND a la même fonction que le mot clé &. Il permet de rechercher les éléments Content Server contenant les deux expressions indiquées dans la requête.

Syntaxe

<expression> AND <expression>

Exemple

Saisissez poisson AND oiseau pour rechercher les éléments Content Server contenant à la fois poisson et oiseau.

&

Description

Le mot clé & a la même fonction que le mot clé AND. Il permet de rechercher les éléments Content Server contenant les deux expressions indiquées dans la requête.

Syntaxe

<expression>  &  <expression>

Exemple

Saisissez poisson & oiseau pour rechercher les éléments Content Server contenant à la fois poisson et oiseau.

AND-NOT

Description

Le mot clé AND-NOT a la même fonction que le mot clé &! . Il permet de rechercher les éléments Content Server contenant l'expression indiquée à gauche du mot clé mais pas l'expression située à sa droite.

Syntaxe

<expression>  AND-NOT <expression>

Exemple

Saisissez poisson AND-NOT oiseau pour rechercher des éléments Content Server contenant le mot poisson mais pas le mot oiseau.

&!

Description

Le mot clé &! a la même fonction que le mot clé AND-NOT. Il permet de rechercher les éléments Content Server contenant l'expression indiquée à gauche du mot clé mais pas l'expression située à sa droite.

Syntaxe

<expression>  &! <expression>

Exemple

Saisissez poisson &! oiseau pour rechercher des éléments Content Server contenant le mot poisson mais pas le mot oiseau.

OR

Description

Le mot clé OR a la même fonction que le mot clé |. Il permet de rechercher les éléments Content Server contenant l'expression indiquée à gauche du mot clé, ceux contenant l'expression située à sa droite ou ceux qui contiennent les deux expressions.

Syntaxe

<expression>  OR  <expression>

Exemple

Saisissez poisson OR oiseau pour rechercher les éléments Content Server contenant le mot poisson ou les éléments Content Server contenant le mot oiseau ou encore les éléments Content Server qui contiennent à la fois poisson et oiseau.

|

Description

Le mot clé | a la même fonction que le mot clé OR. Il permet de rechercher les éléments Content Server contenant l'expression indiquée à gauche du mot clé, ceux contenant l'expression située à sa droite ou ceux qui contiennent les deux expressions.

Syntaxe

<expression>  | <expression>

Exemple

Saisissez poisson | oiseau pour rechercher les éléments Content Server contenant le mot poisson ou les éléments Content Server contenant le mot oiseau ou encore les éléments Content Server qui contiennent à la fois poisson et oiseau.

SOR

Description

Le mot clé SOR a la même fonction que le mot clé OR mais son algorithme de classement de pertinence considère que les expressions situées à la gauche et à la droite du mot clé SOR sont synonymes et les traite comme un terme unique. Cela permet d'éviter que les synonymes aient plus de poids dans le calcul des taux de pertinence pour un élément du résultat de la recherche. Les taux et l'ordre de classement des résultats des requêtes effectuées à l'aide du mot clé SOR sont différents de ceux issus des requêtes effectuées à l'aide du mot clé OR.

Syntaxe

<expression>  SOR  <expression>

Exemple

Saisissez "R.-U." SOR "Royaume-Uni" pour rechercher les éléments Content Server contenant au moins une occurrence du mot R.-U ou du mot Royaume-Uni ou les deux. Content Server affiche les résultats de cette requête dans un ordre différent de ceux de la requête "R.U." OR "Royaume-Uni".

XOR

Description

Le mot clé XOR recherche les éléments Content Server contenant une des deux expressions indiquées dans la requête mais pas les deux. Ce mot-clé est également appelé <OR> exclusif.

Syntaxe

<expression>  XOR  <expression>

Exemple

Saisissez poisson XOR oiseau pour rechercher les éléments Content Server contenant le mot poisson ou le mot oiseau mais pas les deux.

=

Description

Le mot-clé = permet de rechercher les éléments Content Server contenant la locution spécifiée, comme suit :

  • Pour les régions textes de métadonnées, le terme ou l'expression de recherche doivent correspondre à toute la région ou à toute une valeur de composant d'une région à valeur multiple pour être considérés comme une correspondance.

    Par exemple, [qlregion "NomVille"] ="York" correspondrait à « York » dans la région NomVille, mais ne correspondrait pas à « New York ».

  • Pour les recherches de contenu, le terme ou l'expression de recherche peut apparaître n'importe où dans le contenu de l'objet.

    Par exemple, ="York" correspondrait à un objet contenant City of York.

  • Les recherches ne sont pas sensibles à la casse.

    Par exemple, ="York" correspondrait à « YORK ».

  • Les recherches ne sont pas sensibles aux espaces ou signes de ponctuation supplémentaires, tant que ceux-ci ne séparent pas les termes de recherche.

    Par exemple, [qlregion "NomVille"] ="York" correspondrait à NomVille «  : York : ».

  • La correspondance avec un seul composant à valeurs multiples est requise pour les recherches de champs de métadonnées à valeurs multiples.

    Par exemple, si le fichier d'entrée XML d'un objet contient <NomVille>Londres</NomVille><NomVille>New York</NomVille>, une recherche pour [qlregion "NomVille"] ="New York" correspondra à l'objet car « New York » correspond à un composant entier de la région NomVille à valeurs multiples, et une recherche pour [qlregion "NomVille"] ="York" ne correspondrait pas.

Syntaxe

=<terme>

Par exemple, saisissez =York pour rechercher les éléments Content Server contenant soit une valeur région de métadonnées de « York », soit « York » dans le contenu de l'élément.

Exemple

Saisissez = York pour rechercher les éléments Content Server contenant le mot York.

NOT

Description

Le mot-clé NOT a la même fonction que le mot-clé ! et le mot-clé !=. Il recherche les éléments Content Server ne contenant pas l'expression spécifiée.

Syntaxe

<expression> NOT

Exemple

Saisissez NOT oiseau pour rechercher les éléments Content Server ne contenant pas le mot oiseau.

!

Description

Le mot clé ! a la même fonction que le mot clé NOT et le mot clé !=. Il recherche les éléments Content Server ne contenant pas l'expression spécifiée.

Syntaxe

! <expression>

Exemple

Saisissez ! oiseau pour rechercher les éléments Content Server ne contenant pas le mot oiseau.

!=

Description

Le mot clé != a la même fonction que les mots clés NOT et !. Il recherche les éléments Content Server ne contenant pas l'expression spécifiée.

Syntaxe

!= <expression>

Exemple

Saisissez != oiseau pour rechercher les éléments Content Server ne contenant pas le mot oiseau.

<

Description

Le mot clé < permet de rechercher les éléments Content Server contenant des nombres ou des mots inférieurs à celui que vous avez indiqué. Si le terme indiqué est un nombre entier ou un nombre réel, Content Server utilise la position numérique. Autrement, il utilise la position lexicographique.

Syntaxe

< <expression>

Exemple

Saisissez < 20000630 pour rechercher les éléments Content Server contenant au moins un nombre inférieur à 20000630 . Si vous restreignez cette requête à un attribut de date, elle permet de rechercher les éléments Content Server qui sont datés d'avant le 30 juin 2000.

<=

Description

Le mot clé <= permet de rechercher les éléments Content Server contenant des nombres ou des mots inférieurs ou égaux à celui que vous avez indiqué. Si le terme indiqué est un nombre entier ou un nombre réel, Content Server utilise la position numérique. Autrement, il utilise la position lexicographique.

Syntaxe

<= <expression>

Exemple

Saisissez <= 20000630 pour rechercher des éléments Content Server contenant au moins un nombre inférieur ou égal à 20000630. Si vous restreignez cette requête à un attribut de date, elle permet de rechercher les éléments Content Server qui sont datés du 30 juin 2000 ou d'avant.

>

Description

Le mot clé > permet de rechercher les éléments Content Server contenant des nombres ou des mots supérieurs à celui que vous avez indiqué. Si le terme indiqué est un nombre entier ou un nombre réel, Content Server utilise la position numérique. Autrement, il utilise la position lexicographique.

Syntaxe

> <expression>

Exemple

Saisissez > 20000630 pour rechercher les éléments Content Server contenant au moins un nombre ou un mot supérieur à 20000630. Si vous restreignez cette requête à un attribut de date, elle permet de rechercher les éléments Content Server qui sont datés après le 30 juin 2000.

>=

Description

Le mot clé >= permet de rechercher les éléments Content Server contenant des nombres ou des mots supérieurs ou égaux à celui que vous avez indiqué. Si le terme indiqué est un nombre entier ou un nombre réel, Content Server utilise la position numérique. Autrement, il utilise la position lexicographique.

Syntaxe

>= <expression>

Exemple

Saisissez >= 20000630 pour rechercher les éléments Content Server contenant au moins un nombre ou un mot supérieur ou égal à 20000630. Si vous restreignez cette requête à un attribut de date, elle permet de rechercher les éléments Content Server qui sont datés du 30 juin 2000 ou d'après.

QLLIKE

Description

Le mot-clé QLLIKE recherche les résultats dans les régions de métadonnées textes ayant des structures communes dans les numéros de pièces, les URL et les noms de fichiers.

 

Note :
QLLIKE doit être configuré par votre administrateur. Si QLLIKE est utilisé sur une région sans cette configuration, il se comporte comme un opérateur AND. Pour plus d'informations sur la configuration de recherche de votre site, contactez votre administrateur.

Lorsqu'un autre modificateur est spécifié, par exemple QLPHONETIC, il est alors prioritaire.

 

Syntaxe

qllike « <expression>  »

Exemples

Saisissez [qlregion "OTFileName"] qllike «  MonDocument99 » pour rechercher tous les objets Content Server dont le nom contient des mots similaires à MonDocument99.

QLATTRIBUTE

Description

Le mot-clé QLATTRIBUTE limite les expressions QLREGION aux valeurs de région comprenant un attribut spécifique.

Toutes les valeurs de champ OTName et OTDComment dans Content Server reçoivent un attribut lang="x" dans lequel x est le code de langage de métadonnées. QLATTRIBUTE est associé au mot-clé QLREGION pour limiter les recherches aux valeurs de région ayant un attribut particulier. L'unique attribut de région actuellement pris en charge est l'attribut lang. Les deux opérateurs utilisés avec QLATTRIBUTE sont : =, est nul.

Syntaxe

QLATTRIBUTE <expression>

Exemples

Saisissez [qlregion "OTName"] [qlattribute "lang" = "fr"] "voiture" pour rechercher tous les objets Content Server dont le nom français contient le mot voiture.

Saisissez [qlregion "OTDComment"] [qlattribute "lang" est nul] * pour rechercher tous les objets dont la description définie n'a aucun langage associé.

QLPHONETIC

Description

Le mot clé QLPHONETIC recherche les éléments Content Server contenant des mots qui ressemblent phonétiquement à l'expression indiquée dans la requête. Ce mot clé est basé sur un algorithme usuel appelé Soundex. L'algorithme Soundex crée une représentation à 4 chiffres à partir d'un mot ; celle-ci est conçue pour correspondre à la prononciation phonétique du mot d'origine. Soundex est souvent utilisé pour des recherches vagues dans lesquelles une correspondance proche est requise.

Syntaxe

QLPHONETIC <expression>

Exemple

Saisissez QLPHONETIC conte pour rechercher les éléments Content Server contenant les mots comme conte ou compte.

QLREGION

Description

Le mot clé QLREGION recherche les éléments Content Server contenant les termes de la recherche compris dans une région d'index spécifique. Chaque région regroupe un certain type d'informations concernant les éléments indexés, notamment le contenu et les métadonnées des documents. Pour obtenir la liste des régions, contactez votre administrateur Content Server.

Syntaxe

[QLREGION <region>] <expression>

Exemple

Saisissez [QLREGION OTCreatedByName] gdurand pour rechercher les éléments Content Server contenant le mot gdurand dans la région OTCreatedByName.

QLSTEM

Description

Le mot-clé QLSTEM recherche les éléments Content Server représentant les formes singulier ou pluriel de l'expression de requête spécifiée.

Syntaxe

QLSTEM <expression>

Exemple

Saisissez QLSTEM taxe pour rechercher les éléments Content Server contenant soit le mot taxe, soit le mot taxes.

QLTHESAURUS

Description

Le mot clé QLTHESAURUS recherche les éléments Content Server contenant les entrées de thésaurus associées à l'expression spécifiée dans la requête.

Syntaxe

QLTHESAURUS <expression>

Exemple

Saisissez QLTHESAURUS noyau pour rechercher les éléments Content Server contenant au moins une entrée de thésaurus associée au mot noyau, comme noyau, pépin, graine ou noix.

QLRIGHT-TRUNCATION

Description

Le mot clé QLRIGHT-TRUNCATION recherche les éléments Content Server contenant des mots dont le préfixe correspond à l'expression spécifiée dans la requête.

Syntaxe

QLRIGHT-TRUNCATION <expression>

Exemple

Saisissez QLRIGHT-TRUNCATION part pour rechercher les éléments Content Server contenant au moins un mot qui commence par part, comme particulier, partage ou partenaire.

QLLEFT-TRUNCATION

Description

Le mot clé QLLEFT-TRUNCATION recherche les éléments Content Server contenant des mots dont le suffixe correspond à l'expression de requête indiquée.

Syntaxe

QLLEFT-TRUNCATION <expression>

Exemple

Saisissez QLLEFT-TRUNCATION tion pour rechercher les éléments Content Server contenant au moins un mot qui finit par tion, comme action, définition ou spécification.

QLRANGE

Description

Le mot clé QLRANGE recherche les éléments Content Server contenant des mots (ou nombres) compris dans l'intervalle lexicographique (ou numérique) formé par les deux expressions indiquées dans la requête. Les termes que vous indiquez comme bornes de l'intervalle de recherche sont inclus dans cet intervalle. Si les termes indiqués sont des nombres entiers ou des nombres réels, Content Server utilise la position numérique. Autrement, il utilise la position lexicographique.

Vous devez placer l'intervalle de recherche entre guillemets droits, " ", et séparer les termes qui servent de bornes par un tilde ~.

Syntaxe

QLRANGE <  "expression  >~ <expression">

Exemple

Saisissez QLRANGE "girafe ~ gnou" pour rechercher les éléments Content Server contenant au moins un mot de l'intervalle lexicographique qui va de girafe à gnou, comme girafe, girafes, girolle, glace, gnome ou gnou.

Saisissez QLRANGE "20000801 ~ 20000901" pour rechercher les éléments contenant au moins un nombre compris entre 20000801 et 20000901, par exemple entre les dates du 1er août 2000 et du 1er septembre 2000 compris.

QLREGEX

Description

Le mot clé QLREGEX recherche les éléments Content Server contenant une extension de l'expression régulière spécifiée dans la requête. Les expressions de requête utilisées avec le mot clé QLREGEX doivent être délimitées par des guillemets droits, " ". Ce mot-clé ne s'applique pas aux locutions (sur les unités lexicales). Pour plus d'informations sur l'utilisation des expressions régulières, reportez-vous à la rubrique Utilisation des expressions régulières avec LQL.

Syntaxe

QLREGEX <"expression">

Exemple

Saisissez QLREGEX "^chat" pour rechercher les éléments Content Server contenant au moins l'une des extensions de l'expression régulière ^cha, par exemple chat, chatte, chaton, char, chant, chaîne, chanoine, chagrin ou chaque.

PROX

Description

Le mot clé prox[n,t/f] permet de rechercher les éléments Content Server contenant toutes les locutions de requête spécifiées se trouvant à une distance désignée (n) l'une de l'autre, et où l'ordre est classé (t) ou non (f). La distance (n) est calculée comme le nombre d'unités lexicales entre la dernière unité de la première locution et la première de la seconde. La spécification du second paramètre t/f) est facultative et a par défaut la valeur f.

 

Note :
L'opérateur PROX se limite strictement à la simple recherche de termes et de locutions exactes. Il n'est pas générique.

 

Syntaxe

<expression> prox[n,t/f] <expression>

Exemple

Saisissez course prox[10,t] voitures pour rechercher les éléments Content Server contenant le mot course précédant le mot voitures jusqu'à une distance de 10 unités lexicales d'écart.

QLNEAR

Description

Le mot-clé QLNEAR a la même fonction que le mot-clé PROX, mais il est défini sur la valeur par défaut prox[10,f].

Syntaxe

<expression >QLNEAR <expression>

Exemple

Saisissez course QLNEAR voitures pour rechercher les éléments Content Server contenant le mot course et le mot voitures jusqu'à une distance de 10 unités lexicales d'écart, dans n'importe quel ordre.
