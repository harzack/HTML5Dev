Vous pouvez effectuer une recherche Content Server à l'aide de requêtes complexes permettant de cibler précisément les données à récupérer. Lorsque vous créez une requête complexe, vous avez la possibilité de l'enregistrer afin de pouvoir la réutiliser ultérieurement. Pour plus d'informations sur l'enregistrement des requêtes, reportez-vous à la rubrique Utilisation des recherches enregistrées.

Utilisation du langage de requêtes Content Server, LQL

Une requête complexe est une instruction unique, formulée en langage LQL, qui vous permet de cibler avec précision les données que vous souhaitez récupérer. Par exemple, pour rechercher des éléments Content Server contenant le mot rapport mais pas le mot frais, saisissez la requête complexe rapport AND-NOT frais.

Les requêtes LQL initiées depuis la page Recherche avancée renvoient des éléments OpenText Enterprise Library si vous avez spécifié une vue OpenText Enterprise Library dans la section Vues de recherche.

Utilisation des conventions LQL

Les conventions LQL définissent la syntaxe des requêtes complexes. Elles spécifient comment utiliser les caractères génériques, comment définir l'ordre dans lequel Content Server traite les termes de requête et comment définir la syntaxe pour créer des expressions. Pour plus d'informations sur les conventions LQL, reportez-vous à la rubrique Conventions LQL.

Utilisation des mots clés LQL

Les mots clés LQL sont des mots que le langage de requête Livelink se réserve et auxquels il attribue une signification particulière. Lorsque les mots clés LQL sont utilisés avec les conventions du langage de requête Livelink, ils vous permettent de créer des requêtes complexes. Les mots clés LQL ne sont pas sensibles à la casse. Pour plus d'informations sur les mots clés LQL, reportez-vous à la rubrique Mots clés LQL.

Spécification des attributs à l'aide du langage LQL

Le langage LQL constitue une autre méthode de spécification des attributs lors de la recherche d'éléments Content Server. Chaque élément Content Server possède certains attributs contenant des informations supplémentaires sur cet élément. Par défaut, les recherches Content Server ne portent pas sur les attributs des éléments. Toutefois, vous avez la possibilité de spécifier des attributs dans les requêtes de base ; vous pouvez également utiliser le langage LQL pour rechercher un attribut en particulier ou encore un ensemble d'attributs. Pour plus d'informations sur les attributs, consultez la section Utilisation des catégories et des attributs. Pour plus d'informations sur la spécification des attributs à l'aide du langage LQL, reportez-vous à la rubrique Spécification des attributs de requête à l'aide du langage LQL.

 

Note :
Les noms d'attributs sont sensibles à la casse. Pour obtenir la liste des attributs disponibles et leur casse correcte, contactez votre administrateur.

 

Utilisation des expressions régulières avec LQL

Vous pouvez utiliser des expressions régulières à l'aide du langage LQL pour rechercher des éléments Content Server contenant une extension de l'expression régulière de l'expression spécifiée dans la requête. Les expressions de requête utilisées avec le mot-clé QLREGEX doivent être délimitées par des guillemets droits «   ». Pour plus d'informations sur l'utilisation des expressions régulières, reportez-vous à la rubrique Utilisation des expressions régulières avec LQL.

Didacticiel LQL

Le langage de requêtes permet aux utilisateurs de construire des requêtes de recherche pouvant être traitées par le moteur de recherche OpenText. Ces requêtes de recherche sont décrites dans Didacticiel de requête dans la barre de recherche.
