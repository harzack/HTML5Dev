Vous pouvez effectuer une recherche Content Server en créant une requête de base contenant certains mots clés. Une requête de base peut également contenir une combinaison de mots clés et d'autres critères permettant d'élargir ou au contraire d'affiner votre recherche. Lorsque vous créez une requête de base, vous avez la possibilité de l'enregistrer afin de pouvoir la réutiliser ultérieurement. Pour plus d'informations sur les autres types de requêtes, reportez-vous aux rubriques Recherche à l'aide de requêtes en langage naturel ou Recherche à l'aide de requêtes complexes. Pour plus d'informations sur l'enregistrement des requêtes, reportez-vous à la rubrique Utilisation des recherches enregistrées.

Vous pouvez rechercher des éléments OpenText Enterprise Library à condition de disposer des autorisations appropriées pour afficher les vues OpenText Enterprise Library. Pour plus d'informations sur la recherche d'éléments OpenText Enterprise Library, reportez-vous à la rubrique Using the Search Bar to Find Open Text Enterprise Library Services Items.

Utilisation de Texte intégral

La section Texte intégral de la page de recherche Content Server vous permet de créer des requêtes qui contiennent des mots clés ou des requêtes complexes construites à l'aide du langage de requêtes Content Server. Cette section comprend également des composants qui vous permettent d'étendre ou de limiter votre recherche. Pour étendre votre recherche, sélectionnez un opérateur qui indique la relation que vous souhaitez inclure entre le terme saisi et votre recherche. Pour limiter votre recherche, précisez si vous souhaitez effectuer la recherche dans le contenu, dans les métadonnées des éléments ou dans les deux. Le contenu désigne les informations contenues dans les éléments Content Server (le texte d'un document par exemple) ; les métadonnées sont les informations qui définissent chaque élément Content Server comme la date à laquelle le document a été ajouté à Content Server. Vous pouvez également limiter votre recherche en utilisant des opérateurs booléens pour définir précisément la relation entre les différents critères de recherche de votre requête. Pour en savoir plus, voir Recherche Content Server.

 

Note :
Si vous effectuez des recherches dans des métadonnées, votre recherche se limite aux métadonnées pour lesquelles votre administrateur a autorisé les recherches (c'est-à-dire, s'il a appliqué la recherche par défaut). Il est possible que cette option n'inclue pas toutes les métadonnées associées à un élément. Lorsque vous effectuez des recherches dans le contenu et dans les métadonnées, vous faites des recherches dans le contenu et dans les métadonnées pour lesquelles votre administrateur a autorisé les recherches.

 

Utilisation des mots clés

Les mots clés sont des mots ou des groupes de mots que vous spécifiez pour rechercher des éléments Content Server contenant ces mots ou ces groupes de mots. Par exemple, si vous spécifiez les mots clés modèle de plan de développement, Content Server peut vous proposer une liste d'éléments Content Server contenant ces trois mots, n'importe lequel de ces mots ou la locution exacte. Si plusieurs mots sont indiqués entre guillemets, Content Server recherche la locution exacte mentionnée entre guillemets.

Extension d'une recherche

Vous pouvez étendre votre recherche à des éléments Content Server contenant des mots associés aux mots spécifiés dans la requête. Certains opérateurs vous permettent ainsi de spécifier le type de mots associés à inclure dans la recherche. Par exemple, vous pouvez sélectionner un opérateur permettant de rechercher les éléments Content Server contenant des mots qui, phonétiquement, ressemblent au mot clé spécifié. Vous pouvez également sélectionner un opérateur permettant de rechercher les éléments Content Server contenant des mots dérivés de la racine du mot clé spécifié. Par exemple, si vous recherchez les termes ayant la même racine que le mot clé communiquer, vous obtenez la liste des éléments Content Server contenant les termes communiquer, communiquant, communiquent et communiqué.

Lorsque vous utilisez des mots clés avec un opérateur, celui-ci s'applique à tous les mots. Lorsque vous recherchez une la locution exacte, l'opérateur s'applique à l'ensemble de l'expression. Dans une requête complexe, l'opérateur s'applique à tous les mots de la recherche qui ne sont pas des mots clés du langage de requêtes Content Server. Pour plus d'informations sur le langage de requête Content Server, reportez-vous à la rubrique Recherche à l'aide de requêtes complexes.

Limitation d'une requête

Pour limiter une recherche, vous pouvez augmenter le nombre de critères spécifiques utilisés par Content Server dans le cadre de cette recherche. Ces critères regroupent l'utilisation d'opérateurs booléens (Et (AND), Ou (OR) et Sauf (NOT). Ces opérateurs appliquent la logique élémentaire permettant de modifier la relation entre les mots clés. Les critères regroupent également la possibilité de faire porter la recherche sur le contenu ou les métadonnées d'un élément (ou les deux). Les autres types de critères de recherche sont les options avancées : vues de recherche, attributs système, attributs personnalisés au sein d'une catégorie et types XML. Le tableau ci-dessous décrit les critères de recherche disponibles.

Tableau 1-1 : Critère de recherche

Critères Fonction
Dans Tous Permet de rechercher les informations contenues dans chaque élément Content Server, comme dans le texte de documents ou dans les informations de métadonnées dont la recherche a été autorisée par votre administrateur.
Contenu Permet de rechercher les informations contenues dans les différents éléments Content Server, par exemple dans le texte des documents.
Métadonnées Permet de rechercher les informations accompagnant les différents éléments Content Server, par exemple la date d'ajout d'un document à Content Server. Si vous effectuez des recherches dans des métadonnées, votre recherche se limite aux métadonnées pour lesquelles votre administrateur a autorisé les recherches (c'est-à-dire, s'il a appliqué la recherche par défaut). Il est possible que cette option n'inclue pas toutes les métadonnées associées à un élément.
Opérateurs booléens Permet de définir avec précision la relation entre les différents critères de recherche mentionnés dans la requête. Par exemple, si les mots-clés indiqués sont plan et test, rechercher les éléments plan AND test permet d'obtenir des éléments de Content Server contenant plan et test ; plan OR test permet d'obtenir les éléments Content Server contenant plan ou test ou les deux ; plan NOT test permet d'obtenir les éléments Content Server contenant plan, mais pas test.
Vues de recherche Permet de faire porter la recherche sur un ensemble d'informations (ou vue de recherche) Content Server. Par exemple, choisissez la vue Aide pour rechercher des informations dans l'aide utilisateur en ligne de Content Server.
Attributs système Permet de rechercher les éléments Content Server possédant certains attributs. Par exemple, vous pouvez rechercher tous les éléments de Content Server dont l'attribut Créé par correspond à un utilisateur en particulier, ou encore un langage spécifique lorsque vous recherchez un nom ou une description. Pour plus d'informations sur les attributs, reportez-vous à la rubrique Utilisation des catégories et des attributs.
 

Note :
Dans la liste déroulante Date, si vous spécifiez Les 3 derniers mois ou Les 6 derniers mois, l'intervalle de dates défini dans la requête ne correspond pas exactement à 3 ou 6 mois. Le calcul s'effectue plutôt avec, respectivement, 90 jours et 180 jours. Certains mois comptant plus ou moins de 30 jours, la requête effectuée sur cet intervalle de dates peut ne pas afficher certains éléments qui devraient normalement apparaître.

Vous ne pouvez pas rechercher de fichiers XML via une recherche Attributs système. Pour effectuer une recherche sur une région XML, utilisez le composant Types XML pour sélectionner la région DTD.

Lorsque vous choisissez des options dans les listes déroulantes Type de contenu ou Type MIME, votre requête n'est pas modifiée tant qu'aucune option n'est sélectionnée. Lorsque vous sélectionnez Tout type MIME ou Aucun type MIME n'a été défini, une région est définie. L'état par défaut est « vide ».

 
Catégories Cette option permet de rechercher les éléments Content Server possédant certains attributs personnalisés au sein d'une catégorie. Par exemple, vous pouvez rechercher tous les éléments Content Server dont l'attribut personnalisé Nom de l'entreprise est défini sur Open Text et dont l'attribut personnalisé Numéro de fichier est défini sur 93761634 dans la catégorie Informations sur l'entreprise. Pour plus d'informations sur les catégories, reportez-vous à la rubrique Utilisation des catégories et des attributs.
Types XML Permet de rechercher les contenus XML dans Content Server. Choisissez une DTD XML qui définit les types XML, ou les régions XML, que vous souhaitez rechercher. Pour plus d'informations sur les types XML, reportez-vous à la rubrique Types XML.
Emplacement Permet de trouver un emplacement spécifique dans Content Server, tel qu'un dossier, un projet ou un espace de travail particulier.

Spécification des attributs système

Par défaut, la page de recherche Content Server contient de nombreux attributs système. Toutefois, certains attributs système doivent être activés par un administrateur pour être affichés. Par exemple, un administrateur peut activer l'attribut OTDocumentRating de manière à permettre aux utilisateurs de rechercher les documents associés à une valeur d'évaluation donnée. L'administrateur est souvent appelé à activer de nouveaux attributs lors de l'installation d'un nouveau module Content Server ou d'une nouvelle version Content Server. Pour plus d'informations sur les attributs, consultez la section Utilisation des catégories et des attributs.
