Pour effectuer une recherche à l'aide de la recherche avancée :

  1. Dans le menu Outils de la barre de menu générale, choisissez Rechercher.

  2. Dans le composant Texte intégral de la page Recherche avancée Content Server, procédez comme suit :

    1. Choisissez les options dans les listes déroulantes Rechercher, Modificateur et Dans.

    2. Saisissez votre terme de recherche dans la zone de texte.

    3. Cliquez sur le bouton Dupliquer pour ajouter davantage d'options et ainsi affiner votre recherche, si nécessaire.

     

    Note :
    Les composants qui apparaissent dans votre Content Server peuvent être différents de ceux mentionnés ici.

    Lorsque vous effectuez une recherche en utilisant plusieurs composants, par exemple Texte intégral, Requête en langage naturel, etc., ils sont implicitement combinés ou « cumulés » ensemble.

     

  3. Sous Ajouter au formulaire de recherche, cliquez sur Requête en langage naturel et procédez comme suit :

    1. Choisissez une option dans la liste déroulante Mode.

    2. Saisissez une question ou du texte dans la zone de texte.

  4. Cliquez sur Vue de recherche et choisissez la vue de recherche Content Server que vous souhaitez rechercher. Si vous ne sélectionnez pas explicitement une vue de recherche Content Server, la vue de recherche Entreprise est recherchée par défaut.

     

    Note :
    Si une vue de recherche est supprimée, elle apparaîtra toujours dans la liste déroulante des vues de recherche et pourra être renvoyée dans un appel de fonction func=search.getslices. Ceci est dû aux caractéristiques de mémoire cache dans Content Server. Lorsque la mémoire cache sera nettoyée de toutes les threads actives, la vue de recherche n'apparaîtra plus.

     

  5. Cliquez sur Attributs système et procédez comme suit :

    1. Choisissez les options à partir des listes déroulantes Attributs supplémentaires et Type de contenu.

       

      Note :
      Dans la liste déroulante Date, si vous spécifiez Les 3 derniers mois ou Les 6 derniers mois, l'intervalle de dates défini dans la requête ne correspond pas exactement à 3 ou 6 mois. Le calcul s'effectue plutôt avec, respectivement, 90 jours et 180 jours. Certains mois comptant plus ou moins de 30 jours, la requête effectuée sur cet intervalle de dates peut ne pas afficher certains éléments qui devraient normalement apparaître.

      Lorsque vous recherchez des dates, vous pouvez sélectionner l'option « vide ». Il s'agit du paramètre par défaut qui exclut le champ date de la requête de recherche.

      Vous ne pouvez pas rechercher de fichiers XML via une recherche Attributs système. Pour effectuer une recherche sur une région XML, utilisez le composant Types XML pour sélectionner la région DTD.

      Lorsque vous choisissez des options dans les listes déroulantes Type de contenu ou Type MIME, votre requête n'est pas modifiée tant qu'aucune option n'est sélectionnée. Lorsque vous sélectionnez Tout type MIME ou Aucun type MIME n'a été défini, une région est définie. L'état par défaut est « vide ».

       

    2. Cliquez sur pour choisir un utilisateur Content Server à saisir dans la zone de texte Créé par.

  6. Cliquez sur Catégories et choisissez une catégorie si plusieurs attributs permettent d'affiner votre recherche.

    Lorsqu'un attribut Case à cocher est ajouté à une catégorie et que le paramètre Afficher dans la recherche est activé (voir Utilisation des attributs), une liste déroulante est ajoutée au composant Catégorie. L'affichage par défaut de l'attribut indique que deux entrées existent. Dans une entrée, la Case à cocher est désactivée, dans l'autre entrée, la Case à cocher est activée.

    L'attribut Case à cocher vous permet de sélectionner une valeur à partir de la liste déroulante :

    • N'importe lequel : valeur par défaut, vos sélections Modificateur et Menu contextuel Date ne sont pas incluses dans la requête de recherche

    • True : vos sélections Modificateur et Menu contextuel Date doivent être vraies pour être incluses dans les résultats de recherche. Cette valeur est l'équivalent de Case à cocher activée

    • False : vos sélections Modificateur et Menu contextuel Date doivent être fausses pour être incluses dans les résultats de recherche. Cette valeur est l'équivalent de Case à cocher désactivée

    • Défini : seules les valeurs dont la Case à cocher est définie sont renvoyées dans les résultats de recherche.

    • Non défini : toutes les valeurs dont la Case à cocher n'est pas définie sont renvoyées dans les résultats de recherche.

    • Pas vrai : vos sélections Modificateur et Menu contextuel Date ne doivent pas être vraies pour être incluses dans les résultats de recherche.

    • Pas faux : vos sélections Modificateur et Menu contextuel Date ne doivent pas être fausses pour être incluses dans les résultats de recherche.

     

    Note :
    OpenText vous recommande fortement de ne pas dupliquer cette ligne pour ajouter davantage d'options, chaque sélection de recherche étant couverte pour tous les cas possibles.

     

    Lorsque vous recherchez des dates, vous pouvez sélectionner l'option « vide ». Il s'agit du paramètre par défaut qui exclut le champ date de la requête de recherche.

  7. Cliquez sur Types XML et choisissez un Volume DTD XML pour rechercher les contenus XML.

  8. Cliquez sur Emplacement et choisissez un emplacement Content Server à saisir dans la zone de texte.

  9. Facultatif Cliquez sur Types d'élément physique et sélectionnez les types d'élément physique que vous souhaitez utiliser dans la recherche.

  10. Facultatif Cliquez sur Attributs d'élément physique et sélectionnez les attributs d'élément physique disponibles que vous souhaitez utiliser dans la recherche.

  11. Facultatif Cliquez sur Attributs de Records management et sélectionnez les attributs de Records management disponibles que vous souhaitez utiliser dans la recherche.

  12. Facultatif Cliquez sur Attributs d'habilitation de sécurité et sélectionnez les attributs d'habilitation de sécurité disponibles que vous souhaitez utiliser dans la recherche.

  13. Cliquez sur le bouton Rechercher ou Réinitialiser le formulaire de recherche.

     

    Note :
    Lorsque vous cliquez sur Réinitialiser, les termes de recherche retrouveront leurs valeurs initiales.

     

  14. Sous Options d'enregistrement, cliquez sur Enregistrer votre recherche ou Enregistrer en tant que formulaire pour enregistrer votre recherche ou créer un formulaire de recherche.
