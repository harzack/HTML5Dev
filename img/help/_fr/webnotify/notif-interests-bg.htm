Vous pouvez définir des centres d'intérêt des notifications pour suivre l'évolution des types d'événements qui surviennent. Les notifications permettent de suivre l'évolution de deux types de demandes : les demandes générales et spécifiques. Les centres d'intérêt généraux permettent de suivre l'évolution des types d'événement qui surviennent sur l'ensemble de Content Server. Ils s'appliquent à tous les éléments pour lesquels l'autorisation Voir vous a été attribuée. Par exemple, si vous définissez la notification pour la demande Un élément est ajouté, vous recevez une notification à chaque fois qu'un élément est ajouté à Content Server, à condition que l'autorisation Voir vous ait été attribuée pour cet élément.

Les demandes de notification spécifiques permettent de suivre l'évolution des événements qui surviennent dans des recueils ou des éléments spécifiques. Par exemple, vous pouvez définir une notification sur une tâche assignée pour suivre les modifications d'échéance de la tâche assignée. Chaque élément de Content Server possède son propre ensemble de demandes de notification spécifiques.

Si vous disposez des privilèges appropriés, vous pouvez définir les demandes de notification par défaut pour les utilisateurs d'un groupe. Par défaut, vos centres d'intérêt sont copiés à partir de vos centres d'intérêt du groupe par défaut ; toutefois, si vous modifiez vos centres d'intérêt personnels, ceux-ci remplaceront vos centres d'intérêt du groupe par défaut.

 

Note :
Seul l'administrateur ou un utilisateur disposant des droits d'administration système adéquats peut gérer les intérêts par défaut d'un groupe.
