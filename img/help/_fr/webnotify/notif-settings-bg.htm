Les paramètres de notification déterminent la date et l'heure auxquelles les rapports de notification sont générés et envoyés par e-mail.

Vous pouvez définir les paramètres pour votre rapport de notification personnel et pour un rapport de notification de projet si vous êtes coordinateur de projet. Déterminez le type de demandes de notification que vous souhaitez afficher dans chaque rapport, puis définissez la date et l'heure d'envoi. Par exemple, vous pouvez souhaiter être informé toutes les heures des tâches et des étapes du workflow qui sont ajoutées, mais seulement une fois par semaine pour les versions qui sont ajoutées à un document. Pour plus d'informations sur la notification et les rapports, reportez-vous aux rubriques Définition des demandes de notification et Affichage des rapports de notification.

Vous pouvez également définir les paramètres de notification pour les utilisateurs d'un groupe si vous disposez des privilèges appropriés. Les rapports de notification de groupe n'apparaissent pas sur une page distincte. Les notifications définies pour les membres d'un groupe s'affichent dans l'un des rapports de notification personnels de l'utilisateur. Les paramètres de notification de groupe ne remplacent pas les paramètres de notification personnels.

Les paramètres de notification que vous pouvez définir sont les mêmes pour les rapports personnels et de projet. Ils sont décrits dans le tableau suivant.

Paramètre de notification Description
Nom du rapport Le nom du rapport.
Fonctionnement Détermine si une nouvelle fenêtre de navigateur s'ouvre lorsque vous cliquez sur les liens d'un rapport.
Envoi de l'e-mail Permet de recevoir les rapports de notification par e-mail.
Adresse e-mail Lorsque l'envoi par e-mail est activé, vous devez indiquer l'adresse e-mail à laquelle vous souhaitez envoyer le rapport de notification.
Contenu de l'e-mail Détermine la façon dont le rapport est envoyé par e-mail. Corps de texte normal uniquement permet d'envoyer le rapport dans un e-mail en texte normal, sans lien hypertexte. Corps HTML uniquement permet d'envoyer le rapport dans un e-mail au format HTML et d'inclure des liens hypertextes. Corps de texte normal avec pièce jointe HTML permet d'envoyer le rapport en pièce jointe avec un lien vers la page de rapport.
Date et heure Les sections Les jours suivants, Aux heures suivantes et Aux minutes suivantes vous permettent de définir les dates et heures auxquelles sont envoyés les rapports.
Options de suppression Détermine la fréquence à laquelle Content Server efface automatiquement la page de rapport de notification de projet.

 

Note :
Les paramètres de l'e-mail et de la date sont uniquement disponibles pour les notifications personnelles. Le paramètre Eliminer les événements antérieurs à est uniquement disponible pour les notifications de projet.
