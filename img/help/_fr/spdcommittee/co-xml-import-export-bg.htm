Les capacités d'importation et d'exportation natives de Content Server vous permettent de convertir certains objets et informations contenus dans une communauté, en un fichier XML. Ce fichier peut ensuite être importé dans une autre communauté, ailleurs dans Content Server, ou dans une communauté située dans une instance de Content Server différente. Il est également possible d'exporter une communauté.

Exportation

Les types d'objet de communauté suivants (également appelés nœuds) peuvent être exportés :

  • Communauté

  • Sous-communauté

  • Modèles de communauté

     

    Note :
    Il est possible d'exporter une communauté entière et de l'importer dans le volume des modèles de communauté (ce qui a pour effet de transformer la communauté importée en modèle de communauté). Il est impossible de copier une communauté dans le volume des modèles de communauté pour la transformer en modèle. Pour de plus amples informations sur les modèles de communauté, reportez-vous à la section Gestion des modèles de communauté.

     

  • Forums

  • Blogs

  • Archive de messagerie

     

    Note :
    Si vous exportez une archive de messagerie et que vous l'importez ensuite dans une communauté existante, l'archive de messagerie importée sera fusionnée avec la communauté de destination existante.

     

  • Bibliothèque

     

    Note :
    Une bibliothèque de communauté est exportée sous la forme d'un dossier. Ceci signifie que vous pouvez exporter une bibliothèque et importer son contenu dans un autre dossier ou dans une autre bibliothèque de communauté, où la bibliothèque importée apparaîtra sous la forme d'un nouveau dossier.

Note :
Les groupes d'experts ne peuvent pas être exportés individuellement ; cependant, ils peuvent être exportés dans le cadre de l'exportation d'une communauté entière.

 

Lors de l'exportation d'une communauté ou du contenu à l'intérieur d'une communauté, vous pouvez spécifier les paramètres suivants :

  • Portée de l'exportation, incluant les éléments suivants :

    • Ce nœud seulement, soit l'élément seul.

    • Ce nœud et les enfants de premier niveau uniquement, soit l'élément et son contenu immédiat exportable (mais pas les éléments contenus dans ces éléments).

    • Ce nœud et tous les nœuds enfants, soit l'élément et la totalité de son contenu exportable, y compris tous les éléments exportables contenus dans ces éléments.

  • Informations sur les catégories et les attributs. Pour de plus amples informations sur les catégories et attributs, reportez-vous à la section Utilisation des catégories et des attributs.

  • Informations sur la version, soit l'exportation des informations sur la version en cours d'un élément uniquement ou sur toutes les versions Pour de plus amples informations sur les versions, reportez-vous à la rubrique Utilisation des versions.

  • Type de texte de la version, soit l'exportation du contenu de la version sous la forme de texte brut ou de texte codé en base 64.

  • Informations sur les autorisations, soit l'inclusion des informations sur les autorisations d'un élément dans les données exportées. Pour de plus amples informations sur les autorisations, reportez-vous à la section Autorisations.

Importation

Vous pouvez effectuer une importation XML vers les types d'objet ou les emplacements suivants :

  • Communautés

  • Modèles de communauté

  • Dossiers de modèles

  • Volume des modèles de communauté

Pour de plus amples informations sur les modèles de communauté, reportez-vous à la section Gestion des modèles de communauté.

La seule restriction lors de l'importation est que l'objet spécifié par le fichier XML doit être conforme au schéma XML natif de Content Server.
