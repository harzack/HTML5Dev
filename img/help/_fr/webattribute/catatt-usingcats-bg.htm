Les attributs ne pouvant exister qu'au sein d'une catégorie, l'application d'une catégorie à des éléments implique l'application des attributs contenus dans la catégorie. Les catégories définissent l'ordre, la disposition, le type et les valeurs par défaut des attributs qu'elles contiennent. Une catégorie peut contenir un seul attribut. Vous pouvez appliquer plusieurs catégories à un élément.

Tous les utilisateurs peuvent appliquer des catégories lors de l'ajout, de la modification et du déplacement d'éléments. La page Ajouter de la plupart des éléments comprend une section Catégories dans laquelle vous pouvez choisir d'appliquer une catégorie. La plupart des éléments comportent également un onglet Catégories sur leur page Propriétés. Si vous possédez les autorisations appropriées, vous pouvez modifier les informations sur les catégories appliquées à un élément existant ou enlever les affectations de catégories. Vous pouvez appliquer toutes les catégories pour lesquelles vous possédez les autorisations appropriées. Dans la plupart des cas, les catégories sont appliquées au volume des catégories de Content Server. Tous les utilisateurs Content Server peuvent accéder aux catégories qui se trouvent à cet emplacement.

Si un recueil auquel vous ajoutez un élément comporte une catégorie à laquelle sont associés des attributs obligatoires, vous devrez entrer les informations obligatoires lors de l'ajout de l'élément. Une catégorie comportant des attributs obligatoires est indiquée par l'icône Obligatoire .

Vous pouvez aussi appliquer des catégories à tous les éléments d'une collection en sélectionnant Appliquer les catégories dans la liste Autres actions. Si vous avez déjà une collection, vous pouvez la classer en sélectionnant Appliquer le classement dans la liste Autres actions.

Eléments glissés-déposés avec attributs obligatoires

Si vous déplacez un élément à l'aide d'un glisser-déposer vers un recueil comportant des attributs système obligatoires ou une catégorie avec attributs obligatoires, vous devez ajouter les informations obligatoires afin de permettre aux autres utilisateurs d'accéder à cet élément. Vous pouvez accéder à une liste des éléments incomplets soit à partir de la zone Glisser-déposer ou du mode d'exploration d'un recueil, soit via la page Tâches assignées. Lorsque vous accédez à la liste des éléments incomplets, complétez les éléments en ajoutant les informations obligatoires pour chaque élément ou mettez à jour plusieurs éléments à la fois lorsqu'ils nécessitent les mêmes informations obligatoires.

Si vous supprimez un recueil contenant des éléments aux attributs obligatoires incomplets, ces éléments ne sont pas supprimés mais déplacés vers le dossier Eléments incomplets, accessible via la page Tâches assignées. Les autres utilisateurs ne peuvent pas accéder à ces éléments tant qu'ils n'ont pas été déplacés vers un autre recueil. Les éléments situés dans le dossier Eléments incomplets peuvent également être supprimés.

 

Note :
La case à cocher Appliquer les informations requises aux éléments contenus dans les sous-conteneurs vous permet de mettre à jour les éléments incomplets existant dans les sous-dossiers du recueil parent. Cette option ne vous permet cependant pas de supprimer des éléments des sous-dossiers. Si vous sélectionnez cette case et tentez de supprimer tous les éléments sélectionnés, seuls les éléments incomplets dans le recueil parent seront supprimés.

Cette option ne s'applique qu'aux éléments ajoutés à l'aide d'un glisser-déposer. Si un recueil contient une catégorie à laquelle sont associés des attributs obligatoires, et que vous ajoutez un élément à l'aide du menu Ajouter un élément, vous devez ajouter les informations obligatoires en même temps que l'élément. Pour plus d'informations sur l'ajout d'informations aux éléments incomplets, reportez-vous à la rubrique Pour compléter les informations obligatoires des éléments glissés-déposés.

 

Votre administrateur peut définir des notifications pour les éléments glissés-déposés contenant des informations d'attribut incomplètes. La notification vous informant que des éléments sont incomplets est envoyée par e-mail ; la fréquence de ces notifications dépend du programme défini par votre administrateur. Votre administrateur peut également vous accorder une durée définie pour vous permettre d'ajouter des informations obligatoires aux éléments incomplets avant l'envoi d'un e-mail. L'e-mail de notification des éléments incomplets glissés et déposés inclut un lien vers votre page Tâches assignées, où vous pouvez accéder aux recueils contenant des éléments dont les informations d'attribut sont incomplètes. Pour plus d'informations sur la fonction de glisser-déposer des éléments, reportez-vous à la rubrique Utilisation des éléments généraux.

Versions des catégories

Les catégories sont des éléments multi-version, ce qui signifie que Content Server peut gérer plusieurs versions d'une catégorie. Lorsque vous modifiez une catégorie et que vous enregistrez ces modifications, Content Server crée une nouvelle version de la catégorie. Les éléments existants continuent d'utiliser les versions précédentes, tandis que les nouveaux éléments utilisent la version la plus récente. Cela vous permet de modifier les attributs d'une catégorie à n'importe quel moment sans rendre caduques les données d'attribut des éléments existants.

Une nouvelle version de catégorie n'affectant pas automatiquement les éléments existants, vous devez mettre à niveau manuellement l'affectation de catégories d'un élément pour appliquer la version la plus récente de la catégorie de l'élément. Lorsqu'une catégorie est modifiée, l'icône Pouvant être mis à niveau  s'affiche dans l'onglet Catégories de la page Propriétés de chaque élément associé à cette catégorie. Cela indique que vous pouvez mettre à niveau l'affectation de la catégorie de l'élément.

Héritage d'une catégorie

Par défaut, lorsqu'une catégorie est appliquée à un recueil, tous les éléments que vous ajoutez au recueil héritent de la catégorie du recueil. Si vous possédez les autorisations appropriées, vous pouvez désactiver l'héritage des catégories pour un recueil spécifique. Une fois l'élément ajouté, vous pouvez modifier ou supprimer les catégories héritées de l'onglet Catégories de la page Propriétés de l'élément. Lorsque vous modifiez les catégories d'un recueil, vous pouvez appliquer la nouvelle version de la catégorie à tous les sous-éléments de ce recueil.
