Lorsque vous utilisez la fonction de recherche Content Server, vous pouvez inclure un ou plusieurs attributs dans vos critères de recherche. Vous pouvez même rechercher des attributs ayant une valeur spécifique. Les attributs personnalisés permettent également de créer des relations entre les éléments. Les attributs sont indexés et stockés dans la base de données Content Server. Cela signifie que vous pouvez exécuter un rapport LiveReport pour extraire des informations basées sur les attributs.

Types d'attributs

Vous devez ajouter au moins un attribut à chaque catégorie, mais pouvez en ajouter autant que nécessaire. Chaque attribut peut contenir plusieurs valeurs sur plusieurs lignes. Vous devez déverrouiller le nombre de lignes pour autoriser les utilisateurs à dépasser le nombre de lignes par défaut. Pour permettre aux utilisateurs d'utiliser un attribut lors d'une recherche, vous devez activer le paramètre Afficher dans la recherche.

La plupart des attributs peuvent être rendus obligatoires. Si vous appliquez une catégorie à un dossier et que cette catégorie comprend des attributs obligatoires, chaque fois qu'un utilisateur ajoute un élément à ce dossier, il doit indiquer des valeurs pour ces attributs.

Le tableau suivant décrit les types d'attributs personnalisés disponibles dans Content Server.

Type d'attribut Description
Fenêtre contextuelle Ajoute un menu contenant un ensemble de valeurs prédéfinies, telles qu'une liste déroulante : Couleur qui contient les valeurs Rouge, Bleu et Jaune. Cet attribut peut contenir une combinaison, de lettres de chiffres et de symboles. Le nombre maximal de caractères autorisé pour chaque valeur est 254.
Champ de texte Ajoute un champ pouvant contenir du texte alphanumérique, tel qu'un champ Nom, permettant de saisir le nom du client. Cet attribut peut contenir une combinaison, de lettres de chiffres et de symboles. Le nombre par défaut est 32 et la valeur maximale est 254.
Texte : Multiligne. Ajoute un champ pouvant contenir une grande quantité de texte alphanumérique, tel qu'un champ Commentaires permettant de saisir une explication ou une description. Le nombre par défaut (longueur d'affichage) est 32 et la valeur maximale est 254. Si vous dépassez le nombre par défaut, un champ de texte multiligne défilera automatiquement vers la droite jusqu'à atteindre le nombre maximum de caractères.
Case à cocher Ajoute une option permettant d'activer et de désactiver une valeur. Cet attribut ne peut pas être obligatoire.
Utilisateur : Champ Ajoute une option permettant de rechercher et de sélectionner un utilisateur Content Server. Pour plus d'informations sur la recherche de groupes, reportez-vous à la rubrique Recherche d`utilisateurs et de groupes.
Champ Date Ajoute un menu permettant de saisir une date. Vous pouvez inclure un champ heure.
Date : Liste déroulante Ajoute une option qui permet de choisir parmi un ensemble de dates prédéfinies.
Champ : Entier Ajoute un champ permettant de saisir des nombres entiers. Les valeurs peuvent être positives ou négatives, mais vous ne pouvez pas saisir de caractères spéciaux (tels que les symboles monétaires et les décimales).
Champ entier : Liste déroulante Ajoute un champ ou une liste déroulante permettant de choisir des nombres entiers prédéfinis par le créateur de la catégorie. Les valeurs peuvent être positives ou négatives, mais vous ne pouvez pas saisir de caractères spéciaux (tels que les symboles monétaires et les décimales).
Ensemble Type d'attribut particulier d'une catégorie contenant au moins deux attributs très liés. Un ensemble est une catégorie au sein d'une autre catégorie. Par exemple, vous voudrez sans doute connaître la date à laquelle un utilisateur donné a révisé un document (deux attributs : Date et Réviseur), ou le code d'un produit dont se plaint un client (trois attributs : Numéro de code, Nom du produit et Nom du client). Un ensemble ne peut pas être obligatoire. Pour rendre obligatoires les valeurs d'un ensemble, définissez chaque attribut comme obligatoire.
