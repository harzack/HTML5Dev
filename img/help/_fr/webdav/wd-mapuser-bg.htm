Lorsque vous tentez d'accéder à une ressource WebDAV, le système peut vous demander de vous authentifier à l'aide d'un nom d'utilisateur et d'un mot de passe. WebDAV prend notamment en charge les mécanismes d'identification Authentification Basique/Content Server et Autres mécanismes d'authentification.

 

Note :
Pour plus d'informations sur le mécanisme d'authentification utilisé par la ressource WebDAV à laquelle vous souhaitez accéder, consultez votre administrateur.

 

Authentification Basique / Content Server

Lorsque vous accédez à une ressource WebDAV, une boîte de dialogue vous demande de saisir un nom d'utilisateur et un mot de passe. Dans le cadre de l'authentification Basique/Content Server, vous saisissez le même nom d'utilisateur et le même mot de passe que ceux utilisés dans Content Server.

Autres mécanismes d'authentification

Lorsque vous accédez à une ressource WebDAV, une boîte de dialogue vous demande de saisir un nom d'utilisateur et un mot de passe. Saisissez un nom principal comme nom d'utilisateur, par exemple moi@LEDOMAINE.NET. Dans cet exemple, vous taperiez le mot de passe correspondant au nom « moi » d'ouverture de session du domaine « LEDOMAINE.NET » comme mot de passe.

 

Note :
Pour pouvoir vous connecter à la ressource WebDAV, vous devez configurer votre navigateur Internet Explorer de la façon appropriée. Pour plus d'informations sur la configuration d'Internet Explorer, consultez le Guide d'installation et d'administration de WebDAV.

 

Mappage des noms principaux aux utilisateurs Content Server

Lorsque vous vous connectez à WebDAV avec un nom principal authentifié correctement, WebDAV le met en correspondance avec un utilisateur Content Server. Votre administrateur peut configurer WebDAV de manière à utiliser l'une des deux méthodes de mise en correspondance. Il s'agit des méthodes de mappage Nom identique et Géré par l'utilisateur.

Nom identique

Avec la méthode Nom identique, le nom d'utilisateur Content Server est identique au nom d'ouverture de session de domaine du Nom principal. Par exemple, le nom principal « moi@LEDOMAINE.NET » est mappé à l'utilisateur Content Server appelé « moi ».

Géré par l'utilisateur

Avec la méthode de mappage Géré par l'utilisateur, chaque utilisateur Content Server gère une liste de noms principaux mappés avec lui.

Pour déterminer si le mappage Géré par l'utilisateur est configuré dans WebDAV :

  1. Dans Content Server, cliquez sur Outils dans la barre de menu Global, puis sélectionnez Paramètres.

  2. Si l'onglet Mappages d'utilisateurs WebDAV est visible, cela signifie que le mappage Géré par l'utilisateur est configuré dans WebDAV.

Pour afficher et modifier les noms principaux qui vous correspondent :

  1. Dans Content Server, cliquez sur Outils dans la barre de menu Global, puis sélectionnez Paramètres.

  2. Sélectionnez l'onglet Mappages d'utilisateurs WebDAV. La page affiche tous les noms principaux qui vous correspondent.

  3. Cliquez sur Modifier les mappages.

  4. La page Supprimer les mappages s'affiche, permettant de modifier les mappages d'utilisateurs.

Pour afficher et modifier les noms principaux mappés à un utilisateur :

 

Note :
Cette fonctionnalité est disponible uniquement pour les utilisateurs disposant des privilèges administrateur.

 

  1. Dans Content Server, cliquez sur Entreprise dans la barre de menu Global, puis sélectionnez Utilisateurs et groupes.

  2. Sélectionnez un utilisateur.

  3. Cliquez sur Modifier pour l'utilisateur, puis sélectionnez l'onglet Général.

  4. La page affiche tous les noms principaux mappés à l'utilisateur.

  5. Cliquez sur Modifier les mappages.

  6. La page Supprimer les mappages s'affiche, permettant de modifier les mappages d'utilisateurs.
