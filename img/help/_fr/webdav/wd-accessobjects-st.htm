Les objets auxquels l'utilisateur peut accéder avec WebDAV sont appelés des ressources. Des URL sont attribuées aux ressources, ce qui permet d'y accéder comme à des pages Web. Une URL WebDAV pointe vers un serveur Web, un répertoire virtuel et un chemin d'accès dans Content Server. Voici un exemple typique d'URL WebDAV :

http://www.ma societe.com/livelinkdav/entreprise/doc1.doc

où www.masociete.com identifie le serveur Web sur lequel Content Server WebDAV est installé, livelinkdav est le répertoire virtuel qui a été attribué à WebDAV à l'installation et entreprise/doc1.doc est le chemin d'accès à un document Content Server spécifique.
