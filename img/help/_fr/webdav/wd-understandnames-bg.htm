Dans WebDAV, chaque élément Content Server a deux noms. Le nom de l'élément qui apparaît dans l'interface utilisateur Content Server est son libellé d'affichage. Le nom de l'élément qui apparaît dans l'URL WebDAV est son nom d'URL.

Certains logiciels clients WebDAV utilisent les libellés d'affichage au lieu des noms de l'URL dans l'interface utilisateur mais un nom d'URL est toujours utilisé dans l'URL qui permet d'accéder à l'élément. Par défaut, les noms d'URL des éléments sont les mêmes que les libellés d'affichage, aux exceptions suivantes près :

  • Les libellés d'affichage de l'espace de travail Entreprise, du volume des rapports (l'emplacement de tous les rapports LiveReport du système) et le volume des catégories (l'emplacement de toutes les catégories personnalisées du système) sont localisés (traduits) dans les versions non anglaises de Content Server, mais leur nom d'URL reste en anglais : enterprise, reports et categories.

  • Le libellé d'affichage de l'espace de travail Personnel d'un utilisateur est du type ~nom de l'utilisateur Accueil. Il apparaît dans l'interface utilisateur Content Server.

  • Les extensions appropriées pour les ressources document sont ajoutées à leur nom d'URL si elles ne sont pas déjà présentes. Par exemple, le nom d'URL par défaut d'un document Microsoft Word est libellé_d_affichage_du_doc.doc. Certains clients WebDAV ont besoin de cette extension pour déterminer le type de ressource.
