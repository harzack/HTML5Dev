Cette fonction est disponible uniquement dans LiveReport Extensions 3.0.1 et versions supérieures, bien que la version 3.0.0 contienne un paramètre URL appelé &SQLdebug qui fournissait une fonction similaire.

Cette fonction permet de vérifier la source SQL avant l'exécution. La source SQL est présentée après la substitution de tous les paramètres et modèles, ce qui est particulièrement utile lorsqu'un rapport LiveReport est élaboré avec des modèles, puisqu'il montre de façon évidente à quoi ressemble la source compilée finale. Cette option peut être appelée de deux façons :

  1. A l'aide de l'option Afficher la source SQL du menu des fonctions.

  2. A l'aide du bouton Enregistrer et afficher la source dans l'écran Edition du rapport LiveReport.

Une fois l'option appelée à l'aide de l'une de ces deux méthodes, le rapport LiveReport est traité exactement de la même façon que s'il était exécuté, mais la requête réelle n'est jamais réellement exécutée. A la place, un écran d'affichage montre des informations de débogage composées de la version et de la sous-version du module, ainsi que de la source SQL qui aurait dû être exécutée.

La sortie affiche la source SQL qui présente la même requête que celle exécutée si le rapport avait été normalement exécuté.

La page de sortie contient un bouton Retour qui renvoie l'utilisateur à l'écran précédent et un bouton Exécuter le rapport qui exécute réellement la requête présentée dans le champ Source SQL.
