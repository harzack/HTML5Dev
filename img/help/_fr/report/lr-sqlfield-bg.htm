Cette section explique les champs disponibles dans le modèle SQL. Pour des exemples d'utilisation de ces champs, voir Exemples de modèles.

Tableau 2-1 : Champs du modèle SQL

Nom du champ
Description
#
Numéro du modèle. Ce champ montre la référence utilisée pour insérer un modèle dans la requête SQL principale. Le symbole tilde (~) est utilisé avec le numéro de modèle pour indiquer l'emplacement où le modèle doit être inséré. Par exemple, « ~1 » indique l'emplacement où le modèle 1 doit être inséré, à condition que les diverses conditions soient remplies. Notez que si la requête SQL ne contient aucune référence de modèle, le modèle est inséré à la fin de la requête SQL.
Auto-Where
Sélectionnez cette case à cocher pour ajouter automatiquement le mot de filtre correct, à savoir WHERE, AND ou OR, au début d'un modèle. Cette option est utile en l'absence de clause WHERE dans la requête SQL principale et lorsque l'ordre d'insertion des modèles n'est pas connu. Cette option utilise toujours WHERE pour le premier modèle ajouté, puis AND ou OR, en fonction des mots qui figurent dans la source du modèle d'origine, pour chaque modèle suivant. Si la source du modèle ne contient aucun de ces mots, la valeur par défaut sera WHERE ou AND. La façon la plus simple d'utiliser cette fonction consiste à placer OR ou AND comme premier mot dans la source du modèle, puis la fonction remplace ce mot par WHERE si le modèle est le premier à être inséré dans la requête SQL.
Pour en savoir plus, voir Filtrage dynamique à l'aide de la fonction Auto-Where. Pour des exemples d'utilisation de l'option Auto-Where, voir Réutilisation d'un modèle avec une entrée de variable et Réutilisation d'un modèle avec plusieurs entrées de variables.
Source SQL
Le contenu du modèle potentiellement ajouté à la requête SQL est défini dans ce champ. Cette source peut inclure des paramètres normaux sous la forme : %1, %2, ainsi que des marqueurs sous la forme #1, #2 qui sont utilisés avec le paramètre URL, la condition Inclure IF, décrit dans la ligne « Inclure IF » de ce tableau et dans Inclure IF.
Auto Comma
Sélectionnez cette case à cocher pour activer une fonction qui garantit que les variables séparées par une virgule présentent le bon nombre de virgules après la résolution des variables. Exemples de variables séparées par une virgule : %1, %2 et %3. Par exemple, si %2 aboutit à une chaîne vide, la liste est automatiquement convertie pour supprimer les virgules inutiles. Ainsi, 12,,24 est converti en 12,24. Notez que la liste d'origine doit être correcte d'un point de vue syntaxique.
Pour en savoir plus, voir Utilisation de la fonction Auto Comma. Pour des exemples d'utilisation de l'option Auto Comma, voir Inclusion d'un modèle si l'un des paramètres n'est pas vide et Affichage du comportement pour la condition : Certaines entrées ne sont pas définies sur l'indicateur.
Inclure IF
Les conditions d'inclusion d'un modèle sont sélectionnées à l'aide de cette option. Dans les descriptions ci-dessous, le terme « Entrée utilisateur » désigne les paramètres transmis au rapport LiveReport sous la forme &inputlabel1=somedata. La plupart des conditions du modèle dépendent du contenu d'une ou de plusieurs de ces entrées utilisateur et ces dernières peuvent être utilisées librement dans la source du modèle.
Pour plus d'informations sur les conditions disponibles pour le champ Inclure IF, voir Inclure IF.

Inclure IF

Les options disponibles pour le champ Inclure IF sont :

  • Obligatoire (aucune condition) : le modèle est toujours inséré lorsque cette option est sélectionnée. Cette option permet de décomposer le code SQL en plus petits morceaux.

  • Paramètre URL : cette option indique que ce modèle va être inséré si un paramètre nommé se trouve dans l'URL. Lorsque cette option est sélectionnée, un champ de texte Nom du paramètre apparaît. Il permet de préciser le nom du paramètre URL qui entraîne l'insertion du modèle. Ce paramètre dans l'URL permet également d'indiquer les entrées utilisateur qui seront insérées dans le modèle lorsque des marqueurs sous la forme #1 ont été utilisés dans la source du modèle. Ces marqueurs indiquent l'emplacement dans la source où chaque entrée utilisateur sera insérée. Par exemple, un modèle peut être activé (inclus) à l'aide d'un paramètre URL sous la forme : &addtemplate=U2,U3. Cela indique une liste d'entrées utilisateur que le modèle peut employer. Chaque élément de la liste représente une entrée utilisateur. Par exemple, U2 = USERINPUT 2. L'entrée utilisateur indiquée par le premier élément de la liste est insérée à la place du marqueur 1 (#1) donc dans cet exemple, #1 est remplacé par la valeur &inputlabel2. Le second élément identifie U3, de sorte que la valeur &inputlabel3 remplace le marqueur 2 (#2). Plusieurs exemples de cette abstraction sont présentés plus loin dans ce document. Notez que le paramètre URL doit indiquer au moins autant d'entrées utilisateur que dans la source du modèle pour que le modèle soit inclus.

    Voir Utilisation d'un paramètre URL pour contrôler l'insertion du modèle pour obtenir des exemples.

  • Aucune entrée définie sur l'indicateur : cette option indique que l'inclusion du modèle dépend des paramètres des entrées utilisateur utilisées dans la source du modèle sous la forme d'un paramètre normal. Par exemple, %1. Lorsque cette option est sélectionnée, un champ de texte Valeur d'indicateur apparaît. Il indique la valeur utilisée comme indicateur dans les entrées utilisateur transmises. La valeur la plus couramment utilisée est une chaîne vide, mais d'autres valeurs sont possibles, notamment aucun et tous. Cette option indique que le modèle sera inclus uniquement si aucune entrée utilisateur n'a été définie sur l'indicateur spécifié. Par exemple, si un modèle utilise deux paramètres d'entrée et si l'un d'eux a été défini sur une chaîne vide, alors le modèle sera exclu.

    Voir Utilisation des valeurs d'entrée utilisateur pour contrôler l'insertion du modèle pour obtenir des exemples.

  • Certaines entrées ne sont pas définies sur l'indicateur : cette option ressemble beaucoup à la précédente ; toutefois, lorsque celle-ci est sélectionnée, le modèle est inclus sauf si toutes les valeurs d'entrée utilisées ont été définies sur l'indicateur spécifié. Par exemple, si le champ Valeur d'indicateur est vide (considéré comme une chaîne vide) et si deux entrées utilisateur sont utilisées dans le modèle, alors le modèle est inclus sauf si les deux entrées sont définies sur une chaîne vide.

    Voir Utilisation des valeurs d'entrée utilisateur pour contrôler l'insertion du modèle pour obtenir des exemples.

  • Condition de modèle vraie : avec cette option, le modèle est inclus si un autre modèle est jugé vrai ou non. L'option Modèle n° permet de spécifier la condition du modèle précédent à utiliser. Par exemple, si le numéro de modèle est défini sur 2, le modèle avec cette option activée sera inclus uniquement si le modèle 2 est jugé vrai.

    Cette condition est utile lorsque des morceaux de code SQL doivent être insérés dans différentes parties de la source SQL principale, en fonction de la même condition. L'un des morceaux SQL peut avoir les conditions réelles définies pour l'inclusion, tandis que les autres peuvent simplement utiliser l'option Condition de modèle vraie pour faire référence à l'autre modèle.

    Pour un exemple de cette condition, voir Ajout de deux modèles en fonction d'une même condition.

  • Condition de modèle fausse : avec cette option, le modèle est inclus si un autre modèle est jugé faux ou non. L'option Modèle n° permet de spécifier la condition du modèle précédent à utiliser. Par exemple, si le numéro de modèle est défini sur 2, le modèle sera inclus uniquement si le modèle 2 est jugé faux.

    Cette option est utile lorsque vous possédez deux modèles mutuellement exclusifs. Cette condition évite de calculer l'ensemble de conditions approprié pour inclure uniquement un modèle lorsque l'autre est exclu. Pour un exemple de cette condition, voir Inclusion d'un modèle si l'un des paramètres n'est pas vide.
