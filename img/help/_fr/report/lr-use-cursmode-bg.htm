Dans la version LiveReports standard et par défaut avec LiveReport Extensions, les requêtes sont également exécutées à l'aide du mode « Récupération du curseur » qui permet d'extraire une ligne de données à la fois, afin que Content Server puisse limiter le nombre total de résultats renvoyés. Cette limite peut être définie via le champ Limite d'enregistrement à la fois dans la version LiveReports standard et dans LiveReport Extensions. Cette approche permet de protéger la base de données contre les requêtes inappropriées qui renvoient des ensembles de résultats trop volumineux ; toutefois, ce mode est plus lent qu'une requête SQL standard (c'est-à-dire sans le mode de récupération du curseur). Lors de l'élaboration et du test des requêtes, il est recommandé de laisser le mode de récupération du curseur sélectionné et de limiter les jeux de résultats à l'aide de l'option Limite d'enregistrement. Lorsqu'une requête a été corrigée et jugée sûre, ce mode peut être désactivé afin d'améliorer les performances des requêtes. Lorsque cette option est désélectionnée, une alerte est générée pour indiquer que la requête SQL va désormais renvoyer un nombre illimité de résultats, sauf si des contraintes particulières sont ajoutées à la requête SQL. Par exemple : RowNum < 1000 dans Oracle ou TOP 1000 dans MS SQL.
