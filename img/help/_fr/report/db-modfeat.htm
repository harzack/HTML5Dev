By default, LiveReports provide a checkbox to allow database modification from a LiveReport. Anyone who edits a LiveReport can normally enable this feature simply by selecting this checkbox. As of version 3.0.2 of LiveReport Extensions, it is possible to disable this feature with an INI setting as follows:

[reportext]
AllowDBModification=false

There is also a feature called ModificationSQLTerms that allows terms to be defined that would require the Allow Database Modification option. If any of these terms are found in the SQL (or SQL templates) then an error is generated. The default is the equivalent of the following setting being added to the [reportext] section of the opentext.ini file:

ModificationSQLTerms=
UPDATE,UPDATETEXT,WRITETEXT,REMOVE,DROP,CREATE,ALTER,INSERT,COMMIT,EXECUTE,FETCH,REVOKE,ROLLBACK,SAVE,TRUNCATE
