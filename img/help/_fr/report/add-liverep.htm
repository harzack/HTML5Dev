Lorsque vous ajoutez un rapport LiveReport, vous devez indiquer le nombre maximum de résultats à afficher, écrire une instruction SQL pour exécuter la recherche dans la base de données et choisir un format pour le rapport. Pour plus d'informations sur les options de formatage, reportez-vous à la rubrique Formats des rapports LiveReport. D'autres options LiveReport permettent de définir les critères renvoyant des résultats plus précis. Vous pouvez également utiliser les rapports LiveReport pour modifier le contenu d'une base de données, sauf si votre administrateur a activé la sécurité des rapports LiveReport pour empêcher la modification de la base de données à l'aide des rapports LiveReport. Par exemple, vous pouvez créer un rapport LiveReport qui réassigne les tâches non terminées à un utilisateur particulier. Pour obtenir une description détaillée des autres options LiveReport et afficher un exemple de rapport, reportez-vous à la rubrique Exemple de rapport LiveReport. Lorsque vous ajoutez un rapport LiveReport, vous pouvez le décrire, modifier ses catégories et attributs et l'ajouter à un recueil différent. Pour en savoir plus, voir Utilisation des éléments généraux.

L'instruction SQL est l'élément le plus important d'un rapport LiveReport. Content Server fournit les paramètres qui seront utilisés par l'instruction SQL, soit en demandant à l'utilisateur de les fournir lorsqu'il ouvre un rapport, soit en utilisant les valeurs fournies par le système, comme la date actuelle. Le sous-rapport peut utiliser les valeurs qu'il hérite du rapport parent.

 

Note :
Lorsque vous écrivez l'instruction SQL pour un rapport, n'oubliez pas que Content Server n'interprète pas le signe pourcentage (%) lorsqu'il est utilisé dans une clause like. Par conséquent, utilisez dans ce cas deux signes pourcentage (%%).

Par défaut, un rapport LiveReport ne vérifie pas les autorisations des éléments de la base de données. Il affiche tous les enregistrements, même si l'utilisateur exécutant le rapport n'a pas l'autorisation de les voir. Toutefois, vous pouvez restreindre les résultats aux éléments que l'utilisateur a l'autorisation de voir. Par exemple, si vous définissez Paramètre %1 en sélectionnant le type Filtrer les autorisations et que vous ajoutez AND %1 à votre instruction SQL, les résultats de la recherche sont restreints aux éléments que l'utilisateur a l'autorisation de voir.
