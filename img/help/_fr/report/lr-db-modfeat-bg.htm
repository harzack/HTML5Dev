Par défaut, LiveReports contient une case à cocher pour autoriser la modification de la base de données à partir d'un rapport LiveReport. Quiconque modifie un rapport LiveReport peut normalement activer cette fonction en sélectionnant la case à cocher correspondante. L'administrateur peut désactiver cette fonction en modifiant le champ AllowDBModification dans la section [report] du fichier opentext.ini. Pour en savoir plus, voir [report].

Il existe également une fonction appelée ModificationSQLTerms qui permet à l'administrateur de définir des termes qui, le cas échéant, nécessitent l'utilisation de l'option Autoriser les modifications de la base de données. Si l'un de ces termes se trouve dans la requête SQL (ou dans les modèles SQL), une erreur est générée. Pour plus d'informations, voir le champ ModificationSQLTerms dans la section [report] du fichier [report].
