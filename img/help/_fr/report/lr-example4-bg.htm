Utilisation de modèles dans d'autres modèles : Outre la définition de modèles à inclure dans le champ SQL principal, il est également possible d'utiliser des modèles dans d'autres modèles. La syntaxe est la même, par exemple « ~2 » ; toutefois, un modèle ne peut utiliser qu'un modèle présentant un numéro inférieur. Par exemple, le modèle ~2 peut inclure une référence au modèle ~1, mais pas à lui-même ni au modèle ~3, ni à un numéro supérieur à 2.

Utilisation de l'icône Vue du modèle : Chaque modèle comprend une icône spéciale qui permet d'Afficher le comportement du modèle. Cliquez sur cette icône pour ouvrir une fenêtre qui contient une série d'instructions sur le modèle en question. Ces instructions tentent de clarifier le comportement de chaque modèle en fonction de la condition sélectionnée et de la source spécifiée. L'icône Afficher le comportement du modèle se trouve à la fin de chaque ligne du modèle.

Exemple 2-8 : Affichage du comportement pour la condition : Certaines entrées ne sont pas définies sur l'indicateur

Les conditions Certaines entrées ne sont pas définies sur l'indicateur et Aucune entrée définie sur l'indicateur comprennent un tableau de conditions qui permet de comprendre l'effet des paramètres d'entrée utilisateur sur l'inclusion ou l'exclusion du modèle. Dans l'exemple ci-dessous, le tableau montre que si tous les paramètres d'entrée utilisateur sont définis sur une chaîne vide, le modèle est exclu, mais si un paramètre d'entrée utilisateur n'est pas défini sur une chaîne vide, le modèle est inclus.

Dans Configuration du modèle, dans la zone Modèles, définissez ~1 :

  • Ne sélectionnez pas la case Auto-Where.

  • Dans le champ Source SQL, saisissez : « %1,%2,%3 »

  • Sélectionnez la case Auto-Comma.

  • Dans la liste Inclure IF, sélectionnez « Aucune entrée définie sur l'indicateur ».

  • Dans le champ Options, assurez-vous que le champ Valeur d'indicateur est vide.

Si vous cliquez sur l'icône Afficher le comportement du modèle située à la fin de la ligne du modèle ~1, une page contenant les informations suivantes apparaît :

Fenêtre d'explication – La combinaison des champs de cette ligne est interprétée comme suit :

  • Les paramètres : %1,%2,%3 ont été trouvés dans le champ source SQL de ce modèle.

  • Le type de condition est défini sur « Aucune entrée définie sur l'indicateur ». Autrement dit : ces paramètres vont être testés pour déterminer si l'indicateur (Une chaîne vide) a été transmis en tant que valeur.

  • Si aucun des paramètres du champ source SQL n'est défini sur Une chaîne vide, le modèle est inséré dans la requête SQL là où le symbole ~1 se trouve.

  • A l'inverse, si L'UN des paramètres du champ source (%1,%2,%3) est défini sur l'indicateur (Une chaîne vide), le modèle ~1 est exclu de la source SQL.
