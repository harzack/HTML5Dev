In the existing LiveReports module, all parameters passed for substitution in the Report code are named with the prefix: inputlabel. Thus, “Input1” is associated with inputlabel1, “Input2” is associated with inputlabel2 and so on. Starting with LiveReport Extensions Version 3.0.2, it is possible for each Input to define an alternative URL parameter. When each input is added, the URL Parameter field defaults to the appropriate inputlabel<x> parameter but this field allows any string of characters to be defined instead of the default.
