Les paramètres peuvent être transmis aux rapports LiveReport à l'aide des entrées utilisateur. Les entrées sont définies dans l'éditeur de rapport LiveReport en sélectionnant le type d'entrée et en indiquant le texte d'invite correspondant. En outre, il est possible d'éviter l'étape d'invite si toutes les entrées sont indiquées dans l'URL qui exécute le rapport LiveReport. Elles sont définies dans les paramètres URL sous la forme : inputlabel<x> où <x> est un nombre. Par exemple :

http://intranet/Content Server/livelink.exe?func=ll&objId=1234 &objAction=RunReport&viewType=1&inputlabel1=treadstone

Dans cet exemple, la chaîne « treadstone » est transmise à l'entrée utilisateur 1. S'il s'agissait de la seule entrée définie, l'utilisateur ne serait pas invité et le rapport serait immédiatement exécuté.

Les autres options de transmission de paramètres sont :

  1. Possibilité de définir un nouveau nom, autre que inputlabel<x>, pour chaque paramètre. Pour en savoir plus, voir Modification du nom d'un paramètre d'entrée.

  2. Possibilité d'insérer une chaîne de caractères sans guillemets, avec des termes SQL, dans la requête SQL. Pour en savoir plus, voir Insérer une chaîne.

  3. Possibilité d'activer ou de désactiver un mode sécurisé qui restreint les termes prédéfinis transmis dans la requête SQL. Pour en savoir plus, voir Configuration du type d'entrée InsertString.

  4. Un nombre illimité d'entrées peut être demandé ou transmis dans l'URL. Pour en savoir plus, voir Insérer une chaîne.

  5. Paramètres de substitution illimités dans la requête SQL. Pour en savoir plus, voir Création d'entrées ou de paramètres de substitution supplémentaires.

En outre, il existe une fonction Modèle SQL qui permet aux concepteurs de rapports d'inclure ou d'exclure des segments de SQL en fonction de différentes conditions prédéfinies, telles que l'état des entrées sélectionnées ou d'autres paramètres URL. Cette fonction est présentée dans une section distincte de ce document. Pour en savoir plus, voir Modèles SQL.
