Cette fonction est conçue pour les situations dans lesquelles certains utilisateurs doivent pouvoir visualiser la requête du rapport LiveReport sans avoir accès à l'écran d'édition. Dans certains cas, l'accès à l'écran d'édition peut se révéler potentiellement dangereux et incertain. Si certains développeurs ou utilisateurs ont besoin d'accéder au contenu de ces requêtes (pour étudier d'autres exemples, examiner un rapport, etc.), il est possible d'utiliser cette fonction pour leur accorder les autorisations nécessaires pour utiliser cette fonction et non la fonction d'édition.

Figure 2-2 : Afficher la requête

Il existe deux façons de configurer les autorisations de cette fonction. Par défaut, cette fonction est disponible uniquement pour les utilisateurs qui disposent au moins des autorisations Modifier pour le rapport LiveReport. Si l'utilisateur n'a pas été autorisé par l'administrateur à créer des rapports LiveReport dans la page Gérer les privilèges relatifs aux objets et à l'utilisation, il n'a pas accès à l'option Editer, mais peut afficher la requête.

De même, l'administrateur peut modifier ce comportement afin que les utilisateurs disposant des autorisations Voir et Voir le contenu puissent utiliser cette fonction. Pour ce faire, il suffit d'ajouter l'entrée suivante dans la section [ReportExt] du fichier opentext.ini : 

QueryVisibleWithSeeContents=true

Pour toute instance de Content Server avec cette option activée, les utilisateurs disposant des autorisations Voir et Voir le contenu sur un rapport LiveReport pourront utiliser l'option Afficher la requête.
