Dans la version LiveReports standard et par défaut avec LiveReport Extensions, toutes les requêtes SQL s'exécutent dans une transaction de base de données qui verrouille les tables concernées par la requête. Lorsqu'une requête modifie la base de données, il est essentiel qu'elle s'exécute dans une transaction.

 

Note :
OpenText vous déconseille d'utiliser LiveReports pour apporter des modifications à la base de données Content Server.

 

La plupart du temps, les requêtes LiveReport servent uniquement à générer des rapports sur les données, pas à les modifier. Pour les requêtes destinées à lire des données, l'exécution de la requête dans une transaction est déconseillée, car elle verrouille les tables concernées. (La description du concept des transactions n'entre pas dans le cadre de ce document.)

 

Note :
Pour les raisons susmentionnées, certaines fonctions telles qu'Autoriser les modifications de la base de données ne sont pas disponibles une fois le mode Utiliser une transaction désactivé.
