Deux conditions Inclure IF dépendent d'autres modèles. Ces deux conditions sont : Condition de modèle vraie et Condition de modèle fausse. Dans chaque cas, un champ Modèle n° permet de saisir le numéro de chaque condition du modèle à utiliser pour contrôler ce dernier. Pour plus d'informations sur ces conditions, voir la ligne « Inclure IF » dans Champs du modèle SQL. Voir également Inclure IF.

Exemple 2-7 : Ajout de deux modèles en fonction d'une même condition

Cet exemple présente deux modèles dont l'un, conditionnel sur &inputlabel1, n'est pas défini sur une chaîne vide, et l'autre dépend du premier modèle. Dans cet exemple, le développeur a décidé que si le sous-type n'était pas inclus sous la forme d'une clause de filtrage, il n'aurait pas à figurer en tant que colonne. Si &inputlabel1 est une chaîne vide, le modèle Where et la colonne Sélectionner un modèle sont inclus.

  1. Dans Source SQL principale, dans le champ SQL, saisissez :

    select name~2
from sampleTable
~1

  2. Dans Configuration du modèle, dans la zone Modèles, définissez ~1 :

    • Ne sélectionnez pas la case Auto-Where.

    • Dans le champ Source SQL, saisissez : « where subtype = %1 »

    • Ne sélectionnez pas la case Auto-Comma.

    • Dans la liste Inclure IF, sélectionnez « Aucune entrée définie sur l'indicateur ».

    • Dans le champ Options, assurez-vous que le champ Valeur d'indicateur est vide.

  3. Dans Configuration du modèle, dans la zone Modèles, définissez ~2 :

    • Ne sélectionnez pas la case Auto-Where.

    • Dans le champ Source SQL, saisissez : « , subtype »

    • Ne sélectionnez pas la case Auto-Comma.

    • Dans la liste Inclure IF, sélectionnez « Condition de modèle vraie ».

    • Dans le champ Options, dans le champ Modèle n°, saisissez « 1 ».

  4. Dans la zone Entrées, ajoutez une entrée générique. Par exemple, incluez une entrée de type STRING avec une Invite appelée « Subtype ».

Si vous accédez à l'URL suivante :

...&inputLabel1=grommet&nextURL=...

Le code SQL qui en résulte contient, dans le champ Source SQL, « select name , subtype from sampleTable where subtype ='grommet' ».
