Une fois le travail sur une étape terminé, vous pouvez envoyer l'étape pour vérification à un utilisateur ou à un groupe, si activé. Vous pouvez également réorienter l'étape pour révision vers un sous-workflow, si activé. Par exemple, si vous avez besoin d'un accord éditorial, vous pouvez envoyer le rapport à un autre workflow qui automatise le processus de révision éditoriale.

L'envoi d'une étape du workflow pour révision la supprime de votre page Mes tâches assignées et la fait apparaître sur la page Mes tâches assignées de l'utilisateur ou du groupe auquel elle est assignée. A l'issue de la révision, l'étape du workflow réapparaît sur votre page Mes tâches assignées pour que vous puissiez faire suivre le traitement du workflow vers l'étape suivante.

Si le lien Envoyer pour révision apparaît sur la page d'une étape du workflow, vous êtes autorisé à envoyer cette tâche à un autre utilisateur, groupe ou sous-workflow pour révision.

 

Notes :

  • Si vous faites suivre une étape de workflow qui comporte des pièces jointes, vous devez décider d'une manière de transmettre les pièces jointes aux réviseurs. Tous les intervenants au workflow doivent également disposer des autorisations d'accès aux pièces jointes. Pour plus d'informations sur la transmission de pièces jointes, reportez-vous à la rubrique Utilisation du lot de travail.

  • Pour autoriser la révision d'une étape, l'auteur du workflow doit vérifier que la case à cocher Envoyer pour révision sur la page Autorisations de l'étape est sélectionnée.
