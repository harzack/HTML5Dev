Lorsque vous êtes prêt à travailler sur un projet ou sur un processus commercial défini par un workflow, vous pouvez le lancer. Lorsque vous lancez un workflow, Content Server crée une copie de travail (une instance) du schéma de workflow et commence à l'exécuter. Le travail effectué lors de l'exécution du processus commercial est stocké dans une instance de workflow. Chaque fois que le même projet ou le même processus commercial doit être exécuté, lancez une nouvelle instance du Workflow Map.

Vous pouvez lancer un workflow dans Content Server ou en envoyant un e-mail si l'auteur du schéma de workflow a configuré celui-ci pour qu'il puisse être activé par e-mail. Pour plus d'informations sur le lancement d'un workflow dans Content Server, voir Pour lancer un workflow. Pour plus d'informations sur le lancement d'un workflow par e-mail, voir Lancement d'un workflow par e-mail.

Selon la façon dont l'auteur du schéma de workflow a configuré le workflow, vous pouvez être amené à effectuer les opérations suivantes :

  • Ajouter ou modifier le lot de travail composé des instructions, des tâches, des fichiers et des autres informations utilisées dans le workflow. Pour plus d'informations sur le lot de travail, reportez-vous à la rubrique Utilisation du lot de travail.

  • Indiquer un nouveau nom pour le workflow dans le champ Lancer sous.

  • Spécifier une date d'échéance pour le workflow dans le champ Date d'échéance.

  • Indiquer votre nom d'utilisateur et votre mot de passe Content Server.

Après son lancement, le workflow actif apparaît dans votre page Mes workflows ou dans un élément de statut de workflow Content Server, si présent. Les responsables du workflow peuvent également suivre sa progression et le gérer. Pour plus d'informations sur le suivi et la gestion des workflows, voir Gestion des workflows. Pour plus d'informations sur l'élément de statut de workflow Content Server, voir Utilisation d'un élément de statut de workflow Content Server.

 

Note :
Vous devez disposer de l'autorisation Voir le contenu sur un workflow pour pouvoir le lancer. Pour de plus amples informations sur les autorisations, reportez-vous à la section Autorisations.

 

Lancement de workflows avec signature

Si le workflow que vous souhaitez lancer est un workflow avec signature vous permettant de faire circuler un document Content Server à approuver et signer ou à refuser, il se peut que vous deviez réaliser des tâches préalables au lancement de ce workflow. Pour plus d'informations sur le lancement d'un workflow avec signature, reportez-vous à la rubrique Lancement de workflows avec signature.

Lancement d'un workflow par e-mail

Si l'auteur d'un workflow a configuré un schéma de workflow pour qu'il soit activé par e-mail, vous pouvez lancer le workflow en rédigeant et en envoyant un e-mail au schéma de workflow. Un schéma de workflow activé par e-mail utilise l'adresse nom_du_schéma_de_workflow@serveur_eLink, où nom_du_schéma_de_workflow est le nom du schéma de workflow spécifié par l'auteur du schéma de workflow et serveur_eLink le nom du serveur eLink. Pour plus d'informations sur l'éventuelle activation par e-mail d'un schéma de workflow, consultez l'auteur du schéma de workflow ou l'administrateur.

Rédaction et envoi d'un e-mail de lancement

Lorsque vous rédigez, puis envoyez un e-mail pour lancer un workflow, Content Server applique les données spécifiées pour créer le workflow. Le tableau suivant explique de quelle manière les données de l'e-mail de lancement sont utilisées dans le workflow :

Informations d'e-mail Devient…
Objet Titre du workflow
Pièce jointe Une pièce jointe au workflow
Texte du corps du message Commentaires sur l'étape Départ du workflow

 

Notes :

  • Si vous envoyez une pièce jointe à un workflow qui ne les accepte pas, si vous tentez de joindre un fichier dont le type a été bloqué ou si vous n'êtes pas autorisé à ajouter des pièces jointes au dossier Pièces jointes du workflow, ce document n'est pas ajouté au workflow. Content Server envoie un message de statut expliquant que la pièce jointe a été refusée.

  • Les valeurs des champs de formulaire et d'attributs de workflow ne peuvent pas être spécifiées par e-mail.

 

Par exemple, l'e-mail suivant lance un workflow activé par e-mail nommé « Approbation_note_de_frais » :

To: Expense_Report_Approval@elink.myServer.com 
Subject: June Expense Report Approval
Attached: c:\MyExpenses\June_report.xls

From: "John Smith" jsmith@myCompany.com

These are my expenses for the month of June. 
If you have any questions, please let me know. Thanks!

Suite à l'envoi d'un e-mail de lancement, Content Server crée le workflow et le nomme « Approbation note de frais de juin ». Ensuite, il copie la feuille de calcul jointe dans les Pièces jointes du workflow, puis il copie le texte de l'e-mail dans la section Commentaires de l'étape Départ du workflow. Enfin, il fait passer le traitement du workflow à l'étape suivant l'étape Départ.

Si le lancement est réussi, Content Server envoie un e-mail de confirmation. Si le lancement échoue, Content Server envoie un e-mail décrivant l'erreur rencontrée et inclut un lien vers une page permettant de lancer le workflow en ligne. Une fois le workflow lancé, vous pouvez interagir avec lui comme vous le feriez avec un workflow lancé dans Content Server. Par exemple, dans le cadre de l'exemple ci-dessus, Jean Dupont peut suivre le workflow sur la page Mes workflows ou dans un élément de statut de workflow Content Server, s'il est disponible ; les utilisateurs assignés à une tâche de l'étape en cours peuvent visualiser cette tâche sur leur page Mes tâches assignées.

Lorsque le traitement du workflow est terminé, le système envoie un e-mail Achevé à l'initiateur et aux gestionnaires du workflow.

 

Important :
L'administrateur doit configurer les paramètres de workflow eLink globaux ou vous devez configurer vos paramètres de workflow eLink locaux afin de recevoir des e-mails concernant les événements des workflows activés par e-mail.
