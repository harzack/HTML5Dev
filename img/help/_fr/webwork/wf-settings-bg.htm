Vous pouvez spécifier des paramètres contrôlant l'affichage et le traitement des workflows et de leurs tâches assignées (les tâches du workflow qui vous sont assignées). Vous pouvez décider que Content Server affiche automatiquement les tâches assignées suivant celle qui vient d'être exécutée, au lieu de récupérer chaque tâche individuellement ; définir l'affichage d'une tâche assignée de workflow lorsque vous y accédez et que vous la lancez ; déterminer les workflows et les tâches assignées apparaissant sur votre page Mes workflows. Le générateur d'expressions constitue une méthode supplémentaire pour choisir les workflow et les tâches assignées à afficher. Pour plus d'informations sur l'utilisation du générateur d'expressions, reportez-vous à la rubrique Utilisation du générateur d'expressions.

Vous pouvez également assigner un mandataire du workflow pour exécuter les tâches du workflow qui vous sont assignées lorsque vous êtes absent ou lorsque vous n'êtes pas disponible. Lorsque vous avez assigné un mandataire, vos tâches assignées apparaissent sur votre page Tâches assignées ainsi que sur celle de votre mandataire.

 

Notes :

  • Si vous lancez un workflow alors que vous avez sélectionné un mandataire, le workflow apparaît sur votre page Mes workflows ainsi que sur celle de votre mandataire.

  • Selon les restrictions définies par votre administrateur, le cas échéant, vous ne pouvez pas assigner de mandataire de workflow ou indiquer le nombre maximum de workflows à afficher par page.

 

Ordre de classement

Vous pouvez choisir l'ordre dans lequel les workflows (mais pas les tâches) s'affichent sur les pages Mes workflows ou Statut du workflow. Pour ce faire, sélectionnez un ordre de classement. Le tableau ci-dessous décrit les modes de classement. Ils organisent les workflows selon une hiérarchie basée sur plusieurs valeurs. Par exemple, le mode de classement Nom - Hiérarchique trie les workflows en fonction de leur statut : les workflows arrêtés sont toujours affichés après les workflows actifs même si le nom d'un workflow arrêté devrait être classé avant celui d'un workflow actif si le tri était purement alphabétique.
