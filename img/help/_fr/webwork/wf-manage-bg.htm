Un responsable de workflow peut contrôler une instance en cours d'exécution d'un workflow pour s'assurer de son bon déroulement. Un responsable de workflow disposant des autorisations appropriées peut effectuer les actions suivantes :

  • Arrêter, pour mettre définitivement fin au workflow.

  • Interrompre, pour arrêter momentanément l'exécution du workflow.

  • Reprendre, pour reprendre un workflow interrompu.

  • Archiver, pour déplacer les workflows arrêtés ou achevés dans la liste des workflows archivés.

  • Supprimer, pour supprimer les workflows arrêtés ou achevés.

Un responsable de workflow peut également modifier une instance en cours d'exécution si nécessaire. Pour plus d'informations sur la modification et l'exécution du workflow, reportez-vous à la rubrique Modification d'un workflow en cours d'exécution.

Avant d'effectuer une opération, le responsable du workflow doit afficher le workflow. Pour plus d'informations sur l'affichage des workflows, reportez-vous à la rubrique Gestion des workflows et des tâches assignées.

A l'issue de l'exécution d'une instance de workflow, cette dernière prend le statut Achevé(e), à moins que ses propriétés n'incluent une action d'achèvement automatique. Dans ce dernier cas, l'instance du workflow prend le statut Archivé ou le workflow est supprimé de Content Server.

Par défaut, le responsable principal d'un workflow est son initiateur. Cependant, l'auteur du Workflow Map peut choisir d'assigner la gestion complète ou partielle du workflow à un autre utilisateur ou groupe. Le responsable principal dispose de toutes les autorisations de gestion pour le workflow.

 

Notes :

  • Lorsque vous supprimez un workflow, ses pièces jointes sont également supprimées de Content Server. Si vous souhaitez enregistrer les pièces jointes, enregistrez-les à un autre endroit avant de supprimer l'instance du workflow.

  • Si vous activez des notifications pour des événements de workflow, tous les gestionnaires de workflow sont notifiés d'un retard de workflow, mais seul le gestionnaire principal reçoit une notification par e-mail lorsqu'un workflow est terminé.
