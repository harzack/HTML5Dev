Le lot de travail est composé d'instructions, de tâches, de fichiers et d'autres informations utilisées dans un workflow. Le lot de travail comprend les éléments suivants :

  • Pièces jointes

  • Attributs

  • Commentaires

  • Rôles

  • Formulaires, si le module optionnel Forms est installé

Lorsque vous lancez un workflow ou que vous exécutez des tâches qui vous ont été assignées, vous pouvez afficher les informations du lot de travail, les modifier et en ajouter. Selon la configuration du workflow et selon les autorisations dont vous disposez, il est possible que vous ne puissiez pas afficher, ajouter ou modifier tous les éléments du lot de travail.

 

Note :
Pour utiliser ces éléments dans un workflow, chaque élément doit être activé dans le lot de travail.

 

Utilisation des pièces jointes

Une pièce jointe de workflow est un élément Content Server ou un fichier qui contient des informations relatives au workflow. Lors de l'exécution de tâches, vous pouvez avoir pour instructions de consulter, de modifier ou d'ajouter une pièce jointe à un workflow. Par exemple, une pièce jointe peut être un document contenant des instructions concernant la tâche ou une feuille de calcul que vous devez remplir. La section Pièces jointes de l'étape permet d'accéder aux pièces jointes.

Le dossier Pièces jointes est un dossier Content Server particulier qui permet de stocker les pièces jointes à une instance de workflow. Dans le dossier Pièces jointes, vous pouvez ajouter ou copier des éléments Content Server et créer des sous-dossiers de la même manière que dans un dossier classique ou que dans un élément de recueil. Pour plus d'informations sur l'utilisation des éléments Content Server, reportez-vous à la rubrique Aide du Guide de prise en main.

Transmission de pièces jointes

Lorsque vous déléguez une étape ou que vous envoyez une étape de workflow à un sous-workflow pour vérification, vous choisissez de transmettre ou non les pièces jointes du workflow au sous-workflow. La transmission d'une pièce jointe signifie que celle-ci sera disponible à la fois dans le sous-workflow et dans le workflow principal. Si les pièces jointes sont ajoutées au sous-workflow au cours des étapes du workflow, elles sont également disponibles dans le workflow principal. Pour plus d'informations sur la délégation des étapes ou l'envoi des étapes à la révision, reportez-vous aux rubriques Délégation d'une étape et Envoi d'une étape pour révision.

Utilisation des attributs

Les attributs du workflow stockent les informations et en effectuent le suivi tout au long du processus de travail. Lors de l'exécution d'une tâche, vous pouvez avoir pour instructions de consulter, de modifier ou d'ajouter un attribut du workflow. La section Attributs de l'étape permet d'accéder aux attributs et à leur valeur. Si aucun attribut n'est défini pour le lot de travail ou si vous n'avez pas l'autorisation d'afficher les attributs, la section Attributs n'apparaît pas sur la page de l'étape.

 

Note :
Les attributs requis sont indiqués par l'icône Obligatoire .

 

Utilisation des commentaires

Les commentaires de workflow permettent aux intervenants du workflow d'enregistrer leurs impressions sur les tâches d'une étape ou sur le workflow. Lors de l'exécution d'une tâche, vous pouvez avoir à ajouter des commentaires. Si l'auteur du schéma de workflow leur en donne l'autorisation, les intervenants peuvent voir les commentaires effectués dans les étapes précédentes par d'autres intervenants.

Si les commentaires entrés par les intervenants précédents sont affichés, ils sont présentés dans l'ordre chronologique inverse (du commentaire le plus récent au moins récent) dans le champ Commentaires précédents. Chaque commentaire est identifié par les informations des champs Nom de l'étape, Exécutant, Date et Commentaires indiquant respectivement l'étape dans laquelle les commentaires ont été effectués, l'auteur Content Server de ces commentaires ainsi que la date de leur entrée.

 

Note :
Les commentaires que vous indiquez peuvent comporter du code HTML de base. Si vous utilisez des caractères spéciaux, vous devrez peut-être utiliser leurs codes HTML pour qu'ils ne soient pas interprétés comme du code HTML. Par exemple, si vous souhaitez utiliser les symboles inférieur à ou supérieur à (< ou >), vous devez saisir les codes HTML de chaque symbole (&lt; ou &gt;).

 

Utilisation des rôles

Lorsque vous lancez un workflow, vous pouvez demander aux intervenants d'exécuter une opération définie par un rôle lors du traitement du workflow. Par exemple, un workflow de révision de document peut comporter des rôles Superviseur et Réviseur.

 

Note :
Les rôles sont disponibles uniquement si l'auteur du workflow définit des rôles dans le Workflow Map.

 

Utilisation des formulaires

Les formulaires vous permettent de saisir des données dans les champs d'un formulaire joint à une étape. Pour plus d'informations sur l'utilisation des formulaires dans les workflows, reportez-vous à la rubrique Utilisation de formulaires dans les schémas de workflow.

Les formulaires ne sont disponibles que si le module optionnel Content Server Forms est installé et ils sont activés dans le schéma de workflow.
