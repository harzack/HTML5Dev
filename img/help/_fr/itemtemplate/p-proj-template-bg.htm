Un modèle de projet est un ensemble d'éléments prédéfinis qui permet de créer facilement un nouveau projet. Si vous disposez des autorisations appropriées, vous pouvez créer un modèle de projet à partir d'un projet spécifique et l'enregistrer dans Content Server ou l'exporter hors de Content Server. D'autres utilisateurs peuvent ensuite ajouter un projet avec les mêmes informations dans l'emplacement de leur choix.

Vous pouvez également effectuer l'une des opérations suivantes :

  • ajouter des versions du modèle ;

  • exporter le modèle ;

  • définir les propriétés de filtrage du modèle ;

  • définir les propriétés des règles de substitution.

Création de modèles de projet et ajout de versions

Les utilisateurs Content Server disposant des autorisations appropriées peuvent créer un modèle de projet à partir d'un projet actuel ou d'un projet enregistré en tant que fichier XML, puis l'enregistrer dans Content Server.

Lorsque vous créez un modèle de projet, tous les éléments contenus dans le projet sont inclus dans le modèle. Toutefois, si des documents contenus dans le projet utilisent la création de versions avancée, la manière dont ils sont copiés dans le modèle dépend des autorisations de l'utilisateur qui crée le modèle. Pour plus d'informations sur les modèles de projet qui contiennent des documents utilisant la création de versions avancée, reportez-vous à la rubrique Fonctionnement de la création de versions avancée pour les fonctions de Content Server.

Lors de la mise à jour d'un modèle ou de l'ajout d'un nouveau modèle, vous pouvez suivre les modifications manuellement en créant une version du modèle. Content Server n'ajoute pas automatiquement une nouvelle version du modèle lorsqu'il est modifié.

Toutes les versions d'un modèle sont répertoriées dans l'onglet Versions de la page Propriétés Modèle de projet. Vous pouvez conserver la page actuelle en gardant les versions les plus récentes et en éliminant les anciennes. Pour plus d'informations sur l'utilisation des versions et l'ajout d'un projet utilisant une version d'un modèle, reportez-vous aux rubriques Gestion des versions et des générations et Ajout et modification de projets.

 

Note :
Lorsque vous créez un projet à partir d'un modèle de projet ou que vous l'importez, les métadonnées de Records management, Habilitation de sécurité ou Objets physiques du projet sont conservées.

 

Exportation d'un modèle de projet

Si vous disposez des autorisations appropriées, vous pouvez exporter et enregistrer un modèle de projet sous forme de fichier XML sur votre ordinateur. Les éléments, les valeurs de règle de substitution, les informations de filtrage et le reste du contenu du modèle de projet sont conservés dans le fichier XML.

L'exportation d'un projet permet aux autres utilisateurs d'importer le modèle dans un autre système Content Server ou un autre endroit du système Content Server actuel.

Définition des propriétés de filtrage du modèle de projet

Vous pouvez définir les éléments que vous souhaitez inclure ou exclure lors de l'utilisation d'un modèle de projet pour la création d'un projet. Seuls les éléments sélectionnés seront inclus dans le nouveau projet. Afin de vous assurer que le projet contient uniquement les éléments appropriés, vérifiez l'onglet Filtrage de la page Propriétés du modèle avant de créer le projet. Une fois les options de filtrage modifiées, elles ne reviennent pas aux paramètres par défaut.

Le tableau suivant contient les éléments que vous pouvez inclure lors de la création d'un projet à partir d'un modèle.

Elément Inclut l'élément suivant par défaut...
Document composite Dernière version d'un document composite. Vous pouvez également inclure toutes les révisions d'un document composite.
Document Dernière version d'un document. Vous pouvez également inclure toutes les versions d'un document.
Liste de tâches Tous les points de repère, groupes de tâches, tâches et sous-tâches.
Canal Nouvelles d'une chaîne.
Discussion Messages de discussion. Vous pouvez également inclure les réponses aux messages.
Intervenants Coordinateurs et membres d'un projet. Vous pouvez également inclure les invités.

Définition des propriétés des règles de substitution du modèle de projet

Les règles de substitution permettent aux utilisateurs de Content Server de remplacer un mot ou une combinaison de mots par un autre mot ou une autre combinaison pour les éléments inclus dans un modèle de projet. Lorsqu'un projet est créé à partir du modèle, les valeurs substituées s'affichent.

Vous pouvez par exemple créer un projet à partir des informations contenues dans un modèle de projet appelé « Equipe des ventes ». Toutefois, vous souhaitez remplacer toutes les occurrences de « Janvier » par « Février » dans les noms d'éléments. Les règles de substitution facilitent ce processus en vous invitant à saisir des informations dans les champs suivants :

  • Chaîne à remplacer, qui correspond à la valeur actuelle à remplacer. Cette valeur n'est pas affichée dans l'onglet Règle de substitution de la page Propriétés Modèle de projet.

  • Valeur de replacement par défaut, qui correspond à la valeur remplaçant la valeur d'origine. Elle s'affiche dans les projets créés à partir du modèle mis à jour. Cette valeur peut être modifiée dans l'onglet Règle de substitution de la page Propriétés Modèle de projet.

  • Afficher le libellé, qui est le libellé du champ Valeur de remplacement par défaut. Il est affiché dans la section Contenu de la page Ajouter : Projet.

  • Afficher la description, qui est un champ facultatif dans lequel vous pouvez ajouter une description du champ Afficher le libellé. Les utilisateurs peuvent consulter la description dans la section Contenu de la page Ajouter : Projet. Les descriptions sont utiles car le libellé ne suffit pas toujours à identifier la chaîne remplacée.

Pour consulter un exemple d'affichage de ces champs dans un modèle de projet, reportez-vous à la rubrique Exemple de règle de substitution.

Vous pouvez ajouter jusqu'à 15 règles de substitution différentes, ou remplacements de chaînes. Les règles de substitution seront appliquées selon l'ordre dans lequel elles apparaissent dans l'onglet Règle de substitution de la page Propriétés Modèle de projet. Si la valeur du champ Chaîne à remplacer, ou une partie de cette valeur, est comprise dans une valeur Chaîne à remplacer qui s'affiche dans la suite de la liste, la règle de substitution suivante n'est pas appliquée.

 

Note :
Les informations sur la règle de substitution s'affichent dans la section Contenu de la page Propriétés Modèle de projet. Elles s'affichent à la place de la section Options, disponible lorsque vous ajoutez un nouveau projet. Si un modèle ne contient pas d'informations sur la règle de substitution, aucune des sections n'est affichée.
