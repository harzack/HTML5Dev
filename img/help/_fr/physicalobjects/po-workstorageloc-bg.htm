Le module Physical Objects permet de gérer le stockage d'éléments physiques pour les infrastructures gérées et non gérées en créant des localisateurs de stockage. Les localisateurs de stockage sont des marqueurs utilisés pour des sections spécifiques de zones dans une installation de stockage. Par exemple, une zone d'infrastructure de stockage peut comprendre plusieurs rayonnages, le localisateur de stockage ne faisant référence qu'à une seule tablette du rayonnage.

Lorsque vous créez un localisateur de stockage pour des infrastructures gérées, vous pouvez spécifier la fréquence d'accès au localisateur, ainsi que le type et les dimensions du localisateur.

Une fois des localisateurs de stockage créés, vous pouvez modifier leurs paramètres, afficher les éléments physiques qui leur sont assignés, supprimer les localisateurs de stockage auxquels aucun élément physique n'est assigné et recalculer l'espace de stockage disponible. Lorsque vous recalculez l'espace de stockage, vous le recalculez pour tous les localisateurs auxquels sont assignés les codes de zone et d'infrastructure spécifiés. Recalculer l'espace de stockage vous permet de vérifier la quantité réelle d'espace libre disponible. Vous pouvez également vérifier le contenu d'un localisateur en chargeant un document qui contient les données collectées via un scanner distant. Une table affiche les résultats : infrastructure, zone, localisateur, ID unique et statut de la vérification. Si le localisateur et l'ID unique d'un élément sont corrects, mais que l'élément n'a pas été assigné au localisateur, une case à cocher apparaît : vous pouvez alors mettre à jour l'élément afin de créer son assignation. Vous pouvez enregistrer ces informations dans un fichier journal ayant pour nom verifylocator_date/time, date/time indiquant la date et l'heure auxquelles le fichier journal a été généré.
