Un affichage personnalisé est un type d'élément contenant du code HTML. Lorsque vous ajoutez un affichage personnalisé à un espace de travail, un dossier, un projet, une liste de tâches ou un document composite, le code HTML s'intègre à l'interface. Cet affichage personnalisé est inséré entre les menus de navigation et les éléments du recueil ou de l'espace de travail. Vous pouvez conserver le fichier affichagepersonnalisé.html dans votre dossier ou votre espace de travail et empêcher l'apparition de cet affichage en le désactivant. Lorsque vous désactivez un affichage personnalisé, ce dernier s'affiche dans la liste des dossiers, mais n'apparaît pas dans l'interface du dossier ou de l'espace de travail.

La première étape pour ajouter un affichage personnalisé consiste à créer le fichier HTML définissant cet affichage, ainsi que d'autres fichiers, tels que les fichiers image ou les feuilles de style. Par défaut, un affichage personnalisé peut contenir les éléments suivants :

  • Code HTML, JavaScript, VBScript, applets Java ou tout autre type de code pouvant être traité par un navigateur Web. Cependant, votre administrateur peut limiter votre capacité à ajouter certains contenus.

  • Liens vers d'autres éléments de Content Server ou d'autres pages Web.

  • Références à des fichiers audio, image ou à d'autres fichiers stockés dans Content Server ou dans un répertoire situé sur un autre serveur Web accessible.

  • Les recherches enregistrées dans Content Server que les autres utilisateurs peuvent ajouter à leurs favoris ou dont ils peuvent se servir pour réutiliser vos critères de recherche ou que vous pouvez utiliser pour définir la priorité d'affichage des formulaires de recherche personnalisée. Pour plus d'informations sur l'utilisation des vues personnalisées avec des recherches enregistrées, voir Utilisation des recherches enregistrées.

Etant donné que le contenu d'un affichage personnalisé est inséré dans le fichier HTML de l'interface, le navigateur Web ignore les balises telles que <TITLE>, <BODY> ou <FRAMESET>. Cela signifie que les types de codes que vous pouvez inclure sont limités. Par exemple, vous ne pouvez pas créer de cadres dans un affichage personnalisé.

 

Note :
Vous pouvez ajouter un dossier de ressources au même endroit que l'affichage personnalisé afin d'y stocker les images et les autres fichiers que l'affichage personnalisé utilise. Pour éviter la suppression accidentelle de ces fichiers et vous permettre d'optimiser l'organisation de votre espace de travail, il est recommandé de masquer l'affichage personnalisé et le dossier de ressources. Pour plus d'informations sur le masquage d'éléments, reportez-vous à la rubrique Personnalisation des espaces de travail et des recueils.

 

La création et la gestion des affichages personnalisés sont généralement réservées à certains utilisateurs ou groupes. Pour ajouter une vue personnalisée, vous devez disposer du privilège de création d'objet pour les vues personnalisées, ainsi que des autorisations Ajouter les éléments et Modifier sur les recueils appropriés. Toutefois, vous n'avez pas besoin d'autorisation pour modifier les sous-éléments affectés par un affichage personnalisé. Pour modifier une vue personnalisée ajoutée par un autre utilisateur, vous devez disposer de l'autorisation Modifier sur la vue personnalisée.

Utilisation de modèles pour créer des affichages personnalisés

Vous pouvez créer des modèles basés sur des affichages personnalisés existants de Content Server ou basés sur un fichier que vous avez stocké sur votre ordinateur. Lorsque vous ajoutez un modèle, il ne devient pas automatiquement un affichage personnalisé ; vous devez ensuite ajouter un affichage personnalisé basé sur ce modèle, puis vérifier s'il est activé.

Si votre administrateur a ajouté des modèles d'affichage personnalisé au volume des modèles, vous pourrez accéder à ces modèles lors de la création d'un affichage personnalisé. Les modèles peuvent également être ajoutés à un espace de travail ou à un dossier pour lequel vous disposez de l'autorisation Ajouter un élément.

Gestion des affichages personnalisés

Vous déterminez les utilisateurs ou groupes auxquels un affichage personnalisé s'applique en définissant des autorisations pour cet affichage. Par exemple, vous pouvez :

  • Appliquer la vue personnalisée à tous les utilisateurs en assignant à la vue personnalisée l'autorisation Voir le contenu. Ceci permet d'activer l'accès public pour la vue personnalisée.

  • Appliquer la vue personnalisée à des utilisateurs spécifiques uniquement en accordant l'autorisation Voir le contenu à ces utilisateurs ou groupes.

  • Appliquer la vue personnalisée à tous les intervenants d'un projet en accordant aux coordinateurs, aux membres et aux invités d'un projet l'autorisation Voir le contenu pour la vue personnalisée.

  • Appliquer la vue personnalisée aux intervenants d'un projet ayant un rôle spécifique uniquement en accordant aux coordinateurs, aux membres ou aux invités d'un projet l'autorisation Voir le contenu pour cette vue personnalisée.

Pour de plus amples informations sur les autorisations, reportez-vous à la section Gestion des autorisations.

 

Note :
Les nouveaux affichages personnalisés sont activés par défaut, ce qui signifie qu'ils s'affichent automatiquement sans qu'il soit nécessaire de les activer. Si plusieurs affichages personnalisés résident dans le même recueil, ils s'afficheront selon l'ordre alphabétique. Tous les autres seront ignorés, à moins qu'ils soient manuellement activés.
