Les programmes ISA vous aident à définir le cycle de vie d'un objet. Vous pouvez créer un programme ISA pour des classements RM ou des objets gérés RM. Lorsque vous créez un programme ISA, vous pouvez modifier ses propriétés ou le supprimer de l'ISA.

 

Note :
Vous ne pouvez pas modifier un programme IRCH sous Content Server.

 

Si une importation de gestionnaire de rétention IRCH a été effectuée, vous pouvez appliquer un programme IRCH à un ISA.

 

Note :
Le gestionnaire de documents peut désactiver le champ Conservation maximale qui s'affiche lorsque vous appliquez un programme à un élément classé. Dans ce cas, le champ n'apparaît pas dans les recherches de sort final et les programmes ISA.

 

Les paragraphes suivants décrivent les processus et les règles liés à la création d'un programme ISA.

Processus par défaut pour les objets classés et les classements RM

Lorsque vous appliquez un programme à un élément classé, les processus par défaut possibles sont les suivants :

  • Supprimer le format électronique

  • Détruire

  • Exporter

  • Changer le statut

  • Officialiser

    Vous pouvez également cocher la case Créer la présentation correspondant à l'action Officialiser pour créer une présentation de l'élément lorsqu'il est officialisé.

  • Créer la présentation

  • Mettre à jour le fournisseur de stockage

     

    Note :
    Cette option insère une entrée dans l'une des tables de fournisseur de stockage en vue de réévaluer les règles du fournisseur de stockage appliquées à l'objet. Cela permet de déterminer si l'objet doit être déplacé vers un nouveau fournisseur de stockage. Cette option est disponible uniquement lorsque le module Storage Provider est installé.

    La fonction Mettre à jour le fournisseur de stockage place les objets du module Content Move dans une file d'attente afin de les réévaluer et de les déplacer vers un périphérique de stockage différent. Par exemple, si vous spécifiez une règle édictant que les documents dont le statut est INA (inactif) doivent être déplacés vers le fournisseur de stockage A, lors du traitement des résultats d'une recherche de sorts finals, les éléments dont le statut est INA sont déplacés vers le fournisseur de stockage A. Cependant, s'il existe une autre règle présentant une priorité supérieure sur le ou les documents que vous essayez de déplacer, c'est cette règle qui est observée et le document n'est donc pas déplacé. Vous devez par conséquent toujours vérifier l'ordre et la priorité des règles définies pour le module Content Move. Pour des informations plus détaillées au sujet de la configuration des fournisseurs de stockage et des règles, reportez-vous à Gestion de stockage flexible.

     

Lorsque vous appliquez un programme à un classement RM, les processus par défaut possibles sont les suivants :

  • Détruire

  • Exporter

  • Fermer

  • Changer le statut

 

Note :
Lorsque vous définissez l'action à effectuer sur Supprimer le format électronique, Exporter, Fermer, Mettre à jour le fournisseur de stockage, Officialiser ou Changer le statut, une liste des codes de statut disponibles s'affiche. Si vous sélectionnez un nouveau code de statut, celui-ci est appliqué à l'objet après sa suppression, son exportation ou sa fermeture. Le nouveau statut de l'élément s'affiche dans les onglets Infos spécifiques et Détails du document de l'élément sur lequel l'action a été exécutée. Si vous choisissez l'action Changer le statut, vous devez choisir un nouveau statut.

 

Types de règles ISA

Vous pouvez baser un programme ISA sur les cinq types de règles suivants :

  • Date calculée, qui permet de créer un programme ISA qui utilise une date spécifiée et une période spécifiée pour calculer la période de conservation.

  • Calcul selon le calendrier, qui permet de créer un programme ISA qui utilise une date spécifiée et une définition de fin d'année spécifiée pour calculer la période de conservation.

  • Evénementiel, qui permet de créer un programme ISA qui utilise une condition d'événement spécifiée pour calculer la période de conservation. Vous ne pouvez pas créer des programmes ISA événementiels pour les ISA possédant une période de conservation fixe spécifiée.

  • Date précise, qui permet de créer un programme ISA qui utilise une date fixe spécifiée pour calculer la période de conservation.

  • Définitif, qui permet de créer un programme ISA sans action associée à effectuer.

Date calculée et types de règles de calcul selon le calendrier

Les types de dates que vous pouvez sélectionner pour les options Date calculée et Calcul selon le calendrier varient selon que vous créez un programme ISA pour des classements RM ou des objets classés.

Les types de dates disponibles pour les classements RM sont les suivants :

  • Date de la catégorie permet d'utiliser les attributs de date spécifiés par une catégorie choisie. Si vous choisissez Date de la catégorie comme option de date, vous devez cliquer sur le lien Sélectionner une catégorie qui s'affiche pour choisir une catégorie. Si une catégorie possède des attributs de date, une liste des noms d'attribut de date s'affiche. Dans le cas contraire, un message vous indique que la catégorie sélectionnée ne contient pas d'attribut de date. Vous devez choisir une catégorie contenant des attributs de date pour continuer.

  • Date de création permet d'utiliser la date de création de l'objet.

  • Date de dernier ajout permet d'utiliser la date de dernier ajout d'un objet géré à un classement RM. Si le programme ISA est basé sur un classement RM, cette option n'apparaît pas.

  • Date de modification permet d'utiliser la date de dernière modification de l'objet.

  • Date du statut permet d'utiliser la date du statut de l'objet.

Les types de dates disponibles pour les objets classés sont les suivants :

  • Date de création permet d'utiliser la date de création de l'objet.

  • Date d'ajout de la dernière version permet d'utiliser la date à laquelle la dernière version de l'objet a été ajoutée.

  • Date de modification permet d'utiliser la date de dernière modification de l'objet.

  • Dates de début du recueil physique permet d'utiliser la date de début d'un recueil physique.

  • Dates de fin du recueil physique permet d'utiliser la date de fin d'un recueil physique.

  • Date du document permet d'utiliser la date à laquelle l'élément Records management a été créé par défaut.

  • Date de réservation permet d'utiliser la date de dernière réservation de l'objet.

  • Date du statut permet d'utiliser la date spécifiée par le statut.

  • Date de création de version permet d'utiliser la date de création de la version. Cette date peut être utilisée en association avec l'option Eliminer les versions remplacées et permet d'éliminer les versions antérieures à cette date après un délai spécifié. L'action Eliminer les versions est la seule action disponible avec ce type de date.

     

    Note :
    L'option Eliminer les versions remplacées apparaît uniquement si elle a été activée dans l'espace de travail Records Management.

     

Lorsque vous créez l'un de ces types de programme ISA, vous pouvez spécifier une période de conservation fixe, ce qui empêche la destruction des enregistrements avant la période de conservation fixe spécifiée. Un seul programme fixe de conservation est autorisé par ISA.

 

Note :
Si votre administrateur n'a pas activé la conservation fixe, la section correspondante n'apparaît pas dans l'assistant de création de programme de conservation. Vous ne pouvez activer la conservation fixe que pour les objets classés.

Si vous activez la conservation fixe pour les objets classés et que le programme est de type définitif, l'élément ne pourra jamais être supprimé. Tous les autres types de programmes permettent leur suppression, mais elle ne peut jamais avoir lieu avant la date future spécifiée.

Il est impossible d'utiliser simultanément des programmes de type Conservation fixe et Evénementiel.

 

Actions liées au programme ISA

Sur la page Programme ISA - Action à exécuter, vous devez spécifier un processus par défaut à exécuter sur les classements RM ou les objets classés lorsque ceux-ci correspondent aux dates ou aux périodes spécifiées dans le programme ISA. Les processus par défaut sont les suivants :

  • Aucun : aucune action par défaut n'est définie.

  • Supprimer le format électronique : supprime le format électronique.

  • Détruire : détruit les objets.

  • Exporter : exporte les objets. Par défaut, lorsqu'une action d'exportation de sort final est exécutée, seule la version actuelle de l'élément est exportée. Vous pouvez modifier ce paramètre pour inclure toutes les versions dans l'action d'exportation dans l'onglet Paramètres de la recherche de sort final.

    Vous pouvez inclure les présentations d'un élément lorsque vous exécutez l'action Exporter. Pour ce faire, cochez la case Inclure la(les) présentation(s) dans l'onglet Paramètres de la recherche de sort final.

  • Fermer : ferme les objets s'il s'agit de types d'objets de classement RM.

  • Changer le statut : applique aux objets le statut que vous spécifiez dans la liste déroulante Nouveau statut correspondante.

  • Officialiser : retire toutes les autorisations, sauf les autorisations Voir et Voir le contenu s'il s'agit d'objets clés RM.

    Vous pouvez cocher la case Créer la présentation correspondant à l'action Officialiser pour créer une présentation de l'élément lorsqu'il est officialisé. Si vous ne sélectionnez pas l'option Créer la présentation, les résultats de recherche renvoient uniquement les éléments pour lesquels il n'existe pas de présentation.

  • Officialiser

  • Créer la présentation, crée des présentations de l'élément au moment de l'exécution de l'action Recherche de sorts finals. L'utilisation d'une présentation vous permet de convertir les éléments versionables à un format de stockage à long terme. Vous pouvez créer un nouveau code de statut pour la présentation au moment où vous sélectionnez Créer une présentation comme option de traitement par défaut.

  • Mettre à jour le fournisseur d'espace de stockage : marque les objets à réévaluer lorsque les métadonnées RM incluses dans les règles du fournisseur de stockage changent.

     

    Note :
    Le bouton radio Mettre à jour les règles de stockage est disponible uniquement si le module Content Move a été installé.

     

  • Eliminer les versions : permet de saisir le nombre minimum de versions à conserver. Cette action n'apparaît que lorsque Date de création de la version est choisie comme Date à utiliser. Le nombre de versions spécifié sera conservé, sans tenir compte de la date à laquelle les versions ont été ajoutées, ni du délai spécifié pour l'élimination des versions. Lorsque vous exécutez une action d'élimination des versions, les versions sont supprimées, sauf s'il existe des versions verrouillées ou des générations d'un élément. S'il existe une version ne pouvant pas être supprimée pour une quelconque raison (y compris les versions verrouillées ou générations), le nœud n'est pas traité.

    Si Afficher les options de version éliminées a été défini sur Oui par votre gestionnaire de documents, vous pouvez choisir d'éliminer les versions principales d'après l'ancienneté de la version ou la date à laquelle la version a été remplacée par une version plus récente.

 

Note :
Les actions Supprimer le format électronique, Fermer, Exporter, Officialiser et Réévaluer les règles de stockage vous permettent de spécifier un nouveau statut pour l'objet après que l'action a été effectuée. Pour ce faire, cliquez sur un statut dans la liste déroulante Nouveau statut située en regard de l'action par défaut correspondante.

Lorsque vous modifiez un programme ISA, vous pouvez fournir une raison expliquant pourquoi vous avez procédé à la modification dans le champ Raison de la mise à jour. Les informations saisies dans ce champ sont suivies dans l'historique d'expertise.

En cas d'erreur, un journal des erreurs est enregistré dans le répertoire content_server_location\logs\dispositions. Ce journal des erreurs est propre au lot de résultats et utilise l'ID de nœud du sort final et la date/heure du lot de résultats au format nodeID_YYYMMDD_HHmmss.log. Vous pouvez également afficher le journal des erreurs en cliquant sur l'icône Fonctions du nœud de résultat et en choisissant Afficher le journal d'erreur.
