Configuration des champs Détails des documents

Vous pouvez sélectionner les champs à afficher sur la page Ajouter un élément lorsqu'un utilisateur ajoute des éléments gérés. Vous pouvez également définir quels champs sont obligatoires. Si vous définissez un champ comme étant obligatoire sur la page Ajouter un élément, les utilisateurs devront renseigner ce champ lorsqu'ils ajouteront un objet géré à Content Server. Si vous définissez un champ comme étant obligatoire dans l'onglet Détails des documents d'un élément géré, l'élément doit être classé. Les utilisateurs pourront renseigner le champ requis, ou le classement pourra hériter des données d'un objet parent ou encore être défini en tant que classement par défaut. Vous pouvez activer les champs suivants :

  • Date du document

  • Type de document

  • Etat

  • Date du statut

  • Date de réception

  • Essentiel, Période de cycle de mise à jour et Date de prochaine révision

  • Officialiser

  • Support de stockage

  • Acquisition

  • Objet

  • ISA

  • Auteur ou émetteur

  • Destinataire(s)

  • Autre(s) destinataire(s)

  • Organisation d'origine

Configuration des champs de classement RM requis

Vous pouvez définir les champs de classement RM obligatoires. Les utilisateurs devront renseigner ces champs lorsqu'ils ajouteront des objets. Vous pouvez définir les champs de classement RM suivants comme étant obligatoires :

  • Numéro de fichier

  • Date de création

  • Etat

  • Date du statut

  • Essentiel

  • Support de stockage

  • Autorité de sort final

  • Période de cycle de mise à jour

  • ISA

  • Objet

  • Keywords

Insertion d'un renvoi sur la page Ajouter un élément

Lorsqu'elle est activée, la fonctionnalité de renvoi s'affiche sur la page Ajouter un élément et permet aux utilisateurs de créer un renvoi à partir de l'élément qu'ils ajoutent à un autre élément dans Content Server. Vous pouvez activer la fonctionnalité de renvoi à partir de la page Configurer les champs Records management.
