Vous pouvez créer des classements RM dans l'espace de travail de classement, en tant que parties intégrantes d'un arbre de classement ou dans un autre classement RM. Dans le module Records management, vous pouvez accéder à l'espace de travail de classement via l'espace de travail de Records management. Pour plus d'informations sur les classements, consultez l'aide en ligne du module Classifications. Vous pouvez également rechercher des classements RM.

Les classements RM sont précédés d'une icône de classement RM ( ).

Lorsque vous créez un classement RM pour un espace de travail de classement ou en tant que partie intégrante d'un arbre de classement, vous pouvez spécifier un nom, une description, un sujet, des mots clés, ainsi que des informations sur l'autorité de sort final. Vous pouvez également spécifier un ISA (identificateur de séries d'archives) et une catégorie. Si l'administrateur a autorisé plusieurs groupes de gestionnaires de documents basés sur les classements RM, vous pouvez sélectionner un groupe dans la page Ajouter un élément lorsque vous créez un classement RM. Le groupe sélectionné sera le groupe de gestionnaires de documents pour l'objet géré. Si cette option n'est pas sélectionnée, le champ Gestionnaire de documents n'apparaît pas sur la page Ajouter un élément. Pour plus d'informations sur les paramètres Gestionnaire de documents, reportez-vous à Attribution de plusieurs groupes de gestionnaires de documents.

Si vous copiez ou déplacez un élément dans Content Server, vous pouvez décider d'hériter ou non du classement RM existant. Votre administrateur peut définir l'action par défaut devant hériter du classement RM existant quand des éléments sont déplacés ou copiés. Il peut également choisir de masquer l'action par défaut, auquel cas l'option permettant de choisir comment hériter du classement RM existant n'apparaît pas lorsque vous déplacez ou copiez des éléments. Pour plus d'informations sur la copie ou le déplacement d'éléments, consultez l'aide en ligne de Content Server.

Une fois que vous avez créé des classements RM, vous pouvez leur assigner des utilisateurs avec des droits supplémentaires. Les utilisateurs bénéficiant de droits supplémentaires peuvent rechercher des objets avec un classement RM spécifique et y accéder, mais ils ne peuvent pas appliquer ces classements RM aux objets.

Lorsque vous créez un nouvel objet ou que vous classez un objet existant, Content Server vous permet d'assigner un classement RM que vous pouvez d'abord rechercher. Vous pouvez également retirer des classements RM d'un objet classé unique ou de plusieurs objets classés dans une collection ou des résultats de recherche.

 

Note :
Lorsque vous ajoutez un élément à un objet RM classé, cet élément hérite les données de son parent classé. Pour hériter les données d'un objet parent, l'indicateur Hériter à partir de de l'objet géré parent doit être sélectionné dans la page Objets gérés. Ces données héritées incluent les codes de statut, de support essentiel et de support de stockage.

Lorsque vous classez un élément en recherchant les classements RM, la fenêtre de recherche s'affiche dans la langue par défaut du système. Vous avez la possibilité de choisir une autre langue pour la recherche, à condition que cette langue de métadonnées ait été activée par l'administrateur. Pour plus d'informations sur la modification des paramètres de langue du système et l'utilisation de l'éditeur de métadonnées multilingues, reportez-vous à Pour modifier vos paramètres généraux et à Utilisation des éléments généraux.
