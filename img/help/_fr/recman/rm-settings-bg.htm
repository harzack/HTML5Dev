Tableau 1-2 : Paramètres de Records Management

Paramétrage Description Remarques complémentaires
Ajouter des classements RM hérités
Permet d'indiquer si un classement RM hérité d'un objet parent doit être ajouté à un élément en plus du classement sélectionné.
Ce paramètre est disponible uniquement si l'option Classements multiples est activée.
Cette option s'applique uniquement lors de l'ajout de nouveaux éléments à un objet parent.
Classements multiples
Permet de spécifier si des objets Content Server peuvent disposer de plusieurs classements Records management ou d'un seul.
  
Masquer l'option permettant d'hériter du classement RM sur copier et déplacer
Permet aux utilisateurs de choisir si les classements RM doivent être hérités d'un objet parent lorsque les utilisateurs déplacent ou copient des éléments.
Si vous activez le paramètre Hériter du classement RM sur copier et déplacer, les utilisateurs ne seront pas en mesure de choisir comment ils souhaitent hériter des classements RM. Les éléments déplacés ou copiés héritent des classements RM en fonction de la valeur par défaut que vous spécifiez dans le paramètre Hériter du classement RM sur copier et déplacer.
Hériter du classement RM sur copier et déplacer
 Spécifie si les classements RM sont hérités lorsque des éléments sont copiés ou déplacés. Par défaut, les classements RM sont toujours hérités.
Conservation fixe
Permet d'activer la fonctionnalité de conservation fixe.
Lorsque ce paramètre est activé, la case à cocher Conservation fixe apparaît lorsque des utilisateurs créent ou mettent à jour un programme ISA. Si la conservation fixe est définie sur un programme ISA et que l'ISA est assigné à un objet géré, une date est calculée en fonction du programme, ce qui empêche la suppression de l'objet avant la date calculée.
Modifier l'ISA
 Permet d'autoriser des utilisateurs autres qu'un gestionnaire de documents à modifier les ISA.  
Approuver les programmes ISA
 Permet d'approuver manuellement les programmes ISA.
Si cette option est désactivée, les programmes ISA sont auto-approuvés. Ce paramètre est désactivé par défaut, ce qui signifie que les programmes ISA n'exigent pas d'approbation pour être utilisés.
Lorsqu'un ou plusieurs programmes d'un ISA sont en attente d'approbation, l'icône En attente d'approbation s'affiche en regard du nom de l'ISA.
Workflow de suppression
Permet de sélectionner et d'activer un workflow de suppression à utiliser pour supprimer les objets du classement RM.
  
Activer les éléments avec le classement par défaut dans le workflow de suppression
Permet de choisir de lancer un workflow de suppression pour les éléments avec un classement par défaut.
 
Supprimer les enfants
Permet d'autoriser les utilisateurs à supprimer des objets ayant des enfants. Si cette option n'est pas activée, les utilisateurs ne peuvent pas supprimer des objets ayant des enfants.
  
Créer un renvoi
Permet de définir si des renvois doivent être créés automatiquement lorsqu'un objet géré est copié.
Pour que les renvois soient créés automatiquement, le gestionnaire de documents doit également définir un code par défaut dans le tableau de renvoi. Pour obtenir des informations sur la maintenance des codes Records management, reportez-vous à la rubrique Maintenance des codes dans l'aide en ligne utilisateur de Records management.
Activer une version de document
Permet de définir si des renvois doivent être utilisés pour indiquer des versions de document.
Lorsque ce paramètre est activé, le paramètre Nouvelle version de document s'affiche dans la liste déroulante Ajouter : Renvoi de l'onglet Renvoi ainsi que dans le champ Renvoi de la page Ajouter un élément d'un élément géré.
Vous pouvez également sélectionner un Code de version de document.
Ajouter une version à un objet en attente
Permet de définir si une version peut être ajoutée à un objet qui est en attente.
  
Ajouter des éléments à un recueil en attente
Permet de définir si des éléments peuvent être créés dans, ou copiés et déplacés vers un recueil en attente.
 Le réglage par défaut est Oui, ce qui permet l'ajout d'éléments aux recueils lorsque ces derniers sont en attente. Si l'option est définie sur Non et que les utilisateurs essaient d'ajouter des éléments à un recueil en attente, ils reçoivent un message d'erreur
Augmenter le nombre maximum de versions d'objets en attente
 Vous permet d'augmenter le nombre maximum de version d'objets en attente lorsque le nombre maximum actuel de versions est atteint.  
Créer une nouvelle partie RM
 Permet de spécifier si une nouvelle partie RM doit être automatiquement créée lorsqu'une autre partie RM est fermée. Le réglage par défaut est Non. Si le paramètre est défini sur Oui, une nouvelle partie RM est automatiquement créée lorsqu'une autre partie RM est fermée manuellement (via l'onglet Informations spécifiques) ou automatiquement (par le biais d'un processus d'action de sort final). Ce cas de figure ne se produit que lorsque le dossier RM parent n'est pas fermé.
