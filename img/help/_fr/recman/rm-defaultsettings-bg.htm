Tableau 1-1 : Paramètres système par défaut

Paramétrage Description
Classement par défaut
Permet de spécifier un classement par défaut. Si aucun classement par défaut n'est spécifié, tous les objets peuvent être classés par héritage. Si l'indicateur Hériter à partir de n'est pas spécifié pour l'objet parent, il n'hérite pas du classement RM par défaut. Les utilisateurs peuvent toujours choisir un classement RM différent de celui hérité, qui remplacera le classement RM par défaut. Pour plus d'informations sur la définition de classements par défaut pour des types d'objets spécifiques, reportez-vous au Guide d'installation et d'administration du module OpenText Records Management.
Emplacement par défaut pour les éléments créés dans l'espace de travail Classement
 Spécifiez un emplacement par défaut pour les éléments créés au sein de l'espace de travail Classements. Ces éléments comprennent Apparences, Arbres de classement, Vues personnalisées et Classements RM.
Titrage du classement RM
Définissez la possibilité d'utiliser le titrage pour les objets gérés et le classement RM auquel les ajouter.
Si le titrage est activé, et que l'option Utiliser le titrage est sélectionnée pour un objet géré, lorsque les utilisateurs créent un objet géré et spécifient un classement, le nom du classement RM est ajouté au nom ou à la description de l'objet géré. Le nom d'objet apparaît dans la langue spécifiée par l'utilisateur ou le système. Si aucune valeur n'est indiquée pour le nom d'objet dans la langue spécifiée, c'est la valeur par défaut en anglais qui est utilisée.
Nombre de résultats de recherche à renvoyer
Spécifiez le nombre de résultats de recherche à renvoyer sur une page.
Séparateur de numéros de fichiers
 Permet de spécifier un caractère par défaut à afficher entre les numéros de fichiers pour les différents niveaux de classement.
Mode d'affichage du classement RM
Permet de spécifier si les classements s'affichent sous la forme d'une liste ou s'ils sont mis en valeur lors de la navigation dans un volume de classements. Le mode liste place les éléments dans une liste verticale; le mode intégré les place dans une section distincte d'un recueil pour les distinguer des autres.
Parcourir les ISA et les attentes
Permet de définir le mode de navigation utilisé pour parcourir les ISA et les attentes. Ce paramètre détermine le comportement des pages des ISA et des attentes lorsqu'elles sont ouvertes à partir de l'espace de travail Records management. En mode Navigation standard, les pages des ISA et des attentes sont automatiquement renseignées conformément à la liste des ISA ou attentes disponibles. En mode Recherche avancée, il est demandé à l'utilisateur de lancer une recherche pour répertorier les ISA ou attentes disponibles. En présence d'un grand nombre d'ISA ou d'attentes, le mode Recherche avancée doit être activé.
Masquer les icônes
Permet de masquer les icônes officielles et classées lors de la navigation. Si les icônes sont masquées, les icônes Classé et Officiel n'apparaissent pas en regard des éléments classés ou officialisés dans la fenêtre de navigation. Si les icônes sont affichées, les utilisateurs peuvent cliquer dessus et être dirigés vers l'onglet Détails du document.
Actions de gestion des documents en vue Parcourir
Spécifiez si des actions de sélection multiple doivent apparaître pour les fonctions de gestion des documents en vue Parcourir. Si oui, les utilisateurs peuvent appliquer les métadonnées RM, appliquer ou enlever des attentes, appliquer ou enlever des classements, assigner des renvois et finaliser des enregistrements pour plusieurs éléments gérés simultanément.
