Les codes Records management permettent au gestionnaire de documents de normaliser la saisie de données en indiquant les options spécifiques que les utilisateurs peuvent choisir. Pour toutes les tables, il est possible de créer des codes à l'aide de 32 caractères au maximum. Vous pouvez gérer des codes dans les tables suivantes :

  • Acquisition : permet de spécifier les options de récupération et de transfert des objets.

  • Renvoi : permet de spécifier les options qui décrivent les relations entre objets.

    Lorsque vous créez ou modifiez un code de renvoi, vous pouvez également sélectionner une attente existante à associer au renvoi dans la liste déroulante Attente associée. Lorsque le renvoi est appliqué, l'attente associée est appliquée automatiquement aux éléments gérés, en tant qu'attente directe. Une seule attente peut être associée à un code de renvoi.

  • Document essentiel : permet de spécifier les options de définition de l'importance d'un objet.

  • Sort : permet de spécifier les options de description de la fonction d'un programme.

  • Type d'attente : permet de spécifier les options des types d'attentes.

  • Juridiction : permet de spécifier des codes de juridiction, comme Fédéral, Régional, Communal, National ou Cantonal.

  • Type de provenance : permet de modifier les codes de type de provenance et d'en ajouter de nouveaux.

  • Conservation d'ISA : permet de spécifier des options de programmes de conservation.

  • Phase de conservation d'ISA : permet de spécifier les options des phases par lesquelles les objets peuvent passer.

  • Codes de règles d'ISA : permet de spécifier des options de règles.

  • Statut de l'ISA : permet de spécifier les options de statut ISA, par exemple Actif ou Fermé.

  • Statut : permet de spécifier les options de statut, par exemple Actif ou Fermé. Vous pouvez également ajouter des autorisations spécifiques afin de permettre aux utilisateurs et aux groupes d'accéder à un code de statut. Les utilisateurs spécifiés disposent des autorisations que vous leur accordez (Voir, Voir le contenu, Modifier, Modifier les attributs, Ajouter les éléments, Réserver, Supprimer des versions, Supprimer et Modifier) pour l'élément géré lorsqu'un élément comportant le code de statut est ajouté, mis à jour ou enlevé. Les autorisations sont automatiquement ajoutées à la liste d'autorisations de la liste de contrôle d'accès pour l'élément concerné lorsque la valeur du statut est mise à jour. Pour de plus amples informations sur les autorisations de la liste de contrôle d'accès, reportez-vous à la rubrique Autorisations.

  • Support de stockage : permet de spécifier les options de désignation des types de supports de stockage disponibles, par exemple papier ou électronique.

Lorsque vous créez un code, vous pouvez ajouter une description. Vous pouvez également spécifier un code relatif lorsque vous créez des codes de renvoi, un ordre de la phase lorsque vous créez des codes de conservation d'ISA, ainsi qu'une période par défaut lorsque vous créez des codes de type d'attente.

 

Note :
Si vous n'avez créé aucun code, la table affiche un message indiquant qu'elle est vide.

 

Vous avez le droit de modifier et supprimer des codes pour les tables de base de données, de définir un code par défaut ou de supprimer le paramètre par défaut. En mode édition, vous ne pouvez pas modifier les valeurs de code.

Vous créez des codes sur la page Gestion des tables, ou vous les créez à la volée (lorsqu'il n'existe pas de code ou que le code requis n'a pas encore été créé) sur la page où le code va être utilisé. L'option permettant de créer un nouveau code à la volée n'est pas disponible à partir des pages suivantes :

  • Recherche ISA

  • Pages Programmes ISA - Conditions d'événements et Actions à exécuter

  • Onglet Infos spécifiques de sort final

  • Page de résultats de lots de sorts finals

  • Créer une recherche d'acquisitions

  • Onglet Infos spécifique de recherches d'acquisition

  • Mappage du gestionnaire de rétention IRCH
