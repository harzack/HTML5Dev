Une recherche de sorts finals renvoie une liste d'objets Content Server compilée via une recherche dans les programmes ISA, puis dans les objets Content Server correspondant aux critères définis par les programmes ISA. Pour plus d'informations sur les critères de recherche définissables dans la section Processus par défaut pour les programmes ISA et recherches de sort final, reportez-vous à Actions liées au programme ISA.

Vous pouvez créer une recherche manuelle de sorts finals ou bien créer une recherche programmée pour être répétée chaque jour, semaine, mois ou année, dans un intervalle de dates flottantes. Lorsque vous créez une recherche de sorts finals, vous pouvez spécifier un nom, une description, une valeur ISA, un processus par défaut et un type d'objet.

Une recherche d'acquisitions renvoie une liste d'objets Content Server auxquels est assigné un code d'acquisition particulier. Lorsque vous créez une recherche d'acquisitions, vous pouvez spécifier un nom et une description.

Vous pouvez gérer les recherches d'acquisitions et de sorts finals que vous créez en les stockant dans un recueil ou un groupe de sort final créé avant la recherche d'acquisitions ou de sorts finals. Un groupe de sort final fonctionne comme un dossier Content Server et vous permet de regrouper des recherches de sorts finals et d'acquisitions similaires.

Affichage des lots de résultats

Une fois la recherche de sorts finals terminée, l'utilisateur qui a exécuté la recherche manuellement ou le propriétaire de la recherche programmée est informé de la fin du processus par e-mail. L'e-mail affiche le nombre d'éléments retournés et propose un lien vers le lot de résultats. Un lot de résultats est un aperçu des résultats de la recherche de sorts finals. La page Lot de résultats répertorie tous les éléments de la recherche de sorts finals. Pour restreindre les résultats du lot de résultats, vous pouvez filtrer la liste en spécifiant les filtres de recherche suivants : nom, ISA, numéro de fichier, date calculée ou décision de révision. La fenêtre Filtrer les résultats de la recherche s'affiche lorsque vous cliquez sur le bouton Filtrer les résultats de la recherche à la page Lot de résultats.

La page Lot de résultats indique le nombre d'éléments sur lesquels des actions ont été effectuées, ainsi que le nombre d'éléments avec des décisions de révision dont le statut est défini sur En attente, Approuvé ou Rejeté. A la page Lot de résultats, vous pouvez effectuer les opérations suivantes : appliquer des acquisitions, modifier l'action de sort final ou la valeur d'état de l'action, changer des décisions de révision et terminer des révisions. Vous pouvez aussi afficher les résultats par réviseur de la décision et afficher le journal de destruction en cliquant sur les liens à la page Lot de résultats. Pour plus d'informations sur les révisions, reportez-vous à la rubrique Utilisation du processus de révision de sorts finals.

S'ils ont été activés sur la page Paramètres de sort final, des boutons supplémentaires, appelés boutons d'actions de sort final, peuvent être disponibles. Il s'agit, par exemple, du bouton Déplacer, qui vous permet de déplacer des éléments d'un lot de résultats. Contrairement aux autres éléments de Content Server, lorsque vous déplacez un élément à partir d'une page de lot de résultats de sorts finals, celui-ci n'est pas supprimé du lot de résultats. Un paramètre de rejet automatique est également disponible pour chaque bouton d'action de sort final. Lorsqu'il est activé, la décision de révision des éléments déplacés est automatiquement définie sur Rejeté, et les commentaires de révision indiquent que les éléments ont été automatiquement rejetés. Si l'élément déplacé est un dossier dont le contenu apparaît dans les résultats d'une recherche de sort final, le contenu en question est également déplacé et la décision de révision Rejeté est appliquée.

 

Note :
Les utilisateurs doivent disposer au moins des autorisations Voir/Voir le contenu assignées à la recherche de sorts finals pour consulter un lot de résultats de sorts finals.

Le gestionnaire de documents peut désigner les adresses e-mail auxquelles les résultats de la recherche de sorts finals seront envoyés pour révision. La ou les adresses e-mail peuvent être définies au préalable, avant le lancement de la recherche de sorts finals ou d'acquisitions. Le gestionnaire de documents peut désigner un seul utilisateur Content Server et groupe Content Server comme destinataire des résultats, mais d'autres destinataires peuvent être ajoutés en saisissant des adresses e-mail valides dans le champ approprié.

 

Traitement des sorts finals

Une fois la recherche de sorts finals terminée, vous pouvez traiter les résultats à la page Lot de résultats, ce qui signifie que vous effectuez l'action sélectionnée sur les éléments du résultat de la recherche. Si un utilisateur effectue des actions sur tous les éléments d'un lot de sorts finals, le statut du lot devient Actions achevées. Si un utilisateur effectue des actions sur certains éléments d'un lot de sorts finals, mais si d'autres éléments doivent toujours être traités, l'état du lot devient Actions effectuées.

Si les éléments sont détruits via le lot de résultats de sorts finals, un journal de destruction est mis à votre disposition. Vous pouvez créer un journal des résultats et l'enregistrer dans Content Server. La fonction d'affichage du journal de destruction n'apparaît dans le menu Fonctions d'un lot de résultats que lorsque des éléments ont été détruits dans ce lot. Pour plus d'informations sur les actions de traitement par défaut des objets classés et classements RM, reportez-vous à Processus par défaut pour les objets classés et les classements RM.

Vous pouvez configurer les résultats de recherche pour qu'ils soient automatiquement traités à l'issue de la recherche de sorts finals. Ce paramètre n'est pas activé par défaut. Vous devez activer le paramètre Traiter automatiquement les résultats de la recherche dans l'onglet Spécifique de la recherche de sorts finals.

 

Note :
Les éléments dont la décision de révision est définie sur En cours ou Rejeté ne peuvent pas être traités (automatiquement ou manuellement). Pour plus d'informations sur le processus de décision de révision, reportez-vous à la rubrique Utilisation du processus de révision de sorts finals.

 

Configuration des colonnes de métadonnées et exportation des métadonnées

L'onglet Colonnes de la page Propriétés vous permet de configurer les colonnes qui afficheront les métadonnées des recueils et groupes de sorts finals, ainsi que des recherches de sorts finals et d'acquisitions. Les métadonnées s'affichent en regard d'un objet Sort final lorsque vous naviguez dans la vue Parcourir. Les colonnes de métadonnées sont transmises aux objets enfants des recueils et des groupes, et aux lots de résultats des recherches de sorts finals et d'acquisitions. Pour apparaître, les colonnes de métadonnées doivent être configurées dans le volume à facettes de Content Server. Il est possible d'ajouter des colonnes de métadonnées supplémentaires, selon les colonnes créées par l'utilisateur. Les colonnes Date calculée, Emplacement et Action apparaissent uniquement dans la vue des lots de résultats. Pour plus d'informations, reportez-vous à la rubrique Utilisation du panneau de configuration du volume à facettes.

 

Note :
Le seul moyen d'afficher la colonne Numéro de fichier pour les objets Sort final est d'en configurer l'affichage dans l'onglet Colonnes, sauf s'il s'agit d'une colonne configurée globalement.

 

Vous pouvez exporter les métadonnées de résultats de sort final depuis un lot de résultats vers un fichier texte ou Excel. Par défaut, les informations exportées vers le rapport sont les suivantes : type d'objet, ID de nœud, numéro de fichier, date calculée, chemin complet et action réalisée. Vous pouvez sélectionner des champs de métadonnées RM et d'objet physique supplémentaires à inclure dans le rapport d'exportation. Si vous ne souhaitez pas que tous les champs de métadonnées par défaut soient inclus dans les résultats de l'exportation, vous pouvez choisir d'inclure uniquement les champs de métadonnées RM et d'objet physique inclus dans un lot de résultats. Les objets gérés, classements RM, dossiers RM et parties RM dont les champs Nom et Description contiennent des métadonnées multilingues sont exportés lors de l'exportation d'un sort final de la même façon que les autres métadonnées. Pour plus d'informations sur l'éditeur de métadonnées multilingues, consultez Utilisation des éléments généraux.
