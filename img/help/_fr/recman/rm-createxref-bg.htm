Un renvoi permet de lier deux objets Content Server à l'aide de codes. Vous pouvez créer un code de renvoi à la volée, comme si vous ajoutiez un renvoi à un objet géré, ou ajouter les codes au préalable dans la table de codes de renvoi. Pour en savoir plus, voir Gestion des codes.

Vous pouvez ajouter un renvoi à un objet géré après sa création ou pendant sa création à partir de la page Ajouter un élément. Lorsque vous affectez un renvoi, le type de renvoi et l'emplacement de tous les éléments affectés à un renvoi s'affichent. Si l'élément que vous modifiez est assigné à une attente, l'icône d'attente s'affiche en regard du type de renvoi.

Vous pouvez afficher les informations de renvoi et de version d'un élément géré dans l'onglet Renvoi de la page Propriétés. Pour plus d'informations sur la page Propriétés, reportez-vous à la rubrique Affichage et modification des propriétés d`un élément. Vous pouvez également configurer des renvois et les métadonnées correspondantes pour qu'ils apparaissent sous forme de colonne ou de facette dans l'explorateur de Content Server. Pour ce faire, les colonnes et facettes doivent être configurées dans le volume Facettes dans Content Server. Pour en savoir plus, voir Utilisation du panneau de configuration du volume à facettes.

 

Note :
L'option permettant d'ajouter un renvoi à partir de la page Ajouter un élément n'est disponible que si elle a été activée par le gestionnaire de documents.

Vous pouvez configurer le système afin qu'il utilise des renvois pour indiquer que les différents objets de Content Server sont des versions les uns des autres, même si vous ne disposez pas des autorisations vous permettant d'afficher la version actuelle. Par exemple, si vous disposez uniquement de l'autorisation d'afficher la version 1 d'un objet, vous pourrez tout de même ajouter une version à l'objet, même si vous ne pouvez pas en afficher la version actuelle.

 

Si la version originale d'un enregistrement est supprimée, les autres versions de l'enregistrement restent visibles dans l'onglet Renvoi de la page Propriétés de l'élément.

Ajout de renvois pour plusieurs éléments

Vous pouvez sélectionner plusieurs éléments dans une collection, un ensemble de résultats de recherche ou dans l'explorateur d'un espace de travail, et leur appliquer un renvoi vers un autre élément. Si votre administrateur a activé l'accès fonctionnel, la fonctionnalité permettant l'application de renvois ne s'affichera pas sous forme de bouton dans l'espace de travail ni dans la liste déroulante Autres actions d'une collection ou d'un ensemble de résultats de recherche si la fonction Objet géré –- Renvoi – Attribuer ne vous est pas assignée.

Association d'une attente à un renvoi

Vous pouvez associer une attente à un code de renvoi au moment où vous créez ce code. Si vous associez une attente à un renvoi, chaque fois qu'un utilisateur applique ce renvoi à un élément géré, l'attente associée lui est également appliquée. Vous pouvez retirer l'attente associée au renvoi sur la page Ajouter un renvoi. Vous ne pouvez associer qu'une seule attente à un code de renvoi. Si un utilisateur sélectionne une autre attente, l'attente nouvellement sélectionnée remplace la précédente.

Si une attente est appliquée à un élément à l'aide d'un renvoi avec attente associée, cette attente est retirée automatiquement lorsque le dernier renvoi de ce type est retiré. Vous devez disposer des autorisations nécessaires pour retirer des renvois et des attentes. Les autorisations suivantes s'appliquent à la suppression des renvois et des attentes associées :

Si l'accès fonctionnel n'est pas activé :

  • L'autorisation Modifier est nécessaire pour retirer tous les renvois, sauf le dernier.

     

    Note :
    Une attente n'est pas retirée tant qu'il reste un renvoi assigné.

     

  • L'utilisateur doit appartenir au groupe de gestionnaires de documents désigné pour pouvoir supprimer le dernier renvoi avec une attente associée.

  • L'autorisation Supprimer est nécessaire pour supprimer automatiquement le renvoi assigné lors de la suppression d'un élément. L'attente associée est retirée automatiquement de l'élément seulement si le dernier renvoi avec cette attente associée est retiré.

Si l'accès fonctionnel est activé :

  • L'autorisation Modifier (ou Voir le contenu) et la fonction Renvoi – Supprimer sont requises pour retirer tous les renvois sauf le dernier.

  • L'autorisation Modifier (ou Voir le contenu) et les fonctions Renvoi – Supprimer et Attente – Supprimer sont requises pour retirer le dernier renvoi avec une attente associée. L'attente est également retirée lorsque le dernier renvoi avec l'attente associée est supprimé.

  • L'autorisation Supprimer est nécessaire pour supprimer automatiquement les renvois assignés lors de la suppression d'un élément. L'attente associée est retirée automatiquement de l'élément seulement si le dernier renvoi avec cette attente associée est retiré.
