Un ISA (identificateur de séries d'archives) est une méthode permettant d'appliquer des informations de conservation et de suppression aux documents ou archives électroniques. Lorsque vous créez un ISA, vous pouvez spécifier les informations suivantes :

  • ISA : code utilisé pour représenter l'identificateur de séries d'archives.

  • Titre : titre de l'ISA pour le code d'ISA.

  • Description : description de l'ISA (2 000 caractères maximum).

  • Objet : peut éventuellement être utilisé pour stocker des métadonnées supplémentaires pour l'ISA, telles que le groupe d'enregistrements auquel l'ISA sera appliqué, ou les notes d'application décrivant le domaine de l'ISA.

  • Statut : statut de l'ISA.

  • Date du statut : date à laquelle le statut de l'ISA est devenu valide. La date du statut correspond à la valeur provenant du tableau Statut de l'ISA.

  • Sous contrôle de disposition : détermine si l'ISA sera sous contrôle de disposition. Si vous activez ce paramètre, les objets avec l'ISA assigné ne seront pas supprimés. Ce paramètre est désactivé par défaut.

 

Note :
Si vous mettez un ISA sous contrôle de disposition, les éléments assignés à l'ISA ne peuvent pas être supprimés, à l'exception des dispositions.

Vous pouvez ajouter de nouveaux codes de statut dans Gestion de tables. Pour plus d'informations sur la gestion des tables, reportez-vous à la section Gestion des codes.

 

Lorsque vous avez créé des ISA, vous pouvez leur attribuer une justification. Une justification est la raison de la création d'un ISA Vous pouvez également enlever une justification d'un ISA.

 

Note :
Vous pouvez uniquement assigner des justifications à un ISA si votre administrateur vous y a autorisé ; s'il n'a pas activé cette option, vous serez uniquement en mesure d'assigner des ISA aux classements RM.

 

Vous pouvez également afficher les informations d'approbation d'un ISA si votre administrateur a activé cette fonctionnalité. Dans le cas contraire, l'option Informations d'approbation ne s'affiche pas dans le menu Fonctions. Le processus d'approbation ISA oblige les utilisateurs à approuver un programme avant qu'il ne puisse être utilisé. Lorsqu'un programme est approuvé, les informations d'approbation sont enregistrées dans la base de données. Les informations sur les approbations ISA sont utiles pour les gestionnaires de documents et les utilisateurs qui souhaitent se renseigner sur les ISA. Par exemple, si une phase particulière a pour but de modifier l'état des éléments, mais que cette phase est paramétrée pour la destruction, et que cette configuration a été approuvée, un gestionnaire de document peut souhaiter savoir quels utilisateurs ont détruit ou approuvé la destruction des éléments gérés avant la date de destruction indiquée.

Lorsqu'un ou plusieurs programmes d'un ISA sont en attente d'approbation, l'icône En attente d'approbation s'affiche en regard du nom de l'ISA. Les ISA comportant des programmes en attente d'approbation peuvent être assignés à des classements ou appliqués à des documents gérés. Ils ne peuvent en revanche pas servir à effectuer des recherches de sorts finals.

 

Note :
Lorsque le processus d'approbation d'utilisation est désactivé, les programmes sont disponibles pour utilisation immédiate et ne nécessitent pas d'approbation. L'icône En attente d'approbation ne s'affiche pas pour les ISA lorsque la fonctionnalité Processus d'approbation est désactivée.

 

La page d'informations d'approbation ISA fournit les informations suivantes :

Tableau 1-5 : Informations d'approbation ISA

Informations d'approbation Description
Phase Nom de la phase affiché sur le programme. Si une même phase existe plusieurs fois pour un ISA, elle est traitée par le sort final même si les critères des instances restantes de cette phase sur un objet ne sont pas satisfaits. Les éléments ne doivent remplir qu'un des critères de la phase, de sorte que lorsque le critère est satisfait, l'élément est traité. Si un élément remplit les critères de plusieurs instances d'une même phase, il est renvoyé plusieurs fois dans la page Résultats du sort final et les éventuelles actions exécutées apparaissent.
Approuvé par L'utilisateur qui a approuvé le programme. L'utilisateur indiqué n'est pas forcément l'utilisateur qui a effectué l'action dans Content Server.
Date d'approbation La date à laquelle l'utilisateur a approuvé le programme.
Approuvé par L'utilisateur qui a approuvé le programme.
Date d'approbation La date à laquelle l'utilisateur a approuvé le programme.
Commentaire d'approbation Le dernier commentaire sur la page de mise à jour d'approbation.
Période max Si la case Période max est sélectionné alors qu'un programme est en cours de création ou de modification, une coche bleue est affichée dans la colonne Période max de la page d'informations sur l'approbation ISA.
