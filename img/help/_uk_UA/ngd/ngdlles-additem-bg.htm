The Add Item menu is available anywhere you have permission to add items. The options available to you when adding an item vary, depending on the item type. See the help topic for the specific item type for information about adding an item.

When adding items, keep in mind that permissions determine which items you can access. By default, any item that you add inherits its permissions from the container you put it in. For example, if a group has See Contents permission on a Folder, members of that group will have permission to see any Document you add to that Folder. However, it is possible to modify the permissions of individual items in a Folder. You become the Owner of the items you add, and you may have full permissions on those items. For more information about setting permissions, see Setting Permissions.

Providing Metadata

Depending on the configuration of your environment, you may be required to provide metadata, such as Category Attributes, about the items you are adding to a Content Server Folder or other type of container. In some instances, some metadata is automatically extracted from the item and appears in the boxes in the Add dialog box. Depending on your configuration, you may be able to edit these boxes.

If you are adding a group of items to a Content Server container, the Apply to all items with similar types check box may be visible. When you select this check box, the metadata you have specified for that item is applied to all subsequent items in the group that are of the same type.

In the Add dialog box, there are some parameters that are common to almost any type of item you add. These parameters, listed in the following table, may not be required to successfully add an item to a Content Server container.

Parameter Description
Name Content Server supplies a default name for any item you add. For a Document, for example, it uses the name of the file being added. However, providing a descriptive name in the Name box can add information about the item's content and purpose. It also makes it more likely to be correctly identified by a Search Query or a LiveReport.
Description By adding a description, you increase the chances that a Search Query or a LiveReport will correctly identify the item. Content Server indexes the text in the Description box as part of the item's metadata. The more metadata an item has, the better Content Server can identify its content and purpose.
Categories You use the Categories box to apply one or more attributes to an item that you add. For more information about working with Categories, see Using Categories.
Create In By default, an item is added to the current container. However, you can add the item to a different container.

The following example procedure illustrates the boxes you may encounter when adding an item to a Content Server container. You may see additional boxes depending on your configuration and which modules are available to you.

  1. Select the items that you want to add, and then drag them to the appropriate Content Server folder.

  2. On the Add page, do any of the following:

    • In the Description box, describe the item.

    • If available, select the appropriate Classification for the item.

    • Optionally, click Edit and enter the Category information.

    • Select the Apply to all items with similar types check box if you want to apply these values to all subsequent items in the group that are of the same type.

       

      Note:
      If you click Cancel when the items are being added to the container, the operation will stop and no further items will be added to the destination. Files that have already been added will not be deleted.

       

  3. Click OK.
