Using Enterprise Connect, you can access content that resides in Virtual Folders, and perform a number of tasks on that content. A banner just below the Enterprise Connect toolbar indicates when you are working in a Virtual Folder. For more information about creating, configuring, and working with Virtual Folders, see Using Virtual Folders.

 

Note:
Facets are not supported for this release of Enterprise Connect.

 
 

Tip:
You can go to the original location of an item in a Virtual Folder if your administrator has enabled the option. Right-click the item and then choose Go to Original Location.

 

The following sections provide an overview of the Virtual Folder functionality available in Enterprise Connect.

Virtual Folders

You can perform the following tasks when working with Virtual Folders in Enterprise Connect:

  • Browse Virtual Folder content in the Enterprise Connect tree.

  • Open and edit documents that reside in a Virtual Folder.

  • Delete and rename a Virtual Folder.

  • Preview content that resides in a Virtual Folder in the Preview Pane.

  • Add content to a real folder that resides in a Virtual Folder using the Add Item context menu action. The banner will indicate when you are accessing a Virtual Folder or its contents.

  • Create a desktop shortcut to a Virtual Folder.

     

    Note:
    Because of the dynamic nature of content that resides in Virtual Folders, a shortcut to a Virtual Folder or its contents may no longer be valid if the content changes.

     

  • Open and edit documents with the same name that appear in the same Virtual Folder.

     

    Tip:
    To distinguish between documents that have the same name, look in the Location column to see the name of the parent folder of the selected document.

     

Note the following exceptions when working with Virtual Folders in Enterprise Connect:

  • You cannot take Virtual Folders offline.

  • You cannot add content to a Virtual Folder, either by dragging or by using the Enterprise Connect Save As dialog box. The Add Item list or the Add Item context menu are available for Virtual Folders, but you can only add an Appearance, another Virtual Folder, or a Custom View to a Virtual Folder.

  • You cannot search the contents of a Virtual Folder.

Real Folders

When you access a real folder that resides in a Virtual Folder, you will notice the following behaviors:

  • You cannot take real folders or their subfolders offline from within the Virtual Folder. However, you can take the folders offline from their actual location in Content Server.

  • You can access the full context menu when you right-click a real folder.

  • You can add content to real folders that reside in a Virtual Folder.

Documents

When you are working with documents that reside in a Virtual Folder in Enterprise Connect, you will notice the following behaviors:

  • You can open read-only versions of documents, and you can open and edit documents through the Enterprise Connect Open dialog box.

  • You can save a new version of a document to a Virtual Folder using the Save As New Version functionality.

  • In a situation where there are two documents with the same file name visible in the List Pane, you can open and edit either one.

     

    Tip:
    To distinguish between files that have the same name, look in the Location column to see the name of the parent folder of the selected document.

     

  • You can delete and rename a document that resides in a Virtual Folder.

  • You can drag a document from a Virtual Folder to your desktop.

  • You can drag documents to real folders from a Virtual Folder.

  • You cannot take documents that reside in Virtual Folders offline.

  • You are presented with the same context menu actions when you right-click a document that resides in Virtual Folders as you see when using the Content Server Web UI.

  • You can view a document that resides in a Virtual Folder using the Enterprise Connect Viewer.

  • The name of the parent folder in which a document resides is displayed in the Location column in the List Pane.
