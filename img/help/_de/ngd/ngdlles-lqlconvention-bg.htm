Live Query Language -Konventionen definieren die Syntax einer komplexen Abfrage.

Eine Wortfolge ist eine feste Folge von einem oder mehreren Wörtern, die in doppelte Anführungszeichen (" ") gesetzt sind. Beispiel: "gut" und "besser oder schlechter". Bei Wortfolgen, die aus nur einem Wort bestehen, können Sie die doppelten Anführungszeichen weglassen (z. B.: Reis).

Beim Erstellen von Wortfolgen gelten die folgenden Regeln:

  • Ein Abstandzeichen (z. B. Tabulator oder Leerzeichen) gibt das Ende eines Wortes an.

  • Zeichenfolgen mit Satzzeichen, jedoch ohne Leerzeichen, werden von Content Server als eine Wortfolge interpretiert (z. B. "www.opentext.com").

  • Bei Wortfolgen, die aus einem Wort bestehen, können Sie die doppelten Anführungszeichen weglassen (z. B. Reis).

  • Bei reservierten Schlüsselwörtern dürfen die doppelten Anführungszeichen nicht weggelassen werden (z. B.: qlnear), wenn die Schlüsselwörter als Suchkriterien behandelt werden sollen. Wenn Sie z. B. in der Online-Hilfe nach Informationen zum Schlüsselwort qlnear suchen, müssen Sie als Abfrage "qlnear" angeben und das entsprechende Hilfe-Segment auswählen.

Als Platzhalter für Zeichen kann in Livelink-Abfragen das Sternchen (*) verwendet werden. Beispiel: Das Ende eines Wortes, das viele verschiedene Endungen hat, soll mit einem Sternchen dargestellt werden. Geben Sie Buch* ein, um nach Objekten zu suchen die die Wörter Buch, Buchhalter, Buchhaltung usw. enthalten.

Standardmäßig verarbeitet Content Server Abfragebegriffe von links nach rechts. Wenn Sie diese Reihenfolge außer Kraft setzen möchten, können Sie runde Klammern () verwenden. In diesem Fall wertet Content Server zuerst die Ausdrücke in den Klammern aus. Mit (Anbau ODER Bauernhof) UND (Weizen ODER Getreide) suchen Sie beispielsweise nach Objekten, die mindestens ein Vorkommen der Wörter Anbau oder Bauernhof und mindestens ein Vorkommen der Wörter Weizen oder Getreide enthalten.
