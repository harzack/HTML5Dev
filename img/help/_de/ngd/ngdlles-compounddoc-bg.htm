Ein zusammengesetztes Dokument ist ein Container, in dem mehrere Dokumente in der Reihenfolge des Eingangs gespeichert werden. Für das Verwalten von zusammengesetzten Dokumenten stehen im Windows-Explorer verschiedene Funktionen zur Verfügung. So können Sie z. B. eine Ausgabe oder eine Revision des Dokuments erstellen und die im zusammengesetzten Dokument enthaltenen Teildokumente neu organisieren.

Das Hinzufügen von Objekten zu zusammengesetzten Dokumenten erfolgt auf die gleiche Weise wie das Hinzufügen von Objekten zu anderen Containern. Zusammengesetzte Dokumente können aber lediglich die folgenden Objekttypen enthalten:

  • Dokumente

  • Text-Dokumente

  • E-Mails

  • Zusammengesetzte Dokumente

  • Verknüpfungen zu und Fassungen von Dokumenten, Text-Dokumenten und anderen zusammengesetzten Dokumenten

Jedes Objekt, das Sie einem zusammengesetzten Dokument hinzufügen, erhält entsprechend der Reihenfolge beim Hinzufügen eine Nummer. Sie können die Reihenfolge der Objekte aber jederzeit ändern. Zusammengesetzte Dokumente können zusätzlich ein so genanntes Hauptdokument enthalten. Welches Dokument zum Hauptdokument werden soll, können Sie beim Neuorganisieren der Objekte im zusammengesetzten Dokument festlegen. Das Hauptdokument kann ein Inhaltsverzeichnis des Buches, die Indexseite der Web-Site oder das Deckblatt eines Berichts sein. Das Hauptdokument ist immer das erste Element eines zusammengesetzten Dokuments und steht in dessen Struktur an oberster Stelle.

Zusammengesetzte Dokumente erfordern keine durchgehende Nummerierung, d. h., die Nummerierung 1, 17 und 339 ist gültig. Wenn Sie die Objekte neu organisieren, können Sie sie automatisch neu nummerieren lassen und so alle Lücken in der Abfolge schließen.

Zusammengesetzte Objekte in Content Server verfügen über eigene Arten von Versionen, die so genannten Ausgaben und Revisionen. Jede Ausgabe und Revision ist eine statische Version eines zusammengesetzten Dokuments, die zu einem bestimmten Zeitpunkt in der Vergangenheit erstellt wurde. Wenn Sie z. B. ein zusammengesetztes Dokument haben, das ein Software-Handbuch darstellt, können Sie eine Ausgabe erstellen, wenn das Handbuch veröffentlicht wird. Sie können am originalen zusammengesetzten Dokument weiterarbeiten, da Sie durch die Erstellung der Ausgabe gewissermaßen eine Sicherheitskopie des Handbuchs zum Zeitpunkt seiner Veröffentlichung angelegt haben. Ausgaben werden wie folgt nummeriert: 1.0, 2.0, 3.0 usw. Revisionen sind Ausgaben recht ähnlich, eignen sich aber vor allem dann, wenn Sie nur kleinere Änderungen an einem zusammengesetzten Dokument speichern möchten, ohne dazu gleich eine neue Ausgabe erstellen zu müssen. Die Revisionsnummerierung basiert auf der Ausgabennummerierung. So steht z. B. die Revisionsnummer "1.2" für die zweite Revision der ersten Ausgabe.

Bei langen bzw. komplexen zusammengesetzten Dokumenten ist es u. U. schwierig, die Gesamtstruktur zu verwalten, wenn immer nur der Inhalt derselben Strukturebene angezeigt wird. Deshalb können Sie sich die Gliederung von zusammengesetzten Dokumenten anzeigen lassen. Aus der Gliederung, die auf einer Seite dargestellt wird, geht die gesamte Struktur der zusammengesetzten Dokumente hervor. Sie können ihr die Reihenfolge und die hierarchischen Beziehungen der Dokumente untereinander entnehmen. Darüber hinaus enthält die Gliederung auch für jedes Objekt innerhalb des zusammengesetzten Dokuments ein Menü Funktionen, über das Sie mit den einzelnen Objekten arbeiten können. So können Sie z. B. ein Dokument öffnen, um sich dessen Inhalt anzusehen.
