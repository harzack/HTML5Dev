Sie können den Befehl Kopieren/Verschieben in Enterprise Connect verwenden, um einzelne oder mehrere Objekte zwischen zwei Ablageorten innerhalb desselben Content Server-Repositorys zu verschieben. Wenn Sie ein Dokument oder ein Objekt, das kein Container ist, verschieben, können Sie die Standard-Verschiebebefehle von Enterprise Connect verwenden, wie z. B. das Ziehen mit dem Mauszeiger. Wenn Sie einen Container wie z. B. einen Ordner verschieben, können Sie über das Kontextmenü auf den Befehl Kopieren/Verschieben in Enterprise Connect zugreifen.

 

Tipp:
In Windows Explorer können Sie Dokumente und Ordner auch mit der rechten Maustaste verschieben. Wählen Sie das Objekt, das Sie verschieben möchten, ziehen Sie die Objekte mit der rechten Maustaste zum Ziel, und klicken Sie auf Verschieben. Diese Aktion ist verfügbar, wenn Sie Objekte aus einem lokalen Ordner in Enterprise Connect, zwischen zwei Ablageorten im selben Content Server-Repository und von einem Enterprise Connect-Suchergebnis in einen Ordner im selben Content Server-Repository oder einem anderen Plug-in ziehen.

 

Wenn Sie Objekte an einen neuen Ablageort verschieben, werden Sie möglicherweise aufgefordert anzugeben, ob die ursprünglich zugewiesenen Kategorien der Objekte behalten, die dem Zielverzeichnis zugewiesenen Kategorien übernommen oder die Kategorien von ursprünglichem Objekt und Zielobjekt übernommen werden sollen. Je nach Konfiguration Ihres Systems müssen Sie möglicherweise auch festlegen, ob die Objekte eine eventuell zugewiesene OpenText Records Management-Klassifikationen übernehmen sollen.

Ist im Zielverzeichnis bereits ein Objekt mit demselben Dateinamen vorhanden, müssen Sie möglicherweise festlegen, wie der Namenskonflikt behoben werden soll. Je nach Konfiguration sind die folgenden Optionen verfügbar:

  • Sie können angeben, dass das aktuelle Objekt mit einem automatisch generierten Dateinamen in das Ziel verschoben werden soll. Dadurch werden alle mit dem Objekt verbundenen Versionen, Protokolle und Metadaten beibehalten. Wenn Sie mehrere Objekte verschieben, können Sie bestimmen, dass sämtliche Namenskonflikte auf diese Weise behoben werden sollen.

     

    Tipp:
    Wenn Sie über die entsprechenden Berechtigungen verfügen, können Sie die Datei nach Abschluss des Verschiebevorgangs umbenennen.

     

  • Sie können festlegen, dass das aktuelle Objekt nicht in das Zielverzeichnis verschoben werden soll. Wenn Sie mehrere Objekte verschieben, können Sie darüber hinaus angeben, dass keine doppelten Objekte im Batch verschoben werden sollen.

  • Sie können bestimmen, dass die Verschiebeaktion nicht fortgesetzt werden soll. Objekte, die bereits erfolgreich verschoben wurden, verbleiben im Zielverzeichnis.

Sie können einzelne oder mehrere Instanzen der folgenden Objekttypen, die keine Dokumente bzw. Container sind, in einen neuen Ablageort im Content Server-Repository verschieben:

  • Benutzerdefinierte Ansichten

  • Umfragen

  • Verknüpfungen

  • URLs

  • Workflow-Diagramm

  • Dokumente

  • Text-Dokumente

  • Von Content Server-Modulen zur Verfügung gestellte Untertypen, wie z. B. Wiki-Seiten.

 

Hinweise:

  • Über das Kontextmenü können Sie Ordner bzw. ordnerähnliche Content Server-Objekte wie z. B. E-Mail-Ordner, Projekte, zusammengesetzte Dokumente, Workflow-Status, Aufgabenlisten, Nachrichtenkanäle oder Meilensteine verschieben.

  • Diese Funktion steht Ihnen nur zur Verfügung, wenn Sie über die entsprechenden Berechtigungen verfügen. Weitere Informationen zum Festlegen von Berechtigungen finden Sie unter Berechtigungen festlegen.

 
 

Tipp:
Weitere Informationen zum Verschieben von Objekten mithilfe von Enterprise Connect sowie zu den verfügbaren Befehlen finden Sie unter OpenText Enterprise Connect Plug-in for Content Server - Benutzerhandbuch (NGDCORE-UGD).
