Eine benutzerdefinierte Ansicht ist ein Objekttyp, der HTML-Code enthält. Wenn Sie einem Arbeitsbereich, einem Ordner, einem Projekt, einer Aufgabenliste oder einem zusammengesetzten Dokument eine benutzerdefinierte Ansicht hinzufügen, zeigt Content Server diesen HTML-Code als Teil der Oberfläche an. Die benutzerdefinierte Ansicht wird zwischen den Navigationsmenüs und den Objekten im Container bzw. Arbeitsbereich eingefügt.

Der erste Schritt beim Hinzufügen einer benutzerdefinierten Ansicht besteht in der Erstellung der HTML-Datei, die die benutzerdefinierte Ansicht zusammen mit anderen Dateien, wie Bilddateien oder Stylesheets, definiert. Benutzerdefinierte Ansichten können standardmäßig folgende Elemente enthalten:

  • HTML, JavaScript, VBScript, Java-Applets bzw. jeden anderen Code, der von Web-Browsern interpretiert werden kann. Ihr Administrator kann jedoch die Möglichkeit, bestimmte Inhalte hinzuzufügen, einschränken.

  • Verknüpfungen zu anderen Objekten in Content Server oder zu anderen Web-Seiten.

  • Verweise auf Bild-, Audio- oder andere Dateien, die entweder in Content Server oder in einem Verzeichnis auf einem anderen verfügbaren Web-Server gespeichert sind.

Da der Inhalt einer benutzerdefinierten Ansicht in die HTML-Datei eingefügt ist, die die Oberfläche bereitstellt, ignorieren Webbrowser Tags wie <TITLE>, <BODY> oder <FRAMESET>. Daraus ergibt sich, dass es nur eine eingeschränkte Zahl von Codetypen gibt, die verwendet werden können. So können in einer benutzerdefinierten Ansicht z. B. keine Frames erstellt werden.

 

Hinweis:
Sie können dort, wo Sie die benutzerdefinierte Ansicht gespeichert haben, auch einen Ressourcenordner hinzufügen, in dem Bilder und andere Dateien gespeichert werden, auf die die benutzerdefinierte Ansicht zurückgreift. Um ein unbeabsichtigtes Löschen dieser Dateien zu verhindern und den Arbeitsbereich so übersichtlich wie möglich zu halten, empfiehlt es sich, die benutzerdefinierte Ansicht und den Ressourcenordner zu verbergen.

 

Das Erstellen und Verwalten von benutzerdefinierten Ansichten ist im Normalfall nur bestimmten Benutzern oder Gruppen gestattet. Wenn Sie eine benutzerdefinierte Ansicht hinzufügen möchten, müssen Sie über die Berechtigung zum Erstellen von Objekten für benutzerdefinierte Ansichten sowie über die Berechtigungen Objekte hinzufügen und Ändern für die entsprechenden Container verfügen. Für das Ändern von Unterobjekten, die sich auf die benutzerdefinierte Ansicht auswirken können, benötigen Sie jedoch keine spezielle Berechtigung. Um eine von einem anderen Benutzer hinzugefügte benutzerdefinierte Ansicht ändern zu können, brauchen Sie die Berechtigung Ändern für die benutzerdefinierte Ansicht.

Sie können durch Zuweisen von Berechtigungen für die benutzerdefinierte Ansicht festlegen, auf welche Benutzer oder Gruppen die jeweilige benutzerdefinierte Ansicht angewendet werden soll.
