OpenText Prospectors sind Content Server-Objekte, die Ihnen dabei helfen, Informationen zu suchen und zu verwenden. Sie funktionieren wie Filter, die in den vom Content Server indizierten Daten nach Übereinstimmungen mit Ihren Suchkriterien suchen. Prospectors laufen ununterbrochen und bieten damit genaue und stets aktuelle Ergebnisse.

Prospectors können alle Datenquellen überwachen, die vom Content Server indiziert werden, vom Content Server-Repository über lokale Verzeichnisse bis hin zu einer Lotus Notes-Datenbank. Weitere Informationen dazu, welche Datenquellen von Ihrem Content Server-System indiziert werden, erhalten Sie von Ihrem Administrator.

Sie können Prospectors erstellen, indem Sie mithilfe der Live Query Language (LQL) eine Abfrage erstellen, mit der der Typ der Informationen definiert wird, die Sie abrufen möchten.

Nach der Erstellung der Prospectors-Abfrage müssen Sie das Segment angeben, das der Prospector überwachen soll. Der Prospector führt dann die Abfrage aus, um zu überprüfen, ob es neue Objekte gibt, die dem Index hinzugefügt wurden und die die angegebenen Suchkriterien erfüllen. Wenn eine Übereinstimmung gefunden wurde, fügt der Prospector in seiner Ergebnisliste eine Verknüpfung zu diesem Objekt hinzu. Der Prospector aktualisiert daraufhin die Ergebnisliste jedes Mal, wenn dem Index Objekte hinzugefügt oder Objekte aus dem Index entfernt werden. Prospectors können nur solche Indizes überwachen, die vom Administrator dafür eingerichtet wurden.

Wenn Sie Prospectors erstellen, legen Sie auch die Ergebnislisten-Optionen fest. So müssen Sie angeben, wie viele Ergebnisse in welcher Reihenfolge angezeigt werden sollen. Sie können auch auswählen, ob der Content Server Zusammenfassungen der Objekte in der Ergebnisliste anzeigen oder ausblenden soll.

Prospectors können in den meisten Content Server-Ablageorten erstellt werden, wie z. B. im Arbeitsbereich, im Projekt-Arbeitsbereich und in Ihrem persönlichen Arbeitsbereich.
