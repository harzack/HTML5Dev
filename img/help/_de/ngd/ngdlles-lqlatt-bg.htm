Die folgende Syntax zeigt, wie Sie beim Erstellen komplexer Abfragen Attribute angeben:

<Attributsname>    : <Wortfolge>

<Attributsname> bezeichnet hierbei den Namen eines Indexattributs, das dem Attribut für die Abfrage zugewiesen ist. Jedem Attribut ist mindestens ein Attribute zugewiesen. Eine Liste der Attribute für ein bestimmtes Attribut erhalten Sie bei Ihrem Administrator.

Wenn Sie z. B. OTName : "version 1.0" eingeben, sucht Content Server nach allen Objekten, deren Namensattribut die Wortfolge Version 1.0 enthält.

Wenn Sie ein komplexeres Suchkriterium innerhalb eines Attributs angeben möchten, verwenden Sie die Schlüsselwörter der Live Query Language (LQL) und setzen den Ausdruck rechts vom Doppelpunkt in runde Klammern:

<Attributsname>    : <(LQL-Ausdruck)>

<Attributsname> bezeichnet hierbei den Namen eines Indexattributs, das dem Attribut für die Abfrage zugewiesen ist.

Wenn Sie z. B. OTModifyDate : (> 19990601) eingeben, sucht Content Server nach allen Objekten, die nach dem 1. Juni 1999 geändert wurden.

Das Festlegen von Abfrageattributen mit LQL ist für die Suche mit komplexen Abfragen von Bedeutung. Weitere Informationen zu komplexen Anfragen finden Sie unter "Mithilfe komplexer Abfragen suchen".
