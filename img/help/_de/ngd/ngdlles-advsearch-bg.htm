Im Dialogfeld Content Server durchsuchen können Sie Abfragen mit Schlüsselwörtern oder komplexe Abfragen mit Live Query Language (LGL) durchführen. Dieser Abschnitt enthält darüber hinaus Komponenten, mit denen Sie die Suche erweitern oder einschränken können. Durch die Auswahl eines Modifikators können Sie die Suche erweitern. Der Modifikator gibt an, welche verwandten Ausdrücke Sie in die Suche einbeziehen möchten. Für das Eingrenzen der Suche muss angegeben werden, ob im Inhalt oder in den Metadaten der Objekte oder in beiden gesucht werden soll. Inhalt umfasst die in Objekten enthaltenen Informationen, wie z. B. den Text eines Dokuments. Metadaten sind die Informationen, die zu jedem Content Server-Objekt gespeichert werden, wie z. B. das Datum, an dem ein Dokument hinzugefügt wurde. Außerdem können Sie Ihre Suche durch die Verwendung von booleschen Operatoren eingrenzen, um die Beziehungen zwischen den Suchkriterien in Ihrer Abfrage genau zu definieren.

Schlüsselwörter sind Wörter oder Wortfolgen, mit denen Sie nach Objekten suchen, die diese Wörter oder Wortgruppen enthalten. Wenn Sie z. B. die Schlüsselwörter Modell eines Geschäftsplans, festlegen, gibt Content Server eine Liste der Objekte zurück, die alle drei Wörter, eines der drei Wörter oder die exakte Wortfolge enthalten. Wenn Sie mindestens zwei Schlüsselwörter in Anführungszeichen einschließen, sucht Content Server die exakte Wortfolge in den Anführungszeichen.

Wenn Sie nach Objekten suchen möchten, die ähnliche Wörter wie die in der Abfrage angegebenen Wörter enthalten, können Sie die Suche erweitern. Mithilfe von Modifikatoren bestimmen Sie die Art der ähnlichen Begriffe, die in die Suche einbezogen werden. Einer der Modifikatoren gibt beispielsweise Objekte zurück, die Wörter enthalten, die ähnlich klingen wie das angegebene Schlüsselwort. Ein anderer Modifikator gibt die Objekte zurück, die Wörter enthalten, die von der Stammform des angegebenen Schlüsselworts abgeleitet sind. Wenn Sie z. B. nach Wörtern suchen, die mit der Stammform des Schlüsselworts Programm verwandt sind, erhalten Sie eine Liste der Objekte, die Wörter wie Programm, Programme, Programmierer und programmieren enthalten.

Wenn Sie Schlüsselwörter für einen Modifikator angeben, wird der Modifikator auf alle Wörter angewendet. Bei der Suche nach einer exakten Wortfolge gilt der Modifikator für die Wortfolge als Ganzes. In einer komplexen Abfrage wird der Modifikator auf alle Wörter dieser Abfrage angewendet, die keine LQL-Schlüsselwörter sind. Weitere Informationen zu LQL finden Sie unter Mithilfe komplexer Abfragen suchen.

Sie schränken eine Suche ein, indem Sie eine größere Anzahl an Kriterien für den Suchvorgang in Content Server angeben. Verwenden Sie beispielsweise boolesche Operatoren (UND, ODER, NICHT; diese einfache Logik beschreibt das Verhältnis der Schlüsselwörter zueinander), oder lassen Sie ausschließlich den Inhalt und/oder die Metadaten eines Objekts durchsuchen. Des Weiteren stehen die folgenden erweiterten Optionen für Suchkriterien zur Auswahl: Umfang, Systemattribute, benutzerdefinierte Attribute innerhalb einer Kategorie und XML-Typen. In der nachstehenden Tabelle werden die verfügbaren Suchkriterien beschrieben.

Kriterien Suche-Typ
In allem Hiermit durchsuchen Sie die Daten in allen Objekten, z. B. den Text von Dokumenten sowie die Metadaten, die Ihr Systemverwalter als durchsuchbar definiert hat.
Im Inhalt Hiermit durchsuchen Sie die Daten in den einzelnen Objekten, z. B. den Text von Dokumenten.
In den Metadaten Hiermit durchsuchen Sie die Informationen, die zu jedem Objekt gespeichert werden, z. B. das Datum, an dem ein Dokument hinzugefügt wurde. Wenn Sie in den Metadaten suchen, schränken Sie Ihre Suche auf die Metadaten-Informationen ein, die Ihr Administrator als durchsuchbar (d. h. als standardmäßig suchen gekennzeichnet) festgelegt hat. Dies kann alle mit einem Objekt verknüpften Metadaten umfassen, muss aber nicht.
Boolesche Operatoren Hiermit definieren Sie exakt das Verhältnis zwischen den Suchkriterien in der Abfrage. Beispiel: Die Schlüsselwörter lauten Plan und Test. Die Suche nach Plan UND Test gibt Objekte zurück, die sowohl Plan als auch Test enthalten; Plan ODER Test gibt Objekte zurück, die Planoder Testoder beide Optionen enthalten. Plan NICHT Test gibt Objekte zurück, die Plannicht jedoch Test enthalten.
Systemattribute Hiermit suchen Sie nach Objekten, die bestimmte Attribute besitzen. Sie können beispielsweise nach allen Objekten mit dem Attribut Erstellt von suchen, das für einen bestimmten Benutzer festgelegt ist.

Sie können für erweiterte Suchen in Enterprise Connect auch die persönlichen Vorlagen und Systemvorlagen verwenden, die Sie in Content Server erstellen und verwalten. Wenn Sie eine Vorlage in Enterprise Connect auswählen, werden die in der Vorlage angegebenen Suchkriterien und Formulare automatisch auf der Seite Suchen angezeigt. Weitere Informationen zum Arbeiten mit Vorlagen finden Sie unter Mit Suchformularen arbeiten.

 

Hinweis:
Wenn eine Vorlage ein Suchformular enthält, das nicht von Enterprise Connect unterstützt wird, wird diese Komponente nicht auf der Seite Suchen angezeigt. Derzeit werden gesamter Text, Systemattribute, Kategorien, E-Mail, Ablageort und physische Objekttypen unterstützt.
