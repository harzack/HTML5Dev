Sie können den Befehl Kopieren/Verschieben nach Enterprise Connect verwenden, um einzelne oder mehrere Objekte von einem Ablageort zu einem anderen innerhalb desselben Content Server-Repositorys zu kopieren. Wenn Sie ein Dokument oder ein Objekt kopieren, das kein Container ist, können Sie die Standard-Kopierbefehle von Enterprise Connect verwenden. Wenn Sie einen Container wie z. B. einen Ordner kopieren, können Sie über das Kontextmenü auf den Befehl Kopieren/Verschieben nach Enterprise Connect zugreifen.

 

Tipp:
Wenn Sie in Windows Explorer arbeiten, können Sie Dokumente und Ordner auch mithilfe der rechten Maustaste kopieren. Wählen Sie die zu kopierenden Objekte aus, ziehen Sie sie mithilfe der rechten Maustaste in das Zielverzeichnis, und klicken Sie dann auf Kopieren. Diese Aktion ist verfügbar, wenn Sie Objekte aus einem lokalen Ordner in Enterprise Connect, zwischen zwei Ablageorten im selben Content Server-Repository und von einem Enterprise Connect-Suchergebnis in einen Ordner im selben Content Server-Repository oder einem anderen Plug-in ziehen.

 

Wenn Sie ein Objekt an einen neuen Ablageort kopieren, müssen Sie möglicherweise angeben, ob die von Ihnen kopierten Objekte ihre ursprünglich zugewiesenen Kategorien behalten oder die dem Zielverzeichnis zugewiesenen Kategorien bzw. die ursprünglich zugewiesenen und die dem Zielverzeichnis zugewiesenen Kategorien übernehmen sollen. Je nach Konfiguration Ihres Systems müssen Sie möglicherweise auch festlegen, ob die Objekte eine eventuell zugewiesene OpenText Records Management-Klassifikationen übernehmen sollen.

Ist im Zielverzeichnis bereits ein Objekt mit demselben Dateinamen vorhanden, müssen Sie möglicherweise festlegen, wie der Namenskonflikt behoben werden soll. Je nach Konfiguration sind die folgenden Optionen verfügbar:

  • Sie können festlegen, dass das aktuelle Objekt mit einem automatisch erstellten Dateinamen in das Zielverzeichnis kopiert wird. Dadurch werden alle mit dem Objekt verbundenen Versionen, Protokolle und Metadaten beibehalten. Wenn Sie mehrere Objekte kopieren, können Sie bestimmen, dass sämtliche Namenskonflikte auf diese Weise behoben werden sollen.

     

    Tipp:
    Wenn Sie über die entsprechenden Berechtigungen verfügen, können Sie die Datei nach Abschluss des Kopiervorgangs umbenennen.

     

  • Sie können bestimmen, dass das aktuelle Objekt nicht in das Zielverzeichnis kopiert werden soll. Wenn Sie mehrere Objekte kopieren, können Sie außerdem bestimmen, dass keine Duplikate in dem Stapel kopiert werden.

  • Sie können bestimmen, dass die Kopieraktion nicht fortgesetzt werden soll. Objekte, die bereits erfolgreich kopiert wurden, verbleiben im Zielverzeichnis.

Sie können einzelne oder mehrere Instanzen der folgenden Objekttypen, die keine Dokumente bzw. Container sind, in einen neuen Ablageort im Content Server-Repository kopieren:

  • Benutzerdefinierte Ansichten

  • Umfragen

  • Verknüpfungen

  • URLs

  • Workflow-Diagramm

  • Dokumente

  • Text-Dokumente

  • Von Content Server-Modulen zur Verfügung gestellte Untertypen, wie z. B. Wiki-Seiten.

 

Hinweise:

  • Über das Kontextmenü können Sie Ordner bzw. ordnerähnliche Content Server-Objekte wie z. B. E-Mail-Ordner, Projekte, zusammengesetzte Dokumente, Workflow-Status, Aufgabenlisten, Nachrichtenkanäle oder Meilensteine kopieren.

  • Diese Funktion steht Ihnen nur zur Verfügung, wenn Sie über die entsprechenden Berechtigungen verfügen. Weitere Informationen zum Festlegen von Berechtigungen finden Sie unter Berechtigungen festlegen.

 
 

Tipp:
Weitere Informationen zum Kopieren von Objekten mithilfe von Enterprise Connect sowie zu den verfügbaren Befehlen finden Sie unter OpenText Enterprise Connect Plug-in for Content Server - Benutzerhandbuch (NGDCORE-UGD).
