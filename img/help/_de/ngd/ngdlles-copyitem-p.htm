So kopieren Sie ein Objekt:

  1. Wählen Sie die zu kopierenden Objekte, klicken Sie mit der rechten Maustaste darauf, und wählen Sie dann Kopieren/Verschieben nach Enterprise Connect.

  2. Navigieren Sie im Dialogfeld Enterprise Connect zum Zielverzeichnis, und klicken Sie auf Kopieren.

  3. Je nachdem, wie Ihr Systemverwalter Ihre Umgebung konfiguriert hat, werden Sie aufgefordert, einen der folgenden Schritte auszuführen:

    Zweck Vorgehensweise
    Kategorieoptionen definieren Führen Sie im Bereich Kategorien einen der folgenden Schritte aus:
    •

    Klicken Sie auf Original, wenn Sie die den Objekten ursprünglich zugewiesenen Kategorien beibehalten möchten. Die zugewiesenen Kategorien werden im Feld Kategorien angezeigt. Klicken Sie auf Bearbeiten, um die Kategorien gegebenenfalls zu bearbeiten.

     

    Hinweis:
    Wenn Sie mehrere Objekte ausgewählt haben und im Dialogfeld Kopieren auf Bearbeiten klicken, um die zugewiesenen Kategorien zu bearbeiten, wird das Dialogfeld Kopieren für das nächste Objekt angezeigt, in dem Sie die Kategorien für dieses Objekt bearbeiten können. Wenn Sie jedoch nicht auf Bearbeiten klicken, werden die von Ihnen angegebenen Kategorien auf alle verbleibenden ausgewählten Objekte angewendet.

     
    •

    Klicken Sie auf Ziel, wenn Sie Kategorien des neuen Ablageorts auf die Objekte anwenden möchten. Um die dem Ziel zugewiesenen Kategorien anzuzeigen, klicken Sie im Dialogfeld Kategorien bearbeiten auf Bearbeiten und anschließend auf Fertig. Sie können die Kategorien gegebenenfalls bearbeiten.

    •

    Klicken Sie auf Zusammengeführt, um die Kategorien sowohl des ursprünglichen als auch des neuen Ablageorts auf das Objekt oder die Objekte anzuwenden. Um die dem Ziel zugewiesenen Kategorien anzuzeigen, klicken Sie im Dialogfeld Kategorien bearbeiten auf Bearbeiten und anschließend auf Fertig. Sie können die Kategorien gegebenenfalls bearbeiten. Wenn Sie mehrere Objekte ausgewählt haben, wird für jedes Objekt das Dialogfeld Kopieren geöffnet, in dem die Kategorien bei Bedarf bearbeitet werden können.

    Festlegen (falls zutreffend), ob das Objekt RM-Klassifikationen übernehmen soll Führen Sie im Bereich RM-Klassifikationen einen der folgenden Schritte aus:
    •

    Klicken Sie auf Ja, wenn das Objekt die dem Zielverzeichnis zugewiesenen RM-Klassifikationen übernehmen soll.

    •

    Klicken Sie auf Nein, wenn das Objekt die dem Zielverzeichnis zugewiesenen RM-Klassifikationen nicht übernehmen soll.

  4. Ist im Zielverzeichnis bereits ein Objekt mit demselben Namen vorhanden, führen Sie im Dialogfeld Enterprise Connect einen der folgenden Schritte aus:

    • Klicken Sie auf Ja, wenn Sie im Zielverzeichnis eine neue Instanz des Objekts mit einem automatisch generierten Namen erstellen möchten.

    • Klicken Sie auf Ja zu allen, wenn Sie eventuelle Namenskonflikte mit anderen Objekten in dem zu kopierenden Stapel beheben möchten, indem Sie im Zielverzeichnis jeweils eine neue Instanz der Objekte mit einem automatisch generierten Namen erstellen.

    • Klicken Sie auf Nein, wenn das aktuelle Objekt nicht in das Zielverzeichnis kopiert werden soll. Der Kopiervorgang wird fortgesetzt, nachdem Sie diese Option gewählt haben.

    • Klicken Sie auf Nein für alle, wenn keine doppelten Objekte in das Zielverzeichnis kopiert werden sollen. Der Kopiervorgang wird fortgesetzt, nachdem Sie diese Option gewählt haben.

    • Klicken Sie auf Abbrechen, wenn der Kopiervorgang abgebrochen werden soll. Objekte, die erfolgreich kopiert wurden, verbleiben im Zielverzeichnis.

     

    Hinweis:
    Wenn Sie Objekte kopieren und mit Enterprise Connect in Outlook arbeiten, sind die Optionen Nein und Nein für alle wegen Funktionseinschränkungen in Outlook nicht verfügbar. Wenn Sie jedoch das Enterprise Connect-Dialogfeld Kopieren/Verschieben zum Kopieren von Objekten verwenden, stehen diese Optionen zur Verfügung. Diese Optionen sind auch im Enterprise Connect-SchnellZugriff-Bereich verfügbar.

     

  5. Das Dialogfeld Wird kopiert zeigt den Fortschritt des Verschieben-Befehls an.
