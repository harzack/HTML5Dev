The opentext.ini file, which is located in the <content_server_home>/config directory, is the main configuration file for Content Server. It also contains several settings that affect the behavior of Classifications. Occasionally, it may be necessary to change these settings by editing the opentext.ini file directly using the Content Server SDK Builder or a text editing program.

For information about the opentext.ini file, see Understanding the opentext.ini file.

The following settings can appear in the [Classification] section of the opentext.ini file. Some of these settings do not appear by default. For the settings that do not appear by default, Content Server assumes default values, as documented in Table 1-6.

Table 1-6: Classifications opentext.ini Settings

Setting Values Description
AllowAppearances true or false.
By default, this setting does not appear in the opentext.ini file, which is equivalent to AllowAppearances=true.
 Controls whether Appearances can be created in the Classifications Workspace. For more information about Appearances, see Working with Appearances.
AllowCustomViews true or false.
By default, this setting does not appear in the opentext.ini file, which is equivalent to AllowCustomViews=true.
 Controls whether Custom Views can be created in the Classifications Workspace. For more information about Custom Views, see Working with Custom Views.
SubTypes {objID,objID,etc...}.
By default, this setting does not appear in the opentext.ini file, which is equivalent to SubTypes={136,144,145,0}.
 

Note:
To make OpenText Enterprise Library object types classifiable, you must edit the SubTypes value. OpenText Enterprise Library subtypes that can be added are 823 (Folder), 824 (Item), 825 (Variant), and 826 (Compound).

 
 Contains the object IDs of the item types that are available as managed objects. For more information about managed objects, see Configuring Item Types as Classifiable.
DisplayNum An integer between 100 and 2000.
By default, this setting does not appear in the opentext.ini file, which is equivalent to DisplayNum=100.
 

Note:
If you set an integer that is less than 100 or greater than 2000, Content Server assumes the default value 100.

 
 Specifies the maximum number of items that are displayed when a user browses a Classification.
reextract true or false.
By default, this setting does not appear in the opentext.ini file, which is equivalent to reextract=true.
 

Note:
Configuring the reextract setting to true can slow down large Content Server installations.

 
When an item is added to Content Server or modified in some way, Content Server extracts the item. This process assigns metadata to the item, indexes the item so that it is searchable, and if you are using Intelligent Classifications, the process determines which assisted or automatic Classifications best match the item and then assigns or suggests those Classifications. To make the Classification data for each item searchable, whenever a Classification is assigned without matching it to assisted and automatic Classifications again, Content Server must automatically reextract the item.
The reextract setting determines whether Content Server reextracts items after they have been assigned a Classification. If this value is set to true, the Classification data for each item is searchable immediately after it is assigned a Classification. If this value is set to false, the Classification data for each item is not searchable until the item is extracted again.
maxClassifications An integer between 1 and 20.
By default, this setting does not appear in the opentext.ini file, which is equivalent to maxClassifications=5.
 

Important:
Do not change this setting unless instructed to do so by OpenText Customer Support. Higher values might slow down your system, and lower values might negatively affect the accuracy of the Intelligent Classification functionality.

 
 When an item is extracted, the Intelligent Classification functionality determines which assisted and automatic Classifications best match the item. The maxClassifications setting identifies the maximum number of Classifications that the Intelligent Classification functionality assigns to or suggests for an item when it is extracted.
maxImportAgentObjects An integer value.
The default value is 100.
 

Note:
A value of 0 restricts all objects from being classified. A value smaller than 0 disables the parameter.

 
 When processing automatic and assisted Classifications (Intelligent Classifications), the maxImportAgentObjects setting defines the maximum number of Classification IPool objects that can be processed during a single Consumer Five Minute Agent run.
maxLargeImportAgentObjects An integer value.
The default value is 5.
 

Note:
A value of 0 restricts all large objects from being classified. A value smaller than 0 disables the parameter.

The size of large Classification IPool objects is defined by the largeImportAgentObjectSizeKB parameter.

 
 When processing automatic and assisted Classifications (Intelligent Classifications), the maxLargeImportAgentObjects defines the maximum number of large Classification IPool objects that can be processed during a single Consumer Five Minute Agent run.
largeImportAgentObjectSizeKB An integer value.
The default value is 4000.
 When processing automatic and assisted Classifications (Intelligent Classifications), the largeImportAgentObjectSizeKB setting defines the size (in KB) to be used with the maxlargeImportAgentObjectSizeKB setting.
lowScoreThres An integer between 0 and 100.
By default, this setting does not appear in the opentext.ini file, which is equivalent to lowScoreThres=40.
 

Note:
After you change this setting, you must restart Content Server and regenerate the Classification Query File to apply the new setting to Classifications using the new default relevance score threshold. For more information about regenerating the Classification Query File, see Maintaining Classifications.

 
 When an item is extracted, the Intelligent Classification functionality compares the item to each assisted and automatic Classification and then determines relevance scores that represent how well the item satisfies each of the Classification's profiles. The lowScoreThres setting identifies the minimum relevance score necessary for the system to either suggest the item for an assisted Classification or assign an automatic Classification to the item.
siblingCompressRatio An integer between 0 and 100.
By default, this setting does not appear in the opentext.ini file, which is equivalent to siblingCompressRatio=75.
 

Note:
If you set an integer that is less than 0 or greater than 100, Content Server assumes the default value.

 
 When an item is extracted, the Intelligent Classification functionality may determine that several assisted and automatic Classifications within the same parent Classification match the item. The siblingCompressRatio setting determines the percentage of Classifications within a parent Classification that must match the item before the parent Classification is assigned to the item instead.
For example, if siblingCompressRatio is set to 75, lowScoreThres is set to 40, and three of the four assisted Classifications within the automatic Classification named Reports match a document with relevance scores greater than or equal to 40, Content Server assigns the Reports Classification to the document instead of the three assisted Classifications.
RetainIntelClass true or false.
By default, this setting does not appear in the opentext.ini file, which is equivalent to RetainIntelClass=true.
 Specifies whether all previous automatic and assisted Classifications assigned to Content Server items are retained when any new automatic and assisted Classifications are assigned.
If the value is set to true, items retain their previous automatic and assisted Classifications and are assigned new Classifications as well; the previous Classifications must be removed manually.
If the value is set to false, previous automatic and assisted Classifications are deleted, and items are assigned new Classifications.
highScoreBypassThres An integer between 0 and 100.
By default, this setting does not appear in the opentext.ini file, which is equivalent to highScoreBypassThres=70.
 Identifies the minimum relevance score necessary for an item to bypass the siblingCompressRatio setting.
For example, if highScoreBypassThres is set to 70, siblingCompressRatio is set to 75, maxClassifications is set to 5, and three of the four assisted Classifications within the automatic Classification named Reports match a document with relevance scores higher than 70, the parent Classification is not assigned to the item. Instead, Content Server ranks the three assisted Classifications along with other highly relevant Classifications, and then assigns the five most relevant Classifications to the item.
bulkOperations true or false.
By default, this setting does not appear in the opentext.ini file, which is equivalent to bulkOperations=false.
 Enables the multiselect functionality for classified items. Setting this parameter to true allows users to select more than one Content Server item to copy, move, delete, (or collect, if enabled).
The multiselect functionality does not have any Classifications-specific features; therefore, the commands are applied directly to the Content Server item and not the Classification.
bulkClassDisplayNum An integer greater than or equal to 1.
By default, this setting does not appear in the opentext.ini file, which is equivalent to bulkClassDisplayNum=2.
Identifies the maximum number of Classifications that can be displayed for an item on the following pages in Content Server:
•

Classify Search Results page when the page is displaying Classifications that are assigned to one or more items. You use this page when you classify items from the Search Result page.

•

Classify Collection Items page when the page is displaying Classifications that are assigned to one or more items. You use this page when you classify items in a Collection.

If the bulkClassDisplayNum setting is set to a number of Classifications that is less than the number that is assigned to an item, a view all link is displayed. Clicking this link displays all Classifications that are assigned to the item. If assigned Classifications are displayed for multiple items, you can click a link at the top of the page to show all Classifications assigned to all items.
inheritedClassDisplayNum An integer greater than or equal to 0.
By default, this setting does not appear in the opentext.ini file, which is equivalent to inheritedClassDisplayNum=4.
 Identifies the maximum number of Classifications that can be displayed on an item's Add page when the item is inheriting Classifications. An item inherits Classifications when you add it to a container that is classified.
For example, when you add a Document to a Folder with the Classification Confidential, the Document inherits that Classification. If the inheritedClassDisplayNum setting is set to a number of Classifications that is less than the number that the item is inheriting, a view all link is displayed. Clicking this link displays all Classifications that the item is inheriting.
maxPathDisplayLen An integer greater than 2.
By default, this setting does not appear in the opentext.ini file, which is equivalent to maxPathDisplayLen=70.
Identifies the maximum number of characters that can be displayed in the path that represents a hierarchy of Classifications on the following pages in Content Server:
•

An item's Add page when the item is inheriting Classifications. An item inherits Classifications when you add it to a container that is classified.

•

Classify Search Results page when the page is displaying Classifications that are assigned to one or more items. You use this page when you classify items from the Search Result page.

•

Classify Collection Items page when the page is displaying Classifications that are assigned to one or more items. You use this page when you classify items in a Collection.

The hierarchy of the inherited Classifications is displayed as a horizontal path in the following format: Confidential:Presentations:FinancialResults, and so on.
If you configure the maxPathDisplayLensetting to a number of characters that is less than the number of characters in a Classification hierarchy path, the path is truncated to the number of characters that you specify. Characters in the middle of the path are the characters that are not displayed, and are represented with an ellipsis (...).
searchResultPathLinks true or false.
By default, this setting does not appear in the opentext.ini file, which is equivalent to searchResultPathLinks=true.
 Controls whether an alternate, non-hyperlinked, colon-delimited path is displayed. This option is enabled by setting the searchResultPathLinks parameter to false.
By default, the paths display as a sequence of hyperlinked locations from the root of the Classification Tree to the Classification itself.
AssignRemoveClassificationPerm
•

PSee – users have see permissions.

•

PSeeContents – users have see content permission.

•

PModify – users have modify permissions (default is set to PModify).

•

PEditAttributes – users have edit attribute permissions.

•

PDelete – users have delete permissions.

•

PEditPerms – users have edit permissions.

By default, this setting does not appear in the opentext.ini file, which is equivalent to AssignRemoveClassificationPerm=PModify.
Users must have permissions to assign or remove Classifications on the object. The permissions required to perform these operations are set by this parameter.
applyDownBlockSize An integer greater than 1.
By default, this setting does not appear in the opentext.ini file, which is equivalent to applyDownBlockSize=50.
 Sub-items are processed in blocks of set sizes. This parameter allows you to adjust the number of items processed in each request.
