The Content Server XML Export API feature enables the information associated with an Content Server volume or object to be translated into XML code and exported as a file that can be used by another application, such as a data archive. When Classifications is installed, you can use the XML Export API to export, in XML format, the information associated with Classifications; for example, the Classification hierarchies in the Classification Workspace and Content Server items that have Classifications assigned to them. The XML Export API gives you control over the structure of the XML files that Content Server generates, and allows you to create custom skins for the Classifications interface. For more information about Content Server's XML Export API, see the Open Text Content Server - XML Import and Export Guide (LLESCOR-PGD), which is available on the OpenText Knowledge Center.

You perform an XML export by adding parameters to the end of the basic Content Server URL in a Web browser. An example of an XML export URL is:http://<content_server_URL>?func=ll&objId=<[DataID]>&objAction=xmlexport&<optionalparameters>, where:

  • <content_server_URL> represents the URL prefix of the Content Server server,

  • <[DataID]> represents the Content Server ID of the object to be exported (Classification Workspace, Classification Tree, Classification, Document, Folder, and so on),

  • and &<optional parameters> represents parameters that control the content of the resulting XML file.

The XML export query always starts with &xmlexport. Multiple parameters can be added and are connected with an ampersand (&).

You can use any of the standard XML Export API optional parameters when you use the XML Export API with Classifications. You can also use other parameters that are specific to Classifications. The tables below provide information about some standard XML Export API optional parameters (scope and callbackhandlername) and the parameters that are specific to Classifications (xmlclassifieditems and xmlclassificationstatus). For more information about other standard XML Export API optional parameters, see the Open Text Content Server - XML Import and Export Guide (LLESCOR-PGD).

Table 1-7 lists some optional parameters that you can use in an XML export.

 

Note:
When you use the XML Export API, you must have the See and See Contents permissions for a classification object (Classification Workspace, Classification Tree, or Classification) to be exported and all subitems within the object. In addition, you must have the system administration rights privilege to set certain optional parameters. Table 1-7 below also outlines which optional parameters require this privilege.

 

Table 1-7: Optional Parameters

Parameter Description Values
scope= Exports the child Classifications of a given classification object (Classification Workspace, Classification Tree, or Classification) according to the specified child level.
•

base: exports the object only.

•

sub: exports the object and all its children.

•

one: exports the object and its immediate children.

•

<n>, where <n> represents an integer greater than 0: exports the object and its immediate children up to <n> levels below the object.1 for one level below, 2 for two levels below, and so on.

xmlclassifieditems= Exports the classified items of a specific status that belong to a Classification Tree or Classification.
 

Note:
You must have the system administration rights privilege to specify the following statuses: pending, rejected, all.

 
•

true: exports all classified items that are visible in the interface; in other words, with “accepted” status.

•

accepted: exports the classified items that are visible in the interface.

•

pending: exports the items that have been assigned Classifications, but the items are not yet visible in the interface.

•

rejected: exports the classified items that have had their Classifications removed.

•

all: exports all classified items regardless of their status.

callbackhandlername= Name of the callback handler used. When you use the classifications value with the callbackhandlername parameter, any Classifications that have been assigned to classifiable objects are appended to the export of the objects classifications
xmlclassificationstatus= Appends Classifications with a specific status that have been assigned to classifiable objects to the export of the objects.
 

Note:
You must have the system administration rights privilege to specify the following statuses: pending, rejected, all.

 
•

accepted: exports the Classifications that are visible in the interface.

•

pending: exports the Classifications that have been assigned to objects, but the items are not yet visible in the interface.

•

rejected: exports the Classifications that have been removed from objects.

•

all: exports Classifications regardless of their status.

You can combine optional parameters to make an XML export more specific. Table 1-8 below provides examples of URLs containing optional parameters that you can combine.

 

Note:
If you combine optional parameters and Content Server returns unexpected results, contact OpenText Customer Support.

 

Table 1-8: XML Export Examples

URL Description
<content_server_URL>?func=ll&objId=4962&objAction=xmlexport&scope=sub&xmlclassifieditems=true Exports the contents of the Classification Workspace object with object ID 4962 and all associated classified items with “accepted” status
<content_server_URL>?func=ll&objId=2040&objAction=xmlexport&scope=base&xmlclassifieditems=all Exports a Classification with object ID 2040 and all associated classified items, regardless of their status
<content_server_URL>?func=ll&objId=2001&objAction=xmlexport&scope=3&xmlclassifieditems=pending Exports a Classification Tree with object ID 2001, three levels of child Classifications in the Classification Tree, and classified items with “pending” status
<content_server_URL>?func=ll&objId=2245&objAction=xmlexport&scope=one&callbackhandlername=classifications&xmlclassificationstatus=accepted Appends to the export of the object with object ID 2245 and its immediate children all Classifications with “accepted” status that are assigned to the object and its children
<content_server_URL>?func=ll&objId=2401&objAction=xmlexport&callbackhandlername=classifications&xmlclassifieditems=true Appends to the export of the object with object ID 2401 all Classifications that are assigned to the object. Also appended to the export are the classified items with “accepted” status that have been assigned the Classifications that are assigned to the object.
 

Note:
In this type of example (when the &callbackhandlername=classifications&xmlclassifieditems= parameters are used together), only one level of classified items of a Classification is exported to prevent successive export repetitions.

 
<content_server_URL>?func=ll&objId=1365&objAction=xmlexport&xmlclassifieditems=pending&callbackhandlername=classifications&xmlclassificationstatus=rejected Exports the Classification with object ID 1365 and all its child Classifications, and associated classified items with “pending” status. For each classified item, its Classifications with “rejected” status are exported.
