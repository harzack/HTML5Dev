The following table describes the error messages for the Document Conversion Service (DCS).

If these errors persist, contact OpenText Customer Support for additional information.

Table 1-28: Error Messages for the Document Conversion Service

Number Message Description Remedy
100 Unable to read configuration The DCS could not read the opentext.ini file supplied on the command line. Ensure that the opentext.ini file exists and that the DCS has sufficient access to read the file.
101 Unable to write to log file The DCS could not write to the specified logfile parameter. Ensure that the path specified in the Log File field on the Document Conversion Service tab on the Admin Server Properties page is valid.
Check that the DCS has sufficient access to write to the file.
104 Unable to bind to admin port The DCS could not bind to the specified Admin port. Ensure that the Admin port number specified in the Log File field on the Document Conversion Service tab on the Admin Server Properties page is a free socket port. To do this, click the Check port link. If the Admin port is not available, change the port number to a number that you know is free.
105 Admin port not specified The Admin port is a required setting that was not specified. Check that the Admin port is specified on the command line argument. For more information about setting command line arguments, see Command Line Arguments.
107 Missing DCS rules file The DCS rulesfile parameter is a required parameter that was not specified. Verify that the Rules File field on the Document Conversion Service tab on the Admin Server Properties page contains the following path:
rulesfile=Content Server_home/config/dcsviewrules.txt.
108 The DCS rules file could not be opened The specified rulesfile parameter does not exist or could not be read by the DCS. Verify that the Rules File field on the Document Conversion Service tab on the Admin Server Properties page contains the following path:
rulesfile=Content Server_home/config/dcsviewrules.txt.
Check that the DCS has sufficient access to read the file.
109 DCS rules file syntax error The DCS rulesfile parameter has been modified and contains invalid syntax. Return the rules file to the version that was shipped with your current Content Server install.
110 DCS rules file is invalid The DCS rulesfile parameter has been modified and contains illegal operations. Verify that the Rules File field on the Document Conversion Service tab on the Admin Server Properties page contains the following path:
rulesfile=Content Server_home/config/dcsviewrules.txt.
Check that the DCS has sufficient access to read the file.
111 Failed to load DCS library One of the runtime loaded libraries required by the DCS could not be found. The DCS log file should identify the name of the library that could not be loaded. For example, “ERROR: Failed to load the library dcssum.dll”.
Ensure that the Content Server_home/lib and Content Server_home/filters directories are included in the system library path environment variables. On Windows, Content Server_home/bin should be included in the PATH variable. On Solaris, Content Server_home/lib should be included in the LD_LIBRARY_PATH variable.
For more assistance, contact OpenText Customer Support.
112 DCS library is missing lib parameter One of the DCS libraries is missing the lib parameter in its INI section. The DCS log file should identify the name of the library that could not be loaded. For example, “ERROR: Failed to load the library dcssum.dll”.
Verify that the corresponding conversion filter page of the opentext.ini file specifies the name of the required library in the lib parameter.
Consult the corresponding section of the Understanding the opentext.ini file.
113 DCS library is invalid One of the DCS libraries does not conform to the DCS library interface. To identify which library is causing the problem, change the DCS log level to the maximum value output by configuring the following parameter in the [DCS] section of the opentext.ini file:
logLevel=4
After logging the activity of the DCS, contact OpenText Customer Support.
114 DCS library could not be initialized One of the DCS libraries encountered an error at initialization time. Ensure that the Content Server_home/config directory and the Content Server_home/filters directories are included in the system library path environment variables. On Windows, Content Server_home/bin should be included in the PATH variable. On Solaris, Content Server_home/lib should be included in the LD_LIBRARY_PATH variable.
If the problem persists, change the DCS log level to the maximum value output by changing the following parameter in the [DCS] section of the opentext.ini file:
logLevel=4
After logging the activity of the DCS, contact OpenText Customer Support.
117 DCS library could not be initialized One of the DCS libraries encountered an error at initialization time. Ensure that the Content Server_home/config and Content Server_home/filters directories are included in the system library path environment variables. On Windows, Content Server_home/bin should be included in the PATH variable. On Solaris, Content Server_home/lib should be included in the LD_LIBRARY_PATH variable.
If the problem persists, change the DCS log level to the maximum value output by changing the following parameter in the [DCS] section of the opentext.ini file:
logLevel=4
After logging the activity of the DCS, contact OpenText Customer Support.
118 Failed to load DCS request handler The DCS IPool library (dcsipool.dll on Windows) could not be loaded. Ensure that the Content Server_home/config and Content Server_home/filters directories are included in the system library path environment variables. On Windows,Content Server_home/bin should be included in the PATH variable. On Solaris, Content Server_home/lib should be included in the LD_LIBRARY_PATH variable.
If the problem persists, change the DCS log level to the maximum value output by changing the following parameter in the [DCS] section of the opentext.ini file:
logLevel=4
After logging the activity of the DCS, contact OpenText Customer Support.
