The Index Verification Status page displays statistics for the verification process currently running. The page is accessible from a link on the Configure Index Verification page, and is only available to Content Server administrators. If the “run once” mode for the verification has been selected and there is no current verification process running, information about the completed verification is displayed.

As a convenience, links are provided from the status page to the verification administration and report pages.

The verification status page is reached by a link at the bottom the Configure Index Verification page.

Table 1-20: Verification Status

Field Description
Verification The mode selected for how verification is set to run.
Started On Date and time.
Ended On Date and time.
Processing Data ID Represents the progress of the verification.
MaximumDataID Verification completes once the “processing data ID” value reaches this target.
Missing Number of objects missing from the index.
Stale Number of objects with incorrect Last modified dates.
Orphaned Number of objects in the index which should have been removed.
Total Errors Number of missing, stale, and orphaned objects.
 

Note:
The status of the index verification is displayed on the Search Index Verify Log page for one week after the verification completes.
