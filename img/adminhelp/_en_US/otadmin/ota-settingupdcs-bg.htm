The Document Conversion Service (DCS) is a process that converts documents from their native formats to HTML or raw text. Admin servers manage DCSs in Content Server DCS provides two services in Content Server: it converts items so that users can hit highlight or view them in Content Server, and it converts data as it passes through an indexing data flow. In data flows, the DCS is referred to as the Document Conversion process. When you set up the DCS, you determine how it operates for converting items for Content Server operations, such as hit highlighting and viewing. For more information about configuring the DCS as it operates for converting data in data flows, see Configuring Document Conversion Processes. Setting up the DCS involves enabling and configuring it, and monitoring its status.

Enabling the DCS

When you install or upgrade Content Server, you can enable the DCS for your default Admin server to manage. Anytime you add a new Admin server or configure an existing Admin server, you can also enable a DCS for each server to manage. Each Admin server can manage only one DCS. Enabling multiple DCSs allows you to distribute the document conversion load across many Admin servers, therefore improving the overall performance of your system. If an Admin server becomes unavailable, the DCS that it manages also becomes unavailable; however, if other Admin servers are available and manage a DCS, those DCSs can handle document conversion. You can disable a DCS at your site at any time; however, at least one DCS should be enabled so that document conversion can occur.

Configuring DCS Settings

After you enable DCSs, you can configure them. Each DCS has its own configuration file that reflects the settings that you configure for it.

 

Note:
The DCS configuration file is stored in the config directory of a primary Content Server installation.

 

When you enable a DCS, Content Server configures most DCS settings for you. You can change the settings at any time to customize how the DCS operates. An example of settings that Content Server configures automatically are port numbers. Content Server selects available ports for the DCS and configures the DCS port number and the Admin port number. The Admin port number is the number of the port on which the DCS listens for information from the Admin server; the DCS port number is the number of the port that Content Server operations use to communicate with the DCS. Other settings that Content Server may configure automatically include the log file, which is the location of the file that records logging information for the DCS you are configuring; the start directory and the filter path, which are the locations where document conversion filters are stored; and the rules file, which is the location where the document conversion rules file is stored. The document conversion rules file is a file that DCS uses to determine which conversion filters to use.

One setting that is not configured automatically and that you must specify is the temporary file directory. The temporary file directory is a temporary storage location that is required by conversion filters.

 

Note:
Content Server appends the Admin port number to log file names when it creates DCS log files. For example, the log file that corresponds to a DCS that listens on Admin port number 5868, has the file name dcs_5868.log. The start directory location and the filter path should match; however, on Windows operating systems, they must match.

 

Content Server also configures default values for the following settings:

  • Log level, which specifies which type of logging information is recorded (for example, 0 specifies that logging will not occur; 1 specifies that only errors and documents that could not be indexed are recorded; 2 specifies that errors and document conversion events, such as filter loading information are recorded; 3 specifies that errors, document conversion events, and performance statistics are recorded; and 4 specifies that errors, performance statistics, and IPool events are recorded)

  • Document conversion timeout, which specifies the maximum number of seconds that Content Server waits to display converted documents to users before returning an error

  • Number of threads, which specifies the number of document conversion requests that the DCS processes simultaneously

  • Queue size, which specifies the maximum number of documents that the DCS will allow to wait to be converted

 

Note:
The values that you specify for the number of threads and the queue size depend on variables at your Content Server site, including the number of users, number of computers dedicated to DCSs, and the amount of available system resources.

 

Monitoring DCS Status

Monitoring the status of a DCS allows you to maintain its performance. DCSs can have one of the following statuses:

  • Running, which indicates that the DCS is running

  • Not running, which indicates that the DCS is stopped and is not scheduled to run

  • Disabled, which indicates that the DCS has been disabled manually

  • Missing from an Admin server, which indicates that the Admin server that manages the DCS cannot locate the DCS's configuration. You should resynchronize the Admin server to re-create the DCS. For more information about resynchronizing Admin servers, see Resynchronizing Admin Servers

  • Returned an error, which indicates that a DCS error has occurred.

  • Returned an Admin server error, which indicates that an error occurred with the Admin server while it was retrieving the DCS process. In this case, the Admin server may have stopped running.

  • Unknown State, which indicates that the DCS is unable to communicate with the Admin server.
