To configure the JVM for search:

  1. On the administration page, click the Search JVM Configuration link.

  2. In the Relative Path to the JRE field, type the relative path to the Java Runtime Environment that OpenText Customer Support provided to you for use in Content Server.

  3. In the Additional Arguments field, type any JVM arguments you want to add.

  4. Click the Update button.

  5. Stop and start all search processes, and then resynchronize your Admin servers.

 

Note:
Some versions of the JRE, including JRE 6, Update 20, which ships with Content Server, allocate large amounts of memory initially. For lightly populated search partitions, up to 200 MB of memory allocation have been observed. This memory is consumed as partitions fill, and does not appear to affect larger partitions.
