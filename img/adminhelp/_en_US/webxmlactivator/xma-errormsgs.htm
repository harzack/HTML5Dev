The following table describes the error messages for XML Activator processes:

If these errors persist, contact OpenText Customer Support for additional information.

Table 1-26: Error Messages for XML Activators

Number Message Description Remedy
1 Content Server core error An internal error occurred when Content Server attempted to create or set parameters in the configuration file for an XML Activator process. Contact OpenText Customer Support.
2 Unable to read configuration parameter Content Server encountered an error while reading a parameter in the configuration file for the XML Activator process. Check the configuration file to ensure that all parameters have valid values. XML Activator configuration files are stored in the Content Server_home/config directory and have the .xml file extension. The name of a particular XML Activator configuration file is specified in the command line for the process. You can view the command line for a process on the Specific tab of its Properties page.
3 Unable to set configuration parameter Content Server encountered an error while setting a parameter in the configuration file for the XML Activator process. Check the values that you specified when creating or updating the XML Activator process. If you are updating an XML Activator process, you can check the values specified on the Specific tab of its Properties page.
4 Unable to connect to Admin server Content Server could not connect to the Admin server that manages the XML Activator process. Check the Admin server associated with the XML Activator process and ensure that it is available. If you are updating an existing XML Activator process, you can change the Admin server by clicking a value in the Admin Server drop-down list on the Specific tab of the XML Activator Properties page.
5 Unable to initialize the configuration file Content Server could not initialize the configuration file for the XML Activator process. Check your system to ensure that adequate memory resources are available.
6 Unable to find the configuration file Content Server could not find the configuration file for the XML Activator process. Check the configuration file's name to ensure that it matches the configuration file name that is specified in the command line for the XML Activator process. Ensure that the configuration file is not currently in use, and check the file's permissions to ensure that the XML Activator process can read data from it.
XML Activator configuration files are stored in the Content Server_home/config directory and have the .xml file extension. The name of a particular XML Activator configuration file is specified in the command line for the process. You can view the command line for a process on the Specific tab of its Properties page.
7 Unable to write data to the configuration file Content Server could not write data to the configuration file for the XML Activator process. Check the configuration file's permissions to ensure that the XML Activator process can write data to it. You can also check your system to ensure that adequate disk space is available and that the configuration file is not currently locked.
XML Activator configuration files are stored in the Content Server_home/config directory and have the .xml file extension. 30 Unable to set the read directory Content Server could not set the read directory for the XML Activator process. The read directory is the directory in which the process' read IPool is created. Ensure that you specified a valid directory path in the Read data from field when you created the XML Activator process. If you are updating an existing XML Activator process, you can change the read directory by setting a new value in the Read field, which you access by clicking the Information button on the Specific tab of its Properties page. Also, check the permissions on the directory to make sure that the XML Activator process can write data to it.
31 Unable to set read area Content Server could not set the read area for the XML Activator process. The read area is a directory within the read IPool directory. On the Specific tab of the XML Activator Properties page, the read area appears following the Read field, which you access by clicking the Information button on the Specific tab of the XML Activator Properties page. The Read field gives the location of the source IPool directory. The name of the read area will be a number (for example, /2645_0). Check the permissions on this directory to make sure that the XML Activator process can write data to it.

37 Unable to set the metadata list Content Server could not set the metadata list for the XML Activator process. Ensure that you specified a valid metadata list value in the Metadata List field when you created the XML Activator process. If you are updating an existing XML Activator process, you can change the metadata list by setting a new value in the Metadata List field on the Specific tab of the XML Activator Properties page.
The metadata list must include the XML tag name followed by the metadata tag name, which is the name of an existing region in Content Server. The tag names for each mapping must be separated by a comma and a space, and each mapping must be separated by a semi-colon. For example:
xName, OTName;xOwnerIDNumber, OTOwnerID;xFileLocation, OTLocation
Any spaces you include in the tag names will be ignored by Content Server. If more than one instance of a tag occurs in your XML files, or if two or more tags in an XML file are mapped to the same metadata tag, Content Server will index the content of the tags to several instances of the same region. In this case, users will not be able to search that region for this index; however, if Content Server users search the index as a whole, they can access this data.
