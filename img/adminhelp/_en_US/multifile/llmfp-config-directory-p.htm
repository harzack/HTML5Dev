To configure general settings:

  1. In the Multi-File Output Administration on the Content Server Administration page, click the General Settings link.

  2. On the General Settings page, enter the path to the location where you want compressed files added. By default, the path is OPENTEXT\temp\multifile.

     

    Note:
    The OPENTEXT\temp\multifile path is also used for multi-file print.

     

  3. In the SMTP Cache Lifetime field, enter the number of seconds you want the cache to be valid before Content Server attempts to obtain new settings. By default, this field is set to 300.

  4. Select the Enable SSL Accelerator at the Load Balancer check box to accelerate the sending/receiving of data for Web applications. By default, this check box in disabled.

  5. In the Address Limit area, type the maximum number of e-mail addresses allowed when sending e-mails in the Maximum E-mail Addresses. By default, this field is set to 10.

  6. In the Attachment Options section, click one of the following radio buttons:

    • Allow the user to choose, which allows the user to choose whether e-mail attachments are added as files or URLs. This option is selected by default.

    • Use files only, which enables e-mail attachments to be included as files only.

    • Use URLs only, which enables e-mail attachments to be included as URL references to the file location in Content Server.

  7. Enter the number of KB users are allowed to download, e-mail, or print (once a file has been compressed) in the Bandwidth Limiter fields. By default, all fields are set to 5120 KB.

  8. Select the objects which will have download, email, and print functionality applied in the Multifile Objects Configuration field. By default, only Documents and Folders have the download, email, and print functionality applied.

  9. Click Submit to save your setting.
