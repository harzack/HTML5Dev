When you configure community settings, you configure the behavior of all communities in Content Server. Table 1-1 describes the community settings that you can configure.

Table 1-1: Configurable Settings for Communities

Setting Description
Navigation Menu When enabled, every user's My Workspace page displays the communities Navigation Menu, giving quick access to their communities.
Personal Frontpage When enabled, the Personal Frontpage displays in the communities Navigation Menu and in the Personal menu.
Indirect User Functions When enabled, indirect user functions allow facilitators to choose whether certain community functions (such as email functions) will apply to indirect members and direct members, or to direct members only. Direct members of a community are users that are added individually, while its indirect members are users that are added to the community as part of a group.
Disable JavaScript Filtering When enabled,JavaScript Filtering for CoP Objects is disabled. CoP objects that use ckEditor are prone to JavaScript attacks. Hence CoP uses the Antisamy Library that ships with content server to filter such malicious code. If you do not want such filtering, please select the checkbox.
Public Member Info When enabled, this setting gives non-members access to the Member Info page of public communities.
State Overwrite When this setting is enabled, changing the community type also changes the corresponding Public Access permissions (See and See Items) for all sub-items. For example, changing a community type from public to private or unlisted removes the See and See Items permissions from the community's sub-items and makes them accessible to community members only.
Maximum Personal Photo Size This setting determines the maximum photo size, in bytes, that can be displayed in community functions, for example, in forum posts. The default size is 50,000 bytes. The ideal dimensions for the image are 80 by 80 pixels. A value of 0 disables the display of photos.
Calendar Appointments When this setting is enabled, all community members with the Add Items permission can add appointments. When disabled, only facilitators can add appointments.
Calendar View This setting configures the default calendar view for a community as either a calender view or list view, and allows you to set the default start and end time for the calendar.
Metrics When enabled, this setting allows facilitators to view community metrics in the community digest. Metrics are statistics that indicate the general robustness of the community.
Abbreviated Links Abbreviates long URL links with an ellipsis in the middle or at the end of the URL.
HTML E-Mail Footer This field specifies the footer for all HTML emails sent from communities.
Text E-Mail Footer This field specifies the footer for all text emails sent from communities.
Reportable Items Selected item types in this section appear in community reports, such as blogs.
