The community directory is a listing of communities and questions, organizing them into a hierarchy of directories. It also provides a means for users to target where their questions can be viewed, and determine who receives them through email, that is, the users who have subscribed to a particular directory.

You create and maintain the community directory by adding, deleting, renaming, and configuring directories and directory trees. A directory tree is a directory at the root level of the community directory. A directory can only be added to a directory tree and a directory tree must reside on the Community Directory page.

When you configure a directory, you can promote it to featured status to make it more prominent, list it in the same format as most directories, or hide it so that users cannot see it.

 

Warning
Deleting a directory or directory tree, and any directories it contains, will orphan all items that are associated only with that directory or directory tree and any directories it contains. An orphaned item is no longer associated with any directory or directory tree. If a question has been orphaned, it will no longer be accessible to any user. If another item type is orphaned, it can be reassigned a directory or directory tree from its Content Server container.

 

By default, the community directory includes a directory named Other.

Configuring the Community Directory

The Content Server administrator or users with the System Administration Rights privilege can configure the look and feel of the community directory. You configure the community directory by doing the following:

  • Selecting which content modules are active or inactive.

  • Sorting content modules between the left and right columns.

  • Choosing the column layout to make both columns of equal size, or to make one of them wider.

  • Adding one or more sidebars below the community directory's Navigation Menu and determining their content.
