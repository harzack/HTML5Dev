OpenText Communities provide users with more ways to collaborate in Content Server. A community is a container, maintained by a facilitator, that holds specialized information, including Content Server Items and customized Reports. A community is presented in a personalized, newspaper-like interface, which can be built around any focus, for example, a department, a particular subject, or a professional affiliation.

You can perform the following administration tasks for communities:

  • Configure system-wide settings for all communities.

  • Administer community templates, which enables you to create a new community using a default structure and existing content.

  • Configure rated items reports, which list the ratings that users have given to items created by the report's recipient. For more information, see Configuring Rated Items Reports.

  • Configure email restrictions for sending email from within communities.

  • Configure the community directory, which is a listing of all communities in Content Server and system-wide questions. Communities are organized into directory trees and directories by the Content Server administrator, according to concepts with which users choose to identify communities and questions. The community directory also features other community-related functions, for example, recently asked questions for the current directory.

 

Note:
Additional administration tasks in the Communities Administration area pertain to questions and answers. For information about these tasks, see Administering Questions and Answers.
