When a document is edited with an application directly through WebDAV, the application may save the document several times throughout the editing session. Every save that is initiated by this application creates a new version of the document in Content Server, which is usually undesirable because a potentially excessive number of documents are stored, and most are intermediate versions that the author does not want to retain. Version management limits the versions kept in WebDAV to the latest save in an editing session.
