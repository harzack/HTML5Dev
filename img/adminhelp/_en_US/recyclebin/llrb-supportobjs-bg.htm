Content Server Recycle Bin supports recycling and restoring the following objects that are installed by a default installation of Content Server.

 

Note:
Optional modules may introduce additional recyclable objects to Content Server. These objects are not noted below, but do appear on the Configure Recycle Bin page after you install an optional module.

 

Object Can be Recycled Can be Restored
Appearance Yes Yes
Category Yes Yes
Channel Yes Yes
Channel News Item (this object is not displayed on the configuration page) Only if parent Channel is Recycled Only if parent Channel is Restored
Channel News Item Attachment (this object is not displayed on the configuration page) Yes Yes
Collection Yes Yes
Compound Document Yes Yes
Compound Document (Single Doc) Yes Yes
Custom View Yes Yes
Custom View Template Yes Yes
Discussion Yes Yes
Discussion Topic (this object is not displayed on the configuration page) Only if parent Discussion is Recycled Only if parent Discussion is Restored
Discussion Topic Attachment (this object is not displayed on the configuration page) Only if parent Discussion is Recycled Only if parent Discussion is Recycled
Document (including Text Doc) Yes Yes

Task Attachment Yes Yes

Workflow Status Yes Yes
XML DTD Yes Yes

The following restrictions apply to the information in the above table:

  • You cannot recycle or delete an object that is reserved, locked, or has a generation.

  • You cannot recycle or delete a container object (Folder, Project, and so on) that contains an object that is reserved, locked, or has a generation.

  • News Items that have been configured as a news ticker are still displayed while in the Recycle Bin.

  • If a Form Template is deleted, any forms using the Template are disabled.

 

Note:
Objects that are recycled retain their links when they are in the Recycle Bin. Child objects within a container (Folders, Projects, etc.) do not retain their original locations.
