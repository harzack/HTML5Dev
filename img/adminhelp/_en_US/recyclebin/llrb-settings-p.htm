To configure Rules:

  1. Click the Configure Recycle Bin link in the Recycle Bin Administration section on the Content Server Administration page.

  2. For each object type, enable either Recycle Object or Standard Delete.

    If you enable Recycle Object, the object is moved to the Recycle Bin when it is deleted. Objects are still active until permanently removed from the system (although recycled Projects will not show in the Personal Projects list and recycled Favorites are removed). Objects are pseudo-deactivated if the access rights are appropriately removed (see below). Selecting Standard Delete will perform a standard Content Server delete on the entire object, regardless of whether the object is a container that contains object types that are to be recycled. By default, all object types are set to Standard Delete.

     

    Note:
    The number of items shown on this page can vary depending on the optional modules installed. For a complete list of supported objects refer to the table in Working with Supported Objects.

     
     

    Tip:
    You can toggle all of the objects at once by clicking Recycle All or Delete All.

     

  3. Restart Content Server after you make any changes to these settings. If Content Server is deployed in a cluster, restart each instance of Content Server.

To configure Menu and Access Restrictions:

  1. Click the Configure Recycle Bin link in the Recycle Bin Administration section on the Content Server Administration page.

  2. Enable Yes for each item on the Personal global menu that you want users to see. To enable users to access the Recycle Bin volume, you must enable at least one Recycle Bin menu item.

    • The Recycle Bin menu item takes users to the top level of the Recycle Bin.

    • The Recycle Bin – Today menu item takes users to the Recycle Bin folder for the current date.

  3. Select the group that has access to recycled items. The Access Restricted to menu enables the Administrator to select a permission rule to apply to recycled items and to child items of recycled container types. A brief description of each option is given in the table below.

     

    Note:
    Users with System Administration Rights can restore and delete objects from the Recycle Bin, regardless of the permission rule that you select.

     

    Permission Rule Description
    Admin Only All permissions are stripped from the item. Project groups (Guests, Members, and Coordinators) are emptied. The owner of the object is changed to Admin.
    Admin and Owner Only All permissions are stripped from the item. Project groups (Guests, Members, and Coordinators) are emptied.
    Original Permissions The item’s permissions are maintained after it is deleted.

     

    Note:
    Restart Content Server after you make any changes to these settings. If Content Server is deployed in a cluster, restart each instance of Content Server.
