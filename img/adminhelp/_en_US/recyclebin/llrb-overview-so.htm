OpenText Recycle Bin offers users who accidentally delete documents from Content Server the opportunity to recover them easily, without involving their Content Server administrator.

By default, Content Server includes the Document Undelete module, which allows for restoration of deleted documents by a Content Server administrator. The Recycle Bin introduces a volume where supported objects are moved upon deletion and can be retrieved by regular users.

The Recycle Bin package replaces the standard document delete operation. Recycle Bin moves documents that are “deleted” in Content Server to a Recycle Bin Volume. This volume is structured so that deleted items are moved to a folder that is named with the format: <year>:<month>:<day>.

Documents in the Recycle Bin can have their versions, permissions, and attributes maintained. Items can be restored using the Content Server interface by manually moving the document from the Recycle Bin back to its original location or another location.

The Recycle Bin Volume should be maintained on a regular basis. A typical policy is to permanently delete documents that have reached a predefined expiration date.
