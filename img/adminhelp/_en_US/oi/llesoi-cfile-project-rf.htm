project

Description

Use type="project" on the <node> tag, to create, update, or delete a Project.

Syntax

<node type="project" action="create"></node>

Usage

  1. Creating or updating a Project can include a number of subordinate tags: goals Tag, include_channel Tag, include_discussion Tag, include_participants Tag, include_tasklist Tag, initiatives Tag, mission Tag, objectives Tag, public_access Tag, roles Tag, startdate Tag, status Tag, and targetdate Tag. These subordinate tags are detailed below.

  2. Creating, updating, or deleting a Project can also require a number of other subordinate tags, such as the location tag. These other subordinate tags are detailed in Object Importer Tag Descriptions and Syntax.

Example

To create a Project called “Standard Project” in the “Documentation” Folder of the Enterprise Workspace:

<import>
  <node type="project" action="create">
    <location>Enterprise:Documentation</location>
    <title language="en">Standard Project</title>
    <goals>This project will be completed in three weeks.</goals>
    <include_channel>TRUE</include_channel>
    <include_discussion>TRUE</include_discussion>
    <include_participants>TRUE</include_participants>
    <include_tasklist>TRUE</include_tasklist>
    <initiatives clear="true"></initiatives>
    <mission>The mission is to deliver a quality product.</mission>
    <objectives>Resolve all outstanding customer issues.</objectives>
    <public_access>TRUE</public_access>
    <roles>
      <coordinator>jdoe</coordinator>
    </roles>
    <startdate>20110314</startdate>
    <status>Pending</status>
    <targetdate>20110401</targetdate>
  </node>
</import>

To view more project examples, see project Import Control File Examples.

goals Tag

The goals tag is used to create or update the goals for the project. This tag's syntax is: 

<goals>Here are some project goals.</goals>
<goals clear="true"></goals>

Usage Notes
1.

The goals tag cannot exist on its own, and must be used with the <node type="project" tag.

2.

The goals tag is not a required tag.

3.

The goals tag is optional when:

  • <node type="project" action="create">

  • <node type="project" action="update">

  • <node type="project" action="sync">
To clear the value, use the clear="true" attribute.

include_channel Tag

The include_channel tag is used to add a Channel to the Project. This tag's syntax is: 

<include_channel>TRUE</include_channel>

Usage Notes
1.

The include_channel tag cannot exist on its own, and must be used with the <node type="project" tag.

2.

The include_channel tag is not a required tag.

3.

The include_channel tag is optional when:

  • <node type="project" action="create">
Valid values are TRUE, which adds a Channel to the Project, and FALSE, which does not add a Channel to the Project. If this tag is not specified, no Channel will be added to the Project.

Usage Notes
1.

The public_access tag cannot exist on its own, and must be used with the <node type="project" tag.

The public_access tag is not a required tag.

3.

The public_access tag is optional when:

  • <node type="project" action="create">

roles Tag

The roles tag allows you to add additional users or groups to various project roles. This tag's syntax is: 

<roles>
  <coordinator>neil</coordinator>
  <member>geddy</member>
  <guest>alex</guest>
</roles>
<roles>
  <guest type="user">jdoe</guest>
</roles>
<roles>
  <guest type="group">DefaultGroup</guest>
</roles>

Usage Notes
1.

The roles tag cannot exist on its own, and must be used with the <node type="project" tag.

2.

The roles tag is not a required tag.

3.

The roles tag is optional when:

  • <node type="project" action="create">

  • <node type="project" action="update">

  • <node type="project" action="sync">
1.

You cannot remove users or groups from roles using Object Importer.

2.

Any user or group must exist in Content Server before you can reference it using the roles tag. Keep in mind that user and group names are case-sensitive.

3.

The guest subordinate tag of the role tag has an optional attribute: type. The only valid values for type are: user and group. If the type attribute is not specified on the guest tag, Object Importer assumes a value of user.

startdate Tag

The startdate tag allows you to add or update the start date for the Project. This tag's syntax is: 

<startdate>20100926</startdate>

Usage Notes
1.

The startdate tag cannot exist on its own, and must be used with the <node type="project" tag.

2.

The startdate tag is not a required tag.

3.

The startdate tag is optional when:

  • <node type="project" action="create">

  • <node type="project" action="update">

  • <node type="project" action="sync">
Dates must be in the format YYYYMMDD or YYYYMMDDHHMMSS, where HH refers to the 24-hour clock and is a value between 01 and 24. If YYYYMMDD is used, the time defaults to 000000.

status Tag

The status tag allows you to add or update the status of the Project. This tag's syntax is: 

<status>Caution</status>

Usage Notes
1.

The status tag cannot exist on its own, and must be used with the <node type="project" tag.

2.

The status tag is not a required tag.

3.

The status tag is optional when:

  • <node type="project" action="create">

  • <node type="project" action="update">

  • <node type="project" action="sync">
1.

Allowable values for the status tag are: Pending, OnTarget, Caution, or Critical.

2.

If the status tag is not specified during the creation of the project, the status will default to Pending.
