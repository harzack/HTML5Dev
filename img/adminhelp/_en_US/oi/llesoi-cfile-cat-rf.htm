category Tag

Description

The category tag specifies a Category name that is to be applied to an object.

Syntax

<category name="Content Server Categories:Test"></category>

Usage

  1. The category tag is not a required tag.

  2. The category tag is optional when:

    • <node type="<any_type>" action="create">

    • <node type="<any_type>" action="update">

    • <node type="<any_type>" action="sync">

  3. Creating or updating a Category can include a number of subordinate tags: attribute Tag, noninherit Tag, setattribute Tag, and subitems Tag. These subordinate tags are detailed below.

Notes

  1. The Category, with all referenced attributes, must exist in Content Server before you can use it. If the Category, and all referenced attributes, do not exist in Content Server, the Category will not be created and an error message will appear in the log file.

  2. If the Category has previously been applied to the object, directly or through inheritance, only the attribute data specified will be updated. If the Category has not been applied to the object, it will be added, and any attribute data will be updated.

    If the Category has changed in Content Server, the Category on the object must be “updated” before altering the category data using Object Importer. This can be done in the Categories tab for a given object through the standard Content Server interface.

  3. You can apply as many Categories as you want for any object using Object Importer. You cannot remove Categories from any object using Object Importer.

attribute Tag

The attribute tag is used to assign a value to an existing attribute. This tag's syntax is: 

<attribute name="Name">John Doe</attribute>
<attribute name ="xyz" clear="true"></attribute>

Usage Notes Example
1.

The attribute tag is not a required tag.

2.

The attribute tag cannot exist on its own, and must be contained within a category tag.

3.

Multiple line attribute tags are supported in the import control file.
1.

The number and syntax of any attribute's values depends on the values which have been set up, and applied to the attribute, in Content Server.

2.

If the attribute supports multiple values, these can be added by specifying multiple attribute tags in the import control file.

3.

To clear an attribute's value, use clear="true".

4.

Dates must be in the format YYYYMMDD or YYYYMMDDHHMMSS, where HH refers to the 24-hour clock and is a value between 01 and 24. If YYYYMMDD is used, the time defaults to 000000.

5.

When defining a check box attribute type, use TRUE for selected and FALSE for cleared.

6.

All values must be specified when updating a multi-value attribute, (MVA). For example, if an attribute contains five values before the update, and the import control file specifies three values, the MVA will only contain the three values after the update.

 
<import>
  <node type="document" action="update">
    <category name="Content Server Categories:Test">
      <attribute name="Name">John Doe</attribute>
      <attribute name="DateOfBirth">19650216</attribute>
    </category>
  </node>
</import>
<import>
  <node type="document" action="update">
    <category name="Content Server Categories:Test">
      <attribute name="Color">Red</attribute>
      <attribute name="Color">Green</attribute>
      <attribute name="Color">Blue</attribute>
    </category>
  </node>
</import>

noninherit Tag

The noninherit tag determines whether inheritance can be enabled or disabled on Categories. This tag's syntax is: 

<noninherit>FALSE</noninherit>

Usage Notes Example
1.

The noninherit tag is not a required tag.

2.

The noninherit tag cannot exist on its own, and must be contained within a category tag.
1.

The noninherit tag must be set to either TRUE or FALSE.

If the noninherit tag is set to TRUE, inheritance can be enabled on Categories. If the noninherit tag is set to FALSE, inheritance can be disabled on Categories.

2.

The default inheritance behavior is derived from the parent object.

 
<import>
  <node type="document" action="update">
    <category name="Content Server Categories:Test">
      <attribute name="Name">Jane Doe</attribute>
      <noninherit>FALSE</noninherit>
    </category>
  </node>
</import>

setattribute Tag

In addition to regular attributes, setattributes are also supported. The setattribute tag is used to assign a value to an existing setattribute. This tag's syntax is: 

<setattribute name="Fruit"></setattribute>

Usage Example
1.

The setattribute tag is not a required tag.

2.

The setattribute tag cannot exist on its own, and must be contained within a category tag.

3.

The setattribute tag can contain attributes.

 
<import>
  <node type="document" action="update">
    <category name="Content Server Categories:SetTest">
      <setattribute name="Fruit">
        <attribute name="Name">Orange</attribute>
        <attribute name="Color">Orange</attribute>
      </setattribute>
    </category>
  </node>
</import>

subitems Tag

The subitems tag is used to apply a Category, and its attribute values, recursively to all children of the current object. This tag's syntax is: 

<subitems>reapply</subitems>

Usage Notes Example
1.

The subitems tag is not a required tag.

2.

The subitems tag cannot exist on its own, and must be contained within a category tag.
The only valid value for the subitems tag is reapply.
 
<import>
  <node type="document" action="update">
    <category name="Content Server Categories:Test">
      <attribute name="Name">Jane Doe</attribute>
      <subitems>reapply</subitems>
    </category>
  </node>
</import>
