taskgroup

Description

Use type="taskgroup" on the node tag to create, update, or delete a Task Group.

Syntax

<node type="taskgroup" action="create"></node>

Usage

  1. Creating or updating a Task Group can include a subordinate tag, milestone Tag, which is detailed below.

  2. Creating, updating, or deleting a Task Group can also require a number of other subordinate tags, such as the location tag. These other subordinate tags are detailed in Object Importer Tag Descriptions and Syntax.

Example

To create a Task Group with a Milestone, called “New Task Group” and “New Milestone”, attached to the Task List, “New Task List”, in the Enterprise Workspace:

<import>
  <node type="taskgroup" action="create">
    <location>Enterprise:New Task List</location>
    <title language="en">New Task Group</title>
    <milestone>Enterprise:New Task List:New Milestone</milestone>
  </node>
</import>

milestone Tag

Use the milestone tag to assign a Milestone to a Task Group. This tag's syntax is: 

<milestone>Enterprise:New Task List:New Milestone</milestone>
<milestone clear="true"></milestone>

Usage Notes
1.

The milestone tag cannot exist on its own, and must be used with the <node type="taskgroup" tag.

2.

The milestone tag is not a required tag.

3.

The milestone tag is optional when:

  • <node type="taskgroup" action="create">

  • <node type="taskgroup" action="update">

  • <node type="taskgroup" action="sync">
1.

The Milestone must exist in Content Server before it can be assigned to a Task Group. If this tag is not specified during create, no milestone will be assigned to the Task Group.

2.

To unassign the milestone, use the attribute clear="true".
