Example 6-29: To create a “Documents” Folder in the “Projects” Folder of the Enterprise Workspace, giving the “Client” group See & See Contents permissions, while removing permissions from the public group: 

<import>
  <node type="folder" action="create">
    <location>Enterprise:Projects</location>
    <title language="en">Documents</title>
    <description language="en">This folder is to store documents</description>
    <acl group="Client" permissions="100000000"></acl>
    <acl standard="world" permissions="000000000"></acl>
  </node>
</import>

Example 6-30: To update an existing Folder called “Documents” located in the Enterprise Workspace, by adding a Nickname. In this example, the “Add Title to Location” check box is selected:

<import>
  <node type="folder" action="update">
    <location>Enterprise</location>
    <title language="en">Documents</title>
    <nickname>docs</nickname>
  </node>
</import>

Example 6-31: To update an existing Folder called “Documents” located in the Enterprise Workspace, by adding a Nickname. In this example, the “Add Title to Location” check box is cleared:

<import>
  <node type="folder" action="update">
    <location>Enterprise:Documents</location>
    <nickname>docs</nickname>
  </node>
</import>

Example 6-32: To update an existing Folder by removing permissions from the public group, when the “Add Title to Location” check box is selected:

    <title language="en">Documents</title>
    <acl standard="world" permissions="000000000"></acl>
  </node>
</import>

Example 6-33: To update an existing Folder by removing permissions from the public group, when the “Add Title to Location” check box is cleared:

<import>
  <node type="folder" action="update">
    <location>Enterprise:Projects:Documents</location>
    <acl standard="world" permissions="000000000"></acl>

Example 6-34: To rename an existing Folder from “Documents” to “My Documents”, when the “Add Title to Location” check box is cleared:

<import>
  <node type="folder" action="update">
    <location>Enterprise:Projects:Documents</location>
    <title language="en">My Documents</title>
  </node>
</import>
