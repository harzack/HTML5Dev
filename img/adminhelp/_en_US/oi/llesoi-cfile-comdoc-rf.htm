Description

Use type="compounddoc", or type="cd", on the <node> tag to create, update, or delete a Compound Document.

Syntax

<node type="compounddoc" action="create"></node>
<node type="cd" action="update"></node>

Usage

Creating, updating, or deleting a Compound Document can require subordinate tags, such as the location tag. These subordinate tags are detailed in Object Importer Tag Descriptions and Syntax.

Notes

While compounddoc and cd are interchangeable, OpenText recommends that you use compounddoc.

Example

To create a Compound Document called “Prototype”, stored in the “Temp” folder, and shown in the personal workspace of the Admin user:

<import>
  <node type="compounddoc" action="create">
    <location>Admin Home:Temp</location>
    <title language="en">Prototype</title>
  </node>
</import>

To view more compounddoc examples, see compounddoc Import Control File Examples.
