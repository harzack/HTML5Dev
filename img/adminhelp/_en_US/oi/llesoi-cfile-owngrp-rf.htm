Description

Use the ownergroup tag to specify a different owner group for the object without changing permissions.

Syntax

<ownergroup>DefaultGroup</ownergroup>

Usage

  1. The ownergroup tag is not a required tag.

  2. The ownergroup tag is optional when:

    • <node type="<any_type>" action="create">

    • <node type="<any_type>" action="update">

    • <node type="<any_type>" action="sync">

    The exception to this rule relates to type="project". See the Notes section below for further information.

Notes

  1. By default, the ownergroup tag is not applicable for Project objects, type="project". The ownergroup will not be updated on the Project.

  2. Any owner group must exist in the system before you can reference it. Group names are case-sensitive.

Example

<import>
  <node type="folder" action="update">
    <location>Enterprise</location>
    <title language="en">Documents</title>
    <ownergroup>DefaultGroup</ownergroup>
  </node>
</import>
