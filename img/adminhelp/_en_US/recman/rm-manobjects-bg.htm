Tracking Objects and Object Types

You can track RM Classifications, RSI status, additional rights, and Record Date changes. You can also track Holds, comments for removing Holds, Official Marks, RSI and code management, and creation, deletion, and modification of RSI Schedules and Holds. For more information about Event Auditing, see Administering Event Auditing.

 

Note:
If you track RSI or code management, changes made to an RSI or code are saved to the audit table. Because RSIs and codes are not objects, audits are performed on the Records Management Administration and RSI containers.

 

Configuring Managed Object Parameters

On the Managed Objects page, the following options are available for managed object types:

  • Inherit From, which allows child objects to inherit parent object RM Classifications

  • Downgrade Permissions, which ensures the downgrade of permissions if an object is marked as official

  • Use Titling, which allows you to apply titling to an object when the Titling feature is turned on. For more information, see Changing Records Management Default Settings.

  • User Hold, which enables you to specify which types of items can have user based Holds applied to them

  • Default RM Classification, which allows you to set the default RM Classification for a specific object type. Managed object types can have RM Classifications assigned to them and can be managed by any user with the proper permissions.

You can add additional Content Server object types to the Managed Objects page that do not appear by default on the Managed Objects page. The object types that appear on the Add Item menu are Content Server object types. When a Content Server module is installed, the module’s object types appear on the Add Item menu and can be added to the list of managed object types.

 

Important:
When you add a managed object type to the Managed Objects page, it does not get automatically managed. You must select the Managed check box for each object type that you want managed.
