Users need the Form Revisable Storage privilege to be able to select the SQL Table revision mechanism when they create a Form and select a template for which a database table exists. By default, the Form Revisable Storage privilege is unrestricted, meaning that all users can store their Forms' revision data directly in the database, provided that a table exists for the template that they select.

Consider restricting the Form Revisable Storage privilege if database space is a concern in your system. Note that even users who cannot themselves create Forms with SQL Table storage for revisions can be granted permission to use such Forms created by other users. Regardless of the Form Revisable Storage privilege, a user creating a Form always has the option to store intermediate Form data as versions of a Content Server node.
