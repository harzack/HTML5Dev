Security Clearance lets you set which objects support security clearance levels and supplemental markings if the DoD setting is set to Supplemental Markings and Security Clearance Levels. You can add, delete and edit supported object types.

Supplemental Markings and Security Clearance Level fields will appear on various pages in Content Server, if those object types are supported. For example, if Projects (202) is configured as a supported object type, the Supplemental Markings and Security Clearance Level fields appear on the Add Project page.

When you add a supported object type, or edit an existing one, you must select the Inherit From check box to enable child items in a container to inherit the Defined Rules of the parent object. For more information about Defined Rules, see Using Rules to Apply Supplemental Markings and Security Clearance Levels to Items.

 

Note:
If you enable the Supplemental Markings and Security Clearance Levels option in the Configuration Settings section, the supported objects table is pre-populated with default supported object types based on the modules that you have installed on your system. For information about DoD settings, see Configuring Security Clearance Settings.
