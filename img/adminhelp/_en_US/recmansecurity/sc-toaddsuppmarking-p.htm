To add a supplemental marking:

  1. On the Content Server Administration page, click the Security Clearance Levels and Supplemental Markings Management link in the Security Clearance Administration section.

  2. On the Security Clearance Levels and Supplemental Markings Management page, click Supplemental Marking in the Add Item menu.

  3. On the Add Security Clearance Levels and Supplemental Markings page, type a supplemental marking code in the Code field.

  4. Type a description of the supplemental marking in the Description field.

  5. Optional Click the Restricted Data Code radio button to disable users from entering information in the Declassify On and Downgrade On sections of the Set Security Clearance page.

  6. Optional Click the Formerly Restricted Data Code radio button to disable users from entering information in the Declassify On and Downgrade On sections of the Set Security Clearance page.

  7. In the User Session Parameter area, use the arrows to select User Session Parameters to be associated with Supplemental Marking.

     

    Note:
    You can click the View Details or Hide Details links to show or hide the User Session Parameters associated with this Supplemental Marking. If you toggle between the two options, any changes made to the metadata fields are maintained.

    If any of the values are selected, users with the Supplemental Marking must have at least one of the values passed with the user session to see the content with the Supplemental Marking.

     

  8. Click the Submit button.

 

Note:
If a DoD setting is enabled, and you click the Restricted Data Code or Formerly Restricted Data Code radio button, all changes in the Declassify On and Downgrade On fields on the Set Security Clearance page are disregarded.
