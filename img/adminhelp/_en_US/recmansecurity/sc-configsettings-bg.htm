Setting Security Clearance Officers Group

Security Clearance lets you set a Security Clearance Officers group. The Security Clearance Officers Group is an existing Content Server group. The members of this group can update security clearance levels on secure-type objects in Content Server.

Enabling Multiple Restricted Supplemental Marking

The Multiple Restricted Supplemental Marking setting, when enabled, allows users to set more than one Supplemental Marking as a restricted supplemental marking.

 

Note:
This setting is only available when the DoD setting is set to None.

 

Updating Box Contents

The Update Box's Contents setting specifies whether changes to security levels or supplemental markings of a container apply to box contents, as well as sub-items and classified items.

Enforcing Security Levels and Supplemental Markings for System Administrators

This option allows you to enforce Security Levels and Supplemental Markings for users that have the System Administration privilege enabled in their profile. Enabling this option ensures that users can see only items for which they have the proper Supplemental Markings and Security Clearance Levels.

When this option is disabled, users can see all existing codes, create new codes, and browse items without restriction; when the option is enabled, users can only see codes that are assigned to them and items for which they have the proper markings and security levels. Users also cannot create new codes and can only assign codes that are accessible to them and the other users.

 

Note:
The Administrator can still see all items, regardless of Supplemental Markings or Security Clearance Levels.

Only the Administrator can update these settings. Users with the System Administration privilege cannot remove the filtering settings.

 

Setting Security Clearance Actions in Browse View

You can configure which Multi-select buttons appear for users when they are in the Browse View of Content Server. When enabled, the buttons allow users to perform actions on multiple items at one time. The following buttons are available, and appear in the following order when enabled:

  • Update Security Clearance Level

  • Assign Supplemental Markings

  • Remove Supplemental Markings

 

Note:
The additional buttons appear next to the standard Copy, Move, and Delete buttons in Browse View.

The buttons also appear in the Actions list when performing operations on multiple items in a Collection or Search result. These buttons appear by default, regardless of which options are enabled for Browse View.

 

Setting Options to Inherit Security Levels and Supplemental Markings on Copy and Move

You can allow users to inherit the Security Level or Supplemental Markings on items when the items are moved or copied from the parent container to another location in Content Server. By default, the Hide the Option setting is enabled (set to No), which means the Inheritance options are displayed when users move or copy items. When Inheritance is enabled, you can set the default action that is applied to items when they are copied or moved.

 

Note:
If the Configuration Settings on the Configure system for Security Clearance Levels and/or Supplemental Markings page are set to DoD Chapter 2 or DoD Chapter 2 & 3, the Inheritance options cannot be configured.

 

The following default actions are available:

  • Supplemental Markings

    • Ignore, which does not update the Supplemental Marking when it is copied or moved

    • Merge, which merges the Supplemental Markings on the item being copied or moved with the Supplemental Marking of the new container

    • Replace, which replaces the Supplemental Marking on the item being copied or moved with the Supplemental Marking of the new container

  • Security Level

    • Ignore, which does not change the Security Level

    • Upgrade, which upgrades the Security Level of the item being copied or moved to the Security Level of the new container

If the user belongs to the Security Officers group, or has the Edit permission on the item, the Inherit Supplemental Markings and Security Level section is displayed on the Copy or Move page, which allows users to select an Inheritance option when they move or copy an item. The option the user chooses overrides the default Inheritance option specified by the Administrator.

Setting DoD Settings

DoD settings are requirements outlined in the DoD 5015.2 Standards document. Depending on the setting you enable, users can access certain fields relating to security clearance. If the supplemental markings and security levels setting is enabled, users can access supplemental markings and limited security clearance level settings. If you enable the DoD Supplemental Markings requirements outlined in chapter 2 of the document, users can work with supplemental markings. If you enable the DoD Supplemental Markings and Security Clearance requirements outline in chapter 2 and chapter 3 (previously 4) of the document, users can work with both supplemental markings and security clearance levels.

 

Note:
The Records Management module must be installed for the Chapter 2 & 3 (previously 4) settings to be enabled.

 

The Declassifying On Time Frame setting is a period of time, in years, used to verify that the time between a Declassify On date and a Publication Date for a record is less than the period of time specified in the Declassifying on Time Frame setting. If you change the Declassify on Timeframe value, the next time you access the Configure System for Security Clearance Levels and/or Supplemental Markings page, a Recalculate Declassify on Date button appears beside the Declassify On time Frame field. You can click this button to update all the records. When the recalculation is complete, you are presented with summary statistics of the recalculation, which include the potential number of records, records processed, skipped due to a manually assigned date, and skipped due to insufficient permissions. The Recalculate Declassify on Date button appears until all records are updated.
