User Session Parameter Codes are added and managed from the Security Clearance Administration pages. The User Session Parameter Codes are used with Supplemental Markings to provide additional permission filters.

When Supplemental Markings have User Session Parameters associated with them, the following rules apply:

  • If a User Session Parameter code is passed to the log in (program session), a Supplemental Marking with User Session Parameter codes assigned can only be assessed by users that have the appropriate Supplemental Markings and User Session Parameter codes assigned

  • If a User Session Parameter code is not assigned to a Supplemental Marking, users can see the content with the marking from anywhere in, as long as they have that marking.

  • If no codes are passed to the log in (program session), Supplemental Markings with User Session Parameter codes assigned are not satisfied, and users cannot see them. However, users can still see items marked with a Supplemental Marking that does not have User Session Parameter code assigned to it.
