The Access-Control-Allow-Origin setting specifies the hosts that are permitted to communicate with the Content Server Cluster Agent service. By default, this setting does not appear in the config.ini file. This is equivalent to Access-Control-Allow-Origin: * , which means that any host can communicate with the Content Server Cluster Agent service.

  • Syntax:

    Access-Control-Allow-Origin: <server.domain.com> or <ip_address>

  • Values:

    A host name or an IP address.

  • Example:

    Access-Control-Allow-Origin: *.mydomain.com
