The Cluster Settings set patch information for Cluster Management. On the Manage Cluster Settings page, you specify the following information:

  • Update URL sets the URL that is used for updating the system manifest. The default directory for Update URL are as follows:

    • Windows, http://mimage.opentext.com/support/ecm/contentserver/10.5/manifest_win.xml

    • Solaris 10, http://mimage.opentext.com/support/ecm/contentserver/10.5/manifest_sol10.xml

    • Solaris 11, http://mimage.opentext.com/support/ecm/contentserver/10.5/manifest_sol11.xml

    • Oracle Linux 6, http://mimage.opentext.com/support/ecm/contentserver/10.5/manifest_lnx6.xml

    • Red Hat 6, http://mimage.opentext.com/support/ecm/contentserver/10.5/manifest_lnx6.xml

  • Host is the Master System: the system template whose configuration is ultimately replicated to the other instances in the Content Server cluster.

  • Patch Staging Folder is the location on the Content Server instance where the new patches will be staged for the database. The default staging directory is located at <Content_Server_home>/clustermanagement/staging/.
