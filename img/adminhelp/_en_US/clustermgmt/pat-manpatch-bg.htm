The Managing Master System page lists available, staged, and installed hotfixes, and provides tools for downloading and installing hotfixes.

 

Note:
For information on how to reverse the effects of a hotfix deployment, see Rolling Back a Hotfix Deployment (Using the Manifest Recovery) for details.

 

After you click Manage Master System in the Cluster Management section of the Content Server administration page, Cluster Management runs an analysis of your Master System. If the system has been recently validated, you are offered the option of skipping this analysis. Once the analysis completes, an Analysis Report appears showing your current Update Level, Installed Patches, and Installed Optional Modules. Click Continue to display the Manage Master System Patches page.

The Manage Master System Patches Page

The Manage Master System Patches page has the following tabs:

  • Available Patches lists the hotfixes that are available on the OpenText Knowledge Center for download. The hotfixes that appear in this tab depend on the modules that you have installed on your Content Server instance.

  • Staged Patches lists the hotfixes that you have downloaded and placed in the Patch Staging Folder (by default, <Content_Server_home>/clustermanagement/staging/).

     

    Note:
    Hotfixes appear temporarily in the Patch Staging Folder before their content is moved to the Content Server database, so normally the Patch Staging Folder does not contain all of the hotfixes that are listed in the Staged Patches tab. It may contain no hotfixes at all.

     

  • Installed Patches lists the installed hotfixes, indicating their status as one of the following:

    Error

    A hotfix with a status of Error has not been correctly installed. It appears with a red background.

    Unknown

    Cluster Management assigns an Unknown status to hotfixes that do not appear on the Cluster Management manifest file. Hotfixes with a status of Unknown appear with a yellow background. Unknown hotfixes may include patches for third-party or custom modules that are not processed by OpenText.

    Hotfixes with a status of Unknown are not replicated to your Content Server when you use the Manage Cluster functions.

    Removal

    Hotfixes with a status of Removal appear with a yellow background. Cluster Management removes hotfixes that have a status of Removal the next time that you click Deploy on the Staged Patches tab.

    Until they are removed, Hotfixes with a status of Removal are replicated to your Content Server when you use the Manage Cluster functions.

    Verified

    Hotfixes with a status of Verified have been successfully installed.

On each of the tabs, information is provided under the following columns:

Install

This column appears only on the Staged Patches tab. When you enable the check box beside a hotfix and then click Deploy, the hotfix is installed on the Master System.

Unless a hotfix has a dependency, you can install hotfixes in any order. When there are dependencies, Cluster Management ensures that hotfixes are installed in the necessary order.

  • When a hotfix has a dependency on another hotfix, the required hotfix is installed first.

  • When a hotfix that modifies Cluster Management is selected for installation, it is always installed before any other hotfixes.

Name

The name of the hotfix file.

Description

A brief description of the hotfix. The description may include the change made by the hotfix, the issue resolved by the hotfix, and the issue number from the OpenText issue tracking system.

Updates

The code base that the hotfix applies to. If, for example Update 2014-03, Initial Release appears in this column, the hotfix is intended to be applied to either the initial release of the installed version of Content Server or to the installed version of Content Server with Update 2014-03 in place. If None appears in this column or the column is blank, the hotfix is not specific to a particular Content Server code base. It may, for example, be a hotfix that patches a module that is compatible with Content Server with any of various Updates in place.

Modules

The module that the hotfix patches. If None, the hotfix patches core Content Server.

Action

Links appear in this column that allow for various actions:

  • Remove appears only on the Staged Patches tab. Click Remove to delete a staged package.

     

    Tip:
    Use this feature to remove deprecated hotfixes from the Staged Patches tab.

     

  • Details appears on each of the tabs. Click Details to open a Patch Details page that provides information on a hotfix.

The Patch Details Page

The Patch Details page provides the following information on a hotfix:

Name

The name of the hotfix file.

Description

A brief description of the hotfix. The description may include the change made by the hotfix, the issue resolved by the hotfix, and the issue number from the OpenText issue tracking system.

Impacts

A description of the effect that the hotfix has on your overall Content Server deployment. If Database Schema, Content in the File Store, or Search Index appears in this field, it indicates that the named component is changed by applying the hotfix.

 

Important:
If any of the above items are listed in the Impacts field, it is particularly important to take a full backup of your system before applying the hotfix.

 
Modules

The module that the hotfix patches. If None, the hotfix patches core Content Server.

Supersedes

A different hotfix that the hotfix is intended to replace. The different hotfix is automatically removed when you install the current hotfix.

Superseded by

A different hotfix that is intended to replace the hotfix. Normally, you should download and install the different hotfix that appears in this field instead of using the current hotfix.

Dependents

Hotfixes that are listed in this field are required to be installed with the current hotfix.
