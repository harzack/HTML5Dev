OpenText Content Server Cluster Management manages the application of Content Server hotfixes, both on a single instance of Content Server and throughout a Content Server cluster. Cluster Management automatically handles hotfix dependencies and removes superseded hotfixes when a newer or more complete version of a hotfix is applied.

Cluster Management can be used to manage hotfixes on a standalone installation of Content Server, but it is designed with clusters in mind to manage Content Server patching simply and efficiently across a Content Server cluster.

Using Cluster Management to manage hotfixes in a Content Server cluster involves two main operations:

  1. You use Cluster Management to automatically download, stage, and install patches on a single instance of Content Server (called the “master system”).

  2. You use Cluster Management to automatically replicate the master system's hotfix configuration to all Content Server instances in a Content Server cluster.

If you need to roll back the application of a hotfix, Cluster Management provides features and information that facilitate the rollback operation.

In addition to its hotfix management features, Cluster Management Update Analysis allows you to preview the effects that a Content Server Update will have on your system so that you can remove obsolete patches and make appropriate backups before applying an Update.

To begin using Cluster Management, you must first ensure that your Cluster Settings are set properly. See Managing Cluster Settings.
