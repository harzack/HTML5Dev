Content Server events can be enabled for the WebReports object. Because a webreport reportview is treated like a normal Content Server document, all of the standard Content Server events can be enabled and used.

Auditing also needs to be enabled on the Specific tab of each webreport. The Run/Export Audits Enabled option is cleared by default and allows each webreport to have additional audit events enabled so that every RUN or EXPORT event will generate an entry in the AUDIT log. This is intended to provide granular control in order to stop the audit logs from overflowing.
