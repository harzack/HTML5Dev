The Define Special Components form will list all the WebReports that are part of the application.

The form can be used to:

  • Define a default launch component for the application
  • Define an initialize component for the application
  • Assign unique nicknames to WebReports that are part of the application
Default Launch Component

An application can be launched using the URL:
?func=csapps.launchapp&appname=<application name>
Calling this URL will run the WebReport defined as the default launch component.

Initialize Component

A WebReport defined as the initialize component will be run immediately after the application is installed.

Unique Nicknames

WebReports can be given unique nicknames within an application and run using the URL:
?func=csapps.launchapp&appname=<application name>&nickname=<unique nickname>
Form Usage

The form will list all the WebReports that are part of the application.

Click on the radio button to the left of a WebReport to define it as the default launch component for the application.

If you don't want to define a launch component for the application then click on the radio button next to "No Component selected for this function", which is the default option.

Click on the radio button to the left of a WebReport to define it as the default launch component for the application.

If you don't want to define a launch component for the application then click on the radio button next to "No Component selected for this function", which is the default option.

Enter unique nicknames in the field to the right of each WebReport.

The nicknames will be validated on submission of the form, and an error will be thrown if a nickname is not unique within the application. Note that nickname comparisons are case-insensitive.

Click the Submit button to submit the form.
Click the Reset button to clear any changes made since the page was loaded.
Click the Cancel button to cancel and return to the Content Server Application Management page.
