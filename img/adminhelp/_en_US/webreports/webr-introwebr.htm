The WebReports Administration section has the following settings and options:

  • Flush Cache: used to remove all compiled reportviews from the cache location.

  • Install the Requests.js Library: used to provide an interface to install the Requests Library and associated files to the JavaScript library location.

  • Manage Category Data Source Configuration: used to update the maximum number of categories and attribute display parameters.

  • Manage Destination Media types: used to configure the media types that can be used for WebReports output.

  • Manage Search Query Integration: used to manage the ability to invoke WebReports directly from the search screen.

  • Manage Tags and Sub-Tags: use to selectively disable WebReports tags and sub-tags.

  • User/Group WR Trigger Administration: used to determine which User and/or Group can trigger WebReports.

  • Manage WebReports Conversion: used to changed the sleep interval for the conversion agent and to set the input and output directories to manage PDF conversion.

  • WebReports Scheduling: used to changed the sleep interval for the schedule agent, and to enable, disable, or permanently delete individual schedules.

  • Manage WebReports Scripting: used to enable or disable scripting of individual WebReports.

  • Restricting WebReports Services: used to enable, disable or restrict the WebReports services feature.

  • WR Trigger Administration: used to determine which node types can trigger WebReports.

  • Miscellaneous WebReports Settings: used to set the values of miscellaneous keys in the WebReports section of the opentext.ini file.

  • License Key Functions: used to set or change the WebReports License Key and display licensing status.

  • Node Administration: used to identify and update reportviews using out-of-date syntax.

  • WebReports Sub-Tag Builder: used to re-build all subtags including any subtags that have been created in the subtags folder.
