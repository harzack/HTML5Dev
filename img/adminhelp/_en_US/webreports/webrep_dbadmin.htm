The WebReports Database Administration page is used to manage any supported WebReports database actions. These actions include permanently deleting the WebReports tables for a clean uninstall, and downgrading the database schema to the appropriate module version.

  • If you choose to Delete WebReports tables, the entire WebReports schema will be removed and following this you will be taken through the uninstall of the module. All of the WebReports tables - including the WebReports schedule table, stats table, trigger tables, and the Module.VersionInfo entry in the KINI table - will be permanently deleted from the database.


  • If you choose to Downgrade the schema to a 5.0.x level (only new tablse removed), only the new schema added after WebReports 5.0.x will be dropped and following this you will be taken through the uninstall of the module. All data valid in WebReports 5.0.x will be retained and you can then install WebReports 5.0.x. Specifically, the WEBREPORTSUGTRIGGERS table is deleted from the database, and the Module.VersionInfo entry in the KINI table is downgraded to 5.0.x.

For more information about this feature please refer to the WebReports Admin guide.
