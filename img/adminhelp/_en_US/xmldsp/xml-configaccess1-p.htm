1. On the Content Server Administration page in the HTTP Messaging Administration section, click the Configure External HTTP Access link.

  2. On the Configure External HTTP Access page, type the fully qualified URL that external systems can use to access the XML Workflow Interchange step in the XMLDSP url box.

  3. Click Save Changes.

     

    Tip:
    If you have configured your HTTP server to allow HTTP access using a non-standard port number, you can add the port number after the host name. Separate the host name and port number by a colon.

     

For information related to this procedure, see Configuring External HTTP Access.
