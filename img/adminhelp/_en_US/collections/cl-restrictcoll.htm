Administrators can choose to restrict access to the Collection menu option to specific users and groups. The Collection menu option is restricted by selecting the Restrict link next to the Multi-Select Command Usage Type in the Usage Privileges area of the Administer Object and Usage Privileges page.

Restricting access to menu options in Collections

You can also control the options that appear in the More Actions list within a Collection. For example, if Collections is used to collect information for team projects, and only project managers should be able to move items from one Collection to another Collection, you can restrict the Move Items to Another Collection menu option to only those Users or Groups.

To restrict the options that appear in the More Actions list within a Collection to specific users and groups, see the Usage Type: Collections Command in the Usage Privileges area of the Administer Object and Usage Privileges page. The Administer Object and Usage Privileges page is accessed from the Content Server administration page, under the System Administration heading.

The following options are available, by default, to all Users and Groups. You can restrict these options from appearing in the More Actions list within a Collection:

  • Copy Items. For information about this option, see Copying, Moving and Removing Items from a Collection.

  • Move Items. For information about this option, see Copying, Moving and Removing Items from a Collection.

  • Copy Items to Another Collection. For information about this option, see Copying, Moving and Removing Items from a Collection.

  • Move Items to Another Collection. For information about this option, see Copying, Moving and Removing Items from a Collection.

  • Remove Items from Collection. For information about this option, see Copying, Moving and Removing Items from a Collection.

  • Zip & Download Command. For information about this option, see Downloading a Collection as a Spreadsheet.

  • Zip & Email Command. For information about this option, see Downloading a Collection as a Spreadsheet.

  • Apply Categories. For information about this option, see Applying Categories to Collections.

For more information on how to restrict object privileges, see Administering Object and Usage Privileges.

The following options that appear in the More Actions list within a Collection are, by default, restricted to administrators on new and upgraded installations of Content Server Upgrade 5, or later.

  • Queue for Indexing. For information about this option, see Queuing a Collection for Indexing.

  • Delete Items. For information about this option, see Copying, Moving and Removing Items from a Collection.

  • Make Disk Image. For information about this option, see Creating and Working with Disk Images.

  • Download as Spreadsheet. For information about this option, see Downloading a Collection as a Spreadsheet.

  • Collect All Search Results. For information about this option, see Collecting Items from the Search Results Page.

 

Note:
When an administrator creates a Collection from the results of a search, the Collection can be downloaded as a spreadsheet, or multiple spreadsheets are created and compressed into a single ZIP file.

 

Enabling Classifications with Collections

If you have the optional Content Server Classifications module installed, you can control the Usage Privilege Apply Classifications for the Usage Type Collections Command. This allows you to restrict which users can see the Apply Classifications menu options in the More Actions list in a Collection. By default, this setting is Unrestricted. For more information on using Collections, see Using Collections.
