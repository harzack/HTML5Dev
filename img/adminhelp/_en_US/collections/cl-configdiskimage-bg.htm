Once a Collection has been created and items have been collected, you can create a Disk Image of the Collection. A Disk Image enables you to copy all or specific items in a Collection to disk. This can be used to share content in a Collection with individuals or groups outside of the Content Server environment.

Disk Image Manifest File

A Disk Image Manifest File is created when you create a Collection.

The Disk Image includes a generic help file for users when browsing the contents of the manifest file. A link in the header of the manifest file will open the help file in a new browser window. You can edit this help file in a text editor, or replace it with your own customized version, if needed. For details about the information in the Manifest File, see Reading a Disk Image Manifest File.

When you configure the Disk Image creation settings, you can specify information about Collections and collected items that can be captured, the disk sizes that can be chosen, and when email notification should be sent. You can also restrict the Make Disk Image option to specific Content Server Users and Groups.

Specifying the Disk Image Creation Settings

To configure the Make Disk Image option, you must first configure the parameters used when writing Collections to disk in the Disk Image Creation Settings area of the administration pages.

 

Important:
For the Disk Image creation process to work, you must have a Content Server File Cache, also known as remote admin cache, RAC, configured.

 

On the Disk Image Creation Settings page, the following settings can be configured:

Setting Description
Available Disk Sizes Defines the disk sizes available to Users during the Make Disk Image creation process. and specifies the default value that should appear. The disk size is the expected size of the disk output. If the Collection size exceeds the specified size, an additional Disk Image is created.
Collections Defines the Content Server information about the Collection that can be included in the Disk Image, and if Users are able to modify these settings. For example, if you always want the Collection's Description included in Disk Images, and you do not want Users to be able to exclude it, select the Active check box but do not select the User Configurable check box.
Content Defines the Content Server information about the items in the Collection that can be included in the Disk Image and if Users are able to modify these settings. For example, if you always want the Descriptions of Collection items to be included in Disk Images, and do not want Users to be able to exclude them, select the Active check box, but do not select the User Configurable check box.
Notification Options Defines the point in the Disk Image creation process that email notification should be sent. For more information about email notification and the contents of the email messages, see Understanding Disk Image Creation Notification.
Show Max Items per Image Allows Users to specify the number of distinct objects to be included in each image. This value and the Disk File System Format limitations cause multiple output directories to be generated when the amount of output exceeds the value and format limitations. You can clear the check box if you do not want Users to specify a Max Objects per Image setting.
 

Note:
For more information about Disk File System Formats, see Creating and Working with Disk Images.

 
Allow Sample Images Defines whether a psuedo-random sample of the items selected for the Disk Image creation process can be created. This allows your users, or the consumers of the Disk Image, to assess whether a set of data is appropriate for full investigation by statistical sampling. For example, in a Collection of 1 million objects, the mathematics may indicate that a statistical sample of 16,500 items will be 98% accurate 95% of the time.
Metadata Language Options Default Language Defines the default metadata language for the Disk Image. This setting is only available when multiple languages are available on your Content Server system.
  User Configurable Defines whether a User can modify the default metadata language. This setting is only available when multiple languages are available on your Content Server system.
Administrator Notify Enabled Defines whether an email notification is sent to the Content Server administrator.
 

Note:
This option is not configurable by Users during the Make Disk Image creation process.

 
  Notify Level Required. Defines the number of items in a Disk Image creation process at which the administrator is notified by email.
E-Mail Address Required. Defines the email address this notification should be sent to.
Directory Mapping Description Information about the Collections Make Disk Image command.
  Admin Server Defines the Content Server File Cache location where Disk Images are written to. You can click the File Cache Settings link to select a specific File Cache and to change additional values. For more information on File Cache settings, see Setting Up the Admin Server File Cache.
  Map Value Required. Defines the URL to access the Disk Image. This value appears in the confirmation message and notification email message when a Disk Image is created. When specifying this value, you can use an FTP server, or provide a UNC or NFS network share location.

 

Note:
By default, when a Disk Image is created, it is removed from the File Cache one day after creation. You can modify this value by changing the Expire Time setting on the File Cache tab in the Admin Server Properties page. You can access this page quickly by clicking the File Cache Settings link in the Admin Server area on the Disk Image Creation Settings page.

 

For more information about creating Disk Images, see Creating and Working with Disk Images.

Restricting the Make Disk Image Option

The Make Disk Image option appears by default in the More Actions list when viewing a Collection. You can restrict this option to specific Users or Groups by updating the Make Disk Image Usage Privilege. For example, if you have a Group that is responsible for releasing information to third party sources, and you only want that group to be able to copy information to a Disk Image, you can restrict this option to only that Group. For more information on how to restrict object usage privileges, see Administering Object and Usage Privileges.
