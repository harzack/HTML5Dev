To create a secure database connection:

  1. In the Web Forms Database Lookup Administration section, click the Manage Secure Database Connections link.

  2. On the Web Forms Connections page, click Web Forms Database Connection on the Add Item menu.

  3. On the Add: Web Forms Database Connection page, click the type of relational database you wish to connect to on the RDBMS Server Type list.

     

    Note:
    Microsoft SQL Server will only be available as a choice on the RDBMS Server Type menu if Content Server is running on a Microsoft Windows server.

    Based on whether you select the Oracle or Microsoft SQL database system from the RDBMS Server Type list, different fields display in the Add: Web Forms Database Connection page.

     

    • For Microsoft SQL Server, specify the following information:

      Field Contents
      User Name The name of the Microsoft SQL Server user account with permission to access the database.
      Password The password of the Microsoft SQL Server user account with permission to access the database.
      SQL Server Database Name The name of the Microsoft SQL Server Database to which the connection is made.
      SQL Sever Name The name of the Microsoft SQL Server to which the connection is made.

    • For Oracle Server, you must specify the User Name and Password used to connect to the Oracle server, and the Service Name, which is the connect string for the database service to which the connection is made.

  4. The Dependent Lookups box displays regardless of the database system you select. You cannot modify this field while creating a database connection.

  5. Specify Name, Description, Categories and Create In parameters as you would when adding any item to Content Server.

     

    Note:
    The Name you specify for the database connection must be unique among all database connections.

     

  6. Click Add.

For information related to this procedure, see Managing Secure Database Connections.
