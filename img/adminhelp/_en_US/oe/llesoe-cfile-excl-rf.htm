exclude Tag

Description

Each time you run an export, the following object types are exported: alias, compound document, custom view, discussion, document, folder, project, reply, topic, and url. To prevent a particular object type from being exported, use the exclude tag.

Syntax

<exclude>
  <object>alias</object>
</exclude>

Usage

  1. The exclude tag is optional.

  2. If it is used, the exclude tag has one required subordinate tag, the object Tag, which is detailed below.

Notes

If container objects, such as folders, are excluded, objects in the excluded containers will not be exported.

Example

In this example, all objects in the Enterprise Workspace, except Compound Documents and Custom Views, will be exported to c:\docs

<export>
  <node>
    <location>Enterprise</location>
    <docstorage>c:\docs<docstorage>
    <exclude>
      <object>compounddoc</object>
      <object>customview</object>
    </exclude>
  </node>
</export>

object Tag

The object tag sets the name of the object to be excluded. This tag's syntax is: 

<object>discussion</object>

Usage
1.

The object tag is required when the exclude tag is used.

2.

Allowable values for use with the object tag are: alias, compounddoc, customview, discussion, document, folder, project, reply, topic, and url.
