Description

 

Important:
This is a mandatory tag regardless of object type.

 

The location tag specifies the location in Content Server from which Object Exporter will begin exporting files.

 

Note:
Object Exporter provides support for the multilingual feature available with Content Server. When a new instance of Content Server is installed, the administrator selects the language which will function as the system default language. You can view your system default language, and all languages which have been installed and enabled for metatdata, in the Metadata section of the Content Server administration page. Click the Configure Multilingual Metadata link. For more information about configuring multilingual metadata on Content Server, see Configuring Multilingual Metadata in the System Administration section of the Configuring Multilingual Metadata.

Because Object Exporter now supports multilingual metadata, you should note that the location of the object is based on the system default language, if set. If the system default language is not set, the location of the object is based on the first alphabetical language, by code.

 

Syntax

<location>Enterprise</location>

Usage

This is a mandatory tag regardless of object type.

Example

In this example, all supported objects in the Enterprise Workspace will be exported:

<export>
  <node>
    <location>Enterprise</location>
    <docstorage>c:\docs</docstorage>
  </node>
</export>
