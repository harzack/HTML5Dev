deletedusers Tag

Description

By default, Object Exporter will export deleted users and groups. To prevent this data from being exported, use the <deletedusers>FALSE</deletedusers> tag.

Syntax

<deletedusers>FALSE</deletedusers>

Usage

  1. The deletedusers tag is optional.

  2. The deletedusers tag has optional attributes: owner Attribute, ownergroup Attribute, and creator Attribute. These attributes are detailed below.

Example

<export>
  <node>
    <location>Enterprise:Folder_001</location>
    <docstorage>c:\docs<docstorage>
    <deletedusers owner="Admin" ownergroup="DefaultGroup" creator="Admin">FALSE</deletedusers>
  </node>
</export>

owner Attribute

The owner attribute sets a value which can be defaulted to another value, if a deleted user or group is encountered. This attribute's syntax is: 

<deletedusers owner="Admin" ownergroup="DefaultGroup" creator="Admin">FALSE</deletedusers>

Usage
The owner attribute is optional if the deletedusers tag is used.

ownergroup Attribute

The ownergroup attribute sets a value which can be defaulted to another value, if a deleted user or group is encountered. This attribute's syntax is: 

<deletedusers owner="Admin" ownergroup="DefaultGroup" creator="Admin">FALSE</deletedusers>

Usage
The ownergroup attribute is optional if the deletedusers tag is used.

creator Attribute

The creator attribute sets a value which can be defaulted to another value, if a deleted user or group is encountered. This attribute's syntax is: 

<deletedusers owner="Admin" ownergroup="DefaultGroup" creator="Admin">FALSE</deletedusers>

Usage
The creator attribute is optional if the deletedusers tag is used.
