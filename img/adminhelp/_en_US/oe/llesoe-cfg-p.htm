To configure Object Exporter:

  1. Create a working directory, visible to Content Server, where the output file(s) will be created and stored temporarily, before being moved when the process is finished.

  2. Create a log directory where Object Exporter will write log files. Ensure this directory is in the <Content Server_home>\object_exporter path.

  3. Create a destination directory, visible to Content Server, where Object Exporter will move the output file(s) when the process is finished.

  4. On the Content Server administration page, in the Object Exporter Administration section, click Configure.

  5. On the Configure Object Exporter page, in the Directory section:

    1. In the Working field, specify the full path of the directory you created in Step 1. This is the directory where Object Exporter will process the export.

    2. In the Log field, specify the full path of the directory you created in Step 2. This is the directory where Object Exporter will write the log files, and should always be found in the <Content Server_home>\object_exporter path.

    3. In the Destination field, specify the full path of the directory you created in Step 3. This is the directory where Object Exporter will move the output file(s) when the process is finished.

  6. In the Advanced section, in the Objects Per Batch field, specify the number of objects to include in each output file.

  7. Click Update.

  8. Restart the servers when prompted.
