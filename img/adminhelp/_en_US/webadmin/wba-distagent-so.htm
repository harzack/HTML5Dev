The Distributed Agent Status page displays the worker identification numbers of the workers in the cluster, and their status on Content Server. In addition, the following information is displayed for each worker ID:

  • the date and time that the worker was last updated.

  • the status of the worker. Possible values include Idle, Waiting or Running.

  • the description of the status of the worker.

  • the actions that you can perform on that worker. Actions can be one of Pause or Resume.

There is little that an administrator will want to do on the Distributed Agent Status page. This page is mostly intended for informational purposes. However, you might have processes running on your hardware, which processes cause that particular machine to become overloaded. In the event this occurs, you can refresh the status of the workers on the Distributed Agent Status page, and, if warranted, pause a worker on an overloaded machine.
