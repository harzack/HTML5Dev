You can specify how Content Server commits batch copy or delete operations to the databaseâ€”either individually or as a group. Content Server defaults to single processing, which means if a single item fails, then the entire batch rolls back to its original state and nothing becomes committed to the database. In group processing, if an item fails, it does not cause the entire operation to fail. All items that process successfully will commit to the database.

In the Delete section, you can enable the Delay deletion from storage provider option. If this option is enabled, when an item is deleted, its content is not deleted immediately. The content is deleted through the Provider Blob Deletion Failure Retry scheduled activity instead. If you enable the Delay deletion from storage provider setting, you must also enable the Provider Blob Deletion Failure Retry scheduled activity on the Configure Schedule Activity page. For more information, see Administering Blob Deletion Failures.

You can configure the move throttle control settings when moving objects from on location of Content Server to another. For users to move items in Content Server, you must enable the move throttle control and specify a maximum number of objects they are allowed to move at one time.

 

Note:
Nodes cannot be moved even if the number of objects selected for a move operation is less than the maximum limit.
