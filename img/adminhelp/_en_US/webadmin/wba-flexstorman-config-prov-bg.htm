Storage Providers enable you to store Documents outside Content Server. Examples of storage providers include:

  • external directories

  • other types of storage systems, such as databases

  • OpenText Archive Server

Each Content Server instance that uses Flexible Storage has a default Storage Provider that cannot be modified or deleted. If a Storage Provider is not specified, the default Storage Provider is used for all documents added to Content Server.

 

Note:
You cannot delete a Storage Provider when it is in use.

 

After you add a Storage Provider, you must:

  • provide a unique name for your Storage Provider

  • provide the absolute path to your Storage Provider

  • define and order the rules associated with your Storage Provider.

Whenever a document is added to Content Server, the system determines which Storage Provider to use based on the defined rules. For more information about Storage Provider Rules, see Configuring Storage Rules.

Administrators can audit content that is moved from one Storage Provider to another. By default, the Provider Changed audit interest is turned off. The audit details for this event include the original Storage Provider name, the new Storage Provider name, and the name of the user who moved the content. Every Document in Content Server displays the name of its Storage Provider on the Versions tab of the Document's Properties page. Users with the proper permissions can view the entire list of Storage Provider Rules. This list includes the Storage Provider rule, and the name and type of the Storage Provider.
