On the Set Auditing Interests page, you can specify which events are audited. Table 21-1 lists the events that you can audit. The table below lists the auditable events that are available when Content Server is installed. If optional modules are installed, additional auditing interests may be available.

 

Notes:

  • When an audit event is not registered with the registry subsystem, it is logged as Deprecated. A deprecated audit event has an ID of zero (0).

  • The Force auditing of all events performed by System Administrators option can enable the auditing of any auditable actions performed by users with the System Administration privilege, regardless of the other audit interest settings.
