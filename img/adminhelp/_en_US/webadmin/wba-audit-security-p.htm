To configure audit security settings:

  1. Log on to Content Server as the Admin user.

     

    Note:
    You must be logged on as Admin to configure audit security settings. It is not sufficient to log on as a user with the System administration rights privilege.

     

  2. In the System Administration section of the Administration page, click Administer Event Auditing.

  3. On the Administer Event Auditing page, click Audit Security Settings.

  4. On the Audit Security Settings page:

    1. Optional If you want to prevent any user other than the Admin user from purging the audit log or modifying your audit interests, enable Limit Audit Config to "Admin" User.

    2. Optional If you want to prevent any user at all from purging the audit log:

      1. To prevent any user, including the Admin user, from purging the audit log, click Disable Audit Purge.

         

        Important:
        Clicking Disable Audit Purge cannot be undone. If the audit log becomes large, you may see an impact to performance.

         
         

        Tip:
        The Disable Audit Purge button is disabled if audit purge has already been disabled.

         

      2. In the Disable Audit Purge? dialog box, enable I understand the risk involved by enabling this setting and want to continue, and then click OK.

    3. Optional If you want to prevent any user at all from changing the audit interests configuration:

      1. To prevent any user, including the Admin user, from modifying the auditing interests and ensuring that the Set Auditing Interests page becomes read-only, click Lock Audit Interests.

         

        Important:
        Clicking Lock Audit Interests cannot be undone. In the event that new interests are added to Content Server, for example if you install a new module or upgrade an existing module, those new interests cannot be audited.

         
         

        Tip:
        The Lock Audit Interests button is disabled if auditing interests have already been locked.

         

      2. In the Lock Audit Interests? dialog box, enable I understand the risk involved by enabling this setting and want to continue, and then click OK.

    4. Click Submit.
