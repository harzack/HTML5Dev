The [OTAdmin] section contains connection, logging, and index server timeout options for the Admin server.

This page contains information about the following parameters:

logfile

•

loginput
•

•

•

logfile

  • Description:

    Specifies the path and file to which Admin server activity is logged.

     

    Important:
    OpenText recommends that you do not modify the value of this parameter.

     

    If you do modify it, OpenText recommends that you use the Configure Debug Settings page, rather than edit the opentext.ini file directly. For more information, see Configuring Server Logging.

  • Syntax:

    logfile=C:/OPENTEXT/logs/admserv.log

  • Values:

    An absolute path and filename. By default, this parameter does not appear in the opentext.ini file, which means that the log file is named admserv.log and is stored in the <Content Server_home>/logs directory, where <Content Server_home> is the root of your Content Server installation.

loginput

  • Description:

    Specifies the log file to which the admin server logs all incoming information requests from the server.

     

    Important:
    OpenText recommends that you do not modify the value of this parameter.

     

    If you do modify it, OpenText recommends that you use the Configure Debug Settings page, rather than edit the opentext.ini file directly. For more information, see Configuring Server Logging.

  • Values:

    An absolute path and file name. By default, this parameter does not appear in the opentext.ini file, which means no incoming connection logging occurs.

loglevel

  • Description:

    Specifies which Admin server activities are logged.

     

    Important:
    OpenText recommends that you do not modify the value of this parameter.

     

    If you do modify it, OpenText recommends that you use the Configure Debug Settings page, rather than edit the opentext.ini file directly. For more information, see Configuring Server Logging.

  • Syntax:

    loglevel=1

  • Values:

    By default, this parameter does not appear in the opentext.ini file, which is equivalent to loglevel=1.

    Value Description
    1 Specifies the Default logging level, which means that serious problems are logged, along with startup and shutdown activities.
    2 Specifies the Verbose logging level, which means that error messages from child processes and server status are logged.
    3 Specifies the Debug logging level, which produces a detailed log for debugging purposes.

logoutput

  • Description:

    Specifies the name of the Log file to which the admin server logs all status information that the admin server outputs to Content Server.

     

    Important:
    OpenText recommends that you do not modify the value of this parameter.

     

    If you do modify it, Content Server recommends that you use the Configure Debug Settings page, rather than edit the opentext.ini file directly. For more information, see Configuring Server Logging.

  • Values:

    An absolute path and file name. By default, this parameter does not appear in the opentext.ini file, which means no outgoing connection logging occurs.

port

  • Description:

    Specifies the port on which the admin server accepts connection requests.

    You must modify the value of this parameter in the opentext.ini file. For more information, see Understanding the opentext.ini file.

  • Syntax:

    port=5858

  • Values:

    Any open TCP/IP port. The default value is 5858.
