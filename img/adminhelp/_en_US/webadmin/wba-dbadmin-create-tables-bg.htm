A Content Server database comprises a set of relational tables stored in the RDBMS. The last step in creating a new Content Server database, connecting to a new or existing Content Server installation, is to create the Content Server database tables on the Create Content Server Tables page. The Create Content Server Tables page appears after you click the Log-in button on the Administrator Log-in page corresponding to your RDBMS type.

To create Content Server tables under an Oracle RDBMS, you must specify the name and password of the Oracle account that you want to own the tables. To create Content Server tables under a Microsoft SQL Server RDBMS, you must specify the name and password of the RDBMS account that you want to own the tables, as well as the name of the SQL Server database for which the account has log-in privileges.

Before you can create Content Server tables, you must create the RDBMS account and the RDBMS repository (tablespace for Oracle, log and data files and SQL Server database for Microsoft SQL Server) to which the account logs in on the Server Maintenance page corresponding to your RDBMS type.

In addition to creating the RDBMS account and repository, you must also decide whether you want the database to store data internally or externally. For more information about internal and external document storage, see Managing Internal and External Storage.

After you create the RDBMS account and repository and selected the type of data storage, you can create the Content Server tables using the procedure that corresponds to your RDBMS type:

  • Oracle Server

  • Microsoft SQL Server
