Installing a Module

You install a module in the following two stages:

  • First, use the module's InstallShield program or .tar file to start the installation. This process extracts the files and places them in the Content Server_home\staging directory.

  • Second, install the module on your current Content Server service by accessing the Administration page, selecting Module Administration and then selecting Install Modules.

When you begin the installation, you are asked to select the instance of Content Server on which you want the module installed. You can only install a module on one instance of Content Server at a time.

For the first stage of a module installation, if you are installing optional Content Server modules during the initial installation of Content Server, follow the instructions in the Content Server First-Time Installation Guide. to complete the setup of Content Server.

 

Note:
If a module has associated online help files, you must update the Help Index after installing, uninstalling, or upgrading a module in order for the changes to be reflected in the Online Help.

If you installed optional modules during the installation of Content Server, the Help Index is automatically built to reflect help for all installed modules.

 

Installing Modules on Windows

For Windows versions of Content Server modules, you perform the first stage of an installation using an InstallShield installation program. This installation program allows you to select the Content Server instance to which you want to add the module, and places the module's files in a subdirectory of the Content Server_home\staging directory.

The Content Server_home\staging directory does not exist if optional modules have not yet been installed. In this case, the module installation program automatically creates the Content Server_home\staging directory.

Installing Modules on UNIX

For UNIX versions of Content Server modules, you perform the first stage of an installation using a .tar compressed archive file. The extraction of this .tar file places the module's files in a subdirectory of the Content Server_home/staging directory.

The Content Server_home/staging directory does not exist if optional modules have not yet been installed. In this case, the extraction of the .tar file automatically creates the Content Server_home/staging directory.

Uninstalling a Module

Content Server allows you to uninstall any optional modules that are installed, such as Content Server Forms or Content Server Explorer. You can also uninstall certain pre-installed modules; however, OpenText strongly recommends that you do not uninstall them unless instructed to do so by OpenText Customer Support.

Before you proceed to the Uninstall Modules page, you can view a list of all Content Server modules that can be uninstalled from your system, as well as view their dependent modules, from the View Uninstallable Modules page.

Once you proceed to the Uninstall page, if a module is depended on by other modules, it will not have an Uninstall button. You must uninstall the module(s) listed in the Dependents column before you can uninstall the module on which they depend.

Once you have completed the uninstallation process, you must restart your server and Web Application server.
