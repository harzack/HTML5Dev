The Maintain Current Database page displays information about the current Content Server database and a series of links that you use to perform tasks on the database.

For Oracle Server, the Maintain Current Database page displays the following information:

  • RDBMS type

  • Oracle service name (also called database alias)

  • Name of Oracle user that owns the Content Server database tables

For Microsoft SQL Server, the Maintain Current Database page displays the following information:

  • RDBMS type

  • Name (alias) of the Microsoft SQL Server installation (often the name of the host on which it resides)

  • Name of the SQL Server database that contains the Content Server tables

  • Microsoft SQL Server user name that Content Server uses to log in to the SQL Server database that contains the Content Server tables

You perform a number of database maintenance tasks on the Maintain Current Database page. The tasks that you can perform on the Maintain Current Database page differ depending on RDBMS type and whether the database uses external or internal document storage.

For Oracle Server, the Maintain Current Database page contains links that you use to perform the following tasks:

  • View tables

  • View tablespace usage

  • Perform Oracle Server maintenance tasks

  • Purge deleted data (internal document storage only)

  • Upgrade the database

  • Verify the database

For Microsoft SQL Server, the Maintain Current Database page contains links that you use to perform the following tasks:

  • View Content Server tables

  • Perform Microsoft SQL Server maintenance tasks

  • Upgrade the database

  • Verify the database

  • Test or repair the database

Performing RDBMS Maintenance Tasks

The Maintain Current Database page displays a link to the maintenance page corresponding to your RDBMS type. Depending on your RDMS type, this link leads to the Oracle Server Maintenance or Microsoft SQL Server Maintenance page.

On the Oracle Server Maintenance page, you can:

  • Create or extend a tablespace

  • Create an Oracle user

  • Delete an Oracle user and/or the database tables that it owns

On the Microsoft SQL Server Maintenance page, you can:

  • Create or extend data and log disk services

  • Create a SQL Server database

  • Create a user to log in to a SQL Server database

  • Delete a SQL Server database and/or the database tables that it contains

When you connect the current Content Server installation to a new or existing Content Server database, a Congratulations page appears. It confirms that the connection to the database was completed successfully.

On the Congratulations page, you have the following options:

  • Use the Go to the Content Server home page link to access the default Content Server start page.

  • Use the Admin Home link to access the Content Server Administration page.

     

    Note:
    If you connect this Content Server installation to an existing Content Server database and click Continue on the Data Source Resynchronization Confirmation page, you must reconfigure the data source objects, then individually resynchronize all registered Admin servers.
