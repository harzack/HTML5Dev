To create Content Server tables under a Microsoft SQL Server RDBMS:

  1. On the Create Content Server Tables page, click the radio button that corresponds with your RDBMS type.

  2. On the Create Content Server Tables page, click the name of the Microsoft SQL Server database in which you want to create the tables.

  3. In the Create Content Server Tables page, click the name of the Microsoft SQL Server user that you want to own the tables in the Microsoft SQL User Name list. The selected user must have log-in privileges to the selected SQL Server database.

  4. Type the password of the selected Microsoft SQL Server user in the Password field.

  5. If you want Content Server to store documents and other items outside the database, select the External Documents check box and then type the absolute path of the directory where you want the Content Server database to store items in the adjacent field.

  6. Click the Create Tables button.

    Depending on whether you selected internal or external storage and whether this is a new or existing Content Server installation, one of the following pages appears:

    • External Document Directory

    • Install Modules

    • Create New Update Enterprise Index
