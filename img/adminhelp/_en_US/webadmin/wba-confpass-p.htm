To configure password settings:

  1. Click the Configure Password Settings link in the Users and Groups Administration section of the Administration page.

  2. In the Minimum Number of Characters field, type the minimum number of characters required for passwords.

  3. Select the Enable check box for any of the following:

    • Passwords Must Contain a Digit

    • Password Cannot Begin With a Digit

    • Password Cannot End With a Digit

    • Changed Password Must Be Different

    • Change Password at First Login

  4. To set all passwords to expire after a certain period, click the On radio button in the Password Expiration section, and then type an integer in the Days Before Expiration field

    If desired, enable Users with the "can create/modify users" privilege can set password expiration on individual user accounts to:

    • allow users with the Can create/modify groups privilege to set a password expiration on user accounts under their control, and

    • allow users with the User administration rights privilege to set Password Never Expires on user accounts

     

    Tip:
    The Can create/modify groups and User administration rights privileges can be set on the Add New User page. For more information, see Working with Users.

     

  5. To prevent users from reusing a password for a certain number of days, click a value in the Days to Prevent Password Reuse drop-down list.

  6. To prevent users from choosing a new password before a certain number of days, click a value in the Days Required Between Password Changes drop-down list.

  7. Click the Save Changes button.

 

Notes:

  • Changing password expiration settings requires Content Server to update the user database. If you make this change, a dialog box opens when you attempt to save your changes, showing you the progress of the update.

  • The maximum number of characters a password can contain is set to 16 by default. To change this setting, you must modify the opentext.ini file. For more information, see [Passwords].
