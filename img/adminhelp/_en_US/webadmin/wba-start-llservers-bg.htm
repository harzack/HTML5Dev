There are two servers referred to in this section: the Content Server server and the Admin server server. If you keep the default settings during the installation of Content Server, your servers are given the following default names:

  • Content Server (OTCS): referred to in the help as Content Server.

  • Content Server Admin (OTCS): referred to in the help as Admin server.

Unless you specify otherwise, both servers start up automatically during the installation and setup of Content Server.

Some system administration tasks require you to stop and restart Content Serverfor the changes to take effect. By default, the Content Server Setup program sets the Content Server and Admin server to automatically start. OpenText recommends that you do not alter this setting, because Content Server requires that these servers be running to function correctly. However, there may be circumstances, such as when you are troubleshooting a problem, under which you do not want either of the servers to start automatically when the primary Content Server host is rebooted. In such cases, you can temporarily set both servers to manual startup, but make sure that you reset them to automatic startup when you are done.

 

Note:
Whenever you restart Content Server, remember to restart the Web application server as well.

 

You can set up the Content Server and Admin server services to start automatically or manually when the computer that hosts them starts. The procedures for modifying these settings are significantly different for Windows and UNIX.

In this section, the names of the servers used in the examples are the default names.
