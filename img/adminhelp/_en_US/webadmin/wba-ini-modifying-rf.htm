Since most of the configuration parameters appear in the Administration pages, it is not usually necessary to manually edit the opentext.ini file. Do not manually edit the opentext.ini file unless you are explicitly advised to do so from:

  • an instruction in a Content Server document, such as this online document or the Content Server Installation Guide

  • OpenText Customer Support

If you do edit the opentext.ini file directly in a text editor, you must stop and then restart the server and the Web application server to put the changes into effect.

The opentext.ini file is located in the <Content Server_home>/config directory, where <Content Server_home> is the root of your Content Server installation.

The opentext.ini file is organized according to the standard Windows INI-file structure. It is composed of several sections, each of which is headed by a label in square brackets. Each section contains zero or more parameters. These parameters are in the form <name>=<value>. The order of the sections is not important. Section titles and name strings are not case sensitive. Value strings, however, are always case sensitive.

 

Note:
The sections that are described are for default Content Server parameters. Optional modules may add other sections to the opentext.ini file that are not described here.

 

For information about modifying other system-configuration files, see Modifying System Configuration Files.
