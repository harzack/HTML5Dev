To deselect the current Content Server database:

  1. In the Database Administration section of the Content Server administration page, click the Change Current Database link.

  2. Optional On the Deselect Content Server Database page, by default, deselecting the current database will delete the indices listed in the table. To prevent an index from being deleted, in the Delete column, clear the check box next to that index.

  3. Click Continue to complete deselecting the current database.

     

    Important:
    Before you proceed to connect to a different database, you must:

    1. Stop all Admin Servers.

    2. Remove the search.ini and otadmin.cfg files from the <Content Server_home>/config directory of each Admin Server. This is required in order to prevent orphaned processes from being carried to another database.

    3. Start each Admin Server.

    4. After reconnection, each Admin Server must be resynchronized to create new search.ini and otadmin.cfg files.

     
 

Note:
After you deselect the database, you can create a new one or select an existing one to replace it with. For more information, see To Change the Current Database.
