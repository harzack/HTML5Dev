When Open Text releases new versions of Content Server modules, the modules can often be downloaded from the Open Text Knowledge Center, or sometimes they are available on CD-ROM. Open Text recommends that you consult the a module's Release Notes prior to beginning an upgrade installation.

A module is upgraded in two stages. The first stage of the upgrade extracts the necessary files from the .tar or installation program file and places them into the _home/staging directory. Your Content Server system is not yet upgraded; you must access the Administration pages to complete the module upgrade.

 

Note:
When you upgrade a module, the files from the previous module are removed from Content Server and moved back to the _home\staging directory.

 

The way you upgrade a module depends on the type of operating system on which Content Server is currently installed. Once the new files are added to the appropriate locations in the _home/staging directory, the upgrade process is the same for both the Windows and UNIX operating systems.

Upgrading Modules on Windows and UNIX

For Windows versions of Content Server modules, you perform the first stage of an upgrade using an InstallShield installation program. This installation program allows you to select the Content Server instance on which you want to upgrade the module and places the module's files in a subdirectory of the _home/staging directory.

For UNIX versions of Content Server modules, you perform the first stage of an upgrade using a .tar compressed archive file. The extraction of this .tar file places the module's files in a subdirectory of the _home/staging directory.

The _home/staging directory does not exist if optional modules have not yet been installed or pre-installed modules have not been upgraded. In this case, the module installation program (for Windows) or the extraction of the .tar file (for UNIX) automatically creates the _home/staging directory.

Completing a Module Upgrade

You perform the second stage of a module upgrade using the Upgrade Modules page, which is accessed via the Administration page.

Once you have performed the upgrade, the Restart Content Server page is the first page that appears for most modules. However, some module upgrades require you to set certain configuration parameters before restarting the server. For information about setting configuration parameters, see the module's specific documentation.

 

Note:
If you have installed multiple modules, you will see them on the Upgrade Modules page. You can upgrade multiple modules at one time, as long as they have no dependencies on other modules. When a module has a dependency on another module, that module must first be added or upgraded. After you upgrade a module that has online help files associated with it, you must update the help index in order to see the module's associated online help files. For more information, see Updating the User and Admin Online Help.
