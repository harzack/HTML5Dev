Although you can view the Deleted Objects Volume, it is not recommended for performance reasons, as it can potentially contain many thousands of items. Instead, from the Reports Volume, use the LiveReports included with Content Server, or add a custom LiveReport.

From the Reports Volume, you can manually purge or restore documents from the Deleted Objects Volume.

Content Server provides two LiveReports by default that allow you to find and view deleted documents. These LiveReports, which are located in the Reports Volume, are called Deleted Documents and Deleted Documents by [User].

Open Text strongly recommends that you add custom LiveReports to assist you in managing the Document Undelete functionality. For more information about LiveReports, see Administering LiveReports. For information about adding a custom LiveReport, see To Add a LiveReport.

Purging Deleted Documents

When you use the Delete function on a document in the Deleted Objects Volume, the document is individually purged and removed from Content Server.

When you view a deleted document, its name link appears with the Node ID in square brackets at the beginning, as follows: [3188] HRPolicies.doc

Documents deleted from a workflow show their path as Attachments, without any reference to the workflow location.

Restoring Deleted Documents

The document undelete function only applies to Content Server documents.

When you restore a deleted document, the Admin user becomes its owner and the only user with any permissions to that document. The document's original permissions are not restored. For information about setting permissions for a document after it has been restored, see To Grant Permissions to an Item.

If you are restoring a document from a workflow, that document will appear as a workflow attachment. Click the Browse Content Server button to explicitly define the restoration path.
