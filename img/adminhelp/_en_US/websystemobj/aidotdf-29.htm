Metadata is information accompanying each Content Server item, for example, the date that a document was created. Metadata is returned to the Document Filters as a separate parameter (DIHS_META_DOC_INFO) at the time of TextExtraction.

 

Note:
MetaData extraction is controlled by the outputoleinfo setting in Opentext.ini file.

 

During the time of TextExtraction, the filter extracts metadata, but does not remove the data from the content returned by OTDF.

The filter recognizes HTML tags in the metadata and converts them to OT mapped tags. A mapping of HTML to OT tags is kept in an XML file (htmlToOTTag.mapping) located in the OTHOME\config folder. If the file cannot be found or opened, then the original HTML tags are used.

If OTDF is run as an external process from DCS, then no threads are used. If the OTDF process hangs or takes longer than the timeout setting, then it's the responsibility of DCS to end the process.

To make a MIMEType request from OTDF, set a “MimeDetect” boolean DCS parameter to “true” in the DCSrules.txt file.

To make a TextExtraction request for indexing, set the “Operation” parameter to “ExtractText” in the [IM] section of the DCSrules.txt file.

The types of information that are extracted and returned to the DCS as metadata are defined in the metadataTags.txt file, and controlled by the outputoleinfo setting in Opentext.ini file.

You can modify the default names of metadata tags with the OT prefix by adding mappings to the htmltoOTTag.mapping file. For example, to rename the OTDocTitle document title metadata tag to CS10, enter:

OTDocTitle = CS10DocTitle

PDF files that contain compressed cross referenced table documents are supported. The metadata in compressed cross-reference tables and cross-reference streams (XRefStm), can be extracted with standard or custom metadata fields.

Metadata is extracted for all the formats listed in the latest Content Server Release Notes.
