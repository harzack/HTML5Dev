The following table describes the error messages for the Document Conversion process.

If these errors persist, contact OpenText Customer Support for additional information.

Table 1-27: Error Messages for the Document Conversion Process

Number Message Description Remedy
10 An internal error has occurred The Document Conversion process has performed an illegal operation (for example, it addressed memory that does not exist or made illegal system calls). Contact OpenText Customer Support.
11 Unable to recognize the MIME type The Document Conversion filter failed to recognize the MIME type of the file. Turn on logging by setting the following parameter in the [FilterEngine] section of the opentext.ini file:
logfile= <x>where <x> is the absolute path and file name of the log file. Look in the log file for more information, and then contact OpenText Customer Support.
12 Unable to find an input file The Document Conversion process expected to read an input file from the IPool but none exists. Contact OpenText Customer Support.
20 Unable to locate sufficient system resources The Document Conversion process failed because it cannot find enough system resources to run. Free some system resources (memory and disk space) and then try to run the process again.
21 Invalid parameters detected in the [FilterEngine] section of the opentext.ini file The Document Conversion process failed to start because the parameters specified in the [FilterEngine] section of the opentext.ini file are invalid. Examine the parameters that are currently specified in the [FilterEngine] section of the opentext.ini file. Then try to run the Document Conversion process again. The opentext.ini file is located in the config directory of your Content Server installation. OpenText recommends that you do not modify the opentext.ini file unless instructed to do so by OpenText Customer Support.
22 Invalid command line arguments specified The Document Conversion process failed to start because some of the command line arguments specified in the otadmin.cfg file are invalid. Examine the command line arguments that are currently specified in the otadmin.cfg file, and then try to run the Document Conversion process again. The otadmin.cfg file is located in the config directory of your Content Server installation. OpenText recommends that you do not modify the otadmin.cfg file unless instructed to do so by OpenText Customer Support.
23 Invalid input file name specified The Document Conversion process attempted to read a particular file from the IPool but the file does not exist. Try to run the Document Conversion process again. If the problem persists, consult OpenText Customer Support.
24 Invalid Sleep parameter specified in the [FilterEngine] section of the opentext.ini file The value of the Sleep parameter in the [FilterEngine] section of the opentext.ini file is invalid. Ensure that the value of the Sleep parameter is a positive integer, and then try to run the Document Conversion process again.
30 An internal IPool error has occurred The Document Conversion process encountered an IPool error when setting up the IPools. Turn on Document Conversion process logging by setting the following parameter in the [FilterEngine] section of the opentext.ini file:
logfile= <x>where <x> is the absolute path and file name of the log file. Then stop the data flow and turn on IPool logging by modifying the following value in the ipool.cfg file:
<LogLevel>3</LogLevel>
The ipool.cfg file and the IPool log file itself (iplog.fle) are stored in the data flow directory. The opentext.ini file is stored in the config directory of your Content Server installation.
After logging the activity of the Document Conversion process, contact OpenText Customer Support.
31 An internal IPool error has occurred The Document Conversion process encountered an IPool error when starting an IPool transaction. Turn on Document Conversion process logging by setting the following parameter in the [FilterEngine] section of the opentext.ini file:
logfile= <x>
where <x> is the absolute path and file name of the log file. Then stop the data flow and turn on IPool logging by modifying the following value in the ipool.cfg file:
<LogLevel>3</LogLevel>
The ipool.cfg file and the IPool log file itself (iplog.fle) are stored in the data flow directory. The opentext.ini file is stored in the config directory of your Content Server installation. After logging the activity of the Document Conversion process, contact OpenText Customer Support.
32 An internal IPool error has occurred The Document Conversion process encountered an IPool error when ending an IPool transaction. Ensure that the IPool directory is not currently in use. Then turn on Document Conversion process logging by setting the following parameter in the [FilterEngine] section of the opentext.ini file:
logfile= <x>
where <x> is the absolute path and file name of the log file. Then stop the data flow and turn on IPool logging by modifying the following value in the ipool.cfg file:
<LogLevel>3</LogLevel>.
The ipool.cfg file and the IPool log file itself (iplog.fle) are stored in the data flow directory. The opentext.ini file is stored in the config directory of your Content Server installation.
After logging the activity of the Document Conversion process, contact OpenText Customer Support.
33 An internal IPool error has occurred The Document Conversion process encountered an IPool error when stopping an IPool transaction. Turn on Document Conversion process logging by setting the following parameter in the [FilterEngine] section of the opentext.ini file:
logfile= <x>.
where <x> is the absolute path and file name of the log file. Then stop the data flow and turn on IPool logging by modifying the following value in the ipool.cfg file:
<LogLevel>3</LogLevel>
The ipool.cfg file and the IPool log file itself (iplog.fle) are stored in the data flow directory. The opentext.ini file is stored in the config directory of your Content Server installation.
After logging the activity of the Document Conversion process, contact OpenText Customer Support.
34 An internal IPool error has occurred The Document Conversion process encountered an IPool error when shutting down the IPool transport. Ensure that the IPool directories are not currently in use. Then turn on Document Conversion process logging by setting the following parameter in the [FilterEngine] section of the opentext.ini file:
logfile= <x>
where <x>  is the absolute path and file name of the log file. Then stop the data flow and turn on IPool logging by modifying the following value in the ipool.cfg file:
<LogLevel>3</LogLevel>
The ipool.cfg file and the IPool log file itself (iplog.fle) are stored in the data flow directory. The opentext.ini file is stored in the config directory of your Content Server installation.
After logging the activity of the Document Conversion process, contact OpenText Customer Support.
35 Unable to process an IPool message The Document Conversion process could not process a message when it tried to retrieve the message from an IPool. Turn on Document Conversion process logging by setting the following parameter in the [FilterEngine] section of the opentext.ini file:
logfile= <x>
where <x> is the absolute path and file name of the log file. Then stop the data flow and turn on IPool logging by modifying the following value in the ipool.cfg file:
<LogLevel>3</LogLevel>
The ipool.cfg file and the IPool log file itself (iplog.fle) are stored in the data flow directory. The opentext.ini file is stored in the config directory of your Content Server installation.
After logging the activity of the Document Conversion process, contact OpenText Customer Support.
