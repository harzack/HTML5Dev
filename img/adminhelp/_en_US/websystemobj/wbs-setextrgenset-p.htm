1. On the Administration page, in the Search Administration section, click the Configure the Enterprise Extractor Settings link.

  2. On the Configure the Enterprise Extractor Settings page, click the Set Extractor General Settings link.

  3. On the Set Extractor General Settings page, in the General Settings section, choose one of the following log levels in the Log Level drop-down list:

    • Normal – Specifies the Default logging level, which means that serious problems are logged, along with startup and shutdown activities. Minimal Extractor progress is logged.

    • Information – Specifies the Verbose logging level.

    • Detailed – Specifies the Debug logging level, which produces a detailed log for debugging purposes.

       

      Note:
      Content Server Debug Level must be 2 or greater for Extractor logging to function.

       

  4. In the Maximum Items per Extraction field, under Item Count, for:

    • Re-extract

    • Targeted Updates

    • Add / Modify

    • Initiated Workflows

    • All others

    Type a value for the maximum number of new or updated index update requests to extract from the Content Server database each time it runs. The default for each type of iterator is 5000.

    Under Options, use the drop-down list to select:

    • Normal Extraction

    • Pause Extraction

    • Discard Indexing Requests

    Under Item Count, for All others, type a value for the maximum number of all other new or updated items. The default is 5000.

  5. In the Maximum Versions to Extract field, type a value for the maximum number of versions per document that are extracted to the Enterprise [All Versions] slice.

    Click the All versions check box to enable all versions of documents to be updated when changes are made.

     

    Note:
    To prevent the Extractor from slowing down the system, the Maximum Versions to Extract value is limited at the Number of Items value, even when set to unlimited.

     

  6. In the Include Access Control Lists field, click the check box to place Access Control List (ACL) permission information into the IPools.

  7. In the EFS Object Content field, select a radio button to specify how External File Store (EFS) object content is processed:

    • Embed a copy of object content in IPool messages – embedded in IPool messages for increased indexing performance

    • Allow DCS to read object content directly from EFS – passed through some stages of data flows as separate files

       

      Note:
      Data files are referred to in EFS directly, rather than extracted to disk again. The files are not deleted.

      OpenText Customer Support recommends, if this setting is used, that you set your Admin Server to read-only mode for the EFS, to prevent data loss.

       

  8. In the Non-EFS Object Content field, select a radio button to specify how object content that is not External File Store (EFS) is processed:

    • Embed a copy of object content in IPool messages – embedded in IPool messages for increased indexing performance

    • Create a temporary copy of object content that DCS will delete after processing – an object's content is passed through some stages of data flows as separate files

      In the File Path field, type a valid Content Server directory path.

       

      Note:
      The File Path must be defined for this setting to function. Files are extracted to disc in this location, and are deleted once processed further down in the dataflow.

       

  9. In the IPool Limits section, in the Total Size field, specify a limit, in MB, for the maximum size for document content that will be written to the iPools, for all objects.

  10. In the Single Item Content Size field, specify a limit, in MB, for the maximum size for document content that will be written to the iPools, for all objects.

  11. In the Number of Items field, specify the maximum number of extracted items that the Enterprise Extractor process groups into a single IPool message.

  12. In the Exclusion Settings section, in the Item Types field, click the edit / review list link to open Configuring Excluded Item Types to specify the Content Server item or node types that you want to exclude from extraction.

    The number of item types currently excluded is also shown.

  13. In the Volume Types field, click the edit / review list link to open Configuring Excluded Volume Types to specify the Content Server volume types that you want to exclude from extraction.

    The number of volume types currently excluded is also shown.

  14. In the Locations field, click the edit / review list link to open Configuring Excluded Locations to specify the Content Server locations that you want to exclude from extraction.

    The number of locations currently excluded is also shown.

  15. In the Rendition Types field, click the edit / review list link to open Configuring Rendition Types to specify the Content Server rendition types that you want to exclude from extraction.

    The number of rendition types currently excluded is also shown.

  16. To enable gathering of Extractor performance statistics used in charting search indexing throughput or diagnosing Extractor performance, check the Enable Statistics Gathering check box, and set the number of days to keep statistics. The default is 7 days.

  17. In the Days to Keep Statistics text box, specify the number of days to collect data for. The default is 7 days.

  18. Click Update to use the settings you defined, or click Reset to Defaults restore the settings to their state when you opened this page, or Cancel.
