After you have added and configured Content Server data flow processes, you may find that some process libraries cannot read an IPool. After two unsuccessful attempts by a data flow process to read the IPool, the IPool can be quarantined to resolve this issue.

An IPool quarantine occurs on the file level, and the IPool file is made up of one or more IPool objects. An unreadable IPool file is sent to the data_flow/_failure directory.
