You need to modify the opentext.ini file on Windows and Linux operating systems to enable and customize specific conversion capabilities of OpenText Document Filters.

 

Note:
By default, OpenText recommends you do not modify the opentext.ini file unless directed to do so by OpenText Customer Support, but modifications are required to complete the Document Filters installation.

You should also know and understand the full intent and capabilities of the custom filter pack before you modify the opentext.ini file. For more information about the parameters you can modify in the opentext.ini file, see Understanding the opentext.ini file.

 

The opentext.ini file is located in the Content Server_home/config folder, where Content Server_home is the root of your Content Server installation.

When you edit the opentext.ini file directly in a text editor, you must stop and then restart the Content Server Server, Admin server and the Web application server to put the changes in effect.

The order of the sections is not important. Section titles and name strings are not case sensitive. Value strings, however, are always case sensitive.

You must make the following changes to the settings defined in the opentext.ini file.

 

Note:
These settings may be changed further as the result of testing and ongoing development of the OpenText Document Filters application.

 

To modify the opentext.ini file:

  1. Stop the Content Server services.

  2. Browse to the \OPENTEXT\config folder, and open the opentext.ini file in a text editor.

  3. Under [DCS] section, add modx10=DCSIm.

     

    Note:
    The modxXX setting specifies the section of the opentext.ini file that contains the configuration for the OpenText Document Filters. The [DCS] and [DCSView] sections may contain several mod parameters that correspond to different document conversion filters.

    During the installation, you may need change the mod parameter that specifies the DCSIm filter section to the next available modxXX number not currently used.

    Modifying the value of these parameters may prevent the DCS from functioning correctly.

    The character x specifies that the conversion filter process will be running as an external process to DCS worker.

     

  4. Under [DCSView], add modx10=DCSIm.

  5. Add a new [DCSIm] section, and add:

    • lib=DCSIm

    • dllpath=%OTHOME%/filters\image

    • outputoleinfo=TRUE

       

      Note:
      When outputoleinfo=TRUE only OLE metadata tags (listed in the metadataTags.txt file) are retrieved. When outputoleinfo=FALSE, or is not defined, then all metadata tags are retrieved. This setting applies to the Custom Regions in Office 2007 and 2010 documents such as Word, Excel, and PowerPoint.

       

  6. Under [Options], change the value wantHTMLView=FALSE to wantHTMLView=TRUE.

     

    Note:
    This change is needed to enable View as Web Page with the 1.1.5 version of the IM filterpack for supported formats on Windows.

     

  7. Save the opentext.ini file.

  8. Restart the Content Server services.
