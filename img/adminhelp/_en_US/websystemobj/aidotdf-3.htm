OpenText Document Filters (OTDF) is the library that communicates with DCS to detect MIME types, provide text extraction, metadata extraction, and HTML view generation.

The library forwards TextExtraction, MIMEtype detection, and HTML view generation requests from DCS to OpenText Document Filters through the use of filenames or buffer content.

All text extracted by OTDF is returned to DCS using UTF-8 encoding. The text for HTML output can be extracted as metadata only output, or text only output.

OpenText Document Filters creates a temporary file stored on disk, except for Microsoft Office format 95-2013 documents, since it is not yet set up to handle extraction requests on buffer content for most formats.

OTDF will use the TmpDir setting in the TmpDir=directory-path parameter in the [FilterEngine] section of the opentext.ini file as its temp directory location, instead of the previously used Temp Dir setting in the image.ini file.

The location of the temporary files is defined by the ‘tempdir’ setting in the MachineNamexportxxxxxdcsServer.ini file.

When a file buffer is received for a format that OTDF does not support, a temporary file is created and used by OTDF while converting the buffer content. The name of the temporary file is dcs_inDDMMYYHHMMSSfltrTHREADID. Each file is uniquely identified using the day, month, year, hour, minute, second it was created, and the processing thread identifier, for example dcs__in170610124733fltr6184. The file is removed by OTDF when the conversion process is finished.

When a file is too large to return by buffer for pure indexing, or if a View as Web Page request is made that includes PNG images, an output file is created and used by OTDF and then passed back to the DCS. The name of this file is dcs_outDDMMYYHHMMSSfltrTHREADID.

When performing a View as Web Page request OTDF creates an output image file and passes it to the DCS using a file named dcs_outDDMMYYHHMMSSfltrTHREADID_x.png.

The _x value is a further unique identifier since there can be many output image files.

All temporary file names are prepended with a DCS prefix, so these temporary files can be cleaned up by OTDF when the conversion is completed.
