You configure a Document Conversion process by modifying the parameters or command line arguments on the Specific tab of its Properties page. For more information about Document Conversion processes, see Configuring Document Conversion Processes.

Modifying Command Line Arguments

The command line argument values that you specify for a specific Document Conversion process are stored in the otadmin.cfg file. The default and valid values for each argument are described in the [FilterEngine] section of the opentext.ini file. You modify the [FilterEngine] section of the opentext.ini file when you want to globally configure all Document Conversion processes at your Content Server site.

 

Note:
If an argument appears in both the opentext.ini file and the command line of a particular Document Conversion process, the value on the command line overrides the value in the opentext.ini file for the process.

 

The following are the Document Conversion process command line arguments you can modify:

  • defaulttype

  • html

  • logfile

  • template

  • summary

  • summarysentences

The Specific tab of the Document Conversion Properties Page displays the following information:

Setting Values
Name Specify a name for the Document Conversion process.
Description Specify a description for the Document Conversion. This information appears on the General tab when viewing the properties of the Document Conversion.
Host Specify the Admin server on whose host you want the Document Conversion process to run.
Read Data From section:  
IPool Base Area Specify the absolute directory path of the data interchange pool (IPool) that you want the Document Conversion process to read data from.
Process Choose the process that you want to precede the Document Conversion process in the data flow.
 

Note:
If the data flow already contains at least one producer or intermediate process, the names of the processes appear in the Process drop-down list.

 
Write Data To section:  
Add a Write iPool Optionally, click the Add a Write iPool button to configure a second process that follows the Document Conversion process.
iPool Base Area Specify the absolute directory path of the data interchange pool (IPool) to which you want the Document Conversion process to split data to.
Process Choose the process that you want to follow the Document Conversion process in the data flow.
 

Note:
If the data flow already contains at least one intermediate or consumer process, the names of the processes appear in the Process drop-down list.

 
Schedule section:  
Manual Select Manual if you want the Document Conversion to process data when initiated by you, or Scheduled if you want the Document Conversion process to run automatically.
At this time Set the time in the drop-down lists, and click the days of the week you want the Document Conversion process to run.
Interval Select the interval at which you want the Document Conversion process to run.
Wait Set the values in the drop-down lists to select the interval at which you want the Document Conversion process to run.
Persistent Select Persistent which allows the Document Conversion process to run continuously. If the process terminates unexpectedly, its Admin server restarts it automatically.
Sleep Interval Set the values in the drop-down lists to select the interval for which you want the Document Conversion process to sleep before restarting.
Stop Message Port Specify the port number to which Content Server sends a shutdown message, and then shuts down the Document Conversion process properly. Do not modify this setting unless you are instructed to do so by OpenText Customer Support.
Maximum Good Exit Code Specify the maximum error code number for which the Admin server will automatically restart the Document Conversion process if it fails.
 

Tip:
If you type -1 in the Maximum Good Exit Code field, the Document Conversion process will terminate immediately when it stops. The Admin service will not restart if the Document Conversion process fails.

 
Do not modify this setting unless you are instructed to do so by OpenText Customer Support.

 

Element Description
Status Specifies whether or not the Document Conversion process is running.
Enable System Management Controls whether Content Server monitors this process to detect when it returns error messages. If the process returns an error message, Content Server records the message in the database. System management is enabled by default. For more information about configuring Content Server to send you email alerts when this or other data flow processes encounter errors, see To Enable Error-Checking and E-Mail Delivery.
Admin Server Specifies the shortcut of the Admin server on whose host computer the Document Conversion process runs. Do not modify the Admin server shortcut for the Document Conversion process unless necessary (for example, if the Admin server host is no longer available).
Command Template Specifies the Template used to generate the command line for the Document Conversion process. Do not modify this value unless you are instructed to do so by OpenText Customer Support.
Command Line Specifies the command line that Content Server uses to run the Document Conversion process. You cannot modify the value of this field.
Start Directory Specifies the directory in which the Document Conversion process starts. The Document Conversion process must run in the Content Server_home/filters directory. Do not modify this value unless you are instructed to do so by OpenText Customer Support.
Temporary Directory Specifies the directory in which the Document Conversion process stores the temporary files that it requires. By default, the temporary directory is the directory in which your operating system stores temporary files. You can specify a particular directory in which this particular Document Conversion process will store its temporary files. To do this, type -tmpdir in the Switch field, type the absolute path of the directory in the Value field, and then click the Update button. In UNIX operating systems, OpenText recommends that you use the /tmp directory.
Additional Command Line Specifies the optional arguments that you want to add to the command line for the Document Conversion process. You must type the names and values of the arguments that you want to add or change by separating multiple argument/value pairs with spaces:-argument value -argument value.
Information Opens the IPool Information page, which specifies the directory from which the Document Conversion process reads data and the directory to which it writes data. Do not modify these values unless you move an IPool, or you are instructed to do so by OpenText Customer Support.
Start Options Specifies the start settings for the Document Conversion process. There are three start settings:
•

Manual, which lets you start the Document Conversion process manually whenever you want it to run

•

Scheduled, which lets you schedule the day and time on which the Document Conversion process runs

•

Persistent, which runs the Document Conversion process continuously.

Stop Options Specifies the stop settings for the Document Conversion process. By default, Content Server sends a shutdown message to the port specified in the Admin Port field and then shuts down the process properly. The Maximum Good Exit Code field specifies the maximum error code number for which the Admin server will automatically restart the Document Conversion process, if it fails. By default, the value of the Maximum Good Exit Code field is set to -1, which means that the Admin server does not restart the Document Conversion process if it fails. Do not modify these stop options unless you are instructed to do so by OpenText Customer Support.
Start/Stop Allows you to maintain Data Flows, depending on its current state. If the process is running or scheduled to run, the Stop button appears. If the process is stopped, the Start button appears.
Resynchronize Resynchronizes information about this process (in the otadmin.cfg file of the process's Admin server) with the information in the Content Server database.
Update Submits the changes you make on this page to the Content Server server.
Reset Resets the information on this page to its state when opened.
