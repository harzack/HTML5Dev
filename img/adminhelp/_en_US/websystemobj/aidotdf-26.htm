The OpenText Document Filters (OTDF) accept Word, Excel, and PowerPoint data that is passed from a buffer or a file to handle text extraction on the Windows, Solaris and Linux platforms.

In Content Server:

  • for Office 2013 documents, page preview, text extraction, and view as web page – raster are supported for Word, Excel, and PowerPoint, on all supported platforms

  • Print Preview is supported for Excel 2007 or 2010 documents, using the native settings defined in the file for:

    • paper size and orientation

    • page margins

    • auto shapes and charts in normal area

    • Fit to 1 page width, and/or 1 page tall

    • custom scaling “Adjust to”

    • hidden columns and rows

  • the text inside Smart Art in MSOffice 2007 or 2010 files is searched and extracted

  • text extraction and View as Web Page are supported from MSOffice documents for Word, Excel, and PowerPoint files that contain Hong Kong Character Set (HKCS) data, on Windows only

MS Word

OTDF will extract hidden text in Word as though it is not hidden.

For embedded OLE objects in Word, text is extracted only for objects in Office 95 - 2003 formats.

RTF

OTDF will extract the following types of embedded files from RTF documents on the Windows, Solaris, and Linux operating systems:

  • Word 97-2003, 2007, and later

  • RTF

  • Excel 97-2003, 2007, and later

  • PowerPoint 97-2003, 2007, and later

  • PDF

Metadata update

OTDF support default and custom metadata updates on the Windows, Solaris, and Linux operating systems for:

  • Word 2007, and later

  • Excel 2007, and later

  • PowerPoint 2007, and later

  • PDF

 

Note:
In the current version, only ASCII filenames are supported since the file systems on Solaris and Linux do not provide the required support.

 

MS Excel

Document Filters will extract text from hidden columns in Excel as though they are not hidden.

OTDF provide print view support for the Office Open XML Format files (.xlsx) with similar functionality as for Microsoft Excel file format files (.xls).

 

Note:
There is a difference between the native MS-Excel print view and Document Filters because the widths of columns and the heights of rows can be different. In Excel, the column width is device-dependent, for example, on the printer’s DPI setting, while the Document Filters output is device-independent. Consequently, the pagination in a Document Filters output file may not be identical to pagination in an Excel file.

 

Chart text will not be in line with the rest of the spreadsheet, and any comments will not be near the cells to which they refer, but they will be on the same data sheet.

The calculated numeric indexes along axes of graphs are not supported for text extraction, but Labels for the axes will be extracted. For example, a bar chart with values explicitly marked on the graph will be extracted. However, if the bar height is 8.5 and in Excel the y-axis shows “12, 10, 8, 6, 4, 2, 0”, Document Filters will not extract these numbers.
