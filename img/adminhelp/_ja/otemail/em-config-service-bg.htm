Email Services では、いくつかの設定がサポートされています。インデックスを作成するために抽出する電子メール フィールドを選択したり、標準 Content Server 属性をインデックス化するかどうかを選択できます。表示するために電子メール メッセージ本文を保存するかどうかを選択できます。また、ページ設定した電子メール フォルダーに表示する 1 ページあたりのアイテム数を選択することもできます。

 

注意:
電子メールの内容全体を検索するには、Content Server のフル テキスト検索を使用します。Content Server の [詳細検索] から個々の電子メール プロパティを検索できるようにするには、次のステップを実行する必要があります。

  • Email Services 設定で、インデックスが作成されるように電子メール フィールドを設定します。

  • 検索管理で [クエリ可能] が有効になるように電子メール フィールドを設定します。詳細については、Content Server 管理ヘルプ (Content Server インデックスおよび検索管理 – 検索の管理 – インデックス範囲の設定) を参照してください。

 

[電子メール サービスの設定] ページでは、Email Services に関して次の項目を設定できます。

[抽出可能な電子メール フィールド]

インデックスを作成するために抽出する電子メール フィールドを指定します。そのフィールドに対する検索を可能にするには、フィールドを抽出する必要があります。ただし、特定の電子メール フィールドを抽出しないと選択することで検索インデックスのサイズを削減し、メッセージのフル テキスト検索を行うこともできます。

 

注意:
インデックス作成用に特定の電子メール フィールドを抽出しないと選択した場合、他の Content Server 製品の機能に影響が出る可能性があります。

 
[抽出可能な属性]

電子メール用に標準 Content Server 属性のインデックスを作成するかどうかを選択します。インデックス作成のために抽出する項目を制限すると、特に大量の電子メールを扱っている組織では、検索インデックスに必要なメモリの量を少なくすることができます。

 

注意:

  • 標準の Content Server 属性を抽出しないと選択した場合、電子メールの保存や検索に Content Server を使用する全アプリケーションの機能に影響が出る可能性があります。

  • 標準 Content Server 属性を抽出しないと選択した場合、以下の属性が電子メール用にインデックス化されないため、システムでどの電子メールも検索できなくなります。

 

標準 Content Server 属性のインデックスを作成しない場合、次の属性のインデックスも作成されません。

OTSubType OTCreateDate OTTimeCompleted
OTSubTypeName OTCreateTime OTDateStarted
OTOwnerID OTModifyDate OTTimeStarted
OTDataID OTModifyTime OTStatus
OTRootSubType OTAssignedTo OTPriority
OTRightIDs OTAssignedToName OTReserved
OTName OTAssignedToFullName OTReservedDate
OTLocation OTDateAssigned OTReservedTime
OTCDMajor OTTimeAssigned OTReservedBy
OTCDMinor OTDateEffective OTReservedByName
OTDComment OTTimeEffective OTReservedByFullName
OTCatalog OTDateDue OTCreatedBy
OTDCategory OTTimeDue OTCreatedByName

[メッセージ本文の保存]

電子メール ノードを Content Server でプレビュー表示するために、電子メール メッセージの本文をメタデータとしてデータベースに保存するかどうかを選択できます。

 

注意:

  • Email Monitoring では、電子メール メッセージの本文を保存しないことをお勧めします。データベースが巨大になりすぎる可能性があります。

    電子メール メッセージの本文を表示用に保存しないことを選択した場合、電子メールの本文は HTML ビューで表示されなくなります。ただし、代わりに OpenText Document Filters によって生成されるビューで表示する場合は、メッセージの本文を表示することができます。詳細については、OpenText Document Filters - Administrator's Guide (LLESCF-AGD) を参照してください。

 
[追加の電子メール プロパティの抽出]

標準の電子メール プロパティの他に、カスタム電子メール プロパティを抽出して保存することができます。追加の電子メール プロパティを抽出してサーバー上に保存するように Microsoft Outlook を設定するには、[追加の電子メール プロパティの抽出] フィールドにプロパティのリストを追加する必要があります。エントリは 1 行に 1 つずつ、以下の形式で入力する必要があります。

<MailSystem>|<PropertyName>|<表示名>|<範囲名>

説明:

<MailSystem> には、プロパティを抽出するメール環境を指定します。値は OUTLOOK または LOTUS のいずれかです。

<PropertyName> には、メール システムで抽出するプロパティの名前を指定します。

Microsoft Outlook では、MAPI プロパティの<識別番号>を入力します。名前付きプロパティについては、プロパティの GUID とプロパティの名前の両方をコンマで区切り、1 つの文字列または番号として入力する必要があります。

Lotus Notes では、[ドキュメント プロパティ] の<フィールド名>を入力します。

 

注意:
プロパティが見つからない場合は、データベースに保存されていません。また、標準プロパティ (Subject、From、To、CC、SentDate、ReceivedDate) は、常に抽出されてカスタム プロパティとは別に保存されます。

 

<表示名> は Content Server の [プロパティ – 電子メール] タブのプロパティに使用する表示名です。

<範囲名> (オプション) は、インデックスを作成するために検索エンジンが使用する領域名です。追加の電子メール プロパティを検索可能にするには、範囲名を指定する必要があります。電子メール プロパティを検索可能にする必要がない場合は、この操作を省略できます。

例

OUTLOOK|0x0017|Importance Flag|EmailImportance

LOTUS|($Classification)|Lotus Classification

OUTLOOK|{00020329-0000-0000-C000-000000000046},Keywords|Classification|EmailClassification

[署名/暗号化されたメッセージ クラス]

署名されたメッセージや、署名および暗号化された電子メール メッセージに特定の Microsoft Outlook メッセージ クラスを定義することができます。デフォルトでは、Enterprise Connect は Content Server 内の特定のメッセージ クラスが署名されている、または暗号化されていると認識します。

Enterprise Connect は、これらのタイプを正しく処理したり、このセクションに入力されたプロパティを使用することができないので、これらのタイプの電子メールを Content Server に追加したユーザーに対して、適切なメッセージを表示できません。このセクションに含まれるメッセージのリストが完全でない場合は、リストを変更して、追加のメッセージを含めることができます。1 行に 1 つずつメッセージ クラスを入力します。

Email Services の設定を行うには

  1. [Content Server 管理] ページの [電子メール サービス管理] セクションで、[電子メール サービスの設定] リンクをクリックします。

  2. [電子メール サービスの設定] ページで、Email Services のオプションを選択します。

  3. [送信] をクリックします。
