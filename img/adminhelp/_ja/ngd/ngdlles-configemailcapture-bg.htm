保存された電子メールに適用されるサブタイプ、およびユーザーが添付ファイル付きの電子メールを Content Server に追加した時の添付ファイルの処理方法を指定できます。重複電子メールを特定する方法や、ユーザーが重複電子メールをフォルダーに追加する度にユーザーに通知されるか、あるいは重複電子メールは無視されるか指定することもできます。選択する方法は、電子メールおよび添付ファイルを保存する企業プロセスによって異なります。

電子メールのサブタイプ

電子メールのサブタイプ オプションでは、ユーザーが Content Server に追加したすべての電子メールに電子メールのサブタイプ (749) が適用されます。既定では、すべての電子メールが電子メールのサブタイプとして保存されます。次に、添付ファイルの処理方法を指定できます。電子メールの添付ファイルを、個別のコピーとしては保存しないと選択できます。また、元の電子メールと添付ファイルの他に、添付ファイルの各コピーを個別のドキュメントとして追加すると指定できます。必要に応じて、添付ファイルの別個のコピーを追加するか、電子メールおよび添付ファイルを単一のアイテムして保存するかを、電子メールごとにユーザーが決定できるようにすることができます。

 

ヒント:
OpenText Records Management モジュールをインストールし、相互参照機能を有効にしている場合は、相互参照を使用して、電子メールと添付ファイルをリンクできます。詳細については、 OpenText Enterprise Connect - Installation Guide (NGDCORE-IGD).

 

Microsoft Outlook での重複電子メール

既定では、Content Server に追加された電子メールの名前と、そのフォルダーに既に存在している電子メールを比較して、重複電子メールが検出されます。このオプションを選択すると、Enterprise Connect が追加先で重複電子メールを特定する度に、ユーザーに通知されます。設定によっては、ユーザーは重複電子メールの名前変更やメタデータの入力を要求される場合があります。

例えば、重複電子メールを特定するために既定の名前比較を使用しているとします。ユーザーが電子メールをフォルダーに追加し、その後、名前は同じで内容が異なる電子メールを同じフォルダーに追加すると、重複として特定されます。設定によっては、ユーザーは電子メールの名前変更やメタデータの入力を要求されます。

必要に応じて、ユーザーが Content Server に追加した電子メールに割り当てられているメッセージ ID と、そのフォルダーに既に存在している電子メールを比較して、重複電子メールが検出されるように指定することができます。メッセージ ID は受信者ごとに異なります。例えば、同じ電子メールが 2 人の異なるユーザーに送信される場合、電子メールのそれぞれのコピーに固有の ID が割り当てられます。その他の選択肢として、ユーザーが Content Server に追加する電子メールの外部メッセージ ID と、そのフォルダーに既に存在している電子メールの外部 ID を比較して、重複電子メールが検出されるように指定することができます。外部メッセージ ID は、ユーザーのドメイン外から受信される電子メールにのみ適用されます。メッセージ ID の比較オプションを選択した場合は、重複電子メールは対象にコピーまたは移動されず、ユーザーは重複電子メールに関する通知を受信しません。新しい電子メールのみが追加され、設定によっては、ユーザーにメタデータの入力が要求される場合があります。

例えば [重複する電子メールの確認には、メッセージ ID を使用します。] チェック ボックスをオンにしているとします。ユーザーが電子メールのバッチをフォルダーに追加し、そのフォルダーに既に存在している電子メールの 1 つに、バッチ内の電子メールのいずれかと同じメッセージ ID があります。ユーザーは、重複電子メールに関する通知は受信しません。設定によっては、ユーザーは、新しい電子メールのすべてのメタデータの入力を要求される場合があります。もう 1 つのケースとして、2 名のユーザーが同じ送信者から電子メールを受信するとします。両方のユーザーが同じフォルダーに電子メールをコピーした場合、電子メールには異なる ID があるので、その電子メールは重複とは見なされません。

例えば [重複する電子メールの確認には、外部メッセージ ID を使用します。] チェック ボックスをオンにしているとします。ユーザーは、外部ソースから受信した電子メールをフォルダーに追加します。同じ電子メールを受信した同僚がその電子メールのコピーを同じフォルダーに追加すると、その電子メールは重複と特定され追加されません。ユーザーは、重複電子メールに関する通知は受信しません。

 

重要項目:
管理者の場合は、最初に Enterprise Connect を設定する時に、重複電子メールの特定方法を決めることが重要になります。ユーザーが電子メールの操作を開始してからこの設定を変更すると、問題が発生する可能性があります。

 
 

注意:

  • この機能は、Microsoft Outlook にのみ適用されます。

  • この機能は、コンパウンド電子メールには反映されません。

 

コンパウンド電子メールのサブタイプ

コンパウンド電子メールのオプションは、Enterprise Connect に移行しようとしている、コンパウンド電子メールのサブタイプ (サブタイプ 557) を使用してきた OpenText Explorer ユーザーのみを対象としています。このオプションを選択すると、すべての電子メールおよび関連する添付ファイルがコンパウンド電子メールとして Content Server に追加されます。コンパウンド電子メールでは、電子メール メッセージおよび関連する添付ファイルが 1 つのフォルダーに結合されます。コンパウンド電子メールは、ドキュメントを表示するように設定されている場合、Enterprise Connect の リスト ぺイン および Easy Access ペイン内にのみ表示されます。コンパウンド電子メールは、Microsoft Outlook またはエクスプローラーのツリー ペイン、あるいは Microsoft Office の [ファイルを開く] ダイアログ ボックスには表示されません。ユーザーは閲覧ウィンドウでコンパウンド電子メールを表示できます。ユーザーがコンパウンド電子メールをオフラインにすることができるようにするには、コンパウンド電子メールのサブタイプを、オフラインにできるサブタイプのリストに追加する必要があります。詳細については、 オフラインにすることができるサブタイプの指定.

 

重要項目:
コンパウンド電子メールに必要な記憶領域は、標準的な電子メールのサブタイプで必要な記憶領域よりはるかに大きくなります。コンパウンド電子メールのサブタイプを適用する場合は、この要件を満たす十分なディスク容量があることを確認してください。
