ユーザーが Enterprise Connect を使用して電子メールを Content Server に追加する時、命名規則が電子メールに適用されます。既定の命名規則では、電子メール名として電子メールの件名が使用されます。区切り文字や除外は使用されません。次の方法で命名規則をカスタマイズすることができます。

  • 電子メール名で使用されるメッセージ プロパティを定義します。

  • 電子メールの名前を構成する個々のメッセージ プロパティの間に使用する区切り文字を定義します。

  • 電子メール名から除外する文字列を指定します。

  • 電子メールの日付の形式を指定します。

  • 同じ名前の電子メールの処理方法を指定します。

 

注意:

  • ユーザーが電子メールを Content Server に追加する時に、Enterprise Connect は指定された命名規則を適用しますが、ユーザーは必要に応じて電子メールの名前を変更できます。

  • [マイセッティング] ページを Enterprise Connect ユーザーが利用できるようにすると、ユーザーは独自の設定にあわせて電子メールの命名規則を設定できます。詳細については、 ツリー構造とリスト ペインを設定するには.

  • Enterprise Connect が、Content Server コンテナーで既に使用されている電子メールの名前を生成すると、ユーザーは電子メールを追加する前に名前の競合を解決するように求められます。あるいは Enterprise Connect が、追加先に既に存在する電子メールと区別するための接尾辞を付加して、電子メールの新しい名前を生成します。ユーザーが利用できるオプションは、管理設定に応じて異なります。ファイル名の競合の解決の詳細については、OpenText Enterprise Connect Plug-in for Content Server - ユーザー ヘルプ (NGDLLES-H-UGD)を参照してください。

  • ただし電子メールを電子メールのサブタイプとして保存し、メッセージ ID を照合して重複を特定する場合は、重複電子メールは追加先に追加されず、ユーザーは重複に関する通知を受信することはありません。重複コンパウンド電子メールはこのルールから除外されているため、重複コンパウンド電子メールの処理方法を決定する必要があります。
