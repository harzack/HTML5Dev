Content Server に追加したアイテムのメタデータを入力するように、ユーザーに要求するタイミングを指定できます。属性や他の情報が必要な場合に限りユーザーにメタデータの入力を要求するか、Content Server に追加したアイテムごとにメタデータの入力を要求するかを設定できます。

Records Management などの他の Content Server モジュールがインストールされている環境の場合、デフォルトでは、ユーザーが Content Server の 1 つのインスタンス内でアイテムをコピーまたは移動する際は必ず、メタデータを入力するよう求められます。しかし、ユーザーが Content Server の 1 つのインスタンス内でアイテムをコピーまたは移動する場合のうち、ソースと宛先とでカテゴリ要件が異なるという場合のみ、メタデータの入力を求めるように指定することも可能です。例えば、Enterprise Connect および Records Management がインストールされている環境の場合を考えてみましょう。[常に表示] の設定値を使用している場合、ユーザーが Content Server のインスタンス内でドキュメントをコピーまたは移動するたびに、ソースと宛先のメタデータ要件が同じかどうかには関係なく、メタデータを入力するよう求められます。[表示しない] の設定値を使用している場合は、宛先ではユーザーによるメタデータ入力は必要とされないがソース側ではカテゴリと属性が有効になっている場合に、ユーザーにメタデータの入力が求められます。ソース側と宛先のメタデータ要件が同じなら、ユーザーにメタデータの入力は求められません。

メタデータの取得に最も適したオプションを選択してください。
