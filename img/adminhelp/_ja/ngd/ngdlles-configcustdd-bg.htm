ユーザーが特定のタイプのフォルダーをファイル システムから Content Server にドラッグする時に、作成されるオブジェクトのタイプを指定することが必要な場合があります。そのためには、ユーザーがフォルダーを Content Server にドラッグする時に、フォルダー名に特定の拡張子を持つフォルダーが電子メール フォルダーやコンパウンド ドキュメントになるように指定します。この機能は、Microsoft Outlook でのみ利用できます。

デフォルトでは、値は提供されません。複数の拡張子を指定する場合は、コンマを使用して拡張子を区切ります。

例えば、ユーザーが [test.cd] という名前のフォルダーをファイル システムから Content Server にドラッグし、CD という拡張子を持つフォルダーはコンパウンド ドキュメントとして処理されると既に指定していたとします。このとき、CD 拡張子はフォルダー名から削除され、新しいコンパウンド ドキュメントが宛先に作成されます。そのフォルダー内のすべてのアイテムはアクションに組み込まれ、ユーザーは指定した設定にしたがってメタデータを提供するよう指示されます。メイン フォルダーに含まれるサブフォルダーは、このアクションに組み込まれません。宛先に同じ名前のアイテムが既にある場合は、指定済みの命名競合解決ルールが適用されます。許可されていないタイプのアイテムがフォルダーに含まれている場合は、アクションが破棄されます。
