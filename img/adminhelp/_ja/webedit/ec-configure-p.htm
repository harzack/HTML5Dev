Web 編集を設定するには:

  1. Content Server の [管理] ページで、[Web 編集管理] の下にある [Web 編集の設定] をクリックします。

  2. [Web 編集の設定] ページの [エディター] 領域で、使用可能な各エディターのボックスに数値を入力してエディターの設定を行います。

     

    ヒント:
    負の値を指定して、エディターを無効にすることができます。

     

  3. [Office Editor 設定] 領域の [Content Server 表示名] ボックスで、Office Editor デスクトップ クライアントに表示するリポジトリ名を指定します。

     

    ヒント:
    名前を指定しない場合、<host>/<contentserverfolder> 形式が使用されます。次に例を示します。intranet.opentext.com/intranet またはlocalhost:8080/otcs.

     

  4. [Office Editor の設定] 領域で、次のいずれかの操作を実行します。

    • Office Editor クライアントのインストールをユーザーに実行させる場合は、[Enable Installer Download] チェック ボックスをオンにします。これがデフォルトの設定です。

    • Office Editor クライアントのインストールをユーザーに実行させない場合は、[Enable Installer Download] チェック ボックスをオフにします。

  5. [Office Editor の設定] 領域の [新しいバージョンが利用可能になったとき] リストで、次のいずれかの操作を実行します。

    • 最新バージョンの Office Editor クライアントにアップグレードするようユーザーに指示しない場合は、[アップグレードの表示もしくはプロンプトを表示しない] を選択します。これがデフォルトの設定です。

    • 最新バージョンの Office Editor クライアントにアップグレードするかどうかをユーザーが決定できるようにする場合は、[ユーザーにアップグレードするオプションを提供するか、互換性のある古いバージョンの利用を継続する] を選択します。

    • ユーザーに最新バージョンの Office Editor クライアントに直ちにアップグレードしてもらう場合は、[強制的にアップグレード] を選択します。

     

    注意:
    [ユーザーにアップグレードするオプションを提供するか、互換性のある古いバージョンの利用を継続する] または [強制的にアップグレード] のいずれかのオプションを選択する場合は、[インストーラーのダウンロードを有効] チェック ボックスをオンにして、ユーザーが Office Editor クライアントをインストールできるようにする必要があります。

     

  6. [すべての [開く] で doc.Fetch を使用] チェック ボックスで、次のいずれかの操作を実行します。

    • ネイティブ アプリケーションの [Office Editor の MIME タイプ] セクションで指定したように、設定したすべてのドキュメントを開くには、このチェック ボックスをオフにします。このオプションを選択した場合、Office Editor は自動的にドキュメントをドキュメント キャッシュにダウンロードして、そこでドキュメントを開きます。これがデフォルトの設定です。

    • Content Server の既定の実装 (doc.Fetch) を使用してすべてのドキュメントを開くには、このチェック ボックスをオンにします。このオプションを選択した場合、ユーザーがドキュメントを開く時に、Office Editor は使用されません。

  7. [ドキュメント キャッシュを有効化] チェック ボックスで、次のいずれかの操作を実行します。

    1. Office Editor で編集したドキュメントの最近使用したすべてのバージョンを保持するには、このチェック ボックスをオンにします。これがデフォルトの設定です。

    2. Office Editor で編集したドキュメントの最近使用したすべてのバージョンを削除するには、このチェック ボックスをオフにします。

  8. [Office Editor の MIME タイプ] 領域で、次のいずれかの操作を実行します。

    • Office Editor を使用してすべてのタイプのドキュメントを開いて編集する場合は、[Support Office Editor for all documents] をクリックします。

    • Office Editor を使用して特定のドキュメント タイプのみを開いて編集する場合は、[Support Office Editor for selected MIME types] をクリックします。必要に応じて、サポートされる MIME タイプのリストを編集できます。これがデフォルトの設定です。

     

    注意
    この設定に他の MIME タイプを追加できますが、ご使用の環境のターゲット アプリケーションのためにこの機能を検証するには、実際の稼動環境で導入予定のアプリケーションを徹底的にテストする必要があります。この機能で見つかった問題については、OpenText カスタマー サポートに通知する必要があります。この機能は「現状のまま」提供されており、明示的または黙示的なあらゆる種類の保証が付随しません。この機能に関する問題は、解決が保証されず、事例ごとに評価されます。

     
     

    注意:
    WebDAV モジュールがインストールされている場合、サポートされる WebDAV MIME タイプに対して同様のリストが表示されます。このリストは必要に応じて編集できます。

     

  9. [新規ドキュメントの追加] 領域で、[ドキュメントの追加] ページの [新規] ドロップダウン リストに表示するドキュメント タイプを指定します。

  10. [送信] をクリックします。
