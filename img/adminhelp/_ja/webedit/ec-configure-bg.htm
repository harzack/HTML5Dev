[Web 編集管理] ページでは、サポートされているドキュメント タイプの編集時に使用するエディタの指定、Web からの自動インストールの無効化、サポートするドキュメントの MIME タイプの設定、新規追加機能でサポートするドキュメント タイプの指定が可能です。

エディターの優先順位の設定

エディター優先順位を設定することにより、サポートされているドキュメントの編集時に使用するエディターを指定します。設定できるエディターは、使用する Content Server モジュールによって異なります。ただし、OpenText Office Editor および OpenText WebDAV を設定することはできます。エディター優先順位を設定する場合は、各エディターに数値を割り当てます。エディターに割り当てる数値が大きいほど、優先順位が高くなります。

Office Editor 設定の指定

Office Editor デスクトップ クライアントの [Location] カラムに表示するリポジトリ名を指定できます。名前を指定しない場合、Office Editor により <host>/<contentserverfolder> の形式を使用して名前が作成されます。同じ名前で複数のリポジトリがある場合、最後に設定されたリポジトリが使用され、番号が以前の各バージョンに追加されます。例えば、重複するインスタンスの名前は [Content Server (2)] に変更されます。

 

注意:
Enterprise Connect がクライアントにインストールされている場合、指定したリポジトリ名は使用されません。この場合、Enterprise Connect プラグインの名前が代わりに使用されます。

 

[インストーラーのダウンロードを有効] チェック ボックスをオンにすると、ユーザーが Content Server Web UI から Office Editor クライアントをダウンロードしてインストールできるように設定されます。これがデフォルトの設定です。このチェック ボックスをオフにすると、ユーザーはダウンロードとインストールができなくなるため、インストールまたはアップグレードの手段を提供することが必要になります。

ユーザーが Office Editor クライアントの最新のバージョンにアップグレードするタイミングは、[新しいバージョンが利用可能になったとき] リストのオプションを使用して調整できます。古いバージョンの Office Editor クライアントを使用しているユーザーがドキュメントを開くか編集しようとする時の対応に、次のオプションのいずれかを指定できます。

  • ユーザーにアップグレードするよう指示しない場合は、[アップグレードの表示もしくはプロンプトを表示しない] を選択します。これがデフォルトの設定です。

  • ユーザーがアップグレードするかまたは作業を継続するかを選択できるようにするには、[ユーザーにアップグレードするオプションを提供するか、互換性のある古いバージョンの利用を継続する] を選択します。ユーザーには、表示されるリンクを使用して Office Editor クライアントの最新のバージョンにアップグレードするか、または現在のバージョンを使用して作業を継続するかのどちらかを選択するように指示するメッセージが表示されます。

  • アップグレードしてから作業を継続するようユーザーに強制するには、[強制的にアップグレード] を選択します。表示されるリンクを使用して Office Editor クライアントの最新のバージョンにアップグレードするように指示するメッセージが表示されます。

 

注意:
[ユーザーにアップグレードするオプションを提供するか、互換性のある古いバージョンの利用を継続する] または [強制的にアップグレード] のいずれかのオプションを選択する場合は、[インストーラーのダウンロードを有効] チェック ボックスをオンにして、ユーザーが Office Editor クライアントをインストールできるようにする必要があります。

 

Content Server では [開く] 機能の動作を設定できます。Content Server の既定の実装 (doc.Fetch) を使用してすべてのドキュメントを開くように指定できます。この場合、ユーザーがドキュメントを開く時に、Office Editor は使用されません。Office Editor をインストールしなくてもユーザーが Content Server ドキュメントを開けるようにする場合は、このオプションを選択します。あるいは、サポートされるすべての MIME タイプ (サポートの可否は、[MIME Types for Office Editor] セクションで指定されたオプションで決まります) は、Office Editor を使用して開くと指定することができます。この場合、Office Editor は自動的にドキュメントをドキュメント キャッシュにダウンロードして、そこでドキュメントを開きます。

最近使用されたドキュメントのキャッシュは有効または無効にすることができます。デフォルトでは、すべてのドキュメントがドキュメント キャッシュに保持されます。[ドキュメント キャッシュを有効化] チェック ボックスをオフにすると、使用されていないドキュメントはキャッシュから削除されます。例えば、このチェック ボックスをオフにした場合、ユーザーがドキュメントを開いて編集した後に閉じると、そのドキュメントはドキュメント キャッシュから削除されます。競合状態にあるドキュメントは、競合が解決されるまでドキュメント キャッシュに保持されます。また、使用中のドキュメントは、閉じるまでキャッシュに保持されます。この設定は次のタイプのドキュメントにのみ適用されます。

.docx .docm .doc .dotx
.dotm .dot .xlsx .xlsm
.xlsb .xls .xltx .xltm
.xlt .pptx .pptm .ppt
.potx .potm .pot .ppsx
.ppsm .pps .ppam .ppa
.mpp .mpt .vsdx .vssx
.vstx .vsdm .vssm .vstm
.vsd .vdw .vsw .vss
.vst .pdf .wpd  

 

注意:

  • この設定は、Enterprise Connect がインストールされていない場合に、Office Editor を使用して Content Server Web UI から編集されるドキュメントにのみ適用されます。この設定を Enterprise Connect で編集されるドキュメントに適用するには、Enterprise Connect に追加の変更を行う必要があります。詳細については、 OpenText Enterprise Connect - Installation Guide (NGDCORE-IGD).

  • この設定に行った変更を適用するには、Office Editor クライアントを再起動する必要があります。

 

MIME タイプの指定

エディターでサポートされる MIME タイプを指定して、特定のエディターを使用して編集できるドキュメントを指定できます。選択したエディターを対象として、Microsoft Word、Excel、PowerPoint、Project、Visio ドキュメントについて、サポートする MIME タイプを指定します。例えば、Office Editor では、すべての MIME タイプをサポートする、あるいは、特定のタイプのみをサポートすると指定することができます。必要に応じて、サポートされる MIME タイプのリストを編集できます。エディターとして WebDAV を使用している場合も、サポートされる MIME タイプのリストを編集できます。

サポートされる新しいドキュメント タイプの指定

特定のドキュメント タイプを有効または無効にすることで、ドキュメントの新規追加機能でサポートするドキュメント タイプを指定します。例えば、ユーザーによる Microsoft Word ドキュメントの新規追加を有効にし、PowerPoint ドキュメントの新規追加を無効にすることができます。指定したドキュメント タイプは、[ドキュメントの追加] ページの [新規] リストに表示されます。

 

ヒント:
新規追加機能を対象として、既定のドキュメントを変更することができます。ユーザーが新規ドキュメントを追加すると、関連するデスクトップ プログラムでは、管理者が指定した既定のドキュメントが自動的に開きます。例えば、Content Server インストールの support\webedit ディレクトリにある既定の空白ドキュメントを、使用するカスタムの既定ドキュメントに置き換えることで、Word 2010 ドキュメントを変更することができます。
