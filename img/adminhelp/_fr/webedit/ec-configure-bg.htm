Sur la page Gestion des modifications Web, vous pouvez spécifier l'éditeur utilisé pour modifier un type de document pris en charge, désactiver l'installation automatique en ligne, définir le type MIME des documents pris en charge et spécifier les types de documents pris en charge pour la fonctionnalité Ajout de nouveaux documents.

Définition de la priorité de l'éditeur

Définissez la priorité de l'éditeur afin d'indiquer quel éditeur doit être utilisé lors de la modification d'un document pris en charge. Les éditeurs configurables dépendent des modules Content Server utilisés, mais peuvent inclure OpenText Office Editor et OpenText WebDAV. Lorsque vous définissez la priorité de l'éditeur, vous attribuez une valeur numérique à chaque éditeur, les valeurs les plus hautes étant attribuées aux éditeurs prioritaires.

Spécification des paramètres Office Editor

Vous pouvez spécifier le nom du référentiel qui s'affiche dans la colonne Emplacement sur le poste client Office Editor. Si vous ne saisissez aucun nom, Office Editor en créera un en utilisant le format <hôte>/<dossiercontentserver> . Si plusieurs référentiels portent le même nom, le référentiel configuré en dernier est utilisé et un numéro est ajouté à chaque version précédente. Par exemple, une instance dupliquée serait renommée Content Server (2).

 

Note :
Le nom du référentiel que vous spécifiez n'est pas utilisé si Enterprise Connect est installé sur le client. Dans ce cas, le nom du plug-in Enterprise Connect est utilisé.

 

Lorsque vous activez la case à cocher Activer le téléchargement du programme d'installation, vous permettez aux utilisateurs de télécharger et d'installer le client Editeur Office à partir de l'interface utilisateur Web du serveur de contenu. C'est le paramètre par défaut. Si vous désactivez cette case à cocher, les utilisateurs ne pourront pas procéder au téléchargement et à l'installation, et vous devrez vous charger des installations et mises à niveau.

Vous pouvez déterminer le moment où les utilisateurs effectuent leur mise à niveau vers la version plus récente du client Editeur Office au moyen des options figurant dans la liste Quand une nouvelle version est disponible. Lorsque des utilisateurs travaillant avec une ancienne version du client Editeur Office essaient d'ouvrir ou de modifier un document, vous pouvez spécifier l'une des options suivantes :

  • Pour que les utilisateurs ne soient jamais invités à effectuer les mises à niveau, choisissez l'option Ne pas afficher la mise à niveau ni demander aux utilisateurs de l'effectuer. C'est le paramètre par défaut.

  • Pour permettre aux utilisateurs d'installer les mises à niveau ou de poursuivre leur travail, choisissez Fournir aux utilisateurs la possibilité de mettre à niveau ou de continuer à utiliser une version compatible plus ancienne. Les utilisateurs reçoivent un message les invitant à effectuer une mise à niveau vers la version la plus récente du client Editeur Office au moyen du lien fourni, ou à continuer à utiliser la version actuelle.

  • Pour obliger les utilisateurs à effectuer la mise à niveau pour pouvoir continuer, choisissez Forcer les utilisateurs à mettre à niveau. Les utilisateurs reçoivent un message les invitant à effectuer une mise à niveau vers la version la plus récente du client Editeur Office au moyen du lien fourni.

 

Note :
Si vous sélectionnez l'option Fournir aux utilisateurs la possibilité de mettre à niveau ou de continuer à utiliser une version compatible plus ancienne ou Forcer les utilisateurs à mettre à niveau, vous devez également activer la case à cocher Activer le téléchargement du programme d'installation pour permettre aux utilisateurs d'installer le client Editeur Office.

 

Vous pouvez configurer la manière dont la fonction Ouvrir dans Content Server est utilisée. Vous pouvez spécifier que tous les documents s'ouvrent à l'aide de la mise en œuvre par défaut Content Server (doc.Fetch). Dans ce cas, Office Editor n'est pas utilisé quand un utilisateur ouvre un document. Sélectionnez cette option si vous voulez que les utilisateurs puissent ouvrir les documents Content Server sans avoir à installer Office Editor. Vous pouvez également indiquer que tous les types MIME pris en charge, tel que défini par les options spécifiées dans la section Types MIME pour Office Editor, s'ouvrent à l'aide de Office Editor. Dans ce cas, Office Editor télécharge automatiquement le document dans le cache avant de l'ouvrir.

Vous pouvez activer ou désactiver la mise en cache des documents utilisés récemment. Par défaut, tous les documents sont conservés dans le cache de documents. Si vous désactivez la case à cocher Activer la mise en cache des documents, les documents inutilisés sont supprimés de la mémoire cache. Par exemple, si la case à cocher est désactivée et que l'utilisateur ouvre, modifie, puis ferme un document, ce dernier est supprimé du cache de documents. Les documents conflictuels demeurent dans le cache de documents jusqu'à ce que le conflit soit résolu. En outre, les documents en cours d'utilisation demeurent dans le cache jusqu'à ce qu'ils soient fermés. Ce paramètre s'applique uniquement aux types suivants de documents.

.docx .docm .doc .dotx
.dotm .dot .xlsx .xlsm
.xlsb .xls .xltx .xltm
.xlt .pptx .pptm .ppt
.potx .potm .pot .ppsx
.ppsm .pps .ppam .ppa
.mpp .mpt .vsdx .vssx
.vstx .vsdm .vssm .vstm
.vsd .vdw .vsw .vss
.vst .pdf .wpd  

 

Notes :

  • Ce paramètre s'applique uniquement aux documents qui sont modifiés à partir de l'interface utilisateur Web du serveur de contenu au moyen de l'Editeur Office, lorsque Enterprise Connect n'est pas installé. Vous devez effectuer des modifications supplémentaires dans Enterprise Connect pour appliquer ce paramètre aux documents modifiés dans Enterprise Connect. Pour plus d'informations, voir OpenText Enterprise Connect - Installation Guide (NGDCORE-IGD).

  • Les utilisateurs doivent redémarrer le client Editeur Office pour appliquer les modifications apportées à ce paramètre.

 

Spécification des types MIME

Vous pouvez spécifier les types MIME pris en charge par les éditeurs afin d'indiquer quels documents peuvent être modifiés avec un éditeur spécifique. Pour chaque éditeur, vous devez indiquer les types MIME pris en charge pour les documents Microsoft Word, Excel, PowerPoint, Project et Visio. Par exemple, vous pouvez indiquer que Office Editor prend en charge tous les types MIME ou uniquement les types spécifiés. Vous pouvez modifier la liste des types MIME pris en charge à votre convenance. Si vous utilisez WebDAV comme éditeur, vous pouvez également modifier la liste des types MIME pris en charge.

Spécification des types de nouveaux documents pris en charge

Pour spécifier les types de documents pris en charge pour la fonctionnalité Ajouter de nouveaux documents, vous activez ou désactivez les types de documents spécifiques. Par exemple, vous pouvez autoriser les utilisateurs à ajouter de nouveaux documents Microsoft Word, mais pas PowerPoint. Les types de documents pris en charge que vous avez spécifiés s'affichent dans la liste Ajouter un nouveau dans la page Nouveau document.

 

Astuce :
Vous pouvez modifier les documents par défaut pour la fonctionnalité Ajouter un nouveau. Quand un utilisateur choisit d'ajouter un document, le document par défaut que vous avez spécifié s'ouvre automatiquement dans le logiciel associé. Par exemple, vous pouvez modifier un document Word 2010 en remplaçant le document vierge par défaut dans le répertoire support\webedit de votre installation Content Server par le document par défaut personnalisé que vous souhaitez utiliser.
