Pour configurer les paramètres de modification Web :

  1. Dans la page de gestion de Content Server, sous Gestion des modifications Web, cliquez sur Configurer les paramètres de modification Web.

  2. Dans la zone Editeurs de la page Configuration des paramètres de modification Web, spécifiez la priorité de chaque éditeur en entrant une valeur numérique dans la case située en regard de chaque éditeur.

     

    Astuce :
    Pour rendre un éditeur indisponible, entrez une valeur négative.

     

  3. Dans la zone Office EditorParamètres, dans le champ Nom affiché de Content Server , saisissez le nom du référentiel tel qu'il doit s'afficher sur l'ordinateur client Office Editor.

     

    Astuce :
    Si vous ne saisissez aucun nom, le format <hôte>/<dossiercontentserver> est utilisé. Par exemple, intranet.opentext.com/intranet ou localhost:8080/otcs.

     

  4. Dans la section Office EditorParamètres, effectuez l'une des opérations suivantes :

    • Activez la case à cocher Activer le téléchargement du programme d'installation si vous voulez que les utilisateurs installent le client Office Editor. C'est le paramètre par défaut.

    • Désactivez la case à cocher Activer le téléchargement du programme d'installation si vous ne voulez pas que les utilisateurs puissent installer le client Office Editor.

  5. Dans la zone Office EditorParamètres, dans la liste Quand une nouvelle version est disponible, effectuez l'une des actions suivantes :

    • Choisissez Ne pas afficher la mise à niveau ni demander aux utilisateurs de l'effectuer si vous ne voulez pas inviter les utilisateurs à effectuer une mise à niveau vers la version la plus récente du client Office Editor. C'est le paramètre par défaut.

    • Choisissez Fournir aux utilisateurs la possibilité de mettre à niveau ou de continuer à utiliser une version compatible plus ancienne si vous voulez permettre aux utilisateurs d'effectuer une mise à niveau vers la version la plus récente du client Office Editor.

    • Choisissez Forcer les utilisateurs à mettre à niveau si vous voulez que les utilisateurs effectuent immédiatement une mise à niveau vers la version la plus récente du client Office Editor.

     

    Note :
    Si vous sélectionnez l'option Fournir aux utilisateurs la possibilité de mettre à niveau ou de continuer à utiliser une version compatible plus ancienne ou Forcer les utilisateurs à mettre à niveau, vous devez également activer la case à cocher Activer le téléchargement du programme d'installation pour permettre aux utilisateurs d'installer le client Editeur Office.

     

  6. Dans le champ Utiliser doc.Fetch pour toutes les ouvertures, effectuez l'une des opérations suivantes :

    • Décochez la case pour ouvrir tous les documents configurés, comme indiqué dans la section Types de MIME pour Office Editor, dans leurs applications natives. Si vous sélectionnez cette option, Office Editor télécharge automatiquement le document dans le cache avant de l'ouvrir. C'est le paramètre par défaut.

    • Cochez la case pour ouvrir tous les documents à l'aide de la mise en œuvre par défaut Content Server (doc.Fetch). Si vous sélectionnez cette option, Office Editor n'est pas utilisé quand un utilisateur ouvre un document.

  7. Dans la case à cocher Activer la mise en cache des documents, effectuez l'une des opérations suivantes :

    1. Activez la case à cocher pour conserver toutes les versions récemment utilisées des documents modifiés avec Office Editor. C'est le paramètre par défaut.

    2. Désactivez la case à cocher pour supprimer toutes les versions récemment utilisées des documents modifiés avec Office Editor.

  8. Dans la zone Types MIME pour Office Editor, effectuez l'une des opérations suivantes :

    • Cliquez sur Prise en charge de Office Editor pour tous les documents si vous voulez utiliser Office Editor pour ouvrir et modifier tous les types de documents.

    • Cliquez sur Prise en charge de Office Editor pour des MIME sélectionnés si vous voulez utiliser Office Editor pour ouvrir et modifier uniquement les types de documents spécifiés. Vous pouvez modifier la liste des types MIME pris en charge à votre convenance. C'est le paramètre par défaut.

     

    Attention
    Bien qu'il soit possible d'ajouter d'autres types MIME à ce paramètre, pour que cette fonctionnalité soit validée pour les applications cibles de votre environnement, vous devez réaliser un test exhaustif de l'application que vous voulez déployer dans un environnement de production. Tout problème de fonctionnalité doit être communiqué au service clientèle OpenText. Cette fonctionnalité est fournie « en l'état », sans garantie d'aucune sorte, expresse ou implicite. Les problèmes relatifs à cette fonctionnalité seront examinés au cas par cas, sans garantie de résolution.

     
     

    Note :
    Si le module WebDAV est installé, une liste similaire recense les types MIME WebDAV pris en charge. Vous pouvez modifier cette liste à votre convenance.

     

  9. Dans la zone Ajouter de nouveaux documents, cliquez sur les types de documents qui doivent s'afficher dans la page Ajouter un document, dans la liste Ajouter un nouveau.

  10. Cliquez sur Envoyer.
