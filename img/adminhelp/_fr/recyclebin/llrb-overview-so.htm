OpenText Recycle Bin permet aux utilisateurs qui suppriment accidentellement des documents dans Content Server de les récupérer facilement sans faire appel à leur administrateur Content Server.

Par défaut, Content Server inclut le module de récupération de document qui permet à un administrateur Content Server de restaurer des documents supprimés. Le module Corbeille introduit un volume dans lequel sont placés les objets pris en charge à supprimer et à partir duquel les utilisateurs peuvent les récupérer.

Le package Corbeille remplace l'opération standard de suppression de document. La Corbeille déplace les documents « supprimés » dans Content Server vers le volume Corbeille. Ce volume est structuré de manière telle que les éléments supprimés sont déplacés vers un dossier dont le nom correspond au format : <année> :< mois> : <jour>.

Il est possible de maintenir les versions, permissions et attributs des documents dans la corbeille. Les éléments peuvent être récupérés via l'interface Content Server en déplaçant manuellement les éléments de Corbeille vers leur emplacement d'origine ou un autre emplacement.

Le volume de Corbeille doit faire l'objet d'une maintenance régulière. Une politique courante consiste à supprimer définitivement les éléments qui ont atteint une date d'expiration prédéfinie.
