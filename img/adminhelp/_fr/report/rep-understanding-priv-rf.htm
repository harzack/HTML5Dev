Le module Content Server LiveReports permet aux utilisateurs de créer et d'utiliser des éléments Content Server appelés rapports LiveReport. Les rapports LiveReport sont de puissants outils permettant d'accéder directement au SGBDR sous-jacent dans Content Server au moyen d'instructions SQL.

Vous pouvez configurer les rapports LiveReports de façon à :

  • retourner des listes d'éléments qui répondent à certains critères ;

  • rapporter des informations statistiques sur le contenu d'une base de données sous forme de graphique circulaire présentant la proportion des éléments que possède chaque utilisateur ;

  • modifier des informations de la base de données, par exemple, réaffecter toutes les tâches d'un utilisateur qui a quitté l'organisation.

Gestion des privilèges et des droits d'accès aux rapports LiveReport

Etant donné que les rapports LiveReport vous permettent de modifier directement les informations contenues dans la base de données Content Server, OpenText vous conseille d'accorder avec parcimonie le droit aux utilisateurs de créer des rapports LiveReport.

 

Astuce :
Vous pouvez configurer Content Server pour empêcher LiveReports de modifier la base de données Content Server. Voir Configuration de la sécurité des rapports LiveReports.

 

Les utilisateurs n'ayant pas le droit d'en créer peuvent toutefois exécuter des rapports LiveReport s'ils disposent des autorisations nécessaires. Par exemple, vous pouvez permettre à tous les utilisateurs d'exécuter un rapport affichant la liste des éléments qui leur appartiennent ou qu'ils ont réservés. Les utilisateurs peuvent afficher les rapports LiveReports qu'ils sont autorisés à exécuter sur la page Gérer les rapports LiveReport, à laquelle ils accèdent en cliquant sur Rapports dans le menu général Personnel.

En limitant la création de rapports LiveReport à un nombre restreint d'utilisateurs compétents, vous vous prémunissez contre les éventuels dommages qu'ils peuvent infliger au contenu de la base de données. Une connaissance élémentaire de SQL et du schéma Content Server est une condition sine qua non pour pouvoir modifier les rapports LiveReport avec un minimum d'efficacité et de sécurité. Un utilisateur qui a le droit de modifier ou de créer des rapports LiveReport peut y inclure n'importe quelle instruction SQL. Il peut utiliser l'instruction SELECT pour récupérer des informations, mais également les instructions UPDATE et DELETE pour modifier et supprimer des informations. Pour plus d'informations sur les noms des tables et colonnes de la base de données Content Server, reportez-vous au guide Référence du schéma du système.

Vous pouvez octroyer les droits suivants à un utilisateur pour un rapport LiveReport :

  • Voir

  • Voir le contenu

  • Modifier

  • Modifier les attributs

  • Supprimer

  • Modifier les autorisations

Pour modifier des rapports LiveReport existants, un utilisateur doit bénéficier de l'autorisation Modifier pour ces rapports et du privilège de création pour le type d'élément des rapports LiveReports. Pour exécuter un rapport LiveReport, un utilisateur doit disposer de l'autorisation Voir le contenu sur ce rapport. Pour de plus amples informations sur les autorisations, reportez-vous à Autorisations et Gestion des autorisations.

Définition des objets autorisés dans les rapports LiveReport interactifs

Certains rapports LiveReports sont interactifs dans la mesure où ils permettent à leurs utilisateurs de spécifier des informations et de sélectionner des options. Un des types d'entrées pouvant être ajoutés à un rapport LiveReport est le type Objet. Quand ce type d'entrée est inclus dans un rapport LiveReport, il apparaît dans un menu permettant aux utilisateurs de sélectionner le type d'objet qui les intéresse. Par défaut, les types d'éléments que les utilisateurs peuvent sélectionner dans le menu Eléments sont :

  • Canal

  • Discussion

  • Document

  • Dossier

  • Rapport LiveReport

  • Projet

  • Tâche

  • Liste de tâches

  • Sujet

  • URL

Vous pouvez modifier la liste des types d'éléments inclus dans les menus Eléments en modifiant le paramètre objectSybTypes dans la section [reports] du fichier opentext.ini.

Le volume Rapports LiveReport Content Server initial contient un ensemble de rapports LiveReports fourni par OpenText. Vous pouvez modifier ces rapports pour répondre aux besoins de vos utilisateurs et ajouter de nouveaux rapports au volume Rapports LiveReport à l'aide du menu Ajouter un élément ou en les important dans votre système Content Server.

Les utilisateurs peuvent stocker les rapports LiveReports n'importe où dans Content Server, mais la page Gérer les rapports LiveReport affiche uniquement les rapports LiveReports du volume Rapports LiveReport que les utilisateurs sont autorisés à afficher.

Etant donné que le volume Rapports LiveReport est un type de dossier Content Server, vous pouvez y effectuer les mêmes tâches que dans les autres dossiers Content Server. Vous pouvez ajouter de nouveaux rapports LiveReports, octroyer des autorisations d'accès au volume et à son contenu, configurer le volume, etc. Pour plus d'informations sur l'ajout, la modification et l'exécution de rapports LiveReport, reportez-vous à Utilisation des rapports LiveReport de l'aide en ligne utilisateur de Content Server.
