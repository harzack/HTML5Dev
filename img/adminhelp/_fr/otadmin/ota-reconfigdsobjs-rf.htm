Lorsque vous installez une nouvelle instance de Content Server et que vous essayez de vous connecter à une base de données existante, vous devez reconfigurer les serveurs d'administration, les index et les processus de flux de données enregistrés dans la base de données, puis resynchroniser tous les serveurs d'administration enregistrés du site. Pour en savoir plus, voir Resynchronisation des serveurs d'administration.

 

Note :
Si vous resynchronisez un serveur d'administration qui gère un index Entreprise sans déplacer au préalable le répertoire de l'index Entreprise vers le chemin d'accès spécifié dans la base de données (ou sans modifier le chemin pour qu'il corresponde à l'emplacement correct), Content Server crée un index Entreprise vide à l'emplacement spécifié dans la base de données. Dans ce cas, vous devez extraire à nouveau les données Entreprise pour régénérer l'index Entreprise. Pour plus d'informations sur la réextraction de toutes les données, reportez-vous à la rubrique Vérification du contenu et réindexation.

 

Reconfiguration des serveurs d'administration

La reconfiguration d'un serveur d'administration s'effectue en vérifiant qu'il est disponible et que seuls des serveurs d'administration valables sont enregistrés avec lui.

  • Exécutez une commande ping vers tous les serveurs d'administration enregistrés pour vous assurer que le nouvel hôte Content Server peut y accéder.

  • Si l'un des serveurs d'administration enregistrés réside sur un hôte distant, assurez-vous que Content Server est installé sur cet hôte et qu'il fonctionne correctement. Pour plus d'informations sur l'installation de Content Server, reportez-vous au Guide d'installation de Content Server. Lorsque Content Server est installé et que les serveurs d'administration ont démarré, vous devrez peut-être modifier le numéro de port ou le mot de passe stockés dans la base de données.

  • Supprimez l'enregistrement des serveurs d'administration que vous ne souhaitez plus utiliser.

Reconfiguration des index

La reconfiguration d'un index s'effectue en vérifiant que tous les objets source de données liés sont valables et que le chemin de leur répertoire est correct.

  • Assurez-vous que les serveurs d'administration, les moteurs d'index et les répertoires d'indexation sont corrects pour cette nouvelle installation Content Server.

  • Pour utiliser à nouveau les index créés à l'aide de l'installation Content Server dans laquelle la base de données a été créée, assurez-vous que les fichiers d'index existent dans les emplacements du répertoire d'indexation spécifiés dans l'onglet Spécifique de la page Propriétés du moteur d'indexation. Si les répertoires d'indexation ne se trouvent pas aux chemins d'accès spécifiés, modifiez les chemins pour qu'ils correspondent à l'emplacement correct ou Déplacement des index les répertoires d'index vers les chemins spécifiés.

Reconfiguration des processus de flux de données

La reconfiguration des processus de flux de données s'effectue en vérifiant que les serveurs d'administration, les répertoires de lecture et d'écriture et les numéros de port sont corrects pour cette nouvelle installation Content Server.
