Un rapport est créé dès qu'une itération de vérification ou de correction de l'index se termine. Les rapports sont créés au format CSV (valeurs séparées par des virgules) et nommés en fonction de la date de création. Tous les rapports sont stockés dans le dossier des Rapports de vérification du Volume des objets système qui est également accessible par le biais du lien situé en bas de la page Configurer la vérification de l'index.

 

Note :
La colonne n°1 du rapport contient un 0 pour les lignes renfermant des informations de configuration et des statistiques. Elle contient un 1 pour les lignes contenant des données détaillées. Il est ainsi plus facile de classer les objets détaillés en réalisant un tri sur la première colonne.

 

Pour afficher les rapports de vérification

  1. Sur la page Volume des objets système, cliquez sur le lien Dossier Rapports de vérification.

  2. Sur la page Rapports de vérification, vous pouvez cliquer sur chaque rapport pour ouvrir une page Vue d'ensemble d'un document et exécuter un certain nombre de fonctions sur le rapport.

  3. Vous pouvez importer le rapport dans la plupart des applications de feuille de calcul pour consultation et analyse.

Le rapport de vérification de l'index comporte quatre sections : en-tête, configuration, statistiques et détails.

L'en-tête comprend le titre du rapport, la version du logiciel qui a généré le rapport, l'heure de génération du rapport et le nom du système.

La section de configuration est un lot de résultats des paramètres de vérification et de correction au moment où le rapport a été exécuté. Ces informations offrent une vue d'ensemble des données fournies dans le rapport. On y trouve également des informations complémentaires dans l'éventualité où un audit du statut de l'index de recherche serait nécessaire.

La section des statistiques renferme un certain nombre de totaux par types d'erreurs, comme illustré ci-dessous. Il est à noter que les chiffres affichés à la verticale ne sont PAS additionnés.

  • Les Erreurs récemment identifiées sont celles qui ont été identifiées pour la première fois lors de cette itération.

  • Les Tentatives de correction représentent le nombre d'objets pour lesquels une correction a été tentée au cours de cette itération. Lorsque la correction est désactivée, le chiffre sera toujours 0.

  • L'Itération marquée comme ne pouvant être rectifiée est le nombre total d'objets pour lesquels une troisième tentative de correction a été réalisée durant l'itération précédente, sans succès. Les objets ont été marqués comme ne pouvant pas être rectifiés. Aucune autre tentative de correction ne sera réalisée. Le nombre d'objets ne pouvant pas être rectifiés reflète les objets pour lesquels une correction a été tentée. Il ne comprend PAS les objets pour lesquels aucune correction n'a été tentée. Par exemple, les objets erronés sur des partitions en lecture seule ne sont pas inclus dans ce total.

  • La Dernière itération corrigée représente le nombre d'objets pour chaque type d'erreur qui a été corrigé au cours de l'itération précédente et qui n'est plus signalé comme erreur dans l'itération actuelle.

  • Le Total des objets ne pouvant pas être rectifiés représente le nombre d'objets marqués comme ne pouvant pas être rectifiés et pour lesquels aucune correction n'a été tentée. Il comprend les objets ne pouvant pas être rectifiés, les nouveaux comme anciens.

  • Le Total des objets incorrects est le nombre d'objets pour chaque type pour lequel une erreur a été découverte au cours de la dernière itération.

Certaines fonctions pratiques ont été ajoutées pour les statistiques, notamment le nombre total d'objets dans l'index, le nombre d'objets avec des avertissements de contenu ou de métadonnées, et le nombre d'entrées de la liste détaillée des objets.

La liste détaillée des objets est la dernière section du rapport. Elle contient une liste de tous les objets pour lesquels le vérificateur signale une erreur, dans la limite de la valeur maximale indiquée en section Configuration du rapport de vérification.

Pour faciliter la manipulation, le rapport est limité à 10 000 éléments par fichier. Lorsque le nombre d'objets est supérieur, ceux-ci sont placés dans des fichiers distincts et signalés dans la description. Par exemple : (2 sur 7).
