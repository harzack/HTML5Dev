Le service de conversion de documents (DCS) est un processus qui convertit des documents au format HTML ou au format de texte simple. Dans Content Server, le DCS est géré par les serveurs d'administration. Il fournit deux services dans Content Server : il convertit des éléments pour que les utilisateurs puissent les mettre en évidence ou les afficher dans Content Server ; il convertit les données lorsqu'elles sont transmises via un flux de données d'indexation. Dans les flux de données, le DCS est appelé « processus de conversion de documents ». Lorsque vous configurez le DCS, vous déterminez son fonctionnement lors de la conversion d'éléments pour des opérations Content Server, telles que la mise en évidence des résultats et l'affichage. Pour plus d'informations sur la configuration du DCS pour la conversion de données dans des flux de données, reportez-vous à la rubrique Configuration des processus Conversion de documents. La configuration du DCS comprend son activation, sa configuration proprement dite et le contrôle de son statut.

Activation du DCS

Lorsque vous installez ou mettez à niveau Content Server, vous pouvez activer le DCS qui sera géré par votre serveur d'administration par défaut. A chaque fois que vous ajoutez un nouveau serveur d'administration ou configurez un serveur d'administration existant, vous pouvez également activer un DCS qui sera géré par chaque serveur. Chaque serveur d'administration ne peut gérer qu'un seul DCS. En activant plusieurs DCS, vous avez la possibilité de répartir la charge de conversion de documents sur plusieurs serveurs d'administration, ce qui vous permet d'améliorer les performances générales de votre système. Si un serveur d'administration devient indisponible, le DCS qu'il gère est également indisponible ; cependant, si d'autres serveurs d'administration sont disponibles et qu'ils gèrent un DCS, ces DCS peuvent se charger de la conversion de documents. Vous pouvez à tout moment désactiver un DCS sur votre site ; cependant, au moins un DCS doit être activé pour que la conversion de documents puisse avoir lieu.

Configuration des paramètres DCS

Après avoir activé les DCS, vous pouvez les configurer. Chaque DCS dispose de son propre fichier de configuration qui prend en compte les paramètres que vous configurez.

 

Note :
Le fichier de configuration DCS est stocké dans le répertoire config d'une installation Content Server principale.

 

Lorsque vous activez un DCS, Content Server configure pour vous la plupart des paramètres DCS. Vous pouvez à tout moment modifier ces paramètres pour personnaliser le fonctionnement du DCS. Les numéros de port sont l'un des paramètres que Content Server configure automatiquement. Content Server sélectionne les ports disponibles pour le DCS et configure le numéro de port DCS et Admin. Le numéro de port Admin est le numéro du port sur lequel le DCS est en réception pour les informations provenant du serveur d'administration ; le numéro de port DCS est le numéro du port que les opérations Content Server utilisent pour communiquer avec le DCS. Les autres paramètres que Content Server peut configurer automatiquement comprennent le fichier journal, à savoir l'emplacement du fichier enregistrant les informations de journalisation pour le DCS que vous configurez ; le répertoire de lancement et le chemin de filtre, à savoir les emplacements de stockage des filtres de conversion de documents ; le fichier de règles, à savoir l'emplacement de stockage du fichier de règles de conversion de documents. Le fichier de règles de conversion de documents est un fichier utilisé par le DCS pour déterminer les filtres de conversion à utiliser.

Le répertoire des fichiers temporaires est un paramètre qui n'est pas configuré automatiquement ; vous devez donc le spécifier. Le répertoire des fichiers temporaires est un emplacement de stockage temporaire dont ont besoin les filtres de conversion.

 

Note :
Content Server ajoute le numéro du port Admin au nom des fichiers journaux du DCS à leur création. Par exemple, le nom du fichier journal correspondant à un DCS dont le numéro du port Admin est 5868 est dcs_5868.log. L'emplacement du répertoire de lancement et le chemin de filtre devraient correspondre ; cependant, sur les systèmes d'exploitation Windows, ils doivent impérativement correspondre.

 

Content Server configure également les valeurs par défaut des paramètres suivants :

  • Niveau de consignation : cette option permet de spécifier le type d'informations de consignation qui sont enregistrées (par exemple, 0 indique qu'aucune consignation n'aura lieu ; 1 indique que seuls les erreurs et les documents qui n'ont pas pu être indexés sont enregistrés ; 2 indique que les erreurs et les événements de conversion de documents, tels que les informations de chargement des filtres, sont enregistrés ; 3 indique que les erreurs, les événements de conversion de documents et les statistiques de performance sont enregistrés ; 4 indique que les erreurs, les statistiques de performance et les événements d'IPool sont enregistrés).

  • Délai d'attente de la conversion : cette option permet de spécifier le nombre maximal de secondes pendant lesquelles Content Server attend avant de retourner une erreur lorsqu'il essaye d'afficher des documents convertis.

  • Nombre de threads : cette option permet de spécifier le nombre de demandes de conversion de documents que le DCS peut traiter simultanément.

  • Taille de la file d'attente : cette option permet de spécifier le nombre maximal de documents que le DCS peut mettre en attente avant la conversion.

 

Note :
Les valeurs que vous indiquez pour le nombre de threads et la taille de la file d'attente dépendent des variables de votre site Content Server, y compris le nombre d'utilisateurs, le nombre d'ordinateurs consacrés aux DCS et la quantité de ressources système disponibles.

 

Suivi du statut du DCS

Le suivi du statut d'un DCS vous permet de gérer ses performances. Les DCS peuvent avoir les statuts suivants :

  • En cours d'exécution : le DCS est en cours d'exécution.

  • Pas en cours d'exécution : le DCS est arrêté et n'est pas programmé pour être exécuté.

  • Désactivé : le DCS a été manuellement désactivé.

  • Manquant dans le serveur d'administration : le serveur d'administration qui gère le DCS ne peut pas trouver la configuration de celui-ci. Vous devez resynchroniser le serveur d'administration pour recréer le DCS. Pour plus d'informations sur la resynchronisation des serveurs d'administration, reportez-vous à la rubrique Resynchronisation des serveurs d'administration.

  • Erreur renvoyée : une erreur DCS s'est produite.

  • Erreur du serveur d'administration renvoyée : une erreur s'est produite avec le serveur d'administration lors de la récupération du processus DCS. Dans ce cas, le serveur d'administration est peut-être arrêté.

  • Etat inconnu : le DCS ne peut pas communiquer avec le serveur d'administration.
