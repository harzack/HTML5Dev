Pour restaurer un index à l'aide de l'Assistant de restauration :

  1. Redémarrez les flux de données associés à la source de données dont vous souhaitez restaurer l'index.

  2. Redémarrez le processus de recherche associé à la source de données dont vous souhaitez restaurer l'index.

  3. Arrêtez le gestionnaire de sauvegarde associé à la source de données dont vous voulez restaurer les index.

  4. Supprimez tous les fichiers du répertoire de l'index que vous souhaitez restaurer.

     

    Note :
    Le chemin du répertoire de l'index d'une source de données est stocké dans le champ Répertoire d'indexation de l'onglet Spécifique de la page Propriétés pour le processus du moteur d'indexation de la source de données.

    Ne supprimez pas le fichier de signature de l'index ni le fichier FieldModeDefinitions.ini lorsque vous nettoyez le répertoire d'indexation avant d'effectuer la restauration. Si vous supprimez le fichier de signature, le processus de recherche ne démarrera probablement pas en raison de l'absence de ces éléments.

    Si les processus sont encore en cours d'exécution, l'Assistant de restauration échoue étant donné qu'il ne peut pas remplacer des fichiers en cours d'utilisation.

     

  5. Sur la page Volume des objets système, cliquez sur l'icône Fonctions de la source de données dont vous voulez restaurer l'index, puis sélectionnez Maintenance.

  6. Sur la page Maintenance de sources de données, cliquez sur le bouton radio Restaurer l'index, puis cliquez sur le bouton OK. Si vous n'avez pas créé de gestionnaire de sauvegarde auparavant, vous êtes invité à en ajouter un maintenant. Sur la page Confirmation de la création d'un gestionnaire de sauvegarde, cliquez sur le bouton OK. Sur la page Ajouter Gestionnaire de sauvegarde, saisissez les informations requises, puis cliquez sur le bouton Ajouter.

  7. Sur la page Assistant de restauration de l'index de recherche de sources de données, à l'étape Partitions, sélectionnez toutes les partitions à restaurer, puis cliquez sur le bouton Suivant.

  8. A l'étape Image de sauvegarde, pour la première partition que vous souhaitez restaurer, saisissez le chemin absolu du répertoire qui contient la dernière image de sauvegarde dans le champ Emplacement de l'image, puis cliquez sur bouton Suivant.

     

    Note :
    Vous devez inclure le chemin du répertoire dans lequel est stockée l'image de sauvegarde. L'Assistant de restauration a besoin de la dernière image de sauvegarde pour définir l'ensemble d'images à restaurer.

     

  9. A l'étape Statut du fichier, cliquez sur le bouton Vérifier le statut pour actualiser la progression du processus.

  10. Lorsque l'Assistant de restauration renvoie une page indiquant que l'ensemble des images de sauvegarde à restaurer a été déterminé, cliquez sur le bouton Suivant.

  11. A l'étape Statut de restauration, lorsque l'Assistant de restauration affiche la liste des images de sauvegarde à restaurer, cliquez sur le niveau de transmission des messages d'erreur de votre choix dans la liste déroulante Niveau d'alerte :

    • Aucun, qui est l'option par défaut ;

    • Informations, qui permet de notifier au destinataire le statut général et d'autres informations système ;

    • Avertissement, qui permet de notifier au destinataire les erreurs non critiques dans le système ;

    • Erreur, qui permet de notifier au destinataire les erreurs système générales ;

    • Erreur grave, qui permet de notifier au destinataire les erreurs système critiques.

     

    Note :
    Pour plus d'informations sur la transmission des messages d'erreur, reportez-vous à la rubrique Pour activer la recherche d'erreurs et l'envoi d'e-mails.

     

  12. Cliquez sur le bouton Suivant.

  13. A l'étape Image de sauvegarde, lorsque l'Assistant de restauration affiche une image de sauvegarde à restaurer, saisissez le chemin d'accès absolu de l'image de sauvegarde dans le champ Emplacement, puis cliquez sur le bouton Suivant.

     

    Note :
    Vous devez inclure le chemin du répertoire dans lequel est stockée l'image de sauvegarde.

     

  14. Cliquez sur le bouton Vérifier le statut pour actualiser la progression du processus. Lorsque l'Assistant de restauration a terminé de restaurer l'image spécifiée, cliquez sur le bouton Suivant.

  15. A l'étape Résumé, cliquez sur le bouton Suivant pour valider l'index restauré.

  16. A l'étape Validation d'index, cliquez sur le bouton Vérifier le statut pour actualiser la progression du processus. Si vous avez activé la transmission de messages d'erreur pour l'Assistant de restauration, Content Server envoie un message d'alerte lorsque la restauration est terminée. Si vous avez sélectionné plusieurs partitions à restaurer, cliquez sur le bouton OK pour répéter les étapes de l'Assistant de restauration et restaurer les autres partitions. Ne cliquez pas sur le bouton Resynchroniser jusqu'à ce que la restauration de toutes les partitions soit terminée.

  17. Lorsque toutes les partitions ont été restaurées, procédez de l'une des façons suivantes :

    • Cliquez sur le bouton OK pour fermer l'Assistant de restauration sans resynchroniser le serveur d'administration.

    • Cliquez sur le bouton Resynchroniser pour resynchroniser immédiatement le serveur d'administration.

     

    Note :
    Si vous devez restaurer d'autres index gérés par le même serveur d'administration, commencez par les restaurer, consultez ensuite Resynchronisation des serveurs d'administration, puis resynchronisez les serveurs d'administration et redémarrez le gestionnaire de recherche et le flux de données de la source de données.

     

  18. Redémarrez les processus de flux de données et de recherche de la source de données.
