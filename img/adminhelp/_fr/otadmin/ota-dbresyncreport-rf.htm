Si vous connectez l'installation Content Server actuelle à une base de données que vous avez créée avec une installation Content Server différente, Content Server vous invite à resynchroniser tous les serveurs d'administration du site. La resynchronisation permet de trouver l'emplacement des objets système (par exemple les processus de flux de données, les index et les IPools) dans la nouvelle installation Content Server. Pour plus d'informations sur les resynchronisations, reportez-vous à la rubrique Resynchronisation des serveurs d'administration.

Lorsque vous cliquez sur le bouton Resynchroniser sur la page Confirmation de la resynchronisation de la source de données, Content Server affiche la page Resynchroniser le serveur d'administration : Tous les serveurs, qui indique que la resynchronisation de la base de données est en cours. La page est actualisée très fréquemment, indiquant les objets système qui ont été synchronisés par Content Server jusque-là. Pour chaque objet système, la page affiche l'action éventuelle effectuée par Content Server pour resynchroniser l'objet. Lorsque la resynchronisation est terminée, la page Resynchroniser le serveur d'administration Livelink : Tous les serveurs affiche un message de confirmation et vous invite à poursuivre l'administration de la base de données.

Pour quitter la page Resynchroniser le serveur d'administration Livelink : Tous les serveurs, cliquez sur le lien Continuer.

En fonction du statut de l'installation actuelle, Content Server vous invite à Création de l'index Entreprise (s'il n'existe pas) ou affiche la page Félicitations.
