Vous pouvez à tout moment sauvegarder un index grâce à l'Assistant de sauvegarde. L'Assistant de sauvegarde passe en revue toutes les étapes de l'opération de sauvegarde, vous invitant à saisir les informations nécessaires dans un ordre défini.

Pour sauvegarder un index à intervalle régulier, vous pouvez automatiser le processus de sauvegarde. Pour plus d'informations sur le processus de sauvegarde, reportez-vous à la rubrique Automatisation des sauvegardes d'index.

Utilisation de l'Assistant de sauvegarde

Pour pouvoir sauvegarder l'index d'une source de données, cette dernière doit disposer d'un gestionnaire de sauvegarde. Si vous essayez de sauvegarder un index dont la source de données ne dispose pas de gestionnaire de sauvegarde, l'Assistant de sauvegarde vous invite à en créer un. Pour plus d'informations sur les gestionnaires de sauvegarde, reportez-vous à la rubrique Sauvegarde et restauration des index.

Utilisation de l'interface de ligne de commande

Pour sauvegarder un index à partir de l'interface de ligne de commande, appelez l'exécutable backuputil. Vous pouvez transmettre des paramètres à backuputil à l'aide d'un fichier de configuration ou des paramètres de ligne de commande. Cependant, OpenText conseille d'utiliser un fichier de configuration car il vous permet de conserver les paramètres appliqués lors de la sauvegarde.

Le fichier de configuration de sauvegarde, appelé host#.ini par défaut (# étant une série de lettres et de chiffres générée automatiquement), est stocké dans le répertoire d'indexation et est d'abord généré par le gestionnaire de sauvegarde lorsque vous programmez un processus de sauvegarde. Pour que Content Server génère un fichier de configuration, créez un gestionnaire de sauvegarde et programmez un processus de sauvegarde. Vous pouvez également créer un fichier de configuration manuellement.

 

Note :
Lorsque vous sauvegardez un index, Content Server crée un fichier d'historique de sauvegarde, appelé backup.ini, dans le répertoire de l'index. Dans le fichier backup.ini, Content Server enregistre l'historique des sauvegardes générées pour cet index. Le fichier backup.ini permet à Content Server de déterminer ce qui a déjà été sauvegardé dans cet index lorsque vous effectuez des sauvegardes différentielles. Etant donné l'importance de ce fichier, ne nommez pas votre fichier de configuration backup.ini si vous le créez manuellement.
