Le tableau suivant décrit les messages d'erreur générés par le service de conversion de documents (DCS).

Si ces erreurs persistent, contactez le service d'assistance clientèle d'OpenText pour obtenir plus d'informations.

Tableau 1-27 : Messages d'erreur du service de conversion de documents

Numéro Message Description Solution
100 Impossible de lire la configuration Le service de conversion de documents (DCS) n'a pas pu lire le fichier opentext.ini spécifié sur la ligne de commande. Assurez-vous que le fichier opentext.ini existe et que le DCS dispose d'un accès suffisant pour lire le fichier.
101 Impossible d'écrire dans le fichier journal Le DCS n'a pas pu écrire dans le paramètre logfile spécifié. Assurez-vous que le chemin spécifié dans le champ Fichier journal de l'onglet Service de conversion de documents de la page Propriétés du serveur d'administration est correct.
Vérifiez que le DCS dispose d'un accès suffisant pour écrire dans le fichier.
104 Impossible d'effectuer la liaison avec le port Admin Le DCS n'a pas pu être lié au port Admin spécifié. Assurez-vous que le numéro de port Admin spécifié dans le champ Fichier journal de l'onglet Service de conversion de documents de la page Propriétés du serveur d'administration est un port d'échange libre. Pour cela, cliquez sur le lien Vérifier le port. Si le port Admin n'est pas disponible, changez le numéro de port et utilisez un numéro libre.
105 Port Admin non spécifié Le port Admin est un paramètre obligatoire qui n'a pas été spécifié. Assurez-vous que le port Admin est spécifié dans l'argument de la ligne de commande. Pour plus d'informations sur la définition des arguments de ligne de commande, reportez-vous à la rubrique Arguments de ligne de commande.
107 Fichier de règles DCS manquant Le paramètre rulesfile du DCS est un paramètre obligatoire qui n'a pas été spécifié. Assurez-vous que le champ Fichier de règles de l'onglet Service de conversion de documents de la page Propriétés du serveur d'administration contient le chemin suivant :
rulesfile=Content Server_home/config/dcsviewrules.txt
108 Impossible d'ouvrir le fichier de règles DCS Le paramètre rulesfile spécifié n'existe pas ou n'a pas pu être lu par le DCS. Assurez-vous que le champ Fichier de règles de l'onglet Service de conversion de documents de la page Propriétés du serveur d'administration contient le chemin suivant :
rulesfile= Content Server_home/config/dcsviewrules.txt.
Vérifiez que le DCS dispose d'un accès suffisant pour lire le fichier.
109 Erreur de syntaxe du fichier de règles DCS Le paramètre rulesfile du DCS a été modifié et sa syntaxe n'est pas correcte. Utilisez la version du fichier de règles que vous avez obtenue avec votre installation Content Server actuelle.
110 Fichier de règles DCS non valable Le paramètre rulesfile du DCS a été modifié et contient des opérations illégales. Assurez-vous que le champ Fichier de règles de l'onglet Service de conversion de documents de la page Propriétés du serveur d'administration contient le chemin suivant :
rulesfile=Content Server_home/config/dcsviewrules.txt
Vérifiez que le DCS dispose d'un accès suffisant pour lire le fichier.
111 Echec de chargement de la bibliothèque DCS L'une des bibliothèques d'exécution chargées requise par le DCS est introuvable. En règle générale, le fichier journal du DCS identifie le nom de la bibliothèque qui n'a pas pu être chargée. Par exemple, « ERREUR : Echec de chargement de la bibliothèque dcssum.dll ».
Assurez-vous que les répertoires Content Server_home/lib et Content Server_home/filters sont inclus dans les variables d'environnement du chemin de la bibliothèque système. Sous Windows, Content Server_home/bin doit être inclus dans la variable PATH. Sous Solaris, Content Server_home/lib doit être inclus dans la variable LD_LIBRARY_PATH.
Pour plus d'informations, contactez le service d'assistance clientèle d'OpenText.
112 Paramètre lib manquant dans la bibliothèque DCS Le paramètre lib de la section INI de l'une des bibliothèques du DCS est manquant. En règle générale, le fichier journal du DCS identifie le nom de la bibliothèque qui n'a pas pu être chargée. Par exemple, « ERREUR : Echec de chargement de la bibliothèque dcssum.dll ».
Vérifiez que, sur la page de filtre de conversion correspondante du fichier opentext.ini, le nom de la bibliothèque requise est spécifié dans le paramètre lib.
Consultez la section correspondante de Compréhension du fichier opentext.ini.
113 Bibliothèque DCS non valable L'une des bibliothèques du DCS ne s'aligne pas sur l'interface de bibliothèque DCS. Pour identifier la bibliothèque à l'origine de ce problème, définissez le niveau de consignation du DCS sur la valeur maximale en configurant le paramètre suivant dans la section [DCS] du fichier opentext.in :
logLevel=4
Après avoir consigné l'activité du DCS, contactez le service d'assistance clientèle d'OpenText.
114 Impossible d'initialiser la bibliothèque DCS Une erreur s'est produite dans l'une des bibliothèques du DCS au moment de l'initialisation. Assurez-vous que les répertoires Content Server_home/config et Content Server_home/filters sont inclus dans les variables d'environnement du chemin de la bibliothèque système. Sous Windows, Content Server_home/bin doit être inclus dans la variable PATH. Sous Solaris, Content Server_home/lib doit être inclus dans la variable LD_LIBRARY_PATH.
Si le problème persiste, définissez le niveau de consignation du DCS sur la valeur maximale en modifiant le paramètre suivant dans la section [DCS] du fichier opentext.ini :
logLevel=4
Après avoir consigné l'activité du DCS, contactez le service d'assistance clientèle d'OpenText.
117 Impossible d'initialiser la bibliothèque DCS Une erreur s'est produite dans l'une des bibliothèques du DCS au moment de l'initialisation. Assurez-vous que les répertoires Content Server_home/config et Content Server_home/filters sont inclus dans les variables d'environnement du chemin de la bibliothèque système. Sous Windows, Content Server_home/bin doit être inclus dans la variable PATH. Sous Solaris, Content Server_home/lib doit être inclus dans la variable LD_LIBRARY_PATH.
Si le problème persiste, définissez le niveau de consignation du DCS sur la valeur maximale en modifiant le paramètre suivant dans la section [DCS] du fichier opentext.ini :
logLevel=4
Après avoir consigné l'activité du DCS, contactez le service d'assistance clientèle d'OpenText.
118 Echec de chargement du gestionnaire de requêtes DCS La bibliothèque IPool du DCS (dcsipool.dll sous Windows) n'a pas pu être chargée. Assurez-vous que les répertoires Content Server_home/config et Content Server_home/filters sont inclus dans les variables d'environnement du chemin de la bibliothèque système. Sous Windows, Content Server_home/bin doit être inclus dans la variable PATH. Sous Solaris, Content Server_home/lib doit être inclus dans la variable LD_LIBRARY_PATH.
Si le problème persiste, définissez le niveau de consignation du DCS sur la valeur maximale en modifiant le paramètre suivant dans la section [DCS] du fichier opentext.ini :
logLevel=4
Après avoir consigné l'activité du DCS, contactez le service d'assistance clientèle d'OpenText.
