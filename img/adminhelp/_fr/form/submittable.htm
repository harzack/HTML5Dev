Les utilisateurs ont besoin du privilège Réserve permettant l'envoi de formulaires pour pouvoir sélectionner Table SQL comme mécanisme d'envoi lors de la création d'un formulaire et pour pouvoir sélectionner un modèle pour lequel une table de base de données existe.

Par défaut, le privilège Réserve permettant l'envoi de formulaires n'est pas limité : tous les utilisateurs peuvent donc stocker les données envoyées de leurs formulaires directement dans la base de données, à condition qu'il existe une table pour le modèle sélectionné.

Vous pouvez envisager de limiter le privilège Réserve permettant l'envoi de formulaires si l'espace de la base de données pose des problèmes à votre système. Notez que même les utilisateurs ne pouvant pas créer de formulaires qui stockent directement des données envoyées dans la base de données peuvent être autorisés à utiliser les formulaires de ce type créés par d'autres utilisateurs. Quel que soit le privilège Réserve permettant l'envoi de formulaires, un utilisateur qui crée un formulaire a toujours la possibilité de stocker les données envoyées sous forme de versions d'un nœud Content Server.
