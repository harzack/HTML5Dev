Les questions suivantes vont vous aider à choisir la meilleure gestion des formulaires pour votre système :

  • Souhaitez-vous autoriser tous les utilisateurs à ajouter des tables à la base de données Content Server ? Si la réponse est oui, veillez à ce que tous les utilisateurs connaissent le schéma Content Server afin de préserver l'intégrité de votre base de données. Vous pouvez envisager de limiter le privilège de création de modèles de formulaire. Pour plus d'informations sur les privilèges de création de modèles de formulaire, voir Gestion des droits de création.

  • Souhaitez-vous autoriser tous les utilisateurs à stocker des données de formulaire dans des tables qui peuvent exister pour les modèles sélectionnés lors de la création de formulaires ? Si la réponse est non, limitez les Gestion des droits d'utilisation.

  • Souhaitez-vous autoriser tous les utilisateurs à ajouter des formulaires à votre système ? Si la réponse est oui, tous les utilisateurs doivent connaître les mécanismes de révision et d'envoi. Même si vous limitez le privilège de création de formulaires, les utilisateurs peuvent remplir les formulaires prérenseignés. Cette méthode vous permet de réduire le temps de formation de l'utilisateur. Pour plus d'informations sur les privilèges de création de formulaires, voir Gestion des droits de création.

     

    Note :
    Si vous choisissez cette option, OpenText vous recommande de prévoir un répertoire spécial pour les modèles de formulaire. Ce répertoire permettra aux utilisateurs de retrouver facilement les modèles dont ils ont besoin pour ajouter des formulaires au système.

     

  • Comment prévoyez-vous d'utiliser les données de formulaires envoyées ? Si vous souhaitez réaliser des statistiques ou mettre ces données à la disposition d'une application externe, associez les tables de base de données à vos modèles de formulaire. L'option Table SQL est alors disponible pour les mécanismes de révision et d'envoi. Pour en savoir plus, voir Utilisation de formulaires.
