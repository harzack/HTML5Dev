Pour configurer l'administration de l'intégration Directory Services

  1. Dans la section Administration de l'intégration Directory Services de la page Administration de Content Server, cliquez sur le lien Configurer les paramètres d'intégration.

     

    Astuce :
    Vous pouvez utiliser un raccourci pour accéder à cette page directement, http://<nom_serveur_complet>/OTCS/livelink.exe?func=otdsintégration.paramètres.

     

  2. Facultatif Sélectionnez Authentification du serveur Web pour récupérer les informations des utilisateurs authentifiés directement à partir de votre serveur Web.

    1. Dans la zone Variable environnement, saisissez la variable utilisée pour valider les informations d'authentification de l'utilisateur ou conservez la valeur par défaut UTILISATEUR_DISTANT.

      Le paramètre Variable environnement vous permet de choisir la variable à utiliser afin de déterminer le nom d'utilisateur. Par défaut, il est configuré sur UTILISATEUR_DISTANT. D'autres schémas d'authentification peuvent définir Variable environnement sur une valeur différente, comme Siteminder, qui utilise la valeur HTTP_UTILISATEUR_SM.

    2. Dans la zone Format du nom d'utilisateur dans Content Server, sélectionnez l'option correspondant aux noms de connexion des utilisateurs dans Content Server :

      • Nom d'utilisateur uniquement

      • Nom d'utilisateur de type NT

      • Résoudre dans OTDS

    3. Dans la zone Sensibilité à la casse du nom d'utilisateur, cliquez sur Conserver la casse pour conserver le nom d'utilisateur lorsque l'utilisateur se connecte à Content Server ou cliquez sur Minuscule pour convertir le nom d'utilisateur en lettres minuscules lorsque l'utilisateur se connecte à Content Server.

  3. Choisissez une méthode d'authentification à utiliser si Authentification du serveur Web n'est pas sélectionnée ou si les informations d'authentification de l'utilisateur ne peuvent pas être authentifiées en utilisant le serveur Web. Choisissez l'une des options suivantes :

    Content Server Authentification
    Choisissez cette méthode si vous souhaitez que Content Server authentifie uniquement les utilisateurs internes.
    OTDS Authentification
    Choisissez cette méthode si vous souhaitez qu'OpenText Directory Services authentifie les utilisateurs internes et externes pour une authentification unique.

  4. Si vous choisissez Authentification OTDS dans étape 3, configurez vos paramètres d'intégration Directory Services

    1. Dans le champ URL du serveur OTDS, saisissez l'URL du serveur Directory Services.

      L'URL doit contenir le domaine complet et le numéro de port du serveur Directory Services. Voici quelques exemples d'URL :

      • http://<nom_serveur>:<numéro_port>

      • https://<nom_serveur>:<numéro_port>

       

      Note :
      Si votre serveur Directory Services a été installé dans un cluster, vous devez saisir le domaine complet et le numéro de port du répartiteur de charge dans le champ URL du serveur OTDS.

       

    2. Dans le champ URL de connexion OTDS,saisissez l'URL pour vous connecter à OTDS.

      Pour un exemple de convention URL, voir étape 4.a.

    3. Dans le champ Identifiant de la ressource, saisissez l'ID unique qui a été généré lorsque vous avez créé une ressource Content Server synchronisée dans Directory Services.

    4. Dans le champ Vérifier les connexions, cliquez sur Exécuter le test pour confirmer que l'URL saisie dans le champ URL du serveur OTDS est valide.

       

      Note :
      Ce test de connexion ne vérifie pas si le Répertoire de serveur OpenText est correctement configuré avec Content Server. Il vérifie uniquement si l'URL fournie dans le champ URL du serveur OTDS est valide.

       

  5. Cliquez sur Enregistrer.

     

    Note :
    Attendez pour redémarrer Content Server.

     

  6. Sur la page Administration de Content Server, dans la section Configuration du serveur, sélectionnez Configurer les paramètres de sécurité.

  7. Sur la page Configurer les paramètres de sécurité, dans la zone Sites Web de référence sécurisés, assurez-vous que le nom Serveur OTDS de étape 4.a figure dans la liste.

  8. Cliquez sur Enregistrer les modifications.

  9. Redémarrez Content Server.
