La page Administration de l'intégration Directory Services dans Content Server est utilisée pour configurer la manière dont les utilisateurs sont authentifiés avec Content Server. Content Server prend en charge l'authentification unique et les services d'authentification fournis par le serveur Web et OpenText Directory Services (OTDS), ainsi que l'authentification fournie en interne.

Vous pouvez configurer les options d'authentification suivantes selon ce qui convient au serveur Web et au service de répertoire externe que vous souhaitez utiliser avec Content Server :

Authentification du serveur Web

Votre serveur Web peut être configuré pour authentifier les utilisateurs lorsqu'ils accèdent aux URL Content Server. Par exemple, si vous utilisez Microsoft Internet Information Services (IIS), votre serveur peut être configuré pour authentifier les utilisateurs disposant de l'authentification Windows intégrée. Dans ce cas, IIS renseigne la variable de l'environnement du serveur Web REMOTE_USER avec le nom de compte de l'utilisateur authentifié. Le format du <nom d'utilisateur> peut être fourni et configuré dans les formulaires suivants, sélectionnez l'option correspondant au format des noms de connexion des utilisateurs dans Content Server.

  • Nom d'utilisateur uniquement

  • Nom d'utilisateur de type NT qui peut aussi être exprimé par DOMAIN\username.

  • Résoudre dans OTDS La valeur depuis REMOTE_USER est envoyée à OTDS afin de trouver le nom d'utilisateur correspondant dans Content Server. Vous devez configurer OpenText Directory Services. Pour en savoir plus, voir Authentification OTDS et OpenText Directory Services - Administration Guide (OTDS-AGD).

     

    Note :
    Un gestionnaire d'authentification Web Access Management doit être configuré dans OTDS pour que cette option fonctionne.

     

La sensibilité à la casse pour le <nom d'utilisateur> peut être configurée pour préserver la casse ou convertir la casse en minuscules. Vous souhaitez peut-être convertir la casse en minuscules lorsque vous avez une base de données sensible à la casse et que la synchronisation est configurée sur minuscules.

Si l'authentification du serveur Web est activée mais que les informations de l'utilisateur ne sont pas disponibles, l'authentification essaye l'authentification OTDS ou l'authentification Content Server interne selon la méthode configurée.

Authentification OTDS

L'authentification OTDS utilise le serveur OpenText Directory Services (OTDS) pour authentifier les utilisateurs. L'administrateur doit d'abord créer une ressource Content Server dans OTDS. Pour en savoir plus, voir OpenText Directory Services - Administration Guide (OTDS-AGD). Au cours de cette opération, un identificateur unique, appelé ID de ressource, est généré. L'ID de ressource et le serveur URL OpenText Directory Services sont des valeurs obligatoires et doivent être saisis afin de configurer l'authentification OTDS dans Content Server. Pour plus d'informations sur la configuration de OpenText Directory Services, voir OpenText Directory Services - Administration Guide (OTDS-AGD).

Si vous souhaitez activer l'authentification OTDS ultérieurement, vous pouvez enregistrer les données sans activer l'authentification. Une fois l'authentification activée, tous les utilisateurs connectés à ce moment-là peuvent continuer à utiliser Content Server.

Authentification Content Server

L'authentification Content Server peut être utilisée si vous ne souhaitez pas vous servir de l'authentification externe. Content Server peut prendre en charge à la fois les utilisateurs internes et externes. Toutefois, Content Server n'authentifie pas les utilisateurs externes.
