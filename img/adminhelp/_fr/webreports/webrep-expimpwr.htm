Lorsque vous exportez et importez des WebReports vers et depuis XML, vous devez tenir compte des points ci-dessous.

  • Lorsqu'un objet précis est recherché dans un système cible, si plusieurs occurrences de cet objet sont présentes, la routine d'importation utilise actuellement la première occurrence trouvée dans la base de données. En général, il s'agit du premier objet créé.

  • L'exportation et l'importation de WebReports ne prennent pas en charge les aspects suivants :

    • Les catégories ou attributs associés, avec paramètres, à exporter vers un nœud Content Server ne sont pas liés à nouveau vers de nouveaux objets sur un système cible.

    • Les données de planification ne sont actuellement pas exportées ni importées.

Exportation et importation standard

Comme la plupart des objets Content Server, les WebReports peuvent stocker toutes les données de WebReports dans un fichier XML pour une importation ultérieure. La fonctionnalité de base utilise la même syntaxe que d'autres objets Content Server. Par exemple, pour exporter un seul objet WebReport, vous pouvez utiliser la syntaxe suivante : 

?func=ll&objId=10879&objAction=xmlexport&nodeinfo&content=base64&versioninfo=current&scope=sub

&attributeinfo peut être utilisé pour inclure les attributs assignés. &versioninfo=all peut être utilisé si toutes les versions sont requises.

Dans cet exemple, l'objid 10879 fait référence à un objet WebReport. Si un recueil Content Server complet est exporté (au moyen du paramètre &scope=sub) et s'il contient un WebReport, le WebReport est exporté avec exactement les mêmes informations. Comme les WebReports contiennent du texte, l'option &content=base64 doit être utilisée pour toute exportation.

Pour importer un WebReport, utilisez la syntaxe d'importation standard. Par exemple, pour importer un fichier appelé TestImport.xml, stocké sur le disque C:\ de Content Server dans un dossier de Content Server avec l'ID d'objet 22222, utilisez la syntaxe suivante : 

?func=admin.xmlimport&filename=C:\TestImport.xml&objid=22222
 

Note :
Le fichier à importer doit être enregistré/stocké à un emplacement où Content Server peut le récupérer ; dans l'exemple ci-dessus, le disque C:\ est le disque local de Content Server, et non le disque C de l'utilisateur exécutant la commande.
