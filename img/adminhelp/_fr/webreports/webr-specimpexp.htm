WebReports fournit des fonctions standard d'exportation et d'importation, mais aussi des fonctions plus avancées pour faciliter le rétablissement de liens vers des dépendances sur le système de destination. Cette fonctionnalité est utilisée pour résoudre les objets suivants sur le système cible :

  • ID de source de données

  • Tout ensemble de constantes vers LivelinkObject

  • Tout ensemble de constantes vers LivelinkUser

  • Toute valeur par défaut de paramètre définie sur ObjectID

  • Toute valeur par défaut de paramètre définie sur Custom

  • Toute valeur par défaut de paramètre définie sur User

  • Tout ID de constante globale

  • Tout ID de paramètre global

Ces objets sont résolus au moyen d'une série d'actions qui recherchent l'objet correct sur le système cible. Dans certains cas, plusieurs objets correspondent à celui recherché, si bien que des priorités sont appliquées. Les éléments ci-dessous présentent ces priorités. En cas d'échec de l'une des étapes, l'étape suivante est tentée.

  1. Si l'objet recherché est importé dans le cadre de la même opération d'importation XML, le nouvel ID de l'objet importé est utilisé.

  2. Si l'ID de l'objet d'origine existe sur le système cible, le nœud associé est vérifié, afin d'assurer qu'il s'agit du même type d'objet, avec le même nom et les mêmes données.

  3. En l'absence d'ID d'objet équivalent, une requête de base de données est utilisée pour rechercher dans le système cible un objet de même nom et de données équivalentes. Si cet objet est trouvé, ce nouvel ID d'objet est ajouté au WebReport.

Les ID d'utilisateur Content Server sont gérés d'une façon similaire. Cependant, les ID d'utilisateur ne sont pas importés lors d'une importation XML, donc la première méthode de recherche n'est pas appliquée.
