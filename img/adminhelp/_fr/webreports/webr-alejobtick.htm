A partir de WebReports 4.0.1, les bons de travail Adlib eXpress sont totalement pris en charge. Lorsque la case à cocher de la fonction Bon de travail est activée, sur la page Gérer les répertoires de conversion, les utilisateurs peuvent fournir des directives de traitement détaillées par le biais d'un fichier de bon de travail XML. Ce fichier XML peut être stocké dans Content Server, avec tous les objets WebReports partageant un même bon de travail ou des instructions de traitement spécifiques pour chacun d'eux. Voir le site Web Adlib pour en savoir plus sur les bons de travail XML.

Lorsque la fonction Bon de travail est activée, deux nouveaux répertoires de conversion sont proposés à l'utilisateur :

  • Répertoire d'entrée (lors de l'utilisation de bons de travail XML)

  • Répertoire de sortie (lors de l'utilisation de bons de travail XML)

 

  1. Pour que la fonction de conversion puisse être activée, le premier ensemble de répertoires de conversion doit être spécifié. Les autres répertoires destinés aux bons de travail XML sont facultatifs, mais ils sont utiles pour les serveurs exécutant plusieurs instances de Content Server à l'aide de méthodes de conversion différentes. Les versions actuelles d'Adlib eXpress prennent uniquement en charge la conversion avec ou sans bon de travail, mais pas les deux simultanément.

  2. Si seule la conversion avec bon de travail XML est utilisée, l'administrateur doit spécifier les répertoires de conversion dans les deux premiers champs.

  3. Si la conversion avec bon de travail XML est utilisée en même temps que la conversion standard, les répertoires de conversion standard doivent être spécifiés en premier, puis les répertoires pour bon de travail XML.

  4. Les valeurs saisies sur cette page seront utilisées pour remplacer les occurrences de $DEFAULT_PATH spécifiées dans le bon de travail lui-même.

  5. Un exemple de bon de travail de base est disponible dans l'aide en ligne : WebReports : Informations avancées : Conversion PDF.
