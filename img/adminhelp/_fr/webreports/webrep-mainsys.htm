Vous pouvez activer des fonctions Content Server pour WebReports, au moyen des pages Administration Content Server principales.

  • Vous pouvez restreindre la création de WebReports à un groupe d'utilisateurs sélectionnés.

  • Vous pouvez activer des événements Content Server standard pour WebReports.

  • Vous pouvez stocker des WebReports dans le volume Rapports Content Server.

  • Vous pouvez exporter des WebReports vers un e-mail.

Restriction de l'ajout de nouveaux éléments pour WebReports

Comme la plupart des autres objets Content Server, les objets WebReports peuvent être contrôlés au moyen du « générateur d'objets » Content Server. Cela signifie qu'un administrateur peut restreindre la création de WebReports à un groupe d'utilisateurs sélectionnés.

Gestion d'événements WebReports

Vous pouvez activer des événements Content Server pour l'objet WebReports. Un affichage de rapport WebReport est traité comme un document Content Server normal ; tous les événements Content Server standard peuvent donc être activés et utilisés.

Certains événements WebReports à inclure dans l'audit sont activés par défaut. Pour activer les événements désactivés par défaut, utilisez la page Content Server Définir les centres d'intérêt à inclure dans l'audit.

Pour écrire les événements RUN ou EXPORT d'un WebReport dans le journal AUDIT, vous devez activer l'option Exécution/exportation des audits activée dans l'onglet Spécifique de la page Propriétés de chaque WebReport. Cette option est désactivée par défaut afin de permettre un contrôle granulaire pour éviter le débordement des journaux d'audit.

Stockage de WebReports dans le volume Rapports

L'objet WebReport est configuré pour se comporter comme un rapport LiveReport pour toutes les fonctions système associées aux rapports. En particulier, les WebReports peuvent être enregistrés dans le volume Rapports, au moyen de l'onglet Rapport LiveReport dans les menus Personnel ou Projet.

Exportation vers un e-mail

Les WebReports peuvent être exportés vers des emplacements multiples, notamment un e-mail. Pour qu'une exportation vers un e-mail fonctionne correctement, les détails suivants doivent être renseignés dans la page Configurer la notification des pages Administration Content Server :

  • ID de serveur SMTP

  • Port SMTP

  • Content Server Nom d'hôte

 

Note :
Bien que ces détails doivent être renseignés, il est inutile d'activer les notifications Content Server.
