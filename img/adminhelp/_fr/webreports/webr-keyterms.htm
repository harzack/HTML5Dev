Les termes suivants sont utilisés dans cette documentation :

  • Utilisateurs finaux : consommateurs Content Server qui voient la sortie d'un WebReport.

  • Développeurs : tout utilisateur pouvant être impliqué dans la création de nouveaux WebReports ou la modification de WebReports existants pour en changer l'apparence et le comportement. Pour un WebReport donné, les développeurs sont répartis en deux catégories :

    1. ceux pouvant modifier l'affichage de rapport ou y ajouter une version ;

    2. ceux pouvant uniquement extraire ou télécharger l'affichage de rapport.

    Ce dernier cas permet d'autoriser les développeurs à réutiliser un rapport d'affichage existant pour un nouveau WebReport, tout en leur interdisant de modifier l'original.

  • Source de données : tout objet Content Server utilisé afin de produire un ensemble de données pour le WebReport. Il s'agit le plus souvent d'un rapport LiveReport, mais une requête de recherche enregistrée, un modèle de formulaire ou un formulaire sont également possibles.

  • Rapport LiveReport : élément Content Server permettant aux utilisateurs d'obtenir des informations statistiques sur la base de données Content Server et, éventuellement, de modifier celle-ci. La requête SQL d'un rapport LiveReport communique directement avec la base de données. L'administrateur Content Server peut créer de nouveaux rapports LiveReport et accorder à d'autres utilisateurs l'autorisation de les exécuter. Les rapports LiveReport pour lesquels les utilisateurs possèdent des autorisations s'affichent sur l'onglet Rapports LiveReport de la page Rapports de leur espace de travail Personnel.

  • Affichage de rapport : terme utilisé tout au long de ce document en référence au document stocké pour chaque objet WebReport. Ces affichages de rapports contiennent du code Web classique, par exemple HTML, ainsi que des balises personnalisées qui permettent de déterminer l'emplacement où les champs de données de la source de données sont insérés dans la page Web de sortie.

  • Administrateurs système : utilisateurs gérant les autorisations et les droits d'accès pour les WebReports, et pouvant également accéder aux rapports LiveReport et/ou à d'autres sources de données utilisées pour fournir des données aux WebReports. Les différentes tâches et fonctions décrites dans ce document sont conçues pour ces administrateurs.
