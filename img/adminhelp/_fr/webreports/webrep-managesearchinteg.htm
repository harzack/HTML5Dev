Le module WebReports prend en charge une fonction d'intégration de recherche, qui permet de lancer un ou plusieurs WebReports à partir de l'écran Recherche avancée standard.

Pour activer cette fonction, les actions suivantes sont nécessaires :

  1. Vous devez activer cette fonction sur la page Gérer l'intégration de la requête de recherche et spécifier un nom approprié pour le nouveau bouton de recherche.

  2. Une fois la fonction activée, vous devez redémarrer Content Server.

  3. Vous devez définir un ou plusieurs WebReports pour qu'ils utilisent un « lancement de recherche » en tant que source de données. Pour ce faire, sélectionnez l'onglet Source dans la page Propriétés du WebReport.

  4. A partir de la page Gérer l'intégration de la requête de recherche, vous pouvez activer ou désactiver les WebReports utilisant cette source de données.

    Les autorisations des WebReports utilisant cette source de données sont configurées pour restreindre l'accès à ces WebReports aux utilisateurs appropriés uniquement.

Une fois la configuration répertoriée ci-dessus établie, l'écran Recherche avancée affiche le bouton de recherche personnalisé. Lorsque l'utilisateur sélectionne ce bouton de recherche personnalisé, la recherche est lancée dans les WebReports que l'utilisateur est autorisé à exécuter. Si l'utilisateur est autorisé à effectuer une recherche sur plusieurs WebReports, lorsqu'il sélectionne le bouton de recherche personnalisé, la liste de ces WebReports s'affiche et l'utilisateur peut sélectionner celui dans lequel effectuer la recherche.

Remarques techniques sur l'intégration de la recherche

Lorsque plusieurs instances de Content Server sont utilisées, il convient de tenir compte des points suivants :

  1. L'option d'activation est propre au serveur et stockée dans le fichier opentext.ini. Il en résulte que cette fonction peut être restreinte à une instance Content Server spécifique. Inversement, si tous les serveurs d'un cluster doivent se comporter de la même façon, cette fonction doit être activée, et le texte du bouton défini, pour chaque serveur. Pour ce faire, il est également possible de copier le fichier opentext.ini sur chaque serveur.

  2. L'activation ou la désactivation de WebReports distincts porte sur l'ensemble du système. Les WebReports désactivés sont donc indisponibles sur toutes les instances Content Server.

  3. Si cette page est affichée sur un serveur où la fonction est désactivée, la liste des WebReports actifs et/ou inactifs n'est pas visible. Pour que cette liste devienne disponible :

    • Activez l'Intégration de la recherche.

    • Fournissez un nom pour le bouton.

    • Cliquez sur Appliquer.

    Cependant, comme noté ci-dessus, cela n'active pas la fonction, sauf si Content Server est ensuite redémarré.
