Outre les fonctions uniques fournies par WebReports détaillées ci-dessus, des fonctions de base de gestion de documents Content Server sont proposées, afin d'apporter d'autres fonctionnalités aux affichages de rapports, notamment un contrôle complet des versions.

Cela signifie que toutes les options d'administration normalement disponibles pour les documents Content Server peuvent être appliquées aux WebReports. En particulier, toute option d'administration de document, telle que Gérer le contrôle d'élément, s'applique également aux fichiers d'affichages de rapports. Pour plus d'informations, voir :

  • Activation de fonctions Content Server pour WebReports

  • Exportation et importation de WebReports vers XML

  • Définition des autorisations des utilisateurs de WebReports

  • Préférences WebReports dans le fichier opentext.ini

  • Vérifier les mises à jour

Les fichiers d'affichages de rapports sont totalement indexés, ce qui permet de les retrouver au moyen de la fonction de recherche, bien que la plupart des utilisateurs ne bénéficient pas des autorisations de lecture. Pour plus d'informations sur les affichages de rapports, voir :

  • Vues en mémoire cache

  • Affichages de rapports par défaut
