Lors de la désinstallation du module WebReports, lancée à partir de la page Administration du module, le schéma de la base de données du module reste en place, afin d'apporter davantage de flexibilité à l'administrateur système, par exemple pour éviter la perte des informations de planification lorsque WebReports est réinstallé. Seules les versions de WebReports 5.0.x ou ultérieures peuvent être réinstallées dans de telles circonstances.

Si vous souhaitez supprimer définitivement WebReports et son schéma associé ou brider le schéma, sélectionnez Administration de la base de données WebReports dans les pages Administration.

  • Si vous cliquez sur Supprimer les tables WebReports, le schéma de WebReports est entièrement supprimé, puis vous êtes conduit dans la désinstallation du module.

  • Si vous cliquez sur Brider le schéma à 4.x.x, seul le nouveau schéma ajouté après WebReports 4.1.1 est supprimé, puis vous êtes conduit dans la désinstallation du module. Toutes les données valides dans WebReports 4.x.x sont conservées et vous pouvez ensuite installer WebReports 4.x.x.

Si vous désinstallez WebReports sans utiliser l'une de ces options, puis si vous tentez d'installer une version de WebReports antérieure à la version 5.0.x, l'installation échoue et signale une erreur d'incohérence de schéma. Vous devez alors désinstaller WebReports manuellement, puis réinstaller WebReports 5.0.x et utiliser l'une des options ci-dessus ou supprimer manuellement le schéma de la base de données. En cas de doute, n'hésitez pas à contacter l'assistance technique d'OpenText.
