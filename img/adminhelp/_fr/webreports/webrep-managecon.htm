Plusieurs destinations WebReports prennent en charge la conversion de documents, une fonction qui peut être utilisée pour créer des rapports PDF mis en forme. Les destinations sont :

  • E-mail

  • Nœud Content Server

  • Version Content Server

  • Serveur

  • Workflow (en pièce jointe)

Dans les versions 3.x et antérieures de WebReports, la conversion PDF était prise en charge d'une manière légèrement différente. L'utilisateur pouvait sélectionner un type MIME PDF qui, une fois sélectionné, générait l'affichage de rapport à convertir au format PDF. Une des limites résidait dans le fait que l'affichage de rapport converti au format PDF ne pouvait être qu'au format HTML.

A partir de WebReports 4.0, le type MIME et le processus de conversion ont été complètement découplés. Ceci signifie que l'utilisateur peut utiliser WebReports pour créer tout type de document, par exemple WordML ou SpreadsheetML, puis exécuter le processus de conversion sur ce document. Ceci apporte beaucoup plus de flexibilité à l'aspect du document à convertir.

En cas d'utilisation de la fonction de conversion PDF, l'administrateur Content Server doit configurer des options dans la section Gérer la conversion WebReports des pages Administration Content Server.

La page Gérer la conversion WebReports vous permet d'ajouter, de visualiser ou de modifier des informations sur les répertoires de conversion WebReports. Ces répertoires sont utilisés conjointement à un outil de conversion, tel qu'Adlib eXpress, pour effectuer des exports WebReports en PDF, ou en d'autres formats si nécessaire.

Lorsqu'un utilisateur décide qu'il souhaite mettre en forme un WebReport, par exemple au format PDF, l'affichage de rapport est présenté au format défini par cet utilisateur dans le WebReport, puis transféré dans le dossier d'entrée défini sur la page Gérer la conversion WebReports. L'outil de conversion doit être configuré pour « surveiller » ce répertoire. Lorsqu'un fichier est trouvé, l'outil doit le convertir et placer le document qui en résulte dans le dossier de sortie, puis supprimer le fichier d'entrée. Une fois que le document converti apparaît dans le dossier de sortie, WebReports l'envoie à l'emplacement d'exportation précédemment défini et le supprime du dossier de sortie.
