Lorsque les développeurs de WebReports sélectionnent l'option Utiliser un affichage de rapport par défaut, ils peuvent choisir un modèle parmi plusieurs modèles standard plutôt que parcourir leur bureau à la recherche d'un affichage de rapport. Ces modèles sont stockés sous forme de fichiers texte dans Content Server, dans le répertoire du module destiné aux WebReports. Par exemple, <Content Server_home>\module\webreports_x_x_x\defaultreportviews\10_basic_report.txt

L'administrateur Content Server peut modifier ces fichiers et en ajouter de nouveaux en modifiant un fichier existant, en le renommant ou en plaçant des fichiers supplémentaires dans le dossier defaultreportviews. Ces fichiers s'affichent sous la forme d'éléments à sélectionner dans le menu déroulant des affichages de rapports par défaut. Les noms de fichiers des affichages de rapports par défaut doivent se terminer par .txt et commencer par <x>_, où <x> est un nombre déterminant l'ordre de ces affichages de rapports dans le menu.

En général, un affichage de rapport est adapté ou un nouvel affichage de rapport est créé pour :

  • inclure une marque d'entreprise standard dans les rapports ;

  • fournir des affichages de rapports spécifiques pour l'exportation de données Content Server vers d'autres applications. Par exemple, CSV ou WordML.

Pour plus d'informations sur les affichages de rapports prédéfinis de WebReports, consultez l'aide en ligne et le manuel de l'utilisateur.
