• WebReports utilise la mémoire cache Content Server pour déterminer quelle destination correspond à quel fichier converti. Comme pour n'importe quelle mémoire cache, la durée de vie est limitée, ce qui signifie, bien que cela soit peu probable, qu'il est possible que des fichiers convertis présents dans le dossier de sortie soient liés à une mémoire cache expirée. Dans un tel cas, WebReports ne sait pas où envoyer les fichiers, car les informations de destination sont perdues. Cette situation peut se produire si les utilisateurs ont exporté et converti des fichiers dans un système où les agents étaient désactivés sur tous les serveurs. Activer les agents ne traite pas les fichiers en attente, à moins qu'ils ne disposent d'une mémoire cache. De manière générale, si un élément reste plus d'une heure dans le dossier de sortie, il doit être supprimé manuellement.

  • Les agents Content Server sont utilisés pour surveiller le dossier de sortie toutes les 5 minutes, afin de détecter la présence d'un fichier converti. Pour fonctionner, ces agents doivent donc être activés pour la conversion.

  • L'heure exacte d'écriture d'un fichier dans le dossier de sortie, la fréquence à laquelle l'outil de conversion vérifie le contenu de ce dossier, la durée réelle de l'opération de conversion et le moment où celle-ci se produit dans le cycle de vérification de 5 minutes de l'agent sont autant de paramètres qui déterminent le temps nécessaire pour qu'un fichier arrive à la destination d'exportation. Ce délai peut varier de quelques secondes à 30 minutes, en fonction du système.

  • Les dossiers d'un emplacement réseau peuvent être configurés en tant qu'UNC, par exemple : \\192.168.255.20\disk1\PDF Conversion\input

  • Une exportation peut être effectuée depuis n'importe quel Content Server d'un environnement de clusters. Il est donc essentiel de définir le répertoire d'entrée sur chaque machine du cluster. Le dossier de sortie est surveillé uniquement par le Content Server, lorsque l'agent de conversion WebReports est en cours d'exécution. C'est pourquoi il est essentiel de spécifier le dossier de sortie sur ce serveur. Il est toutefois recommandé de spécifier le dossier de sortie sur toutes les instances de Content Server.
