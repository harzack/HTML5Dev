WebReports comporte des préférences de configuration qui peuvent être définies dans le fichier opentext.ini. Le tableau suivant décrit les options possibles. Soyez prudent lorsque vous modifiez des paramètres dans le fichier opentext.ini. Content Server doit être redémarré pour qu'une modification prenne effet. En cas de doute, consultez toujours l'assistance technique d'OpenText.

Tableau 1-2 : Options de configuration de WebReports dans le fichier opentext.ini

Nom du champ Type Description
addsearchbutton Chaîne ou null Libellé du bouton de lancement d'un WebReport à partir d'une recherche Content Server. Doit être défini à partir des pages Administration dans Content Server. Pour plus d'informations sur cette fonction, voir WebReports User Guide (Manuel d'utilisation WebReports).
searchslice Nombre entier Détermine le nombre de résultats à recueillir par l'API de recherche. La modification de ce paramètre affecte les performances des WebReports utilisant la recherche comme source de données. En général, ce paramètre ne doit pas être modifié sans l'avis de l'assistance technique d'OpenText. La valeur par défaut est 300.
searchHardLimit Nombre entier Nombre maximal de résultats qu'un WebReport utilisant la recherche comme source de données peut renvoyer. La valeur par défaut est 50 000.
searchDebug true / false Insère ou non des informations de débogage supplémentaires dans le journal du thread pour l'exécution de WebReports avec la recherche comme source de données. Prend la valeur false par défaut.
EnableOscriptReportviews true / false Autorise ou non la présence d'Oscript dans les affichages de rapports. Prend la valeur true par défaut. Lorsque ce paramètre est défini sur false, aucun oscript n'est compilé ou exécuté dans les affichages de rapports. La fonction d'écriture de scripts est alors complètement désactivée sur Content Server. Pour plus d'informations sur cette fonction, voir WebReports User Guide (Manuel d'utilisation WebReports). Prend la valeur true par défaut.
OscriptAllowFunction1 Liste de chaînes Liste de fonctions de packages Oscript distinctes à autoriser dans un affichage de rapport. Pour plus d'informations sur cette fonction, voir WebReports User Guide (Manuel d'utilisation WebReports).
OscriptAllowWholePkg Liste de chaînes Liste de packages Oscript à autoriser dans un affichage de rapport. Pour plus d'informations sur cette fonction, voir WebReports User Guide (Manuel d'utilisation WebReports).
MaxNestedSubWebReports Nombre entier Profondeur maximale d'imbrication de sous-rapports WebReports. Prend la valeur 5 par défaut si ce champ est absent.
VersionInput Chaîne de chemin de répertoire Répertoire d'entrée pour un convertisseur (p. ex. AdLib eXpress). Doit être modifié au moyen de la page Administration Gérer la conversion WebReports.
ConversionDir Chaîne de chemin de répertoire Répertoire de sortie pour un convertisseur (p. ex. AdLib eXpress). Doit être modifié au moyen de la page Administration Gérer la conversion WebReports.
additionalSearchColumns Liste de chaînes ou liste de listes Colonnes facultatives, mises à la disposition des WebReports utilisant la recherche comme source de données. Les valeurs possibles sont : OTHotWords, OTSummary, parentRecord, Score, shortOTSummary. Vous pouvez les spécifier sous forme de chaînes dans une liste ou sous forme d'une liste de deux listes d'éléments, où le premier élément est la colonne renvoyée par la recherche et le second élément le nom de cette colonne dans la sortie. Notez que ces valeurs sont disponibles au moyen de l'index de recherche. Si la structure sous-jacente de l'index est modifiée dans des versions ultérieures, ces colonnes risquent de ne plus être disponibles.
AllowFormCustomview true / false Contrôle si des WebReports peuvent être ajoutés ou non à des modèles de formulaires sous forme de vues. Pour voir cette fonction, accédez à l'onglet des vues d'un modèle de formulaire. L'option Ajouter Vue WR Power est alors présente si la fonction est activée. Cette fonction est activée par défaut.
WRNodeMIMETypes Liste de chaînes Détermine la liste de types MIME sur lesquels un nœud WebReport peut être défini. Il s'agit en fait du type MIME de l'affichage du rapport. Normalement, le type MIME text/plain est assigné aux WebReports, mais il est parfois utile d'autoriser le type MIME text/html ou d'autres types MIME.
destinationmimetypes Liste de chaînes Type MIME de la sortie créée par des WebReports. Selon les paramètres du client, ce paramètre peut déterminer l'application utilisée pour ouvrir le document résultant.
WRTriggerSubtypes Liste d'entiers Contrôle les sous-types en mesure de lancer des WebReports à partir de la fonction Déclencheur WR. Tous les sous-types sont désactivés par défaut.
DisallowServices true / false Active ou désactive la fonction Services WR. Cette fonction est activée par défaut.
livelinkcategory_maxNumCats Nombre entier Nombre maximal de catégories pouvant être sélectionnées au moyen de la fonction Catégorie comme source de données.
livelinkcategory_NumAttrAcross Nombre entier Nombre maximal d'attributs à afficher sur une seule ligne à partir de la fonction Catégorie comme source de données.
livelinkcategory_attrDelimiter Chaîne Caractère utilisé pour séparer les attributs à valeurs multiples dans la sortie de la fonction Catégorie comme source de données.
livelinkcategory_maxNumRows Nombre entier Nombre maximal de résultats pouvant être récupérés à partir de la fonction Catégorie comme source de données.
AllowFormCustomview true / false Active ou désactive la fonction WR Power View, qui permet à un WebReport d'être utilisé en tant que vue de formulaire.
EnabledLAPIFunctions Liste de chaînes Liste blanche de fonctions LAPI pouvant être exécutées comme une requête AJAX. Les fonctions suivantes sont toujours activées par défaut pour prendre en charge WebReports : {'wrlistnodes', 'wrlistusers', 'setwizarddata', 'getobjectinfo'} ; il est donc inutile de les définir explicitement dans le fichier opentext.ini. Vous pouvez ajouter d'autres fonctions à cette liste ou utiliser la valeur générique {'all'} pour activer toutes les fonctions.
collectConvDocCache Nombre entier Certaines destinations de WebReports autorisent la conversion de la sortie du WebReport avant qu'il soit envoyé à sa destination finale. Le document converti est stocké dans une mémoire cache qui est effacée par un agent 120 minutes plus tard par défaut. Ce paramètre définit une autre durée d'expiration pour cette mémoire cache.
SecureServicesSubtags Liste de chaînes Définit les sous-balises nécessitant des considérations de sécurité particulières lorsqu'elles sont utilisées au moyen de la fonction Services WR. Si l'une des sous-balises spécifiées est incluse dans une requête de services WR, le logiciel attend un jeton sécurisé fourni dans la requête. Par exemple, le code : ?func=webreports.runservice&servicetype=gettagdata&tagdata=1000&subtags=useraction:delete crée un message d'erreur, sauf si un jeton sécurisé valide a été obtenu auparavant (&servicetype=gettoken) et ajouté à la requête comme suit : ?func=webreports.runservice&servicetype=gettagdata&tagdata=1000&subtags=useraction:delete&securerequesttoken=123456.... Ce paramètre est ajouté automatiquement, avec une liste par défaut, lors des installations nouvelles et complètes ; en revanche, si le logiciel est installé lors d'une mise à jour, le paramètre par défaut est ajouté sous forme de commentaire. L'ensemble actuel de sous-balises incluses par défaut est le suivant : SecureServicesSubtags={'CATACTION','NODEACTION','SCACTION','PERMACTION','RMACTION', 'USERACTION','WFACTION','USERINFO','RMINFO','NODEINFO','EMAILINFO','WFINFO','AUDITINFO', 'VERINFO','SCINFO','POINFO'}
UpdateNumber Nombre entier Paramètre utilisé par le logiciel pour gérer ses versions de mises à jour. Ne doit pas être modifié sans l'avis du personnel d'assistance technique.

.
Exemple 1-1 : L'exemple suivant illustre une configuration type :

[WebReports]
addsearchbutton=Custom Report
VersionInput=C:\AdLib eXpress\Input
ConversionDir=C:\AdLib eXpress\Output
OscriptAllowFunction1={'Scheduler.debugbreak','Web.CRLF','Web.Escape','Web.Unescape','Web.DecodeForURL','Web.EncodeForURL','Web.Format','Web.EscapeHTML','Web.EscapeXML'}
OscriptAllowWholePkg={'Assoc','Bytes','Date','List','Math','Pattern','RecArray','Str','String','Boolean','undefined','void','integer','real','record'}
EnableOscriptReportviews=true
MaxNestedSubWebReports=10
additionalSearchColumns={{'OTSummary','Summary'},{'shortOTSummary','shortSummary'},OTHotWords,Score,parentRecord}
