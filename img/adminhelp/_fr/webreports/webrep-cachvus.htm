A chaque exécution d'un WebReport, un fichier correspondant à la version actuelle du fichier d'affichage du rapport est créé et mis en mémoire cache. Cette fonctionnalité est similaire à celle utilisée pour WebForms. Ces vues placées en mémoire cache sont présentes dans le dossier : <Content Server_hom>/support/webreports/views

Comme il est parfois souhaité que les utilisateurs exécutent différentes versions d'un même WebReport, toutes les versions des fichiers mis en cache sont conservées dans ce dossier. L'administrateur peut supprimer le contenu de ce dossier à tout instant. Ces fichiers sont créés dès qu'un utilisateur exécute un WebReport pour la première fois.
