Les tableaux suivants décrivent les paramètres de configuration de Records Management disponibles.

Tableau 1-1 : Paramètres de Records Management

Paramétrage Description Remarques complémentaires
Groupe Records Management Permet d'assigner un groupe au groupe Records Management principal. Les utilisateurs de ce groupe sont en mesure d'effectuer des tâches supplémentaires de Records Management, comme la mise à jour des informations détaillées du document, même s'ils ne disposent pas d'autorisations pour l'objet. Les utilisateurs de ce groupe doivent cependant recevoir des autorisations pour l'espace de travail de Records Management et ses objets, ainsi que l'espace de travail Classement, afin d'exécuter des fonctions associées à Records Management.
Fonction de copie Permet d'empêcher la copie d'objets Content Server. Si vous modifiez ce paramètre, vous devez redémarrer le serveur Content Server et le serveur d'application Web.
Autorisations Voir et Voir le contenu Permet de faire en sorte que les autorisations Voir et Voir le contenu soient modifiables pour l'accès public. Si cette option n'est pas activée, les autorisations Voir et Voir le contenu ne peuvent pas être modifiées pour l'accès public. Si vous modifiez ce paramètre, vous devez redémarrer le serveur Content Server et le serveur d'application Web.
Annulation des autorisations d'un projet Permet d'annuler les autorisations par défaut des projets.  
Autorisations de déplacement Permet de spécifier si les objets déplacés héritent des autorisations et du groupe de propriétaires des nouveaux objets parents.
Si le paramètre Hériter des autorisations est défini sur Oui, vous pouvez empêcher certains sous-types d'hériter des autorisations lors d'un déplacement en saisissant leurs numéros, séparés par une virgule, dans le champ Exclure les sous-types.
Si le paramètre Hériter des autorisations est défini sur Non, le paramètre Hériter du groupe de propriétaires n'est pas disponible.
 Les sous-types Espaces de travail (848) et Projets (202) n'héritent jamais des autorisations en cas de déplacement.
Rendre confidentiel Permet l'utilisation de fonctions de confidentialité. Vous avez également la possibilité d'indiquer si la fonction Enlever la confidentialité ne doit assigner que les autorisations Voir et Voir le contenu à l'accès public.  
Suppression sécurisée DoD Permet de choisir d'activer la suppression sécurisée DoD ou non. Lorsque des documents sont supprimés, une suppression sécurisée remplace le fichier sept fois avant de l'effacer. Une fois la suppression effectuée, le logiciel de récupération du fichier restaure le fichier dans un format lisible.  
Répertoire par défaut de l'exportation Permet de spécifier le répertoire d'exportation par défaut pour les documents et les métadonnées lors du traitement des actions de sort final et d'attente.
En mode d'exportation DoD, un journal d'exportation dont le nom commence par « export_ » est généré dans le répertoire suivant : [contentserver_installation_path/logs/dispositions].
 

Note :
Pour que la procédure d'importation/exportation DoD fonctionne correctement, certaines clés doivent être ajoutées manuellement dans la section [RecMan] du fichier opentext.ini. Pour plus d'informations sur des paramètres DoD spécifiques, contactez le service d'assistance OpenText.

 

Tableau 1-2 : Paramètres de sort final

Paramétrage Description
Nombre d'éléments à rendre dans une réserve Permet de définir le nombre d'éléments à rendre dans une réserve de façon à augmenter ou à diminuer le nombre d'éléments traités au cours d'une activité de sort final. Une réserve est un ensemble d'éléments traités dans le même événement de file d'attente. En limitant le nombre d'éléments traités simultanément, le thread est disponible pour d'autres requêtes Content Server entre les réserves.
Nombre de secondes entre les réserves Permet de définir le nombre de secondes d'intervalle entre les réserves de façon à augmenter ou à diminuer le temps d'attente du moteur de sort final entre chaque traitement de réserve d'éléments dans une activité de sort final.
Nombre de minutes de veille lorsque la réserve est vide Permet de définir le nombre de minutes de veille du moteur de sort final lorsque la réserve est vide.
Valeur de dérivation Permet de définir la valeur utilisée lors de l'exécution d'une recherche de sorts finals sur un recueil. Lors de l'exécution d'une recherche de sorts finals sur un recueil, la recherche parcourt la hiérarchie du recueil à la recherche d'éléments des types d'objets spécifiés. Lorsque le nombre d'éléments parcourus dépasse la valeur de dérivation, la recherche s'interrompt et une recherche standard est effectuée sans tenir compte de la hiérarchie du recueil.
