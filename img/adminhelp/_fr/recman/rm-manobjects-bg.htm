Suivi d'objets et de types d'objets

Vous pouvez effectuer le suivi des modifications de classement, de statut ISA, de droits supplémentaires et de date du document. Vous pouvez également effectuer le suivi des attentes, des commentaires relatifs à la suppression d'une attente, des officialisations, de la gestion des ISA et des codes, ainsi que de la création, de la suppression et de la modification des programmes ISA et des attentes. Pour plus d'informations sur l'audit des événements, reportez-vous à la rubrique Gestion de l`audit des événements.

 

Note :
Si vous effectuez le suivi de la gestion des ISA et des codes, les modifications apportées aux ISA ou aux codes sont enregistrées dans la table d'audit. Etant donné que les ISA et les codes ne sont pas des objets, les audits sont effectués sur la page Administration de Records Management et dans les recueils ISA.

 

Configuration de paramètres d'objets gérés

La page Objets gérés propose les options suivantes pour les types d'objets gérés :

  • Hériter à partir de, qui permet aux objets enfants d'hériter les classements RM de leur élément parent.

  • Brider les autorisations, qui garantit que les autorisations sont bridées lorsqu'un objet est officialisé.

  • Utiliser le titrage, qui permet de titrer un objet lorsque l'option Titrage est activée. Pour en savoir plus, voir Modification des paramètres par défaut de Records management.

  • Attente de l'utilisateur, qui permet de préciser à quels types d'éléments il est possible d'appliquer des attentes utilisateur.

  • Classement RM par défaut, qui permet de définir le classement RM par défaut pour un type d'objet spécifique. Des classements RM peuvent être affectés aux types d'objets gérés, qui peuvent être gérés par tout utilisateur possédant les autorisations appropriées.

Le tableau ci-dessous dresse la liste des objets gérés pris en charge qui s'affichent par défaut sur la page Objets gérés.

Consultez la documentation et les notes de version du module qui vous intéresse pour connaître les autres types d'objets pris en charge.

Vous pouvez ajouter à la page Objets gérés des types d'objets Content Server supplémentaires qui n'apparaissent pas par défaut sur cette page. Les types d'objets qui s'affichent dans le menu Ajouter un élément sont des types d'objets Content Server. Lorsqu'un module Content Server est installé, les types d'objets de ce module apparaissent dans le menu Ajouter un élément et vous pouvez les ajouter à la liste des types d'objets gérés.

 

Important :
Lorsque vous ajoutez un type d'objet géré à la page Objets gérés, celui-ci n'est pas géré automatiquement. Vous devez cocher la case Géré correspondant à chaque type d'objet devant être géré.
