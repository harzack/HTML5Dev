Vous pouvez configurer des paramètres globaux eLink pour les événements de workflow de tous les utilisateurs sur la page Paramètres globaux du workflow eLink. Vous pouvez également définir si des utilisateurs peuvent lancer des workflows en envoyant un e-mail.

Administration de paramètres globaux du workflow eLink

Lorsque vous activez un paramètre global eLink pour un événement du workflow, tous les utilisateurs reçoivent une notification eLink par e-mail pour cet événement, quels que soient les paramètres eLink définis par les utilisateurs pour l'événement sur leur page personnelle Modifier les paramètres eLink. Si vous n'activez pas de paramètre global, le système reconnaît les paramètres des utilisateurs. Par exemple, si vous activez l'événement Un workflow que je gère s'est achevé, les utilisateurs reçoivent un e-mail chaque fois qu'un workflow qu'ils gèrent est terminé, même s'ils ont indiqué qu'ils ne souhaitaient pas recevoir de notification pour l'événement dans leurs paramètres personnels eLink.

 

Important :
Vous devez configurer des paramètres globaux du workflow eLink ou les utilisateurs doivent configurer leurs paramètres locaux du workflow eLink pour recevoir des e-mails pour les événements de workflows dotés de fonctions d'e-mail.

 

Gestion du lancement de workflows par e-mail

eLink offre aux concepteurs de schéma de workflow l'option de définir une adresse e-mail eLink pour les workflows. Les utilisateurs peuvent utiliser cette adresse pour lancer un workflow en envoyant une requête de lancement dans un e-mail.

Vous pouvez autoriser les utilisateurs à lancer des workflows par e-mail en activant le paramètre correspondant. Ce paramètre s'applique à tous les schémas de workflows dans Content Server. Si vous n'activez pas le lancement de workflows par e-mail, les utilisateurs peuvent uniquement lancer des workflows à partir de Content Server.
