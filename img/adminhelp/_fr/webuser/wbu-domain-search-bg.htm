Lorsque vous activez les domaines, vous devez configurer la recherche Content Server pour qu'elle puisse fonctionner au sein de ces domaines. Pour configurer la recherche Content Server, vous devez définir les autorisations dans différentes zones du volume de l'objet système.

L'indexation et la recherche constituent des opérations globales, ce qui signifie qu'elles s'effectuent sur tous les domaines d'un site Content Server. Vous ne créez pas d'index pour un domaine spécifique mais pour le site Content Server tout entier.

Lorsque vous créez l'index Entreprise, les autorisations sont automatiquement définies afin que les utilisateurs d'un domaine ne puissent pas accéder aux informations qui résident dans un autre domaine. Par exemple, les utilisateurs du Domaine A peuvent accéder aux documents pour lesquels ils ont les autorisations dans le Domaine A uniquement. Toutefois, lorsque vous créez un index autre que l'index Entreprise, les autorisations ne sont pas définies automatiquement. Dans ce cas, si vous voulez limitez l'accès aux informations de l'index, vous devez définir les autorisations appropriées. Par exemple, si vous voulez que tous les utilisateurs du Domaine A accèdent aux informations indexées dans la vue de recherche A mais pas à celles de la vue B, vous devez définir les autorisations appropriées sur la vue de recherche A, en accordant l'accès seulement aux membres Content Server faisant partie du Domaine A.

Après avoir activé les domaines, vous devez donner aux utilisateurs l'autorisation d'enregistrer les modèles de recherche dans le dossier Modèles de recherche personnels. Cet ensemble d'autorisations étant probablement le même pour les membres de tous les domaines, vous pouvez définir les autorisations pour le groupe interdomaine que vous créez. Le groupe interdomaine comprend les membres de tous les autres domaines dans votre système Content Server.
