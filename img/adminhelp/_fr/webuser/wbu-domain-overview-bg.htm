Lorsque les domaines sont activés, un champ supplémentaire apparaît sous les champs Nom de l'utilisateur et Mot de passe sur la page Connexion de Content Server. Dans ce champ, nommé Domaine par défaut, l'utilisateur doit spécifier un domaine auquel se connecter. Le libellé d'affichage du champ varie selon le paramètre Nom d'affichage du domaine défini sur la page Configurer le domaine. Par exemple, si vous utilisez des domaines pour créer des espaces de travail distincts pour chaque société d'une organisation, vous pouvez définir le nom d'affichage du domaine sur Société. Les utilisateurs Content Server hors domaine laissent ce champ vide pour se connecter au système.

 

Note :
Content Server prenant en charge les domaines isolés, des utilisateurs et des groupes différents peuvent avoir le même nom d'utilisateur dans différents domaines.

 

Groupes et projets interdomaines

Les groupes interdomaines permettent une collaboration entre les domaines. Par exemple, si vous avez deux groupes d'utilisateurs, chacun dans un domaine différent, qui doivent collaborer sur un projet, vous pouvez créer un groupe interdomaines qui contient ces deux groupes d'utilisateurs.

Un groupe interdomaines peut contenir :

  • des utilisateurs de plusieurs domaines ;

  • des groupes de plusieurs domaines ;

  • des domaines ;

  • d'autres groupes interdomaines.

Vous pouvez également créer un projet interdomaines, en ajoutant des utilisateurs et des groupes de plusieurs domaines, afin de permettre aux groupes de partager des informations.

Pour créer un projet interdomaines, vous devez intégrer dans un projet standard des utilisateurs, des groupes, des domaines et des groupes interdomaines de plusieurs domaines.

Autorisations et privilèges

Seul un administrateur ou un utilisateur disposant des privilèges d'administration système et d'administration utilisateurs/groupes peut ajouter un domaine ou groupe interdomaines, ou accéder à différents domaines dans les pages des autorisations. L'interface des autorisations pour les autres utilisateurs reste la même. Toutefois, l'administrateur peut accorder des autorisations à tous les éléments Content Server entre les domaines, permettant ainsi aux utilisateurs de domaines différents d'accéder à ces éléments, notamment en utilisant la recherche Content Server. De même, seuls les administrateurs peuvent créer des projets interdomaines.

Un domaine entier peut être un groupe de propriétaires pour tout élément.

Si vous voulez que les utilisateurs du domaine puissent accéder aux volumes d'administration tels que le volume Workflow ou le volume Catégories, vous devez créer le groupe interdomaines Tous les domaines. Pour le volume des objets système, le dossier Modèles de recherche personnels, le volume DTD XML et le volume des rapports, vous devez définir manuellement les autorisations pour les utilisateurs du domaine.

Recherche de domaines

La recherche de domaines s'effectue de la même manière que pour des utilisateurs ou des groupes. Lorsque les domaines sont activés, cependant, en plus de la liste déroulante Rechercher et du champ qui commence par, la barre de recherche des utilisateurs et groupes comprend la liste déroulante dans. Par défaut, cette liste est définie sur le domaine du système, mais en dessous d'une ligne en pointillé, elle affiche la liste de tous les domaines du système. Le domaine du système est le domaine de tous les utilisateurs qui n'appartiennent à aucun domaine créé. Par défaut, le domaine du système est nommé Content Server.

Lorsque le domaine système est sélectionné, la liste déroulante Rechercher comprend les options Nom de domaine et Nom de groupe interdomaine. Si vous sélectionnez un domaine spécifique dans la liste déroulante dans, Content Server supprime ces options.

Si vous souhaitez afficher uniquement les utilisateurs et les groupes d'un domaine particulier, cliquez sur le nom de ce domaine dans la liste déroulante dans, puis effectuez votre recherche comme vous le feriez dans le système Content Server.

Configuration et modification de domaines

Tout domaine créé devient automatiquement membre du groupe interdomaines Tous les domaines, s'il existe. Un groupe par défaut est créé pour ce domaine au moment de sa création, avec le même nom que le domaine. Il fonctionne de la même façon que le groupe par défaut du domaine système Content Server. Une fois les domaines activés, vous créez de nouveaux domaines et groupes interdomaines de manière similaire aux utilisateurs et groupes.

 

Note :
Pour compléter un domaine, vous devez ajouter de nouveaux utilisateurs et groupes ; vous ne pouvez pas ajouter d'utilisateurs ou de groupes Content Server existants en tant qu'utilisateurs ou groupes du domaine. Lorsque vous ajoutez un utilisateur ou un groupe à un domaine, cet utilisateur ou ce groupe existe uniquement dans ce domaine, et non dans l'ensemble du système Content Server.

 

Les domaines ne peuvent pas être supprimés ; ils peuvent uniquement être désactivés. Lorsque vous désactivez les domaines, tous les domaines et groupes interdomaines existants sont conservés dans la base de données. Si vous réactivez les domaines, Content Server restaure automatiquement tous les domaines précédemment utilisés.

Un administrateur dispose d'une version spéciale de la barre de recherche Utilisateurs et groupes si la fonction Domaines est activée. En plus de la liste déroulante Rechercher et du champ qui commence par, la liste déroulante Domaine s'affiche. Cette nouvelle liste a par défaut votre domaine système sélectionné, mais elle répertorie tous les domaines du système. Lorsque le domaine système est sélectionné, la liste déroulante Rechercher comprend les options Nom de domaine et Nom de groupe interdomaine.

Le seul attribut d'un domaine que vous pouvez modifier est son nom. Cependant, vous pouvez effectuer les opérations suivantes pour les domaines sur la page Configurer le domaine :

  • Activer ou désactiver les domaines

  • Créer un groupe interdomaines Tous les domaines. Lorsque vous créez le groupe Tous les domaines, Content Server y intègre automatiquement tous les domaines du système Content Server. Le groupe Tous les domaines se met à jour automatiquement à chaque fois que vous ajoutez un autre domaine, pour que vous ne soyez pas obligé de le régénérer manuellement. En plus des domaines, le groupe Tous les domaines peut contenir des utilisateurs et des groupes. Pour les ajouter, vous modifiez le groupe.

  • Spécifiez un nom d'affichage (par exemple, Société) pour le domaine. Le nom apparaît sur les pages de connexion et de profil d'utilisateur.

  • Spécifiez un nom du domaine système pour représenter l'ensemble du système Content Server. Une fois que vous avez défini ce nom, il apparaît au-dessus des pointillés dans la liste déroulante Domaine des barres de recherche pour l'administrateur. Le nom par défaut est Content Server.
