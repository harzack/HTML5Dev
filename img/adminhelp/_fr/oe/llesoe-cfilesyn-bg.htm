Tous les fichiers de contrôle de l'exportation doivent commencer par une balise <export> et se terminer par une balise </export> .

Entre les balises <export></export> on trouve plusieurs paragraphes appelés paragraphes de nœud. Chaque paragraphe de nœud décrit une action relative à un objet donné. Les paragraphes de nœud commencent par une balise <node> et se terminent par une balise </node> .

Chaque fois que vous exécutez une exportation, les types d'objet suivants sont exportés : raccourci, document composite, vue personnalisée, discussion, document, dossier, projet, réponse, sujet et url. Pour éviter qu'un type d'objet spécifique soit exporté, utilisez la balise exclude .

Les pages suivantes contiennent la syntaxe et les descriptions des fichiers de contrôle de l'exportation pour les balises exportateur d'objet.

 

Note :
Object Exporter prend en charge plusieurs emplacements d'exportation.

 

Object Exporter et UTF-8

Si le fichier de contrôle comporte des caractères non-ASCII, ajoutez la ligne suivante, <?xml version="1.0" encoding="UTF-8"?>, en haut du fichier de contrôle et enregistrez ce dernier avec le codage UTF-8.

Exemple 2-1 : Voici un exemple de fichier de contrôle de l'exportation pouvant contenir des caractères non-ASCII :

<?xml version="1.0" encoding="UTF-8"?>
<export>
  <node>
    <location>Enterprise</location>
    <docstorage>c:\docs</docstorage>
  </node>
</export>
 

Note :
La valeur située entre les balises <docstorage></docstorage> doit être composée de caractères contenus dans le jeu de caractères ASCII.
