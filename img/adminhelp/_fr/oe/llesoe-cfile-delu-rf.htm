Balise deletedusers

Description

Par défaut, l'exportateur d'objet exportera les utilisateurs et les groupes supprimés. Pour éviter que ces données soient exportées, utilisez la balise <deletedusers>FALSE</deletedusers>.

Syntaxe

<deletedusers>FALSE</deletedusers>

Utilisation

  1. La balise deletedusers est facultative.

  2. La balise deletedusers a des attributs facultatifs : Attribut owner, Attribut ownergroup et Attribut creator. Ces attributs sont décrits ci-dessous :

Exemple

<export>
  <node>
    <location>Enterprise:Folder_001</location>
    <docstorage>c:\docs<docstorage>
    <deletedusers owner="Admin" ownergroup="DefaultGroup" creator="Admin">FALSE</deletedusers>
  </node>
</export>

Attribut owner

L'attribut owner définit une valeur qui peut être établie par défaut sur une autre valeur si un utilisateur ou un groupe supprimé est rencontré. La syntaxe de cet attribut est la suivante : 

<deletedusers owner="Admin" ownergroup="DefaultGroup" creator="Admin">FALSE</deletedusers>

Utilisation
L'attribut owner est facultatif si la balise deletedusers est utilisée.

Attribut ownergroup

L'attribut ownergroup définit une valeur qui peut être établie par défaut sur une autre valeur si un utilisateur ou un groupe supprimé est rencontré. La syntaxe de cet attribut est la suivante : 

<deletedusers owner="Admin" ownergroup="DefaultGroup" creator="Admin">FALSE</deletedusers>

Utilisation
L'attribut ownergroup est facultatif si la balise deletedusers est utilisée.

Attribut creator

L'attribut creator définit une valeur qui peut être établie par défaut sur une autre valeur si un utilisateur ou un groupe supprimé est rencontré. La syntaxe de cet attribut est la suivante : 

<deletedusers owner="Admin" ownergroup="DefaultGroup" creator="Admin">FALSE</deletedusers>

Utilisation
L'attribut creator est facultatif si la balise deletedusers est utilisée.
