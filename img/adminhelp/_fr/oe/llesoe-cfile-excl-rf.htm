Balise exclude

Description

Chaque fois que vous exécutez une exportation, les types d'objet suivants sont exportés : raccourci, document composite, vue personnalisée, discussion, document, dossier, projet, réponse, sujet et url. Pour éviter qu'un type d'objet spécifique soit exporté, utilisez la balise exclude .

Syntaxe

<exclude>
  <object>alias</object>
</exclude>

Utilisation

  1. La balise exclude est facultative.

  2. Si elle est utilisée, la balise exclude a une balise subordonnée requise, Balise object, décrite ci-dessous.

Notes

Si des objets de recueil, tels que des dossiers, sont exclus, les objets de ces recueils exclus ne seront pas exportés.

Exemple

Dans cet exemple, tous les objets de l'espace de travail Entreprise, sauf les documents composites et les vues personnalisées, seront exportés vers :

<export>
  <node>
    <location>Enterprise</location>
    <docstorage>c:\docs<docstorage>
    <exclude>
      <object>compounddoc</object>
      <object>customview</object>
    </exclude>
  </node>
</export>

Balise object

La balise object définit le nom de l'objet à exclure. La syntaxe de cette balise est la suivante : 

<object>discussion</object>

Utilisation
1.

La balise object est requise lorsque la balise exclude est utilisée.

2.

Les valeurs pouvant être utilisées avec la balise object sont les suivantes : alias (raccourci), compounddoc (document composite), customview (vue personnalisée) , discussion, document, folder (dossier) , project (projet), reply (réponse), topic (sujet)et url.
