OpenText Object Exporter permet d'exporter des objets depuis Content Server vers le système de fichiers. Il est possible de configurer le fichier de contrôle qui gère le processus d'exportation. Le fichier de contrôle permet de spécifier l'emplacement où se trouvent les objets à exporter dans Content Server ainsi que l'endroit du système de fichiers vers lequel seront exportés les objets. Les types d'objet Content Server suivants peuvent être exportés :

Type d'objet Syntaxe de l'exportateur d'objet
Raccourci alias
Document composite compounddoc
Vue personnalisée customview
Discussion discussion
Document document
Dossier folder
Projet project
Réponse reply
Sujet topic
URL url

Utilisation de la configuration de l'exportateur d'objet

La page Configurer l'exportateur d'objet permet de définir comment Object Exporter traite une exportation. Vous devez spécifier les chemins complets vers les répertoires de travail, des journaux et de destination sur la page Configurer Object Exporter. Vous devez d'abord créer ces trois répertoires et vous assurer qu'ils soient visibles pour Content Server :

  • Le répertoire de travail est l'emplacement où le ou les fichiers de sortie sont créés et enregistrés temporairement avant d'être déplacés une fois le processus terminé. Voici un exemple de répertoire de fichier de travail valide : C:\OPENTEXT\object_exporter\working, où C:\OPENTEXT est la valeur par défaut pour Content Server.

  • Le répertoire des journaux est l'emplacement où Object Exporter copiera les fichiers journaux. Pour afficher et supprimer des fichiers journaux de la page Object Exporter Afficher les fichiers journaux d'exportation, le répertoire des fichiers journaux doit se trouver dans <Content Server_home>\object_exporter. Voici un exemple de répertoire de fichier journal valide : C:\OPENTEXT\object_exporter\log, où C:\OPENTEXT est la valeur par défaut pour Content Server.

  • Le répertoire de destination est l'emplacement vers lequel Object Exporter déplacera le ou les fichiers de sortie une fois le processus terminé. Le ou les fichiers de sortie d'une exportation réussie sont en réalité des fichiers de contrôle correspondant à cette exportation réussie. À l'aide de OpenText Object Importer, vous pouvez utiliser ce ou ces fichiers pour importer tous les objets que vous avez récemment exportés, puisque toutes les autorisations, titres, attributs et autres métadonnées de l'objet sont détaillés dans ces fichiers de contrôle de l'importation. Voici un exemple de répertoire de destination valide : C:\OPENTEXT\object_exporter\destination, où C:\OPENTEXT est la valeur par défaut pour Content Server.

 

Important :
Les répertoires de travail, des journaux et de destination doivent être créés et visibles pour Content Server. Vous devez les spécifier sur la page Configurer Object Exporter pour que les exportations puissent avoir lieu.

 

En plus de la création des répertoires spécifiques à Object Exporter détaillée plus haut, vous devez également :

  • Vous assurer que l'adresse e-mail de l'utilisateur Admin est définie dans la page du compte de l'utilisateur Admin. Vous accédez à cette page en sélectionnant Utilisateurs et groupes dans le menu Entreprise. Recherchez l'utilisateur Admin sur la page Utilisateurs et groupes, puis cliquez sur Modifier en regard du nom de l'utilisateur Admin. Entrez l'adresse e-mail de l'utilisateur Admin.

  • Complétez la page Notifications. Il n'est pas nécessaire que la page Notifications soit activée. Pour plus d'informations sur la page Notifications, reportez-vous à la rubrique Configuration des notifications, des rapports, des événements et des paramètres dans Suppression des événements en suspens.

 

Important :
Lorsque l'adresse e-mail de l'utilisateur Admin n'est pas définie ou lorsque la page Notifications n'est pas complétée, les exportations n'ont pas lieu.

 

Sur la page Configurer Object Exporter, vous devez déterminer la configuration du champ Objets par lot. Le champ Objets par lot vous permet de stipuler que les objets exportés doivent être divisés en plusieurs fichiers de sortie, pour éviter d'avoir un seul fichier de très grande taille. Ces fichiers de sortie sont ceux qui sont copiés dans le répertoire Destination que vous avez créé plus haut.

Utilisation de l'exportation manuelle d'objet Object Exporter

Les exportations d'objets sont lancées à partir de la page Exportation manuelle d'objet à l'aide du bouton Démarrer l'exportation. À partir de cette page, vous choisissez le fichier de contrôle de l'exportation que Object Exporter exécutera pour exporter vos données depuis Content Server vers votre système. Sur cette page, vous devez également indiquer le nom du fichier journal sur lequel Object Exporter enregistrera toutes les informations de l'exportation.

Exemple 1-1 : Exemple de fichier de contrôle de l'exportation qui exclut les documents et dossiers de l'exportation :

<export>
  <node>
    <location>Enterprise</location>
    <docstorage>c:\docs</docstorage>
    <exclude>
      <object>document</object>
      <object>folder</object>
    </exclude>
  </node>
</export>

Utilisation des fichiers journaux Object Exporter

Pour afficher et supprimer des fichiers journaux des pages d'administration Content Server, le répertoire de fichiers journaux doit se trouver dans : <Content Server_home>\object_exporter.
