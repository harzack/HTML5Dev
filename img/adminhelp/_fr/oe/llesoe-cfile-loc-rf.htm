Description

 

Important :
Cette balise est obligatoire, quel que soit le type d'objet.

 

La balise location spécifie l'emplacement dans Content Server à partir duquel l'exportateur d'objet commencera à exporter des fichiers.

 

Note :
L'exportateur d'objet prend en charge les fonctions multilingues disponibles dans Content Server. Lorsqu'une nouvelle instance de Content Server est installée, l'administrateur choisit la langue qui sera définie par défaut pour le système. Vous pouvez voir quelle est la langue par défaut de votre système, ainsi que toutes les langues qui ont été installées et activées pour les métadonnées dans la section Métadonnées de la page d'administration de Content Server. Cliquez sur le lien Configurer les métadonnées multilingues. Pour plus d'informations sur la configuration des métadonnées multilingues sur Content Server, consultez la rubrique Configuration de métadonnées multilingues sous la section Administration système dans Configuration de métadonnées multilingues.

Étant donné que l'exportateur d'objet prend désormais en charge les métadonnées multilingues, notez que l'emplacement de l'objet dépend de la langue par défaut du système, lorsque cette dernière a été définie. Si aucune langue par défaut n'a été définie pour le système, l'emplacement de l'objet dépend de la première langue qui apparaît dans l'ordre alphabétique, par code.

 

Syntaxe

<location>Enterprise</location>

Utilisation

Cette balise est obligatoire, quel que soit le type d'objet.

Exemple

Dans cet exemple, tous les objets pris en charge par l'espace de travail Entreprise seront exportés :

<export>
  <node>
    <location>Enterprise</location>
    <docstorage>c:\docs</docstorage>
  </node>
</export>
