Quand les utilisateurs travaillent sur un document, une nouvelle version est créée dans Content Server à chaque enregistrement. En fonction de vos processus, ces versions du document peuvent être requises ou non. En tant qu'administrateur, vous pouvez spécifier comment sont gérées ces versions intermédiaires. Par défaut, seule la version finale du document est retenue à l'issue des modifications. Vous pouvez également indiquer qu'une nouvelle version est créée dans Content Server chaque fois qu'un utilisateur enregistre son travail. Si l'option est activée, l'événement d'audit Enterprise Connect-Version annulée et remplacée signale les versions intermédiaires.
