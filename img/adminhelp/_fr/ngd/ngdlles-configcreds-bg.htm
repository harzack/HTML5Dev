La case à cocher Autoriser le client Enterprise Connect à stocker les informations d'identification de l'utilisateur sur le disque vous permet de déterminer la façon dont les informations d'identification d'Enterprise Connect sont stockées, et d'appliquer cette configuration à tous les utilisateurs d'Enterprise Connect accédant à cette instance de Content Server. Par défaut, lorsque les utilisateurs se connectent à Enterprise Connect, leurs informations d'identification sont stockées sur le système de fichiers local, dans un format crypté. Cela signifie que les utilisateurs ne sont pas invités à fournir leurs informations d'identification chaque fois qu'ils démarrent une nouvelle session Enterprise Connect, sauf si leurs informations d'identification stockées deviennent non valides. Le cas échéant, vous pouvez spécifier que les informations d'identification de l'utilisateur ne doivent pas être stockées, ce qui signifie que les utilisateurs seront invités à fournir leurs informations d'identification chaque fois qu'ils démarrent une nouvelle session Enterprise Connect.

 

Note :
La sélection de cette case à cocher prévaut sur l'option spécifiée dans le paramètre de registre DontStoreCredentialsLocally . Par exemple, si vous désactivez cette case, les informations d'identification ne sont jamais stockées en local, quel que soit le paramètre de registre. Pour plus d'informations, consultez la section OpenText Enterprise Connect - Installation Guide (NGDCORE-IGD) .
