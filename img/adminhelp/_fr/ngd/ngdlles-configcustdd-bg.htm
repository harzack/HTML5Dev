Vous pouvez spécifier le type d'objet qui est créé lorsque les utilisateurs font glisser certains types de dossiers depuis leur système de fichiers vers Content Server. Pour cela, vous spécifiez que les dossiers dont le nom présente certaines extensions deviennent soit des Dossiers d'e-mails ou des Documents composites lorsque les utilisateurs les font glisser vers Content Server. Cette fonctionnalité n'est pas disponible dans Microsoft Outlook.

Par défaut, aucune valeur n'est fournie. Si vous spécifiez plusieurs extensions, utilisez des virgules pour les séparer.

Par exemple, un utilisateur fait glisser un dossier appelé test.cd depuis son système de fichiers vers Content Server, et vous avez spécifié que les dossiers avec l'extension CD doivent être traités en tant que documents composites. L'extension CD est supprimée du nom du dossier et un nouveau document composite est créé à la destination. Tous les éléments résidant dans le dossier sont inclus dans l'action et les utilisateurs seront invités à fournir les métadonnées sur la base des paramètres que vous avez spécifiés. Les sous-dossiers résidant dans le dossier principal ne sont pas inclus dans l'action. Si la destination contient déjà un élément du même nom, les règles de résolution des conflits de dénomination que vous avez spécifiées seront appliquées. Si le dossier contient un type d'élément qui n'est pas autorisé, l'action est abandonnée.
