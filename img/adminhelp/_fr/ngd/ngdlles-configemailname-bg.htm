Lorsque des utilisateurs ajoutent des e-mails à Content Server à l'aide d'Enterprise Connect, des conventions de dénomination sont appliquées à l'e-mail. La convention de dénomination par défaut utilise l'objet de l'e-mail comme nom de l'e-mail, sans séparateurs ni exclusions. Vous pouvez personnaliser les conventions de dénomination comme suit :

  • Définissez les propriétés de message à utiliser dans les noms des e-mails.

  • Définir le séparateur à utiliser entre les propriétés de message qui composent le nom de l'e-mail.

  • Spécifiez les chaînes que vous souhaitez supprimer du nom de l'e-mail.

  • Spécifiez le format de la date dans l'e-mail.

  • Spécifiez comment gérer les e-mails portant le même nom.

 

Notes :

  • Même si Enterprise Connect applique les conventions de dénomination spécifiées lorsque les utilisateurs ajoutent des e-mails à Content Server, les utilisateurs peuvent renommer les e-mails s'ils le souhaitent.

  • Si vous rendez la page Mes paramètres accessible aux utilisateurs d'Enterprise Connect, ceux-ci pourront configurer ces conventions de dénomination des e-mails comme ils le souhaitent. Pour plus d'informations, consultez la section Pour configurer la structure d'arborescence et le volet Liste.

  • Si Enterprise Connect génère un nom pour l'e-mail déjà utilisé dans le recueil Content Server, les utilisateurs seront invités à résoudre le conflit de dénomination pour pouvoir ajouter l'e-mail, ou Enterprise Connect génèrera un nouveau nom pour l'e-mail, avec un suffixe, afin de le différencier de l'e-mail déjà présent dans la destination cible. L'option accessible aux utilisateurs dépend de vos paramètres d'administration. Pour plus d'informations sur la résolution des conflits de dénomination des fichiers, voir OpenText Enterprise Connect Plug-in for Content Server - User Help (NGDLLES-H-UGD).

  • Toutefois, si vous enregistrez les e-mails en tant que sous-types d'e-mails et que vous identifiez les doublons à l'aide de la méthode de comparaison des ID de messages, les doublons d'e-mails ne sont pas ajoutés à la cible et les utilisateurs ne sont pas informés de la présence d'un doublon. Notez que vous serez invité à déterminer la méthode de gestion des e-mails composites, car ils sont exclus de cette règle.
