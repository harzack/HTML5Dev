Lors de la configuration des paramètres de communauté, vous configurez le comportement de toutes les communautés dans Content Server. Tableau 1-1 décrit les paramètres de communauté que vous pouvez configurer.

Tableau 1-1 : Paramètres configurables pour les communautés

Paramètre Description
‎Console de navigation Lorsque ce paramètre est activé, la page Mon espace de travail de chaque utilisateur affiche la Console de navigation des communautés, permettant ainsi à l'utilisateur d'accéder rapidement à ses communautés.
Page d'accueil personnelle Lorsque ce paramètre est activé, la Page d'accueil personnelle s'affiche sur la Console de navigation des communautés et sur le menu Personnel.
Fonctions utilisateurs indirects Lorsque ce paramètre est activé, les fonctions utilisateurs indirects permettent aux modérateurs de choisir si certaines fonctions de communautés (telles que les fonctions e-mail) s'appliqueront aux membres indirects et aux membres directs, ou seulement aux membres directs. Les membres directs d'une communauté sont des utilisateurs qui sont ajoutés individuellement, tandis que les membres indirects sont des utilisateurs qui sont ajoutés par l'intermédiaire d'un groupe ajouté à une communauté.
Désactiver le filtrage JavaScript Lorsque activé, le filtrage JavaScript pour les objets CoP est désactivé. Les objets CoP qui utilisent ckEditor sont sujets à des attaques par JavaScript. Ainsi CoP utilise la Bibliothèque Antisamy livré avec Content Server pour filtrer tel code malveillant. Si vous ne voulez pas un tel filtrage, veuillez cochez la case.
Informations publiques sur les membres Lorsque ce paramètre est activé, les non-membres ont accès à la page d'informations sur les membres des communautés publiques.
Remplacement du statut Lorsque ce paramètre est activé, si vous changez le type de communauté, vous changez également les autorisations d'accès public correspondantes (Voir et Voir les éléments) pour tous les sous-éléments. Par exemple, le fait de faire passer un type de communauté de Public à Privé ou à Non répertorié supprime les autorisations Voir et Voir les éléments des sous-éléments de la communauté. Ceci les rend accessibles aux membres de la communauté uniquement.
Taille max. de la photo personnelle Ce paramètre permet de définir la taille maximale, en octets, de la photo qui peut être affichée dans les fonctions de la communauté (par exemple dans les messages de forum). La taille par défaut est de 50 000 octets. Les dimensions optimales pour une image sont 80 par 80 pixels. Une valeur de 0 désactive l'affichage des photos.
Rendez-vous de calendrier Lorsque ce paramètre est activé, tous les membres d'une communauté bénéficiant des autorisations d'ajout d'éléments peuvent ajouter des rendez-vous. Dans le cas contraire, seul le modérateur est habilité à entrer des rendez-vous.
Affichage du calendrier Ce paramètre permet de configurer l'affichage par défaut du calendrier d'une communauté sur Affichage du calendrier ou Affichage liste et de définir les heures de début et de fin par défaut du calendrier.
Mesures Lorsque ce paramètre est activé, il permet aux modérateurs d'afficher les mesures de la communauté dans le résumé de la communauté. Les mesures sont des statistiques qui indiquent la robustesse générale de la communauté.
Liens abrégés Ce paramètre permet de représenter les URL longues sous une forme raccourcie en remplaçant le début ou la fin des URL par des points de suspension.
Pied de page d'e-mail HTML Ce champ spécifie le pied de page pour tous les e-mails HTML envoyés depuis les communautés.
Pied de page d'e-mail Texte Ce champ spécifie le pied de page pour tous les e-mails texte envoyés depuis les communautés.
Eléments auditables Les types d'élément sélectionnés dans cette section apparaissent dans les rapports de communauté, tels que les blogs.
