OpenText Communities fournit aux utilisateurs d'autres manières de collaborer dans Content Server. Une communauté est un recueil géré par un modérateur qui stocke des informations spécialisées, comme des éléments Content Server et des rapports personnalisés. Une communauté est présentée dans une interface personnalisée ayant l'apparence d'un journal, pouvant être créée autour de différents thèmes, par exemple un service dans une entreprise, un sujet spécifique ou une affiliation professionnelle.

Vous pouvez effectuer les tâches de gestion suivantes pour les communautés :

  • Configuration des paramètres système de toutes les communautés.

  • Gestion des modèles de communauté, permettant de créer une nouvelle communauté en utilisant une structure par défaut et du contenu existant.

  • Configuration de rapports d'éléments évalués, correspondant à des évaluations données par les utilisateurs aux éléments créés par les destinataires des rapports. Pour plus d'informations, voir Configuration des rapports sur les éléments évalués.

  • Configuration des restrictions e-mail pour l'envoi d'e-mails depuis les communautés.

  • Configuration de l'annuaire des communautés, répertoriant l'ensemble des communautés de Content Server et les questions globales. Les communautés sont hiérarchisées dans des structures d'annuaires et des annuaires par l'administrateur Content Server, en fonction des concepts que les utilisateurs ont choisi pour identifier les communautés et les questions. L'annuaire des communautés comporte également d'autres fonctionnalités liées aux communautés (par exemple, les questions récemment posées pour l'annuaire actif).

 

Note :
Les tâches de gestion supplémentaires dans la section Gestion des communautés concernent les Questions/Réponses. Pour de plus amples informations sur ces tâches, reportez-vous à la section Gestion des Questions et Réponses.
