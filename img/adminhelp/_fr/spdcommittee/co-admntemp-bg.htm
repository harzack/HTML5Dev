Les modèles de communauté fournissent une structure et un contenu par défaut pour les communautés. Vous ajoutez et configurez des modèles de communauté pour qu'ils conviennent à une certaine utilisation (par exemple, un groupe de développement de produits), afin de faciliter la configuration des communautés créées récemment. Les modèles de communauté sont stockés dans le volume des modèles de communauté, qui les met à disposition dans l'ensemble du système. Vous pouvez organiser vos modèles de communauté dans le volume en les stockant dans des dossiers de modèles de communauté.

Lorsque vous ajoutez un modèle de communauté, vous le configurez de la même manière que vous configureriez une nouvelle communauté, y compris la mise en forme et l'ajout d'URL sur la page d'accueil de la communauté. Vous pouvez aussi configurer le modèle de communauté comme vous le feriez lors de l'activation d'une communauté, y compris gérer les demandes d'abonnement, configurer les abonnements et gérer les groupes d'experts. Pour de plus amples informations sur la configuration des communautés, reportez-vous à la section Activation d`une communauté.

 

Note :
Lors de l'ajout d'un modèle de communauté, vous devez créer un rôle Modérateur avec toutes les autorisations pour qu'un ou plusieurs utilisateurs puissent gérer les communautés créées avec le modèle.

Des forums peuvent être ajoutés à un modèle de communauté si vous commencez à créer un forum depuis un autre recueil dans Content Server, mais que vous sélectionnez ensuite le modèle de communauté comme destination cible dans lequel placer le forum indiqué dans le champ Créer dans.

 

Au lieu de créer tous les modèles de communauté en partant de zéro, vous pouvez ajouter un modèle de démonstration qui servira de base à tous les modèles ultérieurs. Le modèle de démonstration est préconfiguré afin de fournir aux utilisateurs les caractéristiques de base pour les modèles de communauté et les communautés de votre organisation.
