Lorsque vous créez un modèle de projet, Content Server génère une « copie » du projet à partir duquel vous souhaitez créer ce modèle. Ce cliché comprend l'ensemble du contenu du projet, notamment les intervenants, les tâches, les documents et les discussions. Lorsque vous créez un projet à partir d'un modèle de projet, le modèle est copié dans le nouveau projet. Vous pouvez créer un modèle de projet dans le volume des modèles ou depuis le menu Fonctions d'un projet existant. Pour ce faire, vous devez posséder l'autorisation Ajouter un élément pour le recueil où vous ajoutez un modèle de projet.

Le modèle de vue personnalisée vous permet d'ajouter un fichier Vue personnalisée auquel les utilisateurs peuvent accéder lorsqu'ils créent une vue personnalisée dans un dossier ou un espace de travail. Lorsque vous ajoutez un modèle de vue personnalisée au volume Modèles, les utilisateurs le voient dans le volume Modèles Content Server, qui correspond au recueil apparaissant par défaut quand un utilisateur ajoute une vue personnalisée. Seuls les administrateurs, ou utilisateurs disposant de droits en Administration, peuvent supprimer des modèles de vue personnalisée à partir du volume Modèles.

 

Note :
Si l'autorisation du dossier où se trouve une vue personnalisée avec un formulaire de recherche enregistré est modifiée et si l'utilisateur ne modifie pas l'autorisation du formulaire de recherche enregistré, il se peut qu'il ne puisse pas accéder au dossier.
