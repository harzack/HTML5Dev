Vous pouvez construire le thésaurus dans n'importe quel emplacement pour lequel vous disposez d'une autorisation en écriture et d'un espace disque suffisant. Le fichier du thésaurus peut être assez volumineux, selon le nombre de termes définis dans votre fichier de modèle. Par exemple, si votre fichier de modèle contient 1 000 000 d'entrées, le fichier du thésaurus peut atteindre une taille approximative de 100 Mo. Avant de pouvoir utiliser le thésaurus dans le moteur de recherche, vous devez copier le fichier dans le répertoire de configuration, généralement Content Server_accueil/config. Une fois que vous avez créé le fichier de modèle XML que vous souhaitez utiliser, vous pouvez construire votre thésaurus personnalisé en utilisant la commande buildths.

Pour... Définissez le paramètre suivant...
Spécifier un thésaurus de langue standard DefaultThesaurus=<langue_thésaurus>
Spécifier le thésaurus multi-lingue DefaultThesaurus=thesaurus.eur
Spécifier un thésaurus personnalisé DefaultThesaurus=<nom_thésaurus_personnalisé>
Répertorier plusieurs thésaurus DefaultThesaurus=<nom_thésaurus,langue_thésaurus,nom_thésaurus>
Appliquer un marqueur d'arrêt DefaultThesaurus=-<thésaurus_spécifié>

Syntaxe

Tous les paramètres sont obligatoires.

buildths -name <nom_thésaurus> -infile i<fichier_xml_entrée>

où <thesaurusname>.ths correspond au chemin d'accès au fichier du thésaurus créé et où <inputxmlfile> correspond au chemin d'accès au fichier de modèle XML.

 

Note :
La commande buildths est située dans votre répertoire bin Content Server, généralement nommé Content Server_accueil/bin. Vous pouvez appeler la commande à partir du répertoire bin ou du répertoire Content Server_accueil.
