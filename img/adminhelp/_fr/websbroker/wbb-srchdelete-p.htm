Pour effectuer une recherche dans la liste des éléments supprimés :

  1. Lancez le programme SQL de votre base de données et connectez-vous avec le nom d'utilisateur et le mot de passe appropriés.

  2. Dans une fenêtre de requête, saisissez et exécutez la requête SQL suivante :select DataID from DTree où SubType=402. La valeur obtenue avec cette requête est l'ID du nœud du volume des documents supprimés.

  3. Connectez-vous en tant qu'administrateur Content Server, puis accédez à la page Recherche avancée dans Content Server.

  4. Dans la liste déroulante Rechercher, cliquez sur Requête complexe.

  5. Saisissez "<OTLocation>nnnn" dans le champ de recherche Texte intégral, où nnnn est l'ID de nœud du volume des documents supprimés obtenu à l'étape 2.

  6. Cliquez sur le bouton Enregistrer la requête.

  7. Sur la page Ajouter Requête, saisissez Objets supprimés dans le champ Nom.

  8. Cliquez sur le bouton Parcourir Content Server pour accéder au dossier Système, puis cliquez sur le lien Sélectionner qui se trouve à côté de Dossier Vues de recherche.

  9. Sur la page Ajouter Requête, cliquez sur le bouton Ajouter.

  10. Naviguez jusqu'au dossier Vues de recherche.

  11. Cliquez sur l'icône Fonctions située à côté de la vue de recherche Objets supprimés, puis cliquez sur Autorisations.

  12. Sur la page Autorisations, désactivez les autorisations pour tous les groupes et utilisateurs autres que les administrateurs, puis cliquez sur le bouton Terminé.
