Les modèles de recherche vous permettent de définir l'apparence par défaut de la page Recherche dans Content Server. Vous pouvez configurer des champs et des valeurs spécifiques que vous souhaitez afficher par défaut lorsque les utilisateurs effectuent une recherche utilisant un formulaire de recherche. Par exemple, si des utilisateurs recherchent fréquemment des documents qu'ils ont créés, vous pouvez ajouter l'attribut système Créé par à la page Recherche et l'enregistrer en tant que formulaire de recherche. Créé par s'affichera par défaut à chaque utilisation de ce formulaire de recherche.

Vous pouvez réaliser des tâches d'administration de modèle de recherche lorsque vous vous connectez à Content Server en tant qu'utilisateur Admin. Vous pouvez modifier ou supprimer les modèles système, ainsi que tous les modèles personnels créés par les utilisateurs Content Server. Lorsque vous modifiez un formulaire de recherche pour changer son apparence, les changements apportés sont immédiatement disponibles pour les utilisateurs qui peuvent accéder au formulaire de recherche. Si vous souhaitez qu'un formulaire de recherche ne soit plus à disposition des utilisateurs Content Server, vous pouvez le supprimer. Vous pouvez également exécuter d'autres fonctions Content Server standard (telles que la modification des autorisations, la définition des notifications et l'affichage des informations générales ou spécifiques) sur tous les modèles de recherche présents dans le système.

 

Note :
Si vous avez activé les domaines et que vous supprimez tous les modèles de recherche puis réinitialisez le modèle système par défaut, vous devez réappliquer les autorisations Ajouter un élément et Supprimer pour le dossier Modèles de recherche personnels.

Si vous modifiez le nom du dossier Modèles de recherche personnels d'un utilisateur, cela supprime les modèles enregistrés de l'utilisateur. Votre utilisateur ne peut plus utiliser les modèles existants car ils n'apparaissent plus dans la liste déroulante.

 

Lorsque vous créez un formulaire de recherche, vous pouvez également configurer les options d'affichage des résultats de recherche à associer à ce formulaire de recherche. Pour en savoir plus, voir Configuration des options d'affichage des résultats de recherche.

Modèles de recherche système

Vous pouvez créer jusqu'à 100 modèles de recherche système pour autant de configurations de page Recherche répondant aux différents besoins des utilisateurs. Vous pouvez définir des autorisations pour chaque modèle de recherche de sorte qu'ils ne s'affichent que pour les utilisateurs et les groupes désignés. Les utilisateurs peuvent ensuite sélectionner un modèle de recherche dans une liste permettant d'accéder facilement aux configurations de recherche de leur choix. Lorsque vous créez un modèle de recherche alors que vous êtes connecté en tant qu'utilisateur Admin, le modèle de recherche est enregistré automatiquement en tant que modèle de recherche système. Pour plus d'informations sur la création et l'utilisation des modèles de recherche dans Content Server, reportez-vous à la rubrique Utilisation des formulaires de recherche.

Modèle de recherche système par défaut

Lorsque vous êtes connecté en tant qu'utilisateur doté de droits Admin, vous pouvez également modifier ou réinitialiser le modèle de recherche système par défaut. Lorsque vous vous connectez pour la première fois, Content Server génère un modèle système intitulé Admin - Par défaut. Ce modèle est une copie du modèle système par défaut d'origine. Tous les utilisateurs Content Server y ont accès. Le modèle de recherche système par défaut définit la présentation de la page Recherche Content Server lorsque les utilisateurs Content Server accèdent à la page pour la première fois, lorsqu'aucun modèle de recherche n'existe dans le système ou lorsque les utilisateurs Content Server choisissent de ne pas utiliser de modèles personnels ou système. Vous pouvez modifier le modèle de recherche système par défaut afin de spécifier la présentation de la page Recherche Content Server dans ces cas-là. Si vous voulez supprimer les modifications apportées au modèle système par défaut, vous pouvez réinitialiser le modèle. La réinitialisation supprime puis recrée le modèle système par défaut dans sa configuration d'origine. Pour en savoir plus, voir Pour réinitialiser les modèles de recherche.

 

Note :
Les sites Content Server devant toujours contenir un modèle système par défaut, Content Server réinitialise automatiquement le modèle Admin-Default si vous essayez de le supprimer.

 

Modèles de recherche personnels

Par défaut, tous les utilisateurs Content Server peuvent créer 20 formulaires de recherche personnels, mais si vous voulez permettre aux utilisateurs Content Server d'en créer plus ou moins de 20, vous pouvez changer cette valeur. Cela permet de réguler le nombre de formulaires de recherche personnels sur votre site Content Server (entre 0 et 100 par utilisateur) et de faciliter la gestion des modèles de recherche. Vous pouvez supprimer ou modifier les formulaires de recherche personnels des utilisateurs depuis les pages Administration.

 

Note :
Si vous modifiez le nom du dossier Modèles de recherche personnels d'un utilisateur, cela supprime les modèles enregistrés de l'utilisateur. Votre utilisateur ne peut plus utiliser les modèles existants car ils n'apparaissent plus dans la liste déroulante.

 

Modèles de résultats de recherche

Par défaut, tous les utilisateurs Content Server peuvent créer 20 modèles de résultats de recherche, mais si vous voulez permettre aux utilisateurs Content Server d'en créer plus ou moins de 20, vous pouvez changer cette valeur. Cela permet de réguler le nombre de Modèles de résultats de recherche sur votre site Content Server (entre 0 et 100 par utilisateur) et de faciliter la gestion des modèles de recherche. Vous pouvez supprimer ou modifier les Modèles de résultats de recherche des utilisateurs depuis les pages Administration.
