La page Configurer les paramètres de l'agent de workflow vous permet de spécifier les intervalles d'exécution de l'agent de workflow, d'activer la notification d'erreurs et de définir l'ordre de traitement des tâches. Pour plus d'informations sur l'agent de workflow, voir Description de l`agent de workflow.

Lorsque l'agent de workflow démarre, il recherche dans la base de données Content Server toutes les tâches dont le statut indique qu'elles sont prêtes pour l'agent de workflow. Lorsque l'agent de workflow a terminé le traitement des tâches, il s'arrête jusqu'au prochain intervalle de démarrage programmé.

Les tâches sont traitées par l'agent de workflow ou l'agent distribué. La manière dont les tâches sont traitées dépend de l'ordre de traitement sélectionné. Si l'ordre FIFO est activé, l'agent de workflow traite les tâches. Si l'ordre FIFO est désactivé, l'agent de workflow utilise les agents distribués pour un traitement parallèle des tâches.

Activation de la notification d'erreurs

Vous devez activer la notification d'erreurs de l'Workflow Agent pour vous assurer que les messages d'erreur sont envoyés en cas d'échec du traitement. Si vous désactivez la notification d'erreurs, aucun utilisateur ne reçoit de notification en cas d'échec du traitement.

Si l'agent de workflow rencontre une erreur, il essaie d'envoyer un e-mail qui informe les gestionnaires des workflows et l'exécutant de l'étape de l'erreur et leur demande l'action à opérer pour corriger le problème. Pour envoyer les messages d'erreur, l'agent de workflow a besoin des informations suivantes :

  • Les paramètres SMTP du serveur de messagerie et l'adresse e-mail de l'expéditeur doivent être spécifiés.

  • Les gestionnaires des workflows et les utilisateurs assignés à l'étape doivent spécifier leur adresse e-mail dans leur profil.

Vous devez spécifier les paramètres SMTP du serveur de messagerie dans les sections suivantes :

  • Paramètres SMTP : vous permet de définir les paramètres SMTP du serveur de messagerie.

  • Paramètres de l'e-mail : vous permet de définir l'adresse e-mail de l'expéditeur.

Vous pouvez par ailleurs tester vos paramètres .

Spécification de l'ordre de traitement

Vous pouvez spécifier que l'agent de workflow doit traiter les tâches dans l'ordre dans lequel il les a reçues (ordre FIFO) ou selon un ordre défini par le résultat d'une requête de base de données (cet ordre peut varier d'une requête à l'autre). Si vous n'avez pas beaucoup de tâches à traiter, ce paramètre peut ne pas avoir beaucoup d'influence.

Si vous désactivez l'ordre FIFO, l'agent de workflow ne trie pas les tâches après avoir recherché dans la base de données toutes les tâches à traiter ; il effectue les tâches dans l'ordre dans lequel elles apparaissent dans le résultat de la recherche. Pour augmenter le débit, désactivez l'ordre FIFO.
