Pour programmer l'Workflow Agent :

  1. Cliquez sur le lien Configurer les paramètres de l'agent de workflow dans la zone Administration des workflows.

  2. Indiquez le moment où vous voulez que l'Workflow Agent s'exécute en effectuant l'une des opérations suivantes :

    • Programmez les heures d'exécution de l'agent de workflow en indiquant les heures de début dans les champs suivants :

      • Dans la section Les jours suivants, sélectionnez les cases à cocher correspondant aux jours de la semaine où l'agent de workflow doit être exécuté

      • Dans la section Aux heures suivantes , cochez les cases correspondant aux heures d'exécution de l'agent de workflow.

      • Dans la section Aux minutes suivantes, sélectionnez les cases à cocher correspondant aux intervalles de minutes d'exécution de l'agent de workflow.

    • Dans le champ Intervalle de veille, indiquez le nombre de secondes pendant lesquelles Content Server attend avant d'exécuter de nouveau l'agent de workflow après avoir démarré ou exécuté l'agent de workflow. Si vous voulez par exemple que l'Workflow Agent s'exécute toutes les trois minutes, saisissez 180 dans le champ Intervalle de veille.

     

    Note :
    Vous devez soit programmer des heures de début soit indiquer un intervalle de veille. Si vous programmez des heures de début et que vous indiquez un intervalle de veille, ce dernier est ignoré.

     

  3. Dans la section Envoyer des e-mails d'erreur, sélectionnez si vous voulez que le gestionnaire principal du workflow reçoive un e-mail lorsqu'une erreur de workflow se produit.

  4. Dans la section Traiter les tâches dans l'ordre FIFO, sélectionnez si vous voulez que l'agent de workflow traite les tâches dans l'ordre FIFO. Pour en savoir plus, voir Spécification de l'ordre de traitement.

  5. Cliquez sur Enregistrer les modifications.

  6. Redémarrez Content Server et le serveur d'application Web.

Pour plus d'informations sur cette procédure, consultez la section Gestion des paramètres de l'agent de workflow.
