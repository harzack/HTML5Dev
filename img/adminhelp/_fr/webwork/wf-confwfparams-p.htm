Pour spécifier les paramètres d'utilisateur du workflow :

  1. Dans la section Administration des workflows de la page Administration Content Server, cliquez sur le lien Configurer les paramètres du workflow.

  2. Dans la zone Historique d'expertise, sélectionnez la case à cocher Ne pas inclure de noms d'utilisateurs pour exclure les noms d'utilisateurs des journaux d'audit des workflows. La valeur <Aucun> s'affiche dans la colonne Utilisateur de la page Audit. Désactivez cette case à cocher pour inclure les noms d'utilisateurs dans les journaux d'audit des workflows.

  3. Dans la zone Mandataires, sélectionnez la case à cocher Autoriser les mandataires d'utilisateur pour donner la possibilité aux utilisateurs d'accorder l'autorisation à un autre utilisateur de terminer leurs étapes du workflow en leur absence. Désactivez la case à cocher pour que les étapes du workflow soient obligatoirement effectuées par les utilisateurs.

  4. Dans la zone Statuts du workflow, cliquez sur l'un des éléments suivants dans la liste Affichage pour déterminer les workflows et les instances des processus lancés qui apparaissent par défaut lorsqu'un utilisateur accède aux pages Mes workflows ou Statut du workflow :

    • Tous : permet d'afficher tous les workflows et les instances de processus lancés avec l'autorisation Voir.

    • Lancé : permet d'afficher uniquement les workflows et les instances de processus lancés par l'utilisateur.

    • Géré : permet d'afficher uniquement les workflows et les instances de processus gérés par l'utilisateur.

    • Workflow initialisé : permet d'afficher uniquement les workflows Content Server lancés par l'utilisateur.

    • Workflow géré : permet d'afficher uniquement les workflows Content Server gérés par l'utilisateur.

    • EPS initialisé : permet d'afficher uniquement les instances EPS lancées par l'utilisateur.

    • EPS géré : permet d'afficher uniquement les instances EPS gérées par l'utilisateur.

  5. Dans la zone Statuts du workflow - Afficher, cliquez sur le bouton radio Non archivé ou Archivé pour déterminer le type de workflow qui apparaît par défaut lorsque les utilisateurs accèdent aux pages Mes workflows ou Statut du workflow.

  6. Dans la zone Statuts du workflow - Afficher, cliquez sur l'icône Modifier l'expression, , pour construire des arguments complexes et pour restreindre davantage les workflows qui apparaissent. Par exemple, ("Status = OK" OR "Status = Completed") AND "Due Date > 07/05/2013". Pour en savoir plus, consultez le document Gestion des paramètres d'utilisateur du workflow.

  7. Dans la zone Statuts du workflow - Ordre de tri, cliquez sur la méthode par défaut que vous voulez que Content Server utilise pour trier les workflows sur les pages Mes workflows ou Statut du workflow. Les options correspondent aux colonnes des pages.

  8. Dans la zone Nombre maximum d'éléments par page, saisissez le nombre de workflows et de tâches assignées par défaut qui s'affichent sur les pages Mes workflows ou Statut du workflow.

  9. Dans la zone Nombre maximum d'éléments par page, sélectionnez la case à cocher Permettre aux utilisateurs de changer les paramètres individuels pour permettre aux utilisateurs individuels de modifier le paramètre Nombre maximum d'éléments par page de la page Mes paramètres Workflow. Désactivez la case à cocher pour appliquer à tous les utilisateurs la limite définie dans le champ Nombre maximum d'éléments par page.

  10. Dans la zone Définir le destinataire pour les e-mails d'erreur de workflow, cliquez sur l'un des boutons suivants :

    • Tous les gestionnaires, qui envoie une notification à tous les gestionnaires dès qu'une erreur de workflow se produit.

    • Gestionnaire principal, qui envoie une notification à l'exécutant et au propriétaire de l'instance de workflow.

  11. Cliquez sur Enregistrer les modifications.

  12. Redémarrez Content Server et le serveur d'application Web.

 

Note :
Si des utilisateurs ont modifié les valeurs par défaut des champs Relations de workflow, Ordre de classement et Workflows maximum par page, les valeurs par défaut que vous définissez ne remplacent pas leurs paramètres.

 

Pour plus d'informations sur cette procédure, consultez la section Gestion des paramètres d'utilisateur du workflow.
