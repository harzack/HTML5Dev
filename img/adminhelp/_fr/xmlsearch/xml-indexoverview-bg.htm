Lorsqu'un document XML est ajouté à Content Server, les régions XML sont automatiquement indexées. Pour que les utilisateurs puissent effectuer des recherches dans les régions XML, vous devez ajouter un fichier DTD (Document Type Definition) XML et définir ses régions comme disponibles pour la recherche. Les fichiers DTD XML définissent les éléments et les attributs utilisés dans les documents XML. En rendant disponibles pour la recherche les régions du fichier DTD XML, vous permettez aux utilisateurs Content Server de créer des requêtes visant le contenu de régions XML spécifiques. Si vous ne rendez pas une région XML disponible pour la recherche, les utilisateurs Content Server peuvent effectuer des recherches dans le contenu d'une région en créant des requêtes de texte intégral. Après avoir rendu une région XML disponible pour la recherche, vous créez un modèle de recherche qui affiche par défaut la section Types XML sur la page Recherche.

 

Note :
Les utilisateurs Content Server ne peuvent pas utiliser la section Attributs système de la page Recherche pour rechercher les métadonnées associées à des documents générés par Gestion des processus de flux de données XML Activator. A la place, ils doivent utiliser la section Types XML de la page Recherche.

 

Ajout de fichiers DTD XML

Pour indexer les données XML, vous devez commencer par ajouter à Content Server des fichiers DTD XML ou des exemples de documents XML. Lorsque vous ajoutez un fichier DTD XML à Content Server, Content Server analyse les éléments et les attributs définis par la DTD XML et extrait les régions XML en fonction de ces éléments ou attributs.

Lorsqu'un document est ajouté à Content Server via la commande DTD XML du menu Ajouter un élément, Content Server analyse le contenu. Si Content Server reconnaît le document comme XML, celui-ci est analysé et balisé à nouveau.

Le texte suivant est un exemple de document XML :


<fish>

  <fish_OTATTR>

    <length>18</length>

    <weight>5</weight>

  </fish_OTATTR>

  <fish_OTNODE>I caught a really</fish_OTNODE>

  <commentary>

    <commentary_OTATTR>

      <type>lie</type>

    </commentary_OTATTR>

    <commentary_OTNODE>big</commentary_OTNODE>

  </commentary>

  <fish_OTNODE>one!</fish_OTNODE>

</fish>

Vous pouvez également extraire les régions depuis les exemples de documents XML qui définissent correctement tous les attributs et éléments XML que vous voulez rechercher. Que Content Server extraie les régions XML en fonction des fichiers DTD XML ou des exemples de documents XML, vous devez vous assurer que les documents XML contiennent du code XML correctement formaté et qu'ils sont stockés dans Content Server sous forme de fichiers DTD XML. Si le code des documents XML n'est pas correctement formaté, Content Server extrait le plus de régions possible et affiche un message d'avertissement vous conseillant de corriger le contenu XML. Content Server essaye d'extraire les régions depuis tous les documents que vous ajoutez comme fichier DTD XML. Vous pouvez ajouter des fichiers DTD XML et des exemples de documents XML à n'importe quel emplacement dans Content Server. Cependant, OpenText vous recommande de les stocker dans le volume DTD XML, auquel vous accédez depuis la page Administration.

Rendre des régions XML disponibles pour la recherche

Lorsque vous rendez les régions XML disponibles pour la recherche, vous pouvez effectuer votre sélection dans toutes les régions XML extraites par Content Server à partir d'une DTD XML donnée ou un exemple de document XML. Les régions XML que vous rendez disponibles pour la recherche correspondent aux régions XML dans lesquelles les utilisateurs peuvent effectuer des recherches dans Content Server. Chaque région possède un nom de région et un nom d'affichage. Le nom d'affichage correspond au nom que Content Server affiche sur la page Recherche lorsque les utilisateurs Content Server effectuent une recherche sur des régions XML. Définissez des noms d'affichage descriptifs de façon à ce que les utilisateurs Content Server puissent facilement identifier les régions dans lesquelles effectuer leurs recherches.

Content Server extrait quatre types de régions XML, basés sur les composants de la plupart des documents XML correctement formatés.

  • Régions Elément : représentent un élément XML, ses attributs et son contenu. Ces régions englobent les informations contenues dans les régions Texte et les régions Attribut.

  • Régions Texte : représentent le contenu de données de type caractère (texte brut) d'un élément XML. _OTNODE est ajouté à la fin du nom de ces régions.

  • Régions Attribut : noms et valeurs de l'attribut d'un élément XML. _OTATTR est ajouté à la fin du nom de ces régions. Ces régions englobent les régions Nom de l'attribut.

  • Régions Nom de l'attribut : représentent des noms et valeurs d'attributs spécifiques. Le nom de l'attribut permet de construire une région et la valeur de l'attribut représente le contenu correspondant de cette région.
