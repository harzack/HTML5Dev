Pour définir un flux de données producteur XML Activator :

  1. Sur la page Volume des objets système, cliquez sur Source de données de producteur XML Activator dans le menu Ajouter un élément.

  2. Si vous avez précédemment supprimé une source de données Content Server et que vous voulez associer ses requêtes et formulaires de recherche enregistrés à cette source de données Producteur XML Activator, choisissez le nom de la vue de recherche de la source de données supprimée dans la liste déroulante Remplacement de vue de recherche.

  3. Sur la page Créer une nouvelle source de données de producteur XML Activator, saisissez un identificateur unique pour tous les objets associés à cet index dans le champ Préfixe du processus. Cet identificateur correspond au nom d'affichage des objets associés à cet index sur la page Administration système et au nom d'affichage de la vue de recherche de l'index dans la liste Vues de recherche sur la page Recherche.

  4. Dans le champ Port, saisissez une valeur pour les numéros de port récepteur des processus associés au flux de données. Le numéro de port que vous spécifiez et au moins les onze numéros de port suivants ne doivent pas être utilisés par une autre source de données de votre système. Le nombre de numéros de port consécutifs utilisés dépend du nombre de partitions que vous indiquez dans le champ Partitions. La configuration d'un flux de données producteur XML Activator nécessite huit numéros de port et quatre numéros de port supplémentaire par partition. Les valeurs valables vont de 1025 à 65535.

  5. Dans la liste déroulante Hôte, sélectionnez le raccourci du serveur d'administration dont l'hôte sera utilisé pour exécuter le processus producteur XML Activator.

  6. Dans le champ Répertoire entrant, saisissez le chemin absolu du répertoire dans lequel vous voulez que le processus producteur XML Activator lise les fichiers XML générés par l'application tierce.

  7. Dans le champ Répertoire d'écriture, saisissez le chemin absolu du répertoire dans lequel vous voulez que le processus producteur XML Activator écrive les données qu'il extrait. Par défaut, les valeurs sont automatiquement saisies dans les autres champs de la page Créer un nouveau flux de données de producteur XML Activator ; cependant, vous pouvez modifier ces valeurs afin de personnaliser les répertoires de lecture, d'écriture et d'indexation.

     

    Note :
    Si vous modifiez la valeur que vous aviez auparavant saisie dans le champ Répertoire d'écriture, vous devez mettre à jour de façon manuelle la valeur des autres répertoires.

     

  8. Dans le champ Balise d'opération, saisissez le nom de la balise XML contenant l'opération que XML Activator effectuera sur le contenu des fichiers XML. Cette balise est définie par défaut sur Opération. Content Server ignorera tous les espaces que vous incluez dans les noms de balise.

  9. Dans le champ Balise d'identificateur, saisissez le nom de la balise XML contenant l'identificateur unique du contenu des fichiers XML. Par défaut, le nom défini pour cette balise OTURN. Content Server ignorera tous les espaces que vous incluez dans les noms de balise.

  10. Dans le champ Liste des métadonnées, mettez en correspondance le nom des balises XML avec le nom des balises de métadonnées afin de garantir la compatibilité avec les applications tierces qui écrivent les éléments de données que vous voulez indexer comme métadonnées. La mise en correspondance des balises doit inclure le nom de la balise XML suivi du nom de la balise de métadonnées, qui est le nom d'une Définition des régions d'index dans Content Server. Dans chaque mappage, les noms de balise doivent être séparés par une virgule et un espace. De plus, les mappages sont séparés par des points-virgules. Par exemple :

    xName, OTName;xOwnerIDNumber, OTOwnerID;xFileLocation, OTLocation

    Content Server ignorera tous les espaces que vous incluez dans les noms de balise. Si plusieurs types de balises apparaissent dans votre fichier XML, ou si plus de deux balises sont mappées dans votre fichier XML avec la même balise des métadonnées, Content Server indexe le contenu des balises à plusieurs types de la même région. Dans ce cas, les utilisateurs Content Server ne peuvent pas rechercher cet index dans cette région ; cependant, si les utilisateurs Content Server font une recherche sur l'index entier, il leur sera possible d'accéder à ces données. Si vous mettez en correspondance deux balises XML différentes avec la même balise de métadonnées ou si vous incluez plusieurs instances d'une balise dans vos fichiers XML, les utilisateurs Content Server ne pourront pas rechercher cet index dans la région.
