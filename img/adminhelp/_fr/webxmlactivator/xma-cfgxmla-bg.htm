Vous pouvez effectuer la plupart des tâches administratives liées à un processus XML Activator dans l'onglet Infos spécifiques de la page Propriétés. L'onglet Infos spécifiques de la page Propriétés d'un processus XML Activator contient divers champs qui sont décrits dans le tableau suivant :

Onglet Infos spécifiques de la page Propriétés XML Activator

Champ Description
Etat Spécifie si le processus XML Activator est en cours d'exécution ou non, et donne de brèves informations si le processus n'est pas en cours d'exécution. Vous ne pouvez pas modifier ce champ.
Serveur d'administration Spécifie le raccourci du serveur d'administration sur l'ordinateur hôte duquel s'exécute le processus XML Activator. Si vous modifiez cette valeur, vous devrez probablement effectuer aussi l'une des opérations suivantes :
•

Modifier le chemin du répertoire d'écriture.

•

déplacer le répertoire d'écriture. Si vous le déplacez, vous devez aussi modifier le répertoire de lecture du processus Conversion de documents du flux de données.

Ne modifiez pas l'hôte du serveur d'administration, à moins d'y être absolument obligé (par exemple, si l'hôte du serveur d'administration n'est plus disponible).
Activer la gestion du système Détermine si Content Server contrôle ce processus pour détecter le retour de messages d'erreur. S'il retourne un message d'erreur, Content Server l'enregistre dans la base de données. La gestion du système est activée par défaut. Pour plus d'informations sur la configuration de Content Server en vue de l'envoi d'e-mails d'alerte lorsqu'un processus de flux de données rencontre des erreurs, reportez-vous à la rubrique Pour activer la recherche d'erreurs et l'envoi d'e-mails.
Modèle de commande Spécifie le modèle utilisé pour générer la ligne de commande du processus XML Activator. Ne modifiez pas cette valeur, sauf si le service d'assistance clientèle d'OpenText vous le demande.
Ligne de commande Spécifie la ligne de commande utilisée par Content Server pour exécuter le processus XML Activator. Vous ne pouvez pas modifier la valeur de ce champ.
Répertoire de démarrage Spécifie le répertoire dans lequel le processus XML Activator s'exécute. Le processus XML Activator doit toujours être exécuté dans le répertoire Content Server_home/config. Ne modifiez pas cette valeur, sauf si le service d'assistance clientèle d'OpenText vous le demande.
IPools Vous permet d'accéder aux informations concernant les répertoires de lecture et d'écriture du processus XML Activator. Le répertoire de lecture est celui à partir duquel le processus XML Activator lit les données. Le répertoire d'écriture est celui dans lequel le processus XML Activator écrit les données. Ne modifiez pas les répertoires de lecture et d'écriture sauf si vous avez déplacé l'IPool ou si le service d'assistance clientèle d'OpenText vous le demande.
Fichier journal Spécifie le chemin du fichier journal. Si le champ est vide, la fonction de journalisation est désactivée.
Niveau de consignation Spécifie le niveau de consignation (Valeur par défaut, Détaillé ou Déboguer) appliqué si la journalisation est activée.
•

Par défaut pour enregistrer uniquement les erreurs dans le journal. Il s'agit du niveau de consignation minimum.

•

Détaillé pour enregistrer le statut des messages acheminés à travers les processus et les IPools du flux de données dans le journal. Les erreurs sont également enregistrées.

•

Déboguer pour enregistrer des informations détaillées au sujet du statut du flux de données et des erreurs dans le journal. Compte tenu de l'importante quantité de données générées, OpenText recommande de choisir ce niveau uniquement si vous devez déboguer des problèmes graves sur le flux de données.

Répertoire sortant Spécifie le répertoire dans lequel le processus XML Activator écrit les données au format de fichier XML.
Programme Indique le moment où le processus XML Activator s'exécute. Choisissez :
•

Manuel, l'option par défaut pour lancer manuellement le processus XML Activator.

•

Programmé, pour programmer le jour et l'heure d'exécution du processus XML Activator. Si vous prévoyez des mises à jour fréquentes du répertoire entrant, vous pouvez programmer le jour et l'heure au cours desquels le répertoire sera régulièrement reparcouru par le processus XML Activator. Par exemple, vous pouvez programmer l'exécution de XML Activator toutes les minutes. OpenText vous recommande de programmer l'exécution de XML Activator.

•

Intervalle, pour programmer le processus XML Activator afin qu'il s'exécute à un intervalle spécifique. Cliquez sur les valeurs dans les listes déroulantes pour spécifier les unités de temps et la durée de l'intervalle.

Resynchroniser Resynchronise les informations relatives à ces processus (dans le fichier otadmin.cfg du serveur d'administration du processus) avec les informations de la base de données Content Server.
Mettre à jour Cet élément soumet au serveur les modifications effectuées sur cette page.
Réinitialiser Rétablit les informations de cette page telles qu'elles étaient à l'ouverture de la page.

Pour en savoir plus, voir Configuration requise des fichiers XML Activator.
