Le tableau suivant décrit les messages d'erreur des processus XML Activator.

Si ces erreurs persistent, contactez le service d'assistance clientèle d'OpenText pour obtenir plus d'informations.

Tableau 1-25 : Messages d'erreur des processus XML Activator

Numéro Message Description Solution
1 Erreur Content Server principale Une erreur interne s'est produite lorsque Content Server a tenté de créer ou de définir des paramètres dans le fichier de configuration pour un processus XML Activator. Contactez l'assistance clientèle d'OpenText.
2 Impossible de lire le paramètre de configuration Une erreur Content Server s'est produite lors de la lecture d'un paramètre dans le fichier de configuration pour le processus XML Activator. Vérifiez le fichier de configuration pour vous assurer que tous les paramètres présentent des valeurs valables. Les fichiers de configuration XML Activator sont stockés dans le répertoire de configuration Content Server_home/config et portent l'extension .xml. Le nom d'un fichier de configuration XML Activator particulier est spécifié sur la ligne de commande pour le processus. Vous pouvez voir la ligne de commande du processus dans l'onglet Infos spécifiques de la page Propriétés.
3 Impossible de définir le paramètre de configuration Une erreur Content Server s'est produite lors de la configuration d'un paramètre dans le fichier de configuration pour le processus XML Activator. Vérifiez les valeurs que vous avez spécifiées lors de la création ou de la mise à jour du processus XML Activator. Si vous mettez à jour un processus XML Activator, vous pouvez vérifier les valeurs spécifiées dans l'onglet Infos spécifiques de la page Propriétés.
4 Impossible de se connecter au serveur d'administration. La connexion de Content Server au serveur d'administration qui gère le processus XML Activator a échoué. Vérifiez le serveur d'administration associé au processus XML Activator et assurez-vous qu'il est disponible. Si vous mettez à jour un processus XML Activator existant, vous pouvez changer le serveur d'administration en cliquant sur une valeur dans la liste déroulante Serveur d'administration de l'onglet Infos spécifiques de la page Propriétés XML Activator.
5 Impossible d'initialiser le fichier de configuration Content Server n'a pas pu initialiser le fichier de configuration pour le processus XML Activator. Vérifiez que les ressources mémoires disponibles de votre système sont adéquates.
6 Fichier de configuration introuvable Content Server n'a pas pu trouver le fichier de configuration pour le processus XML Activator. Vérifiez le nom du fichier de configuration pour vous assurer qu'il correspond au nom de fichier de configuration spécifié dans la ligne de commande pour le processus XML Activator. Assurez-vous que le fichier de configuration n'est pas en cours d'utilisation et vérifiez les autorisations du fichier pour garantir que le processus XML Activator peut lire les données qu'il contient.
Les fichiers de configuration XML Activator sont stockés dans le répertoire de configuration Content Server_home/config et portent l'extension .xml. Le nom d'un fichier de configuration XML Activator particulier est spécifié sur la ligne de commande pour le processus. Vous pouvez voir la ligne de commande du processus dans l'onglet Infos spécifiques de la page Propriétés.
7 Impossible d'écrire des données dans le fichier de configuration Content Server n'a pas pu écrire de données dans le fichier de configuration pour le processus XML Activator. Vérifiez que les autorisations du fichier de configuration n'empêchent pas le processus XML Activator d'écrire des données dans celui-ci. Vous pouvez également contrôler votre système pour vous assurer qu'il dispose de suffisamment d'espace disque disponible et que le fichier de configuration n'est pas verrouillé.
Les fichiers de configuration XML Activator sont stockés dans le répertoire de configuration Content Server_home/config et portent l'extension .xml. Le nom d'un fichier de configuration XML Activator particulier est spécifié sur la ligne de commande pour le processus. Vous pouvez voir la ligne de commande du processus dans l'onglet Infos spécifiques de la page Propriétés.

30 Impossible de définir le répertoire de lecture Content Server n'a pas pu définir le répertoire de lecture pour le processus XML Activator. Le répertoire de lecture est celui dans lequel l'IPool de lecture du processus est créé. Assurez-vous d'avoir spécifié un chemin de répertoire valable dans le champ Lire les données à partir de lors de la création du processus XML Activator. Si vous mettez à jour un processus XML Activator existant, vous pouvez changer le répertoire de lecture en définissant une nouvelle valeur dans le champ Lire, auquel vous accédez en cliquant sur le bouton Informations de l'onglet Infos spécifiques de la page Propriétés. Vérifiez également les autorisations du répertoire pour vous assurer que le processus XML Activator peut y écrire des données.
31 Impossible de définir la zone de lecture Content Server n'a pas pu définir la zone de lecture pour le processus XML Activator. La zone de lecture est un répertoire du répertoire de lecture IPool. Dans l'onglet Infos spécifiques de la page Propriétés XML Activator, la zone de lecture apparaît à la suite du champ Lire, auquel vous accédez en cliquant sur le bouton Informations de l'onglet Infos spécifiques de la page Propriétés XML Activator. Le champ Lire donne l'emplacement du répertoire source iPool. Le nom de la zone de lecture sera un nombre (par exemple, /2645_0). Vérifiez les autorisations de ce répertoire pour vous assurer que le processus XML Activator peut y écrire des données.

37 Impossible de définir la liste des métadonnées Content Server n'a pas pu définir la liste des métadonnées pour le processus XML Activator Assurez-vous d'avoir spécifié une liste des métadonnées valable dans le champ Liste des métadonnées lors de la création du processus XML Activator. Si vous mettez à jour un processus XML Activator existant, vous pouvez changer la liste des métadonnées en définissant une nouvelle valeur dans le champ Liste des métadonnées de l'onglet Infos spécifiques de la page Propriétés XML Activator.
La liste des métadonnées doit contenir le nom de la balise XML suivi du nom de la balise des métadonnées, qui est le nom d'une région existante dans Content Server. Dans chaque mappage, les noms de balise doivent être séparés par une virgule et un espace. De plus, les mappages sont séparés par des points-virgules. Par exemple :
xName, OTName;xOwnerIDNumber, OTOwnerID;xFileLocation, OTLocation
Content Server ignorera tous les espaces que vous incluez dans les noms de balise. Si plusieurs types de balises apparaissent dans votre fichier XML, ou si plus de deux balises sont mappées dans votre fichier XML avec la même balise des métadonnées, Content Server indexe le contenu des balises à plusieurs types de la même région. Dans ce cas, les utilisateurs ne peuvent pas rechercher cet index dans cette région ; cependant, si les utilisateurs Content Server font une recherche sur l'index entier, il leur sera possible d'accéder à ces données.
