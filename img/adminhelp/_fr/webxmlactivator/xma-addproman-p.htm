Pour ajouter un processus Producteur XML Activator à un flux de données :

  1. Sur la page Volume des objets système, cliquez sur le lien préfixe_processus Dossier le la source de données de l'index auquel vous voulez ajouter le processus.

  2. Cliquez sur le lien Flux de données du flux de données dans lequel vous voulez ajouter le processus.

  3. Cliquez sur XML Activator dans le menu Ajouter un élément.

  4. Sur la page Ajouter XML Activator, saisissez un nom pour le processus XML Activator dans le champ Nom.

  5. Pour afficher une description du processus XML Activator, saisissez-la dans le champ Description.

  6. Dans la liste déroulante Type d'activateur, cliquez sur Producteur XML Activator.

  7. Dans la liste déroulante Hôte, cliquez sur le raccourci du serveur d'administration dont l'hôte sera utilisé pour exécuter le processus producteur XML Activator.

  8. Saisissez le chemin absolu du fichier journal dans le champ Fichier journal. Saisissez par exemple Content Server_home\index\prefix\logs\nom.log, où nom correspond au nom vous permettant d'associer le fichier journal à ce processus XML Activator.

  9. Dans la liste déroulante Niveau de consignation, cliquez sur l'un des niveaux de consignation suivants :

    • Valeur par défaut : seules les erreurs apparaissent dans le journal. Il s'agit du niveau de consignation minimum.

    • Détaillé : le statut des messages acheminés à travers les processus et les IPools du flux de données apparaît dans le journal. Les erreurs figurent également dans le journal.

    • Déboguer : des informations détaillées au sujet du statut du flux de données et des erreurs apparaissent dans le journal. Compte tenu de l'importante quantité de données générées, OpenText recommande de choisir ce niveau uniquement si vous devez déboguer des problèmes graves sur le flux de données.

  10. Dans le champ Répertoire entrant, saisissez le chemin absolu du répertoire depuis lequel vous voulez que le producteur XML Activator lise les données qui ont été traitées par une application tierce.

  11. Dans le champ Balise d'opération, saisissez le nom de la balise XML contenant l'opération que XML Activator effectuera sur le contenu des fichiers XML. Cette balise est définie par défaut sur Opération. Content Server ignorera tous les espaces que vous incluez dans les noms de balise.

  12. Dans le champ Balise d'identificateur, saisissez le nom de la balise XML contenant l'identificateur unique du contenu des fichiers XML. Par défaut, le nom défini pour cette balise OTURN. Content Server ignorera tous les espaces que vous incluez dans les noms de balise.

  13. Dans le champ Liste des métadonnées, vous mettez en correspondance le nom des balises XML avec le nom des balises de métadonnées afin de garantir la compatibilité avec les applications tierces qui écrivent les éléments de données que vous voulez indexer comme métadonnées. La mise en correspondance des balises doit inclure le nom de la balise XML suivi du nom de la balise de métadonnées, qui est le nom d'une Définition des régions d'index dans Content Server. Dans chaque mappage, les noms de balise doivent être séparés par une virgule et un espace. De plus, les mappages sont séparés par des points-virgules. Par exemple :

    xName, OTName;xOwnerIDNumber, OTOwnerID;xFileLocation, OTLocation

    Content Server ignorera tous les espaces que vous incluez dans les noms de balise. Si plusieurs types de balises apparaissent dans votre fichier XML, ou si plus de deux balises sont mappées dans votre fichier XML avec la même balise des métadonnées, Content Server indexe le contenu des balises à plusieurs types de la même région. Dans ce cas, les utilisateurs Content Server ne peuvent pas rechercher cet index dans cette région ; cependant, si les utilisateurs Content Server font une recherche sur l'index entier, il leur sera possible d'accéder à ces données. Si vous mettez en correspondance deux balises XML différentes avec la même balise de métadonnées ou si vous incluez plusieurs instances d'une balise dans vos fichiers XML, les utilisateurs Content Server ne pourront pas rechercher cet index dans la région.

Pour plus d'informations sur cette procédure, reportez-vous à la rubrique Configuration finale d'un processus producteur XML Activator.
