Un processus producteur XML Activator lit les données (sous forme de fichiers XML) depuis le répertoire spécifié dans le système de fichiers dans lequel l'application tierce a écrit les informations. Le processus producteur XML Activator envoie ensuite les données vers un IPool pour le processus suivant du flux de données.

Vous pouvez utiliser ce processus pour indexer des informations d'applications tierces dans Content Server.

Ajout d'un processus producteur XML

Vous ajoutez un processus producteur XML Activator à un flux de données pour fournir une source aux données qui seront traitées par le flux de données. La première étape lors de l'ajout à un flux de données d'un processus producteur XML Activator consiste à le nommer et à spécifier ses paramètres de configuration.

Lorsque vous ajoutez un processus producteur XML Activator à un flux de données, vous pouvez spécifier le nom des balises d'opération et d'identificateur qui peuvent apparaître dans les Configurations requises des fichiers XML Activator transmises via le flux de données. La balise d'opération spécifie l'action effectuée par Content Server sur les données. La balise d'identificateur est une chaîne continuelle et unique qui identifie l'objet de données (contenu inclus dans un fichier XML spécifique).

Lorsque vous spécifiez des répertoires sur la page Ajouter : XML Activator, vous devez indiquer le chemin des répertoires tel qu'il est mappé/monté sur l'hôte du serveur d'administration dont vous avez sélectionné le raccourci dans la liste déroulante Hôte.

Configuration finale d'un processus producteur XML Activator

Après avoir nommé et spécifié les paramètres de configuration du processus producteur XML Activator, la configuration finale consiste à définir les paramètres de codage binaire, d'IPools et les options de démarrage et d'arrêt.

Les fichiers XML générés par votre application tierce et qui ajoutent ou remplacent les données de votre index comportent du contenu, en règle générale des données binaires ou du texte, identifié par la balise de contenu. La balise de contenu comprend également un attribut qui identifie le type de codage du contenu. Par défaut, les processus de producteur XML Activator sont définis pour reconnaître <Content Encoding="Base64"> comme valeur de la balise de contenu et son attribut dans les fichiers XML, où Content est le nom de la balise de contenu, Encoding est le nom de l'attribut de balise et Base64 est la valeur d'attribut de balise.

Vous changez ces paramètres lorsque vous configurez un processus XML Activator si votre application tierce utilise des noms différents pour les paramètres. Par exemple, si votre application tierce génère les fichiers XML avec une balise de contenu <a b="c">, vous définissez Balise de contenu sur a, Attribut de balise sur b et Valeur d'attribut de balise sur c.

La liste déroulante Codage vous permet d'indiquer le type de codage du contenu de vos fichiers XML.
