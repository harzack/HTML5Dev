Lorsqu'une étape de XML Workflow Interchange envoie un message à un système externe, celui-ci reçoit l'URL lui permettant d'accéder à l'étape. Un serveur HTTP peut fournir différentes adresses Internet et mappages de répertoires virtuels permettant l'accès à une installation Content Server unique. En configurant l'accès HTTP externe, vous indiquez aux systèmes externes le mappage qu'ils doivent utiliser.
