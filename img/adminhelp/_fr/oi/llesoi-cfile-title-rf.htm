Attention
Le caractère deux-points, « : », ne peut pas être utilisé dans un titre car les deux-points servent à délimiter les éléments de chemin d'accès.

 

Description

La balise title permet d'appliquer un titre à un objet. Content Server autorise l'utilisation de plusieurs balises title pour un même objet afin de créer plusieurs titres, dans plusieurs langues, pour cet objet.

Syntaxe

<title language="en">Moving Pictures</title>
<title clear="true" language="en"></title>

Utilisation

  1. La balise title est obligatoire dans le cas suivant :

    • <node type="<any_type>" action="create">

    La seule exception à cette règle concerne type="document". Consultez la section Notes ci-dessous pour plus d'informations.

  2. La balise title peut être obligatoire ou facultative, selon le paramétrage de la case à cocher « Ajouter un titre à l'emplacement » dans l'importateur d'objet, lorsque :

    • <node type="<any_type>" action="update">

    • <node type="<any_type>" action="delete">

  3. La section « Notes » ci-dessous fournit des informations complémentaires sur les conditions associées au paramètre « Ajouter un titre à l'emplacement » . Pour plus d'informations sur la configuration de la case à cocher « Ajouter un titre à l'emplacement », consultez la section Configurer l'importateur d'objet.

  4. La création, mise à jour ou suppression d'un titre peut nécessiter la balise Attribut language, décrite ci-dessous.

Notes

  1. Lors d'une opération action="create", la balise title est obligatoire pour tous les types d'objet sauf les documents. type="document. Si votre importation utilise <node type="document action="create" sans aucun titre spécifié, le nom du document sera utilisé. Par exemple, si la balise <title language="en"></title> n'est pas utilisée lors de l'importation du document <file>c:/temp/import_dir/readme.txt</file>, Content Server utilise « readme.txt » en tant que nom de l'objet et nom de fichier de la version.

  2. Lors d'une opération action="update" ou action="delete", cette balise est obligatoire et doit contenir le nom de l'objet, si la case Ajouter un titre à l'emplacement est cochée. Si la case Ajouter un titre à l'emplacement est décochée, cette balise peut, de façon facultative, être utilisée pour renommer un objet lors d'une mise à jour.

  3. Pour effacer la valeur de la balise title, utilisez l'attribut clear="true".

  4. Si un titre n'existe pas dans une langue activée dans le système vers lequel vous effectuez l'importation, l'attribut clear="true" sera utilisé pour supprimer le titre dans cette langue.

  5. Si deux balises title (ou plus) possèdent le même attribut de langue au sein du même paragraphe de nœud, aucune erreur ne sera générée. L'importateur d'objet définira le titre à l'aide de la valeur contenue dans la dernière balise <title> avec cet attribut de langue.

Exemple

Pour supprimer un document nommé « My Guide » dans le dossier « Documents » , avec un titre répertorié pour l'anglais et un titre répertorié pour le français :

<import>
  <node type="document" action="create">
    <location>Enterprise:Projects:Documents</location>
    <title language="en">My Guide</title>
    <title language="fr">Mon Guide</title>
    <file>c:\temp\my_guide.doc<\file>
  </node>
</import>

Pour consulter d'autres exemples de balises title, consultez la section Exemples de fichier de contrôle d'importation de titres.

Attribut language

Use the language attribute to specify a <language_code> which will create, update, or delete the title of that object associated with that language. La syntaxe de cette balise est la suivante : 

<title language="en">Moving Pictures</title>
<title language="fr">Images Mobiles</title>
<title clear="true" language="de"></title>

Utilisation Notes
1.

L'attribut language est facultatif, pourvu qu'une seule balise <title></title> apparaisse dans le paragraphe de nœud. Si vous appliquez plusieurs balises <title language="en"></title> au paragraphe de nœud, l'attribut language est obligatoire.

 

Astuce :
OpenText vous recommande de toujours utiliser l'attribut language pour la balise title et, s'il n'y a qu'une seule balise title, de spécifier le code de la langue par défaut du système pour l'attribut language .

Si l'attribut language n'est pas utilisé, l'importateur d'objet utilisera la langue par défaut du système.

 
2.

Afin de pouvoir créer ou mettre à jour des titres multilingues, toutes les langues référencées par l'attribut language doivent exister dans Content Server et être activées.
1.

Si <node type="<any_type>" action="create", des titres multilingues seront créés, à condition que la langue concernée existe et qu'elle est activée dans le système. Si <node action="update" ou <node action="sync", les titres multilingues seront mis à jour si le titre existe pour cet attribut language .

 

Attention
Sur la page Configurer l'importateur d'objet, l'une des options proposées est la case à cocher Ajouter un titre à l'emplacement. Si cette case est cochée, toutes les opérations de mise à jour, d'ajout de version et de suppression suivront le schéma suivant : le système prendra la première balise <title> du paragraphe de nœud du script d'importation, ajoutera le contenu de la balise <title> au contenu de la balise <location> et procédera à la mise à jour, l'ajout de version ou la suppression sur cet objet.

Dans le cas où plusieurs titres sont définis à l'intérieur du paragraphe de nœud (comme dans le cas des systèmes multilingues), l'importateur d'objet n'accédera qu'au premier titre pour déterminer l'objet à mettre jour, auquel ajouter une version ou à supprimer. Si le premier titre n'est pas défini dans la langue préférée de l'utilisateur pour les métadonnées, l'objet risque d'être introuvable étant donné que la balise <location> contient un composant dans la mauvaise langue.

Les utilisateurs doivent veiller à ce que le premier titre soit toujours défini dans la même langue que la balise <location> ; ils peuvent aussi choisir de décocher la case Ajouter un titre à l'emplacement sur la page Configurer l'importateur d'objet.
