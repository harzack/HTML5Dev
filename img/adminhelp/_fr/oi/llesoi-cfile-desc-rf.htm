Description

Utilisez la balise description pour fournir une description de l'objet. Content Server autorise l'utilisation de plusieurs balises description pour un même objet afin de créer plusieurs descriptions, dans plusieurs langues, pour cet objet.

Syntaxe

<description language="en">first draft</description>
<description clear="true" language="en"></description>

Utilisation

  1. La balise description n'est pas obligatoire.

  2. La balise description est facultative dans les cas suivants :

    • <node type="<any_type>" action="create">

    • <node type="<any_type>" action="update">

    • <node type="<any_type>" action="addversion">

    • <node type="<any_type>" action="sync">

  3. La création, mise à jour ou suppression d'une description peut nécessiter la balise Attribut language, décrite ci-dessous.

     

    Astuce :
    OpenText vous recommande de toujours utiliser l'attribut language pour la balise description et, s'il n'y a qu'une seule balise description, de spécifier le code de la langue par défaut du système dans l'attribut language .

    Si l'attribut language n'est pas utilisé, l'importateur d'objet utilisera la langue par défaut du système.

     

Notes

  1. Pour effacer la valeur de la balise description, utilisez l'attribut clear="true".

  2. Si une description n'existe pas dans une langue activée dans le système vers lequel vous effectuez l'importation, l'attribut clear="true" sera utilisé pour supprimer la description dans cette langue.

  3. Si deux balises description (ou plus) possèdent le même attribut de langue au sein du même paragraphe de nœud, aucune erreur ne sera générée. L'importateur d'objet définira la description à l'aide de la valeur contenue dans la dernière balise <description> qu'il aura rencontrée avec cet attribut de langue.

Exemple

Pour créer le document « guidelines.doc » avec des descriptions en anglais et en français, lorsque la case « Ajouter un titre à l'emplacement » est cochée :

<import>
  <node type="document" action="create">
    <location>Enterprise:Projects:Documents</location>
    <title language="en">guidelines.doc</title>
    <description language="en">Final Drafts</description>
    <description language="fr">Versions Finales</description>
  </node>
</import>

Pour consulter d'autres exemples de balises description, consultez la section Exemples de fichier de contrôle d'importation de descriptions.

Attribut language

Utilisez l'attribut language pour spécifier un <language_code> pour créer, mettre à jour ou supprimer la description des objets associés à cette langue. La syntaxe de cette balise est la suivante : 

<description language="en">Final Drafts</description>
<description language="fr">Versions Finales</description>
<description clear="true" language="it"></description>

Utilisation Notes
1.

L'attribut language n'est pas obligatoire, pourvu qu'une seule balise <description></description> apparaisse dans le paragraphe de nœud. Si vous appliquez plusieurs balises <description></description> au paragraphe de nœud, l'attribut language est obligatoire.

 

Astuce :
OpenText vous recommande de toujours utiliser l'attribut language pour la balise description .

 
2.

Afin de pouvoir créer ou mettre à jour des descriptions multilingues, toutes les langues référencées par l'attribut language doivent exister dans Content Server et être activées.
1.

Si <node type="<any_type>" action="create">, des descriptions multilingues seront créées, à condition que la langue existe et soit activée sur le système. Si <node action="update" ou <node action="sync", les descriptions multilingues seront mises à jour si la description existe pour cet attribut language .
