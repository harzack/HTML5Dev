Cette section fournit une liste des meilleures pratiques, techniques et conseils pour votre configuration de l'importateur d'objet.

Utilisation du thread d'agent

Lorsque des tâches d'importation fréquentes sont programmées, il peut s'avérer nécessaire de surveiller le thread d'agent pour s'assurer que toutes les activités programmées Content Server sur le site se font rapidement. Dans de rares cas, il peut être conseillé d'affecter un thread d'agent dédié pour programmer les tâches de l'importateur d'objet. Si cela s'avère nécessaire sur votre site, et pour obtenir de l'aide en vue de le configurer, contactez le service d'assistance clientèle OpenText.

Ancre

Si Object Importer est exécuté à partir de l'ancre, ou du serveur d'indexation/de notifications, vous devrez installer Object Importer sur tous les serveurs du cluster car il a un schéma associé.

Fichier de contrôle

Essayez de maintenir le volume à environ 1 000 documents par fichier de contrôle. Il est possible d'accroître le nombre de documents et de dépasser 1 000, mais cela peut avoir un effet négatif sur les performances puisque cela consomme davantage de ressources à la fois sur Content Server et sur le serveur de base de données. Il est possible d'accéder aux fichiers de contrôle et aux fichiers de téléchargement sur des ordinateurs autres que celui sur lequel est installé Content Server, à condition que Content Server puisse y accéder. Toutefois, cela peut ne pas être possible en cas d'incohérence entre les systèmes d'exploitation des ordinateurs, par exemple, si votre système comporte un mélange d'ordinateurs sous Windows et sous Solaris.

Durée de chargement

La durée de chargement peut être influencée par la quantité de métadonnées ajoutées aux documents, la taille moyenne des documents chargés et l'architecture de votre système.

En fonction de la quantité de métadonnées ajoutées aux documents et de la taille moyenne des documents chargés, la durée moyenne de chargement par document est, dans la plupart des cas, légèrement inférieure à une seconde par document.

Importations manuelles et importations programmées

Il existe une option permettant d'effectuer une importation manuelle, mais elle n'est prévue que pour de petites opérations de test par lot. Pour plus d'information sur l'exécution d'une importation manuelle, consultez la section Importer manuellement un document.

La méthode de prédilection pour les importations de plus grande taille consiste à activer leur programmation et mettre les fichiers de contrôle dans le répertoire de chargement désigné. Lorsqu'une valeur a été saisie dans le champ Répertoire de chargement, les importations doivent avoir lieu à partir de ce répertoire. Pour accéder au champ Répertoire de chargement, cliquez sur Configurer les paramètres du serveur dans la section Configuration du serveur des pages d'administration de Content Server.

Serveurs de l'importateur d'objet

L'ajout de serveurs parallèles réduira la durée générale d'importation nécessaire pour la tâche. En général, Object Importer n'impose pas une trop grande contrainte sur l'instance de Content Server. Object Importer peut s'exécuter sur dix serveurs en parallèle. Il convient donc de le contrôler de façon à s'assurer que les ressources matérielles adéquates sont disponibles, en particulier dans des domaines tels que l'espace disque pour les journaux de la base de données. Dans la plupart des cas, il ne devrait être nécessaire d'exécuter Object Importer que sur un seul serveur.

Préparation à l'exécution de Object Importer

  1. Veillez à ce que les utilisateurs et catégories référencés dans le fichier de contrôle de l'importation existent dans Content Server avant le début de l'importation.

  2. Assurez-vous de ne pas surcharger les répertoires de destination. Cela affectera en effet la navigation. Vous avez donc tout intérêt à maintenir un nombre d'objets de destination inférieur à 200.

  3. Veillez à configurer Object Importer comme indiqué dans Configuration de l'importateur d'objet. Configurer l'importateur d'objet inclut, notamment, de définir les répertoires de l'importateur d'objet et de suivre toute la création de la base de données en vue de créer la table de statut de l'importateur d'objet.

Agent de programmation

Activez un seul agent de programmation par serveur sur lequel l'importateur d'objet est exécuté. Veillez à choisir un agent de programmation différent à exécuter sur chaque serveur.

Programmation

Dans la mesure du possible, exécutez Object Importer pendant les heures creuses. Si vous traitez une tâche imposante, il peut être souhaitable de désactiver l'index pendant le chargement.

Répertoire de chargement

Pour accéder au champ Répertoire de chargement, cliquez sur Configurer les paramètres du serveur dans la section Configuration du serveur des pages d'administration de Content Server.

Le répertoire de chargement peut être défini ou désactivé. Lorsque le répertoire de chargement est désactivé, ce qui signifie que le champ Répertoire de chargement est vide, l'importateur d'objet recherche les documents comme indiqué entre les balises <file></file> du fichier de contrôle.

Exemple 2-1 : L'importateur d'objet recherche un document readme.txt dans le répertoire c:\files\:

<file>c:\files\readme.txt</file>

Lorsqu'une valeur est spécifiée dans le champ du répertoire de chargement, l'importateur d'objet recherche le document dans le répertoire indiqué. Par conséquent, le fichier de contrôle doit uniquement contenir le nom du fichier et pas le chemin entier.

Exemple 2-2 : L'importateur d'objet recherche un document readme.txt dans le répertoire indiqué dans la page de configuration :

<file>readme.txt</file>
