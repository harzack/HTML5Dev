taskgroup

Description

Utilisez type="taskgroup" sur la balise node pour créer, mettre à jour ou supprimer un groupe de tâches.

Syntaxe

<node type="taskgroup" action="create"></node>

Utilisation

  1. La création ou la mise à jour d'un groupe de tâches peut inclure une balise subordonnées, Balise milestone, décrite ci-dessous.

  2. La création, mise à jour ou suppression d'un groupe de tâches peut également nécessiter un certain nombre de balises subordonnées, telles que la balise location . Ces autres balises subordonnées sont détaillées dans Descriptions et syntaxe des balises Importateur d'objet.

Exemple

Pour créer un groupe de tâches et une étape clé, appelés respectivement « New Task Group » et « New Milestone », rattachés à la liste de tâches nommée « New Task List »dans l'espace de travail Enterprise :

<import>
  <node type="taskgroup" action="create">
    <location>Enterprise:New Task List</location>
    <title language="en">New Task Group</title>
    <milestone>Enterprise:New Task List:New Milestone</milestone>
  </node>
</import>

Balise milestone

Utilisez la balise milestone pour attribuer une étape clé à un groupe de tâches. La syntaxe de cette balise est la suivante : 

<milestone>Enterprise:New Task List:New Milestone</milestone>
<milestone clear="true"></milestone>

Utilisation Notes
1.

La balise milestone ne peut exister seule et doit être utilisée en association avec la balise <node type="taskgroup" .

2.

La balise milestone n'est pas obligatoire.

3.

La balise milestone est facultative dans les cas suivants :

  • <node type="taskgroup" action="create">

  • <node type="taskgroup" action="update">

  • <node type="taskgroup" action="sync">
1.

L'étape clé doit exister dans Content Server pour pouvoir être assignée à un groupe de tâches. Si cette balise n'est pas spécifiée lors de la création, aucune étape clé n'est assignée au groupe de tâches.

2.

Pour désassigner l'étape clé, utilisez l'attribut clear="true".
