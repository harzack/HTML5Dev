document

Description

Utilisez type="document" sur la balise <node> pour créer, mettre à jour ou supprimer un document.

Syntaxe

<node type="document" action="create"></node>

Utilisation

  1. La création, mise à jour ou suppression d'un document peut inclure un certain nombre de balises subordonnées : Balise file, Balise mime, Balise versioncontrol, Balise versionmajor, Balise versionminor et Balise versiontype. Ces balises subordonnées sont détaillées ci-dessous.

  2. La création, mise à jour ou suppression d'un document peut également nécessiter un certain nombre de balises subordonnées, telles que la balise location. Ces autres balises subordonnées sont détaillées dans Descriptions et syntaxe des balises Importateur d'objet.

Notes

  1. Livelink Enterprise Server 9.6 permet de définir des versions principales et secondaires pour un même document. Object Importer prend en charge cette fonctionnalité, avec versions, et l'ancienne fonctionnalité, sans version. Pour pouvoir utiliser la fonctionnalité sans version, aucune nouvelle opération n'est requise en ce qui concerne le fichier de contrôle d'importation.

  2. Pour pouvoir utiliser la fonctionnalité avec versions, vous devez ajouter la balise <versioncontrol> au fichier de contrôle d'importation lors de la création du document.

  3. Lorsque vous ajoutez une version à document existant placé sous contrôle de version avancée, utilisez la balise <versiontype>. Au lieu de la balise <versiontype> , vous pouvez utiliser la balise <versionmajor> ou <versionminor>. Des informations détaillées sur les balises <versiontype>, <versionmajor>et <versionminor> sont fournies ci-dessous.

  4. Si les balises <versiontype>, <versionmajor> et <versionminor> ne sont pas spécifiées dans le fichier de contrôle d'importation, le document sera ajouté en utilisant l'incrément de version mineure suivant. Si les balises <versiontype>, <versionmajor> et <versionminor> sont spécifiées dans le fichier de contrôle d'importation et que le document ne se trouve pas sous contrôle de version avancée, les balises sont ignorées.

     

    Important :
    L'utilisation des balises <versiontype>, <versionmajor> et <versionminor> dans le même fichier de contrôle d'importation génère une erreur.

     

Exemple

Exemple Pour créer une version mineure d'un document nommé « guidelines.doc » dans le dossier « Documents » de l'espace de travail Entreprise, dossier « Projets » :

<import>
  <node type="document" action="create">
    <location>Enterprise:Projects:Documents</location>
    <file>c:\temp\guidelines.doc</file>
    <versioncontrol>TRUE</versioncontrol>
    <versiontype>MINOR</versiontype>
  </node>
</import>

Pour consulter d'autres exemples de balises document, consultez la section Exemples de fichier de contrôle d'importation de documents.

Balise file

La balise file définit le nom du chemin d'accès absolu du fichier à importer dans Content Server. La syntaxe de cette balise est la suivante : 

<file>c:/documents/readme.txt</file>

Utilisation Notes
1.

La balise file ne peut exister seule et doit être contenue dans une balise <node type="document".

2.

La balise file est obligatoire si Balise mime est utilisé.

3.

La balise file est obligatoire dans les cas suivants :

  • <node type="document" action="create">

  • <node type="document" action="addversion">

4.

La balise file est facultative dans le cas suivant :

  • <node type="document" action="update">
Le chemin d'accès au fichier doit être visible par le serveur traitant le fichier de contrôle d'importation (lecteur local ou mappé). Les fichiers à importer dans Content Server ne doivent pas être placés n'importe où dans l'arborescence de l'instance de Content Server. Par exemple, il est judicieux de placer ces fichiers à l'emplacement suivant : C:/temp/.

Balise mime

La balise mime permet d'affecter le type mime du document. La syntaxe de cette balise est la suivante : 

<mime>image/jpeg</mime>

Utilisation Notes
1.

La balise mime ne peut exister seule. Elle doit être contenue dans une balise <node type="document" et être utilisée en association avec Balise file.

2.

La balise mime est facultative dans les cas suivants :

  • <node type="document" action="create">

  • <node type="document" action="update">

  • <node type="document" action="addversion">

  • <node type="document" action="sync">
Si cette balise n'est pas spécifiée, Content Server essaiera de déterminer le type mime à partir de l'extension du document, en s'appuyant sur le fichier de mappage <Content Server_home>\config\mime.types.

Balise versioncontrol

La balise versioncontrol permet de prendre en charge l'importation de versions de document majeures et mineures. La syntaxe de cette balise est la suivante : 

<versioncontrol>TRUE</versioncontrol>
<versiontype>MINOR</versiontype>

Utilisation Notes
1.

La balise versioncontrol ne peut exister seule et doit être contenue dans une balise <node type="document".

2.

La balise versioncontrol est obligatoire si Balise versiontype est utilisé.

3.

La balise versioncontrol est obligatoire dans le cas suivant :

  • <node type="document" action="addversion">(quand ni la balise versionmajor ni la balise versionminor ne sont utilisées).

4.

La balise versioncontrol est facultative dans les cas suivants :

  • <node type="document" action="create">

  • <node type="document" action="update">

  • <node type="document" action="sync">
1.

Lorsque la valeur est définie sur TRUE, la version importée dans Content Server correspondra à l'incrément de version mineure suivant.

2.

Si vous n'utilisez pas de versions majeures/mineures, n'incluez pas cette balise.

Balise versionmajor

La balise versionmajor peut être utilisée à la place de la balise <versiontype>MAJOR</versiontype> pour indiquer un numéro de version majeure spécifique. La syntaxe de cette balise est la suivante : 

<versionmajor>2</versionmajor>

Utilisation Notes
1.

La balise versionmajor ne peut exister seule et doit être contenue dans une balise <node type="document".

2.

La balise versionmajor est facultative dans les cas suivants :

  • <node type="document" action="addversion">(quand ni la balise versionminor ni la balise versiontype ne sont utilisées).

  • <node type="document" action="create">

  • <node type="document" action="update">

  • <node type="document" action="sync">
L'importateur d'objet vérifie que la version majeure que vous entrez est supérieure à la dernière version ; dans le cas contraire, l'importateur d'objet émet une erreur.

Balise versionminor

La balise versionminor peut être utilisée à la place de la balise <versiontype>MINOR</versiontype> pour indiquer un numéro de version mineure spécifique. La syntaxe de cette balise est la suivante : 

<versionminor>5</versionminor>

Utilisation Notes
1.

La balise versionminor ne peut exister seule et doit être contenue dans une balise <node type="document".

2.

La balise versionminor est facultative dans les cas suivants :

  • <node type="document" action="addversion">(quand ni la balise versionmajor ni la balise versiontype ne sont utilisées).

  • <node type="document" action="create">

  • <node type="document" action="update">

  • <node type="document" action="sync">
L'importateur d'objet vérifie que la version mineure que vous entrez est supérieure à la dernière version ; dans le cas contraire,l'importateur d'objet génère une erreur.

Balise versiontype

La balise versiontype permet de prendre en charge l'importation de versions de document majeures ou mineures. La syntaxe de cette balise est la suivante : 

<versioncontrol>FALSE</versioncontrol>
<versiontype>MAJOR</versiontype>
<versioncontrol>TRUE</versioncontrol>
<versiontype>MINOR</versiontype>

Utilisation Notes
1.

La balise versiontype ne peut exister seule et doit être contenue dans une balise <node type="document". Elle doit aussi être associée à Balise versioncontrol.

2.

La balise versiontype est facultative dans les cas suivants :

  • <node type="document" action="addversion">(quand ni la balise versionmajor ni la balise versionminor ne sont utilisées).

  • <node type="document" action="create">

  • <node type="document" action="update">

  • <node type="document" action="sync">
1.

Lorsque la valeur est définie sur MAJOR, la version importée dans Content Server correspondra à l'incrément de version majeure suivant.

2.

Lorsque la valeur est définie sur MINOR, la version importée dans Content Server correspondra à l'incrément de version mineure suivant.
