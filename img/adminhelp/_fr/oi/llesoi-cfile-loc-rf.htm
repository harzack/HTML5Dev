Description

 

Important :
Cette balise est obligatoire, quel que soit le type d'objet ou l'action d'importation concerné.

 

La balise location indique l'emplacement de l'objet. Le chemin d'accès doit être absolu, commencer en haut de la hiérarchie et les différents niveaux doivent être séparés par deux-points.

Syntaxe

<location>Enterprise:Projects</location>

Utilisation

Cette balise est obligatoire, quel que soit le type d'objet ou l'action d'importation concerné.

Notes

  1. Si action="create" est utilisé :

    La balise <location></location> représente le chemin d'accès dans Content Server où l'objet sera créé. Selon la manière dont l'importateur d'objet a été configuré, le chemin d'accès peut être créé s'il n'existe pas. La création de chemin ne s'applique qu'à l'action action="create".

  2. Si action="update" est utilisé :

    La balise <location></location> représente soit le chemin d'accès dans Content Server menant à l'objet à mettre à jour, soit le chemin d'accès dans Content Server menant à l'objet parent. Le chemin d'accès choisi dépend du paramètre Ajouter un titre à l'emplacement dans la section de configuration. Par exemple, pour mettre à jour un document nommé readme.txt situé dans le dossier « Projets » de l'espace de travail Entreprise :

    Si la case Ajouter un titre à l'emplacement est cochée, la syntaxe est la suivante : 

    <location>Enterprise:Projects</location>
<title language="en">readme.txt</title>

    Si la case Ajouter un titre à l'emplacement est décochée, la syntaxe est la suivante : 

    <location>Enterprise:Projects:readme.txt</location>

  3. Si action="sync" est utilisé :

    La balise <location></location> représente le chemin d'accès dans Content Server où l'objet sera créé ou mis à jour.

  4. Si action="addversion" est utilisé :

    La balise <location></location> représente le chemin d'accès dans Content Server où l'objet sera créé.

  5. Si action="delete" est utilisé :

    La balise <location></location> représente le chemin d'accès dans Content Server menant à l'objet à supprimer. Les règles de syntaxe s'appliquant au paramètre Ajouter un titre à l'emplacement , indiquées précédemment pour action="update"s'appliquent à action="delete" également.

Exemple

Pour créer un projet nommé « Standard Project » dans le dossier « Documentation » de l'espace de travail Entreprise lorsque la case Ajouter un titre à l'emplacement est cochée :

<import>
  <node type="project" action="create">
    <location>Enterprise:Documentation</location>
    <title language="en">Standard Project</title>
  </node>
</import>
