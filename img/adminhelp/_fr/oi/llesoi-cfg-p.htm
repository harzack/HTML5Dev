Pour configurer l'importateur d'objet :

  1. Créez un répertoire de fichier de contrôle, visible pour Content Server, où l'importateur d'objet cherchera les fichiers de contrôle d'importation.

  2. Créez un répertoire de travail, visible pour Content Server, où l'importateur d'objet traitera les fichiers de contrôle d'importation.

  3. Créez un répertoire de journaux où l'importateur d'objet copiera les fichiers journaux. Ce répertoire doit toujours se trouver soit dans : <Content Server_home>\logs soit dans <Content Server_home>\object_importer.

     

    Note :
    Si une valeur a été saisie dans le champ Répertoire de chargement de la page Configurer les paramètres du serveur, les importations doivent avoir lieu à partir de ce répertoire. Les valeurs spécifiées entre les balises <file></file> du fichier de contrôle d'importation doivent contenir uniquement le nom du fichier. Par exemple, <file>readme.txt</file>. Pour plus d'informations sur la page Configurer les paramètres du serveur, consultez la section Configuration des paramètres de base du serveur dans Configuration des paramètres de base du serveur.

     

  4. Dans la section Administration de l'importateur d'objet de la page d'administration de Content Server, cliquez sur Configurer.

  5. Sur la page Configurer l'importateur d'objet, dans la section Répertoire :

    1. Dans le champ Fichier de contrôle, indiquez le chemin complet du répertoire que vous avez créé à l'étape 1. C'est le répertoire dans lequel l'importateur d'objet recherche les fichiers d'importation.

    2. Dans le champ Travail, indiquez le chemin complet du répertoire que vous avez créé à l'étape 2. C'est le répertoire dans lequel l'importateur d'objet traite les fichiers d'importation.

    3. Dans le champ Journal, indiquez le chemin complet du répertoire que vous avez créé à l'étape 3. C'est le répertoire dans lequel l'importateur d'objet écrit les fichiers journaux et qui doit toujours se trouver soit dans <Content Server_home>\logs soit dans <Content Server_home>\object_importer.

  6. Dans la section Avancé :

    1. Dans le champ Adresse sur erreur, spécifiez une adresse e-mail pour la notification des erreurs critiques. Cela n'inclut pas les erreurs liées aux données importées car celles-ci sont enregistrées dans le fichier journal.

    2. Dans le champ Signaler tous les, spécifiez la fréquence de mise à jour de la base de données. Pour des raisons de performance, OpenText recommande que cette valeur soit définie en fonction de la taille du fichier de téléchargement. Si une valeur de zéro, (0), est spécifiée, la mise à jour n'aura pas lieu.

    3. Lors de la création d'objets dans Content Server à l'aide de l'importateur d'objet, vous devez spécifier le chemin de l'objet parent entre les balises <location></location> . Pour créer des dossiers manquants, et ajouter l'objet lorsque le chemin parent n'existe pas, activez la case à cocher Créer la structure du répertoire. Si la case n'est pas cochée, l'objet ne sera pas ajouté et une erreur sera enregistrée.

    4. Pour supprimer le fichier de contrôle du répertoire de travail une fois le traitement terminé, activez la case à cocher Supprimer les fichiers de contrôle. Si la case n'est pas cochée, le fichier de contrôle reste dans le répertoire de travail. Cette case ne s'applique qu'aux importations programmées.

    5. Pour arrêter le traitement lorsqu'une erreur se produit pendant le traitement d'un fichier de contrôle, activez la case à cocher Arrêter le traitement en cas d'erreur. Par défaut, l'importateur d'objet traitera le fichier de contrôle entier, en ignorant toute erreur rencontrée.

    6. La case Ajouter un titre à l'emplacement sert à maintenir la compatibilité entre différentes versions de l'importateur d'objet, de sorte que les fichiers de contrôle existants puissent toujours être utilisés. Pour les nouvelles installations, OpenText vous recommande de laisser cette option désactivée afin de pouvoir renommer les objets.

      Désactivez cette option pour les fichiers de contrôle créés pour, et utilisés dans, Object Importer Version 1.0.x, Object Importer Version 1.1.7 et Object Importer Version 1.1.7.1. Activez cette option pour ceux qui sont utilisés dans les versions de Object Importer postérieures à 1.1.7.1 et antérieures à la version 3.

       

      Attention
      Sachez qu'une fois que vous avez créé des fichiers de contrôle avec une méthode, vous ne devez pas utiliser l'autre méthode car cela pourrait entraîner des résultats indésirables.

       

    7. Pendant une importation programmée, lorsque l'agent est activé, une liste de fichiers figurant dans le répertoire de chargement à ce moment là est noté par Object Importer. Ces fichiers sont ensuite traités, en ignorant tout nouveau fichier qui pourrait être ajouté. Pour sonder le répertoire de chargement jusqu'à ce qu'il n'y ait plus de fichiers à traiter, activez la case à cocher Actualiser les fichiers de contrôle.

    8. Pour pouvoir être en mesure de modifier la « date de modification » d'un document pendant l'opération de création uniquement, activez la case à cocher Date de modification du document. Cette option ne s'applique pas à l'opération de mise à jour et, par conséquent, ne fonctionnera que sur les documents n'ayant qu'une seule version.

      Normalement, Content Server n'autorise pas ce comportement car l'ajout d'une version constitue une modification, ce qui définit automatiquement le champ Date de modification sur la date en cours. Définir le paramètre « date de modification » n'est pas une opération définitive car celle-ci sera réinitialisée sur la date en cours lorsque certaines actions seront réalisées dans Content Server. Ces actions incluent la modification du nom du document ou des autorisations sur le document.

    9. Pour définir la valeur de la balise <owner> sur « Admin » si la valeur est introuvable dans le système, au lieu de renvoyer une erreur, activez la case à cocher Propriétaire par défaut à administrer.

    10. Pour définir la valeur de la balise <createdby> sur « Admin » si la valeur est introuvable dans le système, au lieu de renvoyer une erreur, activez la case à cocher Créateur par défaut à administrer.

  7. Cliquez sur Mettre à jour.
