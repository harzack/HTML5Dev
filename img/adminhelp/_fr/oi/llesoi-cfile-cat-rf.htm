Balise category

Description

La balise category spécifie un nom de catégorie à appliquer à un objet.

Syntaxe

<category name="Content Server Categories:Test"></category>

Utilisation

  1. La balise category n'est pas obligatoire.

  2. La balise category est facultative dans les cas suivants :

    • <node type="<any_type>" action="create">

    • <node type="<any_type>" action="update">

    • <node type="<any_type>" action="sync">

  3. La création ou la mise à jour d'une catégorie peut inclure un certain nombre de balises subordonnées : Balise attribute, Balise noninherit, Balise setattribute et Balise subitems. Ces balises subordonnées sont détaillées ci-dessous.

Notes

  1. La catégorie, ainsi que tous les attributs référencés, doivent exister dans Content Server pour que la balise puisse être utilisée. Si ces éléments n'existent pas dans Content Server, la catégorie ne sera pas créée et un message d'erreur apparaîtra dans le fichier journal.

  2. Si la catégorie a été précédemment appliquée à l'objet, directement ou par héritage, seules les données d'attribut spécifiées seront mises à jour. Dans le cas contraire, la catégorie sera ajoutée et toutes les données d'attribut seront mises à jour.

    Si la catégorie a changé dans Content Server, celle appliquée à l'objet doit être « mise à jour » avant de modifier les données de catégorie à l'aide de Object Importer. Cette opération peut être réalisée dans l'onglet Catégories d'un objet donné via l'interface Content Server standard.

  3. Vous pouvez appliquer autant de catégories que vous le souhaitez aux objets à l'aide de Object Importer. En revanche, vous ne pouvez pas supprimer de catégorie d'un objet à l'aide de Object Importer.

Balise attribute

La balise attribute permet d'attribuer une valeur à un attribut existant. La syntaxe de cette balise est la suivante : 

<attribute name="Name">John Doe</attribute>
<attribute name ="xyz" clear="true"></attribute>

Utilisation Notes Exemple
1.

La balise attribute n'est pas obligatoire.

2.

La balise attribute ne peut exister seule et doit être contenue dans une balise category .

3.

Les balises attribute multilignes sont admises dans le fichier de contrôle d'importation.
1.

Le nombre et la syntaxe des valeurs d'un attribut dépendent des valeurs configurées et appliquées à l'attribut dans Content Server.

2.

Si l'attribut accepte des valeurs multiples, ces dernières peuvent être ajoutées en spécifiant des balises à attributs multiples dans le fichier de contrôle d'importation.

3.

Pour effacer la valeur d'un attribut, utilisez clear="true".

4.

Les dates doivent être au format AAAAMMJJ ou AAAAMMJJHHMMSS, où la valeur HH se rapporte au format 24 heures et doit être comprise entre 01 et 24. Si le format AAAMMJJ est utilisé, l'heure prend par défaut la valeur 000000.

5.

Lorsque vous définissez un type d'attribut pour une case à cocher, utilisez la syntaxe suivante : TRUE pour l'état activé et FALSE pour l'état désactivé.

6.

Toutes les valeurs doivent être spécifiées lors de la mise à jour d'un attribut à valeurs multiples (Multi-Value Attribute, MVA). Par exemple, si un attribut contient cinq valeurs avant la mise à jour et que le fichier de contrôle d'importation indique trois valeurs, le MVA ne contiendra que ces trois valeurs après la mise à jour.

 
<import>
  <node type="document" action="update">
    <category name="Content Server Categories:Test">
      <attribute name="Name">John Doe</attribute>
      <attribute name="DateOfBirth">19650216</attribute>
    </category>
  </node>
</import>
<import>
  <node type="document" action="update">
    <category name="Content Server Categories:Test">
      <attribute name="Color">Red</attribute>
      <attribute name="Color">Green</attribute>
      <attribute name="Color">Blue</attribute>
    </category>
  </node>
</import>

Balise noninherit

La balise noninherit détermine si l'héritage peut être activé ou désactivé pour les catégories. La syntaxe de cette balise est la suivante : 

<noninherit>FALSE</noninherit>

Utilisation Notes Exemple
1.

La balise noninherit n'est pas obligatoire.

2.

La balise noninherit ne peut exister seule et doit être contenue dans une balise category .
1.

La balise noninherit doit être définie sur TRUE ou FALSE.

Si la balise noninherit est définie sur TRUE, l'héritage peut être activé pour les catégories. Si la balise noninherit est définie sur FALSE, l'héritage peut être désactivé pour les catégories.

2.

Le comportement d'héritage par défaut découle de l'objet parent.

 
<import>
  <node type="document" action="update">
    <category name="Content Server Categories:Test">
      <attribute name="Name">Jane Doe</attribute>
      <noninherit>FALSE</noninherit>
    </category>
  </node>
</import>

Balise setattribute

En plus des attributs classiques, les attributs setattribute sont également pris en charge. La balise setattribute permet d'attribuer une valeur à un attribut setattribute existant. La syntaxe de cette balise est la suivante : 

<setattribute name="Fruit"></setattribute>

Utilisation Exemple
1.

La balise setattribute n'est pas obligatoire.

2.

La balise setattribute ne peut exister seule et doit être contenue dans une balise category .

3.

La balise setattribute peut contenir des attributs.

 
<import>
  <node type="document" action="update">
    <category name="Content Server Categories:SetTest">
      <setattribute name="Fruit">
        <attribute name="Name">Orange</attribute>
        <attribute name="Color">Orange</attribute>
      </setattribute>
    </category>
  </node>
</import>

Balise subitems

La balise subitems permet d'appliquer une catégorie et ses valeurs d'attribut de manière récursive à tous les objets enfants de l'objet actuel. La syntaxe de cette balise est la suivante : 

<subitems>reapply</subitems>

Utilisation Notes Exemple
1.

La balise subitems n'est pas obligatoire.

2.

La balise subitems ne peut exister seule et doit être contenue dans une balise category .
La seule valeur acceptée pour la balise subitems est reapply.
 
<import>
  <node type="document" action="update">
    <category name="Content Server Categories:Test">
      <attribute name="Name">Jane Doe</attribute>
      <subitems>reapply</subitems>
    </category>
  </node>
</import>
