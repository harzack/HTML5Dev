Description

Les fichiers de contrôle sont utilisés pour créer, mettre à jour, synchroniser et supprimer des objets. Chaque fichier de contrôle comprend un ou plusieurs paragraphes <node>, décrivant chacun des actions à appliquer à un objet spécifique.

Syntaxe

<import>
  <node type="folder" action="create">
    <location>Enterprise</location>
    <title language="en">Temp</title>
  </node>
</import>

Utilisation

  1. La balise node est obligatoire pour tous les fichiers de contrôle. Les deux attributs de la balise <node> , Attribut type et Attribut action, sont des attributs obligatoires pour chaque balise <node> .

  2. La balise <node> peut également contenir plusieurs balises subordonnées. Ces balises subordonnées sont détaillées dans Descriptions et syntaxe des balises Importateur d'objet.

Attribut action

Les actions suivantes sont autorisées pour l'importation d'objets et de données provenant du système de fichiers dans Content Server :

Action Syntaxe Description
Créer action="create"
Crée des objets, tels que des dossiers, dans Content Server ou importe des données, telles que des documents, dans Content Server à partir du système de fichiers.
Mettre à jour action="update"
Met à jour des métadonnées d'objet générales, des métadonnées d'objet spécifiques, des autorisations et des données de catégorie et d'attribut. Parmi les métadonnées d'objet générales, on peut citer le nom et la description. Parmi les métadonnées d'objet spécifiques, on peut citer le type mime, l'URL et le raccourci.
Synchroniser action="sync"
Effectue une création ou une mise à jour, selon que l'objet existe ou non dans Content Server.
Ajouter une version action="addversion"
Effectue une mise à jour en ajoutant une nouvelle version d'un objet Content Server existant.
Supprimer action="delete"
Supprime des objets dans Content Server.
