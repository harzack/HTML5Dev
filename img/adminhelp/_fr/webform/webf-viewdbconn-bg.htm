Etant donné que les consultations de la base de données Web Forms des versions précédentes présentent un risque d'attaque par injection SQL, les anciennes connexions à la base de données sont désactivées par défaut et peuvent uniquement être effectuées en ajoutant l'entrée suivante dans opentext.ini :
[HForm]
AllowUnsafeDatabaseLookups=true

Le lien Informations de connexion à la base de données issues des versions précédentes de la section Administration des consultations de bases de données Web Forms répertorie les connexions à la base de données Web Forms effectuées avec des versions précédentes du module Web Forms.

Le tableau suivant explique les informations qui s'affichent pour chaque connexion.

Colonne Contenu
Nom Nom de la base de données.
Type Type de la base de données
Nom de l'utilisateur Nom d'utilisateur du compte utilisé pour accéder à la base de données.

 

Note :
Les nouvelles connexions à la base de données peuvent uniquement être créées en utilisant le lien Gérer des connexions sécurisées à la base de données dans la section Administration des consultations de bases de données Web Forms.
