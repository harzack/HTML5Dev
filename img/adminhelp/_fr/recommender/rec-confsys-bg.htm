Le paramétrage système de l'Assistant de recommandation contrôle les opérations internes de la fonction, contrairement aux composants Assistant de recommandation qui contrôlent la présentation aux utilisateurs des données de l'Assistant de recommandation. Pour plus d'informations sur les composants de l'Assistant de recommandation, reportez-vous à la rubrique Configuration des composants de l'Assistant de recommandation.

Le paramétrage système de l'Assistant de recommandation permet de configurer l'agent de l'Assistant de recommandation, qui assure le suivi des habitudes de l'utilisateur, d'afficher des données et de les éliminer des tables de base de données de l'agent.

Exécution de l'agent de l'Assistant de recommandation

L'agent de l'Assistant de recommandation, lorsqu'il s'exécute, enregistre, dans la table de suivi, le comportement de l'utilisateur depuis le dernier enregistrement effectué. Il fait également une synthèse des données précédemment stockées dans la table de suivi et les place dans le tableau de synthèse. Le système utilise alors les données du tableau de synthèse pour créer les recommandations.

Par défaut, l'agent de l'Assistant de recommandation effectue un suivi du comportement de l'utilisateur Admin et des autres utilisateurs. Cependant, si vous ne voulez pas que le comportement de l'utilisateur Admin influence les recommandations, vous pouvez désactiver cette fonction.

 

Note :
Les modifications apportées au suivi de l'utilisateur Admin ne sont pas visibles tant que l'agent de l'Assistant de recommandation ne s'exécute pas.

 

L'agent de l'Assistant de recommandation s'exécute toutes les cinq minutes, en fonction de la quantité de données qu'il doit traiter.

Utilisation du tableau de synthèse

Le tableau de synthèse contient les données utilisées par le système pour générer les recommandations. Le tableau enregistre, pour un utilisateur et un élément donnés, le nombre de fois où une opération donnée a été effectuée. La colonne de tableau correspondant à cette valeur est celle du nombre de références.

Dans les grands tableaux, les données qui influent le plus sur les recommandations sont celles pour lesquelles le nombre de références est le plus élevé. Lorsque l'on visualise de grands tableaux, il est donc utile de filtrer les données par nombre de références.

Pour disposer en permanence de données pertinentes et actualisées, ces données doivent être éliminées à intervalles réguliers. Vous pouvez définir le système pour qu'il élimine automatiquement les données présentes dans le tableau de synthèse depuis un nombre de jours donné ou vous pouvez éliminer l'ensemble des données du tableau manuellement.

Exclusion ou inclusion des types de fichiers

La section [ExcludedMimeTypes] du fichier opentext.ini spécifie les types de fichiers extraits et indexés dans Content Server. La section [ExcludedMimeTypes] est une liste d'exclusion, ce qui signifie que vous pouvez inclure ou exclure un élément de la section Eléments les plus actifs de l'onglet des recommandations usuelles en modifiant la liste. Les éléments dont les types de fichiers sont inclus dans la liste n'apparaîtront pas dans la section Eléments les plus actifs. Les éléments dont les types de fichiers ne sont pas inclus dans la liste apparaîtront dans la section Eléments les plus actifs.
