Pour exporter les données de recommandation d'un utilisateur au format XML, ajoutez &outputxml=true à la fin de l'URL Content Server de la page ou saisissez l'URL suivante : <protocole>://<hôte>/<préfixe_URL> /Content Server.exe?func=Personal.Recommendations&outputxml=true

où :

  • <protocole> est soit http, soit https ;

  • <hôte> est l'adresse complète de l'hôte (par exemple, Content Serverhost.mycorp.com) et

  • <préfixe_URL> est le raccourci du répertoire virtuel mappé au répertoire <Content Server_accueil>/cgi dans le serveur HTTP.

Cette commande gère la sortie des recommandations de l'utilisateur actuellement connecté au serveur Admin.

Le DTD qui définit cette sortie XML est inclus dans les installations telles que <Content Server_home>/module/recommender_<x> _<x>_<x>/config/recommender_xml_output.dtd, où <x>_<x>_<x> est le numéro de version du module Recommender. Par exemple, pour Recommender 10.5.0, le fichier se trouve dans : <Content Server_home> /module/recommender_10_5_0/config/recommender_xml_output.dtd.

Sortie XML

La première partie du résultat en XML est un en-tête qui définit le fichier XML et le type de données que le fichier contient (dans le cas présent, des données de recommandation). Cette section est constituée de la déclaration XML suivante, qui spécifie la version XML, le codage de caractères utilisés dans ce fichier et l'élément PersonalRecommendations.

<?xml version="1.0" ?><PersonalRecommendations>

Le fichier XML contient également, pour chaque composant de l'Assistant de recommandation, une section affichant des informations sur la page des recommandations. Chaque section est définie par une balise.

Tableau 1-3 : Balises XML définissant les composants de l'Assistant de recommandation

Elément Composant
<whatsnew> Nouveautés
<mostactive> Eléments les plus actifs
<toppicks> Les mieux notés
<history> Eléments récemment consultés
<userslikeme> Personnes ayant des centres d'intérêt similaires
<docsinterest> Documents intéressants

Chaque balise contient un attribut DisplayName qui correspond au nom qui apparaît dans l'interface utilisateur graphique pour l'élément (par exemple, <whatsnew DisplayName="What's New">).

Les composants Nouveautés, Eléments les plus actifs, Les mieux notés, Documents intéressants et Eléments récemment consultés sont composés de tableaux d'éléments et sont donc définis par les mêmes balises dans la sortie XML. Le tableau suivant décrit les balises par ordre d'apparition.

 

Note :
Plusieurs balises XML du tableau possèdent le même nom (par exemple, <dc_link>). Ces répétitions sont cependant nécessaires dans la mesure où le tableau répertorie toutes les balises par ordre d'apparition.
