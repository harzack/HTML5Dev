Le contenu des documents indexés est stocké dans des régions. Il existe d'autres régions destinées au stockage des données concernant l'emplacement, le type MIME, la date de création, la taille, ainsi que d'autres informations portant sur les documents indexés. Ces régions non associées au contenu des documents sont parfois désignées sous le terme métadonnées. Dans le cas de l'index Entreprise, les régions de métadonnées correspondent aux attributs des éléments de la base de données Content Server, y compris tous les attributs personnalisés que vous avez éventuellement définis.

Pour un type d'élément particulier (document, dossier, tâche, etc.), le contenu et les métadonnées sont indexés. Les régions décomposent le groupe de données global (contenu et/ou métadonnées) indexé en sous-composants distincts que vous pourrez par la suite rechercher et afficher individuellement. Par exemple, OTAssignedTo et OTDateCompleted sont deux régions de métadonnées d'un index Entreprise standard. Les recherches dans les régions permettent aux utilisateurs de restreindre leur recherche aux parties de l'index qui les intéressent plus particulièrement. La page Configurer les régions de n'importe quel index répertorie toutes les régions de métadonnées qui ont des valeurs dans cet index.

Après la création d'un index (qu'il s'agisse de l'index Entreprise, des index de l'aide, d'un index Directory Walker ou de tout autre type d'index), vous devez sélectionner les régions de métadonnées de l'index que vous souhaitez configurer pour la recherche dans la section Attributs système de la page Recherche Content Server et pour l'affichage sur la page Résultats de recherche. Vous pouvez également sélectionner les régions de métadonnées de l'index dans lesquelles vous souhaitez effectuer des recherches par défaut ou que vous souhaitez trier. Pour plus d'informations sur la configuration des régions d'index pour la recherche, l'affichage, la recherche par défaut ou le tri, voir Configuration des régions d'index.

Régions de workflow

Vous pouvez modifier manuellement votre fichier LLFieldDefinitions.txt pour qu'une région de workflow spécifique soit indexée et puisse être recherchée ou exclue. Pour en savoir plus, consultez le document Pour configurer les régions de workflow pouvant être extraites.

Dans une nouvelle installation Content Server, certaines régions de métadonnées de workflow sont marquées comme DROP ou REMOVE dans le fichier LLFieldDefinitions.txt et ne sont donc pas indexées pour améliorer les performances et l'efficacité du moteur de recherche. Il se peut que votre installation n'utilise pas de nouveau fichier LLFieldDefinitions.txt pour maintenir la compatibilité. Pour plus d'informations, voir la section « Mettre à jour les fichiers de configuration de la recherche » de Vérification du contenu et réindexation.
