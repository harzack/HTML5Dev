Pour configurer une recherche à distance :

  1. Sur la page Administration, dans la section Administration de recherche, cliquez sur le lien Ouvrir le volume des objets système.

  2. Sur la page Système Content Server, cliquez sur le lien Dossier de la source de données de l'entreprise.

  3. Sur la page Dossier de la source de données de l'entreprise, dans le menu Ajouter un élément, cliquez sur Content Server distant.

  4. Sur la page Content Server distant, saisissez un nom pour le processus de source de données dans le champ Nom.

  5. Facultatif Pour afficher une description de Content Server distant, dans l'onglet Général de sa page Propriétés, saisissez une description dans le champ Description.

  6. Dans la liste Hôte, cliquez sur le raccourci du serveur d'administration de l'hôte sur lequel vous voulez exécuter la recherche à distance.

  7. Facultatif Pour permettre à Content Server de contrôler le statut de ce processus, sélectionnez la case à cocher Activer la gestion du système.

  8. Facultatif Pour démarrer le processus de recherche à distance juste après sa création, sélectionnez Démarrer le processus de rechercher à distance. Le processus démarre par défaut, sauf si vous décochez la case.

  9. Facultatif Pour que le serveur d'administration client puisse communiquer avec le serveur d'administration distant, saisissez le numéro de port dans le champ Port Admin.

  10. Facultatif Pour autoriser l'exécution de requêtes de recherche dans le serveur distant, saisissez le numéro de port dans le champ Port de recherche.

  11. Dans la section Paramètres Content Server distants, dans le champ Nom du site, saisissez le nom à afficher sur la page Résultat de recherche lorsqu'une recherche est effectuée sur le serveur distant.

  12. Dans le champ URL, saisissez le chemin d'accès du serveur distant.

  13. Facultatif Dans le champ Paramètres facultatifs de l'API de recherche, vous pouvez saisir des paramètres à inclure dans la requête API de recherche ou cliquer sur le bouton Afficher les vues de recherche distantes... pour récupérer une liste des vues de recherche disponibles sur le serveur distant.

    Exemple de paramètres spécifiques à inclure : &slice=X, où X est le DataID de la vue de recherche ou &template=Y, où Y est le DataID du modèle de recherche.

     

    Note :
    Ce champ ne se limite pas aux paramètres de vues de recherche ou de modèles ; tous les paramètres d'API de recherche pour le site distant peuvent y figurer.

     

  14. Dans le champ Utilisateur Admin, saisissez le nom de connexion de l'administrateur du serveur distant.

  15. Dans le champ Mot de passe de l'administrateur, saisissez le mot de passe de l'administrateur du serveur distant.

  16. Facultatif Pour activer le contrôle d'accès des utilisateurs, procédez comme suit :

    • Sélectionnez Utilisateur Natif uniquement, pour que les recherches s'effectuent sur le serveur distant en tant qu'utilisateur initiateur à partir du système Content Server local.

    • Pour que les recherches sur le serveur distant s'effectuent en tant que simple utilisateur, sélectionnez Utilisateur Public nommé et saisissez un nom dans le champ Utilisateur Public.

    • Pour limiter les résultats de la recherche à l'utilisateur identifié dans le serveur local, mais permettre l'accès public si cet utilisateur n'a pas ouvert de session, sélectionnez Retour de l'utilisateur natif vers l'utilisateur Public nommé, puis saisissez un nom dans le champ Utilisateur Public.

  17. Cliquez sur Ajouter.
