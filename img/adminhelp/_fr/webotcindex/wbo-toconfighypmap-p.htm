Pour configurer un mappage de liens hypertextes :

  1. Consultez le champ Répertoires de l'onglet Infos spécifiques de la page Propriétés de Directory Walker pour identifier le chemin racine commun à tous les répertoires parcourus par le processus Directory Walker. Par exemple, si le champ Répertoires contient c:/dirA/dir1 et c:/dirA/dir2, votre chemin racine commun est c:/dirA.

     

    Note :
    Le chemin racine commun est spécifique à l'environnement du système indexé par le processus Directory Walker.

     

  2. Copiez le préfixe du chemin.

  3. Sur la page Administration système, cliquez sur le lien < préfixe_processus> Dossier de la source de données du dossier de la source de données correspondant à l'index Directory Walker à utiliser par le mappage de liens hypertextes.

  4. Cliquez sur l'icône Fonctions du gestionnaire de recherche, sélectionnez Propriétés, puis Infos spécifiques.

  5. Dans l'onglet Infos spécifiques de la page Propriétés du gestionnaire de recherche, collez le préfixe du chemin dans le champ Rechercher.

  6. Dans le champ Remplacer par, saisissez ce qui suit : http://<Nom_serveur>/<Nom_répertoire_virtuel>.

     

    Important :
    Open Text vous recommande vivement d'examiner l'accès possible au document au moyen de votre serveur Web et, si nécessaire, de restreindre correctement l'accès aux emplacements de dossiers sensibles.

    Lorsque vous créez un mappage de répertoire virtuel pour une source de données non Entreprise, le mappage du serveur Web à l'emplacement des dossiers n'hérite pas automatiquement des autorisations définies pour la vue de recherche. Il est donc possible que des utilisateurs puissent accéder, au moyen du serveur Web, à des documents qu'ils ne sont pas autorisés à voir au moyen de Content Server.

     

  7. Cliquez sur le bouton Mettre à jour.

 

Astuce :
Vous pouvez également effectuer une recherche dans Content Server pour localiser le chemin racine approprié.
