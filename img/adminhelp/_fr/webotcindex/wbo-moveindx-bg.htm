Pour déplacer l'index d'une partition, déplacez le répertoire d'indexation et modifiez les chemins d'accès spécifiés pour le moteur d'indexation et les moteurs de recherche associés à l'index de la partition.

Le répertoire d'indexation d'une partition peut être déplacé vers les deux emplacements suivants :

  • Vous pouvez déplacer le répertoire d'indexation de la partition vers un lecteur local différent sur le même hôte Content Server, ce qui signifie que vous n'avez pas à modifier le serveur d'administration qui gère le moteur d'indexation et les moteurs de recherche correspondants.

  • Vous pouvez déplacer l'index de la partition sur le lecteur d'un autre hôte Content Server, ce qui signifie que vous devez modifier le serveur d'administration qui gère le moteur d'indexation et les moteurs de recherche correspondants.

Pour des raisons de performances, OpenText vous conseille d'opter pour des connexions rapides entre les moteurs d'indexation et moteurs de recherche et les fichiers des répertoires d'indexation correspondants.

Pour déplacer un index d'un hôte vers un autre, vous devez installer plusieurs hôtes Content Server. Pour plus d'informations sur les installations Content Server, reportez-vous au Guide d'installation Content Server. Pour plus d'informations sur l'enregistrement d'un serveur d'administration sur un hôte Content Server distant, reportez-vous à la rubrique Configuration des serveurs d'administration.

Pour afficher les résultats de la recherche sous forme de liens actifs, vous devez créer des mappages de liens hypertextes qui configureront le gestionnaire de recherche d'une source de données Directory Walker afin de traiter les documents par le biais d'un serveur Web. Pour que les mappages de liens hypertextes fonctionnent correctement, vous devez exécuter un serveur Web. Pour plus d'informations sur la configuration des mappages de liens hypertextes, voir Configuration des mappages de liens hypertextes.
