Pour s'assurer que tous les résultats renvoient au document approprié, vous devez créer des mappages de liens hypertextes qui effectueront la configuration du gestionnaire de recherche de manière qu'il traite les documents par le biais d'un serveur Web. Les processus Directory Walker créés dans les versions antérieures de Content Server ne nécessitaient pas de serveur Web, mais ici, un serveur Web est requis. Pour que les mappages de liens hypertextes fonctionnent correctement, vous devez exécuter un serveur Web.

Vous pouvez configurer des mappages de liens hypertextes lorsque vous créez une source de données Directory Walker ou lorsque vous configurez son gestionnaire de recherche séparément. Si vous choisissez de configurer les mappages de liens hypertextes lors de la création d'une source de données Directory Walker, vous n'avez pas besoin de configurer les mappages de liens hypertextes pour le gestionnaire de recherche de la source de données. Pour les autres sources de données non-Entreprise, vous configurez les mappages de liens hypertextes lors de la configuration du gestionnaire de recherche pour la source de données. Voici quelques exemples de sources de données non-Entreprise : la source de données de l'aide administrateur, la source de données Directory Walker, la source de données de l'aide utilisateur et la source de données producteur XML Activator. Pour plus d'informations sur les gestionnaires de recherche, reportez-vous à la rubrique Gestion des processus de recherche.

L'utilisation des mappages de liens hypertextes permet également de modifier le chemin d'accès aux documents indexés après la création de l'index. Par exemple, si les répertoires indexés sont déplacés après la création de l'index, vous modifiez les mappages des liens hypertextes pour le gestionnaire de recherche en question. Si vous déplacez l'index vers un nouvel hôte Content Server, il n'est pas nécessaire de modifier les paramètres de configuration car les mappages des liens hypertextes restent valables, même si le chemin enregistré dans l'index n'est pas valable pour le nouvel hôte.

Vous consignez les répertoires à parcourir lorsque vous configurez un processus Directory Walker. Tous les répertoires consignés doivent avoir une racine commune. Par exemple, si vous définissez le processus Directory Walker pour qu'il parcoure c:/dirA/dir1 et c:/dirA/dir2, vous pouvez créer un mappage de liens hypertextes à partir de c:/dirA. Si vous consignez des répertoires ayant des racines différentes, tels que c:/dirA et c:/dirB ou c:/dirA et d:/dirA, vous ne pouvez pas créer de mappage de liens hypertextes à partir de ces deux chemins. Si vos répertoires n'ont pas de racine commune, envisagez de créer des index Directory Walker individuels pour eux pour qu'ils puissent utiliser les mappages de liens hypertextes. Pour plus d'informations sur la configuration d'une source de données Directory Walker, reportez-vous à la rubrique Indexation des données sur votre système de fichiers.
