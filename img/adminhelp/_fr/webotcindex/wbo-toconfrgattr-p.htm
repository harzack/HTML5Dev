Pour configurer les attributs des régions de l'index :

  1. Sur la page Administration système, cliquez sur le lien préfixe_processus Dossier de la source de données correspondant à la source de données contenant les régions d'index auxquelles vous souhaitez appliquer des attributs de région.

  2. Cliquez sur l'icône Fonctions du gestionnaire de recherche, sélectionnez Propriétés, puis Attributs de région.

  3. Sur l'onglet Attributs de région de la page Propriétés du Gestionnaire de recherche, indiquez les Attributs par défaut pour la langue à appliquer aux régions OTName et OTDComment. Vous pouvez modifier la langue utilisée en saisissant une valeur Xlate au lieu de texte brut.

     

    Note :
    Lors d'une nouvelle installation de Content Server 10.0, aucune valeur n'est définie par défaut pour les attributs de région, donc la zone de texte est vide.

    Dans Content Server 10.0, OTName, OTDComment et OTGUID sont les seules régions utilisant actuellement des attributs, mais d'autres régions seront ajoutées dans des versions ultérieures. OTName et OTDComment sont des régions qui existaient déjà dans des versions antérieures alors qu'OTGUID est une nouvelle région dans Content Server 10.0.

    Lors d'une installation où les sources de données Entreprise sont mises à niveau à partir de Content Server 9.7.1, ou après une mise à jour du contenu dans Content Server 10.0, les attributs des régions OTName et OTDComment sont définis sur la langue des métadonnées par défaut du système et affichés dans la zone de texte.

    Vous pouvez définir la langue des métadonnées pour toutes les valeurs existantes OTName et OTDComment dans un index mis à niveau. Par exemple, pour définir les attributs de régions sur le français, saisissez : 

    OTName="lang","fr"
OTDComment="lang","fr"

     

  4. Cliquez sur le bouton Mettre à jour.
