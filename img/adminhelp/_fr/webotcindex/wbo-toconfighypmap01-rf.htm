Les exemples suivants illustrent les deux modes de configuration des mappages de liens hypertextes dans Content Server.

Pour configurer un mappage de liens hypertextes pour un gestionnaire de recherche :

  1. Indiquez au processus Directory Walker de rechercher un fichier particulier dans un répertoire spécifique, tel que c:/Documents/exemple.doc.

  2. Dans le serveur Web, créez un répertoire correspondant au chemin d'accès à ce fichier, tel que c:/Documents/Nom_répertoire_virtuel.

  3. Copiez tout le chemin.

  4. Sur la page Administration système, cliquez sur le lien préfixe_processus Dossier de la source de données correspondant au dossier de la source de données relatif à l'index Directory Walker qui devra être utilisé par le mappage de liens hypertextes.

  5. Cliquez sur l'icône Fonctions du gestionnaire de recherche, sélectionnez Propriétés, puis Infos spécifiques.

  6. Dans l'onglet Infos spécifiques de la page Propriétés du gestionnaire de recherche, collez le chemin dans le champ Rechercher.

  7. Dans le champ Remplacer par, saisissez ce qui suit : http://Server_name/Virtual_directory_name .

     

    Important :
    Open Text vous recommande vivement d'examiner l'accès possible au document au moyen de votre serveur Web et, si nécessaire, de restreindre correctement l'accès aux emplacements de dossiers sensibles.

    Lorsque vous créez un mappage de répertoire virtuel pour une source de données non Entreprise, le mappage du serveur Web à l'emplacement des dossiers n'hérite pas automatiquement des autorisations définies pour la vue de recherche. Il est donc possible que des utilisateurs puissent accéder, au moyen du serveur Web, à des documents qu'ils ne sont pas autorisés à voir au moyen de Content Server.

     

  8. Cliquez sur le bouton Mettre à jour.

    Sur la page Résultats de recherche, le lien actif vers le document exemple.doc s'affiche désormais comme suit : http://Nom_serveur/Nom_répertoire_virtuel/Documents/exemple.doc.

Pour configurer un mappage de liens hypertextes pour une source de données Directory Walker :

  1. Indiquez au processus Directory Walker de rechercher un fichier particulier dans un répertoire spécifique, tel que c:/Documents/exemple.doc.

  2. Dans le serveur Web, créez un répertoire correspondant au chemin d'accès à ce fichier, tel que c:/Documents/Nom_répertoire_virtuel.

  3. Sur la page Administration système, cliquez sur Source de données Directory Walker dans le menu Ajouter un élément.

  4. Sur la page Créer une nouvelle source de données Directory Walker, collez le préfixe dans le champ Rechercher de la section Informations générales.

  5. Dans le champ Remplacer par, saisissez ce qui suit : http://Server_name/Virtual_directory_name .

     

    Important :
    Open Text vous recommande vivement d'examiner l'accès possible au document au moyen de votre serveur Web et, si nécessaire, de restreindre correctement l'accès aux emplacements de dossiers sensibles.

    Lorsque vous créez un mappage de répertoire virtuel pour une source de données non Entreprise, le mappage du serveur Web à l'emplacement des dossiers n'hérite pas automatiquement des autorisations définies pour la vue de recherche. Il est donc possible que des utilisateurs puissent accéder, au moyen du serveur Web, à des documents qu'ils ne sont pas autorisés à voir au moyen de Content Server.

     

  6. Finalisez la configuration de la source de données Directory Walker.

  7. Cliquez sur le bouton Créer des processus.

    Sur la page Résultats de recherche, le lien actif vers le document example.doc s'affiche désormais comme suit : http://Nom_serveur/Nom_répertoire_virtuel/Documents/exemple.doc.
