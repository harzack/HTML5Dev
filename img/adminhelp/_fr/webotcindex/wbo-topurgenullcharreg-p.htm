To remove a region with a null character from the search index:

  1. Ensure a log level of Info is set for each Index Engine in the Content Server cluster by visiting each Index Engine's Specific tab of their Properties pages.

  2. Stop the Content Server and Admin server services for all Admin servers in the Content Server cluster.

  3. Apply the supplied patch to all Admin servers in the cluster.

  4. To permanently remove regions that have nulls in their name from the index, add the following setting to each [DataFlow_<signature>] section of the search.ini file on each Admin server, for example: 

    [DataFlow_<signature>]
RemoveRegionsWithNull=true

  5. Restart the Admin server services for all Admin servers in the Content Server cluster.

  6. After the Index Engines hydrate metadata information into memory, a file will be created in the install_dir/logs directory of the form: 

    indexengine_{EngineName}_objectIds_regions_null.txt

     

    Note:
    The content of each file is a list of object IDs that were associated with the regions containing null characters in their names.

    Once a particular Index Engine has completed processing, the text Changing ctl file to <#> for Removed Regions with nulls will appear in the Index Engine's log file.

    Regions removed by this process can also be identified in the Index Engine log by entries in the form of "Removing region region name as it contains a null".

    If the grid is started twice with the RemoveRegionsWithNull setting, the files containing the list of ObjectIDs will be overwritten.

     

  7. Once all Index Engines have completed processing, stop the Admin server services in the cluster and remove the setting RemoveRegionsWithNull from the search.ini files.

  8. Restart the Content Server and Admin server services.

  9. Revert the Index Engine log levels to their original values.
