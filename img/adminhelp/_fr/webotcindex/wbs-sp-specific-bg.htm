Un Spider extrait les données d'un serveur Spider qui parcourt et indexe toute quantité de sources de connaissances Web que vous avez définies comme étant pertinentes pour votre entreprise. Il rend l'index obtenu disponible pour effectuer des recherches dans l'interface Web du système Content Server.

Vos utilisateurs ont l'avantage de pouvoir envoyer une requête de recherche unique qui recherche le contenu dans le référentiel, mais également dans les sources Web ciblées. Les utilisateurs reçoivent un ensemble unifié de résultats et peuvent rapidement trouver des informations pertinentes, quel que soit leur emplacement.

Un Spider peut être associé à la fonction Prospecteurs de Content Server pour permettre aux utilisateurs de trouver automatiquement des données pertinentes dans les sources de données parcourues.

 

Note :
Les URL Web qui seront parcourues, ainsi que les caractéristiques du projet, sont définies sur le serveur Spider installé séparément à partir de Content Server. Pour plus d'informations sur le package serveur Spider, contactez le service d'assistance clientèle d'OpenText.

Les processus producteurs Spider ne peuvent être ajoutés qu'au Gestionnaire de flux de données de toute source de données non Entreprise.

La mise en évidence des résultats n'est pas disponible pour les résultats de recherche Spider.

Si Content Server et le processus Spider sont exécutés sous UNIX et se trouvent sur différents hôtes, ils doivent partager un lecteur monté transversalement sous le même nom. Ceci est nécessaire pour permettre aux processus de flux de données Content Server de lire dans le répertoire vers lequel le Spider écrit les données qu'il collecte. Cette restriction ne s'applique pas à Windows. Sous Windows, les lecteurs doivent être mappés entre les deux hôtes. Ils ne doivent pas avoir la même lettre de lecteur.

 

Pour plus d'informations sur l'ajout d'un processus Spider, voir Pour ajouter un Spider.
