Pour configurer les options de recherche pour les régions d'index :

  1. Sur la page Administration, dans la section Administration de recherche, cliquez sur le lien Ouvrir le volume des objets système.

  2. Sur la page Administration système, cliquez sur le lien préfixe_processus Dossier de la source de données correspondant à la source de données contenant les régions d'index à configurer pour la recherche et/ou l'affichage.

  3. Cliquez sur l'icône Fonctions du gestionnaire de recherche, sélectionnez Propriétés, puis Régions.

  4. Dans l'onglet Régions de la page Propriétés du Gestionnaire de recherche, choisissez les options à appliquer pour chaque région affichée :

    1. Saisissez un nouveau nom dans le champ Libellé d'affichage d'une région afin de modifier le nom de la région.

       

      Note :
      Vous ne pouvez pas modifier le Libellé d'affichage de certaines régions dans les deux circonstances suivantes :

      • Le libellé d'affichage des régions d'attributs de catégories est dérivé des noms de catégorie et d'attribut, conformément au modèle [nom de catégorie]:[nom d'attribut]. Dans ce cas, le libellé d'affichage de la région d'attribut est constitué d'un lien vers la définition de la catégorie.

      • Les libellés d'affichage d'un nombre réduit de régions de recherche de base sont codés en dur et ne peuvent être modifiés. Ils incluent notamment : OTDocumentRating, OTLocation, OTName, OTObjectDate, OTObjectSize.

       

    2. Sélectionnez la case à cocher Pour recherche d'une région afin de permettre aux utilisateurs d'y effectuer des recherches.

       

      Note :
      La case à cocher Pour recherche ne s'affiche pas pour les régions d'attributs de catégories. Pour ces régions, la case à cocher Afficher dans la recherche sur la page de définition d'attribut détermine si la région apparaît dans le composant de recherche de catégorie.

      Les attributs de workflow d'index sont stockés dans la région WFATTR. Par défaut, il n'est pas possible d'effectuer des recherches dans ces régions sur la page Recherche avancée. Pour que les recherches soient possibles, sélectionnez la case à cocher Pour recherche de chaque région. Une fois activée, la région d'attribut s'affiche dans le composant des attributs système.

       

    3. Sélectionnez la case à cocher Pour affichage d'une région afin de permettre aux utilisateurs de l'afficher sur la page Résultats de recherche.

    4. Sélectionnez la case à cocher Recherche par défaut d'une région pour que les recherches y soient effectuées par défaut.

    5. Sélectionnez la case à cocher Pouvant être trié(e)(s) d'une région afin de permettre aux utilisateurs de trier les régions concernées sur la page Résultats de recherche.

  5. Cliquez sur le bouton Mettre à jour.

 

Note :
Si une région ne comporte pas de case à cocher Pour affichage, c'est parce qu'aucun élément comportant cet attribut n'a encore été ajouté à la base de données Content Server. Une fois qu'un élément comportant cet attribut est ajouté et indexé, la case à cocher Pour affichage apparaît sur la ligne de la région.

La case à cocher Pour affichage ne dépend pas de la région en cours d'indexation. Elle est toujours présente pour la plupart des régions. Si une région facultative, comme un attribut de catégorie, n'est pas encore indexée, elle ne s'affiche pas sur l'onglet des régions du gestionnaire de recherche.
