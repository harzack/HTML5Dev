Pour configurer une région de workflow pouvant être extraite :

  1. Ouvrez le nouveau fichier type LLFieldDefinitions.txt dans le répertoire Content Server_accueil\config\reference d'un éditeur de texte puis recherchez workflow pour voir quelles régions de workflow sont indexées.

  2. Recherchez la section Content Server Workflow pour voir quelles régions de workflow sont marquées comme DROP ou REMOVE et ne sont pas indexées.

  3. Ouvrez votre fichier actif LLFieldDefinitions.txt dans le répertoire Content Server_accueil\config d'un éditeur de texte.

  4. Dans la section Content Server Workflow de votre fichier actif LLFieldDefinitions.txt, saisissez # au début d'une région que vous voulez indexer, si la ligne commence par DROP ou REMOVE.

    Le signe # indique la directive du moteur d'indexation à rejeter la région.

  5. Enregistrez et fermez votre fichier actif LLFieldDefinitions.txt dans le répertoire Content Server_accueil\config.

  6. Reconstruisez l'index. Pour plus d'informations sur la reconstruction de l'index, reportez-vous à la rubrique Résultats de maintenance de sources de données.
