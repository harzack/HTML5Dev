If an outer region is not listed in the LLFieldDefinitions.txt file, its nested metadata regions will not be indexed. For information about adding regions to the LLFieldDefinitions.txt file, see To Configure Nested Index Regions. Outer regions are listed in the nested section in the LLFieldDefinitions.txt file. This file is located in the Config directory of your Content Server installation. Regions are listed in the following format in the LLFieldDefinitions.txt file:

NESTED    OTTestA

NESTED is a prefix for each outer region, the space between NESTED and OTTestA is a tab space, and OTTestA is the name of the outer region. The outer region's name is case sensitive.

Regions are organized in a hierarchy in Content Server, and the LLFieldDefinitions.txt file reflects this organization. This hierarchy is necessary because each outer region in the list may be nested within another outer region. To ensure that all metadata regions are indexed, the list must contain all outer regions within which a metadata region can appear. For example, if the outer region OTTestB is nested within OTTestA, you must add both OTTestB and OTTestA to the nested section of the LLFieldDefinitions.txt file. If you only add OTTestB and not OTTestA, Content Server will not search within OTTestA and will be unable to find OTTestB.
