Vous pouvez sélectionner les options de recherche suivantes pour les régions d'index :

  • Pour recherche, qui permet aux utilisateurs de choisir les régions qui apparaissent en tant qu'attributs système sur la page Recherche avancée

  • Pour affichage, qui permet aux utilisateurs d'afficher les régions sur la page Résultats de recherche

  • Recherche par défaut, qui spécifie qu'une recherche est automatiquement effectuée dans les régions.

  • Pouvant être trié(e)(s), qui permet aux utilisateurs de trier les régions sur la page Résultats de recherche

  • Filtre, qui permet aux utilisateurs d'utiliser les filtres de recherche avec ces régions sur la page Résultats de recherche. Pour en savoir plus, voir Filtres de recherche.

 

Note :
Il est important de vérifier quelles régions sont configurées pour recherche, pour affichage, pour recherche par défaut et pour tri lors de l'installation d'un nouveau module ou de la mise à niveau de Content Server ou de l'un de ses modules, car le nouveau logiciel peut inclure de nouvelles régions. Vous ne pouvez cependant pas configurer d'options de recherche pour les nouvelles régions avant l'indexation du nouveau logiciel et l'introduction de données dans les régions. Si les régions que vous pensiez voir dans l'onglet Régions du gestionnaire de recherche n'apparaissent pas, cela signifie peut-être qu'elles sont imbriquées. Vous pouvez configurer les régions imbriquées en modifiant le fichier LLFieldDefinitions.txt dans le répertoire Config de votre installation Content Server.

 

Lorsque vous configurez des régions d'index, vous pouvez également rendre visibles les attributs au sein des catégories. Cependant, pour que les attributs d'une catégorie soient par défaut configurés pour la recherche (c'est-à-dire pour qu'ils s'affichent par défaut lorsque la catégorie est ajoutée à la page Recherche), activez la case Afficher dans la recherche lors de l'ajout ou de la modification d'un attribut.

Les utilisateurs peuvent effectuer des recherches dans des régions existantes à l'aide du langage LQL (Live Query Language), quelles que soient les options de recherche spécifiées pour les régions. La spécification de l'option Pour recherche ou Recherche par défaut pour une région élimine le besoin de préciser le nom de la région dans une instruction LQL.
