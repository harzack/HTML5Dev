Vous pouvez utiliser la recherche à distance OpenText Content Server pour effectuer des recherches dans plusieurs installations Content Server. Il vous suffit de configurer une connexion à distance entre les installations en ajoutant et en configurant un système Content Server distant, qui figure dans une source de données et qui est accessible aux vues de recherche. Cela permet à l'administrateur de créer de nouvelles vues de recherche pour effectuer des recherches dans les systèmes Content Server distants ou pour ajouter l'objet aux vues de recherche existantes. Pour en savoir plus sur les vues de recherche, reportez-vous à la rubrique Création des vues de recherche. Lorsque l'administrateur accède au parent de cet objet, ce dernier a un statut particulier ou est inactif, il signale une recherche en cours ou une erreur de processus, à l'instar d'autres processus de recherche. Pour en savoir plus sur les sources de données, consultez la rubrique Ajout et configuration d`une nouvelle base de données. Configuration d'une recherche à distance

Vous configurez un processus de recherche à distance pour établir une connexion entre des systèmes Content Server en ajoutant un système Content Server distant au dossier de la source de données de l'entreprise. Ce processus se comporte comme un navigateur Web ; il communique des informations au système Content Server qui sert de serveur distant en langage HTTP. En outre, ce processus accepte les connexions à partir d'un système Content Server qui sert de serveur client. Lorsque vous ajoutez un système Content Server distant, vous devez configurer l'authentification pour que l'administrateur puisse se connecter au système Content Server distant. Pour en savoir plus sur les processus de recherche, reportez-vous à la rubrique Gestion des processus de recherche.

 

Notes :

  • Cet utilisateur peut être l'administrateur système ou un utilisateur qui dispose des privilèges d'administration du système.

  • Lorsqu'un système Content Server agissant comme un serveur client exécute des opérations sur les résultats de recherche qui doivent être traitées par un système Content Server fonctionnant comme un serveur distant, le serveur client doit être ajouté comme référent valide du serveur distant. La mise en évidence des résultats et les commandes du menu Fonctions sont des exemples d'opérations sur les résultats de recherche. Pour plus d'informations sur l'ajout d'un référent valide à Content Server,voir Configuration des paramètres de sécurité.

 

Vous devez préciser le type d'autorisation dont a besoin l'utilisateur pour effectuer une recherche dans le système Content Server distant. Lorsque vous ajoutez un nouveau système Content Server distant, vous pouvez spécifier l'un des paramètres de contrôle de l'accès des utilisateurs suivants :

  • Utilisateur Natif uniquement, les recherches s'effectuent sur le serveur distant en tant qu'utilisateur initiateur à partir du système Content Server local.

     

    Note :
    Si l'utilisateur n'existe pas encore sur le serveur distant, la requête de recherche renverra un message d'erreur de type « Aucun résultat trouvé » et sera consignée dans le fichier journal en tant qu'avertissement.

     

  • Utilisateur Public nommé, les recherches sur le serveur distant s'effectuent toujours en tant que simple utilisateur. Le nom de l'utilisateur est saisi dans le champ d'entrée.

  • Retour de l'utilisateur Natif à l'utilisateur Public nommé, identique à Utilisateur Natif uniquement si l'utilisateur initiateur à partir du système Content Server local se trouve sur le serveur distant. Si l'utilisateur local ne se trouve pas sur le serveur distant, les recherches s'effectuent sous le nom d'utilisateur spécifié sur le serveur distant.
