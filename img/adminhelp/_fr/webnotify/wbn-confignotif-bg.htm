La page Configurer la notification vous permet d'activer et de désactiver les notifications, de définir l'URL de la page d'accueil Content Server, de configurer les paramètres SMTP, de définir les paramètres d'e-mail, de définir les paramètres de rapport et de définir les paramètres de programme de notification par défaut.

Pour plus d'informations sur l'activation ou la désactivation des notifications, voir Activation et désactivation des notifications.

Modification de l'URL pour les rapports de notification

Notification utilise l'URL de votre serveur pour créer les liens HTTP vers les éléments Content Server des rapports de Notification. De ce fait, vous devez vous assurer que cette URL est correctement spécifiée sur la page Configurer la notification.

Normalement, il n'est pas nécessaire de modifier l'URL qui apparaît dans la zone URL de la page d'accueil de Content Server. Toutefois, si les utilisateurs indiquent que les liens des rapports de notification ne fonctionnent pas, il se peut que l'URL soit incorrecte.

Pour modifier l'URL, respectez le format suivant :

  • <protocole>://<hôte> :<port>/<préfixe_URL>/contentserver.exe si le serveur fonctionne sous Windows.

  • <protocole>://<hôte> :<port>/<préfixe_URL>/contentserver si le serveur fonctionne sous UNIX.

où :

  • <protocole> est soit http, soit https.

  • <hôte> est le nom complet de l'hôte sur lequel réside le serveur HTTP. Par exemple, contentserverhost.mycorp.com.

  • <port> est le port récepteur du serveur HTTP.

  • <préfixe_URL> est le préfixe URL, ou raccourci de répertoire virtuel, mappé au répertoire <Content Server_accueil>/cgi du serveur HTTP.

Utilisation de la messagerie pour envoyer des notifications

La notification utilise des e-mails pour avertir les utilisateurs lorsque certains événements se sont produits et pour informer l'administrateur des erreurs qui ont pu se produire dans le système de notification.

Toutes les adresses e-mail doivent être des adresses e-mail Internet valables, au format compte@domaine, où domaine correspond au domaine du serveur du compte de messagerie. Elles se présentent généralement sous la forme monentreprise.com, monorganisation.org ou monétablissementscolaire.edu.

La notification prend en charge le protocole SMTP (Simple Mail Transport Protocol) pour le transport des e-mails. Pour que Notification puisse fonctionner correctement, vous devez configurer les paramètres SMTP avec précision.

La notification vous permet de tester l'envoi d'e-mails aux adresses qui s'affichent dans les champs Adresse de l'expéditeur par défaut, Adresse de réponse par défaut et Adresse sur erreur de la page Configurer la notification. Vous pouvez également tester l'envoi d'e-mails à d'autres adresses.
