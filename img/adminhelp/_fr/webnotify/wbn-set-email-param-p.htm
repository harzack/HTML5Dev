Pour définir les paramètres d'e-mail :

  1. Sur la page Administration Content Server, dans la section Administration des notifications, cliquez sur Configurer la notification.

  2. Dans la zone Paramètres de l'e-mail de la page Configurer la notification, cliquez sur un des types d'e-mails suivants dans la liste Type de message par défaut :

    • Corps de texte normal uniquement pour envoyer des rapports en texte normal sans attribut. Tous les clients de messagerie sont en mesure de gérer ce type de message.

    • Corps HTML uniquement pour envoyer des rapports au format HTML. Si les utilisateurs disposent de clients de messagerie prenant en charge le format HTML, il s'agit du format le plus pratique dans la mesure où ils peuvent cliquer directement sur les liens de la notification à l'intérieur de l'e-mail. Si les logiciels de messagerie de vos utilisateurs ne peuvent pas afficher le format HTML, ne sélectionnez pas cette option.

    • Corps texte normal avec pièce jointe HTML pour envoyer des rapports en texte normal sans attribut, avec une version HTML en pièce jointe au message. Les utilisateurs peuvent utiliser les pièces jointes HTML dans leur navigateur Web.

  3. Dans le champ Objet par défaut, saisissez la chaîne de caractères que devra utiliser la notification sur la ligne d'objet des e-mails de notification sortants. La valeur par défaut est : Content Server Notification le %1 où 1% est une variable qui insère la date et l'heure en fonction des paramètres de date de date et d'heure correspondants définis sur le serveur.

  4. Si vous avez sélectionné Corps de texte normal avec pièce jointe HTML comme Type de message par défaut à l'étape 2, saisissez la chaîne de caractères que devra utiliser Notification pour nommer les pièces jointes HTML dans le champ Nom du fichier joint par défaut. Ces pièces jointes sont celles qui apparaissent dans les e-mails de notification. La valeur par défaut est : %a%m%j_%H%M.html, où :

    • %a correspond à l'année codée sur chaque chiffre.

    • %m correspond au mois codé sur deux chiffres.

    • %d j correspond au jour du mois, codé sur deux chiffres.

    • %H%M correspond à l'heure et aux minutes, au format 24 heures.

  5. Dans le champ Base de données Content Server par défaut, saisissez le nom de la base de données Content Server actuelle. La notification affiche ce nom dans le champ par défaut. Notification affiche ce nom de base de données dans les en-têtes des rapports.

  6. Dans le champ Adresse de l'expéditeur par défaut, saisissez l'adresse e-mail de l'expéditeur qui apparaîtra dans les e-mails de notification. Par défaut, la notification affiche une adresse e-mail basée sur le nom DNS de l'hôte principal Content Server, par exemple, hostname@domain.com.

  7. Dans le champ Adresse de réponse par défaut, saisissez l'adresse e-mail qui apparaîtra dans les e-mails de la notification comme adresse de réponse. Par défaut, la notification affiche une adresse e-mail basée sur le nom DNS de l'hôte principal Content Server, par exemple, hostname@domain.com.

  8. Dans le champ Objet sur erreur, saisissez la chaîne de caractères que devra utiliser la notification sur la ligne d'objet des messages d'erreurs de la notification sortants. La valeur par défaut est : Content Server Erreur de la notification le %1, où 1% est une variable qui insère la date et l'heure en fonction des paramètres de date et d'heure correspondants définis sur le serveur.

  9. Dans le champ Adresse sur erreur, saisissez l'adresse e-mail à laquelle la notification enverra les messages d'erreur. Par défaut, la notification affiche une adresse e-mail basée sur le nom DNS de l'hôte principal Content Server, par exemple, hostname@domain.com.

  10. Cliquez sur Envoyer.

     

    Note :
    Si vous configurez pour la première fois la notification, vous devez également activer la notification et configurer SMTP et les paramètres de rapport. Pour plus d'informations, voir les rubriques Pour activer des notifications, Pour définir les paramètres SMTP et Pour configurer les paramètres de rapport.
