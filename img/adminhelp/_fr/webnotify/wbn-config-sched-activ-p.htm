Pour configurer des activités programmées de notification :

  1. Sur la page Administration Content Server, dans la section Administration des notifications, cliquez sur Configurer les activités programmées.

  2. Pour activer le producteur d'événements à minuit, dans la section Producteur d'événements à minuit, effectuez les opérations suivantes :

    1. Dans le champ Statut, cliquez sur Activer.

    2. Dans la section Programme d'activités, spécifiez les jours et heures d'exécution souhaitée de l'événement.

  3. Pour activer le nouvel essai après l'échec de suppression du blob du fournisseur, dans la section Nouvel essai après l'échec de suppression du blob du fournisseur, effectuez les opérations suivantes :

    1. Dans le champ Statut, cliquez sur Activer.

    2. Dans la section Programme d'activités, spécifiez les jours et heures d'exécution souhaitée de l'événement.

  4. Pour activer la fonction Glisser et déposer la notification d'éléments incomplets, dans la section Glisser et déposer la notification d'éléments incomplets, effectuez les opérations suivantes :

    1. Dans le champ Statut, cliquez sur Activer.

    2. Dans la section Programme d'activités, spécifiez les jours et heures d'exécution souhaitée de l'événement.

    3. Dans le champ Retard d'exécution, dans la liste Minutes, sélectionnez le nombre de minutes pour le délai.

  5. Pour vider les messages antérieurs, dans le champ Statut de la section Vider les messages antérieurs, cliquez sur Activer.

  6. Pour désactiver les comptes des utilisateurs dont les mots de passe ont expiré, dans la section Faire expirer les mots de passe, effectuez les opérations suivantes :

    1. Dans le champ Statut, cliquez sur Activer.

    2. Dans la section Programme d'activités, spécifiez les jours et heures d'expiration souhaitée des mots de passe.

  7. Pour notifier les utilisateurs lorsque leurs mots de passe sont sur le point d'expirer, dans la section Contrôle de l'expiration du mot de passe, effectuez les opérations suivantes :

    1. Dans le champ Statut, cliquez sur Activer.

    2. Dans la première zone de saisie du champ Notifications par e-mail, spécifiez le nombre de jours avant l'expiration du mot de passe où la première notification est envoyée.

    3. Dans la deuxième zone de saisie du champ Notifications par e-mail, spécifiez le nombre de jours avant l'expiration du mot de passe où les notifications quotidiennes commencent à être envoyées.

  8. Pour informer l'administrateur système lorsque le nombre d'échecs de tentatives de connexion dépasse le nombre maximal autorisé, dans la section Notification d'échec de connexion, effectuez les opérations suivantes :

    1. Dans le champ Statut, cliquez sur Activer.

    2. Dans la section Programme d'activités, spécifiez les jours et heures où l'administrateur est informé.

    3. Dans la première zone de saisie du champ Seuil de notification à l'administrateur, saisissez le nombre maximum d'échecs de tentatives de connexion avant que l'administrateur ne soit informé.

    4. Dans la deuxième zone de saisie du champ Seuil de notification à l'administrateur, saisissez le nombre de minutes pendant lesquelles ces tentatives de connexion se produisent avant que l'administrateur ne soit informé.

  9. Pour activer l'agent client 5 minutes, dans la section Agent client 5 minutes, effectuez les opérations suivantes :

    1. Dans le champ Statut, cliquez sur Activer.

    2. Dans le champ Eliminer les événements, sélectionnez la case à cocher Eliminer sur envoi pour éliminer les événements.

    3. Dans la section Programme d'activités, spécifiez les jours et heures d'exécution souhaitée de l'événement.

  10. Pour transférer les événements de notification de la table NotifyEvents vers LLEventQueue, dans la section Transférer les événements de notification, effectuez les opérations suivantes :

    1. Dans le champ Statut, cliquez sur Activer.

    2. Dans la section Programme d'activités, spécifiez les jours et heures de transfert des données.

  11. Pour activer le producteur d'événements de client, dans la section Producteur d'événements de client, effectuez les opérations suivantes :

    1. Dans le champ Statut, cliquez sur Activer.

    2. Dans le champ Eliminer les événements, sélectionnez la case à cocher Eliminer sur envoi pour éliminer les événements.

    3. Dans la section Programme d'activités, spécifiez les jours et heures où l'administrateur recevra une copie des événements de l'agent de notification des transferts.

  12. Pour convertir des événements de nœud individuels en événements de message pour chaque utilisateur, dans la section Processeur d'événements du noeud, effectuez les opérations suivantes :

    1. Dans le champ Statut, cliquez sur Activer.

    2. Dans le champ Eliminer les événements, sélectionnez Eliminer sur envoi pour éliminer les événements de nœud.

    3. Dans la section Programme d'activités, spécifiez les jours et heures de conversion des événements de nœud en événements de message.

    4. Dans la section Options de processeur, sélectionnez Vérifications d'autorisation supplémentaires pour activer les vérifications d'autorisation supplémentaires pour chaque utilisateur qu'un événement intéresse.

    5. Dans la liste Nombre de processeurs, sélectionnez le nombre d'instances du gestionnaire d'agent NodeEventProcessor qui peuvent être exécutées simultanément.

    6. Sélectionnez Traiter l'utilisateur Admin séparément pour distribuer de manière égale le nombre d'utilisateurs par rapport au nombre d'instances requis.

  13. Cliquez sur Envoyer.
