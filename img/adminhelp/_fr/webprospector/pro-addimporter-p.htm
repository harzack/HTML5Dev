Pour ajouter un processus d'importation de prospecteurs :

  1. Cliquez sur le lien Ouvrir le volume des objets système de la section Administration de recherche sur la page Administration.

  2. Sur la page Volume des objets système, cliquez sur le lien <préfixe_processus> de Dossier de la source de données correspondant à la source de données contenant le flux de données auquel vous voulez ajouter un processus d'importation de prospecteurs.

  3. Cliquez sur le lien <préfixe_processus> du Gestionnaire de flux de données.

  4. Cliquez sur Importateur dans le menu Ajouter un élément.

  5. Sur la page Ajouter Importateur, saisissez un nom pour le processus d'importation de prospecteurs dans le champ Nom. Si vous voulez ajouter une description du processus d'importation de prospecteurs, saisissez-la dans le champ Description.

  6. Dans la liste déroulante Hôte, cliquez sur le raccourci du serveur d'administration Content Server sur l'hôte duquel le processus d'importation de prospecteurs doit s'exécuter.

  7. Pour permettre à Content Server de surveiller le statut de ce processus, cochez la case Activer la gestion du système.

  8. Pour spécifier le répertoire d'exécution du processus d'importation de prospecteurs, saisissez-en le chemin absolu dans le champ Répertoire de démarrage. Si vous ne spécifiez pas de répertoire, le processus d'importation de prospecteurs est exécuté dans le répertoire Content_Server_home/bin.

  9. Dans la liste déroulante Importer la définition de tâche, cliquez sur Prospecteur Si vous voulez configurer la tâche exécutée par le processus d'importation de prospecteurs, cliquez sur le bouton Configurer.

  10. Dans le champ Zone de base iPool de la section Infos sur Interchange Pool, saisissez le chemin absolu de l'IPool à partir duquel le processus Importateur de prospecteurs devra lire les données. Si le flux de données contient déjà au moins un processus, la liste déroulante Processus apparaît.

  11. Cliquez sur le processus qui précèdera le processus d'importation de prospecteurs dans le flux de données dans la liste déroulante Processus.

     

    Astuce :
    Dans la plupart des flux de données, le processus d'importation de prospecteurs suit le processus Distributeur de mise à jour <processes_prefix>.

     

  12. Dans la section Options de lancement, programmez le moment de l'exécution du processus d'importation de prospecteurs. Pour plus d'informations sur la configuration des processus Importateur, reportez-vous à la rubrique Configuration d`un processus Importateur.

  13. Cliquez sur le bouton Ajouter.

  14. Cliquez sur le lien Regénération des fichiers de requêtes du prospecteur dans la section Administration des prospecteurs de la page Administration.

 

Note :
Si le Content Server qui héberge le processus d'importation de prospecteurs est différent du Content Server qui héberge le processus du moteur d'indexation, vous devez saisir le chemin d'accès à son répertoire dans le champ Zone de base iPool de la section Infos sur Interchange Pool, tel qu'il est mappé ou monté sur l'hôte d'importation. Si vous ajoutez un processus d'importation de prospecteurs à un flux de données Directory Walker surveillant un répertoire statique, vous devez exécuter une maintenance pour éliminer le flux de données et reconstruire l'index. Cette action met l'index à jour et permet aux prospecteurs d'obtenir des résultats dans le répertoire statique. Pour plus d'informations sur l'élimination et la réindexation des flux de données, reportez-vous à la rubrique Pour réindexer le contenu Entreprise.
