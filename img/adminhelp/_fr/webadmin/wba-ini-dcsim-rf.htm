Les paramètres de la section [DCSIm] définissent la configuration de Document Filters (IM Filter) d'OpenText utilisés dans le service de conversion de documents (DCS). Document Filters (IM Filter) d'OpenText sont un ensemble de composants du service de conversion de documents et de fichiers associés qui étendent les capacités de détection de type MIME, d'extraction de texte et de conversion de documents de Content Server.

Pour plus d'informations sur DCSIm, voir Filtres de conversion de documents et Document Filters Administrator's Guide (Manuel administrateur Document Filters dans la base de connaissance OpenText : https://knowledge.opentext.com/knowledge/piroot/llescf/v100000/llescf-agd/en/docovw.xml .

dllpath

  • Description :

    Indique le chemin d'accès au répertoire où sont installées les ressources du filtre DCSIm.

  • Syntaxe :

    dllpath=Content Server_home/filters/image

  • Valeurs :

    Un chemin absolu. La valeur par défaut est Content Server_home/filters/image.

lib

  • Description :

    Détermine le nom de la bibliothèque que le DCS doit charger. Une bibliothèque est une liste d'opérations associées à un filtre de conversion que le DCS lit pour effectuer la conversion.

  • Syntaxe :

    lib=DCSIm

  • Valeurs :

    La valeur par défaut est DCSIm. La modification de la valeur de ce paramètre risque d'empêcher le DCS de fonctionner ou de fonctionner correctement. Ne modifiez pas la valeur de ce paramètre, sauf si le service d'assistance clientèle d'OpenText vous le demande.

outputoleinfo

  • Description :

    Indique si le DCSIm doit extraire les propriétés OLE du document et ne récupérer que les balises de métadonnées OLE répertoriées dans le fichier metadataTags.txt. OLE est une technologie d'intégration de programmes prise en charge par tous les programmes Microsoft Office. OLE permet de partager des informations entre différents programmes.

    Si ce paramètre est activé, le DCSIm extrait les propriétés OLE standard, ainsi que toutes les propriétés OLE personnalisées associées à un document.

     

    Note :
    Ce réglage s'applique aux régions personnalisées des documents Office 2007 et 2010 comme Word, Excel et PowerPoint.

     

  • Syntaxe :

    outputoleinfo=TRUE

  • Valeurs :

    Lorsque la valeur est outputoleinfo=TRUE, seules les balises de métadonnées OLE (répertoriées dans le fichier metadataTags.txt file) sont récupérées. Lorsque la valeur est outputoleinfo=FALSE ou n'est pas définie, toutes les balises de métadonnées sont récupérées.

viewpagerange

  • Description :

    Indique quelles pages d'un document DCSIm va convertir au format HTML pour l'option Afficher en tant que page Web.

  • Syntaxe :

    viewpagerange=all

  • Valeurs :

    Une plage d'entiers de 1 à X, ou all. Par exemple, pour convertir uniquement les dix premières pages au format HTML, définissez viewpagerange=1-10. La valeur par défaut est all.

WordExcel2010HtmlViewOn

  • Description :

    Indique si l'option Afficher en tant que page Web pour les formats Word et Excel d'Office 2010 est activée dans DCSIm.

  • Syntaxe :

    WordExcel2010HtmlViewOn=TRUE

  • Valeurs :

    TRUE ou FALSE. Lorsque la valeur est WordExcel2010HtmlViewOn=FALSE ou n'est pas définie, l'option Afficher en tant que page Web pour les formats Word et Excel d'Office 2010 n'est pas activée.

x-timeout

  • Description :

    Indique le temps d'attente maximum (en secondes) avant la fin d'un processus de traitement de conversion de documents. Configurez ce paramètre lorsque la valeur par défaut définie dans le paramètre timeout n'est pas valable. Vous configurez le paramètre timeout pour chaque filtre de conversion. Par exemple, certains filtres de conversion sont plus lents que d'autres. Dans ce cas, la valeur timeout par défaut de 30 secondes peut ne pas être applicable, car le délai de conversion moyen est supérieur à cette valeur. Dans ce cas, vous pouvez modifier la valeur x-timeout en choisissant une valeur plus élevée et plus appropriée.

     

    Avertissement
    OpenText vous recommande vivement de ne pas modifier la valeur de ce paramètre.

     

  • Syntaxe :

    x-timeout=30

  • Valeurs :

    La valeur par défaut est héritée du paramètre timeout de la section [DCSworker] du fichier opentext.ini. La valeur par défaut est 30 secondes.

     

    Note :
    Lorsque les valeurs x-timeout et timeout de la section [DCSworker] du fichier opentext.ini sont spécifiées, la valeur la plus basse est appliquée en premier et le processus de traitement de conversion de documents prend fin.

    Le paramètre x-timeout est actif uniquement lorsque le filtre de conversion qu'il modifie est géré par un processus de traitement. Pour plus d'informations sur la configuration d'un processus de traitement, reportez-vous à la rubrique [DCS].
