Pour exécuter un jeu personnalisé de tests de diagnostic sur votre base de données Content Server :

  1. Sur la page Options de vérification personnalisées, sélectionnez un des tests suivants en cochant la case correspondante :

    • Vérifier le nombre de listes de contrôle d'accès d'un noeud

    • Vérifier le nombre d'enfants d'un recueil

    • Vérifier si un DataID existe pour chaque DocID

    • Vérifier si un ID de fournisseur ProviderData existe pour chaque ID de fournisseur sous DVersData

    • Vérifier la taille du stockage de documents externe

    • Vérifier si chaque fichier représenté dans la table ProviderData existe dans le stockage de documents externe

    • Vérifier le nom de chaque fichier du stockage de documents externe

    • Vérifier l'existence des membres et chefs de groupe

    • Vérifier si chaque groupe enregistré dans KUAFChildren existe

    • Rechercher les nœuds sans ID parent

    • Vérifier l'ID parent de chaque noeud

    • Vérifier si un providerID DVersData existe pour chaque providerID sous ProviderData

    • Vérifier si le refCnt d'un nœud correspond au nombre de versions

  2. Cliquez sur le bouton Etablir diagnostic. La page Rapport de vérification de la base de données Content Server initiale apparaît.
