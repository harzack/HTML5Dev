Content Server requiert une licence avant que les utilisateurs qui ne disposent pas du privilège System administration rights puissent se connecter. Vous pouvez appliquer une licence de production, de non-production ou temporaire à Content Server.

Pour obtenir une licence pour Content Server, vous devez acquérir un fichier de licence auprès d'OpenText, puis l'appliquer dans Content Server. Les licences Content Server sont liées à une empreinte système générée à partir des informations dans votre base de données Content Server. (Les informations sont codées et hachées afin qu'elles ne soient pas lisibles.) Un seul fichier de licence suffit pour obtenir une licence pour différentes instances de Content Server qui se connectent à la même base de données.

Certains modules Content Server requièrent également des licences. Les modules qui requièrent des licences apparaissent sur la page Gérer les licences de la section Présentation des licences de module.
