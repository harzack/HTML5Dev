Un kit de localisation est disponible sur le Knowledge Center OpenText. Avant de pouvoir configurer des langues, vous devez créer un nouveau lot de langue et l'installer sur l'ordinateur où s'exécute Content Server.

Lorsqu'une nouvelle instance de Content Server est installée, la langue par défaut est toujours l'anglais. Les lots de langues français, allemand, japonais et néerlandais sont inclus dans l'installation de Content Server. Si vous souhaitez utiliser l'une de ces cinq langues, il n'est pas nécessaire de créer un nouveau kit de localisation.

 

Important :
OpenText vous recommande vivement de supprimer et de recréer le dossier de la source de données : Aide utilisateur et le dossier de la source de données : Aide administrateur lorsque vous ajoutez ou supprimez des lots de langues ou lorsque le paramètre régional du système est modifié. Pour en savoir plus, voir Création de l`index de l`aide utilisateur ou administrateur.

 

Installation de lots de langues

Lorsque vous disposez du lot de langue que vous souhaitez installer, et en fonction de s'il s'agit d'un lot de langue de système Content Server ou d'un lot de langue de module Content Server, vous devez effectuer l'une des opérations suivantes :

  1. Si vous installez un lot de langue de système Content Server, vous devez décompresser le lot de langue dans le répertoire d'installation racine de Content Server.

    Par exemple, si vous avez installé Content Server sur le répertoire C:\OPENTEXT, décompressez le lot de langue que vous avez téléchargé ou créé dans le répertoire C:\OPENTEXT. Le lot de langue est décompressé dans C:\OPENTEXT\langpkgstaging.

  2. Si vous installez un lot de langue de module Content Server, vous devez décompresser le lot de langue dans le répertoire C:\<Content Server_home>\langpkgstaging\module, où C:\<Content Server_home> correspond à l'emplacement de votre installation Content Server.

 

Note :
Lorsque vous installez un lot de langue, si les paramètres existent déjà dans le fichier opentext.ini, et que la valeur du réglage n'est pas vide, les valeurs sont remplacées par les paramètres du lot de langue que vous installez.

 

Une fois qu'une langue est installée, le code de langue apparaît par ordre alphabétique sous la section Langues de la page Lots de langues installés. Content Serveret chaque module Content Server affichent toutes les langues disponibles ; si une langue n'est pas installée pour un module particulier, un message « Aucune langue installée » s'affiche. Pour de plus amples informations sur les noms et codes de langues, reportez-vous à la section Configuration des langues.

Vous pouvez spécifier un format de nom d'affichage utilisateur différent pour chaque langue. Le nom d'affichage pour les utilisateurs peut être leur ID de connexion ou leur prénom, leur nom et leur initiale intermédiaire. Vous pouvez également décider d'ajouter les ID de connexion des utilisateurs au format de votre choix. Pour en savoir plus, voir Configuration des paramètres utilisateur.

Lorsque vous installez une langue, il peut s'avérer nécessaire de mettre à jour la façon dont les dates et les heures s'affichent pour correspondre à l'affichage standard de cette langue. Par exemple, le format de date par défaut suit le modèle mois/jour/année, mais vous pouvez le modifier en année/mois/jour. Vous pouvez spécifier le format pour chaque langue installée. Pour en savoir plus, voir Définition des formats de date et d'heure.

Mise à niveau de lots de langues

Les lots de langues Content Server peuvent être mis à niveau. OpenText propose des lots de langues mis à jour, pour chacune des langues fournies avec Content Server, avec chaque mise à jour de Content Server.

Il existe deux manières de mettre vos lots de langues à niveau :

  1. Lorsque vous mettez Content Server à niveau vers le correctif ou le Service pack le plus récent, vous pouvez également télécharger les lots de langues les plus récents.

  2. Vous pouvez également télécharger les lots de langues les plus récents depuis la Knowledge Center OpenText et installer ces lots de langues mis à jour sur votre système Content Server.

Lorsque vous avez mis à jour le lot de langue que vous souhaitez installer, vous devez le décompresser dans le répertoire adéquat :

  1. Si vous installez un lot de langue mis à jour sur le système Content Server, décompressez le lot de langue mis à jour dans le répertoire d'installation racine de Content Server.

    Par exemple, si vous avez installé Content Server sur le répertoire C:\OPENTEXT, décompressez le lot de langue mis à jour que vous avez téléchargé dans le répertoire C:\OPENTEXT. Le lot de langue mis à jour est décompressé dans C:\OPENTEXT\langpkgstaging.

  2. Si vous installez un lot de langue mis à jour sur un module Content Server, décompressez le lot de langue mis à jour dans le répertoire C:\<Content Server_home>\langpkgstaging\module, où C:\<Content Server_home> correspond à l'emplacement de votre installation Content Server.

 

Important :
Comme les clients peuvent télécharger et installer des lots de langues complets avec chaque mise à jour de Content Server, chaque lot de langue complet remplace le lot de langue existant qui correspond, à condition que la version du lot de langue que vous installez soit supérieure ou égale au lot de langue actuellement installé.

Content Server génère un message d'erreur si le lot de langue mis à jour que vous essayez d'installer est une version antérieure au lot de langue actuellement installé.

 

Pour en savoir plus, voir Pour mettre à jour un lot de langue pour Content Server.

Désactivation de lots de langues

Les lots de langues ne peuvent pas être désinstallés, mais ils peuvent être désactivés. Pour plus d'informations sur la désactivation d'un lot de langue, voir Configuration des langues.
