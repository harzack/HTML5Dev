La section [XML] contient les paramètres de configuration pour l'exportation XML. Par défaut, cette section apparaît dans le fichier opentext.ini lorsque Content Server est installé. Veuillez contacter le service d'assistance clientèle d'OpenText avant d'apporter des modifications à cette section.

Cette page contient des informations sur les paramètres suivants :

•

Note :
Un paramètre supplémentaire, LogPath, peut être ajouté dans la section [XML] pour acheminer la journalisation de l'activité d'importation et d'exportation XML vers son propre répertoire. Par défaut, les fichiers journaux XML sont stockés avec d'autres fichiers journaux Content Server. Pour en savoir plus, voir Logpath. Si votre système Content Server utilise beaucoup l'importation et l'exportation XML, vous pouvez ajouter la ligne LogPath=<chemin_répertoire_complet> à la section [XML], où <chemin_répertoire_complet> est le chemin du répertoire que vous spécifiez.

Veuillez contacter le service d'assistance clientèle d'OpenText avant d'apporter toute modification à cette section.

 

Encoding

  • Description :

    Définit l'attribut de codage de la déclaration XML. Voici un exemple de déclaration XML : <?xml version="1.0" encoding="ISO-8859-1"?>.

  • Syntaxe :

    encoding=ISO-8859-1

  • Valeurs :

    La valeur par défaut est ISO-8859-1.

StyleSheetMimeType

  • Description :

    Définit la valeur de l'attribut du type sur le lien à la feuille de style incorporée à l'exportation XML. Voici un exemple de déclaration de feuille de style : <?xml-stylesheet type="text/xsl" href="../webadmin/.."?>. Microsoft IE5 nécessite la valeur text/xsl, qui n'est pas un type MIME formellement reconnu.

  • Syntaxe :

    StyleSheetMimeType=text/xsl

  • Valeurs :

    La valeur par défaut est text/xsl.

MaxNodesToExport

  • Description :

    Définit la limite supérieure du nombre de nœuds pouvant être exporté dans une requête d'exportation unique. Zéro, ou un nombre négatif, spécifie un nombre illimité de nœuds.

  • Syntaxe :

    MaxNodesToExport=1000

  • Valeurs :

    La valeur par défaut est 1000.

FetchSize

  • Description :

    Définit la taille de bloc selon laquelle les nœuds sont exportés et enregistrés dans le navigateur.

  • Syntaxe :

    FetchSize=20

  • Valeurs :

    La valeur par défaut est 20.

UnencodedMimetype

  • Description :

    Spécifie les types MIME qui peuvent être exportés non codés lorsque le paramètre d'entrée de contenu de la requête d'exportation XML contient la valeur cdata ou plain. Si vous ne spécifiez pas le type MIME, les valeurs cdata et plain sont ignorées.

  • Syntaxe :

    UnencodedMimetype_1=text/xml

  • Valeurs :

    Types MIME valides.
