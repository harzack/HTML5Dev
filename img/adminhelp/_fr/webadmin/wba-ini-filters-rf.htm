TypeSense

  • Description :

    Contrôle l'affichage des documents et le comportement d'extraction.

  • Syntaxe :

    TypeSense=BROWSER

  • Valeurs :

    BROWSER ou SERVER. La valeur par défaut est BROWSER.

    Si ce paramètre est défini sur BROWSER, l'ouverture d'un document oblige Content Server à se reporter à la section ViewableMimeTypes. Si le type MIME du document s'affiche dans cette section, Content Server exécute une extraction au lieu de tenter d'ouvrir le document. Si ce paramètre est défini sur SERVER, l'ouverture d'un document oblige Content Server à rechercher la liste INSOViewableMIMETypes dans la section [filters]. Si le type MIME du document s'affiche dans cette section, Content Server ouvre le document au lieu d'exécuter une extraction.

autoRecMimeTypes

  • Description :

    Envoie à Content Server une instruction d'exécution de la fonction de reconnaissance automatique des fichiers associés aux types MIME spécifiés.

  • Syntaxe :

    autoRecMimeTypes=application/octet-stream

  • Valeurs :

    Une liste de types MIME valables, séparés par des virgules. La valeur par défaut est application/octet-stream.

cachePath

  • Description :

    Chemin d'accès au répertoire dans lequel vous souhaitez créer les sous-zones de mémoire cache d'affichage.

  • Syntaxe :

    cachePath=C:\OPENTEXT\viewcache\

  • Valeurs :

    Un chemin d'accès absolu est obligatoire. Pas de valeur par défaut.

     

    Note :
    OpenText vous recommande de partager le sous-dossier viewcache, avec des autorisations de partages totales pour vos utilisateurs.

     

eraseNo

  • Description :

    Nombre de sous-zones à créer pour les fichiers convertis.

  • Syntaxe :

    eraseNo=10

  • Valeurs :

    Un entier supérieur ou égal à un. La valeur par défaut est 10.

filterPath

  • Description :

    Emplacement du répertoire contenant les filtres d'affichage des documents.

  • Syntaxe :

    filterPath=C:\OPENTEXT\filters\

  • Valeurs :

    Un chemin absolu. La valeur par défaut est <Content Server_home> \filters\, où <Content Server_accueil> est la racine de votre installation Content Server.

lastIdFile

  • Description :

    Lorsque les fichiers sont convertis au cours d'une fonction d'affichage, Content Server les place dans une sous-zone numérotée de la mémoire cache dont le numéro d'identification correspond au paramètre lastIdFile.

  • Syntaxe :

    lastIdFile=C:\OPENTEXT\viewcache\lastid.dat

  • Valeurs :

    Un chemin absolu et un nom de fichier. Le nom du fichier par défaut est lastid.dat.

logfile

  • Description :

    Pour envoyer à Content Server une instruction d'enregistrement de l'activité des filtres dans un journal, ajoutez l'entrée logfile dans la section [filters]. Ce paramètre active l'enregistrement des filtres et place les données enregistrées dans le fichier que vous spécifiez.

  • Syntaxe :

    logfile=C:\filters\<nom_fichier_journal>

  • Valeurs :

    Un chemin absolu et un nom de fichier. OpenText recommande :

    <Content Server_home>\filters\< logfile_name>, où <Content Server_accueil> est la racine de votre installation Content Server et <nom_fichier_journal> est le nom de votre fichier journal.

relativeLinkMimeTypes

  • Description :

    Lors de l'ouverture de documents ayant un type MIME figurant dans ce paramètre, Content Server convertit les liens relatifs du document vers d'autres fichiers pour qu'un simple clic suffise à atteindre l'élément approprié, l'élément référencé devant être enregistré dans Content Server.

  • Syntaxe :

    relativeLinkMimeTypes=text/html,application/pdf

  • Valeurs :

    Une liste de types MIME, séparés par des virgules. La valeur par défaut est : relativeLinkMimeTypes=text/html,application/pdf
