La section [OTAdmin] contient les options de connexion, de journalisation et de délai d'attente du serveur d'indexation du serveur d'administration.

Cette page contient des informations sur les paramètres suivants :

•

logfile

•

loginput
•

loglevel

•

logoutput
•

port

logfile

  • Description :

    Indique le chemin et le fichier où les activités du serveur d'administration sont consignées.

     

    Important :
    OpenText vous recommande de ne pas modifier la valeur de ce paramètre.

     

    Si vous décidez tout de même de la modifier, OpenText vous recommande d'utiliser la page Configurer les paramètres de débogage plutôt que de modifier directement le fichier opentext.ini. Pour en savoir plus, voir Configuration du journal du serveur.

  • Syntaxe :

    logfile=C:/OPENTEXT/logs/admserv.log

  • Valeurs :

    Un chemin absolu et un nom de fichier. Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui signifie que le fichier journal s'appelle admserv.log et qu'il est stocké dans le répertoire <Content Server_home>/logs, où <Content Server_accueil> est la racine de votre installation Content Server.

loginput

  • Description :

    Indique le fichier journal dans lequel le serveur d'administration enregistre toutes les demandes d'informations entrantes du serveur.

     

    Important :
    OpenText vous recommande de ne pas modifier la valeur de ce paramètre.

     

    Si vous décidez tout de même de la modifier, OpenText vous recommande d'utiliser la page Configurer les paramètres de débogage plutôt que de modifier directement le fichier opentext.ini. Pour en savoir plus, voir Configuration du journal du serveur.

  • Valeurs :

    Un chemin absolu et un nom de fichier. Par défaut, ce paramètre ne figure pas dans le fichier opentext.ini. Il ne se produit donc aucune journalisation de connexion entrante.

loglevel

  • Description :

    Indique quelles activités du serveur d'administration sont consignées dans le journal.

     

    Important :
    OpenText vous recommande de ne pas modifier la valeur de ce paramètre.

     

    Si vous décidez tout de même de la modifier, OpenText vous recommande d'utiliser la page Configurer les paramètres de débogage plutôt que de modifier directement le fichier opentext.ini. Pour en savoir plus, voir Configuration du journal du serveur.

  • Syntaxe :

    loglevel=1

  • Valeurs :

    Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut au paramètre loglevel=1.

    Valeur Description
    1 Désigne le niveau de consignation par défaut. Les problèmes graves ainsi que les activités de démarrage et d'arrêt du système sont consignés.
    2 Désigne le niveau de consignation détaillé. Les messages d'erreur relatifs aux processus enfants et au statut du serveur sont consignés.
    3 Désigne le niveau du journal de débogage qui génère un journal détaillé pour permettre le débogage.

logoutput

  • Description :

    Désigne le nom du fichier journal dans lequel le serveur d'administration enregistre toutes les informations relatives au statut qu'il envoie à Content Server.

     

    Important :
    OpenText vous recommande de ne pas modifier la valeur de ce paramètre.

     

    Si vous décidez tout de même de la modifier, Content Server vous recommande d'utiliser la page Configurer les paramètres de débogage plutôt que de modifier directement le fichier opentext.ini. Pour en savoir plus, voir Configuration du journal du serveur.

  • Valeurs :

    Un chemin absolu et un nom de fichier. Par défaut, ce paramètre ne figure pas dans le fichier opentext.ini. Il ne se produit donc aucune journalisation de connexion sortante.

port

  • Description :

    Désigne le port sur lequel le serveur d'administration accepte les demandes de connexion.

    Vous devez modifier la valeur de ce paramètre dans le fichier opentext.ini. Pour en savoir plus, voir Compréhension du fichier opentext.ini.

  • Syntaxe :

    port=5858

  • Valeurs :

    Tout port TCP/IP ouvert. La valeur par défaut est 5858.
