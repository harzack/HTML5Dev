Vous pouvez modifier les paramètres de performances suivants du serveur sur la page Configurer les paramètres de performances :

  • Seuil de droits d'utilisateur

  • Nombre de sessions

  • Threads serveur

  • Recevoir avant d'envoyer

  • Activer la mise en mémoire cache des éléments Web

  • Taille du tampon pour les fichiers

La page Configurer les paramètres de performances est accessible à partir de la page Administration. Sélectionnez Configuration du serveur. Ensuite, sélectionnez Configurer les paramètres de performances.

Seuil de droits d'utilisateur

Ce paramètre vous permet de définir la façon dont Content Server assure la vérification des autorisations en fonction du nombre de droits dont dispose un utilisateur particulier. La valeur par défaut du seuil de droits d'utilisateur est définie sur 250. Si, pour un utilisateur particulier, ce nombre de droits est égal ou inférieur au nombre spécifié (le seuil), Content Server calcule, selon sa méthode standard, les autorisations dont l'utilisateur bénéficie pour accéder à l'élément voulu. Si ce nombre dépasse le seuil, Content Server adopte une méthode plus récente qui contribue à améliorer les performances des systèmes Content Server où les utilisateurs disposent chacun d'un plus grand nombre d'autorisations.

Content Server effectue la vérification des autorisations à l'aide d'un des deux algorithmes disponibles. Un algorithme effectue la vérification des autorisations à l'aide de requêtes SQL ; l'autre se procure toutes les listes de contrôle d'accès sur un élément, puis vérifie les droits de l'utilisateur sur chaque liste de contrôle d'accès individuelle en utilisant la logique du programme. Content Server choisit entre ces deux algorithmes en fonction du nombre de droits dont dispose l'utilisateur comparé au nombre spécifié dans le champ Seuil des droits d'utilisateur de la page Configurer les paramètres du serveur.

Pour plus d'informations, voir maxRightsString dans la section [options] de la référence de fichier opentext.ini.

Nombre de sessions

Ce paramètre définit le nombre maximal de sessions de connexion de l'utilisateur mises en mémoire cache sur un thread du serveur. La valeur par défaut du nombre de sessions est définie sur 100. Lorsque le nombre maximum de sessions est atteint, la session de connexion utilisateur la plus ancienne est supprimée. Les sessions de connexion de l'utilisateur sont mises en mémoire cache séparément sur chaque thread. Lorsqu'un utilisateur retourne à un thread après la suppression de ses informations de connexion dans la mémoire cache, sa requête suivante met un peu plus de temps à s'exécuter. Plus le nombre maximal de sessions est bas, moins le serveur doit consacrer de mémoire pour suivre les sessions de connexion de l'utilisateur sur chaque thread. Toutefois, plus le nombre est élevé, moins le serveur supprime d'informations de connexion de l'utilisateur dans la mémoire cache. L'utilisation de la mémoire d'un serveur peut être importante pour un système sur lequel plusieurs threads sont exécutés. Vous pouvez essayer différentes valeurs pour le nombre maximal de sessions, selon le nombre d'utilisateurs qui accèdent à votre système Content Server.

Threads serveur

Ce paramètre définit le nombre de threads utilisés par Content Server. Le Nombre de threads par défaut est 8. Vous voudrez peut-être augmenter ou diminuer le nombre de threads, selon :

  • la vitesse d'exécution des requêtes individuelles ;

  • la capacité requise ;

  • votre profil d'utilisation ;

  • la disponibilité des ressources de processeur et de RAM sur votre matériel Content Server.

Le nombre optimal de threads dépend de l'environnement dans lequel l'instance de Content Server est exécutée (par exemple, nombre et vitesse des processeurs, quantité de mémoire physique, vitesse de la connexion réseau et accès aux disques en local ou via le réseau). Il dépend aussi du profil d'utilisation de votre instance de Content Server (par exemple, types de requêtes effectués et fréquences auxquelles chacun de ces types de requêtes a lieu).

Pour déterminer le nombre de threads que votre serveur peut prendre en charge, Open Text vous conseille de tester différentes valeurs. Vous pouvez mesurer les résultats en utilisant les journaux Content Server et les utilitaires disponibles dans le système d'exploitation. Pour plus d'informations, contactez le service d'assistance clientèle d'Open Text.

 

Note :
Ne définissez pas un nombre de threads supérieur à ce nombre.

 

Recevoir avant d'envoyer

Ce paramètre réduit le temps consacré par le serveur au téléchargement et à l'ouverture de documents. Le paramètre par défaut pour Recevoir avant d'envoyer est défini sur FALSE. Lorsque ReceiveBeforeSend=TRUE, les réponses du serveur sont mises en mémoire tampon sur le serveur Web, qui se charge ensuite d'envoyer le contenu au navigateur. Cela permet aux threads serveur disponibles de passer plus rapidement à d'autres requêtes. Du point de vue d'un utilisateur final, l'extraction et le téléchargement des documents prennent autant de temps.

Activer la mise en mémoire cache des éléments Web

Ce paramètre permet au serveur Web de mettre en mémoire cache les documents extraits par Content Server. Le paramètre par défaut pour Activer la mise en mémoire cache des éléments Web est défini sur FALSE. Lorsque la mise en mémoire cache des éléments Web est activée, le serveur valide les documents. Si le processus de validation confirme que la copie en cours du document est en mémoire cache, le document sera transmis par le cache Web au lieu du serveur.

 

Note :
Dans Content Server, la mise en mémoire cache des éléments Web n'est utilisée que dans le cas de requêtes de documents à télécharger ou extraire.

 

Taille du tampon pour les fichiers

Ce paramètre vous permet de configurer la taille du tampon pour les fichiers lors de la copie d'éléments sur et à partir du magasin de fichiers externe. La taille du tampon pour les fichiers doit être saisie en octets et la valeur doit être comprise entre 16 384 (16 Ko) et 2 097 152 (2 Mo). Par défaut, la taille du tampon pour les fichiers utilisée pour copier des éléments est 524 288 (500 Ko).
