Affichage de l'utilisation de l'espace de stockage (Oracle)

Le serveur Oracle utilise une entité appelée espace de stockage pour stocker les tables d'une ou plusieurs bases de données relationnelles. Si votre SGBDR est un serveur Oracle, vous pouvez afficher la quantité d'espace utilisé par les tables Content Server dans l'espace de stockage Oracle.

Sur la page Maintenance de la base de données active, cliquez sur le lien Afficher l'utilisation de Espace de stockage Content Server.

Elimination des données supprimées (Oracle)

Si votre SGBDR est un serveur Oracle et si la base de données Content Server utilise le stockage de documents interne, la page Maintenance de la base de données active affiche le lien Eliminer les données supprimées.

Lorsque les utilisateurs suppriment des documents de la base Content Server, Oracle marque les données comme étant supprimées, bien que celles-ci occupent toujours l'espace disque. Il est donc recommandé de purger les données supprimées à intervalles réguliers pour libérer de l'espace disque.

Le lien Eliminer les données supprimées affiche la page correspondante sur laquelle vous effectuez la suppression.

Si votre SGBDR est un serveur Oracle et que votre base de données Content Server stocke les documents en interne, vous pouvez utiliser la page Eliminer les données supprimées pour supprimer de la table des données BLOB les données ayant le statut supprimé. Vous accédez à la page correspondante en cliquant sur le lien Eliminer les données supprimées sur la page Maintenance de la base de données active.

Lorsqu'un utilisateur Content Server supprime un document stocké en interne dans une base de données Content Server, Oracle ne supprime pas le document du support de stockage, mais il lui attribue le statut supprimé. Par conséquent, le document continue à occuper de l'espace sur le disque, même si les utilisateurs Content Server n'y ont plus accès. Il est donc recommandé de purger les données supprimées de la base de données Content Server à intervalles réguliers pour libérer de l'espace disque.

Pour éliminer des données supprimées, vous devez avoir suffisamment de segments rollback pour éliminer la table de données BLOB. Pour plus d'informations sur l'opération de rollback, reportez-vous à la documentation Oracle.

Visualisation des tables Content Server

Une base de données Content Server enregistre les données dans un ensemble de tables SGBDR. Vous pouvez afficher la liste de ces tables et le nombre de lignes que contient chacune d'entre elles.

La page Utilisation des espaces de stockage Content Server affiche des informations sur l'espace dans lequel réside la base de données Content Server courante, y compris l'espace utilisé.

Pour le serveur Oracle, la page Afficher les tables Content Server affiche les informations suivantes sur les tables de la base Content Server :

  • le nom des tables ;

  • l'espace de stockage Oracle qui contient la table.

  • le nombre de lignes dans la table ;

Pour Microsoft SQL Server, la page Afficher les tables Content Server affiche les informations suivantes sur les tables de la base Content Server :

  • le nom de la table ;

  • le nombre de lignes dans la table ;

  • le type d'objet du serveur SQL ;

  • l'utilisateur de Sybase ou Microsoft SQL Server qui est propriétaire de la table.

Le serveur Oracle utilise une entité appelée espace de stockage pour stocker les tables d'une ou plusieurs bases de données relationnelles. Un espace de stockage peut avoir plusieurs utilisateurs dont chacun possède un ensemble de tables. En général, l'ensemble de tables correspondant à une base de données Content Server réside dans un espace unique, qui peut être partagé avec d'autres ensembles de tables appartenant à des utilisateurs de base de données différents.
