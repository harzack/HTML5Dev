La section [Summarizer] du fichier opentext.ini définit un aspect du service de conversion de documents, DCS. Ce service convertit des documents au format HTML ou texte brut pour permettre leur affichage et indexation. Les DCS sont gérés par les serveurs d'administration. Les services de conversion gérés par le DCS partagent les paramètres dans le fichier opentext.ini. Ces paramètres définissent le comportement des services de conversion. La section [Summarizer], qui fait partie du DCS, est activée par défaut.

Le résumeur Content Server utilise les fonctions de recherche de Content Server pour fournir des résumés de document générés automatiquement. Si le résumeur est activé et si vos options d'affichage dans Content Server sont définies de manière à afficher les résumés, Content Server affiche les résumés de document sur la page Résultat de recherche, une fois que vous avez effectué une recherche. Le résumeur génère des résumés en compilant des phrases d'après leur emplacement dans le document, les structures voisines et la signification statistique des mots dans les phrases. Ces phrases sont identifiées par rapport au fichier de fréquence d'occurrence des mots, qui est un fichier de configuration identifiant la fréquence statistique de mots issus d'un grand nombre de documents.

Le résumeur génère également les locutions clés associées à un document. Les locutions clés sont des locutions issues du document et représentatives du contenu. Par défaut, le résumeur identifie cinq locutions clés en fonction de plusieurs facteurs, notamment la répétition et l'emplacement des locutions dans le document. Si le document ne comporte pas cinq locutions clés appropriées, le résumeur génère autant de locutions clés que possible. Les locutions clés s'affichent également sur la page Résultat de recherche une fois que vous avez effectué une recherche, à condition que les options d'affichage dans Content Server soient définies pour afficher les locutions clés et que le résumeur soit activé.

Le résumeur commence par marquer les fichiers source au moyen de jetons, afin d'identifier les mots, les abréviations, etc. Après avoir défini les éléments dans le fichier source, le résumeur identifie les séries de jetons qui constituent des phrases lisibles. Il analyse également la structure du fichier source afin d'identifier les régions les plus susceptibles de contenir des données de résumé valables. Le résumeur classe ensuite les phrases selon l'importance des mots dans le document et le fichier de fréquence d'occurrence des mots, puis génère le résumé. Si le résumeur ne parvient pas à résumer un fichier source, en raison de la structure de ce dernier ou de la lisibilité des phrases, le résumé se composera des dix premiers mots du fichier. Si le fichier source ne contient aucune unité lexicale, le résumeur ne peut pas générer de résumé ou de locutions clés.

OpenText a conçu et configuré le résumeur afin qu'il traite les documents en anglais, mais vous pouvez le configurer pour obtenir des résumés dans d'autres langues. Si vous souhaitez personnaliser le résumeur, voici quelques informations importantes que vous devez savoir :

  • Le résumeur peut résumer des documents en japonais, en français et en allemand, mais il a été testé pour l'anglais uniquement.

  • Le résumeur traite la plupart des signes de ponctuation de fin de phrase des langues autres que l'anglais.

  • Les documents en langue à codage multioctets peuvent être résumés. La segmentation est basée sur Unicode.

  • Des problèmes peuvent survenir si vous effectuez des résumés dans des langues nécessitant une segmentation basée sur un dictionnaire.

SumAbbrevFile

  • Description :

    Détermine le chemin relatif et le nom du fichier d'abréviations utilisé par le résumeur pour définir les abréviations courantes et toutes les combinaisons de trois lettres ne constituant pas un mot.

  • Valeurs :

    Un chemin et un nom de fichier relatifs à l'emplacement du DCS sur votre installation Content Server. Par défaut, le fichier d'abréviations s'appelle abbrev.eng et il est enregistré dans le répertoire config de votre installation Content Server.

SumDefFile

  • Description :

    Détermine le chemin relatif et le nom du fichier de définitions que le résumeur utilise pour définir son fonctionnement. Le fichier de définition contient cinq nombres (un par ligne) représentant les éléments suivants :

    • Un multiplieur de taux pour les phrases situées dans les 20 premiers pour cent d'un document. Il s'agit probablement de phrases d'introduction, idéales pour constituer un résumé.

    • Un nombre maximum d'unités lexicales autorisées dans une phrase de résumé. Une unité lexicale est une combinaison de lettres, de nombres, de traits et de références d'entité. Le résumeur identifie comme illisibles les phrases qui contiennent plus d'unités lexicales que la limite autorisée. Ces phrases ne seront pas utilisées pour constituer un résumé.

    • Un nombre minimum d'unités lexicales autorisées dans une phrase de résumé. Le résumeur identifie comme illisibles les phrases qui contiennent moins d'unités lexicales que le nombre minimum requis. Ces phrases ne seront pas utilisées pour constituer un résumé.

    • Un taux maximum d'unités non lexicales par rapport aux unités lexicales. Si le taux actuel d'unités non lexicales par rapport aux unités lexicales est supérieur à ce nombre, le résumeur identifie la phrase comme illisible. Cette phrase ne sera pas utilisée pour constituer un résumé.

    • Le nombre de documents utilisés pour former les données relatives à la signification statistique des mots dans le fichier de fréquence d'occurrence des mots.

  • Valeurs :

    Un chemin et un nom de fichier relatifs à l'emplacement du DCS sur votre installation Content Server. Par défaut, le fichier de définitions s'appelle summdef.eng et il est enregistré dans le répertoire config de votre installation Content Server.

SumDocFreqFile

  • Description :

    Détermine le chemin relatif et le nom du fichier de fréquence d'occurrence des mots contenant les données nécessaires au résumeur pour calculer la signification statistique des mots dans les documents. Ce fichier contient une liste de mots présents dans plus de 1 000 documents sur les 1 371 876 utilisés par OpenText pour élaborer l'arrière-plan statistique des paramètres par défaut du résumeur.

  • Valeurs :

    Un chemin et un nom de fichier relatifs à l'emplacement du DCS sur votre installation Content Server. Par défaut, le fichier de fréquence d'occurrence des mots s'appelle docfreq.eng et il est enregistré dans le répertoire config de votre installation Content Server.

SumStopWordFile

  • Description :

    Détermine le chemin relatif et le nom du fichier de mots vides que le résumeur utilise pour définir une liste de mots vides courants. Les mots vides sont des mots n'ajoutant aucune valeur sémantique à une phrase. Il s'agit essentiellement de mots tels que « un(e) », « et » et « le/la ». En différenciant les mots vides et les mots ayant une importance sémantique, le résumeur identifie les mots susceptibles de contribuer à l'intérêt du document, en augmentant la pertinence de ses résumés et locutions clés.

  • Valeurs :

    Un chemin et un nom de fichier relatifs à l'emplacement du DCS sur votre installation Content Server. Par défaut, le fichier des mots vides s'appelle stopword.eng et il est enregistré dans le répertoire config de votre installation Content Server.

SumTagFile

  • Description :

    Désigne le chemin relatif et le nom du fichier de balises contenant une liste des balises de marquage présentes dans les documents. Par exemple, les balises de marquage HTML, XML et SGML. Un chiffre indiquant l'importance de la balise pour le résumeur se trouve en regard de chaque balise. Le tableau suivant présente la signification de chaque chiffre.

    Tableau décrivant la plage des paramètres de signification des balises :

    Numéro Description
    0 Le résumeur ignore ces balises, mais pas les données qu'elles contiennent. Si une phrase débute avant cette balise et la contient, le résumeur ne sépare pas la phrase en deux phrases distinctes. Par défaut, les balises inconnues sont identifiées par ce chiffre.
    1 Le résumeur ignore ces balises, mais pas les données qu'elles contiennent. Si une phrase débute avant cette balise et la contient, le résumeur sépare la phrase en deux phrases distinctes.
    2 Le résumeur ignore ces balises et les données qu'elles contiennent.
    3 Cette valeur marque comme phrases de résumé (de vue d'ensemble) les données contenues dans ces balises. Pour générer le résumé, le résumeur utilise les phrases du titre, du résumé et de la conclusion, dans cet ordre.
    4 Cette valeur identifie comme phrases de conclusion les données contenues dans ces balises. Pour générer le résumé, le résumeur utilise les phrases du titre, du résumé et de la conclusion, dans cet ordre.
    5 Cette valeur identifie comme phrases de titre les données contenues dans ces balises. Pour générer le résumé, le résumeur utilise les phrases du titre, du résumé et de la conclusion, dans cet ordre.

  • Valeurs :

    Un chemin et un nom de fichier relatifs à l'emplacement du DCS sur votre installation Content Server. Par défaut, le fichier des balises s'appelle tags.eng et il est enregistré dans le répertoire config de votre installation Content Server.

lib

  • Description :

    Détermine le nom de la bibliothèque que le DCS doit charger. Une bibliothèque est une liste d'opérations associées à un filtre de conversion que le DCS lit pour effectuer la conversion.

    La modification de la valeur de ce paramètre risque d'empêcher le DCS de fonctionner ou de fonctionner correctement.

     

    Important :
    Ne modifiez pas la valeur de ce paramètre, sauf si le service d'assistance clientèle d'OpenText vous le demande.

     

  • Valeurs :

    La valeur par défaut est dcssum.

récapitulatif

  • Description :

    Indique si le DCS génère des résumés des documents qu'il convertit au format HTML. Si l'utilisateur définit les options d'affichage de recherche pour afficher les résumés, Content Server affiche ces résumés accompagnés des résultats de la recherche sur la page Résultat de recherche.

  • Valeurs :

    TRUE ou FALSE ; la valeur TRUE demande au DCS de générer des résumés. Par défaut, sur la ligne de commande de chaque DCS, le paramètre summary est défini sur TRUE.

    Pour définir le paramètre summary sur FALSE, vous devez modifier Arguments de ligne de commande plutôt que le paramètre du fichier opentext.ini, car les arguments définis sur la ligne de commande priment sur les paramètres généraux définis dans le fichier opentext.ini.

summaryhotwords

  • Description :

    Indique le nombre de liens hypertexte utilisés par le résumeur pour générer le résumé d'un document. En réduisant cette valeur, vous pouvez accélérer la création des résumés mais beaucoup d'autres variables affectent également la vitesse de ce processus.

     

    Important :
    OpenText vous recommande vivement de ne pas modifier la valeur de ce paramètre.

     

  • Valeurs :

    Un entier supérieur ou égal à un. La valeur par défaut est 20.

summarysentences

  • Description :

    Indique le nombre de phrases à générer dans les résumés.

  • Syntaxe :

    summarysentences=5

  • Valeurs :

    Un entier supérieur ou égal à un. Par défaut, ce paramètre n'apparaît pas dans la section [Summarizer] du fichier opentext.ini, ce qui équivaut à summarysentences=5.

summarysize

  • Description :

    Détermine la portion du document converti, en octets, utilisée par Content Server pour générer un résumé. Une valeur peu élevée limite le résumé aux <n> premiers octets du document. La modification de cette valeur risque également d'affecter les performances de conversion des documents plus importants.

     

    Important :
    OpenText vous recommande vivement de ne pas modifier la valeur de ce paramètre.

     

  • Valeurs :

    Tout entier supérieur à un. La valeur par défaut est 256000 octets.

summarytitlesize

  • Description :

    Indique le nombre de caractères à afficher dans les titres des résultats de recherche pour certains documents. Ces documents proviennent de sources de données autres que la source de données Content Server Entreprise (sources de données Directory Walker ou Spider, par exemple).

  • Syntaxe :

    summarytitlesize=30

  • Valeurs :

    Un entier. La valeur par défaut est 30.

x-maxcalls

  • Description :

    Indique le nombre de réutilisations d'un processus pour le traitement des documents. Pendant la conversion de documents, le DCS charge un processus de traitement. Le processus de traitement charge le filtre de conversion approprié et l'utilise pour convertir le document. Pour améliorer les performances, le DCS réutilise le processus de traitement pour plusieurs conversions. Si le processus de traitement rencontre une erreur, il s'interrompt avant d'atteindre la valeur définie dans le fichier opentext.ini.

     

    Important :
    OpenText vous recommande vivement de ne pas modifier la valeur de ce paramètre.

     

  • Valeurs :

    La valeur par défaut est héritée du paramètre maxcalls de la section [DCSworker] du fichier opentext.ini.

     

    Note :
    Le paramètre x-maxcalls est actif uniquement lorsque le filtre de conversion qu'il modifie est géré par un processus de traitement. Pour plus d'informations sur la configuration d'un processus de traitement, reportez-vous à la rubrique Configuration du journal du serveur.

    Si vous définissez une valeur pour x-maxcalls, la valeur de maxcalls sera écrasée dans la section [DCSworker] du fichier opentext.ini.

     

x-timeout

  • Description :

    Indique le temps d'attente maximum (en secondes) avant la fin d'un processus de traitement de conversion de documents. Configurez ce paramètre lorsque la valeur par défaut définie dans le paramètre timeout n'est pas valable. Configurez le paramètre timeout de la page opentext.ini pour chaque filtre de conversion. Par exemple, certains filtres de conversion sont plus lents que d'autres. Dans ce cas, la valeur timeout par défaut de 30 secondes peut ne pas être applicable, car le délai de conversion moyen est supérieur à cette valeur. Dans ce cas, vous pouvez modifier la valeur x-timeout en choisissant une valeur plus élevée.

     

    Important :
    OpenText vous recommande vivement de ne pas modifier la valeur de ce paramètre.

     

  • Valeurs :

    La valeur par défaut est héritée du paramètre timeout de la section [DCSworker] du fichier opentext.ini.

     

    Note :
    Le paramètre x-timeout est actif uniquement lorsque le filtre de conversion qu'il modifie est géré par un processus de traitement. Pour plus d'informations sur la configuration d'un processus de traitement, reportez-vous à la rubrique Configuration du journal du serveur.

    Si vous définissez une valeur pour x-timeout, la valeur de timeout sera écrasée dans la section [DCSworker] du fichier opentext.ini.
