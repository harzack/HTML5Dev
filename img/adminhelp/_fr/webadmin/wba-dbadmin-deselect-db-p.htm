Pour désélectionner la base de données Content Server active :

  1. Dans la section Administration de la base de données de la page Administration Content Server, cliquez sur le lien Modifier la base de données active.

  2. Facultatif Sur la page Désélectionner la base de données Content Server, par défaut, désélectionner la base de données active supprime les index répertoriés dans le tableau. Pour empêcher la suppression d'un index, dans la colonne Supprimer, désactivez la case à cocher en regard de cet index.

  3. Cliquez sur Continuer pour poursuivre la désélection de la base de données active.

     

    Important :
    Avant de poursuivre et de vous connecter à une autre base de données, vous devez :

    1. arrêter tous les serveurs d'administration ;

    2. enlever les fichiers search.ini et otadmin.cfg du répertoire <Content Server_home>/config de chaque serveur d'administration. Cela est nécessaire afin d'empêcher le déplacement des processus orphelins vers une autre base de données.

    3. démarrer chaque serveur d'administration.

    4. Après la reconnexion, chaque serveur d'administration doit être resynchronisé pour créer de nouveaux fichiers search.ini et otadmin.cfg.

     
 

Note :
Après avoir désélectionné la base de données, vous pouvez en créer une nouvelle ou sélectionner une base existante en remplacement. Pour en savoir plus, voir Modification de la base de données active.
