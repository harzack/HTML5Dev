Pour configurer les paramètres de sécurité :

  1. Dans la section Configuration du serveur de la page Administration Content Server, cliquez sur le lien Configurer les paramètres de sécurité.

  2. Sur la page Configurer les paramètres de sécurité, dans le champ Cookies HTTP uniquement, cliquez sur l'un des boutons suivants :

    • Activer, pour inclure l'attribut httpOnly lors de la définition des cookies d'authentification.

    • Désactiver, pour exclure l'attribut httpOnly des cookies d'authentification.

  3. Dans le champ Clé de codage des cookies, saisissez une clé de codage (une chaîne de texte contenant plus d'un caractère).

  4. Saisissez une clé de codage (une chaîne de texte contenant plus d'un caractère) dans le champ Clé de codage des données.

     

    Avertissement
    Même si Content Server utilise une clé par défaut pour le codage, lorsque les champs Clé de codage des cookies et Clé de codage des données sont vierges, OpenText vous recommande vivement de saisir des clés uniques.

     

  5. Dans la zone Informations relatives à l'authentification des cookies, procédez de la façon suivante :

    • Pour utiliser l'adresse IP client comme cookie d'authentification, cliquez sur une valeur de la liste Adresse IP client représentant la portion de l'adresse IP client à comparer.

    • Activez l'une des cases à cocher suivantes pour déterminer les attributs de l'utilisateur à inclure dans le cookie d'authentification :

      • ID du propriétaire

      • Date d'expiration du mot de passe

      • Mot de passe codé dans un sens

    • Cliquez sur l'un des boutons radio suivants, puis saisissez un entier pour le nombre de minutes, le cas échéant, pour gérer l'intervalle d'expiration du cookie :

      • Ne jamais faire expirer

      • Expire <nombre_de_minutes> minutes après la dernière requête

      • Expire <nombre_de_minutes> minutes après la dernière connexion

  6. Dans le champ Date d'expiration des cookies de connexion, sélectionnez l'un des boutons radio suivants et, le cas échéant, saisissez un entier pour le nombre de jours, de façon à spécifier la durée de validité de l'authentification actuelle des cookies :

    • Ne jamais faire expirer

    • Expire dans <nombre_de_jours> jour(s)

  7. Dans la zone Stratégies de connexion :

    • Sélectionnez la case à cocher Désactiver les sessions simultanées depuis plusieurs ordinateurs pour désactiver une session utilisateur si l'utilisateur se connecte à Content Server sur plusieurs ordinateurs.

      Spécifiez la liste des ordinateurs auxquels cette fonction ne s'applique pas dans le champ Sauf pour les hôtes.

    • Sélectionnez la case à cocher Autoriser la connexion par requête HTTP pour permettre aux utilisateurs de se connecter grâce à la requête HTTP. Cette option est désactivée par défaut.

  8. Dans la zone Nouvelles tentatives de saisie du mot de passe :

    • Pour Utilisateur :

      • Sélectionnez la case à cocher Désactiver la connexion en cas de saisie d'un mot de passe incorrect, pour spécifier les actions à effectuer lors de l'échec de la connexion d'un utilisateur.

      • Spécifiez le Nombre de tentatives de connexion autorisées avant que le compte ne soit désactivé.

      • Sélectionnez la case à cocher Envoyer un e-mail à l'administrateur lorsque la connexion est désactivée pour que l'administrateur Content Server reçoive un e-mail dès que le nombre spécifié de tentatives de connexion est dépassé.

    • Pour l'Administrateur Web :

      • Sélectionnez la case à cocher Désactiver la connexion en cas de saisie d'un mot de passe incorrect, pour spécifier les actions à effectuer lors de l'échec de la connexion d'un utilisateur.

      • Spécifiez le Nombre de tentatives de connexion autorisées avant que le compte ne soit désactivé.

      • Spécifiez le Nombre de minutes de désactivation de la connexion avant que l'administrateur Web puisse tenter de se connecter à nouveau.

      • Sélectionnez la case à cocher Envoyer un e-mail à l'administrateur lorsque la connexion est désactivée pour que l'administrateur Content Server reçoive un e-mail dès que le nombre spécifié de tentatives de connexion est dépassé.

    • Pour l'Admin :

      • Sélectionnez la case à cocher Envoyer un e-mail à l'administrateur lorsque la connexion est désactivée pour que l'administrateur Content Server reçoive un e-mail dès que le nombre spécifié de tentatives de connexion est dépassé.

      • Spécifiez le Maximum d'échecs de saisie du mot de passe pour le nombre de fois où un administrateur peut tenter de se connecter à Content Server avant l'envoi d'un e-mail.

  9. Dans la zone Configuration de la page Connexion :

    • Activez la case à cocher Désactiver l'exécution automatique du navigateur pour demander aux navigateurs Web client de ne pas enregistrer ou exécuter de manière automatique les informations relatives à l'identification de l'utilisateur pour Content Server.

    • Entrez un entier non nul dans le champ Nombre de jours d'utilisation du mot de passe pour permettre aux utilisateurs d'enregistrer leur cookie d'authentification pendant un nombre de jours défini.

  10. Dans la zone Exiger le filtrage des arguments, procédez comme suit pour activer le filtrage des arguments de requête :

    • Dans le champ Filtrer la chaîne, entrez les chaînes que vous souhaitez exclure d'une requête Content Server. Par exemple, pour éviter qu'un script ne soit enregistré dans Content Server en tant que document texte, saisissez : <Script>

       

      Note :
      Dans le filtre, évitez d'utiliser un caractère URL courant, tel qu'un point (.) ou une barre oblique (/).

       

    • Dans le champ Caractère séparateur, saisissez le caractère à traiter comme séparateur entre les différents éléments de la liste de filtres, et non comme un élément à part entière.

  11. Dans le champ Affichage de la taille du recueil, activez la case à cocher Masquer le nombre d'éléments pour empêcher l'affichage du nombre d'éléments dans un conteneur.

  12. Dans le champ Expiration du jeton de requête sécurisée, cliquez sur l'un des boutons suivants :

    • Jamais de délai d'attente

    • Délai d'attente <nombre_de_secondes> secondes après la requête précédente, puis saisissez un nombre entier de secondes avant expiration.

  13. Dans le champ Hôtes client Content Server, spécifiez les serveurs à partir desquels les requêtes client doivent être acceptées. Plusieurs adresses IP doivent être séparées par des virgules. Les requêtes provenant de serveurs non répertoriés dans cette liste sont rejetées.

     

    Note :
    La liste d'hôtes client peut également être utilisée par les autres modules Content Server. Ne supprimez pas et ne modifiez pas les adresses IP existantes susceptibles d'apparaître dans ce champ, vous risqueriez d'endommager gravement votre système.

     

  14. Dans le champ Sites Web de référence sécurisés, entrez les adresses Web HTTP que vous voulez autoriser. Si vous ajoutez plusieurs adresses Web HTTP, assurez-vous que chaque adresse apparaît sur une ligne séparée.

  15. Effectuez les opérations suivantes dans la zone Fonctions des documents :

    • Cliquez sur le bouton Activé de la fonction Ouvrir pour permettre aux utilisateurs d'ouvrir les documents.

    • Cliquez sur le bouton Activé de la fonction Afficher en tant que page Web pour permettre aux utilisateurs d'afficher les documents en tant que pages Web.

  16. Dans le champ Inter-domaines sécurisés, saisissez la paire <clé> ;<cible> où <clé> correspond à une balise alphanumérique sensible à la casse utilisée pour enregistrer l'application Web tierce et <cible> est un chemin URL enregistré vers la ressource cible.

  17. Cliquez sur Enregistrer les modifications.

  18. Redémarrez les serveurs et le serveur d'application Web, puis cliquez sur le lien Continuer de la page Redémarrer le serveur.
