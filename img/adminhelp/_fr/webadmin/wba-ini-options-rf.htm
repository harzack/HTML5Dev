EFSCopyBufferSize

  • Description :

    Définit la taille du tampon des fichiers pour les fichiers copiés vers et à partir du stockage de fichier externe. Pour définir ce paramètre, OpenText vous recommande d'utiliser la page Configurer les paramètres de performances, plutôt que de modifier directement le fichier opentext.ini. Pour en savoir plus, voir Configuration des paramètres de performances.

  • Syntaxe :

    EFSCopyBufferSize=<valeur en octets>

  • Valeurs :

    Une valeur en octets, entre 16 384 et 2 097 152. La valeur par défaut est 524288 (500 Ko).

EnableNotification

  • Description :

    Active ou désactive la notification Content Server. Pour définir ce paramètre, OpenText vous recommande d'utiliser la page Configurer la notification, plutôt que de modifier directement le fichier opentext.ini. Pour en savoir plus, voir Activation et désactivation des notifications.

  • Syntaxe :

    EnableNotification=FALSE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est FALSE jusqu'à ce que la notification Content Server soit activée par l'administrateur.

excludeNodeIDs

  • Description :

    Vous permet de spécifier les ID de nœud à exclure du traitement lors de l'ajout de documents à Content Server.

  • Syntaxe :

    excludeNodeIDs=<ID_nœud1>,<ID_nœud 2>.

  • Valeurs :

    ID nœud séparés par une virgule.

Vous pouvez forcer le système à ignorer le paramètre excludeNodeIDs en définissant le paramètre processAllNodeIds sur TRUE.

maxRightsString

  • Description :

    Ce paramètre permet de spécifier laquelle des deux méthodes doit être utilisée par Content Server pour calculer les autorisations d'un utilisateur lors de la demande d'un élément particulier. Le choix est basé sur le nombre de droits détenus par l'utilisateur. Si un utilisateur qui tente d'accéder à un élément dans Content Server a un nombre de droits supérieur ou égal au nombre spécifié, le seuil, Content Server calcule les autorisations de l'utilisateur définies pour cet élément de manière différente. La méthode alternative a pour but d'améliorer les performances lorsque des utilisateurs Content Server ont un grand nombre de droits individuels.

    Pour définir ce paramètre, OpenText vous recommande d'utiliser la page Configurer les paramètres de performances, plutôt que de modifier directement le fichier opentext.ini. Pour en savoir plus, voir Configuration des paramètres de performances.

  • Syntaxe :

    maxRightsString=250

  • Valeurs :

    Un entier supérieur ou égal à zéro. La valeur par défaut est 250.

MaxOpenSessions

  • Description :

    Ce paramètre définit le nombre maximal de sessions de connexion de l'utilisateur mises en mémoire cache sur un thread du serveur.

    Pour définir ce paramètre, OpenText vous recommande d'utiliser la page Configurer les paramètres de performances, plutôt que de modifier directement le fichier opentext.ini. Pour en savoir plus, voir Configuration des paramètres de performances.

  • Syntaxe :

    MaxOpenSessions=100

  • Valeurs :

    Un entier supérieur ou égal à zéro. La valeur par défaut est 100.

processAllNodeIds

    Ajoutez ce paramètre si vous souhaitez que le système ignore le paramètre excludeNodeIDs.

  • Syntaxe :

    processAllNodeIds

  • Valeurs :

    Vrai

wantByteServing

  • Description :

    Active ou désactive l'option de téléchargement page par page des fichiers PDF.

  • Syntaxe :

    wantByteServing=FALSE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est FALSE.

wantLAPILogs

  • Description :

    Définit si Content Server enregistre les valeurs LAPI inArgs et outArgs, reçues et envoyées par le serveur, dans les journaux des threads. Pour en savoir plus, voir Suppression des événements en suspens.

  • Syntaxe :

    wantLAPILogs=FALSE

  • Valeurs :

    TRUE ou FALSE. Par défaut, le paramètre wantLAPILogs ne figure pas dans le fichier  opentext.ini, ce qui équivaut à wantLAPILogs=FALSE. Pour activer cette fonction, vous devez ajouter manuellement wantLAPILogs=TRUE dans la section [options] du fichier opentext.ini.

wantLogs

  • Description :

    Active ou désactive l'enregistrement dans les journaux de connexion. La sortie de journal est enregistrée dans des fichiers appelés connect<n>.log, où <n> est le numéro de thread. Pour en savoir plus, voir Configuration du journal du serveur.

  • Syntaxe :

    wantLogs=FALSE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est FALSE.

wantNotification

  • Description :

    Active ou désactive la notification par messagerie. Ce paramètre n'est pas utilisé dans Content Server. Il n'a pas été utilisé depuis Livelink 9.0.

wantSearch

  • Description :

    Autorise ou interdit les recherches lancées par les utilisateurs. Ce paramètre n'est pas utilisé dans Content Server. Il n'a pas été utilisé depuis Livelink 9.0.

wantSearchLogs

  • Description :

    Active ou désactive l'enregistrement des transactions de recherche dans les journaux. Les données d'entrée, ou requêtes, et les données de sortie, ou résultats, du moteur de recherche (otsearch) sont enregistrées. Les transactions sont enregistrées dans le fichier search.log dans le répertoire <Content Server_home>/logs, où <Content Server_accueil> est la racine de votre installation Content Server. Pour en savoir plus, voir Activation du journal de connexion SQL.

  • Syntaxe :

    wantSearchLogs=FALSE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est FALSE.

wantSecureCookies

  • Description :

    Si Content Server est exécuté sur un serveur HTTPS, SSL, l'indicateur de niveau de sécurité est défini pour le cookie de connexion de l'utilisateur.

  • Syntaxe :

    wantSecureCookies=TRUE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est TRUE.

wantTimings

  • Description :

    Enregistre les séquences des transactions dans les journaux des threads. Pour en savoir plus, voir Configuration du journal du serveur.

  • Syntaxe :

    wantTimings=TRUE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est TRUE.

wantVerbose

  • Description :

    Enregistre dans les journaux des threads tous les arguments envoyés par le navigateur Web au serveur. Pour en savoir plus, voir Configuration du journal du serveur.

  • Syntaxe :

    wantVerbose=TRUE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est TRUE.

wantWeb

  • Description :

    Active l'exécution de Content Server sur le Web. Si vous souhaitez utiliser Content Server uniquement via un module LAPI, vous pouvez définir le paramètre wantWeb sur la valeur FALSE.

  • Syntaxe :

    wantWeb=TRUE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est TRUE.
