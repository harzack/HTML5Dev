La navigation à facettes est l'un des outils de navigation dont disposent vos utilisateurs. Il leur permet de parcourir les éléments dans Content Server à l'aide de groupes de métadonnées, ou facettes, pour affiner progressivement leur recherche. La navigation à facettes est activée par défaut et installée avec des facettes et des catégories de documents par défaut. En tant qu'administrateur, vous devez évaluer les besoins d'utilisation de vos utilisateurs afin de déterminer comment vous allez personnaliser la navigation à facettes. Cette section explique comment configurer la navigation à facettes pour tous les utilisateurs de sorte que les facettes disponibles soient pertinentes par rapport au contenu de votre système. Elle indique comment configurer le bandeau pour tous les utilisateurs, définir vos Gestionnaires des connaissances et ajouter des catégories de documents, des facettes, des arbres de facettes et des colonnes. Pour obtenir des informations concernant la navigation à facettes, consultez la rubrique Utilisation de la navigation à facettes dans l'aide utilisateur.

Activation de la navigation à facettes

L'administrateur décide si la navigation à facettes est activée pour tous les utilisateurs. Lors de l'installation, la navigation à facettes est activée par défaut pour tous les utilisateurs. Ces derniers y accèdent à l'aide du bandeau Filtre de contenu.

Pour en savoir davantage sur l'activation et la configuration du bandeau Filtre de contenu, consultez Configuration du bandeau.

Configuration des gestionnaires des connaissances

L'administrateur doit décider qui, dans votre organisation, connaît suffisamment les contenus de données de l'organisation pour faire office de gestionnaire des connaissances. Les administrateurs sont les seuls à pouvoir définir un gestionnaire des connaissances. L'administrateur système accorde à la personne désignée comme gestionnaire des connaissances, les privilèges et les autorisations nécessaires pour accéder au Volume à facettes de sorte qu'il puisse administrer la configuration de la navigation à facettes. Votre ou vos gestionnaires des connaissances structurent le fonctionnement de la navigation à facettes dans votre organisation. Lorsque vous démarrez, il est utile de créer un groupe de gestionnaires des connaissances dans Content Server, puis d'y ajouter les utilisateurs qui auront le privilège de gestionnaire des connaissances.

Configuration des catégories de documents

Les catégories de documents sont utilisées comme sources de données à facettes. Aucune catégorie de documents ne peut être supprimée si elle est utilisée comme source de données pour une facette. Si vous tentez de supprimer une catégorie de documents qui est utilisée comme source de données pour une facette, une boîte de dialogue s'affiche pour vous informer que la catégorie de documents ne peut pas être supprimée.

Lorsque vous avez créé une catégorie de documents, vous devez la configurer. Vous trouverez ci-dessous un lien où vous pourrez lire la procédure à suivre pour créer et configurer une catégorie de documents. Pendant le processus de configuration, vous devez sélectionner tous les types MIME que vous voulez ajouter à votre nouvelle catégorie de documents dans la page Configurer le raccourci de type MIME. Ces types MIME sont hiérarchisés de la manière suivante :

  • Types MIME utilisés

    C'est une liste de types MIME qui ont déjà été ajoutés à votre système Content Server. Tout document, fichier graphique ou autre élément ayant été ajouté à un emplacement dans Content Server a un type MIME. Ce type MIME est indiqué dans cette section.

  • Raccourcis de types MIME

    Il s'agit d'une liste raccourcis de types MIME qui ont déjà été définis dans toutes les catégories de documents.

  • Types MIME connus

    Il s'agit d'une liste de tous les type MIME connus qui n'ont pas été indiqués au préalable dans la liste du champ types MIME utilisés. Si un Type MIME s'affiche dans ce champ alors il ne s'agit pas d'un type MIME d'un élément ajouté à votre système Content Server.

Configuration du format de date

Les facettes de date ont un auto-transversal avec les niveaux suivants : Date relative, Siècle, Décennie, Année, Mois et Jour. La date relative peut être hier ou aujourd'hui. Les dates correspondant au siècle, à la décennie et à l'année s'affichent sous forme d'année à quatre chiffres. Le jour s'affiche selon le format défini comme format de date court dans votre installation Content Server. La manière dont le bandeau affiche le mois et l'année peut être configurée par l'administrateur.

Par défaut, toute facette de date affichera le mois dans le format Mois Année, le mois correspondant au nom complet de ce dernier et l'année étant indiquée par quatre chiffres. Vous pouvez modifier ce format de date par défaut en modifiant la section [Lang_en_US] du fichier _home/config/opentext.ini.

Les options disponibles sont les suivantes :

  • %b

    Nom du mois abrégé avec trois caractères. Par exemple : Jan, Fév, Mar.

  • %B

    Nom de mois entier. Par exemple : Janvier, Février, Mars.

  • %m

    Mois à deux chiffres. Par exemple : 01, 02, 03.

  • %Y

    L'année, incluant le siècle. Par exemple : 1993, 2002, 2010.

  • %y

    Année à deux chiffres. Par exemple : 93, 02, 10.

Pour plus d'informations sur la modification du fichier opentext.ini, reportez-vous à la rubrique Modification du fichier opentext.ini.
