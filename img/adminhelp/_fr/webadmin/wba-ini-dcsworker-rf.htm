Les paramètres de la section [DCSworker] définissent le comportement des processus de traitement gérés par le service de conversion de documents (DCS). Le DCS convertit des documents au format HTML ou texte brut pour permettre leur affichage et indexation. Le DCS envoie les documents à un processus de traitement nommé dcsworker.exe. Ce processus de traitement délègue les opérations de conversion aux filtres de conversion appropriés après réception des requêtes du DCS. Il charge le filtre de conversion requis et procède à la conversion dans un environnement isolé. Ceci évite que des processus nuisibles n'interrompent le DCS.

La section [DCSworker] du fichier opentext.ini contient des informations sur les paramètres suivants :

•

fastinit

•

lib
•

maxcalls

•

timeout
•

workerexe

fastinit

  • Description :

    Indique que les filtres de conversion ne doivent pas être chargés tant qu'ils ne sont pas requis pour le DCS.

  • Syntaxe :

    fastinit=TRUE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est TRUE.

    La définition de ce paramètre sur TRUE demande au DCS, avant de charger le filtre de conversion, d'attendre qu'il soit requis pour le processus de conversion. La définition de ce paramètre sur TRUE entraîne des performances plus rapides.

lib

  • Description :

    Détermine le nom de la bibliothèque que le DCS doit charger. Une bibliothèque est une liste d'opérations associées à un filtre de conversion que le DCS lit pour effectuer la conversion.

     

    Attention
    La modification de la valeur de ce paramètre risque d'empêcher le DCS de fonctionner ou de fonctionner correctement. Ne modifiez pas la valeur de ce paramètre, sauf si le service d'assistance clientèle d'OpenText vous le demande.

     

  • Valeurs :

    La valeur par défaut est dcsworker.

maxcalls

  • Description :

    Indique le nombre maximum d'opérations de conversion qu'un processus de traitement doit effectuer avant d'être interrompu. Un processus de traitement s'interrompt une fois que son quota de conversion de documents a atteint la valeur définie par le paramètre maxcalls. Ce mécanisme permet d'éviter que les filtres de conversion ayant des fuites de mémoire n'utilisent trop de mémoire.

  • Syntaxe :

    maxcalls=200

  • Valeurs :

    La valeur par défaut est 200.

     

    Note :
    Ce paramètre peut être personnalisé pour chaque filtre de conversion. Il suffit pour cela de configurer le paramètre x-maxcalls dans la section relative au filtre de conversion approprié du fichier opentext.ini.

     

timeout

  • Description :

    Indique le temps d'attente maximum (en secondes) avant la fin d'un processus de traitement de conversion de documents.

    Le DCS contrôle le temps nécessaire aux processus de traitement pour convertir des documents et interrompt les processus qui dépassent cette limite. Ce délai d'attente limite est défini par le paramètre timeout.

    Lorsque le processus de traitement tente de convertir un document corrompu ou comportant une anomalie, il échoue et renvoie un code d'erreur au processus de traitement. Toutefois, certains documents peuvent entraîner un blocage du système de filtrage qui tourne alors dans une boucle de traitement sans fin. Il peut arriver (rarement) que le filtre de conversion pris dans cette boucle sans fin épuise progressivement les ressources du système.

    La valeur timeout par défaut du processus de traitement est de 30 secondes. Passé ce délai, le processus s'interrompt. Toutefois, si le processus de traitement épuise la mémoire rapidement, il risque de consommer toute la mémoire disponible avant l'expiration du délai de 30 secondes. Le cas échéant, vous pouvez diminuer la valeur timeout afin que le filtre de conversion n'épuise pas toute la mémoire avant l'expiration du délai.

    La valeur du paramètre timeout doit être aussi basse que possible, sans toutefois entraîner l'interruption du processus de conversion avant la conversion d'un document valable. Pour obtenir une estimation de la valeur timeout appropriée, vous pouvez activer la journalisation pendant un laps de temps raisonnable, par exemple 24 heures, et prendre note du temps nécessaire au processus pour convertir un document valable. La durée de conversion valable maximale est généralement de quelques secondes. Une fois que vous avez défini cette valeur, vous pouvez modifier la valeur timeout en multipliant par 1,5 ou 2 la durée de conversion valable maximale. Le temps supplémentaire permet de traiter des documents plus gros que ceux qui ont été convertis au cours de la période de test.

  • Syntaxe :

    timeout=30

  • Valeurs :

    Un entier supérieur ou égal à un. La valeur par défaut est 30.

     

    Note :
    Ce paramètre apparaît également dans la section [FilterEngine] du fichier opentext.ini. Ce paramètre exécute la même fonction que le paramètre [FilterEngine] correspondant. Cependant, la définition de la valeur de ce paramètre dans la section [DCSworker] écrase le paramètre de la section [FilterEngine]. Si aucune valeur n'est définie dans la section [DCSworker], la valeur utilisée est celle de la section [FilterEngine].

    Ce paramètre peut être personnalisé pour chaque filtre de conversion. Il suffit pour cela de configurer le paramètre x-timeout dans la section relative au filtre de conversion approprié du fichier opentext.ini.

     

workerexe

  • Description :

    Indique le nom du programme de processus de traitement qu'utilise Content Server pour convertir des documents.

     

    Important :
    OpenText vous recommande vivement de ne pas modifier la valeur de ce paramètre.

     

  • Valeurs :

    Sous Windows, la valeur par défaut est dcsworker.exe. Pour les autres plates-formes, la valeur par défaut est dcsworker.
