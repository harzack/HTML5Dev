La section [report] définit les options des rapports LiveReport Dans les versions de Livelink antérieures à la version 8.1, cette section était intitulée [reports].

objectSubTypes

  • Description :

    Le paramètre objectSubTypes définit les types d'objets Content Server qui s'affichent dans les listes de saisie d'entrées Object de LiveReports. Pour modifier les types d'objets Content Server affichés dans ces listes, modifiez la liste des numéros de types de nœud, également appelés numéros des types d'objets. Pour plus d'informations sur la manière dont les types d'objets sont utilisés dans les rapports LiveReports, voir Privilèges et autorisations pour les rapports LiveReport.

    Pour plus d'informations sur chaque type d'objet, reportez-vous à la rubrique Correspondance entre le nom et le numéro des types de nœud.

    Pour plus d'informations sur les sous-types d'objets, reportez-vous au manuel Content Server Module Development Guide ou inspectez les objets LLNode et WebNode dans le générateur Content Server.

  • Syntaxe :

    objectSubTypes={ 299, 215, 202, 206, 204, 144, 0, 140, 207, 130, 1, 131, 136, 400, 223, 230, 208, 128, 335 }

  • Valeurs :

    La valeur par défaut de ce paramètre est :
    objectSubTypes={ 299, 215, 202, 206, 204, 144, 0, 140, 207, 130, 1, 131, 136, 400, 223, 230, 208, 128, 335 }

    Comme le montre l'exemple, la liste des numéros de types de nœud, ou types d'objet, Content Server corrects doit être séparée par des virgules et toute la liste doit être délimitée par des accolades.

InsertStrEnabled

  • Description :

    Active et désactive le type d'entrée InsertStr.

  • Syntaxe :

    InsertStrEnabled=false

  • Valeurs :

    true ou false. La valeur par défaut est false.

  • Exemple :

    [report]
InsertStrEnabled=true

blockedSQLterms

  • Description :

    Indique tous les termes qui doivent être bloqués lors de l'utilisation dans SQL. Ces termes sont vérifiés si InsertStr a été utilisé et si le Mode sécurisé a été activé. Pour en savoir plus, voir Insérer une chaîne.

  • Syntaxe :

    blockedSQLterms=;,UPDATE,UPDATETEXT,WRITETEXT,REMOVE,DROP,CREATE, ALTER,INSERT,COMMIT,EXECUTE,FETCH,REVOKE,ROLLBACK,SAVE,TRUNCATE,UNION

  • Valeurs :

    Une virgule sépare la liste des termes. La valeur par défaut est : ;,UPDATE,UPDATETEXT,WRITETEXT,REMOVE,DROP,CREATE, ALTER,INSERT,COMMIT,EXECUTE,FETCH,REVOKE,ROLLBACK,SAVE,TRUNCATE,UNION.

ModificationSQLTerms

  • Description :

    Indique une liste de termes qui, si elle est utilisée dans SQL ou dans un modèle SQL, oblige l'utilisateur à définir l'option Autoriser les modifications de la base de données. Pour en savoir plus, voir Fonctions Autoriser les modifications de la base de données.

  • Syntaxe :

    ModificationSQLTerms=UPDATE,UPDATETEXT,WRITETEXT,REMOVE,DROP,CREATE,ALTER, INSERT,COMMIT,EXECUTE,FETCH,REVOKE,ROLLBACK,SAVE,TRUNCATE

  • Valeurs :

    Une virgule sépare la liste des termes. La valeur par défaut est : ;,UPDATE,UPDATETEXT,WRITETEXT,REMOVE,DROP,CREATE, ALTER,INSERT,COMMIT,EXECUTE,FETCH,REVOKE,ROLLBACK,SAVE,TRUNCATE,UNION.

AllowDBModification

  • Description :

    Active ou désactive la fonction Autoriser les modifications de la base de données dans LiveReports. Pour en savoir plus, voir Fonctions Autoriser les modifications de la base de données.

  • Syntaxe :

    AllowDBModification=false

  • Valeurs :

    true ou false. La valeur par défaut est false.

     

    Note :
    Ce paramètre remplace un paramètre précédent appelé lockDBModification. Si vous aviez déjà défini lockDBModification=true, il correspond maintenant au paramètre actuel AllowDBModification=false.

     

  • Exemple :

    [report]
AllowDBModification=true

QueryVisibleWithSeeContents

  • Description :

    Détermine si l'option Afficher la requête est disponible pour les utilisateurs disposant uniquement d'une autorisation Voir le contenu pour le rapport LiveReport. Si l'administrateur définit cette option sur true, les utilisateurs disposant des autorisations Voir et Voir le contenu sur un rapport LiveReport pourront utiliser l'option Afficher la requête. Pour en savoir plus, voir Fonctions Autoriser les modifications de la base de données.

  • Syntaxe :

    QueryVisibleWithSeeContents=true

  • Valeurs :

    true ou false. La valeur par défaut est true.

  • Exemple :

    [report]
QueryVisibleWithSeeContents=false
