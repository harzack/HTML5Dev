La page Administration est le point de départ de la plupart des tâches d'administration système que vous effectuez. Cette page contient des sections présentant des liens vers d'autres pages administratives. Si vous installez des modules facultatifs, les liens vers les pages d'administration du module apparaissent dans une nouvelle section de la page Administration.

La page Administration n'est pas accessible à partir de l'interface utilisateur standard. Pour accéder à la page Administration, les utilisateurs disposant de privilèges d'administration système doivent utiliser une URL et un mot de passe spéciaux. Pour plus de sécurité, vous pouvez limiter l'accès à la page Administration à une liste d'adresses IP approuvées. Pour en savoir plus, voir Limitation des connexions du compte administrateur par adresse IP.

Admin est un compte utilisateur créé spécifiquement pour l'administrateur Content Server. Dans l'aide, l'administrateur Content Server est également appelé l'administrateur. Il s'agit du seul utilisateur autorisé à effectuer certaines tâches administratives. De plus, l'administrateur bénéficie d'un passe-droit qui lui donne automatiquement toutes les autorisations sur tous les éléments et emplacements dans Content Server sans vérification.

Il est important de noter que le mot de passe d'administration système est différent du mot de passe du compte utilisateur Admin. Dans certains cas, l'accès à certaines pages d'administration (telles que celles vous permettant de manipuler les données du système) requiert la connexion en tant qu'administrateur. Vous vous connectez en tant qu'administrateur de la même manière que pour tout autre utilisateur.
