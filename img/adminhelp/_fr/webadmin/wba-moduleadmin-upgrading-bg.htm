Lorsqu'Open Text publie de nouvelles versions de modules Content Server, ces modules peuvent souvent être téléchargés depuis la base de connaissances Open Text, ou sont parfois disponibles sur CD-ROM. Open Text vous recommande de consulter les notes de version d'un module avant de lancer une installation de mise à niveau.

Un module est mis à niveau en deux étapes. La première étape de la mise à niveau extrait les fichiers nécessaires du fichier .tar ou du programme d'installation et les place dans le répertoire _home/staging. Votre système Content Server n'est pas encore mis à niveau ; vous devez accéder aux pages Administration pour achever la mise à niveau du module.

 

Note :
Lorsque vous mettez à niveau un module, les fichiers du module précédent sont supprimés de Content Server et replacés dans le répertoire _home\staging.

 

La manière dont vous mettez à niveau un module dépend du type de système d'exploitation sur lequel Content Server est installé. Une fois les nouveaux fichiers ajoutés aux emplacements appropriés dans le répertoire _home/staging, le processus de mise à niveau est le même pour les systèmes d'exploitation Windows et UNIX.

Mise à niveau de modules sous Windows et UNIX

Pour les versions Windows des modules Content Server, vous effectuez la première phase d'une mise à niveau à l'aide d'un programme d'installation InstallShield. Ce programme d'installation vous permet de sélectionner l'instance de Content Server pour laquelle vous voulez mettre à niveau le module et place les fichiers correspondants dans un sous-répertoire du répertoire _home/staging.

Pour les versions UNIX des modules Content Server, vous effectuez la première phase de la mise à niveau en utilisant un fichier d'archive compressé au format .tar. L'extraction de ce fichier .tar place les fichiers du module dans un sous-répertoire du répertoire _home/staging.

Le répertoire _home/staging n'existe pas si aucun module facultatif n'est encore installé ou si aucun des modules préinstallés n'a encore été mis à niveau. Dans ce cas, le programme d'installation du module (pour Windows) ou l'extraction du fichier .tar (pour UNIX) crée automatiquement le répertoire _home/staging.

Fin de l'installation d'un module

Vous effectuez la seconde étape de la mise à niveau d'un module à l'aide de la page Mettre des modules à niveau, accessible via la page Administration.

Une fois la mise à niveau effectuée, la page Redémarrer Content Server est la première page qui s'affiche pour la plupart des modules. Cependant, certaines mises à niveau de module exigent que vous définissiez certains paramètres de configuration avant de redémarrer le serveur. Pour plus d'informations sur la définition de paramètres de configuration, reportez-vous à la documentation spécifique du module.

 

Note :
Si vous avez installé plusieurs modules, ils apparaissent sur la page Mettre des modules à niveau. Vous pouvez mettre à niveau plusieurs modules simultanément, pourvu qu'ils n'aient pas de dépendances sur d'autres modules. Lorsqu'un module a une dépendance sur un autre module, ce module doit d'abord être ajouté ou mis à niveau. Après la mise à niveau d'un module associé à des fichiers d'aide en ligne, vous devez mettre à jour l'index de l'aide pour voir les fichiers d'aide en ligne associés au module. Pour en savoir plus, voir Mise à jour de l'aide en ligne utilisateur et administrateur.
