Ces autorisations s'appliquent aux éléments de gestion de documents tels que documents, dossiers, documents composites, schémas de workflow, URL, raccourcis et générations. Les autorisations de gestion des documents sont plus détaillées que les autorisations des éléments de travail et permettent de contrôler plus précisément les opérations effectuées sur les éléments de gestion des documents.

La liste suivante décrit les autorisations de gestion des documents :

  • Voir – L'utilisateur ou le groupe peut voir le nom de l'élément.

  • Voir le contenu – L'utilisateur ou le groupe peut voir le nom de l'élément et ouvrir cet élément.

  • Modifier – L'utilisateur ou le groupe peut renommer l'élément et modifier certaines de ses propriétés (pour les éléments de recueil, tels que les dossiers, ceci inclut les propriétés qui sont affichées sur la page Infos sur la présentation de l'élément).

  • Modifier les attributs – L'utilisateur ou le groupe peut assigner une catégorie à un élément ou modifier les attributs personnalisés d'un élément ou sa catégorie.

  • Réserver – L'utilisateur ou le groupe peut réserver l'élément, le modifier, puis le déverrouiller. Il peut également lui ajouter des versions. Cette autorisation n'est disponible que pour les éléments pouvant être réservés, tels que les documents et les schémas de workflow.

  • Ajouter des éléments – L'utilisateur ou le groupe peut ajouter des éléments à l'élément. Cette autorisation n'est disponible que pour les éléments pouvant contenir d'autres éléments, tels que dossiers et documents composites.

  • Supprimer les versions – L'utilisateur ou le groupe peut supprimer des versions de l'élément. Cette autorisation n'est disponible que pour les éléments possédant des versions, tels que les documents et les schémas de workflow.

  • Supprimer – L'utilisateur ou le groupe peut supprimer l'élément.

  • Modifier les autorisations – L'utilisateur ou le groupe peut modifier les autorisations dont d'autres utilisateurs ou groupes bénéficient sur un élément.

Toutes les autorisations sont imbriquées au sein de l'autorisation Voir, ce qui signifie que seuls les utilisateurs et groupes qui bénéficient de l'autorisation Voir peuvent voir un élément ou son contenu, modifier, ajouter, supprimer ou réserver des éléments. Par exemple, vous ne pouvez pas modifier un élément que vous ne pouvez pas voir. De même, les autorisations Modifier les attributs, Réserver, Ajouter les éléments et Supprimer les versions sont imbriquées dans l'autorisation Modifier. Les autorisations Supprimer et Modifier les autorisations sont imbriquées dans l'autorisation Supprimer des versions.

 

Note :
Les autorisations Supprimer des versions et Réserver ne s'appliquent pas directement aux dossiers ou aux documents composites. Elles sont disponibles dans le jeu d'autorisations principalement pour les documents composites et les dossiers, de sorte que vous pouvez spécifier des autorisations par défaut pour les éléments ajoutés au dossier ou au document composite.

 

Lorsque vous sélectionnez une autorisation de gestion des documents, Content Server s'assure que le jeu d'autorisations de base correspondant à cette autorisation est également sélectionné. Si, par exemple, dans l'image ci-dessus, vous activez la case à cocher Réserver alors qu'aucune autre autorisation n'est sélectionnée, Content Server active automatiquement les cases à cocher Voir, Voir le contenu et Modifier.
