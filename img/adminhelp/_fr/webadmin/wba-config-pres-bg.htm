La page Configurer la présentation vous permet de définir la façon dont les éléments suivants sont présentés à l'utilisateur :

  • Configurer des options de recueil et choisir un style de navigation pour le système.

  • Activer ou désactiver la page de vue d'ensemble du document et la page Vue d'ensemble de la version.

  • Configurer un message HTML personnalisé affiché sur l'écran de connexion.

  • Promouvoir certaines fonctions de l'interface utilisateur.

  • Configurer la barre latérale de la vue de navigation et les panneaux de la barre latérale.

  • Activer ou désactiver les vues petites et grandes icônes.

  • Configurer les pages DOCTYPE pour HTML par défaut

  • Choisir l'organisation et la présentation des espaces de travail Projet.

Pour plus d'informations sur la configuration des options de mot de passe utilisateur et de l'affichage du nom d'utilisateur, reportez-vous à la rubrique Configuration des paramètres utilisateur.

Configuration des options de recueil

La page Configurer des options du recueil vous permet de spécifier comment les utilisateurs parcourent les recueils dans Content Server et activent et désactivent les fonctions de pagination et de filtrage.

navigation

Vous pouvez sélectionner le mode d'affichage du chemin de l'élément ou emplacement actuel, à savoir comme liste ou bandeau horizontal. Vous pouvez également autoriser les utilisateurs à sélectionner leur style préféré.

Pagination

Lorsque vous activez les dossiers et espaces de travail pour l'utilisation de la pagination, vous améliorez l'efficacité de la navigation des recueils contenant de grandes quantités d'éléments. Lorsque la pagination est activée, les utilisateurs peuvent trier le contenu dans un recueil de sorte à rendre la recherche d'éléments plus facile. Les administrateurs peuvent configurer le nombre d'éléments à afficher dans les recueils dans Content Server. Si le nombre d'éléments dans un recueil est inférieur au nombre spécifié, les flèches de navigation pour les pages précédente et suivante n'apparaissent pas. La pagination peut également être désactivée pour l'ensemble du système. Lorsque la pagination est désactivée, les utilisateurs voient tous les contenus d'un recueil sur une seule page.

 

Note :
La pagination est activée par défaut.

 

Personnalisation des colonnes

Lorsque cette option est activée, les utilisateurs sont en mesure de personnaliser les colonnes affichées dans la liste de navigation.

 

Note :
Par défaut, la personnalisation des colonnes est désactivée.

 

Eléments mis en valeur

Lorsque cette option est activée, les éléments qui ont été sélectionnés comme des éléments mis en valeur n'apparaîtront plus dans la liste de navigation.

 

Note :
Les éléments mis en valeur apparaissent dans la liste de navigation par défaut.

 

Filtrage

Lorsque vous activez les dossiers et espaces de travail pour l'utilisation du filtrage, vous améliorez l'efficacité de la recherche d'éléments dans un recueil contenant de grandes quantités d'éléments. Lorsque le filtrage est activé, les utilisateurs peuvent rechercher un élément en saisissant le nom de cet élément dans une zone de texte. Les résultats de la recherche sont renvoyés sur une nouvelle page.

Les administrateurs peuvent activer et désactiver le filtrage des recueils dans Content Server. Lorsque le filtrage est désactivé, le champ de recherche du filtre n'apparaît pas dans les recueils Content Server.

 

Note :
Le filtrage est activé par défaut.

La barre latérale Filtre de contenu et le filtrage sont mutuellement exclusifs. Lorsque la barre latérale et le filtrage sont tous deux activés, le filtrage ne sera disponible que sur les recueils qui ne sont pas éligibles pour afficher la barre latérale.

 

Activation ou désactivation de la page de vue d'ensemble d'un document

Vous pouvez activer ou désactiver une page Vue d'ensemble en tant que cible par défaut des liens Document sur une page, des Liens abrégés Document et des liens Version dans l'onglet Versions de la page Propriétés d'un document. Les pages Vue d'ensemble d'un document et Vue d'ensemble de la version fournissent des informations sur un document ou une version, et vous permettent d'y exécuter un certain nombre de fonctions. Les pages Vue d'ensemble d'un document et Vue d'ensemble de la version sont également accessibles à partir du menu Fonctions d'un document ou d'une version.

Si la fonction Afficher en tant que page Web n'est pas disponible (car vous n'avez pas installé de package de filtre de conversion de documents), la page Vue d'ensemble d'un document est également la cible par défaut lorsque les utilisateurs tentent d'afficher un document ou une version d'un document.

Pour en savoir plus, voir Pages de vue d`ensemble d`un document et d`une version.

Configuration du message de l'écran de connexion

Vous pouvez créer un message personnalisé qui apparaîtra dans l'écran de connexion. Chaque fois qu'un utilisateur se connecte à Content Server, ce message s'affiche à partir de l'écran de connexion.

Si vous avez installé plusieurs lots de langue, la page Configurer le message de connexion affiche une liste dans la colonne Langue représentant chaque lot de langue. Dans cette liste, vous devez sélectionner la langue à utiliser pour afficher votre message de connexion.

La page Configurer le message de connexion contient les colonnes suivantes :

  1. Langue : cette colonne affiche les lots de langue installés. Si plusieurs lots de langue sont installés, cette colonne affiche une liste vous permettant de sélectionner le lot de langue à utiliser pour afficher votre message de connexion. Par exemple, « Anglais (Etats-Unis) ».

  2. Code de langue : le code de langue associé à la langue sera affiché dans cette colonne. Ce champ n'est pas modifiable et affiche le code de la langue associé à la langue que vous avez sélectionnée dans la colonne Langue. Par exemple, « _en_US ».

  3. Message de l'écran de connexion HTML : une zone de texte qui vous permet de saisir le message que vous voulez que tous les utilisateurs qui se connectent à Content Server voient sur l'écran de connexion. Vous pouvez saisir du texte brut, HTML ou Javascript dans cette zone de texte.

Configuration des fonctions promues

Vous pouvez faciliter la localisation de certaines fonctions de gestion de documents dans l'interface utilisateur en les promouvant. Les fonctions promues s'affichent de manière plus distincte pour les documents sur la page Vue détaillée des recueils et des espaces de travail, ainsi que les pages Afficher en tant que page Web et Propriétés des documents.

Les fonctions suivantes peuvent être promues :

Fonction Présentation
Télécharger Lien sur la page Vue détaillée de l'espace de travail ou du recueil parent et bouton sur les pages Afficher en tant que page Web et Propriétés.
Modifier Lien sur la page Vue détaillée de l'espace de travail ou du recueil parent et bouton sur les pages Afficher en tant que page Web et Propriétés.
Ouvrir Lien sur la page Vue détaillée de l'espace de travail ou du recueil parent et bouton sur les pages Afficher en tant que page Web et Propriétés.
Ajouter un document Bouton Ajouter un document sur la page de navigation du recueil.
Ajouter un dossier Bouton Ajouter un dossier sur la page de navigation du recueil.

 

Note :
Par défaut Afficher en mode Vue de la liste détaillée et Afficher en mode Propriétés et Afficher en tant que page Web sont activés.

 

Configuration du bandeau

Vous pouvez activer ou désactiver le bandeau Filtre de contenu et les volets du bandeau pour tous les utilisateurs. Par défaut, ils sont activés.

Configuration des vues petites et grandes icônes

Vous pouvez activer ou désactiver les vues petites et grandes icônes dans l'interface utilisateur. Lorsque le paramètre Afficher les vues petites et grandes icônes est activé, les recueils peuvent afficher des éléments dans les vues détaillée et petites et grandes icônes, et les utilisateurs peuvent basculer d'un type de vue à un autre à l'aide des boutons Type de vue. Par défaut, les éléments marqués comme mis en valeur apparaissent dans la section Eléments mis en valeur et dans la liste Vue détaillée. L'administrateur peut modifier ce paramètre par défaut dans la section Eléments mis en valeur.

Lorsque ce paramètre est désactivé, les utilisateurs ne peuvent pas passer d'une vue à une autre. La vue détaillée s'affiche pour tous les recueils sans tenir compte du choix de vue de l'utilisateur. Si vous réactivez le paramètre Afficher les vues petites et grandes icônes, le recueil affiche la vue sélectionnée à l'origine.

 

Note :
Le paramètre Afficher les vues petites et grandes icônes est désactivé par défaut.

 

Configuration des options de miniature

Vous pouvez activer ou désactiver la génération de miniature dans l'interface utilisateur lorsque des objets dans une source de données de l'entreprise sont indexés. Lorsque le paramètres Générer les miniatures est activé, le processus Conversion de documents crée des miniatures du document ou de l'objet. Vous pouvez générer des miniatures pour la version actuelle de l'objet, ainsi que pour toutes les versions et spécifier pour quels types MIME les miniatures doivent être générées. Vous pouvez également activer ou désactiver les miniatures, lorsqu'elles sont disponibles, à afficher à la place des grandes icônes.

 

Note :
Le paramètre Générer les miniatures est activé par défaut.

 

Configuration des pages DOCTYPE pour HTML

DOCTYPE est la déclaration du type de document utilisé par les navigateurs pour afficher correctement les éléments et attributs des pages Web. OpenText recommande que le DOCTYPE soit toujours explicitement déclaré dans vos documents HTML ou XHTML.

 

Note :
La valeur par défaut est : <!DOCTYPE html PUBLIC "-//W3C//DTD HTML 3.2 Final//EN">

 

Configuration des paramètres de présentation du projet

Cette page vous permet de configurer l'apparence et le comportement du projet par défaut suivants :

  • Vous pouvez définir la page de démarrage par défaut d'un projet sur la page Vue d'ensemble du projet ou l'espace de travail Projet.

  • Vous pouvez afficher un menu Projet ou une Barre d'icônes du projet dans tous les espaces de travail Projet.

  • Vous pouvez activer la page Overview.html, qui fournit une vue personnalisée des éléments d'un projet aux intervenants de ce projet. Les sections qui apparaissent sur la page sont configurées par le coordinateur de projet. Lorsque cette option est activée, la page Overview.html apparaît si un projet est ouvert. Par défaut, cette option est désactivée et la page Espace de travail Projet est utilisée comme page de démarrage à l'ouverture d'un projet.

  • Vous pouvez définir le nombre maximum d'éléments autorisés dans chaque section de la page Vue d'ensemble du projet.

  • Vous pouvez modifier les valeurs par défaut de l'indicateur de statut pour tous les projets.

 

Note :
Lorsque vous modifiez un paramètre de projet, la modification s'applique aux nouveaux projets et aux projets existants. Par exemple, si vous désactivez la case à cocher Mes tâches dans la section Navigation, l'icône Mes tâches n'est plus affichée dans la barre d'icônes du projet.
