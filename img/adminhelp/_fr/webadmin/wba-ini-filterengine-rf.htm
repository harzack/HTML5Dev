Les paramètres de la section [FilterEngine] définissent le comportement central du service de conversion de documents, DCS. Le DCS convertit des documents au format HTML ou texte brut pour permettre leur affichage et indexation. Pour ce faire, il utilise des filtres de conversion de documents. Pour plus d'informations sur les filtres de conversion de documents, reportez-vous à la rubrique Filtres de conversion de documents.

Vous pouvez configurer le DCS de trois façons différentes.

  • Vous pouvez configurer globalement tous les DCS de votre site en définissant les paramètres de la section [FilterEngine] du fichier opentext.ini.

  • Vous pouvez configurer globalement les paramètres spécifiques à un filtre de conversion dans l'une des sections suivantes du fichier opentext.ini : [DCS], [DCSworker], [DCSxpdf], [DCSservers], [DCSview], [DCSpipe], [DCSipool], [QDF], [DCSIm] et [Summarizer].

  • Vous pouvez configurer localement un DCS individuel en définissant les arguments de ligne de commande.

Certains paramètres des sections DCS du fichier opentext.ini et certains arguments de ligne de commande du DCS sont également des paramètres de la section [FilterEngine]. Les paramètres et arguments de ligne de commande du DCS exécutent la même fonction que les paramètres [FilterEngine] correspondants. Cependant, les valeurs définies pour ces paramètres dans les sections DCS du fichier opentext.ini ou dans les arguments de ligne de commande du DCS écrasent celles des paramètres de la section [FilterEngine].

dllpath

  • Description :

    Détermine l'emplacement du répertoire contenant les filtres de conversion de Content Server. Un document créé par un traitement de texte, Microsoft Word par exemple, contient des informations de formatage qui ne sont pas nécessaires ou requises pour la recherche. Les filtres de conversion extraient le contenu de texte nécessaire pour lire des fichiers de traitement de texte et effectuer une indexation à partir de ces fichiers.

  • Syntaxe :

    dllpath=C:\OPENTEXT\filters

  • Valeurs :

    Un chemin absolu. La valeur par défaut est <Content Server_home> \filters, où <Content Server_accueil> est la racine de votre installation Content Server.

encoding<n>

  • Description :

    Le paramètre encoding<n> permet de spécifier de multiples valeurs de codage de caractères afin d'assurer une conversion réussie du codage des caractères pour les documents indexés. Le DCS va tenter de détecter le codage de caractères de documents texte en cours d'indexation. Si le DCS n'arrive pas à détecter le codage, il va tenter d'effectuer un codage de caractères UTF-8 en utilisant les valeurs de codage définies pour ce paramètre. Le DCS essaie d'utiliser les valeurs de codage dans l'ordre dans lequel elles sont répertoriées dans la section [FilterEngine]. Le DCS continue de tenter de coder les caractères jusqu'à ce qu'il réussisse ou qu'il n'ait plus de valeurs de codage de caractères valables à tester. La première valeur de codage à laquelle les données peuvent être converties avec succès est considérée comme la valeur de codage appropriée. Chaque entrée encoding<n> de la section [FilterEngine] doit être au format suivant : encoding<n>=<valeur_codage>, où <n> est un nombre unique.

    Dans l'exemple illustré à la section Syntaxe ci-dessous, le DCS tente de convertir les documents en utilisant le codage Shift JIS. Si le codage Shift JIS échoue, le DCS tente d'utiliser le codage EUC-JP.

  • Syntaxe :

    encoding1=Shift JIS
    encoding2=EUC-JP

  • Valeurs :

    Un nom de spécification de codage de caractères. Si aucune valeur n'est définie pour ce paramètre, ISO-8859-1 est utilisé. Si vous n'incluez pas le paramètre encoding<n> dans le fichier opentext.ini, par défaut, le DCS utilise le codage Latin-1. Pour plus d'informations, contactez le service d'assistance clientèle d'OpenText.

logfile

  • Description :

    Détermine l'emplacement et le préfixe du fichier journal de conversion de documents. Le numéro de port admin attribué à ce processus est ajouté au fichier journal, suivi du suffixe .log. Ceci empêche que différents processus écrivent dans le même fichier.

  • Syntaxe :

    logfile=C:\OPENTEXT\logs\dcs

  • Valeurs :

    Un chemin absolu. La valeur par défaut est <Content Server_home> \logs\dcs, où <Content Server_accueil> est la racine de votre installation Content Server.

     

    Note :
    Ce paramètre est obligatoire. Pour désactiver la journalisation, définissez logLevel sur 0.

     

maxfilesize

  • Description :

    Indique la taille maximum, en Mo, d'un fichier pouvant être converti en mémoire.

  • Syntaxe :

    maxfilesize=8

  • Valeurs :

    Un entier supérieur à un. La valeur par défaut est 8.

récapitulatif

  • Description :

    Indique si le DCS génère des résumés des documents qu'il convertit au format HTML. Si l'utilisateur définit ses options d'affichage de recherche pour afficher les résumés, Content Server affiche ces résumés accompagnés des résultats de recherche sur la page Résultat de recherche.

    Par défaut, sur la ligne de commande de chaque DCS, le paramètre summary est défini sur TRUE. Pour définir le paramètre summary sur FALSE, vous devez modifier la ligne de commande de chaque DCS plutôt que le paramètre du fichier opentext.ini, car les arguments définis sur la ligne de commande écrasent les paramètres généraux définis dans le fichier opentext.ini. Pour en savoir plus, voir Arguments de ligne de commande.

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est TRUE.

    La valeur TRUE indique que le DCS génère des résumés. La valeur FALSE indique que le DCS ne génère pas de résumé.

summaryhotwords

  • Description :

    Indique le nombre de liens hypertextes utilisés par le résumeur pour générer le résumé d'un document. En réduisant cette valeur, vous pouvez accélérer la création des résumés mais beaucoup d'autres variables affectent également la vitesse de ce processus.

     

    Important :
    OpenText vous recommande vivement de ne pas modifier la valeur de ce paramètre.

     

  • Valeurs :

    Un entier supérieur ou égal à un. La valeur par défaut est 20.

summarysentences

  • Description :

    Indique le nombre de phrases à générer dans les résumés.

  • Syntaxe :

    summarysentences=5

  • Valeurs :

    Un entier supérieur ou égal à un. Par défaut, ce paramètre n'apparaît pas dans la section [FilterEngine] du fichier opentext.ini, ce qui équivaut à summarysentences=5.

timeout

  • Description :

    Indique le temps d'attente maximum (en secondes) avant la fin d'un processus de traitement.

    Le DCS contrôle le temps nécessaire aux processus de traitement pour convertir des documents et interrompt les processus qui dépassent cette limite. Ce délai d'attente limite est défini par le paramètre timeout.

    Lorsque le processus de traitement tente de convertir un document corrompu ou comportant une anomalie, il échoue et renvoie un code d'erreur au DCS. Toutefois, certains documents peuvent entraîner un blocage du système de filtrage qui tourne alors dans une boucle de traitement sans fin. Il peut arriver (rarement) que le filtre de conversion pris dans cette boucle sans fin épuise progressivement les ressources du système.

    La valeur timeout par défaut du DCS est de 30 secondes. Passé ce délai, le processus de traitement s'interrompt. Toutefois, si le processus de traitement épuise la mémoire rapidement, il risque de consommer toute la mémoire disponible avant l'expiration du délai de 30 secondes. Le cas échéant, vous pouvez diminuer la valeur timeout afin que le filtre de conversion n'épuise pas toute la mémoire avant l'expiration du délai.

    La valeur du paramètre timeout doit être aussi basse que possible, sans toutefois entraîner l'interruption du processus de conversion avant la conversion d'un document valable. Pour obtenir une estimation de la valeur délai d'attente appropriée, vous pouvez activer la journalisation pendant un laps de temps raisonnable, par exemple 24 heures, et prendre note du temps nécessaire au processus pour convertir un document valable. La durée de conversion valable maximale est généralement de quelques secondes. Une fois que vous avez défini cette valeur, vous pouvez modifier la valeur timeout en multipliant par 1,5 ou 2 la durée de conversion valable maximale. Le temps supplémentaire permet de traiter des documents plus gros que ceux qui ont été convertis au cours de la période de test.

    La valeur du paramètre timeout peut également être personnalisée pour chaque filtre de conversion. Pour cela, il vous suffit de définir le paramètre x-timeout dans la section INI de chaque filtre de conversion.

  • Syntaxe :

    timeout=30

  • Valeurs :

    Un entier supérieur ou égal à un. La valeur par défaut est 30.

tmpdir

  • Description :

     

    Note :
    Dans la version Windows de Content Server, cette valeur ne s'applique pas. Tous les fichiers temporaires sont enregistrés dans le répertoire défini par la variable d'environnement TMP.

     

    Ce paramètre spécifie le répertoire que le DCS utilise pour stocker les fichiers de conversion temporaires. Les fichiers de conversion temporaires sont des fichiers avec des extensions .in et .out, et des fichiers temporaires créés par le système de filtrage. La valeur de ce paramètre ne peut pas être supérieure à 256 octets.

    Sur les systèmes d'exploitation UNIX, OpenText vous recommande d'utiliser le répertoire /tmp en tant que répertoire temporaire pour les DCS. Si vous définissez un système de fichiers temporaire, tel que /tmp pour UNIX, les performances des DCS seront considérablement améliorées. Dans les versions UNIX de Content Server, le paramètre tmpdir est défini par défaut sur /tmp lors de l'installation de Content Server.

    Sous UNIX, les processus de traitement pour lesquels est survenue une anomalie au moment de la fermeture peuvent parfois rejeter les fichiers temporaires. OpenText vous recommande de nettoyer régulièrement le répertoire temporaire.

  • Syntaxe :

    tmpdir=/tmp

  • Valeurs :

    Un chemin absolu.

fenêtre

  • Description :

    Définit le nombre maximum de messages IPool à traiter à partir de la zone de lecture IPool avant que le DCS exécute une transaction IPool.

    Ce paramètre est le premier d'un ensemble de trois mécanismes de limitation du débit compris dans la section [FilterEngine] du fichier opentext.ini. Il est suivi des paramètres minwindow et maxtime.

  • Syntaxe :

    window=10

  • Valeurs :

    Un entier supérieur ou égal à un. La valeur par défaut est 10.

     

    Note :
    Si cette valeur est plus élevée que la valeur par défaut, le débit du filtre de conversion augmente mais le DCS utilise davantage d'espace disque.

     

minwindow

  • Description :

    Définit le nombre minimum de messages IPool à traiter à partir de la zone de lecture IPool avant que le DCS exécute une transaction IPool. Si le nombre de messages disponibles dans la zone de lecture est inférieur à la valeur indiquée pour le paramètre minwindow, la transaction sera exécutée après le traitement du dernier message iPool par le DCS.

    Ce paramètre est le deuxième d'un ensemble de trois mécanismes de limitation du débit compris dans la section [FilterEngine] du fichier opentext.ini. Il est précédé du paramètre window et suivi du paramètre maxtime.

  • Syntaxe :

    minwindow=5

  • Valeurs :

    Un entier supérieur ou égal à un. La valeur par défaut est 5.

maxtime

  • Description :

    Indique la durée maximum, en millisecondes, nécessaire au traitement d'une transaction IPool. Lorsqu'une transaction dépasse cette durée, elle est exécutée après le traitement complet du message iPool actif, que les critères window ou minwindow aient été respectés ou non.

    Ce paramètre est le dernier d'un ensemble de trois mécanismes de limitation du débit compris dans la section [FilterEngine] du fichier opentext.ini. Il est précédé des paramètres window et minwindow.

  • Syntaxe :

    maxtime=120000

  • Valeurs :

    Un entier supérieur ou égal à un. La valeur par défaut est 120000, soit deux minutes.
