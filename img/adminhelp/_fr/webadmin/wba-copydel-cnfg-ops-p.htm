Pour modifier les paramètres des opérations de copie, de suppression et de déplacement :

  1. Cliquez sur le lien Configurer les opérations pour la copie, la suppression et le déplacement dans la section Administration système sur la page Administration Content Server.

  2. Sur la page Configurer les opérations pour la copie, la suppression et le déplacement, cliquez sur l'un des boutons radio suivants dans les sections Copier et Supprimer :

    • Exécuter les transactions une fois le traitement des éléments terminé, qui exécute la transaction de base de données lorsque tous les nœuds ont été traités.

       

      Note :
      En cas d'échec de l'un des éléments de la transaction, l'ensemble de la transaction échoue et aucune opération n'est exécutée dans la base de données.

       

    • Exécuter la transaction chaque fois qu'un élément est traité avec succès, qui exécute la transaction de base de données à chaque nœud traité.

       

      Note :
      La fonction Copier copie un nœud à la fois et exécute la transaction de base de données selon un schéma descendant. La fonction Supprimer quant à elle supprime un nœud à la fois et exécute la transaction de base de données selon un schéma ascendant.

       

  3. Facultatif Dans la section Supprimer, sélectionnez la case à cocher Retarder la suppression du fournisseur de stockage pour permettre aux éléments supprimés d'être placés dans la file d'attente et supprimés par l'activité programmée Nouvel essai après l'échec de suppression du blob du fournisseur.

     

    Note :
    Si vous activez cette option, vous devez également activer l'activité programmée Nouvel essai après l'échec de suppression du blob du fournisseur.

     

  4. Dans la section Déplacer, sélectionnez la case à cocher Activer la commande de limitation de déplacement, puis saisissez le nombre maximal d'objets autorisés lors d'un déplacement dans le champ correspondant.

  5. Cliquez sur le bouton Mettre à jour.
