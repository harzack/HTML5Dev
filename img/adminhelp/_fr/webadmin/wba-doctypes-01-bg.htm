Lorsqu'un fichier est téléchargé vers la base de données Content Server, la liste suivante indique les actions réalisées par Content Server pour déterminer le type MIME du document. Si la première action ne parvient pas à déterminer le type MIME, l'action suivante est réalisée.

  1. Content Server recherche les informations sur le type MIME dans le navigateur Web de l'utilisateur.

  2. Content Server recherche l'extension du fichier dans le fichier mime.types pour voir si elle est mappée sur un type MIME dans ce fichier.

  3. Content Server tente d'identifier le type MIME à l'aide d'un programme autorecognition.

  4. Content Server attribue le fichier au premier type MIME figurant dans la liste du fichier mime.types. Par défaut, il s'agit du type application/activemessage.

 

Note :
Les actions par défaut, ainsi que l'ordre dans lequel elles sont réalisées, peuvent être modifiés sur la page Administration système.

 

Non seulement le fichier mime.types assigne les types MIME aux documents téléchargés, mais il contrôle les types MIME affichés dans la liste Type MIME de la page Infos spécifiques des documents dans Content Server. Par défaut, le fichier mime.types comporte environ 100 types MIME.

L'utilisation d'un éditeur de texte vous permet de modifier facilement le fichier mime.types en vue d'ajouter ou de supprimer un type MIME, ou encore de faire correspondre une nouvelle extension de fichier à un type MIME existant.

La liste des types MIME

Les Types MIME se trouvent dans le fichier Content Server_home/config/mime.types pouvant être modifié à l'aide d'un éditeur de texte.

Exemple 8-1 : Exemple de liste de types MIME

# MIME type to file extension mapping file.
#
# This is a whitespace-separated values file where blank lines and text
# following "#" is ignored.
#
# Each line is a separate entry with the following format:
# <mime type> <file extension> ...
application/activemessage
application/andrew-inset
application/applefile
application/atomicmail
application/dca-rft
application/dec-dx
application/mac-binhex40
application/macwriteii
application/msword doc W6BN
application/vnd.ms-excel xls xlb XLS5
application/powerpoint ppt SLD3

Un type MIME est ajouté à la liste au format suivant :

<type_MIME> <extension_fichier1> <extension_fichier2> ...

Exemple de <type_MIME> dans la liste de types MIME ci-dessus : application/msword, les deux <extensions_fichier> associées sont : doc et W6BN. Cependant, lors de l'ajout d'un nouveau type MIME, il n'est pas nécessaire de mapper une extension de fichier sur le type MIME.

Vous pouvez utiliser le caractère # pour ajouter un commentaire à une entrée. Tout ce qui suit le caractère # est considéré comme commentaire et ignoré par Content Server.
