Pour exécuter un jeu personnalisé de tests de diagnostic sur votre base de données Content Server :

Sur la page Options de vérification personnalisées, sélectionnez un des tests suivants en cochant la case correspondante :

  1. Vérifier le nombre de listes de contrôle d'accès d'un noeud

  2. Vérifier le nombre d'enfants d'un recueil

  3. Vérifier si un DataID existe pour chaque DocID

  4. Vérifier si un ID de fournisseur ProviderData existe pour chaque ID de fournisseur sous DVersData

  5. Vérifier l'existence des membres et chefs de groupe

  6. Vérifier si chaque groupe enregistré dans KUAFChildren existe

  7. Rechercher les nœuds sans ID parent

  8. Vérifier l'ID parent de chaque noeud

  9. Vérifier si un providerID DVersData existe pour chaque providerID sous ProviderData

  10. Vérifier si le refCnt d'un nœud correspond au nombre de versions

  11. Cliquez sur le bouton Etablir diagnostic.
