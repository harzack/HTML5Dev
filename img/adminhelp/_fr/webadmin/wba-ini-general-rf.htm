AdminIndexStyle

  • Description :

    Détermine si la page Administration affiche tous les liens et toutes les sections en même temps ou si elle affiche les sections sous forme d'onglets et les liens uniquement sous l'onglet sélectionné.

  • Syntaxe :

    AdminIndexStyle=all

  • Valeurs :

    Les valeurs valides sont all ou tabs. La valeur par défaut est all.

    Valeur Description
    tous Affiche tous les liens et toutes les sections en même temps. Il s'agit de la valeur par défaut.
    tabs Affiche les sections sous forme d'onglets. Seuls les liens de l'onglet sélectionné sont affichés.

AdminMailAddress

  • Description :

    Adresse de messagerie de l'administrateur. Si vous spécifiez ici une adresse e-mail, un lien permettant d'envoyer un e-mail à l'administrateur sera créé sur la page Identification de l'utilisateur. Pour en savoir plus, voir Configuration des paramètres de base du serveur.

  • Syntaxe :

    AdminMailAddress=<nom_utilisateur>@<nom_domaine>.com

  • Valeurs :

    Une adresse électronique valide.

adminpwd

  • Description :

    Mot de passe codé de l'administrateur Content Server.

DBBINNT

Débogage

  • Description :

    Définit le niveau de débogage dans Content Server. Pour plus d'informations, voir Configuration du journal du serveur.

    L'enregistrement dans les journaux amoindrit les performances du serveur. Activez uniquement cette fonction le temps de réunir les informations nécessaires pour résoudre un problème particulier.

    OpenText vous recommande de modifier cette valeur uniquement depuis la page Configurer les paramètres du serveur plutôt que de modifier directement le fichier opentext.ini. Pour plus d'informations, voir Configuration des paramètres de base du serveur.

  • Syntaxe :

    Debug=0

  • Valeurs :

    Les valeurs valides sont 0, 1, 2 et 11. La valeur par défaut est 0, zéro.

    Valeur Description
    0 Pas d'enregistrement. Il s'agit de la valeur par défaut.
    1 Effectue une journalisation minimum du serveur. Production de journaux sur les threads du serveur et d'autres journaux.
    2 Effectue chaque journalisation à partir de Debug=1, ainsi qu'une journalisation plus détaillée des requêtes des threads.
    11 Effectue chaque journalisation à partir de Debug=2, et permet la journalisation du processus client CGI et du moteur de mise à jour de l'index, Extracteur Entreprise. Outre les journaux des threads du serveur et autres journaux, production de journaux sur le client CGI et mise à jour d'index.

     

    Note :
    Lorsque vous utilisez LLServlet au lieu de CGI Content Server, llclient<nnn>.out est remplacé par servletclient<nnn>.out. Un fichier est généré pour chaque numéro de thread, déterminé par la machine virtuelle Java de votre recueil servlet. Chaque fois qu'un navigateur client envoie une requête à LLServlet, un nouveau fichier servletclient<nnn>.out est créé. Si un nom de fichier portant ce numéro <nnn> existe déjà, les informations de la requête sont ajoutées à la fin du fichier existant.

     

DefaultContentRH

  • Description :

    Définit le gestionnaire de requêtes sur le contenu par défaut. Si l'affichage des cadres de Content Server est activé lors de la connexion des utilisateurs, si DefaultRH=LL.FrameHome, ce paramètre définit la page Content Server qui apparaîtra dans le cadre de droite. Si l'affichage des cadres de Content Server n'est pas activé lors de la connexion des utilisateurs, ce paramètre définit la page Content Server qui sera affichée dans la fenêtre du navigateur Web.

    OpenText vous recommande de modifier cette valeur uniquement depuis la page Configurer les paramètres du serveur plutôt que de modifier directement le fichier opentext.ini. Pour plus d'informations, voir Configuration des paramètres de base du serveur.

  • Syntaxe :

    DefaultContentRH=Enterprise.Home

  • Valeurs :

    Nom d'un gestionnaire de requêtes valide. Les valeurs possibles sont : Enterprise.Home, Personal.Home et LL.Index. La valeur par défaut est Enterprise.Home.

    Valeur Description
    Enterprise.Home Il s'agit de la valeur par défaut. Elle spécifie l'affichage de la page Espace de travail Entreprise lors de la connexion de l'utilisateur.
    Personal.Home Spécifie l'affichage de la page Espace de travail Personnel de l'utilisateur lors de la connexion de l'utilisateur.
    LL.Index Spécifie l'affichage de la page A propos de Content Server lors de la connexion de l'utilisateur.

DefaultRH

  • Description :

    Définit le gestionnaire de requêtes par défaut. Ce paramètre définit la page Content Server affichée lors de la connexion des utilisateurs via leur navigateur Web.

  • Syntaxe :

    DefaultRH=Enterprise.Home

  • Valeurs :

    Nom d'un gestionnaire de requêtes valide. Les valeurs possibles sont : Enterprise.Home, Personal.Home, LL.Index et LL.FrameHome. La valeur par défaut est Enterprise.Home.

    Valeur Description
    Enterprise.Home Il s'agit de la valeur par défaut. Elle spécifie l'affichage de la page Espace de travail Entreprise dans une vue Content Server sans cadre lors de la connexion des utilisateurs.
    Personal.Home Spécifie l'affichage de la page Espace de travail Personnel de l'utilisateur dans une vue Content Server sans cadre lors de la connexion des utilisateurs.
    LL.Index Spécifie l'affichage de la page A propos de Content Server dans une vue Content Server sans cadre lors de la connexion des utilisateurs.
    LL.FrameHome Spécifie l'activation de l'affichage avec des cadres de Content Server lors de la connexion des utilisateurs. Lorsque le paramètre DefaultRH est défini sur LL.FrameHome, la barre de menus s'affiche dans le cadre de gauche. Le paramètre DefaultContentRH définit le contenu du cadre de droite.

DisableSelectReservedBy

  • Description :

    A partir de la version 8.0.2 de Livelink, les utilisateurs peuvent réserver un document à un groupe. Cela permet à un membre d'un groupe de réserver un document sans pour autant empêcher les autres membres du groupe de travailler dessus ou de le retourner. Il s'agit pour Content Server d'afficher une page invitant l'utilisateur à spécifier l'utilisateur ou le groupe auquel il réserve le document au moment même du téléchargement de ce document vers le navigateur Web.

    Si le navigateur Web de l'utilisateur est configuré pour ouvrir le type MIME du document, celui-ci est affiché directement dans la fenêtre du navigateur Web et l'utilisateur ne peut pas spécifier l'utilisateur ou le groupe pour lesquels il souhaite réserver le document. Ce problème a notamment été signalé avec les documents HTML dans les navigateurs Microsoft Internet Explorer. L'utilisateur ne peut donc pas réserver de documents de ce type MIME avec la fonction Emprunter.

    Les utilisateurs qui rencontrent ce problème ont deux options :

    1. réserver puis extraire le document, plutôt que d'utiliser la fonction Emprunter ;

    2. Ajoutez DisableSelectReservedBy=TRUE dans la section [general] du fichier opentext.ini. Cette option désactive la fonctionnalité d'emprunt du groupe pour tous les utilisateurs.

  • Syntaxe :

    DisableSelectReservedBy=FALSE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est FALSE.

    La définition de ce paramètre sur FALSE signifie que la fonctionnalité d'emprunt du groupe est activée. Par défaut également, l'entrée ne figure pas dans le fichier opentext.ini.

HTMLCharset

  • Description :

    Par défaut, Content Server ne spécifie pas de codage de jeu de caractères ; le jeu de caractères utilisé dépend de la configuration du navigateur Web de chaque utilisateur. Vous pouvez définir Content Server pour remplacer l'ensemble de caractères utilisé par le navigateur Web sur la page Configurer les paramètres du serveur.

  • Valeurs :

MessageClearInterval

  • Description :

    Le paramètre MessageClearInterval définit en nombre de jours la durée maximale de stockage des messages de notification Content Server dans Content Server.

    Si un utilisateur active l'envoi par messagerie d'un rapport de notification, les messages associés à ce rapport sont effacés de la base de données lors de l'envoi du message électronique. Cependant, si un utilisateur n'active pas l'envoi par e-mail d'un rapport de notification donné, les événements correspondant aux intérêts du rapport sont enregistrés dans la base de données jusqu'à ce que l'utilisateur les ouvre et les supprime via l'onglet Notification de l'espace de travail Personnel. Si les utilisateurs ne consultent pas régulièrement leur onglet Notification, de nombreux messages risquent de s'accumuler dans la base de données. Afin d'éviter ce genre de surcharge, Content Server supprime tous les rapports de notification antérieurs au nombre de jours défini par le paramètre MessageClearInterval.

  • Syntaxe :

    MessageClearInterval=30

  • Valeurs :

    Tout nombre entier de jours. La valeur par défaut est 30.

NavigationOption

  • Description :

    Détermine si les menus de navigation et les icônes de l'interface utilisateur de Content Server acceptent le langage Java.

    OpenText vous recommande de modifier cette valeur uniquement depuis la page Configurer les paramètres du serveur plutôt que de modifier directement le fichier opentext.ini. Pour plus d'informations, voir Configuration des paramètres de base du serveur.

  • Syntaxe :

    NavigationOption=0

  • Valeurs :

    Les valeurs possibles sont : 0, 1 ou 2. La valeur par défaut est 0.

    Valeur Description
    0 Il s'agit de la valeur par défaut. Définit une navigation Java par défaut tout en permettant aux utilisateurs de sélectionner une navigation non-Java.
    1 Définit une navigation uniquement non-Java pour tous les utilisateurs.
    2 Définit une navigation non-Java par défaut tout en permettant aux utilisateurs de sélectionner une navigation Java.

NumOldLogs

  • Description :

    Ce paramètre, une fois ajouté à la section [general] et défini sur 0, zéro, indique à Content Server d'afficher uniquement l'ensemble actuel de journaux de thread. Une fois le serveur redémarré, ce paramètre supprime tous les anciens threads.

    Ce paramètre ne prend pas en compte les journaux de connexion.

  • Syntaxe :

    NumOldLogs=0

  • Valeurs :

    0, zéro, pour afficher uniquement des journaux actuels ou 1, un, pour afficher tous les journaux. Ne s'affiche pas par défaut.

OTHOME

  • Description :

    Chemin complet d'accès à la racine de l'installation Content Server. La marque de réservation Content Server_accueil est utilisée pour désigner ce répertoire racine. Le chemin est défini par l'utilitaire d'installation de Content Server.

  • Valeurs :

    Un chemin d'accès absolu est obligatoire. Sous Windows, le paramètre OTHOME peut prendre la valeur C:\OPENTEXT\, par exemple. Sur les systèmes UNIX, /usr/local/ est un exemple de valeur possible pour le paramètre OTHOME. Cette valeur est définie lors de l'installation de Content Server.
