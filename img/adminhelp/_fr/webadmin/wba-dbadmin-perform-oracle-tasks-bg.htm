Utilisez la page Maintenance du serveur Oracle pour effectuer les tâches suivantes :

  • créer un nouvel espace de stockage ;

  • créer un compte utilisateur Oracle ;

  • agrandir un espace de stockage ;

  • supprimer un compte utilisateur Oracle et/ou ses tables.

Utilisez la page Maintenance du serveur Oracle dans les deux cas suivants :

  • Lorsque vous administrez une base de données Content Server. Dans ce cas, vous accédez à la page Maintenance du serveur Oracle en cliquant sur le lien Tâches de maintenance Serveur Oracle de la page Maintenance de la base de données active.

  • Lorsque vous créez une base de données Content Server pour vous connecter à une installation Content Server nouvelle ou existante. Dans ce cas, vous accédez à la page Maintenance du serveur Oracle en cliquant sur le lien Maintenance du serveur Oracle de la page Créer des tables Content Server.

Dans ces deux cas, cliquez sur le lien Revenir à la page précédente de la page Maintenance du serveur Oracle pour revenir à la page initiale (Maintenance de la base de données active ou Créer des tables Content Server) à partir de laquelle vous aviez affiché la page Maintenance du serveur Oracle.

Si vous accédez à la page Maintenance du serveur Oracle dans le cadre d'une création de base Content Server, effectuez les tâches suivantes dans l'ordre indiqué avant de cliquer sur le lien Revenir à la page précédente :

  • Création d'un nouvel espace de stockage, si nécessaire, destiné à contenir les tables de la base Content Server.

  • Pour supprimer des utilisateurs et des tables Oracle pouvant se connecter à l'espace de stockage.

Création d'un nouvel espace de stockage

Content Server utilise un espace de stockage Oracle pour stocker les tables de la base Content Server. Du fait qu'un espace de stockage peut contenir des tables appartenant à plusieurs utilisateurs Oracle, il n'est pas nécessaire de créer un nouvel espace s'il en existe déjà un avec suffisamment de place.

L'espace de stockage étant créé en même temps que la base Content Server, aucune procédure distincte de création n'est en principe nécessaire.

Création d'un utilisateur Oracle

Toutes les tables de l'espace de stockage appartiennent à des utilisateurs Oracle. Un utilisateur Oracle en particulier possède les tables de la base Content Server. Content Server utilise le nom d'utilisateur Oracle et son mot de passe pour se connecter à la base Content Server, gérée par le serveur Oracle.

L'utilisateur Oracle étant créé en même temps que la base Content Server, aucune procédure distincte de création n'est en principe nécessaire.

Agrandissement d'un espace de stockage

Un espace de stockage contient un ou plusieurs fichiers de données. A mesure que la quantité de données de votre base Content Server augmente, ces fichiers se remplissent et deviennent saturés. Vous pouvez augmenter la taille d'un espace de stockage en lui ajoutant un nouveau fichier de données.

Certaines versions du serveur Oracle Server peuvent être configurées de façon à ce que l'espace de stockage s'agrandisse automatiquement lorsque l'espace vient à manquer. Pour plus d'informations, reportez-vous à la documentation fournie avec votre serveur Oracle.

Suppression d'un utilisateur Oracle et/ou de ses tables

Si la base de données Content Server est endommagée, la seule manière pour la récupérer peut être de supprimer ses tables et même l'utilisateur de base de données, puis de restaurer la base à partir d'une sauvegarde.

Si vous supprimez uniquement les tables, celles créées pour les catégories personnalisées ne sont pas retirées. De ce fait, pour les englober dans la suppression, supprimez l'utilisateur et ses tables.

La page Connexion de l'administrateur du serveur Oracle s'affiche dans les deux cas suivants :

  • Lorsque vous créez une base de données Content Server. La page apparaît une fois que vous cliquez sur le bouton Continuer de la page Sélectionner le type de SGBDR.

  • Vous avez cliqué sur le lien Tâches de maintenance Serveur Oracle de la page Maintenance de la base de données active.

Sur cette page, vous devez indiquer le nom d'utilisateur et le mot de passe d'un compte SGBDR détenant les privilèges Administrateur.
