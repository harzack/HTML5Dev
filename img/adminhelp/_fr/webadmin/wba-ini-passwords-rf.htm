Ces paramètres apparaissent dans la section [Passwords] par défaut. Pour les définir, OpenText vous recommande d'utiliser la page Configurer les paramètres du mot de passe, plutôt que de modifier directement le fichier opentext.ini. Pour en savoir plus, voir Configuration des paramètres utilisateur.

 

Note :
Les paramètres par défaut répertoriés ci-dessous sont applicables aux nouvelles installations Content Server, si Content Server est mis à niveau à partir d'une version précédente et si les valeurs des mots de passe ont été sauvegardées, les valeurs sont conservées dans la version mise à niveau.

Selon les autres paramètres de mot de passe, des paramètres supplémentaires peuvent apparaître dans la section [Passwords]. Pour plus d'informations sur ces paramètres, reportez-vous à la rubrique Configuration des paramètres utilisateur.

 

MaxPasswordLength

  • Description :

    Le nombre maximum de caractères autorisés pour un mot de passe utilisateur est défini sur 16 par défaut. Pour modifier le nombre par défaut de caractères autorisés pour les mots de passe utilisateur, vous devez ajouter manuellement le paramètre MaxPasswordLength à la section [Passwords] du fichier opentext.ini.

  • Syntaxe :

    MaxPasswordLength=<entier pour le nombre maximum de caractères autorisés dans les mots de passe utilisateur>

MinimumLength

  • Description :

    Nombre minimum de caractères requis pour le mot de passe d'un utilisateur.

  • Syntaxe :

    MinimumLength=6

  • Valeurs :

    Un nombre entier compris entre 0 et 16. La valeur par défaut est 6. Si 0 est sélectionné, un champ de mot de passe vierge est accepté. La valeur maximale est 16.

MustContainDigits

  • Description :

    Définit si les mots de passe doivent contenir au moins un caractère numérique.

  • Syntaxe :

    MustContainDigits=TRUE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est TRUE, ce qui signifie qu'un caractère numérique est requis.

ExpirationDays

  • Description :

    Nombre de jours avant l'expiration des mots de passe utilisateur.

  • Syntaxe :

    ExpirationDays=30

  • Valeurs :

    Un entier supérieur ou égal à -1. La valeur par défaut est 30. Une valeur de -1 indique que les mots de passe n'expirent jamais.

ChangePWAtFirstLogin

  • Description :

    Définit si les utilisateurs doivent modifier leurs mots de passe après leur première connexion à Content Server.

  • Syntaxe :

    ChangePWAtFirstLogin=TRUE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est TRUE, ce qui signifie que les utilisateurs modifient leurs mots de passe après leur première connexion à Content Server.

MustBeDifferent

  • Description :

    Définit si les utilisateurs doivent spécifier un nouveau mot de passe lorsqu'ils modifient leur mot de passe.

  • Syntaxe :

    MustBeDifferent=TRUE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est TRUE, ce qui signifie que les mots de passe modifiés ne peuvent pas être identiques aux mots de passe précédents.

preventPwdReuse

  • Description :

    Détermine le nombre de jours avant qu'un mot de passe utilisé précédemment puisse être réutilisé.

  • Syntaxe :

    preventPwdReuse=60

  • Valeurs :

    Un entier supérieur ou égal à zéro. La valeur par défaut est 60. Une valeur définie sur 0 indique que le même mot de passe peut être utilisé immédiatement.
