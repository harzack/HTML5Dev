La section [Client] du fichier opentext.ini contient des options propres à la configuration des clients Content Server.

ErrorRedirectURL

  • Description :

    URL vers laquelle les utilisateurs sont redirigés lorsqu'ils tentent de se connecter à Content Server alors que le serveur Livelink n'est pas en cours d'exécution. Les informations qui s'affichent sur la page par défaut sont transmises dans une variable CGI nommée errid. Les connexions client Servlet ne sont pas redirigées.

  • Syntaxe :

    ErrorRedirectURL=http://www.opentext.com

  • Valeurs :

    Toute URL valable. Par défaut, le paramètre ErrorRedirectURL ne figure pas dans le fichier opentext.ini.

ErrorStatus

  • Description :

    Lorsque le serveur n'est pas en cours d'exécution, ce code est envoyé dans l'en-tête HTTP.

  • Syntaxe :

    ErrorStatus=503

  • Valeurs :

    Un code HTTP valide. Par défaut, le paramètre ErrorStatus ne figure pas dans le fichier opentext.ini.

StrictClientParse

  • Description :

    Impose aux clients d'être stricts lors de l'analyse d'une requête Web d'entrée. Un message d'erreur s'affiche, évitant ainsi au serveur de recevoir des requêtes incomplètes.

  • Syntaxe :

    StrictClientParse=TRUE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est TRUE.

    La définition de ce paramètre sur FALSE n'impose pas au client d'être strict lors de l'analyse d'une requête Web d'entrée.

ReceiveBeforeSend

  • Description :

    Lorsque ce paramètre est défini sur TRUE, il transmet la charge du téléchargement des fichiers au serveur Web. Ceci libère des threads Content Server plus rapidement pour exécuter d'autres fonctions.

  • Syntaxe :

    ReceiveBeforeSend=FALSE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est FALSE.

    La définition de ce paramètre sur TRUE transmet la charge du téléchargement des fichiers au serveur Web.

MaxHeaderBytes

  • Description :

    Définit la taille maximum des en-têtes HTTP en octets. Content Server utilise ce paramètre pour limiter la taille des en-têtes HTTP. Si la taille réelle de l'en-tête HTTP envoyé par Content Server est supérieure à cette limite (si l'en-tête HTTP contient un cookie de grande taille, par exemple), le navigateur peut être incapable d'afficher les pages Web Content Server.

    Par défaut, le paramètre MaxHeaderBytes ne figure pas dans le fichier opentext.ini.

  • Syntaxe :

    MaxHeaderBytes=2000

  • Valeurs :

    Un entier positif représentant la taille maximum des en-têtes en octets. La valeur par défaut est 2000.
