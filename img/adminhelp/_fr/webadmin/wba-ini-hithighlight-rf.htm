La section [HitHighlight] contient les paramètres de configuration pour la mise en évidence des résultats dans Content Server. La mise en évidence des résultats permet aux utilisateurs de mettre en évidence les occurrences des termes recherchés au sein d'un élément de résultat de recherche particulier. Pour en savoir plus, voir Utilisation des résultats de la recherche.

Avant de pouvoir mettre en évidence un élément de résultat de recherche, Content Server convertit l'élément au format HTML à l'aide d'un processus nommé wfwcnv. Dans cette section du fichier opentext.ini, vous pouvez configurer l'environnement dans lequel le processus wfwcnv s'exécute.

cachePath

  • Description :

    Indique le répertoire dans lequel les fichiers convertis générés par le processus wfwcnv sont mis en mémoire cache.

  • Valeurs :

    Un chemin absolu. La valeur par défaut est <Content Server_home>\cache\hh\, où <Content Server_accueil> est la racine de votre installation Content Server.

hhAdminServer

  • Description :

    Indique le raccourci du serveur d'administration permettant de gérer le processus wfwcnv. Un raccourci de serveur d'administration est identique au nom qui apparaît dans la colonne Nom du tableau Serveurs d'administration sur la page Volume de l'objet système. Pour en savoir plus, voir Utilisation du volume des objets système.

  • Valeurs :

    Le raccourci d'un serveur d'administration enregistré avec l'hôte Content Server principal. Par défaut, le paramètre hhAdminServer ne figure pas dans le fichier opentext.ini, ce qui équivaut à hhAdminServer=default.

HHStyle

  • Description :

    Indique la couleur de l'arrière-plan ainsi que la couleur, la taille, le style et l'épaisseur de la police des résultats mis en évidence pour un mot ou une locution clé. Un exemple de style de police est l'italique. Un exemple d'épaisseur de police est le gras.

  • Syntaxe :

    HHStyle={background:red; color:blue}

  • Valeurs :

    Les valeurs définies pour ce paramètre doivent être comprises entre accolades, {}. Chaque entrée se présente sous la forme<option> :<valeur>. Les entrées sont séparées par des points-virgules.

    Le paramètre HHStyle peut contenir cinq options d'affichage : background, color, font-size, font-style ou font-weight. Les valeurs de chacun de ces éléments sont décrites dans le tableau suivant :

    Option Valeur
    background Toute couleur HTML valable. Cette valeur peut être un nom de couleur ou une spécification RVB numérique. Pour plus d'informations, reportez-vous à un tableau ou à un site Internet référençant les couleurs HTML.
    couleur Toute couleur HTML valable. Cette valeur peut être un nom de couleur ou une spécification RVB numérique. Pour plus d'informations, reportez-vous à un tableau ou à un site Internet référençant les couleurs HTML.
    font-size Un entier représentant la taille absolue de la police.
    font-style Trois valeurs sont à votre disposition : normal, italic ou oblique, selon la famille de police utilisée.

hhwfwcnvdir

  • Description :

    Indique le répertoire dans lequel démarre le processus wfwcnv.

  • Valeurs :

    Un chemin absolu. Par défaut, le paramètre hhwfwcnvdir n'apparaît pas dans le fichier opentext.ini, qui est équivalent à hhwfwdnvdir=<Content Server_home>/bin/, où <Content Server_accueil> est la racine de votre installation Content Server.

MaxNumberOfCacheItems

  • Description :

    Indique le nombre maximum de fichiers de conversion pouvant être stockés dans la mémoire cache. Lorsque le nombre d'éléments mis en mémoire cache dépasse la limite fixée, les éléments les moins récemment consultés sont supprimés.

    Un élément placé en mémoire cache se compose d'un ensemble de fichiers convertis associés à un document source donné. Par exemple, si un document source non-HTML contient du texte et des images et que vous le convertissez au format HTML, vous obtenez un fichier HTML et au moins un fichier image. Cet ensemble de fichiers représente l'élément placé dans la mémoire cache pour ce document source.

  • Valeurs :

    Un entier supérieur ou égal à zéro. Par défaut, le paramètre MaxNumberOfCacheItems ne figure pas dans le fichier opentext.ini, ce qui équivaut à MaxNumberOfCacheItems=30.

sectionName

  • Description :

    Indique la section opentext.ini à partir de laquelle le processus wfwcnv lit ses paramètres de filtrage.

  • Valeurs :

    Un nom de section opentext.ini valable. Le nom de la section ne doit pas contenir de crochets droits  [ ] . Par défaut, le paramètre sectionName ne figure pas dans le fichier opentext.ini, ce qui équivaut à sectionName=HHFilterSettings.
