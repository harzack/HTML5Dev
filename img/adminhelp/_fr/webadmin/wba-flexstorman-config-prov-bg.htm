Les fournisseurs de stockage vous permettent de stocker des documents à l'extérieur de Content Server. Les exemples de fournisseurs de stockage incluent :

  • des répertoires externes ;

  • d'autres types de systèmes de stockage tels que des bases de données ;

  • un serveur d'archives OpenText.

Chaque instance Content Server utilisant le stockage flexible dispose d'un fournisseur de stockage par défaut. Celui-ci ne peut pas être modifié ou supprimé. Si aucun fournisseur de stockage n'est spécifié, tous les documents ajoutés à Content Server utilisent le fournisseur de stockage par défaut.

 

Note :
Vous ne pouvez pas supprimer un fournisseur de stockage s'il est utilisé.

 

Après avoir ajouté un fournisseur de stockage, vous devez :

  • fournir un nom unique pour votre fournisseur de stockage ;

  • fournir le chemin absolu de votre fournisseur de stockage ;

  • définir et ordonner les règles associées avec votre fournisseur de stockage.

Lorsqu'un document est ajouté à Content Server, le système détermine le fournisseur de stockage à utiliser en fonction des règles définies. Pour plus d'informations sur les règles des fournisseurs de stockage, reportez-vous à la rubrique Configuration des règles de stockage.

Les administrateurs peuvent vérifier le contenu déplacé d'un fournisseur de stockage vers un autre. Par défaut, Fournisseur modifié est désactivé. Les détails de l'audit pour cet événement incluent le nom du fournisseur de stockage d'origine, le nom du nouveau fournisseur de stockage et le nom de l'utilisateur qui a déplacé le contenu. Dans Content Server, chaque document affiche le nom de son fournisseur de stockage dans l'onglet Versions de la page Propriétés du document. Les utilisateurs dotés des autorisations appropriées peuvent voir la liste complète des règles du fournisseur de stockage. Cette liste comprend la règle du fournisseur de stockage et le nom et le type du fournisseur de stockage.
