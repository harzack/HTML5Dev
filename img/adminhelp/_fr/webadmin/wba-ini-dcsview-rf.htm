Les paramètres de la section [DCSview] définissent les fonctions d'affichage et de mise en évidence des documents par le service de conversion de documents, DCS, dans Content Server. Le DCS convertit des documents au format HTML ou texte brut pour permettre leur affichage et indexation. Les DCS sont gérés par les serveurs d'administration. Les paramètres de la section [DCSview] sont utilisés uniquement lorsque le serveur d'administration n'est pas en cours d'exécution ou que le DCS géré par le serveur d'administration n'est pas activé. Vous pouvez également configurer ce service lors de la configuration d'un serveur d'administration. Les paramètres définis lors de la configuration d'un serveur d'administration remplacent les valeurs de la section [DCSview] du fichier opentext.ini. Pour plus d'informations sur les paramètres que vous pouvez configurer en même temps qu'un serveur d'administration, voir Configuration des paramètres et réglages du serveur.

Certains paramètres de la section [DCSview] du fichier opentext.ini sont également présents dans la section [FilterEngine]. Les paramètres [DCSview] exécutent la même fonction que les paramètres [FilterEngine] correspondants. Cependant, les valeurs définies pour les paramètres de la section [DCSview] écrasent celles des paramètres de la section [FilterEngine]. Si aucune valeur n'est définie dans la section [DCSview], la valeur utilisée est celle de la section [FilterEngine].

logLevel

  • Description :

    Indique les événements et les informations devant figurer dans le fichier journal.

  • Syntaxe :

    logLevel=1

  • Valeurs :

    Un entier 0, 1, 2, 3 ou 4. La valeur par défaut est 1.

    Le tableau suivant contient des descriptions des valeurs valides pour le paramètre logLevel.

    Valeur Description
    0 Désactive la consignation dans le journal.
    1 Seuls les erreurs et les documents ne pouvant pas être indexés sont enregistrés. Il s'agit de la valeur par défaut.
    2 Consigne tous les éléments provenant de logLevel=1, ainsi que les événements DCS. Ces événements comprennent notamment le chargement d'un filtre de conversion et l'arrêt du processus de conversion.
    3 Consigne tous les éléments provenant de logLevel=2, ainsi que les statistiques de conversion de documents et les informations sur les documents. Parmi ces informations figurent l'heure de la conversion, la taille du fichier d'entrée, le type MIME, la balise OTURN et le filtre de conversion utilisé pour convertir le document.
    4 Consigne tous les éléments provenant de logLevel=3, ainsi que les événements du traitement IPool et les messages du journal générés par les filtres de conversion. Ce niveau ne doit être utilisé que pour identifier les problèmes survenant dans le flux de données.

mod

  • Description :

    Détermine la section du fichier opentext.ini contenant la configuration d'un filtre de conversion. La section [DCSview] peut contenir plusieurs paramètres mod correspondant à différents filtres de conversion.

     

    Attention
    La modification de la valeur de ce paramètre risque d'empêcher le DCS de fonctionner ou de fonctionner correctement. Ne modifiez pas les valeurs de ces paramètres, sauf si le service d'assistance clientèle d'OpenText vous le demande.

     

  • Valeurs :

    Le tableau suivant présente les valeurs par défaut pour le paramètre mod.

    Nom Valeur
    modx02 QDF
    modx03 DCStext
    modx04 DCSxpdf
    modx06 DCSmail

     

    Note :
    Le caractère x indique que le processus du filtre de conversion sera exécuté au sein d'un processus de traitement.

     

modreq

  • Description :

    Détermine le gestionnaire de requêtes utilisé par le DCS. Le gestionnaire de requêtes accepte des documents d'entrée d'un type de source spécifique (IPool, mécanisme d'échange ou canal, par exemple) et envoie les documents au DCS afin d'effectuer la conversion. Une fois la conversion effectuée, les documents convertis sont renvoyés au gestionnaire de requêtes et envoyés à leur destination (un client ou un IPool, par exemple).

     

    Attention
    La modification de la valeur de ce paramètre risque d'empêcher le DCS de fonctionner ou de fonctionner correctement. Ne modifiez pas la valeur de ce paramètre, sauf si le service d'assistance clientèle d'OpenText vous le demande.

     

  • Valeurs :

    La valeur par défaut est DCSpipe.

modx10

  • Description :

    Ce paramètre est requis pour utiliser le filtre OTDF avec Content Server sur la plate-forme Linux®.

  • Syntaxe :

    modx10=DCSIm

  • Valeurs :

    La seule valeur autorisée est DCSIm.

NumThreads

  • Description :

    Détermine le nombre maximum de conversions de documents pouvant être effectuées simultanément par le DCS. La modification de la valeur par défaut influence les performances de la conversion de documents.

  • Syntaxe :

    NumThreads=1

  • Valeurs :

    Un entier compris entre 1 et 64. La valeur par défaut est 1.

QueueSize

  • Description :

    Détermine le nombre maximum de documents mis en file d'attente par le DCS lorsque le nombre maximum de conversions de documents simultanées a été atteint.

     

    Important :
    OpenText vous recommande vivement de ne pas modifier la valeur de ce paramètre.

     

  • Valeurs :

    Un entier. La valeur par défaut est 50.

rulesfile

  • Description :

    Détermine le nom du fichier qui définit les règles du traitement de la conversion de documents de différents types MIME.

     

    Important :
    OpenText vous recommande vivement de ne pas modifier la valeur de ce paramètre.

     

  • Valeurs :

    Un chemin absolu. La valeur par défaut est <Content Server_home> \config\dcsviewrules.txt, où <Content Server_accueil> est la racine de votre installation Content Server.

dllpath

  • Description :

    Détermine l'emplacement du répertoire contenant les filtres de conversion de Content Server. Un document créé par un traitement de texte, Microsoft Word par exemple, contient des informations de formatage qui ne sont pas nécessaires ou requises pour la recherche. Les filtres de conversion extraient le contenu de texte nécessaire pour lire des fichiers de traitement de texte et effectuer une indexation à partir de ces fichiers.

  • Syntaxe :

    dllpath=C:\OPENTEXT\filters

  • Valeurs :

    Un chemin absolu. La valeur par défaut est <Content Server_home> \filters, où <Content Server_accueil> est la racine de votre installation Content Server.

logfile

  • Description :

    Détermine l'emplacement et le préfixe du fichier journal de conversion de documents. Le numéro de port admin attribué à ce processus est ajouté au fichier journal, suivi du suffixe .log. Ceci empêche que différents processus écrivent dans le même fichier.

  • Syntaxe :

    logfile=C:\OPENTEXT\logs\dcsview

  • Valeurs :

    Un chemin absolu. La valeur par défaut est <Content Server_home> \logs\dcsview, où <Content Server_accueil> est la racine de votre installation Content Server.

     

    Note :
    Ce paramètre est obligatoire. Pour désactiver la journalisation, définissez logLevel sur 0.
