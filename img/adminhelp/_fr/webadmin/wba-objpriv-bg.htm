Content Server Les privilèges de l'objet régulent la possibilité des utilisateurs à créer certains éléments Content Server.

Par défaut, tous les objets Content Server, excepté les types d'objet Catégorie, Rapport LiveReport, URL, Vue personnalisée et Apparence, sont sans restriction, ce qui signifie que les utilisateurs et les groupes disposent des privilèges nécessaires pour créer les types d'élément dans Content Server. Seul l'utilisateur administrateur possède initialement les privilèges nécessaires pour créer des rapports LiveReport, des URL et des vues personnalisées. Ces paramètres peuvent être modifiés en restreignant ou en ne restreignant pas le type d'objet pour des groupes ou des utilisateurs spécifiques.

Un utilisateur détenant le privilège de création d'élément pour les rapports LiveReport peut exécuter des instructions SQL sur le SGBDR, y compris celles qui modifient la base de données Content Server. Pour maintenir l'intégrité de votre base de données Content Server, il est recommandé de restreindre le privilège de création de rapports à l'utilisateur Admin ou à un petit nombre d'utilisateurs ayant une bonne connaissance de SQL et du schéma Content Server. Dans un souci de protection de votre base de données, l'utilisateur Content Server qui a l'autorisation Modifier sur des rapports LiveReport doit en même temps détenir le privilège de création d'éléments LiveReport.

 

Note :
Les modules facultatifs installés disposent de leurs propres restrictions et paramètres par défaut. Les modules facultatifs peuvent restreindre les privilèges relatifs aux objets et à l'utilisation par défaut.
