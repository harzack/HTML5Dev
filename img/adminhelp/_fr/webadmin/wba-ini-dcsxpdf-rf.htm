La section [DCSxpdf] définit les paramètres du filtre XPDF. Le service de conversion de documents, DCS, utilise le filtre XPDF pour convertir des fichiers PDF au format texte brut en vue de leur indexation.

La section [DCSxpdf] du fichier opentext.ini contient des informations sur les paramètres suivants :

•

dllpath

•

lib
•

startdir

•

x-maxcalls
•

x-timeout

dllpath

  • Description :

    Indique le chemin d'accès au répertoire où sont installées les ressources du filtre XPDF.

  • Syntaxe :

    dllpath=C:\OPENTEXT\filters\xpdf

  • Valeurs :

    Un chemin absolu. La valeur par défaut est <Content Server_home>\filters\xpdf, où <Content Server_accueil> est la racine de votre installation Content Server.

lib

  • Description :

    Détermine le nom de la bibliothèque que le DCS doit charger. Une bibliothèque est une liste d'opérations associées à un filtre de conversion que le DCS lit pour effectuer la conversion.

     

    Attention
    La modification de la valeur de ce paramètre risque d'empêcher le DCS de fonctionner ou de fonctionner correctement. Ne modifiez pas la valeur de ce paramètre, sauf si le service d'assistance clientèle d'OpenText vous le demande.

     

  • Valeurs :

    La valeur par défaut est dcsxpdf.

startdir

  • Description :

    Indique le chemin d'accès au répertoire à partir duquel le filtre XPDF est exécuté.

  • Syntaxe :

    startdir=C:\OPENTEXT\filters\xpdf

  • Valeurs :

    Un chemin absolu. La valeur par défaut est <Content Server_home>\filters\xpdf, où <Content Server_accueil> est la racine de votre installation Content Server.

x-maxcalls

  • Description :

    Indique le nombre de réutilisations d'un processus pour le traitement des documents. Pendant la conversion de documents, le DCS charge un processus de traitement. Le processus de traitement charge le filtre de conversion approprié et l'utilise pour convertir le document. Pour améliorer les performances, le DCS réutilise le processus de traitement pour plusieurs conversions. Si le processus de traitement rencontre une erreur, il s'interrompt avant d'atteindre la valeur définie dans le fichier opentext.ini.

     

    Important :
    OpenText vous recommande vivement de ne pas modifier la valeur de ce paramètre.

     

  • Valeurs :

    La valeur par défaut est héritée du paramètre maxcalls de la section [DCSworker] du fichier opentext.ini.

     

    Note :
    Le paramètre x-maxcalls est actif uniquement lorsque le filtre de conversion qu'il modifie est géré par un processus de traitement. Pour plus d'informations sur la configuration d'un processus de traitement, reportez-vous à la rubrique [DCS].

    Si vous définissez une valeur pour x-maxcalls, la valeur de maxcalls sera écrasée dans la section [DCSworker] du fichier opentext.ini.

     

x-timeout

  • Description :

    Indique le temps d'attente maximum (en secondes) avant la fin d'un processus de traitement de conversion de documents. Configurez ce paramètre lorsque la valeur par défaut définie dans le paramètre timeout n'est pas valable. Vous configurez le paramètre timeout pour chaque filtre de conversion. Par exemple, certains filtres de conversion sont plus lents que d'autres. Dans ce cas, la valeur timeout par défaut de 30 secondes peut ne pas être applicable, car le délai de conversion moyen est supérieur à cette valeur. Dans ce cas, vous pouvez modifier la valeur x-timeout en choisissant une valeur plus élevée.

     

    Important :
    OpenText vous recommande vivement de ne pas modifier la valeur de ce paramètre.

     

  • Valeurs :

    La valeur par défaut est héritée du paramètre timeout de la section [DCSworker] du fichier opentext.ini.

     

    Note :
    Le paramètre x-timeout est actif uniquement lorsque le filtre de conversion qu'il modifie est géré par un processus de traitement. Pour plus d'informations sur la configuration d'un processus de traitement, reportez-vous à la rubrique [DCS].

    Si vous définissez une valeur pour x-timeout, la valeur de timeout sera écrasée dans la section [DCSworker] du fichier opentext.ini.
