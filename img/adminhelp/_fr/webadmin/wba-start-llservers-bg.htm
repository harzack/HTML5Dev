Cette section fait référence à deux serveurs : le serveur Content Server et le serveur d'administration. Si vous conservez les paramètres par défaut pendant l'installation de Content Server, vos serveurs ont les noms par défaut suivants :

  • Content Server (OTCS) : également appelé Content Server dans l'aide.

  • Content Server Admin (OTCS) : également appelé serveur d'administration dans l'aide.

A moins que vous ne spécifiiez autre chose, les deux serveurs démarrent automatiquement au cours de l'installation et de la configuration de Content Server.

Certaines tâches d'administration du système nécessitent l'arrêt et le redémarrage de Content Server pour que les modifications puissent prendre effet. Par défaut, le programme d'installation Content Server définit le démarrage automatique des serveurs Content Server et d'administration. OpenText vous recommande de ne pas modifier ce paramètre car ces serveurs doivent être exécutés pour que Content Server fonctionne correctement. Dans certains cas cependant, par exemple lorsque vous tentez de résoudre un problème, il peut arriver que vous ne souhaitiez pas démarrer automatiquement un ou plusieurs serveurs lors de la réinitialisation de l'hôte Content Server principal. Vous pouvez alors provisoirement choisir de démarrer manuellement les deux serveurs, mais vous devrez ensuite reconfigurer le démarrage automatique.

 

Note :
A chaque fois que vous redémarrez Content Server, n'oubliez pas de redémarrer également le serveur d'application Web.

 

Vous pouvez configurer le démarrage automatique ou manuel des services Content Server et des services du serveur d'administration lorsque leur ordinateur hôte démarre. Les procédures de modification de ces paramètres sont très différentes pour Windows et UNIX.

Dans cette section, les noms des serveurs utilisés dans les exemples sont les noms par défaut.
