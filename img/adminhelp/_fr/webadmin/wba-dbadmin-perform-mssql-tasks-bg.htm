La page Connexion de l'administrateur de Microsoft SQL Server s'affiche dans les cas suivants :

  • Lorsque vous créez une base de données Content Server. La page apparaît une fois que vous cliquez sur le bouton Continuer de la page Sélectionner le type de SGBDR.

  • Lorsque vous avez cliqué sur le lien Tâches de maintenance Microsoft SQL Server sur la page Maintenance de la base de données active.

Sur cette page, vous devez indiquer le nom d'utilisateur et le mot de passe d'un compte SGBDR détenant les privilèges Administrateur.

Utilisez la page Maintenance Microsoft SQL Server pour effectuer les tâches suivantes :

  • créer une nouvelle base de données Microsoft SQL Server ;

  • créer un nouvel utilisateur ;

  • agrandir une base de données Microsoft SQL Server ;

  • supprimer des tables Microsoft SQL Server Content Server ;

  • supprimer une base de données Microsoft SQL Server.

Utilisez la page Maintenance Microsoft SQL Server dans les deux cas suivants :

  • Lorsque vous administrez une base de données Content Server. Dans ce cas, vous accédez à la page Maintenance Microsoft SQL Server en cliquant sur le lien Tâches de maintenance Microsoft SQL Server de la page Maintenance de la base de données active.

Dans ces deux cas, vous devez cliquer sur le lien Revenir à la page précédente de la page Maintenance Microsoft SQL Server pour revenir à la page initiale (Maintenance de la base de données active ou Créer des tables Content Server) à partir de laquelle vous aviez affiché la page Maintenance Microsoft SQL Server.

Suppression de tables de la base de données Content Server ou d'une base de données SQL Server

Si, pour une raison quelconque, votre base de données Content Server est endommagée, la seule manière de la récupérer peut consister à supprimer ses tables, voire la base de données SQL Server dans laquelle elles résident, puis à effectuer une restauration à partir d'une sauvegarde.

Agrandissement d'une base de données SQL Server

Lorsque la quantité de données de la base de données du serveur SQL occupera presque tout l'espace alloué dans le fichier de données, vous devrez augmenter l'espace alloué au fichier de données pour pouvoir agrandir la base de données. Vous pouvez attribuer de l'espace dans plusieurs fichiers de données à une base de données SQL.
