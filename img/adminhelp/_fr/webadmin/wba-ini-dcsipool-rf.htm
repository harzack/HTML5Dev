La section [DCSipool] définit les options spécifiques à la configuration des IPools du service de conversion de documents (DCS). Le DCS convertit des documents au format HTML ou texte brut pour permettre leur affichage et indexation. Les IPools sont des zones de stockage temporaires qui relient les processus en un flux de données. Au fur et à mesure que les données circulent par l'intermédiaire d'un flux de données, elles sont déposées dans les IPools. Le DCS lit les données des IPools afin de les traiter et de les convertir au format HTML.

Certains paramètres de la section [DCSipool] du fichier opentext.ini sont également présents dans la section [FilterEngine]. Les paramètres [DCSipool] exécutent la même fonction que les paramètres [FilterEngine] correspondants. Cependant, les valeurs définies pour les paramètres de la section [DCSipool] écrasent celles des paramètres de la section [FilterEngine]. Si aucune valeur n'est définie dans la section [DCSipool], la valeur utilisée est celle de la section [FilterEngine].

lib

  • Description :

    Détermine le nom de la bibliothèque que le DCS doit charger. Une bibliothèque est une liste d'opérations associées à un filtre de conversion que le DCS lit pour effectuer la conversion.

     

    Attention
    La modification de la valeur de ce paramètre risque d'empêcher l'initialisation du DCS. Ne modifiez pas la valeur de ce paramètre.

     

  • Valeurs :

    La valeur par défaut est dcsipool.

MaxMetaSize

  • Description :

    Indique la quantité maximum, en Mo, de métadonnées pouvant être enregistrées lorsque le document correspondant est en cours de conversion. Les métadonnées supérieures à cette valeur seront enregistrées dans un fichier temporaire jusqu'à la conversion du document associé.

     

    Important :
    OpenText vous recommande vivement de ne pas modifier la valeur de ce paramètre.

     

  • Valeurs :

    Un entier supérieur à un. La valeur par défaut est 8.

     

    Note :
    Si vous indiquez une valeur supérieure à la valeur par défaut, le DCS risque d'utiliser plus de mémoire que nécessaire.

     

maxrequests

  • Description :

    Indique le nombre de documents que le DCS lit depuis l'IPool.

     

    Important :
    OpenText vous recommande vivement de ne pas modifier la valeur de ce paramètre.

     

  • Valeurs :

    La valeur doit correspondre au paramètre NumThreads de la section [DCS] du fichier opentext.ini afin que le processus d'indexation puisse se synchroniser correctement. Pour plus d'informations, référez-vous aux rubriques[DCS]

     

    Note :
    Ce paramètre n'est pas défini dans le fichier opentext.ini par défaut. Lorsqu'aucune valeur n'est définie, la valeur par défaut correspond au paramètre défini pour le paramètre NumThreads dans la section [DCS].

     

MaxFileSize

  • Description :

    Indique la taille maximum, en Mo, d'un fichier pouvant être converti en mémoire.

  • Syntaxe :

    MaxFileSize=8

  • Valeurs :

    Un entier supérieur à un. La valeur par défaut est 8.

fenêtre

  • Description :

    Définit le nombre maximum de messages IPool à traiter à partir de la zone de lecture IPool avant que le DCS exécute une transaction IPool.

    Ce paramètre est le premier d'un ensemble de trois mécanismes de limitation du débit compris dans la section [FilterEngine] du fichier opentext.ini. Il est suivi des paramètres minwindow et maxtime.

  • Syntaxe :

    window=10

  • Valeurs :

    Un entier supérieur ou égal à un. La valeur par défaut est 10.

     

    Note :
    Si cette valeur est plus élevée que la valeur par défaut, le débit du filtre de conversion augmente mais le DCS utilise davantage d'espace disque.

     

minwindow

  • Description :

    Définit le nombre minimum de messages IPool à traiter à partir de la zone de lecture IPool avant que le DCS exécute une transaction IPool. Si le nombre de messages disponibles dans la zone de lecture est inférieur à la valeur indiquée pour le paramètre minwindow, la transaction sera exécutée après le traitement du dernier message iPool par le DCS.

    Ce paramètre est le deuxième d'un ensemble de trois mécanismes de limitation du débit compris dans la section [FilterEngine] du fichier opentext.ini. Il est précédé du paramètre window et suivi du paramètre maxtime.

  • Syntaxe :

    minwindow=5

  • Valeurs :

    Un entier supérieur ou égal à un. La valeur par défaut est 5.

maxtime

  • Description :

    Indique la durée maximum, en millisecondes, nécessaire au traitement d'une transaction IPool. Lorsqu'une transaction dépasse cette durée, elle est exécutée après le traitement complet du message iPool actif, que les critères window ou minwindow aient été respectés ou non.

    Ce paramètre est le dernier d'un ensemble de trois mécanismes de limitation du débit compris dans la section [FilterEngine] du fichier opentext.ini. Il est précédé des paramètres window et minwindow.

  • Syntaxe :

    maxtime=120000

  • Valeurs :

    Un entier supérieur ou égal à un. La valeur par défaut est 120000, soit deux minutes.
