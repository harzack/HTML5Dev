Un rapport système contient des informations détaillées sur votre système Content Server. Il existe deux types de rapports système : le Rapport système Lite et le Rapport système complet. Chaque rapport contient des informations sur les éléments suivants :

  • Système

  • Base de données active

  • Serveur de la base de données

  • Application

  • Utilisateurs et groupes

  • Types de nœud

  • Stockage externe

  • Attributs de date obsolètes

  • OSpaces

  • XDBs

  • Gestionnaires de requêtes

  • Modules personnalisés

  • OSpaces personnalisés

  • Table AgentSchedule

  • Table KIni

  • Informations sur la table LLSystemData

  • OpenText.ini

  • Répertoire d'installation

  • Rapports personnalisés supplémentaires

Le rapport système complet contient également des informations sur les éléments suivants :

  • Versions des nœuds

  • Tables Content Server

  • Colonnes de la table

  • Index

  • espace de stockage

Le rapport généré est un document texte, intitulé sysreport.txt, qui réside dans le répertoire logs de votre installation Content Server. Une fois que vous avez généré un rapport système, le chemin d'accès à l'emplacement où réside le fichier apparaît dans la section Chemin d'accès au fichier de la page Rapport du système Content Server. Vous pouvez cliquer sur le lien du rapport système pour y accéder.

Si un rapport système a déjà été généré, un lien vers le rapport le plus récemment généré apparaît dans l'en-tête de la page Rapport du système. Pour obtenir la version la plus récente du rapport système, vous devez de nouveau générer le rapport.
