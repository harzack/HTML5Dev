Installation d'un module

L'installation d'un module se déroule en deux phases, qui sont les suivantes :

  • Tout d'abord, utilisez le programme InstallShield du module ou le fichier .tar pour démarrer l'installation. Ce processus extrait les fichiers et les place dans le répertoire Content Server_home\staging.

  • Installez ensuite le module sur votre service Content Server actuel en accédant à la page Administration, sélectionnant Administration du module puis Installer des modules.

Lorsque vous commencez l'installation, vous devez sélectionner l'instance de Content Server sur laquelle vous souhaitez installer le module. Vous pouvez uniquement installer un module sur une instance de Content Server à la fois.

Pour la première phase de l'installation d'un module, si vous installez des modules facultatifs Content Server pendant l'installation initiale de Content Server, suivez les instructions du manuel Content Server First-Time Installation Guide (Manuel de l'installation initiale de Content Server) pour terminer l'installation de Content Server.

 

Note :
Si un module est associé à des fichiers d'aide en ligne, vous devez mettre à jour l'index de l'aide après l'installation, la désinstallation ou la mise à niveau d'un module pour que les changements soient reflétés dans l'aide en ligne.

Si vous avez installé des modules facultatifs durant l'installation de Content Server, l'index de l'aide est automatiquement généré en fonction de tous les modules installés.

 

Installation des modules sous Windows

Pour les versions Windows des modules Content Server, effectuez la première phase de l'installation en utilisant un programme d'installation InstallShield. Celui-ci vous permet de sélectionner l'instance de Content Server pour laquelle vous voulez ajouter le module et de placer les fichiers correspondants dans un sous-répertoire du répertoire Content Server_home\staging.

Le répertoire Content Server_home\staging n'existe pas si aucun module facultatif n'est encore installé. Dans ce cas, le programme d'installation du module crée automatiquement le répertoire Content Server_home\staging.

Installation de modules sous UNIX

Pour les versions UNIX des modules Content Server, vous effectuez la première phase de la mise à niveau en utilisant un fichier d'archive compressé au format .tar. L'extraction de ce fichier .tar place les fichiers du module dans un sous-répertoire du répertoire Content Server_home/staging.

Le répertoire Content Server_home/staging n'existe pas si aucun module facultatif n'est encore installé. Dans ce cas, l'extraction du fichier .tar crée automatiquement le répertoire Content Server_home/staging.

Désinstallation d'un module

Content Server vous permet de désinstaller les modules facultatifs installés, tels que Content Server Forms ou Content Server Explorer. Vous pouvez également désinstaller certains modules préinstallés ; cependant, OpenText vous recommande vivement de ne pas le faire sauf si vous y êtes invité par le service d'assistance clientèle d'OpenText.

Avant d'accéder à la page Désinstaller des modules, vous pouvez afficher une liste de tous les modules Content Server pouvant être désinstallés de votre système, ainsi que des modules qui en dépendent, sur la page Afficher les modules pouvant être désinstallés.

Lorsque vous accédez à la page Désinstaller, si d'autres modules dépendent du module sélectionné, le bouton Désinstaller n'est pas disponible. Vous devez désinstaller les modules répertoriés dans la colonne Dépendants avant de pouvoir désinstaller le module dont ils dépendent.

Une fois le processus de désinstallation terminé, vous devez redémarrer vos serveurs et d'application Web.
