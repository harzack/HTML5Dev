Vous pouvez modifier les paramètres suivants du Content Server sur la page Configurer les paramètres du serveur :

  • Préfixe URL du répertoire /support

    Le préfixe URL, également appelé l'alias du répertoire virtuel, est mappé au répertoire support lors de l'installation de Content Server. Par défaut, le répertoire support se trouve à la racine du niveau d'installation : /installation_path/support. Il n'est normalement pas nécessaire de modifier le préfixe URL, mais si vous le modifiez, vous devez effectuer la modification dans Content Server et dans le serveur Web.

    La valeur par défaut du préfixe URL est /img/.

     

    Note :
    Le préfixe URL doit être précédé et suivi d'une barre oblique (/).

     

  • Content Server Mot de passe de l'administrateur

    Le mot de passe de l'administrateur est celui que vous utilisez pour accéder aux pages Administration. Attention, ne confondez pas ce mot de passe avec celui de l'utilisateur Admin. Après l'installation Content Server initiale, il est conseillé de modifier le mot de passe administrateur par défaut. Vous pouvez définir un nouveau mot de passe administrateur sur la page Configurer les paramètres du serveur en sélectionnant la case à cocher Modifier le mot de passe de l'administrateur. Le nouveau mot de passe devient effectif lors de la connexion suivante en tant qu'administrateur.

  • Nom du site

    Le nom du site est celui affiché dans Content Server. Le nom du site doit être simple et convivial. Le nom du site par défaut est Content Server.

  • Adresse e-mail de l'administrateur

    Si vous fournissez une adresse e-mail pour l'administrateur, un lien vers l'adresse apparaît sur la page Identification de l'utilisateur.

  • Traitement par lot de la mise à jour des catégories

    Ce paramètre vous permet de définir la vitesse de réactualisation de la fenêtre de progression lors des opérations de mise à niveau des catégories par lot. La vitesse de réactualisation par défaut est de 200 éléments, ce qui signifie que la fenêtre de progression est toujours actualisée après le traitement de 200 catégories.

  • Configurer la modification/l'organisation

    Ce paramètre vous permet de spécifier un nombre maximum d'éléments que les utilisateurs peuvent placer sur une page. Un maximum de 100 éléments par page est la valeur par défaut.

     

    Note :
    Cette limitation s'applique uniquement aux pages Modifier/Organiser. Les pages associées à une page Modifier/Organiser comprennent les pages Mes favoris et Mes projets. Lorsque la limite Nombre maximum d'éléments par page est dépassée, Content Server divise la page Modifier/Organiser en plusieurs pages. La limite s'applique alors à chaque page individuelle.

     

  • Page d'ouverture utilisateur par défaut

    Ce paramètre vous permet de sélectionner l'une des pages suivantes, là où les utilisateurs sont redirigés lors de leur première connexion :

    • Espace de travail Entreprise qui affiche la page d'accueil de votre entreprise lorsque les utilisateurs se connectent.

    • Mon espace de travail, qui affiche l'espace de travail personnel de chaque utilisateur lorsque les utilisateurs se connectent.

    • A propos de Content Server, qui affiche la page A propos de Content Server lorsque les utilisateurs se connectent.

       

      Note :
      Si vous sélectionnez A propos de Content Server, vous devez décider si vous voulez imposer que les utilisateurs se connectent pour pouvoir voir la page. Si vous voulez imposer que les utilisateurs se connectent, activez la case à cocher Connexion requise pour « A propos de Content Server ». Cette option est désactivée par défaut.

       

  • Administrer les icônes pour les dossiers

    Lorsqu'il est activé, ce paramètre permet aux utilisateurs de sélectionner des icônes supplémentaires pour les dossiers Content Server. Ce paramètre est désactivé par défaut.

  • Durée des indicateurs Nouveau(elle) et Modifié(e)

    Vous pouvez spécifier le nombre de jours où des éléments ont les icônes Nouveau(elle) ou Modifié(e) à côté d'eux lorsque des éléments sont ajoutés ou modifiés. Par défaut, l'icône Nouveau(elle) apparaît pendant 2 jours suite à l'ajout d'un élément et l'icône Modifié(e) pendant 7 jours après la modification d'un élément.

  • Séparateur d'adresses pour l'URL « mailto : » URL

    Vous pouvez modifier le caractère inséré entre plusieurs adresses de destinataires dans les fenêtres de composition de message. Si votre entreprise utilise surtout les applications de messagerie Microsoft, choisissez un point virgule « ; » comme séparateur d'adresse. Si votre entreprise utilise en majorité d'autres applications de messagerie telles que Netscape, choisissez virgule « , » .

  • Options du journal du serveur

    Vous pouvez modifier les options de journalisation du serveur d'administration sur la page Configurer les paramètres du serveur ou sur la page Configurer les paramètres de débogage. Les options de journalisation disponibles sont les suivantes :

    • Pas de journalisation, pour désactiver la consignation dans le journal sur le serveur Content Server. Il s'agit de la valeur par défaut.

    • Journal du thread, qui génère les fichiers journaux suivants dans le répertoire Content Server_home/logs : llserver.out, sockserv1.out , thread<c>.out (un par thread).

    • Journal détaillé du thread, qui génère les mêmes fichiers journaux que Journal du thread, mais en mode détaillé. Le mode détaillé inclut des informations sur les variables d'environnement pertinentes dans les journaux de thread.

    • Journal de CGI et du thread, qui génère les fichiers journaux suivants dans le répertoire Content Server_home/logs : llserver.out, sockserv1.out , thread<n>.out (un par thread), receiver<n>.out (un de chaque par thread), llclient<nnn>.out (un par requête envoyée depuis le navigateur Web d'un utilisateur final au programme CGI), llindexupdate< nnn>.out (un par démarrage de l'Extracteur Entreprise) et indexupdateOut<nnn>.out (un par arrêt de l'Extracteur Entreprise).

       

      Note :
      Lorsque vous utilisez LLServlet au lieu de CGI, llclient< nnn>.out est remplacé par servletclient< nnn>.out. Un fichier est généré pour chaque numéro de thread (déterminé par la machine virtuelle Java de votre recueil servlet). Chaque fois qu'un navigateur client envoie une requête à LLServlet, un nouveau fichier servletclient<nnn>.out est créé. Si un nom de fichier portant ce numéro <nnn> existe déjà, les informations de la requête sont ajoutées à la fin du fichier existant.

       

  • Clé de licence

    Saisissez le Nom de l'entreprise, la Date d'expiration et le Nombre d'utilisateurs.

  • Mode d'accès au clavier étendu

    Le mode de navigation améliorée par le clavier permet de manipuler l'interface utilisateur à l'aide du clavier. Cependant, certaines fonctions dépendantes de Java, telles que l'éditeur de texte, sont désactivées lorsque le mode de navigation améliorée par le clavier est en cours d'utilisation. Cette option est désactivée par défaut. Lorsque vous activez cette option, tous les utilisateurs doivent utiliser ce mode.

  • Ensemble de caractères

    Vous pouvez spécifier l'ensemble de caractères que les navigateurs Web utilisent lors de l'affichage de l'interface utilisateur. Pour plus d'informations, voir HTMLcharset dans la section [general] de la référence du fichier opentext.ini.

  • Répertoire de chargement

    Le paramètre Répertoire de chargement permet de restreindre l'emplacement à partir duquel Content Server accepte le chargement des documents. Le répertoire défini dans ce champ doit être accessible au serveur Web et au serveur Admin. Open Text vous recommande de définir dans ce champ l'intégralité du chemin menant au répertoire.

     

    Note :
    Open Text vous recommande vivement de spécifier un répertoire de chargement. Ne pas renseigner ce champ peut représenter un risque au niveau de la sécurité.
