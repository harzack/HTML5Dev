La section [Security] définit trois options de sécurité Content Server.

Authentification

  • Description :

    Il s'agit d'informations Open Text exclusives.

CGIHosts

  • Description :

    Liste des hôtes dont le serveur accepte les connexions clientes. Ces connexions sont effectuées à partir du programme CGI, qui est appelé cs sur les systèmes UNIX et cs.exe sur les systèmes Windows. Le programme CGI et le serveur doivent résider sur le même hôte ; la valeur par défaut est donc l'adresse IP 127.0.0.1, une adresse IP spéciale désignant l'hôte local.

    Si la liste ne comporte aucun hôte, le serveur accepte des connexions CGI provenant de toutes les adresses IP.

  • Syntaxe :

    CGIHosts=127.0.0.1

  • Valeurs :

    La valeur par défaut est 127.0.0.1. Il s'agit d'une adresse IP spéciale désignant l'hôte local. Dans de rares circonstances, il arrive que certains systèmes n'identifient pas cette adresse IP spéciale comme désignant l'hôte local, localhost. Dans ce cas, remplacez cette adresse IP par l'adresse IP unique ou par le nom de l'hôte local.

hideContainerSize

  • Description :

    Définissez ce paramètre sur TRUE pour que Content Server n'affiche pas le nombre d'éléments présents dans un recueil.

     

    Note :
    Ce paramètre s'applique également aux prospecteurs. Les objets du prospecteur affichent une valeur de taille vide lorsque ce paramètre est défini sur TRUE.

     

    Pour définir ce paramètre, OpenText vous recommande d'utiliser la page Configurer les paramètres de sécurité, plutôt que de modifier directement le fichier opentext.ini. Pour en savoir plus, voir Configuration des paramètres de sécurité.

  • Syntaxe :

    hideContainerSize=FALSE

  • Valeurs :

    TRUE ou FALSE. Par défaut, cette entrée ne figure pas dans le fichier opentext.ini, ce qui équivaut à hideContainerSize=FALSE.
