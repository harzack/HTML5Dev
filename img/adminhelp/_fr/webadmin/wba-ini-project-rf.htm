Par défaut, la section [Project] ne figure pas dans le fichier opentext.ini. Vous n'aurez besoin de l'ajouter que pour modifier la valeur par défaut du paramètre ProjectOutlineSubtypes, décrit ci-dessous.

ProjectOutlineSubtypes

  • Description :

    Par défaut, les projets Content Server contiennent sept sous-types d'objets WebNode. Les valeurs des sous-types se traduisent par un nom de type de nœud ou un libellé qui est inclus dans le champ Inclure les types d'élément sur la page Aperçu du projet d'un projet.

    En ajoutant une section [Project] au fichier opentext.ini, l'administrateur peut modifier les valeurs de sous-type selon les besoins du projet au sein de l'organisation.

  • Syntaxe :

    ProjectOutlineSubtypes={204,215,207,144,140,136,400,0}

  • Valeurs :

    Voici les sept valeurs par défaut :

    Valeur du sous-type de nœud Type de nœud
    204 Liste de tâches
    215 Discussion
    207 Canal
    144 Document
    140 URL
    136 Document composite
    0 Dossier

    Pour consulter la liste complète des correspondances entre le nom et le numéro des types de nœud, reportez-vous à la rubrique Correspondance entre le nom et le numéro des types de nœud.
