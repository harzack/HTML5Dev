La page Maintenance de la base de données Content Server active affiche des informations sur la base de données courante ainsi qu'une série de liens permettant d'exécuter certaines tâches sur la base.

Pour le serveur Oracle, cette page affiche les informations suivantes :

  • le type de SGBDR ;

  • le nom du service Oracle (également appelé raccourci de la base de données) ;

  • le nom de l'utilisateur Oracle propriétaire des tables de la base de données Content Server.

Pour Sybase ou Microsoft SQL Server, la page Maintenir la base de données active affiche les informations suivantes :

  • le type de SGBDR ;

  • Nom (raccourci) de l'installation Sybase ou Microsoft SQL Server (bien souvent le nom de l'hôte sur lequel réside le serveur)

  • le nom de la base de données SQL Server contenant les tables Content Server ;

  • le nom d'utilisateur Microsoft SQL Server utilisé par Content Server pour se connecter à la base de données SQL Server contenant les tables Content Server.

Vous pouvez effectuer un certain nombre de tâches de maintenance sur cette page. Les tâches possibles diffèrent selon le type de SGBDR et le mode de stockage des documents (externe ou interne).

Pour le serveur Oracle, la page Maintenance de la base de données active comporte des liens qui permettent d'effectuer les tâches suivantes :

  • visualiser les tables ;

  • afficher l'utilisation de l'espace de stockage ;

  • exécuter les tâches de maintenance du serveur Oracle ;

  • éliminer les données supprimées (stockage de documents interne uniquement) ;

  • mettre à niveau la base de données ;

  • vérifier la base de données ;

Pour Sybase ou Microsoft SQL Server, la page Maintenance de la base de données active contient des liens qui permettent d'effectuer les tâches suivantes :

  • visualiser les tables Content Server ;

  • Exécution des tâches de maintenance de Sybase ou Microsoft SQL Server

  • mettre à niveau la base de données ;

  • vérifier la base de données ;

exécuter les tâches de maintenance d'un SGBDR.

La page Maintenance de la base de données active affiche un lien vers la page de maintenance correspondant à votre type de SGBDR. Selon le type de SGBDR, ce lien mène à la page Maintenance du serveur Oracle ou Maintenance Microsoft SQL Server.

Sur la page Maintenance du serveur Oracle, vous pouvez :

  • créer ou agrandir un espace de stockage ;

  • créer un utilisateur Oracle ;

  • supprimer un utilisateur Oracle et/ou les tables de la base dont il est propriétaire.

Sur la page Maintenance du serveur Sybase ou Microsoft SQL, vous pouvez :

  • créer ou agrandir des espaces disque de journalisation et de données ;

  • créer une base de données SQL Server ;

  • créer un utilisateur pour une connexion à une base de données du serveur SQL ;

  • supprimer une base de données du serveur SQL et/ou les tables qu'elle contient.

Lorsque vous connectez l'installation Content Server active à une base de données Content Server existante ou nouvelle, une page Félicitations apparaît. Elle confirme que la connexion à la base de données s'est correctement déroulée.

Sur la page Félicitations, vous disposez des options suivantes :

  • Utilisez le lien Allez à la page d'accueil de Content Server pour accéder à la page de démarrage par défaut de Content Server.

  • Utilisez le lien Accueil Admin pour accéder à la page Administration Content Server.

     

    Note :
    Si vous connectez cette installation Content Server à une base de données Content Server existante et cliquez sur le bouton Continuer de la page Confirmation de la resynchronisation de la source de données, vous devez reconfigurer les objets source de données, puis resynchroniser un par un tous les serveurs d'administration Livelink.
