Les paramètres de la section [QDF] définissent la configuration des QDF (Quality Document Filters) utilisés dans le service de conversion de documents (DCS). Les filtres QDF convertissent des documents au format HTML ou texte brut pour permettre leur affichage et indexation. Les détails de cette conversion varient en fonction du format de fichier natif. Par exemple, les filtres QDF convertissent les fichiers Microsoft Outlook au format HTML ou texte brut pour permettre leur indexation. Par défaut, les filtres QDF utilisent la version Unicode d'un fichier Microsoft Outlook si elle est disponible ; si elle n'est pas disponible, les filtres QDF utilisent la version RTF. Si aucune des deux versions n'est disponible, les filtres QDF utilisent la version HTML. Si aucune des trois versions n'est disponible, les filtres QDF utilisent une version au format texte brut. Lorsque les filtres QDF utilisent la version RTF ou HTML d'un fichier Microsoft Outlook pour l'indexation, le contenu est extrait et renvoyé au DCS comme s'il s'agissait d'une pièce jointe. Le mécanisme de filtrage RTF des filtres QDF convertit alors le contenu RTF au format HTML ou texte brut pour permettre son indexation. Pour plus d'informations sur les filtres QDF, reportez-vous à la rubrique Filtres de conversion de documents.

La section [QDF] permet également de configurer l'extraction des propriétés de documents OLE personnalisées à partir des documents Microsoft Office pris en charge. Les noms de région des métadonnées exportées sont identiques aux noms de propriété des documents OLE. On note toutefois deux exceptions : ils comportent le préfixe OTDoc et n'acceptent pas les barres obliques et les espaces.

DefaultLatinEncoding

  • Description :

    Indique l'ensemble de caractères latins par défaut à détecter.

    Un ensemble de caractères peut avoir des schémas multiples. Par exemple, les ensembles de caractères ISO-8859-* et EUC ont chacun plusieurs variations régionales. Du fait de ces variations, vous pouvez spécifier l'ensemble de caractères par défaut que les filtres QDF détecteront.

  • Syntaxe :

    DefaultLatinEncoding=ISO-8859-1

  • Valeurs :

    ISO-8859-1 ~ ISO-8859-15, WinANSI. Par défaut, ce paramètre ne figure pas dans le fichier opentext.ini, ce qui équivaut à DefaultLatinEncoding=ISO-8859-1.

lib

  • Description :

    Détermine le nom de la bibliothèque que le DCS doit charger. Une bibliothèque est une liste d'opérations associées à un filtre de conversion que le DCS lit pour effectuer la conversion. Ce paramètre est requis par le DCS.

    La modification de la valeur de ce paramètre risque d'empêcher le DCS de fonctionner ou de fonctionner correctement.

     

    Important :
    Ne modifiez pas la valeur de ce paramètre, sauf si le service d'assistance clientèle d'OpenText vous le demande.

     

  • Valeurs :

    La valeur par défaut est dcsqdf.

showcdata

  • Description :

    Indique si les sections CDATA des documents XML doivent être extraites pour permettre au DCS de les indexer.

  • Syntaxe :

    showcdata=FALSE

  • Valeurs :

    TRUE ou FALSE ; la valeur FALSE demande au DCS d'ignorer les sections CDATA lors de l'extraction de données à partir de documents XML, pour permettre leur indexation. La valeur par défaut est FALSE.

     

    Note :
    Si vous définissez ce paramètre sur TRUE, les performances de recherche risquent d'être affectées.

     

maxfilesunzip

  • Description :

    Indique le nombre maximal de fichiers extraits par les filtres QDF à partir d'un fichier compressé pendant la conversion. Les fichiers extraits sont déterminés par l'ordre dans lequel le créateur des fichiers les a ajoutés à l'origine.

     

    Note :
    Le traitement récursif des fichiers compressés est pris en charge par la configuration de fichier des règles DCS. Il s'agit d'un fichier LZH dans un fichier LZH, ou d'un fichier ZIP dans un fichier LZH.

     

  • Syntaxe :

    maxfilesunzip=250

  • Valeurs :

    Un entier positif. La valeur par défaut est 250.

MinAsianAvgLength

  • Description :

    Indique la longueur moyenne minimale des unités lexicales multi-octets adjacentes.

    Une fois que les filtres QDF ont déterminé qu'un fichier ne contient pas uniquement des caractères 7 bits, la détection des ensembles de caractères asiatiques est normalement exécutée. Si la valeur des paramètres MinHighLowRatioForSJIS ou MinHighLowRatioForEUC est respectée, les filtres QDF analysent du texte afin de déterminer l'ensemble de caractères utilisé.

    Lors de l'analyse de textes multi-octets, les filtres QDF effectuent des évaluations de texte supplémentaires. Par exemple, les filtres QDF déterminent la longueur moyenne des unités lexicales multi-octets adjacentes afin de déterminer si le texte analysé contient bien du texte asiatique ou simplement quelques unités lexicales. Pour être évaluée, la longueur moyenne des unités lexicales adjacentes doit être supérieure à la valeur minimale définie pour le paramètre MinAsianAvgLength.

     

    Note :
    Lorsqu'ils déterminent la longueur moyenne des unités lexicales multi-octets adjacentes, les filtres QDF ignorent tous les espaces vierges du texte analysé.

    Pour plus d'informations sur les autres conditions à remplir pour permettre l'évaluation de textes multi-octets, reportez-vous aux paramètres MinAsianTextRatio et MinAsianTokens.

     

  • Syntaxe :

    MinAsianAvgLength=6

  • Valeurs :

    Un entier supérieur à 1. La valeur par défaut est 6.

MinAsianTextRatio

  • Description :

    Indique le pourcentage d'octets de texte asiatique qui doit être dépassé lors de la détection de texte asiatique.

    Une fois que les filtres QDF ont déterminé qu'un fichier ne contient pas uniquement des caractères 7 bits, la détection des ensembles de caractères asiatiques est normalement exécutée. Si la valeur des paramètres MinHighLowRatioForSJIS ou MinHighLowRatioForEUC est respectée, les filtres QDF analysent du texte afin de déterminer l'ensemble de caractères utilisé.

    Lors de l'analyse de textes multi-octets, les filtres QDF effectuent des évaluations de texte supplémentaires. Tout d'abord, les filtres QDF déterminent la longueur moyenne d'unités lexicales multi-octets adjacentes. Pour plus d'informations, voir le paramètre MinAsianAvgLength. Ensuite, au moins l'une des deux conditions suivantes doit être remplie :

    • Le pourcentage d'octets d'unités lexicales asiatiques dans le texte dépasse la valeur du paramètre MinAsianTextRatio.

    • Le nombre d'unités lexicales identifiées dépasse la valeur du paramètre MinAsianTokens. Pour plus d'informations, reportez-vous au paramètre MinAsianTokens.

  • Syntaxe :

    MinAsianTextRatio=50

  • Valeurs :

    Un entier compris entre 1 et 100. La valeur par défaut est 50.

MinAsianTokens

  • Description :

    Indique le nombre d'unités lexicales multi-octets qui doivent être identifiées lors d'une détection de texte asiatique.

    Une fois que les filtres QDF ont déterminé qu'un fichier ne contient pas uniquement des caractères 7 bits, la détection des ensembles de caractères asiatiques est normalement exécutée. Si la valeur des paramètres MinHighLowRatioForSJIS ou MinHighLowRatioForEUC est respectée, les filtres QDF analysent du texte afin de déterminer l'ensemble de caractères utilisé.

    Lors de l'analyse de textes multi-octets, les filtres QDF effectuent des évaluations de texte supplémentaires. Tout d'abord, les filtres QDF déterminent la longueur moyenne d'unités lexicales multi-octets adjacentes. Pour plus d'informations, voir le paramètre MinAsianAvgLength. Ensuite, au moins l'une des deux conditions suivantes doit être remplie :

    • Le pourcentage d'octets d'unités lexicales asiatiques dans le texte dépasse la valeur du paramètre MinAsianTextRatio. Pour plus d'informations, reportez-vous au paramètre MinAsianTextRatio.

    • Le nombre d'unités lexicales identifiées dépasse la valeur du paramètre MinAsianTokens.

  • Syntaxe :

    MinAsianTokens=5000

  • Valeurs :

    Un entier supérieur à 0. La valeur par défaut est 5000.

MinHighLowRatioForEUC

  • Description :

    Indique le taux d'octets élevés à faibles qui doit être dépassé pour permettre la détection des ensembles de caractères asiatiques dans les fichiers EUC.

    Une fois que les filtres QDF ont déterminé qu'un fichier ne contient pas uniquement des caractères 7 bits, la détection des ensembles de caractères asiatiques est normalement exécutée. Ce niveau de détection des ensembles de caractères peut prendre du temps et retarder les opérations de détection de type MIME. C'est pourquoi, en fonction du contenu des documents sur votre site Content Server, vous ne souhaiterez peut-être pas utiliser ce niveau de détection ou vous l'utiliserez uniquement lorsqu'un certain taux de caractères à octets élevés à faibles est dépassé dans un document. Vous pouvez modifier ce taux en changeant la valeur du paramètre MinHighLowRatioForEUC.

  • Syntaxe :

    MinHighLowRatioForEUC=60

  • Valeurs :

    Un entier compris entre 1 et 100. La valeur par défaut est 60.

MinHighLowRatioForSJIS

  • Description :

    Indique le taux d'octets élevés à faibles qui doit être dépassé pour permettre la détection des ensembles de caractères asiatiques dans les fichiers Shift-JIS.

    Une fois que les filtres QDF ont déterminé qu'un fichier ne contient pas uniquement des caractères 7 bits, la détection des ensembles de caractères asiatiques est normalement exécutée. Ce niveau de détection des ensembles de caractères peut prendre du temps et retarder les opérations de détection de type MIME. C'est pourquoi, en fonction du contenu des documents sur votre site Content Server, vous ne souhaiterez peut-être pas utiliser ce niveau de détection ou vous l'utiliserez uniquement lorsqu'un certain taux de caractères à octets élevés à faibles est dépassé dans un document. Vous pouvez modifier ce taux en changeant la valeur du paramètre MinHighLowRatioForSJIS.

  • Syntaxe :

    MinHighLowRatioForSJIS=50

  • Valeurs :

    Un entier compris entre 1 et 100. La valeur par défaut est 50.

x-maxcalls

  • Description :

    Indique le nombre de réutilisations d'un processus pour le traitement des documents. Pendant la conversion de documents, le DCS charge un processus de traitement. Le processus de traitement charge le filtre de conversion approprié et l'utilise pour convertir le document. Pour améliorer les performances, le DCS réutilise le processus de traitement pour plusieurs conversions. Si le processus de traitement rencontre une erreur, il s'interrompt avant d'atteindre la valeur définie dans le fichier opentext.ini.

     

    Important :
    OpenText vous recommande vivement de ne pas modifier la valeur de ce paramètre.

     

  • Valeurs :

    La valeur par défaut est héritée du paramètre maxcalls de la section [DCSworker] du fichier opentext.ini.

     

    Note :
    Le paramètre x-maxcalls est actif uniquement lorsque le filtre de conversion qu'il modifie est géré par un processus de traitement. Pour plus d'informations sur la configuration d'un processus de traitement, reportez-vous à la rubrique Configuration du journal du serveur.

    Si vous définissez une valeur pour x-maxcalls, la valeur de maxcalls sera écrasée dans la section [DCSworker] du fichier opentext.ini.

     

x-timeout

  • Description :

    Indique le temps d'attente maximum (en secondes) avant la fin d'un processus de traitement de conversion de documents. Configurez ce paramètre lorsque la valeur par défaut définie dans le paramètre timeout n'est pas valable. Vous configurez le paramètre timeout pour chaque filtre de conversion. Par exemple, certains filtres de conversion sont plus lents que d'autres. Dans ce cas, la valeur timeout par défaut de 30 secondes peut ne pas être applicable, car le délai de conversion moyen est supérieur à cette valeur. Dans ce cas, vous pouvez modifier la valeur x-timeout en choisissant une valeur plus élevée.

     

    Important :
    OpenText vous recommande vivement de ne pas modifier la valeur de ce paramètre.

     

  • Valeurs :

    La valeur par défaut est héritée du paramètre timeout de la section [DCSworker] du fichier opentext.ini.

     

    Note :
    Le paramètre x-timeout est actif uniquement lorsque le filtre de conversion qu'il modifie est géré par un processus de traitement. Pour plus d'informations sur la configuration d'un processus de traitement, reportez-vous à la rubrique Configuration du journal du serveur.

    Si vous définissez une valeur pour x-timeout, la valeur de timeout sera écrasée dans la section [DCSworker] du fichier opentext.ini.
