Pour protéger la pertinence et l'intégrité du journal d'audit Content Server, les utilisateurs habituels n'ont pas accès aux fonctions administratives du journal d'audit Content Server. Par défaut, seuls l'utilisateur administrateur et tout utilisateur bénéficiant du privilège Droits d'administration système peuvent interroger le journal d'audit, définir les événements à inclure dans l'audit et éliminer le journal d'audit. Vous pouvez activer les paramètres de sécurité de l'audit pour restreindre davantage ces possibilités.

 

Astuce :
Il se peut que vous souhaitiez activer les paramètres de sécurité de l'audit si votre entreprise est réglementée et si vous devez obligatoirement prouver l'intégrité du journal d'audit.

Astuce :
Si vous activez l'un de ces niveaux de sécurité, les utilisateurs bénéficiant du privilège Droits d'administration système peuvent interroger le journal d'audit mais ne peuvent ni ajouter, ni supprimer les intérêts à inclure dans l'audit, ni éliminer du journal d'audit.

 

Pour plus d'informations sur la page des intérêts à inclure dans l'audit, voir Gestion des centres d'intérêt de l'audit. Pour plus d'informations sur le journal d'audit, voir Gestion des enregistrements d'audit.

  • Vous pouvez configurer Content Server afin que seul l'utilisateur administrateur puisse définir les événements à inclure dans l'audit et éliminer le journal d'audit.

  • Vous pouvez configurer Content Server afin qu'aucun utilisateur ne puisse éliminer le journal d'audit.

  • Vous pouvez configurer Content Server afin qu'aucun utilisateur ne puisse modifier la configuration des intérêts de l'audit.

 

Important :
Réfléchissez bien avant de décider d'empêcher l'élimination du journal d'audit ou la modification des intérêts de l'audit. Une fois l'une de ces étapes en cours, elle ne peut plus être désactivée. Si vous empêchez par exemple la modification de votre configuration des intérêts de l'audit, vous ne pourrez plus activer de nouveaux centres d'intérêt à inclure dans l'audit ajoutés grâce à un nouveau module Content Server, ni désactiver les centres d'intérêt à inclure dans l'audit activés par défaut.
