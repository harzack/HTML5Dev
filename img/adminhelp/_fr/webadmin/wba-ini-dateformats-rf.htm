La section [dateformats] définit la gestion de la date et de l'heure dans Content Server. Pour modifier ces paramètres, OpenText vous recommande d'utiliser la page Gérer la date/l'heure plutôt que de modifier directement le fichier opentext.ini. Pour en savoir plus, voir Définition des formats de date et d'heure.

InputDateMinYear

  • Description :

    Définit l'année la plus ancienne que les utilisateurs peuvent choisir dans les listes qui leur sont fournies.

  • Syntaxe :

    InputDateMinYear=1990

  • Valeurs :

    Un entier positif. La valeur par défaut est 1990.

    Vous pouvez définir cette valeur aussi bas que vous le souhaitez, il n'y a pas de limite numérique. Toutefois, il est recommandé de la définir sur une valeur qui a du sens dans l'affichage.

InputDateMaxYear

  • Description :

    Définit l'année la plus lointaine (future) que les utilisateurs peuvent choisir dans les listes qui leur sont fournies.

  • Syntaxe :

    InputDateMaxYear=2027

  • Valeurs :

    Un entier positif dont la limite est 400 000. La valeur par défaut est 2027.

    Il est recommandé de la définir sur une valeur qui a du sens dans l'affichage. Une valeur de paramètre trop élevée ralentit le traitement des pages et peut avoir un effet notable sur vos performances.

TwoDigitYears

  • Description :

    Indique si Content Server affiche les années sous la forme de deux ou de quatre chiffres.

  • Syntaxe :

    TwoDigitYears=FALSE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est FALSE.

    La définition de ce paramètre sur TRUE affiche l'année 2002 sous la forme 02. Si vous conservez la valeur par défaut FALSE, l'année 2002 est affichée sous la forme 2002.

WantTimeZone

  • Description :

    Définit si la fonction de décalage horaire est activée.

  • Syntaxe :

    WantTimeZone=FALSE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est FALSE.

    La définition de ce paramètre sur TRUE active la fonction de décalage horaire.

SeparateCentury

  • Description :

    Indique si Content Server fournit une ou deux listes aux utilisateurs pour les saisies d'année.

  • Syntaxe :

    SeparateCentury=FALSE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est FALSE.

    La définition de ce paramètre sur TRUE fournit deux listes aux utilisateurs pour les saisies d'année : une liste pour le siècle, 19 et 20, et une liste pour la décennie, de 00 à 99. Si vous conservez la valeur par défaut FALSE, une seule liste pour les saisies d'année est fournie.
