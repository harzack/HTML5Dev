Vous devez sauvegarder périodiquement Content Server pour protéger les données se trouvant dans votre déploiement Content Server. De plus, OpenText vous recommande de sauvegarder Content Server dès que vous appliquez un correctif ou une mise à jour, installez, désinstallez ou mettez à niveau un module. Ceci est également valable à chaque fois que vous apportez une modification à votre déploiement Content Server.

Un ensemble de sauvegardes Content Server comprend les éléments suivants :

  • un dossier <Content Server_accueil> ;

  • une base de données Content Server ;

  • un magasin de fichiers externe ;

  • un ou plusieurs index de recherche ;

  • tout dossier Content Server se trouvant en dehors du dossier < Content Server_accueil>, par exemple un dossier Chargement.

 

Note :
Si votre déploiement Content Server inclut des modules facultatifs ou des applications intégrées, il peut être nécessaire d'inclure des éléments additionnels dans l'ensemble de sauvegardes.

 

Arrêtez les services Content Server avant d'effectuer une sauvegarde afin que tous les composants Content Server soient synchronisés les uns avec les autres. Effectuer la sauvegarde de Content Server lorsque les services Content Server fonctionnent peut entraîner des incohérences de données. Par exemple, la base de données Content Server peut contenir des références à des éléments Content Server qui n'existent pas dans la sauvegarde du magasin de fichiers externe.

Pour sauvegarder Content Server :

  1. Arrêtez les services Content Server.

  2. Sauvegardez la base de données Content Server à l'aide des outils appropriés pour votre système de gestion de base de données relationnelle Microsoft SQL Server ou Oracle Database.

  3. Sauvegardez les index de recherche Content Server en effectuant une copie du dossier ...\index\<nom_index> et de ses sous-dossiers.

    Exemple: Pour sauvegarder l'index de recherche d'entreprise, copiez le dossier ...\index\Enterprise\, ainsi que tous les sous-dossiers.

     

    Note :
    Voir Sauvegarde et restauration des index pour plus d'informations sur l'automatisation des sauvegardes d'index.

     

  4. Sauvegardez le dossier <Content Server_accueil> en le copiant, ainsi que ses sous-dossiers.

  5. Sauvegardez tout dossier Content Server se trouvant en dehors du dossier <Content Server_accueil>, par exemple le dossier Chargement Content Server.

Pour restaurer un ensemble de sauvegardes Content Server :

  1. Arrêtez les services Content Server.

  2. Restaurez votre base de données Content Server.

  3. Restaurez les index de recherche Content Server.

  4. Restaurez le dossier <Content Server_accueil>.

  5. Restaurez tout dossier Content Server existant en dehors du dossier <Content Server_accueil>.

  6. Démarrez les services Content Server.
