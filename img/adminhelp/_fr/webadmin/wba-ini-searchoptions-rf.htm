brokerObjectCacheExpire

  • Description :

    Détermine la durée en minutes pendant laquelle les données de vue de recherche de la barre de recherche Content Server sont mises en mémoire cache avant l'actualisation.

    Par exemple, si vous ajoutez ou supprimez une vue de recherche, le paramètre brokerObjectCacheExpire détermine la durée en minutes écoulée avant que la vue de recherche apparaisse dans la barre de recherche Content Server ou en soit supprimée.

  • Syntaxe :

    brokerObjectCacheExpire=5

  • Valeurs :

    Un entier supérieur ou égal à zéro. Par défaut, ce paramètre ne figure pas dans le fichier opentext.ini, ce qui équivaut à brokerObjectCacheExpire=5. Définissez le paramètre brokerObjectCacheExpire sur 0 pour indiquer que les données de vue de recherche ne sont pas dans l'antémémoire.

dateTypesAdditions

  • Description :

    Spécifie la liste des régions non définies par défaut, les régions supplémentaires détectées par l'index de recherche, traitées sous forme de dates par le composant de recherche avancée Attributs système. Le traitement des dates permet aux utilisateurs de saisir des dates au moyen d'un widget de calendrier au lieu d'utiliser la syntaxe de texte spéciale.

  • Syntaxe :

    dateTypesAdditions={'region1name ', 'region2name'}

  • Valeurs :

    Liste de régions valides, non définies par défaut, séparées par des virgules. Par défaut, ce paramètre ne figure pas dans le fichier opentext.ini, ce qui équivaut à aucune région spécifiée.

    Voici un exemple d'entrée valide : dateTypesAdditions={'OTEmailSentDate', 'OTEmailReceivedDate'}

DefaultThesaurus

  • Description :

    Détermine la langue du thésaurus utilisé par Content Server pour effectuer les recherches.

  • Syntaxe :

    DefaultThesaurus=English

  • Valeurs :

    Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut au paramètre DefaultThesaurus=English. Pour utiliser un thésaurus de recherche en français ou en allemand, utilisez DefaultThesaurus=French ou DefaultThesaurus=German. Pour utiliser un thésaurus personnalisé, spécifiez DefaultThesaurus=<nom_thésaurus>, où <nom_thésaurus> est le nom de fichier du thésaurus personnalisé créé dans le répertoire config du répertoire d'installation Content Server. Pour plus d'informations sur l'utilisation des thésaurus personnalisés, reportez-vous à la rubrique Création d`un thésaurus personnalisé.

defaultWebSearchDetail

  • Description :

    Spécifie le statut initial des liens Moins de détails et Plus de détails des styles des pages Résultats de recherche Recherche Web et Recherche Web - Thèmes droite. Les utilisateurs peuvent modifier le statut initial en définissant l'autre paramètre de détails.

  • Syntaxe :

    defaultWebSearchDetail=FALSE

  • Valeurs :

    TRUE ou FALSE. Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut à defaultWebSearchDetail=FALSE et définit le statut initial sur le lien Moins de détails. Pour définir le statut initial sur le lien Plus de détails, indiquez defaultWebSearchDetail=TRUE.

EnterpriseTimeToLiveInterval

  • Description :

    Indique le délai (exprimé en secondes) après lequel Content Server expire lors du renvoi des résultats de la recherche.

  • Syntaxe :

    EnterpriseTimeToLiveInterval=180

  • Valeurs :

    Par défaut, ce paramètre ne figure pas dans le fichier opentext.ini, ce qui équivaut à EntepriseTimeToLiveInterval=180.

findSimilar

  • Description :

    Spécifie si l'option Rechercher des éléments similaires est désactivée pour les résultats de recherche, lors de l'utilisation de l'API de recherche avec outputformat=xml (Extensible Markup Language). Vous pouvez également définir une valeur temporaire pour ce paramètre au moyen des paramètres de l'API de recherche.

  • Syntaxe :

    findSimilar=FALSE

  • Valeurs :

    TRUE ou FALSE. Prend la valeur TRUE par défaut.

     

    Note :
    Les valeurs présentes dans le fichier opentext.ini sont les valeurs par défaut. Si vous définissez une valeur dans les paramètres de l'API, elle prime sur la valeur présente dans le fichier opentext.ini. Si aucune valeur n'est spécifiée, ni dans le fichier opentext.ini, ni dans le paramètre de l'API, la valeur par défaut est TRUE.

     

functionMenu

  • Description :

    Spécifie si le menu Fonction est désactivé pour les résultats de recherche lors de l'utilisation de l'API de recherche avec outputformat=xml (Extensible Markup Language). Vous pouvez également définir une valeur temporaire pour ce paramètre au moyen des paramètres de l'API de recherche.

  • Syntaxe :

    functionMenu=false

  • Valeurs :

    TRUE ou FALSE. Prend la valeur TRUE par défaut.

     

    Note :
    Les valeurs présentes dans le fichier opentext.ini sont les valeurs par défaut. Si vous définissez une valeur dans les paramètres de l'API, elle prime sur la valeur présente dans le fichier opentext.ini. Si aucune valeur n'est spécifiée, ni dans le fichier opentext.ini, ni dans le paramètre de l'API, la valeur par défaut est TRUE.

masterCacheDisable

  • Description :

    Indique si la mise en mémoire cache de données d'objets de recherche, tels que les schémas de régions des gestionnaires de recherche et les définitions de vues de recherche, est interdite.

  • Syntaxe :

    masterCacheDisable=TRUE

  • Valeurs :

    TRUE ou FALSE. Par défaut, ce paramètre ne figure pas dans le fichier opentext.ini, ce qui équivaut à masterCacheDisable=FALSE.

MaxImportAgentObjectsPerTransaction

  • Description :

    Spécifie le nombre maximum d'objets provenant de la table DStagingImport à traiter par transaction de base de données. Ce paramètre peut être utilisé en présence d'une très grande quantité de données dans la table, ce qui pourrait causer des problèmes de mémoire.

  • Syntaxe :

    MaxImportAgentObjectsPerTransaction=25

  • Valeurs :

    Un entier supérieur ou égal à 1, ou la valeur -1, ce qui équivaut à illimité. La valeur par défaut est -1. Par défaut, ce paramètre ne figure pas dans le fichier opentext.ini, ce qui équivaut à MaxImportAgentObjectsPerTransaction=-1.

needValidHTTPSCerticate

  • Description :

    Indique si un certificat HTTPS (Hypertext Transfer Protocol Secure) valable doit être présent pour que la conversion de l'URL en fichier local puisse récupérer un document via HTTPS.

  • Syntaxe :

    needValidHTTPSCerticate=FALSE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est FALSE.

NoHiddenItemForResult

  • Description :

    Indique si des éléments cachés apparaissent ou non dans une liste de résultats de recherche.

  • Syntaxe :

    NoHiddenItemForResult=FALSE

  • Valeurs :

    TRUE ou FALSE. Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut à NoHiddenItemForResult=FALSE et inclut les éléments indiqués comme masqués dans une liste de résultats de recherche. Pour exclure les éléments masqués, spécifiez NoHiddenItemForResult=TRUE.

objectTypes et objectTypesName

  • Description :

    Déterminent les types d'éléments Content Server qui apparaissent dans la liste Type d'objet de la section Attributs système sur la page Recherche Content Server. Deux paramètres définissent la liste Type d'objet : objectTypes et objectTypesName.

    Ces paramètres vous permettent d'associer les valeurs des sous-types d'objets aux libellés d'affichage que vous souhaitez voir apparaître dans la liste Type d'objet de la section Attributs système. Classez les paramètres objectTypes et objectTypesName dans l'ordre dans lequel vous souhaitez voir apparaître les options dans la liste Type d'objet. Si vous souhaitez ajouter une option à la liste Type d'objet ou en supprimer, vous devez ajouter ou supprimer la valeur du sous-type dans le paramètre objectTypes et le libellé d'affichage correspondant dans le paramètre objectTypesName.

    Pour les simples valeurs de sous-types bien connues, présentes dans la liste Type d'objet, le paramètre objectTypes correspondant est ignoré en faveur du nom du type d'objet officiel dans Content Server. Cela permet d'assurer que le nom du type est disponible dans chaque langue d'interface utilisateur.

    Utilisez le paramètre objectTypes pour répertorier les valeurs de sous-type des types d'éléments Content Server que vous souhaitez afficher dans la liste Type d'objet de la section Attributs système sur la page Recherche Content Server. Par exemple, dans la configuration par défaut, le type de nœud 144 (documents Content Server) correspond à Documents dans la liste Type d'objet. Dans la liste Type d'objet, il est également possible d'associer plusieurs valeurs du sous-type en utilisant l'opérateur booléen OU pour les appliquer à la même option de la liste Type d'objet. Ces expressions doivent apparaître entre guillemets dans le paramètre objectTypes. Par exemple, dans la configuration par défaut, l'entrée "130 OR 134" (messages et réponses Content Server) correspond à Discussions dans la liste Type d'objet. Pour obtenir une liste complète des valeurs subtype de votre système Content Server, utilisez le générateur du module Content Server SDK afin d'exécuter un script répertoriant toutes les valeurs subtype ou consultez [llview].

    Utilisez le paramètre objectTypesName pour répertorier les chaînes, délimitées par des guillemets simples, que Content Server doit afficher dans la liste Type d'objet pour chaque entrée du paramètre objectTypes. Répertoriez les libellés d'affichage du paramètre objectTypesName dans le même ordre que les entrées objectTypes qu'ils représentent. Par défaut, le paramètre 'All Types' est appliqué.

     

    Astuce :
    Vous pouvez modifier la langue utilisée pour les objectTypes en entrant une valeur Xlate plutôt qu'une valeur numérique.

     

  • Syntaxe :

    objectTypes={144,0,"206|212|204|205","130|134|215",202,128,189}

    objectTypesName={'Documents','Folders','Tasks','Discussions','Projects','Workflow Map','Workflow Status'}

  • Valeurs :

    Par défaut, ces paramètres ne figurent pas dans le fichier opentext.ini. Cela équivaut à la définition des paramètres répertoriée dans la section Syntaxe ci-dessus.

redirectPost

  • Description :

    Indique si les pages de recherche devant effectuer un « POST » des requêtes de recherche sont redirigées vers une requête « GET », afin que le bouton Retour d'un navigateur Internet fonctionne conformément aux attentes des utilisateurs. En outre, la barre de recherche Content Server doit envoyer une requête « GET » lorsque c'est possible.

  • Syntaxe :

    redirectPost=TRUE

  • Valeurs :

    TRUE ou FALSE. Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut à redirectPost=TRUE.

resultCountLookAhead

  • Description :

    Spécifie le nombre de résultats de recherche supplémentaires pour vérifier l'autorisation et, si la fin de l'ensemble de résultats « non autorisés » peut être atteinte dans cet intervalle, le nombre exact doit être affiché.

  • Syntaxe :

    resultCountLookAhead=225

  • Valeurs :

    Un entier compris entre 0 et 225. Par défaut, les utilisateurs peuvent afficher 225 résultats de recherche, ce qui équivaut à resultCountLookAhead=225. Pour plus d'informations sur les résultats de la recherche, reportez-vous à la rubrique Gestion des processus de recherche.

     

    Note :
    La spécification d'une valeur pour ce paramètre ne garantit pas qu'un nombre exact de résultats de recherche est disponible.

     

SearchResultsCacheExpiration

  • Description :

    Spécifie la durée en minutes pendant laquelle les résultats de recherche sont placés dans la mémoire cache de la base de données Content Server.

     

    Important :
    OpenText vous recommande de modifier cette valeur uniquement depuis la page Configurer les paramètres du débogage plutôt que de modifier directement le fichier opentext.ini.

     

  • Valeurs :

    Un entier supérieur ou égal à 0. Par défaut, ce paramètre ne figure pas dans le fichier opentext.ini, ce qui équivaut à SearchResultsCacheExpiration=30.

wantMultipleEnterprise

  • Description :

    Indique si les administrateurs sont autorisés à créer plusieurs sources de données Entreprise.

  • Syntaxe :

    wantMultipleEnterprise=FALSE

  • Valeurs :

    TRUE ou FALSE. Par défaut, ce paramètre ne figure pas dans le fichier opentext.ini, ce qui équivaut à wantMultipleEnterprise=FALSE. Pour autoriser la création de plusieurs sources de données Entreprise, définissez wantMultipleEnterprise=TRUE.
