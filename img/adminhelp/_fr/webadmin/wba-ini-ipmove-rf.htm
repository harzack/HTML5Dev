La section [ipmove] est utilisée pour contrôler l'envoi d'un élément à un iPool particulier.

 

Note :
Il existe un fichier de configuration distinct pour définir tous les arguments, par exemple, \bin\ipmove -inifile opentext.ini -config myconfig, où myconfig est un fichier au même format que le fichier d'initialisation standard. Pour définir les paramètres [ipmove], OpenText vous recommande d'utiliser le fichier myconfig, plutôt que de modifier directement le fichier opentext.ini. Pour en savoir plus, consultez le document Envoi d`un élément à un IPool utilisant ipmove.

 

readfields

  • Description :

    Le paramètre readfields fonctionne, lorsqu'il existe plus d'un iPool d'écriture, avec l'emplacement spécifié dans le champ Chemin d'accès au fichier, dans la section Contenu d'objet non EFS de la page Définition des paramètres généraux de l'Extracteur. Pour en savoir plus, consultez le document Définition des paramètres généraux de l`Extracteur.

    Si l'emplacement d'un chemin de répertoire est spécifié pour le champ Chemin d'accès au fichier, le nom d'origine du chemin d'accès est transmis au premier iPool. Tous les autres iPools qui obtiennent cet élément obtiennent également un exemplaire du fichier temporaire spécifié dans le champ Chemin d'accès au fichier. Il incombe au consommateur iPool de supprimer le fichier temporaire lorsqu'il a consommé le contenu et exécuté la transaction.

    Si un iPool de sortie n'est pas sélectionné, et si readfields n'est pas défini, alors la copie du fichier temporaire spécifié dans l'emplacement Chemin d'accès au fichier n'a pas lieu. En effet, les messages iPool sont copiés rapidement et les entrées ne sont pas analysées. La non-suppression éventuelle du fichier temporaire peut se produire avec un processus Two Way Tee ou avec une fusion.

  • Syntaxe :

    readfields=TRUE

  • Valeurs :

    TRUE ou FALSE. La valeur par défaut est FALSE.

    La définition de readfields=TRUE est uniquement requise en cas d'absence de modèle et en cas de spécification de plusieurs sorties. Si vous spécifiez un modèle, readfields=TRUE est automatiquement défini.
