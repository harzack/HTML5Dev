Les icônes utilisées pour identifier certains types MIME sont contrôlées par les correspondances icône-type MIME dans le fichier Content Server_home/config/mime.gif. Lorsqu'un utilisateur Content Server ajoute un document au système, Content Server note son type MIME et vérifie dans le fichier mime.gif si une icône spécifique a été définie pour le type MIME du nouveau fichier. Par exemple, si un utilisateur ajoute un document Microsoft Word dans Content Server, une icône Microsoft Word apparaît en regard dans Content Server, le fichier GIF de l'icône ayant été mappé sur le type MIME application/msword dans le fichier mime.gif.

Si le type MIME d'un document n'est pas spécifié dans le fichier mime.gif, Content Server utilise une icône Document par défaut .

Vous pouvez modifier le fichier mime.gif pour :

  • associer un nouveau fichier d'icônes ou un fichier d'icônes différent à un ou plusieurs types MIME existants ;

  • associer un nouveau type MIME à un fichier d'icônes existant ;

  • associer un nouveau type MIME à un nouveau fichier d'icônes.

 

Astuce :
Vous pouvez utiliser un éditeur d'image pour créer des icônes personnalisées que vous utiliserez avec les documents que vous ajoutez dans Content Server.

 

Création d'une icône personnalisée

Si vous souhaitez créer une icône personnalisée destinée à être utilisée dans Content Server, nous vous recommandons de créer trois tailles différentes pour chaque icône personnalisée :

  • La condition requise par défaut concerne une icône de 16 × 16 pixels, , que vous créez dans votre répertoire <Content Server_home>/support/webdoc.

  • Si un utilisateur choisit de définir un élément comme mis en valeur, Content Server affiche une icône de 32 × 32 pixels, que vous pouvez créer dans vos répertoires <Content Server_home>/support/webdoc et <Content Server_home>/support/mimetypeimages.

  • Content Server utilise une icône de 145 × 145 pixels sur la page Vue d'ensemble d'un élément. Vous créez cette icône dans votre répertoire <Content Server_home>/support/mimetypeimages.
