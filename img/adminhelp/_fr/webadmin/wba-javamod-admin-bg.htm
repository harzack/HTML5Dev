To take advantage of Content Server features written in Java, you must install and configure an application server which processes servlets, JSPs, etc. However, if you do not want to use Java-based features, you must disable them so that users trying to access them will not get error messages.

Java Virtual Machine (JVM) Settings

In order for Content Server to use JavaObjects, a Java Virtual Machine (JVM) is loaded in each Content Server thread. The default maximum amount of memory that each JVM can consume is 1 GB. A new configuration setting has been added to the Administer Java Module page to allow the administrator to set this limit to a different value to improve system performance. The JVM setting Maximum Heap Size allows you to set this limit.

You cannot set the maximum heap size larger than 2 GB. Once you have made a change to this setting, you must restart the servers.

Remote Method Invocation (RMI)

Adding Java to Content Server requires a protocol that can seamlessly manage communication for Java requests and responses between an application server and regular OScript requests. RMI allows object-to-object communication between two Java Virtual Machines (JVMs), where one JVM can invoke methods belonging to an object stored in another JVM. RMI makes objects in separate JVMs look and act like local objects.

By default, when you install Content Server, it automatically assigns an RMI port number. If necessary, you can change the port number at a later time. Once you have made a change to this setting, you must restart the servers.

Application Server

A configured application server is required for certain features in Content Server. Selecting the Configured checkbox will enable those features. By default, this checkbox is cleared.

Once you have made a change to this setting, you must restart the servers.

Secure Attribute for Cookies

This parameter prevents cookies from being sent over connections that are not secure. When the Secure Attribute For Cookies parameter is enabled, a browser is only allowed to send the cookie if the request is being sent over a secure channel (for example, HTTPS). By default, this setting is not enabled.

Allowed Client IPs

This parameter defines specific client IP addresses from which RMI connections are allowed to Content Server.

Once you have made a change to this setting, you must restart the servers.
