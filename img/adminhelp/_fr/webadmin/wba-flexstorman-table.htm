Cette page répertorie les règles de stockage disponibles dans Content Server. Lorsque la valeur que vous avez définie est assignée à un document, celui-ci est stocké dans le fournisseur de stockage spécifié dans la règle. Des exemples détaillant plusieurs de ces règles de stockage sont disponibles dans Exemples montrant comment définir des règles de stockage.

Règles de stockage par défaut

Les règles de stockage suivantes sont disponibles par défaut lorsque vous installez Content Server :

La taille du fichier en octets est supérieure à '?'

  • Description : si la taille de fichier d'un document dépasse la taille de fichier stipulée dans cette règle, le document est stocké dans le fournisseur de stockage spécifié dans la règle.

  • Valeur représentée par '?' : < taille_fichier> en octets. Par exemple, pour représenter 100 Ko, saisissez « 1048576 ».

  • Syntaxe : La taille de fichier en octets est supérieure à '1048576'

Le nom de la catégorie est '?'

  • Description : si la catégorie spécifiée est assignée à un document, celui-ci est stocké dans le fournisseur de stockage spécifié dans la règle.

  • Valeur représentée par '?' : < nom_catégorie>. Un exemple de nom de catégorie est « maCatégorie ».

  • Syntaxe : Le nom de la catégorie est 'maCatégorie'

Le type Mime est '?'

  • Description : si un document appartient au type MIME spécifié, celui-ci est stocké dans le fournisseur de stockage spécifié dans la règle.

  • Valeur représentée par '?' : < nom_type_MIME>. Un exemple de type MIME est « application/msword ».

  • Syntaxe : Le type MIME est 'application/msword'

La taille du fichier en octets est inférieure à '?'

  • Description : si la taille de fichier d'un document est inférieure à la taille de fichier stipulée dans cette règle, le document est stocké dans le fournisseur de stockage spécifié dans la règle.

  • Valeur représentée par '?' : < taille_fichier> en octets. Par exemple, pour représenter 100 Ko, saisissez « 1048576 ».

  • Syntaxe : La taille de fichier en octets est inférieure à '1048576'

Le nom du noeud contient '?'

  • Description : si le nom du nœud d'un document contient la chaîne spécifiée, celui-ci est stocké dans le fournisseur de stockage spécifié dans la règle. Saisissez toute chaîne pouvant faire partie du nom du nœud.

  • Valeur représentée par '?' : < nom_nœud>. Un exemple de nom de nœud est « test ».

  • Syntaxe : Le nom du nœud contient 'test'

Toujours (valeur ignorée)

  • Description : OpenText vous recommande de ne pas utiliser cette règle de stockage, dans la mesure où elle s'applique à tout type d'objet. Une valeur arbitraire doit être saisie.

Toute autre valeur d'attribut du noeud est '?'

  • Description : si un document a un attribut de nœud supplémentaire et que cet attribut de nœud supplémentaire contient la chaîne spécifiée, le document est stocké dans le fournisseur de stockage spécifié dans la règle. Saisissez toute chaîne pouvant faire partie de l'attribut de nœud.

    Cette règle s'applique aux attributs de nœud supplémentaires pouvant être définis sur la page Administration Gérer les attributs de nœud supplémentaires. Pour en savoir plus, voir Gestion des attributs de nœud supplémentaires.

  • Valeur représentée par '?' : < valeur_attribut_nœud>. Un exemple de la valeur d'attribut de nœud est « test ».

  • Syntaxe : Toute autre valeur d'attribut du nœud est 'test'

La valeur de l'attribut ? non spécifique est ?

  • Description : si un document a un attribut de catégorie nommé mais non spécifié contenant la chaîne spécifiée, le document est stocké dans le fournisseur de stockage spécifié dans la règle. L'attribut de catégorie ne doit pas être spécifié.

  • Valeur représentée par le premier ? : <nom_catégorie>.<nom_attribut> . Un exemple de nom de catégorie et de nom d'attribut est « maCatégorie.monAttr ».

  • Valeur représentée par le deuxième ? : <valeur_attribut>. Un exemple de la valeur d'attribut est « touteValeur ».

  • Syntaxe : La valeur de l'attribut non spécifique maCatégorie.monAttr est touteValeur

La valeur de l'attribut ? est ?

  • Description : si un document a un attribut de catégorie nommé et spécifié contenant la chaîne spécifiée, le document est stocké dans le fournisseur de stockage spécifié dans la règle. L'attribut de catégorie doit être spécifié. Cette règle de stockage est disponible car les attributs peuvent avoir plusieurs valeurs.

  • Valeur représentée par le premier ? : <nom_catégorie>[<x>].< nom_attribut>[<z>], où <x> et <z> sont des entiers positifs représentant la catégorie et l'attribut spécifiques. Un exemple référençant la troisième ligne de l'attribut monAttr dans la catégorie maCatégorie est : « maCatégorie[1].monAttr[3] ».

     

    Astuce :
    Comme il n'y a toujours qu'une seule ligne dans la catégorie, < x> correspond toujours à 1 et doit toujours être inclus.

     
     

    Note :
    Si la catégorie contient l'attribut Ensemble et si vous souhaitez spécifier une valeur dans cet attribut Ensemble, vous devez représenter la catégorie dans ce formulaire : <nom_catégorie>[< x>].<nom_ensemble>[<y>].< nom_attribut>[<z>]

    Un exemple référençant la troisième ligne d'un attribut nommé monAttr, dans la quatrième ligne d'un ensemble nommé monEnsemble, dans la catégorie nommée maCatégorie est : «  maCatégorie[1].monEnsemble[4].monAttr[3] ».

     

  • Valeur représentée par le deuxième ? : <valeur_attribut>. Un exemple de la valeur d'attribut est « touteValeur ».

  • Syntaxe sans ensemble : La valeur d'un attribut maCatégorie[1].monAttr[3] est touteValeur

  • Syntaxe avec ensemble : La valeur d'un attribut maCatégorie[1].monEnsemble[4].monAttr[2] est touteValeur

Toutes les miniatures

  • Description : Cette règle achemine les miniatures vers le fournisseur de stockage logique désigné.

  • Valeur : Aucune valeur n'est requise.

Espace disponible du volume (Mo)

  • Description : cette règle vérifie l'espace libre restant disponible sur le volume sur lequel Content Server tente de stocker le document. Si l'espace libre est inférieur à la valeur de l'espace libre configurée et spécifiée dans la règle, elle indique que le document ne peut pas être stocké sur le volume. Content Server recommence l'opération avec le volume suivant dans la liste jusqu'à ce qu'il puisse enregistrer le document. Si le document ne peut être enregistré sur aucun des volumes, il est stocké dans le volume par défaut.

  • Valeur : une valeur numérique, en Mo, doit être saisie. Un exemple est « 500 ».

Règles de stockage disponibles pour le module OpenText Archive Storage Provider

Certains modules ont des règles spécifiques disponibles uniquement lorsque ce module est installé. Cette section répertorie les règles de stockage ajoutées à Content Server grâce à l'installation du module Content Move, un module facultatif inclus avec le OpenText Archive Storage Provider.

 

Note :
Content Move rend également disponibles les opérateurs logiques AND, OR et NOT, ce qui vous permet de créer des règles complexes.

 

Projet '?'

  • Description : cette règle vérifie si un document est assigné à un projet spécifique que vous sélectionnez dans la liste. Dans ce cas, ce document est stocké dans le fournisseur de stockage spécifié dans la règle.

  • Valeur : le nom du nœud de ce projet.

  • Syntaxe : Projet '< nom_nœud_projet>'

Version la plus récente '?'

  • Description : Si la règle est définie sur TRUE, la dernière version ou la présentation du document sont stockées dans le fournisseur de stockage spécifié dans la règle. Si la règle est définie sur FALSE, toutes les versions et présentations, sauf la dernière version et la dernière présentation, sont stockées dans le fournisseur de stockage spécifié dans la règle.

  • Valeur : TRUE ou FALSE

  • Syntaxe : Dernière version 'TRUE' ou Dernière version 'FALSE'

Date de création/Date de modification '?'

  • Description : cette règle vérifie la date de création du document, ou la date de modification du document, si l'une d'entre elles existe. Si la date de création ou de modification du document est égale à la date spécifiée dans la règle, le document est stocké dans le fournisseur de stockage spécifié dans la règle.

    Vous pouvez également sélectionner une plage d'heure telle que : plus tôt, plus tard, égal, entre, exactement, plus et moins.

  • Valeur : une valeur de date sous la forme JJ/MM/AAAA. Par exemple, « 28/02/2012 ».

     

    Astuce :
    sélectionnez d'abord l'année, puis la date actuelle est prédéfinie.

     

  • Syntaxe : Date de création/Date de modification '28/02/2012'

Volume '?'

  • Description : cette règle vérifie si un document est stocké dans un volume spécifique. Dans ce cas, ce document est stocké dans le fournisseur de stockage spécifié dans la règle.

  • Valeur : sélectionnez un volume dans la liste. Un exemple de volume est : accueil Admin.

  • Syntaxe : Volume 'Accueil Admin'

Type d'objet dans ?

  • Description : cette règle vérifie si un type d'objet spécifique a été assigné à un document. Dans ce cas, ce document est stocké dans le fournisseur de stockage spécifié dans la règle. Vous pouvez sélectionner un ou plusieurs types.

    OpenText vous recommande de sélectionner uniquement les types avec un contenu.

  • Valeur : un type d'objet, avec un contenu, que vous sélectionnez dans la liste. Des exemples de types avec un contenu incluent : 144, 753, 825.

  • Syntaxe : Type d'objet dans 144

Stocké sous ?

  • Description : cette règle vérifie si un document est stocké dans un recueil spécifique que vous sélectionnez dans une liste. Dans ce cas, ce document est stocké dans le fournisseur de stockage spécifié dans la règle.

  • Valeur : le nom du nœud de ce recueil.

  • Syntaxe : Stocké ci-dessous <nom_nœud_recueil>

Stocké sous le type de recueil ?

  • Description : cette règle vérifie si un document est stocké dans un type de recueil spécifié que vous sélectionnez dans une liste. Dans ce cas, ce document est stocké dans le fournisseur de stockage spécifié dans la règle. Vous pouvez sélectionner un ou plusieurs types.

  • Valeur : le type de recueil.

  • Syntaxe : Type de recueil stocké ci-dessous <type_recueil> stocké ci-dessous

Stocké dans l'espace de travail personnel (valeur ignorée)

  • Description : cette règle vérifie si un document est stocké dans un espace de travail Personnel. Dans ce cas, ce document est stocké dans le fournisseur de stockage spécifié dans la règle.

  • Valeur : Nul, rien ne doit être saisi.

  • Syntaxe : Stocké dans l'espace de travail personnel (valeur ignorée)

Règles de stockage disponibles pour le module Classifications

Certains modules ont des règles spécifiques disponibles uniquement lorsque ce module est installé. Cette page répertorie les règles de stockage disponibles ajoutées à Content Server grâce à l'installation du module Classifications.

Classements assignés à un nœud

  • Description : si le classement spécifié, le classement Records management, le dossier Records Management ou la provenance sont assignés à un document, celui-ci est stocké dans le fournisseur de stockage spécifié dans la règle. Cette règle s'applique aux classements RM principaux et secondaires, ainsi qu'aux dossiers RM.

  • Valeur : <Chemin du classement>. Sélectionnez une valeur de classement, classement de Records management, dossier Records Management ou provenance dans la fenêtre Parcourir les classements ou Mes classements favoris.

  • Syntaxe : Classement :< nom de classement, nom de classement RM, nom de dossier RM ou provenance>

Pour plus d'informations sur cette règle, voir Configuration des règles de stockage pour les éléments classés

Règles de stockage disponibles pour le module Enterprise Library

Certains modules ont des règles spécifiques disponibles uniquement lorsque ce module est installé. Cette page répertorie les règles de stockage disponibles pour le module Enterprise Library pour Content Server.

Propriété variante '?' du type EL '?' est '?'

Dans ce cas, la règle vérifie si une valeur de type de propriété de variante spécifique est assignée au document. Dans ce cas, ce document est stocké dans le fournisseur de stockage spécifié dans la règle.

  • Valeur représentée par le premier ? : Enterprise Library <nom_type>

  • Valeur représentée par le deuxième ? : <nom_propriété_variante>

  • Valeur représentée par le troisième ? : <valeur_propriété_variante>

  • Syntaxe : Propriété variante 'abc' du type EL 'def' est 'xyz'

Est un objet Enterprise Library (valeur ignorée)

  • Syntaxe : Est un objet Enterprise Library (valeur ignorée)

L'application Enterprise Library est '?'

  • Description : cette règle vérifie si un document appartient à une application Enterprise Library spécifiée. Dans ce cas, ce document est stocké dans le fournisseur de stockage spécifié dans la règle.

  • Valeur : Enterprise Library <nom_application>.

  • Syntaxe : L'application Enterprise Library est '<nom_application>'

Le type Enterprise Library est '?'

  • Description : cette règle vérifie si un document appartient à un type Enterprise Library spécifié. Dans ce cas, ce document est stocké dans le fournisseur de stockage spécifié dans la règle.

  • Valeur : Enterprise Library <nom_type>.

  • Syntaxe : Le type Enterprise Library est '<nom_type>'

Les règles de stockage disponibles pour les modules Records Management et Security Clearance

Certains modules ont des règles spécifiques disponibles uniquement lorsque ce module est installé. Cette page répertorie les règles de stockage disponibles pour les modules Records management et Security Clearance.

Classement RM principal '?'

  • Description : cette règle vérifie si un classement Records Management spécifique a été assigné à un document. Dans ce cas, ce document est stocké dans le fournisseur de stockage spécifié dans la règle.

  • Valeur : le nom du classement Records Management. Parcourez l'arbre de classement et sélectionnez une valeur.

  • Syntaxe : Classement RM '<nom_classement_RM>'

Essentiel RM '?'

  • Description : cette règle vérifie si une valeur Essentiel Records Management spécifique a été assignée à un document. Dans ce cas, ce document est stocké dans le fournisseur de stockage spécifié dans la règle.

  • Valeur : la valeur Essentiel Records Management. Sélectionnez une valeur Essentiel dans la liste.

  • Syntaxe : Essentiel RM '<valeur_essentiel_RM>'

Statut RM '?'

  • Description : cette règle vérifie si un code de statut Records management spécifique a été assigné à un document. Dans ce cas, ce document est stocké dans le fournisseur de stockage spécifié dans la règle.

  • Valeur : <Code du statut> . Sélectionnez une valeur du code de statut RM dans la liste.

  • Syntaxe : Statut RM '<code_statut_RM>'

Niveau d'habilitation de sécurité '?'

  • Description : cette règle vérifie si un niveau d'habilitation de sécurité spécifique a été assigné à un document. Dans ce cas, ce document est stocké dans le fournisseur de stockage spécifié dans la règle.

    La règle s'applique spécifiquement au Niveau de sécurité choisi ; elle ne s'applique pas aux éléments ayant un Niveau de sécurité inférieur au niveau choisi. S'il existe plusieurs niveaux, il peut être nécessaire de concaténer une ou plusieurs règles d'habilitation de sécurité.

    Vous ne pouvez modifier que les règles qui utilisent les niveaux de sécurité pour lesquels vous avez l'autorisation Voir. Si des règles existantes utilisent des niveaux de sécurité pour lesquels vous n'avez pas l'autorisation Voir, vous constaterez que les règles existent mais que vous ne pouvez pas y accéder. Les documents dont les niveaux de sécurité ont été mis à jour sont placés dans la file d'attente et réévalués par la règle.

  • Valeur : un niveau d'habilitation de sécurité. Sélectionnez un niveau d'habilitation de sécurité dans la liste. Seuls les niveaux d'habilitation de sécurité pour lesquels vous avez l'autorisation Voir s'affichent dans la liste.

  • Syntaxe : Niveau d'habilitation de sécurité '<niveau_SC>'

Marquages supplémentaires '?'

  • Description : cette règle vérifie si des marquages supplémentaires d'habilitation de sécurité spécifiques ont été assignés à un document. Dans ce cas, ce document est stocké dans le fournisseur de stockage spécifié dans la règle.

    Vous ne pouvez modifier que les règles qui utilisent les marquages supplémentaires pour lesquels vous avez l'autorisation Voir. Si des règles existantes utilisent des niveaux de sécurité pour lesquels vous n'avez pas l'autorisation Voir, vous constaterez que les règles existent mais que vous ne pouvez pas y accéder. Les documents dont les marquages supplémentaires ont été mis à jour sont placés dans la file d'attente et réévalués par la règle.

  • Valeur : un marquage supplémentaire d'habilitation de sécurité. Sélectionnez un ou plusieurs marquages supplémentaires dans la liste. Seuls les marquages supplémentaires pour lesquels vous avez l'autorisation Voir s'affichent dans la liste.

  • Syntaxe : Marquages supplémentaires '<marquages_supplémentaires_SC>'
