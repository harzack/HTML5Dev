Vous pouvez définir la façon dont Content Server exécute les opérations de copie ou de suppression par lots dans la base de données (individuellement ou en groupe). Content Server effectue par défaut ces opérations sur une base individuelle ; si un seul élément échoue, l'ensemble du lot retrouve son état d'origine et aucune opération n'est exécutée dans la base de données. Dans un traitement en groupe, l'échec d'un élément n'entraîne pas l'échec de l'ensemble de l'opération. Tous les traitements d'élément corrects sont appliqués dans la base de données.

Dans la section Supprimer, vous pouvez activer l'option Retarder la suppression du fournisseur de stockage. Si cette option est activée, lorsqu'un élément est supprimé, son contenu n'est pas supprimé immédiatement. Le contenu est alors supprimé grâce à l'activité programmée de l'essai après l'échec de suppression du blob du fournisseur. Si vous activez le paramètre Retarder la suppression du fournisseur de stockage, vous devez également activer l'activité programmée Nouvel essai après l'échec de suppression du blob du fournisseur sur la page Configurer l'activité programmée. Pour en savoir plus, voir Gestion des échecs de suppression du blob.

Vous pouvez configurer les paramètres de commande de limitation de déplacement lors du déplacement d'objets d'un emplacement de Content Server à un autre. Pour que les utilisateurs déplacent des éléments dans Content Server, vous devez activer la commande de limitation de déplacement et spécifier un nombre maximum d'objets autorisés à être déplacés simultanément.

 

Note :
Les nœuds ne peuvent pas être déplacés, même si le nombre d'objets sélectionnés pour une opération de déplacement est inférieur au nombre maximum.
