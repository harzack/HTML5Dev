Pour configurer les paramètres de sécurité de l'audit :

  1. Connectez-vous à Content Server en tant qu'utilisateur administrateur.

     

    Note :
    Vous devez être connecté en tant qu'administrateur afin de configurer les paramètres de sécurité de l'audit. Vous connecter en tant qu'utilisateur bénéficiant du privilège Droits d'administration système n'est pas suffisant.

     

  2. Dans la section Administration système de la page Administration, cliquez sur Gérer l'audit des événements.

  3. Sur la page Gérer l'audit des événements, cliquez sur Paramètres de sécurité d'audit.

  4. Sur la page Paramètres de sécurité d'audit :

    1. Facultatif Si vous voulez empêcher tout utilisateur autre que l'utilisateur administrateur d'éliminer le journal d'audit ou de modifier les intérêts de l'audit, activez Limiter la config de l'audit à l'utilisateur Admin.

    2. Facultatif Si vous voulez empêcher tous les utilisateurs d'éliminer le journal d'audit :

      1. Pour empêcher un utilisateur, y compris l'utilisateur administrateur, d'éliminer le journal d'audit, cliquez sur Désactiver l'élimination de l'audit.

         

        Important :
        Après avoir cliqué sur Désactiver l'élimination de l'audit, l'action ne peut pas être annulée. Si le journal d'audit devient trop important, cela peut avoir un impact sur les performances.

         
         

        Astuce :
        Le bouton Désactiver l'élimination de l'audit est désactivé si l'élimination de l'audit a déjà été désactivée.

         

      2. Dans la boîte de dialogue Désactiver l'élimination de l'audit ?, activez Je comprends le risque lié à l'activation de ce paramètre et souhaite poursuivre, puis cliquez sur OK.

    3. Facultatif Si vous voulez empêcher tous les utilisateurs de modifier la configuration des intérêts de l'audit :

      1. Pour empêcher un utilisateur, y compris l'utilisateur administrateur, de modifier les centres d'intérêt à inclure dans l'audit et pour s'assurer que la page Définir les centres d'intérêts à inclure dans l'audit soit en lecture seule, cliquez sur Verrouiller les événements d'audit.

         

        Important :
        Après avoir cliqué sur Verrouiller les événements d'audit, l'action ne peut pas être annulée. Si de nouveaux intérêts sont ajoutés à Content Server, par exemple si vous installez un nouveau module ou si vous mettez à niveau un module existant, ces nouveaux intérêts ne peuvent pas être audités.

         
         

        Astuce :
        Le bouton Verrouiller les événements d'audit est désactivé si les centres d'intérêt à inclure dans l'audit ont déjà été verrouillés.

         

      2. Dans la boîte de dialogue Verrouiller les événements d'audit ?, activez Je comprends le risque lié à l'activation de ce paramètre et souhaite poursuivre, puis cliquez sur OK.

    4. Cliquez sur Envoyer.
