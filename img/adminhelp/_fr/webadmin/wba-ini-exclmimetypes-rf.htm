• Description :

    La section [ExcludedMimeTypes] définit le fonctionnement du processus Extracteur Entreprise. En incluant un type MIME dans cette section, vous indiquez à l'Extracteur Entreprise d'ignorer le contenu des documents de ce type MIME et d'extraire uniquement les métadonnées des documents de ce type MIME dans la base de données Content Server. Les métadonnées comprennent, par exemple, le nom et la date de création du document.

    Il existe de nombreux types de fichiers dont vous ne souhaiterez peut-être pas indexer le contenu, tels que les fichiers audio et image, puisqu'il ne s'agit pas de données textuelles. Lorsque le contenu de ces types de fichiers passe dans le flux de données de l'index Entreprise, le traitement est long et mobilise une grande quantité de ressources, même si les résultats obtenus ne sont finalement d'aucune utilité. C'est pourquoi l'ajout des types MIME de ces types de fichiers et des types de fichiers similaires dans la section [ExcludedMimeTypes] améliore l'efficacité du flux de données de l'index Entreprise en empêchant de transmettre les données que vous ne voulez pas indexer.

     

    Note :
    Lorsqu'un utilisateur ajoute un document à la base de données Content Server, Content Server enregistre le type MIME rapporté par le navigateur Web de l'utilisateur. Assurez-vous que tous les navigateurs Web utilisés pour vous connecter à Content Server sont correctement configurés. Si le navigateur Web d'un utilisateur est mal configuré, il peut parfois rapporter un type MIME incorrect. Si un navigateur Web se trompe et rapporte un type MIME qui apparaît dans la section [ExcludedMimeTypes], le contenu du document n'est ni extrait ni indexé, même si son type MIME fait partie de ceux dont vous voulez indexer le contenu.

     

  • Valeurs :

    Voici un exemple de la section [ExcludedMimeTypes] du fichier opentext.ini :

    [ExcludedMimeTypes]
    audio/basic=TRUE
    audio/mpeg=TRUE
    audio/x-aiff=TRUE
    audio/x-wav=TRUE
    image/gif=TRUE
    image/ief=TRUE
    image/jpeg=TRUE
    image/tiff=TRUE
    image/x-bmp=TRUE
    image/x-cmu-raster=TRUE
    image/x-pcx=TRUE
    image/x-pic=TRUE
    image/x-pict=TRUE
    image/x-portable-anymap=TRUE
    image/x-portable-graymap=TRUE
    image/x-portable-pixmap=TRUE
    image/x-rgb=TRUE
    image/x-xbitmap=TRUE
    image/x-xpixmap=TRUE
    image/x-xwindowdump=TRUE
    video/mpeg=TRUE
    video/quicktime=TRUE
    video/x-msvideo=TRUE
    video/x-sgi-movie=TRUE

     

    Note :
    Bien que les entrées de la section [ExcludedMimeTypes] apparaissent sous la forme <MIME_type>=TRUE, Content Serverignore la valeur qui suit le symbole d'égalité. Le paramétrage d'un type MIME sur FALSE n'a aucun effet. Pour inclure un type MIME particulier pour l'indexation, placez un signe dièse, #, devant l'entrée du type MIME ou supprimez-le.
