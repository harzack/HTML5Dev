Les paramètres de la section [DCS] définissent le comportement de la fonction d'indexation du service de conversion de documents, DCS. Le DCS utilise différents filtres de conversion pour convertir des données en HTML ou en texte brut au sein d'un processus de flux de données intermédiaire. Après avoir converti les documents, le DCS met les données à disposition du processus de distributeur de mise à jour en vue de l'indexation. Pour plus d'informations sur les filtres de conversion, reportez-vous à la rubrique Filtres de conversion de documents.

La section [DCS] du fichier opentext.ini contient les paramètres suivants :

•

logLevel

•

maxoctetsize

•

mod
•

modreq

•

NumThreads

•

QueueSize
•

rulesfile

•

Securedelete

Certains paramètres de la section [DCS] du fichier opentext.ini sont également présents dans la section [FilterEngine]. Les paramètres [DCS] exécutent la même fonction que les paramètres [FilterEngine] correspondants. Cependant, les valeurs définies pour les paramètres de la section [DCS] écrasent celles des paramètres de la section [FilterEngine]. Si aucune valeur n'est définie dans la section [DCS], la valeur utilisée est celle de la section [FilterEngine]. Cette page contient des informations relatives aux paramètres suivants que partagent les sections [DCS] et [FilterEngine] :

  • dllpath

  • logfile

logLevel

  • Description :

    Indique les événements et les informations devant figurer dans le fichier journal.

  • Syntaxe :

    logLevel=1

  • Valeurs :

    Un entier 0, 1, 2, 3 ou 4. La valeur par défaut est 1.

    Le tableau suivant contient des descriptions des valeurs valides pour le paramètre logLevel.

    Valeur Description
    0 Désactive la consignation dans le journal.
    1 Seuls les erreurs et les documents ne pouvant pas être indexés sont enregistrés. Il s'agit de la valeur par défaut.
    2 Consigne tous les éléments provenant de logLevel=1, ainsi que les événements DCS. Ces événements comprennent notamment le chargement d'un filtre de conversion et l'arrêt du processus de conversion.
    3 Consigne tous les éléments provenant de logLevel=2, ainsi que les statistiques de conversion de documents et les informations sur les documents. Parmi ces informations figurent l'heure de la conversion, la taille du fichier d'entrée, le type MIME, la balise OTURN et le filtre de conversion utilisé pour convertir le document.
    4 Consigne tous les éléments provenant de logLevel=3, ainsi que les événements du traitement IPool et les messages du journal générés par les filtres de conversion. Ce niveau ne doit être utilisé que pour identifier les problèmes survenant dans le flux de données.

maxoctetsize

  • Description :

    Détermine la taille maximum, en kilo-octets, des fichiers programme pouvant être convertis par le DCS. Certains documents texte inhabituels peuvent, dans certains cas, être identifiés en tant que fichiers application octet-stream. Au lieu de rejeter un document de ce type, le DCS tente de le convertir en ensemble de caractères UTF-8, ou Unicode. Si la procédure réussit, le document est indexé.

    Les fichiers plus volumineux que la taille définie seront rejetés sans être convertis au format UTF-8. Des valeurs inférieures peuvent empêcher l'indexation de documents texte supplémentaires. Des valeurs supérieures peuvent augmenter le nombre de tokens incorrects indexés par le moteur de recherche, ce qui affecte les performances de recherche.

  • Syntaxe :

    maxoctetsize=256

  • Valeurs :

    Un entier. La valeur par défaut est 256.

mod

  • Description :

    Détermine la section du fichier opentext.ini contenant la configuration d'un filtre de conversion. La section [DCS] peut contenir plusieurs paramètres mod correspondant à différents filtres de conversion.

     

    Attention
    La modification de la valeur de ce paramètre risque d'empêcher le DCS de fonctionner ou de fonctionner correctement. Ne modifiez pas les valeurs de ces paramètres, sauf si le service d'assistance clientèle d'OpenText vous le demande.

     

  • Valeurs :

    Le tableau suivant présente les valeurs par défaut pour le paramètre mod.

    Nom Valeur
    modx01 QDF
    mod03 Summarizer
    modx04 DCSxpdf
    modx05 Languageid
    modx06 DCSmail

     

    Note :
    Le caractère x indique que le processus du filtre de conversion sera exécuté au sein d'un processus de traitement.

    Le processus de filtre Summarizer doit être exécuté au sein du DCS, et non d'un processus de traitement.

     

modreq

  • Description :

    Détermine le gestionnaire de requêtes utilisé par le DCS. Le gestionnaire de requête accepte des documents d'entrée d'un type de source spécifique (IPool, socket ou canal, par exemple) et envoie les documents au DCS afin d'effectuer la conversion. Les documents convertis sont ensuite renvoyés au gestionnaire de requête et envoyés à leur destination (un client ou un IPool, par exemple).

     

    Attention
    La modification de la valeur de ce paramètre risque d'empêcher le DCS de fonctionner ou de fonctionner correctement. Ne modifiez pas la valeur de ce paramètre, sauf si le service d'assistance clientèle d'OpenText vous le demande.

     

  • Valeurs :

    La valeur par défaut est DCSipool.

modx10

  • Description :

    Ce paramètre est requis pour utiliser le filtre OTDF avec Content Server sur la plate-forme Linux®.

  • Syntaxe :

    modx10=DCSIm

  • Valeurs :

    La seule valeur autorisée est DCSIm.

NumThreads

  • Description :

    Détermine le nombre maximum de conversions de documents pouvant être effectuées simultanément par le DCS. La modification de la valeur par défaut influence les performances de la conversion de documents.

  • Syntaxe :

    NumThreads=2

  • Valeurs :

    Un entier compris entre 1 et 64. La valeur par défaut est 2.

QueueSize

  • Description :

    Détermine le nombre maximum de documents mis en file d'attente par le DCS lorsque le nombre maximum de conversions de documents simultanées a été atteint.

     

    Important :
    OpenText vous recommande vivement de ne pas modifier la valeur de ce paramètre.

     

  • Valeurs :

    Un entier. La valeur par défaut est 20.

rulesfile

  • Description :

    Détermine le nom du fichier qui définit les règles du traitement de la conversion de documents de différents types MIME.

     

    Important :
    OpenText vous recommande vivement de ne pas modifier la valeur de ce paramètre.

     

  • Valeurs :

    Un chemin absolu. La valeur par défaut est <Content Server_home> \config\dcsrules.txt, où <Content Server_accueil> est la racine de votre installation Content Server.

Securedelete

  • Description :

    Détermine si des modèles de suppression sécurisée sont utilisés sur les fichiers temporaires générés par le DCS pour les rendre non-récupérables.

    Lors de la création d'un index, le DCS génère des fichiers temporaires pouvant contenir du texte ou d'autres éléments extraits des fichiers en cours d'indexation. Lorsqu'un fichier temporaire n'est plus utile, le flux de données utilise les appels système normaux pour le supprimer. Dans la plupart des cas, ce processus supprime l'entrée du fichier mais conserve une image du fichier sur le disque.

    Les sites sur lesquels la sécurité est importante peuvent définir un niveau de suppression sécurisée, chargé de déterminer la manière dont le système remplace l'image laissée sur le disque. Le paramètre Securedelete définit le niveau de suppression sécurisée pour le DCS.

     

    Note :
    Si vous définissez le paramètre Securedelete sur une valeur autre que la valeur par défaut, OpenText vous recommande de définir le niveau de suppression sécurisée sur la même valeur pour les iPools. Pour plus d'informations sur le niveau de suppression sécurisée, voir Configuration d`une suppression sécurisée des fichiers temporaires.

     

  • Syntaxe :

    Securedelete=0

  • Valeurs :

    Un entier compris entre 0 et 4, zéro et quatre. Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut au paramètre Securedelete=0.

    La définition de ce paramètre sur 0 désactive la fonctionnalité de suppression sécurisée pour le DCS. La définition de ce paramètre sur une valeur supérieure représente des suppressions de fichiers de plus en plus complexes.

dllpath

  • Description :

    Détermine l'emplacement du répertoire contenant les filtres de conversion de Content Server. Un document créé par un traitement de texte, Microsoft Word par exemple, contient des informations de formatage qui ne sont pas nécessaires ou requises pour la recherche. Les filtres de conversion extraient le contenu de texte nécessaire pour lire des fichiers de traitement de texte et effectuer une indexation à partir de ces fichiers.

  • Syntaxe :

    dllpath=C:\OPENTEXT\filters

  • Valeurs :

    Un chemin absolu. La valeur par défaut est <Content Server_home> \filters, où <Content Server_accueil> est la racine de votre installation Content Server.

logfile

  • Description :

    Détermine l'emplacement et le préfixe du fichier journal de conversion de documents. Le numéro de port administrateur attribué à ce processus est ajouté au fichier journal, suivi du suffixe .log. Ceci empêche que différents processus écrivent dans le même fichier.

     

    Note :
    Ce paramètre est obligatoire. Pour désactiver la journalisation, définissez logLevel=0.

     

  • Syntaxe :

    logfile=C:\OPENTEXT\logs\dcs

  • Valeurs :

    Un chemin absolu. La valeur par défaut est <Content Server_home> \logs\dcs, où <Content Server_accueil> est la racine de votre installation Content Server.
