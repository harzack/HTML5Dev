Lors de l'installation de Content Server, il n'existe qu'un seul groupe, appelé Groupe par défaut, qui contient un seul utilisateur, appelé Admin. Le Groupe par défaut est le groupe Service de l'utilisateur Admin. (Un service est le groupe principal dont un utilisateur est membre.) Après avoir terminé l'installation et la configuration initiales de Content Server, vous devez créer des utilisateurs pour chaque personne utilisant Content Server et les organiser en groupes.

Pour des raisons de performances, un groupe ne peut pas contenir plus de 1 000 utilisateurs et sous-groupes (y compris le Groupe par défaut). Si votre système Content Server gère un grand nombre d'utilisateurs, vous pouvez contourner cette limite en créant des groupes Service, auxquels vous pouvez ajouter des sous-ensembles d'utilisateurs. Vous pouvez ensuite désigner chaque service comme membre d'un seul groupe principal. Il devient ainsi possible de travailler avec chaque utilisateur dans le système. Par exemple, pour ajouter chaque utilisateur de Content Server à un projet, il vous suffit d'ajouter le groupe principal au projet.

Si vous supprimez un groupe qui est le groupe de service d'un utilisateur, Content Server réaffecte automatiquement tous les utilisateurs du groupe supprimé au groupe par défaut.

Affichage d\'une liste de groupes

Content Server vous permet d'afficher la liste de tous les groupes ou d'un sous-ensemble de groupes en fonction d'une recherche de correspondance de modèles selon des noms de groupes. Par défaut, Content Server affiche jusqu'à 30 groupes par page de résultat de recherche. Si le résultat de votre recherche comprend plus de 30 éléments, le bouton Suite s'affiche sur la page de résultats, vous permettant d'afficher la page de résultats suivante. Vous pouvez changer cette option par défaut en modifiant le paramètre MaxUsersToListPerPage dans la section [general] du fichier _home/config/opentext.ini.

Interdiction des groupes récursifs

Lors de la création d'un sous-groupe dans Content Server, il est possible de former des groupes récursifs. Par exemple, vous pouvez désigner le Groupe A comme membre du Groupe B, puis le Groupe B comme membre du Groupe A. Les groupes récursifs pouvant réduire les performances du système, vous pouvez choisir d'interdire la formation de groupes récursifs dans Content Server.
