La section [Servers] se rapporte uniquement à l'utilisation du module Content Server Explorer. Dans cette section, l'administrateur fournit une correspondance de chemin de répertoire selon laquelle un serveur principal se connecte à un ou plusieurs serveurs secondaires, afin que les utilisateurs puissent effectuer des opérations sur des éléments entre les serveurs. Par exemple, des opérations d'ouverture, de copie et de suppression.

L'affichage de Content Server Explorer s'apparente à celui de l'Explorateur Windows, familier aux utilisateurs des systèmes d'exploitation Windows. Si un nom de serveur secondaire fait l'objet d'une correspondance dans le fichier opentext.ini, tous les utilisateurs peuvent voir le nom du serveur secondaire dans l'arborescence et ouvrir les dossiers, sous-dossiers et éléments.

Dans l'exemple suivant, remote_0 correspond au nom du serveur Content Server secondaire. Il est suivi par l'URL qui lie le serveur principal au serveur secondaire.

#server_0=remote_0|http://myNT/support/cs.exe"
