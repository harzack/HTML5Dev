Le système de requête en langage naturel Content Server utilise la fonction offerte par le résumeur Content Server pour analyser le texte entré par les utilisateurs lors de l'envoi de requêtes en langage naturel. La section [nlqsearch] contient à peu près les mêmes paramètres que la section [Summarizer]. Toutefois, les valeurs par défaut de ces paramètres peuvent être différentes.

La section [nlqsearch] du fichier opentext.ini contient les paramètres suivants :

•

SumAbbrevFile

•

SumDefFile
•

SumDocFreqFile

•

SumStopWordFile
•

SumTagFile

SumAbbrevFile

  • Description :

    Indique le fichier d'abréviations utilisé par le résumeur Content Server pour définir les abréviations courantes et toutes les combinaisons de trois lettres ne constituant pas un mot.

  • Syntaxe :

    SumAbbrevFile=../config/abbrev.eng

  • Valeurs :

    Un chemin relatif et un nom de fichier. Par défaut, le fichier d'abréviations s'appelle abbrev.eng et il est enregistré dans le répertoire config de votre installation Content Server. Par exemple, <Content Server_home>/config, où <Content Server_accueil> est la racine de votre installation Content Server.

SumDefFile

  • Description :

    Désigne le fichier de définition utilisé par le résumeur Content Server pour définir son fonctionnement. Le fichier de définition contient cinq nombres (un par ligne) représentant les éléments suivants :

    • Un multiplieur de taux pour les phrases situées dans les 20 premiers pour cent d'un document. Il s'agit probablement de phrases d'introduction, idéales pour constituer un résumé.

    • Un nombre maximum d'unités lexicales autorisées dans une phrase de résumé. Une unité lexicale est une combinaison de lettres, de nombres, de traits et de références d'entité. Le résumeur identifie comme illisibles les phrases qui contiennent plus d'unités lexicales que la limite autorisée. Ces phrases ne seront pas utilisées pour constituer un résumé.

    • Un nombre minimum d'unités lexicales autorisées dans une phrase de résumé. Le résumeur identifie comme illisibles les phrases qui contiennent moins d'unités lexicales que le nombre minimum requis. Ces phrases ne seront pas utilisées pour constituer un résumé.

    • Un taux maximum d'unités non lexicales par rapport aux unités lexicales. Si le taux actuel d'unités non lexicales par rapport aux unités lexicales est supérieur à ce nombre, le résumeur identifie la phrase comme illisible. La phrase ne sera pas utilisée pour constituer un résumé.

    • Le nombre de documents utilisés pour former les données relatives à la signification statistique des mots dans le fichier de fréquence d'occurrence des mots.

  • Syntaxe :

    SumDefFile=../config/natlang.eng

  • Valeurs :

    Un chemin relatif à la valeur du paramètre otpath dans la section [OTCommon] du fichier opentext.ini. Par défaut, le fichier de définitions s'appelle natlang.eng et il est enregistré dans le répertoire config de votre installation Content Server. Par exemple, <Content Server_home>/config, où <Content Server_accueil> est la racine de votre installation Content Server.

SumDocFreqFile

  • Description :

    Désigne le fichier de fréquence d'occurrence des mots contenant les données nécessaires au résumeur Content Server pour calculer la signification statistique des mots dans des documents. Ce fichier contient une liste de mots présents dans plus de 1 000 documents sur les 1 371 876 utilisés par OpenText pour élaborer l'arrière-plan statistique des paramètres par défaut du résumeur.

  • Syntaxe :

    SumDocFreqFile=../config/docfreq.eng

  • Valeurs :

    Un chemin relatif et un nom de fichier. Par défaut, le fichier de fréquence d'occurrence des mots s'appelle docfreq.eng et il est enregistré dans le répertoire config de votre installation Content Server. Par exemple, <Content Server_home>/config, où <Content Server_accueil> est la racine de votre installation Content Server.

SumStopWordFile

  • Description :

    Désigne le fichier des mots vides utilisé par le résumeur Content Server pour définir une liste de mots vides courants. Les mots vides sont des mots n'ajoutant aucune valeur sémantique à une phrase. Il s'agit essentiellement de mots tels que un(e), et et le/la. En différenciant les mots vides et les mots ayant une importance sémantique, le résumeur Content Server identifie les mots susceptibles de contribuer à l'intérêt du document, en augmentant la pertinence de ses résumés et locutions clés.

  • Syntaxe :

    SumStopWordFile=../config/nlstopword.eng

  • Valeurs :

    Un chemin relatif et un nom de fichier. Par défaut, le fichier des mots vides s'appelle nlstopword.eng et il est enregistré dans le répertoire config de votre installation Content Server. Par exemple, <Content Server_home>/config, où <Content Server_accueil> est la racine de votre installation Content Server.

SumTagFile

  • Description :

    Spécifie le fichier de balises contenant une liste des balises de marquage qui figurent dans les documents. Par exemple, les balises de marquage HTML, XML et SGML. Un chiffre indiquant l'importance de la balise pour le résumeur Content Server se trouve en regard de chaque balise.

    Tableau des paramètres de signification des balises :

    Numéro Libellé Description
    0 IGNORE_TAG Le résumeur ignore ces balises mais pas les données qu'elles contiennent. Si une phrase débute avant cette balise et la contient, le résumeur ne sépare pas la phrase en deux phrases distinctes. Par défaut, les balises inconnues sont identifiées par ce chiffre.
    1 SENTENCE_BREAKING_IGNORE_TAG Le résumeur ignore ces balises mais pas les données qu'elles contiennent. Si une phrase débute avant cette balise et la contient, le résumeur sépare la phrase en deux phrases distinctes.
    2 IGNORE_BETWEEN_TAGS Le résumeur ignore ces balises et les données qu'elles contiennent.
    3 PRESENTATION Cette valeur marque comme phrases de résumé (de vue d'ensemble) les données contenues dans ces balises. Pour générer le résumé, le résumeur utilise les phrases du titre, du résumé et de la conclusion, dans cet ordre.
    4 CONCLUSION Cette valeur identifie comme phrases de conclusion les données contenues dans ces balises. Pour générer le résumé, le résumeur utilise les phrases du titre, du résumé et de la conclusion, dans cet ordre.
    5 TITLE_MAJOR Cette valeur identifie comme phrases de titre les données contenues dans ces balises. Pour générer le résumé, le résumeur utilise les phrases du titre, du résumé et de la conclusion, dans cet ordre.

  • Syntaxe :

    SumTagFile=../config/tags.eng

  • Valeurs :

    Un chemin relatif et un nom de fichier. Par défaut, le fichier des balises s'appelle tags.eng et il est enregistré dans le répertoire config de votre installation Content Server. Par exemple, <Content Server_home>/config, où <Content Server_accueil> est la racine de votre installation Content Server.
