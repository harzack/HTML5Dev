Une base de données Content Server comporte un ensemble de tables relationnelles qui sont stockées dans un SGBDR. Pour que vous puissiez vous connecter à une installation Content Server nouvelle ou existante, la dernière phase de la création d'une base Content Server consiste à créer les tables de la base Content Server à partir de la page Créer des tables Content Server. La page Créer des tables Content Server apparaît une fois que vous avez cliqué sur le bouton Connexion de la page Connexion de l'administrateur correspondant à votre type de SGBDR.

Pour créer des tables Content Server sous un SGBDR Oracle, vous devez spécifier le nom et le mot de passe du compte Oracle qui doit être propriétaire des tables. Pour créer des tables Content Server sous un SGBDR Microsoft SQL Server, vous devez spécifier le nom et le mot de passe du compte SGBDR qui doit être propriétaire des tables, ainsi que le nom de la base de données SQL Server sur laquelle le compte a des privilèges de connexion.

Avant de créer les tables Content Server, vous devez créer le compte et le référentiel SGBDR (espace de stockage pour les fichiers Oracle, journaux et de données et base de données SQL Server pour Microsoft SQL Server) auxquels le compte se connecte à partir de la page Maintenance du serveur correspondant à votre type de SGBDR.

En plus de créer le compte et le référentiel SGBDR, vous devez choisir un mode de stockage des données, à savoir interne ou externe. Pour plus d'informations sur le stockage de documents interne ou externe, reportez-vous à la rubrique Gestion de stockage interne et externe.

Après avoir créé le compte et le référentiel SGBDR et choisi le type de stockage, vous pouvez créer les tables Content Server en utilisant la procédure correspondant à votre type de SGBDR :

  • Serveur Oracle

  • Microsoft SQL Server
