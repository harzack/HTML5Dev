Les processus Directory Walker permettent d'extraire des données d'un ensemble de fichiers stocké sur votre système de fichiers et de les écrire dans un flux de données où elles sont indexées.

Après avoir créé un processus Directory Walker, vous pouvez modifier les paramètres de navigation et les répertoires parcourus par le processus Directory Walker. Si vous modifiez ces informations, vous devez relancer le processus Directory Walker afin qu'il parcoure une nouvelle fois votre système de fichiers avec les nouvelles informations. La relance du processus Directory Walker permet de mettre à jour l'index en ajoutant de nouveaux fichiers, en mettant à jour les fichiers existants et en enlevant les fichiers supprimés.

Pour parcourir à nouveau les répertoires spécifiés sans modifier les paramètres du processus Directory Walker, cliquez sur l'icône Fonctions du gestionnaire de flux de données correspondant, puis sélectionnez Reprendre. Vous pouvez également parcourir à nouveau les répertoires en cliquant sur l'icône Fonctions du processus Directory Walker, puis en sélectionnant Démarrer.

Le tableau suivant décrit les éléments de l'onglet Infos spécifiques de la page Propriétés du processus Directory Walker.

Onglet Infos spécifiques de la page Propriétés du processus Directory Walker

Elément Description
Etat Spécifie si le processus Directory Walker est exécuté ou non. Vous ne pouvez pas modifier ce champ.
Serveur d'administration Spécifie le raccourci du serveur d'administration sur lequel réside l'hôte qui exécute le processus Directory Walker. En modifiant cette valeur, vous devrez probablement exécuter aussi une des opérations suivantes :
•

Modifier le chemin du répertoire d'écriture.

•

déplacer le répertoire d'écriture. Si vous le déplacez, vous devez aussi modifier le répertoire de lecture du processus Conversion de documents du flux de données.

Ne modifiez pas le raccourci du serveur d'administration destiné au processus Directory Walker sauf en cas de nécessité (par exemple, si l'hôte du serveur d'administration n'est plus disponible).
Activer la gestion du système Détermine si Content Server contrôle ce processus pour détecter le retour de messages d'erreur. S'il retourne un message d'erreur, Content Server l'enregistre dans la base de données. La gestion du système est activée par défaut. Pour plus d'informations sur la configuration de Content Server en vue de l'envoi d'e-mails d'alerte lorsqu'un processus de flux de données rencontre des erreurs, reportez-vous à la rubrique Pour activer la recherche d'erreurs et l'envoi d'e-mails.
Modèle de commande Spécifie le modèle utilisé pour générer la ligne de commande du processus Directory Walker. Ne modifiez pas cette valeur, sauf si le service d'assistance clientèle d'OpenText vous le demande.
Ligne de commande Spécifie la ligne de commande utilisée par Content Server pour exécuter le processus Directory Walker. Vous ne pouvez pas modifier la valeur de ce champ.
Répertoire de démarrage Spécifie le répertoire dans lequel le processus Directory Walker est exécuté. Le processus Directory Walker doit toujours être exécuté dans le répertoire Content Server_accueil/config. Ne modifiez pas cette valeur, sauf si le service d'assistance clientèle d'OpenText vous le demande.
IPools Vous permet d'accéder aux informations sur le répertoire d'écriture du processus Directory Walker. Le répertoire d'écriture est le répertoire dans lequel le processus Directory Walker enregistre les données. Ne modifiez pas le répertoire d'écriture sauf si vous avez déplacé l'IPool ou si le service d'assistance clientèle d'OpenText vous le demande.
Fichier journal Spécifie le chemin d'accès absolu au fichier journal dans lequel Content Server enregistre les activités du processus Directory Walker. Si le champ est vide, la fonction de journalisation est désactivée.
Niveau de consignation Spécifie le niveau de consignation (Valeur par défaut, Détaillé ou Déboguer) appliqué si le journal est activé.
•

Par défaut pour enregistrer uniquement les erreurs dans le journal. Il s'agit du niveau de consignation minimum.

•

Détaillé pour enregistrer le statut des messages acheminés à travers les processus et les IPools du flux de données dans le journal. Les erreurs sont également enregistrées.

•

Déboguer pour enregistrer des informations détaillées au sujet du statut du flux de données et des erreurs dans le journal. Compte tenu de l'importante quantité de données générées, OpenText recommande de choisir ce niveau uniquement si vous devez déboguer des problèmes graves sur le flux de données.

Emplacement de la base de données Historique des fichiers parcourus Spécifie le chemin d'accès aux fichiers de la base de données Historique des fichiers parcourus qui enregistrent des informations au sujet des répertoires que doit parcourir le processus Directory Walker. Lorsqu'un processus Directory Walker parcourt une nouvelle fois un ensemble de répertoires, il compare les fichiers de la base de données Historique des fichiers parcourus aux fichiers stockés dans le répertoire pour identifier les nouveaux fichiers, ainsi que les fichiers mis à jour ou supprimés. Le processus Directory Walker extrait des informations relatives aux fichiers ajoutés, remplacés ou supprimés dans le flux de données. Il n'extrait pas une nouvelle fois la totalité du jeu de fichiers. Ainsi, la mise à jour de l'index s'en trouve plus efficace.
Groupes de répertoires Spécifie les groupes de répertoires parcourus par le processus Directory Walker. Chaque groupe de répertoires contient le nom des répertoires parcourus par le processus Directory Walker. Il définit aussi les types de fichiers que le processus Directory Walker doit inclure et/ou exclure en parcourant les répertoires, les sous-répertoires à parcourir, ainsi que les intervalles de dates et de tailles des fichiers correspondant aux critères. Vous pouvez ajouter, modifier ou supprimer des groupes de répertoires après avoir créé un processus Directory Walker.
Options de démarrage Détermine les options de lancement du processus Directory Walker. En démarrant, le processus parcourt les répertoires spécifiés pour ajouter des données et les mettre à jour en fonction du contenu actuel des répertoires.
•

Manuel, option par défaut permettant de lancer manuellement le processus Directory Walker à chaque fois que vous souhaitez parcourir à nouveau les répertoires spécifiés.

•

Programmé pour programmer le jour et l'heure d'exécution du processus Directory Walker. Si les répertoires parcourus par le processus changent fréquemment, vous pouvez programmer le jour et l'heure auxquels le processus devra parcourir régulièrement les répertoires.

•

Continuel pour exécuter le processus Directory Walker en continu.

Options d'arrêt Détermine les options d'arrêt du processus Directory Walker. Trois options d'arrêt sont disponibles :
•

Terminer arrête le processus immédiatement—sans lui laisser le choix.

•

Port de message d’arrêt pour obliger le serveur d'administration par défaut à envoyer un message d'arrêt au port sur lequel le processus Directory Walker s'exécute. Ce message d'arrêt indique au processus Directory Walker de s'achever normalement. La valeur que vous spécifiez dans ce champ est ajoutée à la ligne de commande lorsque vous mettez à jour les paramètres de l'onglet Infos spécifiques de la page Propriétés du processus Directory Walker. Il s'agit de l'option d'arrêt par défaut du processus Directory Walker.

•

Exécutable d'arrêt pour spécifier un fichier exécutable demandant au processus de s'arrêter

Le champ Valeur maximale du code de sortie correcte définit le numéro de code d'erreur le plus élevé pour lequel le serveur d'administration relancera automatiquement le processus Directory Walker en présence d'une erreur. OpenText vous recommande de définir la Valeur maximale du code de sortie correcte pour un processus Directory Walker sur 99. Ne modifiez pas la Valeur maximale du code de sortie correcte, sauf si le service d'assistance clientèle d'OpenText vous le demande.
Démarrer/Arrêter Cet élément vous permet de gérer des flux de données, selon son état courant. Si le processus est en cours d'exécution ou programmé pour être exécuté, le bouton Arrêter s'affiche. Si le processus est arrêté, le bouton Démarrer s'affiche.
Resynchroniser Cet élément resynchronise les informations relatives à ce processus (dans le fichier otadmin.cfg du serveur d'administration du processus) avec les informations de la base de données Content Server.
Mettre à jour Permet de soumettre au serveur Content Server les modifications que vous avez apportées à cette page.
Réinitialiser Rétablit les informations de cette page à l'état qui était le leur à l'ouverture de la page.
