Pour définir les options du journal et les fichiers de la base de données Historique des fichiers parcourus :

  1. Saisissez le chemin absolu du fichier journal dans le champ Fichier journal. Saisissez par exemple Content Server_home/index/prefix/logs/nom.log, où nom correspond au nom vous permettant d'associer le fichier journal à ce processus Directory Walker.

  2. Dans la liste déroulante Niveau de consignation, cliquez sur l'un des niveaux de consignation suivants :

    • Par défaut pour enregistrer uniquement les erreurs dans le journal. Il s'agit du niveau de consignation minimum.

    • Détaillé pour enregistrer le statut des messages acheminés à travers les processus et les IPools du flux de données dans le journal. Les erreurs sont également enregistrées.

    • Déboguer pour enregistrer des informations détaillées au sujet du statut du flux de données et des erreurs dans le journal. Compte tenu de l'importante quantité de données générées, OpenText recommande de choisir ce niveau uniquement si vous devez déboguer des problèmes graves sur le flux de données.

  3. Dans le champ Emplacement de la base de données Historique des fichiers parcourus, saisissez le chemin absolu du répertoire dans lequel vous souhaitez que le processus Directory Walker enregistre ses fichiers d'historique.

     

    Note :
    Si vous supprimez une base de données Historique du processus Directory Walker et que vous modifiez la casse du répertoire, le processus Directory Walker considère que tous les objets de la base de données sont nouveaux lorsqu'il parcourra de nouveau le répertoire.
