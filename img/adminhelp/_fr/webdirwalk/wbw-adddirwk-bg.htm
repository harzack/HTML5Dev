La plupart des index Content Server sont créés à l'aide des modèles d'index fournis par Content Server. Les modèles d'index vous permettent de créer tous les objets système associés à un index particulier en une seule étape. Vous pouvez par exemple créer l'index Entreprise, un index Directory Walker, un index producteur XML Activator, un index d'aide administrateur ou un index d'aide utilisateur. Pour cela, sélectionnez l'option appropriée dans le menu Ajouter un élément de la page Administration système. Après avoir fourni les informations nécessaires, Content Server génère automatiquement en une seule étape les objets système correspondants (par exemple, le dossier source de données, le gestionnaire de flux de données, le flux de données et tous ses processus, ainsi que le gestionnaire de recherche et ses moteurs de recherche). Vous pouvez également créer des index en ajoutant des objets système individuels à un dossier source de données que vous ajoutez à la page Administration système. Pour plus d'informations sur l'ajout de composants d'index, reportez-vous à la rubrique Création individuelle de composants d'index.

La création d'un processus Directory Walker est nécessaire lorsque vous souhaitez indexer des données stockées sur votre système de fichiers. Pour chaque processus Directory Walker que vous créez, vous devez définir les répertoires de niveau supérieur et les niveaux de sous-répertoires que le processus doit parcourir. Le processus Directory Walker parcourt les répertoires de niveau supérieur que vous avez définis (et leurs sous-répertoires, s'il y a lieu) et extrait les fichiers correspondant aux critères définis.

Vous déterminez les types de fichiers recueillis par le processus Directory Walker en spécifiant les critères d'inclusion et d'exclusion (tels que les modèles de noms de fichier, les intervalles de dates et les intervalles de tailles de fichier). Le processus Directory Walker enregistre les fichiers correspondant à vos critères dans une réserve d'échange de données (IPool). Ils restent accessibles pour le processus suivant du flux de données.

Si vous créez un index Directory Walker à l'aide du modèle d'index fourni par Content Server, Content Server définit la journalisation de l'index en utilisant des valeurs par défaut. Le cas échéant, Content Server génère un fichier journal Directory Walker dans le répertoire de journaux de votre installation Content Server (Content Server_home/logs) et définit le niveau de consignation sur Valeur par défaut. Une fois que le flux de données Directory Walker a été créé, vous pouvez modifier ces valeurs lors de la configuration du processus Directory Walker. Si vous créez un index Directory Walker en ajoutant des objets système individuels sur un dossier source de données, vous devez définir l'emplacement du fichier journal et le niveau de consignation du processus Directory Walker. Un fichier journal du processus Directory Walker décrit le statut actuel du processus : il indique si le démarrage et l'arrêt se sont déroulés correctement et établit une liste des avertissements ou des erreurs qui se sont produits lors de l'extraction des données.

Vous pouvez également configurer un processus Directory Walker pour qu'il stocke des informations dans les fichiers de la base de données Historique des fichiers parcourus. Les fichiers de la base de données Historique des fichiers parcourus contiennent des informations sur les fichiers traités par le processus Directory Walker. Lorsqu'un processus Directory Walker parcourt un ensemble de répertoires déjà parcouru, il compare les fichiers de la base de données Historique des fichiers parcourus aux fichiers enregistrés dans le répertoire pour localiser les nouveaux fichiers, ainsi que les fichiers mis à jour ou supprimés. Le processus Directory Walker extrait des informations relatives aux fichiers ajoutés, remplacés ou supprimés dans le flux de données. Il n'extrait pas une nouvelle fois la totalité du jeu de fichiers. Ainsi, la mise à jour de l'index s'en trouve plus efficace. Lorsque vous créez un processus Directory Walker de manière individuelle, vous devez définir le répertoire dans lequel le processus Directory Walker enregistre ses fichiers de la base de données Historique des fichiers parcourus. Lorsque vous créez un flux de données Directory Walker à l'aide des modèles d'index, Content Server crée automatiquement les fichiers de la base de données Historique des fichiers parcourus. Vous pouvez ensuite modifier l'emplacement des fichiers lors de la configuration du processus Directory Walker.

La création d'un flux de données producteur XML Activator s'effectue en quatre étapes.

Configuration d'un processus Directory Walker

Pour configurer un processus Directory Walker, il vous suffit de le nommer et de définir le serveur d'administration sur l'hôte duquel vous souhaitez exécuter le processus.

Configuration des options de journal et fichiers d'historique

Pour consigner dans un journal l'activité du processus Directory Walker que vous ajoutez à un flux de données, vous pouvez définir le nom du fichier journal à associer au processus et le type d'activité à enregistrer. Si vous ne souhaitez pas consigner dans un journal l'activité du processus Directory Walker, il vous suffit, lors de l'ajout d'un processus Directory Walker, de ne pas définir les informations relatives au journal.

Définissez également l'emplacement des fichiers de la base de données Historique des fichiers parcourus créés par Directory Walker. Lors de l'exécution d'un processus Directory Walker, les informations relatives aux fichiers traités par le processus sont enregistrées dans les fichiers de la base de données Historique des fichiers parcourus. Le processus Directory Walker utilise ces informations lorsqu'il parcourt un répertoire pour la seconde fois afin de localiser les fichiers nouveaux, mis à jour ou supprimés. OpenText vous recommande de stocker les fichiers de la base de données Historique des fichiers parcourus dans le répertoire où le processus Directory Walker écrit les données : le répertoire que vous spécifiez dans le champ Répertoire d'écriture sur la page Ajouter : Directory Walker.

Lorsque vous spécifiez des répertoires sur la page Ajouter : Directory Walker, vous devez indiquer le chemin des répertoires tel qu'ils sont mappés/montés sur l'hôte du serveur d'administration dont vous avez sélectionné le raccourci dans la liste déroulante Hôte.

Configuration des groupes de répertoires

Pour qu'un processus Directory Walker puisse s'exécuter, vous devez identifier les répertoires dans lesquels le processus va rechercher les données. Déterminez les répertoires de niveau supérieur et les niveaux de sous-répertoires que le processus doit analyser. Pour chaque groupe de répertoires défini, vous pouvez également identifier les types de fichiers que le processus Directory Walker doit inclure ou exclure.

Lorsque vous spécifiez des répertoires sur la page Ajouter : Directory Walker, vous devez indiquer le chemin des répertoires tel qu'ils sont mappés/montés sur l'hôte du serveur d'administration dont vous avez sélectionné le raccourci dans la liste déroulante Hôte.

Configuration de l'IPool et des options de démarrage/arrêt

Après avoir configuré le processus Directory Walker, spécifié ses paramètres d'historique des fichiers parcourus et ses paramètres de consignation, puis déterminé les répertoires à analyser, vous devez définir l'emplacement de la réserve d'échange de données (iPool) du processus ainsi que les options de lancement et les options d'arrêt.

Lorsque vous spécifiez des répertoires sur la page Ajouter : Directory Walker, vous devez indiquer le chemin des répertoires tel qu'ils sont mappés/montés sur l'hôte du serveur d'administration dont vous avez sélectionné le raccourci dans la liste déroulante Hôte.

 

Note :
Si vous mélangez les formats de chemin dans le processus Directory Walker, par exemple, un lecteur mappé en tant que répertoire à parcourir et un répertoire UNC dans le champ Exclure, la restriction à exclure est ignorée. La restriction à inclure est également ignorée dans le champ Inclure.
