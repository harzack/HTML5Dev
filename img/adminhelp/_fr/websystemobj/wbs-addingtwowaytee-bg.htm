Un processus T peut être ajouté à un gestionnaire de flux de données d'une source de données. Lorsque vous ajoutez un processus T à un gestionnaire de flux de données, vous pouvez configurer les paramètres suivants :

Paramétrage Valeurs
Nom Indiquez un nom pour le T.
Description Indiquez une description pour le T. Ces informations s'affichent dans l'onglet Général lorsque vous consultez les propriétés du T.
Hôte Indiquez le serveur d'administration sur l'hôte à partir duquel le processus T doit s'exécuter.
Activer la gestion du système Sélectionnez cette option pour permettre à Content Server de surveiller le statut du processus T. Lorsque cette option est activée, si le processus retourne des messages d'erreur, Content Server les enregistre dans la base de données.
 

Astuce :
Pour plus d'informations sur la configuration de Content Server en vue de l'envoi d'e-mails d'alerte lorsqu'un processus de flux de données rencontre des erreurs, reportez-vous à la rubrique Pour activer la recherche d'erreurs et l'envoi d'e-mails.

 
Mode de sortie
Sélectionnez :
•

Copier les messages IPool dans toutes les sorties, pour envoyer tous les messages IPool à cinq processus de sortie IPool maximum

•

Distribuer les objets IPool (contenu de message) parmi toutes les sorties, pour distribuer tous les messages IPool parmi cinq processus de sortie IPool maximum

 

Note :
Lorsque vous sélectionnez le mode de sortie de distribution, les champs Région de filtre et Valeur de filtre sont désactivés dans la zone Ecrire les données dans.

 
Informations sur Interchange Pool
Lire les données à partir de
Pour Zone principale IPool, indiquez le chemin absolu de l'IPool à partir duquel le processus T doit lire les données.
Pour Processus, choisissez le processus qui doit précéder le processus T dans le flux de données.
 

Note :
Si le flux de données contient déjà des processus producteurs ou intermédiaires, leurs noms s'affichent dans la liste déroulante Processus.

 
Ecrire les données dans
Pour Ajouter un lien d'écriture IPool, si vous le souhaitez, cliquez sur le bouton Ajouter un lien d'écriture IPool afin de configurer jusqu'à cinq processus qui suivent le processus T.
Pour Zone principale IPool, indiquez le chemin absolu de l'IPool vers lequel le processus T doit diviser les données.
Pour Processus, choisissez le processus qui doit suivre le processus T dans le flux de données.
 

Note :
Si le flux de données contient déjà au moins un processus intermédiaire ou consommateur, les noms des processus apparaissent dans la liste déroulante Processus.

 
Pour Région de filtre, spécifiez le nom d'une région. Par exemple, vous pouvez spécifier la région de métadonnées OTDCategory. Seuls les objets auxquels cette région est appliquée sont filtrés vers le processus de destination.
 

Astuce :
Pour plus d'informations sur certaines régions de métadonnées dans Content Server, reportez-vous à la rubrique Définition des régions d'index.

 
Pour Valeur de filtre, indiquez une valeur autorisée pour la région spécifiée dans le paramètre Région de filtre. Par exemple, si vous spécifiez la région de métadonnées OTDCategory dans le paramètre Région de filtre, et si vous indiquez l'ID de nœud d'une catégorie dans le champ Valeur de filtre, tous les objets de la région OTDCategory et de la catégorie spécifiée sont filtrés vers le processus de destination.
Options de démarrage
Programme
Sélectionnez :
•

Manuel, si vous souhaitez que le processus T traite les données lorsque vous le lancez vous-même

•

Programmé, si vous souhaitez que le processus T s'exécute automatiquement

Pour A cette heure, définissez l'heure dans les listes déroulantes, puis cliquez sur les jours de la semaine auxquels le processus T doit s'exécuter.
Pour Intervalle, sélectionnez l'intervalle selon lequel le processus T doit s'exécuter.
Pour Attente, définissez les valeurs des listes déroulantes afin de sélectionner l'intervalle entre deux exécutions du processus T.
Sélectionnez Continuel pour que le processus T s'exécute continuellement. En cas d'arrêt inopiné du processus, son serveur d'administration le redémarre automatiquement.
Options d'arrêt
Valeur maximale du code de sortie correcte Indiquez le numéro de code d'erreur le plus élevé pour lequel le serveur d'administration redémarre automatiquement le processus T en cas d'échec.
 

Astuce :
Si vous spécifiez la valeur -1, le processus T se termine immédiatement lorsqu'il s'arrête et le service d'administration ne le redémarre pas en cas d'échec.

 
Ne modifiez pas ce paramètre, sauf si le service d'assistance clientèle d'OpenText vous le demande.

Pour obtenir des instructions sur l'ajout d'un processus T, voir Pour ajouter un T.

 

Note :
Seules les informations ajoutées à Content Server après l'ajout du processus T sont envoyées au processus ajouté au flux de données. Pour envoyer les données d'index actuelles au processus, vous devez effectuer une réindexation. Pour plus d'informations sur la réindexation, voir Vérification du contenu et réindexation.
