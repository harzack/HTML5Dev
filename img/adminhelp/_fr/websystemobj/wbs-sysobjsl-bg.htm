Lorsque les utilisateurs Content Server créent des requêtes sur la page Recherche, ils spécifient des critères de recherche et peuvent choisir une vue de recherche (Content Server recherche par défaut la vue de recherche Entreprise). Content Server applique chaque requête que les utilisateurs Content Server envoient à la vue de recherche qu'ils spécifient. Le nom des vues de recherche disponibles sur un site Content Server s'affiche dans la liste Vues de recherche de la page Recherche et dans la liste déroulante de la barre de recherche qui se trouve dans la partie supérieure de la plupart des autres pages Content Server.

Une fois enregistrée dans Dossier - Vues de recherche, une requête devient une vue de recherche. Cela signifie que le nom de la vue de recherche s'affiche automatiquement dans la liste Vues de recherche de la page Recherche. Si vous enregistrez des requêtes dans un autre dossier Content Server, vous pouvez à tout moment les exécuter à nouveau. Après avoir créé une vue de recherche, vous pouvez cliquer sur son nom dans Dossier - Vues de recherche pour exécuter la requête correspondante et afficher les résultats de la recherche.

Lorsque vous supprimez un flux de données d'indexation, la vue de recherche qui représente l'ensemble de toutes les données de cet index est également supprimée. Ainsi, les requêtes, les formulaires et les modèles de recherche enregistrés et associés à cette vue de recherche n'auront pas d'ensemble de données à rechercher et aucun résultat de recherche ne sera produit. Cependant, lorsque vous créez un index à partir d'un modèle d'index, vous pouvez choisir de remplacer la vue de recherche de l'index supprimé par la vue de recherche du nouvel index. Les requêtes, les formulaires et les modèles de recherche enregistrés sont alors associés au nouvel index.

Vous pouvez concevoir des vues de recherche pour qu'elles répondent aux besoins de votre entreprise. La liste suivante décrit certains types de vue de recherche usuels :

  • Si votre système Content Server contient plusieurs index (par exemple les index Entreprise, Directory Walker, Aide et Content Server Spider), vous pouvez créer une vue de recherche comprenant tous vos index.

  • Vous pouvez créer une vue de recherche basée sur une requête que vous élaborez sur la page Recherche.

  • Vous pouvez créer une vue de recherche basée sur un formulaire de recherche que vous élaborez sur la page Recherche.

  • Vous pouvez créer une vue de recherche qui permet aux utilisateurs Content Server de faire porter la requête sur les informations d'un dossier Content Server spécifique et de ses sous-dossiers.

  • Vous pouvez créer une vue de recherche qui permet aux utilisateurs Content Server de faire porter la requête sur les informations d'un projet spécifique.

Vous pouvez également définir un ordre personnalisé pour les vues de recherche de la barre de recherche et pour une recherche avancée. De plus, vous pouvez activer ou désactiver l'option A partir d'ici de la barre de recherche et définir la taille du composant Vue de recherche pour la recherche avancée. Pour en savoir plus, consultez le document Pour configurer les vues de recherche dans un ordre personnalisé.
