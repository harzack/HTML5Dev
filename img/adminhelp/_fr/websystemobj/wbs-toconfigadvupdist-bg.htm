Onglet Paramètres avancés de la page Propriétés du distributeur de mise à jour

 

Note :
Certains paramètres dans cet onglet sont affectés par des paramètres associés dans l'onglet Spécifique de la page Propriétés d'un schéma de partition. Pour en savoir plus, voir Configuration d'un schéma de partition. Les paramètres affectés sont identifiés ci-dessous dans les descriptions.

 

Elément Description
Paramètres avancés
Délai d'attente autorisé pour le moteur d'indexation Indique le nombre de délais d'attente consécutifs pour le moteur d'indexation qui sont autorisés avant que le processus Distributeur de mise à jour ne s'arrête. La durée considérée comme un délai d'attente est définie par le paramètre Délai d'attente de mise à jour du moteur d'indexation.
Délai d'attente de mise à jour du moteur d'indexation Indique la durée pendant laquelle le processus Distributeur de mise à jour attend une réponse d'un moteur d'indexation avant que la mise à jour d'un index ne soit abandonnée.
Point de vérification simultané de limite d'écriture
Indique le nombre de fichiers de point de vérification du moteur d'indexation qui peuvent être écrits en parallèle. Cette fonction empêche l'écriture de fichiers de point de vérification de saturer l'entrée/la sortie du disque et d'entraîner un « emballement ». Les valeurs possibles sont des entiers compris entre 1 et un très grand nombre, mais une valeur trop élevée n'empêche pas l'« emballement ». La valeur par défaut est 8. Un entier inférieur à 1 désactive cette fonction.
Répertoire de démarrage Indique le répertoire dans lequel le distributeur de mise à jour s'exécute. Ce paramètre doit être un chemin absolu dans le répertoire Content Server_accueil /bin du serveur d'administration qui exécute le distributeur de mise à jour.
Taille du lot
Indique le nombre d'objets par lots transactionnels. La valeur par défaut est 500.
OpenText vous recommande de définir cette valeur sur :
•

100 lorsque Mode Stockage de valeur est sélectionné

•

500 ou plus pour le Mode Mémoire faible

 

Note :
Cette valeur doit être inférieure au paramètre Nombre d'éléments dans la section Limites IPool de la page Définir les paramètres généraux de l'Extracteur. Pour en savoir plus, consultez le document Définition des paramètres généraux de l'Extracteur.

 
Préférence de partitions
Indique le nombre de partitions à remplir avant que des nouvelles ne soient créées. Ce paramètre est facultatif et vide par défaut.
Les moteurs d'indexation peuvent être plus efficaces pour utiliser l'unité centrale et réduire le nombre de fichiers de point de vérification qui doivent être écrits lorsque ce paramètre est à environ 6 ou 8 partitions. OpenText recommande de ne pas modifier la Préférence de partitions sauf si le service d'assistance clientèle d'OpenText vous le demande.
Journalisation
Fichier journal Indique l'emplacement du fichier journal du processus Distributeur de mise à jour. Le fichier journal enregistre des informations sur le processus et peut être utile lors d'un dépannage.
Niveau de débogage Indique le type de message qui sera enregistré dans le fichier journal du processus Distributeur de mise à jour :
•

Niveau d'informations : enregistre tous les types de messages.

•

Niveau de statut : enregistre les messages de statut périodiques, les messages d'avertissement et tous les messages d'erreur.

•

Niveau d'avertissement : enregistre les messages d'avertissement et tous les messages d'erreur.

•

Niveau d'erreur : enregistre les messages d'erreur typiques et graves.

•

Niveau d'erreur grave : enregistre les messages d'erreur graves.

Intervalle de vidage du journal Indique le nombre de nouveaux messages enregistrés dans le journal du processus Distributeur de mise à jour avant que le processus n'écrive sur le disque.
Options du fichier journal Indique les paramètres pour l'écriture des fichiers journaux.
  Actions du fichier journal
Indique comment le fichier journal du processus Distributeur de mise à jour est affecté lorsque le processus redémarre :
•

Ajouter à l'élément existant : toutes les nouvelles informations sont ajoutées à la fin du fichier journal actuel.

•

Créer : le fichier journal remplace le fichier journal précédent.

•

Créer (Enregistrer l'élément existant) : le fichier journal existant est enregistré et renommé, et un nouveau fichier journal est créé sur le modèle du fichier journal existant.

•

Défilement : enregistre et ferme le fichier journal existant et crée un nouveau fichier journal

  Taille du fichier journal Indique une limite, en Mo, sur la taille totale maximale pour les fichiers journaux. La valeur par défaut est 100.
  Journaux de démarrage à tenir Indique le nombre de fichiers journaux à tenir avant de commencer à les remplacer. La valeur par défaut est 5.
  Journaux supplémentaires à tenir Indique le nombre de fichiers journaux supplémentaires à tenir avant de commencer à les remplacer. La valeur par défaut est 10.
Actions
Processus Permet de démarrer ou d'arrêter le processus du moteur d'indexation ou tous les processus d'indexation (le processus Distributeur de mise à jour et les processus du moteur d'indexation appartenant à un système d'indexation et de recherche). Que vous arrêtiez tous les processus d'indexation ou uniquement le processus Distributeur de mise à jour, l'indexation s'arrête. Si les processus d'indexation ne sont pas en cours d'exécution, les boutons Démarrer s'affichent ; si les processus d'indexation sont en cours d'exécution ou s'ils sont programmés, les boutons Arrêter s'affichent.
Processus d'indexation Permet d'arrêter puis de démarrer tous les processus d'indexation appartenant à un système d'indexation et de recherche.
Option Permet de resynchroniser les informations sur le processus Distributeur de mise à jour avec les informations de la base de données Content Server.
