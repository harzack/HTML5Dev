L'index Entreprise est un index de toutes les données stockées dans la base de données Content Server. Dans des circonstances normales, l'index Entreprise est automatiquement créé lors de l'installation de Content Server. A ce moment, les processus qui génèrent l'index Entreprise et assurent sa mise à jour sont initialisés. Pendant l'installation et la configuration initiale de Content Server, la page Créer une nouvelle source de données Entreprise apparaît automatiquement et vous invite à créer l'index Entreprise. Si l'index Entreprise n'est pas créé au moment de l'installation ou s'il a été supprimé par la suite, vous pouvez le générer automatiquement en sélectionnant Source de données Entreprise dans le menu Ajouter un élément de la page Administration système. Dans ce cas, Content Server génère tous les objets système requis pour créer l'index Entreprise en une seule étape. Cependant, si vous voulez créer les objets système de l'index Entreprise individuellement, vous pouvez ajouter un flux de données. Pour plus d'informations, reportez-vous à la rubrique Ajout de processus de flux de données.

Un flux de données Entreprise contient un ou plusieurs processus Extracteur, un ou plusieurs processus de conversion de documents et un processus du distributeur de mise à jour. Un processus Extracteur effectue un contrôle continu de la base de données Content Server pour déterminer si de nouvelles informations ont été ajoutées, modifiées ou supprimées. Il ordonne ensuite au serveur d'envoyer les ajouts, les modifications et les suppressions à un IPool. Un processus Conversion de documents lit à partir de cet IPool, convertit les nouvelles données et les données modifiées au format HTML ou texte brut, puis écrit les données converties et les notifications de suppression dans un autre IPool. Le processus Distributeur de mise à jour lit les données à partir de cet IPool, les transmet aux processus Moteur d'indexation qu'il gère et met l'index Entreprise à jour. Le distributeur de mise à jour et les moteurs d'indexation sont aussi des processus d'un système d'indexation et de recherche.

Un seul processus Extracteur étant habituellement requis pour extraire des données de la base de données Content Server, et un seul processus Conversion de documents étant habituellement requis pour convertir les données au format HTML ou texte brut, OpenText vous recommande de ne prévoir qu'un seul processus Extracteur et un seul processus Conversion de documents pour chaque système Content Server, sauf si les services généraux ou le service d'assistance clientèle d'OpenText vous recommande l'ajout de plusieurs processus dans le cadre d'une stratégie d'indexation de gros volumes. Pour plus d'informations sur l'indexation de gros volumes, reportez-vous à la rubrique Configuration de l'indexation de gros volumes.

L'index Entreprise est mis à jour par incréments pour refléter les modifications les plus récentes apportées à la base de données Content Server. Cela permet de garantir la mise à jour systématique des résultats de recherche reçus par les utilisateurs Content Server.

Il existe deux scénarios classiques de création pour l'index Entreprise : pendant l'installation et la configuration de Content Server ou une fois l'installation et la configuration terminées.

Création d'un index Entreprise au cours de l'installation

Les conditions suivantes doivent être remplies :

  • La page Créer une nouvelle source de données Entreprise est affichée dans votre navigateur Web. Elle fait partie de l'installation et de la configuration de Content Server.

  • Si vous voulez créer l'index Entreprise sur un hôte Content Server secondaire, vous avez déjà installé et enregistré cet hôte. Pour plus d'informations, consultez la section OpenText Content Server - Installation Guide (LLESCOR-IGD).

S'il s'agit d'une installation Content Server principale sur un hôte où il en existe déjà d'autres, assurez-vous que les serveurs et les processus de flux de données correspondant à ces installations existantes sont en cours d'exécution avant de créer les processus de flux de données de cet index Entreprise. Cela permet à Content Server de détecter automatiquement les numéros de port déjà utilisés. Si vous voulez créer l'index Entreprise sur un hôte Content Server distant, configurez le serveur d'administration sur cet hôte avant de configurer le processus Extracteur.

Création d'un index Entreprise après l'installation

Vous pouvez créer un index Entreprise après l'installation si l'index Entreprise d'origine a été supprimé ou si vous n'avez pas créé d'index Entreprise lors de l'installation. L'index ainsi créé inclut un processus Extracteur, un processus de conversion de documents et un processus du distributeur de mise à jour.

Un seul index Entreprise avec l'ensemble des processus standard étant habituellement requis pour extraire des données de la base de données Content Server, OpenText vous recommande de ne prévoir qu'un seul index pour chaque système Content Server, sauf si les services généraux ou le service d'assistance clientèle d'OpenText recommande l'ajout de plusieurs index ou processus dans le cadre d'une stratégie d'indexation de gros volumes. Pour plus d'informations sur l'indexation de gros volumes, reportez-vous à la rubrique Configuration de l'indexation de gros volumes.
