Vous pouvez créer un index de données à l'aide des modèles d'index Content Server. Les modèles créent les objets système associés à une source de données spécifique. Les sources de données peuvent être les suivantes :

  • Les données de l'index Entreprise. Pour en savoir plus, voir Création de l'index Entreprise.

  • Les fichiers du système d'aide utilisateur ou administrateur Content Server. Pour en savoir plus, voir Création de l'index de l'aide utilisateur ou administrateur.

  • Des données spécifiques de votre système de fichiers. Pour en savoir plus, voir Indexation des données sur votre système de fichiers.

  • Les données collectées par les processus d'extraction associés aux modules Content Server facultatifs, tels que le module Content Server Spider.

  • Des données recueillies à partir d'un flux de données producteur XML Activator. Pour en savoir plus, voir Création d'un flux de données producteur XML Activator.

Pour plus d'informations sur le module Content Server Spider, reportez-vous à la documentation qui l'accompagne.

Les modèles d'index fournis par Content Server créent les flux de données qui relient les processus suivants dans une chaîne : un processus producteur (par exemple, Extracteur, Directory Walker, Producteur XML Activator, Content Server Spider), un processus Conversion de documents, un processus Distributeur de mise à jour et un processus Importateur.

Les modèles d'index Content Server créent également les objets système suivants qui sont associés à une source de données particulière : un dossier source de données, un gestionnaire de flux de données (qui contient les processus de flux de données), un gestionnaire de recherche (qui contient les fédérateurs de recherche et les moteurs de recherche), un schéma de partition et une ou plusieurs partitions (en fonction du nombre que vous spécifiez). Pour plus d'informations sur les schémas de partition, reportez-vous à la rubrique Utilisation des schémas de partition. Un fédérateur de recherche, un ou plusieurs moteurs de recherche et moteurs d'indexation sont également créés, en fonction du nombre de partitions que vous spécifiez. Pour chaque partition, un moteur d'indexation est créé et associé automatiquement au processus Distributeur de mise à jour dans le flux de données. Pour chaque partition, un moteur d'indexation est également créé et associé à un fédérateur de recherche. Pour plus d'informations sur les processus du distributeur de mise à jour et les moteurs d'indexation, voir Configuration des processus d'indexation. Pour plus d'informations sur les fédérateurs de recherche et les moteurs de recherche, voir Gestion des processus de recherche. Les partitions, les schémas de partition, le processus Distributeur de mise à jour, les fédérateurs de recherche, les moteurs d'indexation et les moteurs de recherche sont tous des composants d'un système d'indexation et de recherche.

Une fois que vous avez créé un index à l'aide des modèles Content Server, vous pouvez y ajouter des objets système à tout moment. Par exemple, vous pouvez ajouter des partitions, qui permettent aux données d'être distribuées dans d'autres index de plus petite taille (chaque partition gère son propre index de données, qui constitue une portion de l'ensemble de l'index de données). Vous pouvez également ajouter des fédérateurs de recherche (cet ajout entraînant l'ajout simultané de moteurs de recherche). Pour plus d'informations sur l'ajout individuel d'objets système à un index, reportez-vous à la rubrique Création individuelle de composants d'index. Après avoir créé un index, vous pouvez gérer les objets système (par exemple, les processus de recherche et d'indexation et les processus de flux de données) qui lui sont associés. Pour plus d'informations sur la gestion des flux de données, reportez-vous à la rubrique Gestion des flux de données.
