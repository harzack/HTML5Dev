Lorsque le serveur d'administration démarre pour la première fois sur un hôte Content Server principal ou secondaire, Content Server crée les fichiers admserv.log dans le répertoire Content Server_home/logs. Vous pouvez modifier les paramètres des journaux pour le serveur d'administration s'exécutant sur l'hôte Content Server principal en définissant de nouvelles valeurs de journal sur la page Configurer les paramètres du débogage. Pour changer les paramètres des journaux pour des serveurs d'administration s'exécutant sur des hôtes Content Server secondaires, vous devez modifier le fichier Content Server_home/config/opentext.ini sur l'hôte secondaire. Pour plus d'informations sur les serveurs d'administration, reportez-vous à la rubrique Resynchronisation des serveurs d'administration.

Vous pouvez modifier les paramètres de la page Configurer les paramètres du débogage pour modifier les options par défaut des journaux pour les serveurs d'administration s'exécutant sur un hôte Content Server principal. Par défaut, Content Server enregistre toutes les informations de journalisation des serveurs d'administration s'exécutant sur un hôte Content Server principal dans un fichier journal intitulé admserv.log, stocké dans le répertoire des journaux de l'installation Content Server principale. Vous pouvez changer le nom et l'emplacement du fichier journal du serveur d'administration si vous voulez enregistrer ces informations dans un fichier et/ou à un emplacement différent. Vous pouvez également changer le niveau de consignation dans un journal par défaut ou modifier les paramètres sur la page Configurer les paramètres du débogage afin de préciser si vous voulez enregistrer le flux de données entre le serveur et le serveur d'administration.
