Les règles de contrôle de flux de données gèrent les flux de données de votre site Content Server. Elles vous permettent de définir l'état ou la condition que les flux de données peuvent rencontrer et d'associer certaines actions à ces conditions. Par exemple, vous pouvez créer une règle de contrôle de flux de données qui ordonne à Content Server d'arrêter le flux et d'envoyer un e-mail d'alerte relatif aux objets système lorsque des erreurs surviennent dans un processus de flux de données particulier.

 

Note :
Content Server envoie des e-mails d'alerte relatifs aux objets système aux destinataires spécifiés sur la page Configurer l'envoi d'e-mails d'alerte relatifs aux objets système. Pour plus d'informations, reportez-vous à la rubrique Pour activer la recherche d'erreurs et l'envoi d'e-mails.

 

Vous pouvez créer des règles de contrôle de flux de données basées sur le nombre de messages IPool d'un flux de données, le temps qu'un flux de données reste inactif ou bloqué, les erreurs de processus d'un flux de données ou la quantité d'espace disque disponible sur l'ordinateur où est exécuté le flux de données.

Nombre de messages

Une règle de contrôle Nombre de messages démarre ou arrête une alimentation ou un lecteur IPool et envoie des e-mails d'alerte relatifs aux objets système lorsque le nombre de messages IPool d'un flux de données est supérieur ou inférieur à un nombre spécifié. Par exemple, si vous voulez que Content Server envoie un message d'information lorsque le nombre de messages IPool dépasse 200 dans une alimentation iPool, vous pouvez créer une règle de contrôle qui identifie cette condition.

Flux de données discrets

Une règle de contrôle Flux de données silencieux démarre ou arrête une alimentation ou un lecteur IPool et envoie des e-mails d'alerte relatifs aux objets système lorsqu'un flux de données reste silencieux (c'est-à-dire qu'il ne reçoit aucun message IPool) pendant un certain temps. Les flux de données silencieux sont des flux de données dont les processus s'exécutent normalement et ne contiennent pas de messages IPool. Par exemple, si vous voulez que Content Server envoie un message d'erreur lorsqu'un flux de données ne contient aucun message IPool pendant 30 minutes, vous pouvez créer une règle de contrôle qui identifie cette condition.

Cette règle de contrôle est particulièrement utile pour les flux de données comportant un niveau faible, comme les flux de données d'aide, car elle vous permet de fermer automatiquement les flux de données qui ne sont plus actifs. Vous pouvez également utiliser cette règle de contrôle de flux de données pour signaler des problèmes dans un flux de données comportant un volume élevé.

Flux de données bloqués

Une règle de contrôle Flux de données bloqué arrête une alimentation IPool ou le flux de données et envoie des e-mails d'alerte relatifs aux objets système lorsqu'un flux de données est bloqué pendant un certain temps. Les flux de données bloqués sont des flux de données dont les processus s'exécutent, mais ne traitent pas les messages IPool reçus par le flux de données. Par exemple, si vous voulez que Content Server envoie un message d'erreur grave et arrête l'alimentation IPool lorsqu'un flux de données est bloqué pendant plus de six heures, vous pouvez créer une règle de contrôle qui identifie ces conditions.

Erreur de processus dans le flux de données

Une règle de contrôle Erreur du processus dans le flux de données arrête le flux de données et envoie des e-mails d'alerte relatifs aux objets système lorsque des erreurs surviennent pendant l'exécution de l'un des processus du flux de données. Si une erreur de processus se produit, un message décrivant la condition d'erreur spécifique s'affiche dans l'onglet Infos spécifiques de la page Propriétés du processus de flux de données correspondant. Vous pouvez fonder les règles de contrôle de flux de données sur l'existence d'erreurs de processus dans un flux de données, mais vous ne pouvez pas les fonder sur une erreur de processus spécifique.

Espace disque

Une règle de contrôle Espace disque démarre ou arrête une alimentation IPool, un lecteur IPool ou le flux de données et envoie des e-mails d'alerte relatifs aux objets système lorsqu'une certaine quantité d'espace disque est disponible. Par exemple, si vous voulez arrêter un lecteur IPool et envoyer un message d'avertissement lorsqu'une certaine quantité d'espace disque n'est plus disponible, vous pouvez créer une règle de contrôle chargée d'identifier ces conditions.
