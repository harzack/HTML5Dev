La page Configurer les options de recherche contrôle les paramètres des composants de recherche de Content Server.

Toute modification apportée aux paramètres de recherche affecte tous vos utilisateurs lorsque vous cliquez sur Mettre à jour, car les paramètres représentent la configuration de recherche par défaut.

Vous pouvez activer, désactiver et modifier les paramètres par défaut des composants suivants :

  • Rechercher similarités

  • Mise en évidence des résultats du résumé

  • En-tête des résultats de recherche

  • Affichage des résultats de recherche

  • Paramètres de mémoire cache

Configuration de la recherche Rechercher similarités

Le composant Rechercher similarités vous permet d'ajouter la commande Rechercher similarités au menu Fonctions des documents dans les espaces de travail et sur la page Résultats de recherche.

La recherche des similarités est un moyen rapide de rechercher dans Content Server des documents similaires à l'original. La commande parcourt Content Server à la recherche de documents contenant les cinq locutions clés les plus courantes (mots et combinaisons de mots récurrents, notamment les combinaisons incluant des mots peu courants) de l'original. Si le document original ne contient pas de locutions clés, la commande cherche dans Content Server des documents contenant les termes du titre de l'original.

Les locutions clés utilisées pour la recherche des similarités peuvent être différentes des locutions clés du document. Les locutions clés générées avec la requête Rechercher similarités ne correspondent pas à celles indiquées lorsque vous recherchez ce document.

Sur la page Résultats de la recherche, une requête Rechercher similarités utilise les locutions générées par le DCS avant l'indexation. En outre, les locutions clés comportant trois mots sont raccourcies à deux mots.

La fonction Rechercher similarités recherche uniquement des documents par rapport à l'index Entreprise. Si aucun index Entreprise n'est défini pour une installation Content Server, la recherche de similarités ne renvoie aucun résultat, même si des documents similaires peuvent exister dans d'autres vues.

Configuration de la recherche Mise en évidence des résultats du résumé

La section Mise en évidence des résultats du résumé vous permet d'activer la mise en évidence des termes recherchés dans le résumé des documents sur la page Résultats de recherche. La mise en évidence des termes recherchés permet aux utilisateurs Content Server d'analyser rapidement le résumé d'un document sur la page Résultats de recherche pour voir si le document contient les informations demandées.

 

Note :
L'option Mise en évidence des résultats est disponible sur la page Résultats de recherche uniquement pour les variantes OpenText Enterprise Library.

Si une recherche étendue génère 100 termes de recherche ou plus, Content Server met en évidence uniquement les 100 premiers termes. Il s'agit d'une limitation codée en dur connue à des fins d'optimisation des performances.

Si des objets d'e-mail Lotus Notes (objets DXL) sont indexés via une solution de gestion/suivi des e-mails, il est impossible de mettre en évidence ces objets dans une page Résultats de recherche.

 

Configuration de l'en-tête des résultats de recherche

Le champ Modèle de compte des résultats de la section En-tête des résultats de recherche vous permet de configurer l'affichage du composant En-tête des résultats de recherche sur la page Résultats de recherche. Ce composant peut afficher la liste en cours des numéros du premier et dernier résultat de la recherche, le nombre total de résultats estimés et la clé ou le sens du tri.

Les modèles Estimation et Exact représentent les chaînes affichées pour différentes classes d'utilisateurs. Vous pouvez modifier la langue utilisée en saisissant une valeur Xlate au lieu de texte brut.

Le modèle Estimation sert à générer la chaîne d'en-tête des utilisateurs habituels qui obtiennent les comptes de résultats de recherche estimés, par défaut.

Le modèle Exact est destiné aux utilisateurs dotés d'autorisations Administration système et à ceux qui appartiennent au groupe doté du privilège « Afficher les Comptes de résultats non autorisés ». Ces utilisateurs peuvent voir le compte de résultats de recherche réel du moteur de recherche. Pour plus d'informations, voir « Définition des comptes de résultats de recherche » dans Gestion des processus de recherche.

Les boutons Rétablir les valeurs par défaut vous permettent de réinitialiser l'affichage du composant En-tête des résultats de recherche.

Configuration de l'affichage des résultats de recherche

La section Affichage des résultats de recherche configure les paramètres d'affichage dans les pages Résultats de recherche.

 

Note :
Ces options d'affichage ne s'appliquent pas uniformément à tous les styles de résultats de recherche.

 

Option Description
Longueur du nom Indique le nombre maximum de caractères pour les colonnes Nom et Emplacement sur les pages Résultats de recherche. La valeur par défaut est 40.
Longueur de chemin
Indique le nombre maximum de caractères pour le chemin Fil d'Ariane. La valeur par défaut est 80.
Le chemin est tronqué si sa longueur est supérieure à la valeur spécifiée. Lorsqu'un chemin de type fil d'Ariane est généré, la longueur de troncation de chaque élément de celui-ci est prise en considération. L'algorithme de troncation laisse de côté des éléments situés au milieu du chemin et avance jusqu'à ce que le chemin en fil d'Ariane soit affiché entièrement, ou jusqu'à ce que la limite de caractères soit atteinte.
Fil d'Ariane Indique si le chemin de type fil d'Ariane des emplacements de recueil s'affiche sur les pages Résultats de recherche. La valeur par défaut est Vrai.

Configuration de la recherche Paramètres de mémoire cache

La section Paramètres de mémoire cache configure la mémoire cache pour la recherche Content Server et affecte la vitesse à laquelle Content Server génère la page de recherche.

Le tableau suivant répertorie et décrit les options de la section Paramètres de mémoire cache.

Option Description
Agents actuels Affiche le nombre total d'agents actuels sur votre système.
Un agent est une vue de recherche, une requête enregistrée ou un modèle de recherche.
Agents mis en mémoire cache à conserver Vous permet d'indiquer le nombre d'agents utilisés récemment à conserver en mémoire cache. Les valeurs possibles commencent à 0, mais la valeur maximale dépend de la quantité de mémoire contenue dans votre système.
Ensembles de régions actuels Affiche le nombre total d'ensembles de régions sur votre système.
Un ensemble de régions regroupe toutes les régions associées à une source de données particulière.
Ensembles de régions de mémoire cache à conserver Vous permet d'indiquer le nombre de régions utilisées récemment à conserver en mémoire cache. Les valeurs possibles commencent à 0, mais la valeur maximale dépend de la quantité de mémoire contenue dans votre système.
Placer les agents ou les ensembles de régions dans la mémoire cache améliore les performances du système. Toutefois, si vous définissez des valeurs trop élevées, vous risquez de réduire les performances du système.

 

Note :
Placer les agents ou les ensembles de régions dans la mémoire cache améliore les performances du système. Toutefois, si vous définissez des valeurs trop élevées, vous risquez de réduire les performances du système. OpenText vous recommande de ne pas modifier ces paramètres sauf si le service d'assistance clientèle d'OpenText vous le demande.
