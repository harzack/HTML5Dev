Un flux de données dispose de deux types de composants : les processus et les IPools. Pour différentes raisons, vous pouvez avoir besoin de déplacer un processus pour l'exécuter sur un hôte de serveur d'administration différent ou déplacer un iPool à un emplacement différent.

Si plusieurs serveurs d'administration sont enregistrés dans le serveur, vous pouvez déplacer un processus de flux de données d'un hôte de serveur d'administration à un autre. Pour plus d'informations sur l'enregistrement de serveurs d'administration, reportez-vous à la rubrique Configuration des serveurs d'administration.

Si vous déplacez un processus de flux de données, vous devrez peut-être aussi modifier les répertoires de lecture et/ou d'écriture IPool associés à ce processus. Si vous modifiez l'emplacement des répertoires d'écriture et/ou de lecture du processus, vous devrez peut-être aussi corriger les répertoires de lecture et/ou d'écriture des processus environnants au sein d'un flux de données. Après avoir déplacé un processus de flux de données, assurez-vous que tous les répertoires de lecture et d'écriture sont valables. Après avoir déplacé un IPool, testez le flux de données pour vous assurer que les données traversent les IPools correctement.
