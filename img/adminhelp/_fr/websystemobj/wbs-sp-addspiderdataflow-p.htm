Pour ajouter un nouveau flux de données Spider :

  1. Sur la page Administration, dans la section Administration de recherche, cliquez sur le lien Ouvrir le volume des objets système.

  2. Dans le menu Ajouter un élément, cliquez sur Source de données Spider.

  3. Sur la page Créer une nouvelle source de données Spider, dans le champ Préfixe du processus de la section Informations communes, saisissez la chaîne de caractères que vous voulez utiliser comme identificateur pour les processus de flux de données.

  4. Dans la liste déroulante Replacement de vue de recherche, si nécessaire, cliquez sur la vue de recherche Content Server à laquelle plusieurs projets seront indexés.

  5. Dans le champ Partitions, saisissez un nombre pour spécifier le nombre de partitions dans lesquelles cette source de données sera répartie.

  6. Dans le champ Port, saisissez une valeur représentant la série des numéros des ports sur lesquels vous souhaitez que les processus associés à cette source de données Spider soient en réception. Le numéro de port spécifié et les 12 numéros de port suivants (au moins) ne doivent pas être utilisés par une autre source de données dans votre système. Le nombre de numéros de port consécutifs utilisés dépend du nombre de partitions que vous indiquez dans le champ Partitions. La création d'un flux de données Spider nécessite huit numéros de port et, pour chaque partition, quatre numéros de port supplémentaires. Les valeurs valables vont de 1025 à 65530.

  7. Dans la liste déroulante Hôte de la section Informations Spider, cliquez sur le raccourci du serveur d'administration sur l'hôte à partir duquel le processus du flux de données Spider doit s'exécuter.

  8. Dans le champ Répertoire d'écriture, saisissez le chemin absolu du répertoire (relatif au serveur d'administration sur lequel le flux de données Spider s'exécute) sur lequel le processus Spider doit écrire les données, ou utilisez le bouton Parcourir. Vous devez choisir un répertoire placé sur le lecteur de l'hôte Content Server principal, et ce répertoire ne doit pas être le répertoire d'écriture d'autres sources de données Entreprise.

  9. Dans la zone Serveurs Spider, saisissez :

    • Le nom de l'Hôte pour l'installation du serveur Spider. Si nécessaire, sous Action, cliquez sur bouton Dupliquer pour ajouter d'autres serveurs Spider.

    • Le numéro de Port du serveur Spider.

    • Le Mot de passe pour accéder au serveur Spider, si défini.

    • Les Caractéristiques du projet pour cette source de données à indexer. Cliquez sur le lien Création de la liste des projets pour afficher une liste des projets du serveur Spider, ainsi que leurs numéros de port, définis sur l'Hôte, puis cliquez sur les projets à inclure. Vous pouvez également saisir les caractéristiques directement, à l'aide du format nom du projet, nom du port, séparé par ; (point-virgule). Si nécessaire, vous pouvez également mettre les projets en surbrillance puis les supprimer.

    • Sous Action, cliquez sur le bouton Gérer le serveur Spider pour modifier les paramètres du serveur Spider, y compris les URL de site Web et les caractéristiques de navigation.

    • Si nécessaire, cliquez sur le bouton Supprimer pour supprimer les définitions du serveur Spider inutiles.

  10. Dans la section Intermédiaire : Conversion de documents, dans la liste déroulante Hôte, cliquez sur le raccourci du serveur d'administration sur l'hôte à partir duquel le processus Conversion de documents doit s'exécuter.

  11. Dans le champ Répertoire de lecture, saisissez le chemin absolu du répertoire sur lequel vous voulez que le processus Conversion de documents lise les données, ou utilisez le bouton Parcourir. Spécifiez le chemin de répertoire tel qu'il est mappé/monté sur l'hôte du serveur d'administration sur lequel est exécuté le processus de conversion de documents. Ce répertoire doit être identique au Répertoire d'écriture spécifié dans la section Informations Spider.

  12. Dans le champ Répertoire d'écriture, saisissez le chemin absolu du répertoire sur lequel vous voulez que le processus Conversion de documents écrive les données converties, ou utilisez le bouton Parcourir. Spécifiez le chemin de répertoire tel qu'il est mappé/monté sur l'hôte du serveur d'administration sur lequel est exécuté le processus de conversion de documents. OpenText vous recommande vivement de choisir un répertoire placé sur un lecteur local de l'hôte du serveur d'administration qui exécute le processus Conversion de documents. Ce répertoire doit également être différent des répertoires d'écriture d'autres sources de données Entreprise.

  13. Dans la section Client : index Content Server, dans la liste déroulante Hôte, cliquez sur le raccourci du serveur d'administration sur l'hôte à partir duquel le processus Index Content Server doit s'exécuter.

  14. Dans le champ Répertoire de lecture, saisissez le chemin absolu du répertoire sur lequel vous voulez que le processus Index Content Server lise les données, ou utilisez le bouton Parcourir. Spécifiez le chemin de répertoire tel qu'il est mappé/monté sur l'hôte du serveur d'administration à partir duquel est exécuté le processus Index Content Server. Ce répertoire doit être identique au Répertoire d'écriture principal spécifié dans la section Informations Spider.

  15. Dans le champ Répertoire d'écriture, saisissez le chemin absolu du répertoire sur lequel vous voulez que le processus Index Content Server écrive les données converties, ou utilisez le bouton Parcourir. Spécifiez le chemin de répertoire tel qu'il est mappé/monté sur l'hôte du serveur d'administration à partir duquel est exécuté le processus Index Content Server. OpenText vous recommande vivement de choisir un répertoire placé sur un lecteur local de l'hôte du serveur d'administration qui exécute le processus Index Content Server. Ce répertoire doit également être différent des répertoires d'écriture d'autres sources de données Entreprise.

  16. Dans la section Options de lancement, sélectionnez la case à cocher Lancer les processus dans le flux de données pour lancer les processus de flux de données dès leur création.

  17. Cliquez sur le bouton Créer des processus. Content Server crée les processus de flux de données et affiche une page Statut.
