Les tableaux suivants fournissent une liste complète des conditions, actions et rapports disponibles lors de l'ajout de règles de contrôle. Bien que ces options puissent être combinées pour créer des règles de contrôle personnalisées, OpenText vous recommande d'utiliser, dans la mesure du possible, les modèles de règles de contrôle fournis. L'utilisation de ces modèles vous évite de combiner des options incompatibles ou de créer des règles trop complexes qui se contredisent.

Certaines options peuvent être utilisées uniquement lors de la création de règles de contrôle de gestion des partitions. Par exemple, la condition de contrôle de l'Extracteur Content Server et l'action Définir la date du prochain déclenchement peuvent uniquement être utilisées lors de la définition d'une règle de contrôle sur la création automatique de partitions en fonction de la date pour une source de données Entreprise.

Tableau 1-7 : Types de condition

Condition Description
Espace disque du flux de données Permet de spécifier une condition de règle de contrôle basée sur l'espace disque disponible (en Mo) sur l'ordinateur qui exécute un certain flux de données.
Erreur du processus de flux de données Permet de spécifier une condition de règle de contrôle basée sur la présence d'une ou plusieurs erreurs dans le processus de flux de données.
Contrôle de l'extracteur Content Server Permet de spécifier une condition de règle de contrôle basée sur la date de déclenchement associée aux processus Extracteur Content Server. Cette condition doit être utilisée uniquement lors de la mise en œuvre d'une règle de création automatique de partitions basée sur la date dans le cadre d'une solution d'indexation de gros volumes. Pour en savoir plus, voir Ajout de règles de contrôle de la gestion des partitions.
Nombre de messages Permet de spécifier une condition de règle de contrôle basée sur le nombre de messages IPool dans le flux de données.
Espace de la partition Permet de spécifier une condition de règle de contrôle basée sur l'espace utilisé dans une partition. Si un même emplacement compte plusieurs partitions, les conditions s'appliquent à l'ensemble des partitions. Par exemple, si un emplacement comporte cinq partitions et que vous avez créé une condition d'espace de la partition qui indique une capacité de 75 %, toutes les partitions doivent atteindre 75 % avant que la condition ne soit remplie.
Flux de données discrets Permet de spécifier une condition de règle de contrôle basée sur la durée d'inactivité d'un flux de données sans messages IPool en attente à traiter.
Flux de données bloqués Permet de spécifier une condition de règle de contrôle basée sur la durée d'inactivité d'un flux de données avec des messages IPool en attente à traiter.

Tableau 1-8 : Types d'actions

Action Description
Créer une partition Permet de définir les paramètres associés à la création automatique de partitions (par exemple, descendre les partitions et nommer la partition et son répertoire). Cette action doit être utilisée uniquement lors de la mise en œuvre d'une règle de création automatique de partitions dans le cadre d'une solution d'indexation de gros volumes. Pour en savoir plus, voir Ajout de règles de contrôle de la gestion des partitions.
 

Note :
Si vous n'utilisez pas de règles de contrôle de gestion des partitions, vous devez arrêtez le flux de données avant de créer les partitions, puis redémarrer le flux de données et les fédérateurs de recherche une fois les partitions créées.

 
Redémarrer les fédérateurs de recherche Permet de définir une règle de contrôle qui redémarre tous les fédérateurs de recherche d'un site Content Server lorsque la condition spécifiée est remplie. Pour plus d'informations sur les fédérateurs de recherche, reportez-vous à la rubrique Gestion des processus de recherche.
Définir la date du prochain déclenchement Permet de spécifier l'intervalle de la définition de la date de déclenchement suivante pour la condition de contrôle de l'Extracteur Content Server. Cette action doit être utilisée en parallèle avec la condition de contrôle de l'Extracteur Content Server lors de la mise en œuvre d'une règle de création automatique de partitions dans le cadre d'une solution d'indexation de gros volumes. Pour en savoir plus, voir Ajout de règles de contrôle de la gestion des partitions.
Lancer le flux de données Permet de démarrer un flux de données lorsque la condition correspondante est remplie.
Lancer l'alimentation iPool Permet de démarrer le processus de flux de données qui envoie des messages à l'IPool spécifié lorsque la condition correspondante est remplie.
Lancer le lecteur iPool Permet de démarrer le processus de flux de données qui lit les messages de l'IPool spécifié lorsque la condition correspondante est remplie.
Arrêter le flux de données Permet d'arrêter un flux de données lorsque la condition correspondante est remplie.
Arrêter l'alimentation iPool Permet d'arrêter le processus de flux de données qui envoie des messages à l'IPool spécifié lorsque la condition correspondante est remplie.
Arrêter le lecteur iPool Permet d'arrêter le processus de flux de données qui lit les messages de l'IPool spécifié lorsque la condition correspondante est remplie.

Tableau 1-9 : Types de rapports

Rapport Description
Standard Définit la disposition et le contenu de l'e-mail envoyé aux destinataires des e-mails d'alerte du volume des objets système. Si vous configurez une règle de contrôle avec un rapport standard, vous pouvez envoyer un message d'informations, d'avertissement, d'erreur ou d'erreur grave, selon les conditions et les actions définies dans la règle. Pour plus d'informations sur la configuration des destinataires des e-mails d'alerte du volume des objets système et sur les types de messages que vous pouvez envoyez, reportez-vous à la rubrique Ajout de destinataires d'e-mails. Si vous souhaitez personnaliser l'e-mail envoyé avec un rapport standard, vous pouvez ajouter un texte personnalisé contenant jusqu'à 250 caractères.
