En temps normal, Content Server ne requiert qu'un seul processus Extracteur dans une source de données Entreprise pour extraire les données de la base de données Content Server ; il s'agit de la configuration par défaut de Content Server. Cependant, dans certains cas, lorsque le volume d'objets en cours d'indexation est trop important, les sources et les flux de données doivent être personnalisés pour permettre l'indexation de gros volumes. Ainsi, si une entreprise de grande envergure souhaite stocker et indexer l'ensemble de ses e-mails dans Content Server, la quantité de données peut nécessiter une indexation de gros volumes adaptée aux besoins du site.

L'indexation de gros volumes consiste à configurer au moins l'un des éléments suivants :

  • des processus Extracteur Entreprise multiples dans une source de données Entreprise ;

  • des processus de conversion de documents multiples dans une source de données Entreprise ;

  • des sources de données Entreprise multiples.

Pour plus d'informations sur l'ajout des processus Extracteur Entreprise et Conversion de documents, reportez-vous à la rubrique Ajout de processus de flux de données.

Les sources de données Entreprise multiples ne sont disponibles que si vous avez défini dans opentext.ini le paramètre wantMultipleEnterprise de la section [SearchOptions] sur true. Pour en savoir plus, voir La section [SearchOptions].

Considérations lors de la configuration de l'indexation de gros volumes

La combinaison des processus et des sources de données requise pour une installation dépend des données à indexer. OpenText vous recommande de contacter les services généraux ou le service d'assistance clientèle pour une analyse de votre système avant de mettre en œuvre l'indexation de gros volumes. OpenText vous recommande également de planifier l'utilisation des ports et des répertoires partagés pour les sous-zones IPool avant la mise en œuvre. La fonction de flux de données de test est utile à cet égard, notamment pour déterminer si les sous-zones IPool ont été correctement spécifiées. Pour plus d'informations sur la fonction de flux de données de test, reportez-vous à la rubrique Maintenance des flux de données. Pour plus d'informations sur l'affichage des numéros de port sur votre système, reportez-vous à la rubrique Maintenance des serveurs d'administration.

Réextraction des données Entreprise

Lorsque vous ajoutez une source de données Entreprise à Content Server, vous pouvez extraire à nouveau les données Entreprise de la base de données lors de sa création. Si votre base de données n'est pas très volumineuse, la commande de réextraction peut être utile car elle redistribue les données Entreprise dans les sources de données Entreprise, en équilibrant plus efficacement la charge. Cependant, si votre base de données est très volumineuse, la réextraction n'est pas forcément utile car elle risque d'être extrêmement longue. Pour plus d'informations, contactez le service d'assistance clientèle d'OpenText.
