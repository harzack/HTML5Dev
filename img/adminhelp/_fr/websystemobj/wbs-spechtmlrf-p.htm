Pour configurer un processus Conversion de documents, vous devez modifier les paramètres ou les arguments de ligne de commande dans l'onglet Infos spécifiques de sa page Propriétés. Pour plus d'informations sur les processus de conversion de documents, reportez-vous à la rubrique Configuration des processus Conversion de documents.

Modification des arguments de ligne de commande

Les valeurs des arguments de ligne de commande spécifiées pour un processus Conversion de documents spécifique sont stockées dans le fichier otadmin.cfg. Les valeurs par défaut et les valeurs valables pour chaque argument sont décrites dans la section [FilterEngine] du fichier opentext.ini. Vous modifiez la section [FilterEngine] du fichier opentext.ini lorsque vous voulez configurer de manière globale tous les processus Conversion de documents sur votre site Content Server.

 

Note :
Si un argument apparaît dans le fichier opentext.ini et la ligne de commande d'un processus Conversion de documents particulier, la valeur de la ligne de commande remplace la valeur du fichier opentext.ini pour le processus.

 

Voici les arguments de ligne de commande du processus Conversion de documents qui peuvent être modifiés :

  • defaulttype

  • html

  • logfile

  • template

  • récapitulatif

  • summarysentences

L'onglet Infos spécifiques de la page Propriétés du processus de conversion de documents affiche les informations suivantes :

Paramétrage Valeurs
Nom Spécifiez un nom pour le processus Conversion de documents.
Description Spécifiez une description pour le processus Conversion de documents. Ces informations s'affichent dans l'onglet Général lorsque vous consultez les propriétés de la conversion de documents.
Hôte Spécifiez le serveur d'administration sur l'hôte à partir duquel vous souhaitez exécuter le processus Conversion de documents.
Section Lire les données à partir de :  
Zone principale IPool Indiquez le chemin absolu du répertoire de l'IPool à partir duquel le processus Conversion de documents devra lire les données.
Processus Choisissez le processus devant précéder le processus Conversion de documents dans le flux de données.
 

Note :
Si le flux de données contient déjà des processus producteurs ou intermédiaires, leurs noms s'affichent dans la liste déroulante Processus.

 
Section Ecrire les données dans :  
Ajouter un lien d'écriture iPool Vous pouvez également cliquer sur le bouton Ajouter un lien d'écriture iPool pour configurer un second processus après le processus Conversion de documents.
Zone principale iPool Indiquez le chemin absolu du répertoire de l'IPool dans lequel le processus Conversion de documents devra diviser les données.
Processus Choisissez le processus devant suivre le processus Conversion de documents dans le flux de données.
 

Note :
Si le flux de données contient déjà au moins un processus intermédiaire ou consommateur, les noms des processus apparaissent dans la liste déroulante Processus.

 
Section Programme :  
Manuel Sélectionnez Manuel si vous souhaitez que la conversion de documents traite les données lorsqu'elle est lancée par vous, ou Programmé si vous souhaitez exécuter le processus Conversion de documents automatiquement.
A cette heure Définissez l'heure dans les listes déroulantes et cliquez sur les jours de la semaine où vous souhaitez exécuter le processus Conversion de documents.
Intervalle Sélectionnez l'intervalle pendant lequel vous souhaitez exécuter le processus Conversion de documents.
Attente Définissez les valeurs dans les listes déroulantes pour sélectionner l'intervalle où vous souhaitez exécuter le processus Conversion de documents.
Continuel Continuel permet l'exécution continue du processus de Conversion de documents. En cas d'arrêt inopiné du processus, son serveur d'administration le redémarre automatiquement.
Intervalle de veille Définissez les valeurs dans les listes déroulantes pour sélectionner l'intervalle où vous souhaitez mettre en veille le processus Conversion de documents avant de le redémarrer.
Port de message d'arrêt Spécifiez le numéro de port auquel Content Server envoie un message d'arrêt, puis arrête le processus Conversion de documents. Ne modifiez pas ce paramètre, sauf si le service d'assistance clientèle d'OpenText vous le demande.
Valeur maximale du code de sortie correcte Indiquez le numéro de code d'erreur le plus élevé pour lequel le serveur d'administration redémarre automatiquement le processus Conversion de documents en cas d'échec.
 

Astuce :
Si vous saisissez -1 dans le champ Valeur maximale du code de sortie correcte, le processus Conversion de documents s'interrompra aussitôt qu'il s'arrêtera. Le service d'administration ne redémarrera pas en cas d'échec du processus Conversion de documents.

 
Ne modifiez pas ce paramètre, sauf si le service d'assistance clientèle d'OpenText vous le demande.

 

Elément Description
Etat Cet élément précise si le processus Conversion de documents est en cours d'exécution ou non.
Activer la gestion du système Détermine si Content Server contrôle ce processus pour détecter le retour de messages d'erreur. S'il retourne un message d'erreur, Content Server l'enregistre dans la base de données. La gestion du système est activée par défaut. Pour plus d'informations sur la configuration de Content Server en vue de l'envoi d'e-mails d'alerte lorsqu'un processus de flux de données rencontre des erreurs, reportez-vous à la rubrique Pour activer la recherche d'erreurs et l'envoi d'e-mails.
Serveur d'administration Cet élément spécifie le raccourci du serveur d'administration sur l'hôte duquel le processus Conversion de documents s'exécute. Ne modifiez pas le raccourci du serveur d'administration destiné au processus Conversion de documents sauf si cela s'avère nécessaire (par exemple, si l'hôte du serveur d'administration n'est plus disponible).
Modèle de commande Cet élément spécifie le modèle utilisé pour générer la ligne de commande du processus Conversion de documents. Ne modifiez pas cette valeur, sauf si le service d'assistance clientèle d'OpenText vous le demande.
Ligne de commande Cet élément spécifie la ligne de commande utilisée par Content Server pour exécuter le processus Conversion de documents. Vous ne pouvez pas modifier la valeur de ce champ.
Répertoire de démarrage Cet élément spécifie le répertoire dans lequel le processus Conversion de documents démarre. Le processus Conversion de documents doit s'exécuter dans le répertoire Content Server_accueil/filtres. Ne modifiez pas cette valeur, sauf si le service d'assistance clientèle d'OpenText vous le demande.
Répertoire temporaire Cet élément spécifie le répertoire dans lequel le processus Conversion de documents enregistre les fichiers temporaires qui lui sont nécessaires. Par défaut, le répertoire temporaire est celui dans lequel votre système d'exploitation enregistre les fichiers temporaires. Vous pouvez spécifier un répertoire particulier dans lequel ce processus de conversion de documents enregistrera ses fichiers temporaires. Pour cela, saisissez -tmpdir dans le champ Autre. Tapez également le chemin absolu du répertoire dans le champ Valeur, puis cliquez sur le bouton Mettre à jour. Sur les systèmes d'exploitation UNIX, OpenText recommande l'utilisation du répertoire /tmp.
Ligne de commande supplémentaire Cet élément spécifie les arguments facultatifs que vous pouvez ajouter à la ligne de commande du processus Conversion de documents. Vous devez saisir les noms et les valeurs des arguments à ajouter ou à modifier en séparant plusieurs paires argument/valeur par des espaces :-argument value -argument value.
Informations Ce bouton ouvre la page Informations IPool, qui spécifie le répertoire à partir duquel le processus Conversion de documents lit les données et le répertoire dans lequel il écrit les données. Ne modifiez pas ces valeurs, sauf si vous déplacez un IPool ou si le service d'assistance clientèle d'OpenText vous le demande.
Options de démarrage Cet élément spécifie les paramètres de lancement du processus Conversion de documents. Il existe trois paramètres de lancement :
•

Manuel, qui permet de lancer le processus Conversion de documents manuellement dès lors que vous souhaitez l'exécuter ;

•

Programmé, qui permet de programmer le jour et l'heure d'exécution du processus Conversion de documents ;

•

Continuel, qui permet d'exécuter le processus Conversion de documents en continu.

Options d'arrêt Cette section spécifie les paramètres d'arrêt du processus Conversion de documents. Par défaut, Content Server envoie un message d'arrêt au port spécifié dans le champ Port Admin, puis arrête complètement le processus. Le champ Valeur maximale du code de sortie correcte spécifie le numéro de code d'erreur le plus élevé pour lequel le serveur d'administration redémarre automatiquement le processus de conversion de documents en cas d'échec. La valeur du champ Valeur maximale du code de sortie correcte est définie par défaut sur -1 ; ainsi, le serveur d'administration ne redémarre pas le processus de conversion de documents en cas d'échec. Ne modifiez pas ces options d'arrêt, le service d'assistance clientèle d'OpenText vous le demande.
Démarrer/Arrêter Cet élément vous permet de gérer des flux de données, selon son état courant. Si le processus est en cours d'exécution ou programmé pour être exécuté, le bouton Arrêter s'affiche. Si le processus est arrêté, le bouton Démarrer s'affiche.
Resynchroniser Cet élément resynchronise les informations relatives à ce processus (dans le fichier otadmin.cfg du serveur d'administration du processus) avec les informations de la base de données Content Server.
Mettre à jour Cet élément envoie au serveur Content Server les modifications que vous avez apportées à la page.
Réinitialiser Rétablit les informations de cette page à l'état qui était le leur à l'ouverture de la page.
