Lors d'un processus de fusion, les données de plusieurs processus de flux de données sont fusionnées, puis transférées au processus suivant dans le flux de données.

La configuration des processus de fusion s'effectue sur les pages Propriétés. Par exemple, vous pouvez modifier le serveur d'administration dont l'hôte sera utilisé pour exécuter le processus de fusion. Vous pouvez également activer la gestion des objets système, modifier les répertoires de démarrage et de destination, gérer les options de démarrage et d'arrêt ou resynchroniser le processus.

Onglet Infos spécifiques de la page Propriétés de Fusion

Paramétrage Valeurs
Nom Indiquez un nom pour la fusion.
Description Indiquez une description pour la fusion. Ces informations s'affichent dans l'onglet Général lorsque vous consultez les propriétés de la fusion.
Hôte Indiquez le serveur d'administration sur l'hôte à partir duquel le processus Fusion doit s'exécuter.
Activer la gestion du système Sélectionnez cette option pour permettre à Content Server de surveiller le statut du processus Fusion. Lorsque cette option est activée, si le processus retourne des messages d'erreur, Content Server les enregistre dans la base de données.
 

Astuce :
Pour plus d'informations sur la configuration de Content Server en vue de l'envoi d'e-mails d'alerte lorsqu'un processus de flux de données rencontre des erreurs, reportez-vous à la rubrique Pour activer la recherche d'erreurs et l'envoi d'e-mails.

 
Section Lire les données à partir de :  
Zone principale iPool Indiquez le chemin absolu du répertoire de l'IPool à partir duquel le processus Fusion devra lire les données.
Processus Choisissez le processus qui doit précéder le processus Fusion dans le flux de données.
 

Note :
Si le flux de données contient déjà des processus producteurs ou intermédiaires, leurs noms s'affichent dans la liste déroulante Processus.

 
Section Ecrire les données dans :  
Ajouter un lien d'écriture iPool Vous pouvez également cliquer sur le bouton Ajouter un lien d'écriture iPool pour configurer un second processus après le processus Fusion.
Zone principale iPool Indiquez le chemin absolu du répertoire de l'IPool dans lequel le processus Fusion devra diviser les données.
Processus Choisissez le processus qui doit suivre le processus Fusion dans le flux de données.
 

Note :
Si le flux de données contient déjà au moins un processus intermédiaire ou consommateur, les noms des processus apparaissent dans la liste déroulante Processus.

 
Section Programme :  
Manuel Sélectionnez Manuel si vous souhaitez que la fusion traite les données lorsque vous la lancez vous-même, ou Programmé si vous souhaitez exécuter le processus Fusion automatiquement.
A cette heure Définissez l'heure dans les listes déroulantes, et cliquez sur les jours de la semaine où vous souhaitez exécuter le processus Fusion.
Intervalle Sélectionnez l'intervalle pendant lequel vous souhaitez exécuter le processus Fusion.
Attente Définissez les valeurs dans les listes déroulantes pour sélectionner l'intervalle où vous souhaitez exécuter le processus Fusion.
Continuel Sélectionnez Continuel pour que le processus Fusion s'exécute en continu. En cas d'arrêt inopiné du processus, son serveur d'administration le redémarre automatiquement.
Valeur maximale du code de sortie correcte Indiquez le numéro de code d'erreur le plus élevé pour lequel le serveur d'administration redémarre automatiquement le processus Fusion en cas d'échec.
 

Astuce :
Si vous saisissez -1 dans le champ Valeur maximale du code de sortie correcte, le processus Fusion s'interrompra aussitôt qu'il s'arrêtera. Le service d'administration ne redémarrera pas en cas d'échec du processus Fusion.

 
Ne modifiez pas ce paramètre, sauf si le service d'assistance clientèle d'OpenText vous le demande.
