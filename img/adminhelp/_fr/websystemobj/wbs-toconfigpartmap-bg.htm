OpenText vous recommande de ne pas modifier les valeurs par défaut de l'onglet Spécifique de la page Propriétés d'un schéma de partition sans les conseils du service d'assistance clientèle d'OpenText.

Elément Description
Paramètres
Délai d'attente du moteur de recherche Indique la durée pendant laquelle :
•

les fédérateurs de recherche attendent une réponse des moteurs de recherche qu'ils gèrent.

•

les moteurs de recherche évaluent une requête, avant que la requête associée ne soit arrêtée.

Les fédérateurs de recherche attendent jusqu'à 1,25 fois (*) la valeur Délai d'attente du moteur de recherche. Les moteurs de recherche attendent pendant la valeur du délai d'attente.
 

Note :
Lorsque vous établissez une connexion avec le fédérateur de recherche, la valeur de seuil du délai d'attente par défaut pour émettre une requête est très courte et la connexion peut donc facilement être perdue. Vous pouvez augmenter le temps disponible pour émettre une requête en appuyant sur la touche Entrée tout de suite après avoir ouvert le client de recherche. Cela vous laisse 3 minutes pour saisir votre requête avant que le Fédérateur de recherche ne se ferme.

 
Mise à jour de l'intervalle de requête Indique la fréquence à laquelle les moteurs de recherche vérifient si l'index d'une partition a changé.
Taille de la troncature du contenu Indique la taille maximale du contenu converti des documents avant qu'il ne soit tronqué. Seul le contenu tronqué peut faire l'objet de recherches.
Nettoyage du fichier Indique les options qu'un moteur d'indexation utilise pour supprimer les fichiers et les répertoires de l'index d'une partition :
•

Tous : tous les fichiers qui ne sont pas utilisés sont supprimés.

•

Partiel : tous les fichiers qui ne sont pas utilisés sont supprimés, sauf les fichiers metaLog et accumLog. Un fichier metaLog est un fichier qui stocke les modifications apportées aux métadonnées depuis le dernier enregistrement des métadonnées dans un fichier de point de vérification. Un fichier de point de vérification est un fichier qui stocke en mémoire les métadonnées sur le disque. Un fichier accumLog est la représentation de l'accumulateur de contenu sur le disque. Les fichiers metaLog, de point de vérification et accumLog sont stockés dans le répertoire d'indexation associé à l'index d'une source de données spécifique.

•

Aucun : aucun fichier n'est supprimé.

Allocation de mémoire au filtre de recherche Indique l'espace de mémoire supplémentaire alloué pour les facettes de chaque processus du moteur de recherche. Si vous modifiez cette valeur, les processus du moteur de recherche doivent être resynchronisés.
Intervalle de nettoyage du fichier Indique la fréquence à laquelle les moteurs d'indexation vérifient l'index d'une partition afin de savoir si des fichiers ou des répertoires peuvent être supprimés.
Méthode de stockage sur le disque
Indique quelle méthode de stockage est activée dans le moteur de recherche :
•

Mode Mémoire faible : les valeurs de métadonnées de texte sont stockées dans le fichier de point de vérification, le dictionnaire et l'index de métadonnées de texte sont codés dans des fichiers stockés sur le disque. Il s'agit du mode par défaut.

•

Mode Stockage de valeur : le stockage de valeur se trouve sur le disque dans le fichier de point de vérification, l'index et le dictionnaire sont gardés en mémoire

La modification de ce paramètre entraîne la reconstruction des index pour les régions de métadonnées utilisant le stockage en Mode Mémoire faible.
Limites de l'index metalog
Mode Mémoire faible Indique les paramètres pour l'écriture d'un journal de métadonnées en Mode Mémoire faible.
  Taille Indique la taille de l'index, en Mo. La valeur par défaut est 200.
  Objets Indique le nombre d'objets dans l'index. La valeur par défaut est 50000.
  Mises à jour Indique le nombre de mises à jour dans l'index. La valeur par défaut est 5000.
Autres modes de stockage Indique les paramètres pour l'écriture d'un journal de métadonnées pour d'autres modes de stockage.
  Taille Indique la taille de l'index, en Mo. La valeur par défaut est 16.
  Objets Indique le nombre d'objets dans l'index. La valeur par défaut est 5000.
  Mises à jour Indique le nombre de mises à jour dans l'index. La valeur par défaut est 500.
Options de fusion
Fusions Indique si des fusions sont effectuées :
•

Vrai : des fusions d'index sont effectuées.

•

Faux : les fusions d'index ne sont pas effectuées.

OpenText vous recommande de ne pas modifier la valeur par défaut de ce paramètre (à savoir Vrai).
Intervalle de tentative de fusion Indique la fréquence à laquelle les moteurs d'indexation vérifient un index de partition pour déterminer si des fusions d'index sont nécessaires.
Numéro d'index cible Indique le nombre de sous-index à partir duquel une opération de fusion a lieu. Si le nombre de sous-index dépasse la valeur définie pour le paramètre Numéro d'index cible, les sous-index sont fusionnés.
Date la plus ancienne de l'index Indique le nombre de jours pouvant s'écouler avant qu'un sous-index ne soit compressé pour diminuer sa taille. Si la date de création d'un sous-index dépasse le nombre de jours défini pour le paramètre Date la plus ancienne de l'index, le sous-index est compressé.
Taux d'indexation Indique la proportion permettant de déterminer si les sous-index sont susceptibles d'être fusionnés. La valeur par défaut du paramètre Taux d'indexation est 3, ce qui indique que le taux de fusion est de 3:1. Pour plus d'informations sur les opérations de fusion, voir Utilisation des schémas de partition.
Mauvaise heuristique d'objet
Taille minimale de document à prendre en considération Indique le nombre minimal de mots qu'un document peut contenir avant que le document ne puisse pas être indexé. Configurez ce paramètre avec les paramètres Proportion de mots du document, Taux de mots du document restrictif, Longueur moyenne du mot maximale et Longueur moyenne du mot maximale restrictive afin de ne pas indexer les documents. Un document n'est pas indexé s'il dépasse la valeur définie pour le paramètre Taille minimale de document à prendre en considération et si l'une des affirmations suivantes est vraie :
•

il dépasse les valeurs définies pour les paramètres Proportion de mots du document et Longueur moyenne du mot maximale ;

•

il dépasse la valeur définie pour le paramètre Taux de mots du document restrictif ;

•

il dépasse la valeur définie pour le paramètre Longueur moyenne du mot maximale restrictive.

Proportion de mots du document Indique la proportion maximale de mots uniques sur le nombre de mots total dans le contenu d'un document.
Taux de mots du document restrictif Indique une proportion plus restrictive pour les critères définis par le paramètre Proportion de mots du document.
Longueur moyenne du mot maximale Indique la longueur moyenne maximale des mots d'un document. Par exemple, si un document répond aux critères Taille minimale de document à prendre en considération et Proportion de mots du document, si la valeur du paramètre Longueur moyenne du mot maximale est définie à 10 (valeur par défaut) et si la longueur moyenne du mot dans un document est de 12 caractères, le document n'est pas indexé.
Longueur moyenne du mot maximale restrictive Indique une proportion plus restrictive pour les critères définis par le paramètre Longueur moyenne du mot maximale.
Actions
Option Permet de resynchroniser les informations sur le schéma de partition avec les informations de la base de données Content Server.

Réglage des paramètres de mémoire des métadonnées

Dans certains cas, il peut être utile de stocker certaines régions de métadonnées d'un schéma de partition dans la mémoire vive et d'autres, sur disque. La gestion de cette mémoire des métadonnées vous permet d'exploiter au mieux vos ressources mémoire et disque. Les paramètres mémoire par défaut des quatre modes de partition sont définis comme suit :

  • Lecture/Ecriture : tout sur disque, l'emplacement par défaut est le disque ;

  • Mettre à jour uniquement : tout sur disque, l'emplacement par défaut est le disque ;

  • Lecture seule : tout sur disque, l'emplacement par défaut est le disque ;

  • Hors service : tout sur disque, l'emplacement par défaut est le disque ;

 

Avertissement
Lorsque vous modifiez une partition sur En lecture seule, le moteur d'indexation pour cette partition s'arrête et cesse d'indexer tout élément mis à jour ou supprimé.

 
 

Note :
Open Text recommande généralement de ne pas utiliser les modes En lecture seule et Mettre à jour uniquement.
