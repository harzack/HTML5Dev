Vous pouvez définir une région de filtre et une valeur pour contrôler les informations envoyées aux processus de destination d'un processus T. Si vous laissez ces champs vides, toutes les données seront exportées vers le processus de destination.

Paramétrage Valeur
Région de filtre Indiquez le nom de n'importe quelle région. Par exemple, vous pouvez spécifier la région de métadonnées OTDCategory. Seuls les objets auxquels cette région est appliquée seront filtrés vers le processus de destination.
 

Astuce :
Pour plus d'informations sur certaines régions de métadonnées dans Content Server, reportez-vous à la rubrique Définition des régions d'index.

 
Valeur de filtre Indiquez une valeur autorisée pour la région spécifiée dans le paramètre Région de filtre. Par exemple, si vous spécifiez la région de métadonnées OTDCategory dans le paramètre Région de filtre, et si vous indiquez l'ID de nœud d'une catégorie dans le champ Valeur de filtre, tous les objets de la région OTDCategory et de la catégorie spécifiée sont filtrés vers le processus de destination.
 

Astuce :
L'ID de nœud de la plupart des éléments Content Server est identifié dans la section suivante de son URL : func=ll&objid=<valeur> où <valeur> correspond à l'ID de nœud.

 

Vous ne pouvez indiquer qu'une seule valeur Région de filtre et une Valeur de filtre pour chaque entrée Ecrire les données dans.
