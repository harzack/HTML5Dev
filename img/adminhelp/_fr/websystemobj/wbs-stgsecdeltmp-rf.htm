Lors de l'indexation, le service de conversion de documents (DCS) et les IPools dans les flux de données génèrent des fichiers temporaires qui peuvent contenir du texte ou tout autre contenu extrait des fichiers en cours d'indexation. Lorsqu'un fichier temporaire n'est plus utile, le flux de données utilise les appels système normaux pour le supprimer. Dans la plupart des cas, le système supprime l'entrée du fichier dans le tableau des fichiers et permet à l'image du fichier sur le disque d'être remplacée par un nouveau fichier. Cependant, certains logiciels peuvent retrouver le fichier supprimé à partir de l'image, ce qui peut représenter un risque au niveau de la sécurité.

Lorsque vous configurez la suppression sécurisée des fichiers temporaires, vous spécifiez un niveau de suppression sécurisée, qui détermine la façon dont le système remplace les images de fichiers sur le disque. Les valeurs possibles sont des nombres entiers allant de 0 à 4, représentant plusieurs niveaux de suppression sécurisée. Le tableau suivant détaille les différences entre ces valeurs.

Valeur Description
0 Exécute une suppression non sécurisée.
1 Remplace l'image du fichier en un seul passage par un nombre d'octets nul. Cette option est recommandée pour des systèmes de fichiers autres que des fichiers journaux (par exemple, Solaris 9).
2, 3 ou 4 Remplace l'image du fichier par une configuration d'octets plus complexe (dans des configurations binaires non nulles), avec plusieurs passages, rendant la récupération du fichier de plus en plus difficile. La valeur du niveau représente le nombre de passages effectués pour remplacer les données ; la valeur 4 offre le niveau de sécurité le plus élevé. Ces options sont recommandées pour les systèmes de fichiers journaux (par exemple, Windows et Solaris 10).

 

Note :
L'encodage RLE de données sur les lecteurs nécessite l'utilisation de configurations binaires spécifiques pour la suppression totale des données. L'utilisation des niveaux de suppression sécurisée 2, 3 ou 4 ne garantit pas la sécurité des systèmes de fichiers journaux, mais plus le niveau de suppression sécurisée est élevé et plus la probabilité de récupération des données est moindre.

Etant donné que la valeur de chaque niveau de suppression sécurisée représente le nombre de passages effectués lors du remplacement des fichiers temporaires, l'impact sur les performances augmente avec le niveau de suppression sécurisée.

 

Vous devez spécifier le niveau de suppression sécurisée dans plusieurs emplacements. Le premier est le paramètre Securedelete dans la section [DCS] du fichier opentext.ini, qui spécifie le niveau de suppression sécurisée pour le DCS. Pour plus d'informations sur Securedelete, reportez-vous à la rubrique [DCS].

Vous spécifiez le niveau de suppression sécurisée pour les IPools dans les fichiers ipool.cfg des flux de données. Il existe un fichier pour chaque répertoire de flux de données et pour chaque source de données (par exemple, Content Server_home\index\enterprise\data_flow pour la source de données Entreprise). Vous devez spécifier le niveau en ajoutant la ligne suivante avant la ligne </InterchangePool> à la fin du fichier :

<SecureDeleteLevel>x<SecureDeleteLevel>

x étant la valeur du niveau de suppression sécurisée.

 

Note :
Si les répertoires de lecture et d'écriture pour une source de données se trouvent sur des ordinateurs distincts, vous devez ajouter ce paramètre aux fichiers ipool.cfg sur les deux ordinateurs.

 

OpenText vous recommande de spécifier le même niveau de suppression sécurisée pour le DCS et les IPools. Les fichiers temporaires de ces deux composants contiennent souvent les mêmes données. Ainsi, accroître le niveau de suppression sécurisée pour seulement l'un de ces composants n'offre qu'une faible amélioration de la sécurité.
