Le tableau suivant décrit les messages d'erreur relatifs au processus Conversion de documents.

Si ces erreurs persistent, contactez le service d'assistance clientèle d'OpenText pour obtenir plus d'informations.

Tableau 1-26 : Messages d'erreur relatifs au processus Conversion de documents

Numéro Message Description Solution
10 Une erreur interne s'est produite Le processus Conversion de documents a effectué une opération illégale (par exemple, adressage d'un espace mémoire inexistant ou appels système illégaux). Contactez l'assistance clientèle d'OpenText.
11 Impossible de reconnaître le type MIME. Le filtre de conversion de documents n'est pas parvenu à identifier le type MIME du fichier. Activez la consignation en définissant le paramètre suivant dans la section [FilterEngine] du fichier opentext.ini :
logfile= <x>où <x> correspond au nom de chemin et de fichier du fichier journal. Examinez le fichier journal pour obtenir davantage d'informations, puis contactez le service d'assistance clientèle d'OpenText.
12 Impossible de trouver un fichier d'entrée. Le processus Conversion de documents prévoit de lire un fichier d'entrée à partir de l'iPool, mais il n'en existe aucun. Contactez l'assistance clientèle d'OpenText.
20 Impossible de localiser suffisamment de ressources système. Le processus Conversion de documents a échoué car il n'a pas été en mesure de trouver les ressources système suffisantes à son exécution. Libérez certaines ressources système (mémoire et espace disque), puis exécutez à nouveau le processus.
21 Paramètres non valables dans la section [FilterEngine] du fichier opentext.ini. Le processus Conversion de documents n'a pas démarré car les paramètres spécifiés dans la section [FilterEngine] du fichier opentext.ini ne sont pas valables. Examinez les paramètres actuellement spécifiés dans la section [FilterEngine] du fichier opentext.ini. Ensuite, tentez d'exécuter à nouveau le processus Conversion de documents. Le fichier opentext.ini se trouve dans le répertoire config de votre installation Content Server. OpenText vous recommande de ne pas modifier le fichier opentext.ini sauf si le service d'assistance clientèle d'OpenText vous le demande.
22 Arguments de ligne de commande spécifiés non valables Le processus Conversion de documents n'a pas démarré car certains arguments de ligne de commande spécifiés dans le fichier otadmin.cfg ne sont pas valables. Examinez les arguments de la ligne de commande actuellement spécifiés dans le fichier otadmin.cfg, puis tentez d'exécuter à nouveau le processus Conversion de documents. Le fichier otadmin.cfg se trouve dans le répertoire config de votre installation Content Server. OpenText vous recommande de ne pas modifier le fichier otadmin.cfg sauf si le service d'assistance clientèle d'OpenText vous le demande.
23 Nom du fichier d'entrée spécifié non valable Le processus Conversion de documents a tenté de lire un fichier particulier inexistant dans l'iPool. Tentez d'exécuter à nouveau le processus Conversion de documents. Si le problème persiste, consultez le service d'assistance clientèle d'OpenText.
24 Paramètres de veille non valables dans la section [FilterEngine] du fichier opentext.ini. La valeur du paramètre Sleep de la section [FilterEngine] du fichier opentext.ini n'est pas valable. Assurez-vous que la valeur du paramètre Sleep est un entier positif, puis tentez d'exécuter à nouveau le processus Conversion de documents.
30 Une erreur interne IPool s'est produite. Une erreur iPool s'est produite dans le processus Conversion de documents lors de la configuration des iPools. Activez la journalisation du processus Conversion de documents en définissant le paramètre suivant dans la section [FilterEngine] du fichier opentext.ini :
logfile= <x>, où <x> correspond au nom absolu de chemin et de fichier du fichier journal. Ensuite, arrêtez le flux de données et activez la journalisation iPool en modifiant la valeur suivante dans le fichier ipool.cfg.
<LogLevel>3</LogLevel>
Le fichier ipool.cfg et le fichier journal ipool lui-même (iplog.fle) sont enregistrés dans le répertoire de flux de données. Le fichier opentext.ini est stocké dans le répertoire config de votre installation Content Server.
Après avoir journalisé l'activité du processus de conversion de documents, contactez le service d'assistance clientèle d'OpenText.

Activez la journalisation du processus Conversion de documents en définissant le paramètre suivant dans la section [FilterEngine] du fichier opentext.ini :

Après avoir journalisé l'activité du processus de conversion de documents, contactez le service d'assistance clientèle d'OpenText.
