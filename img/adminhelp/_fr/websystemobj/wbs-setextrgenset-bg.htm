Un processus Extracteur Entreprise est un processus de flux de données qui extrait des objets et des métadonnées de Content Server et les écrit dans un flux de données, pour mettre à jour l'index Content Server.

Vous configurez la plupart des paramètres de l'Extracteur Entreprise sur la page Définir les paramètres généraux de l'Extracteur qui contrôle la manière dont les objets sont ajoutés à l'index de recherche. D'autres paramètres peuvent affecter le comportement de l'Extracteur, comme les exclusions de type MIME qui sont définies dans le fichier opentext.ini.

La page Définir les paramètres généraux de l'Extracteur contient les contrôles suivants :

Elément Description
Paramètres généraux
Niveau de consignation
Spécifie le niveau de consignation pour l'extraction et l'indexation des données Extracteur Entreprise. Les données de consignation figurent dans les journaux des threads Content Server :
•

Normal : spécifie le niveau de consignation par défaut, ce qui signifie que l'avancement minimum de l'Extracteur est consigné.

•

Information : spécifie le niveau du journal détaillé.

•

Détaillé : spécifie le niveau du journal de débogage, ce qui génère un journal détaillé pour permettre le débogage.

 

Note :
Content Server Le niveau de débogage doit être à 2 ou supérieur à cette valeur pour que la consignation de l'Extracteur fonctionne.

 
Nombre maximum d'éléments par extraction
Spécifie le nombre maximum de requêtes de mise à jour de l'index de recherche que l'Extracteur traitera lors de l'exécution de chaque type individuel d'itérateur :
•

Extraire à nouveau

•

Mises à jour de pertinence

•

Ajouter/Modifier

•

Workflows lancés

•

Tous les autres

Pour chaque type d'itérateur, utilisez la liste déroulante pour sélectionner une option de traitement :
•

Extraction normale : l'extraction est normale.

•

Suspendre l'extraction : l'extraction n'aura pas lieu pour cet itérateur. Les éléments resteront dans la file d'attente de l'itérateur, et la file d'attente pourra augmenter.

 

Astuce :
Cette option ne doit être utilisée que temporairement, par exemple pour permettre à d'autres itérateurs de traiter davantage de données.

 
•

Rejeter les demandes d'indexation : l'extraction n'aura pas lieu pour cet itérateur. Les éléments présents dans la file d'attente de l'itérateur seront régulièrement supprimés pour l'empêcher d'augmenter. (Non disponible pour Extraire à nouveau.)

 

Astuce :
Cette option peut être utilisée de façon continue.

 
La valeur par défaut pour chaque type est 5000.
Nombre maximum de versions à extraire
Spécifie le nombre maximum de versions par document devant être traitées dans une requête d'extraction. Davantage de versions peuvent figurer dans l'index de recherche.
 

Note :
Si ce paramètre est trop faible, il peut mener au stockage d'informations incorrectes pour d'anciennes versions, en particulier deux versions différentes marquées comme la version actuelle. L'extraction des anciennes versions est requise pour mettre à jour les informations communes à toutes les versions des éléments, comme les dates modifiées, les informations utilisateur et la version actuelle. Les valeurs en dessous de 10 ne sont pas recommandées.

 
Inclure les listes de contrôle d'accès
Lorsqu'elles sont sélectionnées, les informations d'autorisation ACL (liste de contrôle d'accès) sont ajoutées au flux de données et à l'index de recherche.
 

Note :
Il n'est généralement pas recommandé d'inclure des informations ACL. L'extraction ACL diminue les performances d'indexation, et ne fournit pas de valeur pour les installations classiques.

 
Contenu d'objet EFS
Il y a deux manières de présenter les objets stockés dans le système de fichiers d'entreprise au serveur Document Conversion Server (DCS) pour être indexés :
•

Autoriser le DCS à lire directement à partir d'EFS est généralement plus rapide. Les fichiers ne sont pas supprimés. Il est recommandé de s'assurer que le serveur d'administration dispose de l'autorisation En lecture seule dans EFS.

•

L'intégration d'une copie du fichier est privilégiée lorsque la sécurité doit être maximisée, ou lorsque le DCS ne dispose pas d'accès en lecture dans EFS.

Contenu d'objet non EFS
Les systèmes de stockage non EFS les plus courants sont Enterprise Library ou Archive Server. Les objets extraits de ces sources pour être indexés par le DCS peuvent être intégrés dans des messages IPool ou placés dans une zone de travail temporaire.
En sélectionnant une zone de travail temporaire, le Chemin d'accès au fichier spécifié doit être accessible à chaque serveur d'administration, et à Content Server, ainsi qu'au serveur Document Conversion Server. Pour en savoir plus, consultez le document Transmission du contenu d'objet via les flux de données.
Limites IPool
Taille totale
Spécifie une limite, en Mo, de taille totale maximale pour les documents inclus dans un IPool. Ce paramètre ne spécifie pas de limite de taille IPool pour le message IPool lui-même.
Tout entier positif est valide et il n'y a pas de valeur maximale pour ce paramètre. La valeur par défaut est 1999 Mo.
 

Note :
Si un objet qui dépasse la limite figure dans l'IPool, celui-ci se ferme et passe au message suivant.

 
Taille du contenu d'élément unique
Spécifie une limite, en Mo, de taille maximale pour un document qui figurera dans un IPool. Ce paramètre ne spécifie pas de limite de taille IPool pour le message IPool lui-même. Le DCS ne peut pas traiter les fichiers d'une taille supérieure à 2 Go.
Tout entier positif est valide et il n'y a pas de valeur maximale pour ce paramètre. La valeur par défaut est 1024 Mo.
 

Note :
Si un objet qui dépasse la limite figure dans l'IPool, celui-ci se ferme et passe au message suivant.

 
Nombre d'éléments
Définit le nombre maximum d'éléments extraits et regroupés dans un seul message IPool par le processus Extracteur Entreprise. Les messages IPool contiennent les informations dont un index a besoin pour rester à jour. Par exemple, les requêtes pour mettre à jour ou supprimer des éléments indexés. En général, les valeurs comprises entre 100 et 500 sont recommandées.
Si vous définissez un nombre d'éléments trop faible, le temps nécessaire à la gestion IPool et l'indexation des transactions peut ralentir le processus d'indexation. Si cette valeur est trop élevée, les limites de taille des messages IPool risquent d'être dépassées.
Paramètres d'exclusion
Un certain nombre de paramètres sont disponibles pour contrôler les objets qui doivent être extraits et représentés dans l'index de recherche. Ni le contenu, ni les métadonnées des objets exclus ne seront indexés.
Le nombre d'objets actuellement exclus est affiché pour chaque type.
Types d'élément
Cliquez sur le lien liste de modification/révision pour accéder à Configuration des types d'élément exclus afin de voir ou d'ajouter des éléments, et pour obtenir davantage d'informations sur les types d'élément.
Types de volume
Cliquez sur le lien liste de modification/révision pour accéder à Configuration des types de volume exclus afin de voir ou d'ajouter des éléments, et pour obtenir davantage d'informations sur les types de volume.
Emplacements
Cliquez sur le lien liste de modification/révision pour accéder à Configuration des emplacements exclus afin de voir ou d'ajouter des emplacements, et pour obtenir davantage d'informations sur les emplacements.
Types de présentations
Cliquez sur le lien liste de modification/révision pour accéder à Configuration des types de présentation afin de voir et d'ajouter des types de présentations, et pour obtenir davantage d'informations sur les types de présentations.
 

Note :
Ce paramètre ne s'affiche que lorsqu'un module Renditions est installé sur votre système Content Server.
