Pour créer l'index Entreprise au cours de l'installation :

  1. Pour créer l'index Entreprise sur un hôte Content Server secondaire dont vous avez enregistré le serveur d'administration sur cet hôte, actualisez votre page Web pour afficher le nouvel hôte dans la liste déroulante Hôte de la section Informations sur le producteur.

  2. Sur la page Créer une nouvelle source de données Entreprise, saisissez un identificateur unique pour tous les objets système associés à ce flux de données d'indexation dans le champ Préfixe du processus. Si vous souhaitez définir le nombre de partitions qui doivent composer cet index, saisissez un nombre dans le champ Partitions.

  3. Dans le champ Port, saisissez une valeur représentant la série des numéros des ports sur lesquels vous souhaitez que les processus associés à cette source de données soient en réception. Le numéro de port spécifié et les onze numéros de port suivants (au moins) ne doivent pas être utilisés par une autre source de données dans votre système. Le nombre de numéros de port consécutifs utilisés dépend du nombre de partitions que vous indiquez dans le champ Partitions. La création d'un index Entreprise nécessite huit numéros de port et, pour chaque partition, quatre numéros de port supplémentaires. Les valeurs valables vont de 1025 à 65530.

  4. Dans le champ Répertoire d'écriture principal de la section Informations sur le producteur, saisissez le chemin absolu du répertoire où vous voulez que le processus Extracteur enregistre les données. Par défaut, le répertoire d'écriture est le répertoire <Content Server_accueil> /index/entreprise sur l'hôte principal Content Server. Vous devez choisir un répertoire placé sur un lecteur de l'hôte Content Server principal.

  5. Dans la liste déroulante Hôte de la section Informations sur le consommateur et intermédiaires, cliquez sur le raccourci du serveur d'administration sur l'hôte duquel les processus Conversion de documents et Distributeur de mise à jour doivent s'exécuter. Par défaut, ces processus sont exécutés sur un serveur d'administration de l'hôte Content Server principal.

  6. Dans le champ Répertoire de lecture principal, saisissez le chemin absolu du répertoire à partir duquel vous voulez que le processus Conversion de documents lise les données. Spécifiez le chemin de répertoire tel qu'il est mappé/monté sur l'hôte du serveur d'administration sur lequel est exécuté le processus de conversion de documents.

  7. Dans le champ Répertoire d'écriture principal de la section Consommateur et intermédiaire - Informations, saisissez le chemin absolu du répertoire où vous voulez que le processus Conversion de documents enregistre les données converties. Spécifiez le chemin de répertoire tel qu'il est mappé/monté sur l'hôte du serveur d'administration sur lequel est exécuté le processus de conversion de documents. OpenText vous recommande vivement de choisir un répertoire placé sur un lecteur local de l'hôte du serveur d'administration qui exécute le processus Conversion de documents.

  8. Pour lancer les processus de flux de données dès qu'ils sont créés, cochez la case Lancer les processus dans le flux de données.

  9. Cliquez sur le bouton Créer des processus.

  10. Sur la page Statut de la création d'un flux de données, cliquez sur le bouton Continuer.
