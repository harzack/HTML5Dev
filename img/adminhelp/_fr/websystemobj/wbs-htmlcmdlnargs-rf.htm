Outre les arguments répertoriés ci-dessous, vous pouvez spécifier n'importe quel paramètre contenu dans la section [FilterEngine] du fichier opentext.ini sur la ligne de commande correspondant à un processus Conversion de documents. Par exemple, vous pouvez spécifier si un processus Conversion de documents particulier génère des résumés des documents qu'il convertit en transmettant le paramètre -summary sur la ligne de commande. Pour plus d'informations sur les paramètres de la section [FilterEngine], reportez-vous à la rubrique [FilterEngine].

Les arguments définis sur la ligne de commande remplacent les paramètres généraux définis dans le fichier opentext.ini.

Le tableau suivant décrit les paramètres que vous pouvez définir sur la ligne de commande correspondant à un processus Conversion de documents particulier.

Tableau 1-6 : Arguments de ligne de commande du processus Conversion de documents

Argument Description
-readipool Spécifie le chemin absolu du répertoire de flux de données qui contient le sous-répertoire IPool à partir duquel le processus Conversion de documents lit les messages IPool.
-readarea Spécifie le nom du sous-répertoire IPool à partir duquel le processus Conversion de documents lit les messages IPool.
-writeipool Spécifie le chemin absolu du répertoire de flux de données qui contient le sous-répertoire IPool dans lequel le processus Conversion de documents enregistre les fichiers traités.
-writearea Spécifie le nom du sous-répertoire IPool dans lequel le processus Conversion de documents enregistre les fichiers traités.
-sleep Spécifie le nombre de millisecondes nécessaires à la mise en veille du processus Conversion de documents lorsque l'IPool d'entrée est vide. Après chaque période de veille, le processus vérifie s'il y a d'autres messages IPool à traiter dans l'IPool d'entrée. Si ce paramètre n'est pas défini ou s'il est défini sur -1, le processus s'arrête lorsqu'il rencontre un IPool vide. Si vous définissez ce paramètre sur un nombre positif, le processus est continuel. La valeur par défaut est -1.
-inifile Spécifie le chemin absolu du fichier opentext.ini.
-adminport Spécifie le numéro de port sur lequel le processus de conversion du document écoute la commande d'arrêt. Le serveur d'administration par défaut envoie une commande shutdown pour ordonner au processus Conversion de documents de s'arrêter progressivement. Par défaut, Content Server spécifie une valeur pour cet argument de ligne de commande lorsqu'il crée un processus Conversion de documents.
