Un processus Extracteur Entreprise est un processus de flux de données qui extrait des données de la base de données Content Server et les écrit dans un flux de données, où il est indexé.

Configurez les processus Extracteur Entreprise sur leurs pages Propriétés. Par exemple, vous pouvez modifier le serveur d'administration sur l'hôte duquel le processus Extracteur Entreprise s'exécute. Vous pouvez également activer la gestion des objets système, modifier les répertoires de démarrage et de destination, gérer les options de démarrage et d'arrêt ou resynchroniser le processus.

Vous pouvez également configurer le processus Extracteur Entreprise en modifiant les paramètres de la section [LivelinkExtractor] du fichier opentext.ini.

Onglet Infos spécifiques de la page Propriétés du processus Extracteur Entreprise

Elément Description
Etat Indique si le processus Extracteur Entreprise est en cours d'exécution. Vous ne pouvez pas modifier ce champ.
Serveur d'administration Cet élément spécifie le raccourci du serveur d'administration sur l'hôte duquel le processus Extracteur Entreprise s'exécute. Ne modifiez pas le raccourci du serveur d'administration pour le processus Extracteur Entreprise. Ce processus doit s'exécuter sur un hôte de serveur d'administration sur lequel s'exécute le serveur. Il s'agit toujours de l'hôte Content Server principal sur lequel s'exécute le serveur d'administration par défaut (principal).
Activer la gestion du système Détermine si Content Server contrôle ce processus pour détecter le retour de messages d'erreur. S'il retourne un message d'erreur, Content Server l'enregistre dans la base de données. La gestion du système est activée par défaut. Pour plus d'informations sur la configuration de Content Server en vue de l'envoi d'e-mails d'alerte lorsqu'un processus de flux de données rencontre des erreurs, reportez-vous à la rubrique Pour activer la recherche d'erreurs et l'envoi d'e-mails.
Dernière mise à jour Indique la date et l'heure de la dernière exécution du processus Extracteur Entreprise par Content Server. Vous ne pouvez pas modifier ce champ.
Progression de l'extraction complète Indique le statut du processus de l'Extracteur Entreprise. Avant l'exécution d'un processus d'extraction complète, le statut est vide. Lorsque le processus est en cours d'exécution, il indique l'ID du dernier nœud traité et l'ordre du processus (croissant ou décroissant). Le statut change lorsque le processus d'extraction est terminé.
Reprendre l'extraction complète à
Spécifie l'ID du nœud auquel le processus Extraction Entreprise reprend.
 

Astuce :
Cet élément n'a généralement pas besoin d'être modifié, sauf dans certains cas :

  • Un Extracteur peut ignorer un ID de nœud particulier, reconnu comme problématique, si vous saisissez l'ID suivant après celui-ci. Par exemple, si 5440 est un ID de nœud problématique, saisir 5439 permet de l'ignorer, mais il n'est pas nécessaire que 5439 soit un ID de nœud valide.

  • Spécifier -1 annule uniquement le processus d'extraction complète pour l'Extracteur en cours de modification. Sur un système avec plusieurs Extracteurs, vous devez tous les modifier pour annuler complètement l'extraction.

 
 

Note :
Ce paramètre est disponible uniquement lorsque l'Extracteur est arrêté.

 
Modèle de commande Cet élément spécifie le modèle utilisé pour générer la ligne de commande du processus Extracteur Entreprise. Ne modifiez pas cette valeur, sauf si le service d'assistance clientèle d'OpenText vous le demande.
Ligne de commande Indique la ligne de commande utilisée par Content Server pour exécuter le processus Extracteur Entreprise. Vous ne pouvez pas modifier la valeur de ce champ.
Répertoire de démarrage Cet élément spécifie le répertoire dans lequel le processus Extracteur Entreprise s'exécute. Le processus Extracteur Entreprise doit toujours s'exécuter dans le répertoire Content Server_accueil/bin. Ne modifiez pas cette valeur, sauf si le service d'assistance clientèle d'OpenText vous le demande.
Bouton Informations Charge la page Gestion des liens du flux de données, qui spécifie le répertoire dans lequel le processus Extracteur Entreprise écrit les données. Ne modifiez pas cette valeur, sauf si vous avez déplacé un IPool ou si le service d'assistance clientèle d'OpenText vous le demande.
Options de démarrage Cette section permet de spécifier les paramètres de lancement du processus Extracteur Entreprise. Lorsque ce dernier démarre, il parcourt la base de données Content Server, ajoutant et mettant à jour des données selon les besoins.
•

Manuel (option par défaut) permet de lancer le processus Extracteur Entreprise manuellement lorsque vous souhaitez à nouveau parcourir la base de données Content Server.

•

Programmé permet de programmer le jour et l'heure d'exécution du processus Extracteur Entreprise.

•

Continuel entraîne l'exécution en continu du processus Extracteur Entreprise.

Par défaut, le processus Extracteur Entreprise est programmé pour s'exécuter toutes les minutes de façon à répercuter en continu les modifications intervenant dans la base de données Content Server. Selon votre Système Content Server, vous pouvez modifier l'intervalle de programmation.
Options d'arrêt Cette section spécifie les paramètres d'arrêt du processus Extracteur Entreprise. Par défaut, l'option d'arrêt est définie sur Terminer. Il s'agit de l'option d'arrêt la plus adaptée à ce processus. Le champ Valeur maximale du code de sortie correcte spécifie le numéro de code d'erreur le plus élevé pour lequel le serveur d'administration redémarre automatiquement le processus Extracteur Entreprise en cas d'échec. La valeur du champ Valeur maximale du code de sortie correcte est fixée par défaut à -1, ce qui signifie que le serveur d'administration ne redémarrera pas ce processus en cas d'échec. Ne modifiez pas la Valeur maximale du code de sortie correcte, sauf si le service d'assistance clientèle d'OpenText vous le demande.
Démarrer/Arrêter Cet élément vous permet de gérer des flux de données, selon son état courant. Si le processus est en cours d'exécution ou programmé pour être exécuté, le bouton Arrêter s'affiche. Si le processus est arrêté, le bouton Démarrer s'affiche.
Resynchroniser Cet élément resynchronise les informations relatives à ce processus (dans le fichier otadmin.cfg du serveur d'administration du processus) avec les informations de la base de données Content Server.
