Lorsque vous créez des index Content Server à l'aide de modèles d'index, Content Server génère automatiquement les objets système appropriés, y compris le flux de données et tous les processus correspondants. Dans ce cas, Content Server configure chaque processus de flux de données à l'aide des valeurs par défaut. Vous pouvez modifier ces options par défaut pour personnaliser la configuration des processus de flux de données. La modification des options de lancement permet de personnaliser l'exécution des processus de flux de données afin d'améliorer l'efficacité du traitement des données. Par exemple, en fonction du volume de données à traiter sur votre site, vous pouvez programmer l'exécution des processus de flux de données toutes les minutes ou toutes les heures, tous les jours, ou tous les deux jours. Une autre option peut également assurer l'efficacité du traitement des données sur votre site : la configuration des options de lancement pour le processus Extracteur Entreprise et la configuration de règles de contrôle (règles d'application des actions de gestion du flux de données) pour les processus Conversion de documents et Distributeur de mise à jour. Par exemple, lors de la configuration des options de démarrage d'un processus Extracteur Entreprise, vous pouvez programmer son exécution tous les jours à 21:00. Le processus Extracteur Entreprise s'exécute jusqu'à ce qu'il ait extrait tous les objets de la base de données Content Server, puis il transfère les données extraites sous la forme de messages IPool vers l'IPool à partir duquel le processus Conversion de documents effectue sa lecture. Si vous définissez une règle de contrôle pour le processus de conversion de documents, le processus est exécuté chaque fois qu'il y a des messages IPool dans l'IPool à partir duquel il effectue la lecture. S'il n'y a pas de messages IPool, l'exécution du processus de conversion de documents s'arrête jusqu'à ce qu'il reçoive un ou plusieurs messages IPool. Vous pouvez également définir cette règle de contrôle pour le processus Distributeur de mise à jour, processus utilisé pour transmettre les données aux moteurs d'indexation afin qu'elles y soient indexées. Chaque fois qu'il y a des messages IPool dans l'IPool utilisé par le distributeur de mise à jour pour la lecture, le distributeur de mise à jour est exécuté ; son exécution s'arrête lorsqu'il n'y a pas de messages.

Vous pouvez configurer les options de lancement dans l'onglet Infos spécifiques de la page Propriétés des processus de flux de données afin de déterminer si le processus est exécuté en continu, s'il est exécuté selon un programme prédéterminé ou s'il est exécuté lorsque vous le lancez manuellement.

Le tableau suivant décrit les options de lancement les plus appropriées pour les processus de flux de données.

Tableau 1-1 : Options de démarrage par défaut

Type de processus Description de l'option de démarrage par défaut
Extracteur Entreprise Programmez ce processus pour qu'il s'exécute toutes les minutes afin de lui permettre de connaître en permanence les modifications effectuées dans votre base de données Content Server. Bien que vous puissiez modifier l'heure et les jours d'exécution du processus Extracteur Entreprise, OpenText vous recommande de ne pas définir l'option de lancement de ce processus sur Manuel ou Continuel.
Directory Walker Définissez ce processus pour qu'il s'exécute lorsque vous le démarrez manuellement. Il continue ensuite de s'exécuter jusqu'à la fin de ses opérations. Selon le type d'index que vous créez, vous pouvez modifier le paramètre manuel. Utilisez l'option de démarrage Manuel lors de l'indexation de données relativement statiques sur votre système de fichiers. Si vous prévoyez de modifier fréquemment le contenu des répertoires spécifiés, vous pouvez configurer le processus Directory Walker pour s'exécuter de façon préprogrammée (par exemple, une fois par semaine). Vous vous assurez ainsi que l'index reste synchronisé avec l'état actuel des données contenues dans les répertoires parcourus.
Conversion de documents Programmez ce processus pour qu'il s'exécute à chaque minute afin de lui permettre de savoir en permanence si son répertoire de lecture ne contient pas de nouvelles données. Bien que vous puissiez modifier l'heure et les jours d'exécution du processus Conversion de documents, OpenText vous recommande de ne pas définir l'option de lancement de ce processus sur Manuel ou Continuel.
Distributeur de mise à jour Ce processus est exécuté de façon continue pour permettre de savoir en permanence si son répertoire de lecture contient de nouvelles données à indexer. OpenText vous recommande de ne définir aucune autre option de lancement pour le processus Distributeur de mise à jour.
Importateur Programmez ce processus pour qu'il s'exécute toutes les minutes afin de lui permettre de s'adapter en permanence aux modifications effectuées dans votre base de données Content Server. OpenText vous recommande de ne définir aucune autre option de lancement pour le processus Importateur.
Producteur XML Activator Programmez ce processus pour qu'il soit exécuté toutes les minutes de manière à parcourir régulièrement un répertoire. Bien que vous puissiez modifier l'heure et les jours lors de l'exécution du processus Producteur XML Activator, OpenText vous recommande de ne pas définir l'option de lancement de ce processus sur Manuel ou Continuel.
