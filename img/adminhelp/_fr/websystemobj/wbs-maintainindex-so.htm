Après avoir créé l'index d'une source de données, vous pouvez en assurer la gestion en le déplaçant, en rassemblant les fichiers journaux de recherche pour une source de données spécifique ou en vérifiant son contenu et en le réindexant.

Une fois que vous avez construit l'index d'une source de données en créant une ou plusieurs partitions, vous pouvez déplacer l'index d'une partition vers un nouvel emplacement. Le déplacement de l'index d'une partition peut s'avérer nécessaire pour différentes raisons, par exemple, si le lecteur sur lequel réside l'index de la partition est affecté à une autre utilisation, si vous mettez à niveau le matériel sur lequel réside l'index de la partition ou si l'index de la partition dépasse la capacité du lecteur sur lequel il réside.

Vous pouvez rassembler les fichiers journaux et les archiver dans un répertoire de journaux temporaire pour les examiner. Vous pouvez définir les numéros des lignes de fin souhaitées et collecter d'autres fichiers, tels que les search.ini et otadmin.cfg.

Si un index vous semble incomplet, obsolète ou corrompu, certaines tâches administratives vous aideront à détecter et corriger les problèmes potentiels. Les tâches administratives à votre disposition dépendent du type de la source de données utilisée. Par exemple, si vous travaillez avec une source de données Entreprise, vous pouvez vérifier le contenu de l'index Entreprise, extraire à nouveau les données de la base Content Server ou éliminer le flux de données Entreprise et reconstruire l'index. Si vous travaillez avec des sources de données non-Entreprise, vous pouvez extraire à nouveau les données de la source ou éliminer le flux de données et reconstruire l'index.
