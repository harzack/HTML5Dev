Vous pouvez activer le journal de connexion SQL lorsque vous voulez enregistrer la communication qui s'engage entre le serveur et la base de données Content Server. Activez le journal de connexion SQL en activant le journal de connexion sur la page Configurer les paramètres du débogage.

Le nombre de fichiers connectn.log est égal au nombre de threads sur lesquels le serveur est exécuté. Si vous diagnostiquez un problème, vous pouvez temporairement définir le serveur pour qu'il soit exécuté sur un seul thread. Cela vous permettra de rechercher des informations de diagnostic dans un seul fichier. L'exécution sur un seul thread réduisant énormément les performances du serveur, OpenText vous recommande d'opter pour ce type d'exécution pendant les périodes de faible utilisation et de rétablir les paramètres de thread d'origine du serveur après avoir effectué votre diagnostic.

Content Server vous permet d'enregistrer la communication qui s'engage entre le serveur et la base de données Content Server. Lorsque cette option est activée, Content Server enregistre les journaux de connexion à la base de données dans des fichiers connectn.log du répertoire Content Server_home/logs. Par défaut, les journaux de connexion à la base de données sont désactivés.

 

Note :
La taille des journaux de connexion à la base de données peut augmenter rapidement. OpenText vous recommande d'activer le journal de connexion si vous en avez besoin pour diagnostiquer un problème, mais de le désactiver dès que le problème est résolu.
