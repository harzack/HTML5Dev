Le processus Conversion de documents est un processus de flux de données intermédiaire dans un flux de données d'indexation de Content Server. Vous créez des flux de données d'indexation lors de la création de l'index d'une source de données. Le processus Conversion de documents est automatiquement ajouté à un flux de données d'indexation pour une source de données créée à l'aide des modèles Content Server. Lorsque vous créez une source de données en ajoutant ses composants individuellement, vous pouvez ajouter un processus Conversion de documents à un flux de données d'indexation. Après avoir ajouté un processus Conversion de documents à un flux de données d'indexation, vous pouvez le configurer.

Les processus Conversion de documents lisent les données produites par le processus de flux de données précédent à partir d'un IPool, puis ils convertissent ces données de leur format natif au format HTML ou texte brut.

Le processus Conversion de documents convertit les données au format HTML ou texte à l'aide de filtres de conversion. Pour de plus amples informations sur les filtres de conversion utilisés par le processus Conversion de documents, reportez-vous à la rubrique Filtres de conversion de documents.

Dans la plupart des cas, un processus Conversion de documents convertit les données suivantes :

  • des données extraites de la base de données Content Server par le processus Extracteur Entreprise ;

  • des données recueillies par un processus producteur XML à partir d'applications tierces ;

  • des données extraites par le processus Directory Walker à partir de votre système de fichiers ;

  • Données extraites par un processus Content Server Spider à partir du Web

  • des données extraites par un processus Lotus Notes Extractor à partir de bases de données Lotus Notes (si vous achetez et installez Activator pour le module Lotus Notes).

 

Note :
Les documents Excel 97 contenant des liens hypertextes très longs ne sont pas toujours correctement convertis au format HTML.

Si le contenu d'un fichier d'archives (ZIP) dépasse 2 Go lorsqu'il n'est pas compressé, le DCS rejette une partie des données, entraînant une indexation partielle de l'objet.

 

Une fois que le processus Conversion de documents a converti les données au format HTML ou texte, il écrit ces données dans un IPool. Le distributeur de mise à jour lit ces données et les distribue aux moteurs d'indexation. Les moteurs d'indexation utilisent les données converties pour mettre à jour ou créer l'index correspondant. Pour plus d'informations sur la modification des distributeurs de mise à jour ou des moteurs d'indexation, voir Configuration des processus d'indexation.

Vous pouvez personnaliser le processus Conversion de documents dans tous les flux de données d'indexation. Pour personnaliser un processus Conversion de documents, vous devez connaître et sélectionner les filtres de conversion, configurer les arguments de la ligne de commande et définir les paramètres du processus Conversion de documents.
