Contenu d'objet EFS

Lorsque vous définissez le champ Contenu d'objet EFS pour utiliser Incorporer une copie du contenu d'objet dans les messages IPool, le contenu d'objet EFS (magasin de fichiers externe) est incorporé dans les messages IPool pour augmenter les performances d'indexation.

Lorsque vous définissez le champ Contenu d'objet EFS pour Autoriser le DCS à lire le contenu d'objet depuis EFS, le processus Extracteur Entreprise va extraire le contenu d'un document en référençant son emplacement dans EFS. Le processus Conversion de documents (DCS) doit pouvoir accéder au contenu du document grâce à la référence exacte.

Contenu d'objet non EFS

Lorsque vous définissez le champ Contenu d'objet non EFS pour utiliser Incorporer une copie du contenu d'objet dans les messages IPool, le contenu d'objet non EFS (magasin de fichiers non externe) est incorporé dans les messages IPool pour augmenter les performances d'indexation.

Le contenu d'objet non EFS (magasin de fichiers non externe) est incorporé dans les messages IPool pour augmenter les performances d'indexation.

Lorsque vous définissez le champ Contenu d'objet non EFS sur Créer une copie temporaire du contenu d'objet qui sera supprimée par le DCS après traitement, l'Extracteur va utiliser l'emplacement File Path pour le contenu d'objet volumineux binaire (blob) stocké dans la base de données relationnelle (BDR), ainsi que dans Archive Server ou d'autres fournisseurs de stockage tiers. L'Extracteur va traiter le contenu stocké d'un objet et le placer dans un fichier temporaire du répertoire Chemin d'accès au fichier.

 

Note :
Il ne faut pas utiliser d'emplacement temporaire pour les objets non EFS si le flux de données est divisé puisque seul un consommateur IPool peut être responsable de la suppression des fichiers à partir de l'emplacement temporaire.
