Pour créer un index Entreprise après l'installation :

  1. Sur la page Volume des objets système, cliquez sur Source de données Entreprise dans le menu Ajouter un élément. Si vous avez supprimé une source de données Entreprise et que vous souhaitez associer ses requêtes et ses formulaires de recherche enregistrés à cette nouvelle source de données Entreprise, cliquez sur le nom de la vue de recherche Entreprise de la source de données supprimée dans la liste déroulante Remplacement de vue de recherche, puis cliquez sur la vue de recherche Entreprise [Toutes les versions] de la source de données supprimée dans la liste déroulante Remplacement de vue de recherche [Toutes versions].

  2. Saisissez un identificateur unique pour tous les objets système associés à ce flux de données d'indexation dans le champ Préfixe du processus. Cet identificateur correspond au nom d'affichage des objets associés à cet index sur la page Administration système et au nom d'affichage de la vue de recherche de l'index dans la liste Vues de recherche sur la page Recherche. Si vous souhaitez définir le nombre de partitions qui doivent composer cet index, saisissez un nombre dans le champ Partitions.

  3. Dans le champ Port, saisissez une valeur représentant la série des numéros des ports sur lesquels vous souhaitez que les processus associés à cette source de données soient en réception. Le numéro de port spécifié et les douze numéros de port suivants (au moins) ne doivent pas être utilisés par une autre source de données dans votre système. Le nombre de numéros de port consécutifs utilisés dépend du nombre de partitions que vous indiquez dans le champ Partitions. La création d'un index Entreprise nécessite huit numéros de port et, pour chaque partition, quatre numéros de port supplémentaires. Les valeurs valables vont de 1025 à 65535.

  4. Dans la liste déroulante Hôte de la section Informations sur le producteur, cliquez sur le raccourci du serveur d'administration sur l'hôte à partir duquel le processus Extracteur doit s'exécuter.

  5. Dans le champ Répertoire d'écriture principal de la section Informations sur le producteur, saisissez le chemin absolu du répertoire (par rapport au serveur d'administration sur lequel le processus Extracteur est exécuté) dans lequel vous souhaitez que le processus Extracteur écrive les données. Par défaut, le répertoire d'écriture est le répertoire Content Server_home/index/enterprise sur l'hôte Content Server principal par défaut. Vous devez choisir un répertoire placé sur le lecteur de l'hôte Content Server principal, et ce répertoire ne doit pas être le répertoire d'écriture d'autres sources de données Entreprise.

  6. Dans la liste déroulante Hôte de la section Informations sur le consommateur et intermédiaires, cliquez sur le raccourci du serveur d'administration sur l'hôte duquel les processus Conversion de documents et Distributeur de mise à jour doivent s'exécuter. Par défaut, ces processus sont exécutés sur un serveur d'administration de l'hôte Content Server principal.

  7. Dans le champ Répertoire de lecture principal, saisissez le chemin absolu du répertoire à partir duquel vous voulez que le processus Conversion de documents lise les données. Spécifiez le chemin de répertoire tel qu'il est mappé/monté sur l'hôte du serveur d'administration sur lequel est exécuté le processus de conversion de documents. Ce répertoire doit être identique au répertoire d'écriture principal spécifié dans la section Informations sur le producteur.

  8. Dans le champ Répertoire d'écriture principal de la section Consommateur et intermédiaire - Informations, saisissez le chemin absolu du répertoire où vous voulez que le processus Conversion de documents enregistre les données converties. Spécifiez le chemin de répertoire tel qu'il est mappé/monté sur l'hôte du serveur d'administration sur lequel est exécuté le processus de conversion de documents. OpenText vous recommande vivement de choisir un répertoire placé sur un lecteur local de l'hôte du serveur d'administration qui exécute le processus Conversion de documents. Ce répertoire doit également être différent des répertoires d'écriture principaux d'autres sources de données Entreprise.

  9. Pour lancer les processus de flux de données dès qu'ils sont créés, cochez la case Lancer les processus dans le flux de données.

  10. Cliquez sur le bouton Créer des processus.

  11. Sur la page Statut de la création d'un flux de données, cliquez sur le bouton Continuer.
