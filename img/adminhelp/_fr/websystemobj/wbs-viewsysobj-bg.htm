La page Afficher le statut de l'objet système contient trois sections : Statut des objets système actuels, Tests des objets système précédemment exécutés et Tester les objets système.

Statut des objets système actuels

Vous pouvez tester votre objet système sur votre site Content Server (par exemple, les flux de données, processus et moteurs de recherche du volume des objets système) pour déceler les erreurs. Lorsque les objets système comportent des erreurs, Content Server affiche le nom des objets et les erreurs signalées sur la page Afficher le statut de l'objet système. Si un objet comporte une erreur qui empêche l'exécution des processus d'indexation et de recherche (par exemple, le serveur d'administration ne fonctionne pas), l'erreur apparaît comme Grave.

Pour retourner à la page d'administration, cliquez sur le bouton Continuer.

Tests des objets système précédemment exécutés

Cette section fournit des informations concernant les tests des objets système exécutés précédemment.

Par défaut, un agent Content Server teste les objets dans le volume système pour déceler d'éventuelles erreurs toutes les cinq minutes ; cependant, vous pouvez tester les objets système à tout moment lorsque vous soupçonnez des erreurs. Chaque fois qu'un test est effectué, Content Server enregistre les informations suivantes dans son historique :

  • Dernière mise à jour : indique la date et l'heure du dernier test effectué sur le système

  • Notification : indique si le test a été effectué par un agent Content Server

  • Opération réussie : indique si le test a décelé des erreurs

  • Temps écoulé : indique la durée (en secondes) du test

  • Dernier message : indique le nom du service qui a effectué le test Par défaut, il s'agit du service Gestion du système. Si un test échoue, ce champ en donne la raison.

 

Note :
Si un objet système comporte une erreur, vous devez identifier le problème et corriger tous les défauts qu'il peut entraîner. Ainsi, la prochaine fois qu'un rapport d'erreurs d'objet système sera généré, un champ Correction apparaîtra, indiquant qu'une erreur précédemment décelée a été corrigée.

 

Tester les objets système

Que vous activiez ou désactiviez la vérification automatique d'erreurs des objets système, vous pouvez à tout moment tester le système Content Server pour déterminer si les objets système renvoient des erreurs.

 

Note :
Les objets système comprennent les flux de données, les processus, les vues de recherche et les moteurs de recherche du volume des objets système.

 

Lorsque vous testez des objets système pour diagnostiquer des erreurs, Content Server génère un rapport d'erreurs des objets système qui répertorie tous les objets système qui renvoient des messages d'erreur et affiche le texte des messages d'erreur. Vous pouvez également configurer Content Server pour qu'il envoie des e-mails d'alerte relatifs aux objets système aux destinataires que vous configurez sur la page Configurer l'envoi d'e-mails d'alerte relatifs aux objets système. Cette page est accessible via le lien de la section Rapport d'e-mail.

Lorsque la recherche d'erreurs relatives aux objets système et les e-mails d'alerte automatiques sont activés, Content Server enregistre tous les messages d'erreur renvoyés par les objets système dans la base de données Content Server. OpenText vous recommande d'éliminer régulièrement le journal des messages d'erreur des objets système à partir de la base de données Content Server.
