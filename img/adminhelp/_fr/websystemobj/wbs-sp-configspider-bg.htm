Un processus Spider est un processus de flux de données qui extrait des données d'un ou de plusieurs serveurs Spider et les écrit dans un flux de données, où il est indexé.

La configuration des processus Spider s'effectue sur les pages Propriétés. Par exemple, vous pouvez modifier le serveur d'administration sur l'hôte à partir duquel le processus Spider s'exécute. Vous pouvez également sélectionner des projets Spider pour l'indexation et définir leurs caractéristiques, gérer les options de lancement et d'arrêt, ou resynchroniser les processus.

Onglet Spécifique de la page Propriétés Spider :

Elément Description
Etat Cet élément spécifie si le processus Spider est ou non en cours d'exécution. Vous ne pouvez pas modifier ce champ.
Serveur d'administration Cet élément spécifie le raccourci du serveur d'administration sur l'hôte à partir duquel le processus Spider s'exécute. Ne modifiez pas le raccourci du serveur d'administration pour le processus Spider. Ce processus doit s'exécuter sur un hôte de serveur d'administration à partir duquel s'exécute le serveur. Il s'agit toujours de l'hôte Content Server principal sur lequel s'exécute le serveur d'administration par défaut (principal).
Serveur Spider
Projets disponibles sur le serveur Spider identifié par l'hôte, le port et les mots de passe facultatifs. Sélectionnez les projets Spider pour l'indexation dans Content Server en cliquant dessus.
•

Le nom de l'Hôte de chaque module de serveur Spider installé sur votre système Content Server. Si nécessaire, sous Action, cliquez sur bouton Dupliquer pour ajouter d'autres serveurs Spider.

•

Le numéro de Port du serveur Spider.

•

Le Mot de passe pour accéder au serveur Spider, si défini.

•

Les Caractéristiques du projet pour cette source de données à indexer. Cliquez sur le lien Création de la liste des projets pour afficher une liste des projets du serveur Spider, ainsi que leurs numéros de port, définis sur l'Hôte, puis cliquez sur les projets à inclure. Vous pouvez également saisir les caractéristiques directement, à l'aide du format nom du projet, nom du port, séparé par ; (point-virgule). Si nécessaire, vous pouvez également mettre les projets en surbrillance puis les supprimer.

•

Sous Action, cliquez sur le bouton Gérer le serveur Spider pour modifier les paramètres du serveur Spider, y compris les URL de site Web et les caractéristiques de navigation.

•

Si nécessaire, cliquez sur le bouton Supprimer pour supprimer les définitions du serveur Spider inutiles.

Modèle de commande Cet élément spécifie le modèle utilisé pour générer la ligne de commande du processus Spider. Ne modifiez pas cette valeur, sauf si le service d'assistance clientèle d'OpenText vous le demande.
Ligne de commande Cet élément spécifie la ligne de commande utilisée par Content Server pour exécuter le processus Spider. Vous ne pouvez pas modifier la valeur de ce champ.
IPools Cet élément charge la page Gestion des liens du flux de données lorsque vous cliquez sur le bouton Informations, qui spécifie le répertoire dans lequel le processus Spider écrit les données. Ne modifiez pas cette valeur, sauf si vous avez déplacé un IPool ou si le service d'assistance clientèle d'OpenText vous le demande.
Fichier journal Spécifie le chemin absolu du fichier journal dans lequel Content Server enregistre les activités du processus Spider. Si le champ est vide, la fonction de journalisation est désactivée.
Niveau de débogage Spécifie le niveau de consignation (Valeur par défaut, Détaillé ou Déboguer) appliqué si la journalisation est activée.
•

Par défaut pour enregistrer uniquement les erreurs dans le journal. Il s'agit du niveau de consignation minimum.

•

Détaillé pour enregistrer le statut des messages acheminés à travers les processus et les IPools du flux de données dans le journal. Les erreurs sont également enregistrées.

•

Déboguer pour enregistrer des informations détaillées au sujet du statut du flux de données et des erreurs dans le journal. Compte tenu de l'importante quantité de données générées, OpenText recommande de choisir ce niveau uniquement si vous devez déboguer des problèmes graves sur le flux de données.

Programme
Représente le moment où le processus Spider est programmé pour être exécuté :
•

A cette heure : une heure spécifique certains jours de la semaine.

•

Chaque : un intervalle spécifique.

Valeur maximale du code de sortie correcte Numéro de code d'erreur le plus élevé pour lequel le serveur d'administration redémarre automatiquement le Spider en cas d'échec. Ne modifiez pas cette valeur, sauf si le service d'assistance clientèle d'OpenText vous le demande.
Processus Cet élément vous permet de gérer des flux de données, selon son état courant. Si le processus est en cours d'exécution ou programmé pour être exécuté, le bouton Arrêter s'affiche. Si le processus est arrêté, le bouton Démarrer s'affiche.
Option Cet élément resynchronise les informations relatives à ce processus (dans le fichier otadmin.cfg du serveur d'administration du processus) avec les informations de la base de données Content Server.
Mettre à jour Cet élément soumet au serveur les modifications effectuées sur cette page.
Réinitialiser Rétablit les informations de cette page à l'état qui était le leur à l'ouverture de la page.
