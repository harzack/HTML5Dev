Un flux de données est un ensemble de processus liés qui traitent les données à partir d'une seule source dans Content Server. Content Server vous permet de créer un flux de données qui indexe les données d'une source spécifique afin de pouvoir y effectuer des recherches. Vous créez des flux de données d'indexation lorsque vous créez des index. Les processus de flux de données d'un flux de données d'indexation incluent les processus, qui localisent ou extraient les données (par exemple, le processus Extracteur Entreprise, Directory Walker ou XML Activator), le processus Conversion de documents, qui convertit les données de leur format natif au format HTML ou texte brut et le processus Distributeur de mise à jour, qui transmet les données aux moteurs d'indexation pour qu'elles y soient indexées.

Après avoir ajouté les processus qui constituent un flux de données, vous pouvez gérer celui-ci (par exemple, le reprendre, l'interrompre ou le resynchroniser).

Affichage des images de flux de données

La navigation, méthode d'affichage par défaut des flux de données, permet d'afficher des listes de processus et d'IPools dans un flux de données. Lorsque vous affichez une image de flux de données, vous pouvez voir une représentation graphique du mode de liaison entre les processus et les IPools dans un flux de données. Pour plus d'informations sur l'affichage d'une image de flux de données, reportez-vous à la rubrique Affichage des images de flux de données.

Surveillance du statut des flux de données

Vous pouvez surveiller le statut d'un flux de données dans un dossier source de données. Un flux de données peut avoir les statuts suivants :

  • Actif, qui indique que tous les processus du flux de données sont en cours d'exécution ou programmés pour être exécutés.

  • Inactif : n exécutés, qui indique qu'un processus au moins du flux de données est inactif. La valeur n indique le nombre de processus en cours d'exécution ou dont l'exécution est programmée.

  • Inactif, qui indique que tous les processus du flux de données sont inactifs.

  • Problème : n exécutés, qui indique qu'au moins un processus du flux de données renvoie un message d'erreur. La valeur n indique le nombre de processus en cours d'exécution ou dont l'exécution est programmée.

Lorsque vous naviguez dans le gestionnaire de flux de données ou que vous affichez une image de flux de données, vous pouvez surveiller le statut de chaque processus dans un flux de données. Un processus de flux de données peut avoir les statuts suivants :

  • En cours d'exécution, qui indique que le processus est en cours d'exécution.

  • Programmé, qui indique que le processus est programmé pour être exécuté, mais qu'il n'exécute pas de nouveaux statuts.

  • Contact des moteurs d'indexation, qui indique que le processus a terminé l'initialisation et tente de contacter les processus du moteur d'indexation. Ce statut apparaît uniquement pour le processus du distributeur de mise à jour.

  • En attente des moteurs d'indexation, qui indique que le processus n'a pas encore pu contacter de processus du moteur d'indexation.

  • Recherche du distributeur de mise à jour, qui indique que le processus tente de contacter le serveur d'échange afin de déterminer le port utilisé par le processus Distributeur de mise à jour. Ce statut apparaît uniquement pour le processus du moteur d'indexation.

  • Contact du distributeur de mise à jour, qui indique que le processus a trouvé le port du processus du distributeur de mise à jour et a terminé l'initialisation et tente actuellement de contacter le processus. Ce statut apparaît uniquement pour le processus du moteur d'indexation.

  • Création d'un index vide, qui indique que le processus crée l'index pour la première fois. Ce statut apparaît uniquement pour le processus du moteur d'indexation.

  • Chargement de l'index, qui indique que le processus charge un index dans la mémoire. Ce statut, disponible principalement lorsque l'index est volumineux, apparaît uniquement pour les processus du moteur d'indexation et du moteur de recherche.

  • Restauration de l'index, qui indique que le processus charge un index restauré dans la mémoire. Ce statut, disponible principalement lorsque l'index est volumineux, apparaît uniquement pour le processus du moteur d'indexation.

  • En attente des moteurs de recherche, qui indique que le processus attend d'être contacté par un processus du moteur d'indexation. Ce statut apparaît uniquement pour le processus du fédérateur de recherche.

  • En attente de l'index initial, qui indique que le processus attend qu'un processus du moteur d'indexation crée un index initial. Ce statut apparaît uniquement pour le processus du moteur de recherche.

  • Inscription avec le fédérateur de recherche, qui indique que le processus contacte un fédérateur de recherche. Ce statut apparaît uniquement pour le processus du moteur de recherche.

  • Inactif, qui indique que le processus est arrêté et qu'il n'est pas programmé pour être exécuté.

  • Erreur <n>, qui indique que le processus renvoie un message d'erreur. Cliquez sur le message d'erreur pour afficher l'onglet Infos spécifiques de la page Propriétés du processus de flux de données. Vous pouvez alors modifier le paramètre du processus.

  • Inconnu, qui indique que Content Server a pu communiquer avec le serveur d'administration, mais que ce dernier n'a pas pu consigner le statut du processus. Cela peut se produire si le fichier otadmin.cfg du serveur d'administration est endommagé, manquant ou s'il n'est pas synchronisé avec la base de données Content Server.

  • Admin N/A, qui indique que le serveur d'administration n'est pas disponible, ce qui signifie qu'il ne peut pas consigner le statut du processus. Si le serveur d'administration n'est pas disponible, il est possible qu'il n'existe pas.
