Le contrôle de l'accès IP aux composants de recherche vous permet d'augmenter le niveau de sécurité de votre système Content Server en limitant l'accès aux composants utilisés par la recherche. Pour ce faire, enregistrez les adresses IP des ordinateurs pour lesquels vous souhaitez autoriser l'accès. Content Server stocke alors les adresses IP dans la base de données Content Server et dans un fichier distant (otsearch.policy) dans le répertoire config de chaque serveur Content Server (par exemple Content Server_accueil /config). Etant donné que Content Server remplace le fichier distant chaque fois qu'il ajoute ou supprime une adresse IP du fichier, vous ne devez pas modifier le fichier manuellement.

 

Avertissement
Si aucune adresse IP n'est enregistrée, Content Server autorise l'accès à tous les ordinateurs. Toutefois, dès lors que des adresses IP ont été enregistrées, Content Server autorise l'accès uniquement aux ordinateurs possédant ces adresses IP. Par conséquent, vous devez enregistrer les adresses IP de tous les ordinateurs qui hébergent des serveurs et des serveurs d'administration afin d'éviter que le système de recherche ne réponde plus.

 

Si vous enregistrez des adresses IP incorrectes et que vous n'êtes plus en mesure de contrôler l'accès IP au moyen de l'interface Content Server, vous pouvez restaurer l'accès en remplaçant le fichier distant actuel par un fichier distant vide, qui autorisera alors l'accès aux composants de recherche à toutes les adresses IP. Un fichier distant vide, nommé otsearch.policy.bak, est disponible dans le même répertoire que le fichier distant actuel ; ce fichier est généré lors de la première modification du fichier distant. Toutefois, alors que le système de recherche utilise le fichier distant lorsqu'il contrôle l'accès IP, l'interface Content Server affiche les valeurs stockées dans la base de données Content Server. De ce fait, après avoir remplacé le fichier distant, vous devez mettre à jour les valeurs dans l'interface Content Server.
