Un flux de données correspond à une série de processus liés qui échangent des informations. Vous pouvez créer un flux de données personnalisé pour une source de données particulière en ajoutant et en configurant des processus de flux de données. Vous ajoutez les processus de flux de données à un gestionnaire de flux de données. Les processus de flux de données incluent :

  • Directory Walker qui extrait les données d'un ensemble de fichiers stocké sur votre système de fichiers et les écrit dans un flux de données. Pour plus d'informations sur l'ajout d'un processus Directory Walker, reportez-vous à la rubrique Ajout d'un processus Directory Walker. Pour plus d'informations sur la configuration d'un processus Directory Walker, reportez-vous à la rubrique Configuration d'un processus Directory Walker.

  • La Conversion de documents qui convertit des documents au format HTML ou texte brut pour permettre leur affichage et indexation. Pour plus d'informations sur l'ajout d'un processus Conversion de documents, reportez-vous à la rubrique Pour ajouter un processus Conversion de documents. Pour plus d'informations sur la configuration d'un processus Conversion de documents, reportez-vous à la rubrique Configuration du service de conversion de documents.

  • L'Extracteur Entreprise qui est un processus de flux de données qui extrait des données de la base de données Content Server et les écrit dans un flux de données Entreprise, où elles sont indexées. Un seul processus étant habituellement requis pour extraire les données de la base de données Content Server, OpenText vous recommande de ne prévoir qu'un seul processus Extracteur Entreprise pour chaque système Content Server, sauf si les services généraux ou le service d'assistance clientèle d'OpenText recommande l'utilisation de plusieurs Extracteurs Entreprise dans le cadre d'une stratégie d'indexation de gros volumes. Pour plus d'informations sur l'indexation de gros volumes, reportez-vous à la rubrique Configuration de l'indexation de gros volumes. Pour plus d'informations sur l'ajout d'un processus Extracteur Entreprise, reportez-vous à la rubrique Pour ajouter un processus Extracteur Entreprise. Pour plus d'informations sur la configuration d'un processus Extracteur Entreprise, reportez-vous à la rubrique Configuration du processus Extracteur Entreprise.

  • Spider, un processus de flux de données qui extrait les données d'un serveur Spider parcourant des sites intranets, extranets et Web d'entreprises, ou même des sites Web publics, et qui les transmet au gestionnaire de flux de données de toute source de données non Entreprise, où elles sont indexées. Le serveur Spider d'OpenText est un package supplémentaire que vous devez installer séparément de votre système Content Server. Un administrateur peut configurer des projets de parcours de données étendus et sophistiqués, afin de créer des index de contenu externe, où tous les utilisateurs peuvent effectuer des recherches. Pour plus d'informations sur l'indexation de sources Web ciblées, voir Pour ajouter un nouveau flux de données Spider. Pour plus d'informations sur l'ajout et la configuration d'un processus Spider, voir Pour ajouter un Spider et Pour configurer un processus Spider.

  • Importateur : lit les données générées par le processus Distributeur de mise à jour, puis les utilise pour exécuter une tâche. Par exemple, si vous avez installé les Content Server Classifications, l'importateur détermine les classifications assistées et automatiques qui conviennent le mieux aux éléments représentés par les données. Cette fonctionnalité sera ajoutée aux prochaines versions de Content Server et de ses modules. Pour plus d'informations sur l'ajout d'un processus Importateur, reportez-vous à la rubrique Pour ajouter un processus Importateur. Pour plus d'informations sur la configuration d'un processus Importateur, reportez-vous à la rubrique Configuration d'un processus Importateur.

  • Fusion, qui fusionne plusieurs processus Extracteur Entreprise ou Conversion de documents, afin de combiner les messages iPool provenant de processus multiples et de les inscrire dans un seul iPool pour le processus de flux de données suivant. Pour plus d'informations sur l'ajout d'un processus Fusion, reportez-vous à la rubrique Pour ajouter un processus de fusion. Pour plus d'informations sur la configuration d'un processus Fusion, reportez-vous à la rubrique Configuration d'un processus de fusion.

  • Proxy : agit comme un emplacement réservé destiné à un processus au sein d'un flux de données. Ajoutez un proxy pour représenter un processus qu'aucun serveur d'administration ne gère. Par exemple, si vous créez un processus personnalisé non-Content Server (qui crée, manipule ou consomme des données) qui sera utilisé dans un flux de données Content Server, vous pouvez représenter ce processus dans Content Server au moyen d'un proxy. Le processus représenté par le proxy lit les données de l'IPool spécifié, exécute un processus, puis écrit les données dans un autre IPool que vous aurez indiqué. Pour plus d'informations sur l'ajout d'un processus Mandataire, reportez-vous à la rubrique Pour ajouter des proxies. Pour plus d'informations sur la configuration d'un processus Mandataire, reportez-vous à la rubrique Configuration des proxies.

  • T, qui envoie des messages IPool à cinq processus de sortie IPool maximum présentant des fonctionnalités similaires, tels que des processus Importateur IPool. Pour plus d'informations, reportez-vous à la rubrique Ajout et configuration d'un processus T.

  • XML Activator, qui lit les données (sous forme de fichiers XML) depuis le répertoire spécifié dans le système de fichiers dans lequel une application tierce a écrit les informations. Pour plus d'informations sur l'ajout d'un processus Producteur XML Activator, reportez-vous à la rubrique Configuration finale d'un processus producteur XML Activator. Pour plus d'informations sur la configuration d'un processus Producteur XML Activator, reportez-vous à la rubrique Configuration d'un processus XML Activator.

Vous pouvez créer un flux de données d'indexation pour permettre l'indexation des données et la recherche au sein de ces données. La création d'un flux de données d'indexation fait partie du processus de création des index. Après avoir créé un flux de données, vous pouvez le gérer (par exemple, configurer ses processus et surveiller son statut). Pour plus d'informations sur la gestion des flux de données, reportez-vous à la rubrique Gestion des flux de données.

Si vous avez installé un module facultatif (par exemple, le module Content Server Spider), vous pouvez ajouter les processus de flux de données correspondants à un gestionnaire de flux de données. Pour plus d'informations sur l'ajout des processus de flux de données fournis par un module facultatif, reportez-vous à la documentation qui accompagne ce module.
