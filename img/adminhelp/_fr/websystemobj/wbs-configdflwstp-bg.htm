Lorsque vous créez des index Content Server à l'aide de modèles d'index, Content Server génère automatiquement les objets système appropriés, y compris le flux de données et tous les processus correspondants. Dans ce cas, Content Server configure chaque processus de flux de données à l'aide des valeurs par défaut. Vous pouvez modifier ces valeurs par défaut pour personnaliser la configuration des processus. La modification des options d'arrêt permet de personnaliser le moment de l'arrêt des processus de flux de données.

Vous pouvez configurer les options d'arrêt dans l'onglet Infos spécifiques de la page Propriétés de chaque processus de flux de données afin de déterminer les événements qui se produisent lors de l'arrêt d'un processus.

Vous pouvez donner l'ordre à un processus de s'interrompre aussitôt qu'il s'arrête ou envoyer un message d'arrêt au processus lorsqu'il s'arrête. Si vous envoyez un message d'arrêt, vous devez spécifier un numéro de port disponible sur lequel le message est envoyé. Lorsque le processus reçoit le message d'arrêt, il effectue une série d'opérations pour s'arrêter progressivement. Vous pouvez également exécuter un fichier exécutable lors de l'arrêt d'un processus. Ce fichier exécutable (que vous devez créer) peut ensuite arrêter le processus progressivement.

Le tableau suivant décrit les options d'arrêt les plus appropriées pour les processus de flux de données.

Tableau 1-2 : Options d'arrêt par défaut

Type de processus Description de l'option d'arrêt par défaut
Extracteur Entreprise Configurez ce processus pour qu'il s'interrompe aussitôt qu'il s'arrête. Le processus Extracteur Entreprise ne prend pas en charge les options Port des messages d’arrêt ou Exécutable d'arrêt.
Directory Walker Configurez ce processus pour qu'il s'interrompe aussitôt qu'il s'arrête. Vous pouvez également le configurer pour qu'il s'arrête lorsqu'il reçoit un message d'arrêt en indiquant un numéro de port dans le champ Port des messages d’arrêt. Dans ce cas, Content Server envoie un message d'arrêt au processus Directory Walker, lui ordonnant de s'arrêter progressivement.
Conversion de documents Configurez ce processus pour qu'il s'interrompe à la réception d'un message d'arrêt en indiquant un numéro de port dans le champ Port des messages d’arrêt. Dans ce cas, le serveur d'administration envoie un message d'arrêt au processus Conversion de documents, lui ordonnant de s'arrêter progressivement. Bien que vous puissiez configurer également ce processus pour qu'il s'interrompe aussitôt qu'il s'arrête, OpenText vous recommande de ne pas le faire.
Distributeur de mise à jour Configurez ce processus pour qu'il s'interrompe à la réception d'un message d'arrêt en indiquant un numéro de port dans le champ Port des messages d’arrêt. Dans ce cas, Content Server envoie un message d'arrêt au processus Distributeur de mise à jour, lui ordonnant de s'arrêter progressivement. Le processus Distributeur de mise à jour ne prend pas en charge les options Terminer ou Exécutable d'arrêt.
Producteur XML Activator Configurez ce processus pour qu'il s'interrompe à la réception d'un message d'arrêt en indiquant un numéro de port dans le champ Port des messages d’arrêt. Dans ce cas, Content Server envoie un message d'arrêt au processus Producteur XML Activator, lui ordonnant de s'arrêter progressivement. Vous pouvez également configurer ce processus pour qu'il s'interrompe aussitôt qu'il s'arrête ou lorsqu'un fichier exécutable spécifique est exécuté.
