Le volume des objets système est un volume Content Server spécial contenant les objets système qui contrôlent les fonctionnalités d'indexation et de recherche de Content Server. Le volume des objets système vous permet de créer et de tenir à jour les serveurs d'administration et les objets source de données qu'ils gèrent. Vous pouvez ajouter et configurer des serveurs d'administration et créer ou modifier de nombreux types d'index. Pour plus d'informations sur l'administration de serveurs d'administration, reportez-vous à la rubrique Resynchronisation des serveurs d'administration.

Le volume des objets système contient une hiérarchie de dossiers. Le niveau supérieur, aussi appelé « racine », s'affiche sur la page Administration système. La page Administration système contient deux tables : la table Serveurs d'administration et la table Vue détaillée.

Le tableau suivant décrit les informations contenues dans la table Serveurs d'administration de la page Administration système :

Nom de la colonne Description
Par défaut Identifie le raccourci du serveur d'administration affiché par défaut dans les listes déroulantes Serveur d'administration de l'onglet Infos spécifiques de la page Propriétés d'un processus de flux de données, stockées dans le volume des objets système. Une coche s'affiche en regard du raccourci du serveur d'administration par défaut.
Nom Indique le raccourci de chaque serveur d'administration que vous configurez dans Content Server.
Fonctions Contient l'icône Fonctions, qui permet d'accéder aux fonctions associées à un serveur d'administration.
Etat
Affiche le statut actuel de chaque serveur d'administration dans le système. Le statut d'un serveur d'administration peut prendre les valeurs Actif, Non disponible ou Mode sans échec.
Pour plus d'informations sur le mode sans échec, voir Maintenance des serveurs d'administration.
Processus Affiche le nombre de processus gérés par chaque serveur d'administration.
Exécution Affiche le nombre de processus en cours d'exécution, gérés par chaque serveur d'administration. Si l'état du serveur d'administration ne permet pas d'obtenir ces informations, le statut Non disponible s'affiche dans cette colonne.
Erreurs Affiche le nombre de processus ayant rencontré des erreurs et gérés par chaque serveur d'administration. Si l'état du serveur d'administration ne permet pas d'obtenir ces informations, le statut Non disponible s'affiche dans cette colonne.
Statut du DCS Identifie (à l'aide d'une icône) le statut du DCS. Si aucun DCS n'est présent, une icône indiquant qu'aucun serveur DCS n'existe s'affiche dans cette colonne. Vous pouvez cliquer sur cette icône pour accéder à la page Infos spécifiques associée au DCS.

La table Vue détaillée contient un dossier source de données pour chaque index créé dans Content Server, un dossier Vues de recherche et un dossier Modèles de recherche personnels. Les dossiers source de données contiennent tous les objets système associés à un index particulier. Les objets système incluent les éléments suivants : gestionnaires de flux de données, gestionnaires de recherche, gestionnaires de sauvegarde, processus de flux de données, partitions, moteurs de recherche, fédérateurs de recherche et moteurs d'indexation. Le dossier Vues de recherche contient les vues de recherche créées automatiquement par Content Server pour chaque index. Ce dossier contient également toute autre vue de recherche que vous créez. Le dossier Modèles de recherche personnels contient tous les modèles utilisateur et le modèle système par défaut qui vous permettent de configurer la présentation de la page Recherche.

Le tableau suivant décrit les informations contenues dans la table Vue détaillée de la page Administration système :

Nom de la colonne Description
Type Contient une icône identifiant chaque type d'objet source de données dans la table.
Nom Spécifie le nom de chaque objet source de données.
Fonctions Contient l'icône Fonctions, qui permet d'accéder aux fonctions associées à chaque objet source de données.
Taille Spécifie le nombre d'éléments contenus dans chaque objet source de données.
Modifié Spécifie la date et l'heure de la dernière modification apportée à chaque objet source de données.
Etat Affiche le statut actuel de chaque objet source de données dans le système, le cas échéant. Les valeurs affichées dans cette colonne varient selon chaque type d'objet source de données.
