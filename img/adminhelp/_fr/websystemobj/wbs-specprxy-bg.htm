Un proxy agit comme un emplacement réservé destiné à un processus au sein d'un flux de données. Ajoutez un proxy pour représenter un processus qu'aucun serveur d'administration ne gère. Par exemple, si vous créez un processus personnalisé non-Content Server (qui crée, manipule ou consomme des données) qui sera utilisé dans un flux de données Content Server, vous pouvez représenter ce processus dans Content Server au moyen d'un proxy. Configurez les proxies sur leurs pages Propriétés.

Tableau 1-3 : Onglet Infos spécifiques de la page Propriétés du processus proxy

Elément Description
Serveur d'administration Cet élément spécifie le raccourci du serveur d'administration sur l'hôte duquel le processus proxy s'exécute. Il s'agit du serveur d'administration qui contrôle également les autres composants présents dans le flux de données (par exemple, les répertoires de lecture et d'écriture que vous spécifiez). Ne modifiez pas l'hôte du serveur d'administration sauf si cela est nécessaire (par exemple, si l'hôte du serveur d'administration n'est plus disponible). Si vous modifiez le serveur d'administration sur lequel le processus proxy s'exécute, vous devez également modifier les répertoires de lecture et d'écriture associés au proxy.
Bouton Informations Charge la page Gestion des liens du flux de données, qui spécifie le répertoire à partir duquel le processus de proxy lit les données et le répertoire vers lequel le processus de proxy écrit les données.
Mettre à jour Permet de soumettre au serveur Content Server les modifications que vous avez apportées à cette page.
Réinitialiser Rétablit les informations de cette page à l'état qui était le leur à l'ouverture de la page.
