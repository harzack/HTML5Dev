Une image de flux de données offre un moyen alternatif d'afficher les processus de flux de données et les iPools dans un flux de données. Plutôt que d'afficher dans l'ordre alphabétique des listes organisées des processus de flux de données et des IPools dans un flux de données, les images de flux de données affichent des représentations graphiques de la manière dont les processus sont reliés entre eux par des IPools au sein des flux de données. Les liens dans les images de flux de données indiquent la manière dont les IPools relient un processus de flux de données au suivant et dont les messages IPool circulent dans le flux de données.

Les images de flux de données peuvent représenter des processus de flux de données liés séquentiellement ou en parallèle. L'illustration ci-dessous offre un exemple d'image de flux de données pour un flux de données Entreprise standard, dans lequel les messages IPool passent de façon séquentielle du processus Extracteur Entreprise au processus Distributeur de mise à jour, puis en parallèle aux processus Importateur Entreprise Prospecteur et Importateur Entreprise Classement. Dans cet exemple, les mêmes messages iPool sont transmis simultanément au processus Importateur Entreprise Prospecteur et au processus Importateur Entreprise Classement.

Les images de flux de données permettent également de gérer les liens entre les processus de flux de données. Vous pouvez ajouter, supprimer ou afficher des informations sur les liens.

Ajout et suppression de liens

Vous pouvez utiliser une image de flux de données pour relier les processus dans un flux de données. Lors de l'ajout d'un nouveau processus à un flux de données, vous pouvez configurer son lien sur la page utilisée pour créer le nouveau processus ou à l'aide d'une image de flux de données. Les processus qui ne sont pas liés lorsque vous les ajoutez à un flux de données sont affichés sans connexion dans la partie supérieure de l'image du flux de données. La suppression d'un lien entraîne la suppression des connexions IPool entre deux processus de flux de données. Vous pouvez reconnecter les processus de flux de données à tout moment en ajoutant un nouveau lien.

Affichage d'informations sur les liens

Lorsque vous affichez des informations sur un lien, vous pouvez obtenir des informations sur les connexions IPool dans une image de flux de données. Par exemple, vous pouvez afficher le nombre de messages IPool qui ont bien été transmis ou qui attendent d'être transmis dans les processus d'un flux de données. Vous pouvez également surveiller la taille du répertoire, qui correspond à l'espace disque occupé par la sous-zone IPool, et l'espace disque disponible, qui correspond à l'espace disponible sur l'ordinateur occupé par la sous-zone IPool.

 

Note :
La sous-zone IPool stocke les IPools de lecture et d'écriture des processus de flux de données. Deux processus de flux de données partagent chaque sous-zone IPool : un processus lit les données dans la sous-zone pendant que l'autre processus enregistre les données dans la zone.
