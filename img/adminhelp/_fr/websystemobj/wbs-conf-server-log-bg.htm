Content Server vous permet d'enregistrer les activités du serveur (llserver), du programme CGI Content Server (Content Server), du Servlet Content Server et du processus Extracteur Entreprise (indexupdate). Lorsque le journal de débogage est activé, Content Server copie les fichiers associés dans le répertoire Content Server/logs. Cette fonction est désactivée par défaut.

Vous pouvez définir le niveau de débogage dans la liste déroulante Niveau de débogage Content Server de la page Configurer les paramètres du débogage. Lorsque vous modifiez ce paramètre, vous en modifiez également la valeur dans la liste déroulante Journal Content Server de la page Configurer les paramètres du serveur. Dans la liste déroulante Niveau de débogage Content Server, 0, 1, 2 et 11 correspondent respectivement aux options Pas de journalisation, Journal du thread, Journal détaillé du thread et Journal de CGI et du thread dans la liste déroulante Journal Content Server.

Le tableau suivant décrit chaque niveau du journal de débogage :

Tableau 1-4 : Niveaux du journal de débogage

       
Niveau de consignation Fichiers journaux générés Description  
1   llserver.out Généré à chaque redémarrage du serveur.
sockserv1.out Généré à chaque redémarrage du serveur.    
threadn.out Un fichier est généré pour chaque thread. Ce fichier, mis à jour en continu, enregistre les données envoyées sur ses threads par le serveur. Le contenu de ces fichiers dépend en partie des options d'enregistrement de thread.    
2 Niveau 1 + receivern.out Un fichier est généré pour chaque thread pour enregistrer les activités des threads réceptrices.
11 Niveau 2 + llclientnnn.out Un fichier est généré chaque fois que le navigateur Web d'un utilisateur final envoie une requête au programme Content Server CGI. Le numéro nnn est généré par Content Server.
servletclientnnn.out Un fichier est généré pour chaque numéro de thread (déterminé par la machine virtuelle Java de votre recueil Servlet). Chaque fois que le navigateur Web d'un utilisateur final envoie une requête au Servlet Content Server, un nouveau fichier servletclientnnn.out est créé. Si un fichier portant ce numéro (nnn) existe déjà, les informations de la requête sont ajoutées à la fin du fichier existant.    
llindexupdatennn.out Un fichier est généré à chaque lancement du processus Extracteur Entreprise (toutes les minutes). Le numéro nnn est généré par Content Server.    
indexupdateOutnnn.out Un fichier est généré à chaque arrêt du processus Extracteur Entreprise (toutes les minutes). Le numéro nnn est généré par Content Server.    
0   Aucun Aucun fichier de niveau 1, 2 ou 11 n'est généré.

Vous pouvez définir des valeurs sur la page Configurer les paramètres du débogage afin d'activer ou de désactiver la journalisation pour le serveur Content Server et de configurer les options d'enregistrement de thread.

Le nombre de fichiers threadn.out et receivern.out est égal au nombre de threads sur lesquels est exécuté le serveur. Si vous diagnostiquez un problème, vous pouvez temporairement définir le serveur pour qu'il soit exécuté sur un seul thread. Cela vous permettra de rechercher des informations de diagnostic dans un seul fichier. L'exécution sur un seul thread réduisant énormément les performances du serveur, OpenText vous recommande d'opter pour ce type d'exécution pendant les périodes de faible utilisation et de rétablir les paramètres de thread d'origine du serveur après avoir effectué votre diagnostic.

 

Attention
Dans Microsoft Windows Server 2008 et les versions ultérieures, les fichiers journaux écrits par Content Server peuvent être de zéro octet jusqu'à ce que les services Content Server soient redémarrés.
