Onglet Infos spécifiques de la page Propriétés du moteur d'indexation

Elément Description
Etat Cet élément spécifie si le moteur d'indexation est ou non en cours d'exécution. Vous ne pouvez pas modifier ce champ.
Hôte Cet élément spécifie le serveur d'administration sur lequel est exécuté le moteur d'indexation.
Activer la gestion du système Détermine si Content Server contrôle ce processus pour détecter le retour de messages d'erreur. S'il retourne un message d'erreur, Content Server l'enregistre dans la base de données. La gestion du système est activée par défaut. Pour plus d'informations sur la configuration de Content Server en vue de l'envoi d'e-mails d'alerte lorsqu'un processus de flux de données rencontre des erreurs, reportez-vous à la rubrique Pour activer la recherche d'erreurs et l'envoi d'e-mails.
Valeur maximale du code de sortie correcte Lorsqu'un processus rencontre une erreur, il renvoie un code d'erreur. Si le numéro du code d'erreur est inférieur au numéro inscrit dans le champ Valeur maximale du code de sortie correcte, le processus tente de redémarrer. Si le numéro du code d'erreur est supérieur au numéro inscrit dans le champ Valeur maximale du code de sortie correcte, le processus ne tente pas de redémarrer et devra être redémarré manuellement. OpenText recommande de ne pas modifier la Valeur maximale du code de sortie correcte sauf si le service d'assistance clientèle d'OpenText vous le demande.
Port Admin Cet élément spécifie le port sur lequel est exécuté le moteur d'indexation. Le serveur d'administration du processus utilise ce port pour le démarrer et l'arrêter. Cliquer sur le lien Vérifier le port pour vérifier si le numéro de port que vous avez spécifié est disponible.
Port du serveur Cet élément spécifie le port utilisé par les processus d'échange pour communiquer avec le moteur d'indexation. Vous pouvez vérifier la disponibilité du numéro de port spécifié en cliquant sur le lien Vérifier le port.
Répertoire d'indexation Cet élément spécifie le répertoire de l'index de partition auquel appartient le moteur d'indexation.
Partition Cliquez sur le lien du nom de la partition pour configurer la partition à laquelle appartient le moteur d'indexation.
Fichier journal Cet élément spécifie l'emplacement du fichier journal du moteur d'indexation. Le fichier journal enregistre des informations sur le processus et peut être utile lors d'un dépannage.
Actions du fichier journal Cet élément spécifie la manière dont le fichier journal du moteur d'indexation est affecté lorsque le processus redémarre :
•

Ajouter à l'élément existant : ajoute toutes les nouvelles informations à la fin du fichier journal actuel.

•

Créer : remplace le fichier journal précédent.

•

Créer (Enregistrer l'élément existant) : enregistre et renomme le fichier journal existant et crée un nouveau fichier journal sur le modèle du fichier journal existant.

•

Niveau d'avertissement : enregistre les messages d'avertissement et tous les messages d'erreur.

•

Niveau d'erreur : enregistre les messages d'erreur types et graves.

•

Niveau d'erreur grave : enregistre les messages d'erreur graves.

Niveau de débogage Cet élément spécifie le type de message qui sera enregistré dans le fichier journal du moteur d'indexation :
•

Niveau d'informations : enregistre tous les types de messages.

•

Niveau de statut : enregistre les messages de statut périodiques, les messages d'avertissement et tous les messages d'erreur.

Intervalle de vidage du journal Spécifie le nombre de nouveaux messages enregistrés dans le journal du moteur d'indexation avant que le processus le copie sur le disque.
Démarrer/Arrêter Lance ou arrête le moteur d'indexation ou tous les processus d'indexation (le processus Distributeur de mise à jour et les moteurs d'indexation qui appartiennent à une source de données). Que vous arrêtiez tous les processus d'indexation ou un moteur d'indexation individuel, l'indexation s'arrête. Si les processus d'indexation ne sont pas en cours d'exécution, les boutons Démarrer s'affichent ; si les processus d'indexation sont en cours d'exécution ou s'ils sont programmés, les boutons Arrêter s'affichent.
Redémarrer Cet élément arrête puis redémarre tous les processus d'indexation appartenant à une source de données.
Resynchroniser Cet élément resynchronise les informations sur le moteur d'indexation avec les informations présentes dans la base de données Content Server.
Générer le résumé du fichier journal de l'index Cet élément génère un résumé des événements enregistrés dans le fichier journal du moteur d'indexation. Le résumé du fichier journal contient les statistiques relatives à l'index de la partition à laquelle appartient le moteur d'indexation. Pour plus d'informations sur les données incluses dans les résumés de fichier journal d'un index, contactez le service d'assistance clientèle d'OpenText.
Valider l'index Cet élément identifie les problèmes potentiels rencontrés avec l'index d'une source de données en analysant chaque portion de l'index pour voir si les structures de données sont synchronisées.
Mettre à jour Permet de soumettre au serveur les modifications que vous avez apportées à cette page.
Réinitialiser Rétablit les informations de cette page à l'état qui était le leur à l'ouverture de la page.
