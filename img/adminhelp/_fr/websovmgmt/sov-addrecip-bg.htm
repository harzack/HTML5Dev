Content Server peut envoyer cinq types d'e-mails d'alerte relatifs aux objets système pour consigner le statut des objets présents dans le volume de l'objet système :

  • Erreurs graves, qui informe le destinataire des erreurs système critiques, comme l'arrêt d'un moteur de recherche. Les erreurs graves affectent immédiatement la capacité d'un utilisateur Content Server à effectuer certaines opérations dans Content Server.

  • Erreurs, qui prévient le destinataire des erreurs système générales, comme l'arrêt d'un processus de flux de données. la plupart des erreurs ne sont pas immédiatement critiques, mais elles peuvent devenir graves après un certain temps ;

  • Avertissements, qui prévient le destinataire des erreurs non critiques dans le système, comme un flux de données ne répondant plus aux messages Ipool. les avertissements informent les destinataires des erreurs potentielles ;

  • Informations, qui notifie le destinataire du statut général et d'autres informations système.

  • Corrections, qui prévient le destinataire qu'une erreur précédemment notifiée a été rectifiée. Par exemple, si vous configurez Content Server de manière qu'il envoie les erreurs graves à trois administrateurs de votre site Content Server, Content Server peut également envoyer un message de correction une fois que l'un des administrateurs a rectifié l'erreur.

C'est à vous de déterminer les types de messages reçus par chaque destinataire lors de la configuration de l'envoi des e-mails d'alerte relatifs aux objets système. Les e-mails d'alerte relatifs aux objets système sont également associés aux règles de contrôle de flux de données. Pour en savoir plus, voir Ajout de règles de contrôle de flux de données.

Après avoir configuré les alertes relatives aux objets système, les paramètres d'envoi d'e-mails et le serveur de messagerie que Content Server utilise pour envoyer des e-mails d'alerte relatifs aux objets système, vous devez identifier les destinataires de l'e-mail. Vous pouvez configurer Content Server afin qu'il envoie des e-mails d'alerte relatifs aux objets système aux utilisateurs Content Server, à l'administrateur Content Server ou à tout autre compte de messagerie valable. Si vous n'avez pas spécifié d'adresse e-mail pour l'administrateur Content Server sur votre site Content Server, vous ne pouvez pas configurer Content Server afin qu'il envoie des e-mails d'alerte relatifs aux objets système à l'administrateur Content Server.

Par défaut, l'administrateur Content Server et les utilisateurs Admin sont répertoriés dans la section Destinataires de la page Configurer l'envoi d'e-mails d'alerte relatifs aux objets système. Par défaut, ces utilisateurs reçoivent des e-mails d'alerte SOV.

Vous ajoutez des destinataires d'e-mails en spécifiant leur adresse e-mail et en configurant l'ajout de destinataires d'e-mails pour chaque destinataire. Après avoir configuré un destinataire d'e-mails, vous pouvez vérifier la configuration en envoyant un message de test.
