La vérification d'erreurs permet à Content Server de surveiller le statut des objets stockés dans le volume des objets système. Lors de l'activation de la recherche d'erreurs, vous configurez également Content Server pour envoyer des e-mails d'alerte qui consignent le statut des objets dans le volume des objets système. La recherche automatique d'erreurs et l'envoi des e-mails vous permettent de connaître le statut général, les erreurs potentielles ou les problèmes rencontrés par les objets système lors des processus. Vous pouvez configurer Content Server afin d'envoyer des e-mails d'alerte aux utilisateurs Content Server, à l'administrateur Content Server ou à tout autre compte de messagerie.

Activez la recherche d'erreurs si vous voulez que Content Server surveille le statut des objets dans le volume des objets système. Content Server enregistre le statut ou la condition des objets système et envoie ces informations aux Ajout de destinataires d'e-mails des e-mails d'alerte relatifs aux objets système.

 

Note :
Si vous activez la recherche d'erreurs mais que vous ne configurez pas les destinataires des e-mails et les messages, Content Server enregistre les erreurs mais n'avertit personne de leur existence.

 

Pour activer la recherche d'erreurs et l'envoi des e-mails d'alerte relatifs aux objets système, procédez comme suit :

  • Activez Notification Content Server pour que les informations de la page Configurer la notification (par exemple, les informations relatives à l'URL de la page d'accueil Content Server et à la base de données Content Server par défaut) soient accessibles dans les e-mails d'alerte relatifs aux objets système.

  • Activez la recherche d'erreurs.

  • Configurez les destinataires des e-mails et les types de messages.

Désactivez la recherche d'erreurs si vous ne voulez pas que Content Server surveille le statut des objets dans le volume des objets système. Pour désactiver la recherche d'erreurs pour un processus de flux de données ou un moteur de recherche particulier, arrêtez le processus ou le moteur de recherche, puis désactivez la case à cocher Activer la gestion du système correspondante. Par défaut, la recherche d'erreurs est activée. Content Server surveille alors le statut de tous les objets du volume des objets système à l'exception de ceux pour lesquels la gestion du système est désactivée. Par défaut, la gestion du système est activée pour tous les objets système.
