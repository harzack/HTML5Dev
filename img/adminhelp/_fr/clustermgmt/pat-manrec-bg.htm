Lorsque vous installez un correctif, l'agent de cluster crée un répertoire de sauvegarde dans le répertoire <OTACCUEIL>/clustermanagement/recovery. Le fichier recovery_manifest.xml contient des informations sur le déploiement des correctifs, l'étiquetage des entrées, y compris :

  • <supprimer/>, un fichier à supprimer.

  • <Archive_Fichiers_restauration/>, une archive d'un répertoire modifié dans le répertoire <OTACCUEIL> Content Server. Remplace le répertoire modifié par la copie archivée pour supprimer les changements du déploiement de correctifs.

  • <impacts/>, affiche les modifications apportées à Content Server, et répertorie les étapes requises pour effectuer une restauration d'un déploiement de correctifs.

 

Note :
Pour plus d'informations sur la restauration d'un déploiement de correctifs, voir le fichier recovery_instruction.txt dans le répertoire <OTACCUEIL>/clustermanagement/recovery/.
