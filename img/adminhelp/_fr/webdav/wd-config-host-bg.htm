Afin d'équilibrer les charges et d'assurer la sécurité, les utilisateurs installent parfois des serveurs intermédiaires entre le client et le serveur Web exécutant WebDAV, y compris des proxy inversés et des répartiteurs de charge. Lors de la communication entre le serveur Web et le serveur intermédiaire, l'URL utilisée par le client est masquée. Toutes les informations relatives à l'URL renvoyées au client concernent alors l'URL du serveur intermédiaire et sont donc erronées. Pour remédier à cette erreur de transmission des informations, vous pouvez configurer le serveur Web de sorte que l'URL des serveurs intermédiaires soit remplacée par celle utilisée par le client.

Le protocole est lui aussi masqué, de sorte que les informations transmises au client sont également erronées. Ici encore, vous pouvez spécifier le protocole correct. Les protocoles utilisent en général le schéma http, utilisé par défaut, ou https.

Dans l'exemple suivant, les paramètres convertissent une adresse URL incorrecte http://nom_hôte1:8080/livelinkdav en URL correcte https://nom_hôte2/livelinkdav.

Nom d'hôte du proxy = nom_hôte1:8080Hôte défini = nom_hôte2Schéma défini = https
