Pour accéder à WebDAV, le serveur Web d'accès doit être configuré avec des modules d'extension.

Si le serveur Web est local, en d'autres termes s'il se trouve sur le même ordinateur que le serveur Content Server, vous devez le configurer. Si le serveur Web est distant, c'est-à-dire qu'il ne se trouve pas sur le même ordinateur que celui hébergeant Content Server, vous pouvez configurer Microsoft Internet Information Server (IIS) en tant que serveur Web distant ou vous pouvez configurer un autre serveur Web pour Windows ou UNIX/Linux.

 

Important :
Avant de pouvoir configurer un serveur Web distant, vous devez spécifier un numéro de port statique pour le port du serveur coopérant Weblink. Pour plus d'informations sur la définition d'un numéro de port, reportez-vous à la rubrique Spécification du port du serveur coopérant Weblink.

 

Configuration d'un serveur Web lors de l'installation ou de la mise à jour de WebDAV

Lorsque vous ajoutez le module WebDAV à Content Server, vous devez configurer le répertoire virtuel qu'un serveur Web utilisera pour communiquer avec WebDAV. Vous êtes ensuite invité à configurer un serveur Web local. Pour configurer un serveur Microsoft Internet Information Server (IIS) en tant que serveur Web local, vous devez exécuter ou enregistrer un programme d'installation ; pour configurer un serveur Web local non-IIS, vous pouvez télécharger un fichier contenant une description des modifications requises pour le serveur Web spécifié.
