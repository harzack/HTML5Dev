Lorsqu'un dossier contient des catégories nécessitant des attributs, l'interface utilisateur de Content Server oblige les utilisateurs à spécifier les valeurs de ces attributs lors de l'ajout de documents audit dossier. WebDAV ne permet pas l'affichage des interfaces utilisateurs requises pour cette fonctionnalité, l'un des comportements non interactifs suivants doit être utilisé :

  • Si l'installation de Content Server inclut le module Staging personnel, vous pouvez choisir de créer un document dans un dossier Staging personnel. Pour plus d'informations sur le module Staging personnel, voir les notes de publication de OpenText WebDAV.

  • Vous pouvez choisir de créer le document sans spécifier les attributs requis. Vous pouvez au besoin les préciser ultérieurement via l'interface utilisateur de Content Server. Il s'agit de l'option la plus pratique pour l'utilisateur, bien qu'elle vienne à l'encontre de l'obligation de spécifier les attributs dans ces dossiers.

  • Vous pouvez choisir d'autoriser la création des documents tout en supprimant les catégories contenant ces attributs. Cette solution va dans le sens des capacités de l'interface utilisateur de Content Server mais complique le processus de spécification des attributs pour l'utilisateur qui doit retrouver et ajouter les catégories adéquates.

  • Vous pouvez désactiver la création de documents dans les dossiers à l'aide de WebDAV. Avec cette solution, les clients WebDAV échouent et affichent une erreur lorsque l'utilisateur essaie de créer un document, mais les documents existants peuvent toujours être modifiés.

WebDAV vous permet d'omettre certains éléments répertoriés dans le dossier WebDAV pour les utilisateurs ne disposant pas de droits d'administration système, en fonction des besoins et des attentes de votre communauté d'utilisateurs. Vous pouvez omettre des éléments masqués ainsi que des catégories et des volumes de rapport.

Les éléments désignés comme masqués omettent les éléments masqués des listes, de la même façon qu'ils sont omis des pages de navigation de Content Server. Cette option est désactivée par défaut, ce qui permet de créer et modifier facilement les éléments masqués au moyen des clients WebDAV.

Les catégories et les volumes de rapports masqués suppriment ces éléments du dossier racine WebDAV. Ces emplacements sont rarement utilisés par les utilisateurs et la modification WebDAV est généralement déconseillée pour ces objets ; l'option est aussi activée par défaut afin de réduire la complexité des affichages utilisateur.

Si vous masquez les éléments Content Server, aucun utilisateur ne peut les voir dans l'affichage du dossier WebDAV.
