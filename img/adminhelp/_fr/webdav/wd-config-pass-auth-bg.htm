Vous pouvez utiliser l'authentification Passthrough sans le module Directory Services dans les configurations nécessitant l'authentification à identification unique Single Sign-On (SSO) avec le module Content Server Directory Services et l'authentification Kerberos, mais que ces méthodes ne peuvent pas être utilisées pour l'une des raisons suivantes :

  • Le module Directory Services n'est pas installé sur le serveur Content Server (requis pour l'authentification utilisant le module Content Server Directory Services).

  • Le serveur Web utilisé est IIS (déconseillé avec l'authentification Kerberos).

Pour utiliser la méthode d'authentification Passthrough, votre serveur Web doit être configuré de manière à authentifier les utilisateurs sur le répertoire virtuel WebDAV.

 

Note :
Pour des informations détaillées sur la configuration de l'authentification Passthrough, voir « Configuring Passthrough Authentication (Configuration de l'authentification Passthrough)  » dans OpenText WebDAV Installation and Administration Guide (Manuel d'installation et d'administration de OpenText WebDAV).
