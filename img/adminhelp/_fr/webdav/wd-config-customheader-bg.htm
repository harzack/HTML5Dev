L'authentification d'en-tête personnalisé peut souvent être employée lorsque le serveur Web est configuré avec un logiciel tiers pour authentifier les utilisateurs. Lorsqu'un tel produit impose une exigence d'authentification et transmet le nom de l'utilisateur authentifié dans un en-tête HTTP, WebDAV peut être configuré pour accepter le nom d'utilisateur à partir de cet en-tête.

Pour configurer cette méthode d'authentification, le nom de l'en-tête HTTP doit être fourni. Notez que la documentation de votre produit d'authentification tiers peut donner le nom de la variable ISAPI ou CGI équivalente, par exemple « UN_EN-TETE » ou « HTTP_UN_EN-TETE », au lieu du nom de l'en-tête HTTP lui-même, « Un-en-tête ». Cette méthode d'authentification doit être configurée avec le nom d'en-tête réel, qui peut inclure des tirets, mais pas de traits de soulignement, et ne commence pas par « HTTP ».

 

Attention
Des en-têtes HTTP arbitraires pouvant être inclus dans des requêtes Web, cette méthode d'authentification ne doit être utilisée qu'en association avec un produit d'authentification tiers imposant une authentification et contrôlant l'en-tête nommé. Autoriser les requêtes Web à spécifier arbitrairement la valeur de l'en-tête utilisé pour l'authentification permettrait d'usurper sans limite l'identité de tout utilisateur Content Server.
