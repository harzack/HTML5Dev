Méthode d'authentification

WebDAV prend en charge les méthodes d'authentification suivantes :

  • Configuration basique/Authentification Content Server pour authentifier les utilisateurs avec leurs références Content Server ou Configuration de l'authentification à l'aide du module Livelink Directory Services ;

  • Configuration de l'authentification Passthrough pour que le serveur Web authentifie les utilisateurs, que l'authentification des services d'annuaire soit ou non configurée pour Content Server ;

  • Configuration de l'authentification Kerberos pour authentifier les utilisateurs à l'aide de Kerberos et d'un serveur Kerberos spécifique ;

  • Configuration de l'authentification d'en-tête personnalisé pour effectuer l'authentification avec un en-tête HTTP personnalisé, généralement fourni par un produit d'authentification tiers ; et

  • Configuration de l'authentification personnalisée pour spécifier un module d'extension d'authentification personnalisée.

Les options de configuration suivantes sont disponibles pour toutes les méthodes :

Domaines autorisés

Si le serveur Web authentifie les utilisateurs par rapport à un domaine, vous devez configurer WebDAV de manière à accepter les utilisateurs de ce domaine. Pour plus d'informations sur la configuration de WebDAV de manière à autoriser les utilisateurs de domaines spécifiés, reportez-vous à la rubrique Gestion des domaines autorisés.

Mappage utilisateur

Lorsqu'un serveur Web authentifie un utilisateur, WebDAV le mappe sur un utilisateur Content Server spécifique. Plusieurs méthodes permettent de mapper des noms principaux dans WebDAV (utilisateurs authentifiés) à des utilisateurs Content Server. Pour plus d'informations sur ces méthodes de mappage, reportez-vous à la rubrique Gestion de mappages utilisateurs.
