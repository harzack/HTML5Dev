L'authentification Kerberos correspond à la norme MIT V5 Kerberos. Il s'agit de la méthode d'authentification réseau par défaut sur les systèmes Windows 2000 ou ultérieurs. Elle est également utilisée sur les hôtes UNIX.

Là où l'authentification Basique/Content Server demande que le client envoie un nom d'utilisateur et un mot de passe dans sa demande HTTP vers un serveur cible, un client utilisant l'authentification Kerberos envoie des jetons dans le cadre de sa demande au serveur cible. Le client communique avec un intermédiaire (le serveur Kerberos) pour obtenir ses jetons et le serveur cible communique avec le serveur Kerberos pour les vérifier. Le service d'annuaire Active Directory Windows 2000, 2003 ou ultérieur (contrôleur de domaine Windows) ou un autre serveur Kerberos (par exemple, sur un hôte UNIX) peut contenir le serveur Kerberos.

 

Notes :

  • La configuration Kerberos nécessite que le module WebDAV possède un compte de service dans Active Directory, ce qui fait du service WebDAV le destinataire des demandes HTTP envoyées à son serveur. Sur les serveurs Windows, le compte d'ordinateur du serveur représente généralement ce compte de service.?Il faut toutefois savoir qu'IIS utilise également ce compte pour effectuer sa propre authentification. C'est pourquoi OpenText déconseille d'utiliser l'authentification WebDAV Kerberos sur les serveurs Windows intégrant IIS. Il est préférable d'utiliser le module Directory Services.

  • Le module WebDAV ne prend pas en charge l'authentification Kerberos pour les configurations avec BEA WebLogic Server, Apache Tomcat Server ou Sun Java System Application Server. OpenText a déterminé que ce serveur, utilisé dans ce type de configuration, est défaillant dans l'une de ses fonctions principales.

 

Avec l'authentification Kerberos, les clients Microsoft tels qu'Internet Explorer peuvent accéder aux ressources WebDAV disposant de la fonction d'authentification unique (SSO, Single Sign-On). SSO réduit la fréquence à laquelle les utilisateurs doivent fournir leur nom d'utilisateur et leur mot de passe.

Sur les clients Microsoft, SSO nécessite la configuration suivante :

  • Les clients doivent s'exécuter sous Windows 2000 Professionnel SP2 ou version ultérieure.

  • Internet Explorer doit être correctement configuré.

  • Tous les utilisateurs doivent être membres d'un domaine Active Directory.

  • WebDAV doit disposer d'un compte de service approprié dans ce même domaine ou dans un domaine accrédité.

Pour configurer l'authentification des utilisateurs avec Kerberos dans WebDAV, vous devez :

  • configurer le serveur Kerberos en créant un compte de service ;

  • configurer Content Server ;

  • configurer Internet Explorer.

 

Note :
Pour des informations détaillées sur la configuration de l'authentification Kerberos, voir « Configuring Kerberos Authentication (Configuration de l'authentification Kerberos) » dans OpenText WebDAV Installation and Administration Guide (Manuel d'installation et d'administration de OpenText WebDAV).
