Chaque fois qu'un serveur Web authentifie un utilisateur par rapport à un domaine, l'utilisateur est généralement identifié par un nom principal (par exemple, utilisateur@LEDOMAINE.NET). Le nom principal comprend généralement un domaine qui identifie, entre autres, la personne ou l'organisation offrant la garantie de l'identité de l'utilisateur. Dans cet exemple, le nom principal inclut le domaine "LEDOMAINE".

WebDAV peut être configuré de deux manières pour accepter (autoriser) les noms principaux d'après leur domaine.

Autoriser tous les utilisateurs authentifiés

Les noms principaux de tous les domaines sont acceptés.

Accepter les utilisateurs de domaines spécifiés

Les noms principaux d'une liste de domaines spécifiés sont acceptés.
