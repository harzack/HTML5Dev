Lorsqu'un utilisateur se connecte à WebDAV avec un nom principal (par exemple, utilisateur@LEDOMAINE.NET) authentifié avec succès, WebDAV met en correspondance le nom principal et l'utilisateur Content Server qu'il représente. Vous pouvez configurer WebDAV pour utiliser l'une des trois méthodes de mappage : Nom identique, Domaine et nom et Géré par l'utilisateur.

Mappage Nom identique

Avec la méthode de mappage Nom identique, le nom principal mappe sur un nom d'utilisateur Content Server identique au nom d'ouverture de session de domaine du nom principal. Par exemple, le nom principal utilisateur@LEDOMAINE.NET est mappé sur l'utilisateur Content Server utilisateur.

Mappage Domaine et nom

Avec la méthode de mappage Domaine et nom, le nom principal mappe sur un utilisateur de domaine Content Server. Le nom d'utilisateur Content Server est identique au nom d'ouverture de session de domaine du nom principal. Par exemple, le nom principal utilisateur@LEDOMAINE.NET est mappé sur l'utilisateur Content Server utilisateur du domaine Content Server LEDOMAINE.

Mappage Géré par l'utilisateur

Avec la méthode de mappage Géré par l'utilisateur, le nom principal mappe sur un utilisateur Content Server lui-même défini par un utilisateur ou un administrateur. Aucun mappage n'est défini à l'origine, C'est à l'utilisateur ou à l'administrateur Content Server d'ajouter des mappages. Pour plus d'informations sur le mappage Géré par l'utilisateur, consultez OpenText WebDAV User Guide (Manuel de l'utilisateur de OpenText WebDAV).

Mappage des services d'annuaire

Quand le module Content Server Directory Services est installé et configuré correctement, WebDAV peut être configuré pour utiliser le mappage du module des services d'annuaire pour mapper un nom principal sur un utilisateur Content Server, ou l'une des méthodes de mappage identifiées ci-dessus.
