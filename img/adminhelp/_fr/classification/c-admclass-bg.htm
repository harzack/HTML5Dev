OpenText Classifications ajoute des fonctions de classement taxinomiques à Content Server, offrant une alternative à la façon habituelle d'organiser, de gérer et de parcourir les éléments Content Server. Vous pouvez classer des éléments Content Server, tels que les documents, les documents composites, les documents texte et les dossiers. Une fois que les éléments sont classés, les utilisateurs peuvent parcourir les Arbre de classement, en plus de la hiérarchie de dossiers Content Server classique.

Par exemple, si les éléments de votre espace de travail Entreprise sont organisés en fonction des différents services de votre entreprise (ressources humaines, marketing, etc.), l'ajout de classements vous permet de les organiser dans l'espace de travail Classement sur la base d'autres critères (tels que la date, le sujet et le type d'élément), outre le classement par service.

Ce sont les administrateurs qui créent les classements. Les éléments peuvent être affectés aux classements par les administrateurs et par les utilisateurs qui possèdent les autorisations appropriées. Si vous avez activé la fonctionnalité Classement intelligent du module, vous pouvez également proposer ou assigner des classements automatiquement. Pour proposer des classements ou les assigner automatiquement, la fonctionnalité de Classement intelligent se base sur des profils dans lesquels vous avez défini les critères auxquels doivent répondre les éléments.

La section Gestion des classements décrit les tâches suivantes :

  • Activation du Classement intelligent : permet un classement automatique ou assisté des éléments, basé sur des locutions clés générées.

  • Configuration des classements : permet de définir l'accessibilité à un classement et de configurer les règles de fournisseur de stockage.

  • Configuration du profil de classement : inclut l'ajout de documents exemples (documents utilisés pour générer les locutions clés) pour les classements assistés ou automatiques, ainsi que la construction de la Requête de profil.

  • Classification des éléments : comprend la configuration des objets gérés (éléments Content Server classables), la définition du type de gestion et la gestion des objets en attente.

  • Exportation et importation de taxonomies de classements : englobe l'exportation des classements vers un dossier XML, l'importation des classements depuis d'autres installations Content Server et la construction d'une taxonomie XML.

  • Exécution de fonctions de maintenance : permet d'ajuster les fonctions du module et d'assurer le bon fonctionnement du système.

  • Utilisation de la fonction de l'API d'exportation XML Content Server avec Classifications : permet d'exporter des informations relatives au classement au format XML.

Pour plus d'informations sur l'utilisation des classements (par exemple sur le classement des éléments et la recherche dans les classements), reportez-vous à  Utilisation de Classifications.
