Si vous souhaitez élaborer une hiérarchie de classements, sans avoir à ajouter chaque classement à l'espace de travail Classement individuellement, vous pouvez créer un fichier XML définissant une hiérarchie de classements (une taxonomie de classement XML), puis l'importer.

Construction d'une taxonomie XML

Les taxonomies de classement XML se composent d'une série de balises XML définissant une hiérarchie de classements, incluant le nom de tous les classements et les informations de profil correspondantes. Si vous définissez une taxonomie de classement XML manuellement en vue de l'importer dans Content Server, la structure, les balises et les valeurs qu'elle contient doivent répondre à certains critères.

Section d'en-tête

La première partie d'une taxonomie de classement XML consiste en une section d'en-tête, qui définit le fichier XML et l'Arbre de classement en général. Cette section commence par la déclaration XML suivante, qui définit la version XML et le codage de caractères utilisé dans le fichier :

<?xml version=&quot;1.0&quot; encoding=&quot;ISO-8859-1&quot;?>

L'Arbre de classement doit ensuite être défini à l'aide de la balise de début importexport :

<importexport applanguage='USA' appname='ContentServer' appversion='9.1.0' dtdversion='2.5' exportdate='<date_and_time>'>

Tableau 1-1 décrit les attributs de la balise de début importexport :

Tableau 1-1 : Attributs de la balise de début importexport

Attribut Valeur
applanguage Langue de l'application, par exemple, FRA.
appname Nom de l'application, par exemple, Content Server. Cet attribut est obligatoire.
appversion Numéro de version de l'application, par exemple, 9.6.0. Cet attribut est obligatoire.
dtdversion Numéro de version du DTD qui définit les taxonomies de classement XML. La seule valeur possible est 2,5.
exportdate Date et heure d'exportation de l'Arbre de classement à partir d'Content Server. Cet attribut se présente sous la forme <aaaa>-<mm>-<jj>T<hh>:<mn>:<ss> (par exemple, 2002-04-12T21:31:48), où :
<aaaa> représente l'année,
<mm> le mois,
<jj> le jour,
<hh> l'heure,
<mn> le nombre de minutes
et <ss> le nombre de secondes.
Cet attribut est facultatif. Lorsque vous construisez une taxonomie de classement XML manuellement, il est inutile d'inclure cet attribut.

Section de hiérarchie des classements

La seconde partie d'une taxonomie de classement XML définit la hiérarchie des classements et leurs paramètres. Cette section figure entre les balises de début et de fin de l'élément importexport ; elle consiste en un ou plusieurs éléments classificationtree, qui contiennent un ou plusieurs éléments classification.

Les éléments classification définissent les classements comme suit :

<classification managementtype='<type>' name='<nom>' selectable='<valeur booléenne>' subtype='<n>'><query>SELECT &quot;OTObject&quot; ,&quot;OTScore&quot; WHERE "test" ORDEREDBY RankingExpression &quot;test&quot;</query></classification>

 

Note :
<n> dans subtype='<n>' représente un numéro valide de sous-type.

 

Tableau 1-2 décrit les balises classificationtree, classification et query :

Tableau 1-2 : Balises classificationtree, classification et query

Balise Description
classificationtree Cette balise définit une liste de classements. Ses attributs contiennent les paramètres de la liste de classements.
classification Cette balise définit un classement. Ses attributs contiennent tous les paramètres du classement.
requête Cette balise définit la Requête de profil correspondant à un classement lorsque celui-ci a été exporté à partir d'Content Server. N'entrez aucune valeur dans ce champ.

Tableau 1-3 décrit les attributs des balises de début classificationtree et classification :

 

Note :
Lors de la définition de l'attribut name de l'élément Arbre de classement ou classement, vous ne pouvez pas utiliser le symbole des deux points (:).

 

Tableau 1-3 : Attributs des balises classificationtree et classification

Attribut Valeur
managementtype Le type de gestion à attribuer à l'Arbre de classement ou au classement. Les valeurs valables sont : manual, automatic et assisted.
name Le nom de l'Arbre de classement ou du classement. Cette valeur est une chaîne (Rapports annuels, par exemple).
selectable Indique si l'Arbre de classement ou le classement est sélectionnable ou pas. Les valeurs valides sont true et false.
subtype Le numéro de sous-type de l'objet Content Server. Les valeurs admises sont 196 pour les balises Arbre de classement et 199 pour les balises classification.

Tableau 1-4 décrit les attributs facultatifs des balises de début classificationtree et classification :

Tableau 1-4 : Attributs des balises classificationtree et classification

Attribut Valeur
score Nombre saisi par l'utilisateur. Les valeurs admises sont comprises entre 1 et 100 (inclus).
description Description du nœud saisie par l'utilisateur.

Dans cette section, la hiérarchie des classements est définie par l'imbrication d'éléments classification à l'intérieur d'autres éléments classification. Par exemple, si un classement nommé Classement1 en contient un autre nommé Classement2, alors Classement2 sera imbriqué dans Classement1 de la manière suivante :

<classification managementtype='manual' name='Classification1' selectable='true' subtype='199'>
   <classification managementtype='manual' name='Classification2' selectable='true' subtype='199'>
   </classification>
</classification>

La définition de la hiérarchie est soumise aux règles suivantes :

  1. Un seul élément de classificationtree peut figurer au niveau racine de la hiérarchie.

  2. Chaque élément classificationtree et classification peut contenir d'autres éléments classification.

 

Note :
Chaque élément classification et classificationtree peut contenir un élément de requête query (facultatif).

 

A la fin de cette section, vous devez insérer la balise de fin de l'élément classificationtree, à savoir : </classificationtree>.

Fichiers exemples

L'exemple suivant porte sur un Arbre de classement vide nommé Arbre de classement vide. Celui-ci n'est pas sélectionnable et son type de gestion est configuré sur automatique :

<?xml version="1.0" encoding="ISO-8859-1"?>
<importexport applanguage='USA' appname='ContentServer' appversion='9.5.0' dtdversion='2.5' exportdate='2003-07-22T17:34:07'>
   <classificationtree managementtype="automatic" name="Test Classification Tree" selectable="true" subtype='196'>
         <query>SELECT "OTObject" ,"OTScore" WHERE "test" ORDEREDBY RankingExpression "test"</query>
   </classificationtree>
</importexport>

L'exemple suivant porte sur un Arbre de classement plus classique. Le nom de l'arbre est Informations d'entreprise et celui-ci contient trois classements : A propos de l'entreprise, Rapports annuels et Nouvelles. Le classement Nouvelles contient deux autres classements : Communiqués de presse et Bulletins d'information. Au niveau racine, le classement Informations d'entreprise n'est pas sélectionnable et son type de gestion est configuré sur manuel ; par contre, toutes les classements qu'il contient sont sélectionnables et leur type de gestion est configuré sur assisté.

<?xml version="1.0" encoding="ISO-8859-1"?>
<importexport dtdversion='2.5' appname='ContentServer' appversion='9.5.0' applanguage='USA'>
   <classificationtree managementtype='manual' name='Corporate Information' selectable='false' subtype='196'>
      <query></query>
      <classification managementtype='assisted' name='About Us' selectable='true' subtype='199'>
         <query></query>
      </classification>
      <classification managementtype='assisted' name='Annual Reports' selectable='true' subtype='199'>
         <query></query>
      </classification>
      <classification managementtype='assisted' name='News' selectable='true' subtype='199'>
         <query></query>
         <classification managementtype='assisted' name='Press Releases' selectable='true' subtype='199'>
            <query></query>
         </classification>
         <classification managementtype='assisted' name='Newsletters' selectable='true' subtype='199'>
            <query></query>
         </classification>
      </classification>
   </classificationtree>
</importexport>

Importation d'une taxonomie XML

Une fois que vous avez créé une taxonomie XML, vous pouvez importer le fichier XML dans Content Server ; les classements sont alors créés automatiquement. Pour plus d'informations sur l'importation de taxonomies de classement, reportez-vous à la rubrique Exportation et importation de taxonomies de classement.

 

Note :
Si vous n'avez pas activé la fonctionnalité Classement intelligent, le type de gestion est défini sur « manuel » lorsque vous importez des Arbre de classement. Pour plus d'informations sur Classement intelligent, consultez la section Activation de la fonctionnalité de Classement intelligent.
