Les types d'élément qu'il est possible de classer sont appelés des objets gérés. Vous pouvez configurer quatre types d'élément par défaut dans Content Server en tant qu'objets gérés :

  • Documents composites

  • Documents

  • Documents texte

  • Dossiers

Par exemple, si vous définissez le type d'élément Documents comme objet géré, les utilisateurs Content Server peuvent classer les documents. Content Server peut également être configuré de manière à classer automatiquement les documents ou à proposer des classements à l'aide de la fonctionnalité de classement intelligent. Pour plus d'informations sur les classements automatiques et assistés, reportez-vous à la rubrique Ajout de classements.

Si vous souhaitez que d'autres types d'élément Content Server soient configurés en tant qu'objets gérés (par exemple, les URL ou les projets), vous devez les ajouter à la page d'administration Objets gérés. Pour en savoir plus, voir Configuration des objets gérés.

Objets Enterprise Library Services au sein d'un classement

Vous ne pouvez pas classer des types d'objet OpenText Enterprise Library par défaut. Vous devez les rendre classables manuellement en modifiant la valeur du paramètre SubTypes dans la section [Classification] du fichier opentext.ini, puis en les sélectionnant sur la page Objets gérés. Pour plus d'informations, reportez-vous à la rubrique Paramètres du fichier opentext.ini pour Classements.

Une fois configurés, vous pouvez classer des éléments OpenText Enterprise Library à l'aide du classement manuel, assisté ou automatique. Des règles d'héritage de classement s'appliquent aux objets de recueil OpenText Enterprise Library, tels que Dossier et Elément.

 

Note :
Comme les variantes ont un contenu (dans le document et dans l'index de recherche) et que les éléments contiennent des métadonnées, les variantes sont classées indépendamment des éléments. Lorsqu'Content Server classe des objets OpenText Enterprise Library basés sur le contenu, seules les variantes sont classées, et les requêtes basées sur le contenu ne renvoient que des variantes dans les résultats de recherche. Lorsque vous ajoutez une variante, celle-ci hérite des classements de l'élément à condition que l'élément et la variante soient des sous-types classables. Avant de classer des éléments Enterprise Library Services, vous devez les rendre disponibles pour le classement. Pour en savoir plus, voir Classement d'un contenu existant à l'aide de la fonctionnalité de classement intelligent.
