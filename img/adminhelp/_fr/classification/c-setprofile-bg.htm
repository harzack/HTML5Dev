Lorsque vous utilisez la fonctionnalité de Classement intelligent pour les classements de type assisté ou automatique, vous devez définir un profil. Ce profil vous permet de spécifier des critères auxquels les éléments doivent répondre afin que le classement leur soit assigné. Chaque fois que vous ajoutez ou que vous modifiez un élément de type géré, le système l'examine par rapport à chaque profil de classement défini et détermine s'il répond aux critères définis dans le profil pour le classement correspondant. Si l'élément remplit les critères d'un profil de classement, le système effectue l'une des opérations suivantes :

  • S'il s'agit d'un classement de type assisté, le système le propose interactivement sur la page Gérer les objets en attente. Vous pouvez alors l'accepter, le rejeter ou le laisser en attente.

  • S'il s'agit d'un classement de type automatique, le système l'assigne automatiquement à l'élément.

 

Note :
Vous ne pouvez pas définir un profil pour un classement de type manuel.

 

Pour plus d'informations sur les objets gérés, reportez-vous à la rubrique Configuration de types d'éléments comme classables. Pour plus d'informations sur l'acceptation ou le rejet de classements proposés par le système, reportez-vous à la rubrique Maintenance des classements. Pour plus d'informations sur la définition du type de classement, reportez-vous à la rubrique Ajout de classements.

Onglet Profil

S'il s'agit d'un classement de type assisté ou automatique, l'onglet Profil sera visible lorsque vous afficherez une page de propriétés de classement et vous pourrez :

  • Définir le taux de pertinence des scores.

  • Ajouter ou dissocier les documents exemples.

  • Voir les Locutions clés générées.

Chacun de ces critères sert à déterminer si un objet géré répond au profil de classement et s'il doit être classé automatiquement ou si le classement doit être proposé.

Définition du taux de pertinence des scores

Le taux de pertinence des scores détermine le degré de correspondance d'un document à vos paramètres de profil. Plus le taux est élevé, plus la correspondance doit être proche. Cette valeur doit être comprise entre 1 et 100. La valeur par défaut est 40.

 

Note :
Cliquez sur le bouton Définir une requête de profil dans l'onglet Profil, puis sur le bouton Enregistrer en tant que profil de la page de recherche pour que les modifications apportées au taux de pertinence des scores soient effectives.

 

Ajout de documents exemples au profil

Si vous possédez déjà un ou plusieurs documents représentatifs d'un classement, vous pouvez définir un profil de classement rapidement en ajoutant ces documents au profil en tant que documents exemples. Content Server détermine et affiche ensuite les locutions clés (mots et combinaisons de mots récurrents, notamment les combinaisons incluant des mots peu courants) des documents exemples. Ces locutions permettent de déterminer si les éléments respectent le profil.

Une fois que vous avez ajouté les documents exemples, vous devez créer la Requête de profil pour stocker les locutions clés dans le profil. Une Requête de profil est un ensemble de critères de recherche qui définit les paramètres des classements de type automatique et assisté.

Lors de l'ajout d'un document exemple à un profil de classement, Content Server ajoute automatiquement des locutions clés extraites du document dans la page Recherche de Requête de profil. De plus, l'emplacement du document exemple s'affiche dans la colonne Emplacement de l'onglet Profil.

 

Note :
Content Server ne peut extraire des locutions clés que de la dernière version principale parmi les versions principales/secondaires d'un document exemple. Vous pouvez ajouter des documents sans version principale comme documents exemples, mais les locutions clés de ces documents ne contribuent pas au profil de classement.

 

Définition de la Requête de profil

Vous pouvez définir la Requête de profil à l'aide de la page de recherche Content Server. Vous pouvez configurer toute information pouvant faire l'objet de recherches dans Content Server en tant que critère pour les classements intelligents. Pour les objets gérés qui répondent aux critères spécifiés dans la Requête de profil et satisfont le taux de pertinence des scores spécifiés dans le profil de classement, le classement est assigné automatiquement ou proposé, selon le type de gestion défini pour le classement. Par défaut, la page Requête de profil de classement s'ouvre sur des champs de texte supplémentaires dans la section Texte intégral. Ces champs vous permettent de :

  • spécifier des locutions clés ;

  • modifier des locutions clés des documents exemples pour changer les locutions extraites automatiquement ;

  • modifier des locutions clés d'une requête complexe d'une taxonomie de classement importée.

Chaque section contient des instructions.

 

Note :
Vous pouvez modifier des locutions clés dans les champs de documents exemples et des requêtes complexes des taxonomies de classement importées, mais Content Server n'enregistrera pas les modifications.

 

Modèles de recherche de profils de classement

Les modèles de recherche de profils de classement incluent un élément en lecture seule, Portée, qui fait référence aux vues de recherche Enterprise et Enterprise Library. Content Server peut tester des Requêtes de profil sur des sources de données Enterprise et Enterprise Library à l'aide du bouton Rechercher. Lorsque vous êtes satisfait de votre profil et que vous l'avez enregistré, Content Server effectue un classement automatique ou assisté, selon le type de gestion activé sur l'onglet Extras de votre page de propriétés de classement.

Les vues de recherche Enterprise et Enterprise Library sont mises en évidence dans l'élément Portée en lecture seule. Cela indique que les vues sont présélectionnées et qu'elles ne peuvent pas être modifiées.
