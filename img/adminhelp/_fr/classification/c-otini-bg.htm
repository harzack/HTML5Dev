Le fichier opentext.ini, situé dans le répertoire <content_server_home>/config, est le principal fichier de configuration de Content Server. Il contient également plusieurs paramètres agissant sur le comportement de Classements. Il est parfois nécessaire de modifier ces paramètres en modifiant directement le fichier opentext.ini à l'aide du module Content Server SDK Builder ou d'un éditeur de texte.

Pour plus d'informations sur le fichier opentext.ini, reportez-vous à la rubrique Compréhension du fichier opentext.ini.

Les paramètres suivants figurent dans la section [Classification] du fichier opentext.ini. Certains paramètres n'apparaissent pas par défaut. Toutefois, pour ces paramètres, Content Server applique les valeurs par défaut indiquées dans Tableau 1-6.

Tableau 1-6 : Paramètres opentext.ini des classements

Paramétrage Valeurs Description
AllowAppearances true ou false.
Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut au paramètre AllowAppearances=true.
 Ce paramètre permet de s'assurer que des apparences peuvent être créées dans l'espace de travail Classement. Pour plus d'informations sur les apparences, reportez-vous à la rubrique Utilisation des apparences.
AllowCustomViews true ou false.
Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut au paramètre AllowCustomViews=true.
 Ce paramètre permet de vérifier que les affichages personnalisés peuvent être créés dans l'espace de travail Classement. Pour plus d'informations sur les affichages personnalisés, reportez-vous à Utilisation des affichages personnalisés.
SubTypes {objID,objID,etc...}.
Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut au paramètre SubTypes={136,144,145,0}.
 

Note :
Pour rendre les types d'objet OpenText Enterprise Library classables, vous devez modifier la valeur des sous-types. Les sous-types OpenText Enterprise Library pouvant être ajoutés sont 823 (Dossier), 824 (Elément), 825 (Variante) et 826 (Composite).

 
 Contient les ID d'objet des types d'éléments disponibles en tant qu'objets gérés. Pour plus d'informations sur les objets gérés, reportez-vous à la rubrique Configuration de types d'éléments comme classables.
DisplayNum Un entier compris entre 100 et 2000.
Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut au paramètre DisplayNum=100.
 

Note :
Si vous affectez à ce paramètre une valeur inférieure à 100 ou supérieure à 2000, Content Server utilise la valeur par défaut 100.

 
 Définit le nombre maximum d'éléments affichés lorsqu'un utilisateur parcourt un classement.
reextract true ou false.
Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini , ce qui équivaut au paramètre reextract=true.
 

Note :
La configuration du paramètre reextract sur true peut entraîner un ralentissement des installations Content Server de grande taille.

 
A chaque fois que vous ajoutez un élément à Content Server ou que vous modifiez un élément, Content Server procède à l'extraction de cet élément. Ce processus consiste à assigner des métadonnées à l'élément, à l'indexer à des fins de recherche et, si vous utilisez la fonctionnalité Classement intelligent, à identifier les classements assistés et automatiques qui lui conviennent le mieux, puis à assigner ou proposer ces classements. Pour que les données de classement de chaque élément puissent faire l'objet de recherches, Content Server doit réextraire l'élément automatiquement chaque fois qu'un classement lui est assigné, sans le comparer de nouveau à des classements assistés et automatiques.
Le paramètre reextract détermine si Content Server doit réextraire les éléments une fois qu'un classement leur a été assigné. Si vous configurez ce paramètre sur true, les données de classement de chaque élément peuvent faire l'objet de recherches immédiatement après l'assignation d'un classement. Si vous configurez ce paramètre sur false, les données de classement de chaque élément ne peuvent pas faire l'élément de recherches tant que l'élément n'a pas été extrait à nouveau.
maxClassifications Un entier compris entre 1 et 20.
Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut au paramètre maxClassifications=5.
 

Important :
Ne modifiez pas ce paramètre, sauf instruction contraire donnée par OpenText Customer Support. En effet, une valeur plus élevée risquerait de provoquer un ralentissement du système et une valeur plus faible pourrait avoir un effet négatif sur la précision de la fonctionnalité de Classement intelligent.

 
 Lors de l'extraction d'un élément, la fonctionnalité de Classement intelligent identifie les classements automatiques et assistés qui lui conviennent le mieux. Le paramètre maxClassifications définit le nombre maximum de classements que la fonctionnalité de Classement intelligent peut assigner à ou proposer pour un élément lors de son extraction.
maxImportAgentObjects Un nombre entier.
La valeur par défaut est 100.
 

Note :
Une valeur de 0 entraîne des restrictions de classement sur tous les objets. Une valeur inférieure à 0 désactive le paramètre.

 
 Lors du traitement de classements automatiques et assistés (Classement intelligent), le paramètre maxImportAgentObjects définit le nombre maximal d'objets de classement IPool qui peuvent être traités au cours d'une exécution de l'Agent client 5 minutes.
maxLargeImportAgentObjects Un nombre entier.
La valeur par défaut est 5.
 

Note :
Une valeur de 0 entraîne des restrictions de classement sur tous les objets de taille importante. Une valeur inférieure à 0 désactive le paramètre.

La taille des grands objets de classement iPool est définie par le paramètre largeImportAgentObjectSizeKB.

 
 Lors du traitement de classements automatiques et assistés (Classement intelligent), le paramètre maxLargeImportAgentObjects définit le nombre maximal d'objets de classement IPool de taille importante qui peuvent être traités au cours d'une exécution de l'Agent client 5 minutes.
largeImportAgentObjectSizeKB Un nombre entier.
La valeur par défaut est 4000.
 Lors du traitement de classements automatiques et assistés (Classement intelligent), le paramètre largeImportAgentObjectSizeKB définit la taille (en Ko) à utiliser avec le paramètre maxlargeImportAgentObjectSizeKB.
lowScoreThres Un entier compris entre 0 et 100.
Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut au paramètre lowScoreThres=40.
 

Note :
Si vous modifiez ce paramètre, vous devez redémarrer Content Server et régénérer le fichier de requête de classement pour appliquer le nouveau paramètre aux classements qui utilisent le nouveau taux de pertinence des scores par défaut. Pour plus d'informations sur la régénération du fichier de requête de classement, reportez-vous à la rubrique Maintenance des classements.

 
 Lors de l'extraction d'un élément, la fonctionnalité de Classement intelligent compare celui-ci à chaque classement assisté et automatique, puis détermine les taux de pertinence des éléments de chaque profil de classement. Le paramètre lowScoreThres identifie le taux de pertinence minimum nécessaire pour que le système propose un classement assisté ou assigne un classement automatique à l'élément.
siblingCompressRatio Un entier compris entre 0 et 100.
Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut au paramètre siblingCompressRatio=75.
 

Note :
Si vous affectez à ce paramètre une valeur inférieure à 0 ou supérieure à 100, Content Server utilise la valeur par défaut.

 
 La fonctionnalité de Classement intelligent identifie parfois, au sein du même classement parent, plusieurs classements automatiques et assistés correspondant à l'élément. Le paramètre siblingCompressRatio définit le pourcentage de classements devant correspondre à l'élément, au sein d'un classement parent, pour que celui-ci soit assigné à l'élément.
Par exemple, si vous configurez le paramètre siblingCompressRatio sur 75, le paramètre lowScoreThres sur 40 et que trois des quatre classements assistés au sein du classement automatique Rapports correspondent à un document avec des taux de pertinence supérieurs ou égaux à 40, Content Server assigne le classement Rapports au document au lieu de lui assigner les trois classements assistés.
RetainIntelClass true ou false.
Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut au paramètre RetainIntelClass=true.
 Détermine si tous les classements automatiques et assistés précédemment assignés aux éléments Content Server doivent être conservés lorsque de nouveaux classements automatiques et assistés sont assignés à ces éléments.
Si vous configurez ce paramètre sur true, les éléments conservent leurs classements automatiques et assistés précédents et adoptent également les nouveaux classements ; les classements précédents doivent alors être dissociés manuellement.
Si vous configurez ce paramètre sur false, les classements automatiques et assistés précédents sont dissociés et les nouveaux classements sont assignés aux éléments.
highScoreBypassThres Un entier compris entre 0 et 100.
Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut au paramètre highScoreBypassThres=70.
 Identifie le taux de pertinence minimum nécessaire pour qu'un élément ignore le paramètre siblingCompressRatio.
Par exemple, si vous configurez highScoreBypassThres sur 70, siblingCompressRatio sur 75, maxClassifications sur 5 et que trois des quatre classements assistés au sein du classement automatique Rapports correspondent à un document avec des taux de pertinence supérieurs ou égaux à 70, le classement parent n'est pas assigné au document. Mais Content Server classe les trois classements assistés au même titre que les autres classements pertinents, puis il assigne à l'élément les cinq classements qui conviennent le mieux.
bulkOperations true ou false.
Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut au paramètre bulkOperations=false.
 Active la fonctionnalité de sélection multiple pour les éléments classés. En définissant ce paramètre sur true, les utilisateurs peuvent sélectionner plusieurs éléments Content Server à copier, déplacer, supprimer (ou collecter, s'ils y sont autorisés).
La fonctionnalité de sélection multiple ne contient pas de caractéristiques spécifiques de classement. Par conséquent, les commandes sont appliquées directement sur l'élément Content Server et non sur le classement.
bulkClassDisplayNum Un entier supérieur ou égal à 1.
Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut au paramètre bulkClassDisplayNum=2.
Définit le nombre maximum de classements qui peuvent être affichés pour un élément sur les pages suivantes d'Content Server :
•

Page Classer les résultats de recherche lorsque la page affiche les classements qui sont assignés à un ou plusieurs éléments. Utilisez cette page lorsque vous classez les éléments de la page Résultats de la recherche.

•

Page Classer les éléments d'une collection lorsque la page affiche les classements qui sont assignés à un ou plusieurs éléments. Utilisez cette page lorsque vous classez des éléments dans une collection.

Si le paramètre bulkClassDisplayNum est défini sur un nombre de classements inférieur au nombre assigné à un élément, un lien Afficher tout s'affiche. En cliquant sur ce lien, tous les classements assignés à l'élément s'affichent. Si les classements assignés s'affichent pour plusieurs éléments, vous pouvez cliquer sur le lien en haut de la page pour afficher les classements assignés à tous les éléments.
inheritedClassDisplayNum Un entier supérieur ou égal à 0.
Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut au paramètre inheritedClassDisplayNum=4.
 Définit le nombre maximum de classements qui peuvent s'afficher sur la page Ajouter de l'élément lorsque l'élément hérite de classements. Un élément hérite de classements lorsque vous l'ajoutez à un recueil classé.
Par exemple, lorsque vous ajoutez un document à un dossier classé comme confidentiel, le document hérite de ce classement. Si le paramètre inheritedClassDisplayNum est défini sur un nombre de classements inférieur au nombre affecté à un élément, un lien Afficher tout s'affiche. En cliquant sur ce lien, tous les classements hérités par l'élément s'affichent.
maxPathDisplayLen Un entier supérieur à 2.
Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut au paramètre maxPathDisplayLen=70.
Définit le nombre maximum de caractères qui peuvent s'afficher dans le chemin représentant une hiérarchie de classements sur les pages suivantes d'Content Server :
•

Page Ajouter de l'élément lorsque l'élément hérite de classements. Un élément hérite de classements lorsque vous l'ajoutez à un recueil classé.

•

Page Classer les résultats de recherche lorsque la page affiche les classements qui sont assignés à un ou plusieurs éléments. Utilisez cette page lorsque vous classez les éléments de la page Résultats de la recherche.

•

Page Classer les éléments d'une collection lorsque la page affiche les classements qui sont assignés à un ou plusieurs éléments. Utilisez cette page lorsque vous classez des éléments dans une collection.

La hiérarchie des classements hérités s'affiche en tant que chemin horizontal au format suivant : Confidentiel : Présentations : RésultatsFinanciers, etc.
Si le paramètre maxPathDisplayLen est défini sur un nombre de caractères inférieur à celui qui est contenu dans un chemin de hiérarchie de classement, le chemin est tronqué selon le nombre de caractères indiqué. Les caractères se trouvant au milieu du chemin ne s'affichent pas et sont représentés des points de suspension (...).
searchResultPathLinks true ou false.
Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut au paramètre searchResultPathLinks=true.
 Permet de vérifier qu'un autre chemin ne contenant pas de lien hypertexte et délimité par le caractère « : » s'affiche. Pour activer cette option, définissez le paramètre searchResultPathLinks sur false.
Par défaut, les chemins s'affichent sous la forme d'une séquence d'emplacements dotés de liens hypertextes allant de la racine de l'Arbre de classement au classement lui-même.

applyDownBlockSize Un entier supérieur à 1.
Par défaut, ce paramètre n'apparaît pas dans le fichier opentext.ini, ce qui équivaut au paramètre applyDownBlockSize=50.
 Les sous-éléments sont traités en blocs de tailles définies. Ce paramètre vous permet d'ajuster le nombre d'éléments traités lors de chaque requête.
