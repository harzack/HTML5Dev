La rubrique Maintenance des classements décrit les tâches possibles et les paramètres de maintenance relatifs au module Classements. Ces tâches et paramètres vous permettent d'ajuster le fonctionnement du module afin d'obtenir des performances optimales.

Les tâches de maintenance incluent la régénération du fichier de requête des classements, la suppression ou le reclassement d'éléments associés avec un classement, la gestion des objets en attente de classement, l'élimination de classements, l'élimination de la base de données des classements rejetés et la configuration des paramètres du fichier opentext.ini pour les classements.

Régénération du fichier de requête de classement

Le fichier de requête de classement (queries.in) est un fichier système dans lequel sont enregistrées toutes les informations de Requête de profil pour les classements assistés et automatiques. Il se trouve dans le répertoire <content_server_home>/config. Si vous n'avez pas activé la fonctionnalité de Classement intelligent, ce fichier est absent de votre système. Pour en savoir plus, voir Activation de la fonctionnalité de Classement intelligent. Le fichier queries.in est modifié lorsqu'un classement est mis à jour dans Content Server via le bouton Enregistrer en tant que profil, ou lorsqu'un nœud de classement est supprimé.

Vous devez régénérer le fichier de requête de classement afin de mettre à jour les informations de Requête de profil qu'il contient dans les cas suivants :

  • après avoir importé un Arbre de classement ;

  • après avoir remplacé le type de gestion « manuel » d'un classement par « assisté » ou « automatique », si ce classement avait été précédemment configuré sur « assisté » ou sur « automatique » ;

  • lorsque le OpenText Customer Support vous le demande.

Gestion des objets en attente

Lorsque vous ajoutez à Content Server un élément et que cet élément est un objet géré, Content Server examine cet élément et détermine s'il répond aux critères définis dans le profil de chaque classement assisté. Si l'élément répond aux critères du profil, Content Server propose le classement à associer à cet élément. Vous pouvez accepter ce classement, le rejeter ou différer votre décision en laissant le statut de l'élément en attente.

 

Note :
Pour plus d'informations sur l'activation des types d'objet géré, reportez-vous à la rubrique Pour activer des types d'objet géré supplémentaires.

 

Elimination de classements

Vous pouvez supprimer des classements de deux façons différentes : en les dissociant ou en les éliminant. Lorsque vous dissociez un classement, Content Server vérifie que vous disposez de l'autorisation de dissocier ce classement, ainsi que tous les classements qu'il contient, avant de dissocier les éléments en question. De plus, Content Server met à jour la base de données et, le cas échéant, le fichier de requête de classement de chaque classement supprimé.

 

Attention
La durée du processus varie selon la taille de la hiérarchie de classement et peut prendre du temps.

 

Eliminer des classements consiste à dissocier rapidement une hiérarchie de classements de taille importante (par exemple, un Arbre de classement, un classement et tous les classements qu'il contient). Le processus d'élimination, tout comme la dissociation, permet de supprimer tous les classements, mais est plus rapide car les autorisations associées à chaque classement ne sont pas vérifiées. En outre, le fichier de requête de classement n'est mis à jour qu'une seule fois. Vous pouvez, par exemple, éliminer une hiérarchie de classements si vous importez un Arbre de classement comportant un nombre élevé de classements (nombre supérieur à 1 000) et que l'importation échoue.

Elimination de classements rejetés de la base de données

A chaque fois que vous rejetez un classement proposé par le système, ce dernier enregistre le rejet dans une base de données Content Server afin de ne pas proposer ce classement pour le même élément par la suite. Lorsque les données atteignent une certaine taille, elles peuvent toutefois provoquer un ralentissement du système.

En éliminant des classements rejetés de la base de données, vous supprimez des entrées en fonction du nombre de jours dans la base de données que vous avez spécifiée. La valeur par défaut est 45 jours. Les entrées de 45 jours et plus sont éliminées.

 

Note :
Si vous éliminez certains classements de la base de données Content Server, il se peut que des classements rejetés pour certains objets vous soient de nouveau proposés automatiquement par la suite.
