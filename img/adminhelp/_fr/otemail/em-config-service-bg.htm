Email Services prend en charge plusieurs paramètres de configuration. Vous pouvez sélectionner les champs d'e-mail à extraire pour les indexer et décider d'indexer ou non les attributs Content Server standard. Vous pouvez décider si le corps des e-mails doit être stocké pour l'afficher. Vous pouvez également sélectionner le nombre d'éléments à afficher par page dans les dossiers d'e-mails paginés.

 

Note :
Vous pouvez utiliser la recherche Content Server en texte intégral pour faire porter la recherche sur tout le contenu des e-mails. Pour rechercher les propriétés d'un e-mail à partir de la recherche avancée Content Server, suivez cette procédure :

  • Configurez les champs d'e-mail pour qu'ils soient indexés dans la configuration Email Services.

  • Activez l'option Pour recherche des champs d'e-mails dans le gestionnaire de recherche. Pour plus d'informations, reportez-vous à Content Serverl'aide administrateur Content Server(Index et administration de recherche – Gestion de la recherche – Configuration des régions d'index).

 

Sur la page Configurer Email Services, vous pouvez configurer les paramètres suivants de Email Services :

Champs d'e-mail extractibles

Vous pouvez spécifier les champs d'e-mail à extraire pour les indexer. Pour faire l'objet d'une recherche, un champ doit être extrait. Néanmoins, vous pouvez choisir de ne pas extraire certains champs d'e-mail afin de réduire la taille de l'index de recherche et vous reposer sur la recherche intégrale des messages.

 

Note :
Choisir de ne pas extraire certains champs d'e-mail pour l'indexation peut affecter le fonctionnement d'autres produits Content Server.

 
Attributs extractibles

Vous pouvez indexer ou non les attributs Content Server standard des e-mails. Le fait de limiter les éléments qui peuvent être extraits pour l'indexation permet de réduire la mémoire nécessaire pour la recherche d'index, notamment dans les organisations devant gérer des volumes importants d'e-mails.

 

Notes :

  • Choisir de ne pas extraire les attributs Content Server standard peut affecter le fonctionnement de toutes les applications utilisant Content Server pour stocker et rechercher des e-mails.

  • Si vous choisissez de ne pas extraire les attributs Content Server standard, les attributs répertoriés ci-dessous ne sont pas indexés pour les e-mails et ne peuvent pas faire l'objet d'une recherche portant sur les e-mails du système.

Stocker le corps du message

Vous pouvez décider si le corps des e-mails doit être stocké sous forme de métadonnées dans la base de données, afin de l'afficher lorsque les nœuds d'e-mail sont prévisualisés dans Content Server.

 

Notes :

  • Pour Email Monitoring, OpenText ne recommande pas le stockage du contenu du corps des e-mails. Votre base de données pourrait devenir trop importante.

    Si vous choisissez de ne pas stocker le contenu du corps des e-mails pour l'affichage, le contenu du corps des e-mails ne sera pas disponible au format HTML. Cependant, vous pouvez afficher le corps des messages s'ils sont rendus dans l'affichage généré par OpenText Document Filters. Pour en savoir plus, voir OpenText Document Filters - Administrator's Guide (LLESCF-AGD).

 
Extraction des propriétés d'e-mail supplémentaires

En plus des propriétés d'e-mail standard, vous pouvez préciser des propriétés d'e-mail personnalisées à extraire et à stocker. Pour configurer Microsoft Outlook afin qu'il assure l'extraction et le stockage des propriétés supplémentaires des e-mails sur le serveur, vous devez ajouter la liste des propriétés au champ Extraction des propriétés d'e-mail supplémentaires. Vous devez saisir les entrées sur un rang par ligne, au format suivant :

<Système de messagerie>|<Nom de la propriété>|<Nom d'affichage>|<Nom de la région>

où

<Système de messagerie> identifie l'environnement de messagerie à partir duquel l'extraction de propriétés doit être tentée. Les valeurs sont OUTLOOK ou LOTUS.

<Nom de la propriété> identifie le nom de la propriété à extraire dans le système de messagerie.

Pour Microsoft Outlook, saisissez le <Numéro d'identificateur> de la propriété MAPI. Pour les propriétés portant un nom, vous devez saisir à la fois le GUID de la propriété et le nom de la propriété sous la forme d'une chaîne ou d'un nombre, séparés par une virgule.

Pour Lotus Notes, saisissez le <Nom du champ> de la Propriété du document.

 

Note :
Si vous ne parvenez pas à trouver une propriété, elle ne sera pas stockée dans la base de données. Qui plus est, les propriétés standard (Objet, De, À, CC, Date d'envoi et Date de réception) sont toujours extraites et stockées séparément des propriétés personnalisées.

 

<Nom d'affichage> est le nom d'affichage que vous souhaitez utiliser pour la propriété dans l'onglet Propriétés – E-mail de Content Server.

<Nom de zone> (facultatif) correspond au nom de la zone à utiliser par le moteur de recherche pour l'indexation. Si vous souhaitez avoir la possibilité de rechercher la propriété d'e-mail supplémentaire, vous devez préciser un nom de région. Si vous n'avez pas besoin de rechercher la propriété d'e-mail supplémentaire, vous pouvez ignorer ce champ.

Exemple

OUTLOOK|0x0017|Importance Flag|EmailImportance

LOTUS|($Classification)|Classement Lotus

OUTLOOK|{00020329-0000-0000-C000-000000000046},Keywords|Classification|EmailClassification

Classes de messages signés/cryptés

Vous pouvez définir des classes spéciales de messages Microsoft Outlook pour les e-mails signés ou cryptés. Par défaut, Enterprise Connect reconnaît certaines classes de messages signés ou cryptés dans Content Server.

Enterprise Connect ne peut pas traiter correctement ces types d'e-mails et utilise les propriétés saisies dans cette section pour proposer un message approprié aux utilisateurs qui ajoutent ces types d'e-mails dans Content Server. Si la liste des messages compris dans cette section est incomplète, vous pouvez la modifier pour inclure d'autres messages. Saisissez chaque classe de message sur une ligne distincte.

  2. Sur la page Configurer Email Services, sélectionnez les options pour Email Services.

  3. Cliquez sur Envoyer.
