Mit dem Kontrollkästchen Enterprise Connect Client erlauben, die Anmeldeinformationen auf der Festplatte zu speichern können Sie steuern, wie Enterprise Connect-Anmeldeinformationen gespeichert werden, und diese Konfiguration auf alle Enterprise Connect-Benutzer übertragen, die auf diese Instanz von Content Server zugreifen. Wenn sich die Benutzer bei Enterprise Connect anmelden, werden ihre Anmeldeinformationen standardmäßig in einem verschlüsselten Format im lokalen Dateisystem gespeichert. Daher werden die Benutzer nicht bei jedem Start einer neuen Enterprise Connect-Sitzung aufgefordert, ihre Anmeldeinformationen anzugeben, es sei denn, die gespeicherten Anmeldeinformationen sind ungültig geworden. Falls erforderlich, können Sie angeben, dass die Benutzeranmeldeinformationen nicht gespeichert werden, sodass die Benutzer bei jedem Start einer neuen Enterprise Connect-Sitzung aufgefordert werden, ihre Anmeldeinformationen einzugeben.

 

Hinweis:
Die Auswahl für dieses Kontrollkästchen hat Vorrang vor der Option, die in der Registrierungseinstellung DontStoreCredentialsLocally festgelegt ist. Wenn Sie dieses Kontrollkästchen z. B. deaktivieren, werden die Anmeldeinformationen unabhängig von der Registrierungseinstellung niemals lokal gespeichert. Weitere Informationen finden Sie unter OpenText Enterprise Connect - Installation Guide (NGDCORE-IGD) .
