Wenn Benutzer versuchen, eine E-Mail mit Anlagen zu einem Zielspeicherort hinzuzufügen, stellen sie u. U. fest, dass sich in dem Ordner bereits eine Anlage mit demselben Namen befindet. Sie können festlegen, wie Enterprise Connect E-Mail-Anlagen mit demselben Namen behandelt.

 

Hinweis:
Wenn Sie Enterprise Connect-Benutzern die Seite Meine Einstellungen zur Verfügung stellen, können sie diese Namenskonventionen ihren Anforderungen anpassen. Standardmäßig werden die hier festgelegten Einstellungen angewendet. Weitere Informationen finden Sie unter So konfigurieren Sie die Baumstruktur und das Listenfenster.

 

Sie können die Benutzer fragen, ob sie die Anlage mit dem doppelten Namen als neue Version des vorhandenen Dokuments hinzufügen möchten. Sie können auch festlegen, dass dies automatisch erfolgt, sobald Enterprise Connect ein Dokument mit demselben Namen erkennt. Sie können die Benutzer auch fragen, ob sie die Anlage mit dem doppelten Namen als neues Dokument hinzufügen möchten. In diesem Fall wird dem Namen des doppelten Dokuments ein numerisches Suffix hinzugefügt, um es vom ursprünglichen Dokument zu unterscheiden. Beispiel: taxform(2).doc. Sie können auch festlegen, dass dies automatisch erfolgt, sobald Enterprise Connect ein Dokument mit demselben Namen erkennt.

 

Hinweis:
Diese Einstellung gilt nur für E-Mails, die als E-Mail-Untertypen (749) gespeichert wurden.

 

Wenn Sie Records Management (RM) installiert haben und Sie einen Querverweis für E-Mail-Anlagen erstellen, sehen die Benutzer einen Querverweis zur entsprechenden Instanz der Anlage. Beispiel: Ein Benutzer verschiebt eine E-Mail mit einer Anlage namens legalform.doc in Content Server in einen Ordner, in dem bereits ein Dokument mit demselben Namen vorhanden ist. Sie haben Anlage automatisch als neue Dokument-Version hinzufügen ausgewählt. Wenn Querverweise aktiviert sind, wird dem vorhandenen Dokument eine neue Version von legalform.doc hinzugefügt. Wenn der Benutzer die E-Mail an einem neuen Speicherort öffnet, enthält sie eine Verknüpfung zur entsprechenden Instanz von legalform.doc. In diesem Fall wäre es die zweite Version von legalform.doc. Weitere Informationen zum Einrichten von RM-Querverweisen finden Sie unter OpenText Records Management - Installation and Administration Guide (LLESRCM-IGD).
