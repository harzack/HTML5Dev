So behandeln Sie E-Mails mit identischen Namen:

  1. Klicken Sie auf der Seite Content Server-Verwaltung auf Enterprise Connect-Verwaltung und anschließend auf die Verknüpfung E-Mail-Einstellungen für Enterprise Connect konfigurieren.

  2. Gehen Sie zum Bereich Namenserstellung für E-Mails.

  3. Führen Sie in der Liste Wenn eine E-Mail mit dem gleichen Namen zu einem Ordner hinzugefügt wird einen der folgenden Schritte aus:

    • Klicken Sie auf Namen mit einem Suffix generieren, wenn Enterprise Connect automatisch einen neuen Namen für die doppelte E-Mail erstellen soll. In das Ziel wird eine neue Instanz der E-Mail eingefügt. Sie wird durch ein Suffix von der Instanz unterschieden, die im Ziel bereits vorhanden ist. Wählen Sie dann in der nebenstehenden Liste eine der folgenden Optionen:

      • Wählen Sie Benutzer nach Entscheidung fragen, wenn Enterprise Connect Benutzer fragen soll, ob sie die automatisch umbenannte E-Mail zum Zielverzeichnis hinzufügen möchten. Dies ist die Standardeinstellung.

      • Klicken Sie auf Namen automatisch generieren, wenn Enterprise Connect automatisch einen neuen Namen für die doppelte E-Mail ohne Bestätigung vom Benutzer erstellen soll. Der Benutzer wird nicht aufgefordert, den Namenskonflikt zu beheben.

    • Klicken Sie auf Benutzern das Eingeben eines neuen Namens erlauben, wenn Benutzer in der Lage sein sollen, einen neuen Namen für die doppelte E-Mail einzugeben.

  4. Wenn Sie Ihre Wahl getroffen haben, führen Sie einen der folgenden Schritte aus:

    • Klicken Sie auf Senden, um die Änderungen anzuwenden.

    • Klicken Sie auf Zurücksetzen, um zur letzten gespeicherten Konfiguration zurückzukehren.

    • Klicken Sie auf Standard festlegen, um zur Standardkonfiguration zurückzukehren, und anschließend auf OK, um die Standardeinstellungen anzuwenden.

 

Hinweis:
Diese Einstellungen gelten für E-Mails, zusammengesetzte E-Mails und E-Mail-Stub-Nachrichten bei der Verwendung von Windows Explorer, Microsoft Outlook, Lotus Notes und GroupWise.
