Sie können angeben, welcher Untertyp auf gespeicherte E-Mails angewendet werden soll und wie Anlagen behandelt werden sollen, wenn Benutzer E-Mails mit Anlagen zu Content Server hinzufügen. Außerdem können Sie angeben, wie doppelte E-Mails gekennzeichnet und wie Benutzer benachrichtigt werden, wenn sie einem Ordner eine doppelte E-Mail hinzufügen, oder ob doppelte E-Mails ignoriert werden sollen. Die auszuwählenden Methoden sind abhängig von der Vorgehensweise bei der Speicherung von E-Mails und Anlagen in Ihrem Unternehmen.

E-Mail-Untertyp

Mit der Option für E-Mail-Untertypen wird der E-Mail-Untertyp (749) auf alle E-Mails angewendet, die Benutzer zu Content Server hinzufügen. Standardmäßig werden alle E-Mails als der E-Mail-Untertyp gespeichert. Sie können dann angeben, wie Anlagen behandelt werden. Sie können festlegen, dass E-Mail-Anlagen nicht als separate Kopien gespeichert werden sollen oder dass zusätzlich zur ursprünglichen E-Mail und Anlage auch eine separate Kopie der Anlage als Einzeldokument erstellt werden soll. Falls zutreffend, können Sie zulassen, dass Benutzer für jede E-Mail entscheiden, ob getrennte Kopien der Anlagen hinzugefügt werden sollen, oder ob die E-Mail und ihre Anlagen als ein einzelnes Objekt gespeichert werden sollen.

 

Tipp:
Sie können E-Mails und Anlagen mithilfe von Querverweisen verknüpfen, wenn das OpenText Records Management-Modul installiert und diese Funktion aktiviert ist. Weitere Informationen finden Sie unter OpenText Enterprise Connect - Installation Guide (NGDCORE-IGD).

 

Doppelte E-Mails in Microsoft Outlook

Standardmäßig werden doppelte E-Mails erkannt, indem die Namen der E-Mails, die zu Content Server hinzugefügt werden, mit den E-Mails verglichen werden, die sich bereits in dem Ordner befinden. Wenn Sie diese Option wählen, werden Benutzer jedes Mal benachrichtigt, wenn Enterprise Connect eine doppelte E-Mail im Zielverzeichnis herausstellt. Je nach Konfiguration können Benutzer aufgefordert werden, doppelte E-Mails umzubenennen oder Metadaten anzugeben.

Angenommen, Sie verwenden den Standardnamensvergleich zum Herausstellen doppelter E-Mails. Wenn ein Benutzer eine E-Mail einem Ordner hinzugefügt hat und dann demselben Ordner eine weitere E-Mail mit demselben Namen hinzufügt, wird diese auch bei unterschiedlichem Inhalt als Duplikat herausgestellt. Je nach ihrer Konfiguration können Benutzer aufgefordert werden, die E-Mail umzubenennen oder Metadaten anzugeben.

Optional können Sie angeben, dass doppelte E-Mails erkannt werden, indem die Nachrichten-ID der E-Mail, die vom Benutzer zu Content Server hinzugefügt wurde, mit denen der E-Mails verglichen wird, die sich bereits in diesem Ordner befinden. Die Nachrichten-ID ist für jeden Empfänger unterschiedlich. Wenn z. B. dieselbe E-Mail an zwei verschiedene Benutzer gesendet wird, wird jeder E-Mail eine eindeutige Nachrichten-ID zugewiesen. Außerdem können Sie festlegen, dass doppelte E-Mails erkannt werden, indem die externe Nachrichten-ID der E-Mail, die vom Benutzer zu Content Server hinzugefügt wurde, mit der externen ID der E-Mails verglichen wird, die sich bereits in dem Ordner befinden. Die externe Nachrichten-ID gilt nur für E-Mails, die von außerhalb der Domäne des Benutzers empfangen wurden. Wenn Sie die Nachrichten-ID-Vergleichsoptionen wählen, werden doppelte E-Mails nicht kopiert oder in das Zielverzeichnis verschoben, und Benutzer erhalten bei doppelten E-Mails keine Benachrichtigung. Es werden nur neue E-Mails hinzugefügt, und je nach Ihrer Konfiguration werden Benutzer aufgefordert, Metadaten einzugeben.

Angenommen, Sie haben das Kontrollkästchen Nachrichten-ID für die Duplikaterkennung verwenden aktiviert. Ein Benutzer fügt einen Satz E-Mails einem Ordner hinzu, und eine der E-Mails ist bereits mit derselben Nachrichten-ID in diesem Ordner vorhanden. Der Benutzer erhält keine Benachrichtigung zu dieser doppelten E-Mail. Je nach Konfiguration können Benutzer aufgefordert werden, Metadaten für alle neuen E-Mails anzugeben. Oder angenommen, zwei Benutzer erhalten eine E-Mail von demselben Absender. Wenn beide Benutzer die E-Mail in denselben Ordner kopieren, würden die E-Mails wegen ihrer unterschiedlichen IDs nicht als Duplikate gelten.

Angenommen, Sie haben das Kontrollkästchen Externe Nachrichten-ID für die Duplikaterkennung verwenden aktiviert. Ein Benutzer fügt einem Ordner eine von einer externen Quelle eingegangene E-Mail hinzu. Wenn ein Kollege, der dieselbe E-Mail erhalten hat, seine E-Mail in demselben Ordner speichert, wird sie als Duplikat herausgestellt und nicht hinzugefügt. Der Benutzer erhält keine Benachrichtigung zu dieser doppelten E-Mail.

 

Wichtig:
Es ist für Sie als Systemverwalter wichtig, dass Sie beim ersten Einrichten von Enterprise Connect entscheiden, wie doppelte E-Mails behandelt werden. Das Ändern dieser Einstellungen, nachdem Benutzer ihre Arbeit mit E-Mails aufgenommen haben, kann Probleme verursachen.

 
 

Hinweise:

  • Diese Funktion gilt nur für Microsoft Outlook.

  • Diese Funktion hat keine Auswirkungen auf zusammengesetzte E-Mails.

 

Untertyp für zusammengesetzte E-Mails

Die Option für zusammengesetzte E-Mails ist nur für OpenText Explorer-Benutzer vorgesehen, die auf Enterprise Connect umrüsten und bisher den Untertyp (557) für zusammengesetzte E-Mails verwendet haben. Wenn Sie diese Option wählen, werden alle E-Mails und zugehörigen Anlagen als zusammengesetzte E-Mails zu Content Server hinzugefügt. Bei einer zusammengesetzten E-Mail werden die E-Mail-Nachricht und alle dazugehörigen Anlagen in einem einzelnen Ordner gespeichert. Zusammengesetzte E-Mails werden nur im Listenfenster in Enterprise Connect und im SchnellZugriff-Bereich angezeigt, wenn es bzw. er für die Anzeige von Dokumenten konfiguriert wurde. In Microsoft Outlook, der Explorer-Baumstruktur oder im Microsoft Office-Dialogfeld Datei öffnen sind sie nicht sichtbar. Benutzer können zusammengesetzte E-Mails im Lesefenster anzeigen. Wenn Sie möchten, dass Benutzer zusammengesetzte E-Mails offline verwenden können, müssen Sie den Untertyp für zusammengesetzte E-Mails der Liste der Untertypen hinzufügen, an denen offline gearbeitet werden kann. Weitere Informationen finden Sie unter Untertypen angeben, die offline verwendet werden können.

 

Wichtig:
Zusammengesetzte E-Mails benötigen wesentlich mehr Speicherplatz als der standardmäßige E-Mail-Untertyp. Wenn Sie den Untertyp für zusammengesetzte E-Mails anwenden, stellen Sie sicher, dass ausreichend Speicherplatz für diese Anforderung vorhanden ist.
