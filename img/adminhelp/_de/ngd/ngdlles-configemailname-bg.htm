Wenn Benutzer mithilfe von Enterprise Connect E-Mails zu Content Server hinzufügen, werden bestimmte Namenskonventionen auf diese E-Mails angewendet. Standardmäßig wird der Betreff der E-Mail ohne Trennzeichen oder Auslassungen als E-Mail-Name verwendet. Die Namenskonventionen können auf folgende Weise angepasst werden:

  • Definieren Sie, welche Nachrichteneigenschaften in E-Mail-Namen verwendet werden sollen.

  • Definieren Sie das Trennzeichen, das zwischen den einzelnen Nachrichteneigenschaften verwendet werden soll, die den E-Mail-Namen bilden.

  • Geben Sie die Zeichenfolgen an, die aus dem E-Mail-Namen entfernt werden sollen.

  • Geben Sie das Datumsformat in der E-Mail an.

  • Geben Sie an, wie E-Mails mit gleichem Namen behandelt werden sollen.

 

Hinweise:

  • Obwohl Enterprise Connect die angegebenen Namenskonventionen auf E-Mails anwendet, wenn diese vom Benutzer zu Content Server hinzugefügt werden, können E-Mails bei Bedarf auch umbenannt werden.

  • Wenn Sie Enterprise Connect-Benutzern die Seite Meine Einstellungen zur Verfügung stellen, können sie die Voreinstellungen der Namenskonventionen für E-Mails ihren Anforderungen gemäß anpassen. Weitere Informationen finden Sie unter So konfigurieren Sie die Baumstruktur und das Listenfenster.

  • Wenn Enterprise Connect für eine E-Mail einen Namen generiert, der bereits im Content Server-Container verwendet wird, werden Benutzer entweder aufgefordert, den Namenkonflikt zu beheben, bevor die E-Mail hinzugefügt werden kann, oder Enterprise Connect generiert für die E-Mail einen neuen Namen mit einem Suffix, das die E-Mail von der bereits im Zielverzeichnis vorhandenen E-Mail unterscheidet. Die Benutzern zur Verfügung stehende Option ist von Ihren Verwaltungseinstellungen abhängig. Weitere Informationen zum Beheben von Dateinamenskonflikten finden Sie unter OpenText Enterprise Connect Plug-in for Content Server - Benutzerhilfe (NGDLLES-H-UGD).

  • Wenn Sie jedoch E-Mails als E-Mail-Untertypen speichern und Duplikate anhand des Vergleichs von Nachrichten-IDs herausstellen, werden doppelte E-Mails nicht dem Zielverzeichnis hinzugefügt, und Benutzer werden nicht über doppelte E-Mails benachrichtigt. Beachten Sie, dass Sie aufgefordert werden, zu bestimmen, wie doppelte zusammengesetzte E-Mails behandelt werden sollen, da sie von dieser Regel ausgeschlossen sind.
