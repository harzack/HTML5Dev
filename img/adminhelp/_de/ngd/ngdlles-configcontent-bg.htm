Sie können festlegen, wann Benutzer aufgefordert werden, Metadaten zu Objekten einzugeben, die sie zu Content Server hinzufügen. Sie können die Einstellungen so konfigurieren, dass Benutzer nur dann aufgefordert werden, Metadaten einzugeben, wenn ein Attribut oder eine andere Information erforderlich ist, oder um für jedes Objekt, das Sie in Content Server hinzufügen, Metadaten anzugeben.

Wenn andere Content Server-Module, wie Records Management, in Ihrer Umgebung installiert sind, werden die Benutzer standardmäßig immer aufgefordert, Metadaten anzugeben, wenn sie Elemente innerhalb einer Instanz von Content Server kopieren oder verschieben. Sie können allerdings festlegen, dass Benutzer nur dann aufgefordert werden, Metadaten beim Kopieren oder Verschieben von Elementen innerhalb einer Content Server-Instanz anzugeben, wenn unterschiedliche Kategorieanforderungen für die Quelle und das Ziel vorliegen. Beispiel: Enterprise Connect und Records Management sind in Ihrer Umgebung installiert. Wenn Sie die Einstellung Immer verwenden, werden Benutzer bei jedem Kopieren oder Verschieben eines Dokuments innerhalb der Content Server-Instanz aufgefordert, Metadaten anzugeben, unabhängig von den Metadatenanforderungen der Quelle und des Ziels. Wenn Sie die Einstellung Nie verwenden, müssen Benutzer Metadaten angeben, wenn das Ziel keine Angabe von Metadaten fordert, für die Quelle aber Kategorien und Attribute aktiviert sind. Wenn die Metadatenanforderungen der Quelle und des Ziels identisch sind, müssen Benutzer keine Metadaten angeben.

Wählen Sie die Optionen, die Ihren Best Practices bei der Erfassung von Metadaten entsprechen.
