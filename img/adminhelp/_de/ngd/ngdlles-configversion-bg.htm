Wenn Benutzer mit einem Dokument arbeiten, wird jedes Mal, wenn sie ihre Arbeit speichern, in Content Server eine neue Version erstellt. Abhängig von Ihren Prozessen werden diese sich in Arbeit befindlichen Versionen möglicherweise nicht benötigt. Als Systemverwalter können Sie festlegen, wie diese Zwischenversionen behandelt werden sollen. Standardmäßig wird nur die Endversion der Dokumente aus der Bearbeitungssitzung beibehalten. Sie können aber auch festlegen, dass jedes Mal, wenn Benutzer ihre Arbeit speichern in Content Server eine neue Version erstellt werden soll. Wenn diese Funktion aktiviert ist, werden die Zwischenversionen mit dem Protokollierungsereignis Enterprise Connect - Version ersetzt gemeldet.
