Email Services unterstützt mehrere Konfigurationseinstellungen. Sie können die E-Mail-Felder auswählen, die für die Indizierung extrahiert werden sollen, und Sie können angeben, ob Content Server-Standardattribute indiziert werden sollen oder nicht. Darüber hinaus können Sie festlegen, ob der E-Mail-Nachrichtentext zur Ansicht gespeichert werden soll. Außerdem können Sie die Anzahl der Objekte auswählen, die pro Seite in E-Mail-Ordnern angezeigt werden sollen.

 

Hinweis:
Mit der Volltextsuche in Content Server wird der gesamte Inhalt der E-Mails durchsucht. Um bestimmte E-Mail-Eigenschaften mit der erweiterten Suche in Content Server zu suchen, müssen Sie die folgenden Schritte ausführen:

  • Konfigurieren Sie die E-Mail-Felder für die Indizierung in der Email Services-Konfiguration.

  • Konfigurieren Sie die E-Mail-Felder so, dass sie im Suchmanager Abfragbar sind. Weitere Informationen finden Sie in der Content Server-Hilfe für Systemverwalter (Content Server-Index und Suche-Verwaltung, Suche verwalten, Indexattribute konfigurieren).

 

Auf der Seite Email Services konfigurieren können Sie die folgenden Einstellungen für Email Services konfigurieren:

Extrahierbare E-Mail-Felder

Sie können die E-Mail-Felder festlegen, die für die Indizierung extrahiert werden sollen. Erst wenn ein Feld extrahiert ist, können Daten in diesem Feld gesucht werden. Um den Suchindex nicht unnötig zu vergrößern, können Sie jedoch auf die Indizierung bestimmter E-Mail-Felder verzichten. In diesem Fall steht Ihnen die Volltextsuche in den Nachrichten zur Verfügung.

 

Hinweis:
Wenn bestimmte E-Mail-Felder nicht für die Indexerstellung extrahiert werden, kann dies die Funktionen anderer Content Server-Produkte beeinträchtigen.

 
Extrahierbare Attribute

Sie können angeben, ob Content Server-Standardattribute für E-Mails indiziert werden sollen oder nicht. Die Begrenzung der Anzahl der extrahierbaren Felder kann besonders in Unternehmen mit hohem E-Mail-Aufkommen dabei helfen, den für den Such-Index benötigten Speicher zu reduzieren.

 

Hinweise:

  • Wenn Content Server-Standardattribute nicht extrahiert werden, kann dies die Funktionen von allen Anwendungen beeinträchtigen, die mit Content Server nach E-Mails suchen und diese speichern.

  • Wenn Sie auf das Extrahieren der Content Server-Standardattribute verzichten, werden die nachstehenden Attribute nicht für E-Mails indiziert, sodass diese Attribute nicht in den E-Mails im System durchsucht werden können.

 

Wenn Sie keine Content Server-Standardattribute indizieren, werden die folgenden Attribute nicht indiziert:

Nachrichtentext speichern

Sie können angeben, ob der E-Mail-Nachrichtentext in Form von Metadaten in der Datenbank für die Anzeige gespeichert werden soll, wenn E-Mail-Objekte in der Vorschau von Content Server angezeigt werden.

 

Hinweise:

  • Bei der E-Mail-Überwachung empfiehlt OpenText, den E-Mail-Nachrichtentext nicht zu speichern. Die Datenbank könnte ansonsten zu groß werden.

    Wenn Sie den E-Mail-Nachrichtentext nicht für die Anzeige speichern wollen, kann der E-Mail-Text nicht in der HTML-Ansicht dargestellt werden. Sie können den Text von Nachrichten jedoch dennoch anzeigen lassen, wenn diese stattdessen in der von den OpenText Document Filters erzeugten Ansicht dargestellt werden. Weitere Informationen finden Sie unter OpenText Document Filters - Administrator's Guide (LLESCF-AGD).

 
Extrahieren zusätzlicher E-Mail-Eigenschaften

Zusätzlich zu den Standard-E-Mail-Eigenschaften können Sie auch andere E-Mail-Eigenschaften festlegen, die extrahiert und gespeichert werden sollen. Wenn Sie Microsoft Outlook so konfigurieren möchten, dass zusätzliche E-Mail-Eigenschaften extrahiert und auf dem Server gespeichert werden, müssen Sie diese Eigenschaften im Feld Extrahieren weiterer E-Mail-Eigenschaften hinzufügen. Sorgen Sie dabei dafür, dass für jeden Eintrag eine neue Zeile eingefügt wird. Die Einträge sind im folgenden Format einzugeben:

<MailSystem>|<Eigenschaftsname>|<Anzeigename>|<Attributname>

wobei Folgendes gilt:

<MailSystem> gibt die E-Mail-Umgebung an, aus der Sie die Eigenschaften extrahieren möchten. Als Werte kommen OUTLOOK oder LOTUS infrage.

<Eigenschaftsname> gibt den Namen der Eigenschaft im Mail-System an, die extrahiert werden soll.

Geben Sie für Microsoft Outlook die <Kennungsnummer> der MAPI-Eigenschaft ein. Für eine benannte Eigenschaft müssen Sie sowohl die GUID der Eigenschaft als auch den Namen der Eigenschaft als Zeichenfolge oder Nummer getrennt durch ein Komma eingeben.

Geben Sie für Lotus Notes den <Feldnamen> der Dokumenteigenschaft ein.

 

Hinweis:
Wenn eine Eigenschaft nicht gefunden werden kann, wird sie nicht in der Datenbank gespeichert. Außerdem werden Standardeigenschaften (Betreff, Von, An, Cc, Gesendet am, Erhalten am) immer extrahiert und getrennt von den benutzerdefinierten Eigenschaften gespeichert.

 

<Anzeigename> gibt den von Ihnen festgelegten Anzeigenamen für die Eigenschaft an. Dies ist der Name, unter dem die Eigenschaft auf der Content Server-Registerkarte Eigenschaften – E-Mail angezeigt wird.

<Region Name> (optional) ist der Regionsname, den die Suchmaschine bei der Indizierung verwenden soll. Sollen die zusätzlichen E-Mail-Eigenschaften durchsuchbar sein, müssen Sie einen Attributnamen angeben. Ist es nicht erforderlich, dass die E-Mail-Eigenschaften durchsuchbar sind, können Sie diesen Wert übergehen.

Beispiel

OUTLOOK|0x0017|Wichtigkeits-Merker|EmailImportance

LOTUS|($Classification)|Lotus-Klassifikation

OUTLOOK|{00020329-0000-0000-C000-000000000046},Keywords|Classification|EmailClassification

Signierte/Verschlüsselte Nachrichtenklassen

Sie können die speziellen Microsoft Outlook-Nachrichtenklassen für signierte oder verschlüsselte E-Mail-Nachrichten definieren. Enterprise Connect erkennt standardmäßig bestimmte Nachrichtenklassen in Content Server als signiert oder verschlüsselt.

Enterprise Connect kann diese Typen nicht ordnungsgemäß verarbeiten und verwendet die in diesem Bereich eingegebenen Eigenschaften, um Benutzer, die Content Server diese E-Mail-Typen hinzufügen, mit einer entsprechenden Nachricht zu informieren. Wenn die Liste der in diesem Bereich angegebenen Nachrichten unvollständig ist, kann sie um zusätzliche Nachrichten erweitert werden. Geben Sie jede Nachrichtenklasse in einer eigenen Zeile ein.

So konfigurieren Sie Einstellungen für Email Services:

  1. Klicken Sie auf der Seite Content Server-Verwaltung im Bereich Email Services-Verwaltung auf die Verknüpfung Email Services konfigurieren.

  2. Wählen Sie auf der Seite Email Services konfigurieren die Optionen für Email Services.

  3. Klicken Sie auf Senden.
