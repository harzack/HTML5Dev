So konfigurieren Sie die Einstellungen für die Web-Bearbeitung:

  1. Klicken Sie auf der Seite Content Server-Verwaltung unter Verwaltung der Web-Bearbeitung auf Einstellungen für Web-Bearbeitung konfigurieren.

  2. Legen Sie im Bereich Editoren der Seite Einstellungen für Web-Bearbeitung konfigurieren die Editor-Priorität fest, indem Sie im Feld für jeden verfügbaren Editor einen numerischen Wert eingeben.

     

    Tipp:
    Sie können einen Editor deaktivieren, indem Sie einen negativen Wert angeben.

     

  3. Geben Sie im Bereich Office Editor Einstellungen im Feld Content Server-Anzeigename den Namen des Repository an, wie er im Office Editor Desktop-Client erscheinen soll.

     

    Tipp:
    Wenn Sie keinen Namen angeben, wird das Format <host>/<contentserverfolder> verwendet. Zum Beispiel: intranet.opentext.com/intranet oder localhost:8080/otcs.

     

  4. Führen Sie im Bereich Office Editor-Einstellungen einen der folgenden Schritte aus:

    • Aktivieren Sie das Kontrollkästchen Download des Installationsprogramms aktivieren, wenn Benutzer den Office Editor-Client installieren sollen. Dies ist die Standardeinstellung.

    • Deaktivieren Sie das Kontrollkästchen Download des Installationsprogramms aktivieren, wenn Sie nicht möchten, dass Benutzer den Office Editor-Client installieren können.

  5. Führen Sie im Bereich Office Editor-Einstellungen in der Liste Wenn eine neue Version verfügbar ist einen der folgenden Schritte aus:

    • Wählen Sie Keine Anzeige oder Aufforderung für ein Upgrade, wenn die Benutzer nicht aufgefordert werden sollen, ein Upgrade auf die aktuelle Version des Office Editor-Clients durchzuführen. Dies ist die Standardeinstellung.

    • Wählen Sie Benutzer die Option geben, ein Upgrade durchzuführen oder weiterhin eine kompatible alte Version zu verwenden, wenn die Benutzer selbst entscheiden sollen, ob sie ein Upgrade auf die aktuelle Version des Office Editor-Clients durchführen möchten.

    • Wählen Sie Benutzer zum Upgrade zwingen, wenn die Benutzer sofort ein Upgrade auf die aktuelle Version des Office Editor-Clients durchführen sollen.

     

    Hinweis:
    Wenn Sie die Option Benutzer die Option geben, ein Upgrade durchzuführen oder weiterhin eine kompatible alte Version zu verwenden oder die Option Benutzer zum Upgrade zwingen auswählen, müssen Sie auch das Kontrollkästchen Download des Installationsprogramms aktivieren aktivieren, damit die Benutzer den Office Editor-Client installieren können.

     

  6. Führen Sie im Kontrollkästchen doc.Fetch für alle Optionen verwenden einen der folgenden Schritte aus:

    • Deaktivieren Sie das Kontrollkästchen, um alle konfigurierten Dokumente in ihren nativen Anwendungen zu öffnen, wie Sie dies im Bereich MIME-Typen für Office Editor angegeben haben. Wenn Sie diese Option wählen, lädt Office Editor das Dokument automatisch in den Dokumenten-Cache herunter und öffnet es dann von dort aus. Dies ist die Standardeinstellung.

    • Markieren Sie das Kontrollkästchen, um alle Dokumente unter Verwendung der Content Server-Standardimplementierung zu öffnen (doc.Fetch). Wenn Sie diese Option wählen, wird Office Editor nicht verwendet, wenn Benutzer ein Dokument öffnen.

  7. Führen Sie im Kontrollkästchen Dokumentenzwischenspeicherung aktivieren einen der folgenden Schritte aus:

    1. Aktivieren Sie das Kontrollkästchen, um alle zuletzt verwendeten Versionen von Dokumenten zu behalten, die mit Office Editor bearbeitet wurden. Dies ist die Standardeinstellung.

    2. Deaktivieren Sie das Kontrollkästchen, um alle zuletzt verwendeten, mit Office Editor bearbeiteten Versionen von Dokumenten zu löschen.

  8. Führen Sie im Feld MIME-Typen fürOffice Editor einen der folgenden Schritte aus:

    • Klicken Sie für alle Dokumente auf SupportOffice Editor, wenn Sie alle Dokumentarten mit Office Editor öffnen und bearbeiten möchten.

    • Klicken Sie für ausgewählte MIME-Typen auf SupportOffice Editor, wenn Sie nur die angegebenen Dokumentarten mit Office Editor öffnen und bearbeiten möchten. Sie können die Liste der unterstützten MIME-Typen bei Bedarf bearbeiten. Dies ist die Standardeinstellung.

     

    Achtung
    Wenngleich es möglich ist, andere MIME-Typen zu dieser Einstellung hinzuzufügen, um diese Funktion für die Zielanwendungen Ihrer Umgebung zu validieren, müssen Sie jede Anwendung gründlich testen, die Sie in einer Produktionsumgebung einsetzen wollen. Alle erkannten Probleme im Zusammenhang mit den Funktionen sollten dem OpenText-Kundendienst zur Kenntnis gebracht werden. Diese Funktionen werden „so wie sie sind" und ohne ausdrückliche oder stillschweigende Garantien jeglicher Art bereitgestellt. Probleme im Zusammenhang mit diesen Funktionen werden auf Einzelfallbasis ohne Lösungsgarantie geprüft.

     
     

    Hinweis:
    Wenn das WebDAV-Modul installiert ist, wird eine ähnliche Liste für unterstützte WebDAV MIME-Typen angezeigt. Sie können diese Liste bei Bedarf bearbeiten.

     

  9. Klicken Sie im Bereich Neue Dokumente hinzufügen auf die Dokumenttypen, die auf der Seite Dokument hinzufügen in der Liste Neu hinzufügen angezeigt werden sollen.

  10. Klicken Sie auf Senden.
