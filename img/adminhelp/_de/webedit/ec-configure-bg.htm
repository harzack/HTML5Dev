Auf der Seite Verwaltung für Web-Bearbeitung können Sie den Editor festlegen, der verwendet wird, wenn Sie unterstützte Dokumenttypen bearbeiten, die automatische Installation über das Web deaktivieren, den MIME-Typ für unterstützte Dokumente angeben und die unterstützten Dokumenttypen für die Funktion Hinzufügen festlegen.

Priorität des Editors einstellen

Legen Sie die Priorität des Editors fest, um anzugeben, welcher Editor für die Bearbeitung eines unterstützten Dokuments verwendet werden soll. Welche Editoren zum Konfigurieren verfügbar sind, hängt davon ab, welche Content Server-Module verwendet werden. Zu den unterstützten Editoren könnten OpenText Office Editor und OpenText WebDAV zählen. Wenn Sie die Priorität des Editors einstellen, weisen Sie jedem Editor einen numerischen Wert zu. Dabei werden den bevorzugten Editoren höhere Werte zugeordnet.

Office Editor-Einstellungen festlegen

Sie können den Namen des Repositorys so festlegen, wie er in der Spalte Ort im Office Editor-Desktop-Client angezeigt werden soll. Wenn Sie keinen Namen angeben, erstellt Office Editor einen Namen mithilfe des Formats <host>/<contentserverfolder> . Wenn es mehr als ein Repository mit demselben Namen gibt, wird das zuletzt konfigurierte Repository verwendet, und jeder der vorherigen Versionen wird eine Nummer angehängt. So würde eine doppelte Instanz beispielsweise in Content Server (2) umbenannt werden.

 

Hinweis:
Der von Ihnen angegebene Repository-Name wird nicht verwendet, wenn Enterprise Connect auf dem Client installiert ist. In diesem Fall wird stattdessen der Name des Enterprise Connect-Plug-ins verwendet.

 

Wenn Sie das Kontrollkästchen Download des Installationsprogramms aktivieren aktivieren, geben Sie den Benutzern die Möglichkeit, den Office Editor-Client über die Content Server-Web-Benutzeroberfläche herunterzuladen und zu installieren. Dies ist die Standardeinstellung. Wenn Sie dieses Kontrollkästchen deaktivieren, können die Benutzer den Client nicht herunterladen und installieren, und Sie müssen Installationen oder Upgrades selbst veranlassen.

Mit den Optionen in der Liste Wenn eine neue Version verfügbar ist können Sie steuern, wann die Benutzer auf die neueste Version des Office Editor-Clients aktualisieren. Wenn Benutzer, die mit einer älteren Version des Office Editor-Clients arbeiten, ein Dokument öffnen oder bearbeiten möchten, können Sie eine der folgenden Optionen festlegen:

  • Wenn Sie Keine Anzeige oder Aufforderung für ein Upgrade wählen, werden die Benutzer niemals für ein Upgrade aufgefordert. Dies ist die Standardeinstellung.

  • Wählen Sie Benutzer die Option geben, ein Upgrade durchzuführen oder weiterhin eine kompatible alte Version zu verwenden, um den Benutzern die Möglichkeit zu geben, ein Upgrade durchzuführen oder die Arbeit fortzusetzen. Die Benutzer werden aufgefordert, durch Klicken auf die bereitgestellte Verknüpfung auf die neueste Version des Office Editor-Clients zu aktualisieren oder die Arbeit mit der aktuellen Version fortzusetzen.

  • Wählen Sie Benutzer zum Upgrade zwingen, um Benutzer zu zwingen, ein Upgrade durchzuführen, bevor sie ihre Arbeit fortsetzen können. Die Benutzer werden aufgefordert, mithilfe der bereitgestellten Verknüpfung auf die aktuelle Version des Office Editor-Clients zu aktualisieren.

 

Hinweis:
Wenn Sie die Option Benutzer die Option geben, ein Upgrade durchzuführen oder weiterhin eine kompatible alte Version zu verwenden oder die Option Benutzer zum Upgrade zwingen auswählen, müssen Sie auch das Kontrollkästchen Download des Installationsprogramms aktivieren aktivieren, damit die Benutzer den Office Editor-Client installieren können.

 

Sie können konfigurieren, wie sich die Funktion Öffnen in Content Server verhält. Sie können festlegen, dass alle Dokumente unter Verwendung der Content Server-Standardimplementierung (doc.Fetch) geöffnet werden. In diesem Fall wird Office Editor nicht verwendet, wenn Benutzer ein Dokument öffnen. Wählen Sie diese Option, wenn Sie möchten, dass die Benutzer Content Server-Dokumente öffnen können, ohne Office Editor installieren zu müssen. Alternativ dazu können Sie angeben, dass alle unterstützten MIME-Typen, wie von den Optionen festgelegt, die Sie im Bereich MIME-Typen für Office Editor konfigurieren, mit dem Office Editor geöffnet werden. In diesem Fall lädt Office Editor das Dokument automatisch in den Dokumenten-Cache herunter und öffnet es dann von dort aus.

Sie können die Zwischenspeicherung zuletzt verwendeter Dokumente aktivieren oder deaktivieren. Standardmäßig werden alle Dokumente im Dokument-Cache abgelegt. Wenn Sie das Kontrollkästchen Dokumentenzwischenspeicherung aktivieren deaktivieren, werden nicht verwendete Dokumente aus dem Cache gelöscht. Wenn das Kontrollkästchen z. B. deaktiviert ist und der Benutzer ein Dokument öffnet, bearbeitet und schließt, wird das Dokument aus dem Dokument-Cache entfernt. Dokumente mit Konfliktstatus bleiben im Dokument-Cache, bis der Konflikt gelöst ist. Darüber hinaus bleiben derzeit verwendete Dokumente im Cache, bis sie geschlossen werden. Diese Einstellung gilt nur für die folgenden Dokumenttypen.

.docx .docm .doc .dotx
.dotm .dot .xlsx .xlsm
.xlsb .xls .xltx .xltm
.xlt .pptx .pptm .ppt
.potx .potm .pot .ppsx
.ppsm .pps .ppam .ppa
.mpp .mpt .vsdx .vssx
.vstx .vsdm .vssm .vstm
.vsd .vdw .vsw .vss
.vst .pdf .wpd  

 

Hinweise:

  • Diese Einstellung gilt nur für Dokumente, die in der Content Server-Web-Benutzeroberfläche mit Office Editor bearbeitet werden, wobei Enterprise Connect nicht installiert ist. Sie müssen zusätzliche Änderungen in Enterprise Connect vornehmen, um diese Einstellung auf Dokumente anzuwenden, die in Enterprise Connect bearbeitet werden. Weitere Informationen finden Sie unter OpenText Enterprise Connect - Installation Guide (NGDCORE-IGD).

  • Benutzer müssen den Office Editor-Client neu starten, um Änderungen an dieser Einstellung zu übernehmen.

 

MIME-Typen festlegen

Sie können die von den Editoren unterstützten MIME-Typen festlegen, um anzugeben, welche Dokumente beim Verwenden eines bestimmten Editors geändert werden können. Für den ausgewählten Editor müssen Sie die MIME-Typen angeben, die bei Microsoft Word-, Excel-, PowerPoint-. Project- und Visio-Dokumenten unterstützt werden sollen. So können Sie beispielsweise festlegen, dass Office Editor alle MIME-Typen oder nur bestimmte davon unterstützt. Sie können die Liste der unterstützten MIME-Typen bei Bedarf bearbeiten. Wenn Sie WebDAV als Editor verwenden, können Sie die Liste der unterstützten MIME-Typen ebenfalls bearbeiten.

Unterstützte neue Dokumenttypen festlegen

Durch Aktivieren oder Deaktivieren bestimmter Dokumenttypen legen Sie die unterstützten Dokumenttypen für die Funktion Neues Dokument hinzufügen fest. So können Sie beispielsweise angeben, dass Benutzer zwar neue Microsoft Word-Dokumente, nicht aber PowerPoint-Dokumente hinzufügen können. Die von Ihnen angegebenen unterstützten Dokumenttypen werden auf der Seite Dokument hinzufügen in der Liste Neu hinzufügen angezeigt.

 

Tipp:
Sie können die Standarddokumente für die Funktion Neu hinzufügen ändern. Wenn ein Benutzer ein neues Dokument hinzufügen möchte, wird das von Ihnen angegebene Standarddokument automatisch im zugehörigen Desktop-Programm geöffnet. Sie können ein Word 2010-Dokument z. B. ändern, indem Sie das leere Standarddokument in das Verzeichnis support\webedit Ihrer Content Server-Installation legen, in dem sich auch das benutzerdefinierte Standarddokument befindet, das Sie verwenden möchten.
